Seite "BPEL-10g-Transportkonfiguration"

Auf dieser Seite konfigurieren Sie die Transporteinstellungen für einen Business-Service mithilfe des Transportprotokolls "BPEL-10g" (Oracle BPEL Process Manager). Weitere Informationen zur Verwendung von Oracle Service Bus mit dem Oracle BPEL Process Manager finden Sie unter "Oracle BPEL Process Manager Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1147.

Eigenschaft
Beschreibung
Rolle
Der BPEL-Transport wird verwendet, um Anforderungsnachrichten von Oracle Service Bus zum Oracle BPEL Process Manager zu senden. Der Business-Service kann folgende Rollen haben:
  • Synchroner Client
    Für die synchrone Kommunikation mit einem Oracle Service Bus-Client wird als einzige Standortinformation die BPEL-Adresse benötigt. Diese Adresse wird statisch als Endpunkt-URI bzw. dynamisch über die URI-Neuschreibung erfasst.
  • Asynchroner Client
    Für die asynchrone Kommunikation mit einem Oracle Service Bus-Client wird ein Callback von Oracle BPEL Process Manager an Oracle Service Bus auf einer anderen Verbindung als die Anforderung gesendet, und Sie müssen Oracle Service Bus so konfigurieren, dass die korrekte Callback-Adresse zur Verfügung gestellt wird. Weitere Informationen finden Sie unter "Advanced Use Cases (Asynchronous)" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
  • Service Callback
    Wenn der Business-Service als Service Callback an Oracle BPEL Process Manager konzipiert wurde (bei dem Oracle BPEL Process Manager einen externen Service über Oracle Service Bus aufruft), ist die Callback-Adresse nur zur Laufzeit bekannt. Verwenden Sie einen Endpunkt-URI von bpel://callback.

    • Wenn Sie den Business-Service mit dem Marker-URI konfigurieren, richten Sie Ihre Pipeline-Logik so ein, dass der URI dynamisch auf $outbound eingestellt wird, z.B. mit der TransportHeader-Aktion.

    Hinweis: Ein Service Callback Business-Service unterstützt weder Load Balancing noch Failover.
Callback Proxy
Dieses optionale Feld ist nur für die Rolle "Asynchroner Client" verfügbar. Hier können Sie den Proxy-Service wählen (entweder einen SB- oder HTTP-Proxy der Art "Beliebiges SOAP"), mit dem Callbacks an den Oracle Service Bus-Client geleitet werden, von dem die Anforderung stammt. Weitere Informationen finden Sie unter "Advanced Use Cases (Asynchronous)" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
Service-Account
Für die JNDI-Kontextsicherheit, mit der auf den Zustell-Service des Oracle BPEL Process Managers zugegriffen wird. Klicken Sie auf "Durchsuchen", und wählen Sie einen Service-Account. Wird kein Service-Account angegeben, wird ein anonymes Subject verwendet.
Im Hinblick auf den zu konfigurierenden Typ des Service-Accounts (z. B. statisch oder passthrough) gibt es keine Einschränkung, aber die Laufzeit muss auf einen Benutzernamen und ein Kennwort zugreifen können.
Transaktion unterbrechen
Wenn Sie "Transaktion unterbrechen" wählen, wird der Business-Service "nicht-transaktional", auch wenn er von einer Transaktion gestartet wird.
Wenn Sie "Transaktion unterbrechen" nicht wählen:
  • Wenn das Protokoll ein WebLogic Server-unterstütztes Protokoll angibt (t3, iiop, http), wird die Transaktion propagiert.
  • Wenn das Protokoll einen OC4J-Server (ormi, opmn) angibt, gibt der BPEL-Transport einen Ausnahmezustand aus, da OC4J Transaktionspropagierung nicht unterstützt.
Der BPEL-Transport ignoriert die Option "Transaktion unterbrechen" in folgenden Fällen:
  • Der Business-Service wird mit der bestmöglichen Service-Qualität (QoS) aufgerufen. Der BPEL-Transport unterbricht automatisch jede Transaktion, die QoS nicht unterstützt.
  • Der Business-Service wird mit der QoS "genau einmal" aufgerufen, und es ist keine Transaktion vorhanden.
Eine Beschreibung der Transaktionspropagierung finden Sie unter "Transaction Propagation" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1151.
Dispatch Policy
Wählen Sie die Instanz des WebLogic Server Work Managers, die Sie für die Dispatch Policy dieses Endpunkts verwenden möchten. Der Standard-Work Manager wird verwendet, wenn kein anderer Work Manager vorhanden ist.
Weitere Informationen zu Work Manager finden Sie unter "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server unter http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
