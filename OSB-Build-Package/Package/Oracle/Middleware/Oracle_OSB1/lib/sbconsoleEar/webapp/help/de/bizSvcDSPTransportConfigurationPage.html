Seite "DSP-Transportkonfiguration"

Die Seite Business-Service erstellen/bearbeiten – DSP-Transportkonfiguration ist eine von diversen Seiten, auf denen Sie Business-Services erstellen und bearbeiten können. Diese Funktionen sind unter Business-Services erstellen und konfigurieren und Business-Service-Konfigurationen bearbeiten genauer beschrieben.

Weitere Informationen finden Sie unter "DSP and Oracle Data Service Integrator Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1229.

Auf dieser Seite können Sie Transporteinstellungen für Business-Services mit DSP-(Oracle Data Service Integrator-)Transportprotokoll konfigurieren. In Tabelle 18-4 ist die Verwendung dieser Seite beschrieben.

Tabelle 18-4 Seite "Business-Services erstellen/bearbeiten – DSP-Transportkonfiguration"
Option
Verwendung
Debug-Ebene
Geben Sie einen der folgenden Werte an:
  • 0: Keine Debug-Informationen
  • 1: Ausgabe von Informationen in der Anforderungsnachricht
  • 3: Ausgabe von Informationen in der Anforderungsnachricht und in der Antwortnachricht
Service-Account
Klicken Sie auf Durchsuchen, und wählen Sie aus der angezeigten Liste einen Service-Account. Wenn Sie keinen Service-Account angeben, wird ein anonymes Subject verwendet.

Weitere Informationen:

  • Service-Accounts
  • Service-Accounts hinzufügen
Dispatch Policy
Wählen Sie die Instanz des WebLogic Server Work Managers, die Sie für die Dispatch Policy dieses Endpunkts verwenden möchten. Der Standard-Work Manager wird verwendet, wenn kein anderer Work Manager vorhanden ist.
Mit dem Work Manager wird die Antwortnachricht zur Antwortverarbeitung gesendet. Informationen über Work Manager finden Sie unter "Work Manager für das Optimieren geplanter Arbeit verwenden" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server unter http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
