Seite "EJB-Transportkonfiguration"

Die Seite Business-Service erstellen/bearbeiten – EJB-Transportkonfiguration ist eine von diversen Seiten, auf denen Sie Business-Services erstellen und bearbeiten können. Diese Funktionen sind unter Business-Services erstellen und konfigurieren und Business-Service-Konfigurationen bearbeiten genauer beschrieben.

Auf dieser Seite können Sie Transporteinstellungen für Business-Services mit EJB-Transportprotokoll konfigurieren. In Tabelle 18-5 ist die Verwendung dieser Seite beschrieben.

Tabelle 18-5 Seite"Business Service erstellen/bearbeiten – EJB-Transport konfigurieren" 
Option
Verwendung
Subject des Callers übergeben
Wenn Sie dieses Kontrollkästchen aktivieren und keine Service-Accounts konfiguriert sind, wird das authentifizierte Subject in Oracle Service Bus beim Aufrufen des EJB vom Proxy-Service übergeben. Hinweis: Wenn Sie diese Option aktivieren, ist das Feld Service-Account deaktiviert.
Service-Account
Klicken Sie auf Durchsuchen, und wählen Sie aus der angezeigten Liste einen Service-Account. Wenn Sie keinen Service-Account angeben, wird ein anonymes Subject verwendet. Diese Option ist nicht verfügbar, wenn die Option "Subject des Callers übergeben" aktiviert ist.

Weitere Informationen:

  • Service-Accounts
  • Service-Accounts hinzufügen
Unterstützt Transaktion
Aktivieren Sie dieses Kontrollkästchen, damit Transaktionen unterstützt werden.
Client-JAR-Archiv
Klicken Sie auf Durchsuchen, und wählen Sie aus der angezeigten Liste eine JAR-Ressource für den EJB-Client. Weitere Informationen zum Erstellen von JAR-Ressourcen finden Sie unter JARs. Dies ist ein Pflichtfeld.
Converter-JAR
Klicken Sie auf Durchsuchen, und wählen Sie aus der angezeigten Liste eine JAR-Ressource für die EJB-Converter-Klasse. Weitere Informationen über EJB-Client-JAR-Ressourcen und Converter-Klassen finden Sie unter "EJB Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.
Home-Interface
Nur EJB 2.1: Wählen Sie aus den vom JAR-Archiv ausgefüllten Optionen die gewünschte EJBHome-Schnittstelle. Der JNDI-Name in diesem URI-Muster muss mit der hier gewählten EJBHome-Schnittstelle verknüpft sein. Wenn der EJB-Typ nicht den Anforderungen entspricht oder im Client-JAR-Archiv keine EJBHome-Schnittstelle angegeben wurde, wird in Oracle Service Bus eine Warnung angezeigt.
Remote-Interface
Nur EJB 2.1: Dieses Feld wird je nach Konfiguration der Home-Schnittstelle automatisch ausgefüllt.
Business-Schnittstelle
Nur EJB 3.0: Wählen Sie im Client-JAR-Archiv die gewünschte Business-Schnittstelle.
Ziel-Namespace
Dieses Feld wird mit Informationen aus dem JAR-Archiv ausgefüllt.
Stil
Wählen Sie entsprechend Ihren Anforderungen Dokument umbrochen oder RPC. Wenn zwei oder mehrere Methoden der Session-EJB mit dem Status "Stateless" dieselbe Anzahl an Parametern desselben Datentyps haben und Sie möchten, dass die Vorgänge dokumentorientiert sind, wählen Sie "Dokument umbrochen".
Der Stil ist wichtig, da der Inhalt von $body beim EJB-Routing oder -Publishing dem Stil entsprechen muss. Auch beim EJB-Aufruf beeinflusst der Stil die Parameterinhalte, besonders wenn die Option "Dokument umbrochen" gewählt wurde. Außerdem muss mit einem Nutzungsverhalten ein EJB-Business-Service definiert und anschließend ein Proxy-Service mit derselben WSDL erstellt werden, die an das EJB weiterleitet. In diesem Fall ist der Stil von WSDL besonders wichtig, da das Clienttool, dass zum Aufrufen des Proxys verwendet wird, WSDL einschränken kann.
Codierung
Wählen Sie Codiert oder Literal.
Methoden
Die angezeigten Methoden sind die Methoden der gewählten EJB-Remote- oder -Business-Schnittstelle. Wählen Sie die erforderlichen Methoden aus. (Sie können mehrere Methoden wählen.) Sie erweitern die Methode, indem Sie auf das Pluszeichen (+) klicken. Bearbeiten Sie die Standardparameterwerte, und wählen Sie einen Converter aus, sofern ein solcher vorhanden oder erforderlich ist.
Schließen Sie Methoden mit Parametern oder Rückgabetypen aus, die nicht vom JAX-RPC-Stack unterstützt werden, oder verknüpfen Sie solche Argumente mit Converter-Klassen.
Sie können den Standardvorgangsnamen für eine bestimmte Methode ändern. (Standardmäßig entspricht der Vorgangsname dem Methodennamen.) Wenn ein EJB Methoden mit demselben Namen enthält, müssen Sie die Vorgangsnamen so ändern, dass jeder Name nur einmal vorkommt, denn für WSDLs sind eindeutige Vorgangsnamen erforderlich.
Hinweis: Wenn die Zugangsdaten oder Transaktionseinstellungen sich zwischen den Methoden für ein bestimmtes EJB unterscheiden, können Sie die Methoden für einen bestimmten Business-Service anpassen und einen Business-Service pro Methode erstellen. Dadurch erhalten Sie genaue Kontrolle über Transaktionen und Zugangsdaten.
Ausnahmen
Dieses Feld wird angezeigt, wenn eine Methode eine Business Exception auslöst. Wenn eine EJB-Methode eine Exception mit Datentypen auslöst, die nicht von Java Web Services (JWS) unterstützt werden, z.B. eine ArrayList, wählen Sie mit dem Feld "Exceptions" eine Converter-Klasse aus, mit der die Exception in einen Typ konvertiert wird, der von JWS unterstützt wird.
Die Converter-Klasse muss "com.bea.wli.sb.transports.ejb.IType-Converter" implementieren. Converter-Klassen können nur für geprüfte Exceptions und nicht für Laufzeit-Exceptions konfiguriert werden.
Verpacken Sie die Converter-Klasse und die konvertierte Exception-Klasse im Client- oder Converter-JAR-Archiv.
Weitere Informationen finden Sie unter "EJB-Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.
