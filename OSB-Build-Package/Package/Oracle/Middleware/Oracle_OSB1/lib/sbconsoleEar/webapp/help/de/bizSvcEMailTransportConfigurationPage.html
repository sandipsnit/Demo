Seite "E-Mail-Transportkonfiguration"

Die Seite Business-Service erstellen/bearbeiten - E-Mail-Transportkonfiguration ist eine von einer Serie zum Erstellen und Bearbeiten von Business-Services, wie in Business-Services erstellen und konfigurieren und Business-Service-Konfigurationen bearbeiten beschrieben.

Verwenden Sie diese Seite, um Transporteinstellungen für einen Business-Service unter Verwendung des E-Mail-Transportprotokolls zu konfigurieren. Tabelle 18-6 beschreibt, wie die Seite zu verwenden ist.

Tabelle 18-6 Einen Business-Service erstellen/bearbeiten - Seite "E-Mail-Transportkonfiguration"
Option
Verwendung
SSL erforderlich
Wählen Sie diese Option, um über Secure Sockets Layer zu kommunizieren. Die Standard-SSL-Portnummer für LDAP ist 465.
Hinweise
  • Das E-Mail-Serverzertifikat muss im Oracle Service Bus Truststore vorhanden sein, damit SSL funktioniert.
  • Wenn Sie TLS verwenden, müssen Sie das vertrauenswürdige Zertifikat entweder im Standard-Truststore ($JAVA_HOME/jre/lib/security/cacerts) oder in einem anderen Truststore speichern und den Pfad in der Datei $DOMAIN_HOME/bin/setDomainEnv.* hinzufügen, wie unten dargestellt:

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<path to trust store>

SMTP-Server
Wählen Sie den Standard-SMTP-Server für Endpunkt-URI-Einträge wie name@domain_name.com. Wenn Sie in der Endpunkt-URI SMTP-Serverparameter verwenden, wie in Tabelle 18-3 beschrieben, werden diese Serverressourcen anstelle der SMTP-Servereinstellung verwendet.
Wählen Sie keinen SMTP-Server, wenn Sie die Option Mailsession verwenden.
Weitere Informationen über das Erstellen von SMTP-Ressourcen finden Sie unter SMTP-Server hinzufügen.
Mailsession
Geben Sie den JNDI-Name der konfigurierten Mailsession ein, der für die Endpunkt-URI-Einträge name@domain_name.com zu verwenden ist. Wenn Sie JNDI-Mailsessionparameter in der Endpunkt-URI verwenden, wie in Tabelle 18-3 beschrieben, werden anstelle dieser Mailsessioneinstellung diese Mailsessions verwendet.
Geben Sie keine Mailsession ein, wenn Sie die SMTP-Serveroption verwenden.
Name des Absenders
Geben Sie einen Anzeigenamen für den ursprünglichen E-Mail-Account für diesen Service ein.
Absenderadresse
Geben Sie den ursprünglichen E-Mail-Account für diesen Service ein.
Antwort an
Geben Sie einen Anzeigenamen für die Antwort an den E-Mail-Account ein.
Antwortadresse
Geben Sie eine Antwort-E-Mail-Adresse ein.
Verbindungs-Timeout
Geben Sie das Timeout-Intervall in Sekunden an, bevor die Verbindung unterbrochen wird. Wenn Sie 0 eingeben, gibt es kein Timeout.
Anforderungscodierung
Akzeptieren Sie den Standard ISO-8859-1 als die Zeichensatzcodierung für Anforderungen in E-Mail-Transporten, oder geben Sie eine andere Zeichensatzcodierung ein.
