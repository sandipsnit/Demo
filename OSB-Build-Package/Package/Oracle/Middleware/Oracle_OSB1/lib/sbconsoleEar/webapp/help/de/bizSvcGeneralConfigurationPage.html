Seite "Allgemeine Konfiguration"

Die Seite Business-Service erstellen/bearbeiten - Allgemeine Konfiguration ist eine von einer Serie zum Erstellen und Bearbeiten von Business-Services, wie in Business-Services erstellen und konfigurieren und Business-Service-Konfigurationen bearbeiten beschrieben.

Verwenden Sie diese Seite, um allgemeine Konfigurationseinstellungen für einen Business-Service zu verändern.

Wenn Sie einen Business-Service erstellen, ist dies die erste Seite einer Serie von Seiten, die zur Konfiguration dieses Service angezeigt wird. Die Seiten, die nach dieser angezeigt werden, unterscheiden sich je nachdem, welche Auswahl Sie auf dieser Seite treffen.

Tabelle 18-1 beschreibt, wie diese Seite zu verwenden ist.

Tabelle 18-1 Einen Business-Service erstellen/bearbeiten - Seite "Allgemeine Konfiguration"
Option
Verwendung
Service-Name
Geben Sie einen eindeutigen Namen für den Business-Service ein.

Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
Beschreibung
Geben Sie eine Beschreibung ein.
Service-Typ
Ein Service-Typ definiert die Typen und das Packaging der vom Service ausgetauschten Nachrichten. Wählen Sie den Service-Typ, den Sie erstellen möchten:
  • WSDL-Web-Service - Wählen Sie diese Option aus, um einen Business-Service basierend auf einer WSDL zu erstellen. Geben Sie dann den WSDL-Namen an, wie er aus dem Pfad hervorgeht (zum Beispiel: myProject/myFolder/myWSDL). Oder Sie klicken auf Durchsuchen, um eine WSDL auf der Seite WSDL wählen auszuwählen.

    • (Port oder Binding) - Geben Sie den Namen eines Ports ein (definiert in der WSDL), um eine tatsächliche Transportadresse zu beschreiben, oder geben Sie den Namen eines Bindings ein (definiert in der WSDL), um eine Transportadresse zuzuordnen. Wenn Sie Durchsuchen verwenden, um eine WSDL auszuwählen, wie oben beschrieben, listet die Seite WSDL-Definition wählen alle in WSDL definierten Ports und Bindings auf. Wenn Sie auf dieser Seite einen Port oder ein Binding wählen, steht im Feld (Port oder Binding) der gewählte Name.

  • Transporttyp-Service - Wählen Sie diese Option, um einen Service zu erstellen, der EJB- oder Flow-Transport verwendet.
  • Messaging-Service - Wählen Sie diese Option, um einen Service zu erstellen, der Nachrichten mit sehr unterschiedlichen Contenttypen austauscht. Dieser Austausch kann entweder Anforderung/Antwort oder einseitig sein. Er kann auch nur eine Antwort und keine Anfrage haben, wenn er mit der Option HTTP `GET' für den HTTP-Transport verwendet wird. Anders als Web Services müssen Content-Typ der Anforderung und der Anfrage nicht gleich sein.
  • Beliebiger SOAP-Service -Wählen Sie diese Option, um einen SOAP-Service zu erstellen, der über keine explizit definierte konkrete Schnittstelle verfügt.

    • Behalten Sie den Standard SOAP 1.1 bei, oder wählen Sie SOAP 1.2 aus der Dropdown-Liste aus.

  • Beliebiger XML-Service - Wählen Sie diese Option, um einen XML-Service zu erstellen, der über keine explizit definierte konkrete Schnittstelle verfügt.

    • HTTP GET wird nur für Messaging-Services und diesen Service-Typ verwendet.

  • Business-Service - Wählen Sie diese Option, um einen vorhandenen Business-Service zu clonen.

    • Geben Sie den Pfad (Projekt/Ordner) und den Namen des Business-Service ein, oder klicken Sie auf Durchsuchen, um den Business-Service auf der Seite Business-Services - Zusammenfassung auszuwählen.

    Da Oracle Service Bus nicht für mehrere Services den gleichen URI akzeptiert, müssen Sie den URI für den geclonten Service ändern.

  • Proxy-Service - Wählen Sie diese Option, um einen Business-Service basierend auf einem vorhandenen Proxy-Service zu erstellen.
Service-Typ (Fortsetzung)
Hinweis: Wenn ein Service über einen anderen Service erstellt wird, werden Alert-Regeln auf folgende Weise beibehalten:
    • Wird ein Proxy-Service aus einem Business-Service oder ein Business-Service aus einem Proxy-Service erstellt, werden die Alert-Regeln, sofern vorhanden, entfernt.
    • Wird ein Proxy-Service aus einem anderen Proxy-Service oder ein Business-Service aus einem anderen Business-Service erstell, werden die Alert-Regeln, sofern vorhanden, beibehalten.

Wenn Sie fertig sind

Klicken Sie auf Weiter, um auf der nächsten Seite mit der Konfiguration dieses Service fortzufahren, oder klicken Sie auf Letzte, um die Konfiguration auf der Seite Business-Service erstellen/bearbeiten – Übersicht zu prüfen und zu speichern.
