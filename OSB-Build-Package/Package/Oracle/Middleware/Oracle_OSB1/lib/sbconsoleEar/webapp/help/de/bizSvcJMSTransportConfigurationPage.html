Seite "JMS-Transportkonfiguration"

Die Seite Business-Services erstellen/bearbeiten - JMS-Transportkonfiguration gehört zu einer Reihe von Seiten zum Erstellen und Bearbeiten von Business-Services. Diese Aktionen werden in Business-Services erstellen und bearbeiten und in Business-Service-Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037) unter "JMS Transport".

Verwenden Sie diese Seite, um Transporteinstellungen für einen Business-Service über das JMS-Transportprotokoll zu konfigurieren. Tabelle 18-13 beschreibt die Verwendung dieser Seite.

Tabelle 18-13 Business-Services erstellen und bearbeiten - Seite "JMS-Transportkonfiguration (Fortsetzung)"
Option
Verwendung
Zieltyp
Wählen Sie einen Typ für das JMS-Bridge-Ziel:
  • Queue (für Point-to-Point)
  • Topic (für Veröffentlichen/Abonnieren)
Nachrichtentyp
Wählen Sie eine der folgenden Optionen:
  • Byte (für einen Stream nicht interpretierter Byte)
  • Text (für Textnachrichten)
Diese Option ist deaktiviert, wenn Sie einen Java-Nachrichtentyp für die Antwort auswählen.
Antwort-Queues
Diese Option ist nur verfügbar, wenn "Queue" als Zieltyp gewählt wurde.
Wählen Sie eine der folgenden Antwortoptionen:
  • Keine - Keine Antwort erwartet. Wählen Sie diese Option für unidirektionale Vorgänge.
  • Eine für alle Anforderungs-URIs - Sie können einen einzigen URI eingeben, um die Antwort zu verarbeiten, oder andere Antwortkonfigurationsdetails festlegen, wie Codierung und Timeout, und optional einen JMS-Service-Account wählen, um die JMS/JNDI-Zugangsdaten zu übergeben.
  • Eine pro Anforderungs-URI - Diese Option ermöglicht Antwort-Failover. Geben Sie dafür einen Antwort-URI oder ein Ziel für jeden Anforderungs-URI ein.
Sie können optional einen Service-Account für JMS/JNDI-Zugangsdaten für jedes Anforderungs-/Antwortpaar wählen.
Antwortmuster
Diese Option ist nur verfügbar, wenn Sie im Feld "Antwort-Queue" eine Antwortoption wählen.
Wählen Sie eine der folgenden Optionen:
  • Wählen Sie JMSCorrelationID für alle Services mit Ausnahme von JAX-RPC-Services, die auf Oracle WebLogic Server ausgeführt werden.
  • Wählen Sie JMSMessageID für JAX-RPC-Services, die auf Oracle WebLogic Server ausgeführt werden.
Weitere Informationen finden Sie unter "Message ID and Correlation ID Patterns for JMS Request/Response" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051.
Dispatch Policy
Wählen Sie diejenige Instanz von Oracle WebLogic Server Work Manager, die Sie für die Dispatch Policy für diesen Endpunkt verwenden möchten. Falls kein anderer Work Manager vorhanden ist, wird der standardmäßige Work Manager verwendet.
Beispiel: Wenn der Business-Service über ein JMS-Transportprotokoll verfügt, ist der Business-Service-Endpunkt eine MDB-(Message-driven Bean-)JAR-Datei, die Sie mit der Dispatch Policy verknüpfen können.
Weitere Informationen zu Work Manager finden Sie unter "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Anforderungscodierung
Geben Sie den Zeichensatz für die Codierung von Anforderungen ein. Standard ist UTF-8.
Antwortcodierung
Diese Option ist nur verfügbar, wenn im Feld "Antwort-Queues" eine der Antwortoptionen gewählt wurde.
Geben Sie den Zeichensatz für die Codierung von Antworten ein. Standard ist UTF-8.
Antwort-Timeout
Diese Option ist nur verfügbar, wenn im Feld "Antwort-Queues" eine der Antwortoptionen gewählt wurde.
Geben Sie die Anzahl der Sekunden ein, während der auf die Antwort gewartet werden soll, bevor die Verbindung unterbrochen wird. Der Standardwert Null (0) bedeutet, es gibt nie einen Timeout für die Antwort.
Client-JAR-Archiv
Diese Option ist verfügbar, wenn der Service ein Messaging-Service mit Java-Antworttyp ist. Wählen Sie die Client-JAR-Datei, die für das Entfernen von Nachrichten mit Java-Objekten aus der Queue verwendet werden soll. Durch die Auswahl der Client-JAR-Datei wird sichergestellt, dass diese sich im Classpath befindet.

Weitere Informationen dazu finden Sie unter "Sending and Receiving Java Objects in Messages" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Antwort-URI
Diese Option ist verfügbar, wenn Sie die Antwortoption "Eine für alle Anforderungs-URIs" und das Antwortmuster "JMSCorrelationID" wählen.
Geben Sie einen Antwort-URI im folgenden Format ein:
jms://host:port/connection_factory/jndi_destination
Verwenden Sie das folgende Format, wenn Sie mehrere Server als Ziel definieren möchten:
jms://host1:port,host2:port/connection_factory/jndi_destination
Sie können den Host und den Port im Antwort-URI auch auslassen. Beispiel:
jms:///connection_factory/jndi_destination

Wenn Sie Host und Port auslassen, wird die Connection Factory/das Ziel-Lookup auf dem lokalen Server angezeigt. Dies ist nützlich, falls der Anforderungs-URI zu einer fremden Connection Factory bzw. einem fremden Ziel-Lookup führt, Sie aber die Antwort an den lokalen Server senden möchten.

Hinweis: Während bei Oracle WebLogic Server Vorwärtsschrägstriche in JNDI-Namen (Beispiel: "myqueues/myqueue") zulässig sind, stören JNDI-Namen mit Vorwärtsschrägstrichen das URI-Format, das für Oracle Service Bus erforderlich ist, und Sie können diese Namen nicht verwenden. Um dieses Problem zu umgehen, definieren Sie einen JMS-Fremdserver, und referenzieren Sie diesen Fremdserver im URI. Weitere Informationen finden Sie unter "Configure foreign servers" in der Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help unter http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Anforderungsantworten
Diese Option ist verfügbar, wenn Sie die Antwortoption "Eine pro Anforderungs-URI" für das Muster "JMSCorrelationID" wählen, um ein Antwort-Failover bereitzustellen.
Geben Sie für jeden Anforderungs-URI, der auf der allgemeinen Transportseite eingegeben wurde, einen Antwort-URI im zuvor genannten Feld "Antwort-URI" ein.
Sie können einen optionalen Service-Account für JMS/JNDI-Zugangsdaten wählen, die der Service sowohl für Anforderungs- als auch Antwort-Queues verwendet.
Ziel - Antworten
Diese Option ist verfügbar, wenn Sie die Antwortoption "Eine für alle Anforderungs-URIs" für das Muster "JMSMessageID" wählen.
Geben Sie den Namen des Zielservers ein, der die Antworten empfangen soll. Geben Sie einen Antwort-URI in das Feld "Antwort-URI" ein.
Anforderungsverbindungen
Diese Option ist verfügbar, wenn Sie die Antwortoption "Eine pro Anforderungs-URI" für das Muster "JMSMessageID" wählen, um ein Antwort-Failover bereitzustellen.
Geben Sie optional für jeden Anforderungs-URI, der fortlaufend nach Nummer im Feld "Folge-Nr." aufgeführt ist, einen JMS-Connection Factory-Namen ein. Falls Sie keinen Namen eingeben, verwendet der JMS-Transport die Connection Factory aus dem Anforderungs-URI.
Sie können einen optionalen Service-Account für JMS/JNDI-Zugangsdaten wählen, die der Service sowohl für Anforderungs- als auch Antwort-Queues verwendet.
Ziel - Ziele
Diese Option ist verfügbar, wenn Sie die Antwortoption "Eine pro Anforderungs-URI" für das Muster "JMSMessageID" wählen, um ein Antwort-Failover bereitzustellen. Verwenden Sie dieses Feld im Zusammenhang mit Verbindungsanforderungen.
Jeder aufgelistete Zielserver (festgelegt durch die Server in der aktuellen Domain), wie Managed Server in einem Cluster, listet die Anforderungs-URIs nach Folgenummern auf, die den URIs im Feld "Anforderungsverbindungen" entsprechen. Geben Sie für jeden Anforderungs-URI der einzelnen Ziele die Ziel-Queue auf dem Zielserver ein, der die Antworten empfängt.
Hinweis: Da die Entwicklungsumgebung für Oracle Service Bus nur eine Einzelserverumgebung unterstützt, wird nur ein Ziel aufgelistet. Wenn Sie dieses Feld in einer Mehrserverumgebung konfigurieren möchten, stellen Sie den Service in der Laufzeitumgebung bereit, und schließen Sie die Servicekonfiguration in der Oracle Service Bus Administration-Konsole ab.
JMS-Service-Account
Diese Option ist verfügbar, wenn Sie im Feld "Antwort-Queues" die Option "Keine" oder "Eine für alle Anforderungs-URIs" verwenden.
Wählen Sie einen Service-Account, der für die vom JMS-Server verwaltete JMS-Ressource verwendet werden soll. Ein Service-Account ist eine Aliasressource für Benutzer-ID und Kennwort, welche für Anforderung und Antwort benötigt werden. Derselbe Service-Account wird für JMS- und JNDI-Zwecke verwendet.
Das Feld schließt sich gegenseitig mit der Option "Caller Subject übergeben" aus. Verwenden Sie entweder die eine oder die andere Option für die JMS-/JNDI-Authentifizierung.
Weitere Informationen finden Sie unter Service-Accounts.
SSL verwenden
Wählen Sie diese Option nur, wenn die Anforderungen über eine TLS-/SSL-Verbindung gestellt werden.
TLS/SSL (Secure Sockets Layer) stellt sichere Verbindungen bereit, indem zwei über ein Netzwerk verbundene Anwendungen die Identität der jeweils anderen authentifizieren und indem die zwischen den Anwendungen ausgetauschten Daten verschlüsselt werden. Mit Authentifizierung kann ein Server, und optional auch ein Client, die Identität einer Anwendung am anderen Ende des Netzwerks prüfen. Des Weiteren muss der Service eine Authentifizierung durchführen, wenn der Administrator den Zugriff auf einzelne JMS-Ziele (Queues oder Topics) durch Einrichten einer Zugriffskontrolle beim JNDI-Eintrag für das Ziel beschränkt hat und wenn dann ein Lookup des Eintrags in der JNDI-Baumstruktur erfolgt. Die Authentifizierung erfolgt über einen Service-Account oder die Option "Caller Subject übergeben".
Ablauf
Das Zeitintervall (in Millisekunden), nach dem die Nachricht abläuft. Standardwert ist 0, das bedeutet, die Nachricht läuft niemals ab.
Nachrichtenpersistenz aktivieren
Der JMS-Zustellungsmodus, den der Service verwendet und mit dem Sie Zuverlässigkeit und Durchsatz festlegen können. Wählen Sie diese Option (Standard) für eine garantierte Nachrichtenzustellung. Deaktivieren Sie diese Option, wenn Sie den Durchsatz erhöhen möchten und ein gelegentlicher Nachrichtverlust hinnehmbar ist.
Unit Of Order
Geben Sie eine Nachrichten-Unit Of Order ein. Mit einer Nachrichten-Unit Of Order können Nachrichten-Producer Nachrichten in einer einzigen Einheit in Bezug auf die Verarbeitungsreihenfolge zusammenfassen. Diese einzelne Unit Of Order verlangt, dass alle Nachrichten in dieser Einheit in der Reihenfolge ihrer Erstellung verarbeitet werden.
Subject des Callers übergeben
Aktivieren Sie dieses Kontrollkästchen, damit Oracle Service Bus das authentifizierte Subject beim Senden der Nachricht übergibt.
Wenn Sie dieses Feld und die JMS-Ressourcen für Business-Service-Ziele in einer anderen Domain aktivieren, aktivieren Sie globales Vertrauen in beiden Domains. Weitere Informationen finden Sie unter "Configuring Security for a WebLogic Domain" in "Securing Oracle WebLogic Server".
Das Feld schließt sich gegenseitig mit der Option "Service-Account" aus. Verwenden Sie entweder die eine oder die andere Option für die JMS-/JNDI-Authentifizierung.
Wenn Sie dieses Feld und die JMS-Ressourcen für Business-Service-Ziele in einer anderen Domain aktivieren, aktivieren Sie globales Vertrauen in beiden Domains. Weitere Informationen finden Sie unter "Configuring Security for a WebLogic Domain" in "Oracle Fusion Middleware Securing Oracle WebLogic Server" auf der Websitehttp://www.oracle.com/pls/as111170/lookup?id=SECMG402.
JNDI-Timeout
Das JNDI-Verbindungs-Timeout (in Sekunden) wird beim Lookup des Ziels oder der Connection Factory in der JNDI-Baumstruktur verwendet.
Der Standardwert null (0) bedeutet, es tritt nie ein Timeout der Verbindung ein.
