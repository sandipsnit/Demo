Seite "MQ-Transportkonfiguration"

Die Seite Business-Service erstellen/bearbeiten - MQ-Transportkonfiguration ist eine von einer Serie zum Erstellen und Bearbeiten von Business-Services, wie in Business-Services erstellen und konfigurieren und Business-Service Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie unter "MQ Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Bevor Sie beginnen

Konfigurieren Sie eine MQ-Verbindungsressource. Siehe MQ-Verbindung.

MQ-Transport konfigurieren

Verwenden Sie diese Seite, um Transporteinstellungen für einen Business-Service unter Verwendung des nativen MQ-Transportprotokolls zu konfigurieren. Tabelle 18-14 beschreibt, wie die Seite zu verwenden ist.

Tabelle 18-14 Einen Business-Service erstellen/bearbeiten - Seite "MQ-Transportkonfiguration"
Option
Verwendung
Meldungstyp
Wählen Sie eine der folgenden Optionen:
  • Byte (für einen Stream nicht interpretierter Byte)
  • Text (für Textnachrichten)
Ist Antwort erforderlich
Wählen Sie diese Option, um anzugeben, dass eine Antwort erwartet wird, wenn eine ausgehende Nachricht gesendet wurde.
Abrufintervall
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Geben Sie ein Abrufintervall in Millisekunden ein. Der Standardwert ist 1000.
Antwortkorrelationsmuster
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Geben Sie an, ob das Antwortkorrelationsmuster basieren soll auf:
  • MessageID
  • CorrelationID
  • Dynamische Queue - Wählen Sie diese Option, wenn die WebSphere MQ-Implementierung dynamische Queues für die Antwortkorrelation verwendet. Der MQ-Transport unterstützt nur temporäre dynamische Queues.
Korrelationswert automatisch generieren
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Aktivieren Sie dieses Kontrollkästchen, um eine CorrelationID oder MessageID automatisch zu generieren.
Modell-Queue
Nur für Dynamische Queue-Antwortkorrelationsmuster. Geben Sie den Namen der Modell-Queue ein, die verwendet wurde, um die dynamische Queue zu generieren.
MQ-Antwort-URI
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Das Ziel, wo die Antwort veröffentlicht werden soll. Geben Sie einen Antwort-URI im gleichen Format ein, wie den Endpunkt-URI:
mq://local_queue_name?conn=mq_connection_resource
oder, wenn Sie dynamische Queues verwenden:
mq://dynamic_queue_prefix?conn=mq_connection_resource
Der auf 32 Zeichen begrenzte dynamic_queue_prefix, wird verwendet, um die dynamische Queue auf dem MQ-Server zu erstellen. Der Queue-Name wird das Präfix plus einer eindeutigen ID. Wenn beispielsweise dynamic_queue_prefix "Beispiel" ist, wäre der Name der dynamischen Queue "Beispiel123129083821" oder ähnlich.
Sie können auch einen Asterisk (*) als Platzhalter im Antwort-URI der dynamischen Queue verwenden. Zum Beispiel:
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
Wenn Sie im URI keinen dynamic_queue_prefix angeben, verwendet der Transport den Namen für die dynamische Queue, den der MQ-Server generiert. Wenn Sie keine ausdrückliche mq_connection_resource im URI angeben (Best Practice), verwendet der Transport die mq_connection_resource aus dem Endpunkt-URI.
Mehr und detaillierte Informationen finden Sie unter "MQ-Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.
Antwort-Timeout
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Geben Sie die Wartezeit auf eine Antwort, bevor die Verbindung unterbrochen wird, in Sekunden an.
Dispatch Policy
Dispatch Policy bezieht sich auf die Instanz von WLS Work Manager, die Sie für den Service-Endpunkt verwenden möchten.
Weitere Informationen zu Work Manager finden Sie unter "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server unter http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Pooling für dynamische Queue
Nur für Dynamic Queue-Antwortkorrelationsmuster. Wählen Sie diese Option, wenn Sie möchten, dass der Service im Pool befindliche Verbindungen zu dynamischen Queues verwendet.
Wenn Sie einen getrennten Verbindungs-Pool für dynamische Queues verwenden möchten, sollten Sie erwägen, eine dedizierte MQ-Verbindungsressource für dynamische Queues zu konfigurieren.
Wählen Sie diese Option nicht, wenn Sie eine neue dynamische Queue-Instanz für jede Anforderung erstellen möchten (und die Queue nach jeder Antwort endgültig löschen wollen).
Endpunkt-URI `PUT'-Optionen
Wählen Sie die MQ PUT-Nachrichtenoptionen aus den folgenden:
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
Sie können mehrere Optionen trennen, entweder mit "|" oder "+". Sie können beispielsweise Folgendes angeben:
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
Die MQ PUT-Nachrichtenoptionen werden angewendet, wenn die Nachricht in die ausgehende Queue gestellt wird.
MQ-URI für unerkannte Antworten
Geben Sie den URI für die Queue ein, an die die unerkannte Antwortnachricht gesendet werden soll. Beachten Sie, dass diese Einstellung nur aktiviert wird, wenn das Kontrollkästchen Korrelationswert automatisch generieren aktiviert wurde.
Wenn Sie für dieses Feld keinen Wert eingeben, werden nicht erkannte Antwortnachrichten gelöscht.
RFH2-Header verarbeiten
Wählen Sie diese Option, um WebSphere MQ RFH2-Header in einer Message-Payload zu parsen, und generieren Sie automatisch einen RFH2Headers-Transport-Header, der die RFH2-Daten enthält.
Wenn Sie diese Option nicht wählen, wird die Payload so übergeben, wie sie empfangen wurde.

Weitere Informationen dazu, wie der MQ-Transport RFH2-Header verarbeitet, finden Sie im Abschnitt "About RFH2 Headers" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
