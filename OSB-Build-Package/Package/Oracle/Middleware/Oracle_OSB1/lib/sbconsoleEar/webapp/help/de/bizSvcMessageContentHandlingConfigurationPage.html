Seite "Nachrichtenübermittlungskonfiguration"

Die Seite Business-Service erstellen/bearbeiten – Nachrichtenübermittlung konfigurieren ist eine von diversen Seiten, auf denen Sie Business-Services erstellen und bearbeiten können. Diese Funktionen sind unter Business-Services erstellen und konfigurieren und Business-Service-Konfigurationen bearbeiten genauer beschrieben.

Auf dieser Seite können festlegen, wie Nachrichtencontent in Oracle Service Bus übermittelt wird. In Tabelle 18-20 sind die Optionen für die Nachrichtenübermittlung bei Business-Services beschrieben. Für verschiedene Transporttypen stehen unterschiedliche Optionen zur Verfügung.

Tabelle 18-20 Seite "Business-Services erstellen/bearbeiten – Nachrichtenübermittlung konfigurieren" 
Option
Verwendung
XOP-/MTOM-Unterstützung
Mit dieser Funktion können Sie ausgehende Nachrichten im MTOM/XOP-Format codieren. Die SOAP-Nachrichtenübertragungsoptimierung (MTOM; Message Transmission Optimization Mechanism) ist eine Methode zum Senden von Binärdaten an und von Web Services. MTOM verwendet XOP (XML-binary Optimized Packaging) zum Übertragen der Binärdaten.
Oracle Service Bus unterstützt XOP/MTOM mit den folgenden Transporten:
  • HTTP/S
  • Lokal
  • SB
Aktivieren Sie das Kontrollkästchen Aktiviert, damit ausgehende Nachrichten vom Business-Service im MTOM/XOP-Format codiert werden. Hinweis: Diese Option ist bei importierten Business-Services, die auf Konfigurationen von früheren Releases basieren, deaktiviert.
Wenn die XOP/MTOM-Unterstützung aktiviert ist, wählen Sie eine der folgenden Optionen, um festzulegen, wie Binärdaten in den Nachrichtenkontextvariablen $header und $body verarbeitet werden sollen:
  • Binärdaten nach Referenz aufnehmen (Standard): Ersetzen Sie in einer ausgehenden Antwortnachricht beim Erstellen der Nachrichtenkontextvariable $body die xop:Include-Elemente durch ctx:binary-content-Elemente.
  • Binärdaten nach Wert aufnehmen: Ersetzen Sie in einer ausgehenden Antwortnachricht beim Erstellen der Nachrichtenkontextvariable $body die xop:Include-Elemente durch Base64-codierte Textversionen von entsprechenden Binärdaten.
Hinweis: Wenn die XOP/MTOM-Unterstützung für einen Business-Service aktiviert ist, muss nicht jede ausgehende Nachricht im MTOM-Format codiert sein. Diese Einstellung gibt stattdessen an, dass der Business-Service eine MTOM-Payload verarbeiten kann.
Da in Oracle Service Bus keine Kombination von MTOM und SwA unterstützt wird, meldet das System einen Laufzeitfehler, wenn Oracle Service Bus versucht, eine ausgehende Anforderung an einen Business-Service zu senden, der Business-Service MTOM/XOP-fähig ist und die Nachrichtenkontextvariable $attachments nicht null ist.
Anhänge
Oracle Service Bus unterstützt das Streaming von MIME-Anhängen mittels HTTP/S-Transport.
Mit dieser Funktion können Sie Anhänge in ausgehenden Antwortnachrichten als Datenträgerdatei speichern und die Daten dann durch Streaming verarbeiten, ohne den Anhangscontent zwischenzuspeichern. Dadurch können große Anhänge von Business-Services zuverlässig und effizient verarbeitet werden.
Aktivieren Sie das Kontrollkästchen Seitenanhänge auf Datenträger, damit Anhänge in ausgehenden Antwortnachrichten vom Business-Service eingelesen werden.
Hinweis: Wenn Sie die XOP/MTOM-Unterstützung aktivieren, ist die Option "Anhänge" nur verfügbar, wenn Sie unter "XOP-/MTOM-Unterstützung" die Option Binärdaten nach Referenz aufnehmen aktivieren. Zudem müssen Payloads mit Anhängen RFC 822 entsprechen. Besonders Zeilen mit Internet-Headern müssen mit CRLF (Wagenrücklauf/Zeilenvorschub) beendet werden.
Ergebnis-Caching
Wenn Sie Business-Services aufrufen, deren Ergebnisse sich selten ändern, wird die Business-Service-Performance durch Ergebnis-Caching verbessert, da gecachte Ergebnisse an den Client zurückgegeben werden, statt einen externen Service direkt aufzurufen.
Damit Sie das Ergebnis-Caching für Business-Services verwenden können, müssen Sie diese Funktion zunächst global aktivieren. Dieser Vorgang ist unter Ergebnis-Caching global aktivieren beschrieben.
Konfigurieren Sie das Ergebnis-Caching in den Erweiterten Einstellungen wie folgt:
  • Aktivieren Sie die Option Unterstützt. Anschließend werden alle anderen Felder aktiviert.
  • Ausdrucks-Namespaces: Wenn Sie für die Konfiguration des Ergebnis-Caching Ausdrücke verwenden (mit dem Cachetokenausdruck, der Ablaufzeit oder beiden) können Sie die Namespaces und entsprechenden Präfixe eingeben, die in den Ausdrücken verwendet werden sollen. In diesem Feld können Sie auch eine Liste mit vorhandenen Namespaces anzeigen.
  • Cachetokenausdruck: In Oracle Service Bus werden Cacheschlüssel verwendet, um gecachte Ergebnisse zum Abrufen oder Ausfüllen zu identifizieren, und der Cachetoken des Cacheschlüssels gibt den eindeutigen Bezeichner an. Sie können einen Ausdruck (den Cachetokenausdruck) verwenden, um den Cachetoken des Cacheschlüssels zu generieren und ein gecachtes Ergebnis für den Business-Service eindeutig zu identifizieren.

    • Wenn Sie den Cachetoken aus einem Wert in der Anforderung (im Proxy-Service-Nachrichtenfluss, der den Business-Service aufruft) generieren möchten, verwenden Sie einen Ausdruck, der den Wert aus der Pipeline $body, $header, $operation oder $transportMetaData ($outbound/ctx:transport/ctx:request oder $outbound/ctx:transport/ctx:response) abruft. Beispiel: Sie können den Cachetoken von einer Kunden-ID in der Nachricht $body abrufen.

    Der Cachetokenausdruck muss in eine Zeichenfolge oder einen Wert mit einfachem Content aufgelöst werden, z.B. ein Attribut oder ein Element ohne untergeordnete Elemente. Wenn der Ausdruck null ergibt oder einen Fehler verursacht, werden die Ergebnisse nicht gecacht.

    Hinweis: Sie können den Cachetoken auch aus der Anforderung generieren, ohne einen Cachetokenausdruck in der Business-Service-Konfiguration festzulegen. Fügen Sie hierzu einen Wert in $outbound/ctx:transport/ctx:request/ctx:cache in der Proxy-Service Pipeline ein. Alle Werte in diesem Cachetoken überschreiben den Cachetokenausdruck in der Business-Service-Konfiguration.

  • Ablaufzeit: Die Ablaufzeit oder die Gültigkeitsdauer bestimmt, wann ein Eintrag im Ergebniscache des Business-Service gelöscht wird. Wenn für die Dauer null (0) angegeben wird, wird der Eintrag nicht gelöscht. Ein negativer Wert bedeutet, dass Ergebnisse nicht im Cache gespeichert werden. Wählen Sie eine der folgenden Optionen:
 
    • Standardwert verwenden: Der Standardwert entspricht dem Ablaufverzögerungswert in der DateiDOMAIN_HOME/config/osb/coherence/osb-coherence-config.xml. Er beträgt 5 Minuten.
    • Dauer: Legen Sie eine bestimmte Zeit fest.
    • XQuery-Ausdruck:·Wie·beim·Festlegen·des·Cachetokens·bei·einem·Ausdruck·können·Sie·einen·XQuery-Ausdruck·eingeben,·der·einen·Ablaufzeitwert·von·der·Anforderung·oder·der·Antwort·erhält.· Wählen·Sie·Anforderung·oder·Antwort.
