Seite "Nachrichtentyp-Konfiguration"

Die Seite Business-Service erstellen/bearbeiten - Nachrichtentypkonfiguration ist eine von mehreren Seiten für das Erstellen und Bearbeiten von Business-Services. Weitere Informationen hierzu finden Sie unter Business-Services erstellen und konfigurieren und unter Business-Service-Konfigurationen bearbeiten.

Auf dieser Seite können Sie Nachrichtentypen für einen Business-Service mit dem Typ "Messaging Service" konfigurieren.

Die Binding-Definition für Messaging Services besteht aus dem Konfigurieren des Contenttyps der ausgetauschten Nachrichten. Der Content-Typ für die Antwort muss nicht mit dem für die Anforderung identisch sein. Deswegen wird die Antwort separat konfiguriert (Beispiel: Der Service kann eine MFL-Nachricht annehmen und einen XML-Bestätigungseingang zurückgeben).

Hinweis: Business-Services mit E-Mail-, Datei-, FTP- oder SFTP-Transport und dem Typ "Messaging Service" unterstützen nur einseitiges Messaging, der Antwortnachrichtentyp muss auf Keine gesetzt sein. Wenn Sie eine andere Option als Keine wählen, ist das Datei-, FTP- oder SFTP-Protokoll auf der Seite "Transportkonfiguration" nicht verfügbar.

In Tabelle 18-2 wird die Verwendung der Seite Business-Service erstellen/bearbeiten - Nachrichtentypkonfiguration beschrieben.

Tabelle 18-2 Seite "Business-Service erstellen/bearbeiten - Nachrichtentypkonfiguration"
Option
Verwendung
Anforderungsnachrichtentyp
Wählen Sie einen Nachrichtentyp für die Anforderungsnachricht:
  • Keine - Wählen Sie diese Option, wenn keine Anforderungsnachricht vorhanden ist (Beispiel mit HTTP GET).
  • Binär - Wählen Sie diese Option, wenn der Content-Typ der Nachricht unbekannt oder unwichtig ist.
  • Text - Wählen Sie diese Option, wenn die Nachricht auf Text beschränkt werden kann.
  • MFL - Wählen Sie diese Option, wenn die Nachricht ein Binärdokument ist, das einer MFL-Definition entspricht. Sie können nur eine MFL-Datei konfigurieren.

    • Für MFLs können Sie auf Durchsuchen klicken, um eine MFL aus dem MFL-Browser auszuwählen. Klicken Sie anschließend auf Weiterleiten.

  • XML - Wählen Sie diese Option, wenn es sich bei der Nachricht um ein XML-Dokument handelt. Um Typinformationen anzugeben, können Sie den XML-Schematyp des ausgetauschten XML-Dokuments deklarieren.
  • Java - Wählen Sie diese Option, wenn die gesendete Anforderung ein Java-Objekt enthält. Der JMS-Transport wird für Nachrichten mit Java-Objekten verwendet. Weitere Informationen finden Sie im Abschnitt "Sending and Receiving Java Objects in Messages" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Antwortnachrichtentyp
Wählen Sie einen Nachrichtentyp für die Antwortnachricht:
  • Keine - Wählen Sie diese Option, wenn keine Antwortnachricht vorhanden ist.
  • Binär - Wählen Sie diese Option, wenn der Content-Typ der Nachricht unbekannt oder unwichtig ist.
  • Text - Wählen Sie diese Option, wenn die Nachricht auf Text beschränkt werden kann.
  • MFL - Wählen Sie diese Option, wenn die Nachricht ein Binärdokument ist, das einer MFL-Definition entspricht. Sie können nur eine MFL-Datei konfigurieren.

    • Für MFLs können Sie auf Durchsuchen klicken, um eine MFL aus dem MFL-Browser auszuwählen. Klicken Sie anschließend auf Weiterleiten.

  • XML - Wählen Sie diese Option, wenn es sich bei der Nachricht um ein XML-Dokument handelt. Um Typinformationen anzugeben, können Sie den XML-Schematyp des ausgetauschten XML-Dokuments deklarieren.
  • Java - Wählen Sie diese Option, wenn die empfangene Antwort ein Java-Objekt enthält. Der JMS-Transport wird für Nachrichten mit Java-Objekten verwendet. Weitere Informationen finden Sie im Abschnitt "Sending and Receiving Java Objects in Messages" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
