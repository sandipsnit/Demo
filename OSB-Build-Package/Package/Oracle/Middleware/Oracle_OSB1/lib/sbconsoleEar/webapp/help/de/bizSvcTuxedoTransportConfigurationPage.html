Seite "Tuxedo-Transportkonfiguration"

Die Seite Business-Service erstellen/bearbeiten - Tuxedo-Transportkonfiguration ist eine Seite von mehreren, auf denen Business-Services erstellt und bearbeitet werden können, wie in Business-Services erstellen und konfigurieren und Business-Service-Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie unter "Tuxedo Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Auf dieser Seite können Sie die Transporteinstellungen für einen Business-Service konfigurieren, der das Tuxedo-Transportprotokoll verwendet. In Tabelle 18-18 wird die Verwendung dieser Seite beschrieben.

Tabelle 18-18 Business-Service erstellen/bearbeiten - Seite "Tuxedo-Transport konfigurieren" (Fortsetzung)
Option
Verwendung
Feldtabellenklassen
Geben Sie die Namen der Klassen ein, die den erhaltenen FML/FML32-Puffer beschreiben. Diese werden für die FML/FML32-to-XML-Konvertierungsroutinen verwendet, um die Feldnamen den Elementnamen zuzuordnen. Dies ist eine durch Leerzeichen getrennte Liste mit voll qualifizierten Klassennamen.
Klassen anzeigen
Geben Sie den Namen der Klasse(n) ein, die den erhaltenen oder gesendeten VIEW/VIEW32-Puffer beschreiben. Diese werden für die VIEW-to-XML oder die VIEW32-to-XML-Konvertierungsroutinen verwendet, um die Feldnamen den Elementnamen zuzuordnen. Dies ist eine durch Leerzeichen getrennte Liste mit voll qualifizierten Klassennamen.
JAR-Klassen
Wählen Sie eine JAR-Ressource, die eine JAR-Datei mit den Klassen FML/FML32 oder VIEW/VIEW32 enthält, die für diesen Endpunktvorgang benötigt werden.
Remote-Zugriffspunkte
Wählen Sie einen aus der Dropdown-Liste aus, die mit dem Import verknüpft ist. Diese Liste enthält die in WTC konfigurierten Remote-Zugriffspunkte. Ein Business-Service kann nicht ohne verknüpften Remote-Zugriffspunkt erstellt werden.
Wenn kein Remote-Zugriffspunkt vorhanden ist oder wenn Sie einen neuen erstellen möchten, klicken Sie auf Neu. Geben Sie den entsprechenden Zugriffspunktnamen und die Netzwerkadresse in den nebeneinander liegenden Feldern ein. Nach der Validierung des Endpunkts wird der Zugriffspunkt für jeden WTC-Server zur WTC-Konfiguration hinzugefügt. Ist kein WTC-Server vorhanden, wird einer erstellt.
Wenn mehr als ein URI angegeben wurde, gibt es ein Remote-Zugriffspunktfeld pro URI, und der URI wird zu informativen Zwecken angezeigt. Wenn mehr als ein URI vorhanden ist, benötigt jeder einen anderen Remote-Zugriffspunkt. Wenn der angegebene URI bereits einer vorhandenen WTC-Ressource entspricht, wird der entsprechende Remote-Zugriffspunkt angezeigt, kann jedoch nicht geändert werden.
Lokale Zugriffspunkte
Dieses Feld wird nur angezeigt, wenn Sie im Feld Remote-Zugriffspunkt den Wert Neu wählen.
Wählen Sie aus der Dropdown-Liste einen lokalen Zugriffspunkt aus, der mit dem neu erstellten Remote-Zugriffspunkt verknüpft werden soll. Wenn keiner vorhanden ist oder wenn Sie einen neuen erstellen möchten, wählen Sie Neu. Geben Sie den entsprechenden Namen des lokalen Zugriffspunktes und die Lokale Netzwerkadresse in die nebeneinander liegenden Felder ein.
Typ des Anforderungspuffers
Wählen Sie den Typ des Puffers aus, den der Remote-Tuxedo-Service erhalten soll.
Untertyp des Anforderungspuffers
Diese Option ist aktiviert, wenn der Wert des vorherigen Feldes Anforderungspuffertyp entweder VIEW oder VIEW32 ist. Geben Sie den Untertyp des Puffers ein, der mit dem Anforderungspuffer verknüpft werden soll.
Antwort erforderlich?
Aktivieren Sie dieses Kontrollkästchen, um einen bidirektionalen Aufruf anzugeben. Wird dieses Feld nicht angekreuzt, wird der zugrunde liegende tpcall mit dem Kennzeichen TPNOREPLY aufgerufen und asynchron eine Nullantwort versendet.
Transaktion unterbrechen?
Aktivieren Sie dieses Kontrollkästchen, um die Transaktion, sofern vorhanden, zu unterbrechen. Dies kann sinnvoll sein, wenn der Remote-Service keine Transaktionen unterstützt.
Dispatch Policy
Wählen Sie die Instanz des WebLogic Server Work Managers, die Sie für die Dispatch Policy dieses Endpunkts verwenden möchten. Der Standard-Work Manager wird verwendet, wenn kein anderer Work Manager vorhanden ist.
Dieser Work Manager wird verwendet, um im Falle eines OneWay-Aufrufs asynchron eine Nullantwort zu versenden. Weitere Informationen zu Work Managern finden Sie unter "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server unter http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Anforderungscodierung
Geben Sie eine Zeichensatzcodierung für Anforderungen in Tuxedo-Transporten an.
Antwortcodierung
Geben Sie eine Zeichensatzcodierung für Antworten in Tuxedo-Transporten an.
Timeout
Geben Sie in Sekunden an, wie lange die Transportlaufzeit auf Antworten wartet. Geben Sie eine Ganzzahl ein, die größer oder gleich null ist. Wenn kein Wert angegeben wird oder der Wert null ist (Standardwert), findet der Timeout für Antworten nach der BLOCKTIME (Blockierungszeit) statt. Dies ist die maximale Anzahl von Sekunden, die der lokale WTC-Zugriffspunkt für das Blockieren eines Aufrufs zulässt.
Transformationsstil
Wählen Sie eine der folgenden Optionen:
  • Keine: (Standardwert) Die Reihenfolge der Felder muss nicht beachtet werden.
  • Sortiert: Die Felder werden bei allen Vorkommnissen in der richtigen Reihenfolge dargestellt.
  • Sortiert und gruppiert: Wenn die Felder logisch wie Datensätze strukturiert sind, werden sie nach Auftreten sortiert und nach Datensatz gruppiert.
