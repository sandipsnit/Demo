Seite "WS-Transportkonfiguration"

Die Seite Business-Service erstellen/bearbeiten - WS-Transportkonfiguration ist eine Seite von mehreren, auf denen Business-Services erstellt und bearbeitet werden können, wie in Business-Services erstellen und konfigurieren und Business-Service-Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie unter "WS Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083.

Auf dieser Seite können Sie die Transporteinstellungen für einen Business-Service konfigurieren, der das WS-Transportprotokoll verwendet. In Tabelle 18-19 wird die Verwendung dieser Seite beschrieben.

Tabelle 18-19 Business-Service erstellen/bearbeiten - Seite "WS-Transportkonfiguration"
Option
Verwendung
Antwort-Timeout
Geben Sie an, wie viele Sekunden auf eine Antwort gewartet werden soll.
Wenn Sie dieses Feld leer lassen, gibt es keine Zeitüberschreitung. Der Business-Service wartet auf die Dauer des Reihenfolge-Timeouts, der in der RM-Policy konfiguriert wurde.
Wenn Sie null (0) eingeben, ist kein Timeout definiert, es gibt also keine Zeitüberschreitung.
Service-Account
Geben Sie den Service-Account ein, oder klicken Sie auf Durchsuchen, um einen aus der angezeigten Liste auszuwählen.
Der Service-Account gibt an, welche Zugangsdaten verwendet werden, wenn für diesen Service eine HTTP-Basisauthentifizierungs-Policy vorliegt.
Erweiterte Einstellungen
Queue-Fehlermeldungen
Aktivieren Sie das Kontrollkästchen, um das Senden von Fehlermeldungen an die konfigurierte Fehler-Queue zu aktiveren.
Fehler-Queue-URI
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Fehler-Queue-Meldungen gewählt ist.
Geben Sie den URI der JMS-Queue für das Speichern von Fehlermeldungen in folgendem Format ein:

jms://host:port/connFactoryJndiName/queueJndiName

Hinweis: Während bei Oracle WebLogic Server Vorwärtsschrägstriche in JNDI-Namen (Beispiel: "myqueues/myqueue") zulässig sind, stören JNDI-Namen mit Vorwärtsschrägstrichen das URI-Format, das für Oracle Service Bus erforderlich ist, und Sie können diese Namen nicht verwenden. Um dieses Problem zu umgehen, definieren Sie einen JMS-Fremdserver, und referenzieren Sie diesen Fremdserver im URI. Weitere Informationen finden Sie unter "Configure foreign servers" in der Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help unter http://www.oracle.com/pls/as111170/lookup?id=WLACH.

JMS-Fehler-Queue-Service-Account
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Fehler-Queue-Meldungen gewählt ist.
Geben Sie den Service-Account ein, oder klicken Sie auf Durchsuchen, um einen aus der angezeigten Liste auszuwählen.
Der Service-Account gibt die Zugangsdaten an, die bei JNDI-Suchen und dem Senden von Fehlermeldungen an die Fehler-Queue verwendet werden.
SSL für Fehler-Queue verwenden
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Fehler-Queue-Meldungen gewählt ist.
Aktivieren Sie das Kontrollkästchen, um SSL für die Verbindung zur Fehler-Queue zu verwenden.
