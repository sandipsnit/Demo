Vorgangseinstellungen für Business-Services konfigurieren

Auf der Seite Vorgangseinstellungen können Sie Business-Services und das Überwachen bestimmter Business-Services aktivieren und deaktivieren. Auf ähnliche Weise können Sie diese Service-spezifischen Vorgangseinstellungen auf der Seite Vorgänge > Smart Search mit folgenden Ausnahmen angeben: Sie können kein Aggregationsintervall festlegen und keine Dringlichkeitsstufe für Warnungen angeben. Weitere Informationen finden Sie unter Vorgangseinstellungen suchen und aktualisieren.

Die Laufzeitauswirkungen der Einstellungen für die Service-Ebene hängen von ihren jeweiligen globalen Einstellungen ab. Sie müssen sowohl die globalen als auch die Einstellungen auf Service-Ebene aktivieren, damit ein Service zur Laufzeit komplett aktiviert ist. Außerdem muss auch der Service-Status aktiviert sein. Siehe ###Globale Einstellungen aktivieren.

Sie können Überwachungskonfigurationseinstellungen auch dann ändern und speichern, wenn der Service nicht zur Laufzeit aktiviert wird. Beispiel: Sie können das Aggregationsintervall auch dann ändern und speichern, wenn die Service-Überwachung deaktiviert ist. Auf diese Weise können Sie Einstellungen bearbeiten und später aktivieren.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Ressourcenbrowser > Business-Services.
  3. Klicken Sie auf den Namen des Business-Service.

    4. Auf der Seite Business-Service anzeigen werden die Informationen aus der Tabelle 25-8 angezeigt.

    Tabelle 25-8: Business-Service-Informationen 
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der diesen Service erstellt, bearbeitet oder in die Konfiguration importiert hat.
    Zuletzt geändert am
    Das Datum oder die Uhrzeit, zu der der Benutzer diesen Service erstellt, bearbeitet oder in die Konfiguration importiert hat. Klicken Sie auf den Link von Datum und Uhrzeit, um die Änderungshistorie dieser Ressource anzuzeigen. Siehe Seite "Änderungshistorie anzeigen"
    Referenzen
    Die Anzahl der Objekte, die dieser Service referenziert. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Siehe ###Referenzen für Ressourcen anzeigen
    Referenziert von
    Die Anzahl der Objekte, die diesen Service referenzieren. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Siehe ###Referenzen für Ressourcen anzeigen
    Beschreibung
    Eine Beschreibung des Service, sofern vorhanden.

  4. Klicken Sie auf die Registerkarte Vorgangseinstellungen.

    5. Sie müssen sich in einer aktiven Session befinden, um die in der Tabelle 25-9 angezeigten Business-Service-Vorgangseinstellungen aktivieren zu können.

    Tabelle 25-9 Business-Service-Vorgangseinstellungen 
    Ziel
    Vorgehensweise
    Status deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session aktivieren, stoppt der Service das Verarbeiten der Nachrichten.
    Wenn Sie einen Service deaktivieren, löscht das System nicht mehr alle zuvor für diesen Service erfassten Überwachungsstatistiken.
    Status aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session aktivieren, nimmt der Service das Verarbeiten der Nachrichten wieder auf.
    Offlinesetzen von Business-Service-Endpunkt-URIs ermöglichen
    Wiederholungsintervall für Offline-URIs festlegen
    Aktivieren Sie das Kontrollkästchen Mit Wiederholungsintervall aktivieren.
    Wenn Sie diese Option aktivieren, entfernt der Business-Service nicht antwortende Endpunkt-URIs zur Laufzeit (setzt sie offline), so dass nur antwortende URIs für Wiederholungsversuche und für das Verarbeiten von Folgeanforderungen verwendet werden.
    Wenn Sie nicht antwortende URIs so lange offline setzen möchten, bis Sie eine Korrektur vorgenommen haben, und die URIs wieder aktivieren, geben Sie kein Wiederholungsintervall an. Beispiel: Ein Wiederholungsintervall von null gibt an, dass der Endpunkt unbegrenzt offline bleibt. Um ihn wieder online zu setzen, verwenden Sie die Aktion Endpunkt online markieren auf der Seite "Endpunkt-URIs". Siehe ###Metriken für Business-Service-Endpunkt-URIs anzeigen.
    Wenn Sie den Business-Service so konfigurieren möchten, dass nicht antwortende URIs offline gesetzt und nach einem bestimmten Zeitintervall für die Folgenachrichtenverarbeitung erneut auf denselben URI-Endpunkt zugegriffen wird, geben Sie ein Wiederholungsintervall an. Geben Sie in den Feldern Stunden, Minuten und Sekunden an, wie lange die nicht antwortenden URIs aus der Verarbeitungsschleife aktiver URIs herausgehalten werden sollen.
    Der URI wird für das angegebene Zeitintervall offline gesetzt, dann wird ein neuer Versuch gestartet. Wenn der Endpunkt antwortet, wird der URI wieder online gesetzt. Andernfalls bleibt er offline, und der Prozess wiederholt sich.
    Sie können nur dann ein Wiederholungsintervall angeben, wenn Sie die Option Mit Wiederholungsintervall aktivieren gewählt haben.
    Überwachung deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session aktivieren, stoppt das System das Erfassen der Überwachungsstatistiken für den Service und löscht alle zuvor erfassten Statistiken.
    Überwachung aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session aktivieren, startet das System das Erfassen der Überwachungsstatistiken für den Service.
    Aggregationsintervall für einen Service festlegen
    Wählen Sie ein Intervall in Stunden oder Minuten und klicken Sie dann auf Aktualisieren. Wenn Ihre Auswahl von Stunden höher ist als 1, ist der Standardwert für Minuten stets null. Wenn die Auswahl der Stunden jedoch 0 oder 1 ist, können Sie Intervalle in Minuten konfigurieren. Siehe ###Aggregationsintervall für einen Service festlegen.
    Sie können die Einstellung des Aggregationsintervalls unabhängig vom Aktivieren der Überwachung ändern und speichern.
    Um die Überwachung zur Laufzeit zu aktivieren, stellen Sie sicher, dass das Kontrollkästchen Überwachung aktivieren aktiviert ist.
    SLA-Alerts deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert. Das System stoppt die Evaluierung aller für diesen Service konfigurierten Alert-Regeln. Daher erhalten Sie keine zu diesem Service gehörigen SLA-Alerts mehr.
    SLA-Alerts aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert. Geben Sie in der Dropdown-Liste Wenn Schweregradsstufe an, dass SLA-Alerts auf die angegebene Stufe oder höher beschränkt werden (Auflistung von der umfassendsten bis zur restriktivsten Stufe): Normal (Standardwert), Warnung, Unbedeutend, Gravierend, Kritisch und Schwerwiegend.
    Service-SLA-Alerts hängen sowohl vom globalen Status als auch vom Status der Service-Ebene ab. Beide müssen für SLA-Alerts zur Laufzeit aktiviert sein. Siehe SLA-Alerts global aktivieren.
    Fluss der Nachrichten einschränken, die an die festgelegten Endpunkte gesendet wurden, die für den Business-Service konfiguriert wurden
    Aktivieren Sie das Kontrollkästchen Aktiviert neben Throttling-Status. Wenn es aktiviert ist, werden die Nachrichten nach Priorität verarbeitet. Sie können Nachrichten optional mit Routingoptionen eine Priorität zuweisen. Ansonsten werden die Nachrichten nach dem Prinzip "First-In-First-Out" sortiert.
    Weitere Informationen finden Sie unter "Throttling" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG614.
    Anzahl der Nachrichten einschränken, die an die festgelegten Endpunkte verteilt werden, die für den Business-Service konfiguriert wurden
    Wenn Throttling-Status aktiviert ist, muss ein Wert in dieses Feld eingegeben werden.
    Geben Sie einen Wert für Maximaler gleichzeitiger Zugriff ein. Dieser Wert muss eine positive Ganzzahl sein. Er darf nicht null sein.
    Wenn dieser Wert überschritten wird, werden eingehende Nachrichten in eine Queue (Throttling Queue) gestellt, sofern eine solche vorhanden ist. Andernfalls werden die Nachrichten verworfen.
    Wenn die Queue voll ist, werden Nachrichten mit einer niedrigeren Priorität entfernt, und ein Laufzeitfehler wird ausgegeben. Wenn ein Error Handler konfiguriert ist, wird dieser ausgelöst. Die neu eingehenden Nachrichten werden in die Queue gestellt.
    Angeben, ob Nachrichten, die an Business-Services gesendet wurden und deren maximaler gleichzeitiger Zugriff überschritten wurde, in eine Queue gestellt werden sollen
    Festlegen, wie viele Nachrichten maximal in der Queue gehalten werden sollen
    Wenn Throttling-Status aktiviert ist, muss ein Wert in dieses Feld eingegeben werden.
    Geben Sie einen Wert für die Throttling Queue ein. Dieser muss eine positive Ganzzahl sein. Oder übernehmen Sie den Standardwert, null. Dies bedeutet, dass keine Throttling Queue erstellt wird. Wenn der Wert für "Throttling Queue" null ist (keine Throttling Queue) und die Anzahl der Nachrichten den Wert in Maximaler gleichzeitiger Zugriff überschreitet, werden die Nachrichten verworfen, und ein Fehler tritt auf. Wenn ein Error Handler konfiguriert ist, wird dieser ausgelöst.
    Wenn Sie Throttling zur Laufzeit deaktivieren, werden alle Nachrichten in der Throttling Queue verarbeitet.
    Wenn Sie einen Business-Service löschen oder umbenennen, werden alle Nachrichten in der Throttling Queue verworfen.
    Maximale Dauer für Nachrichten in der Queue angeben
    Wenn Throttling-Status aktiviert ist, ist die Eingabe eines Wertes in dieses Feld optional.
    Geben Sie einen Wert für Nachrichtenablauf an. Dieser Wert muss eine positive Ganzzahl sein. Der Wert null bedeutet, dass die Nachricht nie abläuft.
    Dieser Wert wird auf der Ebene der Queue eingerichtet. Alle Nachrichten in der Queue haben denselben Nachrichtenablauf. Wenn dieser überschritten wird, werden die Nachrichten unverarbeitet aus der Queue entfernt.
    Nachrichten-Tracing für einen Business-Service aktivieren

    Aktivieren Sie das Kontrollkästchen Aktiviert neben dem Feld Nachrichten-Tracing. Wenn Sie das Nachrichten-Tracing aktiviert haben, protokolliert das System die zwischen der Oracle Service Bus-Pipeline und dem Business-Service ausgetauschten Nachrichten (ausgehende Anforderungs- und Antwortnachrichten).

    Hinweis: Um das Tracing in der Log-Datei oder im Standardausgang (Serverkonsole) anzuzeigen, muss die Anmeldung bei Oracle WebLogic Server auf die folgenden Dringlichkeitsstufen eingestellt werden:

    Mindestdringlichkeit für das Log: Info
    Log-Datei: Info
    Standardausgang: Info

    Weitere Informationen zum Einstellen der Dringlichkeitsstufen für das Log finden Sie im entsprechenden Thema unter Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

    Beachten Sie, dass im betreffenden Fall auch protokollierte ausgehende Nachrichten Wiederholungsanzahl, Fehlercode und Fehlermeldung beinhalten können.
    Wählen Sie aus der Dropdown-Liste Detailebene eine der folgenden Detailebenen:
    • Terse: Zeigt Datum, Zeit, Service-Art, Service-Name und URI an
    • Header: Zeigt Terse-Daten zusammen mit der XML-Darstellung der Nachrichtenmetadaten an
    • Vollständig: Zeigt Header-Daten zusammen mit der RAW-Payload sowie vorhandenen Anhängen an
    Wenn Sie in der Dropdown-Liste Detailebene den Wert "Vollständig" wählen, geben Sie die maximale Größe (in Kilobyte) für die Nachrichten-Payload im Feld Payload Tracing-Grenzwert an. Geben Sie außerdem die Standardcodierung für das Logging der Payload im Feld Standardcodierung an. Dies kann sinnvoll sein, wenn Sie binäre Payloads oder SOAP-Nachrichten mit Binäranhängen protokollieren.
    Der Wert für die Standardcodierung kann "Base64" oder jede von Java unterstützte Codierung sein, wie in http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html definiert.
    Wenn Sie das Feld Standardcodierung leer lassen, verwendet Oracle Service Bus für die Payload die Standardcodierung des Hosts. Die Standardcodierung hängt von einer Kombination aus JVM, dem zugrunde liegenden Betriebssystem (BS) und den Gebietsschemaeinstellungen auf BS-Ebene ab.
    Wenn die im Feld Standardcodierung angegebene Einstellung nicht verwendet werden kann (z.B. weil sie für die Konfiguration keine gültige Option ist), verwendet Oracle Service Bus die Base64-Codierung für die Payload.
    Nachrichten-Tracing für einen Business-Service deaktivieren
    Deaktivieren Sie neben Nachrichten-Tracing das Kontrollkästchen Aktiviert.
    Ergebnis-Caching aktivieren und deaktivieren
    Wenn Sie hier Aktiviert wählen, wird das Ergebnis-Caching für den Business-Service aktiviert. Sie müssen sicherstellen, dass das Ergebnis-Caching global aktiviert ist. Siehe Ergebnis-Caching global aktivieren.
    Wenn Sie das Ergebnis-Caching für einen Business-Service deaktivieren, indem Sie die Option "Aktiviert" deaktivieren und auf Aktualisieren klicken, werden alle für diesen Business-Service gecachten Ergebnisse verworfen.
