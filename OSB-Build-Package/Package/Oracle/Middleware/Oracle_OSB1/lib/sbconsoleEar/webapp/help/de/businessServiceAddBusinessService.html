Business-Services erstellen und konfigurieren

In diesem Abschnitt wird beschrieben, wie Business-Services erstellt und konfiguriert werden.

Für JCA-Services können Sie Business-Services aus JCA-Bindings (Informationen hierzu finden Sie unter Business-Services aus einer JCA-Binding-Ressource generieren) oder aus Oracle Enterprise Repository (Informationen hierzu finden Sie unter Business-Services aus Oracle Enterprise Repository generieren generieren.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer, und wählen Sie das Projekt oder den Ordner, zu dem Sie den Business-Service hinzufügen möchten.
  3. Wählen Sie auf der Seite "Projekt-/Ordneransicht" in der Dropdown-Liste Ressource erstellen die Option Business-Service.
  4. Geben Sie unter Business-Service erstellen/bearbeiten - auf der Seite "Allgemeine Konfiguration" einen Namen für den Service ein, und wählen Sie den zu erstellenden Service-Typ.

    Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
  5. Klicken Sie auf Weiter. Die folgenden Seiten sind je nach den auf der ersten Seite angegebenen Optionen unterschiedlich. Geben Sie die entsprechenden Informationen auf den nachfolgenden Seiten ein, bis Sie zur Zusammenfassungsseite gelangen. Um den Service in der aktuellen Session zu speichern, klicken Sie anschließend auf Speichern.

    6. Ausführliche Anweisungen zum Ausfüllen der einzelnen Seiten finden Sie unter:

    • Allgemeine Konfiguration
    • Seite "Nachrichtentyp-Konfiguration"
    • Seite "Transportkonfiguration"
    • Seite "Protokoll-spezifische Transportkonfiguration"
    • Seite "SOAP-Binding-Konfiguration"
    • Seite "Nachrichtenübermittlungskonfiguration"
    • Seite "Zusammenfassung"
    • Seite "Business-Service Policys"
    • Seite "Business-Service-Sicherheit"

Wenn ein Business-Service aus einer WSDL mit WS-Policy-Anhängen erstellt wird, werden die Policys auf der Seite "Protokollspezifische Transportkonfiguration" angezeigt. Wenn in einer der WS-Policys des Service eine Authentifizierung angegeben ist, müssen Sie einen Service-Account wählen. Ein Proxy-Service, der an diesen Business-Service weiterleitet, verwendet diesen Service-Account für die Authentifizierung beim Business-Service.

Business-Services aus einer JCA-Binding-Ressource generieren

Sie können einen JCA-Business-Service aus einer Outbound-JCA-Binding-Ressource in Oracle Service Bus generieren. Weitere Informationen zu JCA-Binding-Ressourcen finden Sie in Kapitel 12: JCA-Bindings.

So generieren Sie einen JCA-Business-Service aus einem JCA-Binding:

  1. Klicken Sie in der Oracle Service Administration-Konsole im Change Center auf Erstellen oder Bearbeiten, sofern Sie sich nicht bereits im Erstellungs- oder Bearbeitungsmodus befinden.
  2. Klicken Sie im Ressourcenbrowser auf JCA-Bindings.
  3. Suchen Sie das JCA-Binding, aus dem Sie einen Service generieren möchten, und klicken Sie auf das Symbol Aktion.
  4. Bestätigen Sie im angezeigten Fenster den Namen der WSDL und des zu generierenden Service, wählen Sie den Speicherort für diese neuen Ressourcen aus, und klicken Sie auf Generieren.

    5. In Oracle Service Bus werden der Service und die entsprechende WSDL generiert.

  5. Ändern Sie gegebenenfalls andere Konfigurationsdetails zum generierten Service, beispielsweise den Endpunkt-URI.

Business-Services aus Oracle Enterprise Repository generieren

Sie können einen Business-Service mit der Oracle Service Bus-Entwicklungsumgebung aus Oracle Enterprise Repository generieren. Weitere Informationen finden Sie unter "Generating a Business-Service from Oracle Enterprise Repository" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1797.
