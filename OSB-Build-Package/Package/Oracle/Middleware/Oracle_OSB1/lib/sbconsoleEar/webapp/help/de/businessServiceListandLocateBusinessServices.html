Business-Services suchen

  1. Führen Sie eine der folgenden Aktionen aus:
    • Wählen Sie Project Explorer, um die Seite Projektansicht oder die Seite Projekt-/Ordneransicht anzuzeigen. Navigieren Sie dann durch die Projekte und Ordner, um nach dem Business-Service zu suchen.
    • Wählen Sie Ressourcenbrowser > Business-Services. Die Seite Business-Services - Überblick zeigt die Informationen aus Tabelle 18-21. Eine detaillierte Beschreibung der Eigenschaften finden Sie unter Business-Service-Konfigurationen bearbeiten.
  2. Um die Anzahl der Elemente in der Liste zu beschränken, können Sie nach Name, Pfad oder beidem filtern. Geben Sie in den Feldern Name und Pfad unter Suche den Namen und/oder den Pfad des oder der Ziele ein, und klicken Sie dann auf die Schaltfläche Suche.

    3. Der Pfad ist der Projektname und der Name des Ordners, in dem der Business-Service abgelegt ist.

    Die Platzhalter * und ? sind zulässig. Die Suche beachtet die Groß- /Kleinschreibung.

    Klicken Sie auf Alle anzeigen, um alle Business-Services in der Domain anzuzeigen. Damit löschen Sie die Suchparameter der vorhergehenden Suche.

    Tabelle 18-21 Seite "Business-Services - Überblick"
    Eigenschaft
    Beschreibung
    Name
    Ein eindeutiger Name des Business-Service. Klicken Sie auf den Namen, um die Seite Business-Service anzeigen - Konfigurationsdetails anzuzeigen.
    Siehe Business-Service-Konfigurationen bearbeiten .
    Pfad
    Der Pfad ist der Projektname und der Name des Ordners, in dem der Business-Service abgelegt wurde. Beispiel: UDDI/BusServices/OSB_services.
    Klicken Sie auf den Pfad eines Business-Service, um die Seite Projekt-/Ordneransicht mit dem Business-Service anzuzeigen.
    Aktionen
    Führen Sie einen der folgenden Schritte aus:
    • Klicken Sie auf das Symbol Testkonsole starten, um die Testkonsole aufzurufen, mit deren Hilfe Sie den Entwurf Ihrer Services und Transformationen validieren und testen. Für Business-Services können Sie die Testkonsole nur zur Laufzeit verwenden, also wenn die Session aktiviert ist. Für Transformationen können Sie die Testkonsole innerhalb oder außerhalb einer Session verwenden. Siehe Services testen und Transformationen testen.
    • Das Symbol WSDL exportieren wird für WSDL-basierte Business-Services angezeigt. Klicken Sie auf dieses Symbol, um eine WSDL zu exportieren, die Sie anschließend mit einem externen Tool anzeigen oder ändern können.

      • Beachten Sie, dass diese Funktion von der Ressourcenexportfunktion im Systemverwaltungsmodul abweicht. Jene verwenden Sie zum Bereitstellen und Verschieben von Ressourcen zwischen zwei Domains. Siehe WSDL exportieren.

    • Das Symbol WSDL generieren wird für Transporttyp-Business-Services angezeigt, wie EJB und JEJB (nicht aber für Split Join Flow Services). Klicken Sie auf dieses Symbol, um eine WSDL zu generieren, die Sie anschließend anzeigen oder ändern können. Siehe WSDL generieren.
    Optionen
    • Klicken Sie auf das Symbol Löschen, um den Service zu löschen. Das Symbol Warnung wegen Löschvorgang wird angezeigt, wenn andere Ressourcen auf diese Ressource referenzieren. Sie können die Ressource mit einer Warnungsbestätigung löschen. Dies kann zu Konflikten wegen ungelöster Referenzen auf die gelöschte Ressource führen. Weitere Informationen finden Sie unter Business-Services löschen.
