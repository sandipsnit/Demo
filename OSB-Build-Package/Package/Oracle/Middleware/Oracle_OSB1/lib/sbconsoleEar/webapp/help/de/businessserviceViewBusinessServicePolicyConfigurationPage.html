Seite "Business-Service Policys"

Die Seite Business-Service anzeigen - Policy-Konfiguration ist eine von mehreren Seiten für das Bearbeiten von Business-Services. Weitere Informationen hierzu finden Sie unter Business-Services erstellen und konfigurieren und unter Seite Business-Service anzeigen - Konfigurationsdetails.

Auf dieser Seite konfigurieren Sie Policy-Einstellungen für einen Business-Service vom Servicetyp "WSDL-basierter XML-Webservice", "Messaging Service", "Beliebiger SOAP-Service" oder "Beliebiger XML-Service". In Tabelle 18-23 wird die Verwendung der Seite beschrieben. Um OWSM-Policys zusammen mit den Proxyservices "WSDL-basierter XML-Webservice", "Messaging Service" oder "Beliebiger XML-Service" verwenden zu können, muss das Protokoll auf HTTP gesetzt sein.

Beachten Sie, dass bei WSDL-basierten Services an den Service gebundene WLS 9.2-Policys in der effektiven WSDL angegeben (eingebettet) werden. Abstrakte Policys werden vorverarbeitet, bevor sie eingebettet werden. OWSM-Policys werden nach Referenz gebunden und nicht in die effektive WSDL eingebettet.

Service Policy-Konfiguration
Das Feld "Service Policy-Konfiguration" enthält folgende Optionen:

OWSM-Policys

  • Aus OWSM-Policy Store: Policys werden im zentralen von Oracle Enterprise Manager verwalteten Policy-Speicher von Oracle Web Services Manager definiert. Die Policys unterstützen die Token-Profile WS-Security 1.0/1.1, SAML 1.1/2.0, Kerberos und andere Branchenstandards.

    Mit dieser Option fügen Sie Policys aus einer vordefinierten Liste hinzu. Klicken Sie unter "Policys auf Serviceebene" auf "Hinzufügen", um die Liste anzuzeigen. Gegebenenfalls können Sie die Filterfunktion verwenden, um nach den gewünschten Policys zu suchen. Für Business-Services werden nur clientseitige Policys angezeigt.

    Falls eine WSDL, mit deren Hilfe Sie einen Business-Service erstellt haben, eingebettete Standard-WS-Security-Policys enthält, gibt Oracle Service Bus einen Fehler zurück und zeigt einen Konflikt an. Um diesen Konflikt aufzuheben, ersetzen Sie die eingebetteten WSDL-Policys durch kompatible Oracle Web Services Manager Policys, indem Sie Aus OWSM-Policy Store wählen und auf Kompatible hinzufügen klicken. Oracle Service Bus versucht, die geeignetste Policy im Oracle Web Services Manager Policy Store zu finden, und zwar basierend auf der Standard-Policy, die in WSDL eingebettet ist. Der Algorithmus kann keine, eine oder mehrere passende Policys zurückgeben.

    Wenn Oracle Service Bus nach dem Klicken auf Kompatible hinzufügen keine kompatiblen Policys findet, klicken Sie auf Hinzufügen, und wählen Sie die passenden Oracle Web Services Manager-Policys. Wenn Oracle Service Bus mehr als eine passende Policy zurückgibt, löschen Sie diejenigen Policys, die Sie nicht verwenden wollen.

    Hinweis: Diese Option ist nur aktiviert, wenn Sie Ihre Domain so konfiguriert haben, dass sie Oracle Web Services Manager verwendet. Weitere Informationen finden Sie unter "Securing Oracle Service Bus with Oracle Web Services Manager" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

WLS 9-Policys

  • Von WSDL: Wählen Sie diese Option, wenn die Service-Policy der WSDL zugeordnet ist, auf der der Service basiert. Diese Policys unterstützen WS-Security 1.0, SAML 1.1 und andere Branchenstandards.

    Mit dieser Option können Sie (schreibgeschützte) Anforderungs- und Antwort-Policys aus der WSDL anzeigen.

    Hinweis: Wenn bei der Business-Service-Konfiguration ein Fehler auftritt, weil die WS-Security-Policys nicht von Oracle Service Bus unterstützt werden, verwenden Sie die Option "Aus OWSM-Policy Store", um die eingebetteten WSDL-Policys durch kompatible OWSM-Policys zu ersetzen.

  • Von vordefinierter Policy oder WS-Policy-Ressource - Wählen Sie diese Option, um Policys auf Serviceebene, Policys auf Vorgangsebene (in diesem Fall gilt die Policy sowohl für Anforderungs- als auch für Antwortnachrichten), Anforderungs-Policys und Antwort-Policys von der Oracle Service Bus-Administrationskonsole hinzuzufügen.

    Policys werden entweder in Oracle WebLogic Server vordefiniert oder in Oracle Service Bus mit einer WS-Policy-Ressource vom Benutzer definiert. Diese Policys unterstützen WS-Security 1.0, SAML 1.1 und andere Branchenstandards.

    Mit dieser Option können Sie Policys auf der Ebene von Service, Vorgang, Anforderung und Antwort hinzufügen. Klicken Sie im entsprechenden Bereich auf "Hinzufügen", um Policys aus einer Liste auszuwählen.

Hinweis: Die Policy Binding-Modelle schließen sich gegenseitig aus. Sie dürfen nur einen Policy-Typ in einem Service verwenden. Wenn Sie Policys direkt an den Service binden, werden sämtliche WSDL-basierten Policys ignoriert.

Nach dem Hinzufügen von Policys können Sie auf der Seite "Sicherheit" Overrides bereitstellen.
