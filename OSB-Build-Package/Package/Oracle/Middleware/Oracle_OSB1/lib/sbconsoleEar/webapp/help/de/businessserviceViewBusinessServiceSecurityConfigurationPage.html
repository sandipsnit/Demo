Seite "Business-Service-Sicherheit"

Die Seite Business-Service anzeigen - Sicherheitskonfiguration gehört zu einer Reihe von Seiten zum Bearbeiten von Business-Services. Diese Aktion ist auf den Seiten Business-Services erstellen und konfigurieren und Business-Service anzeigen - Konfigurationsdetails beschrieben.

Welche Felder auf dieser Seite verfügbar sind, hängt von der Policy-Verwendung im Business-Service ab. Beispiel: Wenn der Service OWSM-Policys (empfohlen) verwendet wird, werden auf der Sicherheitsseite Policy Overrides angezeigt. Weitere Informationen finden Sie unter "Securing Oracle Service Bus with Oracle Web Services Manager" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

An den Service gebundene WLS 0.2-Policys werden eingebettet in der effektiven WSDL angegeben. Abstrakte Policys werden vor dem Einbetten vorverarbeitet. OWSM-Policys werden durch Referenzen gebunden, und nicht in die effektive WSDL eingebettet.

Sie verwenden diese Seite für das Konfigurieren der Sicherheitseinstellungen für einen Business-Service, der eine WSDL-basierte Policy besitzt oder der benutzerdefinierte Policy Bindings verwendet. Tabelle 18-24 beschreibt die Verwendung der Seite.

Tabelle 18-24 Business-Service anzeigen - Sicherheitsseite

Bearbeiten
Service-Account
Klicken Sie auf Durchsuchen, und wählen Sie aus der angezeigten Liste einen Service-Account. Wenn Sie keinen Service-Account angeben, wird ein anonymes Subject verwendet.
Hinweis: Diese Option ist nicht für Services verfügbar, die Oracle Web Service Manager Policys verwenden, da Benutzer-ID-Daten von Override-Schlüsseln bereitgestellt werden.

Geben Sie für OWSM-Policys beliebige, zulässige Overrides ein.
Weitere Informationen finden Sie in Tabelle 18-23 und unter "Securing Oracle Service Bus with Oracle Web Services Manager" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.
