Session aktivieren

  1. Klicken Sie auf Aktivieren im Change Center. Die Seite Session aktivieren zeigt den Sessionnamen, Benutzernamen und ein Beschreibungsfeld an.

    2. Erstellen einer Session und Verwerfen einer Session erfolgt, gleich was für Aktivitäten im System laufen. Wenn jedoch gerade eine andere Session aktiviert wird, zeigt die Seite Session aktivieren einen Fehler an, der den Benutzer mit den ausstehenden WebLogic Server-Änderungen anzeigt. Weitere Informationen zum Lösen von Konflikten zwischen in Ihrer Session gemachten Änderungen und anderen aktivierten Sessions, finden Sie unter Konflikte anzeigen und auflösen.

  2. Bei Validierungsfehlern wird eine Fehlermeldung angezeigt, die Ihnen mitteilt, dass Ihre Session Validierungsfehler beinhaltet. Bevor Sie fortfahren, müssen Sie Konfigurationskonflikte anzeigen und lösen. Informationen zum Lösen von Konflikten finden Sie unter Konflikte anzeigen und auflösen.

    3. Sie werden die Session nicht aktivieren können, bis Sie alle Konflikte angezeigt haben. Wenn während der Anzeige vorhandener Konflikte neue entstehen, bevor Sie aktivieren, wird eine Meldung angezeigt, die Sie über die neuen Konflikte informiert.

  3. Geben Sie im Feld Beschreibung eine Beschreibung ein. Diese Beschreibung wird in der Spalte Beschreibung angezeigt, wenn Sie auf Änderungen anzeigen unter Change Center klicken, um Konfigurationsänderungen aufgrund von Sessionaktivierungen anzuzeigen.
  4. Klicken Sie zur Aktivierung der Session auf Absenden. Wenn in der Zwischenzeit keine neuen Konflikte entstanden sind, endet die Session, und die Konfiguration wird zur Laufzeit bereitgestellt.
Hinweis: Wenn Sie versuchen, eine Session mit einem JMS-Endpunkt-URI auf einem anderen Server (ein anderer Einzelserver als der, auf dem Sie arbeiten, oder ein Managed Server in einem Cluster) zu aktivieren, stellen Sie sicher, dass der Zielserver verfügbar ist.
Hinweis: Oracle Service Bus lässt die Registrierung von Proxy-Services mit JMS-Transport nicht zu, wenn die JMS-Endpunkt-URL ein Ziel angibt, das nicht erreichbar ist. Anders ausgedrückt, für JMS-Services prüft Oracle Service Bus, ob die angegebene Connection Factory vorhanden ist. Ist sie nicht vorhanden, tritt ein Fehler bei der Sessionaktivierung auf.
