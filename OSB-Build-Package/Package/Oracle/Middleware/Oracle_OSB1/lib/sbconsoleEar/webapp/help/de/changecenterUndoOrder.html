Tasks rückgängig machen – Reihenfolge

Sie können Tasks in einer beliebigen Reihenfolge rückgängig machen (vorausgesetzt, dass die einzelnen Undo-Aktionen gültige Daten ergeben). Beim Undo-Vorgang wird der Wert einer Ressource auf den Wert gesetzt, den sie vor der Änderung an dieser Ressource besaß.

Falls es sich bei der rückgängig gemachten Task um eine Task zum Erstellen eines Objekts handelte, liegt kein vorheriger Status vor, auf den ein Objekt zurückgesetzt werden kann. Das heißt, dass kein Objekt vorhanden war, bevor diese Task ausgeführt wurde. Beim Undo-Vorgang wird das neue Objekt aus der Session gelöscht. In diesem Fall treten Fehler für die Objekte auf, die auf das gelöschte Objekt verweisen. Diese Fehler werden im Change Center auf der Seite Konflikte anzeigen angezeigt.

Wenn Sie nicht in einer Session arbeiten, können Sie die zuvor aktivierten Sessions auf der Seite Konfigurationsänderungen anzeigen anzeigen. Sie können diese Sessions rückgängig machen. Sie dürfen keine zuvor aktivierte Session rückgängig machen, wenn diese Undo-Aktion zu einem Fehler in der Laufzeitkonfiguration führen würde. Beispiel: Wenn Sie versuchen, eine Sessionaktivierung rückgängig zu machen, bei der ein Objekt entfernt wird, auf das von einem anderen Objekt verwiesen wird, ist diese Undo-Aktion nicht zulässig.

Eine Undo-Aktion kann rückgängig gemacht werden. Unter [Task] rückgängig machen klicken Sie in der Spalte Optionen auf das Symbol Rückgängig. In Oracle Service Bus werden Undo-Aktionen uneingeschränkt unterstützt. Das heißt, Sie können die Undo-Aktion rückgängig machen, mit der eine vorherige Undo-Aktion rückgängig gemacht wurde, und so weiter.
