Konflikte anzeigen und auflösen

Auf dieser Seite können Sie Diagnosemeldungen zu Fehlern in der Konfiguration anzeigen und Konflikte zwischen Änderungen in Ihrer Session und anderen aktivierten Sessions anzeigen und auflösen. Wenn Sie auf den Link "Konflikte anzeigen" klicken, wird auch die Anzahl der aktuellen Konflikte in der Session angezeigt.

Konflikte anzeigen
  1. Klicken Sie unter Change Center auf Konflikte anzeigen. Auf der Seite Konflikte anzeigen werden je nach Art der Konflikte alle oder einige der folgenden Informationen angezeigt:
    • Fehler (ein Fehler-Symbol weist auf nicht festschreibbare, kritische Fehler hin) informieren Sie über kritische Konflikte innerhalb der Konfiguration. Sie können Änderungen erst dann festschreiben, wenn die Konflikte aufgelöst sind. Weitere Informationen finden Sie in Tabelle 2-6 und unter Konflikte auflösen.
    • Gleichzeitige Aktualisierungen (ein Warnung-Symbol weist auf festschreibbare, nicht kritische Konflikte hin) warnen Sie bei Änderungen, die mit anderen aktivierten Sessions nicht kompatibel sind. Weitere Informationen finden Sie in Tabelle 2-7.
    • Informationsmeldungen (ein Warnung-Symbol weist auf festschreibbare, nicht kritische Konflikte hin) informieren Sie über nicht kritische Konflikte innerhalb der Konfiguration.

  2. Sie können Details zu einem bestimmten Konflikt anzeigen, indem Sie in der Spalte Name auf den Link des entsprechenden Konflikts klicken.

Konflikte auflösen

Wenn Sie einen Konflikt auflösen möchten, machen Sie sich mithilfe der Informationen in der Spalte Meldungen (siehe Tabelle 2-6) vertraut, und bearbeiten Sie dann das Objekt, das den Konflikt verursacht.

Ein Konflikt aufgrund einer gleichzeitigen Aktualisierung wird dadurch verursacht, dass eine Ressource in der aktuellen Session geändert wird und bereits in einer anderen Session geändert und aktiviert wurde (siehe Tabelle 2-7). In diesem Fall haben Sie zwei Möglichkeiten, den Konflikt aufzulösen:

  • Klicken Sie unter Change Center auf Aktivieren. Dadurch werden Ihre Änderungen zur Laufzeit gespeichert, d.h. Sie überschreiben Änderungen, die bei der Aktivierung in einer früheren Session an Core-Daten vorgenommen wurden. Änderungen, die von einem anderen Benutzer in einer anderen Session vorgenommen und gespeichert wurden, werden somit von Ihren Änderungen in dieser Session überschrieben.
  • Klicken Sie in der Tabelle in der Spalte Synchronisieren auf das Symbol Synchronisieren (siehe Tabelle 2-7). Dadurch wird in dieser Session der Status des Objekts wiederhergestellt, mit dem es zur Laufzeit gespeichert wurde.
