Konfigurationsänderungen anzeigen

Auf dieser Seite werden je nachdem, ob Sie sich in einer Session (das heißt, Sie haben im Change Center auf Bearbeiten geklickt) oder außerhalb einer Session befinden, unterschiedliche Informationen angezeigt.

  • Wenn Sie sich in einer Session befinden, wird auf der Seite Konfigurationsänderungen anzeigen eine Liste der Konfigurationsänderungen angezeigt, die Sie während der aktuellen Session vorgenommen haben. Weitere Informationen finden Sie unter Konfigurationsänderungen in einer Session anzeigen.
  • Wenn Sie sich außerhalb einer Session befinden, wird auf der Seite Konfigurationsänderungen anzeigen eine Liste der Konfigurationsänderungen von vorherigen Sessionaktivierungen angezeigt. Weitere Informationen finden Sie unter Konfigurationsänderungen von Sessionaktivierungen anzeigen.
Konfigurationsänderungen in einer Session anzeigen
  1. Wenn dies noch nicht geschehen ist, klicken Sie auf Erstellen, um eine neue Session zum Vornehmen von Änderungen an der aktuellen Konfiguration zu erstellen. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Nehmen Sie mindestens eine Änderung an der Konfiguration vor.
  3. Klicken Sie unter Change Center auf Änderungen anzeigen. Auf der Seite Konfigurationsänderungen anzeigen werden für jede Änderung, die Sie während der aktuellen Session an der Konfiguration vorgenommen haben, die in Tabelle 2-2 dargestellten Informationen angezeigt.

    4. Tabelle 2-2 Konfigurationsänderungen innerhalb einer Session
    Eigenschaft
    Beschreibung
    Aufgabe
    Eine Beschreibung der implementierten Task. Die Task ist ein Link zur Seite Task-Details. Weitere Informationen finden Sie unter Task-Details anzeigen.
    Ausführungszeit
    Datum und Uhrzeit, zu der die Task ausgeführt wurde.
    Benutzer
    Der Name des Benutzers, der die Änderung implementiert hat.
    Task-Status
    Der Status der Task:
    Abgeschlossen - Die Task wurde abgeschlossen.
    Rückgängig gemacht - Die Task wurde rückgängig gemacht.
    Undo abgeschlossen - Der Undo-Vorgang wurde abgeschlossen.
    Rückgängig
    Der Name des Benutzers, der die Task rückgängig machte. Dieses Feld enthält den Wert Keine, wenn die Task nicht rückgängig gemacht wurde.
    Optionen
    Um die Ausführung der Task zurückzusetzen, klicken Sie auf das Symbol Rückgängig.
    Wenn Sie in einer Session arbeiten, können Sie Tasks in beliebiger Reihenfolge rückgängig machen.
    Nach dem Rückgängigmachen einer Task wird das Objekt der Task auf den Status zurückgesetzt, den es vor der Ausführung der fraglichen Task aufwies. Dabei müssen Sie allerdings beachten, dass alle Tasks, die für dasselbe Objekt nach der rückgängig gemachten Task ausgeführt wurden, ebenfalls rückgängig gemacht werden. Weitere Informationen finden Sie unter Tasks rückgängig machen.

Konfigurationsänderungen von Sessionaktivierungen anzeigen
  1. Stellen Sie sicher, dass Sie sich nicht in einer Session befinden. Weitere Informationen finden Sie unter Sessions verlassen.
  2. Klicken Sie unter Change Center auf Änderungen anzeigen. Auf der Seite Konfigurationsänderungen anzeigen werden für jede zuvor aktivierte Session die in Tabelle 2-3 dargestellten Informationen angezeigt.

    3. Tabelle 2-3 Aktivierte Konfigurationsänderungen
    Eigenschaft
    Beschreibung
    Aufgabe
    Eine Beschreibung der aktivierten Session. Die Task ist ein Link zur Seite Task-Details, auf der die in der Session ausgeführten Vorgänge angezeigt werden. Weitere Informationen finden Sie unter Task-Details anzeigen.
    Beschreibung
    Eine zusätzliche Beschreibung der aktivierten Session, sofern eine Beschreibung eingegeben wurde, als die Session auf der Seite Session aktivieren aktiviert wurde.
    Ausführungszeit
    Datum und Uhrzeit, zu der die Session aktiviert wurde.
    Benutzer
    Der Name des Benutzers, der die Session aktivierte.
    Task-Status
    Der Status der Session:
    Aktiviert - Die Session wurde aktiviert.
    Teilweise aktiviert - Wird während der Sessionaktivierung angezeigt, wenn auf einen oder mehrere Server in einem Cluster kein Zugriff möglich ist und damit das Aktivieren der Session auf diesen Servern verhindert wird. Wenn der Zugriff auf die Server wieder möglich ist, werden alle nicht aktivierten Änderungen auf diesen Servern aktiviert.
    Rückgängig gemacht - Die zuvor aktivierte Session wurde rückgängig gemacht, und alle in der Session ausgeführten Vorgänge wurden verworfen.
    Undo aktiviert - Der Undo-Vorgang wurde aktiviert.
    In Bearbeitung - Wird angezeigt, wenn eine Sessionaktivierung ausgeführt wird, da Sessionaktivierungen lange dauern können.
    Nicht erfolgreich - Wird angezeigt, wenn eine Sessionaktivierung nicht erfolgreich war. In Oracle Service Bus werden Fehler bei Sessionaktivierungen protokolliert, Fehler aufgrund einzelner Updates innerhalb einer Session allerdings nicht.
    Rückgängig
    Der Name des Benutzers, der diese Task rückgängig machte. Dieses Feld enthält den Wert Keine, wenn die Sessionaktivierung nicht rückgängig gemacht wurde.
    Optionen
    Um die Sessionaktivierung und die in dieser Session ausgeführten Vorgänge zurückzusetzen, klicken Sie auf das Symbol Rückgängig. Sie können Sessionaktivierungen in beliebiger Reihenfolge rückgängig machen.
    Klicken Sie auf das Symbol In Session rückgängig machen, um eine neue Session zu erstellen, die vorab mit den Tasks aufgefüllt wurde, die zum Rückgängigmachen der in der Session aktivierten Änderungen erforderlich sind. Weitere Informationen finden Sie unter Vorgänge in einer Session rückgängig machen.
