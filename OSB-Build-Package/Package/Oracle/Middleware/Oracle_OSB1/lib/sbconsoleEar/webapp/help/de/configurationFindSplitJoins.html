Split Joins suchen

  1. Wählen Sie Vorgänge > Smart Search.
  2. Wählen Sie aus der Dropdown-Liste Typ die Option Split Joins, klicken Sie dann auf Suchen. Auf der Seite Übersicht der Split Joins finden Sie die Informationen aus Tabelle 26-4.
  3. So können Sie die Anzahl der Elemente in der Liste einschränken oder bestimmte Business-Services suchen:
    • Klicken Sie auf das Symbol Öffnen, um zusätzliche Suchfilter anzuzeigen. Wenn Sie mehrere Suchkriterien angeben, werden nur Ergebnisse angezeigt, die allen Kriterien entsprechen.
    • Nach Split Join-Name und -Pfad filtern. Geben Sie in den Feldern Name und Pfad den Namen und den Pfad des gesuchten Ziels ein, dann klicken Sie auf Suchen. Der Pfad ist zugleich auch der Projektname und der Name des Ordners, in dem der Split Join abgelegt ist. Sie können die Platzhalter * und ? für eine allgemeinere Suche einsetzen.
    • Nach Split Joins mit Logging filtern. Wählen Sie aus der Dropdown-Liste Logs den Logging-Status des Split Join aus: "Aktiviert" oder "Deaktiviert". Klicken Sie dann auf Suchen. Mit der Wahl "Alle" ignorieren Sie das Suchkriterium "Logging".
    • Nach Ausführungs-Tracing filtern. Wählen Sie aus der Dropdown-Liste Ausführungs-Tracing den Ausführungs-Tracing-Status des Split Join aus: "Aktiviert" oder "Deaktiviert". Klicken Sie dann auf Suchen. Mit der Wahl "Alle" ignorieren Sie das Suchkriterium "Service Logging".

      • Tabelle 26-4 Split Join - Zusammenfassende Informationen 
      Eigenschaft
      Beschreibung
      Name
      Name, der dem Split Join zugewiesen ist. Der Name ist ein Link zur Seite Vorgangseinstellungen. Siehe Split Joins bearbeiten.
      Pfad
      Projekt, das mit dem Split Join verknüpft ist. Wenn der Split Join in einem Projektordner abgelegt ist, wird dieser Ordner ebenfalls aufgelistet. Der Pfad wird in folgendem Format angezeigt:
      project-name/root-folder/ . . ./parent-folder
      Der Pfad ist ein Link zum entsprechenden Pfad im Project Explorer.
      Typ
      Übergeordneter Service-Typ: Split Join.
      Logs
      Der Logging-Status: Aktiviert oder Deaktiviert und die aktivierte sowie darüberliegende Ebenen: Debug (D), Info (I), Warnung (W) oder Fehler (E).
      Ausführungs-Tracing
      Laufzeit-Tracing-Status des Split Join: "Aktiviert" oder "Deaktiviert".

Auf dieser Seite können Sie auch folgende Aufgaben ausführen:

  • Wenn Sie den Suchfilter entfernen und alle Split Joins anzeigen möchten, klicken Sie auf Alle anzeigen.
  • Klicken Sie auf Zurücksetzen, um Ihre Änderungen zu verwerfen und die Seite mit den aktuell gespeicherten Einstellungen und denselben Suchkriterien zu aktualisieren.
  • Um innerhalb einer Session Einstellungen auf Service-Ebene zu aktivieren oder zu deaktivieren, aktivieren oder deaktivieren Sie das Kontrollkästchen daneben und klicken auf Aktualisieren.
    • Hinweis: Die Auswirkungen zur Laufzeit der Split Join Level-Einstellungen hängen von den entsprechenden globalen Einstellungen ab. Sie müssen sowohl die globalen als auch die Service Level-Einstellungen für einen Service aktivieren, damit diese zur Laufzeit vollständig aktiviert sind.
  • Passen Sie die Anzeige der Informationen zu den Vorgangseinstellungen an, indem Sie auf das Symbol Tabellen-Customizer klicken. Weitere Informationen finden Sie unter Tabellenansichten anpassen.
