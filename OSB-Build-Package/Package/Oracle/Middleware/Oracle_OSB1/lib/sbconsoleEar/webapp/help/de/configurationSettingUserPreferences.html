Benutzervoreinstellungen festlegen

Auf dieser Seite passen Sie die Anzeigeoptionen und -einstellungen der Oracle Service Bus Administration-Konsole an und speichern sie. Sie können Ihre Benutzervoreinstellungen aktualisieren (ändern), ohne eine Änderungssession aktivieren zu müssen.

Tabelle 26-7 beschreibt die Optionen und Steuerelemente auf der Seite.

Tabelle 26-7 Benutzereinstellungen - Optionen und Steuerelemente 
Ziel
Vorgehensweise
Homepage ändern, das heißt die Seite anpassen, die geöffnet wird, wenn Sie die Oracle Service Bus Administration-Konsole starten.
Wählen Sie aus der ersten Dropdown-Liste eines der Hauptmenüs: Vorgänge, Ressourcenbrowser, Project Explorer, Sicherheitskonfiguration oder Systemverwaltung.
Wählen Sie aus der zweiten Dropdown-Liste eine der Untermenü-Auswahlmöglichkeiten. (Der Inhalt der Untermenüs hängt von Ihre Auswahl im Hauptmenü ab.)
Suchfilter anzeigen
Wählen Sie Ja oder Nein.
Standardmäßig werden Suchfilter nicht angezeigt (geschlossen). Wenn Sie Ja wählen, werden die Suchfilter immer angezeigt (geöffnet).
Stage-Anmerkungen anzeigen
Wählen Sie Ja oder Nein.
Standardmäßig werden Pipeline Stage-Anmerkungen nicht angezeigt (geschlossen). Wenn Sie Ja wählen, werden Stage-Anmerkungen immer angezeigt (geöffnet).
RessourcenGanzzahl anzeigen
Wählen Sie Ja oder Nein.
Standardmäßig werden Ressourcenmetadaten-Abschnitte angezeigt (geöffnet). Wenn Sie Nein wählen, werden Ressourcenmetadaten nicht angezeigt (geschlossen).
Im Dashboard angeben, wie häufig die Oracle Service Bus Administration-Konsole die Datenanzeige aktualisieren soll.
Wählen Sie aus der Dropdown-Liste Dashboard-Aktualisierungsrate die Option Keine Aktualisierung, 1, 2, 3, 5, 10, 20, 30 oder 60 Minuten als Aktualisierungsrate für das Dashboard.
Die Standardrate ist Keine Aktualisierung, aber Sie können eine andere vordefinierte Zeit einstellen. Beispiel: Wenn Sie 5 Minuten wählen, wird das Dashboard alle 5 Minuten aktualisiert.
Achtung: Wenn Sie einen anderen Wert als den Standardwert Keine Aktualisierung als Aktualisierungsrate für das Dashboard wählen, aktualisiert der Browser das Dashboard in regelmäßigen Abständen. Dadurch wird ein Timeout der Session verhindert, selbst wenn Sie nicht an der Oracle Service Bus Administration-Konsole arbeiten.
Achtung: Oracle empfiehlt, die Dashboard-Aktualisierungsrate mit Vorsicht anzuwenden. Lassen Sie eine Session in der Oracle Service Bus Administration-Konsole nie unbeaufsichtigt. Während die Session aktiv ist, können Dritte über Ihren Rechner darauf zugreifen, ohne sich erneut authentifizieren zu müssen.
Im Dashboard das Zeitintervall für die Anzeige historischer Daten für Alerts angeben
Wählen Sie aus der Dropdown-Liste Dauer der Alert-Historie eine der folgenden Optionen 30 Minuten, 1, 2, 3 oder 6 Stunden als Zeitintervall für die Anzeige historischer Alert-Daten aus. Der Standardwert ist 30 Minuten (zeigt Alerts an, die während der letzten 30 Minuten erhalten wurden), aber Sie können auch eine andere vordefinierte Dauer angeben. Beispiel: Wenn Sie 1 Stunde wählen, zeigt das Dashboard diejenigen Alerts an, die im Laufe der letzten Stunde eingegangen sind.
