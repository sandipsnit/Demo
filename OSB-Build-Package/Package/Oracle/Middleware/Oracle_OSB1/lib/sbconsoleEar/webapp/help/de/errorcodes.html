Fehlercodes

In diesem Abschnitt werden die Fehler beschrieben, die beim Ausführen eines Proxy-Service in Oracle Service Bus auftreten können. Die mit diesen Fehlern verknüpften Fehlercodes erscheinen innerhalb des Elements der Kontextvariablen fault. Sie können mit folgender XQuery-Anweisung auf den Wert zugreifen:

$fault/ctx:errorCode/text()

Zu den Fehlern werden fehlerspezifische Details im Element fault angegeben.

Tabelle A-1 enthält eine Liste der Oracle Service Bus-Fehlercodes.

Sicherheitsfehlercodes (386000 bis 386999)
BEA-386000
Allgemeiner Sicherheitsfehler. Mögliche Ursachen sind:
  • Die Authentifizierung über ein benutzerdefiniertes Token, einen benutzerdefinierten Benutzernamen oder ein benutzerdefiniertes Kennwort wurde konfiguriert, aber die Authentifizierung fand bereits über WS-Security statt.
 
BEA-386100
Authentifizierung nicht erfolgreich. Mögliche Ursachen sind:
  • XPath ist so konfiguriert, dass nur der Text oder der Header bearbeitet wird, aber es ist kein solcher Nachrichtenteil vorhanden (Binding Layer gibt null zurück).
  • Die XPath-Engine löst beim Auswerten des XPath-Ausdrucks eine Exception aus.
  • Das Ergebnis von XPath des Benutzernamens oder XPath des Benutzerkennwortes ist ein Knoten, der nicht den Typ TEXT oder ATTR aufweist.
  • XPath gibt mehrere Knoten zurück.
  • Ein Benutzername/Kennwort wurde abgerufen, aber die Authentifizierung war nicht erfolgreich.
  • Ein benutzerdefiniertes Token wurde abgerufen, aber die Identity Assertion war nicht erfolgreich (der CSS-Identity Assertion Service löst eine LoginException oder eine andere Exception aus).
 
BEA-386101
Fehlendes Authentifizierungs-Token. Mögliche Ursachen sind:
  • XPath des Benutzernamens oder XPath des Benutzerkennwortes geben ein leeres Ergebnis oder eine leere Zeichenfolge zurück.
  • Der XPath des benutzerdefinierten Tokens gibt ein leeres Ergebnis, den Attributwert null oder den Textwert Null zurück.
 
BEA-386102
Autorisierung auf Nachrichtenebene verweigert. Mögliche Ursachen sind:
  • Die Zugriffskontroll-Policy auf Nachrichtenebene lehnt den Zugriff auf den Proxy-Service ab.
 
BEA-386103
Der Algorithmus der Proxy-Service-Vorgangsauswahl kann den Vorgangsnamen aus der Anforderung nicht bestimmen oder gibt einen ungültigen Vorgang zurück (einen Vorgang, der nicht in der WSDL oder null ist). Mögliche Ursachen sind:
  • Beim Berechnen des Vorgangs ist ein Fehler aufgetreten.
  • Der Algorithmus der Vorgangsauswahl gibt null zurück.
  • Der Algorithmus der Vorgangsauswahl gibt einen Vorgang zurück, der nicht zu den Vorgängen in der WSDL gehört.

Schemas mit Fehlerdetails

Unten werden Schemadefinitionen für XML-Snippets dargestellt, die zu den in Tabelle A-1 aufgelisteten Fehlercodes gehören. Diese XML-Snippets werden innerhalb der Kontextvariablen "fault" unter dem Element <ctx:details> angezeigt. Diese Schemas stammen nicht alle aus einem einzigen Namespace. Während ein gegebener Fehlercode normalerweise ein bestimmtes XML-Snippet impliziert, werden einige XML-Snippets von mehreren Fehlercodes verwendet. 

<xs:schema targetNamespace="http://www.oracle.com/wli/sb/stages/transform/config"
           xmlns:tns="http://www.oracle.com/wli/sb/stages/transform/config"
	   xmlns:xs="http://www.w3.org/2001/XMLSchema"
           xmlns:ref="http://www.oracle.com/wli/sb/reference"
           xmlns:sdk="http://www.oracle.com/wli/sb/stages/config"
	   elementFormDefault="qualified"
	   attributeFormDefault="unqualified">

SOAP-Fault-Antwort empfangen 

    <xs:element name="ReceivedFaultDetail" type="tns:ReceivedFaultDetail"/>
    <xs:complexType name="ReceivedFaultDetail">
        <xs:sequence>
            <xs:element name="faultcode" type="xs:QName"/>
            <xs:element name="faultstring" type="xs:string"/>
            <xs:element name="detail" minOccurs="0" >
                <xs:complexType mixed="true" >
                    <xs:sequence>
                        <xs:any namespace="##any" minOccurs="0" maxOccurs="unbounded" processContents="lax" />
                    </xs:sequence>
                    <xs:anyAttribute namespace="##any" processContents="lax" />
                </xs:complexType>
            </xs:element>
            <!-- HTTP-specific status/response code -->
            <xs:element name="http-response-code" type="xs:int" minOccurs="0"/>
        </xs:sequence>
    </xs:complexType>

Eine unbekannte Antwort wurde empfangen
