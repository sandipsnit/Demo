Einführung in Oracle Service Bus

Oracle Service Bus ist ein konfigurationsbasierter, Policy-gesteuerter Unternehmens-Service-Bus. Er gewährleistet hoch skalierbare und zuverlässige, Service-orientierte Integration, Service Management und herkömmliches Message Brokering in heterogenen IT-Umgebungen. Er vereint intelligentes Message Brokering mit Routing und Nachrichtentransformierung sowie Service-Überwachung und -Verwaltung in einem einheitlichen Softwareprodukt. Über die Oracle Service Bus Administration-Konsole können Sie die Service- und Policy-Konfigurationen steuern und System- und Vorgangsaufgaben überwachen. Oracle Service Bus stützt sich auf die Laufzeitfunktionen von Oracle WebLogic Server. Weitere Informationen finden Sie unter Oracle Fusion Middleware Concepts and Architecture for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBCA.

Benennungseinschränkungen bei Ressourcen

Bei der Benennung von Verzeichnissen oder Ressourcen in einer Oracle Service Bus-Konfiguration sind folgende Zeichen zulässig:

    • Alle Java-Identifier-Zeichen, einschließlich Java-Schlüsselwörter, wie es in den Abschnitten "Identifiers" und "Keywords" der Java Language Specification unter http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#3.8
      beschrieben ist
    • Leerzeichen, Punkte und Bindestriche in Namen (nicht voran- oder nachgestellt)

Zeichen wie / \ * : " < > ? | sind nicht zulässig.
