JCA-Bindings bearbeiten

Auf der Seite JCA-Binding-Details anzeigen können Sie die Details eines JCA-Bindings anzeigen und ändern.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Suchen Sie das JCA-Binding wie unter JCA-Bindings suchen beschrieben.
  3. Klicken Sie auf den Namen des JCA-Bindings. Auf der Seite JCA-Binding anzeigen werden die in Tabelle 12-2 beschriebenen Informationen angezeigt.

    4. Tabelle 12-2 JCA-Binding-Details 
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der dieses JCA-Binding erstellt oder es in die Konfiguration importiert hat.
    Zuletzt geändert am
    Datum und Uhrzeit des Zeitpunkts, an dem der Benutzer das JCA-Binding erstellt oder in die Konfiguration importiert hat. Klicken Sie auf den Link mit Datum und Uhrzeit, um die Änderungshistorie dieser Ressource anzuzeigen. Weitere Informationen finden Sie unter Seite "Änderungshistorie anzeigen".
    Referenzen
    Gibt einen Link zur WSDL-Ressource an, die das JCA-Binding referenziert. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Referenziert von
    Die Anzahl der Objekte, die dieses JCA-Binding referenzieren. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste mit allen Objekten anzuzeigen. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Symbol "Aktion"
    Wenn sich das Change Center im Bearbeitungsmodus befindet, können Sie mit dem Symbol "Aktion" Oracle Service Bus-WSDL und einen JCA-Service aus dem JCA-Binding generieren.
    JCA-Datei
    Zeigt eine schreibgeschützte Ansicht eines JCA-Bindings an.
    Abstrakte WSDL
    Zeigt verknüpfte WSDL eines JCA-Bindings an. Wenn das JCA-Binding keine WSDL referenziert, klicken Sie auf Referenzen bearbeiten, um verknüpfte WSDL zu referenzieren.
    TopLink-XML
    Zeigt die erforderliche TopLink- oder EclipseLink-Mapping-Datei eines JCA-Bindings an. Wenn die JCA-Binding-Ressource nicht die erforderliche Mapping-Datei referenziert, klicken Sie auf Referenzen bearbeiten, um die erforderliche Mapping-Datei zu wählen. Dieses Feld wird nur angezeigt, wenn die JCA-Datei eine XML-Ressource deklariert.
    XSLT
    Zeigt die erforderliche XSLT-Datei eines JCA-Bindings an. Wenn die JCA-Binding-Ressource nicht die erforderliche XSLT-Ressource referenziert, klicken Sie auf Referenzen bearbeiten, um die erforderliche XSLT-Datei zu wählen. Dieses Feld wird nur angezeigt, wenn die JCA-Datei eine XSLT-Ressource deklariert, beispielsweise für einen Socket-Adapter.

  4. Wenn Sie die Beschreibung oder JCA-Datei ändern möchten, klicken Sie auf Bearbeiten.

    5. Sie können den Ressourcennamen nicht ändern.

  5. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  6. Um die Referenzen zu den erforderlichen Ressourcen eines JCA Bindings zu ändern oder noch nicht aufgelöste Referenzen zu beseitigen, klicken Sie auf Referenzen bearbeiten, und wählen Sie die entsprechenden Ressourcen.
  7. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
