Alert-Regelbedingungen definieren

Auf dieser Seite können Sie Bedingungen für die Alert-Regel konfigurieren. Sie müssen mindestens eine Bedingung angeben. Wenn Sie mehrere Bedingungen angeben, müssen Sie sie mit den And/Or-Operatoren kombinieren.

  1. Wählen Sie im Feld Aggregationsintervall für die Bedingung wählen die Anzahl der Stunden und Minuten für das Aggregationsintervall dieser Bedingung. Das Aggregationsintervall bestimmt die Häufigkeit, mit der die Bedingung getestet wird. Die Bedingung wird jedes Mal getestet, wenn im Überwachungssubsystem genug Daten-Samples für 1 Aggregationsintervall aggregiert wurden.

    2. Beispiel: Wenn Sie das Aggregationsintervall auf 1 Stunde setzen, wird die Bedingung jedes Mal getestet, wenn Daten für eine Stunde verfügbar sind. Die Bedingung wird zum ersten Mal am Ende der ersten Stunde getestet. Danach wird die Bedingung alle 10 Minuten getestet, weil das Sampling-Intervall für ein Aggregationsintervall von 1 Stunde auf 10 Minuten gesetzt ist.

  2. Zunächst definieren Sie einen einfachen Ausdruck. Zwei oder mehr einfache Ausdrücke können zu einem komplexen Ausdruck zusammengefasst werden. Um einen komplexen einfachen Ausdruck zu definieren, gehen Sie wie folgt vor:
    1. Im ersten Dropdown-Feld wählen Sie Anzahl, Minimum, Maximum, Durchschnitt oder Status.
    2. Im nächsten Dropdown-Feld wählen Sie einen Operanden.

      3. Je nachdem, ob Sie im ersten Dropdown-Feld Anzahl, Minimum, Maximum, Durchschnitt oder Status wählen, ist die Liste der Operanden unterschiedlich. Beispiel: Wenn Sie "Minimum", "Maximum" oder "Durchschnitt" wählen, ist der Operand "Antwortzeit" verfügbar. Mit diesem Operanden können Sie die Antwortzeit in Millisekunden (ms) festlegen. Die verfügbaren Operanden hängen auch von der Konfiguration des Service selbst ab. Die Anzahl der Operanden ist je nachdem, ob ein Service Pipelines, Route-Knoten, Vorgänge und so weiter beinhaltet, unterschiedlich.

      Weitere Informationen zum Generieren von Alerts auf der Grundlage des Endpunkt-URI-Status finden Sie unter "Managing Endpoint URIs for Business-Services" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG595.

      Wenn Sie Anzahl wählen, werden in diesem Feld die in Tabelle 25-21 dargestellten Operanden angezeigt.

      Tabelle 25-21 Operand-Details zu "Anzahl"
      Eigenschaft
      Beschreibung
      Anzahl Cachetreffer
      Bei Business-Services, in denen Ergebnis-Caching verwendet wird, erhöht sich die Anzahl der Cachetreffer jeweils um 1, wenn der Cache für eine Antwort an einen Client verwendet wird.
      Fehleranzahl
      Die Fehleranzahl erhöht sich jeweils um 1, wenn die Nachrichtenverarbeitung einen Fehler zurückgibt.
      Fehlerrate (%)
      Die aufgetretene Fehlerrate im Verhältnis zur Gesamtanzahl von Nachrichten, die im angegebenen Aggregationsintervall erfolgreich verarbeitet wurden.
      Anzahl der Nachrichten
      Die Gesamtanzahl der verarbeiteten Nachrichten.
      Erfolgsrate (%)
      Die erfolgreich verarbeiteten Nachrichten im Verhältnis zur Gesamtanzahl von Nachrichten im angegebenen Aggregationsintervall.
      Failover-Anzahl
      Nur für Business-Services: Die Anzahl der Failover.
      <Anforderungs-Pipeline>.
      Anzahl Fehler
      Nur für Proxy-Services: Die Anzahl der fehlerhaften Nachrichten, die von der Anforderungs-Pipeline verarbeitet wurden.
      <Anforderungs-Pipeline>.
      Anzahl Nachrichten
      Nur für Proxy-Services: Die Anzahl der Nachrichten, die von der Anforderungs-Pipeline verarbeitet wurden.
      Anzahl Validierungsfehler
      Nur für Proxy-Services, die eine Validieren-Aktion in der Pipeline haben: Die Anzahl der Validierungsfehler.
      Anzahl WSS-Fehler
      Je nach dem Transport für den Service (beispielsweise mit HTTP) verfügbar: Die Anzahl der verarbeiteten fehlerhaften Web Service Security-(WSS-)Nachrichten. Dieser Zähler ist nur für WSDL-basierte Services verfügbar und wird aktualisiert, wenn ein WSS-Fehler auftritt.

    3. Im nächsten Feld wählen Sie den gewünschten Vergleichsoperator: =, !=, > oder <.
    4. Im nächsten Feld geben Sie den Wert ein, mit dem verglichen werden soll.
    5. Klicken Sie auf Hinzufügen.
  3. So definieren Sie einen komplexen Ausdruck:
    1. Wiederholen Sie die Schritte a - e in Schritt 2, um einen einfachen Ausdruck zu definieren.
    2. Wiederholen Sie die Schritte a - e erneut, um einen weiteren einfachen Ausdruck zu definieren.
    3. Aktivieren Sie das Kontrollkästchen für die beiden Ausdrücke.
    4. Klicken Sie auf And oder auf Or, um die Ausdrücke in einem komplexen Ausdruck zusammenzufassen.

      5. Die Option Teilen ist verfügbar, nachdem Sie mehrere Ausdrücke zusammengefasst haben. Mit dieser Option können Sie komplexe Ausdrücke wieder in einzelne einfache Ausdrücke aufteilen.

Wenn Sie fertig sind

Klicken Sie auf Vorherige, um zur Seite Allgemeine Konfiguration zurückzukehren. Wenn Sie auf Letzte klicken, wird die Seite Neue Alert-Regel - [Service-Name] angezeigt. Fahren Sie mit dem Abschnitt Alert-Regelkonfiguration überprüfen fort.
