Allgemeine Informationen für Alert-Regeln konfigurieren

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Ressourcenbrowser > Proxy-Services oder Business-Services.
  3. Klicken Sie auf den Service-Namen.
  4. Klicken Sie in der Registerkarte SLA-Alert-Regeln auf die Option Neue hinzufügen.
  5. Geben Sie im Feld Regelname einen Namen für die Alert-Regel ein. Dieses Feld ist obligatorisch.
  6. Geben Sie im Feld Alert-Übersicht eine Kurzbeschreibung für die Alert-Regel ein. Dieser Text wird auch als Betreffzeile für eine E-Mail-Nachricht verwendet, sofern diese Alert-Regel mit einer E-Mail-Alert-Benachrichtigung konfiguriert ist, und darf nicht mehr als 80 Zeichen enthalten. Wenn Sie keine Alert-Übersicht angeben, wird der Standardtext "Oracle Service Bus-Alert" verwendet.
  7. Geben Sie im Feld Regelbeschreibung eine Beschreibung für die Alert-Regel ein.
  8. Geben Sie im Feld Alert-Ziel den Namen für die Alert-Zielressource ein. Sie können auch auf Durchsuchen klicken, um die Seite Alert-Ziel wählen anzuzeigen. Wählen Sie ein Alert-Ziel aus der Liste, und klicken Sie auf Weiterleiten.

    9. Standardmäßig wird der Alert immer an die Oracle Service Bus Administration-Konsole geleitet. Wenn Sie andere Alert-Zielressourcen wählen möchten, finden Sie weitere Informationen unter Alert-Ziele hinzufügen und unter Alert-Ziele suchen.

  9. Geben Sie im Feld Startzeit eine Startzeit im Format HH:MM ein. Beispiel: 09:00 Uhr. In den Feldern Startzeit und Endzeit können Sie ein Zeitfenster angeben, in dem die Regel an jedem Tag vor dem Ablaufdatum aktiv ist. Beispiel: Zwischen 9:00 und 17:00 Uhr täglich.
  10. Geben Sie im Feld Endzeit eine Endzeit im Format HH:MM ein. Beispiel: 17:00 Uhr. In den Feldern Startzeit und Endzeit können Sie ein Zeitfenster angeben, in dem die Regel an jedem Tag vor dem Ablaufdatum aktiv ist. Beispiel: Zwischen 9:00 und 17:00 Uhr täglich.

    11. Die Alert-Regel ist ab der angegebenen Startzeit bis zur angegebenen Endzeit täglich aktiv, bis die Regel abläuft.

  11. Geben Sie im Feld Regelablaufdatum ein Ablaufdatum im Format MM/TT/JJJJ ein. Beispiel: 12/31/2010. Die Regel läuft zum angegebenen Datum um 23:59 Uhr ab. Wenn Sie kein Datum angeben, läuft die Regel nie ab.
  12. Geben Sie im Feld Regel aktiviert den Standardwert Ja an, um sicherzustellen, dass die Regel aktiviert ist. Um diese Regel zu deaktivieren, wählen Sie Nein.
  13. Wählen Sie im Feld Alert-Dringlichkeit eine der folgenden Optionen:
    • Normal
    • Warnung
    • Unbedeutend
    • Gravierend
    • Kritisch
    • Schwerwiegend
  14. Wählen Sie im Feld Alert-Häufigkeit eine der folgenden Einstellungen, die in Tabelle 25-20 dargestellt sind.

    15. Tabelle 25-20 Einstellung "Alert-Häufigkeit"
    Eigenschaft
    Beschreibung
    Jedes Mal
    Wenn Sie diese Option wählen, werden die Aktionen in der Alert-Regel jedes Mal ausgeführt, wenn die Auswertung der Alert-Regel True ergibt. Beispiel: Wenn Sie die Bedingung festlegen, dass die durchschnittliche Antwortzeit größer als 300 Millisekunden beträgt, empfangen Sie jedes Mal einen Alert, wenn die Auswertung dieser Bedingung True ergibt.
    Die Häufigkeit, mit der eine Alert-Regel ausgewertet wird, hängt vom Aggregationsintervall und vom Sample-Intervall ab, das mit dieser Regel verknüpft ist. Wenn das Aggregationsintervall auf 5 Minuten gesetzt ist, beträgt das Sample-Intervall 1 Minute. Die Regeln werden jedes Mal ausgewertet, wenn 5 Daten-Samples verfügbar sind. Deswegen wird die Regel erstmals ungefähr 5 Minuten nach deren Erstellung und danach jede Minute ausgewertet.
    Einmal benachrichtigen
    Wenn Sie diese Option wählen, werden die in der Regel enthaltenen Aktionen zum ersten Mal ausgeführt, wenn die Auswertung der Regel True ergibt. Es werden keine weiteren Alerts generiert, bis die Bedingung zurückgesetzt wird und die Auswertung erneut True ergibt. Beispiel: Wenn Sie die Bedingung festlegen, dass die durchschnittliche Antwortzeit weniger als 300 Millisekunden beträgt, empfangen Sie einen Alert, wenn die Auswertung dieser Bedingung zum ersten Mal True ergibt. Sie empfangen jedoch keine weiteren Alerts, bis die Auswertung der Bedingung False und anschließend wieder True ergibt. Der Alert-Zeitstempel wird aktualisiert und wird im Dashboard angezeigt.

  15. Wählen Sie für die Option Verarbeitung weiterer Regeln stoppen den Wert Ja, wenn Sie die Ausführung weiterer Regeln abbrechen möchten, wenn die Auswertung einer der mit einem Service verknüpften Regeln True ergibt. Mit dieser Option können Sie die Auswertung weiterer Regeln stoppen, wenn mehrere Regeln mit einem bestimmten Service verknüpft sind. Behalten Sie den Standardwert Nein bei, um mit der Verarbeitung der Regeln fortzufahren.

    16. Diese Option verhält sich wie die Option Keine weiteren Regeln anwenden im Regel-Assistenten von Microsoft Outlook.

Wenn Sie fertig sind

Klicken Sie auf Weiter, um die Seite Neue Alert-Regel - Konfiguration der Alert-Regelbedingungen anzuzeigen. Fahren Sie mit dem Abschnitt Alert-Regelbedingungen definieren fort.
