Serverinformationen anzeigen

Auf der Seite Serverintegrität finden Sie folgende Elemente:

  • Auf dem Registerkarten-Header Serverintegrität kann ein Heruntergefahren-Symbol und eine numerische Anzeige vorhanden sein, die die Anzahl der heruntergefahrenen Server anzeigt.
    • Wenn Sie auf das Symbol klicken, werden die angezeigten Server nach den Kriterien Heruntergefahren Integrität und SHUTDOWN-Status angezeigt.
    • Die Anzahl steht für die Anzahl der heruntergefahrenen Server.
  • Ein Kreisdiagramm, das die Integrität der Server in der Domain veranschaulicht. Sie können auf den serverbezogenen Bereich im Diagramm klicken, um weitere Details anzuzeigen.
  • Die Logübersicht. Klicken Sie auf den Zahlen-Link, um das Log anzuzeigen. Der Link führt zur Anzahl der Nachrichten für einen bestimmten Schweregrad. Beispiel Warnung: 1, klicken Sie auf die Nummer 1. Siehe Details der Domain-Log-Dateien anzeigen.
  • Felder, die Serverinformationen abhängig von den angepassten Einstellungen in der Tabelle Serverübersicht anzeigen. Beispiel, Servername, Integrität, Cluster-Name, Rechnername, Serverstatus und aktive Dauer des Servers. Weitere Informationen zu den Feldern finden Sie in Tabelle 25-16.
  • Standardmäßig ist die Dashboard-Aktualisierungsrate auf Keine Aktualisierung eingestellt. Um das Aktualisierungsintervall für das Dashboard zu ändern, lesen Sie Benutzervoreinstellungen festlegen.
Achtung: Wenn Sie einen anderen Wert als den Standardwert Keine Aktualisierung als Aktualisierungsrate für das Dashboard wählen, aktualisiert der Browser das Dashboard in regelmäßigen Abständen. Dadurch wird ein Timeout der Session verhindert, selbst wenn Sie nicht an der Oracle Service Bus Administration-Konsole arbeiten.
Achtung: Oracle empfiehlt, die Dashboard-Aktualisierungsrate mit Vorsicht anzuwenden. Lassen Sie eine Session nie unbeaufsichtigt. Während die Session aktiv ist, können Dritte ohne erneute Authentifizierung über Ihren Rechner Zugriff darauf erhalten.
  • Das Symbol Tabellen-Customizer. Siehe Tabellenansichten anpassen.
  • So beschränken Sie die Anzahl der Elemente in der Liste oder suchen nach speziellen Servern:
    • Klicken Sie auf das Symbol Öffnen, um zusätzliche Suchfilter anzuzeigen. Wenn Sie mehrere Suchkriterien angeben, werden nur Ergebnisse angezeigt, die allen Kriterien entsprechen.
    • Nach Integritätsstatus filtern. Wählen Sie aus der Dropdown-Liste Integrität eine Option, um die Server auf die angegebenen Integritätsstatusebene zu beschränken. Klicken Sie dann auf Suchen.

      • Aktivieren Sie das Kontrollkästchen oder höher, um die Suche auf die gewünschte Integritätsstatusebene oder höher zu beschränken (von der weitreichendsten bis zur restriktivsten Stufe): OK, Warnung, Überladen, Kritisch und Schwerwiegend.

    • Nach Server filtern. Wählen Sie in der Dropdown-Liste Server die Option "Alle" oder den Servernamen. Klicken Sie dann auf Suchen.
    • Nach Cluster-Name filtern. Wählen Sie in der Dropdown-Liste Cluster-Name die Option "Alle" oder den Namen des mit dem Server verknüpften Clusters. Klicken Sie dann auf Suchen.
    • Nach Rechnername filtern. Wählen Sie in der Dropdown-Liste Rechnername die Option "Alle" oder den Namen des mit dem Server verknüpften Rechners. Klicken Sie dann auf Suchen.
    • Nach Status filtern. Wählen Sie in der Dropdown-Liste Status die Option "Alle" oder den Serverstatus RUNNING, FAILED oder SHUTDOWN. Klicken Sie dann auf Suchen.
    • Klicken Sie auf Alle anzeigen, um die Suchfilter zu entfernen und alle Server anzuzeigen.

Auf der Seite Serverintegrität werden die folgenden Informationen zu den einzelnen Servern angezeigt, die in Tabelle 25-16 aufgelistet sind. Die Anzeige hängt von den Filtereinstellungen ab, die Sie vorgenommen haben.

Tabelle 25-16 Serverübersicht - Details 
Eigenschaft
Beschreibung
Server
Name des Servers. Der Name ist ein Link zur Seite Serverdetails anzeigen. Siehe Serverdetails anzeigen.
Integrität
Integritätsstatus des Servers:
  • OK - Der Server arbeitet problemlos.
  • OK/Keine Verarbeitung - Der Server befindet sich im ADMIN-Status. Ein Zugriff ist nur zu administrativen Zwecken möglich. Alle Oracle Service Bus-Nachrichten zur Laufzeit sind nicht erfolgreich.
  • Warnung - Der Server könnte zukünftig Störungen aufweisen. Prüfen Sie die Server-Logs und das entsprechende RuntimeMBean auf nähere Einzelheiten.
  • Überladen - Dem Server wurde mehr Arbeit zugewiesen als seine konfigurierte Schwelle zulässt. Gegebenenfalls weist er Arbeit zurück.
  • Kritisch - Ausstehender Serverfehler. Eine sofortige Maßnahme ist erforderlich, um einen Serverausfall zu verhindern. Prüfen Sie die Server-Logs und das entsprechende RuntimeMBean auf nähere Einzelheiten.
  • Heruntergefahren - Der Server ist ausgefallen und muss erneut hochgefahren werden.
  • Kritisch - Ausstehender Serverfehler. Eine sofortige Maßnahme ist erforderlich, um einen Serverausfall zu verhindern. Prüfen Sie die Server-Logs und das entsprechende RuntimeMBean auf nähere Einzelheiten.
Cluster-Name
Name des mit dem Server verknüpften Clusters.
Rechnername
Name des mit dem Server verknüpften Rechners.
Status
Status des Servers.
  • RUNNING
  • ADMIN
  • FAILED
  • SHUTDOWN
Produktive Zeit
Zeit, die der Server aktiv ist.
