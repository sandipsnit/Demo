Details der Service-Überwachung anzeigen

Auf der Seite Service-Integrität werden Metriken für Services angezeigt, für die das Überwachen aktiviert ist. Informationen zum Aktivieren der Überwachung für Services finden Sie unter Vorgangseinstellungen für Proxy-Services konfigurieren und Vorgangseinstellungen für Business-Services konfigurieren.

Standardmäßig ist die Dashboard-Aktualisierungsrate auf Keine Aktualisierung eingestellt. Um das Aktualisierungsintervall für das Dashboard zu ändern, lesen Sie Benutzervoreinstellungen festlegen.

Achtung: Wenn Sie einen anderen Wert als den Standardwert Keine Aktualisierung als Aktualisierungsrate für das Dashboard wählen, aktualisiert der Browser das Dashboard in regelmäßigen Abständen. Dadurch wird ein Timeout der Session verhindert, selbst wenn Sie nicht an der Oracle Service Bus Administration-Konsole arbeiten.
Achtung: Oracle empfiehlt, die Dashboard-Aktualisierungsrate mit Vorsicht anzuwenden. Lassen Sie eine Session nie unbeaufsichtigt. Während die Session aktiv ist, können Dritte ohne erneute Authentifizierung über Ihren Rechner Zugriff darauf erhalten.

Im Header der Registerkarte Service-Integrität werden möglicherweise folgende Symbole angezeigt:

  • Ein Symbol Warnung und ein Zahlenindikator.
    • Wenn Sie auf das Symbol klicken, werden die angezeigten Services nach dem Kriterium Mit Fehler gefiltert.
    • Die Zahl gibt die Anzahl der fehlerhaften Services seit der letzten Aktualisierung der Seite an.
  • Ein Symbol Endpunkte offline und ein Zahlenindikator.
    • Wenn Sie auf das Symbol klicken, werden die angezeigten Services nach dem Kriterium Mit Offlineendpunkt-URIs gefiltert.
    • Die Zahl gibt die Anzahl der Services mit Endpunkten an, die seit der letzten Aktualisierung der Seite offline gegangen sind.

Auf der Seite Service-Integrität werden Service-Metriken und Überwachungsinformationen angezeigt, die in Tabelle 25-5 beschrieben sind.

  1. Führen Sie im Feld Statistiken anzeigen einen der folgenden Schritte aus:
    • Wenn Sie Überwachungsstatistiken für alle Services für den Zeitraum des aktuellen Aggregrationsintervalls anzeigen möchte, wählen Sie Aktuelles Aggregationsintervall. In der Ansicht "Aktuelles Aggregationsintervall" wird eine bewegliche Statistik der Service-Metriken angezeigt.
    • Wenn Sie Überwachungsstatistiken für den Zeitraum seit dem letzten Zurücksetzen der Statistiken für einen Service anzeigen möchten, wählen Sie Seit der letzten Rücksetzung. In der Ansicht "Seit der letzten Rücksetzung" wird eine laufende Anzahl der Metriken angezeigt.
  2. Wählen Sie aus der Dropdown-Liste einen Server, um die Metriken für diesen Server anzuzeigen.

    3. Wenn ein Cluster vorhanden ist, werden standardmäßig die clusterweiten Metriken angezeigt. Wählen Sie einen einzelnen Managed Server aus, um die Metriken für diesen Server anzuzeigen.

  3. So schränken Sie die Anzahl der Listenobjekte ein oder suchen bestimmte Services:
    • Klicken Sie auf das Symbol Öffnen, um zusätzliche Suchfilter anzuzeigen. Wenn Sie mehrere Suchkriterien angeben, werden nur Ergebnisse angezeigt, die allen Kriterien entsprechen.
    • Nach Service-Name filtern. Geben Sie im Feld Name den Namen des gesuchten Ziels oder Platzhalterzeichen ein (mit den Platzhaltern * und ? können Sie eine allgemeinere Suche durchführen), dann klicken Sie auf Suchen.

      • Diese Suchmethode ist dann vorzuziehen, wenn der Name des Service in allen Projekten und Pfaden eindeutig ist.

    • Nach Service-Pfad filtern: Geben Sie im Feld Pfad den Pfad des gesuchten Ziels ein. Geben Sie im Feld Pfad den Pfad des Suchzieles ein (der Projektname und der Name des Ordners, in dem sich die Services befinden). Mit den Platzhaltern * und ? können Sie eine allgemeinere Suche durchführen. Alle Services in diesem Pfad werden angezeigt.

      • Das Format für das Feld Pfad ist folgendermaßen:
      project-name/root-folder/ . . ./parent-folder

      Wenn ein Service sich direkt unter dem Projekt befindet, sieht das Format so aus: project-name

    • Nach Service-Name und Pfad filtern. Diese Suchmethode ist dann vorzuziehen, wenn sich mehr als ein Service mit demselben Namen in verschiedenen Verzeichnissen befindet.
    • Filtern Sie nach übergeordnetem Service. Wählen Sie aus der Dropdown-Liste Service den übergeordneten Service-Typ aus: Proxy-Services oder Business-Services. Klicken Sie anschließend auf Suchen. Aktivieren Sie die Option "Alle Services", um die Filterkriterien für übergeordnete Server zu ignorieren.
    • Filtern Sie nach Services mit Nachrichten. Wählen Sie die Option Mit Nachrichten, und klicken Sie auf Suchen.
    • Filtern Sie nach Services mit Alerts. Wählen Sie die Option Mit Alerts, und klicken Sie auf Suchen.
    • Filtern Sie nach Services mit Fehlern. Wählen Sie die Option Mit Fehler, und klicken Sie auf Suchen.
    • Filtern Sie nach Services mit Offline-URIs. Wählen Sie die Option Mit Offlineendpunkt-URIs, und klicken Sie auf Suchen.
    • Filtern Sie nach Services, die von einem bestimmten Proxy-Service aufgerufen werden. Geben Sie den Namen und Pfad eines Proxy-Service in das vorhandene Feld ein, oder klicken Sie auf Durchsuchen, um die Seite Proxy-Service wählen anzuzeigen. Wählen Sie aus der Liste einen Proxy-Service aus, und klicken Sie auf Weiterleiten. Klicken Sie anschließend auf Suchen. Die vom angegebenen Proxy-Service aufgerufenen Business-Services werden angezeigt.

      • Platzhalterzeichen können im Feld Aufrufender Proxy nicht verwendet werden.

    • Klicken Sie auf Alle anzeigen, um die Suchfilter zu deaktivieren und alle Services anzuzeigen.

      • Tabelle 25-5 Service-Überwachungsdetails
      Eigenschaft
      Beschreibung
      Name
      Der dem Service zugewiesene Name. Der Name ist ein Link zur Seite Service-Überwachungsdetails. Weitere Informationen finden Sie unter ###Service-Metriken anzeigen.
      Pfad
      Das mit diesem Service verknüpfte Projekt. Wenn der Service sich in einem Projektordner befindet, wird dieser Ordner auch aufgelistet. Der Pfad wird in folgendem Format angezeigt:
      project-name/root-folder/ . . ./parent-folder
      Der Pfad ist ein Link zum entsprechenden Pfad im Project Explorer.
      Service-Typ
      Der übergeordnete Service-Typ: Proxy-Service oder Business-Service
      Aggregationsintervall
      Dieses Feld wird nur angezeigt, wenn Sie im Feld Statistiken anzeigen den Eintrag Aktuelles Aggregationsintervall gewählt haben.
      Dies ist das aktuelle Aggregationsintervall für das Überwachen dieses Service in Stunden und Minuten. Das Intervall wird auf der Seite Vorgangseinstellungen festgelegt. Weitere Informationen finden Sie unter Aggregationsintervall für einen Service festlegen.
      Durchschn. Antwortzeit
      Die durchschnittliche Antwortzeit (in Millisekunden), die der Service benötigt hat, um Nachrichten innerhalb des aktuellen Aggregationsintervalls oder seit dem letzten Zurücksetzen zu verarbeiten.
      Nachrichten
      Die Anzahl der mit diesem Service verknüpften Nachrichten innerhalb des aktuellen Aggregationsintervalls oder seit dem letzten Zurücksetzen
      Fehler
      Die Anzahl der mit diesem Service verknüpften Fehlermeldungen innerhalb des aktuellen Aggregationsintervalls oder seit dem letzten Zurücksetzen
      Alerts
      Dieses Feld ist standardmäßig ausgeblendet.
      Die Anzahl der mit diesem Service verknüpften Alerts (SLA- und Pipeline Alerts) innerhalb des aktuellen Aggregationsintervalls oder seit dem letzten Zurücksetzen
      Bei einem einzelnen Managed Server in einem Cluster wird für Proxy-Services die Anzahl der Pipeline Alerts angezeigt, die nur während des aktuellen Aggregationsintervalls oder seit dem letzten Zurücksetzen mit diesem Service verknüpft sind.
      Bei Business-Services wird für einzelne Managed Servers "N/V" angezeigt.
      SLA-Alerts
      Die Anzahl der mit diesem Service verknüpften SLA-Alerts innerhalb des aktuellen Aggregationsintervalls oder seit dem letzten Zurücksetzen
      Bei einem einzelnen Managed Server in einem Cluster sind SLA-Alerts nicht verfügbar (es werden nur clusterweite SLA-Alerts angezeigt).
      Pipeline Alerts
      Nur für Proxy-Services:
      Die Anzahl der mit diesem Service verknüpften Pipeline Alerts innerhalb des aktuellen Aggregationsintervalls oder seit dem letzten Zurücksetzen
      Endpunkt-URI-Status
      Nur bei Business-Services der Status von Endpunkt-URIs. Der Status ist ein Link zur Seite Endpunkt-URIs. Weitere Informationen finden Sie unter Metriken für Business-Service-Endpunkt-URIs anzeigen.
      Bei einer Einzelknoten-Domain (individuelle Serverinstanzen):
      • Online: Alle Endpunkt-URIs für den Business-Service sind online.
      • Offline: Alle Endpunkt-URIs für den Business-Service sind offline.
      • Teilweise: Mindestens einer der Endpunkt-URIs für den Business-Service ist offline.
      Bei einer geclusterten Domain:
      • Wenn das Feld Server auf "Cluster" oder einen der Managed Servers gesetzt ist, weist der Status Online darauf hin, dass alle Endpunkt-URIs im gesamten Cluster bzw. auf dem gewählten Managed Server online sind.
      • Wenn das Feld Server auf "Cluster" oder einen der Managed Servers gesetzt ist, weist der Status Offline darauf hin, dass alle Endpunkt-URIs im gesamten Cluster bzw. auf dem gewählten Managed Server offline sind.
      • Wenn das Feld Server auf "Cluster" gesetzt ist, weist der Status Teilweise darauf hin, dass mindestens ein Endpunkt-URI für den Business-Service auf mindestens einem Server offline ist oder dass einer der Endpunkt-URIs auf allen Servern offline ist. Die anderen Endpunkt-URIs für denselben Business-Service sind dennoch auf einem oder allen Servern verfügbar.
      • Wenn das Feld Server auf einen der Managed Server gesetzt ist, weist der Status Teilweise darauf hin, dass mindestens ein Endpunkt-URI für den Business-Service auf dem gewählten Managed Server offline ist.
      Aktion
      Dieses Feld wird nur angezeigt, wenn Sie im Feld Statistiken anzeigen den Eintrag Seit der letzten Rücksetzung gewählt haben.
      In dieser Spalte können Sie für einen bestimmten Service auf das Symbol Zurücksetzen klicken, um die Statistiken für diesen Service zurückzusetzen. Wenn Sie diesen Vorgang bestätigen, werden alle Überwachungsstatistiken, die seit dem letzten Zurücksetzen der Statistiken für den Service gesammelt wurden, vom System gelöscht. Es werden jedoch keine Statistiken gelöscht, die während des "Aktuellen Aggregationsintervalls" für den Service gesammelt werden. Nachdem Sie auf das Symbol Zurücksetzen geklickt haben, beginnt das System sofort, erneut Überwachungsstatistiken für den Service zu sammeln.
