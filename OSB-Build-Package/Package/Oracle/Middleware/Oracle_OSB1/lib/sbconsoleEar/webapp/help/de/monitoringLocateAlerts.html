Alerts suchen

  1. Wählen Sie Vorgänge > Dashboard.
  2. Klicken Sie auf die Registerkarte SLA-Alerts oder Pipeline Alerts, um SLA- oder Pipeline Alerts anzuzeigen.
  3. Klicken Sie auf den Link Erweiterte Alert-Historie. Auf den Seiten Erweiterte SLA-Alert-Historie und Erweiterte Pipeline Alert-Historie werden detaillierte Informationen zur Alert-Historie angezeigt.
  4. So schränken Sie die Anzahl der Objekte in der Liste ein oder suchen nach bestimmten Alerts:
    • Klicken Sie auf das Symbol Öffnen, um zusätzliche Suchfilter anzuzeigen. Wenn Sie mehrere Suchkriterien angeben, werden nur Ergebnisse angezeigt, die allen Kriterien entsprechen.
    • Filtern Sie nach dem Datumsbereich. Führen Sie hierfür einen der folgenden Schritte aus:
      • Wählen Sie Alle, um den Datumsbereich und das Zeitintervall als Filterkriterien zu ignorieren und alle Alerts anzuzeigen.
      • Geben Sie in die Felder Von und Bis ein Datum und eine Uhrzeit im Format MM/TT/JJ HH:MM AM|PM ein, und klicken Sie auf Suchen. Es werden alle Alerts angezeigt, die zum angegebenen Datum und Zeitraum aufgetreten sind.
      • Geben Sie im Feld Während der letzten die Anzahl der Tage sowie Stunden und Minuten ein, und klicken Sie auf Suchen. Es werden alle Alert angezeigt, die im angegebenen Zeitraum aufgetreten sind.
    • Filtern Sie nach Alert-Schweregrad. Wählen Sie aus der Dropdown-Liste Alert-Schweregrad die gewünschte Option, um Alerts auf die angegebene Schweregradstufe einzuschränken, und klicken Sie auf Suchen.

      • Aktivieren Sie das Kontrollkästchen oder höher, um die Suche auf die angegebene Schweregradstufe oder höher zu beschränken (von geringer bis zu starker Einschränkung): Normal, Warnung, Unbedeutend, Gravierend, Kritisch und Schwerwiegend.

    • Filtern Sie nach übergeordnetem Service. Klicken Sie auf Durchsuchen, um die Seite Service wählen anzuzeigen. Wählen Sie den übergeordneten Service aus der Liste, und klicken Sie auf Weiterleiten. Klicken Sie auf Löschen, um das Filterkriterium für den übergeordneten Service zu deaktivieren, oder klicken Sie auf Suchen.
    • Filtern Sie nach Alert-Name (beliebige Zeichenfolge) (nur bei SLA-Alerts). Geben Sie eine beliebige Zeichenfolge einschließlich Platzhalterzeichen ein, und klicken Sie auf Suchen. Weitere Informationen finden Sie unter Abfragesprache des WebLogic-Diagnose-Frameworks.
    • Filtern Sie nach Alert-Übersicht (beliebige Zeichenfolge) (nur bei Pipeline Alerts). Geben Sie eine beliebige Zeichenfolge einschließlich Platzhalterzeichen ein, und klicken Sie auf Suchen. Weitere Informationen finden Sie unter Abfragesprache des WebLogic-Diagnose-Frameworks. Wenn Sie nach Pipeline Alert-Aktionen suchen möchten, die ohne Alert-Übersicht erstellt wurden, geben Sie einen Oracle Service Bus-Alert ein.
    • Klicken Sie auf Alle anzeigen, um die Suchfilter zu deaktivieren und alle Alerts anzuzeigen.

Auf den Seiten Erweiterte SLA-Alert-Historie und Erweiterte Pipeline Alert-Historie können Sie auch folgende Schritte ausführen:

  • Wählen Sie Kreisdiagrammansicht, um die Alert-Daten in einem Kreisdiagramm anzuzeigen.
  • Wählen Sie Balkendiagrammansicht, um dieselben Daten in einem Balkendiagramm anzuzeigen.
  • Wählen Sie Tabellenansicht, um die Daten in Tabellenansicht anzuzeigen.
  • Klicken Sie auf Aktualisieren, um eine aktualisierte Version der Daten anzuzeigen.
  • Wenn Sie einzelne Alerts löschen möchten, aktivieren Sie das entsprechende Kontrollkästchen, und klicken Sie unten auf der Seite auf Löschen.
  • Wenn Sie alle Alerts oder Alerts, die innerhalb eines bestimmten Datums- und Zeitbereiches aufgetreten sind, löschen möchten, klicken Sie oben auf der Seite auf den Link Alert-Historie löschen. Weitere Informationen finden Sie unter Alerts löschen.

Abfragesprache des WebLogic-Diagnose-Frameworks

Alerts werden mit dem WebLogic-Diagnose-Framework gespeichert, das über eine eigene Abfragesprache einschließlich Platzhaltern verfügt. Verwenden Sie zum Filtern von Alerts in der erweiterten Alert-Historie die Syntax, die unter "WLDF Query Language" in Oracle Fusion Middleware Configuring and Using the Diagnostics Framework for Oracle WebLogic Server unter http://www.oracle.com/pls/as111170/lookup?id=WLDFC316 beschrieben ist.
