Details von Domain-Log-Dateien anzeigen

Auf dieser Seite können Sie die Details der Einträge von Domain-Log-Dateien anzeigen, die in Tabelle 25-18 beschrieben sind.

Table 25-18 Domain-Log-Details 
Eigenschaft
Beschreibung
Nachricht
Eine Beschreibung des Ereignisses oder der Bedingung.
Datum
Zeigt Uhrzeit und Datum der Erstellung der Meldung in einem für das Gebietsschema geeigneten Format an. Die JVM (Java Virtual Machine), die jede WebLogic Server-Instanz ausführt, ruft Informationen zur lokalen Zeitzone und zum Format vom Betriebssystem des Host-Rechners ab.
Subsystem
Gibt das Subsystem von WebLogic Server an, in dem die Meldung generiert wurde. Beispiel: Enterprise Java Bean (EJB) Container oder Java Messaging Service (JMS).
Nachrichten-ID
Ein eindeutiger sechsstelliger Bezeichner
Alle Meldungs-IDs, die von WebLogic Server-Systemmeldungen generiert werden, beginnen mit BEA und liegen im numerischen Bereich 0-499999.
Schweregrad
Gibt das Ausmaß der Wirkung oder den Schweregrad des in der Meldung angegebenen Ereignisses an.
  • Alert: Ein bestimmter Service befindet sich in einem nicht verwendbaren Zustand, während andere Teile des Systems weiterhin funktionieren. Eine automatische Wiederherstellung ist nicht möglich. Zur Behebung dieses Problems ist eine sofortige Bearbeitung durch den Administrator erforderlich.
  • Kritisch: Das System oder ein Service hat einen Fehler gemeldet. Eine Wiederherstellung des Systems ist möglich, es kann jedoch zu einem vorübergehenden Verlust oder einer dauerhaften Beeinträchtigung des Service kommen.
  • Besonderer Schweregrad: Der Server befindet sich in einem nicht brauchbaren Zustand. Dieser Schweregrad zeigt einen schwerwiegenden Systemfehler an.
  • Fehler: Es ist ein Benutzerfehler aufgetreten. Der Fehler kann vom System oder der Anwendung ohne Unterbrechung und mit geringer Beeinträchtigung des Service verarbeitet werden.
  • Info: Wird zum Melden normaler Vorgänge verwendet und ist eine Informationsmeldung von geringerer Wichtigkeit.
  • Hinweis: ist eine Informationsmeldung von größerer Wichtigkeit.
  • Warnung: Es ist ein verdächtiger Vorgang oder eine verdächtige Konfiguration aufgetreten, der bzw. die jedoch möglicherweise keine Auswirkungen auf den normalen Ablauf hat.
Rechner
Der DNS-Name des Rechners, auf dem die Serverinstanz gehostet wird
Server
Der Name der WebLogic Server-Instanz, auf der die Nachricht generiert wurde.
Thread
Die ID, die die JVM dem Thread zuweist, in dem die Nachricht generiert wurde.
Benutzer-ID
Die Benutzer-ID, unter der das verknüpfte Ereignis ausgeführt wurde
Transaktions-ID
Nur für Meldungen vorhanden, die im Kontext einer Transaktion protokolliert wurden.
Kontext-ID
Kontextinformationen, mit denen Meldungen von einer bestimmten Anforderung oder Anwendung korreliert werden
Details
Eine Beschreibung des Ereignisses oder der Bedingung.
Ursache
Die Ursache der Meldung.
Aktion
Die auszuführende Aktion.
