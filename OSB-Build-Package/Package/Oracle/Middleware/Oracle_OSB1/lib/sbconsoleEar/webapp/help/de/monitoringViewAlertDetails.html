Alert-Details anzeigen

Auf dieser Seite werden Einzelheiten zu bestimmten Alerts einschließlich der in Tabelle 25-4 beschriebenen Informationen angezeigt.

Tabelle 25-4 Alert-Details 
Eigenschaft
Beschreibung
Name des Alerts
Nur bei SLA-Alerts:
Der dem Alert zugewiesene Name. Der Name ist ein Link zur Seite Alert-Regelkonfiguration. Weitere Informationen finden Sie unter Alert-Regelkonfiguration anzeigen.
Alert-Zusammenfassung
Nur bei Pipeline Alerts:
Eine kurze Beschreibung der Pipeline Alert-Aktion oder des Oracle Service Bus-Alerts (der Standardwert für die Pipeline Alert-Aktionen, die ohne Alert-Übersicht erstellt wurden)
Beschreibung
Eine Beschreibung dieser Alert-Regel
Zeitstempel
  • Bei Pipeline Alerts: Datum und Uhrzeit des Alerts
  • Bei SLA-Alerts: Datum und Uhrzeit des Zeitpunkts, an dem die Alert-Regel zuletzt als wahr ausgewertet wurde
Schweregrad
Der benutzerdefinierte Schweregrad des Alerts:
  • Schwerwiegend
  • Kritisch
  • Gravierend
  • Unbedeutend
  • Warnung
  • Normal
Service
Der Name und Pfad des Service, der mit diesem Alert verknüpft ist. Das Feld ist ein Link zur Seite Service-Überwachungsdetails (nur, wenn der Service aktiv überwacht wird). Weitere Informationen finden Sie unter ###Service-Metriken anzeigen.
Service-Typ
Der übergeordnete Service-Typ: Proxy-Service oder Business-Service
Da Pipeline Alerts nur von Proxy-Services definiert werden können, sind bei Pipeline Alerts alle übergeordneten Services Proxy-Services.
Server
Der Name des Servers, auf dem der Alert generiert wurde.
Nur bei SLA-Alerts: Es wird "N/V" angezeigt.
Anmerkung
Ein Textfeld, in das Sie Notizen für diesen Alert eingeben können. In der Alert-Historie wird für Alerts mit Anmerkungen das Symbol Anmerkung angezeigt.
