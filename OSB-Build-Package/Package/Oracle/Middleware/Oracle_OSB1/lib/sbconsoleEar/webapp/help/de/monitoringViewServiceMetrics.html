Service-Metriken anzeigen

Auf dieser Seite können Sie die Überwachungsdetails bestimmter Services aus Tabelle 25-10 und die Service-Metrikenaus Tabelle 25-11 anzeigen.

Tabelle 25-10 Service-Überwachungsdaten 
Eigenschaft
Beschreibung
Service-Name
Name und Pfad des Service
Service-Typ
Der übergeordnete Service-Typ: Proxy-Service oder Business-Service
Statistiken anzeigen
Um die Überwachungsstatistik für den Service für den Zeitraum des aktuellen Aggregationsintervalls anzuzeigen, wählen Sie Aktuelles Aggregationsintervall. Die Ansicht für das aktuelle Aggregationsintervall zeigt eine veränderliche Statistik der Service-Metriken an.
Um die Überwachungsstatistik für diesen Service für den Zeitraum seit dem letzten Zurücksetzen der Statistik anzuzeigen, wählen Sie Seit der letzten Rücksetzung. Die Ansicht "Seit der letzten Rücksetzung" zeigt die laufende Anzahl der Metriken an.
Server
Wählen Sie einen Server aus der Dropdown-Liste, um die Metriken für diesen Server anzuzeigen.
Wenn ein Cluster vorhanden ist, werden standardmäßig die clusterweiten Metriken angezeigt. Wählen Sie einen einzelnen Managed Server aus, um die Metriken für diesen Server anzuzeigen.
Wenn Sie die Metriken für einen Managed Server in einem Cluster anzeigen, wird die SLA-Alert-Anzahl als "N/V" angezeigt (nur die Cluster-weite SLA-Alert-Anzahl wird angezeigt).
Aggregationsintervall
Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, wird das aktuelle Aggregationsintervall für die Überwachung dieses Service in Stunden und Minuten eingestellt. Sie legen dieses Intervall auf der Seite Vorgangseinstellungen fest. Siehe ###Aggregationsintervall für einen Service festlegen.
Statistiken zurücksetzen
Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, können Sie die Überwachungsstatistiken für diesen Service zurücksetzen. Klicken Sie unten auf der Seite auf Statistiken zurücksetzen.

Tabelle 25-11 Service-Metriken anzeigen 
Eigenschaft
Beschreibung
SLA-Alert-Anzahl
  • Die Alert-Anzahl basiert auf der Ansicht Statistiken anzeigen entweder für das aktuelle Aggregationsintervall oder für den Zeitraum seit der letzten Rücksetzung.
  • Der Schweregrad und der Zeitstempel des Alerts werden nur in der Ansicht Aktuelles Aggregationsintervall angezeigt. In der Ansicht Seit der letzten Rücksetzung wird nur die Anzahl angezeigt.
  • Der Schweregrad für den dringlichsten Alert im aktuellen Aggregationsintervall. Wenn mehrere Alerts vorhanden sind, wird der höchste Alert-Schweregrad im aktuellen Aggregationsintervall angezeigt. Wenn keine Alerts im aktuellen Aggregationsintervall vorliegen, wird NORMAL angezeigt.
  • Der Zeitstempel zeigt an, wann der Server zum letzten Mal nach Alert-Daten abgefragt wurde. Außerdem wird der Alert-Status zu der Zeit angezeigt, zu der der Server abgefragt wurde. Wenn keine Alerts vorhanden waren, als der Server abgefragt wurde, wird NORMAL sowie die Abfragezeit angezeigt.
Pipeline-Alert-Anzahl
Nur für Proxy-Services:
  • Die Pipeline-Anzahl basiert auf der Ansicht Statistiken anzeigen entweder für das aktuelle Aggregationsintervall oder für den Zeitraum seit der letzten Rücksetzung.
  • Der Schweregrad und der Zeitstempel des Alerts werden nur in der Ansicht Aktuelles Aggregationsintervall angezeigt. In der Ansicht Seit der letzten Rücksetzung wird nur die Anzahl angezeigt.
  • Der Schweregrad für den dringlichsten Alert im aktuellen Aggregationsintervall. Wenn mehrere Alerts vorhanden sind, wird der höchste Alert-Schweregrad im aktuellen Aggregationsintervall angezeigt. Wenn keine Pipeline-Alerts im aktuellen Aggregationsintervall vorliegen, wird NORMAL angezeigt.
  • Der Zeitstempel zeigt an, wann der Server zum letzten Mal nach Pipeline-Alert-Daten abgefragt wurde. Außerdem wird der Alert-Status zu der Zeit angezeigt, zu der der Server abgefragt wurde. Wenn keine Alerts vorhanden waren, als der Server abgefragt wurde, wird NORMAL sowie die Abfragezeit angezeigt.
Min. Antwortzeit
Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies die minimale Antwortzeit, die dieser Service für die Nachrichtenausführung im Zeitraum des aktuellen Aggregationsintervalls benötigt hat.
Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, dann ist dies die minimale Antwortzeit, die dieser Service für die Nachrichtenausführung im Zeitraum seit der letzten Rücksetzung der Statistiken benötigt hat.
Max. Antwortzeit
Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies die maximale Antwortzeit, die dieser Service für die Nachrichtenausführung im Zeitraum des aktuellen Aggregationsintervalls benötigt hat.
Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, dann ist dies die maximale Antwortzeit, die dieser Service für die Nachrichtenausführung im Zeitraum seit der letzten Rücksetzung der Statistiken benötigt hat.

Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies die gesamte durchschnittliche Antwortzeit (in Millisekunden), die dieser Service für die Nachrichtenausführung im Zeitraum des aktuellen Aggregationsintervalls benötigt hat.
Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, dann ist dies die gesamte durchschnittliche Antwortzeit, die dieser Service für die Nachrichtenausführung im Zeitraum seit der letzten Rücksetzung der Statistiken benötigt hat.

Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies die gesamte Anzahl der Nachrichten, die dieser Service im Zeitraum des aktuellen Aggregationsintervalls ausgeführt hat.

Failover-Anzahl
Nur für Business-Services:
Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies die Anzahl der Failovers, die dieser Service im Zeitraum des aktuellen Aggregationsintervalls auszuführen versucht hat.
Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, dann ist dies die Anzahl der Failovers, die dieser Service im Zeitraum seit der letzten Rücksetzung der Statistiken auszuführen versucht hat.
Erfolgsverhältnis
Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies das Erfolgsverhältnis dieses Service im Zeitraum des aktuellen Aggregationsintervalls.
Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, dann ist dies das Erfolgsverhältnis dieses Service im Zeitraum seit der letzten Rücksetzung der Statistiken.
Beispiel: Wenn der Service 9 von 10 Nachrichten erfolgreich ausgeführt hat, liegt das Erfolgsverhältnis bei 90 %.
Fehlerverhältnis
Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies das Fehlerverhältnis dieses Service im Zeitraum des aktuellen Aggregationsintervalls.
Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, dann ist dies das Fehlerverhältnis dieses Service im Zeitraum seit der letzten Rücksetzung der Statistiken.
Beispiel: Wenn der Service 1 von 10 Nachrichten nicht oder fehlerhaft ausgeführt hat, dann liegt das Fehlerverhältnis bei 10 %.

Für Business-Services: N/V wird angezeigt.
Für Proxy-Services:
Wenn Sie im Feld Statistiken anzeigen den Wert Aktuelles Aggregationsintervall gewählt haben, dann ist dies die Anzahl der Nachrichten mit Validierungsfehlern, die dieser Service im Zeitraum des aktuellen Aggregationsintervalls ausgeführt hat.

Anzahl Treffer
Nur für Business-Services:
Die Anzahl der Treffer zeigt an, wie oft ein Business-Service-Ergebniscache verwendet wurde.
Trefferrate
Die Summe von Trefferanzahl und Nachrichtenanzahl.
Minimale Throttling-Zeit

Wenn Sie im Feld Statistiken anzeigen den Wert Seit der letzten Rücksetzung gewählt haben, dann ist dies die Zeit, die die Nachrichten im Zeitraum seit der letzten Rücksetzung der Statistiken mindestens in der Throttling Queue verbracht haben.
