Benutzerdefinierte WS-Policys

Hinweis: In künftigen Versionen von Oracle Service Bus werden Oracle Web Services Manager-Policys die WLS 9.2-Policys ersetzen. In dieser Version von Oracle Service Bus werden die WLS 9.2-Policys weiterhin unterstützt, aber es empfiehlt sich, die Oracle Web Services Manager-Policys bei der Erstellung neuer Services zu verwenden, damit Sie auf eine spätere Einschränkung des Supports der WLS 9.2-Policys vorbereitet sind. Weitere Informationen finden Sie unter "Securing Oracle Service Bus with Oracle Web Services Manager" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681.

Web Services Policy Framework (WS-Policy) ist ein erweiterbares XML-basiertes Framework, das die Konfiguration eines Web-Services mit domainspezifischen Sicherheitserklärungen erweitert und die Sicherheitsanforderungen und -möglichkeiten des Web-Service spezifiziert. In Oracle Service Bus stellt die Konfiguration der Sicherheit auf Nachrichtenebene in Proxy und Business-Services einen der Hauptverwendungszwecke von WS-Policy dar.

Oracle Service Bus WS-Policys verwenden

Oracle Service Bus umfasst drei einfache WS-Policy-Dateien, mithilfe derer Sie vorgeben können, dass Clients SOAP-Nachrichten autorisieren, digital verschlüsseln oder digital signieren müssen: Auth.xml, Encrypt.xml und Sign.xml. Oracle empfiehlt die Verwendung dieser drei vordefinierten Dateipakete so oft wie möglich, sofern Sie keine speziellen Sicherheitsanforderungen haben.

Weitere Informationen zur Verwendung dieser Policys finden Sie unter "Using WS-Policy in Oracle Service Bus Proxy and Business-Services" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.

Benutzerdefinierte WS-Policys verwenden

Wenn die Oracle Service Bus WS-Policy-Anweisungen Ihre Sicherheitsanforderungen nicht erfüllen, können Sie Ihre eigenen WS-Policys (benutzerdefinierte WS-Policys) erstellen, diese in Oracle Service Bus importieren und dann von WSDL auf sie Bezug nehmen. (Die Oracle Service Bus WS-Policy-Anweisungen sind schreibgeschützt.)

Weitere Informationen zum Erstellen und Bezugnehmen auf benutzerdefinierte WS-Policys finden Sie unter "Using WS-Policy in Oracle Service Bus Proxy and Business-Services" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.
