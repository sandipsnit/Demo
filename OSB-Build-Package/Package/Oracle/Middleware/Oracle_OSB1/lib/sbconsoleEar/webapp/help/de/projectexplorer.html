Mit Projekten, Ordnern und Ressourcen arbeiten

Im Project Explorer werden Konfigurationen und Ressourcen in Projekten und Ordnern verwaltet. Im Project Explorer können Sie die Seite Projektansicht anzeigen, in der alle Projekte in der Domain aufgelistet sind, und die Seiten Projektansicht/Ordneransicht, in der Ordner und Ressourcen in jedem Projekt aufgelistet sind.

Projekte sind nicht-hierarchische, sich nicht überschneidende Gruppierungskonstrukte auf der obersten Ebene.

Alle Oracle Service Bus-Ressourcen (z. B. Services, WS-Policys, WSDLs, XQuery-Transformationen) befinden sich in genau einem Projekt. Projekte überschneiden sich nicht. Ressourcen können direkt unter einem Projekt erstellt werden, oder sie können weiter in Ordner unterteilt werden. Sie können jede Ressource unabhängig von dem Projekt, in dem sie sich befindet, referenzieren.

Wenn Sie eine Domain in Oracle Service Bus erstellen, wird ein Standardprojekt erstellt.

Projekte und Ordner benennen

Projektnamen und Ordnernamen sind auf 64 Zeichen beschränkt. Befolgen Sie die Benennungseinschränkungen für Ressourcen.

Namen, Länge und Ebenen der Verschachtelung von Projekten und Ordnern werden letztendlich nur durch die Grenzen Ihres Betriebssystems beschränkt. Die Erstellung von Ordnern mit sehr langen Namen oder eine starke Verschachtelung der Ordner kann zu Fehlern führen, wenn Ihr Betriebssystem damit überlastet wird.

Ressourcennamen mit Projekten und Ordnern angeben

Projekte und Ordner geben die Namen der Oracle Service Bus-Ressourcen an. Eine Referenz zu einer Ressource ist folgendermaßen aufgebaut:

project-name/root-folder/.../parent-folder/resource-name

Wenn sich eine Ressource direkt unter einem Projekt befindet, ist die Referenz folgendermaßen aufgebaut:

project-name/resource-name
