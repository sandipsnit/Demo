Ressourcen aus einer Zip-Datei laden

Sie können eine Zip- oder JAR-Datei wählen, die die Ressource enthält, die Sie laden möchten; Oracle Service Bus findet und importiert die Ressourcen aus dieser Zip-Datei.

Ressourcen wie WSDLs oder XML-Schemas können volle Bäume abhängiger Ressourcen definieren. Die Funktion "Bulk laden" ermöglicht es Ihnen, eine Gruppe Ressourcen auf einmal zu laden. Sie müssen die Abhängigkeiten nicht manuell lösen.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer, wählen Sie dann das Projekt oder den Ordner, zu dem Sie Ressourcen hinzufügen möchten. Die Seite Projekt-/Ordneransicht wird angezeigt.
  3. Von der Dropdown-Liste Ressourcen erstellen wählen Sie Komprimierte Ressourcen. Die Seite Ressourcen laden wird angezeigt.
  4. Geben Sie im Feld Zip-Datei wählen eine zu importierende Zip-Datei an. Geben Sie den Namen der Datei an (angegeben durch ihren Pfad), oder klicken Sie auf Durchsuchen, suchen Sie die Datei, und klicken Sie auf Öffnen.
  5. Prüfen Sie die Zuordnung der Ressourcentypen zu den Dateierweiterungen. Ändern Sie entsprechend Ihrer Bedürfnisse Erweiterungen, oder fügen Sie welche hinzu. Trennen Sie mehrere Erweiterungen durch Komma.

    6. Jede Datei in der Zip-Datei ist eine mögliche Ressource, und die Dateierweiterung ist im Allgemeinen der Weg, den Dateityp zu identifizieren. Oracle Service Bus definiert eine Standarderweiterung für jede Ressource, aber wenn Sie eine Ressource mit zusätzlichen Erweiterungen beschreiben, ermöglicht Ihnen diese Zuordnung, diese Erweiterungen für den Loader zu deklarieren.

    Nachdem Sie die Zip-Datei und die Erweiterungszuordnung definiert haben, scannt Oracle Service Bus-Loader den Inhalt der Zip-Datei und versucht, jeder Datei einen Ressourcentyp zuzuordnen. Eine Datei ohne Erweiterung oder eine, für die in der Zuordnung keine Erweiterung definiert wurde, wird als nicht bekannt eingestuft und automatisch vom Laden ausgeschlossen.

    Bei bekannten Dateien wird als Name der Ressource der Name der Datei, jedoch ohne Erweiterung, übernommen. Diese Ordnerstruktur der Zip-Datei wird im Zielordner der Oracle Service Bus Administration-Konsole erneut erstellt.

    Eine Erweiterung kann nur einem Dateityp zugeordnet werden.

  6. Klicken Sie auf Weiter.
  7. Prüfen Sie auf der Seite Ressourcen laden - Geladene Ressourcen prüfen die Ressourcen, die Sie importieren. Sie können die Kontrollkästchen neben den Ressourcen deaktivieren, die Sie nicht in diesen Import einschließen wollen.
  8. Klicken Sie auf Import, um die gewählten Ressourcen zu importieren.
  9. Die Seite Ressourcen laden - Importergebnis zeigt die importierten Ressourcen an. Klicken Sie auf Weitere Laden, wenn Sie Ressourcen einer anderen Datei laden möchten, oder fahren Sie mit anderen Aufgaben fort, indem Sie im linken Navigationsfenster eine andere Seite wählen.
  10. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
