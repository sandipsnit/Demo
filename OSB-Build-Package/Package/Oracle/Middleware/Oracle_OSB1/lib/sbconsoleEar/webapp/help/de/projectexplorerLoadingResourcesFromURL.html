Ressourcen aus einem URL laden

Sie können einen Bulk-Import der Ressourcen vornehmen, die in einem URL oder im Dateisystem verfügbar sind. Per Bulk-Import können Sie eine Root-Ressource, wie eine WSDL, zusammen mit ihren abhängigen Elementen, wie andere WSDLs und Schemas, importieren. Die Abhängigkeitszuordnung wird automatisch aufgelöst.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer, wählen Sie dann das Projekt oder den Ordner, zu dem Sie Ressourcen hinzufügen möchten. Die Seite Projekt-/Ordneransicht wird angezeigt.
  3. Wählen Sie aus der Dropdown-Liste Ressource erstellen die Option Ressourcen aus URL. Die Seite Ressourcen laden - Ressourcen aus URL laden wird angezeigt.
  4. Geben Sie im Feld URL/Pfad den URL oder Pfad der Ressource an.

    5. Um eine lokale Ressource anzugeben, können Sie das Datei-Protokoll mit einem Dateinamen verwenden, der vollständig durch den Pfad angegeben ist. Beispiel: file:///c:/osbresources/ForeachAction.jar

  5. Geben Sie im Feld Ressourcenname den Namen der Ressource ein.
  6. Wählen Sie in der Dropdown-Liste Ressourcentyp den Typ der Ressource.
  7. Klicken Sie auf Weiter, um die Seite Ressourcen laden - Geladene Ressourcen prüfen anzuzeigen. Deaktivieren Sie das Kontrollkästchen neben den Ressourcen, die Sie nicht in den Import einschließen möchten.
  8. Klicken Sie auf Import, um die gewählten Ressourcen zu importieren.
  9. Die Seite Ressourcen laden - Importergebnis zeigt die importierten Ressourcen an. Klicken Sie auf Weitere Laden, wenn Sie Ressourcen einer anderen Datei laden möchten, oder fahren Sie mit anderen Aufgaben fort, indem Sie im linken Navigationsfenster eine andere Seite wählen.
  10. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
