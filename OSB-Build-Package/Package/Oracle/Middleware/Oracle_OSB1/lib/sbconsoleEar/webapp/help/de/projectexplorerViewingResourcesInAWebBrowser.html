Ressourcen in einem Webbrowser anzeigen

Oracle Service Bus-Ressourcen mit folgenden URLs in einem Standard-Webbrowser anzeigen

    • Pfad für WSDL-Ressourcen:
      http://host:port/sbresource?WSDL/project_path/wsdlname
      oder
      http://host:port/proxy_service_endpoint_URI?WSDL (für WSDL-basierte HTTP-Proxyservices)
    • Pfad für Proxyservice-Ressourcen:
      http://host:port/sbresource?PROXY/project_path/proxyname
      http://host:port/sbresource?ORAPROXY/project_path/proxyname (für Proxyservices mit verknüpften Oracle Web Services Manager-Policys)
    • Pfad für Business-Service-Ressourcen:
      http://host:port/sbresource?BIZ/project_path/bizname
      http://host:port/sbresource?ORABIZ/project_path/bizname (für Business-Services mit verknüpften Oracle Web Services Manager-Policys)
    • Pfad für WSS-Ressourcen:
      http://host:port/sbresource?POLICY/project_path/policyname
    • Pfad für MFL-Ressourcen:
      http://host:port/sbresource?MFL/project_path/mflname
    • Pfad für SCHEMA-Ressourcen:
      http://host:port/sbresource?SCHEMA/project_path/schemaname

    Hinweis: Wenn Sie Sonderzeichen in den Ressourcennamen verwenden, müssen die URLs, mit denen die Ressourcen in Oracle Service Bus angegeben werden, in UTF-8 codiert werden, um Sonderzeichen zu escapen.

    Sie können auch WSDLs abrufen, die Oracle Web Services Manager-Policys enthalten, damit die Policys den unterstützten WS-Policy- und WS-Security-Policy-Standards entsprechen. Weitere Informationen finden Sie unter "Advertising WSDLs to Support WS Standards" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1877.
