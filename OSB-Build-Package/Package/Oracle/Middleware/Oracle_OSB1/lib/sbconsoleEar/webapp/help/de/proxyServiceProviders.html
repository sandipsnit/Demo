Service-Schlüsselprovider

Service-Schlüsselprovider·enthalten·PKI-Zugangsdaten (PKI = Public Key Infrastructure), die von Proxy-Services zum Entschlüsseln von eingehenden SOAP-Nachrichten sowie für die ausgehende Authentifizierung und digitale Signaturen verwendet wird. PKI-Zugangsdaten sind ein Private Key gepaart mit einem Zertifikat, das für digitale Signaturen und zur Verschlüsselung (für Web Service Security) und für ausgehende SSL-Authentifizierung verwendet werden kann. Das Zertifikat enthält den Public Key, der dem Private Key entspricht.

Hinweis: Wenn Sie einen Service-Schlüsselprovider verwenden möchten, müssen Sie einen PKI-Zugangsdaten-Zuordnungsprovider konfigurieren. Weitere Informationen finden Sie unter "Configuring the WebLogic Security Framework: Main Steps" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498.

Ein einzelner Service-Schlüsselprovider kann alle der folgenden PKI-Zugangsdaten enthalten:

  • Ein Schlüsselpaar für digitale Verschlüsselung

    • Dieses Schlüsselpaar wird von Proxy-Services für die Entschlüsselung von eingehenden SOAP-Nachrichten verwendet, die verschlüsselt wurden, um mit einer Web Service Policy konform zu sein. Wenn der Service-Schlüsselprovider digitale Verschlüsselung unterstützen soll, muss der mit dem PKI-Zugangsdaten-Mapper verknüpfte Keystore mindestens ein X.509-Zertifikat enthalten, das Verschlüsselung unterstützt.

  • Ein Schlüsselpaar für digitale Signaturen

    • Dieses Schlüsselpaar wird von Proxy-Services verwendet, wenn der Endpunkt ein Web Service ist und dieser erfordert, dass Clients ein oder mehrere Teile eines SOAP-Envelope signieren.

  • Ein Schlüsselpaar für SSL-Clientauthentifizierung (wechselseitige SSL)

    • Dieses Schlüsselpaar wird von Proxy-Services zur Authentifizierung verwendet, wenn sie als Client während einer eingehenden TLS/SSL-Verbindung (Secure Sockets Layer) dienen, das heißt, beim Routing einer Nachricht an einen HTTPS-Business-Service oder -Proxy-Service, für den eine Clientzertifikatauthentifizierung erforderlich ist.

Sie können denselben Service-Schlüsselprovider für mehrere Proxy-Services verwenden.
