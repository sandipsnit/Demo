Service-Schlüsselprovider hinzufügen

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer, und wählen Sie ein Projekt oder einen Ordner, in dem der Service-Schlüsselprovider hinzugefügt werden soll. Die Seite Projekt-/Ordneransicht wird angezeigt.
  3. Wählen Sie in der Dropdown-Liste Ressource erstellen die Option Service-Schlüsselprovider, um die Seite Neuen Service-Schlüsselprovider erstellen anzuzeigen.
  4. Geben Sie im Feld Name des Service-Schlüsselproviders einen eindeutigen Namen für diesen Service-Schlüsselprovider ein.

    Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
  5. Geben Sie im Feld Beschreibung eine Beschreibung für den Service-Schlüsselprovider ein.
  6. Führen Sie einen der folgenden Schritte aus, die in Tabelle 17-2 dargestellt sind.

    7. Tabelle 17-2 Authentifizierungsoptionen
    Hinzufügen eines Schlüsselpaares für...
    Vorgehensweise
    Digitale Verschlüsselung
    1. Klicken Sie neben Verschlüsselungsschlüssel auf Durchsuchen.

      2. Im Fenster Alias wählen für Verschlüsselungsschlüssel werden die Schlüsselaliasnamen aus dem Keystore angezeigt, die vom PKI-Zugangsdaten-Mapper Ihrer Realm verwendet werden.

    2. Geben Sie im Fenster Alias wählen für Verschlüsselungsschlüssel das Kennwort ein, das Sie für einen sicheren Zugriff auf den Keystore verwenden. (Das Kennwort wird beim Erstellen des Keystores festgelegt.)
    3. Wählen Sie einen Schlüsselalias, der einem X.509-Zertifikat zugeordnet ist und eine Verschlüsselung unterstützt.
    4. Klicken Sie auf Weiterleiten.
    Wenn Sie diesen Service-Schlüsselprovider einem Proxy-Service zuordnen, wird das X.509-Zertifikat von Oracle Service Bus in die WSDL des Proxy-Service eingebettet. Dann wird dieses Zertifikat vom Proxy-Service zum Verschlüsseln der Nachrichten verwendet, die er an den zugehörigen Endpunkt sendet. Der Proxy-Service verwendet den Private Key in den PKI-Zugangsdaten, um die vom Endpunkt zurückgegebenen Nachrichten zu entschlüsseln.
    Digitale Signaturen
    1. Klicken Sie neben Digitaler Signaturschlüssel auf Durchsuchen.

      2. Im Fenster Alias wählen für Digitaler Signaturschlüssel werden die Schlüsselaliasnamen aus dem Keystore angezeigt, die vom PKI-Zugangsdaten-Mapper Ihrer Realm verwendet werden.

    2. Geben Sie im Fenster Alias wählen für Digitaler Signaturschlüssel das Kennwort ein, das Sie für einen sicheren Zugriff auf den Keystore verwenden. (Das Kennwort wird beim Erstellen des Keystores festgelegt.)
    3. Wählen Sie einen Schlüsselalias.
    4. Klicken Sie auf Weiterleiten.
    SSL-Clientauthentifizierung (wechselseitige SSL)
    1. Klicken Sie neben SSL-Clientauthentifizierungsschlüssel auf Durchsuchen.

      2. Im Fenster Alias wählen für SSL-Clientauthentifizierungsschlüssel werden die Schlüsselaliasnamen aus dem Keystore angezeigt, die vom PKI-Zugangsdaten-Mapper Ihrer Realm verwendet werden.

    2. Geben Sie im Fenster Alias wählen für SSL-Clientauthentifizierungsschlüssel das Kennwort ein, das Sie für einen sicheren Zugriff auf den Keystore verwenden. (Das Kennwort wird beim Erstellen des Keystores festgelegt.)
    3. Wählen Sie einen Schlüsselalias.
    4. Klicken Sie auf Weiterleiten.

  7. Klicken Sie auf Speichern. Der Service-Schlüsselprovider wird in der aktuellen Session gespeichert.
  8. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
