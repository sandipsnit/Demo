Service-Schlüsselprovider löschen

Wenn Sie einen Service-Schlüsselprovider löschen, löscht Oracle Service Bus auch den damit verknüpften Aliasnamen zu den Schlüsselpaar-Bindings aus dem Provider der PKI-Zugangsdatenzuordnungen. Oracle Service Bus löscht das damit verknüpfte Schlüsselzertifikatpaar nicht aus dem Keystore.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wenn ein Proxy-Service für die Verwendung des Service-Schlüssels konfiguriert wurde, entfernen Sie den Service-Schlüsselprovider aus dem Proxy-Service. Siehe Proxy-Service-Konfigurationen bearbeiten.
  3. Wählen Sie Ressourcenbrowser > Service-Schlüsselprovider, um die Seite Übersicht der Service-Schlüsselprovider anzuzeigen.
  4. Klicken Sie im Feld Optionen des zu löschenden Service-Schlüsselproviders auf das Symbol Löschen. Der Service-Schlüsselprovider wird in der aktuellen Session gelöscht. Wenn ein Business oder Proxy-Service für die Verwendung eines Service-Accounts konfiguriert wurde, zeigt das Symbol Warnung wegen Löschvorgang an, dass Sie den Service-Schlüsselprovider mit einer Warnbestätigung löschen können. Dies kann jedoch zu Konflikten aufgrund von nicht aufgelösten Referenzen vom Service zum gelöschten Service-Schlüsselprovider führen.
  5. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
