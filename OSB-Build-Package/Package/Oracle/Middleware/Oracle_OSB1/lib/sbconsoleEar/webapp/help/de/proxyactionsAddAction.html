Aktionen im Nachrichtenfluss hinzufügen und bearbeiten

Aktionen sind die Elemente von Pipeline Stages, Error Handler Stages, Route-Knoten und Verzweigungsknoten, in denen festgelegt wird, wie Nachrichten auf ihrem Weg durch den Proxy-Service definiert werden sollen.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

Weiterhin wird angenommen, dass Sie bereits einen Pipeline-Stage, einen Route-Knoten und/oder einen Error Handler-Stage hinzugefügt haben. Siehe:

  • Pipeline-Paarknoten zu Nachrichtenfluss hinzufügen
  • Stages zu Pipelines hinzufügen
  • Proxy-Service-Error-Handler hinzufügen

So fügen Sie eine Aktion zu einem Nachrichtenfluss hinzu:

  1. Wählen Sie die Komponente aus, zu der Sie eine Aktion hinzufügen möchte. Beispiel: Klicken Sie auf das Symbol Stage, dann auf Stage bearbeiten, oder klicken Sie auf das Symbol Route-Knoten und dann auf Route bearbeiten.
  2. Führen Sie je nachdem, welche Aktionen bereits zum Stage oder Route-Knoten hinzugefügt wurden, einen der folgenden Schritte aus:
    • Wurden noch keine Aktionen hinzugefügt, wird auf der Seite Stage-Konfiguration bearbeiten nur das Symbol Aktion hinzufügen angezeigt. Klicken Sie auf dieses Symbol, und wählen Sie dann einen Aktionstyp.
    • Wurden bereits eine oder mehrere Aktionen hinzugefügt, werden auf der Seite Stage-Konfiguration bearbeiten ein oder mehrere Symbole für diese Aktionen angezeigt, z.B. die Symbole Publizieren oder Routing. Klicken Sie auf das gewünschte Symbol, dann auf Aktion hinzufügen, und wählen Sie dann einen Aktionstyp.
    • Einige Aktionen, z.B. Anforderungs- und Antwortaktionen in Publishing-Aktionen, enthalten den Link Aktion hinzufügen, wenn eine Aktion angebracht ist. Klicken Sie auf das Symbol, und wählen Sie dann einen Aktionstyp.

      • Hinsichtlich dessen, welche Aktionen in einem Nachrichtenfluss aneinandergehängt werden können, gibt es keine Einschränkungen.

  3. Wenn Sie mit dem Hinzufügen von Aktionen fertig sind, können Sie weitere Aktionen im Stage oder im Route-Knoten hinzufügen, wie in Tabelle 21-2 beschrieben.

    4. Tabelle 21-2 Stage-Konfiguration-Tasks bearbeiten 
    Ziel
    Führen Sie diesen Schritt durch:
    Aktion löschen
    Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Diese Aktion löschen.
    Aktion nach unten verschieben (zurückstufen)
    Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Aktion nach unten verschieben. Die Aktion wird unter die nächste in diesem Stage enthaltene Aktion verschoben.
    Diese Option wird nur angezeigt, wenn ein Stage zwei oder mehr Aktionen enthält.
    Aktion nach oben verschieben (hochstufen)
    Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Aktion nach oben verschieben. Die Aktion wird über die vorherige in diesem Stage enthaltene Aktion verschoben.

    Diese Option wird nur angezeigt, wenn ein Stage zwei oder mehr Aktionen enthält.

    Aktion ausschneiden
    Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Ausschneiden.
    Aktion zu kopieren
    Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Kopieren.
    Aktion einfügen, die Sie ausgeschnitten oder kopiert haben
    Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Aktion einfügen.
    Sie können Aktionen über mehrere Stages kopieren und einfügen. Bei den Aktionen Zuweisen, Ersetzen oder Einfügen müssen Sie jedoch folgende Punkte beachten:
    • Alle variablenbezogenen und benutzerdefinierten Namespaces von der Stage-Quelle (Kopie) werden als benutzerdefinierte Namespaces im Stage-Ziel (Einfügung) hinzugefügt.
    • Doppelte Namespaces (identisch in Stage-Quelle und -Ziel) werden nicht kopiert.
    • Unvereinbare Namespaces (Namespace-Deklarationen, die dasselbe Präfix, aber unterschiedliche URIs verwenden) werden kopiert. Benutzer können die Konfiguration speichern, sie jedoch erst aktivieren, wenn die unvereinbaren Namespace-Deklaration in Stage B entfernt sind.
    Stage validieren
    Klicken Sie auf der Seite Stage-Konfiguration bearbeiten auf Validieren, um alle in diesem Stage konfigurierten Aktionen zu validieren.

  4. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  5. Setzen Sie die Erstellung des Nachrichtenflusses auf der Seite Nachrichtenfluss bearbeiten fort, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.
  6. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  7. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
