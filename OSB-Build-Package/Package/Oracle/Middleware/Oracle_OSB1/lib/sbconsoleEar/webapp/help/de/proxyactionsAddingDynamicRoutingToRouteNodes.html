Dynamisches Routing zu Route-Knoten hinzufügen

Ordnet einer Nachricht eine Route auf der Basis der Informationen in einer XQuery-Ressource zu.

Hierbei handelt es sich um eine abschließende Aktion. Das heißt, Sie können nach dieser keine weiteren Aktionen hinzufügen. Diese Aktion kann allerdings Anforderungs- und Antwortaktionen beinhalten.

Weitere Informationen zum Routing finden Sie unter "Modeling Message Flow in Oracle Service Bus" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBAG181.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

So fügen Sie einem Route-Knoten dynamisches Routing hinzu:

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das Symbol Route-Knoten und anschließend auf Route bearbeiten. Die Seite Stage-Konfiguration bearbeiten wird angezeigt.
  3. Klicken Sie auf das Symbol Aktion hinzufügen, und wählen Sie Kommunikation > Dynamisches Routing.
  4. Klicken Sie auf <Ausdruck>. Der XQuery-Ausdruckseditor wird angezeigt.
  5. Geben Sie im XQuery-Ausdruckseditor einen XQuery-Ausdruck ein, dessen Ergebnis in etwa wie folgt aussieht:

    6. <ctx:route>

        <ctx:service isProxy='true'>{$service}</ctx:service>

        <ctx:operation>{$operation}</ctx:operation>

    </ctx:route>

    Hinweis: Wenn ein Proxy-Service aufgerufen wird, muss das Attribut isProxy auf "true" gesetzt sein. Wenn ein Business Service aufgerufen wird, stellen Sie isProxy auf "false" ein.
    Hinweis: - Der Service-Name ist der vollständig qualifizierte Service-Name.
    Hinweis: - Das Element "operation" ist optional.
  6. Klicken Sie auf Speichern.
  7. Um eine Aktion hinzuzufügen, klicken Sie im Feld Anforderungsaktionen auf Aktion hinzufügen. Wählen Sie anschließend eine Aktion, die Sie mit dem Service verknüpfen möchten. Sie können mehrere Aktionen hinzufügen. Weitere Informationen zu dem Typ der hinzuzufügenden Aktionen finden Sie in der Aktionstabelle unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  8. Um eine Aktion hinzuzufügen, klicken Sie im Feld Antwortaktionen auf Aktion hinzufügen. Wählen Sie anschließend eine Aktion, die Sie mit dem Service verknüpfen möchten. Sie können mehrere Aktionen hinzufügen. Weitere Informationen zu dem Typ der hinzuzufügenden Aktionen finden Sie in der Aktionstabelle unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  9. Klicken Sie auf Speichern.
  10. Setzen Sie die Erstellung des Nachrichtenflusses auf der Seite Nachrichtenfluss bearbeiten fort, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.
  11. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  12. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
