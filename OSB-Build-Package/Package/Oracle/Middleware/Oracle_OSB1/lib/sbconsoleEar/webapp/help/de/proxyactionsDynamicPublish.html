Dynamische Publishing-Aktionen hinzufügen

Verwenden Sie eine dynamische Publishing-Aktion, um eine Nachricht in einem Service zu publizieren, der von einem XQuery-Ausdruck angegeben wurde.

Weitere Informationen zum Veröffentlichungsverhalten finden Sie unter "Performing Transformations in Message Flows" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBAG205.

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Aktion hinzufügen > Kommunikation > Dynamisch publizieren.
  3. Klicken Sie auf <Expression>.
  4. Geben Sie im XQuery-Ausdruckseditor einen XQuery-Ausdruck ein, oder wählen Sie eine XQuery-Ressource aus, die ein ähnliches Ergebnis hat: 
    5. <ctx:route>

    <ctx:service 
isProxy="false">project/folder/businessservicename</ctx:service>

    <ctx:operation>foo</ctx:operation>

    </ctx:route>
    Hinweis: Wenn ein Proxy-Service aufgerufen wird, muss das Attribut isProxy auf "true" gesetzt sein. Wenn ein Business Service aufgerufen wird, stellen Sie isProxy auf "false" ein.
    Das Element operation ist optional.
  5. Klicken Sie auf Speichern.
  6. Klicken Sie im Feld Anforderungsaktionen auf Aktion hinzufügen, um eine Aktion hinzuzufügen, dann wählen Sie eine Aktion aus, die Sie mit diesem Service verknüpfen möchten. Sie können mehr als eine Aktion hinzufügen. Weitere Informationen darüber, welche Typen von Aktionen Sie hinzufügen können, finden Sie in der Aktionstabelle unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
