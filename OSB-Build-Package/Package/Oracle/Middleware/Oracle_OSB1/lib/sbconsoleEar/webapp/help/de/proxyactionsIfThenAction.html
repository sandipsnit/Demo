Wenn-Dann-Aktionen hinzufügen

Führen Sie eine Aktion oder eine Gruppen von Aktionen bedingt und basierend auf dem Booleschen Ergebnis eines XQuery-Ausdrucks mit einer If-Then-Aktion aus.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

So fügen Sie eine If-Then-Aktion hinzu

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, und wählen Sie Aktion hinzufügen > Flusssteuerung > If...Then.
  3. Klicken Sie auf <Expression>, um die Seite "XQuery-Bedingungseditor anzuzeigen.

    4. Die von Ihnen erstellte Bedingung wird als Test verwendet, der vor der Eingabe der then()-Klausel als Standard-If-Then-Logik aufgeführt wird. Siehe Inline XQuery- und XPath-Ausdrücke erstellen und bearbeiten.

  4. Wenn Sie die Bearbeitung der XQuery-Bedingung abgeschlossen haben, klicken Sie auf Aktion hinzufügen, und wählen Sie dann eine Aktion, die Sie mit der Bedingung verknüpfen möchten. Weitere Informationen zum hinzuzufügenden Aktionstyp finden Sie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten.

    5. Im Route-Knoten können Sie nur Routing-, dynamische Routing- und Routing-Tabellenaktionen wählen. Diese Aktionen können jedoch Anforderungs- und Antwortaktionen enthalten.

  5. Wenn es Ihre Logik erfordert, klicken Sie auf das Symbol If...Then.... Klicken Sie dann auf Else-If-Bedingung hinzufügen oder Else-Bedingung hinzufügen, um Else-If- oder Else-Bedingungen hinzuzufügen. Klicken Sie auf Aktion hinzufügen, um Aktionen mit diesen Bedingungen zu verknüpfen.

    6. Bedingungsaktionen können verschachtelt werden. Es gibt jedoch eine Verschachtelungsgrenze von vier kumulativen Ebenen im Stage-Editor. Wenn Sie versuchen, eine fünfte Ebene einzuführen, wird die Verschachtelungsaktion nicht angezeigt. Zu kumulativen Ebenen zählen alle Verzweigungsaktionen, If-Then-Bedingungen, Publishing und Routing-Tabellen. Beispiel: Sie haben zwei Bedingungsebenen. Eine Publishing-Tabelle mit einer eingebetteten Routing-Tabelle ergibt zusammen vier Ebenen. Wenn Sie versuchen, eine weitere Bedingungsaktion hinzuzufügen (zur letzten Publishing-Tabelle), wird diese nicht angezeigt.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
