Java-Callout-Aktionen hinzufügen

Mit der Java-Callout-Aktion können Sie eine Java-Methode oder einen EJB-Business-Service aus dem Nachrichtenfluss heraus aufrufen.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

So fügen Sie eine Java-Callout-Aktion hinzu

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, danach klicken Sie auf Aktion hinzufügen > Nachrichtenverarbeitung > Java-Callout.
  3. Klicken Sie auf <Method>. Die Seite JAR wählen wird angezeigt. Wählen Sie eine JAR-Ressource aus der Liste aus. Die Seite Klasse und Methode wählen wird geöffnet.
  4. Klicken Sie in der Liste der Java-Klassen auf das + neben der entsprechenden Klasse, um eine Liste der Methoden anzuzeigen. Wählen Sie eine Methode, und klicken Sie auf Weiterleiten. Die Java-Callout-Aktion wird folgendermaßen auf der Seite Stage bearbeiten angezeigt:
    • <Method> wird durch den Namen der in den Schritten 2 und 3 gewählten Java-Methode ersetzt. Dieser Name bildet den Link zur Seite Klasse und Methode wählen. Klicken Sie auf diesen Link, um die Auswahl der Java-Methoden zu ändern.

      • Die Methode muss eine statische Methode sein.

    • Parameter: Ein <Expression>-Link zur Seite XQuery-Ausdruckseditor wird für jedes von der Java-Methode benötigte Argument angezeigt. Eine Beschriftung für jeden Link zeigt den Datentyp für das Argument an, der einer der folgenden sein kann:
      • java.lang.String
      • Primitive-Typen und ihre entsprechenden Klassentypen (z.B. int versus java.lang.Integer)
      • java.lang.BigDecimal und java.lang.BigInteger (Diese Typen werden in Finanzberechnungen verwendet, wenn keine Abrundungsfehler oder Überläufe toleriert werden)
      • nur org.apache.xbeans.XmlObject und keine typisierten XML-Beans
      • byte[]
      • java.lang.String[] (NUR EINGABE)
      • XmlObject [ ] (NUR EINGABE)
      • javax.activation.DataSource
    • Parameter als Referenz zurückgeben: Mit dieser Option wird der Rückgabewert eines Java-Callout-Aufrufs unabhängig von seinem eigentlichen Typ zu einem <java-content ref="jcid"> Referenzelement, wobei jcid der Schlüssel zum Objekt im Pipeline-Objekt-Repository ist. Geben Sie im Ergebniswertefeld den Namen der Variablen ein, die die Java-Contentreferenz enthalten soll. Diese Option ermöglicht Ihnen zusätzlich zur Pipeline XML die Arbeit mit einem referenzierten Objekt im Nachrichtenfluss, um Passthrough bereitzustellen, Nachrichten mit Java-Callout- und Inline-Aktionen anzureichern oder Nachrichtentransformationen zwischen Java und Nicht-Java-Transporten durchzuführen. Weitere Informationen finden Sie unter "Sending and Receiving Java Objects in Messages" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
    • Ergebnis: Feld Ergebnis, in das Sie die Variable eingeben, der das Ergebnis zugeordnet werden soll. Die Beschriftung für das Feld gibt den Datentyp des Ergebnisses an.
      • Hinweis: Wenn das Ergebnis ein Byte-Array ist (das einzige mögliche zurückgegebene Array), wird das XML-Element mit Binär-Inhalt zurückgegeben.
    • Service-Account anhängen: Ein Service-Account-Link ermöglicht Ihnen die Angabe eines optionalen Service-Accounts, wenn ein Sicherheitskontext für diese Java-Methode vorliegt. Weitere Informationen über Sicherheitskontexte und Service-Accounts finden Sie unter Service-Accounts.

      • Im Falle von festen und zugeordneten Service-Accounts werden Benutzerkennung/Kennwort vom Service-Account im lokalen System authentifiziert, und der Sicherheitskontext wird im Java-Callout propagiert. Im Falles eines Passthrough wird der Sicherheitskontext im Java-Callout propagiert. Dieser Kontext entspricht dem Kontext auf Nachrichtenebene, sofern dieser definiert wurde (mit WS-Sicherheit). Falls nicht, entspricht er dem Kontext auf der Transportebene.

  5. Klicken Sie unter Parameter auf <Expression>. Die Seite XQuery-Ausdruckseditor wird angezeigt. Mit dem XQuery-Ausdruckseditor können Sie die von der Java-Methode benötigten Argumente angeben. Siehe Inline-XQuery- und XPath-Ausdrücke erstellen und bearbeiten.

    6. Wenn der Typ des Eingabewerts nicht dem deklarierten Typ des Eingabearguments entspricht, versucht Oracle Service Bus, die Eingabewerte automatisch in den deklarierten Typ des Eingabearguments zu konvertieren. Beispiel: Die Zeichenfolge "123" wird in die Ganzzahl 123 konvertiert, wenn der deklarierte Typ des Eingabearguments dem Java-Primitive int entspricht.

  6. Weisen Sie im Feld Ergebnis eine Variable für das von der Java-Methode zurückgegebene Ergebnis zu.
  7. Wenn ein Sicherheitskontext für die Java-Methode vorhanden ist, aktivieren Sie das Kontrollkästchen, und klicken Sie auf <Service-Account>. Die Seite Service-Account wählen wird angezeigt. Wählen Sie das erforderliche Service-Account aus der Liste, und klicken Sie auf Weiterleiten.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
