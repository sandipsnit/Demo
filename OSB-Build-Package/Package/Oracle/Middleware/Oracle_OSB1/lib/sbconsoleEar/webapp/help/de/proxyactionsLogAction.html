Log-Aktionen hinzufügen

Mit der Log-Aktion können Sie eine zu protokollierende Nachricht erstellen und eine Gruppe von Attributen definieren, mit der die Nachricht protokolliert wird.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

Hinweis: Um Log-Daten in der Log-Datei oder im Standardausgang (Serverkonsole) anzuzeigen, muss die Anmeldung bei Oracle WebLogic Server auf die folgenden Dringlichkeitsstufen eingestellt werden:

Mindestdringlichkeit für das Log: Info
Log-Datei: Info
Standardausgang: Info

Weitere Informationen zum Einstellen der Dringlichkeitsstufen für das Log finden Sie im entsprechenden Thema unter Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

 

So fügen Sie eine Log-Aktion hinzu:

  1. Stellen Sie sicher, dass das Logging global aktiviert ist. Weitere Informationen finden Sie unter Logging global aktivieren.
  2. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  3. Klicken Sie auf das entsprechende Symbol, und wählen Sie Aktion hinzufügen > Reporting > Log.
  4. Klicken Sie auf <Expression>. Die Seite XQuery-Ausdruckseditor wird angezeigt. Sie geben den Nachrichtenkontext an, der über XQuery-Ausdrücke in Kontextvariablen protokolliert werden soll. Weitere Informationen finden Sie unter Inline XQuery- und XPath-Ausdrücke erstellen und bearbeiten.
  5. Geben Sie im Feld Anmerkung Hinweise für diese Log-Aktion ein. Diese Hinweise werden zusammen mit dem Ergebnis des zuvor definierten Ausdrucks protokolliert.
  6. Wählen Sie eine der folgenden Optionen aus der Dropdown-Liste Schweregrad.

    7. Tabelle 21-5 Schweregrade für Log-Aktionen
    Schweregrad
    Allgemeine Verwendung
    Info
    Wird für das Erstellen von Berichten zu normalen Vorgängen verwendet; Informationsnachricht der unteren Ebene.
    Warnung
    Ein verdächtiger Vorgang oder eine verdächtige Konfiguration ist aufgetreten, die den normalen Vorgang jedoch nicht beeinflusst.
    Fehler
    Ein Benutzerfehler ist aufgetreten. Das System oder die Anwendung kann den Fehler ohne Unterbrechung und einer geringen Verschlechterung des Service verarbeiten.
    Debuggen
    Solange sich Ihre Anwendung in der Entwicklung befindet, ist für Sie möglicherweise das Erstellen und Verwenden von Nachrichten hilfreich, die verbose Beschreibungen von Aktivitäten der unteren Ebene innerhalb der Anwendung enthalten.
    Stellen Sie sicher, dass die Log-Dringlichkeitsstufe in den Vorgangseinstellungen des Proxy-Service mit der Dringlichkeitsstufe der Log-Aktion übereinstimmt. Informationen zu den Betriebseinstellungen des Proxy-Service finden Sie unter Vorgangseinstellungen für Proxy-Services konfigurieren.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
