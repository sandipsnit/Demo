Publishing-Tabellenaktionen hinzufügen

Mit der Aktion für die Publishing-Tabelle können Sie eine Nachricht für null oder mehr statisch angegebene Services veröffentlichen. Anhand der Switch-artigen Bedingungslogik wird zur Laufzeit bestimmt, welche Services für das Publishing verwendet werden.

Weitere Informationen zum Veröffentlichungsverhalten finden Sie unter "Performing Transformations in Message Flows" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBAG205.

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Wählen Sie Aktion hinzufügen > Kommunikation > Publishing-Tabelle.
  3. Klicken Sie auf <Expression>. Die Seite XQuery-Ausdruckseditor wird angezeigt. Erstellen Sie einen XQuery-Ausdruck, der zur Laufzeit den Wert zurückgibt, anhand dessen die Routing-Entscheidung getroffen wird. Weitere Informationen finden Sie unter Inline XQuery- und XPath-Ausdrücke erstellen und bearbeiten.
  4. Wählen Sie einen Vergleichsoperator aus der Liste Operator. Geben Sie dann im angrenzenden Feld einen Wert ein, mit dem der vom XQuery-Ausdruck zurückgegebene Wert verglichen wird.
  5. Klicken Sie auf <Service>, um einen Service auszuwählen, für den Nachrichten veröffentlicht werden sollen, wenn die Auswertung des Ausdrucks für den angegebenen Wert "true" ergibt. Die Seite Service wählen wird angezeigt.
  6. Wählen Sie einen Service aus der Liste, und klicken Sie dann auf Weiterleiten. Dies ist der Ziel-Service für die Nachricht.
  7. Wenn für den Service Vorgänge definiert sind, können Sie den Vorgang angeben, der aufgerufen werden soll, indem Sie ihn aus der Liste Aufrufen wählen.
  8. Wenn Ausgangs- und Eingangsvorgang identisch sein sollen, aktivieren Sie das Kontrollkästchen Eingangsvorgang für Ausgang verwenden.
  9. Um zu konfigurieren, wie die Nachricht gepackt und an den Service gesendet werden soll, klicken Sie im Feld Anforderungsaktionen auf Aktion hinzufügen. Wählen Sie anschließend eine oder mehrere Aktionen, die Sie mit dem Service verknüpfen möchten. Weitere Informationen zu dem Typ der hinzuzufügenden Aktionen finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
    10. Hinweis: Die Verschachtelung ist im Stage Editor auf vier kumulative Ebenen begrenzt. Wenn Sie versuchen, eine fünfte Ebene hinzuzufügen, wird diese Verschachtelungsaktion nicht angezeigt. Kumulative Ebenen umfassen alle Verzweigungsaktionen: If... Then...-Bedingungen, Publishing-Tabellen und Route-Tabellen. Beispiel: Wenn Sie zwei Ebenen mit Konditionalen haben und dann eine Tabelle mit einer Route-Tabelle darin veröffentlichen, haben Sie insgesamt vier Ebenen. Wenn Sie dann versuchen, einen weiteren Konditional (zur letzten Publishing-Tabelle) hinzuzufügen, wird dieser Konditional nicht angezeigt.
  10. Um einen neuen Fall einzufügen, klicken Sie auf das Symbol Fall und wählen die Option Neuen Fall einfügen.
  11. Wiederholen Sie die Schritte 4 bis 8 für den neuen Fall.
  12. Fügen Sie zusätzliche Fälle entsprechend Ihrer Geschäftslogik hinzu.
  13. Klicken Sie für den letzten Fall, den Sie in Folge definieren, auf das Symbol Fall. Um am Ende einen Standardfall hinzuzufügen, wählen Sie Standardfall einfügen.
  14. Konfigurieren Sie den Standardfall. In der Konfiguration dieses Falles wird das Routing-Verhalten definiert, falls keiner der vorhergehenden Fälle erfüllt wird.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
