Antwortaktionen hinzufügen

Mit der Antwortaktion geben Sie an, dass eine sofortige Antwort an den Aufrufenden gesendet werden soll.

Die Antwortaktion kann in der Anforderungs-, Antwort- oder Fehler-Pipeline verwendet werden. Sie können die Aktion so konfigurieren, dass die Antwort als erfolgreich oder nicht erfolgreich zurückgegeben wird. Bei einer nicht erfolgreichen Antwort mit eingehendem HTTP-Transport gibt die Antwortaktion an, dass eine sofortige Antwort an den Aufrufenden gesendet wird.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

So fügen Sie eine Antwortaktion hinzu

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, und wählen Sie Aktion hinzufügen > Flusssteuerung > Antworten.
  3. Wählen Sie Erfolgreich, um zu antworten, dass die Nachricht erfolgreich war, oder wählen Sie Nicht erfolgreich, um zu antworten, dass die Nachricht einen Fehler enthielt.

    4. Eine nicht erfolgreiche Antwort bricht eine von Oracle Service Bus gestartet Transaktion ab.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
