Berichtsaktionen hinzufügen

Verwenden Sie die Berichtsaktion zum Erstellen von Nachrichtenberichten für einen Proxy-Service.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

So fügen Sie eine Berichtsaktion hinzu

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, und wählen Sie dann Aktion hinzufügen > Reporting > Bericht.
  3. Klicken Sie auf <Expression>. Die Seite XQuery-Ausdruckseditor wird angezeigt. Weitere Informationen finden Sie unter Inline XQuery- und XPath-Ausdrücke erstellen und bearbeiten. Der XQuery-Ausdruck wird zum Erstellen der Daten verwendet, die an das Oracle Service Bus-Dashboard gemeldet werden.
  4. Wenn Sie mit dem Bearbeiten des XQuery-Ausdrucks fertig sind, klicken Sie auf Schlüssel hinzufügen. Daraufhin werden zwei Felder angezeigt: das Feld Schlüsselname und das Feld Schlüsselwert, das den XPath-Link enthält, auf den Sie klicken können, um einen XPath-Ausdruck zu bearbeiten. Außerdem wird das Feld "In Variable" angezeigt, in dem Sie eine Kontextvariable eingeben können.

    5. Mithilfe von Schlüsselwertpaaren können Sie aus einer beliebigen Nachrichtenkontextvariablen oder Nachrichten-Payload Schlüssel-IDs extrahieren und den Rest der Nachricht ignorieren. Dank der Schlüssel können Sie Nachrichten ganz einfach identifizieren. Sie werden im Reporting-Modul als Berichtsindexe angezeigt. Weitere Informationen finden Sie in Berichtmeldungen anzeigen und in Nachrichtendetails anzeigen.

    1. Geben Sie einen Schlüsselnamen in das Feld Schlüsselname ein.
    2. Klicken Sie auf <XPath>. Die Seite XPath-Ausdruck bearbeiten wird angezeigt. Weitere Informationen finden Sie unter Inline XQuery- und XPath-Ausdrücke erstellen und bearbeiten.
    3. Geben Sie eine Kontextvariable in das Feld Variable ein. Weitere Informationen zu Kontextvariablen finden Sie in Anhang A, Nachrichtenkontext.
    4. Wenn Sie weitere Schlüsselwerte hinzufügen möchten, klicken Sie auf das Symbol Schlüssel und anschließend auf Schlüssel hinzufügen. Wenn Sie einen Schlüssel entfernen möchten, klicken Sie auf das Symbol Schlüssel und anschließend auf Diesen Schlüssel löschen.

Beispiel: Stellen Sie sich eine Berichtsaktion vor, die auf einem Error Handler in einer Phase konfiguriert wurde. Die Aktion meldet den Content der Kontextvariablen fault im Fall eines Fehlers. Die Berichtsaktion wird wie folgt konfiguriert:

  • Schlüsselname = errorCode
  • Schlüsselwert = ./ctx:errorCode in Variable fault

Jedes Mal, wenn diese Aktion zur Laufzeit ausgeführt wird, wird eine Nachricht über den Reporting-Datenfluss gemeldet. In der folgenden Tabelle sind die Ergebnisse nach zweifacher Ausführung der Berichtsaktion dargestellt.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
