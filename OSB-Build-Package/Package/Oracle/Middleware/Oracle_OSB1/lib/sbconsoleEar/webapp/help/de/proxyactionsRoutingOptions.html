Routing-Optionsaktionen hinzufügen

Mit der Aktion "Routing-Optionen" können Sie eine oder alle der folgenden Eigenschaften für die ausgehende Anforderung in der Variablen $outbound ändern: die Parameter "URI", "Quality of Service", "Modus" und "Wiederholen". Obwohl diese Eigenschaften mit den Aktionen "Zuweisen", "Einfügen", "Ersetzen" oder "Löschen" in der Variablen $outbound geändert werden können, kann diese Task mit der Aktion "Routing-Optionen" einfacher ausgeführt werden. Dabei sind keine Kenntnisse von XPath, XQuery oder der Struktur der Kontextvariablen $outbound erforderlich.

Die Aktion "Routing-Optionen" kann nur verwendet werden, wenn die Kontextvariable $outbound gültig ist. Sie kann den folgenden Aktionen hinzugefügt werden:

  • Veröffentlichen
  • Dynamisches Publishing
  • Publishing-Tabelle
  • Service Callout
  • Routing
  • Dynamisches Routing
  • Routing-Tabelle

Weitere Informationen zum Routing finden Sie unter "Modeling Message Flow in Oracle Service Bus" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBAG181.

 

So konfigurieren Sie die Aktion "Routing-Optionen":

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, und wählen Sie Aktion hinzufügen > Kommunikation > Routing-Optionen.
  3. Führen Sie die folgenden Schritte aus:
    • So legen Sie den URI für die ausgehende Nachricht fest: Wählen Sie URI, und klicken Sie auf XQuery-Ausdruckseditor. Geben Sie einen Ausdruck ein, der einen URI zurückgibt. Dadurch wird der URI für den aufgerufenen Service außer Kraft gesetzt.
    • So legen Sie das Quality of Service-Element fest: Wählen Sie Quality of Service, und wählen Sie die Option "Quality of Service" aus der Dropdown-Liste. Dadurch wird der automatisch berechnete Standardwert außer Kraft gesetzt.
    • So legen Sie den Modus fest: Wählen Sie Modus, und wählen Sie in der Dropdown-Liste "Anforderung" oder "Anforderung+Antwort".
      • Hinweis: Der Modus wird normalerweise automatisch festgelegt, je nach dem Interface des aufgerufenen Service. Dies trifft jedoch in einigen Fällen wie bei beliebigen SOAP- oder beliebigen XML-Services nicht zu.
    • So legen Sie das Wiederholungsintervall fest: Wählen Sie Wiederholungsintervall, und geben Sie die Anzahl der Sekunden zwischen den Wiederholungen an. Dadurch wird der Standardwert außer Kraft gesetzt, der für den aufgerufenen Service konfiguriert wurde.
    • So legen Sie die Anzahl der Wiederholungen fest: Wählen Sie Anzahl Wiederholungen, und geben Sie die Anzahl der Wiederholungen an, die das System ausführen muss, bevor die Aktion abgebrochen wird. Dadurch wird der Standardwert außer Kraft gesetzt, der für den aufgerufenen Service konfiguriert wurde.
    • So legen Sie die Nachrichtenpriorität fest: Wählen Sie Priorität, und klicken Sie auf XQuery-Ausdruckseditor. Geben Sie einen Ausdruck ein, der eine positive Ganzzahl zurückgibt.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
