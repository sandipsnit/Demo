Service Callout-Aktionen hinzufügen

Mit einer Service Callout-Aktion können Sie einen synchronen (blockierenden) Callout für einen Oracle Service Bus-registrierten Proxy oder Business-Service konfigurieren.

Weitere Informationen zu Aktionen zum Serviceaufruf finden Sie unter "Constructing Service Callout Messages" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBAG208.

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, und wählen Sie dann Aktion hinzufügen > Kommunikation > Service Callout.
  3. Klicken Sie auf <Service>. Der Service-Browser wird angezeigt.
  4. Wählen Sie aus der Liste der registrierten Proxy oder Business-Services einen Service aus, und klicken Sie auf Weiterleiten.
  5. Wenn Sie in Schritt 3 weiter oben einen Service auf WSDL-Basis ausgewählt haben, dessen Vorgänge auf den Service angewendet werden können, werden diese Vorgänge in der Dropdown-Liste Aufruf <Vorgang angezeigt. Wählen Sie einen für den Service aufzurufenden Vorgang aus.

    Hinweis: Wenn Sie, aus welchen Gründen auch immer, einen Vorgang für Oracle Service Bus aufrufen, ist nicht sicher, dass nur der ausgewählte Vorgang aufgerufen wird. Wenn Sie beispielsweise VorgangA aufrufen, eine Nachricht jedoch einen Aufruf von Vorgang B enthält, wird VorgangB ebenfalls aufgerufen.
  6. Geben Sie an, wie Sie die Anforderungs- und Antwortnachrichten konfigurieren möchten, indem Sie eine der folgenden Optionen wählen:
    • Wählen Sie SOAP-Body konfigurieren, um den SOAP-Body zu konfigurieren. Wenn Sie diese Option aktivieren, können Sie $body direkt verwenden.
      • Hinweis: Diese Option unterstützt SOAP-RPC-Codierung, was beim Konfigurieren von Payload-Parametern oder -Dokumenten nicht der Fall ist.
    • Wählen Sie Payload-Parameter konfigurierenoder Payload-Dokument konfigurieren, um die Payload zu konfigurieren.
  7. Nachfolgende Konfigurationsoptionen hängen vom Service-Typ ab, den Sie in Schritt 3 gewählt haben, sowie von der Art der Konfigurationsoptionen, die Sie in Schritt 5 für diesen Service festgelegt haben. In Tabelle 21-3 werden die Optionen für jeden einzelnen Service-Typ angezeigt.

    8. Tabelle 21-4 enthält Anweisungen für jede der in Tabelle 21-3 aufgeführten Optionen.

    Tabelle 21-4 Service Callout-Konfigurationsoptionen 
    Für diese Optionen...
    Führen Sie diese Schritte aus ...
    "SOAP-Anforderungs-Body" und
    "SOAP-Antwort-Body"
    So konfigurieren Sie diese Optionen
    • Geben Sie im Feld SOAP-Anforderungs-Body den Namen einer Variable ein, die die XML-Datei des SOAP-Body-Elements für die Callout-Anforderung enthält.
    • Geben Sie in das Feld SOAP-Antwort-Body den Namen einer Variablen ein, an die die XML-Datei des SOAP-Body-Elements für die Antwort gebunden wird.
    "SOAP-Anforderungs-Header" und
    "SOAP-Antwort-Header"
    So konfigurieren Sie diese Optionen
    • Geben Sie im Feld SOAP-Anforderungs-Header den Namen einer Variablen ein, die die XML-Datei des SOAP-Header-Elements für die Callout-Anforderung enthält.

      • Brechen Sie das Eingabedokument für den SOAP-Anforderungs-Header mit <soap-env:Header>...</soap-env:Header> um.

    • Geben Sie im Feld SOAP-Antwort-Header den Namen einer Variablen ein, an die die XML des SOAP-Headers bei der Antwort (sofern vorhanden) gebunden wird.
    "Anforderungsparameter" und
    "Antwortparameter"
    So konfigurieren Sie diese Optionen
    • Geben Sie in die Felder Anforderungsparameter die Namen der Variablen ein, die während der Laufzeit ausgewertet werden, um Werte für die Anforderungsparameter bereitzustellen.

      • Sie müssen nur die Core-Payload-Dokumente in der Eingabevariablen bereitstellen, das SOAP-Package wird von Oracle Service Bus erstellt. Das heißt, brechen Sie das Eingabedokument nicht mit <soap-env:Body>...</soap-env:Body> um.

      Beispiel: Wenn Sie eine Body-Eingabevariable erstellen, die für diesen Anforderungsparameter verwendet wird, definieren Sie den Content dieser Variablen mit der XPath-Anweisung body/* (um den Wrapper soap-env:Body zu entfernen), nicht mit $body (wodurch der Wrapper soap-env:Body erhalten bleibt).

    • Geben Sie in die Felder Antwortparameter die Namen der Variablen ein, denen die Antworten zur Laufzeit zugewiesen werden.
    "Anforderungsdokument" und
    "Antwortdokument"
    So konfigurieren Sie diese Optionen
    • Geben Sie im Feld Anforderungsdokumentvariable den Namen einer Variablen ein, der ein Anforderungsdokument zugewiesen werden soll.

      • Bei Services vom Typ SOAP-Dokument wird die Variable zur Laufzeit ausgewertet und bildet den Body der an den Service gesendeten SOAP-Nachricht. Bei beliebigenXML-Services, wird die Variable zur Laufzeit ausgewertet und bildet den Body der an den Service gesendeten XML-Nachricht.

      Bei Services vom Typ SOAP-Dokument und bei allen XML-Services müssen Sie nur die Core-Payload-Dokumente in der Eingabevariablen bereitstellen, das SOAP-Package wird von Oracle Service Bus erstellt. Das heißt, brechen Sie das Eingabedokument nicht mit <soap-env:Body>...</soap-env:Body> um.

      Beispiel: Wenn Sie eine Body-Eingabevariable erstellen, die für diesen Anforderungsparameter verwendet wird, definieren Sie den Content dieser Variablen mit der XPath-Anweisung body/* (um den Wrapper soap-env:Body zu entfernen), nicht mit $body (wodurch der Wrapper soap-env:Body erhalten bleibt).

      Bei Messaging Services wird die Variable ausgewertet und bildet den Body der Nachricht, basierend auf dem Datentyp, der vom Service erwartet wird. Für Variablen, die bei Messaging Services verwendet werden, gelten folgende Einschränkungen:

      • Bei Services, die Binärdaten erwarten, muss die Variable ein Element ctx:binary-content besitzen.
      • Bei Services, die MFL-Daten erwarten, muss die Variable das XML-Äquivalent besitzen.
      • Bei Services, die Textdaten erwarten, ist die Variable eine Zeichenfolge.
    • Geben Sie im Feld Antwortdokumentvariable den Namen der Variablen ein, denen das Antwortdokument zur Laufzeit zugewiesen wird.

  8. Sie können optional einen oder mehrere Transport-Header hinzufügen. Weitere Informationen finden Sie unter Transport-Header-Aktionen hinzufügen.
    9. Hinweis: Zusätzlich zu den angegebenen Transport-Headern werden auch von der Oracle Service Bus-Binding Layer Header hinzugefügt. Weitere Informationen finden Sie unter "Configuring Transport Headers in Message Flows" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.
