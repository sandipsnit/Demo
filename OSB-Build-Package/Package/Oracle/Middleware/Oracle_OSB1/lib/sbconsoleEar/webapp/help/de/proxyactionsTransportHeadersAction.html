Transport-Header-Aktionen hinzufügen

Mit einer Transport-Header-Aktion können Sie die Header-Werte in Nachrichten festlegen.

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das entsprechende Symbol, und wählen Sie Aktion hinzufügen > Kommunikation > Transport-Header.
  3. Wählen Sie in der Liste Transport-Header einrichten für eine der folgenden Optionen, um zur Laufzeit festzulegen, welche der Speicherorte für den Nachrichtenkontext geändert werden sollen:
    • Ausgehende Anforderung - Mit dieser Option können Sie Header-Werte für ausgehende Anforderungen (die Nachrichten, die von einem Proxy-Service in Route-, Publishing- oder Service Callout-Aktionen ausgesendet werden) festlegen. Dieses Header-Element befindet sich im Nachrichtenkontext an folgender Stelle:

      • $outbound/ctx:transport/ctx:request/tp:headers

    • Eingehende Antwort - Mit dieser Option können Sie Header-Werte für eingehende Antworten (die Antwortnachrichten, die ein Proxy-Service an Clients zurücksendet) festlegen. Dieses Header-Element befindet sich im Nachrichtenkontext an folgender Stelle:

      • $inbound/ctx:transport/ctx:response/tp:headers

  4. Optional können Sie Alle Header durch Pipeline übergeben wählen, um alle Header aus der eingehenden Nachricht an die ausgehende Nachricht zu übergeben oder umgekehrt. Jeder Header in der Quell-Header-Gruppe wird in die Ziel-Header-Gruppe kopiert, wobei alle vorhanden Werte in der Ziel-Header-Gruppe außer Kraft gesetzt werden.

    5. Weitere Informationen zum Verwenden dieser Option in Verbindung mit der Header-spezifischen Übergeben-Option finden Sie unter "Configuring Transport Headers in Message Flows" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

  5. Führen Sie für jeden Header, den Sie hinzufügen möchten, die folgenden Schritte aus:
    1. Klicken Sie in der Tabelle Transport-Header auf Header hinzufügen, um Felder für das Konfigurieren des Headers anzuzeigen.
    2. Um einen Header anzugeben, führen Sie einen der folgenden Schritte aus:
      • Wählen Sie in der Spalte Name einen Header-Namen aus der Dropdown-Liste. Die Liste enthält alle vordefinierten Header-Namen für den Zieltransport (Beispiel: Content-Type für HTTP-Transporte, JMSCorrelationID für JMS-Transporte und so weiter).
      • Geben Sie im Feld Andere einen Header-Namen ein. Falls dieser Header-Name nicht zu den vordefinierten Headern für den Transport dieses Service gehört, wird daraus gemäß der Definition der Transportspezifikation ein Benutzer-Header.
    3. Wählen Sie eine der Optionen in der Spalte Aktion, um die Vorgehensweise zum Festlegen des Header-Wertes anzugeben:

      4. Header als Ausdruck festlegen

      Wenn Sie diese Option wählen, können Sie den Wert des Headers mit einem XQuery- oder XSLT-Ausdruck festlegen. Dabei kann es sich um einen einfachen Ausdruck (Beispiel: "text/xml") oder um einen komplexen XQuery- oder XSLT-Ausdruck handeln.

      Da mit der Oracle Service Bus-Transportebene die XML-Darstellung aller Header als Zeichenfolgenwerte definiert wird, werden die Ergebnisse aller Ausdrücke in Zeichenfolgen konvertiert, bevor der Header-Wert festgelegt wird. Bei Ausdrücken ohne Ergebnis wird der Header-Wert auf eine leere Zeichenfolge gesetzt. Sie können einen Header nicht mit einem Ausdruck löschen.

      Achtung: Nicht alle Header-Einstellungen, die Sie in dieser Aktion angeben können, werden zur Laufzeit berücksichtigt. Weitere Informationen dazu, welche Header Sie für einen bestimmten Transport festlegen können und welche dieser Header berücksichtigt werden, finden Sie unter "Configuring Transport Headers in Message Flows" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

      Header löschen

      Gibt an, dass der Header aus den Metadaten der Anforderung oder der Antwort gelöscht wird.

      Header aus eingehender Anforderung kopieren (wenn Sie Transport-Header für die ausgehende Anforderung festlegen)
            oder
      Header aus ausgehender Antwort kopieren (wenn Sie Transport-Header für die eingehende Antwort festlegen)

      Gibt an, dass dieser Header direkt aus dem entsprechenden Header desselben Namens aus der eingehenden Nachricht in die ausgehende Nachricht kopiert wird und umgekehrt. Beispiel: Wenn Sie den SOAPAction-Header für eine ausgehende Anforderung festlegen möchten und Sie die Option Header aus eingehender Anforderung kopieren wählen, wird zur Laufzeit der Wert aus dem SOAPAction-Anforderungs-Header von $inbound kopiert. Bei eingehenden Antwort-Headern ist der Antwort-Header von $outbound die Quelle des zu kopierenden Headers.

      Wenn die Option Header ... kopieren für einen Header gewählt wird, der in der Quelle nicht vorhanden ist, wird diese Option ignoriert, und im Ziel wird für diesen Header keine Aktion ausgeführt.

      Weitere Informationen zum Verwenden dieser Option in Verbindung mit der globalen Option Alle Header durch Pipeline übergeben finden Sie unter "Configuring Transport Headers in Message Flows" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

  6. Um weitere Header zur Tabelle hinzuzufügen, klicken Sie auf das Symbol Header, und klicken Sie anschließend auf Header hinzufügen.

    7. Die Tabelle wird um eine zusätzliche Zeile erweitert, die eine neue Gruppe von Optionen enthält, mit denen Sie einen weiteren Transport-Header konfigurieren können. Sie können dieser Tabelle so viele Header wie nötig hinzufügen. Sie müssen die Header in der Tabelle nicht sortieren, weil die Laufzeit beim Generieren der entsprechenden XML die Namespaces deklariert und die Header-Elemente in die richtige Reihenfolge bringt.

Wenn Sie fertig sind

Wenn Sie die Konfiguration dieser Aktion abgeschlossen haben, konfigurieren Sie weitere Aktionen, oder speichern Sie Ihre Konfiguration, wie unter Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten beschrieben.

 

Cookies in ausgehenden HTTP-Transport-Headern einrichten

In einem Fluss von HTTP-Meldungen werden Cookies wie folgt im Transport-Header eingerichtet:

  • Mit einem komplexen HML-Ausdruck (Standard in Oracle Service Bus)
  • Mit einem Zeichenfolgenausdruck (Option zum Serverstart erforderlich)

Cookies als komplexe XML-Ausdrücke einrichten

Um ein Cookie mit einem komplexen XML-Ausdruck, dem Standardformat in Oracle Service Bus, einzurichten, konfigurieren Sie den Wert des HTTP-Cookie-Headers in der ausgehenden Anforderung mit der folgenden Syntax für Ausdrücke:

<cookie-values xmlns="http://www.bea.com/wli/sb/transports/http">
    <value>{fn:concat("cookie_name", "=", "cookie_value")}</value>
</cookie-values>

Cookie mit einem Zeichenfolgenausdruck einrichten

Um den Cookie-Header in der ausgehenden Anforderung mit einer Zeichenfolge einzurichten, müssen Sie die folgende Option in den startWebLogic-Befehl der Domain einfügen:

-Dcom.bea.osb.http.cookieAsNoComplexElement=true

Nachdem Sie den Server mit dieser Option neu gestartet haben, können Sie einen HTTP-Cookie-Header mit einem Zeichenfolgenausdruck einrichten. Beispiel:
$cookie_name = "cookie_value"
