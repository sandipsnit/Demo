Routing-Tabellen zu Route-Knoten hinzufügen

Eine Routing-Tabelle besteht aus einer Gruppe von Routes, die in eine Switch-artige Bedingungstabelle gewrappt sind. Hierbei handelt es sich um ein kurzfristiges Konstrukt, mit dem unterschiedliche Routes auf Basis der Ergebnisse eines einzigen XQuery-Ausdrucks gewählt werden können.

Die Verschachtelung ist im Stage Editor auf vier kumulative Ebenen begrenzt. Wenn Sie versuchen, eine fünfte Ebene hinzuzufügen, wird diese Verschachtelungsaktion nicht angezeigt. Kumulative Ebenen umfassen alle Verzweigungsaktionen: If... Then...-Bedingungen, Publishing-Tabellen und Route-Tabellen. Beispiel: Wenn Sie zwei Ebenen mit Konditionalen haben und dann eine Tabelle mit einer Route-Tabelle darin veröffentlichen, haben Sie insgesamt vier Ebenen. Wenn Sie dann versuchen, einen weiteren Konditional (zur letzten Publishing-Tabelle) hinzuzufügen, wird dieser Konditional nicht angezeigt.

Weitere Informationen zum Routing finden Sie unter "Modeling Message Flow in Oracle Service Bus" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBAG181.

Bevor Sie beginnen

In diesen Anweisungen wird davon ausgegangen, dass Sie bereits einen Nachrichtenfluss auf der Seite Nachrichtenfluss bearbeiten bearbeiten, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.

So fügen Sie eine Routing-Tabelle zu einem Route-Knoten hinzu:

  1. Navigieren Sie an die Stelle, an der Sie die Aktion hinzufügen möchten. Eine Beschreibung hierzu finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.
  2. Klicken Sie auf das Symbol Route-Knoten und anschließend auf Route bearbeiten. Die Seite Stage-Konfiguration bearbeiten wird angezeigt.
  3. Klicken Sie auf das Symbol Aktion hinzufügen, und wählen Sie Kommunikation > Routing-Tabelle. Die Routing-Tabellenaktion wird angezeigt.
  4. Wählen Sie in der Liste Operator einen Vergleichsoperator, und geben Sie im angrenzenden Feld einen Wertausdruck ein.
  5. Klicken Sie auf <Service>. Die Seite Service wählen wird angezeigt.
  6. Wählen Sie einen Service aus der Liste, und klicken Sie auf Weiterleiten.
  7. Wenn Sie einen Vorgang für den Service aufrufen möchten, wählen Sie einen Vorgang aus der Liste Vorgang.
  8. Wenn Ausgangs- und Eingangsvorgang identisch sein sollen, aktivieren Sie das Kontrollkästchen Eingangsvorgang für Ausgang verwenden.
  9. Klicken Sie zum Hinzufügen einer Aktion im Feld Anforderungsaktionen auf Aktion hinzufügen, und wählen Sie eine Aktion, die Sie mit dem Service verknüpfen möchten. Sie können mehrere Aktionen hinzufügen.
  10. Klicken Sie zum Hinzufügen einer Aktion im Feld Antwortaktionen auf Aktion hinzufügen, und wählen Sie eine Aktion, die Sie mit dem Service verknüpfen möchten. Sie können mehrere Aktionen hinzufügen.

    11. Weitere Informationen zu den Anforderungstypen und Antwortaktionen, die Sie hinzufügen können, finden Sie unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.

  11. Um einen neuen Fall einzufügen, klicken Sie auf das Symbol Fall und wählen die Option Neuen Fall einfügen.
  12. Wiederholen Sie die Schritte 2 bis 7 für den neuen Fall. Sie können auf das Symbol Fall klicken und anschließend Standardfall einfügen wählen, um am Ende einen Standardfall hinzuzufügen, dessen Routes gewählt werden, wenn keiner der vorhergehenden Fälle erfüllt wird.
  13. Klicken Sie auf Speichern.
  14. Setzen Sie die Erstellung des Nachrichtenflusses auf der Seite Nachrichtenfluss bearbeiten fort, wie in Nachrichtenfluss erstellen, bearbeiten und anzeigen erläutert.
  15. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  16. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
