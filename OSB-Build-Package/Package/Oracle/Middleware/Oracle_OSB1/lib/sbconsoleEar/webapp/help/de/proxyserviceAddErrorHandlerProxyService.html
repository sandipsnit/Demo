Proxy-Service-Error-Handler hinzufügen

Bevor Sie beginnen

Bei diesen Anweisungen wird vorausgesetzt, dass Sie bereits einen Nachrichtenfluss auf der Seite "Nachrichtenfluss bearbeiten" bearbeiten. Weitere Informationen finden Sie unter Nachrichtenfluss erstellen, bearbeiten und anzeigen.

So fügen Sie einen Proxy-Service Error Handler hinzu

  1. Klicken Sie auf das Symbol Proxy-Service, und klicken Sie dann auf Service Error Handler hinzufügen. Die Seite Error Handler bearbeiten wird angezeigt.
  2. Klicken Sie auf das Symbol Error Handler und anschließend auf Stage hinzufügen.
  3. Klicken Sie auf das Symbol Stage, und klicken Sie dann auf Stage bearbeiten. Die Seite Stage-Konfiguration bearbeiten wird angezeigt.
  4. Klicken Sie auf Aktion hinzufügen, und wählen Sie die gewünschte Aktion.

    5. Ein Error Handler ist eine Pipeline und wird daher wie andere Pipelines konfiguriert. Beispiel: Mit der Veröffentlichungsaktion senden Sie Fehlernachrichten an andere Services, mit der Zuweisungsaktion ändern Sie Kontextvariablen und so weiter. Siehe Aktionen in Nachrichtenflüssen hinzufügen und bearbeiten. Es bestehen keine Einschränkungen hinsichtlich der Verkettung von Aktionen.

    Drei häufig verwendete Fehleraktionen sind Auslösefehler, Antworten und Wiederaufnehmen.

  5. Sie können auf der Seite Stage-Konfiguration bearbeiten weitere Aktionen hinzufügen und andere Bearbeitungen vornehmen.
  6. Um die Aktualisierungen in der aktuellen Session festzuschreiben, klicken Sie auf der Seite Stage-Konfiguration bearbeiten auf Speichern.
  7. Um die Aktualisierungen in der aktuellen Session festzuschreiben, klicken Sie auf der Seite Error Handler bearbeiten auf Speichern.

Wenn Sie fertig sind

Fahren Sie auf der Seite Nachrichtenfluss bearbeiten mit dem Erstellen des Nachrichtenflusses fort. Weitere Informationen finden Sie unter Nachrichtenfluss erstellen, bearbeiten und anzeigen.
