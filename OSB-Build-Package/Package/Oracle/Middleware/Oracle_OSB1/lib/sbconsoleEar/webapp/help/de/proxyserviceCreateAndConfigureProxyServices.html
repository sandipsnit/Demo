Proxy-Services erstellen und konfigurieren

In diesem Abschnitt wird beschrieben, wie Proxy-Services erstellt und konfiguriert werden.

Für JCA-Services können Sie Proxy-Services aus JCA-Bindings generieren, wie in Proxy-Service aus einer JCA-Binding-Ressource generieren erläutert.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer und dann das Projekt oder den Ordner, zu dem Sie den Proxy-Service hinzufügen möchten.
  3. Auf der Seite Projektansicht/Ordneransicht wählen Sie Proxy-Service aus der Dropdown-Liste Ressource erstellen aus.
  4. Geben Sie auf der Seite Allgemeine Konfiguration einen Namen für den Service ein, und wählen Sie den zu erstellenden Service-Typ.

    Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
  5. Klicken Sie auf Weiter. Die folgenden Seiten sind je nach den auf der ersten Seite angegebenen Optionen unterschiedlich. Geben Sie die entsprechenden Informationen auf den nachfolgenden Seiten ein, bis Sie zur Zusammenfassungsseite gelangen. Um den Service in der aktuellen Session zu speichern, klicken Sie anschließend auf Speichern.

    6. Ausführliche Anweisungen zum Ausfüllen der einzelnen Seiten finden Sie unter:

    • Allgemeine Konfiguration
    • Seite "Nachrichtentyp-Konfiguration"
    • Seite "Transportkonfiguration"
    • Seiten "Protokollspezifische Transportkonfiguration"
    • Seite "Konfiguration der Vorgangsauswahl"
    • Seite "Nachrichtenbehandlungs-Policy"
    • Seite "Zusammenfassung"
    • Seite "Proxy-Service Policy"
    • Seite "Proxy-Service-Sicherheit"
  6. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
  7. Wenn Sie einen Proxy-Service erstellt haben, müssen Sie als Nächstes seinen Nachrichtenfluss konfigurieren. Der Nachrichtenfluss definiert, wie ein Proxy-Service definiert wird. Weitere Informationen hierzu finden Sie unter Nachrichtenfluss erstellen, bearbeiten und anzeigen.

Proxy-Service aus einer JCA-Binding-Ressource generieren

Sie können einen JCA-Proxy-Service aus einer eingehenden JCA-Binding-Ressource in Oracle Service Bus generieren. Weitere Informationen zu JCA-Binding-Ressourcen finden Sie in Kapitel 12, JCA-Bindings.

So generieren Sie einen JCA-Proxy-Service aus einem JCA-Binding:

  1. Klicken Sie in der Oracle Service Administration-Konsole im Change Center auf Erstellen oder Bearbeiten, sofern Sie sich nicht bereits im Erstellungs- oder Bearbeitungsmodus befinden.
  2. Klicken Sie im Ressourcenbrowser auf JCA-Bindings.
  3. Suchen Sie das JCA-Binding, aus dem Sie einen Service generieren möchten, und klicken Sie auf das Symbol Aktion.
  4. Bestätigen Sie im angezeigten Fenster den Namen der WSDL und des zu generierenden Service, wählen Sie den Speicherort für diese neuen Ressourcen aus, und klicken Sie auf Generieren.

    5. In Oracle Service Bus werden der Service und die entsprechende WSDL generiert.

  5. Ändern Sie gegebenenfalls andere Konfigurationsdetails zum generierten Service, beispielsweise den Endpunkt-URI.
