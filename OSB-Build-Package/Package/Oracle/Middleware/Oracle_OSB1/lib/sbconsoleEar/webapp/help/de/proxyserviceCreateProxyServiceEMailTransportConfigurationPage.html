Seite "E-Mail-Transportkonfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - E-Mail-Transportkonfiguration gehört zu einer Reihe von Seiten zum Erstellen und Bearbeiten von Proxy-Services. Diese Aktionen sind in Proxy-Services erstellen und konfigurieren und in Proxy-Service-Konfigurationen bearbeiten beschrieben.

Sie verwenden diese Seite, um Transporteinstellungen für einen Proxy-Service mithilfe des E-Mail-Transportprotokolls zu konfigurieren. Tabelle 19-4 liefert eine Beschreibung zur Verwendung dieser Seite.

Tabelle 19-4 Seite "Proxy-Service erstellen und bearbeiten - E-Mail-Transportkonfiguration" 
Option
Verwendung
SSL erforderlich
Wählen Sie diese Option, um über Secure Sockets Layer zu kommunizieren. Die standardmäßigen SSL-Portnummern lauten:
  • POP3(S): 995
  • IMAP: 993
Hinweis: Das E-Mail-Serverzertifikat muss im Oracle Service Bus Truststore vorhanden sein, damit SSL funktioniert.
Service-Account
Geben Sie einen Service-Accountnamen ein, oder klicken Sie auf Durchsuchen, um Service-Accounts im Browser zu suchen.
Managed Server
Dieses Feld ist nur in einer geclusterten Domain verfügbar.
Wählen Sie den Managed Server, der als Polling-Server dienen soll. Alle Managed Server können die Nachricht verarbeiten, aber nur ein Server kann einen Abruf für die Nachricht ausführen.
Abrufintervall
Geben Sie ein Abrufintervall (in Sekunden) ein.
E-Mail-Protokoll
Wählen Sie POP3 oder IMAP als Servertyp für den E-Mail-Account.
Lesegrenzwert
Geben Sie die maximale Anzahl Nachrichten ein, die pro Polling Sweep gelesen werden sollen. Die Eingabe von 0 bedeutet kein Grenzwert.
Nach Referenz übergeben
Aktivieren Sie dieses Kontrollkästchen, um die Datei im Archivierungsverzeichnis bereitzustellen und sie als Referenz im Header zu übergeben.
Wenn Sie einen neuen Service erstellen, ist die Option Nach Referenz übergeben standardmäßig aktiviert, und Sie müssen den Speicherort für das Archivierungsverzeichnis angeben.
Anhänge nach Referenz übergeben
Aktivieren Sie dieses Kontrollkästchen, um den Anhang im Archivierungsverzeichnis bereitzustellen und ihn als Referenz im Header zu übergeben.
Wenn die Option Nach Referenz übergeben aktiviert ist, gilt standardmäßig und implizit für die Option Anhänge nach Referenz übergeben der Status "true", und Sie müssen den Speicherort für das Archivierungsverzeichnis angeben.
Aktion nach dem Lesen
Wählen Sie, was mit einer Nachricht nach dem Lesen geschehen soll:
  • Archivieren - Die Nachricht wird archiviert.
  • Löschen - Die Nachricht wird gelöscht.
  • Verschieben - Die Nachricht wird verschoben. Verschieben ist nur zusammen mit dem IMAP-Protokoll verfügbar.
Anhänge
Wählen Sie, wie Anhänge behandelt werden sollen:
  • Archivieren - Die Anhänge werden im Archivierungsverzeichnis gespeichert.
  • Ignorieren - Die Anhänge werden ignoriert.
IMAP-Zielordner
Geben Sie den Ordner an, in den die Nachricht verschoben wird, wenn das Feld Aktion nach dem Lesen auf Verschieben gesetzt ist.
Downloadverzeichnis
Geben Sie einen temporären Speicherort für den Download von E-Mails an.
Archivierungsverzeichnis
Geben Sie den Pfad zum Archivierungsspeicherort an, wenn das Feld Aktion nach dem Lesen auf Archivieren gesetzt ist.
Dieses Feld ist obligatorisch, wenn die Option Nach Referenz übergeben oder Anhänge nach Referenz übergeben gewählt wurde.
Fehlerverzeichnis
Geben Sie den Verzeichnispfad des Dateisystems an, in den die Nachricht und eventuelle Anhänge geschrieben werden sollen, falls Probleme auftreten.
Anforderungscodierung
Akzeptieren Sie den Standard ISO-8859-1 als Zeichensatz für die Codierung von Anforderungen in E-Mail-Transporten, oder geben Sie eine andere Zeichensatzcodierung ein.

Wenn Sie fertig sind

Klicken Sie auf Weiter, um mit der Konfiguration dieses Service auf der nächsten Seite fortzufahren. Oder klicken Sie auf Letzte, um diese Konfiguration auf der Seite Zusammenfassung unter Proxy-Service erstellen/bearbeiten zu prüfen und zu speichern.
