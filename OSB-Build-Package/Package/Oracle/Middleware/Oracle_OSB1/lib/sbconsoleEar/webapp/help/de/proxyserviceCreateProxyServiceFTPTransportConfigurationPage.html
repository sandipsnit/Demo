Seite "FTP-Transportkonfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - FTP-Transportkonfiguration ist eine Seite von mehreren, auf denen Proxy-Services erstellt und bearbeitet werden können, wie in Proxy-Services erstellen und konfigurieren und Proxy-Service-Konfigurationen bearbeiten beschrieben.

Auf dieser Seite können Sie die Transporteinstellungen für einen Proxy-Service konfigurieren, der das FTP-Transportprotokoll verwendet. In Tabelle 19-6 wird die Verwendung dieser Seite beschrieben.

Tabelle 19-6 Proxy-Service erstellen/bearbeiten - Seite "FTP-Transportkonfiguration"
Option
Verwendung
Benutzer-Authentifizierung
Wenn der Benutzer des FTP-Servers anonym ist, wählen Sie Anonym. Wenn es sich beim Benutzer des FTP-Servers um einen extern konfigurierten Account handelt, wählen Sie Externer Benutzer.
Identität (E-Mail-ID)
Dieses Feld ist nur dann verfügbar, wenn die Option Benutzer-Authentifizierung auf Anonym gesetzt wurde.
Geben Sie die E-Mail-ID des anonymen Benutzers ein.
Service-Account
Dieses Feld ist nur dann verfügbar, wenn die Option Benutzer-Authentifizierung auf Externer Benutzer gesetzt wurde.
Geben Sie den Service-Account für den Benutzer ein. In dieses Feld muss ein Wert eingegeben werden, wenn die Option Benutzerauthentifizierung auf Externer Benutzer gesetzt ist.
Nach Referenz übergeben
Aktivieren Sie dieses Kontrollkästchen, um die Datei im Archivierungsverzeichnis bereitzustellen und sie als Referenz im Header zu übergeben.
Remote Streaming
Aktivieren Sie dieses Kontrollkästchen, um die FTP-Dateien zum Zeitpunkt der Verarbeitung direkt vom Remote-Server zu streamen. Wenn Sie diese Option aktivieren, entspricht das Archivierungsverzeichnis dem Remote-Verzeichnis auf dem Rechner des Remote-FTP-Servers. Daher sollten Sie das Archivierungsverzeichnis relativ zum FTP-Benutzerverzeichnis angeben.
Dateimaske
Geben Sie den regulären Ausdruck für die auszuwählenden Dateien ein. Der Standardwert ist *.*.
Managed Server
Dieses Feld ist nur in einer geclusterten Domain verfügbar.
Wählen Sie den Managed Server, der als Polling-Server dienen soll. Alle Managed Server können die Nachricht verarbeiten, aber nur ein Server kann einen Abruf für die Nachricht ausführen.
Abrufintervall
Geben Sie das Abrufintervall in Sekunden an. Der Standardwert ist 60.
Lesegrenzwert
Geben Sie an, wie viele Nachrichten maximal pro Abruf-Sweep gelesen werden sollen. Geben Sie 0 ein, um keine Begrenzung anzugeben. Der Standardwert ist 10.
Aktion nach dem Lesen
Wählen Sie aus, was passieren soll, wenn die Nachricht gelesen wurde.
  • Archivieren - Die Nachricht wird archiviert.
  • Löschen - Die Nachricht wird gelöscht.
Transfermodus
Wählen Sie für den Transfermodus ASCII oder Binär aus.
Archivierungsverzeichnis
Geben Sie den Pfad zum Archivierungsverzeichnis an, wenn die Option Post Read-Aktion auf Archivieren gesetzt ist. In dieses Feld muss ein Wert eingegeben werden, wenn die Option Nach Referenz übergeben aktiviert ist.
Hinweis: Die Verzeichnisse "Archivieren", "Herunterladen" und "Fehler" sind absolute Pfade, und sie werden automatisch erstellt. Wenn Sie einen relativen Pfad angeben, werden die Dateien relativ zum Java-Prozess erstellt, der den WebLogic Server startet.
Downloadverzeichnis
Geben Sie das Verzeichnis auf Ihrem lokalen Computer an, auf den die Dateien beim Dateitransfer heruntergeladen werden.
Hinweis: Die Verzeichnisse "Archivieren", "Herunterladen" und "Fehler" sind absolute Pfade, und sie werden automatisch erstellt. Wenn Sie einen relativen Pfad angeben, werden die Dateien relativ zum Java-Prozess erstellt, der den WebLogic Server startet.
Fehlerverzeichnis
Geben Sie an, wohin die Dateien im Falle eines Problems verschickt werden sollen.
Hinweis: Die Verzeichnisse "Archivieren", "Herunterladen" und "Fehler" sind absolute Pfade, und sie werden automatisch erstellt. Wenn Sie einen relativen Pfad angeben, werden die Dateien relativ zum Java-Prozess erstellt, der den WebLogic Server startet.
Anforderungscodierung
Übernehmen Sie den Standardwert UTF-8 als die Zeichensatzcodierung für Anforderungen in FTP-Transporten.
Unterverzeichnisse scannen
Aktivieren Sie dieses Kontrollkästchen, wenn alle Verzeichnisse rekursiv durchsucht werden sollen.
Nach Eingang sortieren
Aktivieren Sie dieses Kontrollkästchen, wenn die Ereignisse in der Reihenfolge ihres Auftretens geliefert werden sollen.
Timeout
Geben Sie das Socket-Timeout-Intervall in Sekunden an, bevor die Verbindung unterbrochen wird. Wenn Sie 0 eingeben, gibt es keinen Timeout.
Anzahl Wiederholungen
Geben Sie die Anzahl der Wiederholungsversuche an, falls die FTP-Verbindung nicht erfolgreich ist.

Wenn Sie fertig sind

Klicken Sie auf Weiter, um mit der Konfiguration dieses Service auf der nächsten Seite fortzufahren. Oder klicken Sie auf Letzte, um diese Konfiguration auf der Seite Zusammenfassung unter Proxy-Service erstellen/bearbeiten zu prüfen und zu speichern.
