Seite "Dateitransportkonfiguration"

Die Seite Proxy-Service erstellen/bearbeiten – Dateitransportkonfiguration ist eine von diversen Seiten, auf denen Sie Proxy-Services erstellen und bearbeiten können. Diese Funktionen sind unter Proxy-Services erstellen und konfigurieren und Proxy-Service-Konfigurationen bearbeiten genauer beschrieben.

Auf dieser Seite können Sie Transporteinstellungen für Proxy-Services mit Dateitransportprotokoll konfigurieren. In Tabelle 19-5 ist die Verwendung dieser Seite beschrieben.

Tabelle 19-5 Seite "Proxy-Service erstellen/bearbeiten – Dateitransportkonfiguration 
Option
Verwendung
Dateimaske
Gibt die Dateien an, die vom Proxy-Service abgefragt werden sollen. Wenn der URI ein Verzeichnis ist und *.* angegeben wurde, dann fragt der Service alle Dateien im Verzeichnis ab. Nur die Platzhalterzeichen * und ? sind in der Dateimaske zulässig. Reguläre Ausdrücke werden nicht unterstützt.
Managed Server
Dieses Feld ist nur in einer geclusterten Domain verfügbar.
Wählen Sie den Managed Server, der als Polling-Server dienen soll. Alle Managed Server können die Nachricht verarbeiten, aber nur ein Server kann einen Abruf für die Nachricht ausführen.
Abrufintervall
Geben Sie das Abrufintervall in Sekunden an. Der Standardwert ist 60.
Lesegrenzwert
Geben Sie die maximale Anzahl der zu lesenden Nachrichten pro Abruf ein. Geben Sie 0 ein, wenn Sie die Anzahl nicht einschränken möchten. Der Standardwert ist 10.
Nach Eingang sortieren
Aktivieren Sie dieses Kontrollkästchen, um anzugeben, dass Ereignisse in der Reihenfolge ihres Eingangs zugestellt werden sollen.
Hinweis: Wird diese Option bei einem Proxy-Service aktiviert, der in einer geclusterten Umgebung ausgeführt wird, werden Nachrichten immer an denselben Server gesendet. Das heißt, die serverübergreifende Lastverteilung wird ignoriert, wenn diese Option aktiviert ist.
Unterverzeichnisse scannen
Aktivieren Sie dieses Kontrollkästchen, um rekursiv alle Verzeichnisse zu scannen.
Nach Referenz übergeben
Aktivieren Sie dieses Kontrollkästchen, um die Datei im Archivierungsverzeichnis bereitzustellen und sie als Referenz im Header zu übergeben.
Aktion nach dem Lesen
Wählen Sie, was mit einer Nachricht nach dem Lesen geschehen soll:
  • Archivieren - Die Nachricht wird archiviert.
  • Löschen - Die Nachricht wird gelöscht.
Stage-Verzeichnis
Geben Sie ein vorübergehendes Verzeichnis zum Zwischenspeichern der Dateien während ihrer Verarbeitung ein.
Das Stage-Verzeichnis darf sich nicht im Abrufverzeichnis befinden (das im URL des Dateitransport-Proxy-Services angegebene Verzeichnis, z.B. file:///c:/dir1/dir2).
Archivierungsverzeichnis
Geben Sie den Pfad des Archivierungsverzeichnisses an, wenn die Option Aktion nach dem Lesen auf Archivieren gesetzt wurde. Wenn Sie die Option "Nach Referenz übergeben" aktiviert haben, ist das Feld "Archivierungsverzeichnis" ein Pflichtfeld.
Das Archivierungsverzeichnis darf sich nicht im Abrufverzeichnis befinden.
Fehlerverzeichnis
Geben Sie den Speicherort ein, an den Nachrichten und Anhänge gesendet werden, wenn ein Problem aufgetreten ist.
Das Fehlerverzeichnis darf sich nicht im Abrufverzeichnis befinden.
Anforderungscodierung
Übernehmen Sie die Standardeinstellung UTF-8 als Zeichensatzcodierung für Anforderungen in Dateitransporten, oder geben Sie eine andere Zeichensatzcodierung ein.
