Seite "Allgemeine Konfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - Allgemeine Konfiguration ist eine von einer Serie zum Erstellen und Bearbeiten von Proxy-Services, wie inProxy-Services erstellen und konfigurieren und Proxy-Service-Konfigurationen bearbeiten beschrieben.

Verwenden Sie diese Seite, um allgemeine Konfigurationseinstellungen für einen Proxy-Service zu verändern.

Wenn Sie einen Proxy-Service erstellen, ist dies die erste Seite, die von einer Serie von Seiten zur Konfiguration dieses Service angezeigt wird. Die Seiten die nach dieser angezeigt werden, unterscheiden sich je nachdem, welche Auswahl Sie auf dieser Seite treffen.

Tabelle 19-1 beschreibt, wie diese Seite zu verwenden ist.

Tabelle 19-1 Einen Proxy-Service erstellen/bearbeiten - Seite "Allgemeine Konfiguration"
Option
Verwendung
Service-Name
Geben Sie einen eindeutigen Namen für den Proxy-Service ein.

Befolgen Sie die benennungseinschränkungen für Ressourcen bei der Namensgebung.
Beschreibung
Geben Sie eine Beschreibung ein.
Service-Typ
Wählen Sie den zu erstellenden Proxy-Service:
  • WSDL Web-Service - Wählen Sie diese Option aus, um einen Proxy-Service basierend auf einer WSDL zu erstellen. Geben Sie dann den WSDL-Namen an, wie er aus dem Pfad hervorgeht (zum Beispiel: myProject/myFolder/myWSDL). Oder Sie klicken auf Durchsuchen, um eine WSDL auf der Seite WSDL-Definition wählen auszuwählen.

    • (Port oder Binding) - Geben Sie den Namen eines Ports ein (definiert in der WSDL), um eine tatsächliche Transportadresse zu beschreiben, oder geben Sie den Namen eines Bindings ein (definiert in der WSDL), um es einer Transportadresse zuzuordnen. Wenn Sie Durchsuchen verwenden, um eine WSDL auszuwählen, werden auf der Seite WSDL-Definition wählen alle definierten Ports und Bindings aufgelistet.

Hinweis: Wenn Sie den SOAP-Body-Typ für Vorgänge verwenden, stellen Sie sicher, dass die WSDL nicht zwei Vorgänge mit der gleichen Eingabenachricht enthält. Der Vorgang SOAP-Body-Typ kann durch Untersuchen der Eingabenachricht nicht eindeutig identifiziert werden.
  • Transporttyp-Service - Wählen Sie diese Option, um einen JEJB-Proxy-Service zu erstellen.
  • Messaging Service - Wählen Sie diese Option, um einen Service zu erstellen, der Nachrichten eines Datentyps erhalten und mit Nachrichten eines anderen Datentyps antworten kann. Dieser Austausch kann entweder Anforderung/Antwort oder unidirektional sein.

    • (HTTP GET wird nur bei den Service-Typen Beliebiger XML-Service und Messaging-Service unterstützt.)

  • Beliebiger SOAP-Service -Wählen Sie diese Option, um einen SOAP-Service zu erstellen, der über keine explizit definierte konkrete Schnittstelle verfügt.

    • Wählen Sie in der Dropdown-Liste SOAP 1.1 oder SOAP 1.2, um die zu verwendende SOAP-Version festzulegen.

  • Beliebiger XML-Service - Wählen Sie diese Option, um einen XML-Service zu erstellen, der über keine explizit definierte konkrete Schnittstelle verfügt.

    • (HTTP GET wird nur bei den Service-Typen Beliebiger XML-Service und Messaging-Service unterstützt.)

  • Business-Service - Wählen Sie diese Option, um einen Proxy-Service mit einem Route-Knoten zu erstellen, der auf den von Ihnen gewählten Business-Service weiterleitet. Sie können keinen Proxy-Service von einem Transporttyp-Business-Service erstellen. Wenn Sie einen Proxy-Service von einem DSP-Transport-Business-Service erstellen, schaltet Oracle Service Bus den Transporttyp des Proxy-Service auf HTTP, weil DSP-Transport für Proxy-Services nicht verwendet werden kann. Sie können den Transporttyp für den Proxy-Service in einen anderen verfügbaren Transport ändern.

    • Geben Sie den Pfad (Projekt/Ordner) und den Namen des Business-Service ein; oder klicken Sie auf Durchsuchen, um den Business-Service auf der Seite Business-Service wählen auszuwählen.

Service-Typ (Fortsetzung)
  • Proxy-Service - Wählen Sie diese Option, um einen vorhandenen Proxy-Service zu clonen. Da Oracle Service Bus nicht für mehrere Services den gleichen URI akzeptiert, müssen Sie den URI für den geclonten Service ändern.
Hinweis: Wenn ein Service über einen anderen Service erstellt wird, werden Alert-Regeln auf folgende Weise beibehalten:
    • Wird ein Proxy-Service aus einem Business-Service oder ein Business-Service aus einem Proxy-Service erstellt, werden die Alert-Regeln, sofern vorhanden, entfernt.
    • Wird ein Proxy-Service aus einem anderen Proxy-Service oder ein Business-Service aus einem anderen Business-Service erstell, werden die Alert-Regeln, sofern vorhanden, beibehalten.
Service-Schlüsselprovider
Geben Sie gegebenenfalls den Pfad (Projekt/Ordner ) und Namen eines Service-Schlüsselproviders ein, oder klicken Sie auf Durchsuchen , um einen auf der Seite Service-Schlüsselprovider wählen auszuwählen.
Ein Service-Schlüsselprovider ist nur in bestimmten Fällen erforderlich:
  • Ausgehende wechselseitige TLS/SSL, wobei der Proxy-Service Nachrichten an die HTTPS-Services weiterleitet, die eine Clientzertifikatauthentifizierung erfordern.
  • In einigen Web-Service Sicherheitsszenarien, beispielsweise, wenn der Proxy-Service die Verschlüsselung der Nachrichten erfordert.
Um einen für Web Service Security aktivierten Proxy-Service hinzuzufügen, müssen Sie den Proxy-Service aus einer WSDL (Port oder Binding) mit WS-Policy-Anhängen erstellen.
Mehr Informationen finden Sie unter Service-Schlüsselprovider. Mehr über die Erstellung eines Service-Schlüsselproviders unter Service-Schlüsselprovider hinzufügen.
In Registry publizieren
Die Option wird nur angezeigt, wenn eine Standard-UDDI-Registry vorhanden ist.
Wählen Sie diese Option, um den Proxy-Service automatisch in der Standard-Registry zu publizieren. Mehr Informationen dazu unter Auto-Publish verwenden.

Wenn Sie fertig sind

Klicken Sie auf Weiter, um mit der Konfiguration dieses Service auf der nächsten Seite fortzufahren. Oder klicken Sie auf Letzte, um diese Konfiguration auf der Seite Zusammenfassung unter Proxy-Service erstellen/bearbeiten zu prüfen und zu speichern.
