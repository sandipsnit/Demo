Seite "JCA-Transportkonfiguration"

Über diese Seite konfigurieren Sie Transporteinstellungen mit dem JCA-Transportprotokoll. Weitere Informationen zur Verwendung des JCA-Transports finden Sie unter "JCA Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBDV910 und in Generating a Proxy-Service from a JCA Binding Resource.

Tabelle 19-8 JCA-Transportkonfiguration
Option
Beschreibung
JCA-Datei
Klicken Sie auf Durchsuchen, um ein JCA-Binding auszuwählen. Das JCA-Binding definiert verschiedene Aspekte des Service, z.B. Details zum verwendeten Adapter, das Binding mit der WSDL- und der TopLink- oder EclipseLink-Mapping-Datei sowie Eigenschaften der für den Service erforderlichen Aktivierungs-/Interaktionsspezifikation.
Wenn Sie ein gültiges JCA-Binding gewählt haben, werden die übrigen Felder für die Transportkonfiguration aktiviert.
Weitere Informationen zu JCA-Bindings finden Sie in Kapitel 12, JCA-Bindings.
Adaptername
Dieser schreibgeschützte Wert gibt den Namen des Adapters an, der vom JCA-Service verwendet wird.
Adaptertyp
Dieser schreibgeschützte Wert gibt den Adaptertyp an.
Dispatch Policy
Wählen Sie die Instanz des WebLogic Server Work Managers, die Sie für die Dispatch Policy dieses Endpunkts verwenden möchten. Der Standard-Work Manager wird verwendet, wenn kein anderer Work Manager vorhanden ist.
Weitere Informationen zu Work Manager finden Sie unter "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server unter http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
JNDI-Service-Account
Diese Option gilt für JNDI-Kontextsicherheit und dient dem Zugriff auf die vom EIS-Adapter verwaltete Connection Factory. Klicken Sie auf Durchsuchen, und wählen Sie einen Service-Account aus. Wenn Sie keinen Service-Account angeben, wird ein anonymes Subject verwendet.
Bei JCA-Business-Services gibt es keine Einschränkungen hinsichtlich des Typs des zu konfigurierenden JNDI-Service-Accounts (z.B. "Statisch" oder "Passthrough"). Zur Laufzeit muss jedoch der Zugriff auf einen Benutzernamen und ein Kennwort möglich sein. Bei JCA-Proxy-Services können nur statische JNDI-Service-Accounts verwendet werden.
Weitere Informationen zu JNDI-Service-Accounts finden Sie in Kapitel "JCA Transport", im Abschnitt "Security" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV916.
Endpunkteigenschaften
Mit diesem Feld können Sie Werte für Endpunkteigenschaften angeben, z.B. Wiederholungsversuche für den vom Service verwendeten Adaptertyp.
Eine Liste der unterstützten JCA-Endpunkteigenschaften finden Sie unter "Endpoint Properties" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBDV1841.
Dynamische Endpunkteigenschaften
Mit dieser Option können Sie Parameter für JCA-kompatible Services anfordern. Beispiel: Sie können eine dynamische Endpunkteigenschaft verwenden, um Datenbank-Abfrageparameter an den Oracle-JCA-Adapter für die Datenbank zu übergeben.
Weitere Informationen zu Abfragen mit Parametern finden Sie unter "Oracle JCA Adapter for Database" im Oracle Fusion Middleware User's Guide for Technology Adapters unter http://www.oracle.com/pls/as111170/lookup?id=TKADP2117.
Geben Sie ein Name-/Wertpaar für jede dynamische Endpunkteigenschaft ein. Der Schlüssel der Endpunkteigenschaft entspricht dem Abfrageparameternamen.
Immer Konfiguration aus JCA-Datei verwenden
Mit dieser Option legen Sie fest, ob die Eigenschaften der Aktivierungsspezifikation (Proxy-Services) und die Eigenschaften der Interaktionsspezifikation (Business-Services) immer aus der JCA-Datei verwendet werden oder nicht.
Ist diese Option aktiviert (Standard), kommuniziert der JCA-Transport mit dem JCA-Framework unter Verwendung der Eigenschaften der Aktivierungs-/Interaktionsspezifikation in der JCA-Datei.
Ist die Option deaktiviert, können Sie die Eigenschaften der Aktivierungs-/Interaktionsspezifikation überschreiben.
Informationen zu den Auswirkungen des erneuten Deployments bei der Verwendung dieser Option finden Sie im Kapitel "JCA Transport", im Abschnitt "Endpoint Redeployment" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV940.
Vorgangsname
Es wird ein schreibgeschützter Name des gewählten WSDL-Vorgangs angezeigt. Ein Vorgang kann über eigene Eigenschaften der Aktivierungs-/Interaktionsspezifikation verfügen, die im Feld "Eigenschaften der Aktivierungs-/Interaktionsspezifikation" angezeigt werden.
Eigenschaften der Aktivierungs-/Interaktionsspezifikation
Eigenschaften der Aktivierungsspezifikation ist der Feldname für Proxy-Services, Eigenschaften der Interaktionsspezifikation ist der Feldname für Business-Services.
Wenn es sich bei dem Service um einen eingehenden Service handelt, der durch eine EIS-Anwendung aufgerufen wird, werden in diesem Feld die Eigenschaften der Aktivierungsspezifikation für den JCA-Inbound-Vorgang angezeigt, der im Feld "Vorgangsname" angegeben ist.
Sie können die Eigenschaften der Aktivierungs-/Interaktionsspezifikation überschreiben, indem Sie die Option Immer Konfiguration aus JCA-Datei verwenden deaktivieren.
Hinweis: Bei Oracle Adapter Suite-Adaptern werden die Eigenschaften der Aktivierungs-/Interaktionsspezifikation schreibgeschützt angezeigt. Oracle Adapter Suite-Adapter speichern ihre eigenen Konfigurationen, die Sie in den Verwaltungstools von Oracle Adapter Suite ändern müssen.
Verbindungseigenschaften (veraltet)
Informationen zu älteren JCA-Services, die Verbindungseigenschaften mit dem Modus "Nicht verwaltet" verwenden (in diesem Release verworfen), finden Sie im Abschnitt zu den Optionen für die Verbindungskonfiguration unter http://download.oracle.com/docs/cd/E13159_01/osb/docs10gr3/jcatransport/transport.html#wp1105451.

Weitere Informationen zu Endpunkteigenschaften und Eigenschaften der Aktivierungs-/Interaktionsspezifikation finden Sie im Oracle Fusion Middleware User's Guide for Technology Adapters unter http://www.oracle.com/pls/as111170/lookup?id=TKADP.
