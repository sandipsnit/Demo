Seite "JEJB-Transportkonfiguration"

Auf dieser Seite konfigurieren Sie die Transporteinstellungen mit dem JEJB-Transportprotokoll. Weitere Informationen zur Verwendung des JEJB-Transports finden Sie unter "JEJB Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036.

Tabelle 19-9 JEJB-Transportkonfiguration
Option
Beschreibung
Dispatch Policy
Wählen Sie die Instanz des WebLogic Server Work Managers, die Sie für die Dispatch Policy dieses Endpunkts verwenden möchten. Der Standard-Work Manager wird verwendet, wenn kein anderer Work Manager vorhanden ist.
Weitere Informationen zu Work Manager finden Sie unter "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server unter http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
EJB-Spec-Version
Wählen Sie die EJB-Version der Remote-EJB-Schnittstelle.
XMLBeans nach Wert weiterleiten
Wählen Sie diese Option, wenn Sie möchten, dass der Transport eine "Inline"-XML-Darstellung des POJO-Arguments generiert (ein XMLObject), auf dessen Parameter Sie zugreifen können und die Sie mit XQuery-Ausdrücken bearbeiten können.
Hinweis: Typdaten sind inline für XMLObjects, die nach Wert weitergeleitet werden, nicht verfügbar. Wenn Sie diese Option verwenden, können Sie das typisierte XMLObject nicht als Argument in einem Java Callout in einer Proxy-Service Pipeline weiterleiten.
Aktivieren Sie diese Option nicht, wenn Sie POJO nach Referenz weiterleiten möchten, was auch für eine bessere Performance sorgt.
Transaktionsattribut
Wählen Sie eine der folgenden Optionen für das Verarbeiten von Transaktionen:
Supports - Der Transport akzeptiert eine eingehende Transaktion. Quality of Service ist "genau einmal", wenn der Vorgang in einer Transaktion aufgerufen wird, bzw. "Best-Effort", wenn der Vorgang außerhalb einer Transaktion aufgerufen wird.
Required - Der Transport akzeptiert eine eingehende Transaktion. Ist keine laufende Transaktion vorhanden, startet der Transport eine. Quality of Service ist "genau einmal".
RequiresNew - Der Transport startet immer eine neue Transaktion und unterbricht dabei eine ausgehende Transaktion. Die Quality of Service ist "genau einmal".
Mandatory - Der Transport ruft die Methode in der vorhandenen Transaktion auf. Quality of Service ist "genau einmal".
NotSupported - Der Transport unterbricht eine vorhandene Transaktion und nimmt sie bei Aufruf wieder auf. Quality of Service ist "Best-Effort".
Never - Der Transport ruft die Methode in einer Transaktion nicht auf. Quality of Service ist "Best-Effort".
Client-JAR-Archiv
Klicken Sie auf Durchsuchen, und wählen Sie eine EJB-Client-JAR-Ressource aus der angezeigten Liste. Die Client-JAR enthält das Remote- oder Business-Interface für das Remote EJB. Die Client-JAR ist als generische Archivierungsressource registriert.
Home-Interface
Nur EJB 2.1 - Wählen Sie das erforderliche EJBHome-Interface aus den Optionen, die die Client-JAR aufgefüllt hat.
Remote-Interface
Nur EJB 2.1: Dieses Feld wird automatisch ausgehend von der Konfiguration der Home-Schnittstelle vorgegeben.
Business-Schnittstelle
Nur EJB 3.0: Wählen Sie die Business-Schnittstelle aus dem Client-JAR-Archiv aus, die Sie aufrufen möchten.
Ziel-Namespace
Dieses Feld wird mit Informationen aus dem JAR-Archiv ausgefüllt.
Methoden
Wählen Sie die erforderlichen Methoden. Klicken Sie auf +, um die Methode zu erweitern und um die Standardparameterwerte zu bearbeiten.
Sie können den Namen des Standardvorgangs für eine bestimmte Methode ändern. Standardmäßig entspricht der Vorgangsname dem Methodennamen. Wenn ein EJB Methoden mit demselben Namen (überladen) enthält, müssen Sie die Vorgangsnamen ändern, damit sie eindeutig sind. In WSDLs werden eindeutige Vorgangsnamen benötigt.

Wenn Sie fertig sind

Klicken Sie auf Weiter, um mit der Konfiguration dieses Service auf der nächsten Seite fortzufahren. Oder klicken Sie auf Letzte, um diese Konfiguration auf der Seite Zusammenfassung unter Proxy-Service erstellen/bearbeiten zu prüfen und zu speichern.
