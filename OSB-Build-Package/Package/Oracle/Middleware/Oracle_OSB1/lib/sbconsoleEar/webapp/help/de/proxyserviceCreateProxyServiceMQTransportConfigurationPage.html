Seite "MQ-Transportkonfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - MQ-Transportkonfiguration gehört zu einer Reihe von Seiten zum Erstellen und Bearbeiten von Proxy-Services. Diese Aktionen sind in Proxy-Services erstellen und konfigurieren und in Proxy-Service-Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie unter "MQ Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Bevor Sie beginnen

Konfigurieren Sie eine MQ-Verbindungsressource. Siehe MQ-Verbindung.

MQ-Transport konfigurieren

Sie verwenden diese Seite, um Transporteinstellungen für einen Proxy-Service mithilfe des nativen MQ-Transportprotokolls zu konfigurieren. Tabelle 19-11 liefert eine Beschreibung zur Verwendung dieser Seite.

Tabelle 19-11 Seite "Proxy-Service erstellen und bearbeiten - MQ-Transportkonfiguration" 
Option
Verwendung
Abrufintervall
Geben Sie ein Abrufintervall in Millisekunden ein. Der Standardwert ist 1000.
Ist Antwort erforderlich
Wählen Sie diese Option, um anzugeben, dass eine Antwort erwartet wird, wenn eine ausgehende Nachricht gesendet wurde.
Antwortkorrelationsmuster
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Geben Sie an, ob das Antwortkorrelationsmuster auf MessageID oder CorrelationID basiert sein soll.
MQ-Antwort-URI
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Ziel, in dem die Antwort veröffentlicht werden soll. Geben Sie einen Antwort-URI im selben Format wie beim Endpunkt-URI ein: mq://<local-queue-name>?conn=<mq-connection-resource-ref>
Antwortnachrichtentyp
Diese Option ist nur verfügbar, wenn das Kontrollkästchen Ist Antwort erforderlich aktiviert wurde.
Wählen Sie eine der folgenden Optionen:
  • Byte (für einen Stream nicht interpretierter Byte)
  • Text (für Textnachrichten)
Dispatch Policy
Wählen Sie eine Dispatch Policy für diesen Endpunkt.
Die Dispatch Policy bezieht sich auf die WLS Work Manager-Instanz, die Sie für den Service-Endpunkt verwenden möchten. Weitere Informationen zu Work Manager finden Sie unter "Using Work Managers to Optimize Scheduled Work" im Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Backout-Schwellenwert
Geben Sie ein, wie oft die Pipeline eine Nachricht wiederholen soll, bevor die Nachricht in die Queue umgeleitet wird, die im Feld URI für unzustellbare Nachrichten angegeben ist.
Wenn Sie in diesem Feld keinen Wert eingeben, wird die Nachricht ohne Wiederholungsversuche gleich in die Queue für unzustellbare Nachrichten umgeleitet.
MQ-URI für unzustellbare Nachrichten
Geben Sie den URI der DeadLetter Queue an, zu der Anforderungsnachrichten umgeleitet werden sollen, nachdem die Anzahl der Wiederholungsversuche erreicht ist, welche im Feld Backout-Schwellenwert festgelegt wurde.
Wenn Sie für dieses Feld keinen Wert angeben, wird die Nachricht an die Queue zurückgeschickt und bei jedem Polling von dem MQ-Transport ignoriert, nachdem die Anzahl von Wiederholungsversuchen, die im Feld Backout-Schwellenwert angegeben wird, erreicht ist. Der URI für unzustellbare Nachrichten hat dasselbe Format wie der Endpunkt-URI.
"GET"-Optionen für Endpunkt-URI
Geben Sie eine der folgenden MQ GET-Nachrichtenoptionen ein:
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
Sie können mehrere Optionen trennen, entweder mit "|" oder "+". Sie können beispielsweise Folgendes angeben:
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
Die MQ GET-Nachrichtenoptionen werden angewendet, wenn eine Nachrichte aus der eingehenden Queue gelesen wird.
RFH2-Header verarbeiten
Wählen Sie diese Option, um WebSphere MQ RFH2-Header in einer Message-Payload zu parsen, und generieren Sie automatisch einen RFH2Headers-Transport-Header, der die RFH2-Daten enthält.
Wenn Sie diese Option nicht wählen, wird die Payload so übergeben, wie sie empfangen wurde.

Weitere Informationen darüber, wie der MQ-Transport RFH2-Header verarbeitet, finden Sie im Abschnitt "About RFH2 Headers" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
