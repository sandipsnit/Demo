Seite "Nachrichtentyp-Konfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - Nachrichtentyp konfigurieren ist eine Seite von mehreren, auf denen Proxy-Services erstellt und bearbeitet werden können, wie in Proxy-Services erstellen und konfigurieren und Proxy-Service-Konfigurationen bearbeiten beschrieben.

Auf dieser Seite können Sie Nachrichtentypen für einen Proxy-Service konfigurieren, dessen Typ "Messaging Service" lautet.

Die Binding-Definition für Messaging Service besteht in der Konfiguration des Content-Type der ausgetauschten Nachrichten. Der Content-Type für die Antwort muss nicht derselbe wie bei der Anforderung sein. Daher wird die Antwort separat konfiguriert (beispielsweise könnte der Service eine MFL-Nachricht annehmen und einen XML-Bestätigungseingang zurückgeben).

Hinweis: Bei E-Mail-, Datei-, FTP- oder SFTP-Transport-Proxy-Services, deren Typ ein Messaging Service ist, der nur OneWay-Messaging unterstützt, sollte der Antwortnachrichtentyp auf Keine gesetzt sein. Wenn Sie eine andere Option als Keine aktivieren, stehen E-Mail-, Datei-, FTP- oder SFTP-Protokolle auf der Seite "Transportkonfiguration" nicht zur Verfügung.

Tabelle 19-2 erläutert die Verwendung der Seite "Proxy-Service erstellen/bearbeiten - Nachrichtentyp konfigurieren"

Tabelle 19-2 Proxy-Service erstellen/bearbeiten - Seite "Nachrichtentyp konfigurieren"
Option
Verwendung
Anforderungsnachrichtentyp
Wählen Sie einen Nachrichtentyp für die Anforderungsnachricht:
  • Keine - Wählen Sie diese Option, wenn keine Anforderungsnachricht vorhanden ist.
  • Binär - Wählen Sie diese Option, wenn der Content-Typ der Nachricht unbekannt oder unwichtig ist.
  • Text - Wählen Sie diese Option, wenn die Nachricht auf Text beschränkt werden kann.
  • MFL - Wählen Sie diese Option, wenn die Nachricht ein Binärdokument ist, das einer MFL-Definition entspricht. Geben Sie den MFL-Dateinamen ein (durch den Pfad gekennzeichnet), oder klicken Sie auf Durchsuchen, um eine Datei auszuwählen.

    • Sie können nur eine MFL-Datei konfigurieren:

    Hinweis: Damit mehrere MFL-Dateien unterstützt werden, definieren Sie den Content als Binär- oder Textformat und verwenden die Aktion "MFL" im Nachrichtenfluss für die Konvertierung zu XML.

  • XML - Wählen Sie diese Option, wenn die Nachricht ein XML-Dokument ist. Geben Sie den XML-Dateinamen ein (durch den Pfad gekennzeichnet), oder klicken Sie auf Durchsuchen, um eine Datei auszuwählen.

    • Geben Sie optional Daten zum Typ ein, indem Sie (im Feld Element oder Typ) den XML-Schematyp des ausgetauschten XML-Dokuments erklären.

  • Java - Wählen Sie diese Option, wenn in der Anforderung ein Java-Objekt gesendet wird. Der JMS-Transport wird für Nachrichten mit Java-Objekten verwendet. Weitere Informationen finden Sie im Abschnitt "Sending and Receiving Java Objects in Messages" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Antwortnachrichtentyp
Wählen Sie einen Nachrichtentyp für die Antwortnachricht:
  • Keine - Wählen Sie diese Option, wenn keine Antwortnachricht vorhanden ist.
  • Binär - Wählen Sie diese Option, wenn der Content-Typ der Nachricht unbekannt oder unwichtig ist.
  • Text - Wählen Sie diese Option, wenn die Nachricht auf Text beschränkt werden kann.
  • MFL - Wählen Sie diese Option, wenn die Nachricht ein Binärdokument ist, das einer MFL-Definition entspricht. Geben Sie den MFL-Dateinamen ein (durch den Pfad gekennzeichnet), oder klicken Sie auf Durchsuchen, um eine Datei auszuwählen.

    • Sie können nur eine MFL-Datei konfigurieren:

    Hinweis: Damit mehrere MFL-Dateien unterstützt werden, definieren Sie den Content als Binär- oder Textformat und verwenden die Aktion "MFL" im Nachrichtenfluss für die Konvertierung zu XML.

  • XML - Wählen Sie diese Option, wenn die Nachricht ein XML-Dokument ist. Geben Sie den XML-Dateinamen ein (durch den Pfad gekennzeichnet), oder klicken Sie auf Durchsuchen, um eine Datei auszuwählen.

    • Geben Sie optional Daten zum Typ ein, indem Sie (im Feld Element oder Typ) den XML-Schematyp des ausgetauschten XML-Dokuments erklären.

  • Java - Wählen Sie diese Option, wenn die empfangene Antwort ein Java-Objekt enthält. Der JMS-Transport wird für Nachrichten mit Java-Objekten verwendet. Weitere Informationen finden Sie im Abschnitt "Sending and Receiving Java Objects in Messages" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
