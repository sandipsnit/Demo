Seite "SFTP-Transportkonfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - SFTP-Transportkonfiguration ist eine Seite von mehreren, auf denen Proxy-Services erstellt und bearbeitet werden können, wie in Proxy-Services erstellen und konfigurieren und Proxy-Service-Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie unter "HTTP and Poller Transports" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV947.

Auf dieser Seite können Sie die Transporteinstellungen für einen Proxy-Service konfigurieren, der das SFTP-Transportprotokoll verwendet. In Tabelle 19-13 wird die Verwendung dieser Seite beschrieben.

Tabelle 19-13 Proxy-Service erstellen/bearbeiten - Seite "SFTP-Transportkonfiguration"
Option
Verwendung
Benutzer-Authentifizierung
Wählen Sie eine der folgenden Optionen:
  • Benutzername- und Kennwort-Authentifizierung: Gibt an, dass ein statischer Service-Account mit dieser Authentifizierungsmethode verknüpft ist und dass der Client mithilfe der angegebenen Zugangsdaten authentifiziert wird.
  • Host-basierte Authentifizierung: Gibt an, dass ein Benutzername- und Service-Schlüsselprovider diese Authentifizierungsmethode verwenden muss. Jeder Benutzer, der eine Verbindung von einem bekannten Host herstellt, wird mithilfe des Privatschlüssels des Hosts authentifiziert.
  • Public Key-Authentifizierung: Gibt an, dass ein Benutzername- und Service-Schlüsselprovider diese Authentifizierungsmethode verwenden muss. Jeder Benutzer hat seinen eigenen Privatschlüssel.
Service-Account
Geben Sie den Service-Account für den Benutzer ein, oder klicken Sie auf Durchsuchen, um Service-Accounts von einem Browser auszuwählen.
Service-Schlüsselprovider
Diese Option ist nur verfügbar, wenn host-basierte oder Public Key-Authentifizierung gewählt ist.
Geben Sie einen Service-Schlüsselprovider im Feld Service-Schlüsselprovider an. Sie können auf Durchsuchen klicken, um einen Service-Schlüsselprovider von einem Browser auszuwählen. Dieses Feld muss einen Wert enthalten.
Benutzername
Diese Option ist nur verfügbar, wenn host-basierte oder Public Key-Authentifizierung gewählt ist.
Geben Sie den Benutzernamen ein.
Nach Referenz übergeben
Aktivieren Sie dieses Kontrollkästchen, um die Datei im Archivierungsverzeichnis bereitzustellen und sie als Referenz im Header zu übergeben.
Remote Streaming
Aktivieren Sie dieses Kontrollkästchen, um die SFTP-Dateien zum Zeitpunkt der Verarbeitung direkt vom Remote-Server zu streamen. Wenn Sie diese Option aktivieren, entspricht das Archivierungsverzeichnis dem Remote-Verzeichnis auf dem Rechner des Remote-SFTP-Servers. Daher sollten Sie das Archivierungsverzeichnis relativ zum SFTP-Benutzerverzeichnis angeben.
Dateimaske
Geben Sie den regulären Ausdruck für die auszuwählenden Dateien ein. Der Standardwert ist *.*.
Managed Server
Dieses Feld ist nur in einer geclusterten Domain verfügbar.
Wählen Sie den Managed Server, der als Polling-Server dienen soll. Alle Managed Server können die Nachricht verarbeiten, aber nur ein Server kann einen Abruf für die Nachricht ausführen.
Abrufintervall
Geben Sie in Sekunden, in welchem Intervall die Datei vom angegebenen Ort abgerufen werden soll. Der Standardwert ist 60.
Lesegrenzwert
Geben Sie an, wie viele Nachrichten maximal pro Abruf-Sweep gelesen werden sollen. Geben Sie 0 ein, um keine Begrenzung anzugeben. Der Standardwert ist 10.
Aktion nach dem Lesen
Wählen Sie aus, was passieren soll, wenn die Nachricht gelesen wurde.
  • Archivieren - Die Nachricht wird archiviert.
  • Löschen - Die Nachricht wird gelöscht.
Archivierungsverzeichnis
Geben Sie den Pfad zum Archivierungsverzeichnis an, wenn die Option Post Read-Aktion auf Archivieren gesetzt ist. In dieses Feld muss ein Wert eingegeben werden, wenn die Option Nach Referenz übergeben aktiviert ist.
Hinweis: Die Verzeichnisse "Archivieren", "Herunterladen" und "Fehler" sind absolute Pfade, und sie werden automatisch erstellt. Wenn Sie einen relativen Pfad angeben, werden die Dateien relativ zum Java-Prozess erstellt, der den WebLogic Server startet.
Downloadverzeichnis
Geben Sie das Verzeichnis auf Ihrem lokalen Computer an, auf den die Dateien beim Dateitransfer heruntergeladen werden.
Hinweis: Die Verzeichnisse "Archivieren", "Herunterladen" und "Fehler" sind absolute Pfade, und sie werden automatisch erstellt. Wenn Sie einen relativen Pfad angeben, werden die Dateien relativ zum Java-Prozess erstellt, der den WebLogic Server startet.
Fehlerverzeichnis
Geben Sie an, wohin die Dateien im Falle eines Problems verschickt werden sollen.
Hinweis: Die Verzeichnisse "Archivieren", "Herunterladen" und "Fehler" sind absolute Pfade, und sie werden automatisch erstellt. Wenn Sie einen relativen Pfad angeben, werden die Dateien relativ zum Java-Prozess erstellt, der den WebLogic Server startet.
Anforderungscodierung
Übernehmen Sie den Standardwert UTF-8 als die Zeichensatzcodierung für Anforderungen in SFTP-Transporten.
Bevorzugte Cipher Suite
Wählen Sie die zu verwendende Cipher Suite, wenn Sie mit dem Server kommunizieren möchten.
Wenn Sie den Standardwert Laufzeitstandard verwenden verwenden, wird die Liste der unterstützten Cipher Suites an den Server gesendet, und die Versuche werden für jede Suite so lange wiederholt, bis eine Übereinstimmung gefunden wird.
Bevorzugter Datenintegritätsalgorithmus
Wählen Sie den Bulk-Hashing-Algorithmus für Datenintegritätsprüfungen.
Wenn Sie den Standardwert verwenden, Laufzeitstandardwert verwenden, sendet Oracle Service Bus den bevorzugten Algorithmus hmac-sha1 an den Server. Wenn dieser vom Server nicht unterstützt wird, wird die Liste der unterstützten Algorithmen an den Server gesendet und jeder Algorithmus wird erprobt, bis eine Übereinstimmung gefunden wird.
Bevorzugter Public Key-Algorithmus
Wählen Sie den asymmetrischen Schlüsselalgorithmus für Public Key-Kryptografie.
Wenn Sie den Standardwert verwenden, Laufzeitstandardwert verwenden, sendet Oracle Service Bus den bevorzugten Algorithmus ssh-dss an den Server. Wenn dieser vom Server nicht unterstützt wird, wird die Liste der unterstützten Algorithmen an den Server gesendet und jeder Algorithmus wird erprobt, bis eine Übereinstimmung gefunden wird.
Bevorzugter Schlüsselaustauschalgorithmus
Verwenden Sie das Standardschlüsselaustauschprotokoll, um den Sessionschlüssel zur Verschlüsselung der Nachricht zu verhandeln.
Wenn Sie den Standardwert verwenden, Laufzeitstandardwert verwenden, sendet Oracle Service Bus den bevorzugten Algorithmus diffie-hellman-group1-sha1 an den Server. Wenn dieser vom Server nicht unterstützt wird, wird die Liste der unterstützten Algorithmen gesendet und jeder Algorithmus wird erprobt, bis eine Übereinstimmung gefunden wird.
Bevorzugter Komprimierungsalgorithmus
Wählen Sie, ob laufende Daten mit ZLIB komprimiert werden sollen. Wählen Sie ZLIB zur Komprimierung von Daten; sonst wählen Sie Keine. Der Standardwert ist Keiner.
Unterverzeichnisse scannen
Aktivieren Sie dieses Kontrollkästchen, wenn alle Verzeichnisse rekursiv durchsucht werden sollen.
Nach Eingang sortieren
Aktivieren Sie dieses Kontrollkästchen, wenn die Ereignisse in der Reihenfolge ihres Auftretens geliefert werden sollen.
Timeout
Geben Sie das Socket-Timeout-Intervall in Sekunden an, bevor die Verbindung unterbrochen wird. Wenn Sie 0 eingeben, gibt es keinen Timeout. Der Standardwert ist 60.
Anzahl Wiederholungen
Geben Sie die Anzahl der Wiederholungsversuche an, falls die SFTP-Verbindung nicht erfolgreich ist.

Wenn Sie fertig sind
