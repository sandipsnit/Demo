Seite "Tuxedo-Transportkonfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - Tuxedo-Transportkonfiguration gehört zu einer Reihe von Seiten zum Erstellen und Bearbeiten von Proxy-Services. Diese Aktionen sind in Proxy-Services erstellen und konfigurieren und in Proxy-Service-Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie unter "Tuxedo Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Sie verwenden diese Seite, um Transporteinstellungen für einen Proxy-Service mithilfe des Tuxedo-Transportprotokolls zu konfigurieren. Tabelle 19-14 liefert eine Beschreibung zur Verwendung dieser Seite.

Tabelle 19-14 Seite "Proxy-Service erstellen und bearbeiten - Tuxedo-Transportkonfiguration" 
Option
Verwendung
Feldtabellenklassen
Geben Sie den Namen der Klasse(n) ein, die den empfangenen FML/FML31-Puffer beschreiben. Sie dienen als FML/FML32-in-XML-Konvertierungsroutinen, um Feldnamen zu Elementnamen zuzuordnen. Diese durch Leerstellen getrennte Liste enthält vollständig angegebene Klassennamen.
Klassen anzeigen
Geben Sie den Namen der Klasse(n) ein, die den empfangenen VIEW/VIEW32-Puffer beschreiben. Sie dienen als VIEW-in-XML oder VIEW32-in-XML-Konvertierungsroutinen, um Feldnamen zu Elementnamen zuzuordnen. Diese durch Leerstellen getrennte Liste enthält vollständig angegebene Klassennamen.
Die X_C_TYPE- und X_COMMON-Tuxedo-Puffertypen werden genauso wie VIEW/VIEW32-Puffer behandelt.
Enthält eine eingehende Anforderung einen VIEW-Puffer, muss die entsprechende VIEW-Klasse im CLASSPATH von Oracle Service Bus angegeben sein.
JAR-Klassen
Wählen Sie eine JAR-Ressource, die eine JAR-Datei mit FML/FML32- oder VIEW/VIEW32-Klassen enthält, die für diesen Endpunktvorgang erforderlich sind.
Lokaler Zugriffspunkt
Wählen Sie aus der zum Export gehörigen Dropdown-Liste einen lokalen Zugriffspunkt. Die Dropdown-Liste enthält in WTC konfigurierte lokale Zugriffspunkte. Sie können keinen Proxy-Service erstellen, wenn es keinen zugeordneten lokalen Zugriffspunkt gibt.
Wenn kein lokaler Zugriffspunkt vorhanden ist, oder um einen neuen zu erstellen, wählen Sie Neu. Geben Sie in den Feldern Name des lokalen Zugriffspunktes und Lokale Netzwerkadresse die entsprechenden Werte ein. Nach Validierung des Endpunktes wird der Zugriffspunkt der WTC-Konfiguration der einzelnen Server hinzugefügt. Ist kein WTC-Server vorhanden, wird einer erstellt.
Sie können einen vorhandenen Zugriffspunktnamen eingeben, nachdem Sie die Option Neu gewählt haben. Dadurch werden die vorhandenen Daten mit den neuen Parametern aktualisiert. Sie können lediglich Hostname und Port-Nummern ändern.
Remote-Zugriffspunkt
Dieses Feld wird angezeigt, wenn Sie im Feld Lokaler Zugriffspunkt die Option Neu wählen.
Wählen Sie aus der Dropdown-Liste einen Remote-Zugriffspunkt, der mit dem neu erstellten lokalen Zugriffspunkt verknüpft werden soll. Wenn keiner vorhanden ist, oder um einen neuen zu erstellen, wählen Sie Neu. Geben Sie in den Feldern Name des Zugriffspunkts und Netzwerkadresse die entsprechenden Werte ein.
Sie können einen vorhandenen Zugriffspunktnamen eingeben, nachdem Sie die Option Neu gewählt haben. Dadurch werden die vorhandenen Daten mit den neuen Parametern aktualisiert. Sie können lediglich Hostname und Port-Nummern ändern.
Der Remote-Zugriffspunkt wird auch zum Authentifizierungs-Principal für die WTC-Verbindung eingehender Anforderungen. Optional können Sie einen Benutzer mit derselben Zugriffspunkt-ID in der Standardsicherheits-Realm erstellen, um eingehenden Aufrufe zuzulassen. Wählen Sie dazu "Ja" aus der Dropdown-Liste "Benutzer erstellen?". Das Kennwort wird mithilfe einer temporären Variablen zufällig erzeugt, um Sicherheitsprobleme zu vermeiden.
Antwortpuffertyp
Diese Option ist nur verfügbar, wenn das Feld Antwort erforderlich? gewählt wurde.
Wählen Sie den Puffertyp, den der Remote Tuxedo-Client empfangen soll.
Antwortpuffer-Subtyp
Diese Option ist nur verfügbar, wenn die Option Antwort erforderlich? gewählt und der Wert Antwortpuffertyp auf VIEW oder VIEW32 gesetzt wurde.
Geben Sie den Antwortpuffer-Subtyp ein, der mit dem Antwortpuffer verknüpft werden soll.
Antwort erforderlich?
Aktivieren Sie dieses Kontrollkästchen, wenn der Service eine Antwort senden soll.
Diese Option ist standardmäßig aktiviert.
Die Aktivierung dieser Option wird aufgehoben und ist nicht verfügbar, wenn der Service-Typ Messaging Service und der Antwortnachrichtentyp Keine lauten.
Anforderungscodierung
Geben Sie eine Zeichensatzcodierung für Anforderungen in Tuxedo-Transporten an.
Antwortcodierung
Geben Sie eine Zeichensatzcodierung für Antworten in Tuxedo-Transporten an.
Transformationsstil
Wählen Sie eine der folgenden Optionen:
  • Keine: (Standardwert) Die Reihenfolge der Felder muss nicht beachtet werden.
  • Sortiert: Die Felder werden bei allen Vorkommnissen in der richtigen Reihenfolge dargestellt.
  • Sortiert und gruppiert: Wenn die Felder logisch wie Datensätze strukturiert sind, werden sie nach Auftreten sortiert und nach Datensatz gruppiert.
