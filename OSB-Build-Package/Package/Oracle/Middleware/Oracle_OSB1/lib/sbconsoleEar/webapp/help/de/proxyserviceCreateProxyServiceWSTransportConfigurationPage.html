Seite "WS-Transportkonfiguration"

Die Seite Proxy-Service erstellen/bearbeiten - WS-Transportkonfiguration gehört zu einer Reihe von Seiten zum Erstellen und Bearbeiten von Proxy-Services. Diese Aktionen sind in Proxy-Services erstellen und konfigurieren und in Proxy-Service-Konfigurationen bearbeiten beschrieben.

Weitere Informationen finden Sie unter "WS Transport" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083.

Sie verwenden diese Seite, um Transporteinstellungen für einen Proxy-Service mithilfe des WS-Transportprotokolls zu konfigurieren. Tabelle 19-15 liefert eine Beschreibung zur Verwendung dieser Seite.

Tabelle 19-15 Seite "Proxy-Service erstellen und bearbeiten - WS-Transportkonfiguration" 
Option
Verwendung
Dispatch Policy
Wählen Sie eine Dispatch Policy für diesen Endpunkt, oder verwenden Sie die Standard-Dispatch Policy.
Die Dispatch Policy bezieht sich auf die WLS Work Manager-Instanz, die Sie für den Service-Endpunkt verwenden möchten. Weitere Informationen zu Work Manager finden Sie unter "Using Work Managers to Optimize Scheduled Work" im Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Anzahl Wiederholungen
Anzahl der Versuche, eine Nachricht an die Pipeline zu senden.
Tritt eine unbehandelte Exception in der Anforderungs-Pipeline eines Proxy-Service auf, wird die eingehende WS-Transportnachricht so oft erneut der Pipeline zugestellt, bis die festgelegte Anzahl Wiederholungen erreicht ist. Dieser Wert ist für eine zuverlässige Verarbeitung der WS-Transportnachrichten wichtig.
Verzögerung bei Wiederholungen
Anzahl Sekunden, die das System wartet, bevor es nach einem Fehler erneut eine Nachricht an die Pipeline zu senden versucht.
