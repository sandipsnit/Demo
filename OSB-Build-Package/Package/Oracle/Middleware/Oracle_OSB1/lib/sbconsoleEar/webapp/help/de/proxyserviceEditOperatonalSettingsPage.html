Vorgangseinstellungen für Proxy-Services konfigurieren

Auf der Seite Vorgangseinstellungen können Sie einen Proxy-Service und das Überwachen für bestimmte Proxy-Services aktivieren und deaktivieren. Außerdem können Sie diese Service-spezifischen Vorgangseinstellungen auch auf der Seite Vorgänge > Smart Search angeben. Dabei gelten folgende Ausnahmen: Sie können kein Aggregationsintervall festlegen und keinen Schweregrad für die Ausgabe von Alerts oder für die Protokollierung angeben. Weitere Informationen finden Sie unter Vorgangseinstellungen suchen und aktualisieren.

Die Laufzeitauswirkungen der Einstellungen für die Service-Ebene hängen von ihren jeweiligen globalen Einstellungen ab. Sie müssen sowohl die globalen als auch die Einstellungen auf Service-Ebene aktivieren, damit ein Service zur Laufzeit komplett aktiviert ist. Außerdem muss auch der Service-Status aktiviert sein. Siehe Globale Einstellungen aktivieren.

Sie können Überwachungskonfigurationseinstellungen auch dann ändern und speichern, wenn der Service nicht zur Laufzeit aktiviert wird. Beispiel: Sie können das Aggregationsintervall auch dann ändern und speichern, wenn die Service-Überwachung deaktiviert ist. Auf diese Weise können Sie Einstellungen bearbeiten und später aktivieren.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Ressourcenbrowser > Proxy-Services.
  3. Klicken Sie auf den Namen des Proxy-Service.

    4. Auf der Seite Proxy-Service anzeigen werden die in Tabelle 25-6 dargestellten Informationen angezeigt.

    Tabelle 25-6 Informationen zu Proxy-Services
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der diesen Service erstellt, bearbeitet oder in die Konfiguration importiert hat.
    Zuletzt geändert am
    Das Datum oder die Uhrzeit, zu der der Benutzer diesen Service erstellt, bearbeitet oder in die Konfiguration importiert hat. Klicken Sie auf den Link von Datum und Uhrzeit, um die Änderungshistorie dieser Ressource anzuzeigen. Siehe Seite "Änderungshistorie anzeigen"
    Referenzen
    Die Anzahl der Objekte, die dieser Service referenziert. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Siehe Referenzen auf Ressourcen anzeigen
    Referenziert von
    Die Anzahl der Objekte, die diesen Service referenzieren. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Siehe Referenzen auf Ressourcen anzeigen
    Beschreibung
    Eine Beschreibung des Service, sofern vorhanden.

  4. Klicken Sie auf die Registerkarte Vorgangseinstellungen.

    5. Sie müssen sich in einer aktiven Session befinden, um die in Tabelle 25-7 dargestellten Vorgangseinstellungen für Proxy-Services konfigurieren zu können.

    Tabelle 25-7 Vorgangseinstellungen für Proxy-Services konfigurieren
    Ziel
    Vorgehensweise
    Status deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session aktivieren, stoppt der Service das Verarbeiten der Nachrichten.
    Wenn Sie einen Service deaktivieren, löscht das System nicht mehr alle zuvor für diesen Service erfassten Überwachungsstatistiken.
    Status aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session aktivieren, nimmt der Service das Verarbeiten der Nachrichten wieder auf.
    Überwachung deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session deaktivieren, stoppt das System das Erfassen der Überwachungsstatistiken für den Service und löscht alle zuvor erfassten Statistiken.
    Überwachung aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert neben Überwachung. Wählen Sie in der Dropdown-Liste Pipeline-Überwachung bei einen Wert, um das Überwachen auf die angegebene Stufe oder höher zu beschränken (die Stufen sind von der umfassendsten zur einschränkendsten Stufe aufgelistet): "Aktion", "Pipeline" und "Service".
    Wenn Sie die Session aktivieren, werden vom System Überwachungsstatistiken für den Service erfasst.
    Aggregationsintervall für einen Service festlegen
    Wählen Sie ein Intervall in Stunden oder Minuten aus, dann klicken Sie auf Aktualisieren. Wenn Ihre Auswahl von Stunden höher ist als 1, ist der Standardwert für Minuten stets null. Wenn die Auswahl der Stunden jedoch 0 oder 1 ist, dann können Sie Intervalle in Minuten konfigurieren. Siehe Aggregationsintervall für einen Service festlegen.
    Sie können die Einstellung des Aggregationsintervalls unabhängig vom Aktivieren der Überwachung ändern und speichern.
    Um das Überwachen zur Laufzeit zu aktivieren, stellen Sie sicher, dass das Kontrollkästchen Überwachung aktivieren gewählt ist.
    SLA-Alerts deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert. Das System stoppt die Evaluierung aller für diesen Service konfigurierten Alert-Regeln. Daher erhalten Sie keine zu diesem Service gehörigen SLA-Alerts mehr.
    SLA-Alerts aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert. Geben Sie in der Dropdown-Liste Wenn Schweregradstufe an, dass SLA-Alerts auf die angegebene Stufe oder höher beschränkt werden (Auflistung von der umfassendsten bis zur restriktivsten Stufe): Normal (Standardwert), Warnung, Unbedeutend, Gravierend, Kritisch und Schwerwiegend.
    Service-SLA-Alerts hängen sowohl vom globalen Status als auch vom Status der Service-Ebene ab. Beide müssen für SLA-Alerts zur Laufzeit aktiviert sein. Siehe SLA-Alerts global aktivieren.
    Pipeline Alerts deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert. Wenn Sie die Session aktivieren, beendet das System die Ausführung von Pipeline Alert-Aktionen. Aus diesem Grund erhalten Sie keine Pipeline Alerts mehr zum Service.
    Pipeline Alerts aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert. Wählen Sie in der Dropdown-Liste Wenn Schweregradstufe einen Wert, um Pipeline Alerts auf den angegebenen Schweregrad oder höher zu beschränken (die Stufen sind von der umfassendsten zur einschränkendsten Stufe aufgelistet): "Normal" (Standardwert), "Warnung", "Unbedeutend", "Gravierend", "Kritisch" und "Schwerwiegend".
    Berichtsaktion für einen Proxy-Service deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert neben Erstellen von Nachrichtenberichten. Wenn Sie die Session aktivieren, beendet das System die Ausführung von Berichtsaktionen. Aus diesem Grund erhalten Sie keine Nachrichtenberichte mehr zum Service.
    Berichtsaktion für einen Proxy-Service aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert neben Erstellen von Nachrichtenberichten.
    Logging-Ausgabe für eine Pipeline Log-Aktion deaktivieren
    Deaktivieren Sie das Kontrollkästchen Aktiviert neben Logging. Wenn Sie die Session aktivieren, beendet das System die Ausführung von Log-Aktionen. Aus diesem Grund erhalten Sie keine Logging-Ausgaben mehr zum Service.
    Logging-Ausgabe für eine Pipeline Log-Aktion aktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert neben Logging. Wählen Sie in der Dropdown-Liste Wenn Schweregradstufe einen Wert, um die Logging-Ausgabe auf die angegebene Stufe oder höher zu beschränken (die Stufen sind von der umfassendsten zur einschränkendsten Stufe aufgelistet): "Debug" (Standardwert), "Info", "Warnung" und "Fehler".
    Ausführungs-Tracing zur Laufzeit für einen Proxy-Service aktivieren

    Aktivieren Sie das Kontrollkästchen Aktiviert neben Ausführungs-Tracing. Nachdem Sie das Ausführungs-Tracing aktiviert haben, werden vom System verschiedene Details protokolliert, die aus dem Nachrichtenflusskontext und dem Nachrichtenkontext ausgesucht wurden. Zu diesen Details gehören: Stage-Name, Pipeline- oder Route-Knotenname sowie der aktuelle Nachrichtenkontext.

    Hinweis: Um das Tracing in der Log-Datei oder im Standardausgang (Serverkonsole) anzuzeigen, muss die Anmeldung bei Oracle WebLogic Server auf die folgenden Dringlichkeitsstufen eingestellt werden:

    Mindestdringlichkeit für das Log: Info
    Log-Datei: Info
    Standardausgang: Info

    Weitere Informationen zum Einstellen der Dringlichkeitsstufen für das Log finden Sie im entsprechenden Thema unter Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

    Laufzeit-Tracing für einen Proxy-Service deaktivieren
    Aktivieren Sie das Kontrollkästchen Aktiviert neben Ausführungs-Tracing.
    Nachrichten-Tracing für einen Proxy-Service aktivieren

    Aktivieren Sie das Kontrollkästchen Aktiviert neben Nachrichten-Tracing. Nachdem Sie das Nachrichten-Tracing aktiviert haben, werden vom System Nachrichten protokolliert, die zwischen der Oracle Service Bus-Pipeline und dem Proxy-Service (eingehende Anforderungs- und Antwort- sowie ausgehende Anforderungs- und Antwortnachrichten) ausgetauscht wurden.

    Hinweis: Um das Tracing in der Log-Datei oder im Standardausgang (Serverkonsole) anzuzeigen, muss die Anmeldung bei Oracle WebLogic Server auf die folgenden Dringlichkeitsstufen eingestellt werden:

    Mindestdringlichkeit für das Log: Info
    Log-Datei: Info
    Standardausgang: Info

    Weitere Informationen zum Einstellen der Dringlichkeitsstufen für das Log finden Sie im entsprechenden Thema unter Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

    Beachten Sie, dass im betreffenden Fall auch protokollierte ausgehende Nachrichten Wiederholungsanzahl, Fehlercode und Fehlermeldung beinhalten können.
    Wählen Sie aus der Dropdown-Liste Detailebene eine der folgenden Detailebenen:
    • Terse: Zeigt Datum, Zeit, Service-Art, Service-Name und URI an
    • Header: Zeigt Terse-Daten zusammen mit der XML-Darstellung der Nachrichtenmetadaten an
    • Vollständig: Zeigt Header-Daten zusammen mit der RAW-Payload sowie vorhandenen Anhängen an
    Wenn Sie in der Dropdown-Liste Detailebene den Wert "Vollständig" wählen, geben Sie die maximale Größe (in Kilobyte) für die Nachrichten-Payload im Feld Payload Tracing-Grenzwert an. Geben Sie außerdem die Standardcodierung für das Logging der Payload im Feld Standardcodierung an. Dies kann sinnvoll sein, wenn Sie binäre Payloads oder SOAP-Nachrichten mit Binäranhängen protokollieren.
    Der Wert für die Standardcodierung kann "Base64" oder jede von Java unterstützte Codierung sein, wie in http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html definiert.
    Wenn Sie das Feld Standardcodierung leer lassen, verwendet Oracle Service Bus für die Payload die Standardcodierung des Hosts. Die Standardcodierung hängt von einer Kombination aus JVM, dem zugrunde liegenden Betriebssystem (BS) und den Gebietsschemaeinstellungen auf BS-Ebene ab.
    Wenn die im Feld Standardcodierung angegebene Einstellung nicht verwendet werden kann (z.B. weil sie für die Konfiguration keine gültige Option ist), verwendet Oracle Service Bus die Base64-Codierung für die Payload.

    Deaktivieren Sie neben Nachrichten-Tracing das Kontrollkästchen Aktiviert.
