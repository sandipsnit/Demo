Einem Proxy-Service zugeordnete WSDL exportieren

Sie können die WSDL eines WSDL-basierten Proxy-Service exportieren, sodass Sie die WSDL in einem externen Tool (beispielsweise einer IDE) anzeigen oder ändern können. Die WSDL wird als JAR-Datei exportiert.

Beachten Sie, dass diese Funktion sich von der Funktion "Ressourcen exportieren" im Modul "Systemadministration", die Sie verwenden, um Ressourcen zwischen zwei Domains zu verschieben und zwischenzuspeichern, unterscheidet. SieheRessourcen exportieren.

Bevor Sie beginnen

Sie können eine WSDL nur exportieren, wenn Sie sich außerhalb einer Session befinden. Siehe Change Center verwenden.

So exportieren Sie eine WSDL:

  1. Führen Sie eine der folgenden Aktionen aus:
    • Wählen Sie Ressourcenbrowser > Proxy-Services, um die Seite Zusammenfassung der Proxy-Services anzuzeigen.
    • Wählen Sie Project Explorer, und wählen Sie das Projekt oder den Ordner, in dem der Proxy-Service enthalten ist, den Sie als WSDL exportieren möchten. Die Seite "Projekt-/Ordneransicht" wird angezeigt.
  2. Klicken Sie in der Zeile des Proxy-Service, dessen WSDL Sie exportieren möchten, auf das Symbol WSDL exportieren. Sie werden in einem Dialogfeld aufgefordert, die exportierte JAR-Datei zu öffnen oder zu speichern.
  3. Klicken Sie im Dialogfeld auf Öffnen, um die Datei zu öffnen, oder klicken Sie auf Speichern, um sie zu speichern.
