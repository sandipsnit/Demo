Seite "Proxy-Service Policy"

Die Seite Proxy-Service anzeigen - Policys ist eine Seite von mehreren, auf denen Proxy-Services bearbeitet werden können, wie in Proxy-Services erstellen und konfigurieren und Proxy-Service-Konfigurationen bearbeiten beschrieben.

Auf dieser Seite konfigurieren Sie Policy-Einstellungen für einen Proxyservice vom Servicetyp "WSDL-Webservice", "Messaging Service", "Beliebiger SOAP-Service" oder "Beliebiger XML-Service".In Tabelle 19-20 wird die Verwendung dieser Seite beschrieben. Um OWSM-Policys zusammen mit den Proxyservices WSDL-Web Service, Messaging Service oder Beliebiger XML-Service verwenden zu können, muss das Protokoll auf HTTP gesetzt sein.

Für WSDL-basierte Services werden WLS 9.2-Policys, die an den Service angebunden sind, in der effektiven WSDL zugänglich gemacht (inline eingereiht). Abstrakte Policys werden vor der Inline-Einreihung vorverarbeitet. OWSM-Policys werden nach Referenz angebunden und nicht in die effektive WSDL eingereiht.

Service Policy-Konfiguration
Das Feld "Service Policy-Konfiguration" enthält folgende Optionen:
  • Aus OWSM-Policy Store: Policys werden im zentralen von Oracle Enterprise Manager verwalteten Policy Store von Oracle Web Services Manager definiert. Die Policys unterstützen die Token-Profile WS-Security 1.0/1.1, SAML 1.1/2.0, Kerberos und andere Branchenstandards.

    Suchen Sie die gewünschten Policys mit der Filterfunktion. Für Proxy-Services werden nur Service-Policys angezeigt.

    • Weitere Informationen finden Sie unter "Securing Oracle Service Bus with Oracle Web Services Manager" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

  • Von WSDL: Wählen Sie diese Option, wenn die Service-Policy der WSDL zugeordnet ist, die dem Service zugrunde liegt. Die Policys unterstützen WS-Security 1.0, SAML 1.1 und andere Branchenstandards.

    • Mit dieser Option können Sie (schreibgeschützte) Anforderungs- und Antwort-Policys aus der WSDL anzeigen.

  • Unter Von vordefinierter Policy oder WS-Policy-Ressource können Sie Policys auf Serviceebene, Policys auf Vorgangsebene (in diesem Fall gilt die Policy sowohl für Anforderungs- als auch für Antwortnachrichten), Anforderungs-Policys und Antwort-Policys von der Oracle Service Bus Administration-Konsole hinzufügen.

    Policys werden entweder in Oracle WebLogic Server vordefiniert oder im Oracle Service Bus mit einer WS-Policy-Ressource vom Benutzer definiert. Diese Policys unterstützen WS-Security 1.0, SAML 1.1 und andere Branchenstandards.

    • Mit dieser Option können Sie Policys auf Service-, Vorgangs-, Anforderungs- und Antwortebene hinzufügen.

Hinweis: Die Policy Binding-Modelle schließen sich gegenseitig aus. Sie dürfen nur einen Policy-Typ in einem Service verwenden. Wenn Sie Policys direkt an den Service binden, werden sämtliche WSDL-basierten Policys ignoriert.
