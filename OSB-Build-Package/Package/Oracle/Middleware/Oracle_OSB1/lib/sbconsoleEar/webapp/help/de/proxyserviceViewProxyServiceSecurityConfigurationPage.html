Seite "Proxy-Service-Sicherheit"

Die Seite Proxy-Service anzeigen - Sicherheit ist eine von mehreren Seiten für das Bearbeiten von Proxy-Services. Weitere Informationen hierzu finden Sie unter Proxy-Services erstellen und konfigurieren und unter Proxy-Service-Konfigurationen bearbeiten.

Auf dieser Seite können Sie Sicherheitseinstellungen für einen Proxy-Service konfigurieren.

Die Felder auf dieser Seite hängen von der Verwendung von Policys im Proxy-Service ab. Beispiel: Wenn der Proxy-Service OWSM-Policys verwendet (empfohlen), wird der Abschnitt "Policy Overrides" auf der Seite "Sicherheit" angezeigt. Weitere Informationen finden Sie unter "Securing Oracle Service Bus with Oracle Web Services Manager" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Bei WLS 9.2-Policys ist die Konfiguration für benutzerdefinierte Benutzernamen/Kennwörter und benutzerdefinierte Token ähnlich. In beiden Fällen geben Sie XPath-Ausdrücke an, mit denen Oracle Service Bus die erforderlichen Informationen finden kann. Die Root dieser XPath-Ausdrücke lautet wie folgt:

  • Wenn das Service Binding "Beliebiges SOAP" oder "WSDL-SOAP" ist, verwenden Sie soap-env:Envelope/soap-env:Header.
  • Wenn das Service Binding nicht SOAP-basiert ist, verwenden Sie soap-env:Body.

Alle XPath-Ausdrücke müssen in einem gültigen XPath-Format angegeben werden. In den XPath-Ausdrücken muss die XPath-Syntax "declare namespace" verwendet werden, um verwendete Namespaces zu deklarieren. Beispiel:

declare namespace
ns='http://webservices.example.com/MyExampleService';)

Web Services-Sicherheitskonfiguration
WS-Sicherheits-Header verarbeiten
Wenn eine Clientanforderung einen WS-Sicherheits-Header enthält, können Sie entscheiden, ob dieser Header verarbeitet werden soll oder nicht:
  • Wenn Sie "Ja" wählen: In einem Szenario mit einer aktiven Zwischenstufe wendet der Client WS-Security auf die Anforderungs- und/oder Antwortnachrichten an. Der Proxy-Service verarbeitet den Sicherheits-Header und erzwingt die WS-Security Policy. Diese Option wird automatisch auf "Ja" gesetzt, wenn ein Proxy-Service auf einer WSDL basiert, die eine Sicherheits-Policy hat, oder wenn Sie dem Service auf der Seite "Service Policy-Konfiguration" eine Policy anfügen.
  • Wenn Sie "Nein" wählen: In einem WS-Security Passthrough-Szenario wendet der Client WS-Security auf die Anforderungs- und/oder Antwortnachrichten an. Der Proxy-Service verarbeitet den Sicherheits-Header nicht. Stattdessen übergibt er die gesicherte Anforderungsnachricht unverändert an einen Business-Service.
Zugriffskontrolle
Transportzugriffskontrolle
Sie können für alle Proxy-Services eine Policy auf Transportebene erstellen. Dabei wird eine Sicherheitsprüfung durchgeführt, wenn ein Client versucht, eine Verbindung mit dem Proxy-Service herzustellen. Nur Anforderungen von Benutzern, die in der Policy auf Transportebene enthalten sind, dürfen bearbeitet werden.
Nachrichtenzugriffskontrolle
Bei einer Zugriffskontroll-Policy auf Nachrichtenebene wird eine Sicherheitsprüfung durchgeführt, wenn ein Client versucht, einen Proxy-Service mit Sicherheit auf Nachrichtenebene aufzurufen. In folgenden Fällen können Sie eine Zugriffskontroll-Policy auf Nachrichtenebene erstellen:
  • Für Proxy-Services, bei denen es sich um aktive Web Service Security-Zwischenstufen handelt. Das heißt, der WS-Sicherheits-Header wird verarbeitet.
  • Für Proxy-Services, für die Sie auf dieser Seite eine benutzerdefinierte Authentifizierung festlegen.
Nur Benutzer, die in der Policy auf Nachrichtenebene enthalten sind, dürfen den Vorgang aufrufen.
Benutzerdefinierte Authentifizierung
 (Nachrichtenebene)
Authentifizierungstyp:
Wählen Sie eine der folgenden Optionen:
  • Wählen Sie Keine, wenn der Proxy-Service keine benutzerdefinierte Authentifizierung verwenden soll.
  • Name und Kennwort des benutzerdefinierten Benutzers

    • Wenn Sie diese Option wählen, müssen auch Werte in den Feldern XPath des Benutzernamens und XPath des Benutzerkennworts eingegeben werden.

  • Benutzerdefiniertes Token

    • Wenn Sie diese Option wählen, müssen Sie auch einen Tokentyp wählen und Werte im Feld Token-XPath eingeben.

XPath des Benutzernamens
Diese Option ist nur verfügbar, wenn die Option Benutzerdefinierte Authentifizierungseinstellungen auf Name und Kennwort des benutzerdefinierten Benutzers gesetzt ist. Diese Option ist erforderlich, sofern sie verfügbar ist.
Geben Sie den Benutzernamen als XPath-Ausdruck ein.
Der XPath-Ausdruck wird gegebenenfalls anhand der Nachrichten-Header oder der Payload ausgewertet. Dadurch kann in Oracle Service Bus der Benutzername für die benutzerdefinierte Authentifizierung ermittelt werden.
XPath des Benutzerkennworts
Diese Option ist nur verfügbar, wenn die Option Benutzerdefinierte Authentifizierungseinstellungen auf Name und Kennwort des benutzerdefinierten Benutzers gesetzt ist. Diese Option ist erforderlich, sofern sie verfügbar ist.
Geben Sie das Kennwort als XPath-Ausdruck ein.
Der XPath-Ausdruck wird gegebenenfalls anhand der Nachrichten-Header oder der Payload ausgewertet. Dadurch können in Oracle Service Bus die Kennwortwerte für die benutzerdefinierte Authentifizierung ermittelt werden.
Token-Typ
Diese Option ist nur verfügbar, wenn die Option Benutzerdefinierte Authentifizierungseinstellungen auf Benutzerdefiniertes Token gesetzt ist. Diese Option ist erforderlich, sofern sie verfügbar ist.
Wählen Sie den Tokentyp aus der Dropdown-Liste. Es sind nur die aktiven Tokentypen verfügbar, die für einen WebLogic Server-Identity Assertion-Provider konfiguriert wurden. Weitere Informationen finden Sie unter "Configuring Identity Assertion Providers for Custom Tokens" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660.
Token-XPath
Diese Option ist nur verfügbar, wenn die Option Benutzerdefinierte Authentifizierungseinstellungen auf Benutzerdefiniertes Token gesetzt ist. Diese Option ist erforderlich, sofern sie verfügbar ist.
Geben Sie einen XPath-Ausdruck für einen Pfad zum benutzerdefinierten Token ein. In Oracle Service Bus wird der Token-XPath-Ausdruck gegebenenfalls anhand der Nachrichten-Header oder der Payload ausgewertet, um das Token für die benutzerdefinierte Authentifizierung zu ermitteln.
Kontexteigenschaften
Geben Sie optional eine oder mehrere Kontexteigenschaften an, um zusätzliche Kontextinformationen an den Sicherheitsprovider für die Authentifizierung (Name und Kennwort des benutzerdefinierten Benutzers) oder der Identity Assertion (Benutzerdefiniertes Token) zu übergeben.
Mit Kontexteigenschaften können Sie (über das ContextHandler-Interface) zusätzliche Informationen an das WebLogic-Sicherheits-Framework übergeben, sodass ein Sicherheitsprovider Kontextinformationen abrufen kann. Weitere Informationen finden Sie unter "Context Properties Are Passed to Security Providers" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500.
Geben Sie den Eigenschaftsnamen als literale Zeichenfolge und den Werte-Selector als gültigen XPath-Ausdruck ein. (XPath-Ausdrücke können ebenfalls literale Zeichenfolgen sein.)
Die XPath-Ausdrücke werden anhand desselben Nachrichtenteiles ausgewertet, der für das benutzerdefinierte Token, den benutzerdefinierten Benutzernamen oder das benutzerdefinierte Kennwort verwendet wird. Das heißt, die XPath-Ausdrücke für Werte-Selector werden für SOAP-basierte Proxy-Services anhand des Headers und für nicht SOAP-basierte Proxy-Services anhand der Payload ausgewertet.
Der XPath-Ausdruck wird zur Laufzeit ausgewertet und stellt den Wert der Eigenschaft dar. Bei einem ContextHandler handelt es sich eigentlich um eine Namen-/Wertliste. Daher muss ein Sicherheitsprovider wissen, nach welchen Namen er suchen soll. Aus diesem Grund werden die XPath-Ausdrücke nur ausgewertet, wenn ein Sicherheitsprovider den Wert einer dieser benutzerdefinierten Eigenschaften anfordert.
Um diese Kontexteigenschaft hinzuzufügen, klicken Sie auf Eigenschaft hinzufügen. Sie können mehrere Kontexteigenschaften hinzufügen.
