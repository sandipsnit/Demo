Informationen zu XQuery-Editorlayouts und -Tasks

Der XQuery-Ausdruckseditor, der XQuery-Bedingungseditor und der XPath-Ausdruckseditor bestehen jeweils aus den folgenden Komponenten:

  • Paletten
  • Workspace
  • Property Inspector

Paletten

Der linke Bereich der einzelnen Editoren enthält jeweils die unten genannten Paletten. Klicken Sie in einem beliebigen Editor auf den Namen einer Palette, um sie anzuzeigen. Jede Palette enthält Entitäten, die Sie in Ausdrücke im Editor einfügen können.

  • In der Palette Namespace-Definitionen sind standardmäßige Oracle Service Bus Namespaces, variable Namespaces und benutzerdefinierte Namespaces aufgelistet. Sie können neue Namespaces definieren, die dann zu der Liste der benutzerdefinierten Namespaces hinzugefügt werden.

    • Weitere Informationen finden Sie in:

    • Ausdrücke in den Textfeldern des Editor-Workspace erstellen
    • Namespaces zur Verwendung in Inline-Ausdrücken erstellen
  • In der Palette XQuery-Funktionen sind eine Reihe Standard-XQuery-Funktionen aufgelistet. Wenn Sie eine Funktion in einen Ausdruck einfügen, werden Platzhalter für Parameterwerte verwendet, die Sie eingeben müssen.

    • Weitere Informationen finden Sie in:

    • Ausdrücke in den Textfeldern des Editor-Workspace erstellen
    • Benutzerdefinierte XPath-Funktionen in XQuery-Editoren erstellen
    • Externe XQuery-Ressourcen an eingebettete XQuerys binden
  • Die Palette Variablenstrukturen liefert eine Reihe von Tools zum Einfügen von Variablen und Pfaden in Variablen, und zwar mithilfe von XPath-Ausdrücken.

    • Variablenstrukturen sind grafische Darstellungen von Variablen oder Variablenpfaden, die im Editor angezeigt werden. Mit ihrer Hilfe können Sie Variablenstrukturen darstellen und eingebettete XQuery-Ausdrücke erstellen, die auf den Content der Variablen verweisen.

    Hinweis: Variablenstrukturen erstellen keine Variablen. Variablen werden zur Laufzeit als das Ziel der Aktion "Zuweisen" in der Stage erstellt.

    Oracle Service Bus stellt verschiedene vordefinierte Nachrichtenkontextvariablen, wie (Anhänge, Text, Titel, Ausgehend und Eingehend, bereit, deren Kontext Sie als Variablenstruktur anzeigen können. Sie können außerdem eigene Variablenstrukturen definieren.

    Weitere Informationen hierzu finden Sie unter

    • Ausdrücke in den Textfeldern des Editor-Workspace erstellen
    • Variablenstrukturen in den XQuery-Editoren erstellen

Workspace

Rechts auf der Seite befindet sich der Workspace zum Erstellen von XQuery-Ausdruck, XQuery-Bedingung oder XPath. In den drei Editoren ist der Workspace jeweils unterschiedlich.

Property Inspector

In allen drei Editoren wird der Property Inspector unten rechts auf der Seite angezeigt. Wenn Sie ein Element aus einer der Paletten wählen, um es dem Ausdruck hinzuzufügen, wird dieses Element im Property Inspector angezeigt. Sie können das Element dann in den Workspace einfügen. Siehe Ausdrücke in den Textfeldern des Editor-Workspace erstellen.
