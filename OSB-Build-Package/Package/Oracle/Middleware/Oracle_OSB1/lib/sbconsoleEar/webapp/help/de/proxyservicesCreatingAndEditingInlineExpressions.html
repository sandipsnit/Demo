Inline XQuery und XPath-Ausdrücke erstellen und bearbeiten

Wenn Sie auf der Seite "Stage-Konfiguration bearbeiten" Aktionen zu Stages oder Route-Knoten hinzufügen, wird auf der Seite eine Template-Struktur angezeigt, von der Sie zur Eingabe von Konfigurationsdetails aufgefordert werden. In Abbildung 22-1 wird ein Beispiel dargestellt.

Abbildung 22-1 Beispiel für ein Aktionskonfigurations-Template

Wenn es für den Kontext zutreffend ist, werden im Template Links für den Zugriff auf XQuery-Editoren bereitgestellt. Hier können Sie Ausdrücke erstellen, die je nach Kontext in der Aktion inline ausgeführt werden.

Bevor Sie beginnen

Bei diesen Anweisungen wird vorausgesetzt, dass Sie auf der Seite "Stage-Konfiguration bearbeiten" eine Aktion einer Pipeline Stage, einer Error Handler Stage oder eines Route-Knotens erstellen oder bearbeiten. Weitere Informationen finden Sie unter Nachrichtenfluss erstellen, bearbeiten und anzeigen und unter Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.

So erstellen oder ändern Sie einen Inline-Ausdruck:

  1. Suchen Sie auf der Seite Stage-Konfiguration bearbeiten die Stelle in der Aktion, an der Sie den Ausdruck hinzufügen oder bearbeiten möchten.
  2. Klicken Sie auf den Ausdruckslink, um den für den Kontext geeigneten Editor zu öffnen. Wenn noch kein Ausdruck definiert wurde, gibt der Link Auskunft darüber, welche Art von Ausdruck Sie an dieser Position verwenden können:
    • Klicken Sie auf <Expression>, um einen XQuery-Ausdruck zu erstellen. Sie können auch eine XQuery- oder XSLT-Ressource importieren, die außerhalb von Oracle Service Bus erstellt wurde, und sie in die Inline XQuery einbinden.
    • Klicken Sie auf <Condition>, um einen XQuery-Konditionalausdruck für eine if... then...-Aktion zu erstellen.
    • Klicken Sie auf <XPath>, um einen XPath-Ausdruck für eine Nachrichtenkontextvariable zu erstellen.

      • Wenn in einer Position bereits ein Ausdruck definiert wurde, wird der Link <Expression>, <Condition> oder <XPath> durch einen Link ersetzt, in dem der Ausdruck selbst (Beispiel: true()[icon], ) oder ein Teil eines zu langen Ausdrucks (Beispiel: $body/urn:POSta...[icon]) angezeigt wird. Klicken Sie auf den Ausdruck (oder den Teil des Ausdrucks), um den Ausdruck im geeigneten Editor anzuzeigen.

  3. Erstellen Sie den Ausdruck gemäß den Beschreibungen in den folgenden Themen:
    • Ausdrücke in den Textfeldern des Editor-Workspace erstellen
    • Namespaces zur Verwendung in Inline-Ausdrücken erstellen
    • Variablenstrukturen in den XQuery-Editoren erstellen
    • Benutzerdefinierte XPath-Funktionen in XQuery-Editoren erstellen
    • Externe XQuery-Ressourcen an eingebettete XQuerys binden
    • Externe XSLT-Ressourcen an Inline XQuerys binden
    • Dynamische XQuery-Ausdrücke an Inline XQuerys binden
    • XQuery-Vergleichsausdrücke mit der Builder-Option eingeben
    • Unäre Ausdrücke mit der Builder-Option eingeben
  4. Führen Sie optional einen oder beide der folgenden Schritte aus:
    • Klicken Sie auf Validieren, um den Ausdruck zu validieren.
    • Um den Ausdruck zu testen, klicken Sie auf Testen. Siehe XQuery-Tests durchführen.
  5. Klicken Sie auf Speichern, um den Editor zu schließen und den Ausdruck in die Aktion einzufügen.

Wenn Sie fertig sind

Setzen Sie die Konfiguration der Aktion fort, wie in Aktionen im Nachrichtenfluss hinzufügen und bearbeiten erläutert.
