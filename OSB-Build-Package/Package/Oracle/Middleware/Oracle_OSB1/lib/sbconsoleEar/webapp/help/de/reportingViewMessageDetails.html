Nachrichtendetails anzeigen

Auf dieser Seite können Sie Details einer bestimmten Nachricht anzeigen.

  1. Suchen Sie die Nachricht. Siehe Berichtmeldungen anzeigen.
  2. Klicken Sie auf den Berichtsindex. Auf der Seite Nachrichtendetails anzeigen werden die Daten der Allgemeinen Konfiguration aus der Tabelle 27-3 angezeigt.

    3. Tabelle 27-3 Nachrichtenkonfigurationsparameter 
    Eigenschaft
    Beschreibung
    Nachrichten-ID
    Die eindeutige ID dieser Nachricht.
    Datenbank-Zeitstempel
    Datum und Uhrzeit, zu der die Nachricht in der Datenbank registriert wurde.
    Logging-Zeitpunkt
    Datum und Uhrzeit, zu der die Nachricht geloggt wurde.
    Server-Name
    Der Name des Servers, auf dem die Nachricht generiert wurde.
    Status
    Status der Pipeline, in der die Nachricht generiert wurde.
    ANFORDERUNG - Gibt an, dass die Reporting-Aktion in einer Anforderungs-Pipeline ausgeführt wurde.
    ANTWORT - Gibt an, dass die Reporting-Aktion in einer Antwort-Pipeline ausgeführt wurde.
    FEHLER - Die Aktion wird im Error Handler auf Service-Ebene ausgeführt.
    Knotenname
    Der Knotenname der Pipeline, in der die Nachricht generiert wurde.
    Pipeline-Name
    Der Name der Pipeline, in der die Nachricht generiert wurde.
    Stage-Name
    Der Name des Stage, in dem die Nachricht generiert wurde.

    Auf der Seite Eingehender Service werden die Informationen aus der Tabelle 27-4 angezeigt.

    Tabelle 27-4 Eingehender Service - Details 
    Eigenschaft
    Beschreibung
    Name
    Der mit dieser Nachricht verknüpfte eingehende Proxy-Service. Dies ist ein Link zur Seite Proxy-Service anzeigen - Konfigurationsdetails. Siehe Proxy-Service-Konfigurationen bearbeiten.
    URI
    Der mit dem Proxy-Service verknüpfte URI.
    Vorgang
    Der mit dieser Nachricht verknüpfte eingehende Vorgang.

    Auf der Seite Ausgehender Service werden die Informationen aus der Tabelle 27-5 angezeigt.

    Tabelle 27-5 Ausgehender Service - Details 
    Eigenschaft
    Beschreibung
    Name
    Der mit dieser Nachricht verknüpfte ausgehende Business-Service. Dies ist ein Link zur Seite Business-Service anzeigen - Konfigurationsdetails. Siehe Business-Service-Konfigurationen bearbeiten.
    URI
    Der URI zum Endpunkt des ausgehenden Business-Service.
    Vorgang
    Name des Vorgangs, der in dem ausgehenden Service aufgerufen wird.

    Auf der Seite Berichtsindex werden die Informationen aus der Tabelle 27-6 angezeigt.

    Tabelle 27-6 Berichtsindex - Details 
    Eigenschaft
    Beschreibung
    Berichtsindextext
    Berichtindizes sind Schlüsselwertpaare, die Sie verwenden, um Schlüssel-IDs von einer Nachrichtenkontextvariablen zu extrahieren. Schlüssel sind für das Filtern von Nachrichten praktisch. Sie verwenden die Berichtaktion, um Schlüsselwertpaare hinzuzufügen. Siehe Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.

    Auf der Seite Fault werden die Informationen aus der Tabelle 27-7 angezeigt.

    Tabelle 27-7 Fault-Informationen - Details 
    Eigenschaft
    Beschreibung
    Fehlercode
    Der mit dieser Nachricht verknüpfte Fehlercode, sofern vorhanden.
    Sie geben bei der Konfiguration der Auslösefehleraktion Fehlercodes an. Siehe "Umgang mit Fehlern im Nachrichtenfluss" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG232 und Proxy-Service Error-Handler hinzufügen.
    Grund
    Der Grund für den Fehlercode.
    Details
    Die mit diesem Fehlercode verknüpften Fault-Details. Diese Details stellen, sofern vorhanden, eine Stack Trace dar, an der ein bestimmter Fehler aufgetreten ist. Sie können aufgrund der Größenbeschränkung in der Datenbank abgeschnitten werden. Diese Beschränkung liegt bei 2048 Zeichen.

    Auf der Seite Hauptteil des Berichts werden die Informationen aus der Tabelle 27-8 angezeigt.

    Tabelle 27-8 Hauptteil des Berichts - Details 
    Eigenschaft
    Beschreibung
    Hauptteil des Berichts - Text
    Dieser Link öffnet einen anderen Browser, der die die Details des Hauptteils des Berichts anzeigt. Sie verwenden einen Ausdruck in der Berichtsaktion, um den Hauptteil des Berichts zu erfassen. Siehe Vorgehensweise für die Berichtsaktion in Aktionen im Nachrichtenfluss hinzufügen und bearbeiten.

  3. Klicken Sie auf OK, wenn Sie mit dem Anzeigen der Details einer bestimmten Nachricht fertig sind.
