Alert-Ziele hinzufügen

Ein Alert-Ziel ist eine Zieladresse für Alert-Benachrichtigungen in Oracle Service Bus. Sie erstellen Alert-Zielressourcen auf der Seite Alert-Ziel erstellen.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie erst Project Explorer und dann ein Projekt oder einen Ordner, dem das Alert-Ziel hinzugefügt werden soll. Die Seite Projekt-/Ordneransicht wird angezeigt.
  3. Wählen Sie aus der Dropdown-Liste Ressource erstellen die Option Alert-Ziel, um die Seite Alert-Ziel erstellen anzuzeigen.
  4. Geben Sie im Feld Ressourcenname einen eindeutigen Namen für dieses Alert-Ziel ein. Dies ist ein obligatorisches Feld.

    Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
  5. Geben Sie im Feld Ressourcenbeschreibung eine Beschreibung des Alert-Ziels ein.
  6. Wählen Sie eines oder alle der folgenden Ziele, die in diese Alert-Zielressource mit aufgenommen werden sollen.
    7. Hinweis: Alerts werden zur Laufzeit in der Oracle Service Bus Administration-Konsole aggregiert. Dort können Sie unter Vorgänge > Dashboards angezeigt werden.
    • SNMP-Trap: Wenn Sie "SNMP-Trap" angeben, werden Alerts als SNM-Traps gesendet und können mit jedem beliebigen Enterprise Monitoring System (ESM) eines Drittanbieters verarbeitet werden
    • Reporting: Wenn Sie "Reporting" angeben, werden Alerts an das Oracle Service Bus Reporting-Modul gesendet und können mit einem benutzerdefinierten Reporting Provider erfasst werden, der seinerseits mit dem Reporting API Provider von Oracle Service Bus entwickelt werden kann. So können Drittanbieter Alerts in benutzerdefiniertem Java-Code empfangen und verarbeiten.
    • Alert Logging: Wenn Sie Ja wählen, werden an dieses Alert-Ziel gesendete Alerts im Alert Log protokolliert.
  7. Einen E-Mail-Empfänger der Definition für die Alert-Ressourcendefinition hinzufügen. Siehe E-Mail-Empfänger hinzufügen.
  8. Ein JMS-Ziel der Definition für die Alert-Ressourcendefinition hinzufügen. Siehe JMS-Ziele hinzufügen.
  9. Klicken Sie auf Speichern. Das neue Alert-Ziel wird in der aktuellen Session gespeichert.
  10. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
