Split Joins hinzufügen

Bevor Sie beginnen

Split Joins benötigen eine .flow-Datei, die erstellt wird, wenn Sie einen Split Join mit den Oracle Service Bus-Plug-Ins für Eclipse erstellen oder wenn Sie eine Split Join-Konfigurations-JAR-Datei in das Plug-In importieren. Die .flow-Datei kann Referenzen zu Proxy-Services, Business-Services oder zu externen WSDL-Ressourcen haben. Ressourcen, von denen der Split Join abhängig ist, müssen auf dem Server vorhanden sein, bevor der Split Join aktiviert wird.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer und dann ein Projekt oder einen Ordner, in dem Sie den Split Join hinzufügen möchten. Die Seite Projektansicht/Ordneransicht wird angezeigt.
  3. Wählen Sie von der Dropdown-Liste Ressource erstellen den Wert Split Join aus, um die Seite Neue Split Join-Ressource erstellen anzuzeigen.

    4. Geben Sie die in der Tabelle 9-2 angezeigten Parameter an.

    Tabelle 9-2 Split Join-Parameter 
    Eigenschaft
    Verwendung
    Ressourcenname
    Geben Sie einen eindeutigen Namen für diesen Split Join ein.

    Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
    Ressourcenbeschreibung
    Geben Sie eine Beschreibung für den Split Join ein.
    Split Join
    Klicken Sie auf Durchsuchen, um eine .flow-Datei in Ihrem System zu suchen. Diese befindet sich normalerweise unter MW_HOME\user_projects\workspaces\your_workspace.
    Dispatch Policy
    Wählen Sie die Instanz des WebLogic Server Work Managers, mit der Threads (parallele Verzweigungen) für die Ausführung des Split Joins erstellt werden sollen. Der Standard-Work Manager wird verwendet, wenn kein anderer Work Manager vorhanden ist.

  4. Klicken Sie auf Speichern. Der Split Join wird in der aktuellen Session gespeichert.
    5. Hinweis: Ressourcen, von denen der Split Join abhängig ist, müssen auf dem Server vorhanden sein, bevor der Split Join aktiviert wird.
  5. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
