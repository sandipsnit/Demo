MQ-Verbindungen hinzufügen

Bevor Sie beginnen

In Oracle Fusion Middleware Supported System Configurations unter http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html finden Sie Informationen zu unterstützten Versionen von WebSphere MQ in Oracle Service Bus. Die WebSphere MQ-Client-Library ist nicht im Oracle Service Bus-Installationsprogramm enthalten. Sie erhalten die WebSphere MQ-Client-Library bei der Installation von WebSphere MQ.

Damit Sie die MQ-Verbindungsressourcen und den MQ-Transport in Oracle Service Bus konfigurieren und verwenden können, müssen Sie die MQ-Client-Library dem Classpath hinzufügen. Eine Möglichkeit ist das Kopieren der Library "com.ibm.mq.jar library" in das Verzeichnis DOMAIN_NAME/lib, wobei DOMAIN_NAME der Name des Verzeichnisses ist, in dem sich die Domain befindet (in der Regel MW_HOME\user_projects\domains\DOMAIN_NAME).

So fügen Sie eine MQ-Verbindungsressource hinzu

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer und anschließend ein Projekt oder einen Ordner, dem Sie die MQ-Verbindungsressource hinzufügen möchten. Die Seite Projekt-/Ordneransicht wird angezeigt.
  3. Wählen Sie aus der Dropdown-Liste Ressource erstellen unter Utility den Eintrag MQ-Verbindung, um die Seite Neue MQ-Verbindungsressource erstellen anzuzeigen.

    4. Geben Sie die in Tabelle 8-2 angezeigten Verbindungsparameter an.

    Tabelle 8-2 MQ-Verbindungsparameter 
    Eigenschaft
    Verwendung
    Ressourcenname
    Geben Sie einen eindeutigen Namen für diese MQ-Verbindungsressource ein.

    Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
    Ressourcenbeschreibung
    Geben Sie eine Beschreibung für die MQ-Verbindungsressource ein.
    Verbindungstyp
    Wählen Sie einen der folgenden Modi zum Herstellen einer Verbindung mit dem MQ Queue Manager aus:
    • TCP-Modus: Verwenden Sie TCP/IP, um eine Verbindung zu einem Queue Manager herzustellen, der sich nicht auf demselben Rechner befindet wie Oracle Service Bus.
    • Binding-Modus: Verwenden Sie diesen Modus, um eine Verbindung zu einem Queue Manager herzustellen, der sich auf demselben Rechner befindet wie Oracle Service Bus.
    MQ-Hostname
    Nur bei Verbindungen über den TCP-Modus:
    Geben Sie den Hostnamen des MQ Queue Managers ein.
    MQ-Port-Nummer
    Nur bei Verbindungen über den TCP-Modus:
    Geben Sie die Port-Nummer des MQ Queue Manager-Listeners ein.
    MQ Queue Manager-Name
    Geben Sie den Namen des MQ Queue Managers ein, mit dem die Verbindung hergestellt werden soll.
    Queue Manager-CCSID
    Nur bei Verbindungen über den TCP-Modus:
    Der codierte Zeichensatzbezeichner (Coded Character Set Identifier, CCSID), der zum Herstellen einer Verbindung verwendet werden soll. Der CCSID wird hauptsächlich für die Internationalisierungsunterstützung verwendet.
    Weitere Informationen finden Sie unter "WebSphere MQ Fundamentals" unter http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
    MQ Queue Manager-Kanalname
    Nur bei Verbindungen über den TCP-Modus:
    Geben Sie den Kanalnamen für die Queue Manager-Serververbindung ein.
    SSL erforderlich
    Nur bei Verbindungen über den TCP-Modus:
    Aktivieren Sie das Kontrollkästchen für die Verwendung von SSL beim Senden von Nachrichten. Nur serverseitige SSL wird aktiviert, wenn die Option Wechselseitige SSL erforderlich nicht aktiviert ist.
    Cipher Suite
    Diese Option ist nur dann verfügbar, wenn das Kontrollkästchen SSL erforderlich aktiviert ist.
    Wählen Sie den Cipher Suite-Algorithmus aus, der von der SSL verwendet werden soll.
    Der Cipher Suite-Algorithmus dient der Verschlüsselung und Entschlüsselung der Nachrichtenübermittlung zwischen dem WebSphere MQ-Server und dem WebSphere MQ-Client. Daher muss bei der Verwendung von SSL bei der Kommunikation mit einem WebSphere MQ-Server ein Cipher Suite-Algorithmus angegeben werden.
    Wechselseitige SSL erforderlich
    Diese Option ist nur dann verfügbar, wenn das Kontrollkästchen SSL erforderlich aktiviert ist.
    Aktivieren Sie dieses Kontrollkästchen, um sowohl die clientseitige als auch die serverseitige SSL-Authentifizierung zu aktivieren.
    Referenz auf den Service-Schlüsselprovider
    Wenn Sie die Option Wechselseitige SSL erforderlich aktiviert haben, müssen Sie eine Referenz auf den Service-Schlüsselprovider zum Abrufen des entsprechenden Keystore- und Truststore-Informationen angeben.
    Geben Sie den Pfad (Projekt/Ordner) und den Namen eines Service-Schlüsselproviders ein, und klicken Sie auf Durchsuchen, um einen Provider auf der Seite Service-Schlüsselprovider wählen auszuwählen.
    Referenz auf statischen Service-Account
    Nur bei Verbindungen über den TCP-Modus:
    Für die Benutzername- und Kennwort-Authentifizierung erforderlich. Geben Sie den Pfad (Projekt/Ordner) und den Namen eines statischen Service-Accounts ein, und klicken Sie auf Durchsuchen, um Service-Accounts in einem Browser auszuwählen.
    WebSphere MQ-Version
    Wählen Sie die WebSphere MQ-Version aus:
    • 5.3
    • 6.0
    • 7.0

    Hinweis: Versionen 5.3 und 6.0 werden nicht mehr von IBM unterstützt. Diese Versionen sind in Oracle Service Bus veraltet.
    Größe des MQ-Connection Pools
    Geben Sie die Größe des MQ-Connection Pools ein.
    Timeout für MQ-Verbindung
    Geben Sie ein Zeitintervall in Sekunden ein, nach dem nicht verwendete Verbindungen unterbrochen werden. Der Standardwert ist 1800 Sekunden.
    Max. Wartezeit für MQ-Verbindung
    Geben Sie die maximale Wartezeit in Millisekunden an, bis eine Verbindung verfügbar sein soll. Wenn innerhalb dieses Zeitraums keine Verbindung hergestellt wird, löst Oracle Service Bus eine Exception aus. Der Standardwert liegt bei 3000 Millisekunden.

  4. Klicken Sie auf Speichern. Die MQ-Verbindungsressource wird erstellt und in der aktuellen Session gespeichert.
  5. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
