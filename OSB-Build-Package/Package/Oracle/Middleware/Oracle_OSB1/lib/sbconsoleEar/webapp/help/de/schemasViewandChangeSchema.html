XML-Schemata bearbeiten

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Suchen Sie das XML-Schema, wie in XML-Schemas suchen erläutert.
  3. Klicken Sie auf den Namen des XML-Schemas. Auf der Seite XML-Schemadetails anzeigen werden die Daten aus Tabelle 4-2, Tabelle 4-3 und Tabelle 4-4. angezeigt.

    4. Tabelle 4-2 XML-Schema-Details 
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der dieses XML-Schema erstellt oder in die Konfiguration importiert hat.
    Zuletzt geändert am
    Das Datum oder die Uhrzeit, zu der der Benutzer dieses XML-Schema erstellt oder in die Konfiguration importiert hat. Klicken Sie auf den Link von Datum und Uhrzeit, um die Änderungshistorie dieser Ressource anzuzeigen. Siehe Seite "Änderungshistorie anzeigen".
    Referenzen
    Die Anzahl der Objekte, die dieses XML-Schema referenziert. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Siehe Referenzen auf Ressourcen anzeigen.
    Referenziert von
    Die Anzahl der Objekte, die dieses XML-Schema referenzieren. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Wenn dieses XML-Schema beispielsweise von einer bestimmten WSDL referenziert wird, wird die WSDL beim Klicken auf den Link als Referenz angezeigt. Siehe Referenzen auf Ressourcen anzeigen.
    Beschreibung
    Eine Beschreibung dieses XML-Schemas, sofern vorhanden.

    Die strukturelle Ansicht des XML-Schemas wird standardmäßig angezeigt und enthält die Informationen aus der Tabelle 4-3.

    Tabelle 4-3 XML-Strukturdetails 
    Eigenschaft
    Beschreibung
    Ziel-Namespace
    Der Namespace wird verwendet, um die Definitionen zu qualifizieren, die im XML-Schema enthalten sind.
    XML-Schemadefinitionen
    Die mit diesem XML-Schema verknüpften Attribute und Gruppen:
    • XML-Schematypen
    • XML-Schemaelemente
    • XML-Schemaimporte
    • XML-Schema umfasst/definiert neu
    Schemastatus
    Status des XML-Schemas:
    • Gültig
      Ein XML-Schema wird als gültig angezeigt, wenn alle Orte für XML-Schemata, die vom aktuellen XML-Schema eingeschlossen werden, angegeben werden. Weiter werden alle Orte für verschachtelte XML-Schemata (XML-Schemata, die in der Folge von eingeschlossenen XML-Schemata eingeschlossen werden) angegeben.
    • Ungültig
      Ein XML-Schema wird als ungültig angezeigt, wenn ein oder mehrere Orte für das XML-Schema nicht angegeben werden. Weiter kann es sein, dass der Ort für ein oder mehrere verschachtelte XML-Schemata (XML-Schemata, die in der Folge von eingeschlossenen XML-Schemata eingeschlossen werden) nicht angegeben ist. Um ein ungültiges XML-Schema aufzulösen, klicken Sie auf Referenzen bearbeiten. Siehe Nicht aufgelöste XML-Schemareferenzen auflösen.

  4. Klicken Sie auf Textansicht, um eine Textansicht der XML-Schemadetails anzuzeigen. Die Textansicht umfasst die Informationen aus der Tabelle 4-4.

    5. Tabelle 4-4 XML-Schema-Textdetails
    Eigenschaft
    Beschreibung
    Ziel-Namespace
    Der Namespace wird verwendet, um die Definitionen zu qualifizieren, die im XML-Schema enthalten sind.
    XML-Schema
    Der Text für dieses XML-Schema.

  5. Um Änderungen an den Feldern vorzunehmen, klicken Sie auf Bearbeiten. Weitere Beschreibungen zu den Feldern finden Sie unter XML-Schemas hinzufügen.
  6. Nehmen Sie die entsprechenden Bearbeitungen vor.
  7. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  8. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
