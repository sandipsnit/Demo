Sicherheitskonfiguration

Verwenden Sie das Modul Sicherheitskonfiguration, wenn Sie Sicherheitsdaten erstellen und ändern möchten, die in Oracle Service Bus eingehende Sicherheit und administrative Sicherheit verwendet werden.

Sicherheit auf eingehender Transportebene und auf Nachrichtenebene verwendet die Benutzer-, Gruppen- und Rollendaten, um eingehende Clientanforderungen zu authentifizieren. Sie wendet Zugriffskontroll-Policys an, um festzustellen, welche authentifizierten Benutzer autorisiert sind, Proxy und Business-Services zu verwenden.

Administrative Sicherheit verwendet die Benutzer-, Gruppen- und Rollendaten, um festzustellen, welche authentifizierten Benutzer autorisiert sind, Oracle Service Bus-Konfigurationsdaten zu erstellen oder zu ändern oder um Oracle Service Bus-Performance zu überwachen. Weitere Informationen finden Sie unter Benutzer, Gruppen, Sicherheitsrollen und Policys verstehen. Siehe auch "Administrative Sicherheit konfigurieren" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus und http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.

Hinweis: Sie können Benutzer, Gruppen oder Rollen nicht exportieren, wenn Sie eine Konfiguration exportieren, denn diese Objekte befinden sich in Sicherheitsproviderspeichern. Sie müssen diese Objekte erneut erstellen, wenn Sie die exportierte Konfiguration importieren oder WebLogic Server-Tools verwenden (falls verfügbar), um diese zu exportieren und zu importieren.
