Zugriffs-Policys der Nachrichtenebene bearbeiten

Auf der Seite Policy-Details anzeigen können Sie die Zugriffskontroll-Policy auf Nachrichtenebene eines Proxy-Service bearbeiten, der ein Web Service ist und so konfiguriert wurde, dass er Sicherheit auf Nachrichtenebene erfordert. Auf dieser Seite werden die in Tabelle 24-11 beschriebenen Informationen angezeigt.

Tabelle 24-11 Policy-Details 
Eigenschaft
Beschreibung
Proxy-Service-Name
Zeigt den Namen des Proxy-Service an, für den Sie auf der Seite Zugriffskontrolle für Proxy-Services die Option Policys anzeigen gewählt haben.
Provider
Zeigt die Autorisierungsprovider an, die für die Realm konfiguriert wurden.
Vorgang
Zeigt die Vorgänge des Proxy-Service an, die gesichert werden können.
Policy-Bedingungen
Zeigt die Bedingungen an, die bestimmen, welche Benutzer die Vorgänge aufrufen können, die unter Service-Vorgänge aufgeführt sind.

  1. Melden Sie sich bei der Oracle Service Bus Administration-Konsole als Benutzer mit WebLogic Server Admin-Berechtigung an. Nur Benutzer in der Admin-Rolle können die Sicherheitskonfigurationsdaten ändern. Siehe "Configuring Administrative Security" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.
  2. Suchen Sie die Access Policy. Weitere Informationen finden Sie unter Zugriffskontroll-Policys suchen.
  3. Klicken Sie auf der Seite Sicherheit in der Spalte Nachrichtenzugriffskontrolle auf den Namen des Proxy-Service oder des Vorgangs in diesem Proxy-Service. Daraufhin wird der Policy-Editor angezeigt.

    4. Sie können eine Zugriffskontroll-Policy aus einer Session heraus konfigurieren.

  4. Wählen Sie im Feld Autorisierungsprovider einen Autorisierungsprovider aus. Oracle empfiehlt die Verwendung von XACMLAuthorizer.
    5. Hinweis: Oracle Service Bus hat die Unterstützung für den Standard-WebLogic-Autorisierungsprovider verworfen. Stattdessen empfiehlt Oracle die Verwendung des WebLogic-XACML-Autorisierungsproviders.
  5. Fügen Sie Policy-Bedingungen hinzu. Weitere Informationen finden Sie unter Policy-Bedingungen hinzufügen.
  6. Klicken Sie nach der Eingabe der Bedingungen im Bereich Policy-Bedingungen auf Speichern.
