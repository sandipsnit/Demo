Benutzer, Gruppen, Sicherheitsrollen und Policys verstehen

Dieser Abschnitt enthält die folgenden Themen:

  • Benutzer
  • Gruppen
  • Rollen
  • Zugriffskontroll-Policys
  • Sicherheitskonfiguration Daten und Sessions

Benutzer

Benutzer sind Entitäten, die authentifiziert werden können. Ein Benutzer kann eine Person oder eine Software-Entität sein, wie zum Beispiel ein Web-Services-Client. Sie müssen jedem Benutzer eine eindeutige Identität (Name) innerhalb einer Sicherheits-Realm geben.

Normalerweise fallen die Benutzer, die Sie erstellen, in zwei Kategorien.

  • Clientbenutzer, die auf Ihre Proxy-Services oder Business-Services zugreifen.

    • Wenn Sie eine große Anzahl Clientbenutzer erstellen, sollten Sie erwägen, sie in Sicherheitsgruppen zu organisieren.

  • Benutzer mit Administrativrechten, die die Oracle Service Bus Administration-Konsole verwenden, um Proxy-Services, Business-Services und andere Oracle Service Bus-Ressourcen zu erstellen oder zu verändern.

    • Oracle Service Bus verwendet rollenbasierte Sicherheit für seine administrativen Funktionen. Anstatt Zugriffsrechte direkt den Benutzern zu erteilen, erteilt Oracle Service Bus administrative Rechte nur an Sicherheitsrollen. Damit ein Benutzer administrative Rechte erhält, platzieren Sie den Benutzer in einer der Standardsicherheitsgruppen, das ist eine der vordefinierten Sicherheitsrollen.

Gruppen

Wenn Sie die Verwaltung einer großen Gruppe von Benutzern erleichtern möchten, können Sie die Benutzer in Namensgruppen organisieren. Dann erteilen Sie statt Zugriffsrechten oder Rollenidentitäten für einzelne Benutzer, Rechte oder Identitäten an Gruppen.

Administrative Sicherheitsgruppen

Oracle Service Bus stellt Standardsicherheitsgruppen bereit, damit Benutzer die Möglichkeit erhalten, auf administrative Funktionen wie Erstellen von Proxy-Services zuzugreifen. Jede Gruppe ist eine der vordefinierten Oracle Service Bus-Sicherheitsrollen, denen administrative Rechte zugewiesen wurden.

Weitere Informationen finden Sie unter "Administrative Sicherheit konfigurieren" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.

Rollen

Eine Sicherheitsrolle ist eine Identität, die einem Benutzer oder einer Gruppe basierend auf Bedingungen in der Laufzeitumgebung zugeteilt werden kann. Wenn Sie Zugriffskontroll-Policys erstellen, können Sie Zugriffsrechte für eine Rolle, Gruppe oder einen Benutzer erteilen.

Offlinesetzen können Sie zwei Gruppen erstellen, MyCustomersEast und MyCustomersWest. Sie erstellen eine Sicherheitsrolle mit dem Namen PrivilegedCustomer und erstellen Bedingungen, sodass die Gruppe MyCustomersWest diese Rolle von 8:00 bis 20:00 EST inne hat und die Gruppe MyCustomersEast die Rolle von 20:00 bis 8:00 Uhr EST. Dann erstellen Sie eine Zugriffskontroll-Policy für einen Proxy-Service, der der Rolle PrivilegedCustomer Zugriff auf den Service erteilt. Unterschiedliche Benutzer werden zu unterschiedlichen Zeiten Zugriff haben, je nachdem ob Sie zu der Gruppe MyCustomersEast und MyCustomersWest gehören.

Administrative Sicherheitsrollen

Oracle Service Bus liefert vier vordefinierte Sicherheitsrollen (plus vier vordefinierte Rollen vom WebLogic Server), die administrative Rechte erteilen. Sie können die Zugriffsrechte für die administrativen Sicherheitsrollen in Oracle Service Bus nicht ändern, aber Sie können die Bedingungen ändern, unter denen ein Benutzer oder eine Gruppe in einer der Rollen ist.

Weitere Informationen über diese Rollen und die für jede Rolle verfügbaren Rechte finden Sie unter "Administrative Sicherheit konfigurieren" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.

Zugriffskontroll-Policys

Eine Zugriffskontroll-Policy gibt die Bedingungen an, unter denen Benutzer, Gruppen oder Rollen auf einen Proxy-Service zugreifen. Sie können zum Beispiel eine Policy erstellen, die Benutzer in der Rolle GoldCustomer immer erlaubt, auf einen Proxy-Service zuzugreifen, und der Benutzern in der Rolle SilverCustomer erlaubt, nur nach 24 Uhr während der Woche auf den Proxy-Service zuzugreifen.

Sie können für alle Proxy-Services eine Policy auf Transportebene erstellen. Dabei wird eine Sicherheitsprüfung durchgeführt, wenn ein Client versucht, eine Verbindung mit dem Proxy-Service herzustellen. Nur Anforderungen von Benutzern, die in der Policy auf Transportebene enthalten sind, dürfen bearbeitet werden.

Bei einer Zugriffskontroll-Policy auf Nachrichtenebene wird eine Sicherheitsprüfung durchgeführt, wenn ein Client versucht, einen Proxy-Service mit Sicherheit auf Nachrichtenebene aufzurufen. In folgenden Fällen können Sie eine Zugriffskontroll-Policy auf Nachrichtenebene erstellen:

  • Für Proxy-Services, die aktive Web Service Security-Zwischenstufen sind
  • Für Proxy-Services, die benutzerdefinierte Authentifizierung auf Nachrichtenebene haben

Nur Benutzer, die in der Policy auf Nachrichtenebene enthalten sind, dürfen den Vorgang aufrufen.

Sicherheitskonfiguration Daten und Sessions

Benutzer, Gruppen und Rollen sind in Sicherheitsprovidern gespeichert, die nicht von Oracle Service Bus-Sessions beherrscht werden. Daher können Sie diese Daten erstellen oder ändern, wenn Sie innerhalb oder außerhalb einer Session sind. Hinzufügungen oder Änderungen an diesen Daten haben sofortige Wirkung und stehen in allen Sessions zur Verfügung. Wenn Sie eine Session, in der Sie die Daten hinzugefügt oder geändert haben, verwerfen, werden die Sicherheitsdaten nicht verworfen.

Zugriffskontroll-Policys werden bei Autorisierungsprovidern gespeichert. Und das Oracle Service Bus-Repository enthält jetzt eine Referenz darauf.

Zugriffskontroll-Policys werden innerhalb einer Oracle Service Bus-Session verwaltet und nicht außerhalb der Session. Da die Änderungen innerhalb einer Session gemacht werden, können Sie diese weglassen oder verwerfen, wie bei anderen Ressourcen.

Obwohl ACLs in der Oracle Service Bus Administration-Konsole verwaltet werden können, können Sie Policys außerhalb von Oracle Service Bus ändern. Allerdings kann durch das Ändern der Policys außerhalb von Oracle Service Bus der Verweis in Oracle Service Bus veraltet oder ungültig werden.

Daher sollten Sie für ein konsistentes Management entweder ACLs komplett außerhalb von Oracle Service Bus-Sessions verwalten (mit Autorisierungsprovider MBeans oder Drittparteien Autorisierungsprovider-Tools) oder sie komplett in den Oracle Service Bus-Sessions verwalten. Eine Kombination der beiden Ansätze kann in einer inkonsistenten Ansicht der Policys enden.
