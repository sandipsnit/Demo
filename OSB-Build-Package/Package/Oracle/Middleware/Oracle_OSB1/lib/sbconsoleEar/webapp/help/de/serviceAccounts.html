Service-Accounts

Ein Service-Account liefert einen Benutzername und ein Kennwort, das Proxy-Services und Business-Services für die ausgehende Authentifizierung oder Authentifizierung zu einer lokalen oder Remote-Ressource verwendet, wie ein FTP-Server oder ein JMS-Server. Wenn beispielsweise ein Business-Service seinen Benutzernamen und ein Kennwort für die Authentifizierung auf Transportebene mit einem Web Service liefern soll, erstellen Sie einen Service-Account, der den Benutzernamen und das Kennwort angibt, dann konfigurieren Sie den Business-Service so, dass er in seiner ausgehenden Anforderung die Service-Account-Zugangsdaten enthält.

Die Benutzernamen und Kennwörter, die Sie in diese Service-Accounts eingeben, werden für die ausgehende Authentifizierung oder für die Bereitstellung von Zugangsdaten zu lokalen oder Remote-Ressourcen verwendet. Benutzernamen und Kennwörter, die Sie im Modul Sicherheitskonfiguration der Oracle Service Bus Administration-Konsole eingeben, werden für die eingehendeAuthentifizierung und die Authentifizierung von administrativen Anforderungen verwendet. Siehe Service-Accounts festlegen.
