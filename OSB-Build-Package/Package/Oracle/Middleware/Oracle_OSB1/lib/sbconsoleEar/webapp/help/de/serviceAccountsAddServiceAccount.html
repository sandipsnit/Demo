Service-Accounts hinzufügen

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Project Explorer und anschließend ein Projekt oder einen Ordner, dem Sie den Service-Account hinzufügen möchten. Die Seite Projekt-/Ordneransicht wird angezeigt.
  3. Wählen Sie in der Dropdown-Liste Ressource erstellen die Option Service-Account, um die Seite Neuen Service-Account erstellen anzuzeigen.
  4. Geben Sie im Feld Ressourcenname einen eindeutigen Namen für diesen Service-Account ein.

    Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
  5. Geben Sie im Feld Ressourcenbeschreibung eine Beschreibung für den Service-Account ein.
  6. Führen Sie unter Ressourcentyp eine der folgenden Maßnahmen durch:
    • Wenn Sie einen Service-Account erstellen möchten, der die Benutzernamen und -kennwörter angibt, die es von eingehenden Client-Anforderungen empfängt, wählen Sie Passthrough.
    • Wenn Sie einen Service-Account erstellen möchten, der einen Benutzernamen und ein Kennwort angibt, die Sie mit der Service-Accountkonfiguration speichern, wählen Sie Statisch.
    • Wenn Sie einen Service-Account erstellen möchten, der den Benutzernamen eines oder mehrerer authentifizierter Clients dem angegebenen Benutzernamen oder Kennwort zuordnet, wählen Sie Zuordnung.
  7. Führen Sie je nach Ressourcentyp, den Sie in Schritt 6 gewählt haben, einen der in Tabelle 16-2 beschriebenen Schritte aus.

    8. Tabelle 16-2 Optionen für Ressourcentypen 
    gewählter Ressourcentyp
    Führen Sie diese Schritte aus
    Passthrough
    Klicken Sie auf Letzte.
    Statisch
    1. Klicken Sie auf Weiter.
    2. Geben Sie Benutzername und Kennwort in die Felder Benutzername, Kennwort und Kennwort bestätigen ein.
    3. Klicken Sie auf Letzte.
    Zuordnung
    Wenn Sie einen Service-Account erstellen möchten, der den Benutzernamen eines oder mehrerer authentifizierter Clients dem angegebenen Benutzernamen oder Kennwort zuordnet, führen Sie folgende Schritte aus:
    1. Klicken Sie auf Weiter.
    2. Führen Sie in der Tabelle "Autorisierten Remote-Benutzer eingeben" folgende Schritte aus:
      1. Geben Sie in die Felder Remote-Benutzername, Kennwort und Kennwort bestätigen den Benutzernamen und das Kennwort ein, den bzw. das Sie in ausgehenden Anforderungen senden möchten.
      2. Klicken Sie auf Hinzufügen.

        3. Die Benutzerzuordnung wird der Tabelle "Remote-Benutzer" hinzugefügt.

      3. (Optional) Fügen Sie der Tabelle "Autorisierten Remote-Benutzer eingeben" weitere Remote-Benutzer hinzu.
    3. Klicken Sie auf Weiter.
    4. Wenn Sie Remote-Benutzernamen und·-Kennwörtern autorisierte Clients hinzufügen möchten, führen Sie in der Tabelle "Autorisierten lokalen Benutzer eingeben" einen der folgenden Schritte aus:
      1. Geben Sie im Feld Lokaler Benutzername den Namen ein, der einen Client identifiziert, der in seiner eingehenden Anforderung authentifiziert wurde.

        2. Wenn Sie diesen Benutzer noch nicht in das Sicherheitskonfigurations-Modul der Oracle Service Bus Administration-Konsole hinzugefügt haben, tun Sie dies, bevor Sie diese Zuordnung in einer Laufzeitumgebung verwenden. Weitere Informationen finden Sie unter Benutzer hinzufügen. In Oracle Service Bus können Sie eine Zuordnung für einen nicht vorhandenen lokalen Benutzer erstellen, die Zuordnung wird jedoch nie einem authentifizierten Benutzer entsprechen und nie verwendet werden.

      2. Wählen Sie aus der Liste Remote-Benutzername den Benutzernamen, den Sie in ausgehenden Anforderungen für den authentifizierten Benutzer senden möchten, den Sie im Feld "Lokaler Benutzername" angegeben haben.
      3. Klicken Sie auf Hinzufügen.
    5. Wenn Sie anonyme Clients Remote-Benutzernamen zuordnen möchten, führen Sie folgende Schritte aus:
      1. Aktivieren Sie das Kontrollkästchen Anonyme Anforderungen zuordnen.
      2. Wählen Sie aus der Liste "Remote-Benutzer wählen" den Benutzernamen, den Sie in ausgehenden Anforderungen für alle anonymen Benutzer senden möchten.
    6. Klicken Sie auf Letzte.

  8. Klicken Sie auf Speichern. Der Service-Account wird erstellt und in der aktuellen Session gespeichert.
  9. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
