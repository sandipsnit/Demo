Service-Accounts bearbeiten

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Suchen Sie den Service-Account, wie in Service-Accounts suchen erläutert.
  3. Klicken Sie auf den Namen des Service-Accounts. Auf der Seite Service-Account-Details anzeigen werden die Daten aus der Tabelle 16-3 angezeigt.

    4. Tabelle 16-3 Service-Account-Details 
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der diesen Service-Account erstellt oder in die Konfiguration importiert hat.
    Zuletzt geändert am
    Das Datum oder die Uhrzeit, zu der der Benutzer diesen Service-Account erstellt oder in die Konfiguration importiert hat. Klicken Sie auf den Link von Datum und Uhrzeit, um die Änderungshistorie dieser Ressource anzuzeigen. Siehe Seite "Änderungshistorie anzeigen".
    Referenzen
    Die Anzahl der Objekte, die dieser Service-Account referenziert. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Siehe Referenzen auf Ressourcen anzeigen.
    Referenziert von
    Die Anzahl der Objekte, die diesen Service-Account referenzieren. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste der Objekte anzuzeigen. Wenn dieser Service-Account beispielsweise als JMS-Service-Account in einem Proxy-Service mit JMS-Transportprotokoll referenziert wird, wird der Proxy-Service beim Klicken auf den Link als Referenz angezeigt. Siehe Referenzen auf Ressourcen anzeigen.
    Beschreibung
    Eine Beschreibung dieses Service-Accounts, sofern vorhanden.

  4. Um Änderungen an den Feldern vorzunehmen, klicken Sie auf Bearbeiten. Weitere Beschreibungen zu den Feldern finden Sie unter Service-Accounts hinzufügen.

    5. Sie können das Feld Ressourcenname nicht ändern.

  5. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  6. Um die Session zu beenden und die Konfiguration zur Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
Hinweis: Wenn der von Ihnen geänderte Service-Account zur Authentifizierung mit einem WebLogic JMS-Server verwendet wird, kann es sein, dass der JMS-Server Ihre Änderungen bis zu 60 Sekunden lang nicht erkennt. Standardmäßig prüft WebLogic Server JMS Berechtigungen für jedes Ziel alle 60 Sekunden. Um dieses Verhalten zu ändern, modifizieren Sie den WebLogic Server-Startbefehl, sodass folgende Systemeigenschaft in die Frequenz (in Sekunden) geändert wird, in der die Berechtigungen von WebLogic JMS-Server geprüft werden sollen: weblogic.jms.securityCheckInterval
Der Wert 0 (null) für diese Eigenschaft stellt sicher, dass bei jeder Aktion Senden, Empfangen und getEnumeration in einer JMS-Ressource die Berechtigungen geprüft werden.
