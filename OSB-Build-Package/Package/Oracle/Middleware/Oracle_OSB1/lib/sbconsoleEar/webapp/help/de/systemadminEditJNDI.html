JNDI-Provider bearbeiten

Auf dieser Seite können Sie die Details eines JNDI-Providers anzeigen und gegebenenfalls die Konfiguration bearbeiten. Zudem können Sie den JNDI-Provider-Cache zurücksetzen und die Verbindung zum JNDI-Provider testen.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Suchen Sie den JNDI-Provider. Weitere Informationen finden Sie unter JNDI-Provider anzeigen.
  3. Klicken Sie auf den Namen des JNDI-Providers.

    4. Auf der Seite JNDI-Provider anzuzeigen – Konfiguration werden die in Tabelle 30-2 beschriebenen Informationen angezeigt.

    Tabelle 30-2 Informationen zum JNDI-Provider 
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der diese JNDI-Providerressource erstellt, geändert oder in die Konfiguration importiert hat.
    Zuletzt geändert am
    Datum und Uhrzeit des Zeitpunkts, an dem der Benutzer die JNDI-Providerressource erstellt, geändert oder in die Konfiguration importiert hat. Klicken Sie auf den Link mit Datum und Uhrzeit, um die Änderungshistorie dieser Ressource anzuzeigen. Weitere Informationen finden Sie unter Seite "Änderungshistorie anzeigen".
    Referenzen
    Die Anzahl der Objekte, die dieser JDS-Provider referenziert. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste mit allen Referenzen anzuzeigen. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Referenziert von
    Die Anzahl der Objekte, die diesen JNDI-Provider referenzieren. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste mit allen Objekten anzuzeigen. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Beschreibung
    Eine Beschreibung dieses JNDI-Providers.
    JNDI-Cache
    Aktiviert oder Deaktiviert. Ist diese Option aktiviert, werden der JNDI-Kontext und JNDI-Objekte lokal gecacht, wodurch die Performance beim Objekt-Lookup verbessert wird. Oracle empfiehlt, den JNDI-Cache zu aktivieren.
    Provider-URL
    Der URL für den JNDI-Provider.
    JNDI-Anforderungs-Timeout
    Das JNDI-Anforderungs-Timeout in Millisekunden. Der Standardwert ist 0 und bedeutet, dass es kein Timeout gibt.
    Benutzername und Kennwort
    Der Benutzername für den Zugriff auf einen gesicherten JNDI-Provider. Das Kennwort kann geändert werden.

  4. Klicken Sie auf Bearbeiten, um die Seite JNDI-Provider bearbeiten – [Providername] anzuzeigen.

    5. Das Feld Name kann nicht bearbeitet werden.

  5. In das Feld Beschreibung können Sie eine kurze Beschreibung des JNDI-Providers eingeben.
  6. In das Feld Provider-URL können Sie den URL des JNDI-Providers bearbeiten.
  7. Sie können die Felder Benutzer und Kennwort bearbeiten.

    8. Wenn der JNDI-Provider mit einem Benutzernamen und Kennwort konfiguriert wurde, kann der Benutzername nicht bearbeitet werden. Sie können jedoch die Felder Neues Kennwort und Kennwort bestätigen bearbeiten. Wenn der JNDI-Provider nicht mit einem Benutzernamen und Kennwort konfiguriert wurde, dann kann das Feld Benutzername bearbeitet werden, und Sie können einen neuen Benutzernamen eingeben.

  8. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  9. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
