Ressourcen exportieren

Auf der Seite Ressourcen exportieren können Sie Projekte exportieren oder Ressourcen exportieren. Wenn Sie ein vollständiges Projekt exportieren möchten, können Sie ein oder mehrere Projekte für den Export wählen. Sie können die Projekte trotzdem erweitern, um die Ordner und Ressourcen anzuzeigen, aber die Auswahl ist nur auf Projektebene möglich. Außerdem kann die Option Abhängigkeiten einbeziehen nicht gewählt werden.

Sie können unabhängig davon Projekte und Ressourcen exportieren, ob Sie innerhalb oder außerhalb einer Session arbeiten. Wenn Sie den Export innerhalb einer Session durchführen, handelt es sich bei den Ressourcen um Sessionressourcen, und die Konfiguration kann unvollständig sein oder Konflikte aufweisen. Wenn Sie den Export außerhalb einer Session durchführen, handelt es sich bei den Ressourcen um aktivierte Core-Ressourcen.

In Oracle Service Bus können die Benutzer, Gruppen oder Rollen nicht exportiert werden, die Sie im Modul "Sicherheitskonfiguration" der Oracle Service Bus Administration-Konsole erstellen. Außerdem können auch keine Zugangsdatenzuordnungen oder andere Sicherheitsproviderdaten exportiert werden, die Sie in der WebLogic Server-Administrationskonsole erstellen. Stattdessen können Sie diese Daten mit der WebLogic Server-Administrationskonsole exportieren. Weitere Informationen finden Sie unter "Migrating Security Data" in Oracle Fusion Middleware Securing Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=SECMG319.

  1. Wählen Sie Systemverwaltung > Ressourcen exportieren.

    2. Auf der Seite Ressourcen exportieren wird die Liste der Objekte in Ihrer Konfiguration angezeigt. Dabei werden der Name, der Typ, der Zeitpunkt der letzten Änderung und die Referenzen für die einzelnen Objekte angezeigt.

    Bei Services stellt der Zeitstempel der letzten Änderung die Modifikationen dar, die an der Service-Definition sowie am Service-Transport oder an der Pipeline-Konfiguration vorgenommen wurden. Allerdings werden keine Änderungen wiedergegeben, die an den für diesen Service definierten Alerts vorgenommen wurden.

  2. Wählen Sie, ob Sie Projekte exportieren oder Ressourcen exportieren möchten.
    3. Hinweis: Durch das Exportieren von Projekten werden möglicherweise Ressourcen gelöscht, wenn Sie die JAR-Datei des vollständigen Projekts importieren. Weitere Informationen finden Sie unter Ressourcen importieren.
  3. Wählen Sie die zu exportierenden Projekte oder Ressourcen aus. Wenn Sie Projekte exportieren gewählt haben, können Sie die Kontrollkästchen für einzelne Ressourcen nicht deaktivieren.
    1. Erweitern Sie die Projektordner. Der Name, der Typ, der Zeitpunkt der letzten Änderung sowie die Referenzen für die einzelnen im Projekt enthaltenen Ressourcen werden angezeigt.
    2. Aktivieren Sie die Kontrollkästchen zu den Projekten oder Ressourcen, die Sie exportieren möchten.
    3. Deaktivieren Sie die Kontrollkästchen zu den Projekten oder Ressourcen, die Sie nicht exportieren möchten.
  4. Wenn Sie Ressourcen exportieren gewählt haben, aktivieren oder deaktivieren Sie das Kontrollkästchen Abhängigkeiten einbeziehen. Mit dieser Option können Sie alle anderen Ressourcen exportieren, auf die diese Ressource verweist.
  5. Klicken Sie auf Exportieren, um eine JAR-Datei zu erstellen und zu exportieren.

Wenn Sie fertig sind

Fahren Sie gegebenenfalls mit dem Abschnitt Exportsicherheitseinstellungen angeben fort.
