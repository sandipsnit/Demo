Business-Services aus einer UDDI-Registry importieren

Sie können folgende Business-Service-Typen aus einer UDDI-Registry in Oracle Service Bus importieren:

  • WSDL-Services über HTTP-Transport
  • Oracle Service Bus-Proxy-Services, die in einer UDDI-Registry veröffentlicht wurden. Dieses Feature wird vorwiegend in Oracle Service Bus-Deployments mit mehreren Domains verwendet, bei denen Proxy-Services aus einer Domain Proxy-Services in einer anderen Domain ermitteln und Daten zu dieser Domain weiterleiten müssen.

Wenn Sie nur eine Registry konfiguriert haben, stellt Oracle Service Bus automatisch eine Verbindung zu dieser Registry her, ruft alle Business-Entitäten ab und füllt das Suchformular auf. Wenn Sie mehrere Registrys konfiguriert haben, werden Sie vom Importassistenten aufgefordert, zunächst die Registry auszuwählen, aus der Sie Services importieren möchten.

Beim Importieren wird der Inquiry-URL verwendet, um einen bestimmten Service zu suchen, da eine Registry mehrere unterschiedliche Service-Typen aufweist.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Systemverwaltung > Aus UDDI importieren.
  3. Wählen Sie in der Dropdown-Liste Name der Import-Registry den Namen der Registry, aus der Sie den Service importieren möchten, und klicken Sie auf Weiter.

    4. Wenn in Oracle Service Bus eine einzige Registry konfiguriert ist, wird die anfängliche Seite zum Wählen der Registry nicht angezeigt, da es sich hierbei um die Standard-Registry handelt. Die Seite zum Wählen der Registry wird nur angezeigt, wenn mindestens zwei Registrys definiert sind.

    Auf der Seite Aus UDDI importieren können Sie bestimmte Business-Services suchen und importieren. Sie können einen Service nach Business-Entität, nach Muster oder nach beidem suchen.

  4. Wählen Sie in der Dropdown-Liste Entitätsname den Namen der Business Entität (Beispiel: Dokument-Services), um nach Business-Entität zu suchen.
  5. Geben Sie im Feld Service-Name das Muster (Beispiel: a%) ein, nach dem Sie suchen möchten. Es wird eine Liste der Business-Services angezeigt, die den eingegebenen Suchkriterien entsprechen.
  6. Wählen Sie die zu importierenden Services aus, und klicken Sie auf Weiter.

    7. Wenn Sie den gewünschten Service nicht finden, kann das daran liegen, dass Sie nicht zu der Sicherheitsgruppe mit Berechtigungen zum Anzeigen der entsprechenden Datensätze gehören.

  7. Wählen Sie die Binding-Vorlagen, aus denen Sie Business-Services erstellen möchten, und klicken Sie auf Weiter.

    8. Wenn für einen gewählten Service mehrere Binding-Vorlagen vorhanden sind, wird aus jeder dieser Binding-Vorlagen ein Business-Service erstellt. In diesem Fall werden Sie aufgefordert, Ihre Auswahl auf die zu importierenden Binding-Vorlagen zu beschränken.

  8. Wählen Sie auf der Seite Aus UDDI importieren - Importspeicherort wählen ein Projekt in der Dropdown-Liste Projekt. Wählen Sie einen Ordner, in den Sie Ihre Services und die zugehörigen Ressourcen importieren möchten, und klicken Sie auf Weiter.
  9. Auf der Seite Aus UDDI importieren - Services überprüfen und importieren wird eine Liste aller Ressourcen (Business-Services, MFLs, Schemas, WSDLs und so weiter) angezeigt, die im System erstellt werden. Für Ressourcen, die nicht importiert werden können, wird eine Warnmeldung angezeigt.
    Standardmäßig sind alle Elemente in der Liste für den Import markiert.
    1. Deaktivieren Sie das Kontrollkästchen neben einer bestimmten Ressource, wenn sich die Ressource bereits in Ihrem System befindet und Sie die Abhängigkeiten manuell auflösen möchten.
    2. Sie können Abhängigkeiten auf der Seite Konflikte anzeigen im Change Center auflösen, wenn der Importvorgang abgeschlossen ist. Weitere Informationen zum Anzeigen und Auflösen von Konflikten finden Sie unter Konflikte anzeigen und auflösen.
  10. Um den Importvorgang zu starten, klicken Sie auf Importieren.

    11. In einem Popup-Fenster wird der Fortschritt des Importvorgangs angezeigt. Wenn der Importvorgang abgeschlossen ist, wird das Ergebnis auf der Seite Importübersicht angezeigt. Wenn der Importvorgang erfolgreich abgeschlossen wurde, wird oben auf der Seite eine Erfolgsmeldung angezeigt. Wenn eine oder mehrere Ressourcen nicht importiert werden konnten, wird eine Fehlermeldung angezeigt.

    • Navigieren Sie zur Spalte Status der betreffenden Ressource, um die Fehlerursache anzuzeigen.
    • Um weitere Konflikte anzuzeigen und zu beheben, klicken Sie auf Konflikte anzeigen.
  11. Um weitere Services zu importieren, klicken Sie auf Weitere importieren.
  12. Um Änderungen an Ihrer Auswahl vorzunehmen, klicken Sie auf Zurück.
