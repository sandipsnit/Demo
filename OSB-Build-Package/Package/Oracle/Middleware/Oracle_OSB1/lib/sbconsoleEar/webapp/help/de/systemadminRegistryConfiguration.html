Konfigurierte UDDI-Registrys anzeigen

Auf dieser Seite können Sie eine Standard-UDDI-Registry bestimmen (weitere Informationen finden Sie unter Standard-UDDI-Konfiguration einrichten) und UDDI-Registrys anzeigen. Außerdem können Sie nach zuvor konfigurierten Registrys suchen.

  1. Geben Sie im Bereich "UDDI-Registrys" im Feld Name den Namen der Registry ein, die Sie suchen möchten, oder geben Sie Platzhalterzeichen ein. Dabei können Sie das Sternchen (*) für mehrere Zeichen und das Fragezeichen (?) für einzelne Zeichen verwenden. Bei der Suche werden alle Datensätze zurückgegeben, die das Namenskriterium erfüllen.
  2. Klicken Sie auf Suchen.
  3. Klicken Sie auf Alle anzeigen, um die Suchfilter zu entfernen und alle Registrys anzuzeigen, die für das Arbeiten mit Oracle Service Bus konfiguriert sind.

    4. In Tabelle 29-1 werden die Konfigurationseinstellungen der UDDI-Registry angezeigt.

    Tabelle 29-1 Konfigurationseinstellungen der UDDI-Registry 
    Eigenschaft
    Beschreibung
    Registry-Name
    Der Name der Registry. Der Name wird einer Registry zugeordnet, wenn sie zum ersten Mal erstellt wird. Sie können den Namen einer Registry nicht bearbeiten, nachdem der Eintrag gespeichert wurde.
    Abfrage-URL
    Der URL des Inquiry-API-Endpunktes, der für das Suchen und Importieren von Services verwendet wird.
    Publishing-URL
    Der URL des Publishing-API-Endpunktes, der für das Veröffentlichen von Services verwendet wird.
    Sicherheits-URL
    Der URL des Sicherheits-API-Endpunktes, der für das Abrufen eines Authentifizierungstokens verwendet wird, sodass Sie Daten in der Registry veröffentlichen können.
    Abonnement-URL
    Der URL des Abonnement-API-Endpunktes, der für das Abonnieren von Registry-Änderungen, das Erstellen eines Registry-Abonnements und das Empfangen von Änderungen an importierten Services verwendet wird.
    Benutzername
    Der Benutzername, der für die Benutzerauthentifizierung erforderlich ist.
    Kennwort (Kennwort bestätigen)
    Das Kennwort, das für die Benutzerauthentifizierung erforderlich ist.
    tModels in Registry laden
    Lädt die tModels in die gewählte Registry. Diese Option muss nur einmal pro Registry aktiviert werden.
    Automatischen Import aktivieren
    Synchronisiert Services automatisch mit der UDDI-Registry.
