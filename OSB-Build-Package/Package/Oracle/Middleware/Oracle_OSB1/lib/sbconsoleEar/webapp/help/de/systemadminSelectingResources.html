Zu importierende Projekte und Ressourcen wählen

  1. Wenn Sie die JAR-Datei einer Ressource importieren und sicherstellen möchten, dass alle zugeordneten Ressourcen importiert werden, wählen Sie Abhängigkeiten einbeziehen.
    2. Hinweis: Die Option Abhängigkeiten einbeziehen kann nicht aktiviert werden, wenn Sie die JAR-Datei eines vollständigen Projekts oder eine Konfigurationsdatei mit einer älteren Version als 2.6 importieren.
  2. Klicken Sie auf das Symbol Öffnen, um die Erweiterten Einstellungen anzuzeigen und die Werte im importierenden System beizubehalten.
  3. Wenn Sie eine Ressource erneut importieren und dabei die vorgenommenen Anpassungen an der vorhandenen Ressource beibehalten möchten, wählen Sie Umgebungsvariablen beibehalten.
    4. Hinweis: Anpassungen an Referenzen innerhalb einer vorhandenen Ressource können nicht beibehalten werden. Weitere Informationen finden Sie unter Anpassungsdateien erstellen.
  4. Wenn Sie die Ressource "Globale Vorgangseinstellungen" oder andere Ressourcen wie Business oder Proxy-Services importieren, die zugeordnete Vorgangseinstellungen auf Service-Ebene beinhalten, und die Vorgangswerte im importierenden System beibehalten möchten, wählen Sie Vorgangswerte beibehalten.
  5. Um das Überschreiben vorhandener sicherheitsbezogener Konfigurationen in aktualisierten Ressourcen zu verhindern, wählen Sie eine der folgenden Optionen:
    1. Sicherheits- und Policy-Konfiguration beibehalten - Behält die Sicherheitskonfiguration (mit Ausnahme der Zugriffskontroll-Policys) und die Verweise auf WS-Policys bei, die direkt an den Service (und nicht an die WSDL) gebunden sind.

      2. Weitere Informationen finden Sie unter "Security" im Oracle Fusion Middleware Developer's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472.

    2. Zugangsdaten beibehalten - Behält PKI-Zugangsdaten in Service-Schlüsselprovidern, Benutzernamen und Kennwörter in Service-Accounts und Benutzernamens- und Kennwortzugangsdaten in SMTP-Servern, Proxyservern, JNDI-Providern und UDDI-Registrys bei.
    3. Zugriffskontroll-Policys beibehalten - Behält Zugriffskontroll-Policys bei.

      4. Informationen zur Ressourcenübersicht sind in Tabelle 28-1 dargestellt.

      Tabelle 28-1 Ressourceninformationen
      Eigenschaft
      Beschreibung
      Name
      Der Name des Projekts.
      Globale Ressourcen, wie UDDI-Registry-Informationen, SMTP-Server, Proxyserver, JNDI-Provider und globale Vorgangseinstellungen, werden in den entsprechenden Ordnern unter dem Projekt System angezeigt.
      Typ
      Der Ressourcentyp (Projekt).
      Erweitern Sie den Projektordner, um Ressourcen anzuzeigen und zu deaktivieren.
      Vorgänge
      Zeigt --- oder !Deletions! an.
      Erweitern Sie den Projektordner, um die Vorgänge (Erstellen, Aktualisieren, Löschen) anzuzeigen, die für die Ressourcen durchgeführt werden.
      Um das Löschen bestimmter Ressourcen zu verhindern, können Sie sie deaktivieren. Ressourcen werden nur für das Löschen geplant, wenn es sich bei der importierten JAR-Datei um die JAR-Datei eines vollständigen Projekts handelt und wenn sich im importierenden System Projektressourcen befinden, die in der importierten JAR-Datei nicht vorhanden sind.
      Referenzen
      Erweitern Sie den Projektordner, um die Anzahl der Objekte anzuzeigen, auf die die einzelnen Ressourcen verweisen.

  6. Wählen Sie die Objekte, für die Sie Vorgänge (Erstellen, Aktualisieren oder Löschen) ausführen möchten.
    1. Um die einzelnen Objekte in der JAR-Datei anzuzeigen, erweitern Sie den Ordner für die jeweilige Datei. Das Kontrollkästchen, das zu dem jeweiligen Objekt gehört, ist aktiviert.

      2. In Tabelle 28-2 werden die folgenden Ressourcendateidetails für die einzelnen Objekte angezeigt.

      Tabelle 28-2 Ressourcendateidetails 
      Eigenschaft
      Beschreibung
      Name
      Der Name und der Pfad zur Ressource. Der Pfad ist der Projektname und der Name des Ordners, in dem die Ressource gespeichert ist.
      Typ
      Die Ressourcentypen in Oracle Service Bus.
      Hinweis: Wenn eine Ressource sensible Sicherheitsdaten enthält, wird ein Symbol Verschlüsselung angezeigt.
      Achtung: Wenn Sie einen Service-Account oder einen Service-Schlüsselprovider wählen, können unter Umständen vorhandene Sicherheitsdaten überschrieben werden. Sie können dies verhindern, indem Sie die Sicherheitseinstellungen beim Import beibehalten. Weitere Informationen finden Sie unter Service-Accounts oder Service-Schlüsselprovider importieren.
      Vorgänge
      Erstellen, Aktualisieren oder Löschen.
      Ressourcen werden nur für das Löschen geplant, wenn es sich bei der importierten JAR-Datei um die JAR-Datei eines vollständigen Projekts handelt und wenn sich im importierenden System Projektressourcen befinden, die in der importierten JAR-Datei nicht vorhanden sind.
      Referenzen
      Die Anzahl der Objekte, auf die diese Ressource verweist.

    2. Deaktivieren Sie die Kontrollkästchen neben den Ressourcen, die Sie nicht importieren möchten.
  7. Klicken Sie auf Importieren.

Wenn Sie fertig sind

Fahren Sie gegebenenfalls mit dem Abschnitt Sicherheitseinstellungen für den Import festlegen fort. Sie können auch mit dem Abschnitt Importzusammenfassung prüfen fortfahren.
