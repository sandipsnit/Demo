Testkonsole

Die Oracle Service Bus-Testkonsole ist eine browserbasierte Testumgebung für die Validierung und den Test Ihres Systemdesigns. Sie können das Textobjekt konfigurieren (Proxy-Service, Business-Service oder Ressource), den Test ausführen und die Ergebnisse in der Testkonsole anzeigen. In einigen Fällen können Sie den Code per Tracing verfolgen und den Status der Nachricht an bestimmten Trace-Punkten untersuchen. Weitere Informationen zur Testkonsole finden Sie unter "Using the Test Console" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG357.

Nur Benutzer mit den Rollen IntegrationAdmin und IntegrationDeployer können die Testkonsole verwenden. Weitere Informationen zu den Rollen finden Sie unter "Roles in Oracle Service Bus" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus auf der Website http://www.oracle.com/pls/as111170/lookup?id=OSBAG463.

Hinweis: Wenn Sie eine Fehlermeldung erhalten, die besagt, dass der Testkonsolen-Service nicht ausgeführt wird, setzen Sie die Admin-Server-Listening-Adresse auf eine bestimmte gültige Adresse, wie localhost. Navigieren Sie in der Oracle WebLogic Server-Konsole zu Umgebung > Server > admin_server_name > Konfiguration > Allgemein, um die Listening-Adresse einzustellen.
