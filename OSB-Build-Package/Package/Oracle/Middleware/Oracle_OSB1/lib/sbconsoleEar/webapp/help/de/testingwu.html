Proxy-Services testen

Sie können folgende Proxy-Service-Typen testen: alle XML, alle SOAP, Messaging, XML und SOAP. Sie können SOAP-Proxy-Services mit Web Service Security (WSSRAWPolicys testen. Siehe Web Service Security in Proxy-Service-Testdaten konfigurieren.

Hinweis: Wenn die Testkonsole ein Proxy mit Authentifizierung durch benutzerdefinierten Token auf HTTP-Ebene aufruft, wird die Authentifizierungsprüfung nicht durchgeführt.
Achtung: Werden Proxy-Services getestet, wenn die Option "Direkter Aufruf" aktiviert ist, werden einige wichtige Sicherheitsschritte umgangen, einschließlich der Zugriffskontrolle. Oracle empfiehlt, dass Sie den Test-Service nicht in Produktionsumgebungen verwenden.
  1. Klicken Sie unter Change Center auf Aktivieren, um die Testfunktion in der Oracle Service Bus Administration-Konsole zu aktivieren.

    2. Sie können Proxy-Services aus dem Ressourcenbrowser oder Project Explorer testen.

  2. Wählen Sie Ressourcenbrowser > Proxy-Services, um die Seite Zusammenfassung der Proxy-Services anzuzeigen.
  3. Unter Aktionen klicken Sie auf das Symbol Testkonsole starten , das mit dem zu testenden Proxy-Service verbunden ist.

    4. Die Testkonsole öffnet die Seite Proxy-Service-Test. Klicken Sie beispielsweise in den mit dem Produkt gelieferten Beispielen auf das mit dem Proxy-Service LoanGateway1 verknüpfte Symbol.

    Hinweis: In einer geclusterten Domain können Sie in der Testkonsole keine konfigurierten Business-Services oder Proxy-Services testen, die zu einem Business-Service weiterleiten.
  4. Bei SOAP- und XML-Services wählen Sie den WSDL-Vorgang, den Sie testen möchten.
  5. Konfigurieren Sie die Testdaten für den Proxy-Service. Das müssen die Daten sein, die der Proxy-Service vom Client erwartet.

    6. Standard ist, dass beide Testkonfigurationsoptionen, Direkter Aufruf und Einschließlich Tracing, deaktiviert sind. Sie können die Option Direkter Aufruf entfernen, das entfernt auch die Option Einschließlich Tracing. Dadurch wird der Test mit der Methode Indirekter Aufruf ausgeführt, wobei die Nachricht durch die Transportebene gesendet wird.

    Sie können die Option Direkter Aufruf verwenden. Aktivieren Sie Direkter Aufruf, und deaktivieren Sie das Ausführungs-Tracing. Deaktivieren Sie hierzu einfach das Kontrollkästchen Einschließlich Tracing.

    Um das Nachrichten-Tracing anzuzeigen, deaktivieren Sie "Direkter Aufruf".

    Hinweis: Um das Tracing in der Log-Datei oder im Standardausgang (Serverkonsole) anzuzeigen, muss die Anmeldung bei Oracle WebLogic Server auf die folgenden Dringlichkeitsstufen eingestellt werden:

Mindestdringlichkeit für das Log: Info
Log-Datei: Info
Standardausgang: Info

Weitere Informationen zum Einstellen der Dringlichkeitsstufen für das Log finden Sie im entsprechenden Thema unter Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

  1. Klicken Sie auf Ausführen. Die Seite Proxy-Service-Test zeigt die Ergebnisse an. Informationen zur Interpretation der Testergebnisse finden Sie unter Proxy-Services-Testergebnisse anzeigen.
  2. Um den Test erneut auszuführen, klicken Sie auf Zurück. Wiederholen Sie die Schritte ab Schritt 5.
