Proxy-Service-Testergebnisse anzeigen

In Tabelle 32-2 werden die Testergebnisse für Proxy-Services beschrieben. Tracing ist nur aktiviert, wenn Sie sowohl Direkter Aufruf als auch Einschließlich Tracing gewählt haben.

Tabelle 32-2 Testergebnisse für Proxy-Services 
Abschnitt
Beschreibung
Proxy-Service-Name
Der Name des zu testenden Proxy-Service
Testkonsolenaktionen
Wenn Sie auf Zurück klicken, wird die vorherige Browserseite angezeigt.
Mit Schließen schließen Sie die Testkonsole.
Anforderungsdokument
Die von der Testkonsole an den Proxy-Service gesendete Anforderungsnachricht.
Dieser Abschnitt ist ausgeblendet, wenn die Anforderungsnachricht durch die Testkonsole nicht geändert wurde. Er ist für SOAP-Services erweitert, die in der Registerkarte Formular konfiguriert wurden, oder bei Anwendung von WS-Security.
Wenn WS-Security verwendet wurde, enthält dieser Abschnitt zwei SOAP-Nachrichten: die Nachricht Text löschen und die SOAP-Nachricht gesichert.
Antwortdokument
Die Nachrichtenantwort
Bei SOAP-Services mit einer WS-Security-Antwort enthält dieser Abschnitt zwei SOAP-Nachrichten. Die erste Nachricht ist die vom Client erhaltene gesicherte SOAP-Nachricht. Die zweite SOAP-Nachricht ist die entsprechende Nachricht im Klartext.
Antwortmetadaten
Die mit der Nachrichtenantwort empfangenen Metadaten
Tracing von Proxy-Services
Beim Tracing wird der Status der Nachricht auf ihrem Weg durch das System angezeigt. Wenn die Option Direkter Aufruf nicht aktiviert wurde, wird kein Tracing durchgeführt. Weitere Informationen zum Tracing finden Sie unter Tracing von Proxy-Services.
