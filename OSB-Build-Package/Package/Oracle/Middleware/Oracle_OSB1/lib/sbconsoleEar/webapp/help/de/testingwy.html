Business-Services testen

Sie können folgende Typen von Business-Services testen: beliebige XML, beliebige SOAP, Messaging, XML und SOAP. Sie können SOAP-Business-Services mit Web Service-Sicherheits-Policys testen. Weitere Informationen finden Sie unter "Testing Services with Web Service Security" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG384.

Hinweis: In einer geclusterten Domain können Sie in der Testkonsole keine konfigurierten Business-Services oder Proxy-Services testen, die zu einem Business-Service weiterleiten.
  1. Klicken Sie unter Change Center auf Aktivieren, um die Testfunktion in der Oracle Service Bus Administration-Konsole zu aktivieren.
  2. Wählen Sie Ressourcenbrowser > Business-Services, um die Seite Business-Services - Überblick anzuzeigen.
  3. Klicken Sie unter Aktionen auf das Symbol Testkonsole starten, das mit dem zu testenden Service verknüpft ist. Die Testkonsole wird mit der Seite Business-Service-Test geöffnet. Wenn Sie beispielsweise die mit dem Produkt gelieferten Tutorials verwenden, klicken Sie auf das mit loanSaleProcessor verknüpfte Symbol.
  4. Bei SOAP- und XML-Services wählen Sie den WSDL-Vorgang, den Sie testen möchten.
  5. Konfigurieren Sie die Testdaten für den Business-Service (die Eingabedaten sollten der Nachricht entsprechen, die vom Proxy-Service an den Business-Service gesendet wird). Die Optionen Direkter Aufruf und Einschließlich Tracing stehen zur Verfügung, wenn Test-Proxy-Services für Business-Services nicht verfügbar sind. Standardmäßig werden Business-Services mit der Option Direkter Aufruf getestet. Dies bedeutet, dass die Nachrichten durch die Transportschicht geleitet werden.

    6. Um das Nachrichten-Tracing anzuzeigen, deaktivieren Sie "Direkter Aufruf".

    Hinweis: Um das Tracing in der Log-Datei oder im Standardausgang (Serverkonsole) anzuzeigen, muss die Anmeldung bei Oracle WebLogic Server auf die folgenden Dringlichkeitsstufen eingestellt werden:

Mindestdringlichkeit für das Log: Info
Log-Datei: Info
Standardausgang: Info

Weitere Informationen zum Einstellen der Dringlichkeitsstufen für das Log finden Sie im entsprechenden Thema unter Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server auf der Website http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

  1. Klicken Sie auf Ausführen.

    2. Die Seite Business-Service-Test zeigt die Ergebnisse an. Weitere Informationen finden Sie unter Business-Service-Testergebnisse anzeigen.

Anhänge in Business-Services testen

In diesem Abschnitt wird beschrieben, wie mit der Testkonsole Anhänge in Business-Services getestet werden.

Dies wird am Beispiel eines WSDL-basierten HTTP-Business-Service erläutert, bei dem eine ZIP-Datei mit einem "submitAttachment"-Vorgang als Anhang einer SOAP-Nachricht gesendet wird. Bei anderen binären Anhängen müssen die Einstellungen entsprechend geändert werden.

    1. Rufen Sie in der Testkonsole für den Business-Service die XML-Seite auf, und geben Sie den folgenden SOAP-Envelope für die Testnachricht ein:
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. Im Bereich "Anhang" der Testkonsole:
      1. Contenttyp: application/zip (gemäß Definition in der WSDL)
      2. Content-ID: zipFile (muss mit dem Wert href="cid:zipFile" in der Testnachricht übereinstimmen)
      3. Datei: Klicken Sie auf Durchsuchen, und wählen Sie die ZIP-Datei, die Sie im SOAP-Envelope angegeben haben.
      4. Klicken Sie auf Hinzufügen.

    2. Klicken Sie auf Ausführen, um die Testnachricht mit dem Anhang zu senden.

Das erfolgreiche Versenden des Anhangs wird in der Serverkonsole mit einer Nachricht ähnlich der Folgenden bestätigt. Diese Nachricht wird vom submitAttachment-Vorgang protokolliert:

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
