Tracing von Proxy-Services

Tracing wird aktiviert, wenn Sie Proxy-Services mit einem direkten Aufruf testen. Das Kontrollkästchen Einschließlich Tracing wird mit der Option Direkter Aufruf automatisch aktiviert. Wenn Sie kein Tracing durchführen möchten, deaktivieren Sie das Kontrollkästchen Einschließlich Tracing. Bei aktiviertem Tracing sind die Details zum Trace in den Testergebnissen enthalten. Mit Tracing können Sie Systemfehler protokollieren und für eine Fehlerbehebung isolieren. In den Trace-Informationen werden der Pfad in der Anforderung und die Antwortpfade im Code angezeigt.

  1. Klicken Sie unter Change Center auf Aktivieren, um die Testfunktion in der Oracle Service Bus Administration-Konsole zu aktivieren.
  2. Wählen Sie Ressourcenbrowser > Proxy-Services, um die Seite Proxy-Services - Überblick anzuzeigen.
  3. Klicken Sie unter Aktionen auf das Symbol Testkonsole starten, das dem zu testenden Service zugeordnet ist. In der Testkonsole wird die Seite Proxy-Service-Test aufgerufen.
  4. Konfigurieren Sie die Testdaten für den Proxy-Service. Sie müssen die Optionen Direkter Aufruf und Einschließlich Tracing aktivieren, um das Tracing zu aktivieren. Weitere Informationen finden Sie unter Proxy-Service-Testdaten konfigurieren.
  5. Klicken Sie auf Ausführen. Auf der Seite Proxy-Service-Test werden die Testergebnisse für den Service und die Tracing-Informationen angezeigt.
  6. Führen Sie einen Bildlauf nach unten zum Abschnitt Aufrufs-Trace durch.

    7. Dieser Abschnitt enthält eine grafische Darstellung des Nachrichtenflusses. Sie können die Nachricht durch den Service tracen und den Status der Nachricht bei bestimmten Punkten im Trace anzeigen. Die Trace-Punkte werden automatisch festgelegt.

  7. Klicken Sie auf das Pluszeichen (+), um den Nachrichtenfluss zu erweitern und weitere Details anzuzeigen.

    8. Beim Anzeigen des Trace können Sie auch den Nachrichtenfluss in der Oracle Service Bus Administration-Konsole anzeigen. Dadurch können Sie den Trace mit den Stages und Aktionen im Nachrichtenfluss verknüpfen. Sie können den Nachrichtenfluss ändern und den Trace erneut ausführen, um die Ausgabe anzuzeigen.
