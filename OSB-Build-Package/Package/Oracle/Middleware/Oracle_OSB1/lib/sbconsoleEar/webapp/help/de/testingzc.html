MFL-Transformationen testen

Bei einem MFL-Dokument (Message Format Language) handelt es sich um ein spezielles XML-Dokument zur Beschreibung des Layouts von Binärdaten. MFL-Ressourcen unterstützen zwei Transformationen. Jede Transformation ihrerseits akzeptiert nur eine Eingabe und liefert nur eine Ausgabe: XML-zu-Binärdaten oder Binärdaten-zu-XML.

Tabelle 32-5 beschreibt MFL-Konfigurationsoptionen.

MFL-Transformationskonfiguration
XML-Eingabe
Erforderlich für XML-zu-Binärdaten-Transformationen:
Das XML-Schema für das MFL-Dokument kann abgeleitet werden. Ein Beispiel-XML-Dokument wird automatisch in das Textfeld eingegeben.
Die XML-Eingabe kann datei- oder textbasiert sein. Das Referenzieren einer Datei für eine Eingabe gilt vorrangig gegenüber der Texteingabe. Suchen Sie nach der Datei, die im Test verwendet werden soll.
Binäreingabe
Erforderlich für Binärdaten-zu-XML-Transformationen:
Die Binäreingabe kann datei- oder textbasiert sein. Das Referenzieren einer Datei für eine Eingabe gilt vorrangig gegenüber der Texteingabe. Suchen Sie nach der Datei, die im Test verwendet werden soll.

Sie können die Entwurfszeit oder die Laufzeit testen.

  1. Klicken Sie auf Aktivieren, wenn Sie die Laufzeit testen möchten. Aktivieren Sie die Session nicht, um die Entwurfszeit zu testen.
  2. Wählen Sie Ressourcenbrowser > MFLs, um die Seite Übersicht der MFL-Dateien anzuzeigen.
  3. Klicken Sie unter Aktionen auf das Symbol Testkonsole starten, das mit der zu testenden Ressource verknüpft ist. Auf der Testkonsole wird die Seite Ressource testen geöffnet.
  4. Wählen Sie die zu testende Transformation.
  5. Konfigurieren Sie die Testdaten für die MFL-Ressource. Weitere Informationen finden Sie in Tabelle 32-5.
  6. Klicken Sie auf Ausführen:. Auf der Seite Ressource testen werden die Ergebnisse angezeigt.
  7. Klicken Sie auf Zurück, um erneut zu prüfen. Sie können die Testkonsole schließen, ändern und die Ressource erneut prüfen.
