XSLT-Transformationen testen

XSLT-Transformationen (eXtensible Stylesheet Language Transformations) beschreiben XML-zu-XML-Zuordnungen in Oracle Service Bus. Zum Testen einer XSLT-Ressource müssen Sie das Eingabe-XML-Dokument übermitteln, und die Testkonsole gibt das Ausgabe-XML-Dokument zurück. XSLT-Transformationen können weitere benannte Parameter enthalten. Alle von der Transformation benötigten Parameter werden auf der Konfigurationsseite angezeigt. Standardwerte stehen zur Verfügung, können jedoch überschrieben werden.

Tabelle 32-6 beschreibt die XSLT-Konfigurationsoptionen.

Eingabe und Parameter
Das Eingabedokument und die Parameter zum Testen der XSLT-Ressource.
XML-Eingabe
Die XML-Eingabe kann datei- oder textbasiert sein. Die Referenzierung einer Datei für die Eingabe hat Vorrang vor textlichen Eingaben. Suchen und wählen Sie die Datei aus, die Sie in Ihrem Text verwenden möchten. In dieses Feld muss ein Wert eingegeben werden.
<param_name> ([] als XML)
param_name ist ein benannter XSLT-Parameter.
Es gibt zwei Eingabetypen: XML und primitiver Datentyp (Zeichenfolge, Ganzzahl, Float usw.). Der Standardeingabetyp ist "Zeichenfolge". Aktivieren Sie das zum Parameternamen gehörige Kontrollkästchen, um einen Parameter des Typs "XML" festzulegen.

Sie können die Entwurfszeit oder die Laufzeit testen.

  1. Klicken Sie auf Aktivieren, wenn Sie die Laufzeit testen möchten. Aktivieren Sie die Session nicht, um die Entwurfszeit zu testen.
  2. Wählen Sie Project Explorer > XSLTs, um die Seite Übersicht der XSLTs anzuzeigen.
  3. Klicken Sie unter Aktionen auf das Symbol Testkonsole starten, das mit der zu testenden Ressource verknüpft ist. Auf der Testkonsole wird die Seite Ressource testen geöffnet.
  4. Konfigurieren Sie die Testdaten für die Ressource im Abschnitt Eingabe und Parameter dieser Seite. Weitere Informationen finden Sie in Tabelle 32-6.
  5. Klicken Sie auf Ausführen:. Auf der Seite Ressource testen werden die Ergebnisse angezeigt.
  6. Klicken Sie auf Zurück, um erneut zu prüfen. Sie können die Testkonsole schließen, ändern und die Ressource erneut prüfen.
