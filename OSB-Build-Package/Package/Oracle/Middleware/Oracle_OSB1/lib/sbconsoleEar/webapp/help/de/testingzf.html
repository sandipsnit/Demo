XQuery-Ausdruck und XQuery-Bedingungseditoren verwenden

Mit XQuery-Ausdrücken erstellen Sie bei der Ausführung des Nachrichtenflusses den Datencontent für die Nachrichtenkontextvariablen (oder einen Teil der Nachrichtenkontextvariablen). Sie können die Testkonsole direkt im XQuery-Bedingungseditor verwenden, um die Definition des Ausdrucks zu testen.

Ebenso verwenden Sie XQuery-Bedingungen, um boolesche Bedingungen im Nachrichtenfluss auszuwerten. Sie können die Testkonsole direkt im XQuery-Bedingungseditor verwenden, um die Definition der Bedingung zu testen.

Eine XQuery kann mehrere Eingaben annehmen und eine Ausgabe zurückgeben. Jede Eingabe entspricht einer nicht gebundenen XQuery-Variablen, die in der XQuery definiert wurde. Der Wert einer XQuery-Eingabe kann ein primitiver Datentyp sein (Zeichenfolge, Integer, Datum und so weiter), ein XML-Dokument oder eine Folge dieser Typen. Der Ausgabewert kann ein primitiver Datentyp sein (Zeichenfolge, Integer, Datum und so weiter), ein XML-Dokument oder eine Sequenz dieser Typen.

Hinweis: Die Testkonsole unterstützt keine Sequenzen bei der Eingabe.

Dieser Abschnitt enthält ein Eingabefeld für jede der nicht gebundenen XQuery-Variablen.
<param_name> ([] als XML)
param_name ist der Name der entsprechenden nicht gebundenen XQuery-Variablen.
In der Testkonsole wird ein einzeiliges Bearbeitungsfeld angezeigt, wenn es sich um einen einfachen Typ handelt. Ein mehrzeiliges Bearbeitungsfeld wird angezeigt, wenn die Daten XML sind. Eine Kombinationseingabe (<param_name> ([] als XML)) wird verwendet, wenn die Variable nicht typisiert ist. Sie müssen den Variablentyp angeben. Aktivieren Sie das Kontrollkästchen, um einen Parameter von Typ XML zu kennzeichnen.
Eine XML-Eingabe kann dateibasiert oder textbasiert sein. Ein Verweis auf eine Datei für die Eingabe hat Vorrang vor einer Texteingabe. Suchen und markieren Sie die Datei, die Sie im Test verwenden möchten.
Eingaben in der Testkonsole basieren auf dem Typ, damit leichter verständlich ist, welcher Datentyp eingegeben werden muss. Der Standardtyp ist XML.

  1. Greifen Sie auf die Testkonsole zu, wenn Sie eine Aktion im Nachrichtenfluss einer Pipeline bearbeiten.
    • Wie Sie auf den XQuery-Ausdruckseditor zugreifen, ist hier beschrieben Inline XQuery- und XPath-Ausdrücke erstellen und bearbeiten.
    • Wie Sie auf den XQuery-Bedingungseditor zugreifen, ist hier beschrieben Inline XQuery- und XPath-Ausdrücke erstellen und bearbeiten.

      • Die Seite XQuery-Testausdrücke wird angezeigt. Alle erforderlichen Eingabevariablen werden auf der Seite angezeigt.

  2. Konfigurieren Sie die Testdaten für die XQuery im Abschnitt Variablen auf der Seite. Weitere Informationen dazu in Tabelle 32-8.
  3. Klicken Sie Ausführen. Die Testseite zeigt die Ergebnisse an.
  4. Sobald ein Test abgeschlossen ist, können Sie durch Klicken auf Zurück einen neuen Test durchführen. Um nach den an der XQuery vorgenommenen Änderungen einen neuen Test auszuführen, schließen Sie die Testkonsole, und öffnen Sie sie erneut, damit die Änderungen wirksam werden.
