XPath-Ausdruckseditor verwenden

Mit XPath-Ausdrücken können Sie einen Teil der Kontextvariablen einer XML-Nachricht wählen. Mit der Testkonsole im XPath-Ausdruckseditor können Sie die Definition des XPath-Ausdrucks testen. Ein XPath-Ausdruck verwendet ein einziges XML-Dokument als Eingabe und generiert eine Sequenz von XML-Dokumenten, primitiven Typen oder beidem als Ausgabe.

In Tabelle 32-9 werden die Konfigurationsoptionen von XPath-Ausdrücken beschrieben.

Tabelle 32-9 XPath-Optionen konfigurieren

Variablen
Dieser Abschnitt enthält ein einziges Eingabefeld, das dem XML-Dokument entspricht, mit dem dieser XPath-Ausdruck getestet wird.
 
Die XML-Eingabe kann datei- oder textbasiert sein. Das Referenzieren einer Datei für eine Eingabe gilt vorrangig gegenüber der Texteingabe. Suchen Sie nach der Datei, die im Test verwendet werden soll.

  1. Rufen Sie die Testkonsole auf, wenn Sie eine Aktion im Nachrichtenfluss einer Pipeline bearbeiten. Weitere Informationen zum Aufrufen des XPath-Ausdruckseditors finden Sie unter Inline XQuery und XPath-Ausdrücke erstellen und bearbeiten.
  2. Konfigurieren Sie die Testdaten für den XPath-Ausdruck im Abschnitt Variablen der Seite. Weitere Informationen finden Sie in Tabelle 32-9.
  3. Klicken Sie Ausführen. Die Testseite zeigt die Ergebnisse an.
  4. Wenn Sie einen Test abgeschlossen haben, können Sie die Schaltfläche Zurück verwenden, um einen neuen Test auszuführen. Um einen neuen Test nach dem Vornehmen von Änderungen am XPath-Ausdruck auszuführen, müssen Sie die Testkonsole schließen und erneut öffnen, damit die Änderungen wirksam werden.
