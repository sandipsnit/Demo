UDDI

UDDI-Registrys (UDDI für Universal Description, Discovery and Integration) werden in einem Unternehmen verwendet, um Web Services gemeinsam zu nutzen. UDDI liefert den Rahmen, in dem Sie Ihr Geschäft, dessen Services und die technischen Details zu den Services, die Sie zur Verfügung stellen möchten, klassifizieren können.

Um einen Service in einer Registry zu publizieren, müssen Sie den Service-Typ und die Datenstruktur kennen, die diesen Service in der Registry darstellen. Einem Eintrag in der Registry sind bestimmte Eigenschaften zugeordnet, und diese Eigenschaftentypen werden beim Erstellen der Registry definiert. Sie können Ihren Service in einer Registry publizieren und ihn für andere Organisationen zur Verfügung stellen. In Oracle Service Bus entwickelte Proxy-Services können in einer UDDI-Registry publiziert werden. Oracle Service Bus kann mit jeder Registry interagieren, die mit der UDDI-Version 3.0 kompatibel ist.

Weitere Informationen finden Sie unter "UDDI" im Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus unter http://www.oracle.com/pls/as111170/lookup?id=OSBAG392.
