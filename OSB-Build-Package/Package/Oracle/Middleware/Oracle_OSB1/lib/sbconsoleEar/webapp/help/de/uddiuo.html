UDDI-Registrys hinzufügen

Auf dieser Seite können Sie UDDI-Registrys hinzufügen und konfigurieren. Nach dem Konfigurieren der Registrys können Sie Oracle Service Bus-Proxy-Services darin veröffentlichen oder Business-Services daraus importieren, um sie in einem Oracle Service Bus-Proxy-Service zu verwenden.

Sie müssen sich in einer aktiven Session befinden, um eine Registry hinzuzufügen.

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Systemadministration > UDDI-Registrys.
  3. Klicken Sie im Bereich "UDDI-Registrys" auf Registry hinzufügen.
  4. Um die Registry zu konfigurieren, führen Sie folgende Schritte aus. Obligatorische Felder sind mit einem Sternchen gekennzeichnet. In Tabelle 29-1 werden die Eigenschaften beschrieben, die beim Hinzufügen einer Registry festgelegt werden müssen.
    1. Geben Sie im Feld Name einen Namen für die Registry ein.

      Befolgen Sie die Benennungseinschränkungen für Ressourcen bei der Namensgebung.
    2. Geben Sie im Feld Inquiry-URL einen Inquiry-URL in folgendem Format ein: http://host:port/APPLICATION_SERVER_CONTEXT/uddi/inquiry
    3. Geben Sie im Feld Publishing-URL einen Publishing-URL in folgendem Format ein: http://host:port/APPLICATION_SERVER_CONTEXT/uddi/publishing
    4. Geben Sie im Feld Sicherheits-URL einen Sicherheits-URL in folgendem Format ein: http://host:port/APPLICATION_SERVER_CONTEXT/uddi/security
    5. Geben Sie im Feld Abonnement-URL einen Abonnement-URL in folgendem Format ein:
      http://host:port/APPLICATION_SERVER_CONTEXT/uddi/subscription

      6. Standardmäßig ist der Wert für APPLICATION_SERVER_CONTEXT im Installationsprogramm registry.

    6. Geben Sie im Feld Benutzername den Benutzernamen für die Anmeldung bei der Registry-Konsole ein.
    7. Geben Sie im Feld Kennwort (Kennwort bestätigen) das Kennwort für die Anmeldung bei der Registry-Konsole ein.
    8. Aktivieren Sie das Kontrollkästchen tModels in Registry laden, um die Oracle Service Bus-tModels in dieser Registry zu veröffentlichen.

      9. Dieses Feld ist nur erforderlich, wenn in dieser Registry Proxy-Services veröffentlicht werden.

    9. Aktivieren Sie das Kontrollkästchen Automatischen Import aktivieren, um importierte Services automatisch mit der UDDI-Registry zu synchronisieren. Alle Services, die importiert werden, wenn diese Option aktiviert ist, werden mit der UDDI-Registry synchron gehalten.
      10. Hinweis: Bei der automatischen Synchronisierung handelt es sich um einen Hintergrundprozess. Sie können diesen Vorgang nicht mit der Sessionfunktion Rückgängig umkehren. Das Rückgängigmachen einer automatischen Synchronisierungsänderung ist nicht permanent, weil der Service im nächsten Synchronisierungszyklus erneut synchronisiert wird. Wenn ein importierter Service nicht mit der UDDI-Registry synchron sein soll, müssen Sie die Zuordnung dieses Service aufheben, um weitere Updates von der Registry zu vermeiden. Weitere Informationen finden Sie unter Zuordnung von Services aufheben.
  5. Klicken Sie auf Validieren, um die angegebenen URLs zu validieren.
  6. Klicken Sie auf Speichern, um die Registry mit den angegebenen Einstellungen zu konfigurieren und in der aktuellen Session zu speichern.
  7. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
