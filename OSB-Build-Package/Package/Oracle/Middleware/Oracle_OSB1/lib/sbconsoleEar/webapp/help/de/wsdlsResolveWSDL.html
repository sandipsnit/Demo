Nicht aufgelöste WSDL-Referenzen auflösen

Auf der Seite Referenzen einer WSDL-Ressource bearbeiten können Sie nicht aufgelöste WSDL-Referenzen auflösen, indem Sie die Zuordnung für WSDL-Referenzen, wie WSDL-Importe und XML-Schemaimporte, konfigurieren. Eine WSDL kann auch eine nicht aufgelöste Referenz zu einer WS-Policy haben.

Nicht aufgelöste WSDL- oder XML-Schemareferenz auflösen
  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Ressourcenbrowser > WSDLs.
  3. Klicken Sie auf der Seite Übersicht der WSDLs in der Spalte WSDL-Name auf den Namen der entsprechenden WSDL, um die Details der nicht aufgelösten Referenz anzuzeigen. Die Seite WSDL-Details anzeigen wird angezeigt.
  4. Klicken Sie auf Referenzen bearbeiten, um die Seite Referenzen einer WSDL-Ressource bearbeiten anzuzeigen.
  5. Wählen Sie im Feld Ressourcentyp den Wert WSDL oder XML-Schema.
  6. Klicken Sie auf Durchsuchen. Je nach Ressourcentyp wird der WSDL-Browser oder der XML-Schema-Browser angezeigt.
  7. Wählen Sie im WSDL-Browser oder im XML-Schema-Browser eine WSDL oder ein XML-Schema und anschließend eine Definition aus dem Fenster Definitionen.
  8. Klicken Sie auf Weiterleiten. Die gewählte WSDL oder das XML-Schema werden im Feld Ressourcenname angezeigt.
  9. Klicken Sie auf Speichern, um die Referenz aufzulösen.
  10. Klicken Sie auf der Seite WSDL-Details anzeigen auf OK. Der Status der WSDL wird als Gültig angezeigt.
Nicht aufgelöste WS-Policy-Referenz auflösen
  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Wählen Sie Ressourcenbrowser > WSDLs.
  3. Klicken Sie auf der Seite Übersicht der WSDLs in der Spalte WSDL-Name auf den Namen der entsprechenden WSDL, um die Details der nicht aufgelösten Referenz anzuzeigen. Die Seite WSDL-Details anzeigen wird angezeigt.
  4. Klicken Sie auf Referenzen bearbeiten, um die Seite Referenzen einer WSDL-Ressource bearbeiten anzuzeigen.
  5. Klicken Sie im Feld WS-Policy-Name auf Durchsuchen. Daraufhin wird der Policy-Browser angezeigt.
  6. Wählen Sie im Policy-Browser eine WS-Policy.
  7. Klicken Sie auf Weiterleiten. Die gewählte WS-Policy wird im Feld Ressourcenname angezeigt.
  8. Klicken Sie auf Speichern, um die Referenz aufzulösen.
  9. Klicken Sie auf der Seite WSDL-Details anzeigen auf OK. Der Status der WSDL wird als Gültig angezeigt.
