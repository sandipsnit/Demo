WSDLs bearbeiten

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Suchen Sie die WSDL. Weitere Informationen hierzu finden Sie unter WSDLs suchen.
  3. Klicken Sie auf den WSDL-Namen. Auf der Seite WSDL-Details anzeigen werden die in Tabelle 6-2, Tabelle 6-3 und Tabelle 6-4 dargestellten Informationen angezeigt.

    4. Tabelle 6-2 WSDL-Details 
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der diese WSDL erstellt oder in die Konfiguration importiert hat.
    Zuletzt geändert am
    Datum und Uhrzeit, zu dem der Benutzer diese WSDL erstellt oder in die Konfiguration importiert hat. Klicken Sie auf den Link mit dem Datum und der Uhrzeit, um die Änderungshistorie zu dieser Ressource anzuzeigen. Weitere Informationen finden Sie unter Seite "Änderungshistorie anzeigen".
    Referenzen
    Die Anzahl der Objekte, auf die diese WSDL verweist. Wenn solche Referenzen vorhanden sind, klicken Sie auf den numerischen Link, um eine Liste der Objekte anzuzeigen. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Referenziert von
    Die Anzahl der Objekte, die auf diese WSDL verweisen. Wenn solche Referenzen vorhanden sind, klicken Sie auf den numerischen Link, um eine Liste der Objekte anzuzeigen. Beispiel: Wenn Sie den Port oder das Binding dieser WSDL als Service-Typ für einen bestimmten Business-Service oder Proxy-Service wählen, wird der Business-Service oder Proxy-Service als Referenz aufgelistet, wenn Sie auf den Link klicken. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Beschreibung
    Eine Beschreibung dieser WSDL, sofern vorhanden.

    Standardmäßig wird die strukturelle Ansicht der WSDL angezeigt, in der die in Tabelle 6-3 dargestellten Informationen enthalten sind.

    Tabelle 6-3 WSDL-Strukturdetails
    Eigenschaft
    Beschreibung
    Ziel-Namespace
    Der Namespace, der zum Spezifizieren der in der WSDL enthaltenen Definitionen verwendet wird.
    WSDL-Definitionen
    Die Attribute und Gruppen, die der WSDL zugeordnet sind:
    • WSDL-Port-Typen
    • WSDL-Bindings
    • WSDL-Ports
    • XML-Schematypen
    • XML-Schemaelemente
    • WSDL-Importvorgänge
    • XML-Schemaimporte
    • WS-Policy-Referenzen
    • Implizite WS-Policy-Referenzen
    WSDL-Status
    Der Status der WSDL:
    • Gültig:

      • Alle Speicherorte für die in der aktuellen WSDL enthaltenen XML-Schemas oder WSDLs sind angegeben und sind gültig. Außerdem sind alle Speicherorte für verschachtelte XML-Schemas oder WSDLs (XML-Schemas oder WSDLs, die in den enthaltenen WSDLs oder XML-Schemas enthalten sind) angegeben und gültig.

    • Ungültig

      • Mindestens·einer·der·Speicherorte·für·die in·der·aktuellen·WSDL·enthaltenen·XML-Schemas·oder·WSDLs·ist·nicht·angegeben·oder·ist·ungültig.· Außerdem·ist·der·Speicherort·für·mindestens·ein·verschachteltes·XML-Schema·oder·eine·verschachtelte·WSDL·(XML-Schemas·oder·WSDLs,·die·in·den·enthaltenen·WSDLs·oder·XML-Schemas·enthalten·sind)·nicht·angegeben·oder·ungültig.·Um·Referenzen·auf·ein·ungültiges·XML-Schema·oder·eine·ungültige·WSDL·aufzulösen,·klicken·Sie·auf·"Referenzen bearbeiten". Weitere Informationen finden Sie unter Nicht aufgelöste WSDL-Referenzen auflösen.

  4. Klicken Sie auf Textansicht, um eine Textansicht der WSDL-Details anzuzeigen. In der Textansicht sind die in Tabelle 6-4 dargestellten Informationen enthalten.

    5. Tabelle 6-4 WSDL-Textdetails
    Eigenschaft
    Beschreibung
    Ziel-Namespace
    Der Namespace, der zum Spezifizieren der in der WSDL enthaltenen Definitionen verwendet wird.
    Text
    Der Text für diese WSDL.

  5. Um eine Änderung an den Feldern vorzunehmen, klicken Sie auf Bearbeiten. Beschreibungen zu den Feldern finden Sie unter WSDLs hinzufügen.
  6. Nehmen Sie die entsprechenden Bearbeitungen vor.
  7. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
  8. Um die Session zu beenden und die Konfiguration für die Laufzeit bereitzustellen, klicken Sie unter Change Center auf Aktivieren.
Hinweis: Auf der Seite WSDL-Details anzeigen wird möglicherweise eine Schaltfläche Separate Callbacks angezeigt. Diese Schaltfläche wird nur angezeigt, wenn die WSDL Port-Typen mit Callback-Vorgängen aufweist, die umgeschrieben werden können. Wenn Sie auf diese Schaltfläche klicken, wird die WSDL umgeschrieben.
