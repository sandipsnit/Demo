XSL-Transformationen bearbeiten

  1. Wenn dies noch nicht geschehen ist, klicken Sie zum Erstellen einer neuen Session auf Erstellen oder zum Aufrufen einer vorhandenen Session auf Bearbeiten. Weitere Informationen finden Sie unter Change Center verwenden.
  2. Suchen Sie wie unter XSL-Transformationen suchen beschrieben nach XSL-Transformationen.
  3. Klicken Sie auf den Namen der XSL-Transformation. Auf der Seite Details der SXL-Transformation anzeigen werden die in Tabelle 14-2 beschriebenen Informationen als strukturelle Ansicht angezeigt.

    4. Tabelle 14-2 Details der XSL-Transformationsressource 
    Eigenschaft
    Beschreibung
    Zuletzt geändert von
    Der Benutzer, der diese XSL-Transformation erstellt oder in die Konfiguration importiert hat
    Zuletzt geändert am
    Datum und Uhrzeit des Zeitpunkts, an dem der Benutzer die XSL-Transformation erstellt oder in die Konfiguration importiert hat. Klicken Sie auf den Link mit Datum und Uhrzeit, um die Änderungshistorie dieser Ressource anzuzeigen. Weitere Informationen finden Sie unter Seite "Änderungshistorie anzeigen".
    Referenzen
    Die Anzahl der Objekte, die diese XSL-Transformation referenziert. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste mit allen Objekten anzuzeigen. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Referenziert von
    Die Anzahl der Objekte, die diese XSL-Transformation referenzieren. Wenn solche Referenzen vorhanden sind, klicken Sie auf den Link mit der Anzahl, um eine Liste mit allen Objekten anzuzeigen. Beispiel: Wenn Sie diese XSL-Transformation einer Variablen im Nachrichtenfluss eines bestimmten Proxy-Service zugewiesen haben, wird der Proxy-Service als Referenz angezeigt, wenn Sie auf den Link klicken. Weitere Informationen finden Sie unter Referenzen auf Ressourcen anzeigen.
    Beschreibung
    Eine Beschreibung dieser XSL-Transformation
    Ziel-Namespace
    Mit dem Namespace werden die Definitionen in der XSL-Transformation gekennzeichnet.
    XSLT-Abhängigkeiten
    Die mit der XSLT verknüpften Attribute und Gruppen:
    • Der XSLT-Speicherort
    Status
    Der Status der XSLT-Transformation:
    • Gültig:

      • Alle Speicherorte der XSL-Transformation, die in der aktuellen XSL-Transformation eingeschlossen sind, wurden angegeben und sind gültig. Zudem wurden alle Speicherorte für verschachtelte XSL-Transformationen (XSL-Transformationen. die danach von den eingeschlossenen XSL-Transformationen eingeschlossen wurden) angegeben und sind gültig.

    • Ungültig

      • Ein oder mehrere Speicherorte der XSL-Transformation, die in der aktuellen XSL-Transformation eingeschlossen sind, wurden nicht angegeben oder sind ungültig. Zudem sind möglicherweise eine oder mehrere verschachtelte XSL-Transformationen (XSL-Transformationen, die anschließend von den enthaltenen XSL-Transformationen eingefügt wurden) nicht angegeben oder nicht gültig. Um ungültige XSL-Transformationen aufzulösen, klicken Sie auf Referenzen bearbeiten.Weitere Informationen finden Sie unter Nicht aufgelöste XLS-Transformationsreferenzen auflösen.

  4. Klicken Sie auf Textansicht, um eine Textansicht der XSL-Transformationsdetails anzuzeigen. Die Textansicht umfasst die in Tabelle 14-3 beschriebenen Informationen.

    5. Tabelle 14-3 Details des XSL-Transformationstext 
    Eigenschaft
    Beschreibung
    Ziel-Namespace
    Mit dem Namespace werden die Abhängigkeiten in der XSL-Transformation gekennzeichnet.
    XSLT
    Der vollständige Text der XSLT-Transformation.

  5. Wenn Sie Felder ändern möchten, klicken Sie auf Bearbeiten. Eine Beschreibung der Felder finden Sie in XSL-Transformationen hinzufügen.

    6. Sie können das Feld Ressourcenname nicht ändern.

  6. Klicken Sie auf Speichern, um die Aktualisierungen in der aktuellen Session zu speichern.
