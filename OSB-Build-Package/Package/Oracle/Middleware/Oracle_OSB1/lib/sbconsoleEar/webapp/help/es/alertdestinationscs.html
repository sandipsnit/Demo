Adición de Destinatarios de Correo Electrónico

Antes de agregar un destino de correo electrónico, debe configurar un servidor SMTP (consulte Adición de Servidores SMTP) o una sesión de JavaMail en WebLogic Server (consulte la sección sobre la configuración de acceso a JavaMail en Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help (Oracle Fusion Middleware: Ayuda en Pantalla de la Consola de Administración de Oracle WebLogic Server) en http://download.oracle.com/docs/cd/E15523_01/apirefs.1111/e13952/taskhelp/mail/CreateMailSessions.html). Si no hay ningún recurso de servidor SMTP o sesión de JavaMail configurada disponible, no puede configurar un destinatario de correo electrónico.

  1. Seleccione Enviar Alerta a través de Correo Electrónico de la lista desplegable y haga clic en Agregar para mostrar la página Agregar Destinatario de Correo Electrónico.
  2. En el campo Destinatarios del Correo, introduzca un destinatario de correo electrónico con el formato: mailto:nombre_usuario@nombre_host

    3. Este campo es necesario.

    Para especificar varios destinatarios de correo electrónico, introduzca sus nombres de usuario y de host en una lista separada por comas. Por ejemplo, mailto:nombre_usuario@nombre_host [,nombre_usuario_1@nombre_host_1]...[,nombre_usuario_n@nombre_host_n]

    Sólo el primer destinatario necesita el prefijo "mailto:".

  3. En el campo Servidor SMTP, seleccione el nombre del servidor SMTP para el correo electrónico saliente. Este campo no es necesario si se selecciona una sesión de correo en el siguiente paso.
  4. En el campo Sesión de Correo, seleccione una sesión de correo disponible. Este campo no es necesario si se selecciona un servidor SMTP en el paso anterior.
  5. En el campo Nombre del Remitente, proporcione el nombre del remitente para la notificación de alerta. Este campo es opcional.
  6. En el campo Dirección de Origen, proporcione una dirección de correo electrónico válida. Este campo es necesario si se especifica un valor para el campo "Nombre del Remitente".
  7. En el campo Nombre de Respuesta, proporcione el nombre al que se puede enviar una respuesta. Este campo es opcional.
  8. En el campo Dirección de Respuesta, proporcione una dirección de correo electrónico a la que se puede enviar una respuesta. Este campo es necesario si se especifica un valor para el campo "Nombre de Respuesta".
  9. En el campo Timeout de Conexión, introduzca el número de segundos que debe esperar una conexión una respuesta del servidor antes de que se produzca un timeout. El valor por defecto es 0.
  10. En el campo Codificación de Solicitud, introduzca un valor de codificación de juego de caracteres. El valor de codificación por defecto es iso-8859-1.
  11. Haga clic en Guardar. El destinatario del correo electrónico se agrega a la lista de destinatarios de este destino de alerta y se guarda en la sesión actual.
