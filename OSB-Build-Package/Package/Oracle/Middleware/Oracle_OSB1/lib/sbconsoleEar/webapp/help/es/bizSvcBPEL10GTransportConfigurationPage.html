Página Configuración de Transporte de BPEL-10g

Utilice esta página para configurar el transporte de un servicio de negocio mediante el protocolo de transporte de BPEL-10g (Oracle BPEL Process Manager). Para obtener más información sobre el uso de Oracle Service Bus con Oracle BPEL Process Manager, consulte la sección sobre el transporte de Oracle BPEL Process Manager en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1147.

Descripción
Rol
El transporte de BPEL se utiliza para enviar mensajes de solicitud de Oracle Service Bus a Oracle BPEL Process Manager. El servicio de negocio puede servir a uno de los siguientes roles:
  • Cliente Síncrono
    Para la comunicación síncrona con un cliente de Oracle Service Bus, la única información de ubicación necesaria es la dirección de BPEL. Esta dirección se captura de forma estática como URI de punto final y/o de forma dinámica mediante reescritura de URI.
  • Cliente Asíncrono
    Para la comunicación asíncrona con un cliente de Oracle Service Bus, se envía una devolución de llamada de Oracle BPEL Process Manager a Oracle Service Bus en una conexión diferente a la solicitud, y debe configurar Oracle Service Bus para proporcionar la dirección de devolución de llamada correcta. Para obtener más información, consulte la sección sobre casos de uso avanzados (asíncronos) en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
  • Devolución de Llamada de Servicio
    Si el servicio de negocio está diseñado para ser una devolución de llamada de servicio para Oracle BPEL Process Manager (donde Oracle BPEL Process Manager llama a un servicio externo mediante Oracle Service Bus), la dirección de devolución de llamada se conoce sólo en tiempo de ejecución. Utilice un URI de punto final de bpel://callback.

    • Si configura el servicio de negocio con el URI de marcador, configure la lógica de pipeline para definir de forma dinámica el URI en $outbound; por ejemplo, con la acción TransportHeader.

    Nota: Un servicio de negocio de devolución de llamada de servicio no soporta el equilibrio de carga ni el failover.
Proxy de Devolución de Llamada
Este campo opcional está disponible sólo para el rol Cliente Asíncrono. Este campo permite seleccionar el servicio de proxy (debe ser un proxy SB o HTTP del tipo Cualquier SOAP) que se utilizará para direccionar las devoluciones de llamada al cliente de Oracle Service Bus que realizó la solicitud. Para obtener más información, consulte la sección sobre casos de uso avanzados (asíncronos) en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
Cuenta de Servicios
Para la seguridad de contexto de JNDI, se utiliza para acceder al servicio de entrega de Oracle BPEL Process Manager. Haga clic en Examinar y seleccione una cuenta de servicio. Si no se especifica ninguna, se utiliza un sujeto anónimo.
No existe ninguna restricción en cuanto al tipo de cuenta de servicio que se puede configurar, como estática o de traspaso, pero en tiempo de ejecución se debe poder acceder a un nombre de usuario y contraseña.
Suspender Transacción
Al seleccionar Suspender Transacción, el servicio de negocio se vuelve no transaccional incluso si lo llama una transacción.
Si no selecciona Suspender Transacción:
  • Si el protocolo indica un protocolo soportado por WebLogic Server (t3, iiop, http), se propaga la transacción.
  • Si el protocolo indica un servidor OC4J (ormi, opmn), el transporte de BPEL emite una excepción, ya que OC4J no soporta la propagación de transacciones.
El transporte de BPEL ignora la opción Suspender Transacción en las siguientes situaciones:
  • El servicio de negocio se llama con la calidad de servicio (QoS) "Mayor Esfuerzo". El transporte de BPEL suspende automáticamente cualquier transacción que no soporta QoS.
  • El servicio de negocio se llama con QoS definido en "Una Sola Vez" y no hay ninguna transacción.
Para obtener una descripción de la propagación de transacciones, consulte la sección sobre propagación de transacciones en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1151.
Política de Distribución
Seleccione la instancia del gestor de trabajos de WebLogic Server que desea utilizar para la política de distribución de este punto final. Se utiliza el gestor de trabajos por defecto si no existe ningún otro.
Para obtener más información sobre los gestores de trabajo, consulte la sección sobre el uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
