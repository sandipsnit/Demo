Página Configuración de Transporte de EJB

La página Crear/Editar un Servicio de Negocio: Configuración de Transporte de EJB es una de una serie de páginas para la creación y edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Edición de Configuraciones de Servicio de Negocio.

Utilice esta página para configurar el transporte de un servicio de negocio mediante el protocolo de transporte de EJB. En la Tabla 18-5 se describe el uso de esta página.

Tabla 18-5 Crear/Editar un Servicio de Negocio: Página Configuración de Transporte de EJB

Transferir Sujeto del Emisor de Llamada
Active esta casilla de control para que Oracle Service Bus transfiera el sujeto autenticado del servicio de proxy al llamar a EJB si no se ha configurado ninguna cuenta de servicio. Tenga en cuenta que el campo Cuenta de Servicios está desactivado si se selecciona esta opción.
Cuenta de Servicios
Haga clic en Examinar y seleccione una cuenta de servicio de la lista mostrada. Si no se especifica ninguna, se utiliza un sujeto anónimo. Esta opción no está disponible si selecciona la opción Transferir Sujeto del Emisor de Llamada.

Soporta Transacción
Active esta casilla de control para especificar el soporte de la transacción.
Jar de Cliente
Haga clic en Examinar y seleccione un recurso JAR de cliente de EJB de la lista mostrada. Para obtener más información sobre la creación de recursos JAR, consulte JAR. Este campo es necesario.
Jar de Conversor
Haga clic en Examinar y seleccione un recurso JAR de clase de conversor EJB de la lista mostrada. Para obtener más información sobre los recursos JAR de cliente de EJB y clases de conversor, consulte la sección sobre transporte de EJB en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.
Interfaz Raíz
Sólo EJB 2.1. Seleccione la interfaz de EJBHome necesaria de las opciones proporcionadas por JAR. El nombre de JNDI de este ejemplo de URI debe estar asociado con la interfaz de EJBHome que se seleccione aquí. Si EJB no es el tipo que se necesita o no se especifica una interfaz de EJBHome en el JAR de cliente, Oracle Service Bus muestra una advertencia.
Interfaz Remota
Sólo EJB 2.1. Este campo se rellena automáticamente dependiendo de la configuración de la interfaz raíz.
Interfaz de Negocio
Sólo EJB 3.0. Seleccione la interfaz de negocio en el JAR de cliente que desea utilizar.
Espacio de Nombres de Destino
Este campo se rellena con la información seleccionada del JAR.
Estilo
Seleccione Documento Ajustado o RPC según sus necesidades. Si uno o varios métodos del EJB de sesión sin estado tienen el mismo número y tipo de dato de parámetros, y desea que las operaciones estén orientadas al documento, debe especificar que sean de documento ajustado.
El estilo es importante porque al direccionar o publicar en EJB, $body debe tener contenido que coincida con el estilo. Además al llamar a EJB, el estilo afecta al contenido del parámetro, en especial para documento ajustado. En segundo lugar, un patrón de uso debe definir un servicio de negocio de EJB y luego crear un servicio de proxy con el mismo WSDL que se direccione a EJB. En este caso, se debe prestar atención al estilo del WSDL porque la herramienta del cliente utilizada para llamar al proxy puede tener limitaciones en cuanto al estilo del WSDL.
Codificación
Seleccione Codificado o Literal.
Métodos
Los métodos mostrados son los de la interfaz de negocio o remota de EJB que seleccione. Seleccione los métodos necesarios (puede elegir varios). Haga clic en + para ampliar el método. Edite los valores de parámetro por defecto y seleccione un conversor, si se proporciona (o es necesario).
Debe excluir los métodos con parámetros o tipos de retorno no soportados por la pila de JAX-RPC, o bien, debe asociar esos argumentos a las clases de conversor.
Puede cambiar el nombre de operación por defecto para un determinado método. (Por defecto, el nombre de operación es el nombre del método.) Si EJB contiene métodos con el mismo nombre, debe cambiar los nombres de operación para que sean únicos; WSDL necesita nombres de operación únicos.
Nota: Si las credenciales o la configuración de transacción son diferentes en los métodos de un determinado EJB, puede personalizar los métodos de un determinado servicio de negocio y crear un servicio de negocio por método. Esto proporciona un control más detallado de las transacciones y las credenciales.
Excepciones
Este campo aparece si un método emite una excepción de negocio. Si un método de EJB emite una excepción indicando que tiene tipos de dato no soportados por los Servicios Web de Java (JWS), como ArrayList, utilice el campo Excepciones para seleccionar una clase de conversor que convierta la excepción a un tipo soportado por JWS.
La clase de conversor debe implantar com.bea.wli.sb.transports.ejb.ITypeConverter. Las clases de conversor sólo se pueden configurar para las excepciones comprobadas y no para las excepciones de tiempo de ejecución.
Empaquete la clase de conversor y la clase de excepción convertida en el JAR de cliente o conversor.
Para obtener más información, consulte la sección sobre transporte de EJB en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.
