Página Configuración de Transporte de Correo Electrónico

La página Crear/Editar un Servicio de Negocio: Configuración de Transporte de Correo Electrónico es una de una serie de páginas para la creación y edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Edición de Configuraciones de Servicio de Negocio.

Utilice esta página para configurar el transporte de un servicio de negocio mediante el protocolo de transporte de correo electrónico. En la Tabla 18-6 se describe el uso de esta página.

Tabla 18-6 Crear/Editar un Servicio de Negocio: Página Configuración de Transporte de Correo Electrónico 
Opción
Para crear o editar...
SSL Necesario
Seleccione esta opción para comunicarse mediante la capa de conexión segura. El número de puerto SSL por defecto para SMTP es 465.
Notas:
  • El certificado del servidor de correo electrónico debe estar presente en el almacén confianza de Oracle Service Bus para que SSL funcione.
  • Si utiliza TLS, debe colocar el certificado confianza en el almacén confianza por defecto ($JAVA_HOME/jre/lib/security/cacerts) o colocarlo en cualquier almacén confianza y agregar la ruta de acceso en el archivo $DOMAIN_HOME/bin/setDomainEnv.*, tal y como se muestra a continuación:

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<path to trust store>

Servidor SMTP
Seleccione el servidor SMTP por defecto que se utilizará para las entradas de URI de punto final de nombre@nombre_dominio.com. Si proporciona parámetros del servidor SMTP en el URI de punto final, como se describe en la Tabla 18-3, se utilizarán esos recursos del servidor en lugar de este valor de Servidor SMTP.
No seleccione un servidor SMTP si utiliza la opción Sesión de Correo.
Para obtener más información sobre la creación de recursos de SMTP, consulte Adición de Servidores SMTP.
Sesión de Correo
Introduzca el nombre de JNDI de la sesión de correo configurada que se utilizará para las entradas de URI de punto final de nombre@nombre_dominio.com. Si proporciona parámetros de sesión de correo de JNDI en el URI de punto final, como se describe en la Tabla 18-3, se utilizarán esas sesiones de correo en lugar de este valor de Sesión de Correo.
No introduzca una sesión de correo si utiliza la opción Servidor SMTP.
Nombre del Remitente
Introduzca un nombre mostrado para la cuenta de correo electrónico de origen para este servicio.
Dirección de Origen
Introduzca la cuenta de correo electrónico de origen para este servicio.
Nombre de Respuesta
Introduzca un nombre mostrado para la cuenta de correo electrónico de respuesta.
Dirección de Respuesta
Introduzca una dirección de correo electrónico para la respuesta.
Timeout de Conexión
Introduzca el intervalo de timeout, en segundos, antes de que se cierre la conexión. Si introduce 0, no hay timeout.
Codificación de Solicitud
Acepte el valor por defecto ISO-8859-1 como codificación del juego de caracteres para las solicitudes de transportes de correo electrónico o introduzca otra diferente.
