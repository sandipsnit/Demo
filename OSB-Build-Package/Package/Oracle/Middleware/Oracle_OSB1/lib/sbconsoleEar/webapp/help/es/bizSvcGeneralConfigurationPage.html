Página Configuración General

La página Crear/Editar un Servicio de Negocio: Configuración General es una de una serie de páginas para la creación y edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Edición de Configuraciones de Servicio de Negocio.

Utilice esta página para modificar la configuración general de un servicio de negocio.

Al crear un servicio de negocio, ésta es la primera página que aparece de una serie de páginas para la configuración del servicio. Las páginas mostradas después de ésta dependen de las opciones seleccionadas.

En la Tabla 18-1 se describe el uso de esta página.

Tipo de Servicio
Un tipo de servicio define los tipos y el empaquetamiento de los mensajes intercambiados por el servicio. Seleccione el tipo de servicio de negocio que desee crear:
  • Servicio Web WSDL: Seleccione esta opción para crear un servicio de negocio basado en WSDL. A continuación, introduzca el nombre de WSDL, cualificado por su ruta de acceso (por ejemplo, miProyecto/miCarpeta/miWSDL). Como alternativa, haga clic en Examinar para seleccionar un WSDL de la página Seleccione un WSDL.

    • (puerto o enlace): Introduzca el nombre de un puerto (definido en WSDL) para describir la dirección de transporte real, o bien, introduzca el nombre de un enlace (definido en WSDL) para asignar una dirección de transporte. Si utiliza Examinar para seleccionar un WSDL, como se ha descrito anteriormente, en la página Seleccione una definición de WSDL aparecen los puertos y enlaces definidos en el WSDL. Al elegir un puerto o enlace en esa página, se rellena el campo (puerto o enlace) con el nombre seleccionado.

  • Servicio de Transporte según Configuración: Seleccione esta opción para crear un servicio que utilice transporte de EJB o flujo.
  • Servicio de Mensajes: Seleccione esta opción para crear un servicio que intercambie mensajes de muy diferente tipo de contenido. Estos intercambios pueden ser de solicitud/respuesta o unidireccionales. También puede tener una respuesta sin solicitud si se utiliza con la opción de HTTP `GET' para el transporte de HTTP. A diferencia de los servicios web, el tipo de contenido de la solicitud y respuesta no tiene que ser el mismo.
  • Cualquier Servicio SOAP: Seleccione esta opción para crear un servicio SOAP que no tenga una interfaz concreta definida explícitamente.

    • Mantenga el valor por defecto SOAP 1.1 o seleccione SOAP 1.2 de la lista desplegable.

  • Cualquier Servicio XML: Seleccione esta opción para crear un servicio XML que no tenga una interfaz concreta definida explícitamente.

    • HTTP GET sólo se soporta para servicios de mensajes y este tipo de servicio.

  • Servicio de Negocio: Seleccione esta opción para clonar un servicio de negocio existente.

    • Introduzca la ruta de acceso (proyecto/carpeta) y el nombre del servicio de negocio; o bien, haga clic en Examinar para seleccionarlo de la página Resumen de Servicios de Negocio.

    Ya que Oracle Service Bus no acepta el mismo URI para varios servicios, debe cambiar el URI del servicio clonado.

  • Servicio de Proxy: Seleccione esta opción para crear un servicio de negocio basado en un servicio de proxy existente.
Tipo de Servicio (continuación)
Nota: Cuando se crea un servicio a partir de otro servicio, las reglas de alertas se mantienen de la siguiente forma:
    • Cuando se crea un servicio de proxy a partir de un servicio de negocio o un servicio de negocio a partir de un servicio de proxy, se eliminan las reglas de alertas, si las hay.
    • Cuando se crea un servicio de proxy a partir de otro servicio de proxy o un servicio de negocio a partir de otro servicio de negocio, se mantienen las reglas de alertas, si las hay.

Después de Terminar

Haga clic en Siguiente para seguir configurando este servicio en la siguiente página; o haga clic en Último para revisar y guardar esta configuración en Crear/Editar un Servicio de Negocio: Página Resumen.
