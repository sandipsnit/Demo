Página Configuración de Transporte de HTTP

El transporte de HTTP soporta puntos finales de HTTP y HTTPS.

La página Crear/Editar un Servicio de Negocio: Configuración de Transporte de HTTP es una de una serie de páginas para la creación y edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Edición de Configuraciones de Servicio de Negocio.

Utilice esta página para configurar el transporte de un servicio de negocio mediante el protocolo de transporte de HTTP. En la Tabla 18-10 se describe el uso de esta página.

Timeout de Lectura
Introduzca el intervalo de timeout de lectura, en segundos.
Un valor cero (0) indica que no hay timeout.
Timeout de Conexión
Introduzca el intervalo de timeout de conexión en segundos. Si caduca el timeout antes de establecer la conexión, Oracle Service Bus emite un error de conexión.
Un valor cero (0) indica que no hay timeout.
Método de Solicitud HTTP
Este parámetro permite utilizar uno de los siguientes métodos HTTP en una solicitud:
  • POST: Transfiere todos los datos, sin límite de longitud, directamente por la conexión de socket como parte de su cuerpo de solicitud HTTP. El intercambio es invisible para el cliente y la URL no cambia. Para las solicitudes basadas en REST, POST indica al transporte que realice una operación de creación/sustitución o realice una acción con la solicitud.
  • GET: Como parte de la solicitud, puede incluir información propia que describa mejor lo que se debe obtener. Esta información se transfiere como una secuencia de caracteres agregada a la URL de la solicitud en una cadena de consulta. Puede utilizar GET en un servicio de negocio con un tipo de servicio "Cualquier Servicio XML" o "Servicio de Mensajes" si la opción Tipo de Mensaje de Solicitud se ha definido en "Ninguno". Para las solicitudes basadas en REST, GET recupera una representación de un recurso remoto.
  • PUT: Puede utilizar PUT en un servicio de negocio con un tipo de servicio "Cualquier Servicio XML" o "Servicio de Mensajes". PUT indica al transporte que realice una operación de creación/sustitución con una solicitud basada en REST, como la descarga de un archivo en una ubicación conocida.
  • HEAD: Puede utilizar HEAD en un servicio de negocio con un tipo de servicio "Cualquier Servicio XML" o "Servicio de Mensajes" si la opción Tipo de Mensaje de Respuesta se ha definido en "Ninguno". HEAD indica al transporte que obtenga la información de cabecera de un recurso remoto en lugar de la representación completa del recurso en una solicitud basada en REST.
  • DELETE: Puede utilizar PUT en un servicio de negocio con un tipo de servicio "Cualquier Servicio XML" o "Servicio de Mensajes". DELETE indica al transporte que realice una operación de supresión con una solicitud basada en REST.
Nota: Si ya se ha definido un método en la variable $outbound/transport/request/http:http-method, ese valor tiene prioridad respecto a los métodos que seleccione para Método de Solicitud HTTP.
Autenticación
Seleccione una de las siguientes opciones:
  • Ninguna: Especifica que no es necesaria la autenticación para acceder a este servicio.
  • Básica: Especifica que es necesaria la autenticación básica para acceder a este servicio.

    • La autenticación básica indica a WebLogic Server que autentique el cliente con un nombre de usuario y contraseña según los proveedores de autenticación configurados en el dominio de seguridad, como un servicio de directorio Lightweight Directory Access Protocol (LDAP) y Windows Active Directory. El cliente debe enviar su nombre de usuario y contraseña en la cabecera de la solicitud HTTP.

    No se recomienda en absoluto el uso de la autenticación básica en HTTP porque la contraseña se envía en texto sin cifrado. Sin embargo, es seguro el envío de contraseñas por HTTPS porque dispone de un canal cifrado.

    Advertencia: Por defecto, todos los usuarios (autorizados y anónimos) pueden acceder a un servicio de negocio. Para limitar su acceso, cree una política de autorización de nivel de transporte. Consulte Edición de Políticas de Acceso de Nivel de Transporte.

  • Certificado de Cliente: Especifica la comunicación cifrada y la autenticación de cliente segura (SSL bidireccional). Para obtener más información, consulte la sección sobre configuración de seguridad de nivel de transporte en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
Cuenta de Servicios
Introduzca una cuenta de servicio. Una cuenta de servicio es un recurso de alias para un nombre de usuario y una contraseña. Este campo es necesario si se ha seleccionado el campo Autenticación Básica Necesaria.

Política de Distribución
Seleccione la instancia del gestor de trabajos de WebLogic Server que desea utilizar para la política de distribución de este punto final. Se utiliza el gestor de trabajos por defecto si no existe otro. Para obtener más información sobre los gestores de trabajo, consulte la sección sobre uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificación de Solicitud
Acepte el valor por defecto iso-8859-1 como codificación del juego de caracteres para las solicitudes de transportes de HTTP o introduzca otra diferente.
Codificación de Respuesta
Acepte el valor por defecto iso-8859-1 como codificación del juego de caracteres para las respuestas de transportes de HTTP o introduzca otra diferente.
Servidor Proxy
Introduzca un recurso de servidor proxy o haga clic en Examinar para seleccionar una entrada de la lista de recursos de servidor proxy configurados.
Seguir Redireccionamientos HTTP
Active esta casilla de control para especificar que se deben seguir automáticamente los redireccionamientos HTTP (que son solicitudes con un código de respuesta 3xx). Se produce un redireccionamiento al enviar una solicitud saliente a la URL de un servicio de negocio y ese servicio devuelve un código de respuesta (por ejemplo, 302) que indica que la URL ya no es válida y se debe enviar a otra.
Si activa la casilla de control Seguir Redireccionamientos HTTP, Oracle Service Bus reenvía automáticamente la solicitud a la nueva URL sin que tenga que realizar ninguna acción. Desactive la casilla si no desea que se sigan automáticamente los redireccionamientos HTTP.
Usar Modo de Flujo Fraccionado
Seleccione esta opción si desea utilizar codificación de transferencia fraccionada de HTTP para el envío de mensajes.
Nota: No utilice el flujo fraccionado si usa la opción Seguir Redireccionamientos HTTP. El redireccionamiento y la autenticación no se pueden manejar automáticamente en modo fraccionado.
Persistencia de Sesión
Seleccione esta opción si desea utilizar la persistencia de sesión (también se conoce como afinidad de sesión) para las solicitudes HTTP para el servicio de negocio.
Nombre de ID de Sesión Persistente
Si Persistencia de Sesión está activada, introduzca un nombre de identificación para la sesión.
