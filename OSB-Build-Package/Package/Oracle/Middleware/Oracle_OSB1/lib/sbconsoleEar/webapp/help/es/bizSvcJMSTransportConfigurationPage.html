Página Configuración de Transporte de JMS

La página Crear/Editar un Servicio de Negocio: Configuración de Transporte de JMS es una de una serie de páginas para la creación y edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Edición de Configuraciones de Servicio de Negocio.

Para obtener más información, consulte la sección sobre transporte de JMS en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Utilice esta página para configurar el transporte de un servicio de negocio mediante el protocolo de transporte de JMS. En la Tabla 18-13 se describe el uso de esta página.

Tabla 18-13 Crear/Editar un Servicio de Negocio: Página Configuración de Transporte de JMS (Continuación) 
Opción
Para crear o editar...
Tipo de Destino
Seleccione un tipo para el destino de bridge de JMS:
  • Cola (para punto a punto)
  • Tema (para publicación/suscripción)
Tipo de Mensaje
Seleccione una de las siguientes opciones:
  • Bytes (para flujo de bytes no interpretados)
  • Texto (para mensajes de texto)
Esta opción está desactivada si selecciona Java como tipo de mensaje para la respuesta.
Colas de Respuesta
Esta opción sólo está disponible si se selecciona Cola para Tipo de Destino.
Seleccione una de las opciones de respuesta:
  • Ninguno: No se espera respuesta. Defina esta opción para las operaciones unidireccionales.
  • Uno para Todos los URI de Solicitud: Permite introducir un solo URI para manejar la respuesta, así como definir otros detalles de configuración de respuesta como codificación y timeout, y tiene la opción de seleccionar una cuenta de servicio de JMS para transferir credenciales de JMS/JNDI.
  • Uno por URI de Solicitud: Esta opción proporciona failover de respuesta, lo que permite introducir un URI de respuesta o destino para cada URI de solicitud.
Puede seleccionar una cuenta de servicio para las credenciales de JMS/JNDI en cada par solicitud/respuesta.
Patrón de Respuesta
Esta opción sólo está disponible si selecciona una nueva opción de respuesta en el campo Colas de Respuesta.
Seleccione una de las siguientes opciones:
  • Seleccione JMSCorrelationID para todos los servicios excepto JAX-RPC que se ejecuten en Oracle WebLogic Server.
  • Seleccione JMSMessageID para los servicios JAX-RPC que se ejecuten en Oracle WebLogic Server.
Para obtener más información, consulte la sección sobre patrones de identificador de mensaje e identificador de correlación para solicitud/respuesta de JMS en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051.
Política de Distribución
Seleccione la instancia del Gestor de Trabajos de Oracle WebLogic Server que desea utilizar para la política de distribución de este punto final. Se utiliza el Gestor de Trabajos por defecto si no existe ningún otro.
Por ejemplo, si el servicio de negocio tiene un protocolo de transporte de JMS, el punto final del servicio de negocio es un archivo JAR de MDB (bean controlado por mensaje) que puede asociar a la política de distribución en cuestión.
Para obtener más información sobre los gestores de trabajo, consulte la sección sobre uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificación de Solicitud
Introduzca el juego de caracteres para las solicitudes de codificación. El valor por defecto es UTF-8.
Codificación de Respuesta
Esta opción sólo está disponible si selecciona una de las opciones de respuesta en el campo Colas de Respuesta.
Introduzca el juego de caracteres para las respuestas de codificación. El valor por defecto es UTF-8.
Timeout de Respuesta
Esta opción sólo está disponible si selecciona una de las opciones de respuesta en el campo Colas de Respuesta.
Introduzca la cantidad de tiempo, en segundos, que se espera la respuestas antes de cerrar la conexión. El valor por defecto, cero (0), significa que la respuesta nunca tiene timeout.
Jar de Cliente
Esta opción está disponible cuando el servicio es un servicio de mensajes con un tipo de respuesta de Java. Seleccione el JAR de cliente que va a utilizar para eliminar de la cola los mensajes que contienen objetos Java. La selección del JAR de cliente garantiza que está en la classpath.

Para obtener más información, consulte la sección de envío y recepción de objetos Java en los mensajes en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
URI de Respuesta
Esta opción está disponible si selecciona la opción de respuesta Uno para Todos los URI de Solicitud y el patrón de respuesta JMSCorrelationID.
Introduzca un URI de respuesta con el formato:
jms://host:puerto/fábrica_conexiones/destino_jndi
Para utilizar varios servidores, utilice el siguiente formato:
jms://host1:puerto,host2:puerto/fábrica_conexiones/destino_jndi
También puede omitir el host y puerto en el URI de respuesta. Por ejemplo:
jms:///fábrica_conexiones/destino_jndi

Si se omiten, la consulta de fábrica de conexiones/destino se produce en el servidor local. Esto es útil, por ejemplo, si el URI de solicitud va a una fábrica de conexiones/destino en el extranjero, pero desea que la respuesta se envíe al servidor local.

Nota: aunque Oracle WebLogic Server permite el uso de barras inclinadas en los nombres de JNDI (por ejemplo, "miscolas/micola"), los nombres de JNDI con barras inclinadas interfieren en el formato de URI que necesita Oracle Service Bus, por lo que no se pueden utilizar esos nombres. Para solucionar este problema, defina un servidor ajeno de JMS y haga referencia a él en el URI. Para obtener más información, consulte la sección sobre configuración de servidores ajenos en Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help (Ayuda en Pantalla de la Consola de Administración de Oracle Fusion Middleware Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Respuestas de la Solicitud
Esta opción está disponible si selecciona la opción de respuesta Uno por URI de Solicitud y el patrón de respuesta JMSCorrelationID para proporcionar failover de respuesta.
Para cada URI de solicitud introducida en la página de transporte genérico, introduzca un URI de respuesta, como se describe en el campo anterior URI de Respuesta.
Puede seleccionar una cuenta de servicio opcional para las credenciales de JMS/JNDI que utiliza el servicio para las colas de solicitud y respuesta.
Objetivo: Respuestas
Esta opción está disponible si selecciona la opción de respuesta Uno para Todos los URI de Solicitud y el patrón de respuesta JMSMessageID.
Introduzca el nombre del servidor de destino que va a recibir las respuestas e introduzca un URI de respuesta, como se describe en el campo URI de Respuesta.
Conexiones de Solicitud
Esta opción está disponible si selecciona la opción de respuesta Uno por URI de Solicitud y el patrón de respuesta JMSMessageID para proporcionar failover de respuesta.
Para cada URI de solicitud, identificado secuencialmente mediante números en el campo Número de secuencia, puede introducir un nombre de fábrica de conexiones de JMS. Si no introduce un nombre, el transporte de JMS utiliza la fábrica de conexiones de la URI de solicitud.
Puede seleccionar una cuenta de servicio opcional para las credenciales de JMS/JNDI que utiliza el servicio para las colas de solicitud y respuesta.
Objetivo: Destinos
Esta opción está disponible si selecciona la opción de respuesta Uno por URI de Solicitud y el patrón de respuesta JMSMessageID para proporcionar failover de respuesta. Utilice este campo junto con Conexiones de Solicitud.
Cada servidor de destino de la lista (determinados por los servidores del dominio actual), como los servidores gestionados en un cluster, muestra los URI de solicitud por número de secuencia, que corresponde a los del campo Conexiones de Solicitud. Para cada URI de solicitud de cada destino, introduzca el valor de cola Destino en el servidor de destino que recibe las respuestas.
Nota: ya que el entorno de desarrollo de Oracle Service Bus sólo soporta un entorno de un servidor, sólo se muestra un destino. Para configurar este campo en un entorno de varios servidores, despliegue este servicio en el entorno de tiempo de ejecución y realice la configuración de servicio en la consola de administración de Oracle Service Bus.
Cuenta de Servicio de JMS
Esta opción está disponible si selecciona la opción Ninguno o Uno para Todos los URI de Solicitud en el campo Colas de Respuesta.
Seleccione la cuenta de servicio que se utilizará para el recurso de JMS gestionado por el servidor de JMS. Una cuenta de servicio es un recurso de alias para un nombre de usuario y contraseña, que se utiliza para la solicitud y la respuesta. La misma cuenta de servicio se utiliza tanto para JMS como para JNDI.
Este campo se excluye si se utiliza la opción Transferir Sujeto del Emisor de Llamada y viceversa. Utilice una de las dos para la autenticación de JMS/JNDI.
Para obtener más información, consulte Cuentas de Servicio.
Usar SSL
Seleccione esta opción sólo si las solicitudes se realizan mediante una conexión TLS/SSL.
TLS/SSL (Secure Sockets Layer) proporciona conexiones seguras al permitir que dos aplicaciones se conecten mediante una red para autenticar la identidad de la otra y mediante el cifrado de los datos intercambiados entre las aplicaciones. La autenticación permite que un servidor y, como opción, un cliente verifique la identidad de la aplicación en el otro extremo de una conexión de red. Además, si el administrador tiene acceso restringido a los destinos individuales de JMS (colas o temas) al definir el control de acceso en la entrada de JNDI para el destino, el servicio se debe autenticar al consultar la entrada en el árbol de JNDI. Realice la autenticación con una cuenta de servicio o con la opción Transferir Sujeto del Emisor de Llamada.
Caducidad
Intervalo de tiempo, en milisegundos, tras el cual caduca el mensaje. El valor por defecto es 0, lo que significa que el mensaje nunca caduca.
Activar Persistencia de Mensajes
Modo de entrega de JMS que utiliza el servicio, permite equilibrar la fiabilidad con el rendimiento. Seleccione esta opción (por defecto) para una entrega de mensajes garantizada. No la seleccione para mejorar el rendimiento si es tolerable la pérdida ocasional de mensajes.
Unidad de Orden
Introduzca una unidad de orden de los mensajes. Permite a los productores de mensajes agrupar los mensajes en una sola unidad con respecto al orden de procesamiento. La unidad de orden única requiere que todos los mensajes de esa unidad se procesen en secuencia en el orden en que se crearon.
Transferir Sujeto del Emisor de Llamada
Active esta casilla de control para que Oracle Service Bus transfiera el sujeto autenticado al enviar un mensaje.
Si activa este campo y el destino del servicio de negocio son recursos de JMS en un dominio diferente, active la protección global en ambos dominios. Consulte Configuring Security for a WebLogic Domain (Configuración de Seguridad para un Dominio de WebLogic) en Securing Oracle WebLogic Server (Protección de Oracle WebLogic Server).
Este campo se excluye si se utiliza la opción Cuenta de Servicio y viceversa. Utilice una de las dos para la autenticación de JMS/JNDI.
Si activa este campo y el destino del servicio de negocio son recursos de JMS en un dominio diferente, active la protección global en ambos dominios. Consulte la sección sobre configuración de seguridad para un dominio de WebLogic) en Oracle Fusion Middleware Securing Oracle WebLogic Server (Protección de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=SECMG402.
Timeout de JNDI
Timeout de conexión de JNDI (en segundos) utilizado al consultar el destino o la fábrica de conexiones en el árbol de JNDI.
El valor por defecto, cero (0), significa que la conexión nunca sufre un timeout.
