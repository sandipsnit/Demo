Página Configuración de Transporte de MQ

La página Crear/Editar un Servicio de Negocio: Configuración de Transporte de MQ es una de una serie de páginas para la creación y edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Edición de Configuraciones de Servicio de Negocio.

Para obtener más información, consulte la sección sobre transporte de MQ en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Antes de Empezar

Configure un recurso de conexión de MQ. Consulte Conexiones de MQ.

Para configurar el transporte de MQ

Utilice esta página para configurar el transporte de un servicio de negocio mediante el protocolo de transporte de MQ. En la Tabla 18-14 se describe el uso de esta página.

Tabla 18-14 Crear/Editar un Servicio de Negocio: Página Configuración de Transporte de MQ 
Opción
Para crear o editar...
Tipo de Mensaje
Seleccione una de las siguientes opciones:
  • Bytes (para flujo de bytes no interpretados)
  • Texto (para mensajes de texto)
Se Necesita Respuesta
Seleccione esta opción para especificar que se espera una respuesta después de enviar el mensaje saliente.
Intervalo de Sondeo
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Introduzca un intervalo de sondeo en milisegundos. El valor por defecto es 1000.
Patrón de Correlación de Respuesta
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Especifique si el patrón de correlación de respuesta se debe basar en:
  • Identificador de Mensaje
  • Identificador de Correlación
  • Cola Dinámica: Seleccione esta opción si la implantación de WebSphere MQ utiliza colas dinámicas para la correlación de respuesta. El transporte de MQ soporta sólo colas dinámicas temporales.
Valor de Correlación de Generación Automática
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Active esta casilla de control para generar automáticamente un identificador de correlación o de mensaje.
Cola de Modelo
Sólo para el patrón de correlación de respuesta de cola dinámica. Introduzca el nombre de la cola de modelo utilizada para generar la cola dinámica.
URI de Respuesta de MQ
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Destino en el que se debe publicar la respuesta. Introduzca un URI de respuesta en el mismo formato que el URI de punto final:
mq://nombre_cola_local?conn=recurso_conexión_mq
o bien, si utiliza colas dinámicas:
mq://prefijo_cola_dinámica?conn=recurso_conexión_mq
El prefijo de la cola dinámica, que se limita a 32 caracteres, se utiliza para crear la cola dinámica en el servidor de MQ. El nombre de la cola se convierte en el prefijo más un identificador único. Por ejemplo, si el prefijo de la cola dinámica es example, el nombre de la cola dinámica sería example123129083821.
También puede utilizar un asterisco (*) como comodín en el URI de respuesta de la cola dinámica. Por ejemplo:
mq://prefijo_cola_dinámica*?conn=recurso_conexión_mq
mq://prefijo_cola_dinámica*
mq://*
Si no proporciona un prefijo de cola dinámica en el URI, el transporte utiliza el nombre de cola dinámica generado por el servidor de MQ. Si no proporciona un recurso de conexión de MQ explícito en el URI (recomendado), el transporte utiliza el recurso de conexión de MQ del URI de punto final.
Para obtener más información, consulte la sección sobre transporte de MQ en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.
Timeout de Respuesta
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Introduzca el número de segundos que se esperará una respuesta antes de cerrar la conexión.
Política de Distribución
La política de distribución hace referencia a la instancia del Gestor de Trabajo de WLS que desea utilizar para el punto final del servicio.
Para obtener más información sobre los gestores de trabajo, consulte la sección sobre el uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Pool de Cola Dinámica
Sólo para el patrón de correlación de respuesta de cola dinámica. Seleccione esta opción si desea que el servicio utilice conexiones de pool para las colas dinámicas.
Si desea utilizar un pool de conexiones diferente para las colas dinámicas, puede configurar un recurso de conexión de MQ dedicado para las colas dinámicas.
No seleccione esta opción si desea crear una instancia de cola dinámica nueva en cada solicitud (y destruir la cola después de la respuesta).
Opciones `PUT' de URI de punto final
Introduzca las opciones de mensaje PUT de MQ de entre las siguientes:
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
Puede utilizar "|" o "+" para separar varias opciones. Por ejemplo, puede especificar lo siguiente:
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
Las opciones de mensaje PUT de MQ se aplican cuando el mensaje se coloca en la cola saliente.
URI de Respuesta No Reconocida de MQ
Introduzca el URI que representa la cola a la que se deben enviar los mensajes de respuesta no reconocidos. Tenga en cuenta que este valor está disponible sólo si se activa la casilla de control Valor de Correlación de Generación Automática.
Si no especifica un valor para este campo, los mensajes de respuesta no reconocidos se suprimen.
Cabeceras RFH2 de Proceso
Seleccione esta opción para analizar las cabeceras RFH2 de WebSphere MQ desde una carga útil de mensaje y generar automáticamente una cabecera de transporte RFH2Headers que contenga los datos de RFH2.
Si no selecciona esta opción, la carga útil se transfiere como recibida.

Para obtener más información sobre la forma en que el transporte de MQ maneja las cabeceras RFH2, consulte la sección sobre cabeceras RFH2 en la Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
