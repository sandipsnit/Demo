Página Configuración de Transporte de SOA-DIRECT

En la Tabla 18-17 se describen las opciones de configuración específicas del transporte para el transporte de SOA-DIRECT.

Para obtener más información sobre el transporte de SOA-DIRECT, consulte la sección sobre transporte de SOA-DIRECT en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV909.

Tabla 18-17 Crear/Editar un Servicio de Negocio: Configuración de Transporte de SOA-DIRECT
Propiedad
Descripción
Cuenta de Servicio de JNDI
Opcional. Especifica las credenciales de seguridad para la consulta de JNDI del servicio de SOA de destino. La cuenta de servicio debe ser estática. Haga clic en Examinar y seleccione una cuenta de servicio. Si no se especifica ninguna, se utiliza un sujeto anónimo.
Rol
Necesario. Identifica el patrón de comunicación que utiliza el servicio. Seleccione una de las siguientes opciones:
  • Cliente Síncrono (valor por defecto): En este rol, ya que no hay devolución de llamada asíncrona, está desactivada la opción Proxy de Devolución de Llamada. También está desactivada la opción Versión de WS-Addressing.
  • Cliente Asíncrono: En este rol, ya que normalmente se necesita una devolución de llamada asíncrona, puede identificar un proxy de devolución de llamada y debe seleccionar una versión de WS-Addressing. La opción asíncrona sólo está permitida si el servicio de WSDL es de tipo SOAP.
  • Devolución de Llamada de Servicio: Este rol es para la devolución de llamada asíncrona a un servicio de SOA después de una llamada de servicio externo.

    • El equilibrio de carga o el failover no están disponibles para los servicios en el rol Devolución de Llamada de Servicio.

Proxy de Devolución de Llamada
Opcional. Se activa sólo para el rol Cliente Asíncrono.
Esta opción permite especificar el servicio de proxy que recibe la devolución de llamada. Al seleccionar un proxy de devolución de llamada, si no se proporciona WS-Addressing en la solicitud o en el pipeline del servicio de proxy, Oracle Service Bus rellena automáticamente la propiedad ReplyTo en la cabecera de SOAP. Debe seleccionar un servicio de proxy de WSDL que utilice el transporte de SB (para RMI) y el servicio de proxy de devolución de llamada debe entender WS-Addressing.
Se utilizan las propiedades de WS-Addressing que se envían en la solicitud o se definen en el pipeline de servicio de proxy en lugar del proxy de devolución de llamada definido en esta opción.
Si no especifica un proxy de devolución de llamada y la solicitud no contiene propiedades ReplyTo, debe proporcionar estas propiedades en la cabecera de SOAP mediante el pipeline de servicio de proxy.
Proxy de Fallos
Esta opción no está soportada actualmente. Debe configurar los servicios de devolución de llamada para que manejen los fallos en un patrón asíncrono.
Versión de WS-Addressing
Necesario. Se activa sólo para el rol Cliente Asíncrono.
Especifique la versión por defecto de WS-Addressing que se utilizará si no se proporciona ninguna WS-Addressing en la solicitud o en el pipeline de servicio de proxy. Se utilizan las propiedades de WS-Addressing que se envían en la solicitud o se definen en el pipeline de servicio de proxy en lugar de la versión de WS-Addressing definida en esta opción.
Para las versiones no coincidentes de WS-Addressing entre los entornos, realice las transformaciones necesarias en el pipeline de servicio de proxy. Para obtener más información, consulte la sección sobre ejemplos de transformación en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1827.
Política de Distribución
Seleccione la instancia del gestor de trabajos de WebLogic Server que desea utilizar para la política de distribución de este punto final. Se utiliza el gestor de trabajos por defecto si no existe ningún otro.
Para obtener más información sobre los gestores de trabajo, consulte la sección sobre el uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Transferir Sujeto del Emisor de Llamada
Opcional. Active esta casilla de control para que Oracle Service Bus transfiera el sujeto autenticado del servicio de proxy al llamar al servicio de SOA. La opción Cuenta de Servicios de Llamada, alternativa a Transferir Sujeto del Emisor de Llamada, está desactivada si se selecciona esta opción.
Nota: Asegúrese de que se ha activado el dominio protegido entre el cliente y el servidor de destino si están en diferentes dominios. Para obtener más información, consulte la sección de información importante sobre soporte de seguridad entre dominios en Oracle Fusion Middleware Securing Oracle WebLogic Server (Protección de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=SECMG403.
Cuenta de Servicios de Llamada
Opcional. Alternativa a Transferir Sujeto del Emisor de Llamada, que permite especificar las credenciales de seguridad personalizadas seleccionando una cuenta de servicio para la llamada de RMI. Puede especificar cualquier tipo de cuenta de servicio (Transferencia, Estática, Asignación).
Haga clic en Examinar y seleccione una cuenta de servicio. Si no se especifica ninguna, se utiliza un sujeto anónimo.
