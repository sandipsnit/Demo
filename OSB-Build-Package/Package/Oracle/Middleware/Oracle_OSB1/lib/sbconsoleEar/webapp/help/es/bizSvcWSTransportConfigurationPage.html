Página Configuración de Transporte de WS

La página Crear/Editar un Servicio de Negocio: Configuración de Transporte de WS es una de una serie de páginas para la creación y edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Edición de Configuraciones de Servicio de Negocio.

Para obtener más información, consulte la sección sobre transporte de WS en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083.

Utilice esta página para configurar el transporte de un servicio de negocio mediante el protocolo de transporte de WS. En la Tabla 18-19 se describe el uso de esta página.

Tabla 18-19 Crear/Editar un Servicio de Negocio: Página Configuración de Transporte de WS 
Opción
Para crear o editar...
Timeout de Respuesta
Introduzca el número de segundos que se esperará una respuesta.
Si se deja este campo en blanco, indica que no existe timeout de respuesta. El servicio de negocio esperará lo que dure el timeout de secuencia configurado en la política de RM.
Si introduce un valor cero (0), no existe un timeout; por lo tanto, nunca sufrirá un timeout.
Cuenta de Servicios
Introduzca una cuenta de servicio o haga clic en Examinar para seleccionar una de la lista mostrada.
La cuenta de servicio especifica las credenciales que se utilizan cuando existe una política de autenticación básica de HTTP en este servicio.
Valores Avanzados
Poner Mensajes de Error en Cola
Active la casilla de control para permitir el envío de mensajes de error a la cola de errores configurada.
URI de Cola de Errores
Esta opción sólo está disponible si se ha activado la casilla de control Poner Mensajes de Error en Cola.
Introduzca el URI de la cola de JMS para almacenar los mensajes de error, en el siguiente formato:

jms://host:puerto/nombre_jndi_fábrica_conexiones/nombre_jndi_cola

Nota: aunque Oracle WebLogic Server permite el uso de barras inclinadas en los nombres de JNDI (por ejemplo, "miscolas/micola"), los nombres de JNDI con barras inclinadas interfieren en el formato de URI que necesita Oracle Service Bus, por lo que no se pueden utilizar esos nombres. Para solucionar este problema, defina un servidor ajeno de JMS y haga referencia a él en el URI. Para obtener más información, consulte la sección sobre configuración de servidores ajenos en Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help (Ayuda en Pantalla de la Consola de Administración de Oracle Fusion Middleware Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Cuenta de Servicios de Cola de Errores de JMS
Esta opción sólo está disponible si se ha activado la casilla de control Poner Mensajes de Error en Cola.
Introduzca una cuenta de servicio o haga clic en Examinar para seleccionar una de la lista mostrada.
La cuenta de servicio especifica las credenciales que se utilizan para las consultas de JNDI y para el envío de mensajes de error a la cola de errores.
Usar SSL para Cola de Errores
Esta opción sólo está disponible si se ha activado la casilla de control Poner Mensajes de Error en Cola.
Active la casilla de control para utilizar SSL para la conexión a la cola de errores.
