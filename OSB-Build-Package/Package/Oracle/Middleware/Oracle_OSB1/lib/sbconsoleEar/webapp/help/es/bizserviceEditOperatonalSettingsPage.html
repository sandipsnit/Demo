Configuración Operativa de Servicios de Negocio

Utilice la página Configuración Operativa para activar y desactivar un servicio de negocio y la supervisión de determinados servicios de negocio. Del mismo modo, puede especificar esta misma configuración operativa específica del servicio en la página Operaciones > Búsqueda Inteligente, con las siguientes excepciones: No puede definir un intervalo de agregación ni especificar un nivel de gravedad de alerta. Para obtener más información, consulte Búsqueda y Actualización de la Configuración Operativa.

Los efectos en tiempo de ejecución de la configuración de nivel de servicio dependen de su configuración global correspondiente. Debe activar tanto la configuración global como la de nivel de servicio para que un servicio esté completamente activado en tiempo de ejecución. También se debe activar el estado del servicio. Consulte Activación de la Configuración Global.

Puede cambiar y guardar la configuración de supervisión incluso si no se va a activar el servicio en tiempo de ejecución. Por ejemplo, puede cambiar y guardar el intervalo de agregación incluso si está desactivada la supervisión de servicio. De esta forma, puede editar la configuración y activarla más tarde.

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Seleccione Explorador de Recursos > Servicios de Negocio.
  3. Haga clic en el nombre del servicio de negocio.

    4. La página Ver Servicio de Negocio muestra la información de la Tabla 25-8.

    Tabla 25-8 Información de Servicio de Negocio

    Con Referencia de

    Descripción del servicio, si existe.

  4. Seleccione el separador Configuración Operativa.

    5. Debe estar en una sesión activa para realizar la configuración operativa del servicio de negocio mostrada en la Tabla 25-9.

    Tabla 25-9 Configuración Operativa del Servicio de Negocio 
    Para...
    Realice esta acción...
    Desactivar el estado
    Desactive la casilla de control Activado. Cuando activa la sesión, el servicio deja de procesar los mensajes.
    Al desactivar un servicio, el sistema ya no suprime todas las estadísticas de supervisión recopiladas previamente para el servicio.
    Activar el estado
    Active la casilla de control Activado. Cuando activa la sesión, el servicio reanuda el procesamiento de los mensajes.
    Activar los URI de punto final de servicio de negocio que se van a poner fuera de línea
    Definir un intervalo de reintentos para los URI fuera de línea
    Active la casilla de control Activar con Intervalo de Reintentos.
    Al seleccionar esta opción, el servicio de negocio elimina los URI de punto final sin respuesta (los pone fuera de línea), en tiempo de ejecución, de forma que sólo se utilicen los URI con respuesta para los reintentos y para procesar posteriores solicitudes.
    Si desea mantener los URI sin respuesta fuera de línea hasta que realice una acción correctiva y luego volver a activarlos, no proporcione un intervalo de reintento. Por ejemplo, un intervalo de reintento cero indica que el punto final permanece fuera de línea de forma indefinida. Para que vuelva a estar en línea, utilice la acción Marcar Punto Final en Línea en la página URI de Punto Final. Consulte Visualización de Métricas de URI de Punto Final de Servicios de Negocio.
    Si desea configurar el servicio de negocio para que ponga los URI sin respuesta fuera de línea y se vuelva a intentar acceder al mismo punto final de URI para el procesamiento de mensajes posteriores después de un determinado tiempo, especifique un valor de intervalo de reintentos. Utilice los campos de horas, minutos y segundos para especificar el tiempo que se mantienen los URI sin respuesta fuera del bucle de procesamiento de URI activos.
    El URI se mantiene fuera de línea durante el tiempo especificado y luego se vuelve a intentar. Si el punto final responde, el URI se vuelve a poner en línea, o permanece fuera de línea y se repite el proceso.
    Puede especificar un intervalo de reintentos sólo si ha seleccionado la opción Activar con Intervalo de Reintentos.
    Desactivar la supervisión
    Desactive la casilla de control Activado. Al activar la sesión, el sistema deja de recopilar estadísticas de supervisión para el servicio y suprime las estadísticas ya recopiladas.
    Activar la supervisión
    Active la casilla de control Activado. Al activar la sesión, el sistema empieza a recopilar estadísticas de supervisión para el servicio.
    Definir un intervalo de agregación para un servicio
    Seleccione el intervalo en horas o minutos y haga clic en Actualizar. Si la selección de horas excede de 1, la selección por defecto de minutos es siempre cero. Sin embargo, si la selección de horas es 0 o 1, puede configurar intervalos en minutos. Consulte Definición del Intervalo de Agregación para un Servicio.
    Puede cambiar y guardar el valor de intervalo de agregación independientemente de si ha activado la supervisión.
    Para activar la supervisión en tiempo de ejecución, asegúrese de que la casilla de control Activar Supervisión está activada.
    Desactivar alertas de SLA
    Desactive la casilla de control Activado. El sistema deja de evaluar las reglas de alertas configuradas para el servicio; por lo tanto, deja de recibir alertas de SLA asociadas al servicio.
    Activar alertas de SLA
    Active la casilla de control Activado. En la lista desplegable Activar Alertas en nivel, seleccione la restricción de alertas de SLA en el nivel especificado o por encima (del nivel más inclusivo al más restrictivo): Normal (valor por defecto), Advertencia, Menor, Grave, Crítico y Fatal.
    Las alertas de SLA de servicio dependen de los estados de supervisión global y de nivel de servicio; ambos deben estar activados para que las alertas de SLA se activen en tiempo de ejecución. Consulte Activación de Alertas de SLA Globalmente.
    Restringir el flujo de mensajes enviados al juego de puntos finales configurados en el servicio de negocio
    Active la casilla de control Activado junto a Estado de Regulación. Si se activa, los mensajes se procesan por prioridad. Puede asignar a los mensajes una prioridad mediante las opciones de direccionamiento; de lo contrario, los mensajes salen de la cola en el orden de llegada.
    Para obtener más información, consulte la sección sobre regulación en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG614.
    Restringir el número de mensajes distribuidos a un juego de puntos finales configurados en el servicio de negocio
    Si se ha activado Estado de Regulación, este campo es necesario.
    Especifique un valor para Simultaneidad Máxima; debe ser un valor entero positivo, no puede ser cero.
    Si se excede, los mensajes entrantes se ponen en una cola de espera (regulación), si existe. Si no, se desechan los mensajes.
    Si la cola está llena, se eliminan los mensajes con una prioridad menor y se emite un error de tiempo de ejecución. Si se configura un manejador de errores, se disparará. Los nuevos mensajes entrantes se pondrán en cola.
    Especificar si los mensajes enviados a los servicios de negocio que han excedido su límite de simultaneidad de mensajes se deben mantener en una cola de espera
    Definir el número máximo de mensajes que se pueden mantener en la cola de espera
    Si se ha activado Estado de Regulación, este campo es necesario.
    Especifique un valor para Cola de Regulación; debe ser un valor entero positivo o mantenga el valor por defecto, cero, lo que significa que no hay cola de regulación. Si el valor de la cola de regulación es cero (sin cola de regulación) y el número de mensajes excede el valor de Simultaneidad Máxima, se desechan los mensajes y se emite un error. Si se configura un manejador de errores, se disparará.
    Si desactiva la regulación en tiempo de ejecución, se procesarán todos los mensajes de la cola de regulación.
    Al suprimir o cambiar el nombre de un servicio de negocio, se desechan todos los mensajes de la cola de regulación.
    Especificar la duración máxima de los mensajes en la cola de espera
    Si se ha activado Estado de Regulación, este campo es opcional.
    Especifique un valor para Caducidad del Mensaje en milisegundos; debe ser un valor entero positivo. Un valor de cero significa que el mensaje no caducará nunca.
    Este valor se define a nivel de cola; todos los mensajes de la cola tendrán el mismo valor de Caducidad del Mensaje. Si se excede, los mensajes se eliminan de la cola sin procesarse.
    Activar el rastreo de mensajes para un servicio de negocio

    Active la casilla de control Activado junto a Rastreo de Mensajes. Después de activar el rastreo, el sistema registra los mensajes intercambiados entre el pipeline de Oracle Service Bus y el servicio de negocio (mensajes de respuesta y solicitud saliente).

    Nota: para ver el rastreo en el archivo log o en la salida estándar (consola del servidor), el registro de Oracle WebLogic Server debe estar definido en los siguientes niveles de gravedad:

    Gravedad Mínima que Registrar: Información
    Archivo Log: Información
    Salida Estándar: Información

    Para obtener información sobre la definición de niveles de gravedad de log, consulte la sección sobre el uso de los niveles de gravedad de en Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server (Configuración de Archivos Log y Filtrado de Mensajes de Log de Oracle Fusion Middleware para Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

    Tenga en cuenta que, cuando se aplica, los mensajes salientes registrados también pueden incluir el número de reintentos, código de error y mensaje de error.
    En la lista desplegable Nivel de Detalle, seleccione uno de los siguientes niveles de detalle:
    • Breve: Muestra la fecha, hora, tipo de servicio, nombre de servicio y URI.
    • Cabeceras: Muestra la información breve más la representación XML de los metadatos del mensaje.
    • Completo: Muestra la información de las cabeceras más la carga útil raw, incluidos los anexos, si los hay.
    Si selecciona Completo de la lista desplegable Nivel de Detalle, especifique el tamaño máximo (en KB) para la carga útil de los mensajes mediante el campo Límite de Rastreo de Carga Útil. Especifique también la codificación por defecto para el registro de la carga útil en el campo Codificación por Defecto. Puede resultar útil al registrar cargas útiles binarias o mensajes SOAP con anexos binarios.
    El valor de codificación por defecto puede ser Base64 o cualquier codificación soportada por Java como se define en: http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html.
    Si se deja el campo Codificación por Defecto vacío, Oracle Service Bus utiliza la codificación por defecto del host para la carga útil. La codificación por defecto depende de una combinación de los valores de JVM, sistema operativo (SO) subyacente y configuración regional de nivel del sistema operativo.
    Si no se puede utilizar el valor especificado en el campo Codificación por Defecto (por ejemplo, si no es una opción válida para la configuración), Oracle Service Bus utiliza la codificación Base64 para la carga útil.
    Desactivar el rastreo de mensajes para un servicio de negocio
    Desactive la casilla de control Activado junto a Rastreo de Mensajes.
    Activar y desactivar el almacenamiento en caché de resultados
    Al seleccionar Activado se activa el almacenamiento en caché de resultados para el servicio de negocio. También debe asegurarse de que el almacenamiento en caché de resultados está activada a nivel global. Consulte Activación de la Caché de Resultados Globalmente.
    Si desactiva el almacenamiento en caché de resultados de un servicio de negocio desactivando la opción Activado y haciendo clic en Actualizar, se vacían todos los resultados en la caché para ese servicio de negocio.
