Página Políticas de Servicio de Negocio

La página Ver Servicio de Negocio: Políticas es una de una serie de páginas para la edición de servicios de negocio, como se describe en Creación y Configuración de Servicios de Negocio y Página Ver Servicio de Negocio: Detalles de Configuración.

Utilice esta página para configurar los valores de política para un servicio de negocio con un tipo de servicio web XML basado en WSDL, servicios de mensajes, servicios SOAP o servicios XML. En la Tabla 18-23 se describe el uso de esta página. Para que las políticas de OWSM se utilicen con los servicios de proxy web XML basado en WSDL, servicios de mensajes o servicios XML, el protocolo se debe definir en HTTP.

Tenga en cuenta que para los servicios basados en WSDL, se muestran (en línea) las políticas de WLS 9.2 enlazadas al servicio en el WSDL efectivo. Las políticas abstractas se procesan previamente antes de ponerse en línea. Las políticas de OWSM se enlazan por referencia, no se ponen en línea en el WSDL efectivo.

Configuración de Política de Servicio
A continuación se muestran las opciones del campo Configuración de Política de Servicio:

Políticas de OWSM

  • De Almacén de Políticas de OWSM: las políticas están definidas en el almacén central de políticas de Oracle Web Services Manager gestionado por Oracle Enterprise Manager. Estas políticas soportan WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken Profile y otros estándares del sector.

    Con esta opción, puede agregar políticas de una lista predefinida. Haga clic en Agregar en Políticas de Nivel de Servicio para mostrar la lista. Utilice los filtros si es necesario para encontrar las políticas deseadas. Para los servicios de negocio, sólo se muestran las políticas de cliente.

    Si un WSDL utilizado para crear un servicio de negocio contiene políticas de WS-Security estándar embebidas, Oracle Service Bus devuelve un error y muestra un conflicto. Para resolverlo, sustituya las políticas de WSDL embebidas con políticas de Oracle Web Services Manager compatibles seleccionando De Almacén de Políticas de OWSM y haciendo clic en Agregar Compatible. Oracle Service Bus realiza las operaciones necesarias para intentar encontrar la política con mayor porcentaje de coincidencia del almacén de políticas de Oracle Web Services Manager basada en la política estándar embebida en el WSDL. Es posible que el algoritmo no devuelva ningún valor o que aparezcan varias políticas coincidentes.

    Si después de hacer clic en Agregar Compatible Oracle Service Bus no encuentra políticas compatibles, haga clic en Agregar y seleccione las políticas de Oracle Web Services Manager correspondientes. Si Oracle Service Bus devuelve más de una política compatible, suprima las políticas que no desea utilizar.

    Nota: esta opción se activa sólo si ha configurado su dominio para utilizar Oracle Web Services Manager. Para obtener más información, consulte "Securing Oracle Service Bus with Oracle Web Services Manager" (Seguridad de Oracle Service Bus con Oracle Web Services Manager" en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Políticas WLS 9

  • De WSDL: seleccione esta opción si la política de servicio está asociada al WSDL en el que está basado el servicio. Estas políticas soportan WS-Security 1.0, SAML 1.1 y otros estándares del sector.

    Con esta opción puede ver (sólo lectura) las políticas de solicitud y respuesta desde WSDL.

    Nota: si recibe un error en la configuración del servicio de negocio sobre las políticas de WS-Security no soportadas por Oracle Service Bus, utilice la opción De Almacén de Políticas de OWSM para sustituir las políticas de WSDL embebidas por políticas de OWSM compatibles.

  • De Política Predefinida o Recurso de Política de Servicios Web, seleccione esta opción para agregar políticas de nivel de servicio, políticas de nivel de operación (en cuyo caso, la política se aplica a los mensajes de solicitud y respuesta), políticas de solicitud y políticas de respuesta desde la consola de administración de Oracle Service Bus.

    Las políticas están predefinidas en Oracle WebLogic Server o definidas por el usuario en Oracle Service Bus con un recurso de política de servicios web. Estas políticas soportan WS-Security 1.0, SAML 1.1 y otros estándares del sector.

    Con esta opción, puede agregar políticas en los niveles de servicio, operación, solicitud y respuesta. Haga clic en Agregar en la sección correspondiente para seleccionar políticas de una lista.

Nota: Los modelos de enlace de política se excluyen mutuamente. Debe utilizar un solo tipo de política en un servicio. Si enlaza las políticas directamente al servicio, se ignoran todas las políticas basadas en WSDL.

Después de agregar las políticas, puede proporcionar sustituciones en la página Seguridad.

 

Después de Terminar

Haga clic en Actualizar para guardar esta configuración o en Restablecer para deshacer los cambios.
