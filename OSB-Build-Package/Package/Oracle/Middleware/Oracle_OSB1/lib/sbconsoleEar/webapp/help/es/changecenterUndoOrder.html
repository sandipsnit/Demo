Orden para Deshacer Tareas

Puede deshacer las tareas en cualquier orden (siempre que las acciones de deshacer individuales den como resultado datos válidos). La operación de deshacer define el valor de un recurso en el valor que tenía antes del cambio en ese recurso.

En el caso de que la tarea que se está deshaciendo ha creado un objeto, no existe un estado anterior al que pueda volver el objeto, es decir, no existía el objeto antes de realizar esta tarea. De hecho, la operación de deshacer elimina el nuevo objeto de la sesión. En este caso, se producen errores en los objetos que hacen referencia al que se está suprimiendo. Puede consultar estos errores en la página Ver Conflictos del Centro de Cambios.

Cuando no trabaje en una sesión, puede acceder a la página Ver Cambios de Configuración para ver las sesiones activadas anteriormente. Puede deshacer estas sesiones. El sistema no permite deshacer una sesión activada si se producirá un error en la configuración en tiempo de ejecución debido a la operación de deshacer. Por ejemplo, si intenta deshacer una activación de sesión que causa la eliminación de un objeto al que hace referencia otro objeto, no se permite esa acción de deshacer.

Es posible deshacer una acción de deshacer. En la columna Opciones de Proceso de Deshacer de [tarea], haga clic en el icono Deshacer. Oracle Service Bus soporta una cantidad ilimitada de deshacer. Esto significa que puede deshacer lo que se ha deshecho que deshizo una acción de deshacer anterior, y así sucesivamente.
