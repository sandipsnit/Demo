Códigos de Error

En esta sección se describen los errores de Oracle Service Bus que se pueden producir al ejecutar un servicio de proxy. Los códigos de error asociados a estos errores aparecen dentro del elemento de la variable de contexto fault. Puede acceder al valor con la siguiente sentencia XQuery:

$fault/ctx:errorCode/text()

Los errores están acompañados de detalles específicos del error dentro del elemento fault.

En la Tabla A-1 se muestran los códigos de error de Oracle Service Bus.

Códigos de error de seguridad (386000...386999)
BEA-386000
Error de seguridad general. Entre los motivos posibles se incluyen los siguientes:
  • La autenticación de elemento personalizado o de nombre de usuario/contraseña personalizado está configurada pero la autenticación ya se ha realizado mediante WS-Security.
 
BEA-386100
Fallo de autenticación. Entre los motivos posibles se incluyen los siguientes:
  • XPath se ha configurado para actuar como cuerpo o cabecera pero no existe esa parte del mensaje (el nivel de enlace devuelve nulo).
  • El motor de XPath devuelve una excepción al evaluar la expresión XPath.
  • El resultado del elemento XPath de nombre de usuario o contraseña es un nodo que no es del tipo TEXT o ATTR.
  • XPath devuelve más de un nodo.
  • Se ha recuperado un nombre de usuario/contraseña pero ha fallado la autenticación.
  • Se ha recuperado un elemento personalizado pero ha fallado la afirmación de identidad (el servicio de afirmación de identidad de CSS devuelve LoginException u otra excepción).
 
BEA-386101
Falta el elemento de autenticación. Entre los motivos posibles se incluyen los siguientes:
  • El elemento XPath de nombre de usuario o de contraseña devuelve un resultado vacío o una cadena vacía.
  • El XPath del elemento personalizado devuelve un resultado vacío, valor de atributo nulo o valor de texto nulo.
 
BEA-386102
Autorización a nivel de mensaje denegada. Entre los motivos posibles se incluyen los siguientes:
  • La política de control de acceso a nivel de mensaje deniega el acceso al servicio de proxy.
 
BEA-386103
El algoritmo de selección de operación de servicio de proxy no puede determinar el nombre de la operación de la solicitud o devuelve una operación no válida (que no está en WSDL o es nula). Entre los motivos posibles se incluyen los siguientes:
  • Se ha producido un error al calcular la operación.
  • El algoritmo de selección de operación devuelve nulo.
  • El algoritmo de selección de operación devuelve una operación que no es una de las operaciones declaradas por WSDL.

Esquemas de Detalles del Error

A continuación se muestran las definiciones de esquema para los fragmentos XML que acompañan a los códigos de error de la Tabla A-1. Estos fragmentos XML aparecen dentro de la variable de contexto de fallo en el elemento <ctx:details>. No todos estos esquemas proceden de un solo espacio de nombres. Además, mientras que normalmente un determinado código de error implica un fragmento XML concreto, algunos fragmentos XML se utilizan en varios códigos de error. 

<xs:schema targetNamespace="http://www.oracle.com/wli/sb/stages/transform/config"

	   xmlns:xs="http://www.w3.org/2001/XMLSchema"
           xmlns:ref="http://www.oracle.com/wli/sb/reference"
           xmlns:sdk="http://www.oracle.com/wli/sb/stages/config"
	   elementFormDefault="qualified"
	   attributeFormDefault="unqualified">
