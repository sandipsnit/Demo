Introducción a Oracle Service Bus

Oracle Service Bus es un bus de servicio de empresa basado en políticas y configuraciones. Proporciona integración orientada a servicios, gestión de servicios y distribución de mensajes tradicional en entornos de TI heterogéneos de alta escalabilidad y fiabilidad. Combina la distribución inteligente de mensajes con el direccionamiento y la transformación de mensajes, junto con supervisión y administración de servicios en un producto de software unificado. La consola de administración de Oracle Service Bus permite controlar las configuraciones de servicios y políticas así como supervisar las tareas operativas y del sistema. Oracle Service Bus se basa en utilidades de Oracle WebLogic Server en tiempo de ejecución. Para obtener más información, consulte Oracle Fusion Middleware Concepts and Architecture for Oracle Service Bus (Conceptos y Arquitectura de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBCA.

Restricciones de Nomenclatura de Recursos

Al dar nombre a un directorio o recurso en una configuración de Oracle Service Bus, se permiten los siguientes caracteres:

    • Todos los caracteres de identificador Java, incluidas las palabras clave de Java, como se describen en las secciones de identificadores y palabras clave de las especificaciones del lenguaje Java en http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#3.8

    • Espacios en blanco, puntos y guiones en los nombres (que no vayan al principio o al final)

No se permiten los caracteres como / \ * : " < > ? |.
