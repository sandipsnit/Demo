Adición de Enlaces de JCA

La forma más fácil de crear enlaces de JCA en Oracle Service Bus es mediante la importación en bloque de recursos de JCA que haya creado (archivo .jca, WSDL asociado y archivo de asignación de TopLink o EclipseLink). En la importación en bloque, Oracle Service Bus genera automáticamente un enlace de JCA de los archivos .jca, recursos de WSDL de los WSDL de JCA asociados y recursos de documento XML de los archivos de asignación, y mantiene las dependencias entre los archivos.

Para obtener más información sobre la importación, consulte Carga de Recursos de un Archivo Zip, Carga de Recursos de una URL e Importación de Recursos.

Después de importar los recursos de JCA, puede generar servicios de proxy o de negocio con ellos, como se describe en Generación de un Servicio de Proxy a partir de un Recurso de Enlace de JCA y Generación de un Servicio de Negocio a partir de un Recurso de Enlace de JCA.

Si no importa en bloque los recursos de JCA, puede crear manualmente los enlaces de JCA, como se describe en los siguientes pasos:

  1. En Oracle JDeveloper cree un WSDL de servicio JCA, un archivo JCA (.jca) correspondiente y, si es necesario, otros recursos como archivos de asignación TopLink o EclipseLink, y archivos XSLT.

    2. Para obtener más información, consulte Oracle Fusion Middleware User's Guide for Technology Adapters (Guía del Usuario de Oracle Fusion Middleware para Technology Adapters) en http://www.oracle.com/pls/as111170/lookup?id=TKADP.

  2. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión en la consola de administración de OSB o haga clic en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  3. En la consola de administración de OSB, cree o importe los recursos de JCA que ha creado en Oracle JDeveloper, como los recursos WSDL, XML y XSLT.
  4. Cree el enlace de JCA.
    1. Seleccione Explorador de Proyectos y elija la ubicación de proyecto o carpeta para el enlace de JCA. Aparece la página Vista de Proyectos/Carpetas.
    2. En el campo Crear Recurso, seleccione Enlace de JCA para mostrar la página Crear Nuevo Recurso de Enlace de JCA.
    3. En el campo Archivo JCA, haga clic en Examinar para seleccionar el archivo JCA.

      4. El campo Nombre del Recurso se rellena automáticamente con el nombre del archivo (menos la extensión del archivo).

      En lugar de seleccionar un archivo, puede copiar y pegar el contenido de un archivo JCA en el área de texto del campo Archivo JCA.

    4. En el campo Nombre del Recurso, introduzca o cambie el nombre del enlace de JCA. Los nombres de los enlaces de JCA deben ser únicos.

      Siga las Restricciones de Nomenclatura de Recursos para obtener ayuda con la nomenclatura.
    5. En el campo Descripción del Recurso, introduzca una descripción para este enlace de JCA.
    6. Haga clic en Guardar. El enlace de JCA se guarda en la sesión actual.

      7. Aparece una advertencia de validación si el enlace de JCA no hace referencia aún a recursos necesarios, como un WSDL y un archivo de asignación.

  5. Haga clic en el nombre del nuevo enlace de JCA para mostrar la página Ver un Enlace de JCA.
  6. Haga clic en Editar Referencias.
  7. En la página Editar la Referencia de un Recurso de Enlace de JCA, haga clic en Examinar para seleccionar los recursos de referencia de enlace de JCA, como el WSDL, un archivo de asignación de TopLink/EclipseLink y un archivo XSLT.
  8. Haga clic en Guardar.
  9. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.

Después de agregar los recursos del enlace de JCA y configurar las dependencias de los recursos (asociando un WSDL) o, si ha importado recursos de JCA que mantienen sus dependencias, puede generar servicios de proxy y de negocio a partir de los enlaces de JCA. Para obtener más información, consulte Generación de un Servicio de Proxy a partir de un Recurso de Enlace de JCA y Generación de un Servicio de Negocio a partir de un Recurso de Enlace de JCA.
