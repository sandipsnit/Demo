Definición de Condiciones de Reglas de Alertas

Utilice esta página para definir condiciones para la regla de alerta. Debe especificar al menos una condición. Si especifica varias, debe utilizar los operadores And/Or para combinarlas.

  1. En el campo Seleccione Intervalo de Agregación para la Condición, seleccione el número de horas y minutos para definir el intervalo de agregación para esta condición. El intervalo de agregación determina la frecuencia con la que se prueba la condición. La condición se prueba cada vez que el subsistema de supervisión agrega suficientes datos de ejemplo para constituir un intervalo de agregación.

    2. Por ejemplo, si selecciona un intervalo de agregación de 1 hora, la condición se prueba cada vez que hay disponible datos suficientes para una hora. La primera vez que se prueba la condición es al final de la primera hora. Después, se prueba cada 10 minutos porque el intervalo de ejemplo de un intervalo de agregación de 1 hora está definido en 10 minutos.

  2. Se empieza por definir una expresión simple. Se puede combinar dos o varias expresiones simples para formar una expresión compleja. Para definir una expresión simple compleja, haga lo siguiente:
    1. En el primer campo desplegable, seleccione Recuento, Mínimo, Máximo, Medio o Estado.
    2. En el siguiente campo desplegable, seleccione un operando.

      3. Dependiendo de si ha seleccionado Recuento, Mínimo, Máximo, Medio o Estado en el primer campo desplegable, varía la lista de operandos. Por ejemplo, si selecciona Mínimo, Máximo o Medio está disponible el operando Tiempo de Respuesta. Utilice este operando para definir el tiempo de respuesta en milisegundos (mseg). Los operandos disponibles también dependen de la configuración del propio servicio. El número de operandos varía dependiendo de si el servicio tiene pipelines, nodos de direccionamiento, operaciones, etc.

      Para obtener más información sobre la generación de alertas según el estado de URI de punto final, consulte la sección sobre gestión de URI de punto final para servicios de negocio en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG595.

      Si selecciona Recuento, este campo muestra los operandos de la Tabla 25-21.

    3. En el siguiente campo, seleccione el operador de comparación seleccionado: =, !=, > o <.
    4. En el siguiente campo, introduzca el valor para la comparación.
    5. Haga clic en Agregar.
  3. Para definir una expresión compleja:
    1. Repita los pasos a - e del paso 2 para definir una expresión simple.
    2. Vuelva a repetir los pasos a - e para definir otra expresión simple.
    3. Active la casilla de control de cada expresión.
    4. Haga clic en And u Or para combinar las expresiones en una expresión compleja.

      5. La opción Dividir está disponible después de combinar varias expresiones. Esta opción se utiliza para dividir expresiones complejas en sus expresiones simples individuales.

Después de Terminar

Haga clic en Anterior para volver a la página Configuración General. O bien, haga clic en Último para acceder a la página Nueva Regla de Alerta: [nombre de servicio]. Continúe en Revisión de la Configuración de Reglas de Alertas.
