Localización de Alertas

  1. Seleccione Operaciones > Panel de Control.
  2. Seleccione el separador Alertas de SLA o Alertas de Pipeline para mostrar las alertas de SLA o de pipeline.
  3. Haga clic en el enlace Historial de Alertas Ampliado. Las páginas Historial de Alertas de SLA Ampliado e Historial de Alertas de Pipeline Ampliado muestran información detallada sobre el historial de alertas.
  4. Para limitar el número de elementos de la lista o para localizar alertas concretas:
    • Haga clic en el icono Abrir para mostrar filtros de búsqueda adicionales. Si especifica varios criterios de búsqueda, sólo se devolverán los resultados que coincidan con todos los criterios.
    • Filtre por rango de fechas. Realice una de las siguientes acciones:
      • Seleccione Todo para ignorar los criterios de filtro de rango de fecha e intervalo de tiempo y mostrar todas las alertas.
      • En los campos Desde y Hasta, introduzca la fecha y hora con el formato MM/DD/AA HH:MM AM|PM y, a continuación, haga clic en Buscar. Se muestran las alertas producidas en el rango de fecha y hora especificado.
      • En el campo Durante Último(s), introduzca el número de días, especifique las horas y los minutos y, a continuación, haga clic en Buscar. Se muestran las alertas producidas en el rango de tiempo especificado.
    • Filtre por gravedad de la alerta. En la lista desplegable Gravedad de Alerta, seleccione si desea limitar las alertas al nivel de gravedad especificado y, a continuación, haga clic en Buscar.

      • Active la casilla de control o superior para restringir la búsqueda al nivel especificado o superior (del nivel más inclusivo al más restrictivo): Normal, Advertencia, Menor, Mayor, Crítico y Fatal.

    • Filtre por servicio principal. Haga clic en Examinar para mostrar la página Seleccionar Servicio. Seleccione el servicio principal de la lista y haga clic en Enviar. Haga clic en Limpiar para eliminar los criterios de filtro de servicio principal o haga clic en Buscar.
    • Sólo para las alertas de SLA, filtre por la opción Nombre de Alerta (Cualquier Cadena). Introduzca cualquier cadena, incluidos los comodines y, a continuación, haga clic en Buscar. Para obtener más información, consulte Lenguaje de Consulta WebLogic Diagnostics Framework.
    • Sólo para las alertas de pipeline, filtre por la opción Resumen de Alertas (Cualquier Cadena). Introduzca cualquier cadena, incluidos los comodines y, a continuación, haga clic en Buscar. Para obtener más información, consulte Lenguaje de Consulta WebLogic Diagnostics Framework. Para buscar las acciones de alertas de pipeline diseñadas sin texto de resumen de alerta, introduzca Alerta de Oracle Service Bus.
    • Haga clic en Ver Todo para eliminar los filtros de búsqueda y mostrar todas las alertas.

Utilice también las páginas Historial de Alertas de SLA Ampliado e Historial de Alertas de Pipeline Ampliado para realizar las siguientes acciones:

  • Para mostrar los datos de alerta en un gráfico de tarta, seleccione Vista de Gráfico de Tarta.
  • Para mostrar los mismos datos en un gráfico de barras, seleccione Vista de Gráfico de Barras.
  • Para mostrar los datos en formato de tabla, seleccione Vista de Tabla.
  • Para mostrar una versión actualizada de los datos, haga clic en Refrescar.
  • Para suprimir alertas individuales, active las casillas de control correspondientes y haga clic en Suprimir al final de la página.
  • Para suprimir todas las alertas o las que se han emitido en un rango de fecha y hora especificado, haga clic en el enlace Depurar Historial de Alertas al final de la página. Consulte Depuración de Alertas.

Lenguaje de Consulta WebLogic Diagnostics Framework

Las alertas se almacenan mediante WebLogic Diagnostics Framework, que proporciona su propio lenguaje de consulta, incluidos los comodines. Para filtrar las alertas del historial de alertas ampliado, utilice la sintaxis descrita en la sección sobre lenguaje de consulta WLDF en Oracle Fusion Middleware Configuring and Using the Diagnostics Framework for Oracle WebLogic Server (Configuración y Uso de Diagnostics Framework para Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=WLDFC316.
