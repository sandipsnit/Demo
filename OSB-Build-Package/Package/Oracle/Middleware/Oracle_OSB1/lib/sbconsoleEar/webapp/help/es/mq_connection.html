Conexiones de MQ

Nota: Consulte Oracle Fusion Middleware Supported System Configurations (Configuraciones de Sistema de Oracle Fusion Middleware Soportadas) en http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html para averiguar qué versiones de WebSphere MQ soporta Oracle Service Bus. La biblioteca de cliente de WebSphere MQ no se conecta con Installer de Oracle Service Bus. Obtenga la biblioteca de cliente de WebSphere MQ de su instalación de WebSphere MQ.

Las conexiones de MQ son recursos que se pueden compartir y reutilizar en varios servicios de negocio y de proxy de MQ. Estos servicios deben estar conectados a un gestor de colas de MQ antes de acceder a una cola de MQ. Los recursos de conexión de MQ proporcionan los parámetros de conexión necesarios para conectarse a un gestor de colas de MQ.

Cada conexión de MQ tiene un pool de conexiones. Cada servicio de negocio o de proxy que utiliza un determinado recurso de conexión de MQ para obtener una conexión a un gestor de colas concreto usa el mismo pool de conexiones que se ha creado para ese recurso. De esta forma, varios servicios de negocio y de proxy que utilizan el mismo gestor de colas comparten un pool de conexiones.

Para obtener más información sobre recursos de conexión de MQ de Oracle Service Bus y transportes de MQ nativos, consulte la sección sobre transporte de MQ en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Para obtener más información sobre aspectos fundamentales de WebSphere MQ, consulte http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
