Políticas de Servicios Web Personalizadas

Nota: En futuras versiones de Oracle Service Bus, las políticas de Oracle Web Services Manager sustituirán a las políticas de WLS 9.2. Aunque esta versión de Oracle Service Bus sigue soportando las políticas de WLS 9.2, debería empezar a utilizar las políticas de Oracle Web Services Manager en la creación de nuevos servicios para prepararse para cuando deje de estar soportada la política de WLS 9.2. Para obtener más información, consulte la sección sobre protección de Oracle Service Bus con Oracle Web Services Manager en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681.

El marco de política de servicios web (Política de Servicios Web) es un marco basado en XML extensible que extiende la configuración de un servicio web con afirmaciones de seguridad específicas del dominio y especifica los requisitos de seguridad, previsiones y capacidades del servicio web. En Oracle Service Bus, uno de los usos principales de la política de servicio web es configurar la seguridad a nivel de mensaje en servicios de proxy y de negocio.

Uso de las Políticas de Servicios Web de Oracle Service Bus

Oracle Service Bus incluye tres archivos simples de política de seguridad web que puede utilizar para pedir a los clientes que autoricen, cifren digitalmente o firmen digitalmente mensajes de SOAP: Auth.xml, Encrypt.xml y Sign.xml. Oracle recomienda que a menos que tenga necesidades de seguridad específicas, utilice estos archivos preempaquetados con la mayor frecuencia posible.

Para obtener más información sobre estas políticas, consulte la sección sobre el uso de política de servicio web en servicios de proxy y de negocio de Oracle Service Bus en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.

Uso de Políticas de Servicios Web Personalizadas

Si las sentencias de política de servicio web de Oracle Service Bus no satisfacen sus necesidades de seguridad, puede escribir sus propias políticas de servicio web (políticas de servicio web personalizadas), importarlas a Oracle Service Bus y hacer referencia a ellas desde WSDL. (Las sentencias de política de servicio web de Oracle Service Bus son de sólo lectura.)

Para obtener más información sobre la creación y referencia a políticas de servicio web, consulte la sección sobre el uso de la política de servicio web en servicios de proxy y de negocio de Oracle Service Bus en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.
