Visualización de Recursos en un Explorador Web

Para ver los recursos de Oracle Service Bus en un explorador web estándar, utilice las siguientes direcciones URL

    • Ruta de acceso para los recursos de WSDL:
      http://host:port/sbresource?WSDL/project_path/wsdlname
      o
      http://host:port/proxy_service_endpoint_URI?WSDL (para servicios de proxy HTTP basados en WSDL)
    • Ruta de acceso para los recursos de servicio de proxy:
      http://host:port/sbresource?PROXY/project_path/proxyname
      http://host:port/sbresource?ORAPROXY/project_path/proxyname (para los servicios de proxy con políticas de Oracle Web Services Manager asociadas)
    • Ruta de acceso para los recursos de servicio de negocio:
      http://host:puerto/sbresource?BIZ/ruta_acceso_proyecto/bizname
      http://host:puerto/sbresource?ORABIZ/ruta_acceso_proyecto/bizname (para servicios de negocio con políticas de Oracle Web Services Manager asociadas)
    • Ruta de acceso para los recursos de WSS:
      http://host:port/sbresource?POLICY/project_path/policyname
    • Ruta de acceso para los recursos de MFL:
      http://host:port/sbresource?MFL/project_path/mflname
    • Ruta para los recursos de SCHEMA:
      http://host:port/sbresource?SCHEMA/project_path/schemaname

    Nota: Si utiliza caracteres especiales en sus nombres de recurso, las URL utilizadas para exponer los recursos de Oracle Service Bus deben estar codificadas en UTF-8 para poder utilizar caracteres especiales de escape.

    También puede recuperar los WSDL que contengan políticas de Oracle Web Services Manager de forma que las políticas cumplan con las políticas de seguridad y de WS soportadas. Para obtener más información, consulte la sección sobre cómo anunciar WSDL para soportar los estándares de WS en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1877.
