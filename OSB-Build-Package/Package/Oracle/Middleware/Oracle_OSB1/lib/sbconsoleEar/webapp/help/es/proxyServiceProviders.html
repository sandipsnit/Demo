Proveedores de Claves de Servicio

Un proveedor de claves de servicio contiene credenciales de infraestructura de clave pública (PKI) que los servidores de proxy utilizan para descifrar los mensajes de SOAP entrantes y para firmas digitales y autenticación saliente. Una credencial PKI es una clave privada emparejada con un certificado que se puede utilizar para firmas digitales y cifrado (para seguridad del servicio web) y para autenticación SSL saliente. El certificado contiene la clave pública que corresponde a la clave privada.

Nota: Para utilizar un proveedor de claves de servicio, debe configurar un proveedor de asignación de credenciales PKI. Consulte la sección sobre configuración de seguridad a nivel de transporte en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498.

Un único proveedor de claves de servicio puede contener todas las credenciales PKI siguientes:

  • Un par de claves para cifrado digital

    • Los servicios de proxy utilizan este par de claves para descifrar mensajes de SOAP entrantes que se han cifrado para cumplir una sentencia de política de servicios web. Si desea que el proveedor de claves de servicio soporte el cifrado digital, el almacén de claves asociado a la asignación de credenciales PKI debe contener al menos un certificado X.509 que soporte el cifrado.

  • Un par de claves para firmas digitales

    • Los servicios de proxy utilizan este par de claves cuando su punto final es un servicio web y éste necesita que los clientes firmen una o más partes de un elemento Envelope de SOAP.

  • Un par de claves para la autenticación de cliente SSL (SSL bidireccional)

    • Los servicios de proxy utilizan este par de claves para la autenticación cuando actúan como un cliente durante una conexión TLS/SSL (Secure Sockets Layer) saliente; es decir, al direccionar un mensaje a un servicio de negocio HTTPS o un servicio de proxy que necesita la autenticación de certificado de cliente.

Puede utilizar el mismo proveedor de claves de servicio para varios servicios de proxy.
