Adición de Proveedores de Claves de Servicio

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Seleccione Explorador de Proyectos y elija el proyecto o carpeta donde desea agregar el proveedor de claves de servicio. Aparece la página Vista de Proyectos/Carpetas.
  3. Desde la lista desplegable Crear Recurso, seleccione Proveedor de Claves de Servicio para mostrar la página Crear un Nuevo Proveedor de Claves de Servicio.
  4. En el campo Nombre de Proveedor de Claves de Servicio, introduzca un nombre único para este proveedor de claves de servicio.

    Siga las Restricciones de Nomenclatura de Recursos para obtener ayuda con la nomenclatura.
  5. En el campo Descripción, introduzca una descripción para el proveedor de claves de servicio.
  6. Realice uno de los pasos siguientes, que se muestran en la Tabla 17-2.

    7. Tabla 17-2 Opciones de Autenticación
    Para Agregar un Par de Claves para...
    Realice estos pasos...
    Cifrado digital
    1. Junto a Clave de Cifrado, haga clic en Examinar.

      2. La ventana Seleccionar Alias para Clave de Cifrado muestra los alias de clave del almacén de claves que utiliza la asignación de credenciales PKI del dominio.

    2. En la ventana Seleccionar Alias para Clave de Cifrado, introduzca la contraseña que utiliza para el acceso seguro al almacén de claves. (Defina la contraseña al crear el almacén de claves.)
    3. Seleccione un alias de clave que se asigna a un certificado X.509 y que soporta el cifrado.
    4. Haga clic en Enviar.
    Al asociar este proveedor de claves de servicio a un servicio de proxy, Oracle Service Bus embebe el certificado X.509 en el WSDL del servicio de proxy. A continuación, el servicio de proxy utiliza este certificado para cifrar los mensajes que envía a su punto final. El servicio de proxy utiliza la clave privada de la credencial PKI para descifrar los mensajes que devuelve el punto final.
    Firmas digitales
    1. Junto a Clave de Firma Digital, haga clic en Examinar.

      2. La ventana Seleccionar Alias para Clave de Firma Digital muestra los alias de clave del almacén de claves que utiliza la asignación de credenciales PKI del dominio.

    2. En la ventana Seleccionar Alias para Clave de Firma Digital, introduzca la contraseña que utiliza para el acceso seguro al almacén de claves. (Defina la contraseña al crear el almacén de claves.)
    3. Seleccione un alias de clave.
    4. Haga clic en Enviar.
    Autenticación de cliente SSL (SSL bidireccional)
    1. Junto a Clave de Autenticación de Cliente SSL, haga clic en Examinar.

      2. La ventana Seleccionar Alias para Clave de Autenticación de Cliente SSL muestra los alias de clave del almacén de claves que utiliza la asignación de credenciales PKI del dominio.

    2. En la ventana Seleccionar Alias para Clave de Autenticación de Cliente SSL, introduzca la contraseña que utiliza para el acceso seguro al almacén de claves. (Defina la contraseña al crear el almacén de claves.)
    3. Seleccione un alias de clave.
    4. Haga clic en Enviar.

  7. Haga clic en Guardar. Se guardará el proveedor de claves de servicio en la sesión actual.
  8. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.
