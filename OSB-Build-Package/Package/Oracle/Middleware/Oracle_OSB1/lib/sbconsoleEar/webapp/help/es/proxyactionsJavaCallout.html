Adición de Acciones de Llamada de Java

Utilice esta acción para llamar a un método Java, o servicio de negocio de EJB, desde el flujo de mensajes.

Antes de Empezar

En estas instrucciones se supone que ya está editando un flujo de mensajes en la página Editar Flujo de Mensajes, como se explica en Creación, Edición y Visualización de Flujos de Mensajes.

Para agregar una acción de llamada de Java

  1. Navegue a donde desea agregar la acción, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
  2. Haga clic en el icono adecuado y, a continuación, seleccione Agregar una Acción > Procesamiento de Mensajes > Llamada de Java.
  3. Haga clic en <Método>. Aparece la página Seleccionar un JAR. Seleccione un recurso JAR de la lista. Aparece la página Seleccionar una Clase y Método.
  4. En la lista de clases Java que aparece, haga clic en el + junto a la clase adecuada para mostrar una lista de métodos. Seleccione un método y haga clic en Enviar. La acción de llamada de Java aparece en la página Editar Etapa, de la siguiente forma:
    • <Método> se sustituye por el nombre del método Java seleccionado en los pasos 2 y 3. Este nombre es un enlace a la página Seleccionar una Clase y Método. Puede hacer clic en este enlace para cambiar la selección del método Java.

      • El método debe ser estático.

    • Parámetros: Se proporciona un enlace <Expresión> a la página Editor de Expresiones XQuery para cada argumento que necesita el método Java. Una etiqueta para cada enlace indica el tipo de dato del argumento, que será uno de los siguientes:
      • Java.lang.String
      • Tipos primitivos y sus tipos de clase correspondiente (por ejemplo, int frente a java.lang.Integer)
      • java.lang.BigDecimal y java.lang.BigInteger (estos tipos se utilizan en cálculos financieros en los que no se toleran errores de redondeo o desbordamientos)
      • Sólo org.apache.xbeans.XmlObject y no beans xml.
      • byte[]
      • java.lang.String[] (sólo entrada)
      • XmlObject [ ] (sólo entrada)
      • javax.activation.DataSource
    • Devolver Parámetro como Referencia: Esta opción convierte el valor de retorno de una invocación de llamada de Java en un elemento de referencia de <java-content ref="jcid"> sin tener en cuenta su tipo real, donde jcid es la clave del objeto en el repositorio de objetos de pipeline. En el campo de valor del resultado, introduzca el nombre de la variable que va a contener la referencia del contenido de Java. Esta opción permite trabajar con un objeto de referencia en el flujo del mensaje además del XML de pipeline para proporcionar la transferencia, realizar el enriquecimiento del mensaje con las llamadas de Java y acciones en línea, o realizar la transformación del mensaje entre los transportes que son de Java y los que no. Para obtener más información, consulte la sección de envío y recepción de objetos Java en los mensajes en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
    • Resultado: Campo Resultado en el que se introduce la variable a la que se va a asignar el resultado. La etiqueta del campo indica el tipo de dato del resultado.
      • Nota: Si el resultado es una matriz de byte (la única matriz posible devuelta), se devuelve el elemento XML de contenido binario.
    • Asociar Cuenta de Servicio: Un enlace Cuenta de Servicios permite especificar una cuenta de servicio opcional si hay un contexto de seguridad para este método Java. Para obtener más información sobre contextos de seguridad y cuentas de servicio, consulte Cuentas de Servicio.

      • En el caso de cuentas de servicio fijas o asignadas, el identificador de usuario/contraseña de la cuenta de servicio se autentica en el sistema local y el contexto de seguridad se propaga a la llamada de Java. En el caso de passthru, el contexto de seguridad se propaga a la llamada de Java. Este contexto es el contexto a nivel de mensaje si se ha definido (con WS-Security). De lo contrario, es el contexto a nivel de transporte.

  5. En Parámetros, haga clic en <Expresión>. Aparece la página Editor de Expresiones XQuery. Utilice este editor para proporcionar los argumentos necesarios para el método Java. Consulte Creación y Edición de Expresiones XQuery y XPath en Línea.

    6. Si el tipo del valor de entrada que introduce no coincide con el tipo de argumento de entrada declarado, Oracle Service Bus intenta convertir automáticamente los valores de entrada al tipo declarado del argumento de entrada. Por ejemplo, un valor de cadena de "123" se convertiría en entero 123 si el tipo declarado del argumento de entrada es un primitivo java int.

  6. En el campo Resultado, asigne una variable para el resultado devuelto por el método Java.
  7. Si hay un contexto de seguridad para el método Java, active la casilla de control y haga clic en <Cuenta de Servicios>. Aparece la página Seleccionar Cuenta de Servicio. Seleccione la cuenta de servicio necesaria de la lista y haga clic en Enviar.

Después de Terminar

Cuando termine de configurar esta acción, siga configurando otras acciones o guarde la configuración, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
