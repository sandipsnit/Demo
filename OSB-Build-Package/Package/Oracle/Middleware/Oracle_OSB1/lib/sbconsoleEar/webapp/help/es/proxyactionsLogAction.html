Adición de Acciones de Log

Utilice esta acción para construir el mensaje que se va a registrar y para definir un juego de atributos con los que se registra el mensaje.

Antes de Empezar

En estas instrucciones se supone que ya está editando un flujo de mensajes en la página Editar Flujo de Mensajes, como se explica en Creación, Edición y Visualización de Flujos de Mensajes.

Nota: para ver los datos de log en el archivo log o en la salida estándar (consola del servidor), el registro de Oracle WebLogic Server debe estar definido en los siguientes niveles de gravedad:

Gravedad Mínima que Registrar: Información
Archivo Log: Información
Salida Estándar: Información

Para obtener información sobre la definición de niveles de gravedad de log, consulte la sección sobre el uso de los niveles de gravedad de en Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server (Configuración de Archivos Log y Filtrado de Mensajes de Log de Oracle Fusion Middleware para Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

 

Para agregar una acción de log

  1. Asegúrese de que el registro está activado a nivel global. Para obtener más información, consulte Activación Global de Registro.
  2. Navegue a donde desea agregar la acción, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
  3. Haga clic en el icono adecuado y, a continuación, seleccione Agregar una Acción > Informe > Log.
  4. Haga clic en <Expresión>. Aparecerá la página Editor de Expresiones XQuery. Especifique el contexto de mensaje que se registrará mediante expresiones XQuery en las variables de contexto. Consulte Creación y Edición de Expresiones XQuery y XPath en Línea.
  5. En el campo Anotación, introduzca notas para esta acción de registro. Estas notas se registrará junto con el resultado de la expresión definida anteriormente.
  6. En la lista desplegable nivel de gravedad, seleccione una de las siguientes opciones:

    7. Tabla 21-5 Niveles de Gravedad de Acción de Log 
    Nivel de Gravedad
    Uso Común
    Información
    Se utiliza para informar de operaciones normales; mensaje informativo de bajo nivel.
    Advertencia
    Se ha producido una operación o configuración sospechosa pero puede que no afecte al funcionamiento normal.
    Error
    Se ha producido un error de usuario. El sistema o la aplicación puede manejar el error sin interrupciones y con una degradación limitada del servicio.
    Depurar
    Mientras la aplicación está en desarrollo, le puede resultar útil crear y utilizar mensajes que proporcionen descripciones detalladas de la actividad de nivel bajo en la aplicación.
    Asegúrese de que el nivel de gravedad de log de la configuración operativa del servicio de proxy es el mismo que el nivel de gravedad de acción de log. Para obtener información sobre la configuración operativa de los servicios de proxy, consulte Configuración Operativa de Servicios de Proxy.

Después de Terminar

Cuando termine de configurar esta acción, siga configurando otras acciones o guarde la configuración, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
