Adición de Acciones de Opciones de Direccionamiento

Utilice la acción Opciones de Direccionamiento para modificar algunas o todas las propiedades siguientes para una solicitud saliente en $outbound: Parámetros URI, Calidad del Servicio, Modo, Reintentar. Aunque estas propiedades se pueden modificar con las acciones Asignar, Insertar, Sustituir o Suprimir en $outbound, el uso de Opciones de Direccionamiento proporciona una forma más sencilla de realizar esta tarea, sin necesidad de conocimientos de XPath, XQuery o la estructura de la variable de contexto $outbound.

La acción Opciones de Direccionamiento sólo se pueden utilizar donde la variable de contexto $outbound sea válida. Se puede agregar a las siguientes acciones:

  • Publicar
  • Publicación Dinámica
  • Publicar Tabla
  • Llamada de Servicio
  • Direccionamiento
  • Direccionamiento Dinámico
  • Tabla de Direccionamiento

Para obtener más información sobre el direccionamiento, consulte la sección sobre el modelado de flujos de mensajes en Oracle Service Bus en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus), en http://www.oracle.com/pls/as111170/lookup?id=OSBAG181.

 

Para configurar una acción de opciones de direccionamiento:

  1. Navegue a donde desea agregar la acción, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
  2. Haga clic en el icono adecuado y, a continuación, seleccione Agregar una Acción > Comunicación >Opciones de Direccionamiento.
  3. Realice algunos o todos los pasos siguientes:
    • Para definir el URI para el mensaje saliente: Seleccione URI y haga clic en el Editor de Expresiones XQuery. Introduzca una expresión que devuelva un URI. Esto sustituye el URI del servicio llamado.
    • Para definir el elemento Calidad del Servicio: Seleccione Calidad del Servicio y la opción Calidad del Servicio de la lista desplegable. Esto sustituye el valor por defecto que se calcula automáticamente.
    • Para definir el modo: Seleccione Modo y una solicitud o solicitud-respuesta de la lista desplegable.
      • Nota: Normalmente esto ya está definido automáticamente, según la interfaz del servicio llamado. Sin embargo, en algunos casos como servicios Cualquier SOAP o Cualquier XML, esto no es así.
    • Para definir el intervalo de reintentos: Seleccione Intervalo de Reintentos y especifique el número de segundos entre reintentos. Esto sustituye el valor por defecto configurado con el servicio llamado.
    • Para definir el recuento de reintentos: Seleccione Recuento de Reintentos y especifique el número de reintentos que debe intentar el sistema antes de interrumpir la acción. Esto sustituye el valor por defecto configurado con el servicio llamado.
    • Para definir la prioridad de mensaje: Seleccione Prioridad y haga clic en el Editor de Expresiones XQuery. Introduzca una expresión que devuelva un entero positivo.

Después de Terminar

Cuando termine de configurar esta acción, siga configurando otras acciones o guarde la configuración, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
