Adición de Acciones de Llamada de Servicio

Utilice una acción de llamada de servicio para configurar una llamada síncrona (de bloqueo) en un servicio de negocio o proxy registrado por Oracle Service Bus.

Para obtener más información sobre las acciones de llamada de servicio, consulte la sección sobre la construcción de mensajes de llamada de servicio en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus), en http://www.oracle.com/pls/as111170/lookup?id=OSBAG208.

  1. Navegue a donde desea agregar la acción, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
  2. Haga clic en el icono adecuado y, a continuación, seleccione Agregar una Acción > Comunicación > Llamada de Servicio.
  3. Haga clic en <Servicio>. Aparecerá el explorador de servicios.
  4. Seleccione un servicio de la lista de servicios de negocio o proxy registrados y, a continuación, haga clic en Enviar.
  5. Si el servicio seleccionado en el paso 3 anterior está basado en WSDL y tiene operaciones a las que se puede llamar en el servicio, dichas operaciones se muestran en la lista desplegable llamando a <operación>. Seleccione la operación a la que se debe llamar en el servicio.

    Nota: la selección de una operación, lo cual necesita Oracle Service Bus por muchos motivos, no garantiza que sólo se llame a la operación seleccionada. Por ejemplo, si selecciona OperaciónA pero un mensaje también contiene una llamada a la OperaciónB, también se llama a OperaciónB.
  6. Especifique cómo desea configurar los mensajes de solicitud y respuesta seleccionando una de las siguientes opciones:
    • Seleccione Configurar Cuerpo SOAP para configurar el cuerpo SOAP. Al seleccionar esta opción, puede utilizar $body directamente.
      • Nota: Esta opción soporta RPC de SOAP codificado, que no se soporta al configurar un documento o parámetros de carga útil.
    • Seleccione Configurar Parámetros de Carga Útil o Configurar Documento de Carga Útil para configurar la carga útil.
  7. Las posteriores opciones de configuración dependen del tipo de servicio seleccionado en el paso 3 y en el tipo de opciones de configuración seleccionadas para dicho servicio en el paso 5. En la Tabla 21-3 se muestran las opciones disponibles para cada tipo de servicio.

    8. En la Tabla 21-4 siguiente se proporcionan instrucciones para cada una de las opciones mostradas en la Tabla 21-3 anterior.

    Tabla 21-4 Opciones de Configuración de Llamada de Servicio 
    Para estas Opciones...
    Siga estos Pasos...
    Cuerpo de la Solicitud SOAP y
    Cuerpo de la Respuesta de SOAP
    Para configurar estas opciones:
    • En el campo Cuerpo de la Solicitud SOAP, introduzca el nombre de una variable para contener el XML del elemento Cuerpo SOAP para la solicitud de llamada.
    • En el campo Cuerpo de la Respuesta de SOAP, introduzca el nombre de una variable a la que estará enlazado el XML del elemento Cuerpo SOAP en la respuesta.
    Cabecera de la Solicitud SOAP y
    Cabecera de la Respuesta de SOAP
    Para configurar estas opciones:
    • En el campo Cabecera de la Solicitud SOAP, introduzca el nombre de una variable para contener el XML del elemento Cabecera SOAP para la solicitud de llamada.

      • Debe encapsular el documento de entrada para la cabecera de la solicitud SOAP con <soap-env:Header>...</soap-env:Header>.

    • En el campo Cabecera de la Respuesta de SOAP, introduzca el nombre de una variable a la que estará enlazado el XML de las cabeceras de SOAP en la respuesta, si la hay.
    Parámetros de Solicitud y
    Parámetros de Respuesta
    Para configurar opciones:
    • En los campos Parámetros de Solicitud, introduzca nombres para las variables que se evaluarán en tiempo de ejecución para proporcionar valores para los parámetros de solicitud.

      • Debe proporcionar sólo los documentos de carga útil de núcleo en la variable de entrada: Oracle Service Bus crea el paquete SOAP por el usuario. En otras palabras, no encapsule el documento de entrada con <soap-env:Body>...</soap-env:Body>.

      Por ejemplo, al crear una variable de entrada de cuerpo utilizada para este parámetro de solicitud, definirá el contenido de dicha variable mediante la sentencia de XPath body/* (para eliminar el encapsulador soap-env:Body), no $body (cuyo resultado es mantener el encapsulador soap-env:Body).

    • En los campos Parámetros de Respuesta, introduzca los nombres de las variables a las que se asignarán las respuestas en tiempo de ejecución.
    Documento de Solicitud y
    Documento de Respuesta
    Para configurar estas opciones:
    • En el campo Variable del Documento de Solicitud, introduzca el nombre de una variable a la que asignar un documento de solicitud.

      • Para servicios de tipo Documento SOAP, la variable se evalúa en tiempo de ejecución para formar el cuerpo del mensaje de SOAP enviado al servicio. Para servicios Cualquier XML, la variable se evalúa en tiempo de ejecución para formar el cuerpo del mensaje XML enviado al servicio.

      Para los servicios de tipo documento SOAP y para servicios Cualquier XML, proporcione sólo los documentos de carga útil de núcleo en la variable de entrada: Oracle Service Bus crea el paquete SOAP por el usuario. En otras palabras, no encapsule el documento de entrada con <soap-env:Body>...</soap-env:Body>.

      Por ejemplo, al crear una variable de entrada de cuerpo utilizada para este parámetro de solicitud, definirá el contenido de dicha variable mediante la sentencia de XPath body/* (para eliminar el encapsulador soap-env:Body), no $body (cuyo resultado es mantener el encapsulador soap-env:Body).

      Para servicios de mensajes, la variable se evalúa para formar el cuerpo del mensaje, según el tipo de dato que espera el servicio. Las siguientes restricciones se aplican a variables utilizadas con servicios de mensajes:

      • Para servicios que esperan datos binarios, las variables deben tener un elemento ctx:binary-content.
      • Para servicios que esperan datos MFL, la variable debe tener el equivalente de XML.
      • Para servicios que esperan datos de texto, la variable es una cadena.
    • En el campo Variable del Documento de Respuesta, introduzca el nombre de la variable a la que se asignarán un documento de respuesta en tiempo de ejecución.

  8. Opcionalmente, agregue una o más cabeceras de transporte. Para obtener más información, consulte Adición de Acciones de Cabecera de Transporte.
    9. Nota: Además de las cabeceras de transporte especificadas, el nivel de enlace de Oracle Service Bus agrega cabeceras. Para obtener más información, consulte la sección sobre configuración de cabeceras de transporte en flujos de mensajes en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

Después de Terminar

Cuando termine de configurar esta acción, siga configurando otras acciones o guarde la configuración, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
