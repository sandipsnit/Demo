Adición de Acciones de Cabeceras de Transporte

Utilice una acción de cabecera de transporte para definir los valores de cabecera en mensajes.

  1. Navegue a donde desea agregar la acción, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.
  2. Haga clic en el icono adecuado y, a continuación, seleccione Agregar una Acción > Comunicación > Cabeceras de Transporte.
  3. En la lista Definir Cabeceras de Transporte para, seleccione una de las opciones siguientes, para especificar al tiempo de ejecución cuál de las ubicaciones de contexto de mensaje se deben modificar:
    • Solicitud Saliente: Seleccione esta opción para definir valores de cabecera para solicitudes salientes (los mensajes enviados por un servicio de proxy en acciones de direccionamiento, publicación o llamada de servicio). Este elemento de cabecera está en el contexto de mensaje de la siguiente forma:

      • $outbound/ctx:transport/ctx:request/tp:headers

    • Respuesta Entrante: Seleccione esta opción para definir valores de cabecera para respuestas entrantes (los mensajes de respuesta que envía un servicio de proxy a los clientes). Este elemento de cabecera está en el contexto de mensaje de la siguiente forma:

      • $inbound/ctx:transport/ctx:response/tp:headers

  4. Opcionalmente, seleccione Transferir Todas las Cabeceras Mediante Pipeline para transferir todas las cabeceras desde el mensaje entrante al mensaje saliente o viceversa. Cada cabecera del juego de origen de cabeceras se copiarán en el juego de cabeceras de destino, sobrescribiendo cualquier valor existente en el juego de cabeceras de destino.

    5. Para obtener más información sobre el uso de esta opción junto con la opción de traspaso específica de la cabecera, consulte la sección sobre configuración de cabeceras de transporte en flujos de mensajes) en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

  5. Realice los siguientes pasos para cada cabecera que desee agregar:
    1. En la tabla Cabeceras de Transporte, haga clic en Agregar Cabecera para visualizar campos para configurar la cabecera.
    2. Especifique una cabecera realizando lo siguiente:
      • En la lista desplegable de la columna Nombre, seleccione un nombre de cabecera. La lista contiene todos los nombres de cabecera predefinidos para el transporte de destino (por ejemplo, Tipo de Contenido para transportes HTTP, JMSCorrelationID para transportes de JMS, etc.).
      • Introduzca un nombre de cabecera en el campo Otra. Si dicho nombre de cabecera no es una de las cabeceras predefinidas para el transporte de este servicio, se convierte en una cabecera de usuario, como define la especificación de transporte.
    3. Seleccione una de las opciones de la columna Acción para especificar cómo definir el valor de las cabeceras:

      4. Definir Cabecera en Expresión

      Al seleccionar esta opción, puede utilizar una expresión XQuery o XSLT para definir el valor de la cabecera. La expresión puede ser simple (por ejemplo, "texto/xml") o una expresión XQuery o XSLT compleja.

      Puesto que el nivel de transporte de Oracle Service Bus define la representación XML de todas las cabeceras como valores de cadena, el resultado de cualquier expresión se convierte en una cadena antes de que se defina el valor de cabecera. Las expresiones que no devuelven nada dan como resultado el valor de cabecera definido en la cadena vacía. No puede suprimir una cabecera mediante una expresión.

      Atención: No toda la configuración de cabecera que puede especificar en esta acción se utiliza en tiempo de ejecución. Para obtener más información sobre cuál de las cabeceras de un transporte determinado puede definir y cuál de estos juegos se utilizan en tiempo de ejecución, consulte la sección sobre configuración de cabeceras de transporte en flujos de mensajes en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

      Suprimir Cabecera

      Especifica que la cabecera se elimina de los metadatos de solicitud o respuesta.

      Copiar Cabecera de Solicitud Entrante (si define cabeceras de transporte para la solicitud saliente)
            o bien
      Copiar Cabecera de Respuesta Saliente (si define cabeceras de transporte para la respuesta entrante)

      Especifica que esta cabecera se copia directamente de la cabecera correspondiente del mismo nombre del mensaje entrante en el mensaje saliente y viceversa. Por ejemplo, si desea definir la cabecera SOAPAction para una solicitud saliente, al seleccionar Copiar Cabecera de Solicitud Entrante, el tiempo de ejecución copia el valor de la cabecera de solicitud SOAPAction de $inbound. En caso de cabeceras de respuesta entrante, el origen de la cabecera que se va a copiar es las cabeceras de respuesta de $outbound.

      Si se selecciona la opción Copiar Cabecera... para una cabecera que no existe en el origen, se ignora esta opción y no se realiza ninguna acción en el destino para esta cabecera.

      Para obtener más información sobre el uso de esta opción junto con la opción Transferir Todas las Cabeceras Mediante Pipeline global, consulte la sección sobre configuración de cabeceras de transporte en flujos de mensajes) en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

  6. Para agregar cabeceras adicionales a la tabla, haga clic en el icono Cabecera y, a continuación, haga clic en Agregar Cabecera.

    7. La tabla se amplía para incluir una fila adicional, que incluye un nuevo juego de opciones que puede utilizar para configurar otra cabecera de transporte. Puede agregar tantas cabeceras como sea necesario a esta tabla. No tiene que ordenar las cabeceras de la tabla, porque el tiempo de ejecución declara espacios de nombres y coloca elementos de cabecera en su orden correcto al generar el XML correspondiente.

Después de Terminar

Cuando termine de configurar esta acción, siga configurando otras acciones o guarde la configuración, como se describe en Adición y Edición de Acciones en Flujos de Mensajes.

 

Definición de Cookies en Cabeceras de Transporte HTTP Salientes

En los flujos de mensajes HTTP, puede definir cookies en la cabecera de transporte de las siguientes maneras:

  • Con una expresión XML compleja (valor por defecto de Oracle Service Bus)
  • Con una expresión de cadena (necesita la opción de inicio del servidor)

Definición de Cookies como Expresiones XML Complejas

Para definir una cookie con una expresión XML compleja, que es el formato por defecto de Oracle Service Bus, configure el valor de la cabecera de cookie HTTP en la solicitud saliente con la siguiente sintaxis de expresión:

<cookie-values xmlns="http://www.bea.com/wli/sb/transports/http">
    <value>{fn:concat("nombre_cookie", "=", "valor_cookie")}</value>
</cookie-values>

Definición de Cookies con Expresiones de Cadena

Para definir la cabecera de cookie con una cadena en la solicitud saliente, debe agregar la siguiente opción al comando startWebLogic del dominio:

-Dcom.bea.osb.http.cookieAsNoComplexElement=true

Después de reiniciar el servidor con esta opción, puede definir una cabecera de cookie HTTP con una expresión de cadena. Por ejemplo:
$cookie_name = "valor_cookie"
