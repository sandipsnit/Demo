Página Configuración de Transporte de Correo Electrónico

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte de Correo Electrónico es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte de correo electrónico. En la Tabla 19-4 se describe el uso de esta página.

Tabla 19-4 Página Crear/Editar un Servicio de Proxy: Configuración de Transporte de Correo Electrónico 
Opción
Para crear o editar...
SSL Necesario
Seleccione esta opción para comunicarse mediante la capa de conexión segura. Los números de puerto SSL por defecto son:
  • POP3(S): 995
  • IMAP: 993
Nota: el certificado del servidor de correo electrónico debe estar presente en el almacén confianza de Oracle Service Bus para que SSL funcione.
Cuenta de Servicios
Introduzca un nombre de cuenta de servicio o haga clic en Examinar para seleccionar las cuentas de servicio desde un explorador.
Servidor Gestionado
Este campo está disponible sólo en un dominio de cluster.
Seleccione el servidor gestionado que actuará como servidor de sondeo. Todos los servidores gestionados pueden procesar el mensaje, pero sólo uno puede sondear el mensaje.
Intervalo de Sondeo
Introduzca un intervalo de sondeo, en segundos.
Protocolo de Correo Electrónico
Seleccione POP3 o IMAP como tipo de servidor para la cuenta de correo electrónico.
Límite de Lectura
Especifique el número máximo de mensajes que se van a leer por barrido de sondeo. Introduzca 0 para no especificar ningún límite.
Transferir por Referencia
Active esta casilla de control para almacenar en área temporal el archivo en el directorio de archivo y transferirlo como referencia en las cabeceras.
Por defecto, al crear un nuevo servicio, se selecciona la opción Transferir por Referencia y debe especificar la ubicación del directorio de archivo.
Transferir Anexos por Referencia
Active esta casilla de control para almacenar en área temporal los anexos en el directorio de archivo y transferirlos como referencia en las cabeceras.
Por defecto, cuando se selecciona la opción Transferir por Referencia, la opción Transferir Anexos por Referencia es implícitamente verdadera y debe especificar la ubicación del directorio de archivo.
Acción Posterior a Lectura
Seleccione lo que sucede con un mensaje una vez leído:
  • Archivar: El mensaje se archiva.
  • Suprimir: El mensaje se suprime.
  • Mover: El mensaje se mueve. Mover sólo está disponible con el protocolo IMAP.
Anexos
Seleccione cómo se manejan los anexos:
  • Archivar: Los anexos se guardan en el directorio de archivo.
  • Ignorar: Los anexos se ignoran.
Carpeta de Movimiento de IMAP
Introduzca la carpeta a la que se mueve el mensaje si el campo Acción Posterior a Lectura está definido en Mover.
Directorio de Descarga
Introduzca una ubicación temporal para descargar correos electrónicos.
Directorio de Archivo
Especifique la ruta de acceso a la ubicación de archivo si el campo Acción Posterior a Lectura está definido en Archivar.
Este campo es necesario si se selecciona la opción Transferir por Referencia o Transferir Anexos por Referencia.
Directorio de Error
Introduzca la ruta de acceso del directorio del sistema de archivos para escribir el mensaje y cualquier anexo si hay un problema.
Codificación de Solicitud
Acepte el valor por defecto ISO-8859-1 como codificación del juego de caracteres para las solicitudes de transportes de correo electrónico o introduzca otro diferente.

Después de Terminar

Haga clic en Siguiente para seguir configurando este servicio en la siguiente página; o haga clic en Último para revisar y guardar esta configuración en la página Crear/Editar un Servicio de Proxy: Resumen.
