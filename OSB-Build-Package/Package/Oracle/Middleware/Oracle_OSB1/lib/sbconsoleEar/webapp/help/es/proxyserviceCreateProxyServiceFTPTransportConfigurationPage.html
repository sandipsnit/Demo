Página Configuración de Transporte de FTP

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte FTP es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte FTP. En la Tabla 19-6 se describe el uso de esta página.

Tabla 19-6 Página Crear/Editar un Servicio de Proxy: Configuración de Transporte FTP 
Opción
Para crear o editar...
Autenticación de Usuario
Seleccione anónimo si el usuario del servidor FTP es anónimo o usuario externo si el usuario del servidor FTP es una cuenta configurada externamente.
Identidad (Identificador de Correo Electrónico)
Este campo está disponible si la opción Autenticación de Usuario se define en anónimo.
Introduzca el identificador de correo para el usuario anónimo.
Cuenta de Servicios
Este campo está disponible si la opción Autenticación de Usuario se define en usuario externo.
Introduzca la cuenta de servicio del usuario. Es un campo necesario cuando la opción Autenticación de Usuario está definida en el usuario externo.
Transferir por Referencia
Active esta casilla de control para almacenar en área temporal el archivo en el directorio de archivo y transferirlo como referencia en las cabeceras.
Flujo Remoto
Active esta casilla de control para transmitir los archivos de FTP directamente desde el servidor remoto en el momento del procesamiento. Al seleccionar esta opción, el directorio de archivo es el directorio remoto en la máquina del servidor FTP remoto. Por lo tanto, debe especificar el directorio de archivo como relativo al directorio de usuario de FTP.
Máscara de Archivo
Introduzca la expresión normal para los archivos que se van a seleccionar. El valor por defecto es *.*.
Servidor Gestionado
Este campo está disponible sólo en un dominio de cluster.
Seleccione el servidor gestionado que actuará como servidor de sondeo. Todos los servidores gestionados pueden procesar el mensaje, pero sólo uno puede sondear el mensaje.
Intervalo de Sondeo
Introduzca un intervalo de sondeo, en segundos. El valor por defecto es 60.
Límite de Lectura
Especifique el número máximo de mensajes que se van a leer por barrido de sondeo. Introduzca 0 para no especificar ningún límite. El valor por defecto es 10.
Acción Posterior a Lectura
Seleccione lo que sucede con un mensaje una vez leído.
  • Archivar: El mensaje se archiva.
  • Suprimir: El mensaje se suprime.
Modo de Transferencia
Seleccione ASCII o binario como modo de transferencia.
Directorio de Archivo
Especifique la ruta de acceso a la ubicación de archivo si la opción Acción Posterior a Lectura está definida en Archivar. Este campo es necesario si se ha seleccionado la opción Transferir por Referencia.
Nota: Los directorios de archivo, descarga y error son rutas de acceso absolutas y se crean automáticamente. Si especifica una ruta de acceso relativa, los archivos se crean con respecto al proceso Java que inicia WebLogic Server.
Directorio de Descarga
Introduzca el directorio en la máquina local en la que se descargan los archivos durante la transferencia de archivos.
Nota: Los directorios de archivo, descarga y error son rutas de acceso absolutas y se crean automáticamente. Si especifica una ruta de acceso relativa, los archivos se crean con respecto al proceso Java que inicia WebLogic Server.
Directorio de Error
Introduzca la ubicación en la que se publican los mensajes si hay un problema.
Nota: Los directorios de archivo, descarga y error son rutas de acceso absolutas y se crean automáticamente. Si especifica una ruta de acceso relativa, los archivos se crean con respecto al proceso Java que inicia WebLogic Server.
Codificación de Solicitud
Acepte el valor por defecto UTF-8 como codificación del juego de caracteres para las solicitudes de transportes FTP.
Explorar Subdirectorios
Active esta casilla de control para explorar de forma recurrente todos los directorios.
Ordenar por Llegada
Active esta casilla de control para entregar eventos en el orden de llegada.
Timeout
Introduzca el intervalo de timeout del socket, en segundos, antes de que se cierre la conexión. Si introduce 0, no hay timeout.
Recuento de Reintentos
Especifique el número de reintentos para fallos de conexión FTP.

Después de Terminar
