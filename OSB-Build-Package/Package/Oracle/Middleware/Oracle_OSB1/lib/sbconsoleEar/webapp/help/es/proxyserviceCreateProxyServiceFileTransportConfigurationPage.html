Página Configuración de Transporte de Archivo

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte de Archivo es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte de archivo. En la Tabla 19-5 se describe el uso de esta página.

Tabla 19-5 Página Crear/Editar un Servicio de Proxy: Configuración de Transporte de Archivo 
Opción
Para crear o editar...
Máscara de Archivo
Especifique los archivos que debe sondear el servicio de proxy. Si el URI es un directorio y se especifica *.*, el servicio sondeará todos los archivos del directorio. Sólo se permiten los caracteres comodín * y ? en la máscara de archivo. No se soportan expresiones regulares.
Servidor Gestionado
Este campo está disponible sólo en un dominio de cluster.
Seleccione el servidor gestionado que actuará como servidor de sondeo. Todos los servidores gestionados pueden procesar el mensaje, pero sólo uno puede sondear el mensaje.
Intervalo de Sondeo
Introduzca un intervalo de sondeo, en segundos. El valor por defecto es 60.
Límite de Lectura
Especifique el número máximo de mensajes que se van a leer por barrido de sondeo. Introduzca 0 para no especificar ningún límite. El valor por defecto es 10.
Ordenar por Llegada
Active esta casilla de control para especificar qué eventos se entregan en el orden de llegada.
Tenga en cuenta que cuando esta opción está seleccionada para un servicio de proxy que se ejecuta en un entorno de cluster, los mensajes siempre se envían al mismo servidor. En otras palabras, el equilibrio de carga entre servidores se ignora cuando se selecciona esta opción.
Explorar Subdirectorios
Active esta casilla de control para explorar de forma recurrente todos los directorios.
Transferir por Referencia
Active esta casilla de control para almacenar en área temporal el archivo en el directorio de archivo y transferirlo como referencia en las cabeceras.
Acción Posterior a Lectura
Seleccione lo que sucede con un mensaje una vez leído:
  • Archivar: El mensaje se archiva.
  • Suprimir: El mensaje se suprime.
Directorio de Almacenamiento en Zona Intermedia
Introduzca un directorio intermedio para almacenar temporalmente los archivos mientras se procesan.
No coloque el directorio de almacenamiento en zona intermedia dentro del directorio de sondeo (el directorio identificado en la URL del servicio de proxy de transporte de archivo; por ejemplo, file:///c:/dir1/dir2).
Directorio de Archivo
Especifique la ruta de acceso a la ubicación de archivo si la opción Acción Posterior a Lectura está definida en Archivar. El campo Directorio de Archivo también es necesario si ha seleccionado el campo Transferir por Referencia.
No coloque el directorio de archivo dentro del directorio de sondeo.
Directorio de Error
Introduzca la ubicación en la que se publican los mensajes y anexos si hay un problema.
No coloque el directorio de error dentro del directorio de sondeo.
Codificación de Solicitud
Acepte el valor por defecto UTF-8 como codificación del juego de caracteres para las solicitudes de transportes de archivo o introduzca otro diferente.

Después de Terminar
