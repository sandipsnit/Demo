Página Configuración General

La página Crear/Editar un Servicio de Proxy: Configuración General es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Utilice esta página para modificar la configuración general de un servicio de proxy.

Al crear un servicio de proxy, ésta es la primera página que aparece de una serie de páginas para la configuración del servicio. Las páginas mostradas después de ésta dependen de las opciones seleccionadas.

En la Tabla 19-1 se describe el uso de esta página.

Tabla 19-1 Página Crear/Editar un Servicio de Proxy: Configuración General 
Opción
Para crear o editar...
Nombre del Servicio
Introduzca un nombre único para el servicio de proxy.

Siga las Restricciones de Nomenclatura de Recursos para obtener ayuda con la nomenclatura.
Descripción
Introduzca una descripción.
Tipo de Servicio
Seleccione el tipo de servicio de proxy que se va a crear:
  • Servicio Web WSDL: Seleccione esta opción para crear un servicio de proxy basado en WSDL. A continuación, introduzca el nombre de WSDL, cualificado por su ruta de acceso (por ejemplo, miProyecto/miCarpeta/miWSDL). Como alternativa, haga clic en Examinar para seleccionar un WSDL de la página Seleccione una definición de WSDL.

    • (puerto o enlace): Introduzca el nombre de un puerto (definido en WSDL) para describir la dirección de transporte real, o bien introduzca el nombre de un enlace (definido en WSDL) para asignar una dirección de transporte. Si utiliza Examinar para seleccionar un WSDL, en la página Seleccione una definición de WSDL aparecen los puertos y enlaces definidos en el WSDL.

Nota: Si va a utilizar Tipo de Cuerpo SOAP para operaciones, asegúrese de que el WSDL no tiene dos operaciones con el mismo mensaje de entrada. La operación Tipo de Cuerpo SOAP no se puede identificar de forma única inspeccionando el mensaje de entrada.
  • Servicio de Transporte según Configuración: Seleccione esta opción para crear un servicio de proxy de JEJB.
  • Servicio de Mensajes: Seleccione esta opción para crear un servicio que pueda recibir mensajes de un tipo de dato y responder con mensajes de otro tipo de dato distinto. Estos intercambios pueden ser solicitud/respuesta o unidereccional.

    • (HTTP GET sólo está soportado en tipos de servicio Cualquier XML y servicio de mensajes.)

  • Cualquier Servicio SOAP: Seleccione esta opción para crear un servicio SOAP que no tenga una interfaz concreta definida explícitamente.

    • Seleccione SOAP 1.1 o SOAP 1.2 de la lista desplegable para especificar la versión de SOAP que se va a utilizar.

  • Cualquier Servicio XML: Seleccione esta opción para crear un servicio XML que no tenga una interfaz concreta definida explícitamente.

    • (HTTP GET sólo está soportado en tipos de servicio Cualquier XML y servicio de mensajes.)

  • Servicio de Negocio: Seleccione esta opción para crear un servicio de proxy con un nodo de direccionamiento que direccione al servicio de negocio seleccionado. No puede crear un servicio de proxy a partir de un servicio de negocio de transporte según configuración. Si crea un servicio de proxy a partir de un servicio de negocio de transporte de DSP, Oracle Service Bus cambia el tipo de transporte del servicio de proxy a HTTP, porque el transporte de DSP no se puede utilizar para servicios de proxy. Puede cambiar el tipo de transporte del servicio de proxy a cualquier otro transporte disponible.

    • Introduzca la ruta de acceso (proyecto/carpeta) y el nombre del servicio de negocio; o bien haga clic en Examinar para seleccionarlo en la página Seleccionar Servicio de Negocio.

Tipo de Servicio (continuación)
  • Servicio de Proxy: Seleccione esta opción para clonar un servicio de proxy existente. Puesto que Oracle Service Bus no acepta el mismo URI para varios servicios, debe cambiar el URI del servicio clonado.
Nota: Cuando se crea un servicio a partir de otro servicio, las reglas de alertas se mantienen de la siguiente forma:
    • Cuando se crea un servicio de proxy a partir de un servicio de negocio o un servicio de negocio a partir de un servicio de proxy, se eliminan las reglas de alertas, si las hay.
    • Cuando se crea un servicio de proxy a partir de otro servicio de proxy o un servicio de negocio a partir de otro servicio de negocio, se mantienen las reglas de alertas, si las hay.
Proveedor de Claves de Servicio
Si es necesario, introduzca la ruta de acceso (proyecto/carpeta) y el nombre de un proveedor de claves de servicios, o bien haga clic en Examinar para seleccionar uno de la página Seleccionar Proveedor de Claves de Servicio.
Sólo se necesita un proveedor de claves de servicio en determinados casos:
  • TLS/SSL bidireccional saliente, donde el servicio de proxy direcciona mensajes a servicios HTTPS que necesitan autenticación mediante certificado de cliente.
  • En algunos casos de seguridad del servicio web, por ejemplo, si el servicio de proxy necesita que se cifren los mensajes.
Para agregar un servicio de proxy con seguridad de servicio web activada, debe crear el servicio de proxy de WSDL (puerto o enlace) con WS-Policy Attachments.
Para obtener más información, consulte Proveedores de Claves de Servicio. Para obtener más información sobre cómo crear un proveedor de claves de servicio, consulte Adición de Proveedores de Claves de Servicio.
Publicar en Registro
Esta opción sólo aparece cuando existe un registro UDDI por defecto.
Seleccione esta opción para publicar el servicio de proxy en el registro por defecto automáticamente. Para obtener más información, consulte Uso de Publicación Automática.

Después de Terminar

Haga clic en Siguiente para seguir configurando este servicio en la siguiente página; o haga clic en Último para revisar y guardar esta configuración en la página Crear/Editar un Servicio de Proxy: Resumen.
