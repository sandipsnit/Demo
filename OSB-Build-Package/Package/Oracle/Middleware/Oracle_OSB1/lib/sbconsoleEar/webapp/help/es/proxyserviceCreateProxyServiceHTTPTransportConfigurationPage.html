Página Configuración de Transporte de HTTP

El transporte HTTP ahora soporta puntos finales HTTP y HTTPS.

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte HTTP es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte HTTP. En la Tabla 19-7 se describe el uso de esta página.

Autenticación
Seleccione una de las siguientes opciones:
  • Ninguna: Especifica que la autenticación no es necesaria.
  • Básica: Especifica que es necesaria la autenticación básica para acceder a este servicio.

    • La autenticación básica indica a WebLogic Server que autentique el cliente con un nombre de usuario y contraseña según los proveedores de autenticación configurados en el dominio de seguridad, como un servicio de directorio Lightweight Directory Access Protocol (LDAP) y Windows Active Directory. El cliente debe enviar su nombre de usuario y contraseña en la cabecera de la solicitud HTTP.

    No se recomienda en absoluto el uso de la autenticación básica en HTTP porque la contraseña se envía en texto sin cifrado. Sin embargo, es seguro el envío de contraseñas por HTTPS porque dispone de un canal cifrado.

    Advertencia: Por defecto, todos los usuarios (autorizados y anónimos) pueden acceder a un servicio de proxy. Para limitar su acceso, cree una política de autorización a nivel de transporte. Consulte Edición de Políticas de Acceso a Nivel de Transporte.

  • Certificado de Cliente: Especifica la comunicación cifrada y la autenticación de cliente segura (SSL bidireccional). Para obtener más información, consulte la sección sobre configuración de seguridad de nivel de transporte en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
  • Autenticación Personalizada: Especifica que se incluye un elemento de autenticación en una cabecera HTTP. La identidad del cliente se establece mediante este elemento proporcionado por el cliente. Debe configurar un proveedor de afirmación de identidad que asigne el elemento a un usuario de Oracle Service Bus.

    • El elemento de autenticación personalizada puede ser de cualquier tipo de elemento activo soportado por un proveedor de afirmación de identidad de WebLogic Server configurado.

Política de Distribución
Seleccione una política de distribución para este punto final. Déjelo en blanco para utilizar la política de distribución por defecto.
La política de distribución hace referencia a la instancia del gestor de trabajos de WLS que desea utilizar para el punto final de servicio. Para obtener más información sobre los gestores de trabajos, consulte la sección sobre el uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificación de Solicitud
  • Para transportes entrantes HTTP:

    • Si el parámetro de codificación de juego de caracteres de la cabecera Tipo de Contenido no se especifica en la solicitud del cliente, introduzca un parámetro de codificación de juego de caracteres. Si no introduce un valor, el campo se establece por defecto en ISO-8859-1.

  • Para transportes salientes HTTP:

    • Si no ha configurado una codificación de solicitud, Oracle Service Bus en tiempo de ejecución decide la codificación más adecuada mientras realiza una solicitud al servicio de negocio. En un caso de no traspaso, la codificación de caracteres por defecto es UTF-8 en tiempo de ejecución. Sin embargo, si es un caso de traspaso, el tiempo de ejecución traspasará la codificación recibida con la respuesta saliente.

Codificación de Respuesta
Acepte el valor por defecto ISO-8859-1 como codificación del juego de caracteres para las respuestas de transportes HTTP o introduzca otro diferente.
Cabecera de Autenticación
Introduzca la cabecera HTTP (excepto Autorización) de la que Oracle Service Bus extraerá el elemento. Este campo sólo está disponible si ha activado la casilla de control Autenticación Personalizada.
Por ejemplo, client-xyz-token.
Tipo de Elemento de Autenticación
Seleccione un tipo de elemento de autenticación. Sólo están disponibles los tipos de elementos activos configurados para un proveedor de afirmación de identidad. (Consulte Configuring Identity Assertion Providers for Custom Tokens (Configuración de Proveedores de Afirmación de Identidad para Elementos Personalizados) para obtener más información). Este campo solo está disponible si ha activado la casilla de control Autenticación Personalizada.
