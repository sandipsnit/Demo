Página Configuración de Transporte de JEJB

Utilice esta página para configurar el transporte mediante el protocolo de transporte de JEJB. Para obtener más información sobre el uso del transporte de JEJB, consulte la sección sobre transporte de JEJB en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036.

Política de Distribución
Seleccione la instancia del gestor de trabajos de WebLogic Server que desea utilizar para la política de distribución de este punto final. Se utiliza el gestor de trabajos por defecto si no existe ningún otro.

Versión de Especificación de EJB
Seleccione la versión de EJB de la interfaz de EJB remota.
XMLBeans transferidos por valor
Seleccione esta opción si desea que el transporte genere una representación XML "en línea" de los argumentos POJO (XMLObject) a cuyos parámetros puede acceder y manipular con expresiones de XQuery.
Nota: La información introducida no está disponible en línea para XMLObjects transferidos por valor. Si utiliza esta opción, no puede transferir el XMLObject introducido como argumento en una llamada de Java de un pipeline de servicio de proxy.
No seleccione esta opción si desea transferir POJO por referencia, que también proporciona un mejor rendimiento.
Atributo de Transacción
Seleccione una de las opciones siguientes para el manejo de transacciones:
Soporta: El transporte acepta una transacción entrante. La calidad del servicio es Una Sola Vez si la operación se llama en una transacción y Mayor Esfuerzo si la operación se llama fuera de una transacción.
Necesario: El transporte acepta una transacción entrante. Si no existe ninguna transacción continua, el transporte empieza una. La calidad del servicio es Una Sola Vez.
RequiresNew: El transporte siempre empieza una nueva transacción, suspendiendo una transacción continua. La calidad del servicio es Una Sola Vez.
Obligatorio: El transporte llama al método en la transacción existente. La calidad del servicio es Una Sola Vez.
NotSupported: El transporte suspende una transacción existente y la reanuda en la llamada. La calidad del servicio es Mayor Esfuerzo.
Nunca: El transporte no llama al método en la transacción. La calidad del servicio es Mayor Esfuerzo.
JAR de Cliente
Haga clic en Examinar y seleccione un recurso JAR de cliente de EJB de la lista mostrada. El JAR de cliente contiene la interfaz de negocio o remota para el EJB remoto. El JAR de cliente se registra como recurso de archivo genérico.
Interfaz Raíz
Sólo EJB 2.1: Seleccione la interfaz de EJBHome necesaria de las opciones proporcionadas por el JAR de cliente.
Interfaz Remota
Sólo EJB 2.1. Este campo se rellena automáticamente dependiendo de la configuración de la interfaz raíz.
Interfaz de Negocio
Sólo EJB 3.0. Seleccione la interfaz de negocio del JAR de cliente que desea llamar.
Espacio de Nombres de Destino
Este campo se rellena con la información seleccionada del JAR.
Métodos
Seleccione los métodos necesarios. Haga clic en + para ampliar el método, lo que permite editar los valores del parámetro por defecto.
Puede cambiar el nombre de operación por defecto para un determinado método. Por defecto, el nombre de operación es el nombre del método. Si EJB contiene métodos con el mismo nombre (sobrecarga), debe cambiar los nombres de operación para que sean únicos. WSDL necesita nombres de operación únicos.
