Página Configuración de Transporte de JMS

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte de JMS es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Para obtener más información, consulte la sección sobre transporte de JMS en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte de JMS. En la Tabla 19-10 se describe el uso de esta página.

Opción
Para crear o editar...
Tipo de Destino
Seleccione una de las siguientes opciones:
  • Cola (para un tipo de destino de punto a punto)
  • Tema (para un destino de publicación/suscripción)
Se Necesita Respuesta
Esta opción sólo está disponible si se selecciona Cola para Tipo de Destino.
Seleccione esta opción para especificar que se espera una respuesta después de enviar el mensaje saliente.
Patrón de Respuesta
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Seleccione una de las siguientes opciones:
  • Seleccione JMSMessageID para los servicios JAX-RPC que se ejecuten en Oracle WebLogic Server.
  • Seleccione JMSCorrelationID para todos los demás servicios. Al seleccionar esta opción, también debe introducir un URI de respuesta.
Tipo de Mensaje de Respuesta
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Seleccione una de las siguientes opciones:
  • Bytes (para flujo de bytes no interpretados)
  • Texto (para mensajes de texto)
Esta opción está desactivada si selecciona Java como tipo de mensaje para la respuesta.
Jar de Cliente
Esta opción está disponible cuando el servicio es un servicio de mensajes con un tipo de solicitud de Java. Seleccione el JAR de cliente que va a utilizar para eliminar de la cola los mensajes que contienen objetos Java. La selección del JAR de cliente garantiza que está en la classpath.

Para obtener más información, consulte la sección de envío y recepción de objetos Java en los mensajes en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Política de Distribución
Seleccione la instancia del Gestor de Trabajos de Oracle WebLogic Server que desea utilizar para la política de distribución de este punto final. Se utiliza el Gestor de Trabajos por defecto si no existe ningún otro.
Por ejemplo, si el servicio de negocio tiene un protocolo de transporte de JMS, el punto final del servicio de negocio es un archivo JAR de MDB (bean controlado por mensaje) que puede asociar a la política de distribución en cuestión.
Para obtener más información sobre los gestores de trabajos, consulte la sección sobre el uso de gestores de trabajos para optimizar el trabajo programado) en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificación de Solicitud
Introduzca el juego de caracteres para las solicitudes de codificación. El valor por defecto es UTF-8.
Codificación de Respuesta
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Introduzca el juego de caracteres para las respuestas de codificación. El valor por defecto es UTF-8.
Timeout de Respuesta de Cliente
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Introduzca el número de segundos que esperar una respuesta del servicio antes de borrar la conexión. Esto sólo se aplica si el cliente es otro servicio de proxy en el mismo dominio.
URI de Respuesta
Esta opción sólo está disponible si se selecciona JMSCorrelationID para Patrón de Correlación de Respuesta.
Introduzca un URI de respuesta con el formato:
jms://host:puerto/fábrica_conexiones/destino_jndi
Para utilizar varios servidores, utilice el siguiente formato:
jms://host1:puerto,host2:puerto/fábrica_conexiones/destino_jndi
También puede omitir el host y puerto en el URI de respuesta. Por ejemplo:
jms:///fábrica_conexiones/destino_jndi

Si se omiten, la consulta de fábrica de conexiones/destino se produce en el servidor local. Esto es útil, por ejemplo, si el URI de solicitud va a una fábrica de conexiones/destino en el extranjero, pero desea que la respuesta se envíe al servidor local.

Nota: aunque Oracle WebLogic Server permite el uso de barras inclinadas en los nombres de JNDI (por ejemplo, "miscolas/micola"), los nombres de JNDI con barras inclinadas interfieren en el formato de URI que necesita Oracle Service Bus, por lo que no se pueden utilizar esos nombres. Para solucionar este problema, defina un servidor ajeno de JMS y haga referencia a él en el URI. Para obtener más información, consulte la sección sobre configuración de servidores ajenos en Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help (Ayuda en Pantalla de la Consola de Administración de Oracle Fusion Middleware Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Fábrica de Conexiones de Respuesta
Esta opción sólo está disponible si se selecciona JMSMessageID para Patrón de Correlación de Respuesta.
Introduzca un URI de fábrica de conexiones de respuesta.
Si no se especifica una fábrica de conexiones, se utiliza la fábrica de conexiones de la solicitud para la respuesta.
Cuenta de Servicio de JMS
Seleccione la cuenta de servicio que se utilizará para el recurso de JMS gestionado por el servidor de JMS. Una cuenta de servicio es un recurso de alias para un nombre de usuario y contraseña, que se utiliza para la solicitud y la respuesta. La misma cuenta de servicio se utiliza tanto para JMS como para JNDI.
Para obtener más información, consulte Cuentas de Servicio.
Usar SSL
Seleccione esta opción sólo si las solicitudes se realizan mediante una conexión TLS/SSL.
TLS/SSL (Secure Sockets Layer) proporciona conexiones seguras al permitir que dos aplicaciones se conecten mediante una red para autenticar la identidad de la otra y mediante el cifrado de los datos intercambiados entre las aplicaciones. La autenticación permite que un servidor y, como opción, un cliente verifiquen la identidad de la aplicación en el otro extremo de una conexión de red. Además, si el administrador tiene acceso restringido a los destinos individuales de JMS (colas o temas) al definir el control de acceso en la entrada de JNDI para el destino, el servicio se debe autenticar al consultar la entrada en el árbol de JNDI.
Selector de Mensajes
Introduzca una expresión de selector de mensajes.
Sólo se procesan mensajes con propiedades que coinciden con la expresión.
ID de Cliente de JMS

Sólo para los temas, introduzca el ID de cliente que se va a utilizar para el suscriptor. Si no se introduce ningún valor, se genera automáticamente un ID de cliente. La asignación de un ID personalizado le ayuda a identificar este componente al supervisar el servicio. Tenga en cuenta que esto sólo se aplica para suscriptores duraderos.

El ID de cliente de JMS es uno de los diferentes descriptores de MDB que se utilizan para generar los nombres de suscriptor y el valor de ID de cliente para los suscriptores del tema. La definición de un ID de cliente de JMS facilita la identificación y visualización de los suscriptores de un tema concreto mediante el nombre del suscriptor o el ID de cliente.

Para obtener más información, consulte la sección sobre transporte de JMS en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.
Suscripción Duradera
Esta opción sólo está disponible si se selecciona Tema para Tipo de Destino.
Active esta casilla de control si la suscripción es duradera o déjela en blanco si la suscripción no lo es.
Recuento de Reintentos
Introduzca el número de intentos de entrega para un mensaje antes de moverlo a un destino de error. Este campo sólo se aplica a destinos de JMS de Oracle WebLogic Server.
Intervalo de Reintentos
Introduzca el tiempo, en milisegundos, antes de volver a entregar mensajes con rollback o recuperados. Este campo sólo se aplica a destinos de JMS de Oracle WebLogic Server.
Destino de Error
Introduzca el nombre del destino final para mensajes que han alcanzado su límite de nueva entrega. Este campo sólo se aplica a destinos de JMS de Oracle WebLogic Server.
Política de Caducidad
Seleccione una política de caducidad que utilizar cuando se encuentra un mensaje caducado en un destino. Este campo sólo se aplica a destinos de JMS de Oracle WebLogic Server.
Se Necesita XA
Active esta casilla de control si su fábrica de conexiones es XA.
Este valor se tiene en cuenta cuando la fábrica de conexiones remotas no está disponible. Si su fábrica de conexiones está disponible y este valor es true, asegúrese de que la fábrica de conexiones se define como transaccional.
Timeout de Transacción
Introduzca la cantidad de tiempo en segundos que el servicio de proxy de JMS debe esperar para que se procese una transacción. Esta opción sólo se aplica a los servicios mediante una fábrica de conexiones XA. El valor por defecto es 600 segundos.
Distribución de Mensajes de Tema
Esta opción está disponible al seleccionar Tema en Tipo de Destino. Distribución de Mensajes de Tema proporciona distintas opciones para manejar los mensajes de JMS de entrada y proporcionar una alta disponibilidad y failover.

Seleccione una de las siguientes propiedades para determinar de qué forma manejan los mensajes de JMS de entrada los beans controlados por mensajes:
  • Una Copia por Aplicación (por defecto): Si el servicio de proxy de JMS se despliega en un cluster, esta opción proporciona una amplia disponibilidad y escalabilidad garantizando que el mensaje de JMS de entrada se procesa sólo en uno de los servidores disponibles en un cluster.
  • Una Copia por Servidor: Seleccione esta opción si desea que los mensajes de JMS de entrada publicados en un tema los reciba el servicio de proxy de todos los miembros del cluster.
  • Compatibilidad: seleccione esta opción si desea que los mensajes de JMS de entrada se procesen en un servidor gestionado específico o en todos los servidores gestionados de un cluster. Si selecciona esta propiedad, seleccione un servidor de destino.
Nota: Para destinos de tema desplegados en una versión de Oracle WebLogic Server anterior a la 10.3.4, la única opción válida es Compatibilidad.
Las opciones de "una copia" seleccionadas para Distribución de Mensajes de Tema sustituye la política de uso compartido de suscripciones y/o la política de identificador de cliente configuradas en la fábrica de conexiones de JMS.
destino
Esta opción sólo está disponible en un cluster de Oracle Service Bus si selecciona Compatibilidad en la opción Distribución de Mensajes de Tema. Seleccione el servidor de destino que manejará los mensajes de JMS entrantes. Si selecciona una de las opciones de "una copia" en Distribución de Mensajes de Tema, este campo muestra el nombre del cluster.

Si no define un destino, la instancia de servicio de proxy de JMS (lectura de mensajes fuera del tema) de cada servidor gestionado del cluster recibe una copia del mensaje.
Timeout de JNDI
Timeout de conexión de JNDI (en segundos) utilizado al consultar el destino o la fábrica de conexiones en el árbol de JNDI.
