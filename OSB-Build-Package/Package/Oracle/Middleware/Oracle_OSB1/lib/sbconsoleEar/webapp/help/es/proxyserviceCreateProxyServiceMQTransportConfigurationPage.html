Página Configuración de Transporte de MQ

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte de MQ es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Para obtener más información, consulte la sección sobre transporte de MQ en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Antes de Empezar

Configure un recurso de conexión de MQ. Consulte Conexiones de MQ.

Para configurar el transporte de MQ

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte de MQ nativo. En la Tabla 19-11 se describe el uso de esta página.

Tabla 19-11 Página Crear/Editar un Servicio de Proxy: Configuración de Transporte de MQ 
Opción
Para crear o editar...
Intervalo de Sondeo
Introduzca un intervalo de sondeo en milisegundos. El valor por defecto es 1000.
Se Necesita Respuesta
Seleccione esta opción para especificar que se espera una respuesta después de enviar el mensaje saliente.
Patrón de Correlación de Respuesta
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Especifique si el patrón de correlación de respuesta se debe basar en MessageID y CorrelationID.
URI de Respuesta de MQ
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Destino en el que se debe publicar la respuesta. Introduzca un URI de respuesta en el mismo formato que el URI de punto final: mq://<nombre_cola_local>?conn=<referencia_recurso_conexión_mq>
Tipo de Mensaje de Respuesta
Esta opción sólo está disponible si se ha activado la casilla de control Se Necesita Respuesta.
Seleccione una de las siguientes opciones:
  • Bytes (para flujo de bytes no interpretados)
  • Texto (para mensajes de texto)
Política de Distribución
Seleccione una política de distribución para este punto final.
La política de distribución hace referencia a la instancia del gestor de trabajos de WLS que desea utilizar para el punto final de servicio. Para obtener más información sobre los gestores de trabajos, consulte la sección sobre el uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Umbral de Interrupción
Introduzca un valor que represente el número de veces que el pipeline debe volver a intentar un mensaje antes de redireccionarlo a la cola especificada en el campo URI de Mensajes con Problemas de Entrega de MQ.
Si no especifica un valor para este campo, el mensaje se redirecciona a la cola de mensajes con problemas de entrega sin ningún reintento.
URI de Mensajes con Problemas de Entrega de MQ
Introduzca el URI de la cola de mensajes con problemas de entrega a la que deben direccionarse los mensajes de solicitud después del número de reintentos especificados en el campo Umbral de Interrupción.
Si no especifica un valor para este campo, el mensaje se devuelve a la cola y el transporte de MQ lo ignora para cada sondeo después del número de reintentos especificados en el campo Umbral de Interrupción. El URI de mensajes con problemas de entrega utiliza el mismo formato que el URI de punto final.
Opciones 'GET' del URI de Punto Final
Introduzca las opciones de mensaje GET de MQ de entre las siguientes:
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
Puede utilizar "|" o "+" para separar varias opciones. Por ejemplo, puede especificar lo siguiente:
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
Los opciones de mensaje GET de MQ se aplican al leer un mensaje de la cola entrante.
Cabeceras RFH2 de Proceso
Seleccione esta opción para analizar las cabeceras RFH2 de WebSphere MQ desde una carga útil de mensaje y generar automáticamente una cabecera de transporte RFH2Headers que contenga los datos de RFH2.
Si no selecciona esta opción, la carga útil se transmite como recibida.

Para obtener más información sobre la forma en que el transporte de MQ maneja las cabeceras RFH2, consulte la sección sobre cabeceras RFH2 en la Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
