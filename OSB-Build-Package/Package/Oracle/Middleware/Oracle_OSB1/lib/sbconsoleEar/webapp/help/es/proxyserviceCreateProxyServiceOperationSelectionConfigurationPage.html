Página Configuración de Selección de Operaciones

La página Crear/Editar un Servicio de Proxy: Configuración de Selección de Operaciones es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy. Esta página sólo se muestra si el servicio que está creando tiene operaciones.

Utilice esta página para forzar el cumplimiento de WS-I (sólo para servicios SOAP 1.1) y elija el algoritmo de selección que utilizar para determinar la operación llamada por este servicio de proxy. Esta opción sólo está disponible para servicios SOAP o XML definidos a partir de un WSDL.

La especificación de WSDL define un algoritmo por defecto para calcular qué operación se llama según el tipo de mensaje de SOAP recibido. Sin embargo, hay casos (por ejemplo, problemas de rendimiento, problemas de firma/cifrado o que el algoritmo por defecto no es aplicable) en los que puede que tenga que seleccionar la operación según otros medios.

Oracle Service Bus proporciona algoritmos adicionales. Cada uno de ellos sigue el mismo patrón y se basa en la evaluación de una expresión para obtener un valor que, a continuación, se utiliza para buscar la operación correspondiente en una tabla estática.

Oracle Service Bus normalmente es muy comprensivo si a un mensaje entrante le faltan datos de forma que la operación no se puede determinar o tiene datos que no corresponden a una operación válida. Ambas condiciones dan como resultado u valor de $operation vacío. En lugar de rechazar todos estos mensajes, Oracle Service Bus no inicializa la variable de operación en el contexto, sino que sigue procesando el mensaje.

Sin embargo, los requisitos de seguridad se fuerzan si el servicio de proxy se basa en WSDL y al menos una de las siguientes condiciones es verdadera:

  • El WSDL tiene una política WS-Security y el proxy es un intermediario activo.
  • El proxy tiene autenticación personalizada a nivel de mensaje (elemento personalizado o usuario/contraseña).

Si se cumplen estas condiciones, hay una comprobación en tiempo de ejecución para asegurarse de que el algoritmo de selección de operaciones devuelve un nombre de operación válido. Si la selección de operaciones devuelve null o una operación que no está en WSDL, se rechaza el mensaje y se emite un error.

Aplicar Cumplimiento de WS-I

Active esta casilla de control para especificar si el servicio debe cumplir el perfil básico definido por la organización de interoperabilidad de servicios web.
Cuando un servicio está marcado como compatible con WS-I, las comprobaciones se realizan en los mensajes enviados a y desde dicho servicio. Para los proxies, las comprobaciones se realizan en mensajes de solicitud recibidos por el proxy. Para servicios llamados (es decir, servicios llamados por un proxy mediante una acción de llamada de servicio o nodo de direccionamiento), las comprobaciones se realizan en los mensajes de respuesta recibidos de dichos servicios. Algoritmo de Selección
Seleccione una de las opciones siguientes y realice cualquier paso adicional necesario:
  • Cabecera de Transporte: Seleccione este algoritmo para definir la cabecera de transporte que contiene el valor de búsqueda. A continuación:
    • En el campo Nombre de Cabecera, introduzca la cabecera de transporte que extrae el valor utilizado como clave para seleccionar la operación llamada.
    • En el campo Asignación de Operaciones, especifique el valor para cada operación en el campo Valor. El valor se utiliza como clave de la operación.
  • Cabecera SOAPAction: Seleccione este algoritmo para especificar que se realice la asignación de operación automáticamente desde el WSDL asociado a este servicio de proxy.
  • WS-Addressing: Seleccione este algoritmo para especificar que el valor de búsqueda se incluye en la etiqueta Acción de WS-Addressing situada en las cabeceras de SOAP del mensaje de SOAP. A continuación, en el campo Asignación de Operaciones, especifique el valor para cada operación en el campo Valor. El valor se utiliza como clave de la operación.
  • Cabecera de SOAP: Seleccione este algoritmo para definir una expresión XPath que se evaluará en las cabeceras de SOAP. Esto le permite obtener el valor de búsqueda. A continuación:
    • En el campo Expresión XPath, introduzca la expresión XPath que extrae el valor utilizado como clave para seleccionar la operación llamada.
    • En el campo Asignación de Operaciones, especifique el valor para cada operación en el campo Valor. El valor se utiliza como clave de la operación.
  • Tipo de Cuerpo SOAP: Éste es el algoritmo por defecto definido por la especificación de WSDL para calcular qué operación se llama según el tipo del mensaje de SOAP recibido.

    • Si el servicio de proxy se configura para un caso de traspaso de seguridad del servicio web con un cuerpo cifrado, no puede seleccionar este algoritmo. Una excepción similar se aplica a cabeceras de SOAP cifradas para traspaso.

    Si tiene un WSDL con dos operaciones con el mismo mensaje de entrada, no seleccione este algoritmo para las operaciones, porque la operación no se puede identificar de forma exclusiva inspeccionando el mensaje de entrada.

  • Tipo de Carga Útil: Disponible sólo para servicios XML basados en un puerto de WSDL o enlace de WSDL.

Esta opción está disponible sólo cuando la opción Algoritmo de Selección está definida en Cabecera SOAP.
Especifique la expresión XPath que extrae el valor utilizado como clave para seleccionar la operación llamada.
Asignación de Operaciones
Esta opción está disponible sólo cuando la opción Algoritmo de Selección está definida en Cabecera de Transporte, WS-Addressing o Tipo de Cuerpo SOAP.
Especifique el valor para cada operación en el campo Valor. El valor se utiliza como clave de la operación.
