Página Configuración de Transporte de SFTP

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte de SFTP es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Para obtener más información, consulte la sección sobre transporte de HTTP y sondeo en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV947.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte de SFTP. En la Tabla 19-13 se describe el uso de esta página.

Tabla 19-13 Página Crear/Editar un Servicio de Proxy: Configuración de Transporte de SFTP 
Opción
Para crear o editar...
Autenticación de Usuario
Seleccione una de las siguientes opciones:
  • Contraseña de Nombre de Usuario: Especifica que una cuenta de servicio estática está asociada a este método de autenticación y el cliente se autentica con las credenciales proporcionadas.
  • Basado en Host: Especifica que se necesita un nombre de usuario y un proveedor de claves de servicio para utilizar este método de autenticación. Cualquier usuario que se conecte desde un host conocido se autentica con la clave privada del host.
  • Clave Pública: Especifica que se necesita un nombre de usuario y un proveedor de claves de servicio para utilizar este método de autenticación. Cualquier usuario tiene su propia clave privada.
Cuenta de Servicios
Introduzca la cuenta de servicio para el usuario o haga clic en Examinar para seleccionar las cuentas de servicio desde un explorador.
Proveedor de Claves de Servicio
Esta opción sólo está disponible si se ha seleccionado Basado en Host o Clave Pública.
Introduzca un proveedor de claves de servicio en el campo Proveedor de Claves de Servicio. Puede hacer clic en Examinar para seleccionar proveedores de clave de servicio desde un explorador. Este campo es necesario.
Usuario
Esta opción sólo está disponible si se ha seleccionado Basado en Host o Clave Pública.
Introduzca el nombre de usuario.
Transferir por Referencia
Active esta casilla de control para almacenar en área temporal el archivo en el directorio de archivo y transferirlo como referencia en las cabeceras.
Flujo Remoto
Active esta casilla de control para transmitir los archivos de SFTP directamente desde el servidor remoto en el momento del procesamiento. Al seleccionar esta opción, el directorio de archivo es el directorio remoto en la máquina del servidor SFTP remoto. Por lo tanto, debe especificar el directorio de archivo como relativo al directorio de usuario de SFTP.
Máscara de Archivo
Introduzca la expresión normal para los archivos que se van a seleccionar. El valor por defecto es *.*.
Servidor Gestionado
Este campo está disponible sólo en un dominio de cluster.
Seleccione el servidor gestionado que actuará como servidor de sondeo. Todos los servidores gestionados pueden procesar el mensaje, pero sólo uno puede sondear el mensaje.
Intervalo de Sondeo
Introduzca el intervalo en segundos con el que se sondea el archivo de la ubicación especificada. El valor por defecto es 60.
Límite de Lectura
Especifique el número máximo de mensajes que se van a leer por barrido de sondeo. Introduzca 0 para no especificar ningún límite. El valor por defecto es 10.
Acción Posterior a Lectura
Seleccione lo que sucede con un mensaje una vez leído.
  • Archivar: El mensaje se archiva.
  • Suprimir: El mensaje se suprime.
Directorio de Archivo
Especifique la ruta de acceso a la ubicación de archivo si la opción Acción Posterior a Lectura está definida en Archivar. Este campo es necesario si se ha seleccionado la opción Transferir por Referencia.
Nota: Los directorios de archivo, descarga y error son rutas de acceso absolutas y se crean automáticamente. Si especifica una ruta de acceso relativa, los archivos se crean con respecto al proceso Java que inicia WebLogic Server.
Directorio de Descarga
Introduzca el directorio en la máquina local en la que se descargan los archivos durante la transferencia de archivos.
Nota: Los directorios de archivo, descarga y error son rutas de acceso absolutas y se crean automáticamente. Si especifica una ruta de acceso relativa, los archivos se crean con respecto al proceso Java que inicia WebLogic Server.
Directorio de Error
Introduzca la ubicación en la que se publican los mensajes si hay un problema.
Nota: Los directorios de archivo, descarga y error son rutas de acceso absolutas y se crean automáticamente. Si especifica una ruta de acceso relativa, los archivos se crean con respecto al proceso Java que inicia WebLogic Server.
Codificación de Solicitud
Acepte el valor por defecto UTF-8 como codificación del juego de caracteres para las solicitudes de transportes de SFTP.
Conjunto de Cifrado Preferido
Seleccione el conjunto de cifrado que se usará para comunicarse con el servidor.
Si utiliza el valor por defecto, Usar Valor por Defecto de Tiempo de Ejecución, se envía al servidor la lista de conjuntos de cifrado soportados y se intenta hasta que se encuentre una coincidencia.
Algoritmo de Integridad de Datos Preferido
Seleccione el algoritmo de hash masivo para las comprobaciones de integridad de datos.
Si utiliza el valor por defecto, Usar Valor por defecto de Tiempo de Ejecución Oracle Service Bus envía el algoritmo preferido hmac-sha1 al servidor. Si el servidor no lo soporta, se envía al servidor la lista de algoritmos soportados y se prueba cada uno de ellos hasta que se encuentre una coincidencia.
Algoritmo de Clave Pública Preferido
Seleccione el algoritmo de clave asimétrica para el cifrado de clave pública.
Si utiliza el valor por defecto, Usar Valor por Defecto de Tiempo de Ejecución, Oracle Service Bus envía el algoritmo preferido ssh-dss en el servidor. Si el servidor no lo soporta, se envía al servidor la lista de algoritmos soportados y se prueba cada uno de ellos hasta que se encuentre una coincidencia.
Algoritmo de Intercambio de Claves Preferido
Seleccione el protocolo de intercambio de clave por defecto para negociar la clave de sesión para cifrar el mensaje.
Si utiliza el valor por defecto, Usar Valor por Defecto de Tiempo de Ejecución, Oracle Service Bus envía el algoritmo preferido diffie-hellman-group1-sha1 al servidor. Si el servidor no lo soporta, se envía la lista de algoritmos soportados y se prueba cada uno de ellos hasta que se encuentre una coincidencia.
Algoritmo de Compresión Preferido
Seleccione si desea comprimir los datos en ejecución mediante zlib. Seleccione zlib para comprimir los datos; de lo contrario, seleccione Ninguno. El valor por defecto es Ninguno.
Explorar Subdirectorios
Active esta casilla de control para explorar de forma recurrente todos los directorios.
Ordenar por Llegada
Active esta casilla de control para entregar eventos en el orden de llegada.
Timeout
Introduzca el intervalo de timeout del socket, en segundos, antes de que se cierre la conexión. Si introduce 0, no hay timeout. El valor por defecto es 60.
Recuento de Reintentos
Especifique el número de reintentos para fallos de conexión SFTP.
