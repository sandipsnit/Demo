Página Configuración de Transporte de Tuxedo

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte de Tuxedo es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Para obtener más información, consulte la sección sobre transporte de Tuxedo en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte de Tuxedo. En la Tabla 19-14 se describe el uso de esta página.

Tabla 19-14 Página Crear/Editar un Servicio de Proxy: Configuración de Transporte de Tuxedo 
Opción
Para crear o editar...
Clases de Tablas de Campo
Introduzca el nombre de las clases que describan el buffer de FML/FML32 recibido. Se utilizan para las rutinas de conversión de FML/FML32 a XML para asignar nombres de campo a nombres de elemento. Es una lista separada por espacios de nombres de clase completamente cualificados.
Clases de Vista
Introduzca el nombre de las clases que describan el buffer de VIEW/VIEW32 recibido o enviado. Se utilizan para las rutinas de conversión de VIEW a XML o VIEW32 a XML para asignar nombres de campo a nombres de elemento. Es una lista separada por espacios de nombres de clase completamente cualificados.
Los tipos de buffer de Tuxedo X_C_TYPE y X_COMMON se manejan de la misma forma que los buffers VIEW/VIEW32.
Si una solicitud entrante contiene VIEW, se debe especificar la clase VIEW correspondiente en CLASSPATH de Oracle Service Bus.
Jar de Clases
Seleccione un recurso de JAR que contenga un archivo JAR con las clases FML/FML32 o VIEW/VIEW32 necesarias para esta operación de punto final.
Punto de Acceso Local
Seleccione un punto de acceso local de la lista desplegable asociada a la exportación. La lista desplegable contiene los puntos de acceso local configurados en WTC. No se puede crear un servicio de proxy si no hay ningún punto de acceso local asociado.
Si no existe ningún punto de acceso local o para crear uno nuevo, seleccione Nuevo. Introduzca el nombre del punto de acceso local y la dirección de red local correspondientes en los campos adyacentes. Tras la validación del punto final, se agrega el punto de acceso a la configuración de WTC para cada servidor de WTC. Si no existe ningún servidor de WTC, se crea uno.
Puede introducir un nombre de punto de acceso existente después de seleccionar la opción Nuevo. Esto hace que se actualice la información existente con los nuevos parámetros. Sólo puede cambiar el nombre de host y número de puerto.
Punto de Acceso Remoto
Este campo aparece sólo si selecciona Nuevo en el campo Punto de Acceso Local.
En la lista desplegable, seleccione el punto de acceso remoto que desea asociar al nuevo punto de acceso local creado. Si no existe ninguno o para crear uno nuevo, seleccione Nuevo. Introduzca el nombre del punto de acceso y la dirección de red correspondientes en los campos adyacentes.
Puede introducir un nombre de punto de acceso existente después de seleccionar la opción Nuevo. Esto hace que se actualice la información existente con los nuevos parámetros. Sólo puede cambiar el nombre de host y número de puerto.
El punto de acceso remoto también será el principal de autenticación para la conexión de WTC para solicitudes entrantes. Opcionalmente, puede crear un usuario con el mismo identificador de punto de acceso en el dominio de seguridad por defecto para permitir llamadas entrantes. Para ello, seleccione Sí en la lista desplegable ¿Crear Usuario? La contraseña se generará aleatoriamente mediante una variable temporal para evitar problemas de seguridad.
Tipo de Buffer de Respuesta
Esta opción sólo está disponible si se selecciona el campo ¿Respuesta Obligatoria?.
Seleccione el tipo de buffer que recibirá el cliente de Tuxedo remoto.
Subtipo de Buffer de Respuesta
Esta opción sólo está disponible cuando se selecciona la opción ¿Respuesta Obligatoria? y el valor de Tipo de Buffer de Respuesta es VIEW o VIEW32.
Introduzca el subtipo de buffer con el que asociar el buffer de respuesta.
¿Respuesta Obligatoria?
Active esta casilla de control si se espera que este servicio envíe una respuesta.
El estado por defecto es que se seleccione esta opción.
Esta opción se borra y no está disponible si el tipo de servicio es Servicio de Mensajes y el tipo de mensaje de respuesta es Ninguno.
Codificación de Solicitud
Especifique una codificación de juego de caracteres para las solicitudes de los transportes de Tuxedo.
Codificación de Respuesta
Especifique una codificación de juego de caracteres para las respuestas de los transportes de Tuxedo.
Estilo de Transformación
Seleccione una de las siguientes opciones:
  • Ninguno: (Valor por defecto) Puede que no se respete el orden de los campos.
  • Ordenado: Los campos se presentan con todas sus incidencias en el orden correcto.
  • Agrupado y Ordenado: Si los campos tienen una estructura lógica de registros, los campos se ordenan por incidencia y se agrupan por registro.
