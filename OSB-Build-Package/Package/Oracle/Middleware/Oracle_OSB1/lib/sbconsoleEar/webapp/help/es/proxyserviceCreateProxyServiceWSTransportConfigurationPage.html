Página Configuración de Transporte de WS

La página Crear/Editar un Servicio de Proxy: Configuración de Transporte de WS es una de una serie de páginas para la creación y edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Para obtener más información, consulte la sección sobre transporte de WS en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083.

Utilice esta página para configurar el transporte de un servicio de proxy mediante el protocolo de transporte de WS. En la Tabla 19-15 se describe el uso de esta página.

Tabla 19-15 Página Crear/Editar un Servicio de Proxy: Configuración de Transporte de WS

Para crear o editar...
Política de Distribución
Seleccione una política de distribución para este punto final o utilice la política de distribución por defecto.
La política de distribución hace referencia a la instancia del gestor de trabajos de WLS que desea utilizar para el punto final de servicio. Para obtener más información sobre los gestores de trabajos, consulte la sección sobre el uso de gestores de trabajos para optimizar el trabajo programado en Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server (Configuración de Entornos de Servidor de Oracle Fusion Middleware para Oracle WebLogic Server) en http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Recuento de Reintentos
Número de veces que se debe reintentar la entrega de un mensaje al pipeline.
Si se produce una excepción no tratada en el pipeline de solicitudes de un servicio de proxy, el mensaje de transporte de WS entrante se volverá a entregar al pipeline hasta el número de veces especificado por el recuento de reintentos. Este valor es importante para un procesamiento fiable de mensajes de transporte de WS.
Retraso de Reintento
Número de segundo que se pausa el sistema antes de reintentar enviar un mensaje al pipeline después de un error.
