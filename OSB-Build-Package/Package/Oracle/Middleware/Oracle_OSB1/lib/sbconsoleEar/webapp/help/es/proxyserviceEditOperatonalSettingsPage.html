Configuración Operativa de Servicios de Proxy

Utilice la página Configuración Operativa para activar y desactivar un servicio de proxy y la supervisión de determinados servicios de proxy. Del mismo modo, puede especificar esta misma configuración operativa específica del servicio en la página Operaciones > Búsqueda Inteligente, con las siguientes excepciones: No puede definir un intervalo de agregación ni especificar un nivel de gravedad de alerta o registro. Para obtener más información, consulte Búsqueda y Actualización de Configuración Operativa.

Los efectos en tiempo de ejecución de la configuración de nivel de servicio dependen de su configuración global correspondiente. Debe activar tanto la configuración global como la de nivel de servicio para que un servicio esté completamente activado en tiempo de ejecución. También se debe activar el estado del servicio. Consulte Activación de la Configuración Global.

Puede cambiar y guardar la configuración de supervisión incluso si no se va a activar el servicio en tiempo de ejecución. Por ejemplo, puede cambiar y guardar el intervalo de agregación incluso si está desactivada la supervisión de servicio. De esta forma, puede editar la configuración y activarla más tarde.

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Seleccione Explorador de Recursos > Servicios de Proxy.
  3. Haga clic en el nombre del servicio de proxy.

    4. La página Ver Servicio de Proxy muestra la información de la Tabla 25-6.

    Tabla 25-6 Información de Servicio de Proxy 
    Propiedad
    Descripción
    Modificado por
    Usuario que ha creado o editado este servicio, o lo ha importado a la configuración.
    Fecha de Modificación
    Fecha y hora a la que el usuario ha creado o editado este servicio, o lo ha importado a la configuración. Haga clic en el enlace de fecha y hora para ver el historial de cambios de este recurso. Consulte Página Ver Historial de Cambios.
    Referencias
    Número de objetos a los que hace referencia este servicio. Si existen estas referencias, haga clic en el enlace numérico para ver una lista de los objetos. Consulte Visualización de Referencias a Recursos.
    Con Referencia de
    Número de objetos que hacen referencia a este servicio. Si existen estas referencias, haga clic en el enlace numérico para ver una lista de los objetos. Consulte Visualización de Referencias a Recursos.
    Descripción
    Descripción del servicio, si existe.

  4. Seleccione el separador Configuración Operativa.

    5. Debe estar en una sesión activa para realizar la configuración operativa del servicio de proxy mostrada en la Tabla 25-7.

    Tabla 25-7 Configuración Operativa del Servicio de Proxy 
    Para...
    Realice esta acción...
    Desactivar el estado
    Desactive la casilla de control Activado. Cuando activa la sesión, el servicio deja de procesar los mensajes.
    Al desactivar un servicio, el sistema ya no suprime todas las estadísticas de supervisión recopiladas previamente para el servicio.
    Activar el estado
    Active la casilla de control Activado. Cuando activa la sesión, el servicio reanuda el procesamiento de los mensajes.
    Desactivar la supervisión
    Desactive la casilla de control Activado. Al activar la sesión, el sistema deja de recopilar estadísticas de supervisión para el servicio y suprime las estadísticas ya recopiladas.
    Activar la supervisión
    Active la casilla de control Activado junto a Supervisión. En la lista desplegable Activar Supervisión de Pipeline en nivel, seleccione restringir la supervisión al nivel especificado o por encima (del nivel más inclusivo al más restrictivo): Acción, Pipeline y Servicio.
    Al activar la sesión, el sistema empieza a recopilar estadísticas de supervisión para el servicio.
    Definir un intervalo de agregación para un servicio
    Seleccione el intervalo en horas o minutos y haga clic en Actualizar. Si la selección de horas excede de 1, la selección por defecto de minutos es siempre cero. Sin embargo, si la selección de horas es 0 o 1, puede configurar intervalos en minutos. Consulte Definición del Intervalo de Agregación para un Servicio.
    Puede cambiar y guardar el valor de intervalo de agregación independientemente de si ha activado la supervisión.
    Para activar la supervisión en tiempo de ejecución, asegúrese de que la casilla de control Activar Supervisión está activada.
    Desactivar alertas de SLA
    Desactive la casilla de control Activado. El sistema deja de evaluar las reglas de alertas configuradas para el servicio; por lo tanto, deja de recibir alertas de SLA asociadas al servicio.
    Activar alertas de SLA
    Active la casilla de control Activado. En la lista desplegable Activar Alertas en nivel, seleccione la restricción de alertas de SLA en el nivel especificado o por encima (del nivel más inclusivo al más restrictivo): Normal (valor por defecto), Advertencia, Menor, Grave, Crítico y Fatal.
    Las alertas de SLA de servicio dependen de los estados de supervisión global y de nivel de servicio; ambos deben estar activados para que las alertas de SLA se activen en tiempo de ejecución. Consulte Activación de Alertas de SLA Globalmente.
    Desactivar alertas de pipeline
    Desactive la casilla de control Activado. Al activar la sesión, el sistema deja de ejecutar las acciones de alerta de pipeline; por lo tanto, deja de recibir alertas de pipeline asociadas al servicio.
    Activar alertas de pipeline
    Active la casilla de control Activado. En la lista desplegable Activar Alertas en nivel, seleccione restringir las alertas de pipeline en el nivel especificado o por encima (del nivel más inclusivo al más restrictivo): Normal (valor por defecto), Advertencia, Menor, Grave, Crítico y Fatal.
    Desactivar la acción de informe de un servicio de proxy
    Desactive la casilla de control Activado junto a Informes de Mensajes. Al activar la sesión, el sistema deja de ejecutar las acciones de alerta de informe; por lo tanto, deja de recibir informes de mensajes asociados al servicio.
    Activar la acción de informe de un servicio de proxy
    Active la casilla de control Activado junto a Informes de Mensajes.
    Desactivar la salida de registro para una acción de log de pipeline
    Desactive la casilla de control Activado junto a Registro. Al activar la sesión, el sistema deja de ejecutar las acciones de log; por lo tanto, deja de recibir la salida de registro asociada al servicio.
    Activar la salida de registro para una acción de log de pipeline
    Active la casilla de control Activado junto a Registro. En la lista desplegable Activar Registro en nivel, seleccione restringir la salida de registro al nivel especificado o por encima (del nivel más inclusivo al más restrictivo): Depurar (valor por defecto), Información, Advertencia y Error.
    Activar el rastreo de ejecución en tiempo de ejecución para un servicio de proxy

    Active la casilla de control Activado junto a Rastro de Ejecución. Después de activarlo, el sistema registra varios detalles seleccionados del contexto de flujo de mensajes y del contexto de mensaje. Estos detalles incluyen: Nombre de la etapa, nombre del nodo de direccionamiento o pipeline, y contexto de mensaje actual.

    Nota: para ver el rastreo en el archivo log o en la salida estándar (consola del servidor), el registro de Oracle WebLogic Server debe estar definido en los siguientes niveles de gravedad:

    Gravedad Mínima que Registrar: Información
    Archivo Log: Información
    Salida Estándar: Información

    Para obtener información sobre la definición de niveles de gravedad de log, consulte la sección sobre el uso de los niveles de gravedad de en Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server (Configuración de Archivos Log y Filtrado de Mensajes de Log de Oracle Fusion Middleware para Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

    Desactivar el rastreo en tiempo de ejecución para un servicio de proxy
    Desactive la casilla de control Activado junto a Rastreo de Ejecución.
    Activar el rastreo de mensajes para un servicio de proxy

    Active la casilla de control Activado junto a Rastreo de Mensajes. Después de activarlo, el sistema registra los mensajes intercambiados entre el pipeline de Oracle Service Bus y el servicio de proxy (mensajes respuesta y solicitud entrantes, así como mensajes de respuesta y solicitud salientes).

    Nota: para ver el rastreo en el archivo log o en la salida estándar (consola del servidor), el registro de Oracle WebLogic Server debe estar definido en los siguientes niveles de gravedad:

    Gravedad Mínima que Registrar: Información
    Archivo Log: Información
    Salida Estándar: Información

    Para obtener información sobre la definición de niveles de gravedad de log, consulte la sección sobre el uso de los niveles de gravedad de en Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server (Configuración de Archivos Log y Filtrado de Mensajes de Log de Oracle Fusion Middleware para Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

    Tenga en cuenta que, cuando se aplica, los mensajes salientes registrados también pueden incluir el número de reintentos, código de error y mensaje de error.
    En la lista desplegable Nivel de Detalle, seleccione uno de los siguientes niveles de detalle:
    • Breve: Muestra la fecha, hora, tipo de servicio, nombre de servicio y URI.
    • Cabeceras: Muestra la información breve más la representación XML de los metadatos del mensaje.
    • Completo: Muestra la información de las cabeceras más la carga útil raw, incluidos los anexos, si los hay.
    Si selecciona Completo de la lista desplegable Nivel de Detalle, especifique el tamaño máximo (en KB) para la carga útil de los mensajes mediante el campo Límite de Rastreo de Carga Útil. Especifique también la codificación por defecto para el registro de la carga útil en el campo Codificación por Defecto. Puede resultar útil al registrar cargas útiles binarias o mensajes SOAP con anexos binarios.
    El valor de codificación por defecto puede ser Base64 o cualquier codificación soportada por Java como se define en: http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html.
    Si se deja el campo Codificación por Defecto vacío, Oracle Service Bus utiliza la codificación por defecto del host para la carga útil. La codificación por defecto depende de una combinación de los valores de JVM, sistema operativo (SO) subyacente y configuración regional de nivel del sistema operativo.
    Si no se puede utilizar el valor especificado en el campo Codificación por Defecto (por ejemplo, si no es una opción válida para la configuración), Oracle Service Bus utiliza la codificación Base64 para la carga útil.
    Desactivar el rastreo de mensajes para un servicio de proxy
    Desactive la casilla de control Activado junto a Rastreo de Mensajes.
