Localización de Servicios de Proxy

  1. Realice una de las siguientes acciones:
    • Seleccione Explorador de Proyectos para que aparezca la página Vista de Proyectos o la página Vista de Proyectos/Carpetas. A continuación, navegue por los proyectos o carpetas para buscar el servicio de proxy.
    • Seleccione Explorador de Recursos > Servicios de Proxy. La página Resumen de Servicios de Proxy muestra la información que aparece en la Tabla 19-18. Para obtener una descripción más detallada de las propiedades, consulte Edición de Configuraciones de Servicio de Proxy.
  2. Para limitar el número de elementos de la lista, puede filtrar por nombre, ruta de acceso o ambos. En los campos Nombre y Ruta de Acceso debajo de Buscar, introduzca el nombre y/o la ruta de acceso de los destinos y haga clic en el botón Buscar.

    3. La ruta de acceso es el nombre del proyecto y el nombre de la carpeta donde reside el servicio de proxy.

    Se permiten los comodines * y ?. La búsqueda es sensible a mayúsculas/minúsculas.

    Haga clic en Ver Todo para mostrar todos los servicios de proxy del dominio. Se borran los parámetros de la búsqueda anterior.

    Tabla 19-18 Página Resumen de Servicios de Proxy 
    Propiedad
    Descripción
    Nombre
    Nombre único del servicio de proxy. Haga clic en el nombre para mostrar la página Ver Servicio de Proxy: Detalles de Configuración.
    Consulte Edición de Configuraciones de Servicio de Proxy.
    Ruta de Acceso
    La ruta de acceso es el nombre del proyecto y el nombre de la carpeta donde reside el servicio de proxy, por ejemplo, UDDI/BusServices/OSB_services.
    Haga clic en la ruta de acceso del servicio de proxy para mostrar la página Vista de Proyectos/Carpetas que la contiene.
    Acciones
    Realice una de las siguientes acciones:
    • Haga clic en el icono Editar Flujo de Mensajes para editar el flujo de mensajes del servicio de proxy. Consulte Creación, Edición y Visualización de Flujos de Mensajes.
    • Haga clic en el icono Iniciar Consola de Prueba para llamar a la consola de prueba, que se utiliza para validar y probar el diseño de los servicios y transformaciones. Para los servicios de negocio, sólo puede utilizar la consola de prueba en tiempo de ejecución; es decir, cuando esté activada la sesión. Para las transformaciones, puede utilizarla tanto dentro como fuera de una sesión. Consulte Prueba de Servicios y Prueba de Transformaciones.
    • El icono Exportar WSDL aparece para los servicios de negocio basados en WSDL. Haga clic en este icono para exportar un WSDL, para poder verlo o modificarlos en una herramientas externa como IDE.

      • Tenga en cuenta que es diferente a la funcionalidad Exportar Recursos del módulo Administración del Sistema, que se utiliza para mover y crear etapas de los recursos entre dos dominios. Consulte Exportación de WSDL.

    Opciones
    Haga clic en el icono Suprimir para suprimir el servicio. Aparece un icono Advertencia de Supresión si hay otros recursos que hacen referencia a éste. Puede suprimirlo confirmando la advertencia. Esto puede causar conflictos debido a referencias no resueltas al recurso suprimido. Para obtener más información, consulte Supresión de Servicios de Proxy.
