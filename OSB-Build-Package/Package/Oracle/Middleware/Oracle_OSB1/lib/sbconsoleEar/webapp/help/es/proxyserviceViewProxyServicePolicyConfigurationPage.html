Página Política de Servicio de Proxy

La página Ver Servicio de Proxy: Políticas es una de una serie de páginas para la edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Utilice esta página para configurar los valores de política para un servicio de proxy con un tipo de servicio web WSDL, servicios de mensajes, servicios SOAP o servicios XML.En la Tabla 19-20 se describe el uso de esta página. Para que las políticas de OWSM se utilicen con el servicio web WSDL, servicio de mensajes o servicios de proxy del servicio XML, el protocolo se debe definir en HTTP.

Para los servicios basados en WSDL, se muestran (en línea) las políticas de WLS 9.2 enlazadas al servicio en el WSDL efectivo. Las políticas abstractas se procesan previamente antes de ponerse en línea. Las políticas de OWSM se enlazan por referencia, no se ponen en línea en el WSDL efectivo.

Configuración de Política de Servicio
A continuación se muestran las opciones del campo Configuración de Política de Servicio:
  • De Almacén de Políticas de OWSM: las políticas están definidas en el almacén central de políticas de Oracle Web Services Manager gestionado por Oracle Enterprise Manager. Estas políticas soportan WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken Profile y otros estándares del sector.

    Utilice el filtrado, si es necesario, para localizar las políticas deseadas. Para los servicios de proxy, sólo se muestran las políticas del servicio.

    • Para obtener más información, consulte la sección sobre protección de Oracle Service Bus con Oracle Web Services Manager en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

  • De WSDL: seleccione esta opción si la política de servicio está asociada al WSDL en el que está basado el servicio. Estas políticas soportan WS-Security 1.0, SAML 1.1 y otros estándares del sector.

    • Con esta opción, puede ver (sólo lectura) las políticas de solicitud y respuesta desde WSDL.

  • De Política Predefinida o Recurso de Política de Servicios Web, donde puede agregar políticas de nivel de servicio, políticas de nivel de operación (en cuyo caso, la política se aplica a los mensajes de solicitud y respuesta), políticas de solicitud y políticas de respuesta desde la consola de administración de Oracle Service Bus.

    Las políticas están predefinidas en Oracle WebLogic Server o definidas por el usuario en Oracle Service Bus con un recurso de política de servicios web. Estas políticas soportan WS-Security 1.0, SAML 1.1 y otros estándares del sector.

    • Con esta opción, puede agregar políticas de nivel de servicio, operación, solicitud y respuesta.

Nota: Los modelos de enlace de política se excluyen mutuamente. Debe utilizar un solo tipo de política en un servicio. Si enlaza las políticas directamente al servicio, se ignoran todas las políticas basadas en WSDL.

Después de terminar

Haga clic en Actualizar para guardar esta configuración o en Restablecer para deshacer los cambios.
