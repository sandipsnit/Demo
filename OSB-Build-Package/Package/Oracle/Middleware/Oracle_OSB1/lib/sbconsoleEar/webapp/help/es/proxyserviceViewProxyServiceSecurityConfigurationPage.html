Página Seguridad de Servicio de Proxy

La página Ver Servicio de Proxy: Seguridad es una de una serie de páginas para la edición de servicios de proxy, como se describe en Creación y Configuración de Servicios de Proxy y Edición de Configuraciones de Servicio de Proxy.

Utilice esta página para la configuración de seguridad de un servicio de proxy.

Los campos disponibles en esta página dependen del uso de las políticas del servicio de proxy. Por ejemplo, si el servicio utiliza políticas de OWSM (recomendado), aparece Sustituciones de Políticas en la página Seguridad. Para obtener más información, consulte la sección sobre protección de Oracle Service Bus con Oracle Web Services Manager en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Para políticas WLS 9.2, la configuración de usuario/contraseña personalizados y de elemento personalizado es similar. En ambos casos, especifique expresiones XPath para permitir a Oracle Service Bus localizar la información necesaria. El raíz de estas expresiones XPath es el siguiente:

  • Utilice soap-env:Envelope/soap-env:Header si el enlace de servicio es AnySOAP o WSDL-SOAP.
  • Utilice soap-env:Body si el enlace de servicio no se basa en SOAP.

Todas las expresiones XPath deben tener un formato de XPath válido. Las expresiones XPath deben utilizar la sintaxis de "declare namespace" de XPath para declarar cualquier espacio de nombres utilizado, de la siguiente forma:

declare namespace
ns='http://webservices.example.com/MyExampleService';)

Configuración de Seguridad de Servicios Web
Procesar Cabecera de WS-Security
Si una solicitud de cliente incluye una cabecera de seguridad de WS-Security, decide si procesar esta cabecera:
  • Seleccione Sí: En un caso intermedio activo, el cliente aplica WS-Security a los mensajes de solicitud y/o respuesta. El servicio de proxy procesa la cabecera de seguridad y fuerza la política WS-Security. Esta opción se define automáticamente en Sí cuando un servicio de proxy está basado en un WSDL que tiene una política de seguridad o cuando asocia una política al servicio en la página Configuración de Política de Servicio.
  • Seleccione No: En un caso de traspaso de WS-Security, el cliente aplica WS-Security a los mensajes de solicitud y/o respuesta. El servicio de proxy procesa la cabecera de seguridad. En su lugar, transfiere el mensaje de solicitud protegido intacto a un servicio de negocio.
Control de Acceso
Control de Acceso a Transporte
Para todos los servicios de proxy, puede crear una política de nivel de transporte, que aplique una comprobación de seguridad cuando un cliente intenta establecer una conexión con el servicio de proxy. Sólo se permite procesar solicitudes de usuarios que se muestran en la política de nivel de transporte.
Control de Acceso a Mensajes
Una política de control de acceso de nivel de mensaje aplica una comprobación de seguridad cuando un cliente intenta llamar a un servicio de proxy con seguridad a nivel de mensaje. Puede crear una política de control de acceso de nivel de mensaje en los siguientes casos:
  • Para servicios de proxy que son intermediarios de seguridad de servicios web activos; es decir, procese la cabecera de WS-Security.
  • Para servicios de proxy para los que defina una autenticación personalizada en esta página.
Sólo los usuarios que se muestran en la política de nivel de mensaje pueden llamar a la operación.

Tipo de Autenticación
Seleccione una de las siguientes opciones:
  • Seleccione Ninguno si el servicio de proxy no utilizará la autenticación personalizada.
  • Nombre de Usuario y Contraseña Personalizados

    • Al seleccionar esta opción, también debe introducir valores en los campos XPath de Nombre de Usuario y XPath de Contraseña de Usuario.

  • Elemento Personalizado

    • Al seleccionar esta opción, también debe seleccionar un valor para Tipo de Elemento e introduzca valores para el campo XPath de Elemento.

XPath de Nombre de Usuario
Esta opción sólo está disponible cuando la opción Configuración de Autenticación Personalizada está definida en Nombre de Usuario y Contraseña Personalizados. Cuando está disponible, esta opción es necesaria.
Introduzca el nombre de usuario como expresión XPath.
La expresión XPath se evalúa con respecto a las cabeceras de mensajes o carga útil, según corresponda, que permiten a Oracle Service Bus obtener el nombre de usuario para la autenticación personalizada.
XPath de Contraseña de Usuario
Esta opción sólo está disponible cuando la opción Configuración de Autenticación Personalizada está definida en Nombre de Usuario y Contraseña Personalizados. Cuando está disponible, esta opción es necesaria.
Introduzca la contraseña como expresión XPath.
La expresión XPath se evalúa con respecto a las cabeceras de mensajes o carga útil, según corresponda, que permiten a Oracle Service Bus obtener los valores de contraseña para la autenticación personalizada.
Tipo de Elemento
Esta opción sólo está disponible cuando la opción Configuración de Autenticación Personalizada está definida en Elemento Personalizado. Cuando está disponible, esta opción es necesaria.
Seleccione el tipo de elemento de la lista desplegable. Sólo están disponibles los tipos de elementos activos configurados para un proveedor de afirmaciones de identidad de WebLogic Server. Consulte la sección sobre la configuración de proveedores de afirmaciones de identidad para elementos personalizados en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660.
XPath de Elemento
Esta opción sólo está disponible cuando la opción Configuración de Autenticación Personalizada está definida en Elemento Personalizado. Cuando está disponible, esta opción es necesaria.
Introduzca una expresión XPath para especificar una ruta de acceso al elemento personalizado. Oracle Service Bus evalúa la expresión XPath de elemento con respecto las cabeceras de mensaje o carga útil, según corresponda, para obtener el elemento para la autenticación personalizada.
Propiedades de Contexto
Opcionalmente, especifique una o más propiedades de contexto para transferir información de contexto adicional al proveedor de seguridad de autenticación (Nombre de Usuario y Contraseña Personalizados) o de afirmaciones de identidad (Elemento Personalizado).
Propiedades de Contexto proporcionan una forma (interfaz ContextHandler) para transferir información adicional al marco de seguridad de WebLogic de forma que un proveedor de seguridad pueda obtener información contextual. Consulte la sección sobre propiedades de contexto que se transfieren a proveedores de seguridad en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500.
Introduzca el nombre de la propiedad como cadena de literales y el selector de valores como expresión XPath válida. (Las expresiones XPath también pueden ser cadenas de literales.)
Las expresiones XPath se evalúan con respecto a la misma parte del mensaje utilizada para el elemento personalizado o nombre de usuario/contraseña personalizados. Es decir, las expresiones XPath de selector de valores para servicios de proxy basados en SOAP se evalúan con respecto a la cabecera y la carga útil para servicios de proxy no basados en SOAP.
La expresión XPath se evalúa en tiempo de ejecución para producir el valor de la propiedad. ContextHandler es esencialmente una lista de nombres/valores y, como tal, necesita que un proveedor de seguridad sepa qué nombres buscar. Por lo tanto, las expresiones XPath sólo se evalúan si un proveedor de seguridad pregunta el valor de una de estas propiedades definidas por el usuario.
Haga clic en Agregar Propiedad para agregar esta propiedad de contexto. Puede agregar varias propiedades de contexto.
