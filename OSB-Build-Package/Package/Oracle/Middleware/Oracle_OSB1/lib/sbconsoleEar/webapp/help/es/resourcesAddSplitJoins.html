Adición de Divisiones-Uniones

Antes de Empezar

Las divisiones-uniones requieren un archivo .flow que se crea cuando se crea una división-unión con los plugins de Oracle Service Bus para Eclipse, o cuando se importa un JAR de configuración División-Unión al plugin. El archivo .flow puede incluir referencias a servicios de proxy, servicios de negocio o recursos WSDL externos. Los recursos de los que dependa la división-unión deberán estar presentes en el servidor antes de activar la división-unión.

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Seleccione Explorador de Proyectos y seleccione a continuación el proyecto o la carpeta a donde desee agregar la división-unión. Se muestra la página Vista de Carpetas/Proyectos.
  3. Desde la lista desplegable Crear Recurso, seleccione División-Unión para mostrar la página Crear un Nuevo Recurso de División-Unión.

    4. Especifique los parámetros que se muestran en la Tabla 9-2.

    Tabla 9-2 Parámetros de División-Unión
    Propiedad
    Para crear o editar...
    Nombre del Recurso
    Introduzca un nombre único para esta división-unión.

    Siga las Restricciones de Nomenclatura de Recursos para obtener ayuda con la nomenclatura.
    Descripción del Recurso
    Introduzca una descripción para la división-unión.
    División-Unión
    Haga clic en Examinar para localizar en el sistema un archivo .flow. Generalmente están en MW_HOME\proyectos_usuario\espacios_trabajo\su_espacio_trabajo.
    Política de Distribución
    Seleccione la instancia del gestor de trabajos de WebLogic Server que se utilizó para crear threads (ramas paralelas) para ejecutar la división-unión. Si no hay otro gestor de trabajos, se utilizará el gestor de trabajos por defecto.

  4. Haga clic en Guardar. Se creará la división-unión y se guardará en la sesión actual.
    5. Nota: Para poder activar la división-unión, es necesario que los recursos de los que depende estén presentes previamente.
  5. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.
