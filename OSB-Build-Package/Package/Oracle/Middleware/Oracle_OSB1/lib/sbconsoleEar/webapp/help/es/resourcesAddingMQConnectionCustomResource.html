Adición de Conexiones de MQ

Antes de Empezar

Consulte Oracle Fusion Middleware Supported System Configurations (Configuraciones de Sistema de Oracle Fusion Middleware Soportadas) en http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html para averiguar qué versiones de WebSphere MQ soporta Oracle Service Bus. La biblioteca de cliente de WebSphere MQ no se conecta con Installer de Oracle Service Bus. Obtenga la biblioteca de cliente de WebSphere MQ de su instalación de WebSphere MQ.

Para configurar y usar recursos de conexión de MQ, así como el transporte de MQ en Oracle Service Bus, es necesario que agregue la biblioteca de cliente de MQ a la classpath. Una opción es copiar la biblioteca com.ibm.mq.jar en el directorio NOMBRE_DOMINIO/lib, donde NOMBRE_DOMINIO corresponde al nombre del directorio donde se encuentra su dominio, por lo general MW_HOME\user_projects\domains\NOMBRE_DOMINIO.

Para agregar un recurso de conexión MQ

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Seleccione Explorador de Proyectos y seleccione a continuación el proyecto o la carpeta a donde desee agregar el recurso de conexión de MQ. Se muestra la página Vista de Carpetas/Proyectos.
  3. En la lista desplegable Crear Recurso, seleccione Conexión de MQ bajo Utilidad para abrir la página Crear un Nuevo Recurso de Conexión de MQ.

    4. Especifique los parámetros de conexión que se muestran en la Tabla 8-2.

    Tabla 8-2 Parámetros de Conexión de MQ
    Propiedad
    Para crear o editar...
    Nombre del Recurso
    Introduzca un nombre único para este recurso de conexión de MQ.

    Siga las Restricciones de Nomenclatura de recursos para obtener ayuda con la nomenclatura.
    Descripción del Recurso
    Introduzca una descripción para el recurso de conexión de MQ.
    Tipo de Conexión
    Seleccione uno de los siguientes modos para conectarse con el gestor de colas de MQ:
    • modo de tcp: utilice TCP/IP para conectarse a un gestor de colas que no resida en la misma máquina que Oracle Service Bus.
    • modo de enlace: Utilice el modo de enlace para conectarse con un gestor de colas que se encuentre en la misma máquina que Oracle Service Bus.
    Nombre del Host de MQ
    Sólo para conexiones en modo tcp:
    Introduzca el nombre de host del gestor de colas de MQ.
    Número de Puerto de MQ
    Sólo para conexiones en modo tcp:
    Introduzca el número de puerto del listener de gestor de colas de MQ.
    Nombre del Gestor de Colas de MQ
    Introduzca el nombre del gestor de colas de MQ al que se quiera conectar.
    CCSID de Gestor de Colas
    Sólo para conexiones en modo tcp:
    El identificador de juego de caracteres codificado (CCSID) que se empleará al establecerse una conexión. El CCSID se utiliza principalmente para garantizar soporte internacional.
    Para obtener más información, consulte WebSphere MQ Fundamentals (Aspectos Fundamentales de WebSphere MQ) en http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
    Nombre del Canal del Gestor de Colas de MQ
    Sólo para conexiones en modo tcp:
    Introduzca el nombre del canal de conexión del servidor del gestor de colas.
    SSL Necesario
    Para conexiones en modo tcp:
    Seleccione la casilla de control para utilizar SSL para enviar mensajes. Sólo se activará el SSL de servidor cuando la opción SSL Bidireccional Necesario no esté seleccionada.
    Serie Cipher
    Esta opción sólo está disponible cuando está seleccionada la casilla de control SSL Necesario.
    Seleccione el algoritmo Serie Cipher que debe usar SSL.
    El algoritmo Serie Cipher se utiliza para cifrar y descifrar mensajes entre el servidor de WebSphere MQ y el cliente de WebSphere MQ. Es por lo tanto necesario especificar un algoritmo Serie Cipher cuando se use SSL para comunicarse con un servidor de WebSphere MQ.
    SSL Bidireccional Necesario
    Esta opción sólo está disponible cuando está seleccionada la casilla de control SSL Necesario.
    Seleccione esta casilla de control para activar la autenticación SSL de cliente y de servidor.
    Referencia al Proveedor de Claves de Servicio
    Si selecciona SSL Bidireccional Necesario, deberá proporcionar una referencia al proveedor de claves de servicio para obtener el almacén de claves adecuado y la información de almacenamiento protegido.
    Introduzca la ruta de acceso (proyecto/carpeta) y el nombre de un proveedor de claves de servicio, o haga clic en Examinar para seleccionar un proveedor en la página Seleccionar Proveedor de Claves de Servicio.
    Referencia a la Cuenta de Servicio Estática
    Sólo para conexiones en modo tcp:
    Este valor es necesario para poder autenticar el nombre de usuario y la contraseña. Introduzca la ruta de acceso (proyecto/carpeta) y el nombre de una cuenta de servicio estática, o haga clic en Examinar para seleccionar cuentas de servicio desde un explorador.
    Versión de WebSphere MQ
    Seleccione la versión de WebSphere MQ:
    • 5.3
    • 6.0
    • 7.0

    Nota: IBM ya no soporta las versiones 5.3 y 6.0. Estas versiones son anticuadas en Oracle Service Bus.
    Tamaño del Pool de Conexiones de MQ
    Introduzca el tamaño del pool de conexiones de MQ.
    Timeout de Conexión de MQ
    Introduzca el intervalo de tiempo, en segundos, transcurrido el cuál se destruirán las conexiones no utilizadas. El valor por defecto es de 1800 segundos.
    Espera Máxima de Conexión de MQ
    Introduzca la espera máxima, en milisegundos, correspondiente al tiempo que se debe esperar a que se establezca una conexión. Si no se consigue establecer conexión dentro del intervalo de tiempo indicado, Oracle Service Bus devolverá una excepción. El valor por defecto es 3000 milisegundos.

  4. Haga clic en Guardar. Se creará el recurso Conexión de MQ y se guardará en la sesión actual.
  5. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.
