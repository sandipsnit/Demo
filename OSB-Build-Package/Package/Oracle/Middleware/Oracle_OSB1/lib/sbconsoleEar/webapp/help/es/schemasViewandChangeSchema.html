Edición de Esquemas XML

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Localice el esquema XML tal y como se describe en Localización de Esquemas XML.
  3. Haga clic en el nombre del esquema XML. La página Ver Detalles de Esquema XML muestra la información que se ve en la Tabla 4-2, la Tabla 4-3 y la Tabla 4-4.

    4. Tabla 4-2 Detalles de Esquemas XML 
    Propiedad
    Descripción
    Modificado por
    Usuario que ha creado este esquema XML o lo ha importado a la configuración.
    Fecha de Modificación
    Fecha y hora en la que el usuario ha creado este esquema XML o lo ha importado a la configuración. Haga clic en el enlace de fecha y hora para ver el historial de cambios de este recurso. Consulte Página Ver Historial de Cambios.
    Referencias
    Número de objetos a los que hace referencia este esquema XML. Si existen estas referencias, haga clic en el enlace numérico para ver una lista de los objetos. Consulte Visualización de Referencias a Recursos.
    Con Referencia de
    El número de objetos que hacen referencia a este esquema XML. Si existen tales referencias, haga clic en el enlace numérico para ver una lista con los objetos. Por ejemplo, si un WSDL concreto hace referencia a este esquema XML, el WSDL aparecerá en la lista como un referencia cuando haga clic en el enlace. Consulte Visualización de Referencias a Recursos.
    Descripción
    Descripción del esquema XML, si la hay.

    Por defecto, la vista del esquema XML es estructural e incluye la información que se ve en la Tabla 4-3.

    Tabla 4-3 Detalles de Estructuras XML
    Propiedad
    Descripción
    Espacio de Nombres de Destino
    El espacio de nombres que se utiliza para cualificar cualquier definición incluida en el esquema XML.
    Definiciones de Esquemas XML
    Los atributos y los grupos asociados al esquema XML:
    • Tipos de Esquemas XML
    • Elementos de Esquema XML
    • Importaciones de Esquemas XML
    • Inclusiones/Redefiniciones de Esquema XML
    Estado de Esquema
    Estado del esquema XML:
    • Válido
      Los esquemas XML aparecen como válidos cuando aparecen especificadas todas las ubicaciones de los esquemas XML dentro del esquema XML actual. Además, también se especificarán todas las ubicaciones de los esquemas XML anidados (esquemas XML que a su vez incluyen los esquemas XML incluidos).
    • No Válido
      Los esquemas XML aparecen como no válidos cuando una o varias de las ubicaciones del esquema XML han quedado sin especificar. Además, es posible que no se haya especificado la ubicación de uno o varios esquemas XML anidados (esquemas XML que a su vez incluyen los esquemas XML). Para resolver un esquema XML no válido, haga clic en Editar Referencias. Consulte Resolución de Referencias de Esquemas XML No Resueltas.

  4. Haga clic en Vista de Texto para mostrar una vista de texto de los detalles del esquema XML. La vista de texto incluye la información que se ve en la Tabla 4-4.

    5. Tabla 4-4 Detalles de Texto de Esquemas XML
    Propiedad
    Descripción
    Espacio de Nombres de Destino
    El espacio de nombres que se utiliza para cualificar cualquier definición incluida en el esquema XML.
    Esquema XML
    El texto del esquema XML.

  5. Para hacer cambios en los campos, haga clic en Editar. Consulte Adición de Esquemas XML para ver las descripciones de estos campos.
  6. Realice las ediciones adecuadas.
  7. Haga clic en Guardar para confirmar las actualizaciones en la sesión actual.
  8. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.
