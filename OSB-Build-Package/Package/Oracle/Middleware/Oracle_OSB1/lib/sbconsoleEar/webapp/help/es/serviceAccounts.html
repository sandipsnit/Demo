Cuentas de Servicio

Las cuentas de servicio proporcionan un nombre de usuario y una contraseña que los servicios de proxy y de negocio utilizan para las tareas de autenticación saliente o de autenticación en un recurso local o remoto, como un servidor FTP o un servidor JMS. Por ejemplo, si se requiere un servicio de negocio para poder proporcionar un nombre de usuario y una contraseña para hacer una autenticación en el nivel de transporte con un servicio web, podrá crear una cuenta de servicio que especifique el nombre de usuario y la contraseña y, a continuación, configurar el servicio de negocio para que incluya las credenciales de la cuenta de servicio en sus solicitudes salientes.

Los nombres de usuario y las contraseñas que introduzca en las cuentas de servicio se utilizarán en las tareas de autenticación saliente o para proporcionar credenciales a recursos locales o remotos. Los nombres de usuario y las contraseñas que introduzca en el módulo Configuración de Seguridad de la consola de administración de Oracle Service Bus se utilizarán para las tareas de autenticación entrante y para las solicitudes administrativas de autenticación. Consulte Especificación de Cuentas de Servicio.
