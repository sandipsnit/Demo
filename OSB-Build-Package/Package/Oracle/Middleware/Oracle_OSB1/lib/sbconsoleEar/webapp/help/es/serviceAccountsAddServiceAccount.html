Adición de Cuentas de Servicio

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Seleccione Explorador de Proyectos y elija el proyecto o carpeta donde desea agregar la cuenta de servicio. Aparece la página Vista de Proyectos/Carpetas.
  3. Desde la lista desplegable Crear Recurso, seleccione Cuenta de Servicios para mostrar la página Crear una Nueva Cuenta de Servicio.
  4. En el campo Nombre del Recurso, introduzca un nombre único para esta cuenta de servicio.

    Siga las Restricciones de Nomenclatura de Recursos para obtener ayuda con la nomenclatura.
  5. En el campo Descripción del Recurso, introduzca una descripción para la cuenta de servicio.
  6. En Tipo de Recurso, realice una de las acciones siguientes:
    • Para crear una cuenta de servicio que proporcione los nombres de usuario y las contraseñas que recibirá de las solicitudes de cliente entrantes, seleccione Transferencia.
    • Para crear una cuenta de servicio que proporcione un nombre de usuario y una contraseña que se guardará con la configuración de la cuenta de servicio, seleccione Estático.
    • Para crear una cuenta de servicio que asigne el nombre de usuario de uno o varios clientes autenticados a nombres de usuario y contraseñas que usted especifique, seleccione Asignación.
  7. Dependiendo del tipo de recurso que haya seleccionado en el paso 6, proceda con uno de los pasos a continuación descritos en la Tabla 16-2.

    8. Tabla 16-2 Opciones de Tipo de Recursos 
    Tipo de Recurso Seleccionado
    Realice Estos Pasos
    Transferencia
    Haga clic en Último.
    Estático
    1. Haga clic en Siguiente.
    2. Introduzca el nombre de usuario y la contraseña en los campos Usuario, Contraseña y Confirmar Contraseña.
    3. Haga clic en Último.
    Asignación
    Para crear una cuenta de servicio que asigne el nombre de usuario de uno o varios clientes a nombres de usuario y contraseñas que usted especifique, realice lo siguiente:
    1. Haga clic en Siguiente.
    2. En la tabla Introducir Usuario Remoto Autorizado, realice lo siguiente:
      1. En los campos Nombre de Usuario Remoto, Contraseña y Confirmar Contraseña, introduzca el nombre de usuario y la contraseña que desee enviar en las solicitudes salientes.
      2. Haga clic en Agregar.

        3. La asignación del usuario se agrega a la tabla Usuarios Remotos.

      3. (Opcional) Agregue más usuarios remotos a la tabla Introducir Usuario Remoto Autorizado.
    3. Haga clic en Siguiente.
    4. Para asignar clientes autorizados a nombres de usuarios remotos y contraseñas, realice lo siguiente en la tabla Introducir Usuario Local Autorizado:
      1. En el campo Nombre de Usuario Local, introduzca el nombre que identifique a un cliente que se haya autenticado en su solicitud saliente.

        2. Si aún no ha agregado este usuario en el módulo Configuración de Seguridad de la consola de administración de Oracle Service Bus, hágalo antes de utilizar esta asignación en un entorno de tiempo de ejecución. Consulte Adición de Usuarios. Oracle Service Bus le permite crear una asignación para un usuario local que no exista, si bien esta asignación no coincidirá nunca con un usuario autenticado y no se usará nunca.

      2. En la lista Nombre de Usuario Remoto, seleccione el nombre de usuario que desee enviar en las solicitudes salientes correspondientes al usuario autenticado que haya especificado en el campo Nombre de Usuario Local.
      3. Haga clic en Agregar.
    5. Para asignar clientes anónimos a nombres de usuario remotos, realice lo siguiente:
      1. Seleccione la casilla de control Asignar Peticiones Anónimas.
      2. En la lista Seleccionar Usuario Remoto, seleccione el nombre de usuario que quiera enviar en las solicitudes salientes correspondientes a todos los usuarios anónimos.
    6. Haga clic en Último.

  8. Haga clic en Guardar. Se creará la cuenta de servicio y se guardará en la sesión actual.
  9. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.
