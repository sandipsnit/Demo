Edición de Cuentas de Servicio

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Localice la cuenta de servicio tal y como se describe en Localización de Cuentas de Servicio.
  3. Haga clic en el nombre de la cuenta de servicio. La página Ver Detalles de Cuenta de Servicio muestra la información que se ve en la Tabla 16-3.

    4. Tabla 16-3 Detalles de Cuentas de Servicio
    Propiedad
    Descripción
    Modificado por
    El usuario que ha creado esta cuenta de servicio o la ha importado a la configuración.
    Fecha de Modificación
    Fecha y hora en la que el usuario ha creado esta cuenta de servicio o la ha importado a la configuración. Haga clic en el enlace de fecha y hora para ver el historial de cambios de este recurso. Consulte Página Ver Historial de Cambios.
    Referencias
    Número de objetos a los que hace referencia esta cuenta de servicio. Si existen estas referencias, haga clic en el enlace numérico para ver una lista de los objetos. Consulte Visualización de Referencias a Recursos.
    Con Referencia de
    El número de objetos que hacen referencia a esta cuenta de servicio. Si existen estas referencias, haga clic en el enlace numérico para ver una lista con los objetos. Por ejemplo, si ha seleccionado esta cuenta de servicio como una cuenta de servicio JMS en un servicio de proxy con un protocolo de transporte de JMS, el servicio de proxy aparecerá como una referencia cuando se haga clic en el enlace. Consulte Visualización de Referencias a Recursos.
    Descripción
    Descripción de la cuenta de servicio, si existe.

  4. Para hacer cambios en los campos, haga clic en Editar. Consulte Adición de Cuentas de Servicio para ver las descripciones de estos campos.

    5. No puede cambiar el campo Nombre del Recurso.

  5. Haga clic en Guardar para confirmar las actualizaciones en la sesión actual.
  6. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.
Nota: Si la cuenta de servicio que ha modificado se utiliza para autenticarse con un servidor JMS de WebLogic, es posible que el servidor JMS tarde hasta 60 segundos en reconocer la modificación. Por defecto, el JMS de WebLogic Server comprueba los permisos de cada destino cada 60 segundos. Para cambiar esta configuración, modifique el comando de inicio de WebLogic Server de modo que establezca la siguiente propiedad del sistema en la frecuencia (en segundos) con la que desea que el JMS de WebLogic Server compruebe los permisos: weblogic.jms.securityCheckInterval
Un valor de 0 (cero) en esta propiedad garantiza que se realice una comprobación por cada acción de enviar, recibir y getEnumeration que se realice en un recurso de JMS.
