Especificación de Cuentas de Servicio

Puede usar la misma cuenta de servicio para varios servicios de negocio y varios servicios de proxy. Para especificar el nombre de usuario y la contraseña que suministrará una cuenta de servicio, puede recurrir a cualquiera de las siguientes técnicas:

  • Estático

    • Requiere que guarde un nombre de usuario y una contraseña con la configuración de la cuenta de servicio. La cuenta de servicio codifica el nombre de usuario y la contraseña en la solicitud saliente.

  • Traspaso de nombre de usuario y contraseña

    • Hace que la cuenta de servicio proporcione el nombre de usuario y la contraseña que recibirá de una solicitud de cliente entrante. Por ejemplo, si una solicitud entrante HTTP BASIC contiene "pat" y "contraseñadepat" como nombre de usuario y contraseña, la cuenta de servicio codificará "pat" y "contraseñadepat" en la solicitud saliente.

    Ya que esta técnica exige que las solicitudes del cliente incluyan nombres de usuario y contraseñas sin cifrado, sólo se podrá aplicar cuando las solicitudes de cliente utilicen el protocolo HTTP BASIC, un sistema de autenticación donde el elemento de nombre de usuario de seguridad para servicios web lleva la contraseña sin cifrar, o un elemento de nombre de usuario y contraseña personalizado.

    Oracle recomienda utilizar esta técnica sólo cuando Oracle Service Bus y el punto final pertenezcan al mismo dominio de autenticación. Utilice esta técnica por ejemplo cuando direccione mensajes dentro de una única organización y tanto Oracle Service Bus como el consumidor de mensajes se autentiquen frente a un servidor LDAP común.

    Esta técnica se enfrenta a las siguientes restricciones:

    • No se puede usar en solicitudes salientes que autentiquen Oracle Service Bus con un servidor local o remoto o con un recurso del sistema, como por ejemplo un servidor FTP o un servidor JMS.
    • No se puede usar con la función de XQuery fn-bea:lookupBasicCredentials. Para obtener más información, consulte la sección sobre implantación de XQuery en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG444.
      • Nota: Si su proxy funciona como intermediario WSS activo, podrá usar WS-Security para cifrar un elemento nombre de usuario de WS-Security o un nombre de usuario o contraseña personalizados. En esta instancia, el traspaso del nombre de usuario o la contraseña funciona porque el proxy cifrará primero la solicitud para después tener acceso al nombre de usuario y a la contraseña no cifrados.
  • Asignación de usuarios

    • Exige que correlacione (asigne) el nombre de usuario resultado de autenticar una solicitud entrante de un cliente (el nombre de usuario local) con el nombre de usuario y la contraseña que se especifique (el nombre de usuario y la contraseña remotos). Cuando la cuenta de servicio recibe una solicitud de un cliente autenticado que ha sido asignado, proporcionará el nombre de usuario remoto y la contraseña adecuados para la solicitud saliente del servicio de proxy o del servicio de negocio.

    Si el cliente se autentica tanto en el nivel de transporte como en el de mensajes, la cuenta de servicio asignará el nombre de usuario en el nivel de mensajes al nombre de usuario remoto y la contraseña.

    También puede asignar un nombre de usuario anónimo a un nombre de usuario remoto y una contraseña.

    Esta técnica se enfrenta a las siguientes restricciones:

    • No se puede usar en solicitudes salientes que autentiquen Oracle Service Bus con un servidor local o remoto o con un recurso del sistema, como por ejemplo un servidor FTP o un servidor JMS.
    • No se puede usar con la función de XQuery fn-bea:lookupBasicCredentials. Para obtener más información, consulte la sección sobre implantación de XQuery en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG444.
