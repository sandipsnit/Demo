Selección de Proyectos y Recursos que Importar

  1. Si está importando un archivo de recurso JAR y quiere asegurarse de que se importen todos los recursos asociados, seleccione Incluir Dependencias.
    2. Nota: Incluir Dependencias no se puede seleccionar cuando se está importando un archivo JAR de un proyecto completo o un archivo de configuración anterior a la versión 2.6.
  2. Haga clic en el icono de Abrir para mostrar las opciones de Configuración Avanzada a través de las cuales preservar valores en el sistema de importación.
  3. Si va a volver a importar un mismo recurso pero desea proteger todas las personalizaciones que se hayan hecho en el recurso existente, seleccione Preservar Variables de Entorno.
    4. Nota: No se podrá preservar la personalización de referencias dentro de un recurso existente. Consulte Creación de Archivos de Personalización.
  4. Cuando vaya a importar el recurso de configuración operativa global u otros recursos como servicios de negocio o de proxy que contengan configuraciones operativas en el nivel de servicios, y quiera preservar los valores operativos en el sistema de importación, seleccione Preservar Valores Operativos.
  5. Para evitar sobrescribir la configuración existente relacionada con la seguridad en recursos que se están actualizando, seleccione las opciones siguientes que estime oportunas:
    1. Preservar Configuración de Política y Seguridad: Preserva la configuración de seguridad (excluyendo las políticas de control de acceso) y las referencias a las políticas de WS enlazadas directamente al servicio (en lugar de enlazadas a WSDL).

      2. Para obtener más información, consulte la sección sobre seguridad en Oracle Fusion Middleware Developer's Guide for Oracle Service Bus (Guía del Desarrollador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472.

    2. Preservar Credenciales: Preserva las credenciales PKI en los proveedores de claves de servicio, nombre de usuario y contraseñas en las cuentas de servicio y el nombre de usuario y credenciales de contraseña en los servidores SMTP, proveedores de JNDI y registros UDDI.
    3. Preservar Políticas de Control de Acceso: Preserva políticas de control de acceso.

      4. La información del resumen de recursos se muestra en la Tabla 28-1.

      Tabla 28-1 Información de Recursos
      Propiedad
      Descripción
      Nombre
      El nombre del proyecto.
      Los recursos globales, como información sobre registros UDDI, servidores SMTP, servidores proxy, proveedores de JNDI y la configuración operativa global aparecen bajo el proyecto Sistema, en sus respectivas carpetas.
      Tipo
      El tipo de recurso, Proyecto.
      Amplíe la carpeta Proyecto para ver recursos y anular la selección de recursos.
      Operaciones
      Muestra --- o Supresiones.
      Amplíe la carpeta Proyecto para ver las operaciones (creación, actualización y supresión) que se aplicarán en los recursos.
      Para evitar que se supriman recursos, anule la selección de los mismos. Los recursos están programados para suprimirse sólo cuando el JAR que se está importando es un JAR de proyecto completo y hay recursos de proyecto en el sistema de importación que no están presentes en el archivo JAR importado.
      Referencias
      Amplíe la carpeta Proyecto para mostrar el número de objetos al que hace referencia cada recurso.

  6. Seleccione los objetos sobre los que quiera aplicar la acción (creación, actualización o supresión).
    1. Para mostrar cada uno de los objetos contenidos en el archivo JAR, amplíe la carpeta del archivo. La casilla de control asociada con cada objeto aparecerá seleccionada.

      2. Los detalles de archivos de recursos a continuación, visibles para cada objeto, se muestran en la Tabla 28-2.

      Tabla 28-2 Detalles de Archivos de Recursos 
      Propiedad
      Descripción
      Nombre
      El nombre y la ruta de acceso del recurso. La ruta de acceso corresponde al nombre del proyecto y el nombre de la carpeta en la que reside el recurso.
      Tipo
      Los tipos de recursos de Oracle Service Bus.
      Nota: Si un recurso tiene datos de seguridad sensibles, aparecerá un icono de Cifrado.
      Atención: Si selecciona una cuenta de servicio o un proveedor de claves de servicio, es posible que sobrescriba datos de seguridad que ya existan, a menos que especifique preservar valores de la configuración de seguridad durante el proceso de importación. ConsulteImportación de Cuentas de Servicio o Proveedores de Claves de Servicio.
      Operaciones
      Crear, actualizar o suprimir.
      Los recursos están programados para suprimirse sólo cuando el JAR que se está importando es un JAR de proyecto completo y hay recursos de proyecto en el sistema de importación que no están presentes en el archivo JAR importado.
      Referencias
      El numero de objetos al que hace referencia este recurso.

    2. Desactive las casillas de control junto a los recursos que no quiera importar.
  7. Haga clic en Importar.

Después de Terminar

De ser oportuno, continúe con la sección Especificación de la Configuración de Seguridad en la Importación. O bien, continúe con Revisión del Resumen de Importación.
