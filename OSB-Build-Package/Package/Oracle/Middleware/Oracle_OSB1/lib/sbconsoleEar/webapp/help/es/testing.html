Consola de Prueba

La consola de prueba de Oracle Service Bus es un entorno de prueba basado en explorador que se utiliza para validar y probar el diseño del sistema. Puede configurar el objeto de prueba (un servicio proxy, un servicio de negocio o un recurso), ejecutar la prueba y ver los resultados en la consola de prueba. En algunos casos, podrá también rastrear el código y examinar el estado del mensaje en puntos de rastreo concretos. Para obtener más información acerca de la consola de prueba, consulte la sección sobre cómo usar la consola de prueba en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG357.

Sólo pueden usar la consola de prueba los usuarios con rol IntegrationAdmin e IntegrationDeployer. Para obtener más información acerca de los roles, consulte la sección sobre roles en Oracle Service Bus en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG463.

Nota: si recibe un error que indica que el servicio de consola de prueba no se está ejecutando, configure la dirección de recepción del servidor de administración en una dirección específica válida, como un host local. En la consola de Oracle WebLogic Server, vaya a Entorno > Servidores > nombre_servidor_administración > Configuración > General para definir la dirección de recepción.
