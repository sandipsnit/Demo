Prueba de Servicios de Proxy

Puede probar los siguientes tipos de servicios de proxy: Cualquier XML, cualquier SOAP, mensajes, XML y SOAP. Puede probar los servicios de proxy SOAP con políticas de Seguridad de Servicio Web (WSS). Consulte la sección sobre seguridad del servicio web en Configuración de Datos de Prueba de Servicios de Proxy.

Nota: Cuando la consola de prueba llama a un proxy con autenticación de tokens personalizados HTTP, no se realiza la comprobación de autenticación.
Atención: La realización de pruebas de servicios de proxy con la opción de llamada directa activada se saltará algunos pasos importantes de seguridad, como es el control de acceso. Oracle le recomienda que no utilice el servicio de prueba en sistemas de producción.
  1. Haga clic en Activar en Centro de Cambios para activar la función de prueba en la consola de administración de Oracle Service Bus.

    2. Puede probar los servicios de proxy desde el Explorador de Recursos o el Explorador de Proyectos.

  2. Seleccione Explorador de Recursos > Servicios de Proxy para mostrar la página Resumen de Servicios de Proxy.
  3. En Acciones, haga clic en el icono Iniciar Consola de Prueba asociado al servicio de proxy que desea probar.

    4. La consola de prueba abre la página Prueba de Servicios de Proxy. Por ejemplo, utilizando los ejemplos que se proporcionan con el producto, haga clic en el icono asociado al servicio de proxy LoanGateway1.

    Nota: En un dominio en cluster, no puede utilizar la consola de prueba para probar ningún servicio de negocio configurado ni servicio de proxy que se direcciona a un servicio de negocio.
  4. Para los servicios SOAP y XML, seleccione la operación WSDL que desee probar.
  5. Configure los datos de prueba para el servicio de proxy. Deben ser los datos que el servicio de proxy espera del cliente.

    6. Por defecto, ambas opciones de configuración de la prueba,Llamada Directa e Incluir Rastreo, están activadas. Puede desactivar la opción Llamada Directa, que desactivará asimismo la opción Incluir Rastreo. Al hacerlo, la prueba se realiza con el método de llamada indirecta por el cual el mensaje se envía a través del nivel de transporte.

    Puede utilizar la opción de Llamada Directa (dejar seleccionada la opción Llamada Directa) y desactivar el rastreo de ejecución; para ello, sólo tiene que desactivar la casilla de control Incluir Rastreo.

    Para ver el rastreo de mensajes, desactive Llamada Directa.

    Nota: para ver el rastreo en el archivo log o en la salida estándar (consola del servidor), el registro de Oracle WebLogic Server debe estar definido en los siguientes niveles de gravedad:

Gravedad Mínima que Registrar: Información
Archivo Log: Información
Salida Estándar: Información

Para obtener información sobre la definición de niveles de gravedad de log, consulte la sección sobre el uso de los niveles de gravedad de en Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server (Configuración de Archivos Log y Filtrado de Mensajes de Log de Oracle Fusion Middleware para Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

  1. Haga clic en Ejecutar. La página Prueba de Servicios de Proxy muestra los resultados. Para obtener información sobre la interpretación de los resultados de la prueba, consulte Visualización de Resultados de la Prueba de Servicios de Proxy.
  2. Para volver a ejecutar la prueba, haga clic en Atrás. Repita del paso 5 en adelante.
