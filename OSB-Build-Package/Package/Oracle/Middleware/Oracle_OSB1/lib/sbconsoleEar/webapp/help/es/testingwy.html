Prueba de Servicios de Negocio

Puede probar los siguientes tipos de servicios de negocio: Cualquier XML, cualquier SOAP, mensajes, XML y SOAP. Puede probar los servicios de negocio de SOAP con las políticas de seguridad de servicio web. Para obtener más información, consulte la sección sobre la prueba de servicios con seguridad de servicio web en Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus (Guía del Administrador de Oracle Fusion Middleware para Oracle Service Bus) en http://www.oracle.com/pls/as111170/lookup?id=OSBAG384.

Nota: En un dominio en cluster, no puede utilizar la consola de prueba para probar ningún servicio de negocio configurado ni servicio de proxy que se direcciona a un servicio de negocio.
  1. Haga clic en Activar en Centro de Cambios para activar la función de prueba en la consola de administración de Oracle Service Bus.
  2. Seleccione Explorador de Recursos > Servicios de Negocio para mostrar la página Resumen de Servicios de Negocio.
  3. En Acciones, haga clic en el icono Iniciar Consola de Prueba asociado al servicio que desee probar. La consola de prueba abre la página Prueba de Servicios de Negocio. Por ejemplo, utilizando los tutoriales que se proporcionan con el producto, haga clic en el icono asociado a loanSaleProcessor.
  4. Para los servicios SOAP y XML, seleccione la operación WSDL que desee probar.
  5. Configure los datos de prueba para el servicio de negocio (los datos de entrada deben ser el mensaje que envía el servicio de proxy al servicio de negocio). Las opciones Llamada Directa e Incluir Rastreo que están disponibles para las pruebas de los servicios de proxy no están disponibles para los servicios de negocio. Por defecto, los servicios de negocio se prueban utilizando la opción Llamada Directa, lo que quiere decir que los mensajes se transfieren por la capa de transporte.

    6. Para ver el rastreo de mensajes, desactive Llamada Directa.

    Nota: para ver el rastreo en el archivo log o en la salida estándar (consola del servidor), el registro de Oracle WebLogic Server debe estar definido en los siguientes niveles de gravedad:

Gravedad Mínima que Registrar: Información
Archivo Log: Información
Salida Estándar: Información

Para obtener información sobre la definición de niveles de gravedad de log, consulte la sección sobre el uso de los niveles de gravedad de en Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server (Configuración de Archivos Log y Filtrado de Mensajes de Log de Oracle Fusion Middleware para Oracle WebLogic Server), en http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

  1. Haga clic en Ejecutar.

    2. La página Prueba de Servicios de Negocio muestra los resultados. Para obtener más información, consulte Visualización de Resultados de la Prueba de Servicios de Negocio.

Prueba de Anexos en Servicios de Negocio

En esta sección se describe la forma de utilizar la consola de prueba para probar los anexos de los servicios de negocio.

Con fines de ejemplo, esta sección asume que un servicio de negocio HTTP basado en WSDL utiliza una operación "submitAttachment" para enviar un archivo ZIP como anexo en un mensaje de SOAP. Modifique la configuración según corresponda para los diferentes tipos de anexos binarios.

    1. En la consola de prueba para el servicio de negocio, vaya a la página XML e introduzca el siguiente elemento Envelope de SOAP para el mensaje de texto:
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. En la sección Anexo de la consola de prueba:
      1. Tipo de Contenido: aplicación/zip (como se define en el WSDL)
      2. Identificador de Contenido: zipFile (debe coincidir con el valor de href="cid:zipFile" en el mensaje de texto)
      3. Archivo: Haga clic en Examinar y seleccione el archivo ZIP que haya especificado en el elemento Envelope de SOAP.
      4. Haga clic en Agregar.

    2. Haga clic en Ejecutar para enviar el mensaje de prueba con el anexo.

Para confirmar que el anexo se ha enviado correctamente, busque en la consola del servidor un mensaje similar al siguiente, que registra la operación submitAttachment:

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
