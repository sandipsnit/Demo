Rastreo de Servicios de Proxy

El rastreo se activa cuando se prueban los servicios de proxy utilizando una llamada directa. La casilla de control Incluir Rastreo se activa por defecto con la opción Llamada Directa. Si no desea que se realice un rastreo, desactive la casilla de control Incluir Rastreo. Cuando se activa, los resultados de la prueba incluyen los detalles del rastreo. Utilice el rastreo para hacer un seguimiento de los problemas del sistema y aislarlos para su corrección. La información de rastreo muestra la ruta de acceso tomada a través de las rutas de acceso de solicitud y respuesta del código.

  1. Haga clic en Activar en Centro de Cambios para activar la función de prueba en la consola de administración de Oracle Service Bus.
  2. Seleccione Explorador de Recursos > Servicios de Proxy para mostrar la página Resumen de Servicios de Proxy.
  3. En Acciones, haga clic en el icono Iniciar Consola de Prueba asociado al servicio que desee probar. La consola de prueba abre la página Prueba de Servicios de Proxy.
  4. Configure los datos de prueba del servicio de proxy. Las opciones Llamada Directa e Incluir Rastreo deben estar seleccionadas para activar el rastreo. Consulte Configuración de Datos de Prueba de Servicios de Proxy.
  5. Haga clic en Ejecutar. La página Prueba de Servicios de Proxy muestra los resultados de la prueba para el servicio y la información de rastreo.
  6. Desplácese hacia abajo hasta la sección Rastreo de Llamada.

    7. Esta sección muestra una representación gráfica del flujo de mensajes. Puede rastrear el mensaje a través del servicio y ver el estado del mensaje en los puntos del rastreo previamente seleccionados. Los puntos de rastreo se definen automáticamente.

  7. Haga clic en el signo + (más) para ampliar el flujo de mensajes y ver más detalles.

    8. Mientras visualiza el rastreo también puede ver el flujo de mensajes en la consola de administración de Oracle Service Bus. Esto le permite relacionar el rastreo con las etapas y acciones del flujo de mensajes. Puede modificar el flujo de mensajes y volver a ejecutar el rastreo para ver la salida.
