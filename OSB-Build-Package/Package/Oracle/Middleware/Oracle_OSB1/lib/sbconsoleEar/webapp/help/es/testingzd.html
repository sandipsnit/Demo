Prueba de Transformaciones de XSLT

Las XSLT (Transformaciones de Lenguaje de Hoja de Estilo Extensible) describen las asignaciones de XML a XML en Oracle Service Bus. Para probar un recurso de XSLT debe proporcionar el documento XML de entrada y la consola de prueba devuelve el documento XML de salida. Las transformaciones de XSLT pueden tener parámetros adicionales con nombre. En la página de configuración se muestran todos los parámetros necesarios para la transformación. Puede sustituir los valores por defecto disponibles.

En la Tabla 32-6 se describen las opciones de configuración de XSLT.

Entrada XML
La entrada XML puede basarse en archivo o en texto. La referencia a un archivo para la entrada tiene prioridad sobre la entrada de texto. Busque y seleccione el archivo que desee utilizar en la prueba. Es necesaria una entrada XML.
<nombre_parámetro> ([] como XML)
nombre_parámetro es un parámetro XSLT con nombre.
Hay dos tipos de entrada: XML y primitivo (cadena, entero, flotante, etc.). El tipo de entrada por defecto es la cadena. Active la casilla de control asociada al nombre de parámetro para identificar un parámetro del tipo XML.

Puede probar el tiempo de diseño o el tiempo de ejecución.

  1. Haga clic en Activar si desea probar el tiempo de ejecución. No active la sesión para probar el tiempo de diseño.
  2. Seleccione Explorador de Proyectos > XSLT para mostrar la página Resumen de XSLT.
  3. En Acciones, haga clic en el icono Iniciar Consola de Prueba asociado al recurso que desee probar. La consola de prueba abre la página Prueba de Recursos.
  4. Configure los datos de prueba para el recurso en la sección Entrada y Parámetros de la página. Para obtener más información, consulte la Tabla 32-6.
  5. Haga clic en Ejecutar. La página Prueba de Recursos mostrará los resultados.
  6. Para repetir la prueba, haga clic en Atrás. Puede cerrar la consola de prueba, modificar y volver a probar el recurso.
