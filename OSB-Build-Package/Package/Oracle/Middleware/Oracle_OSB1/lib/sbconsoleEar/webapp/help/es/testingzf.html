Uso del Editor de Expresiones y el Editor de Condición de XQuery

Utilice expresiones XQuery para crear el contenido de datos de las variables de contexto de mensaje (o parte de una variable de contexto de mensaje) durante la ejecución del flujo de mensajes. Puede utilizar la consola de prueba directamente en el Editor de Expresiones XQuery para probar la definición de la expresión.

De modo similar, utilice condiciones de XQuery para evaluar las condiciones booleanas del flujo de mensajes. Puede utilizar la consola de prueba directamente en el Editor de Condición de XQuery para probar la definición de la condición.

Un XQuery puede tomar varias entradas y devolver una salida. Cada entrada corresponde a una variable no enlazada de XQuery definida en XQuery. El valor de una entrada de XQuery puede ser un valor primitivo (cadena, entero, fecha, etc.), un documento XML o una secuencia de estos tipos. El valor de salida puede ser un valor primitivo (cadena, entero, fecha, etc.), un documento XML o una secuencia de estos tipos.

Nota: la consola de prueba no soporta secuencias en la entrada.

Esta sección contiene un campo de entrada para cada una de las variables no enlazadas de XQuery.
<nombre_parámetro> ([] como XML)
nombre_parámetro es el nombre de la variable no enlazada de XQuery correspondiente.
En la consola de prueba aparece un cuadro de edición de una sola línea si es de tipo simple. Si los datos son XML aparece un cuadro de edición de varias líneas. Se utiliza una entrada combinada (<nombre_parámetro> ([] como XML)) cuando es una variable sin tipo. Debe declarar el tipo de variable. Seleccione la casilla de control para identificar un parámetro de tipo XML.
Una entrada XML puede basarse en archivo o en texto. La referencia a un archivo para la entrada tiene prioridad sobre la entrada de texto. Busque y seleccione el archivo que desee utilizar en la prueba.
La entrada en la consola de prueba se presenta según el tipo para facilitar la comprensión del tipo de dato que debe introducir. El tipo por defecto es XML.

  1. Acceda a la consola de prueba cuando edite una acción en el flujo de mensajes de un pipeline.
    • Para acceder al Editor de Expresiones XQuery, consulte Creación y Edición de Expresiones XQuery y XPath en Línea.
    • Para acceder al Editor de Condición de XQuery, consulte Creación y Edición de Expresiones XQuery y XPath en Línea.

      • Aparece la página Expresión de Prueba de XQuery. En esta página se muestran todas las variables de entrada necesarias.

  2. Configure los datos de prueba para la XQuery en la sección Variables de la página. Para obtener más información, consulte la Tabla 32-8.
  3. Haga clic en Ejecutar. La página de prueba muestra los resultados.
  4. Una vez completada la prueba, puede hacer clic en Atrás para ejecutar una nueva prueba. Para ejecutar otra prueba después de hacer cambios en XQuery, debe cerrar y volver a abrir la consola de prueba para que se apliquen los cambios.
