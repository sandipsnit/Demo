Adición de Registros UDDI

Utilice esta página para agregar y configurar los registros UDDI. Después de configurar los registros, puede publicar en ellos los servicios de proxy de Oracle Service Bus o bien importar desde ellos los servicios de negocio que se utilizarán en un servicio de proxy de Oracle Service Bus.

Debe estar en una sesión activa para agregar un registro.

  1. Si aún no lo ha hecho, haga clic en Crear para crear una nueva sesión o en Editar para acceder a una sesión existente. Consulte Uso del Centro de Cambios.
  2. Seleccione Administración del Sistema > Registros UDDI.
  3. En el panel Registros UDDI, haga clic en Agregar Registro.
  4. Para configurar el registro, complete los siguientes pasos. Los asteriscos indican los campos necesarios. Consulte la Tabla 29-1 para obtener las descripciones de las propiedades que deben definirse al agregar un registro.
    1. En el campo Nombre, introduzca un nombre para el registro.

      Siga las Restricciones de Nomenclatura de Recursos para obtener ayuda con la nomenclatura.
    2. En el campo URL de Consulta, introduzca una URL de consulta con el formato: http://host:puerto/APPLICATION_SERVER_CONTEXT/uddi/consulta
    3. En el campo URL de Publicación, introduzca una URL de publicación con el formato http://host:puerto/APPLICATION_SERVER_CONTEXT/uddi/publicación
    4. En el campo URL de Seguridad, introduzca una URL de seguridad con el formato http://host:puerto/APPLICATION_SERVER_CONTEXT/uddi/seguridad
    5. En el campo URL de Suscripción, introduzca una URL de suscripción con el formato
      http://host:puerto/APPLICATION_SERVER_CONTEXT/uddi/suscripción

      6. El valor por defecto para APPLICATION_SERVER_CONTEXT en Installer es registry.

    6. En el campo Usuario, introduzca el nombre de usuario para conectarse a la consola de registro.
    7. En el campo Contraseña /(Confirmar Contraseña), introduzca la contraseña para conectarse a la consola de registro.
    8. Active la casilla de control Cargar tModel en el Registro para publicar tModels de Oracle Service Bus en el registro.

      9. Este campo sólo es necesario para publicar servicios de proxy en el registro.

    9. Active la casilla de control Activar Importación Automática para sincronizar automáticamente los servicios importados con el registro UDDI. Cualquier servicio que se importa con esta opción seleccionada se mantendrá sincronizado con el registro UDDI.
      10. Nota: La sincronización automática es un proceso en segundo plano; no se puede invertir con la función Deshacer de la sesión. La acción de deshacer un cambio de sincronización automática no es permanente, ya que el servicio se volverá a sincronizar en el siguiente ciclo de sincronización. Si desea que un servicio importado permanezca no sincronizado con el registro UDDI, debe desasociar este servicio para evitar que se produzcan más actualizaciones desde el registro. Consulte Desasociar Servicios.
  5. Haga clic en Validar para validar las URL que ha proporcionado.
  6. Haga clic en Guardar para configurar el registro con los valores proporcionados y guardarlo en la sesión actual.
  7. Para terminar la sesión y desplegar la configuración en tiempo de ejecución, haga clic en Activar en Centro de Cambios.
