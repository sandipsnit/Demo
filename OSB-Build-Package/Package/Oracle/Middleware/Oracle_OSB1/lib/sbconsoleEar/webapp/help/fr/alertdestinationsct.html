Ajout de destinations JMS

  1. Sélectionnez Envoyer une alerte vers une destination JMS dans la liste déroulante et cliquez sur Ajouter pour afficher la page Ajouter une destination JMS.
  2. Dans le champ URI de destination, entrez l'URI de la destination JMS au format suivant : jms://host:port/factoryJndiName/destJndiName.

    3. Ce champ est obligatoire.

    Remarque : lorsque Oracle WebLogic Server autorise les barres obliques dans les noms JNDI, tels que "myqueues/myqueue", les noms JNDI avec des barres obliques interfèrent avec le format URI requis par Oracle Service Bus, et vous ne pouvez pas utiliser ces noms. Pour passer outre ce problème, définissez un serveur étranger JMS et référencez-le dans l'URI. Pour plus d'informations, reportez-vous à Configurer les serveurs étrangers dans l'aide en ligne d'Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help à l'adresse http://www.oracle.com/pls/as111170/lookup?id=WLACH.

  3. Dans le champ Type de destination, sélectionnez File d'attente ou Rubrique.
  4. Dans le champ Type de message, sélectionnez Octets ou Texte.
  5. Dans le champ Encodage de la demande, indiquez la valeur d'encodage du jeu de caractères. La valeur par défaut est UTF-8.
  6. Cliquez sur Enregistrer. La destination est ajoutée à la liste des destinations JMS à inclure pour cette destination d'alertes et elle est enregistrée dans la session en cours.
