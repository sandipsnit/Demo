Page EJB - Configuration du transport

La page Créer/Modifier un service fonctionnel - EJB - Configuration du transport est une des pages permettant de créer et modifier des services fonctionnels, comme indiqué dans Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Utilisez cette page pour configurer les paramètres de transport d'un service fonctionnel utilisant le protocole de transport EJB. Le tableau 18-5 explique comment utiliser cette page.

Tableau 18-5 Page Créer/Modifier un service fonctionnel - EJB - Configuration du transport
Option
Procédure
Transmettre le sujet de l'appelant
Lorsque vous cochez cette case, Oracle Service Bus transmet le sujet authentifié à partir du service proxy lors de l'appel de l'EJB et lorsque aucun compte de service n'est configuré. Le champ Compte de service est désactivé lorsque cette option est sélectionnée.
Compte de service
Cliquez sur Parcourir et sélectionnez un compte de service dans la liste affichée. Si vous n'indiquez aucun compte de service, un sujet anonyme est utilisé. Cette option n'est pas disponible si vous choisissez Transmettre le sujet de l'appelant.

Pour plus d'informations :

  • Comptes de service
  • Ajout de comptes de service
Prend en charge la transaction
Cochez cette case pour indiquer la prise en charge de la transaction.
Fichier JAR client
Cliquez sur Parcourir et sélectionnez une ressource JAR de client EJB dans la liste affichée. Pour plus d'informations sur la création de ressources JAR, reportez-vous à fichiers JAR. Ce champ est obligatoire.
JAR convertisseur
Cliquez sur Parcourir et sélectionnez une ressource JAR de classe de convertisseur EJB dans la liste affichée. Pour plus d'informations sur les classes de convertisseur et ressources JAR de client EJB, reportez-vous à "EJB Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.
Interface locale
(EJB 2.1 uniquement) Sélectionnez l'interface EJBHome requise parmi les options fournies par le fichier JAR. Le nom JNDI dans cet exemple d'URI doit être associé à l'interface EJBHome sélectionnée. Si le type de l'EJB n'est pas celui requis ou que vous n'indiquez aucune interface EJBHome dans le fichier JAR client, Oracle Service Bus affiche un avertissement.
Interface distante
(EJB 2.1 uniquement) Ce champ est automatiquement rempli en fonction de la configuration de l'interface locale.
Interface fonctionnelle
(EJB 3.0 uniquement) Sélectionnez l'interface fonctionnelle dans le fichier JAR client à utiliser.
Espace de noms cible
Ce champ est rempli en fonction des informations fournies par le fichier JAR.
Style
Sélectionnez Encapsulé dans un document ou RPC selon vos exigences. Si le nombre et le type de données des paramètres figurant dans deux méthodes (ou plus) utilisées par l'EJB de la session sans conservation de statut (stateless) sont identiques, et que vous souhaitez que les opérations soient orientées document, vous devez indiquer qu'elles doivent être encapsulées dans un document.
Le style est un élément important car, lors du routage ou de la publication vers l'EJB, le contenu de $body doit correspondre au style. En outre, lors de l'appel d'un EJB, le style a une incidence sur le contenu du paramètre, en particulier lorsqu'il est encapsulé dans le document. Par ailleurs, un des modèles d'utilisation consiste à définir un service fonctionnel EJB, puis à créer un service proxy à l'aide du même WSDL que celui utilisé pour le routage vers l'EJB. Dans ce cas, vous devez prendre en compte le style du WSDL, car l'outil client permettant d'appeler le proxy peut comporter des restrictions quant au style du WSDL.
Encodage
Sélectionnez Encodé ou Littéral.
Méthodes
Les méthodes affichées correspondent à celles que vous avez choisies pour l'interface fonctionnelle ou EJB distante. Sélectionnez les méthodes requises (vous pouvez en indiquer plusieurs). Cliquez sur le signe + pour développer la méthode. Modifiez les valeurs de paramètre par défaut et sélectionnez un convertisseur, si celui-ci est fourni (ou obligatoire).
Vous devez exclure les méthodes dont les paramètres ou types de retour ne sont pas pris en charge par la pile JAX-RPC, ou associer ces arguments à des classes de convertisseur.
Vous pouvez modifier le nom d'opération par défaut d'une méthode donnée. (Par défaut, le nom de l'opération est identique à celui de la méthode.) Si un EJB contient plusieurs méthodes portant le même nom, vous devez modifier les noms d'opération de manière à ce qu'ils soient uniques. Les WSDL exigent des noms d'opération uniques.
Remarque : si les informations d'identification et de connexion ou les paramètres de transaction sont différents selon les méthodes pour un même EJB, vous pouvez personnaliser les méthodes d'un service fonctionnel donné et créer un service fonctionnel par méthode. Cela permet un contrôle précis des transactions et des informations d'identification et de connexion.
Exceptions
Ce champ apparaît lorsqu'une méthode génère une business exception. Si une méthode EJB entraîne une exception dont les types de données ne sont pas pris en charge par Java Web Services (JWS) (comme ArrayList), le champ Exceptions permet de sélectionner une classe de convertisseur afin de convertir l'exception en un type pris en charge par JWS.
La classe de convertisseur doit implémenter com.bea.wli.sb.transports.ejb.ITypeConverter. Vous ne pouvez configurer des classes de convertisseur que pour les exceptions vérifiées et non pour les exceptions d'exécution.
Packagez la classe de convertisseur et la classe d'exception convertie dans le JAR du client ou du convertisseur.
Pour plus d'informations, reportez-vous à "EJB Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.
