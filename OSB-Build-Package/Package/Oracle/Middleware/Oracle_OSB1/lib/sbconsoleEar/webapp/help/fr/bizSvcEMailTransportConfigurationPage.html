Page Courrier électronique - Configuration du transport

La page Créer/Modifier un service fonctionnel - Courrier électronique - Configuration du transport est une des pages permettant de créer et modifier des services fonctionnels, comme indiqué dans Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Utilisez cette page pour configurer les paramètres de transport d'un service fonctionnel utilisant le protocole de transport de messagerie électronique. Le tableau 18-6 explique comment utiliser cette page.

Tableau 18-6 Page Créer/Modifier un service fonctionnel - Courrier électronique - Configuration du transport 
Option
Procédure
SSL obligatoire
Sélectionnez cette option pour communiquer via SSL (Secure Sockets Layer). Le numéro de port SSL par défaut pour SMTP est 465.
Remarque :
  • Le serveur de messagerie doit figurer dans le truststore Oracle Service Bus pour que SSL fonctionne.
  • Si vous utilisez le protocole TLS, vous devez placer le certificat d'approbation dans le truststore par défaut ($JAVA_HOME/jre/lib/security/cacerts) ou placez-le dans n'importe quel truststore et ajoutez le chemin d'accès dans le fichier $DOMAIN_HOME/bin/setDomainEnv.*, comme indiqué ci-dessous :

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<chemin du truststore>

Serveur SMTP
Sélectionnez le serveur SMTP par défaut à utiliser pour les entrées d'URI endpoint de name@domain_name.com. Si vous spécifiez des paramètres de serveur SMTP dans l'URI endpoint, comme indiqué dans le tableau 18-3, ces ressources de serveur sont utilisées à la place du paramètre de serveur SMTP défini ici.
Vous ne devez pas sélectionner de serveur SMTP si vous choisissez l'option Session de messagerie.
Pour plus d'informations sur la création de ressources SMTP, reportez-vous à Ajout de serveurs SMTP.
Session de messagerie
Indiquez le nom JNDI de la session de messagerie configurée à utiliser pour les entrées d'URI endpoint de name@domain_name.com. Si vous spécifiez des paramètres de session de messagerie JNDI dans l'URI endpoint, comme indiqué dans le tableau 18-3, ces sessions de messagerie sont utilisées à la place du paramètre défini ici.
Vous ne devez pas indiquer de session de messagerie si vous choisissez l'option Serveur SMTP.
Nom de l'émetteur
Saisissez le nom d'affichage du compte de messagerie électronique de l'expéditeur pour ce service.
Adresse de l'émetteur
Indiquez le compte de messagerie électronique de l'expéditeur pour ce service.
Nom de réponse aux messages
Saisissez le nom d'affichage du compte de messagerie électronique de réponse.
Adresse de réponse aux messages
Indiquez l'adresse électronique à laquelle la réponse doit être envoyée.
Temporisation de la connexion
Indiquez l'intervalle de temporisation, en secondes, au bout duquel la connexion est supprimée. La valeur 0 signifie qu'il n'y a jamais temporisation.
Encodage de la demande
Acceptez l'encodage de jeu de caractères par défaut, ISO-8859-1, pour les demandes dans les transports de messagerie électronique ou indiquez un encodage de jeu de caractères différent.

Après avoir fini

Cliquez sur Suivant pour poursuivre la configuration du service sur la page suivante ou cliquez sur Dernier pour vérifier et enregistrer cette configuration sur la page Créer/Modifier un service fonctionnel - Récapitulatif.
