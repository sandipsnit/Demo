Configuration générale, page

La page Créer/Modifier un service fonctionnel - Configuration générale est une des pages permettant de créer et modifier des services fonctionnels, comme indiqué dans Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Utilisez cette page pour modifier les paramètres de configuration générale d'un service fonctionnel.

Lorsque vous créez un service fonctionnel, cette page est la première qui apparaît sur tout un ensemble de pages permettant de configurer le service. Les pages affichées ensuite dépendent des choix effectués sur celle-ci.

Le tableau 18-1 explique comment utiliser cette page.

Tableau 18-1 Page Créer/Modifier un service fonctionnel - Configuration générale 
Option
Procédure
Nom du service
Saisissez un nom unique pour le service fonctionnel.

Pour obtenir des conseils sur la dénomination, suivez les restrictions concernant la dénomination des ressources.
Description
Saisissez une description.
Type de service
Le type de service définit les types et le packaging des messages échangés par le service. Sélectionnez le type de service fonctionnel à créer :
  • Service Web WSDL : sélectionnez cette option pour créer un service fonctionnel basé sur un WSDL. Saisissez ensuite le nom du WSDL, qualifié par son chemin (par exemple, myProject/myFolder/myWSDL). Vous pouvez également cliquer sur Parcourir pour faire votre choix à partir de la page Sélectionner une ressource WSDL.

    • (port ou liaison) : entrez le nom d'un port (défini dans le WSDL), pour décrire une adresse de transport réelle, ou celui d'une liaison (définie dans le WSDL), pour établir une correspondance avec une adresse de transport. Si vous cliquez sur Parcourir pour sélectionner un WSDL (comme décrit ci-dessus), la page Sélectionner une définition WSDL répertorie l'ensemble des ports et liaisons définis dans le WSDL. Lorsque vous choisissez un port ou une liaison sur cette page, le champ (port ou liaison) contient le nom sélectionné.

  • Service doté d'un type de transport : sélectionnez cette option pour créer un service qui utilise le transport de flux ou EJB.
  • Service de messagerie : sélectionnez cette option pour créer un service qui échange des messages dont le type de contenu varie considérablement. Ces échanges peuvent être bidirectionnels (demande/réponse) ou unidirectionnels. Vous pouvez également l'utiliser pour recevoir uniquement des réponses, sans demandes, avec l'option`GET' HTTP pour le transport HTTP. Contrairement aux services Web, les types de contenu de la demande et de la réponse ne doivent pas nécessairement être identiques.
  • N'importe quel service SOAP : sélectionnez cette option afin de créer un service SOAP pour lequel aucune interface concrète n'est explicitement définie.

    • Vous pouvez conserver la valeur par défaut, SOAP 1.1, ou sélectionnez SOAP 1.2 dans la liste déroulante.

  • N'importe quel service XML : sélectionnez cette option afin de créer un service XML pour lequel aucune interface concrète n'est explicitement définie.

    • L'option GET HTTP n'est prise en charge que pour les services de messagerie et ce type de service.

  • Service fonctionnel : sélectionnez cette option pour cloner un service fonctionnel existant.

    • Entrez le chemin (project/folder) et le nom du service fonctionnel ou cliquez sur Parcourir pour sélectionner un service fonctionnel sur la page Récapitulatif des services fonctionnels.

    Dans la mesure où Oracle Service Bus n'accepte pas un URI identique pour plusieurs services, vous devez modifier l'URI du service cloné.

  • Service proxy : sélectionnez cette option pour créer un service fonctionnel basé sur un service proxy existant.
Type de service (suite)
Remarque : lorsque vous créez un service à partir d'un autre, les règles d'alerte sont gérées de la façon suivante :
    • Lorsque vous créez un service proxy à partir d'un service fonctionnel et inversement, les règles d'alerte existantes sont supprimées.
    • Lorsque vous créez un service proxy à partir d'un autre service proxy ou que vous créez un service fonctionnel à partir d'un autre service fonctionnel, les règles d'alerte existantes sont conservées.

Après avoir fini

Cliquez sur Suivant pour poursuivre la configuration du service sur la page suivante ou cliquez sur Dernier pour vérifier et enregistrer cette configuration sur la page Créer/Modifier un service fonctionnel - Récapitulatif.
