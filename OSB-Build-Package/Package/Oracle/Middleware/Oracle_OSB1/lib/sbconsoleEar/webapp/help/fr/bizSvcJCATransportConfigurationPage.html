Page JCA - Configuration du transport

Cette page permet de configurer les paramètres de transport d'un service fonctionnel utilisant le protocole de transport JCA. Pour plus d'informations sur l'utilisation du transport JCA, reportez-vous à "JCA Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV910, et à Génération d'un service fonctionnel à partir d'une ressource de liaison JCA.

Option
Description
Fichier JCA
Cliquez sur Parcourir pour sélectionner une liaison JCA. Celle-ci définit différents aspects du service, tels que les détails relatifs à l'adaptateur utilisé, la liaison au fichier de mapping EclipseLink ou WSDL et TopLink, ainsi que les propriétés de spécification d'activation/interaction requises par le service.
Une fois que vous avez sélectionné une liaison JCA valide, les autres champs relatifs à la configuration du transport sont activés.
Pour plus d'informations sur les liaisons JCA, reportez-vous au chapitre 12, Liaisons JCA.
Nom d'adaptateur
Valeur en lecture seule indiquant le nom de l'adaptateur utilisé par le service JCA.
Type d'adaptateur
Valeur en lecture seule indiquant le type de l'adaptateur.
Stratégie de répartition
Sélectionnez l'instance du gestionnaire de travaux WebLogic Server à utiliser dans la stratégie de répartition de cette adresse. Le gestionnaire de travaux par défaut est utilisé s'il n'en existe pas d'autre.
Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Compte de service JNDI
Pour la sécurité du contexte JNDI, permet d'accéder à la fabrique de connexions gérée par l'adaptateur EIS. Cliquez sur Parcourir et sélectionnez un compte de service. Si aucun n'est spécifié, un sujet anonyme est utilisé.
Pour les services fonctionnels JCA, il n'existe aucune restriction concernant le type de compte de service JNDI pouvant être configuré (statique ou pass-through), mais le système doit pouvoir accéder à un nom utilisateur et un mot de passe à l'exécution. Les services proxy JCA ne peuvent utiliser que des comptes de service JNDI statiques.
Pour plus d'informations sur les comptes de service JNDI, reportez-vous à la section "Security" du chapitre "JCA Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante http://www.oracle.com/pls/as111170/lookup?id=OSBDV916.
Propriétés d'adresse
Permet d'affecter des valeurs aux propriétés d'adresse, telles que le nombre de nouvelles tentatives pour le type d'adaptateur utilisé par le service.
Pour obtenir la liste des propriétés d'adresse JCA prises en charge, reportez-vous à "Endpoint Properties" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1841.
Propriétés d'adresse dynamiques
Permet de transmettre des paramètres de demande à des services compatibles JCA. Vous pouvez, par exemple, utiliser une propriété d'adresse dynamique pour transmettre des paramètres d'interrogation de base de données à l'adaptateur JCA Oracle pour la base de données.
Pour plus d'informations sur l'utilisation des paramètres d'interrogation, reportez-vous à "Oracle JCA Adapter for Database" dans le manuel Oracle Fusion Middleware User's Guide for Technology Adapters à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=TKADP2117.
Entrez une paire nom/valeur pour chaque propriété d'adresse dynamique à fournir. La clé de propriété d'adresse correspond au nom du paramètre d'interrogation.
Toujours utiliser la configuration provenant du fichier JCA
Détermine si les propriétés de spécification d'activation (services proxy) et les propriétés de spécification d'interaction (services fonctionnels) provenant du fichier JCA sont toujours utilisées.
Lorsque cette option est sélectionnée (comportement par défaut), le transport JCA interagit avec la structure JCA à l'aide des propriétés de spécification d'activation/interaction du fichier JCA.
Si vous désélectionnez cette option, vous pouvez remplacer les propriétés de spécification d'activation/interaction.
Pour plus d'informations sur l'impact de cette option sur le redéploiement, reportez-vous à la section "Endpoint Redeployment" du chapitre "JCA Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV940.
Nom d'opération
Affiche le nom en lecture seule de l'opération WSDL sélectionnée. Une opération peut disposer de ses propres propriétés de spécification d'activation/interaction. Celles-ci apparaissent dans les champs Propriétés de spécification d'activation et Propriétés de spécification d'interaction, respectivement.
Propriétés de spécification d'activation/d'interaction
Le champ Propriétés de spécification d'activation concerne les services proxy, tandis que le champ Propriétés de spécification d'interaction est utilisé pour les services fonctionnels.
S'il s'agit d'un service entrant appelé par une application EIS, ce champ affiche les propriétés de spécification d'activation de l'opération entrante JCA spécifiée dans le champ Nom d'opération.
Vous pouvez remplacer les propriétés de spécification d'activation/interaction en désélectionnant l'option Toujours utiliser la configuration provenant du fichier JCA.
Remarque : les propriétés de spécification d'activation/interaction des adaptateurs Oracle Adapter Suite sont en lecture seule. Les adaptateurs Oracle Adapter Suite stockent leurs propres configurations. Pour les modifier, vous devez utiliser les outils de gestion Oracle Adapter Suite.
Propriétés de connexion (propriétaires)
Pour les services JCA hérités utilisant des propriétés de connexion en mode non géré (en phase d'abandon dans cette version), reportez-vous aux options de configuration de la connexion à l'adresse suivante : http://download.oracle.com/docs/cd/E13159_01/osb/docs10gr3/jcatransport/transport.html#wp1105451.

Pour plus d'informations sur les propriétés de spécification d'adresse et d'activation/interaction, reportez-vous au manuel Oracle Fusion Middleware User's Guide for Technology Adapters, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=TKADP.
