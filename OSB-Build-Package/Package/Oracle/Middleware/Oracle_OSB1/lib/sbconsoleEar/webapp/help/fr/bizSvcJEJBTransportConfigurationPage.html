Page JEJB - Configuration du transport

Cette page permet de configurer les paramètres de transport d'un service fonctionnel utilisant le protocole de transport JEJB. Pour plus d'informations sur l'utilisation du transport JEJB, reportez-vous à "JEJB Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036.

Option
Description
Stratégie de répartition
Sélectionnez l'instance du gestionnaire de travaux WebLogic Server à utiliser dans la stratégie de répartition de cette adresse. Le gestionnaire de travaux par défaut est utilisé s'il n'en existe pas d'autre.
Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Version de spécification EJB
Sélectionnez la version EJB de l'interface EJB distante.
Transmettre les beans XML sous forme de valeur
Lorsque vous sélectionnez cette option, le transport génère une représentation XML "imbriquée" des arguments POJO (objet XMLObject). Vous pouvez accéder à leurs paramètres et les manipuler au moyen d'expressions XQuery.
Remarque : les informations de type ne sont pas imbriquées pour les objets XMLObjects transmis sous forme de valeur. Lorsque vous sélectionnez cette option, vous ne pouvez pas transmettre l'objet de type XMLObject en tant qu'argument de callout Java dans le pipeline d'un service proxy.
Ne sélectionnez pas cette option si vous souhaitez transmettre l'objet POJO sous forme de référence. Cela améliore également les performances.
Transmettre le sujet de l'appelant
Lorsque vous sélectionnez cette option à la place de Compte de service, Oracle Service Bus transmet le sujet authentifié du service proxy lors de l'appel de l'EJB.
Compte de service
Cliquez sur Parcourir et sélectionnez un compte de service JNDI dans la liste affichée. Si vous n'indiquez aucun compte de service, un sujet anonyme est utilisé. Pour plus d'informations, reportez-vous au chapitre 16, Comptes de service.
Fichier JAR client
Cliquez sur Parcourir et sélectionnez une ressource JAR de client EJB dans la liste affichée. Le fichier JAR client contient l'interface fonctionnelle ou distante du service distant. Il est inscrit en tant que ressource d'archive générique.
Interface locale
(EJB 2.1 uniquement) Sélectionnez l'interface EJBHome requise parmi les options fournies par le fichier JAR.
Interface distante
(EJB 2.1 uniquement) Ce champ est automatiquement rempli en fonction de la configuration de l'interface locale.
Interface fonctionnelle
(EJB 3.0 uniquement) Sélectionnez l'interface fonctionnelle du fichier JAR client à appeler.
Espace de noms cible
Ce champ est rempli en fonction des informations fournies par le fichier JAR.
Méthodes
Sélectionnez les méthodes requises. Cliquez sur le signe + pour développer une méthode et modifier les valeurs par défaut des paramètres.
Vous pouvez modifier le nom d'opération par défaut d'une méthode donnée. Par défaut, le nom de l'opération est identique à celui de la méthode. Si un EJB contient plusieurs méthodes portant le même nom (surcharge), vous devez modifier les noms d'opération de manière à ce qu'ils soient uniques. Les WSDL exigent des noms d'opération uniques.
