Page JMS - Configuration du transport

La page Créer/Modifier un service fonctionnel - JMS - Configuration du transport est une des pages permettant de créer et modifier des services fonctionnels, comme indiqué dans Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Pour plus d'informations, reportez-vous à "JMS Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Utilisez cette page pour configurer les paramètres de transport d'un service fonctionnel utilisant le protocole de transport JMS. Le tableau 18-13 explique comment utiliser cette page.

Tableau 18-13 Page Créer/Modifier un service fonctionnel - JMS - Configuration du transport (suite)
Option
Procédure
Type de destination
Sélectionnez le type de la destination de pont JMS :
  • File d'attente (point à point)
  • Rubrique (publication/abonnement)
Type de message
Sélectionnez l'une des options suivantes :
  • Octets (flux d'octets non interprétés)
  • Texte (messages texte)
Cette option est désactivée si vous sélectionnez un type de message Java pour la réponse.
Files d'attente de réponse
Cette option n'est disponible que si vous sélectionnez File d'attente dans Type de destination.
Sélectionnez l'une des options de réponse suivantes :
  • Aucun : aucune réponse n'est attendue. Utilisez cette option pour les opérations unidirectionnelles.
  • Un élément pour toutes les URI de demande : permet d'entrer un URI unique pour la gestion de la réponse, de définir certains détails de configuration de la réponse (tels que l'encodage et la temporisation) et éventuellement de sélectionner un compte de service JMS pour la transmission des informations d'identification et de connexion JMS/JNDI.
  • Un élément par URI de demande : permet le basculement de réponse en cas d'incident. Vous pouvez entrer un URI de réponse ou une destination pour chaque URI de demande.
Vous pouvez également sélectionner un compte de service pour les informations d'identification et de connexion JMS/JNDI de chaque paire demande/réponse.
Modèle de réponse
Cette option n'est disponible que si vous sélectionnez une option de réponse dans le champ Files d'attente de réponse.
Sélectionnez l'une des options suivantes :
  • Sélectionnez JMSCorrelationID pour les services autres que JAX-RPC exécutés sur Oracle WebLogic Server.
  • Sélectionnez JMSMessageID pour les services JAX-RPC exécutés sur Oracle WebLogic Server.
Pour plus d'informations, reportez-vous à "Message ID and Correlation ID Patterns for JMS Request/Response" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051.
Stratégie de répartition
Sélectionnez l'instance du gestionnaire de travaux Oracle WebLogic Server à utiliser dans la stratégie de répartition de cette adresse. Le gestionnaire de travaux par défaut est utilisé s'il n'en existe pas d'autre.
Par exemple, si le service fonctionnel utilise un protocole de transport JMS, son adresse consiste en un fichier JAR MDB (bean orienté message) pouvant être associé à la stratégie de répartition définie.
Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Encodage de la demande
Indiquez le jeu de caractères utilisé pour l'encodage des demandes. La valeur par défaut est UTF-8.
Encodage de la réponse
Cette option n'est disponible que si vous sélectionnez une option de réponse dans le champ Files d'attente de réponse.
Indiquez le jeu de caractères utilisé pour l'encodage des réponses. La valeur par défaut est UTF-8.
Temporisation de réponse
Cette option n'est disponible que si vous sélectionnez une option de réponse dans le champ Files d'attente de réponse.
Indiquez la durée, en secondes, au bout de laquelle la connexion est supprimée en l'absence de réponse. La valeur zéro (0) (valeur par défaut) signifie qu'il n'y a jamais temporisation de la réponse.
Fichier JAR client
Cette option est disponible dans le cas d'un service de messagerie présentant un type de réponse Java. Sélectionnez le JAR client à utiliser lors du retrait de la file d'attente des messages contenant des objets Java. La sélection du JAR client garantit qu'il figure dans la variable d'environnement CLASSPATH.

Pour plus d'informations, reportez-vous à "Sending and Receiving Java Objects in Messages" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
URI de réponse
Cette option est disponible lorsque vous sélectionnez l'option Un élément pour toutes les URI de demande et le modèle de réponse JMSCorrelationID.
Entrez un URI de réponse au format suivant :
jms://host:port/connection_factory/jndi_destination.
Pour cibler plusieurs serveurs, utilisez le format suivant :
jms://host1:port,host2:port/connection_factory/jndi_destination.
Vous pouvez aussi n'indiquer ni l'hôte, ni le port dans l'URI de réponse. Par exemple :
jms:///connection_factory/jndi_destination.

Lorsque vous n'indiquez ni l'hôte, ni le port, la recherche de fabrique de connexions/destination s'effectue sur le serveur local. Ce comportement s'avère utile, par exemple, lorsque l'URI de demande pointe vers une fabrique de connexions/destination étrangère, mais que vous souhaitez envoyer la réponse au serveur local.

Remarque : lorsque Oracle WebLogic Server autorise les barres obliques dans les noms JNDI, tels que "myqueues/myqueue", les noms JNDI avec des barres obliques interfèrent avec le format URI requis par Oracle Service Bus, et vous ne pouvez pas utiliser ces noms. Pour passer outre ce problème, définissez un serveur étranger JMS et référencez-le dans l'URI. Pour plus d'informations, reportez-vous à Configurer les serveurs étrangers dans l'aide en ligne d'Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help à l'adresse http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Réponses aux demandes
Cette option est disponible lorsque vous sélectionnez l'option Un élément par URI de demande pour le modèle JMSCorrelationID, afin de permettre le basculement de réponse en cas d'incident.
Pour chaque URI de demande saisi sur la page Transport générale, entrez un URI de réponse, comme indiqué dans la description du champ URI de réponse ci-dessus.
Vous pouvez sélectionner un compte de service facultatif pour les informations d'identification et de connexion JMS/JNDI utilisées par le service, pour les files d'attente de demande et de réponse.
Cible - Réponses
Cette option est disponible lorsque vous sélectionnez l'option Un élément pour toutes les URI de demande pour le modèle JMSMessageID.
Saisissez le nom du serveur cible qui recevra les réponses et entrez un URI de réponse, comme indiqué dans la description du champ URI de réponse.
Connexions de demande
Cette option est disponible lorsque vous sélectionnez l'option Un élément par URI de demande pour le modèle JMSMessageID, afin de permettre le basculement de réponse en cas d'incident.
Pour chaque URI de demande, identifié de façon séquentielle par le numéro spécifié dans le champ Num. séq, vous pouvez éventuellement entrer le nom d'une fabrique de connexions JMS. Si vous n'indiquez aucun nom, le transport JMS utilise la fabrique de connexion provenant de l'URI de demande.
Vous pouvez sélectionner un compte de service facultatif pour les informations d'identification et de connexion JMS/JNDI utilisées par le service, pour les files d'attente de demande et de réponse.
Cible - Destinations
Cette option est disponible lorsque vous sélectionnez l'option Un élément par URI de demande pour le modèle JMSMessageID, afin de permettre le basculement de réponse en cas d'incident. Utilisez ce champ en association avec Connexions de demande.
Les serveurs cible énumérés (déterminés par les serveurs du domaine en cours), comme les serveurs gérés d'un cluster par exemple, répertorient les URI de demande par numéro de séquence. Ceux-ci correspondent à ceux spécifiés dans le champ Connexions de demande. Pour chaque URI de demande sur chaque cible, indiquez la file d'attente de destination du serveur cible qui reçoit les réponses.
Remarque : dans la mesure où l'environnement de développement Oracle Service Bus ne prend en charge qu'un serveur, une seule cible est répertoriée. Pour configurer ce champ dans un environnement multiserveur, déployez le service sur l'environnement d'exécution, puis poursuivez sa configuration dans la console d'administration d'Oracle Service Bus.
Compte de service JMS
Cette option est disponible lorsque vous sélectionnez l'option Aucun ou Un élément pour toutes les URI de demande dans le champ Files d'attente de réponse.
Sélectionnez un compte de service à utiliser avec la ressource JMS gérée par le serveur JMS. Un compte de service représente la ressource d'alias d'un ID utilisateur et d'un mot de passe, utilisée à la fois pour la demande et pour la réponse. Le même compte de service est employé pour les ressources JMS et JNDI.
Ce champ et l'option Transmettre le sujet de l'appelant s'excluent mutuellement. Vous devez utiliser soit l'un soit l'autre pour l'authentification JMS/JNDI.
Pour plus d'informations, reportez-vous à Comptes de service.
Utiliser SSL
Ne sélectionnez cette option que si les demandes sont effectuées via une connexion TLS/SSL.
Le protocole TLS/SSL (Secure Sockets Layer) fournit des connexions sécurisées en permettant à deux applications qui se connectent sur un réseau d'authentifier l'identité de l'autre et en cryptant les données échangées entre les applications. L'authentification permet à un serveur, et éventuellement à un client, de vérifier l'identité de l'application à l'autre extrémité de la connexion réseau. En outre, si l'administrateur restreint l'accès à certaines destinations JMS (files d'attente ou rubriques) en définissant un contrôle d'accès sur l'entrée JNDI de la destination, le service doit procéder à une authentification lors de la recherche de l'entrée dans l'arborescence JNDI. Choisissez l'authentification au moyen d'un compte de service ou de l'option Transmettre le sujet de l'appelant.
Expiration
Délai, en millisecondes, au bout duquel le message arrive à expiration. La valeur par défaut est 0 ; elle indique que le message n'arrive jamais à expiration.
Activer la persistance des messages
Mode de distribution JMS utilisé par le service, assurant un certain équilibre entre fiabilité et débit. Sélectionnez cette option (celle-ci est activée par défaut) pour garantir la distribution de tous les messages. Désélectionnez cette option pour améliorer le débit (certains messages peuvent toutefois être perdus).
Unité d'ordre
Indiquez l'unité d'ordre des messages. Cette unité permet aux émetteurs de regrouper des messages dans une même unité basée sur l'ordre de traitement. Cette unité d'ordre exige que tous les messages qu'elle contient soient traités de façon séquentielle, dans leur ordre de création.
Transmettre le sujet de l'appelant
Lorsque vous cochez cette case, Oracle Service Bus transmet le sujet authentifié lors de l'envoi d'un message.
Si vous activez cette option et que le service fonctionnel cible les ressources JMS d'un domaine différent, vous devez activer la sécurisation globale sur les deux domaines. Reportez-vous à "Configuring Security for a WebLogic Domain" dans le manuel Oracle Fusion Middleware Securing Oracle WebLogic Server.
Ce champ et l'option Compte de service s'excluent mutuellement. Vous devez utiliser soit l'un soit l'autre pour l'authentification JMS/JNDI.
Si vous activez cette option et que le service fonctionnel cible les ressources JMS d'un domaine différent, vous devez activer la sécurisation globale sur les deux domaines. Reportez-vous à "Configuring Security for a WebLogic Domain" dans le manuel Oracle Fusion Middleware Securing Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=SECMG402.
Temporisation JNDI
Temporisation, en secondes, de la connexion JNDI utilisée lors de la recherche de la destination ou de la fabrique de connexions dans l'arborescence JNDI.
La valeur par défaut est zéro (0) ; elle indique qu'il n'y a jamais temporisation de la connexion.
