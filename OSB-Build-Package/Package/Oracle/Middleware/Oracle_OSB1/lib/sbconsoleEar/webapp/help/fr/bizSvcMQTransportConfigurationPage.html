Page MQ - Configuration du transport

La page Créer/Modifier un service fonctionnel - MQ - Configuration du transport est une des pages permettant de créer et modifier des services fonctionnels, comme indiqué dans Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Pour plus d'informations, reportez-vous à "MQ Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Avant de commencer

Configurez une ressource de connexion MQ. Reportez-vous à Connexions MQ.

Pour configurer le transport MQ

Utilisez cette page pour configurer les paramètres de transport d'un service fonctionnel utilisant le protocole de transport MQ natif. Le tableau 18-14 explique comment utiliser cette page.

Tableau 18-14 Page Créer/Modifier un service fonctionnel - MQ - Configuration du transport 
Option
Procédure
Type de message
Sélectionnez l'une des options suivantes :
  • Octets (flux d'octets non interprétés)
  • Texte (messages texte)
Réponse requise ?
Sélectionnez cette option pour indiquer qu'une réponse est attendue lorsqu'un message sortant est envoyé.
Intervalle d'appel
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Indiquez un intervalle d'appel, en millisecondes. La valeur par défaut est 1 000.
Modèle de corrélation de réponse
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Indiquez si le modèle de corrélation de réponse est basé sur l'un des éléments suivants :
  • ID de message
  • ID de corrélation
  • File d'attente dynamique : sélectionnez cette option si votre implémentation MQ WebSphere utilise des files d'attente dynamiques pour la corrélation de réponse. Le transport MQ ne prend en charge que les files d'attente dynamiques temporaires.
Générer automatiquement la valeur de corrélation
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Cochez cette case pour générer automatiquement un ID de corrélation ou de message.
File d'attente modèle
(Modèle de corrélation de réponse de file d'attente dynamique uniquement.) Saisissez le nom de la file d'attente modèle permettant de générer la file d'attente dynamique.
URI de réponse MQ
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Destination vers laquelle la réponse doit être publiée. Entrez un URI de réponse dans un format identique à celui de l'URI endpoint, à savoir :
mq://local_queue_name?conn=mq_connection_resource
ou, si vous utilisez des files d'attente dynamiques :
mq://dynamic_queue_prefix?conn=mq_connection_resource.
L'élément dynamic_queue_prefix, limité à 32 caractères maximum, permet de créer la file d'attente dynamique sur le serveur MQ. Le nom de la file d'attente est formé du préfixe et d'un ID unique. Par exemple, si l'élément dynamic_queue_prefix est "example", le nom de la file d'attente dynamique peut être "example123129083821".
Vous pouvez également utiliser l'astérisque (*) comme caractère générique dans l'URI de réponse de file d'attente dynamique. Exemple :
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
Si vous n'indiquez aucun élément dynamic_queue_prefix dans l'URI, le transport utilise le nom de file d'attente dynamique généré par le serveur MQ. Si vous ne spécifiez aucun élément mq_connection_resource explicite dans l'URI (indiquer cet élément constitue la meilleure pratique), le transport utilise l'élément mq_connection_resource de l'URI endpoint.
Pour plus d'informations, reportez-vous à "MQ Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.
Temporisation de réponse
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Indiquez la durée, en secondes, au bout de laquelle la connexion est supprimée en l'absence de réponse.
Stratégie de répartition
La stratégie de répartition fait référence à l'instance du gestionnaire de travaux WLS à utiliser pour l'adresse du service.
Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Pool de files d'attente dynamiques
(Modèle de corrélation de réponse de file d'attente dynamique uniquement.) Sélectionnez cette option si vous souhaitez que le service utilise des connexions incluses dans des pools pour atteindre les files d'attente dynamiques.
Si vous souhaitez utiliser un pool de connexions différent pour les files d'attentes dynamiques, configurez une ressource de connexion MQ dédiée aux files d'attente dynamiques.
Ne sélectionnez pas cette option si vous souhaitez créer une file d'attente dynamique à chaque demande (et la détruire après réception de la réponse).
Options `PUT' de l'URI endpoint
Entrez les options de message PUT MQ suivantes que vous souhaitez utiliser :
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
Vous pouvez utiliser "|" ou "+" pour séparer les différentes options. Indiquez par exemple :
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
Les options de message PUT MQ sont appliquées lorsque le message est placé dans la file d'attente sortante.
URI de réponse non reconnue MQ
Entrez l'URI représentant la file d'attente vers laquelle les messages de réponse non reconnus doivent être envoyés. Ce paramètre n'est activé que si vous cochez la case Générer automatiquement la valeur de corrélation.
Si vous n'indiquez aucune valeur dans ce champ, les messages de réponse non reconnus sont supprimés.
Traiter les en-têtes RFH2
Sélectionnez cette option pour analyser les en-têtes RFH2 MQ WebSphere des données traitées de message et générer automatiquement un en-tête de transport RFH2Headers contenant les données RFH2.
Si vous ne sélectionnez pas cette option, les données traitées sont transmises telles qu'elles sont reçues.

Pour plus d'informations sur le mode de gestion des en-têtes RFH2 par le transport MQ, reportez-vous à About RFH2 Headers dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
