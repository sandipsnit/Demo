Page Configuration du type de message

La page Créer/Modifier un service fonctionnel - Configuration du type de message est une des pages permettant de créer et modifier des services fonctionnels, comme indiqué dans Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Utilisez cette page pour configurer les types de message d'un service fonctionnel de type service de messagerie.

La définition de liaison des services de messagerie consiste à configurer le type de contenu des messages échangés. Le type de contenu de la réponse ne doit pas nécessairement être identique à celui de la demande. Par conséquent, la réponse est configurée séparément (par exemple, le service peut accepter un message MFL et renvoyer un accusé de réception au format XML).

Remarque : les services fonctionnels de transport de messagerie électronique, de fichier, FTP et SFTP de type service de messagerie prennent en charge la messagerie unidirectionnelle uniquement. Le type de message de réponse doit être défini sur aucun. Si vous sélectionnez une autre option, les protocoles de fichier, FTP et SFTP ne seront pas disponibles sur la page Configuration du transport.

Le tableau 18-2 explique comment utiliser la page Créer/Modifier un service fonctionnel - Configuration du type de message.

Tableau 18-2 Page Créer/Modifier un service fonctionnel - Configuration du type de message 
Option
Procédure
Type du message de demande
Sélectionnez le type de message de demande :
  • Aucun : sélectionnez cette option s'il n'y a pas de message de demande (GET HTTP, par exemple).
  • Binaire : sélectionnez cette option si le type de contenu du message est inconnu ou n'est pas important.
  • Texte : sélectionnez cette option si le message peut être limité à du texte.
  • MFL : sélectionnez cette option si le message est un document binaire conforme à une définition MFL. Vous ne pouvez configurer qu'un fichier MFL.

    • Vous pouvez également cliquer sur Parcourir pour sélectionner un fichier MFL à partir du navigateur MFL. Cliquez ensuite sur Soumettre.

  • XML : sélectionnez cette option si le message est un document XML. Pour fournir des informations de type, vous pouvez déclarer le type de schéma XML du document XML échangé.
  • Java : sélectionnez cette option si un objet Java est envoyé dans la demande. Le transport JMS est utilisé pour les messages de type Objet Java. Pour plus d'informations, reportez-vous à "Sending and Receiving Java Objects in Messages" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Type de message de réponse
Sélectionnez le type de message de réponse :
  • Aucun : sélectionnez cette option s'il n'y a pas de message de réponse.
  • Binaire : sélectionnez cette option si le type de contenu du message est inconnu ou n'est pas important.
  • Texte : sélectionnez cette option si le message peut être limité à du texte.
  • MFL : sélectionnez cette option si le message est un document binaire conforme à une définition MFL. Vous ne pouvez configurer qu'un fichier MFL.

    • Vous pouvez également cliquer sur Parcourir pour sélectionner un fichier MFL à partir du navigateur MFL. Cliquez ensuite sur Soumettre.

  • XML : sélectionnez cette option si le message est un document XML. Pour fournir des informations de type, vous pouvez déclarer le type de schéma XML du document XML échangé.
  • Java : sélectionnez cette option si un objet Java est reçu dans la réponse. Le transport JMS est utilisé pour les messages de type Objet Java. Pour plus d'informations, reportez-vous à "Sending and Receiving Java Objects in Messages" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.

Après avoir fini
