Page SOA-DIRECT - Configuration du transport

Le tableau 18-17 décrit les options de configuration propres au transport SOA-DIRECT.

Pour plus d'informations sur le transport SOA-DIRECT, reportez-vous à "SOA-DIRECT Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV909.

Tableau 18-17 Créer/Modifier un service fonctionnel - SOA-DIRECT - Configuration du transport
Propriété
Description
Compte de service JNDI
Facultatif. Indique les informations d'identification et de connexion de sécurité pour la recherche JNDI du service SOA cible. Le compte de service doit être statique. Cliquez sur Parcourir et sélectionnez un compte de service. Si vous n'indiquez pas de compte de service, un sujet anonyme est utilisé.
Rôle
Obligatoire. Identifie le modèle de communication utilisé par le service. Sélectionnez l'une des options suivantes :
  • Client synchrone : (par défaut) dans ce rôle, étant donné qu'il n'existe pas de callback asynchrone, l'option Proxy de callback est désactivée. L'option Version d'adressage WS est également désactivée.
  • Client asynchrone : dans ce rôle, étant donné que le callback asynchrone est généralement obligatoire, vous pouvez identifier un proxy de callback et vous devez sélectionner une version d'adressage WS. L'option asynchrone n'est activée que lorsque le service WSDL est de type SOAP.
  • Callback de service : ce rôle est destiné à renvoyer le callback asynchrone à un service SOA après l'appel d'un service externe.

    • L'équilibrage de charge ou le basculement en cas d'incident ne sont pas disponibles pour les services qui ont le rôle Callback de service.

Proxy de callback
Facultatif. Activé uniquement pour le rôle Client asynchrone.
Cette fonction permet d'indiquer le service proxy qui reçoit les callbacks. Lorsque vous sélectionnez un proxy callback, si aucun adressage WS n'est fourni par la demande ou le pipeline du service proxy, Oracle Service Bus remplit automatiquement la propriété ReplyTo dans l'en-tête SOAP. Vous devez sélectionner un service proxy WSDL utilisant le transport SB (pour RMI) et le service proxy de callback doit comprendre l'adressage WS.
Les propriétés d'adressage WS envoyées dans la demande ou définies dans le pipeline du service proxy remplacent le proxy de callback défini dans cette option.
Si vous n'indiquez pas de proxy de callback et que la demande ne contient pas de propriétés ReplyTo, vous devez indiquer des propriétés ReplyTo dans l'en-tête SOAP via le pipeline du service proxy.
Proxy d'erreur
Cette option n'est actuellement pas prise en charge. Vous devez configurer vos services de callback pour gérer les erreurs dans un modèle asynchrone.
Version d'adressage WS
Obligatoire. Option activée seulement pour le rôle Client asynchrone.
Indiquez la version d'adressage WS par défaut à utiliser lorsqu'aucun adressage WS n'est fourni dans la demande ou le pipeline du service proxy. Les propriétés d'adressage WS envoyées dans la demande ou définies dans le pipeline du service proxy sont utilisées à la place de la version d'adressage WS définie dans cette option.
En cas de non-concordance de la version d'adressage WS dans les environnements, exécutez toutes les transformations nécessaires dans le pipeline du service proxy. Pour plus d'informations, reportez-vous à Transformation Examples dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1827.
Stratégie de répartition
Sélectionnez l'instance du gestionnaire de travaux WebLogic Server à utiliser dans la stratégie de répartition de cette adresse. Le gestionnaire de travaux par défaut est utilisé s'il n'en existe pas d'autre.
Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Transmettre le sujet de l'appelant
Facultatif. Sélectionnez cette option pour qu'Oracle Service Bus transmette le sujet authentifié à partir du service proxy lorsqu'il appelle le service SOA. L'option Compte de service d'appel, une alternative à Transmettre le sujet de l'appelant, est désactivée lorsque vous sélectionnez cette option.
Remarque : assurez-vous que la sécurisation de domaine est activée entre le client et le serveur cible s'ils sont dans dans des domaines différents. Pour plus d'informations, reportez-vous à Important Information Regarding Cross-Domain Security Support dans le manuel Oracle Fusion Middleware Securing Oracle WebLogic Server, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=SECMG403.
Compte de service d'appel
Facultatif. Il s'agit d'une alternative à Transmettre le sujet de l'appelant qui vous permet d'indiquer des informations d'identification et de connexion de sécurité personnalisées en sélectionnant un compte de service pour un appel RMI. Vous pouvez indiquer n'importe quel type de compte de service (Pass-through, Statique, Mise en correspondance).
Cliquez sur Parcourir et sélectionnez un compte de service. Si vous n'indiquez pas de compte de service, un sujet anonyme est utilisé.
