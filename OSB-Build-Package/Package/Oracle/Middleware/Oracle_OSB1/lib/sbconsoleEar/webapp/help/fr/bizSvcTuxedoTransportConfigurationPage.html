Page Tuxedo - Configuration du transport

La page Créer/Modifier un service fonctionnel - Tuxedo - Configuration du transport est l'une des pages destinées à la création et à la modification de services fonctionnels, décrites dans les rubriques Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Pour plus d'informations, reportez-vous à Tuxedo Transport dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Cette page permet de configurer les paramètres de transport pour un service fonctionnel utilisant le protocole de transport Tuxedo. Le tableau 18-18 explique comment l'utiliser.

Tableau 18-18 Page Créer/Modifier un service fonctionnel - Tuxedo - Configuration du transport (suite)
Option
Procédure
Classes de table de champs
Entrez le nom des classes décrivant le tampon FML/FML32 reçu. Elles sont utilisées pour les sous-programmes routines de conversion FML/FML32-to-XML pour mettre en correspondance les noms de champ et d'élément. Il s'agit d'une liste des noms qualifiés complets des classes, séparés par des espaces.
Classes de vues
Entrez le nom des classes décrivant le tampon VIEW/VIEW32 reçu ou envoyé. Elles sont utilisées pour les sous-programmes de conversion VIEW-to-XML ou VIEW32-to-XML pour mettre en correspondance les noms de champ et d'élément. Il s'agit d'une liste des noms qualifiés complets des classes, séparés par des espaces.
Fichier JAR de classes
Sélectionnez une ressource JAR contenant un fichier JAR appartenant aux classes FML/FML32 ou VIEW/VIEW32 nécessaires pour cette opération endpoint.
Points d'accès distants
Sélectionnez un point d'accès distant dans la liste déroulante associée à l'import. Cette liste contient des points d'accès distants configurés dans WTC. Un service fonctionnel ne peut pas être créé s'il n'est pas associé à un point d'accès distant.
S'il n'existe aucun point d'accès distant ou pour en créer un, sélectionnez Nouveau. Entrez le nom du point d'accès et l'adresse réseau correspondants dans les champs adjacents. Après validation de l'adresse, le point d'accès est ajouté à la configuration WTC pour chaque serveur WTC. S'il n'en existe pas, un serveur WTC est créé.
Lorsque plusieurs URI sont indiqués, il y a un champ de point d'accès distant par URI et l'URI apparaît à des fins d'information. S'il existe plusieurs URI, chacun doit avoir un point d'accès distant. Si l'URI indiqué correspond déjà à une ressource WTC existante, le point d'accès distant correspondant apparaît, mais ne peut pas être modifié.
Points d'accès locaux
Ce champ apparaît uniquement lorsque vous sélectionnez Nouveau dans le champ Point d'accès distant.
Dans la liste déroulante, sélectionnez un point d'accès local à associer avec le point d'accès distant que vous venez de créer. S'il n'existe aucun point d'accès ou pour en créer un, sélectionnez Nouveau. Entrez le nom du point d'accès local et l'adresse réseau locale correspondants dans les champs adjacents.
Type de tampon de demande
Sélectionnez le type de tampon que le service Tuxedo distant recevra.
Sous-type de tampon de demande
Cette option est activée si la valeur précédente du type de tampon de demande est VIEW ou VIEW32. Entrez le sous-type de tampon à associer avec le tampon de demande.
Réponse requise ?
Cochez cette case pour indiquer un appel bidirectionnel. Si elle n'est pas cochée, le tpcall sous-jacent est appelé avec l'indicateur TPNOREPLY et une réponse NULL est envoyée de façon asynchrone.
Suspendre la transaction ?
Cochez cette case pour suspendre la transaction, le cas échéant. Cela est utile lorsque le service distant ne prend pas en charge les transactions.
Stratégie de répartition
Sélectionnez l'instance du gestionnaire de travaux WebLogic Server à utiliser dans la stratégie de répartition de cette adresse. Le gestionnaire de travaux par défaut est utilisé s'il n'en existe pas d'autre.
Ce gestionnaire de travaux est utilisé pour envoyer de manière asynchrone une réponse NULL dans le cas d'un appel unidirectionnel. Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à Using Work Managers to Optimize Scheduled Work dans Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Encodage de la demande
Indiquez un encodage de jeu de caractères pour les demandes dans des transports Tuxedo.
Encodage de la réponse
Indiquez un encodage de jeu de caractères pour les réponses dans des transports Tuxedo.
Temporisation
Indiquez la durée maximale (en secondes) durant laquelle l'exécution du transport attend des réponses, par une valeur entière supérieure ou égale à 0. Si elle n'est pas indiquée ou égale à 0 (par défaut), le délai de temporisation des réponses ira jusqu'à la valeur BLOCKTIME, nombre maximal de secondes que le point d'accès WTC local autorise pour un appel bloquant.
Style de transformation
Sélectionnez l'une des options suivantes :
  • Aucun : (par défaut) l'ordre des champs peut ne pas être respecté.
  • Ordonné : les champs sont présentés avec toutes leurs occurrences dans l'ordre correct.
  • Groupé et ordonné : si les champs sont structurés logiquement en tant qu'enregistrements, ils sont classés par occurrence et regroupés par enregistrement.
