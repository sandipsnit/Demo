Page WS - Configuration du transport

La page Créer/Modifier un service fonctionnel - WS - Configuration du transport est l'une des pages destinées à la création et à la modification de services fonctionnels, décrites dans les rubriques Création et configuration de services fonctionnels et Modification des configurations des services fonctionnels.

Pour plus d'informations, reportez-vous à WS Transport dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083.

Cette page permet de configurer les paramètres de transport pour un service fonctionnel utilisant le protocole de transport WS. Le tableau 18-19 explique comment l'utiliser.

Tableau 18-19 Page Créer/Modifier un service fonctionnel - WS - Configuration du transport 
Option
Procédure
Temporisation de réponse
Entrez le nombre de secondes d'attente pour une réponse.
Si vous ne remplissez pas ce champ, il n'y a pas de temporisation de réponse. Le service fonctionnel attendra pendant la durée de temporisation de séquence configurée dans la stratégie de messagerie fiable.
Lorsque vous entrez la valeur zéro (0), il n'y a pas de temporisation. La temporisation ne sera donc jamais effectuée.
Compte de service
Entrez un compte de service ou cliquez sur Parcourir pour en sélectionner un dans la liste affichée.
Le compte de service indique les informations d'identification et de connexion à utiliser lorsqu'une stratégie d'authentification de base HTTP est employée pour ce service.
Paramètres avancés
Messages d'erreur en file d'attente
Cochez cette case pour activer l'envoi des messages d'erreur dans la file d'attente configurée.
URI de la file d'attente des erreurs
Cette option est disponible uniquement lorsque la case Messages d'erreur en file d'attente est cochée.
Entrez l'URI de la file d'attente JMS pour stocker les messages d'erreur au format suivant :

jms://host:port/connFactoryJndiName/queueJndiName

Remarque : lorsque Oracle WebLogic Server autorise les barres obliques dans les noms JNDI, tels que "myqueues/myqueue", les noms JNDI avec des barres obliques interfèrent avec le format URI requis par Oracle Service Bus, et vous ne pouvez pas utiliser ces noms. Pour passer outre ce problème, définissez un serveur étranger JMS et référencez-le dans l'URI. Pour plus d'informations, reportez-vous à Configurer les serveurs étrangers dans l'aide en ligne d'Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help à l'adresse http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Compte de service de la file d'attente des erreurs JMS
Cette option est disponible uniquement lorsque la case Messages d'erreur en file d'attente est cochée.
Entrez un compte de service ou cliquez sur Parcourir pour en sélectionner un dans la liste affichée.
Le compte de service indique les informations d'identification et de connexion à utiliser pour les recherches JNDI et l'envoi des messages d'erreur vers la file d'attente.
Utiliser SSL pour la file d'attente des erreurs
Cette option est disponible uniquement lorsque la case Messages d'erreur en file d'attente est cochée.
Cochez cette case pour utiliser SSL pour vous connecter à la file d'attente des erreurs.
