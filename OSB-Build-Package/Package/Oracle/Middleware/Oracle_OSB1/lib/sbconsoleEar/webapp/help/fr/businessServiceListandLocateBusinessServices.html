Localisation de services fonctionnels

  1. Effectuez l'une des opérations suivantes :
    • Sélectionnez Explorateur de projets pour afficher la page Vue des projets ou la page Vue du projet/Vue de dossier. Ensuite, parcourez les projets et les dossiers pour localiser le service fonctionnel.
    • Sélectionnez Navigateur Web de ressources > Services fonctionnels. La page Récapitulatif des services fonctionnels affiche les informations présentées dans le tableau 18-21. Pour obtenir une description plus détaillée des propriétés, reportez-vous à Modification des configurations des services fonctionnels.
  2. Pour restreindre le nombre d'éléments dans la liste, vous pouvez filtrer par nom, par chemin ou les deux. Dans les champs Nom et Chemin, sous Rechercher, entrez le nom et/ou le chemin des cibles, puis cliquez sur le bouton Rechercher.

    3. Le chemin est le nom du projet et le nom du dossier dans lequel réside le service fonctionnel.

    Les caractères génériques * et ? sont autorisés. La recherche distingue les majuscules des minuscules.

    Cliquez sur Tout visualiser pour afficher tous les services fonctionnels du domaine. Cela efface les paramètres de recherche de la recherche précédente.

    Tableau 18-21 Page Récapitulatif des services fonctionnels
    Propriété
    Description
    Nom
    Nom unique pour le service fonctionnel. Cliquez dessus pour afficher la page Visualiser un service fonctionnel - Détails de configuration.
    Reportez-vous à Modification des configurations des services fonctionnels.
    Chemin
    Le chemin est le nom du projet et le nom du dossier dans lequel réside le service fonctionnel, par exemple, UDDI/BusServices/OSB_services.
    Cliquez sur le chemin d'un service fonctionnel pour afficher la page page Vue du projet/Vue de dossier qui le contient.
    Actions
    Exécutez l'une des actions suivantes :
    • Cliquez sur l'icône Lancer la console de test pour appeler la console de test, que vous utilisez pour valider et tester la conception de vos services et de vos transformations. Pour les services fonctionnels, vous pouvez utiliser la console de test uniquement lors de l'exécution, c'est-à-dire lorsque la session est activée. Pour les transformations, vous pouvez utiliser la console de test, que vous soyez dans une session ou non. Reportez-vous aux rubriques relatives au test des services et au test des transformations.
    • L'icône Exporter le WSDL apparaît pour les services fonctionnels de type WSDL. Cliquez dessus pour exporter une ressource WSDL que vous pouvez ensuite visualiser ou modifier dans un outil externe tel qu'un IDE.

      • Cette fonction est différente de la fonction Exporter les ressources du module Administration du système, que vous utilisez pour déplacer et préparer les ressources entre deux domaines. Reportez-vous à Export d'un WSDL.

    • L'icône Générer le WSDL apparaît pour les services fonctionnels dotés d'un type de transport, tels qu'EJB et JEJB (mais pas pour les services de flux Fractionnement-Fusion). Cliquez sur cette icône pour générer un WSDL, que vous pourrez ensuite visualiser ou modifier. Reportez-vous à Génération d'un WSDL.
    Options
    • Cliquez sur l'icône Supprimer pour supprimer le service. Une icône Avertissement relatif à la suppression apparaît lorsque d'autres ressources font référence à cette ressource. Vous pouvez supprimer la ressource à condition de confirmer l'opération lorsqu'un avertissement s'affiche. Toutefois, des conflits peuvent survenir, les références à la ressource supprimée n'étant pas·résolues. Pour plus d'informations, reportez-vous à Suppression de services fonctionnels.
