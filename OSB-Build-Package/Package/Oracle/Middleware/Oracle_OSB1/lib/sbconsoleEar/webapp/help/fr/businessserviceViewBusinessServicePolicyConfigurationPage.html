Page Stratégies de service fonctionnel

La page Visualiser un service fonctionnel - Configuration des stratégies est l'une des pages destinées à la création et à la modification de services fonctionnels, décrites dans les rubriques Création et configuration de services fonctionnels et à la page Visualiser un service fonctionnel - Détails de configuration.

Utilisez cette page pour configurer les paramètres de stratégie des services fonctionnels de type Service Web XML WSDL, Service de messagerie, N'importe quel service SOAP ou N'importe quel service XML. Le tableau 18-23 explique comment l'utiliser. Pour pouvoir utiliser les stratégies OWSM avec un service proxy Web XML WSDL, un service proxy de messagerie ou n'importe quel service proxy XML, vous devez définir le protocole sur HTTP.

Pour les services de type WSDL, les stratégies WLS 9.2 liées au service sont exposées (en mode Inline) dans le WSDL en vigueur. Les stratégies de type Abstract sont pré-traitées avant d'être mises en mode Inline. Les stratégies OWSM ne sont pas mises en mode Inline dans le WSDL en vigueur mais liées par référence.

Configuration des stratégies de service
Voici les options du champ Configuration des stratégies de service :

Stratégies OWSM

  • A partir de la banque de stratégies OWSM : les stratégies sont définies dans la banque de stratégies centrale d'Oracle Web Services Manager gérée par Oracle Enterprise Manager. Ces stratégies prennent en charge WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken Profile et d'autres normes du secteur.

    Avec cette option, vous pouvez ajouter des stratégies à partir d'une liste prédéfinie. Cliquez sur Ajouter sous Stratégies de niveau service pour afficher la liste. Utilisez le filtrage si nécessaire pour trouver les stratégies souhaitées. Pour les services fonctionnels, seules les stratégies côté client sont affichées.

    Si un WSDL utilisé pour créer un service fonctionnel contient des stratégies de sécurité WS standard imbriquées, Oracle Service Bus génère une erreur et affiche un conflit. Pour résoudre ce conflit, remplacez les stratégies WSDL imbriquées par des stratégies Oracle Web Services Manager compatibles en sélectionnant A partir de la banque de stratégies OWSM et en cliquant sur Ajouter les stratégies compatibles. Oracle Service Bus tente, dans la mesure du possible, de trouver la stratégie correspondant le plus dans la banque de stratégies Oracle Web Services Manager en fonction de la stratégie standard imbriquée dans le WSDL. L'algorithme peut renvoyer une ou plusieurs stratégies correspondantes, ou n'en renvoyer aucune.

    Après avoir cliqué sur Ajouter les stratégies compatibles, si Oracle Service Bus ne trouve aucune stratégie compatible, cliquez sur Ajouter et sélectionnez les stratégies Oracle Web Services Manager appropriées. Si Oracle Service Bus renvoie plusieurs stratégies compatibles, supprimez celles que vous ne souhaitez pas utiliser.

    Remarque : cette option n'est activée que si vous avez configuré votre domaine pour utiliser Oracle Web Services Manager. Pour plus d'informations, reportez-vous à Securing Oracle Service Bus with Oracle Web Services Manager dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Stratégies WLS 9

  • A partir du langage WSDL : sélectionnez cette option si la stratégie de service est associée au langage WSDL sur lequel le service est basé. Ces stratégies prennent en charge WS-Security 1.0, SAML 1.1 et d'autres normes du secteur.

    Cette option vous permet de visualiser (en lecture seule) les stratégies de demande et de réponse à partir du WSDL.

    Remarque : si vous recevez une erreur de configuration du service fonctionnel concernant des stratégies WS-Security non prises en charge par Oracle Service Bus, utilisez l'option A partir de la banque de stratégies OWSM pour remplacer les stratégies WSDL imbriquées par des stratégies OWSM compatibles.

  • A partir d'une stratégie prédéfinie ou d'une ressource de stratégie WS : cette option permet d'ajouter, à partir de la console d'administration d'Oracle Service Bus, des stratégies de niveau service, des stratégies de niveau opération (auquel cas la stratégie s'applique aux messages de demande et de réponse), des stratégies de demande et des stratégies de réponse.

    Les stratégies sont prédéfinies dans Oracle WebLogic Server ou définies par l'utilisateur dans Oracle Service Bus avec une ressource de stratégie WS. Ces stratégies prennent en charge WS-Security 1.0, SAML 1.1 et d'autres normes du secteur.

    Cette option vous permet d'ajouter des stratégies aux niveaux service, opération, demande et réponse. Cliquez sur Ajouter sous la section appropriée pour sélectionner des stratégies dans une liste.

Remarque : les modèles de liaison de stratégie s'excluent mutuellement. Vous devez utiliser un seul type de stratégie dans un service. Lorsque vous liez des stratégies directement au service, toutes les stratégies de type WSDL sont ignorées.

Après avoir ajouté des stratégies, vous pouvez indiquer des remplacements sur la page Sécurité.

 

Après avoir fini

Cliquez sur Mettre à jour pour enregistrer cette configuration ou sur Réinitialiser pour annuler vos modifications.
