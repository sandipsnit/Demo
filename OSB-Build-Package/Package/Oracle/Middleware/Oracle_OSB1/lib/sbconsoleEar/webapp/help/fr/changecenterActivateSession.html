Activation de sessions

  1. Cliquez sur Activer dans le centre des modifications. La page Activer une session affiche le nom de la session, le nom utilisateur et un champ descriptif.

    2. Les sessions peuvent être créées et supprimées, indépendamment des autres activités dans le système. Cependant, si une autre session est en cours d'activation, la page Activer une session affiche une erreur désignant l'utilisateur ayant des modifications du serveur WebLogic en attente. Pour plus d'informations sur la résolution des conflits entre les modifications effectuées dans votre session et d'autres sessions activées, reportez-vous à Visualisation et résolution des conflits.

  2. En cas d'erreurs de validation, un message d'erreur apparaît, indiquant que votre session comporte des erreurs de validation. Visualisez et résolvez les conflits de configuration avant de continuer. Pour plus d'informations sur la résolution des conflits, reportez-vous à Visualisation et résolution des conflits.

    3. Vous ne pourrez pas activer la session avant d'avoir visualisé tous les conflits. Si de nouveaux conflits surviennent alors que vous visualisez les conflits existants, avant l'activation, un message instantané apparaît pour vous informer des nouveaux conflits.

  3. Entrez une description dans le champ Description. Elle apparaît dans la colonne Description lorsque vous cliquez sur Visualiser les modifications sous Centre des modifications pour afficher les modifications de configuration dues à l'activation de sessions.
  4. Pour activer la session, cliquez sur Soumettre. Si aucun nouveau conflit n'est survenu entre-temps, la session se termine et la configuration est déployée vers l'exécution.
Remarque : Lorsque vous essayez d'activer une session avec un URI endpoint JMS sur un autre serveur (un serveur unique autre que celui sur lequel vous travaillez ou un serveur géré dans un cluster), assurez-vous que le serveur de destination est disponible.
Remarque : Oracle Service Bus n'autorise pas l'inscription de services proxy avec le transport JMS si l'URL endpoint JMS indique une destination inaccessible. En d'autres termes, pour les services JMS, Oracle Service Bus vérifie si la fabrique de connexions indiquée existe. Si ce n'est pas le cas, une erreur d'activation de session survient.
