Ordre d'annulation des tâches

Vous pouvez annuler les tâches dans n'importe quel ordre (à condition que chaque action d'annulation génère des données valides). L'opération d'annulation définit la valeur d'une ressource en fonction de sa valeur avant sa modification.

Si la tâche en cours d'annulation a créé un objet, il n'existe aucun état précédent dans lequel il peut être restauré. En d'autres termes, aucun objet n'existait avant la réalisation de la tâche. En réalité, l'opération d'annulation supprime le nouvel objet de la session. Dans ce cas, des erreurs surviennent pour les objets qui font référence à l'objet supprimé. Vous pouvez visualiser ces erreurs sur la page Visualiser les conflits dans le centre des modifications.

Lorsque vous ne travaillez pas dans une session, vous pouvez accéder à la page Visualiser les modifications de configuration pour voir les sessions précédemment activées. Vous pouvez annuler ces sessions. Le système ne vous permet pas d'annuler une session précédemment activée si cette action d'annulation entraîne une erreur de configuration de l'exécution. Par exemple, si vous essayez d'annuler l'activation d'une session entraînant la suppression d'un objet référencé par un autre, cette action d'annulation n'est pas autorisée.

Il est possible d'annuler une action d'annulation. Dans la colonne Options de la zone Annulation de l'élément [tâche], cliquez sur l'icône Annuler. Oracle Service Bus prend en charge un nombre illimité d'annulations. Vous pouvez donc annuler l'annulation qui avait annulé une annulation précédente et ainsi de suite.
