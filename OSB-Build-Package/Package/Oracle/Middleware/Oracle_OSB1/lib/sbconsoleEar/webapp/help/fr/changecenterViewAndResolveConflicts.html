Visualisation et résolution des conflits

Cette page permet de visualiser les messages de diagnostic concernant les erreurs de configuration, et de visualiser et résoudre les conflits entre les modifications apportées dans votre session et d'autres sessions activées. Le lien de visualisation des conflits affiche également le nombre de conflits actifs dans la session.

Visualisation des conflits
  1. Cliquez sur Visualiser les conflits sous Centre des modifications. La page Visualiser les conflits affiche tout ou partie des catégories d'informations suivantes selon la nature des conflits :
    • Les erreurs (l'icône Erreur représente les conflits critiques impossibles à valider) vous informent des conflits critiques dans votre configuration. Vous ne pouvez pas valider les modifications avant de résoudre les conflits. Reportez vous au tableau 2-6 et à Résolution de conflits.
    • Les mises à jour simultanées (l'icône Avertissement représente les conflits non critiques pouvant être validés) vous avertissent des modifications incompatibles avec d'autres sessions activées. Reportez-vous au tableau 2-7.
    • Les messages d'information (l'icône Avertissement représente les conflits non critiques pouvant être validés) vous informent des conflits non critiques de votre configuration.

  2. Pour afficher les détails d'un conflit en particulier, cliquez sur le lien dans la colonne Nom de ce conflit.

    3. Si l'objet que vous avez modifié dans une session a changé lors de l'exécution depuis que vous avez commencé à modifier la session en cours, la page Visualiser les conflits affiche les informations présentées dans le tableau 2-7.

Résolution de conflits

Pour résoudre un conflit, utilisez les informations fournies dans la colonne Messages (comme décrit dans le tableau 2-6) pour comprendre le problème, puis modifiez l'objet à l'origine du conflit pour résoudre le problème.

Si vous avez un conflit de mise à jour simultanée, c'est-à-dire un conflit qui survient si la ressource modifiée dans la session en cours a déjà été modifiée et activée par une autre session (comme décrit dans le tableau 2-7), vous pouvez le résoudre de deux manières :

  • Cliquez sur Activer sous Centre des modifications. Cela enregistre vos modifications lors de l'exécution, ce qui signifie que vous remplacez les modifications déployées dans les données centrales par l'activation d'une précédente session. En d'autres termes, les modifications enregistrées par l'autre utilisateur dans une autre session sont écrasées par vos modifications dans cette session.
  • Cliquez sur l'icône Synchroniser dans la colonne Synchroniser du tableau (comme décrit dans le tableau 2-7. Cette action rétablit, dans cette session, l'état dans lequel cet objet a été enregistré lors de l'exécution.
