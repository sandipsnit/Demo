Visualisation des modifications de configuration

Cette page affiche différentes informations si vous êtes dans une session (c'est-à-dire si vous avez cliqué sur Modifier dans le centre des modifications) ou si vous êtes en dehors d'une session.

  • Lorsque vous êtes dans une session, la page Visualiser les modifications de configuration affiche la liste des modifications de configuration que vous avez apportées pendant la session en cours. Reportez-vous à Visualisation des modifications de configuration dans une session.
  • Lorsque vous êtes en dehors d'une session, la page Visualiser les modifications de configuration affiche la liste des modifications de configuration générées par l'activation de précédentes sessions. Reportez-vous à Visualisation des modifications de configuration générées par l'activation de sessions.
Visualisation des modifications de configuration dans une session
  1. Si ce n'est pas encore fait, cliquez sur Créer pour créer une session et modifier la configuration en cours. Reportez-vous à Utilisation du centre des modifications.
  2. Apportez au moins une modification à la configuration.
  3. Cliquez sur Visualiser les modifications sous Centre des modifications. Pour chaque modification apportée à la configuration pendant la session en cours, la page Visualiser les modifications de configuration affiche les informations présentées dans le tableau 2-2.

    4. Tableau 2-2 Modifications de configuration dans une session
    Propriété
    Description
    Tâche
    Description de la tâche implémentée. La tâche est un lien vers la page Détails de la tâche. Reportez-vous à Visualisation des détails de tâche.
    Temps d'exécution
    Date et heure auxquelles la tâche a été exécutée.
    Utilisateur
    Nom de l'utilisateur qui a implémenté la modification.
    Statut de tâche
    Statut de la tâche :
    Terminé : la tâche est terminée.
    Annulé : la tâche a été annulée.
    Annulation terminée : l'annulation est terminée.
    Annulé par
    Nom de l'utilisateur qui a annulé la tâche. Ce champ affichera Aucun si la tâche n'a pas été annulée.
    Options
    Cliquez sur l'icône Annuler pour inverser l'exécution de la tâche.
    Lorsque vous travaillez dans une session, vous pouvez annuler les tâches dans n'importe quel ordre.
    Après l'annulation d'une tâche, l'objet de la tâche reprend l'état qu'il avait avant l'exécution de la tâche en question. Cependant, toutes les tâches réalisées sur le même objet après la tâche annulée sont également annulées. Reportez-vous à Annulation des tâches.

Visualisation des modifications de configuration générées par l'activation de sessions
  1. Assurez-vous que vous n'êtes pas dans une session. Reportez-vous à Sessions existantes.
  2. Cliquez sur Visualiser les modifications sous Centre des modifications. Pour chaque session précédemment activée, la page Visualiser les modifications de configuration affiche les informations présentées dans le tableau 2-3.

    3. Tableau 2-3 Modifications de configuration activées
    Propriété
    Description
    Tâche
    Description de la session activée. La tâche est un lien vers la page Détails de la tâche, qui affiche les opérations réalisées dans la session. Reportez-vous à Visualisation des détails de tâche.
    Description
    Description supplémentaire de la session activée, si une description a été entrée lorsque la session a été activée via la page Activer une session.
    Temps d'exécution
    Date et heure d'activation de la session.
    Utilisateur
    Nom de l'utilisateur qui a activé la session.
    Statut de tâche
    Statut de la session :
    Activé : la session a été activée.
    Partiellement activé : apparaît pendant l'activation de la session si les serveurs d'un cluster sont inaccessibles, ce qui empêche l'activation de cette session sur ces serveurs. Lorsque les serveurs deviennent accessibles, toutes les modifications inactivées seront alors activées sur ces serveurs.
    Annulé : la session précédemment activée et toutes les opérations réalisées dans cette session ont été annulées.
    Annulation activée : l'annulation a été activée.
    En cours : apparaît si l'activation de la session est en cours, car l'activation des sessions peut parfois prendre un certain temps.
    Echec : apparaît en cas d'échec de l'activation d'une session. Oracle Service Bus suit les échecs d'activation de session mais pas les échecs dus à chaque mise à jour au cours d'une session.
    Annulé par
    Nom de l'utilisateur qui a annulé la tâche. Ce champ affichera Aucun si l'activation de session n'a pas été annulée.
    Options
    Cliquez sur l'icône Annuler pour inverser l'activation de session et les opérations réalisées dans cette session. Vous pouvez annuler les activations de session dans n'importe quel ordre.
    Cliquez sur l'icône Annuler la tâche dans la session pour créer une session pré-remplie avec les tâches nécessaires pour annuler les modifications activées dans la session. Reportez-vous à Annulation dans une session.
