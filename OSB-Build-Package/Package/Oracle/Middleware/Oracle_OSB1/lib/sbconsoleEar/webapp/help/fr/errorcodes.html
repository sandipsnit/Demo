Codes d'erreur

Cette section décrit les erreurs Oracle Service Bus pouvant survenir lors de l'exécution d'un service proxy. Les codes associés à ces erreurs apparaissent dans l'élément de la variable de contexte fault. Vous pouvez accéder à cette valeur à l'aide de l'instruction XQuery suivante :

$fault/ctx:errorCode/text()

Les erreurs sont accompagnées des détails associés dans l'élément fault.

Le tableau A-1 répertorie les codes d'erreur Oracle Service Bus.

Codes d'erreur de sécurité (de 386000 à 386999)
BEA-386000
Erreur de sécurité générale. Raisons possibles :
  • L'authentification personnalisée du jeton ou du nom utilisateur/mot de passe est configurée, mais l'authentification via la sécurité WS a déjà commencé.
 
BEA-386100
Echec de l'authentification. Raisons possibles :
  • L'expression XPath a été configurée de façon à agir sur le corps ou l'en-tête mais cette partie de message n'existe pas (la liaison des couches renvoie une valeur NULL).
  • Le moteur XPath génère une exception lors de l'évaluation de l'expression Xpath.
  • Le résultat de l'expression XPath de nom utilisateur ou de mot de passe est un noeud qui n'est pas de type TEXT ou ATTR.
  • L'expression XPath renvoie plusieurs noeuds.
  • Un nom utilisateur/mot de passe a été extrait mais l'authentification a échoué.
  • Un jeton personnalisé a été extrait mais l'assertion d'identité a échoué (le service d'assertion d'identité CSS génère une exception LoginException ou toute autre exception).
 
BEA-386101
Jeton d'authentification absent. Raisons possibles :
  • L'expression XPath de nom utilisateur ou de mot de passe renvoie un résultat ou une chaîne vide.
  • L'expression XPath de jeton personnalisé renvoie un résultat vide, une valeur d'attribut ou une valeur de texte NULL.
 
BEA-386102
Autorisation de niveau message refusée. Raisons possibles :
  • La stratégie de contrôle d'accès de niveau message refuse l'accès au service proxy.
 
BEA-386103
L'algorithme de sélection de l'opération de service proxy ne peut pas déterminer le nom de l'opération à partir de la demande ou il renvoie une opération non valide (qui n'est pas dans le WSDL ou est NULL). Raisons possibles :
  • Erreur lors du calcul de l'opération.
  • L'algorithme de sélection de l'opération renvoie une valeur NULL.
  • L'algorithme de sélection de l'opération renvoie une opération qui ne fait pas partie de celles déclarées par le WSDL.

Schémas des détails des erreurs

Vous trouverez ci-dessous les définitions de schéma des fragments de code XML accompagnant les codes d'erreur répertoriés dans le tableau A-1. Ces fragments de code XML apparaissent dans la variable de contexte fault sous l'élément <ctx:details>. Ces schémas ne sont pas tous issus d'un espace de noms unique. En outre, alors qu'un code d'erreur donné implique généralement un fragment de code XML particulier, certains fragments de code XML sont utilisés par plusieurs codes d'erreur. 

<xs:schema targetNamespace="http://www.oracle.com/wli/sb/stages/transform/config"
           xmlns:tns="http://www.oracle.com/wli/sb/stages/transform/config"
	   xmlns:xs="http://www.w3.org/2001/XMLSchema"
           xmlns:ref="http://www.oracle.com/wli/sb/reference"
           xmlns:sdk="http://www.oracle.com/wli/sb/stages/config"
	   elementFormDefault="qualified"
	   attributeFormDefault="unqualified">

            <xs:element name="faultcode" type="xs:QName"/>
            <xs:element name="faultstring" type="xs:string"/>
            <xs:element name="detail" minOccurs="0" >
                <xs:complexType mixed="true" >
