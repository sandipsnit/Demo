Liaisons JCA

Oracle SOA Suite permet l'intégration à un système back-end Enterprise Information Systems (EIS) via les adaptateurs J2EE Connector Architecture (JCA). Avec Oracle JDeveloper et d'autres environnements de développement JCA pris en charge, vous pouvez configurer les adaptateurs et créer les ressources d'adaptateur JCA (fichiers .jca et WSDL abstraits) utilisés pour l'intégration back-end.

Vous pouvez communiquer avec les composants EIS compatibles JCA via Oracle Service Bus (OSB) ; cela vous permet d'employer les ressources JCA sous forme de ressources de liaison JCA. Avec les liaisons JCA, vous pouvez générer les services proxy et fonctionnels qui utilisent le transport JCA Oracle Service Bus pour communiquer avec les applications EIS via les adaptateurs JCA.

Cette section, qui décrit comment ajouter, modifier, localiser et supprimer les liaisons JCA dans Oracle Service Bus, contient les rubriques suivantes :

  • Localisation des liaisons JCA
  • Ajout de liaisons JCA
  • Modification des liaisons JCA
  • Suppression des liaisons JCA

Pour plus d'informations sur le transport JCA, reportez-vous à "JCA Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV910.

Pour plus d'informations sur les adaptateurs JCA Oracle SOA Suite, reportez-vous au manuel Oracle Fusion Middleware User's Guide for Technology Adapters à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=TKADP.
