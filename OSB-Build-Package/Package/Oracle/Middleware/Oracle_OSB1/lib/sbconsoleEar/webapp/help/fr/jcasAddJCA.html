Ajout de liaisons JCA

La méthode la plus simple de création de liaisons JCA dans Oracle Service Bus consiste à importer globalement les ressources JCA créées (fichier .jca, WSDL associé, et fichier de mapping TopLink ou EclipseLink). Lors de l'import global, Oracle Service Bus génère automatiquement une liaison JCA à partir des fichiers .jca, des ressources WSDL à partir des WSDL JCA associés, et des ressources de document XML à partir des fichiers de mapping. De plus, il gère les dépendances entre les fichiers.

Pour plus d'informations sur l'import, reportez-vous à Chargement de ressources à partir d'un fichier ZIP, Chargement de ressources à partir d'une URL et Import de ressources.

Une fois vos ressources JCA importées, vous pouvez générer les services proxy ou fonctionnels à partir de ces dernières, comme décrit dans Génération d'un service proxy à partir d'une ressource de liaison JCA et Génération d'un service fonctionnel à partir d'une ressource de liaison JCA.

Si vous n'importez pas globalement vos ressources JCA, vous pouvez créer manuellement les liaisons JCA comme suit :

  1. Dans Oracle JDeveloper, créez un WSDL de service JCA, un fichier JCA (.jca) correspondant et, si nécessaire, d'autres ressources telles que des fichiers de mapping TopLink ou EclipseLink et des fichiers  XSLT.

    2. Pour plus d'informations, reportez-vous au manuel Oracle Fusion Middleware User's Guide for Technology Adapters à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=TKADP.

  2. Si ce n'est pas déjà fait, cliquez sur Créer pour créer une session dans la console d'administration OSB ou sur Modifier pour indiquer une session existante. Reportez-vous à Utilisation du centre des modifications.
  3. Dans la console d'administration OSB, créez ou importez les ressources JCA que vous avez créées dans Oracle JDeveloper, telles que les ressources WSDL, XML et XSLT.
  4. Créez la liaison JCA.
    1. Sélectionnez Explorateur de projets, puis choisissez l'emplacement du projet ou du dossier de la liaison JCA. La page Vue du projet/Vue de dossier apparaît.
    2. Dans le champ Créer une ressource, sélectionnez Liaison JCA pour afficher la page Créer une ressource de liaison JCA.
    3. Dans le champ Fichier JCA, cliquez sur Parcourir pour sélectionner le fichier JCA.

      4. Le champ Nom de la ressource est automatiquement rempli avec le nom du fichier (sans son extension).

      Au lieu de sélectionner un fichier, vous pouvez copier et coller le contenu d'un fichier JCA dans la zone de texte du champ Fichier JCA.

    4. Dans le champ Nom de la ressource, entrez ou modifiez le nom de la liaison JCA. Les noms de liaison JCA doivent être uniques.

      Pour obtenir des conseils sur la dénomination, suivez les restrictions concernant la dénomination des ressources.
    5. Dans le champ Description de la ressource, entrez la description de cette liaison JCA.
    6. Cliquez sur Enregistrer. La liaison JCA est enregistrée dans la session en cours.

      7. Un avertissement de validation apparaît si la liaison JCA ne référence pas encore les ressources obligatoires, telles qu'un WSDL et un fichier de mapping.

  5. Cliquez sur le nom de la nouvelle liaison JCA afin de l'afficher sur la page Visualiser une liaison JCA.
  6. Cliquez sur Modifier les références.
  7. Sur la page Modifier la référence d'une ressource de liaison JCA, cliquez sur Parcourir pour sélectionner les ressources de référence de la liaison  JCA, telles que le WSDL, un fichier de mapping TopLink ou EclipseLink et un fichier  XSLT.
  8. Cliquez sur Enregistrer.
  9. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.

Une fois que vous avez ajouté les ressources de liaison JCA et configuré les dépendances de ressource (en associant un WSDL), ou si vous avez importé les ressources JCA qui gèrent leurs dépendances, vous pouvez générer des services proxy et fonctionnels à partir des liaisons JCA. Pour plus d'informations, reportez-vous à Génération d'un service proxy à partir d'une ressource de liaison JCA et Génération d'un service fonctionnel à partir d'une ressource de liaison JCA.
