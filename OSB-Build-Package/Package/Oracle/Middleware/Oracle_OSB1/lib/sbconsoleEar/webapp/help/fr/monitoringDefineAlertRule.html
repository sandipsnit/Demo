Définition des conditions de règle d'alerte

Utilisez cette page pour définir les conditions de la règle d'alerte. Vous devez indiquer au moins une condition. Si vous en indiquez plusieurs, vous devez utiliser les opérateurs And/Or pour les combiner.

  1. Dans le champ Sélectionnez l'intervalle d'agrégation pour la condition, sélectionnez le nombre d'heures et de minutes pour définir l'intervalle d'agrégation de cette condition. L'intervalle d'agrégation détermine la fréquence de test de la condition. Cette dernière est testée chaque fois que le sous-système de surveillance rassemble suffisamment d'échantillons de données pour constituer un intervalle d'agrégation.

    2. Par exemple, si vous sélectionnez un intervalle d'agrégation d'une heure, la condition est testée chaque fois qu'une quantité de données correspondant à une heure est disponible. La condition est testée pour la première à la fin de la première heure. Après quoi, elle est testée toutes les 10 minutes car l'intervalle d'échantillonnage pour un intervalle d'agrégation d'une heure est défini sur 10 minutes.

  2. Vous devez commencer par définir une expression simple. Vous pouvez combiner au moins deux expressions simples pour former une expression complexe. Pour définir une expression simple, procédez comme suit :
    1. Dans le premier champ déroulant, sélectionnez Nombre, Minimum, Maximum, Moyenne ou Statut.
    2. Dans le champ déroulant suivant, sélectionnez un opérande.

      3. La liste des opérandes varie selon que vous avez sélectionné Nombre, Minimum, Maximum, Moyenne ou Statut dans le premier champ déroulant. Par exemple, si vous sélectionnez Minimum, Maximum ou Moyenne, l'opérande Temps de réponse est disponible. Utilisez-le pour définir le temps de réponse en millisecondes (ms). Les opérandes disponibles dépendent également de la configuration du service. Le nombre d'opérandes varie selon qu'un service comporte des pipelines, des noeuds de routage, des opérations, etc.

      Pour plus d'informations sur la génération des alertes basées sur le statut d'URI endpoint, reportez-vous à "Managing Endpoint URIs for Business Services" dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG595.

      Lorsque vous sélectionnez Nombre, ce champ affiche les opérandes indiqués dans le tableau 25-21.

    3. Dans le champ suivant, sélectionnez l'opérateur de comparaison voulu : =, !=, > ou <.
    4. Dans le champ suivant, indiquez la valeur de comparaison.
    5. Cliquez sur Ajouter.
  3. Pour définir une expression complexe, procédez comme suit :
    1. Répétez les étapes a à e de l'étape 2 pour définir une expression simple.
    2. Répétez encore une fois ces étapes pour définir une autre expression simple.
    3. Cochez la case des deux expressions.
    4. Cliquez sur AND ou OR pour combiner les expressions et former une expression complexe.

      5. L'option Fractionnement est disponible lorsque vous avez combiné plusieurs expressions. Elle permet de fractionner les expressions complexes en différentes expressions simples.

Après avoir fini

Cliquez sur Préc pour revenir à la page Configuration générale. Cliquez sur Dernier pour afficher la page Nouvelle règle d'alerte - [nom de service]. Passez à la section Vérification de la configuration des règles d'alerte.
