Configuration des informations générales concernant les règles d'alerte

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Sélectionnez Navigateur Web de ressources > Services proxy ou Services fonctionnels.
  3. Cliquez sur le nom du service.
  4. Cliquez sur l'onglet Règles d'alerte de contrat de niveau de service, puis sur Ajouter une valeur.
  5. Dans le champ Nom de la règle, saisissez le nom de la règle d'alerte. Ce champ est obligatoire.
  6. Dans le champ Récapitulatif des alertes, entrez une brève description de la règle d'alerte. Ce texte est également utilisé comme ligne d'objet d'un message électronique si la règle d'alerte est configurée avec une notification par courrier électronique. Cette description ne doit pas contenir plus de 80 caractères. Si vous n'indiquez pas de récapitulatif d'alerte, le texte par défaut, Alerte Oracle Service Bus, sera utilisé à la place.
  7. Dans le champ Description de la règle, entrez la description de la règle d'alerte.
  8. Dans le champ Destination de l'alerte, saisissez le nom de la ressource de destination d'alerte ou cliquez sur Parcourir pour afficher la page Sélectionner une destination d'alertes. Sélectionnez une destination d'alerte dans la liste et cliquez sur Soumettre.

    9. Par défaut, l'alerte sera toujours envoyée à la console d'administration d'Oracle Service Bus. Pour configurer et sélectionner d'autres ressources de destination d'alerte, reportez-vous à Ajout de destinations d'alertes et Localisation de destinations d'alertes.

  9. Dans le champ Heure de début, indiquez une heure de début au format HH:MM. Par exemple, saisissez 09:00. Les champs Heure de début et Heure de fin permettent d'indiquer une fenêtre de temps au cours de laquelle la règle est active chaque jour avant la date d'expiration. Par exemple, tous les jours entre 9:00 et 17:00.
  10. Dans le champ Heure de fin, indiquez une heure de fin au format HH:MM. Par exemple, saisissez 17:00. Les champs Heure de début et Heure de fin permettent d'indiquer une fenêtre de temps au cours de laquelle la règle est active chaque jour avant la date d'expiration. Par exemple, tous les jours entre 9:00 et 17:00.

    11. La règle d'alerte est active tous les jours pendant la période indiquée (via la définition des heures de début et de fin) jusqu'à ce que la règle expire.

  11. Dans le champ Date d'expiration de la règle, indiquez une date d'expiration au format MM/DD/YYYY. Par exemple, saisissez 12/31/2010. La règle expire à 23:59 à la date indiquée. Si vous ne définissez aucune date, la règle n'expire jamais.
  12. Dans le champ Règle activée, conservez Oui comme valeur par défaut pour vous assurer que cette règle est activée ou sélectionnez Non pour désactiver cette règle.
  13. Dans le champ Gravité de l'alerte, sélectionnez une des valeurs suivantes :
    • Normal
    • Avertissement
    • Mineur
    • Majeur
    • Critique
    • Fatal
  14. Dans le champ Fréquence d'alerte, sélectionnez l'un des paramètres suivants, répertoriés dans le tableau 25-20.

    15. Tableau 25-20 Définition de la fréquence d'alerte
    Propriété
    Description
    A chaque fois
    Si vous sélectionnez cette option, les actions comprises dans la règle d'alerte sont exécutées chaque fois que la règle d'alerte renvoie la valeur True. Par exemple, si vous définissez la condition selon laquelle le temps moyen de réponse est supérieur à 300 millisecondes, vous recevez une alerte chaque fois que cette condition renvoie la valeur True.
    Le nombre d'évaluations d'une règle d'alerte dépend de l'intervalle d'agrégation et de l'intervalle échantillon associés à la règle. Si l'intervalle d'agrégation est défini sur 5 minutes, l'intervalle d'échantillon est d'une minute. Les règles sont évaluées chaque fois que 5 échantillons de données sont disponibles. Par conséquent, la règle est évaluée pour la première fois environ 5 minutes après sa création, puis toutes les minutes.
    Notifier une fois
    Si vous sélectionnez cette option, les actions comprises dans la règle sont exécutées la première fois que la règle renvoie la valeur True. En outre, pour qu'une autre alerte soit générée, la condition doit se réinitialiser et renvoyer de nouveau la valeur True. Par exemple, si vous définissez une condition selon laquelle le temps moyen de réponse est inférieur à 300 millisecondes, vous recevez une alerte la première fois que cette condition renvoie la valeur True. Toutefois, vous ne recevrez pas d'autre alerte tant que la condition ne renverra pas la valeur False, puis de nouveau True. L'horodatage de l'alerte est mis à jour et affiché dans le tableau de bord.

  15. Sélectionnez Oui pour l'option Arrêter le traitement des règles supplémentaires si vous souhaitez abandonner l'exécution d'autres règles lorsque l'une des règles associées à un service renvoie la valeur True. Utilisez cette option pour arrêter l'évaluation des prochaines règles lorsque plusieurs règles sont associées à un service en particulier. Conservez Non comme valeur par défaut pour continuer le traitement des règles.

    16. Le comportement de cette option est identique à celui de l'option Arrêter le traitement des règles supplémentaires dans l'assistant Gestion des messages de Microsoft Outlook.

Après avoir fini

Cliquez sur Suivant pour afficher la page Nouvelle règle d'alerte - Configuration des conditions. Passez à la section Définition des conditions de règle d'alerte.
