Visualisation des informations sur le serveur

La page Etat du serveur contient les éléments suivants :

  • Dans l'en-tête d'onglet Etat du serveur, une icône Arrêté et un indicateur numérique précisant le nombre de serveurs arrêtés peuvent être affichés :
    • Lorsque vous cliquez sur l'icône, les serveurs affichés sont filtrés selon les critères Arrêté pour Etat général et SHUTDOWN pour Etat.
    • L'indicateur numérique précise le nombre de serveurs arrêtés.
  • Un graphique à secteurs illustrant l'état général des serveurs du domaine. Vous pouvez cliquer sur la zone du graphique correspondant au serveur pour afficher plus de détails.
  • Le récapitulatif des journaux. Pour afficher le journal, cliquez sur le lien numérique indiquant le nombre de messages pour un niveau de gravité donné. Par exemple, Alerte : 1, cliquez sur le chiffre 1. Reportez-vous à Visualisation des détails des fichiers journaux de domaine.
  • Des champs affichant les informations sur le serveur, selon vos paramètres personnalisés pour la table Récapitulatif du serveur. Par exemple, le nom du serveur, son état général, le nom de cluster, le nom de l'ordinateur, l'état du serveur et la durée d'exécution du serveur. Pour en savoir plus sur ces champs, reportez-vous au tableau 25-16.
  • Par défaut, le taux d'actualisation du tableau de bord est défini sur Pas d'actualisation. Pour modifier l'intervalle de ce taux, reportez-vous à Définition des préférences utilisateur.
Attention : Lorsque vous sélectionnez un intervalle de fréquence d'actualisation du tableau de bord autre que celui par défaut (Pas d'actualisation), le navigateur Web actualise le tableau de bord à intervalles réguliers. Ceci évite la temporisation de votre session, même lorsque vous n'interagissez pas avec la console d'administration d'Oracle Service Bus.
Attention : Oracle recommande d'utiliser la fonctionnalité de fréquence d'actualisation du tableau de bord avec précaution. Vous ne devez jamais laisser une session sans surveillance. Lorsque votre session est active, d'autres personnes peuvent y accéder depuis votre ordinateur, sans qu'elles n'aient à s'authentifier à nouveau.
  • L'icône Outil de personnalisation de table. Reportez-vous à Personnalisation des vues tabulaires.
  • Pour limiter le nombre d'éléments dans la liste ou localiser des serveurs spécifiques, procédez comme suit :
    • Cliquez sur l'icône Open pour afficher les autres filtres de recherche. Si vous indiquez plusieurs critères de recherche, seuls les résultats correspondant à tous les critères seront renvoyés.
    • Filtrer par statut de l'état général. Dans la liste déroulante Etat général, sélectionnez une valeur pour limiter les serveurs au niveau du statut de l'état général indiqué, puis cliquez sur Rechercher.

      • Cochez la case ou supérieur pour limiter votre recherche au niveau du statut de l'état général ou aux niveaux supérieurs (ces niveaux sont affichés du plus général au plus restrictif) : OK, Avertissement, Surchargé, Critique et Fatal.

    • Filtrer par serveur. Dans la liste déroulante Serveur, sélectionnez Tout ou le nom du serveur, puis cliquez sur Rechercher.
    • Filtrer par nom de cluster. Dans la liste déroulante Nom de cluster, sélectionnez Tout ou le nom du cluster associé au serveur, puis cliquez sur Rechercher.
    • Filtrer par nom d'ordinateur. Dans la liste déroulante Nom de l'ordinateur, sélectionnez Tout ou le nom de l'ordinateur associé au serveur, puis cliquez sur Rechercher.
    • Filtrer par état. Dans la liste déroulante Etat, sélectionnez Tout ou l'état du serveur : RUNNING, FAILED ou SHUTDOWN, puis cliquez sur Rechercher.
    • Cliquez sur Tout visualiser pour enlever les filtres de recherche et afficher tous les serveurs.

La page Etat du serveur affiche les informations suivantes pour chaque serveur (tableau 25-16), selon les paramètres de filtre indiqués.

Tableau 25-16 Détails récapitulatifs du serveur
Propriété
Description
Serveur
Nom du serveur. Ce nom est un lien permettant d'accéder à la page Visualiser les détails du serveur. Reportez-vous à Visualisation des détails du serveur.
Etat général
Statut de l'état général du serveur :
  • OK : le serveur fonctionne sans problème.
  • OK/Pas de traitement : l'état du serveur est ADMIN. Il n'est disponible que pour les opérations d'administration. Tous les messages Oracle Service Bus d'exécution échoueront.
  • Avertissement : le serveur peut rencontrer des problèmes dans le futur. Pour plus de détails, consultez les journaux du serveur et le MBean d'exécution correspondant.
  • Surchargé : le volume de travail affecté au serveur est supérieur au seuil configuré ; il se peut qu'il refuse davantage de travail.
  • Critique : échec du serveur en suspens ; vous devez entreprendre une action immédiatement pour éviter un échec. Pour plus de détails, consultez les journaux du serveur et le MBean d'exécution correspondant.
  • Arrêté : le serveur est en échec et doit être relancé.
  • Critique : échec du serveur en suspens ; vous devez entreprendre une action immédiatement pour éviter un échec. Pour plus de détails, consultez les journaux du serveur et le MBean d'exécution correspondant.
Nom de cluster
Nom du cluster associé au serveur.
Nom d'ordinateur
Nom de l'ordinateur associé au serveur.
Etat
Etat du serveur :
  • RUNNING
  • ADMIN
  • FAILED
  • SHUTDOWN
Temps d'exécution
Durée depuis laquelle le serveur est exécuté.
