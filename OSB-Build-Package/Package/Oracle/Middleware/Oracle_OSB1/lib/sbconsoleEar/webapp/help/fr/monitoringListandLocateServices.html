Visualisation des informations sur la surveillance des services

La page Etat du service affiche les mesures des services pour lesquels la surveillance est activée. Pour savoir comment activer la surveillance des services, reportez-vous à Configuration des paramètres opérationnels des services proxy et Configuration de paramètres opérationnels pour les services fonctionnels.

Par défaut, le taux d'actualisation du tableau de bord est défini sur Pas d'actualisation. Pour modifier l'intervalle de ce taux, reportez-vous à Définition des préférences utilisateur.

Attention : Lorsque vous sélectionnez un intervalle de fréquence d'actualisation du tableau de bord autre que celui par défaut (Pas d'actualisation), le navigateur Web actualise le tableau de bord à intervalles réguliers. Ceci évite la temporisation de votre session, même lorsque vous n'interagissez pas avec la console d'administration d'Oracle Service Bus.
Attention : Oracle recommande d'utiliser la fonctionnalité de fréquence d'actualisation du tableau de bord avec précaution. Vous ne devez jamais laisser une session sans surveillance. Lorsque votre session est active, d'autres personnes peuvent y accéder depuis votre ordinateur, sans qu'elles n'aient à s'authentifier à nouveau.

Les icônes suivantes peuvent être affichées dans l'en-tête d'onglet Etat du service :

  • L'icône Avertissement et un indicateur numérique.
    • Lorsque vous cliquez sur cette icône, les services affichés sont filtrés en fonction des critères Comporte des erreurs.
    • L'indicateur numérique précise le nombre de services présentant des erreurs depuis la dernière actualisation de la page.
  • L'icône Adresses hors ligne et un indicateur numérique.
    • Lorsque vous cliquez sur cette icône, les services affichés sont filtrés en fonction des critères Comporte des URI endpoint hors ligne.
    • L'indicateur numérique précise le nombre de services présentant des adresses passées hors ligne depuis la dernière actualisation de la page.

La page Etat du service affiche les mesures de service et les informations sur la surveillance présentées dans le tableau 25-5.

  1. Dans le champ Afficher les statistiques, effectuez l'une des opérations suivantes :
    • Pour afficher les statistiques de surveillance de chaque service pour la période de l'intervalle d'agrégation en cours, sélectionnez Intervalle d'agrégation en cours. La vue Intervalle d'agrégation en cours présente une vue mobile des statistiques des mesures de service.
    • Pour afficher les statistiques de surveillance pour la période écoulée depuis la dernière réinitialisation des statistiques d'un service, sélectionnez Depuis la dernière réinitialisation. La vue Depuis la dernière réinitialisation affiche le nombre de mesures en cours d'exécution.
  2. Sélectionnez un serveur dans la liste déroulante pour afficher les mesures associées.

    3. S'il existe un cluster, les mesures globales du cluster sont affichées par défaut. Sélectionnez un serveur géré individuel pour afficher les mesures associées.

  3. Pour limiter le nombre d'éléments dans la liste ou localiser des services spécifiques, procédez comme suit :
    • Cliquez sur l'icône Open pour afficher les autres filtres de recherche. Si vous indiquez plusieurs critères de recherche, seuls les résultats correspondant à tous les critères seront renvoyés.
    • Filtrer par nom de service. Dans le champ Nom, saisissez le nom de la cible de recherche ou entrez des caractères génériques (utilisez les caractères génériques * et ? pour exécuter une recherche plus générale), puis cliquez sur Rechercher.

      • Cette méthode de recherche est conseillée si le nom du service est unique dans tous les projets et chemins.

    • Filtrer par chemin de service. Dans le champ Chemin, entrez le chemin de la cible de recherche, c'est-à-dire le nom de projet et le nom du dossier dans lequel les services résident. Vous pouvez utiliser les caractères génériques * et ? pour effectuer une recherche plus générale. Tous les services qui résident dans ce chemin sont affichés.

      • Le format du champ Chemin est le suivant :
      project-name/root-folder/ . . ./parent-folder

      Si un service se trouve directement sous le projet, le format est le suivant : project-name

    • Filtrer par nom et chemin de service. Cette méthode de recherche est conseillée si plusieurs services portant le même nom résident dans différents chemins.
    • Filtrer par service parent. Dans la liste déroulante Service, sélectionnez le type de service parent (Services proxy ou Services fonctionnels), puis cliquez sur Rechercher. Sélectionnez Tous les services afin que les critères de filtre de service parent ne soient pas pris en compte.
    • Filtrer par service avec des messages. Sélectionnez l'option Comporte des messages, puis cliquez sur Rechercher.
    • Filtrer par service avec des alertes. Sélectionnez l'option Comporte des alertes, puis cliquez sur Rechercher.
    • Filtrer par service avec des erreurs. Sélectionnez l'option Comporte des erreurs, puis cliquez sur Rechercher.
    • Filtrer par service avec des URI hors ligne. Sélectionnez l'option Comporte des URI endpoint hors ligne, puis cliquez sur Rechercher.
    • Filtrer par service appelé par un service proxy spécifique. Saisissez le nom et le chemin d'un service proxy dans le champ prévu à cet effet ou cliquez sur Parcourir pour afficher la page Sélectionner un service proxy. Sélectionnez le service de proxy dans la liste et cliquez sur Soumettre. Cliquez sur Rechercher. Les services fonctionnels appelés par le service proxy que vous avez indiqué sont affichés.

      • Les caractères génériques ne sont pas pris en charge dans le champ Appelé par le proxy.

    • Cliquez sur Tout visualiser pour enlever les filtres de recherche et afficher tous les services.

      • Tableau 25-5 Informations sur la surveillance des services
      Propriété
      Description
      Nom
      Nom affecté au service. Ce nom est un lien permettant d'accéder à la page Détails de la surveillance des services. Reportez-vous à Visualisation des mesures de service.
      Chemin
      Projet associé au service. Si le service réside dans un dossier de projet, ce dossier est également répertorié. Le chemin est affiché au format suivant :
      project-name/root-folder/ . . ./parent-folder
      Ce chemin est un lien permettant d'accéder au chemin correspondant dans l'explorateur de projets.
      Type de service
      Type du service parent : service proxy ou service fonctionnel.
      Intervalle d'agrégation
      Ce champ n'apparaît que si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques.
      Intervalle d'agrégation en cours défini pour la surveillance du service, en heures et en minutes. Vous devez définir cet intervalle sur la page Paramètres opérationnels. Reportez-vous à Définition de l'intervalle d'agrégation d'un service.
      Temps de réponse moyen
      Temps de réponse moyen (en ms) nécessaire au service pour exécuter les messages au cours de l'intervalle d'agrégation en cours ou depuis la dernière réinitialisation.
      Messages
      Nombre de messages associés au service pour l'intervalle d'agrégation en cours ou depuis la dernière réinitialisation.
      Erreurs
      Nombre de messages d'erreur associés au service pour l'intervalle d'agrégation en cours ou depuis la dernière réinitialisation.
      Alertes
      Ce champ est masqué par défaut.
      Nombre d'alertes (de contrat de niveau de service et de pipeline) associées au service pour l'intervalle d'agrégation en cours ou depuis la dernière réinitialisation.
      Pour un serveur géré individuel dans un cluster et les services proxy, ce champ indique le nombre d'alertes de pipeline uniquement associées au service pour l'intervalle d'agrégation en cours ou depuis la dernière réinitialisation.
      Pour les serveurs gérés individuels et les services fonctionnels, ce champ présente la valeur N/A.
      Alertes de contrat de niveau de service
      Nombre d'alertes de contrat de niveau de service associées au service pour l'intervalle d'agrégation en cours ou depuis la dernière réinitialisation.
      Pour un serveur géré individuel dans un cluster, le champ Alertes de contrat de niveau de service présente la valeur N/A (seules les alertes de contrat de niveau de service dans le cluster sont affichées).
      Alertes de pipeline
      Pour les services proxy uniquement :
      Nombre d'alertes de pipeline associées au service pour l'intervalle d'agrégation en cours ou depuis la dernière réinitialisation.
      Statut d'URI endpoint
      Pour les services fonctionnels uniquement, indique le statut des URI endpoint. Le statut est un lien permettant d'accéder à la page URI endpoint. Reportez-vous à Visualisation des mesures des URI endpoint de service fonctionnel.
      Pour un domaine à noeud unique (instances de serveur individuel) :
      • En ligne : tous les URI endpoint du service fonctionnel sont en ligne.
      • Hors ligne : tous les URI endpoint du service fonctionnel sont hors ligne.
      • Partiel : au moins un des URI endpoint du service fonctionnel est hors ligne.
      Pour un domaine inclus dans un cluster :
      • Lorsque le champ Serveur est défini sur Cluster ou sur un des serveurs gérés, le statut En ligne indique que tous les URI endpoint sont en ligne respectivement dans le cluster ou sur le serveur géré sélectionné.
      • Lorsque le champ Serveur est défini sur Cluster ou sur un des serveurs gérés, le statut Hors ligne indique que tous les URI endpoint sont hors ligne respectivement dans le cluster ou sur le serveur géré sélectionné.
      • Lorsque le champ Serveur est défini sur Cluster, le statut Partiel indique qu'au moins un des URI endpoint du service fonctionnel est hors ligne sur au moins un des serveurs ou qu'un de ces URI est hors ligne sur tous les serveurs. Toutefois, les autres URI endpoint de ce même service fonctionnel sont toujours disponibles sur un serveur ou sur tous les serveurs.
      • Lorsque le champ Serveur est défini sur l'un des serveurs gérés, le statut Partiel indique qu'au moins un des URI endpoint du service fonctionnel est hors ligne sur le serveur géré sélectionné.
      Action
      Ce champ n'apparaît que si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques.
      Dans cette colonne, vous pouvez cliquer sur l'icône Réinitialiser associée à un service spécifique afin de réinitialiser ses statistiques. Lorsque vous confirmez cette opération, le système supprime toutes les statistiques de surveillance collectées pour ce service depuis la dernière réinitialisation des statistiques. Toutefois, le système ne supprime pas celles collectées pendant l'intervalle d'agrégation en cours de ce service. En outre, une fois que vous avez cliqué sur l'icône Réinitialiser, le système recommence immédiatement à collecter les statistiques de surveillance du service.
