Purge des alertes

Utilisez cette page pour supprimer définitivement les alertes de contrat de niveau de service ou de pipeline indiquées.

Remarque : Cette action ne peut pas être annulée.
  1. Sélectionnez Opérations > Tableau de bord.
  2. Sélectionnez l'onglet Alertes de contrat de niveau de service ou Alertes de pipeline, puis cliquez sur le lien Historique étendu des alertes.
  3. En haut à droite de la page Historique étendu des alertes, cliquez sur Purger l'historique des alertes de contrat de niveau de service ou Purger l'historique des alertes de pipeline.
  4. Effectuez l'une des opérations suivantes, présentées dans le tableau 25-3.

    5. Tableau 25-3 Options de purge des alertes
    Opération
    Exécutez les étapes suivantes...
    • Purger toutes les alertes de contrat de niveau de service
    • Purger toutes les alertes de pipeline
    1. Sélectionnez Purger toutes les alertes.
    2. Cliquez sur Purger.
    • Purger les alertes de contrat de niveau de service au cours d'une période donnée
    • Purger les alertes de pipeline au cours d'une période donnée
    1. Sélectionnez Purger les alertes à partir de... jusqu'à...
    2. Dans le champ Purger à partir de, indiquez un mois, un jour, une année et une heure.
    3. Dans le champ Purger jusqu'à, indiquez un mois, un jour, une année et une heure.
    4. Cliquez sur Purger.

Remarque : Si le serveur à partir duquel les alertes sont purgées n'est pas disponible, la structure de journalisation rend un enregistrement de la purge persistant et interroge le serveur toutes les 30 minutes pour vérifier sa disponibilité. Lorsque la structure de journalisation détermine que le serveur est disponible, elle purge les alertes.
