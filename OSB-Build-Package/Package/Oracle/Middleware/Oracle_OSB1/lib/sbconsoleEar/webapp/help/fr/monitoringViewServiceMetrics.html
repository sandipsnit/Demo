Visualisation des mesures de service

Utilisez cette page pour visualiser les détails sur la surveillance de services spécifiques, présentés dans le tableau 25-10, et les mesures de service présentées dans le tableau 25-11.

Tableau 25-10 Informations sur la surveillance des services
Propriété
Description
Nom de service
Nom et chemin du service.
Type de service
Type du service parent : service proxy ou service fonctionnel.
Affichage des statistiques
Pour afficher les statistiques de surveillance du service pour la période de l'intervalle d'agrégation en cours, sélectionnez Intervalle d'agrégation en cours. La vue Intervalle d'agrégation en cours présente une vue mobile des statistiques des mesures de service.
Pour afficher les statistiques de surveillance du service pour la période écoulée depuis la dernière réinitialisation des statistiques, sélectionnez Depuis la dernière réinitialisation. La vue Depuis la dernière réinitialisation affiche le nombre de mesures en cours d'exécution.
Serveur
Sélectionnez un serveur dans la liste déroulante pour afficher les mesures associées.
S'il existe un cluster, les mesures globales du cluster sont affichées par défaut. Sélectionnez un serveur géré individuel pour afficher les mesures associées.
Si vous affichez les mesures d'un serveur géré individuel dans un cluster, le champ Nombre d'alertes de contrat de niveau de service présente la valeur N/A (seul le nombre d'alertes de contrat de niveau de service du cluster est affiché).
Intervalle d'agrégation
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit de l'intervalle d'agrégation en cours défini pour la surveillance de ce service, en heures et minutes. Vous devez définir cet intervalle sur la page Paramètres opérationnels. Reportez-vous à Définition de l'intervalle d'agrégation d'un service.
Réinitialiser les statistiques
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, vous pouvez réinitialiser les statistiques de surveillance de ce service. Pour ce faire, cliquez sur Réinit. statistiques au bas de la page.

Tableau 25-11 Mesures de service
Propriété
Description
Nombre d'alertes de contrat de niveau de service
  • Le nombre d'alertes est basé sur la vue Afficher les statistiques pour l'intervalle d'agrégation en cours ou la période écoulée depuis la dernière réinitialisation.
  • La gravité et l'horodatage de l'alerte ne sont affichés que dans la vue Intervalle d'agrégation en cours. Pour la vue Depuis la dernière réinitialisation, seul le nombre apparaît.
  • Gravité de l'alerte la plus grave dans l'intervalle d'agrégation en cours. Lorsqu'il existe plusieurs alertes, le niveau de gravité le plus élevé dans l'intervalle d'agrégation en cours est affiché. S'il n'existe aucune alerte dans cet intervalle, la valeur NORMAL apparaît.
  • L'horodatage indique l'heure à laquelle le serveur a été interrogé pour la dernière fois afin d'obtenir des informations d'alerte. Il indique aussi le statut de l'alerte lors de la dernière interrogation du serveur. S'il n'y avait pas d'alerte à ce moment-là, la valeur NORMAL apparaît et l'heure d'interrogation est affichée.
Nombre d'alertes de pipeline
Pour les services proxy uniquement :
  • Le nombre d'alertes de pipeline est basé sur la vue Afficher les statistiques, pour l'intervalle d'agrégation en cours ou la période écoulée depuis la dernière réinitialisation.
  • La gravité et l'horodatage de l'alerte ne sont affichés que dans la vue Intervalle d'agrégation en cours. Pour la vue Depuis la dernière réinitialisation, seul le nombre apparaît.
  • Gravité de l'alerte la plus grave dans l'intervalle d'agrégation en cours. Lorsqu'il existe plusieurs alertes, le niveau de gravité le plus élevé dans l'intervalle d'agrégation en cours est affiché. S'il n'existe aucune alerte de pipeline dans cet intervalle, la valeur NORMAL apparaît.
  • L'horodatage indique l'heure à laquelle le serveur a été interrogé pour la dernière fois afin d'obtenir des informations d'alerte de pipeline. Il indique aussi le statut de l'alerte lors de la dernière interrogation du serveur. S'il n'y avait pas d'alerte à ce moment-là, la valeur NORMAL apparaît et l'heure d'interrogation est affichée.
Temps de réponse minimal
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du temps de réponse minimal nécessaire à ce service pour exécuter les messages pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du temps de réponse minimal nécessaire à ce service pour exécuter les messages au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Temps de réponse maximal
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du temps de réponse maximal nécessaire à ce service pour exécuter les messages pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du temps de réponse maximal nécessaire à ce service pour exécuter les messages au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Temps de réponse moy. global
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du temps moyen global (en millisecondes) nécessaire à ce service pour exécuter les messages pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du temps moyen global nécessaire à ce service pour exécuter les messages au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Nombre de messages
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du nombre total de messages exécutés par ce service pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du nombre total de messages exécutés par ce service au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Nombre d'erreurs
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du nombre de messages d'erreur exécutés par ce service pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du nombre de messages présentant des erreurs, exécutés par ce service au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Nombre de changements de serveur
Pour les services fonctionnels uniquement :
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du nombre de basculements tentés par ce service pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du nombre de basculements tentés par ce service au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Ratio succès
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du taux de réussite de ce service pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du taux de réussite de ce service au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Par exemple, si le service a exécuté 9 messages sur 10, le taux de réussite est de 90 %.
Ratio échec
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du taux d'échec de ce service pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du taux d'échec de ce service au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Par exemple, si le service n'a pas pu exécuter 1 message sur 10, le taux d'échec est de 10 %.
Erreurs de sécurité WS
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du nombre de messages présentant des erreurs de sécurité WS, exécutés par ce service pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du nombre de messages présentant des erreurs de sécurité WS, exécutés par ce service au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Erreurs de validation
Pour les services fonctionnels : la valeur N/A apparaît.
Pour les services proxy :
Si vous avez sélectionné Intervalle d'agrégation en cours dans le champ Afficher les statistiques, il s'agit du nombre de messages présentant des erreurs de validation, exécutés par ce service pendant l'intervalle d'agrégation en cours.
Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du nombre de messages présentant des erreurs de validation, exécutés par ce service au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
Nombre de succès
Pour les services fonctionnels uniquement :
L'option Nombre de succès indique le nombre d'utilisations d'un cache des résultats de service fonctionnel.
Taux de succès
Somme du nombre de succès et du nombre de messages.
Temps de ralentissement min

Si vous avez sélectionné Depuis la dernière réinitialisation dans le champ Afficher les statistiques, il s'agit du temps minimal que les messages ont passé dans la file d'attente de ralentissement au cours de la période écoulée depuis la dernière réinitialisation des statistiques.
