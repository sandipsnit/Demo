Connexions MQ

Remarque : Pour connaître les versions de MQ WebSphere prises en charge dans Oracle Service Bus, reportez-vous à Oracle Fusion Middleware Supported System Configurations, à l'adresse suivante : http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html. La bibliothèque client MQ WebSphere n'est pas fournie avec le programme d'installation Oracle Service Bus. Vous pouvez l'obtenir à partir de l'installation de votre MQ WebSphere.

Les connexions MQ sont des ressources partageables qui peuvent être réutilisées dans plusieurs services proxy et fonctionnels MQ. Les services proxy et fonctionnels MQ doivent être connectés à un gestionnaire de files d'attente MQ pour pouvoir accéder à une file d'attente MQ. Les ressources de connexion MQ fournissent les paramètres obligatoires de connexion au gestionnaire de files d'attente MQ.

Chaque ressource de connexion MQ comporte un pool de connexions. Tous les services fonctionnels ou proxy ayant recours à une ressource de connexion MQ donnée pour établir une connexion à un gestionnaire de files d'attente particulier utilisent le même pool de connexions que celui créé pour cette ressource. Par conséquent, plusieurs services fonctionnels ou proxy employant le même gestionnaire de files d'attente partagent un pool de connexions.

Pour en savoir plus sur les ressources de connexion MQ Oracle Service Bus et les transports MQ natifs, reportez-vous à "MQ Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Pour en savoir plus sur les bases de MQ WebSphere, consultez l'adresse suivante : http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
