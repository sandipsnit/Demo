Stratégies WS personnalisées

Remarque : Dans les futures versions d'Oracle Service Bus, les stratégies Oracle Web Services Manager remplaceront les stratégies WLS 9.2. Même si cette version d'Oracle Service Bus prend toujours en charge les stratégies WLS 9.2, il est conseillé d'utiliser les stratégies Oracle Web Services Manager pour la création de services afin de vous préparer à un éventuel abandon de la prise en charge des stratégies WLS 9.2. Pour plus d'informations, reportez-vous à "Securing Oracle Service Bus with Oracle Web Services Manager" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681.

La structure de stratégie de services Web (stratégie WS) est une structure XML extensible qui étend la configuration d'un service Web avec des assertions de sécurité propres à un domaine et qui indique les exigences, attentes et fonctions de sécurité du service Web. Dans Oracle Service Bus, une des principales utilisations de la stratégie WS est la configuration de la sécurité de niveau message dans les services proxy et fonctionnels.

Utilisation des stratégies WS Oracle Service Bus

Oracle Service Bus contient trois fichiers de stratégie WS simples que vous pouvez utiliser pour demander aux clients d'autoriser, et de crypter ou signer numériquement les messages SOAP : Auth.xml, Encrypt.xml et Sign.xml. Oracle recommande d'utiliser ces fichiers prépackagés aussi souvent que possible, excepté si vous avez des besoins de sécurité particuliers.

Pour plus d'informations sur l'utilisation de ces stratégies, reportez-vous à "Using WS-Policy in Oracle Service Bus Proxy and Business Services" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.

Utilisation des stratégies WS personnalisées

Si les instructions de stratégie WS Oracle Service Bus ne répondent pas à vos besoins en matière de sécurité, vous pouvez écrire vos propres stratégies WS (stratégies WS personnalisées), les importer dans Oracle Service Bus et vous y reporter à partir du WSDL. (Les instructions de stratégie WS Oracle Service Bus sont accessibles en lecture seule.)

Pour plus d'informations sur la création et le recours à des stratégies WS personnalisées, reportez-vous à "Using WS-Policy in Oracle Service Bus Proxy and Business Services" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.
