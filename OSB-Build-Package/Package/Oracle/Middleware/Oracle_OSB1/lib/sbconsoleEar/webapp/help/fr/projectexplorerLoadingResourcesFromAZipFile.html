Chargement de ressources à partir d'un fichier ZIP

Vous pouvez sélectionner un fichier ZIP ou JAR contenant les ressources à charger. Oracle Service Bus localise et importe les ressources à partir de ce fichier ZIP.

Les ressources de type WSDL ou schéma XML peuvent définir des arborescences complètes de ressources dépendantes. La fonctionnalité de chargement en masse permet de télécharger tout un ensemble de ressources en une seule fois, sans vous obliger à résoudre les dépendances manuellement.

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Sélectionnez Explorateur de projets, puis choisissez le projet ou le dossier auquel ajouter les ressources. La page Vue du projet/Vue de dossier apparaît.
  3. Dans la liste déroulante Créer une ressource, sélectionnez Ressources compressées. La page Charger des ressources apparaît.
  4. Dans le champ Sélectionner un fichier ZIP, indiquez le fichier ZIP à importer. Entrez le nom du fichier (qualifié par son chemin) ou cliquez sur Parcourir pour localiser le fichier, puis cliquez sur Open.
  5. Vérifiez la correspondance des types de ressource avec les extensions de fichier. Modifiez ou ajoutez des extensions, si nécessaire. Séparez les différentes extensions par des virgules.

    6. Chaque fichier contenu dans le fichier ZIP représente une ressource potentielle. L'extension d'un fichier constitue la méthode la plus courante pour identifier le type de ce fichier. Oracle Service Bus définit une extension par défaut pour chaque ressource. Si vous disposez d'autres extensions pour décrire ces ressources, cette correspondance vous permet de les déclarer au chargeur.

    Une fois le fichier ZIP et la correspondance d'extensions définis, le chargeur Oracle Service Bus analyse le contenu du fichier ZIP et tente d'associer un type de ressource à chaque fichier. Un fichier ne portant aucune extension ou dont l'extension n'est pas définie dans la correspondance est considéré comme un fichier inconnu et il est automatiquement exclu du chargement.

    Pour les fichiers connus, le nom de la ressource est le nom du fichier sans son extension. La structure de dossier du fichier ZIP est recréée dans le dossier cible de la console d'administration d'Oracle Service Bus.

    Une extension ne peut être associée qu'à un seul type de fichier.

  6. Cliquez sur Suivant.
  7. Sur la page Charger des ressources - Vérifier les ressources chargées, vérifiez les ressources à importer. Désélectionnez les cases en regard des ressources à exclure de l'import.
  8. Pour importer les ressources sélectionnées, cliquez sur Importer.
  9. La page Charger des ressources - Importer le résultat affiche les ressources importées. Vous pouvez cliquer sur Charger un autre élément pour importer des ressources à partir d'un autre fichier ou effectuer d'autres tâches en sélectionnant une autre page dans le panneau de navigation de gauche.
  10. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
