Visualisation de ressources dans un navigateur Web

Vous pouvez visualiser les ressources Oracle Service Bus dans un navigateur Web standard à l'aide des URL suivantes :

    • Chemin des ressources WSDL :
      http://host:port/sbresource?WSDL/project_path/wsdlname
      ou
      http://host:port/proxy_service_endpoint_URI?WSDL (pour les services proxy HTTP de type WSDL)
    • Chemin des ressources de service proxy :
      http://host:port/sbresource?PROXY/project_path/proxyname
      http://host:port/sbresource?ORAPROXY/project_path/proxyname (pour les services proxy associés à des stratégies Oracle Web Services Manager)
    • Chemin des ressources de service fonctionnel :
      http://host:port/sbresource?BIZ/project_path/bizname
      http://host:port/sbresource?ORABIZ/project_path/bizname (pour les services fonctionnels associés à des stratégies Oracle Web Services Manager)
    • Chemin des ressources WSS :
      http://host:port/sbresource?POLICY/project_path/policyname
    • Chemin des ressources MFL :
      http://host:port/sbresource?MFL/project_path/mflname
    • Chemin des ressources SCHEMA :
      http://host:port/sbresource?SCHEMA/project_path/schemaname

    Remarque : si vous utilisez des caractères spéciaux dans les noms de ressources, les URL représentant ces ressources dans Oracle Service Bus doivent être encodées en UTF-8 afin d'ajouter un caractère d'échappement aux caractères spéciaux.

    Vous pouvez également extraire des WSDL contenant des stratégies Oracle Web Services Manager afin que les stratégies respectent les normes WS-Security et WS-Policy standard. Pour plus d'informations, reportez-vous à "Advertising WSDLs to Support WS Standards" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1877.
