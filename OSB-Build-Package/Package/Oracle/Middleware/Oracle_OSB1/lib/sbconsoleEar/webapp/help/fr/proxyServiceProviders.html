Fournisseurs de clés de service

Un fournisseur de clés de service contient des informations d'identification et de connexion d'infrastructure PKI utilisées par les services proxy pour décrypter les messages SOAP entrants, ainsi que pour les authentifications sortantes et les signatures numériques. Ces informations sont des clés privées associées à un certificat pouvant être utilisées pour les signatures numériques et le cryptage (pour la sécurité de services Web) et pour l'authentification SSL sortante. Le certificat contient la clé publique correspondant à la clé privée.

Remarque : Pour utiliser un fournisseur de clés de service, vous devez configurer un fournisseur de mise en correspondance des informations d'identification et de connexion d'infrastructure PKI. Reportez-vous à "Configuring the WebLogic Security Framework: Main Steps" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498.

Un seul fournisseur de clés de service peut contenir toutes les informations d'identification et de connexion d'infrastructure PKI suivantes :

  • Une paire de clés pour le cryptage numérique

    • Les services proxy utilisent cette paire de clés afin de·décrypter les messages SOAP entrants cryptés pour respecter les instructions d'une stratégie de services Web. Si vous souhaitez que le fournisseur de clés de service prenne en charge le cryptage numérique, le fichier de clés associé au composant de correspondance des informations d'identification et de connexion d'infrastructure PKI doit contenir au moins un certificat X.509 prenant en charge le cryptage.

  • Une paire de clés pour les signatures numériques

    • Les services proxy utilisent cette paire de clés lorsque l'adresse est un service Web exigeant que les clients signent au moins une partie de l'enveloppe SOAP.

  • Une paire de clés pour l'authentification client SSL (SSL bidirectionnel)

    • Les services proxy utilisent cette paire de clés pour l'authentification lorsqu'ils agissent en tant que client lors d'une connexion TLS/SSL (Secure Sockets Layer) sortante ; c'est-à-dire lorsqu'un message est acheminé vers un service proxy ou un service fonctionnel HTTPS qui exige une authentification par·certificat client.

Vous pouvez utiliser le même fournisseur de clés de service pour plusieurs services proxy.
