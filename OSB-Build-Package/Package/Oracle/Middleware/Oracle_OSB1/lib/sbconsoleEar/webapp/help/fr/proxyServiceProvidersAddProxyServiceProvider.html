Ajout de fournisseurs de clés de service

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Sélectionnez Explorateur de projets, puis choisissez·un projet ou un dossier dans lequel ajouter le fournisseur de clés de service. La page Vue du projet/Vue de dossier apparaît.
  3. Dans la liste déroulante Créer une ressource, sélectionnez Fournisseur de clés de service pour afficher la page Créer un fournisseur de clés de service.
  4. Dans le champ Nom du fournisseur de clés de service, indiquez un nom unique pour ce fournisseur de clés de service.

    Pour obtenir des conseils sur la dénomination, suivez les restrictions concernant la dénomination des ressources.
  5. Dans le champ Description, entrez une description du fournisseur de clés de service.
  6. Exécutez n'importe laquelle des étapes affichées ci-dessous dans le tableau 17-2.

    7. Tableau 17-2 Options d'authentification
    Ajouter une paire de clés pour...
    Exécutez les étapes suivantes...
    Cryptage numérique
    1. En regard de Clé de cryptage, cliquez sur Parcourir.

      2. La fenêtre Sélectionner un alias pour la clé de cryptage affiche les alias de clés figurant dans le fichier de clés utilisé par le composant de correspondance d'informations d'identification et de connexion d'infrastructure PKI de votre domaine.

    2. Dans la fenêtre, Sélectionner un alias pour la clé de cryptage, entrez le mot de passe que vous utilisez pour sécuriser l'accès au fichier de clés. (Vous définissez le mot de passe lorsque vous créez le fichier de clés.)
    3. Sélectionnez un alias de clé qui correspond à un certificat X.509 et qui prend en charge le cryptage.
    4. Cliquez sur Soumettre.
    Lorsque vous associez ce fournisseur de clés de service à un service proxy, Oracle Service Bus imbrique le certificat X.509 dans le WSDL du service proxy. Ce dernier utilise ensuite le certificat pour crypter les messages qu'il envoie à son adresse. Le service proxy utilise la clé privée des informations d'identification et de connexion d'infrastructure PKI pour décrypter les messages que l'adresse renvoie.
    Signatures numériques
    1. En regard de Clé de signature numérique, cliquez sur Parcourir.

      2. La fenêtre Sélectionner un alias pour la clé de signature numérique affiche les alias de clés figurant dans le fichier de clés utilisé par le composant de correspondance d'informations d'identification et de connexion d'infrastructure PKI de votre domaine.

    2. Dans la fenêtre Sélectionner un alias pour la clé de signature numérique, entrez le mot de passe que vous utilisez pour sécuriser l'accès au fichier de clés. (Vous définissez le mot de passe lorsque vous créez le fichier de clés.)
    3. Sélectionnez un alias de clé.
    4. Cliquez sur Soumettre.
    Authentification client SSL (SSL bidirectionnel)
    1. En regard de Clé d'authentification client SSL, cliquez sur Parcourir.

      2. La fenêtre Sélectionner un alias pour la clé d'authentification client SSL affiche les alias de clés figurant dans le fichier de clés utilisé par le composant de correspondance d'informations d'identification et de connexion d'infrastructure PKI de votre domaine.

    2. Dans la fenêtre Sélectionner un alias pour la clé d'authentification client SSL, entrez le mot de passe que vous utilisez pour sécuriser l'accès au fichier de clés. (Vous définissez le mot de passe lorsque vous créez le fichier de clés.)
    3. Sélectionnez un alias de clé.
    4. Cliquez sur Soumettre.

  7. Cliquez sur Enregistrer. Le fournisseur de clés de service est enregistré dans la session en cours.
  8. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
