Ajout et modification d'actions dans les flux de messages

Les actions constituent les éléments des phases du pipeline et du gestionnaire d'erreur, ainsi que des noeuds de routage et de branchement, qui déterminent la manière dont les messages seront définis lors de leur transit dans un service proxy.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Elles supposent également que vous ayez déjà ajouté une phase de pipeline, un noeud de routage et/ou une phase de gestionnaire d'erreur. Reportez-vous à :

  • Ajout de noeuds de paire de pipeline aux flux de messages
  • Ajout de phases aux pipelines
  • Ajout de gestionnaires d'erreur de service proxy

Pour ajouter une action à un flux de messages, procédez comme suit :

  1. Sélectionnez le composant auquel ajouter une action. Par exemple, cliquez sur l'icône Phase, puis sélectionnez Modifier la phase ou cliquez sur l'icône Noeud de routage, puis sélectionnez Modifier le routage.
  2. Selon que vous avez ajouté ou non des actions à la phase ou au noeud de routage, effectuez l'une des opérations suivantes :
    • Si aucune action n'a encore été ajoutée, la page Modifier la configuration de la phase affiche uniquement l'icône Ajouter une action. Cliquez sur celle-ci, puis sélectionnez un type d'action.
    • Si vous avez déjà ajouté des actions, la page Modifier la configuration de la phase affiche des icônes représentant ces actions (par exemple, les icônes Publication ou Routage). Cliquez sur l'icône appropriée, puis sur Ajouter une action et sélectionnez un type d'action.
    • Certaines actions, telles que les actions de demande et de réponse dans les actions de publication, incluent un lien Ajouter une action lorsqu'une action est appropriée. Cliquez sur l'icône, puis sélectionnez un type d'action.

      • Il n'existe aucune restriction concernant les actions pouvant être chaînées dans un flux de messages.

  3. Lorsque vous avez terminé d'ajouter des actions, vous pouvez les configurer dans la phase ou le noeud de routage, comme indiqué dans le tableau 21-2.

    4. Tableau 21-2 Tâches de modification de la configuration de la phase 
    Opération
    Procédure
    Supprimer une action
    Cliquez sur l'icône appropriée, puis sur Supprimer l'action.
    Déplacer une action vers le bas (rétrograder)
    Cliquez sur l'icône appropriée, puis sur Déplacer l'action vers le bas. L'action est déplacée en dessous de l'action suivante figurant dans la phase.
    Cette option n'est disponible que si la phase comprend plusieurs actions.
    Déplacer une action vers le haut (promouvoir)
    Cliquez sur l'icône appropriée, puis sur Déplacer l'action vers le haut. L'action est déplacée au-dessus de l'action précédente figurant dans la phase.

    Cette option n'est disponible que si la phase comprend plusieurs actions.

    Couper une action
    Cliquez sur l'icône appropriée, puis sur Couper.
    Copier une action
    Cliquez sur l'icône appropriée, puis sur Copier.
    Coller une action que vous avez coupée ou copiée
    Cliquez sur l'icône appropriée, puis sur Coller l'action.
    Vous pouvez copier et coller des actions entre différentes phases. Toutefois, dans le cas des actions d'affectation, de remplacement ou d'insertion, vous devez prendre en compte les points suivants :
    • Les espaces de noms basés sur une variable et définis par l'utilisateur dans la phase source (éléments copiés) sont ajoutés en tant qu'espaces de noms définis par l'utilisateur dans la phase cible (éléments collés).
    • Les espaces de noms en double (espaces de noms identiques dans les phases source et cible) ne sont pas copiés.
    • Les espaces de noms en conflit (déclarations d'espace de noms utilisant le même préfixe mais des URI différents) sont copiés. Les utilisateurs pourront enregistrer la configuration, mais ils ne pourront l'activer que lorsque les déclarations d'espace de noms en conflit dans la phase B seront enlevées.
    Valider une phase
    Sur la page Modifier la configuration de la phase, cliquez sur Valider pour valider toutes les actions configurées dans la phase.

  4. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  5. Sur la page Modifier le flux de messages, poursuivez la construction du flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.
  6. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  7. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
