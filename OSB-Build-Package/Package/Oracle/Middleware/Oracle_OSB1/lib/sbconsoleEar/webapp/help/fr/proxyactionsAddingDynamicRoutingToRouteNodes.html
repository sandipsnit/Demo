Ajout d'un routage dynamique aux noeuds de routage

Permet d'affecter une route à un message en fonction des informations de routage disponibles dans une ressource XQuery.

Il s'agit d'une action finale ; cela signifie que vous ne pouvez pas ajouter d'autres actions après celle-ci. Cette action peut toutefois inclure des actions de demande et de réponse.

Pour plus d'informations, reportez-vous à "Modeling Message Flow in Oracle Service Bus" dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG181.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter un routage dynamique à un noeud de routage, procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône Noeud de routage, puis sur Modifier le routage. La page Modifier la configuration de la phase apparaît.
  3. Cliquez sur l'icône Ajouter une action, puis sélectionnez Communication > Routage dynamique.
  4. Cliquez sur <Expression>. L'éditeur d'expressions XQuery apparaît.
  5. Dans l'éditeur d'expressions XQuery, entrez une expression XQuery dont le résultat est semblable à ce qui suit :

    6. <ctx:route>

        <ctx:service isProxy='true'>{$service}</ctx:service>

        <ctx:operation>{$operation}</ctx:operation>

    </ctx:route>

    Remarque : Si un service proxy est appelé, l'attribut isProxy doit être défini sur True. Si un service fonctionnel est appelé, définissez isProxy sur False.
    Remarque : le nom de service correspond au nom qualifié complet du service.
    Remarque : l'élément "operation" est facultatif.
  6. Cliquez sur Enregistrer.
  7. Dans le champ Actions de demande, cliquez sur Ajouter une action pour ajouter une action, puis sélectionnez l'action à associer au service. Vous pouvez ajouter plusieurs actions. Pour plus d'informations sur les différents types d'action que vous pouvez ajouter, reportez-vous au tableau dans Ajout et modification d'actions dans les flux de messages.
  8. Dans le champ Actions de réponse, cliquez sur Ajouter une action pour ajouter une action, puis sélectionnez l'action à associer au service. Vous pouvez ajouter plusieurs actions. Pour plus d'informations sur les différents types d'action que vous pouvez ajouter, reportez-vous au tableau dans Ajout et modification d'actions dans les flux de messages.
  9. Cliquez sur Enregistrer.
  10. Sur la page Modifier le flux de messages, poursuivez la construction du flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.
  11. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  12. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
