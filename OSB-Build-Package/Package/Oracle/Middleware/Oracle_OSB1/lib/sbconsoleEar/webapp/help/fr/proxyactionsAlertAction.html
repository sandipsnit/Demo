Ajout d'actions d'alerte

L'action d'alerte permet de générer des alertes basées sur le contexte de message d'un pipeline et de les envoyer à une destination d'alerte. Contrairement aux alertes de contrat de niveau de service, les notifications générées par l'action d'alerte sont principalement dédiées à des fins fonctionnelles, ou servent à signaler des erreurs, et non à la surveillance de l'état du système. Vous devez ainsi configurer et choisir les destinations d'alertes dans cette optique. Pour plus d'informations sur les destinations d'alertes, reportez-vous à Destinations d'alertes.

Si les alertes de pipeline ne sont pas activées pour le service ou au niveau du domaine, l'action d'alerte configurée est ignorée lors du traitement du message.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter une action d'alerte, procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Génération de rapports > Alerte.
  3. Cliquez sur <Destination>. La page Sélectionner une destination d'alertes apparaît. Sélectionnez une destination d'alerte dans la liste, puis cliquez sur Soumettre.

    4. Par défaut, l'alerte sera toujours transmise dans la console d'administration d'Oracle Service Bus.

  4. Cliquez sur <Expression>. La page Editeur d'expressions XQuery apparaît. Indiquez le contexte de message à ajouter au message d'alerte au moyen d'expressions XQuery sur des variables de contexte. Reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.
  5. Dans le champ Récapitulatif des alertes, entrez une brève description de l'alerte. Celle-ci servira de ligne d'objet pour les notifications par courrier électronique. Elle peut contenir jusqu'à 80 caractères. Si vous n'indiquez aucune description, la ligne d'objet prédéfinie "Alerte Oracle Service Bus" est utilisée.
  6. Dans la liste déroulante Niveau de gravité, sélectionnez le niveau de gravité de l'alerte parmi les options suivantes : Normal, Avertissement, Mineur, Majeur, Critique et Fatal.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
