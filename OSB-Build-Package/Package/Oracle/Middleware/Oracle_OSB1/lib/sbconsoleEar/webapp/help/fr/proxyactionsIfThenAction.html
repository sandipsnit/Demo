Ajout d'actions If...Then...

L'action If...Then... permet d'exécuter une action ou un ensemble d'actions selon des conditions, en fonction du résultat booléen d'une expression XQuery.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter une action If...Then..., procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Contrôle de flux > If...Then.
  3. Cliquez sur <Condition> pour afficher la page Editeur de conditions XQuery.

    4. La condition que vous créez sert de test et est exécutée avant la saisie de la clause then(), selon la logique If... Then... standard. Reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.

  4. Une fois la condition XQuery modifiée, cliquez sur Ajouter une action, puis sélectionnez l'action à associer à la condition. Pour plus en savoir plus sur les différents types d'action que vous pouvez ajouter, reportez-vous à Ajout et modification d'actions dans les flux de messages.

    5. Dans le noeud de routage, vous ne pouvez sélectionner que les actions de routage, de routage dynamique ou de table de routage. Ces actions peuvent toutefois inclure des actions de demande et de réponse.

  5. Selon les exigences de votre logique, cliquez sur l'icône If...Then..., puis sur Ajouter une condition else-if ou Ajouter une condition else pour ajouter des conditions else-if ou else. Cliquez sur Ajouter une action pour associer les actions à ces conditions.

    6. Il est possible d'imbriquer plusieurs conditions d'action. Il existe néanmoins une limite d'imbrication de quatre niveaux cumulés dans l'éditeur de phases. Si vous tentez d'ajouter un cinquième niveau, l'action d'imbrication n'apparaît pas. Les niveaux cumulés incluent toutes les actions de branchement : les conditions If... Then..., les tables de publication et les tables de routage. Vous pouvez ainsi avoir deux niveaux d'action de condition et une table de publication contenant une table de routage, ce qui donne quatre niveaux au total. Si vous tentez d'ajouter une action de condition supplémentaire (à la table de publication), elle n'apparaîtra pas.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
