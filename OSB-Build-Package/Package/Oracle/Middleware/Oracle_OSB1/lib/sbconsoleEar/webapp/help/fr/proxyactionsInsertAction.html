Ajout d'actions d'insertion

L'action d'insertion permet d'insérer le résultat d'une expression XQuery à un emplacement défini relatif aux noeuds sélectionnés par une expression XPath. L'action d'insertion fait partie des actions de mise à jour.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter une action d'insertion, procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Traitement des messages > Insérer.
  3. Cliquez sur <Expression> pour modifier une expression XQuery. L'expression XQuery permet de créer les données à insérer dans un emplacement donné d'une variable nommée. La page Editeur d'expressions XQuery apparaît. Reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.
  4. Une fois l'expression modifiée, sélectionnez un emplacement relatif dans la liste déroulante. L'emplacement relatif permet de contrôler l'endroit où l'insertion est effectuée par rapport au résultat de l'expression XPath, à savoir :
    • Avant : en tant que frère avant chaque élément ou attribut sélectionné par l'expression XPath ;
    • Après : en tant que frère après chaque élément ou attribut sélectionné par l'expression XPath ;
    • en tant que premier enfant de : en tant que premier enfant de chaque élément identifié par l'expression XPath. Une erreur est générée si l'expression XPath renvoie des attributs ;
    • en tant que dernier enfant de : en tant que dernier enfant de chaque élément identifié par l'expression XPath. Une erreur est générée si l'expression XPath renvoie des attributs.
  5. Cliquez sur XPath. La page Editeur d'expressions XPath apparaît. Reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.

    6. Pour être valide, une configuration doit répondre aux exigences suivantes :

    • Les expressions XQuery et XPath renvoient toutes les deux des éléments.
    • Les expressions XQuery et XPath renvoient toutes les deux des attributs, dans ce cas l'expression XQuery doit renvoyer des attributs.
  6. Une fois l'expression XPath modifiée, entrez une variable de contexte dans le champ Variable. L'expression XPath évalue le contenu de cette variable. Pour en savoir plus sur les variables de contexte, reportez-vous à l'annexe A, Contexte de message.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
