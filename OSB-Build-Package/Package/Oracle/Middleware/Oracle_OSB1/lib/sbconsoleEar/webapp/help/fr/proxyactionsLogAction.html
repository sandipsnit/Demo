Ajout d'actions de journalisation

L'action de journalisation permet de créer un message à journaliser et de définir l'ensemble d'attributs qui l'accompagne.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Remarque : pour visualiser les données de journalisation dans le fichier journal ou de sortie standard (console du serveur), vous devez définir la journalisation d'Oracle WebLogic Server sur les niveaux de gravité suivants :

Niveau de gravité minimal à journaliser : Info
Fichier journal : Info
Sortie standard : Info

Pour plus d'informations sur la configuration des niveaux de gravité de journalisation, reportez-vous à Using Log Severity Levels dans le manuel Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server à l'adresse http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

 

Pour ajouter une action de journalisation, procédez comme suit :

  1. Assurez-vous que la journalisation est activée globalement. Pour plus d'informations, reportez-vous à Activation globale de la journalisation.
  2. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  3. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Génération de rapports > Journaliser.
  4. Cliquez sur <Expression>. La page Editeur d'expressions XQuery apparaît. Indiquez le contexte de message à journaliser au moyen d'expressions XQuery sur des variables de contexte. Reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.
  5. Dans le champ Annotation, entrez les remarques concernant l'action de journalisation. Celles-ci sont journalisées, ainsi que le résultat de l'expression précédemment définie.
  6. Sélectionnez l'une des options disponibles dans la liste déroulante Niveau de gravité.

    7. Tableau 21-5 Action de journalisation - Niveaux de gravité
    Niveau de gravité
    Utilisation standard
    Infos
    Utilisé pour signaler des opérations de rapport normales ; il s'agit d'un message d'informations de bas niveau.
    Avertissement
    Indique qu'une opération ou une configuration suspecte a eu lieu, sans incidence sur le fonctionnement normal.
    Erreur
    Signale une erreur utilisateur. Le système ou l'application peut gérer l'erreur sans interruption, mais avec une légère dégradation du service.
    Déboguer
    Lors du développement d'une application, il peut s'avérer utile de créer et d'utiliser des messages fournissant des descriptions en mode verbose de l'activité de bas niveau au sein de l'application.
    Assurez-vous que le niveau de gravité de journalisation des paramètres opérationnels du service proxy et le niveau de gravité de l'action de journalisation sont les mêmes. Pour plus d'informations sur les paramètres opérationnels du service proxy, reportez-vous à Configuration des paramètres opérationnels des services proxy.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
