Ajout d'actions de table de publication

L'action de table de publication permet de publier un message sur des services spécifiés de façon statique ou sur aucun. La logique de condition de permutation de style sert à déterminer, au moment de l'exécution, les services qui seront utilisés pour la publication.

Pour plus d'informations sur le comportement de publication, reportez-vous à Performing Transformations in Message Flows dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse http://www.oracle.com/pls/as111170/lookup?id=OSBAG205.

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Sélectionnez Ajouter une action > Communication > Publier une table.
  3. Cliquez sur <Expression>. La page Editeur d'expressions XQuery apparaît. Créez une expression XQuery. Celle-ci renverra, au moment de l'exécution, la valeur sur laquelle la méthode de routage sera basée. Reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.
  4. Dans la liste Opérateur, sélectionnez un opérateur de comparaison. Dans le champ adjacent, entrez la valeur par rapport à laquelle la valeur renvoyée par l'expression XQuery sera évaluée.
  5. Cliquez sur <Service> et sélectionnez le service vers lequel les messages seront publiés si l'expression renvoie la valeur True pour la valeur que vous avez indiquée. La page Sélectionner un service apparaît.
  6. Sélectionnez un service dans la liste, puis cliquez sur Soumettre. Il s'agit du service cible pour le message.
  7. Si des opérations sont définies pour ce service, vous pouvez sélectionner l'opération à appeler dans la liste des éléments d'appel.
  8. Si vous souhaitez que l'opération sortante soit identique à l'opération entrante, cochez la case Utiliser une opération entrante pour l'opération sortante.
  9. Pour configurer la méthode de packaging et d'envoi du message vers le service, dans le champ Actions de demande, cliquez sur Ajouter une action, puis sélectionnez les actions à associer au service. Pour en savoir plus sur les différents types d'action que vous pouvez ajouter, reportez-vous à Ajout et modification d'actions dans les flux de messages.
    10. Remarque : il existe une limite d'imbrication de quatre niveaux cumulés dans l'éditeur de phases. Si vous tentez d'ajouter un cinquième niveau, l'action d'imbrication n'apparaît pas. Les niveaux cumulés incluent toutes les actions de branchement : les conditions If... Then..., les tables de publication et les tables de routage. Vous pouvez ainsi avoir deux niveaux d'action de condition et une table de publication contenant une table de routage, ce qui donne quatre niveaux au total. Si vous tentez d'ajouter une action de condition supplémentaire (à la table de publication), elle n'apparaîtra pas.
  10. Pour insérer un nouveau cas, cliquez sur l'icône Cas, puis sélectionnez Insérer un nouveau cas.
  11. Répétez les étapes 4 à 8 pour le nouveau cas.
  12. Ajoutez autant de cas que nécessaire conformément à votre logique applicative.
  13. Cliquez sur l'icône Cas du dernier cas défini dans la séquence, puis sélectionnez Insérer un cas par défaut pour ajouter un cas par défaut à la fin.
  14. Configurez le cas par défaut. La configuration de ce dernier détermine le comportement de routage appliqué si aucun des cas précédents n'est satisfait.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
