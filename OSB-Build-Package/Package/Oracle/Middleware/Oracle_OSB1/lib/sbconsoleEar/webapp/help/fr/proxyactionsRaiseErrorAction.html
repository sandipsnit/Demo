Ajout d'actions de génération d'erreur

L'action de génération d'erreur permet de générer une exception avec un code d'erreur (chaîne) et une description.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter une action de génération d'erreur, procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Contrôle de flux > Générer une erreur.
  3. Dans le champ Code d'erreur, entrez le code d'erreur à générer.
  4. Dans le champ Message d'erreur, saisissez la description du code d'erreur.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.

Transactions

Dans le cas d'un service transactionnel, le déclenchement d'une action de génération d'erreur entraîne l'abandon de la transaction soit dans la demande (asynchrone), soit dans la réponse ou la demande (synchrone). Par exemple, vous pouvez effectuer l'introspection des messages et déterminer les conditions déclenchant une action de génération d'erreur même si aucune erreur SOAP ne survient, et l'action de génération d'erreur entraîne l'abandon de la transaction.
