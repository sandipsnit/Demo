Ajout d'actions de réponse

L'action de réponse permet d'indiquer qu'une réponse doit immédiatement être envoyée à l'appelant.

L'action de réponse peut être utilisée dans le pipeline des demandes, des réponses ou des erreurs. Vous pouvez la configurer pour que son résultat soit une réponse de succès ou d'échec. Dans le cas d'une réponse d'échec, si le transport entrant est de type HTTP, l'action de réponse indique qu'une réponse est immédiatement envoyée à l'appelant.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter une action de réponse, procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Contrôle de flux > Répondre.
  3. Sélectionnez Réussite pour répondre que le message a réussi ou Echec pour répondre que le message contient une erreur.

    4. Si vous sélectionnez Echec et que la transaction a été démarrée par Oracle Service Bus, celle-ci est abandonnée.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
