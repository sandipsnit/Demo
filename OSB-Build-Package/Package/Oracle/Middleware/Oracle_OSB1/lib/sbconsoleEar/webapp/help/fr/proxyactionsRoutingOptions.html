Ajout d'actions d'options de routage

L'action d'options de routage permet de modifier les propriétés suivantes pour la demande sortante dans $outbound : l'URI, la qualité de service, le mode et les tentatives. Bien que ces propriétés puissent être modifiées au moyen des actions d'affectation, d'insertion, de remplacement ou de suppression sur $outbound, l'utilisation des options de routage permet d'exécuter cette tâche plus facilement et ne requiert pas la connaissance des expressions XPath et XQuery ou de la structure de la variable de contexte $outbound.

L'action d'options de routage ne peut être utilisée que si la variable de contexte $outbound est valide. Vous pouvez l'ajouter aux actions suivantes :

  • Publier
  • Publication dynamique
  • Publier une table
  • Appel de service
  • Routage
  • Routage dynamique
  • Table de routage

Pour plus d'informations, reportez-vous à "Modeling Message Flow in Oracle Service Bus" dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG181.

 

Pour configurer une action d'options de routage, procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Communication > Options de routage.
  3. Exécutez l'une des étapes suivantes, ou l'ensemble d'entre elles :
    • Pour définir l'URI du message sortant, procédez comme suit : sélectionnez URI et cliquez sur l'éditeur d'expressions XQuery. Entrez une expression qui renvoie un URI. Cela remplace l'URI spécifié pour le service appelé.
    • Pour définir l'élément de qualité de service, procédez comme suit : sélectionnez Qualité de service, puis choisissez l'option du même nom dans la liste déroulante. Cela remplace le paramètre par défaut qui est calculé automatiquement.
    • Pour définir le mode, procédez comme suit : sélectionnez Mode, puis choisissez demande ou demande-réponse dans la liste déroulante.
      • Remarque : normalement, ce paramètre est défini automatiquement en fonction de l'interface du service appelé. Il arrive cependant que ce ne soit pas le cas, comme lorsque N'importe quel service SOAP ou N'importe quel service XML est sélectionné.
    • Pour définir l'intervalle entre les tentatives, procédez comme suit : sélectionnez Intervalle entre les tentatives et indiquez le délai, en secondes, entre les tentatives. Cela remplace le paramètre par défaut configuré avec le service appelé.
    • Pour définir le nombre de tentatives, procédez comme suit : sélectionnez Nombre de tentatives et indiquez le nombre de tentatives effectuées par le système avant l'abandon de l'action. Cela remplace le paramètre par défaut configuré avec le service appelé.
    • Pour définir la priorité du message, procédez comme suit : sélectionnez Priorité et cliquez sur l'éditeur d'expressions XQuery. Entrez une expression qui renvoie un entier positif.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
