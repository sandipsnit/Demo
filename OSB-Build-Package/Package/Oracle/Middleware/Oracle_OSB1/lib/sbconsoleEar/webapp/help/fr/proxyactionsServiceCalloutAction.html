Ajout d'actions d'appel de service

L'action d'appel de service permet de configurer un appel synchrone (bloquant) à un service fonctionnel ou proxy inscrit auprès d'Oracle Service Bus.

Pour plus d'informations sur les actions d'appel de service, reportez-vous à Constructing Service Callout Messages dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse http://www.oracle.com/pls/as111170/lookup?id=OSBAG208.

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône appropriée, puis sélectionnez Ajouter une action > Communication > Appel de service.
  3. Cliquez sur <Service>. Le navigateur Web de services apparaît.
  4. Sélectionnez un service dans la liste des services fonctionnels ou proxy inscrits, puis cliquez sur Soumettre.
  5. Si le service choisi à l'étape 3 ci-dessus est de type WSDL et dispose d'opérations pouvant être appelées sur le service, ces opérations sont répertoriées dans la liste déroulante d'appel Opération>. Sélectionnez une opération à appeler sur le service.

    Remarque : la sélection d'une opération, requise par Oracle Service Bus pour plusieurs raisons, ne garantit pas que seule l'opération sélectionnée est appelée. Par exemple, si vous sélectionnez l'opération A, mais qu'un message contient également un appel pour l'opération B, l'opération B sera également appelée.
  6. Indiquez la méthode de configuration des messages de demande et de réponse à utiliser en sélectionnant l'une des options suivantes :
    • Sélectionnez Configurer le corps SOAP pour configurer le corps SOAP. Si vous sélectionnez cette option, vous pouvez utiliser $body directement.
      • Remarque : cette option prend en charge l'encodage SOAP-RPC, qui n'est pas pris en charge lors de la configuration des paramètres ou documents de données traitées.
    • Sélectionnez Configurer les paramètres de données traitées ou Configurer le document de données traitées pour configurer les données traitées.
  7. Les options de configuration suivantes varient en fonction du type de service sélectionné à l'étape 3 et du type d'option de configuration choisi pour ce service à l'étape 5. Le tableau 21-3 présente les options disponibles pour chaque type de service.

    8. Le tableau 21-4 ci-dessous fournit des instructions pour chacune des options énumérées dans le tableau 21-3 ci-dessus.

    Tableau 21-4 Options de configuration d'appel de service 
    Pour les options suivantes...
    Exécutez les étapes suivantes...
    Corps de la demande SOAP et
    Corps de la réponse SOAP
    Pour configurer ces options, procédez comme suit :
    • Dans le champ Corps de la demande SOAP, entrez le nom de la variable qui doit contenir le XML de l'élément de corps SOAP pour la demande d'appel.
    • Dans le champ Corps de la réponse SOAP, entrez le nom de la variable à laquelle le XML de l'élément de corps SOAP de la réponse sera lié.
    En-tête de demande SOAP et
    En-tête de réponse SOAP
    Pour configurer ces options, procédez comme suit :
    • Dans le champ En-tête de demande SOAP, entrez le nom de la variable qui doit contenir le XML de l'élément d'en-tête SOAP pour la demande d'appel.

      • Vous devez encapsuler le document d'entrée pour l'en-tête de demande SOAP avec <soap-env:Header>...</soap-env:Header>.

    • Dans le champ En-tête de réponse SOAP, entrez le nom de la variable à laquelle le XML de l'élément d'en-tête SOAP de la réponse, sera lié, le cas échéant.
    Paramètres de demande et
    Paramètres de réponse
    Pour configurer ces options, procédez comme suit :
    • Dans les champs Paramètres de demande, entrez le nom des variables qui seront évaluées lors de l'exécution pour fournir les valeurs des paramètres de demande.

      • Vous devez fournir uniquement les documents de données traitées principaux dans la variable d'entrée. En effet, Oracle Service Bus crée le package SOAP à votre place. En d'autres termes, n'encapsulez pas le document d'entrée avec <soap-env:Body>...</soap-env:Body>.

      Par exemple, lors de la création d'une variable d'entrée de corps pour ce paramètre de demande, vous devez définir le contenu de cette variable au moyen de l'instruction XPath body/* (afin d'enlever le wrapper soap-env:Body) et non $body (qui a pour résultat la conservation du wrapper soap-env:Body).

    • Dans les champs Paramètres de réponse, entrez le nom des variables auxquelles affecter les réponses lors de l'exécution.
    Document de demande et
    Document de réponse
    Pour configurer ces options, procédez comme suit :
    • Dans le champ Variable de document de demande, entrez le nom de la variable à laquelle affecter un document de demande.

      • Pour les services de type document SOAP, la variable est évaluée lors de l'exécution afin de constituer le corps du message SOAP envoyé au service. Pour les services XML, la variable est évaluée lors de l'exécution afin de constituer le corps du message XML envoyé au service.

      Pour les services de type document SOAP et XML, vous devez fournir uniquement les documents de données traitées principaux dans la variable d'entrée. En effet, Oracle Service Bus crée le package SOAP à votre place. En d'autres termes, n'encapsulez pas le document d'entrée avec <soap-env:Body>...</soap-env:Body>.

      Par exemple, lors de la création d'une variable d'entrée de corps pour ce paramètre de demande, vous devez définir le contenu de cette variable au moyen de l'instruction XPath body/* (afin d'enlever le wrapper soap-env:Body) et non $body (qui a pour résultat la conservation du wrapper soap-env:Body).

      Pour les services de messagerie, la variable est évaluée afin de constituer le corps du message en fonction du type de données attendu par le service. Les restrictions suivantes s'appliquent aux variables utilisées avec les services de messagerie :

      • Pour les services qui attendent des données binaires, les variables doivent contenir un élément ctx:binary-content.
      • Pour les services qui attendent des données MFL, la variable doit contenir l'équivalent XML.
      • Pour les services qui attendent des données textuelles, la variable doit être une chaîne.
    • Dans le champ Variable de document de réponse, entrez le nom de la variable à laquelle affecter un document de réponse lors de l'exécution.

  8. Si vous le souhaitez, vous pouvez ajouter des en-têtes de transport. Pour plus d'informations, reportez-vous à Ajout d'actions d'en-têtes de transport.
    9. Remarque : outre les en-têtes de transport que vous spécifiez, des en-têtes sont ajoutés par la couche de liaison Oracle Service Bus. Pour plus d'informations, reportez-vous à "Configuring Transport Headers in Message Flows" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

Après avoir fini

Une fois la configuration de cette action terminée, vous pouvez configurer d'autres actions ou enregistrer votre configuration, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
