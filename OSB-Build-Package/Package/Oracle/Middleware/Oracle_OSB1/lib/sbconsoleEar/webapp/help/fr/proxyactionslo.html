Ajout de tables de routage aux noeuds de routage

Une table de routage est un ensemble de routes encapsulées dans une table de condition de permutation de style. Il s'agit d'une construction abrégée permettant de sélectionner différentes routes en fonction des résultats d'une seule expression XQuery.

Il existe une limite d'imbrication de quatre niveaux cumulés dans l'éditeur de phases. Si vous tentez d'ajouter un cinquième niveau, l'action d'imbrication n'apparaît pas. Les niveaux cumulés incluent toutes les actions de branchement : les conditions If... Then..., les tables de publication et les tables de routage. Vous pouvez ainsi avoir deux niveaux d'action de condition et une table de publication contenant une table de routage, ce qui donne quatre niveaux au total. Si vous tentez d'ajouter une action de condition supplémentaire (à la table de publication), elle n'apparaîtra pas.

Les tables de routage permettent d'identifier des services cible pour les messages et de configurer la méthode de routage des messages vers ces services.

Il s'agit d'une action finale ; cela signifie que vous ne pouvez pas ajouter d'autres actions après celle-ci. Cette action peut toutefois inclure des actions de demande et de réponse.

Pour plus d'informations, reportez-vous à "Modeling Message Flow in Oracle Service Bus" dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG181.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter une table de routage à un noeud de routage, procédez comme suit :

  1. Accédez à l'emplacement auquel ajouter l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
  2. Cliquez sur l'icône Noeud de routage, puis sur Modifier le routage. La page Modifier la configuration de la phase apparaît.
  3. Cliquez sur l'icône Ajouter une action, puis sélectionnez Communication > Table de routage. L'action de table de routage apparaît.
  4. Dans la liste Opérateur, sélectionnez un opérateur de comparaison, puis entrez une expression de valeur dans le champ adjacent.
  5. Cliquez sur <Service>. La page Sélectionner un service apparaît.
  6. Sélectionnez un service dans la liste, puis cliquez sur Soumettre.
  7. Pour appeler une opération sur ce service, sélectionnez une opération dans la liste Opération.
  8. Si vous souhaitez que l'opération sortante soit identique à l'opération entrante, cochez la case Utiliser une opération entrante pour l'opération sortante.
  9. Dans le champ Actions de demande, cliquez sur Ajouter une action pour ajouter une action, puis sélectionnez l'action à associer au service. Vous pouvez ajouter plusieurs actions.
  10. Dans le champ Actions de réponse, cliquez sur Ajouter une action pour ajouter une action, puis sélectionnez l'action à associer au service. Vous pouvez ajouter plusieurs actions.

    11. Pour plus d'informations sur les types de demande et de réponse que vous pouvez ajouter, reportez-vous à Ajout et modification d'actions dans les flux de messages.

  11. Pour insérer un nouveau cas, cliquez sur l'icône Cas, puis sélectionnez Insérer un nouveau cas.
  12. Répétez les étapes 2 à 7 pour le nouveau cas. Vous pouvez cliquer sur l'icône Cas, puis sélectionner Insérer un cas par défaut pour ajouter, à la fin, un cas par défaut dont les routes sont sélectionnées si aucun des cas précédents n'est satisfait.
  13. Cliquez sur Enregistrer.
  14. Sur la page Modifier le flux de messages, poursuivez la construction du flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.
  15. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  16. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
