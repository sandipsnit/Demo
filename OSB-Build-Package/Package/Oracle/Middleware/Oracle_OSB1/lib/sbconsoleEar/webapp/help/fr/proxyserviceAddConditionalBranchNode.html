Ajout de branchements conditionnels aux flux de messages

Un noeud de branchement permet au traitement de suivre un seul des multiples chemins possibles. Un branchement est défini par une table de commutateurs basée sur XPath. Chaque branchement de la table indique une condition (par exemple, <500) qui est évaluée dans l'ordre dans le flux de messages par rapport à une seule expression XPath (par exemple, ./ns: PurchaseOrder/ns:totalCost sur $body). La condition qui est satisfaite en premier détermine le branchement qui est suivi. Si aucune condition de branchement n'est satisfaite, c'est le branchement par défaut qui est utilisé. Un noeud de branchement peut comporter plusieurs descendants dans le flux de messages : un pour chaque branchement, y compris le branchement par défaut.

Si le service proxy n'est pas basé sur un WSDL et qu'il reçoit plusieurs types de document comme entrée, il est conseillé d'utiliser un noeud de branchement conditionnel.

Un branchement conditionnel est défini par une table de consultation dans laquelle chaque branchement est balisé avec une valeur de chaîne simple et unique. Une variable incluse dans le contexte de message est désignée comme variable de consultation pour ce noeud et, à l'exécution, sa valeur est utilisée pour déterminer le branchement à suivre. Si aucun branchement ne correspond à la valeur de la variable de consultation, c'est le branchement par défaut qui est utilisé. Vous devez concevoir le service proxy de telle sorte que la valeur de la variable de consultation soit définie avant que le noeud de branchement soit atteint.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter un branchement conditionnel à un flux de messages, procédez comme suit :

  1. Cliquez sur l'icône Noeud de paire de pipeline ou Noeud de branchement, puis sur Créer un noeud de branchement conditionnel. Le noeud de branchement conditionnel est ajouté et tous les noeuds existants après le noeud de branchement inséré sont déplacés vers le branchement par défaut du nouveau noeud de branchement conditionnel.
  2. Pour modifier le nom par défaut du noeud de branchement et ajouter une description, procédez comme suit :
    1. Cliquez sur l'icône de branchement conditionnel, puis sur Modifier le nom et l'annotation.
    2. Modifiez le nom et la description, comme souhaité.
    3. Cliquez sur Enregistrer.
  3. Pour ajouter des définitions de branchement, cliquez sur l'icône de branchement conditionnel, puis sur Modifier le branchement. La page Modifier le noeud de branchement apparaît.
  4. Procédez comme suit :
    1. Dans le champ Chemin sélectionné, cliquez sur Modifier pour ajouter une expression XPath indiquant le chemin. Reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.
    2. Dans le champ Variable, entrez une variable de contexte.
    3. Dans le champ Opérateur, sélectionnez un opérateur de comparaison.
    4. Dans le champ Valeur, entrez la valeur du branchement.
    5. Dans le champ Etiquette, entrez l'étiquette du branchement.
  5. Si vous le souhaitez, sous Options, effectuez les opérations suivantes :
    • Cliquez sur Ajouter un nouveau branchement pour ajouter une nouvelle définition de branchement à ce noeud de branchement.
    • Cliquez sur Supprimer ce branchement pour supprimer une définition de branchement.
    • Cliquez sur Déplacer le branchement vers le haut ou Déplacer le branchement vers le bas pour modifier la position des définitions de branchement. Cette option n'apparaît que s'il existe plusieurs définitions de branchement.
  6. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  7. Sur la page Modifier le flux de messages, continuez à construire le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.
  8. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  9. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
