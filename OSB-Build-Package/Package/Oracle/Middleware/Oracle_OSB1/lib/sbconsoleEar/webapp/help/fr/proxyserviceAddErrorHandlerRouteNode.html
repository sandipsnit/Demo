Ajout de gestionnaires d'erreur de noeud de routage

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Elles supposent également que vous ayez déjà créé un noeud de routage, comme expliqué dans Ajout de noeuds de routage aux flux de messages.

Pour ajouter un gestionnaire de noeud de routage, procédez comme suit :

  1. Cliquez sur l'icône Noeud de routage, puis sur Ajouter un gestionnaire d'erreur. La page Modifier le gestionnaire d'erreur apparaît.
  2. Cliquez sur l'icône Gestionnaire d'erreur, puis sur Ajouter une phase.
  3. Cliquez sur l'icône Phase, puis sur Modifier la phase. La page Modifier la configuration de la phase apparaît.
  4. Cliquez sur Ajouter une action, puis sélectionnez l'action à ajouter.

    5. Un gestionnaire d'erreur est un pipeline ; il est donc configuré comme n'importe quel autre pipeline. Par exemple, vous pouvez utiliser l'action Publier pour envoyer des notifications d'erreur à d'autres services, l'action Affecter pour modifier les variables de contexte, etc. Pour plus d'informations sur le type d'action à ajouter, reportez-vous à la procédure appropriée dans Ajout et modification d'actions dans les flux de messages. Il n'existe aucune restriction concernant le chaînage des actions entre elles.

    Les trois actions sur les erreurs les plus couramment utilisées sont les actions de génération d'erreur, de réponse et de reprise.

  5. Ajoutez d'autres actions et apportez des modifications supplémentaires, comme souhaité, à partir de la page Modifier la configuration de la phase.
  6. Sur la page Modifier la configuration de la phase, cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  7. Sur la page Modifier le gestionnaire d'erreur, cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.

Après avoir fini

Sur la page Modifier le flux de messages, continuez à construire le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.
