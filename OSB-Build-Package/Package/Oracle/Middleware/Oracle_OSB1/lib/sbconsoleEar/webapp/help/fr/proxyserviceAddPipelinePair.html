Ajout de noeuds de paire de pipeline aux flux de messages

Les flux de messages peuvent inclure ou non des noeuds de paire de pipeline : pipelines des demandes et des réponses pour le service proxy (ou pour les opérations du service), et pipelines de gestionnaire d'erreur, qui peuvent être définis pour les phases, les pipelines et les services proxy. Les pipelines peuvent inclure plusieurs phases, qui contiennent elles-mêmes plusieurs actions.

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter un noeud de paire de pipeline à un flux de messages, procédez comme suit :

  1. Cliquez sur l'icône Service proxy, puis sur Ajouter une paire de pipelines.
  2. Pour modifier le nom par défaut du noeud de paire de pipeline et ajouter une description, procédez comme suit :
    1. Cliquez sur l'icône Noeud de paire de pipeline, puis sur Modifier le nom et l'annotation.
    2. Modifiez le nom et la description, comme souhaité.
    3. Cliquez sur Enregistrer.
      4. Remarque : lorsque vous renommez un pipeline ou un noeud de routage, il se peut que le nombre de messages affiché sur la page Tableau de bord du module Surveillance ne corresponde pas à celui des autres composants, car les compteurs de pipeline sont remis à zéro. En effet, Oracle Service Bus considère l'action de changement de nom comme une action de suppression et de recréation. Après une période équivalant à l'intervalle de surveillance du service, les nombres devraient à nouveau correspondre.
  3. Pour ajouter des phases au pipeline, reportez-vous à Ajout de phases aux pipelines.
  4. Pour ajouter des opérations aux phases du pipeline, Reportez-vous à Ajout et modification d'actions dans les flux de messages.
  5. Sur la page Modifier le flux de messages, continuez à construire le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.
  6. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  7. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
