Ajout de noeuds de routage aux flux de messages

Avant de commencer

Les instructions suivantes supposent que vous soyez déjà en train de modifier un flux de messages sur la page Modifier le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.

Pour ajouter un noeud de routage à un flux de messages, procédez comme suit :

  1. Cliquez sur l'icône Noeud de paire de pipeline d'une paire de pipelines, puis sur Ajouter un routage.
  2. Pour modifier le nom par défaut du noeud de routage et ajouter une description, cliquez sur l'icône Noeud de routage, puis sur Modifier le nom et l'annotation. Modifiez le nom et la description comme souhaité, puis cliquez sur Enregistrer.
    3. Remarque : lorsque vous renommez un pipeline ou un noeud de routage, il se peut que le nombre de messages affiché sur la page Tableau de bord du module Surveillance ne corresponde pas à celui des autres composants, car les compteurs de pipeline sont remis à zéro. En effet, Oracle Service Bus considère l'action de changement de nom comme une action de suppression et de recréation. Après une période équivalant à l'intervalle de surveillance du service, les nombres devraient à nouveau correspondre.
  3. Pour ajouter des actions au noeud de routage, cliquez sur l'icône Noeud de routage, puis sur Modifier le routage. La page Modifier le flux de messages apparaît. Pour plus d'informations sur les différentes actions que vous pouvez ajouter aux noeuds de routage, reportez-vous aux sections suivantes :
    • Ajout d'actions If...Then...
    • Ajout d'un routage dynamique aux noeuds de routage
    • Ajout d'actions de routage aux noeuds de routage
    • Ajout de tables de routage aux noeuds de routage
    • Ajout de gestionnaires d'erreur de noeud de routage
  4. Sur la page Modifier le flux de messages, continuez à construire le flux de messages, comme indiqué dans Création, modification et visualisation de flux de messages.
  5. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  6. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
