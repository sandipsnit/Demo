Page Courrier électronique - Configuration du transport

La page Créer/Modifier un service proxy : Courrier électronique – Configuration du transport est une des pages permettant de créer et de modifier des services proxy, comme indiqué dans Création et configuration de services proxy et Modification des configurations des services proxy.

Utilisez cette page pour configurer les paramètres de transport d'un service proxy utilisant le protocole de transport de messagerie électronique. Le tableau 19-4 explique comment utiliser cette page.

Tableau 19-4 Page Créer/Modifier un service proxy : Courrier électronique – Configuration du transport
Option
Procédure
SSL obligatoire
Sélectionnez cette option pour communiquer via SSL (Secure Sockets Layer). Les numéros de port SSL par défaut sont les suivants :
  • POP3(S) : 995
  • IMAP : 993
Remarque : le serveur de messagerie doit figurer dans le truststore Oracle Service Bus pour que SSL fonctionne.
Compte de service
Entrez le nom d'un compte de service ou cliquez sur Parcourir pour sélectionner un compte de service à partir d'un navigateur Web.
Serveur géré
Ce champ n'est disponible que pour les domaines inclus dans un cluster.
Sélectionnez le serveur géré qui doit agir en tant que serveur d'interrogation. Tous les serveurs gérés peuvent traiter le message, mais un seul peut l'interroger.
Intervalle d'appel
Indiquez un intervalle d'appel, en secondes.
Protocole de messagerie électronique
Sélectionnez le type de serveur POP3 ou IMAP pour le compte de messagerie électronique.
Limite de lecture
Indiquez le nombre maximal de messages à lire par balayage d'interrogation. Entrez 0 pour indiquer un nombre illimité.
Transmission par référence
Cochez cette case pour stocker le fichier dans le répertoire d'archivage et le transmettre comme référence dans les en-têtes.
Par défaut, lorsque vous créez un service, l'option Transmission par référence est sélectionnée et vous devez spécifier l'emplacement du répertoire d'archivage.
Transmission des pièces jointes par référence
Cochez cette case pour stocker les pièces jointes dans le répertoire d'archivage et les transmettre comme référence dans les en-têtes.
Par défaut, quand l'option Transmission par référence est sélectionnée, le paramètre Transmission des pièces jointes par référence est implicitement défini sur True et vous devez spécifier l'emplacement du répertoire d'archivage.
Action après lecture
Indiquez ce qu'il advient d'un message une fois que celui-ci a été lu :
  • Archivage : le message est archivé.
  • Supprimer : le message est supprimé.
  • Déplacer : le message est déplacé. L'option Déplacer n'est disponible qu'avec le protocole IMAP.
Pièces jointes
Indiquez la façon dont les pièces jointes doivent être gérées :
  • Archivage : les pièces jointes sont enregistrées dans le répertoire d'archivage.
  • Ignorer : les pièces jointes ne sont pas prises en compte.
Dossier de déplacement IMAP
Indiquez le dossier dans lequel le message est déplacé lorsque le champ Action après lecture est défini sur Déplacer.
Répertoire de téléchargement
Indiquez l'emplacement temporaire dans lequel les courriers électroniques sont téléchargés.
Répertoire d'archivage
Indiquez le chemin de l'emplacement d'archivage à utiliser lorsque le champ Action après lecture est défini sur Archivage.
Ce champ est obligatoire si vous sélectionnez l'option Transmission par référence ou Transmission des pièces jointes par référence.
Répertoire d'erreurs
Indiquez le chemin du répertoire du système de fichiers dans lequel écrire le message et les pièces jointes en cas de problème.
Encodage de la demande
Acceptez l'encodage de jeu de caractères par défaut, ISO-8859-1, pour les demandes de transport de messagerie électronique ou indiquez un encodage de jeu de caractères différent.

Après avoir fini

Cliquez sur Suivant pour poursuivre la configuration du service sur la page suivante ou cliquez sur Dernier pour vérifier et enregistrer cette configuration sur la page Créer/Modifier un service proxy : Récapitulatif.
