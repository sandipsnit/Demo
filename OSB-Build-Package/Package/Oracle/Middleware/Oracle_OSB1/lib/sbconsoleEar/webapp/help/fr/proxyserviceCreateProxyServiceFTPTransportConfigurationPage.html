Page FTP - Configuration du transport

La page Créer/Modifier un service proxy : FTP – Configuration du transport est une des pages permettant de créer et de modifier des services proxy, comme indiqué dans Création et configuration de services proxy et Modification des configurations des services proxy.

Utilisez cette page pour configurer les paramètres de transport d'un service proxy utilisant le protocole de transport FTP. Le tableau 19-6 explique comment utiliser cette page.

Tableau 19-6 Page Créer/Modifier un service proxy : FTP – Configuration du transport
Option
Procédure
Authentification utilisateur
Sélectionnez anonyme si l'utilisateur du serveur FTP est anonyme ou utilisateur externe s'il s'agit d'un compte configuré en externe.
Identité (adresse électronique)
Ce champ n'est disponible que si vous définissez l'option Authentification utilisateur sur anonyme.
Entrez l'adresse électronique de l'utilisateur anonyme.
Compte de service
Ce champ n'est disponible que si vous définissez l'option Authentification utilisateur sur utilisateur externe.
Indiquez le compte de service de l'utilisateur. Ce champ est obligatoire lorsque l'option Authentification utilisateur est définie sur utilisateur externe.
Transmission par référence
Cochez cette case pour stocker le fichier dans le répertoire d'archivage et le transmettre comme référence dans les en-têtes.
Transmission en continu distante
Lorsque vous cochez cette case, les fichiers FTP sont directement transmis en continu à partir du serveur distant au moment du traitement. Lorsque vous sélectionnez cette option, le répertoire d'archivage correspond au répertoire distant du serveur FTP distant. Par conséquent, vous devez indiquer le répertoire d'archivage comme étant relatif au répertoire utilisateur FTP.
Masque de fichier
Entrez l'expression régulière définissant les fichiers à sélectionner. La valeur par défaut est *.*.
Serveur géré
Ce champ n'est disponible que pour les domaines inclus dans un cluster.
Sélectionnez le serveur géré qui doit agir en tant que serveur d'interrogation. Tous les serveurs gérés peuvent traiter le message, mais un seul peut l'interroger.
Intervalle d'appel
Indiquez un intervalle d'appel, en secondes. La valeur par défaut est 60.
Limite de lecture
Indiquez le nombre maximal de messages à lire par balayage d'interrogation. Entrez 0 pour indiquer un nombre illimité. La valeur par défaut est 10.
Action après lecture
Indiquez ce qu'il advient d'un message une fois que celui-ci a été lu.
  • Archivage : le message est archivé.
  • Supprimer : le message est supprimé.
Mode de transfert
Sélectionnez le mode de transfert ASCII ou binaire.
Répertoire d'archivage
Indiquez le chemin de l'emplacement d'archivage à utiliser lorsque l'option Action après lecture est définie sur Archivage. Ce champ est obligatoire si vous avez sélectionné l'option Transmission par référence.
Remarque : les répertoires d'archivage, de téléchargement et des erreurs représentent des chemins absolus ; ils sont créés automatiquement. Si vous indiquez un chemin relatif, les fichiers sont créés relativement au processus Java qui démarre le serveur WebLogic.
Répertoire de téléchargement
Indiquez le répertoire de l'ordinateur local dans lequel sont téléchargés les fichiers lors de leur transfert.
Remarque : les répertoires d'archivage, de téléchargement et des erreurs représentent des chemins absolus ; ils sont créés automatiquement. Si vous indiquez un chemin relatif, les fichiers sont créés relativement au processus Java qui démarre le serveur WebLogic.
Répertoire d'erreurs
Indiquez l'emplacement dans lequel les messages sont envoyés en cas de problème.
Remarque : les répertoires d'archivage, de téléchargement et des erreurs représentent des chemins absolus ; ils sont créés automatiquement. Si vous indiquez un chemin relatif, les fichiers sont créés relativement au processus Java qui démarre le serveur WebLogic.
Encodage de la demande
Acceptez l'encodage de jeu de caractères par défaut, UTF-8, pour les demandes dans les transports FTP.
Balayer les sous-répertoires
Cochez cette case pour balayer tous les répertoires de façon récursive.
Trier par ordre d'arrivée
Cochez cette case pour indiquer que les événements sont livrés dans leur ordre d'arrivée.
Temporisation
Indiquez l'intervalle de temporisation de socket, en secondes, au bout duquel la connexion est supprimée. La valeur 0 signifie qu'il n'y a jamais temporisation.
Nombre de tentatives
Indiquez le nombre de tentatives en cas d'échec de la connexion FTP.

Après avoir fini

Cliquez sur Suivant pour poursuivre la configuration du service sur la page suivante ou cliquez sur Dernier pour vérifier et enregistrer cette configuration sur la page Créer/Modifier un service proxy : Récapitulatif.
