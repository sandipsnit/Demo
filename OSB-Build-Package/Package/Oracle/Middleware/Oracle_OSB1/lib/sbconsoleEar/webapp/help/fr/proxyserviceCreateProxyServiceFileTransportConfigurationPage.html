Page Fichier - Configuration du transport

La page Créer/Modifier un service proxy : Fichier – Configuration du transport est une des pages permettant de créer et de modifier des services proxy, comme indiqué dans Création et configuration de services proxy et Modification des configurations des services proxy.

Utilisez cette page pour configurer les paramètres de transport d'un service proxy utilisant le protocole de transport de fichier. Le tableau 19-5 explique comment utiliser cette page.

Tableau 19-5 Page Créer/Modifier un service proxy : Fichier – Configuration du transport
Option
Procédure
Masque de fichier
Indique les fichiers qui doivent être interrogés par le service proxy. Si l'URI représente un répertoire et que vous indiquez *.*, le service interrogera tous les fichiers du répertoire. Seuls les caractères génériques * et ? sont autorisés dans Masque de fichier. Les expressions régulières ne sont pas prises en charge.
Serveur géré
Ce champ n'est disponible que pour les domaines inclus dans un cluster.
Sélectionnez le serveur géré qui doit agir en tant que serveur d'interrogation. Tous les serveurs gérés peuvent traiter le message, mais un seul peut l'interroger.
Intervalle d'appel
Indiquez un intervalle d'appel, en secondes. La valeur par défaut est 60.
Limite de lecture
Indiquez le nombre maximal de messages à lire par balayage d'interrogation. Entrez 0 pour indiquer un nombre illimité. La valeur par défaut est 10.
Trier par ordre d'arrivée
Cochez cette case pour indiquer que les événements sont livrés dans leur ordre d'arrivée.
Lorsque cette option est sélectionnée pour un service proxy exécuté dans un environnement inclus dans un cluster, les messages sont toujours envoyés au même serveur. En d'autres termes, l'équilibrage de charge entre les serveurs n'est pas pris en compte lorsque cette option est sélectionnée.
Balayer les sous-répertoires
Cochez cette case pour balayer tous les répertoires de façon récursive.
Transmission par référence
Cochez cette case pour stocker le fichier dans le répertoire d'archivage et le transmettre comme référence dans les en-têtes.
Action après lecture
Indiquez ce qu'il advient d'un message une fois que celui-ci a été lu :
  • Archivage : le message est archivé.
  • Supprimer : le message est supprimé.
Répertoire intermédiaire
Indiquez le répertoire intermédiaire à utiliser pour le stockage temporaire des fichiers pendant leur traitement.
Ne placez pas le répertoire intermédiaire dans le répertoire d'interrogation (c'est-à-dire le répertoire identifié dans l'URL du service proxy de transport de fichier ; par exemple, file:///c:/dir1/dir2).
Répertoire d'archivage
Indiquez le chemin de l'emplacement d'archivage à utiliser lorsque l'option Action après lecture est définie sur Archivage. Le champ Répertoire d'archivage est également obligatoire si vous avez sélectionné le champ Transmission par référence.
Ne placez pas le répertoire d'archivage dans le répertoire d'interrogation.
Répertoire d'erreurs
Indiquez l'emplacement dans lequel les messages et les pièces jointes sont envoyés en cas de problème.
Ne placez pas le répertoire des erreurs dans le répertoire d'interrogation.
Encodage de la demande
Acceptez l'encodage de jeu de caractères par défaut, UTF-8, pour les demandes dans les transports de fichier ou indiquez un encodage de jeu de caractères différent.

Après avoir fini

Cliquez sur Suivant pour poursuivre la configuration du service sur la page suivante ou cliquez sur Dernier pour vérifier et enregistrer cette configuration sur la page Créer/Modifier un service proxy : Récapitulatif.
