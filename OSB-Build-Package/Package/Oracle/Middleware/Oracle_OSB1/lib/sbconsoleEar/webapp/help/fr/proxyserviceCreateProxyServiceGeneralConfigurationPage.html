Configuration générale, page

La page Créer/Modifier un service proxy : Configuration générale est une des pages permettant de créer et de modifier des services proxy, comme indiqué dans Création et configuration de services proxy et Modification des configurations des services proxy.

Utilisez cette page pour modifier les paramètres de configuration générale d'un service proxy.

Lorsque vous créez un service proxy, cette page est la première qui apparaît sur tout un ensemble de pages permettant de configurer le service. Les pages affichées ensuite dépendent des choix effectués sur celle-ci.

Le tableau 19-1 explique comment utiliser cette page.

Tableau 19-1 Page Créer/Modifier un service proxy : Configuration générale 
Option
Procédure
Nom du service
Entrez un nom unique pour le service proxy.

Pour obtenir des conseils sur la dénomination, suivez les restrictions concernant la dénomination des ressources.
Description
Saisissez une description.
Type de service
Sélectionnez le type de service proxy à créer :
  • Service Web WSDL : sélectionnez cette option pour créer un service proxy basé sur un WSDL. Saisissez ensuite le nom du WSDL, qualifié par son chemin (par exemple, myProject/myFolder/myWSDL). Vous pouvez également cliquer sur Parcourir pour faire votre choix à partir de la page Sélectionner une définition WSDL.

    • (port ou liaison) : entrez le nom d'un port (défini dans le WSDL) pour décrire une adresse de transport réelle, ou celui d'une liaison (définie dans le WSDL), pour établir une correspondance avec une adresse de transport. Si vous cliquez sur Parcourir pour sélectionner un WSDL, la page Sélectionner une définition WSDL répertorie l'ensemble des ports et liaisons définis.

Remarque : si vous voulez utiliser le type de corps SOAP pour les opérations, assurez-vous que le WSDL ne comporte pas deux opérations présentant le même message d'entrée. L'opération de type de corps SOAP ne peut pas être identifiée de façon unique en examinant le message d'entrée.
  • Service doté d'un type de transport : sélectionnez cette option pour créer un service proxy JEJB.
  • Service de messagerie : sélectionnez cette option pour créer un service pouvant recevoir des messages contenant un type de données particulier et répondre par des messages d'un type de données différent. Ces échanges peuvent être de type demande/réponse ou unidirectionnels.

    • (L'option GET HTTP est prise en charge uniquement pour les services de type N'importe quel service XML et Service de messagerie.)

  • N'importe quel service SOAP : sélectionnez cette option afin de créer un service SOAP pour lequel aucune interface concrète n'est explicitement définie.

    • Sélectionnez SOAP 1.1 ou SOAP 1.2 dans la liste déroulante pour indiquer la version SOAP à utiliser.

  • N'importe quel service XML : sélectionnez cette option afin de créer un service XML pour lequel aucune interface concrète n'est explicitement définie.

    • (L'option GET HTTP est prise en charge uniquement pour les services de type N'importe quel service XML et Service de messagerie.)

  • Service fonctionnel : sélectionnez cette option pour créer un service proxy comportant un noeud de routage acheminant vers le service fonctionnel sélectionné. Vous ne pouvez par créer de service proxy à partir d'un service fonctionnel doté d'un type de transport. Si vous créez un service proxy à partir d'un service fonctionnel de transport DSP, Oracle Service Bus permute le type de transport du service proxy vers HTTP, car le transport DSP ne peut pas être utilisé pour les services proxy. Vous pouvez définir le type de transport du service proxy sur tous les autres transports disponibles.

    • Entrez le chemin (project/folder) et le nom du service fonctionnel ou cliquez sur Parcourir pour sélectionner un service fonctionnel sur la page Sélectionner un service fonctionnel.

Type de service (suite)
  • Service proxy : sélectionnez cette option pour cloner un service proxy existant. Dans la mesure où Oracle Service Bus n'accepte pas un URI identique pour plusieurs services, vous devez modifier l'URI du service cloné.
Remarque : lorsque vous créez un service à partir d'un autre, les règles d'alerte sont gérées de la façon suivante :
    • Lorsque vous créez un service proxy à partir d'un service fonctionnel et inversement, les règles d'alerte existantes sont supprimées.
    • Lorsque vous créez un service proxy à partir d'un autre service proxy ou que vous créez un service fonctionnel à partir d'un autre service fonctionnel, les règles d'alerte existantes sont conservées.
Fournisseur de clés de service
Si nécessaire, entrez le chemin (project/folder) et le nom du fournisseur de clés de service, ou cliquez sur Parcourir pour en sélectionner un sur la page Sélectionner un fournisseur de clés de service.
Un fournisseur de clés de service n'est obligatoire que dans les cas suivants :
  • TLS/SSL bidirectionnel sortant, où le service proxy achemine les messages vers les services HTTPS qui requièrent une authentification de certificat client.
  • Dans certains scénarios de sécurité de service Web, lorsque le service proxy requiert le cryptage des messages, par exemple.
Pour ajouter un service proxy pour lequel la sécurité de service Web est activée, vous devez créer le service proxy à partir d'un WSDL (port ou liaison) comportant des pièces jointes de stratégie WS.
Pour plus d'informations, reportez-vous à Fournisseurs de clés de service. Pour savoir comment créer un fournisseur de clés de service, reportez-vous à Ajout de fournisseurs de clés de service.
Publier vers le registre
Cette option apparaît uniquement si un registre UDDI par défaut est présent.
Sélectionnez cette option pour publier automatiquement le service proxy dans le registre par défaut. Pour plus d'informations, reportez-vous à Utilisation de la publication automatique.

Après avoir fini

Cliquez sur Suivant pour poursuivre la configuration du service sur la page suivante ou cliquez sur Dernier pour vérifier et enregistrer cette configuration sur la page Créer/Modifier un service proxy : Récapitulatif.
