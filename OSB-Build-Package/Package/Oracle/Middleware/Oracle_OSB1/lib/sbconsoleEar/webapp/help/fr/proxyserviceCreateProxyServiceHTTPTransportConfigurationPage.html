Page HTTP - Configuration du transport

Le transport HTTP prend désormais en charge les adresses HTTP et HTTPS.

La page Créer/Modifier un service proxy : HTTP – Configuration du transport est une des pages permettant de créer et de modifier des services proxy, comme indiqué dans Création et configuration de services proxy et Modification des configurations des services proxy.

Utilisez cette page pour configurer les paramètres de transport d'un service proxy utilisant le protocole de transport HTTP. Le tableau 19-7 explique comment utiliser cette page.

Tableau 19-7 Page Créer/Modifier un service proxy : HTTP – Configuration du transport
Option
Procédure
HTTPS obligatoire
Cochez cette case pour les adresses HTTPS entrantes.
Pour plus d'informations, reportez-vous à "Configuring Transport-Level Security" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
Authentification
Sélectionnez l'une des options suivantes :
  • Aucune : indique qu'aucune authentification n'est nécessaire.
  • De base : indique qu'une authentification de base est nécessaire pour accéder à ce service.

    • Lorsque l'authentification de base est activée, WebLogic Server authentifie le client à l'aide d'un nom utilisateur et d'un mot de passe auprès des fournisseurs d'authentification configurés dans le domaine de sécurité, comme un service d'annuaire LDAP (Lightweight Directory Access Protocol) et Windows Active Directory. Le client doit envoyer ses nom utilisateur et mot de passe dans l'en-tête de demande HTTP.

    Il est fortement déconseillé d'utiliser l'authentification de base sur HTTP, car les mots de passe sont envoyés en clair. Vous pouvez en revanche envoyer des mots de passe via HTTPS en toute sécurité, car ce protocole présente un canal crypté.

    Avertissement : par défaut, tous les utilisateurs (autorisés et anonymes) peuvent accéder à un service proxy. Pour limiter l'accès à ce dernier, vous devez créer une stratégie d'autorisation basée sur le transport. Reportez-vous à Modification des stratégies d'accès au niveau du transport.

  • Certificat client : indique qu'une communication cryptée et une authentification client forte (SSL bidirectionnelle) sont utilisées. Pour plus d'informations, reportez-vous à "Configuring Transport-Level Security" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
  • Authentification personnalisée : indique qu'un jeton d'authentification est contenu dans un en-tête HTTP. L'identité du client est établie grâce à l'utilisation de ce jeton fourni par le client. Vous devez configurer un fournisseur d'assertion d'identité qui met en correspondance le jeton avec un utilisateur Oracle Service Bus.

    • Le jeton d'authentification personnalisée peut correspondre à tout type de jeton pris en charge par un fournisseur d'assertion d'identité WebLogic Server.

Stratégie de répartition
Sélectionnez une stratégie de répartition pour cette adresse. Laissez ce champ vide pour utiliser la stratégie de répartition par défaut.
La stratégie de répartition fait référence au gestionnaire de travaux WLS utilisé pour l'adresse de service. Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Encodage de la demande
  • Pour les transports HTTP entrants :

    • Si aucun encodage de jeu de caractères n'est défini pour l'en-tête Content-Type dans le champ Demande client, indiquez-en un. Si vous n'entrez aucune valeur, le champ est défini par défaut sur ISO-8859-1.

  • Pour les transports HTTP sortants :

    • Si vous n'avez pas configuré l'encodage de la demande, l'exécution d'Oracle Service Bus détermine l'encodage le plus approprié via une demande au service fonctionnel. En cas de scénario non pass-through, l'encodage de caractères est défini par défaut sur UTF-8 lors de l'exécution. En revanche, s'il s'agit d'un scénario pass-through, l'exécution transmettra l'encodage reçu avec la réponse sortante.

Encodage de la réponse
Acceptez l'encodage de jeu de caractères par défaut, ISO-8859-1, pour les demandes dans les transports HTTP ou indiquez un encodage de jeu de caractères différent.
En-tête d'authentification
Indiquez l'en-tête HTTP (n'importe lequel sauf Authorization) à partir duquel Oracle Service Bus doit extraire le jeton. Ce champ n'est disponible que si vous avez coché la case Authentification personnalisée.
Par exemple, client-xyz-token.
Type de jeton d'authentification
Sélectionnez un type de jeton d'authentification. Seuls les types de jetons actifs configurés pour un fournisseur d'assertion d'identité sont disponibles. (Pour plus d'informations, reportez-vous à Configuration des fournisseurs d'assertion d'identité pour les jetons personnalisés.) Ce champ n'est disponible que si vous avec coché la case Authentification personnalisée.
