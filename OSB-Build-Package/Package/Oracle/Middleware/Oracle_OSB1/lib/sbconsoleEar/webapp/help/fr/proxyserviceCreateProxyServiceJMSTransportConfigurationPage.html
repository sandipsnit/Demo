Page JMS - Configuration du transport

La page Créer/Modifier un service proxy - JMS - Configuration du transport est l'une des pages destinées à la création et à la modification de services proxy, décrites dans les rubriques Création et configuration de services proxy et Modification des configurations des services proxy.

Pour plus d'informations, reportez-vous à "JMS Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Cette page permet de configurer les paramètres de transport pour un service proxy utilisant le protocole de transport JMS. Le tableau 19-10 explique comment l'utiliser.

Tableau 19-10 Page Créer/Modifier un service proxy - JMS - Configuration du transport
Option
Procédure
Type de destination
Sélectionnez l'une des options suivantes :
  • File d'attente (pour un type de destination point à point)
  • Rubrique (pour une destination publication/abonnement)
Réponse requise ?
Cette option est disponible uniquement lorsque l'option File d'attente est sélectionnée dans Type de destination.
Sélectionnez cette option pour indiquer qu'une réponse est attendue lorsqu'un message sortant est envoyé.
Modèle de réponse
Cette option est disponible uniquement lorsque la case Réponse requise ? est cochée.
Sélectionnez l'une des options suivantes :
  • Sélectionnez JMSMessageID pour les services JAX-RPC exécutés sur Oracle WebLogic Server.
  • Sélectionnez JMSCorrelationID pour tous les autres services. Lorsque vous sélectionnez cette option, vous devez également entrer un URI de réponse.
Type de message de réponse
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Sélectionnez l'une des options suivantes :
  • Octets (flux d'octets non interprétés)
  • Texte (messages texte)
Cette option est désactivée si vous sélectionnez un type de message Java pour la réponse.
Fichier JAR client
Cette option est disponible dans le cas d'un service de messagerie présentant un type de demande Java. Sélectionnez le JAR client à utiliser lors du retrait de la file d'attente des messages contenant des objets Java. La sélection du JAR client garantit qu'il figure dans la variable d'environnement CLASSPATH.

Pour plus d'informations, reportez-vous à "Sending and Receiving Java Objects in Messages" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Stratégie de répartition
Sélectionnez l'instance du gestionnaire de travaux Oracle WebLogic Server à utiliser dans la stratégie de répartition de cette adresse. Le gestionnaire de travaux par défaut est utilisé s'il n'en existe pas d'autre.
Par exemple, si le service fonctionnel utilise un protocole de transport JMS, son adresse consiste en un fichier JAR MDB (bean orienté message) pouvant être associé à la stratégie de répartition définie.
Pour obtenir des informations sur les gestionnaires de travaux, reportez-vous à Using Work Managers to Optimize Scheduled Work dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Encodage de la demande
Indiquez le jeu de caractères utilisé pour l'encodage des demandes. La valeur par défaut est UTF-8.
Encodage de la réponse
Cette option est disponible uniquement lorsque la case Réponse requise ? est cochée.
Indiquez le jeu de caractères utilisé pour l'encodage des réponses. La valeur par défaut est UTF-8.
Temporisation de réponse client
Cette option est disponible uniquement lorsque la case Réponse requise ? est cochée.
Entrez la durée d'attente, en secondes, pour obtenir une réponse du serveur avant de supprimer la connexion. Cela s'applique uniquement si le client est un autre service proxy dans le même domaine.
URI de réponse
Cette option est disponible uniquement lorsque JMSCorrelationID est sélectionné dans Modèle de corrélation de réponse.
Entrez un URI de réponse au format suivant :
jms://host:port/connection_factory/jndi_destination.
Pour cibler plusieurs serveurs, utilisez le format suivant :
jms://host1:port,host2:port/connection_factory/jndi_destination.
Vous pouvez aussi n'indiquer ni l'hôte, ni le port dans l'URI de réponse. Par exemple :
jms:///connection_factory/jndi_destination.

Lorsque vous n'indiquez ni l'hôte, ni le port, la recherche de fabrique de connexions/destination s'effectue sur le serveur local. Ce comportement s'avère utile, par exemple, lorsque l'URI de demande pointe vers une fabrique de connexions/destination étrangère, mais que vous souhaitez envoyer la réponse au serveur local.

Remarque : lorsque Oracle WebLogic Server autorise les barres obliques dans les noms JNDI, tels que "myqueues/myqueue", les noms JNDI avec des barres obliques interfèrent avec le format URI requis par Oracle Service Bus, et vous ne pouvez pas utiliser ces noms. Pour passer outre ce problème, définissez un serveur étranger JMS et référencez-le dans l'URI. Pour plus d'informations, reportez-vous à Configurer les serveurs étrangers dans l'aide en ligne d'Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help à l'adresse http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Fabrique de connexions de réponse
Cette option est disponible uniquement lorsque JMSMessageID est sélectionné dans Modèle de corrélation de réponse.
Entrez un URI de fabrique de connexions de réponse.
Lorsqu'aucune fabrique de connexions n'est indiquée, la fabrique de connexions de la demande est utilisée pour la réponse.
Compte de service JMS
Sélectionnez un compte de service à utiliser avec la ressource JMS gérée par le serveur JMS. Un compte de service représente la ressource d'alias d'un ID utilisateur et d'un mot de passe, utilisée à la fois pour la demande et pour la réponse. Le même compte de service est employé pour les ressources JMS et JNDI.
Pour plus d'informations, reportez-vous à Comptes de service.
Utiliser SSL
Ne sélectionnez cette option que si les demandes sont effectuées via une connexion TLS/SSL.
Le protocole TLS/SSL (Secure Sockets Layer) garantit des connexions sécurisées en autorisant la connexion de deux applications sur un réseau pour authentifier l'identité de l'autre et en cryptant les données échangées entre ces applications. L'authentification permet à un serveur, et éventuellement à un client, de vérifier l'identité de l'application à l'autre extrémité d'une connexion réseau. De plus, si l'administrateur a limité l'accès à chaque destination JMS (files d'attente ou rubriques) en paramétrant le contrôle d'accès de l'entrée JNDI correspondant à la destination, le service doit s'authentifier lorsqu'il recherche l'entrée dans l'arborescence JNDI.
Sélecteur de message
Entrez une expression de sélecteur de messages.
Seuls les messages dont les propriétés correspondent à l'expression sont traités.
ID de client JMS

Pour les rubriques uniquement, entrez l'ID de client à utiliser pour l'abonné. Si aucune valeur n'est indiquée, l'ID de client est généré automatiquement. L'affectation d'un ID personnalisé vous aide à identifier le composant lors de la surveillance du service. Ceci n'est efficace que pour les abonnés durables.

L'ID client JMS est l'un des descripteurs de beans orientés message utilisés pour générer des noms d'abonné et la valeur ClientID pour les abonnés du sujet. La définition d'un ID client JMS facilite l'identification et la visualisation des abonnés pour un sujet spécifique en regardant le nom d'abonné ou l'ID client.

Pour plus d'informations, reportez-vous à "JMS Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.
Inscription de longue durée
Cette option est disponible uniquement si Rubrique est sélectionné dans Type de destination.
Cochez cette case si l'abonnement est durable. Si ce n'est pas le cas, laissez-la vide.
Nombre de tentatives
Entrez le nombre de tentatives de distribution dont un message dispose avant d'être placé dans la destination d'erreur. Ce champ s'applique uniquement aux destinations JMS d'Oracle WebLogic Server.
Intervalle entre les tentatives
Entrez la durée, en millisecondes, avant que les messages annulés (rollback) ou récupérés soient redistribués. Ce champ s'applique uniquement aux destinations JMS d'Oracle WebLogic Server.
Destination d'erreur
Entrez le nom de la destination cible des messages pour lesquels le nombre limite de tentatives a été atteint. Ce champ s'applique uniquement aux destinations JMS d'Oracle WebLogic Server.
Règle d'expiration
Sélectionnez la stratégie d'expiration à utiliser lorsqu'un message arrivé à expiration est détecté dans une destination. Ce champ s'applique uniquement aux destinations JMS d'Oracle WebLogic Server.
XA obligatoire ?
Cochez cette case si la fabrique de connexions est de type XA.
Cette valeur est prise en compte lorsque la fabrique de connexions distante n'est pas disponible. Si la fabrique de connexions est disponible et que cette option a la valeur True, assurez-vous que la fabrique de connexions est définie comme étant transactionnelle.
Temporisation de transaction
Entrez le délai (en secondes) d'attente de traitement d'une transaction par le service proxy JMS. Cette option s'applique uniquement aux services utilisant une fabrique de connexions XA. La valeur par défaut est de 600 secondes.
Diffusion des messages de rubrique
Cette option est disponible lorsque vous sélectionnez Rubrique pour l'option Type de destination. Diffusion des messages de rubrique propose différentes options permettant de traiter les messages JMS entrants et offrant des fonctions de haute disponibilité et de basculement en cas d'incident.

Sélectionnez l'une des propriétés suivantes pour déterminer comment les beans orientés message traitent les messages JMS entrants :
  • Une copie par application : (option par défaut) si le service proxy JMS est déployé vers un cluster, cette option offre une disponibilité et une évolutivité élevées en garantissant le traitement d'un message JMS entrant uniquement sur l'un des serveurs disponibles du cluster.
  • Une copie par serveur : sélectionnez cette option si vous souhaitez que les messages JMS entrants publiés dans une rubrique soient reçus par le service proxy de chaque membre du cluster.
  • Compatibilité : sélectionnez cette option si vous souhaitez que les messages JMS entrants soient traités sur un serveur géré spécifique ou sur l'ensemble des serveurs gérés dans un cluster. Si vous sélectionnez cette propriété, sélectionnez un serveur cible.
Remarque : pour les destinations de rubrique déployées sur Oracle WebLogic Server avant la version 10.3.4, Compatibilité est la seule option valide.
Les options de type Une copie que vous sélectionnez pour Diffusion des messages de rubrique remplacent les options Stratégie de partage d'abonnement et/ou Stratégie d'ID de client configurées sur la fabrique de connexions JMS.
Cible
Cette option est disponible uniquement dans un cluster Oracle Service Bus lorsque vous sélectionnez Compatibilité pour l'option Diffusion des messages de rubrique. Sélectionnez le serveur cible qui traitera les messages JMS entrants. Si vous sélectionnez l'une des options Une copie pour Diffusion des messages de rubrique, ce champ affiche le nom du cluster.

Si vous ne définissez pas une cible, l'instance de service proxy JMS (lecture des messages hors de la rubrique) sur chaque serveur géré dans le cluster obtient une copie du message.
Temporisation JNDI
Temporisation, en secondes, de la connexion JNDI utilisée lors de la recherche de la destination ou de la fabrique de connexions dans l'arborescence JNDI.
