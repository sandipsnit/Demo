Page MQ - Configuration du transport

La page Créer/Modifier un service proxy - MQ - Configuration du transport est l'une des pages destinées à la création et à la modification de services proxy, décrites dans les rubriques Création et configuration de services proxy et Modification des configurations des services proxy.

Pour plus d'informations, reportez-vous à "MQ Transport" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Avant de commencer

Configurez une ressource de connexion MQ. Reportez-vous à Connexions MQ.

Pour configurer le transport MQ

Cette page permet de configurer les paramètres de transport pour un service proxy utilisant le protocole de transport Native MQ. Le tableau 19-11 explique comment l'utiliser.

Tableau 19-11 Page Créer/Modifier un service proxy - MQ - Configuration du transport
Option
Procédure
Intervalle d'appel
Indiquez un intervalle d'appel, en millisecondes. La valeur par défaut est 1 000.
Réponse requise ?
Sélectionnez cette option pour indiquer qu'une réponse est attendue lorsqu'un message sortant est envoyé.
Modèle de corrélation de réponse
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Indiquez si le modèle de corrélation de réponse doit être basé sur MessageID ou CorrelationID.
URI de réponse MQ
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Destination vers laquelle la réponse doit être publiée. Entrez un URI de réponse au même format que l'URI endpoint :mq://<local-queue-name>?conn=<mq-connection-resource-ref>
Type de message de réponse
Cette option n'est disponible que si vous cochez la case Réponse requise ?.
Sélectionnez l'une des options suivantes :
  • Octets (flux d'octets non interprétés)
  • Texte (messages texte)
Stratégie de répartition
Sélectionnez une stratégie de répartition pour cette adresse.
La stratégie de répartition fait référence au gestionnaire de travaux WLS utilisé pour l'adresse de service. Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Seuil de coupure
Entrez une valeur représentant le nombre de fois que le pipeline doit renvoyer un message avant de le rediriger vers la file d'attente indiquée dans le champ URI de lettre morte.
Si vous n'indiquez pas de valeur dans ce champ, le message est redirigé vers la file d'attente des lettres mortes, sans aucune tentative de renvoi.
URI de lettre morte MQ
Entrez l'URI de la file d'attente des lettres mortes vers laquelle les messages de demande doivent être redirigés après le nombre de tentatives indiqué dans le champ Seuil de coupure.
Si vous n'indiquez pas de valeur pour ce champ, le message est renvoyé à la file d'attente et ignoré par le transport MQ pour chaque interrogation après le nombre de tentatives indiqué dans le champ Seuil de coupure. L'URI de lettre morte utilise le même format que l'URI endpoint.
Options 'GET' de l'URI d'adresse
Entrez les options de message MQ GET en choisissant parmi les suivantes :
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
Vous pouvez utiliser "|" ou "+" pour séparer les différentes options. Indiquez par exemple :
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
Les options de message MQ GET sont appliquées lors de la lecture d'un message de la file d'attente entrante.
Traiter les en-têtes RFH2
Sélectionnez cette option pour analyser les en-têtes RFH2 MQ WebSphere des données traitées de message et générer automatiquement un en-tête de transport RFH2Headers contenant les données RFH2.
Si vous ne sélectionnez pas cette option, les données traitées sont transmises telles qu'elles sont reçues.

Pour plus d'informations sur le mode de gestion des en-têtes RFH2 par le transport MQ, reportez-vous à About RFH2 Headers dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
