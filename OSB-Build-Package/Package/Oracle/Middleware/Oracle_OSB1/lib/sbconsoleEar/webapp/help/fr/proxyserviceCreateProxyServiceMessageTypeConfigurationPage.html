Page Configuration du type de message

La page Créer/Modifier un service proxy - Configuration du type de message est l'une des pages destinées à la création et à la modification de services proxy, décrites dans les rubriques Création et configuration de services proxy et Modification des configurations des services proxy.

Cette page permet de configurer les types de message pour un service proxy de type Service de messagerie.

La définition de liaison pour les services de messagerie consiste à configurer le type de contenu des messages échangés. Le type de contenu de la réponse ne doit pas nécessairement être le même que celui de la demande. Par conséquent, la réponse est configurée séparément (par exemple, le service peut accepter un message MFL et renvoyer un accusé de réception XML).

Remarque : Les services proxy de transport Courrier électronique, Fichier, FTP ou SFTP de type Service de messagerie prennent uniquement en charge la messagerie unidirectionnelle ; le type de message de réponse doit être défini sur aucun. Si vous sélectionnez une autre option, les protocoles Courrier électronique, Fichier, FTP ou SFTP ne seront pas disponibles sur la page de configuration du transport.

Le tableau 19-2 explique comment utiliser la page Créer/Modifier un service proxy - Configuration du type de message.

Tableau 19-2 Page Créer/Modifier un service proxy - Configuration du type de message
Option
Procédure
Type du message de demande
Sélectionnez le type de message de demande :
  • Aucune : sélectionnez cette option en l'absence de message de demande.
  • Binaire : sélectionnez cette option si le type de contenu du message est inconnu ou n'est pas important.
  • Texte : sélectionnez cette option si le message peut être limité à du texte.
  • MFL : sélectionnez cette option si le message est un document binaire conforme à une définition MFL. Entrez le nom de fichier MFL (qualifié par son chemin) ou cliquez sur Parcourir pour sélectionner un fichier.

    • Vous ne pouvez configurer qu'un seul fichier MFL.

    Remarque : pour prendre en charge plusieurs fichiers MFL, définissez le contenu sur Binaire ou Texte et utilisez l'action MFL du flux de messages pour le convertir en XML.

  • XML : sélectionnez cette option si le message est un document XML. Entrez le nom du fichier XML (qualifié par son chemin), ou cliquez sur Parcourir pour sélectionner un fichier.

    • Fournissez éventuellement des informations sur le type en déclarant (dans le champ Elément ou Type) le type de schéma XML du document XML échangé.

  • Java : sélectionnez cette option si un objet Java est envoyé dans la demande. Le transport JMS est utilisé pour les messages de type Objet Java. Pour plus d'informations, reportez-vous à "Sending and Receiving Java Objects in Messages" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Type de message de réponse
Sélectionnez le type de message de réponse :
  • Aucun : sélectionnez cette option s'il n'y a pas de message de réponse.
  • Binaire : sélectionnez cette option si le type de contenu du message est inconnu ou n'est pas important.
  • Texte : sélectionnez cette option si le message peut être limité à du texte.
  • MFL : sélectionnez cette option si le message est un document binaire conforme à une définition MFL. Entrez le nom de fichier MFL (qualifié par son chemin) ou cliquez sur Parcourir pour sélectionner un fichier.

    • Vous ne pouvez configurer qu'un seul fichier MFL.

    Remarque : pour prendre en charge plusieurs fichiers MFL, définissez le contenu sur Binaire ou Texte et utilisez l'action MFL du flux de messages pour le convertir en XML.

  • XML : sélectionnez cette option si le message est un document XML. Entrez le nom du fichier XML (qualifié par son chemin), ou cliquez sur Parcourir pour sélectionner un fichier.

    • Fournissez éventuellement des informations sur le type en déclarant (dans le champ Elément ou Type) le type de schéma XML du document XML échangé.

  • Java : sélectionnez cette option si un objet Java est reçu dans la réponse. Le transport JMS est utilisé pour les messages de type Objet Java. Pour plus d'informations, reportez-vous à "Sending and Receiving Java Objects in Messages" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
