Page Configuration de la sélection d'opération

La page Créer/Modifier un service proxy - Configuration de la sélection d'opération est l'une des pages destinées à la création et à la modification de services proxy, décrites dans les rubriques Création et configuration de services proxy et Modification des configurations des services proxy. Cette page apparaît uniquement si le service que vous créez contient des opérations.

Cette page permet d'imposer la conformité WS-I (pour les services SOAP 1.1 uniquement) et de choisir l'algorithme de sélection à utiliser pour déterminer l'opération appelée par ce service proxy. Cette option est disponible uniquement pour les services SOAP et XML définis à partir d'une ressource WSDL.

La spécification WSDL définit un algorithme par défaut pour déterminer l'opération appelée en fonction du type du message SOAP reçu. Cependant, dans certains cas (par exemple, problèmes de performances, problèmes de signature/cryptage ou si l'algorithme par défaut n'est pas applicable), vous devez sélectionner l'opération en fonction d'autres moyens.

Oracle Service Bus propose d'autres algorithmes. Ils suivent tous le même modèle et sont basés sur l'évaluation d'une expression pour obtenir une valeur qui est ensuite utilisée pour rechercher l'opération correspondante dans une table statique.

Oracle Service Bus est généralement très tolérant s'il manque des données dans un message entrant, ce qui fait que l'opération ne peut pas être déterminée, ou s'il a des données qui ne correspondent pas à une opération valide. Pour ces deux situations, le champ $operation est vide. Au lieu de rejeter tous les messages de ce type, Oracle Service Bus n'initialise pas la variable de l'opération dans le contexte mais continue malgré tout à traiter le message.

Cependant, des exigences de sécurité sont imposées si le service proxy est basé sur WSDL et si au moins l'une des conditions suivantes est vraie :

  • Le WSDL a une stratégie de sécurité WS et le proxy est un intermédiaire actif.
  • Le proxy dispose d'une authentification personnalisée au niveau du message (jeton personnalisé ou nom utilisateur/mot de passe).

Si ces conditions sont remplies, une vérification d'exécution est réalisée pour s'assurer que l'algorithme de sélection de l'opération renvoie un nom d'opération valide. Si la sélection de l'opération renvoie NULL ou une opération qui n'est pas dans le WSDL, le message est rejeté et une erreur survient.

Imposer la conformité WS-I
Pour les services SOAP 1.1 uniquement :
Cochez cette case si vous souhaitez indiquer si le service est conforme au profil de base défini par la Web Services Interoperability Organization.
Lorsqu'un service est conforme à la WS-I, des vérifications sont effectuées sur les messages envoyés vers et depuis ce service. Pour les proxies, des vérifications sont effectuées sur les messages de demande reçus par le proxy. Pour les services appelés (les services appelés par un proxy via une action d'appel de service ou un noeud de routage), des vérifications sont effectuées sur les messages de réponse reçus de ces services. C'est la propriété de conformité WS-I du service appelé et non le proxy qui détermine la réalisation de vérifications sur les messages reçus à partir du service appelé. Si vous indiquez des tests de conformité WS-I pour un service appelé, le flux de messages génère une erreur pour les erreurs de réponse.
Algorithme de sélection
Sélectionnez-en un parmi les suivants et effectuez les étapes supplémentaires nécessaires :
  • En-tête de transport : sélectionnez cet algorithme pour définir l'en-tête de transport contenant la valeur de recherche. Ensuite :
    • Dans le champ En-tête, entrez l'en-tête de transport qui extrait la valeur utilisée comme clé pour sélectionner l'opération appelée.
    • Dans le champ Mise en correspondance d'opération, indiquez la valeur pour chaque opération dans le champ Valeur. Cette valeur est utilisée comme clé de l'opération.
  • En-tête SOAPAction : sélectionnez cet algorithme pour indiquer que la mise en correspondance de l'opération doit être faite automatiquement à partir du WSDL associé à ce service proxy.
  • Adressage WS : sélectionnez cet algorithme pour indiquer que la valeur de recherche est contenue dans la balise d'adressage WS Action située dans les en-têtes SOAP du message SOAP. Sous le champ Mise en correspondance d'opération, indiquez ensuite la valeur pour chaque opération dans le champ Valeur. Cette valeur est utilisée comme clé de l'opération.
  • En-tête SOAP : sélectionnez cet algorithme pour définir l'expression XPath à évaluer dans les en-têtes SOAP. Cela vous permet d'obtenir la valeur de recherche. Ensuite :
    • Dans le champ Expression XPath, indiquez l'expression XPath qui extrait la valeur utilisée comme clé pour sélectionner l'opération appelée.
    • Dans le champ Mise en correspondance d'opération, indiquez la valeur pour chaque opération dans le champ Valeur. Cette valeur est utilisée comme clé de l'opération.
  • Type de corps SOAP : il s'agit de l'algorithme par défaut défini par la spécification WSDL pour calculer l'opération appelée en fonction du type de message SOAP reçu.

    • Si le service proxy est configuré pour un scénario pass-through de sécurité de service Web avec un corps crypté, vous ne pouvez pas sélectionner cet algorithme. Le même avertissement s'applique aux en-têtes SOAP pass-through cryptés.

    Si vous avez un WSDL qui dispose de deux opérations avec le même message d'entrée, ne sélectionnez pas cet algorithme pour des opérations, car l'opération ne peut pas être identifiée de manière unique en examinant le message d'entrée.

  • Type de données traitées : disponible uniquement pour les services XML basés sur un port ou une liaison WSDL.
