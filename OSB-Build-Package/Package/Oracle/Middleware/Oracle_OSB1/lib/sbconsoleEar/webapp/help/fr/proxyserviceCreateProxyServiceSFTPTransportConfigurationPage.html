Page SFTP - Configuration du transport

La page Créer/Modifier un service proxy - SFTP - Configuration du transport est l'une des pages destinées à la création et à la modification de services proxy, décrites dans les rubriques Création et configuration de services proxy et Modification des configurations des services proxy.

Pour plus d'informations, reportez-vous à HTTP and Poller Transports dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV947.

Cette page permet de configurer les paramètres de transport pour un service proxy utilisant le protocole de transport SFTP. Le tableau 19-13 explique comment l'utiliser.

Tableau 19-13 Page Créer/Modifier un service proxy - SFTP - Configuration du transport
Option
Procédure
Authentification utilisateur
Sélectionnez l'une des options suivantes :
  • Authentification du nom utilisateur/mot de passe : indique qu'un compte de service statique est associé à cette méthode d'authentification et que le client est authentifié à l'aide des informations d'identification et de connexion fournies.
  • Authentification basée sur l'hôte : indique qu'un nom utilisateur et un fournisseur de clés de service sont obligatoires pour utiliser cette méthode d'authentification. Tout utilisateur se connectant à partir d'un hôte connu est authentifié à l'aide de la clé privée de l'hôte.
  • Authentification de clé publique : indique qu'un nom utilisateur et un fournisseur de clés de service sont obligatoires pour utiliser cette méthode d'authentification. Chaque utilisateur a sa propre clé privée.
Compte de service
Entrez le compte de service de l'utilisateur ou cliquez sur Parcourir pour sélectionner des comptes de service à partir d'un navigateur Web.
Fournisseur de clés de service
Cette option est disponible uniquement lorsque l'une des options Basé sur l'hôte ou Authentification de clé publique est sélectionnée.
Entrez un fournisseur de clés de service dans le champ Fournisseur de clés de service. Cliquez sur Parcourir pour sélectionner des fournisseurs de clés de service à partir d'un navigateur Web. Ce champ est obligatoire.
Nom utilisateur
Cette option est disponible uniquement lorsque l'une des options Basé sur l'hôte ou Authentification de clé publique est sélectionnée.
Entrez le nom utilisateur.
Transmission par référence
Cochez cette case pour stocker le fichier dans le répertoire d'archivage et le transmettre comme référence dans les en-têtes.
Transmission en continu distante
Cochez cette case pour transmettre en continu les fichiers SFTP directement à partir du serveur distant lors du traitement. Lorsque vous sélectionnez cette option, le répertoire d'archivage est le répertoire distant sur l'ordinateur serveur SFTP distant. Par conséquent, vous devez indiquer le répertoire d'archivage relatif au répertoire utilisateur SFTP.
Masque de fichier
Entrez l'expression régulière définissant les fichiers à sélectionner. La valeur par défaut est *.*.
Serveur géré
Ce champ n'est disponible que pour les domaines inclus dans un cluster.
Sélectionnez le serveur géré qui doit agir en tant que serveur d'interrogation. Tous les serveurs gérés peuvent traiter le message, mais un seul peut l'interroger.
Intervalle d'appel
Entrez l'intervalle en secondes auquel le fichier est interrogé à partir de l'emplacement indiqué. La valeur par défaut est 60.
Limite de lecture
Indiquez le nombre maximal de messages à lire par balayage d'interrogation. Entrez 0 pour indiquer un nombre illimité. La valeur par défaut est 10.
Action après lecture
Indiquez ce qu'il advient d'un message une fois que celui-ci a été lu.
  • Archivage : le message est archivé.
  • Supprimer : le message est supprimé.
Répertoire d'archivage
Indiquez le chemin de l'emplacement d'archivage à utiliser lorsque l'option Action après lecture est définie sur Archivage. Ce champ est obligatoire si vous avez sélectionné l'option Transmission par référence.
Remarque : les répertoires d'archivage, de téléchargement et des erreurs représentent des chemins absolus ; ils sont créés automatiquement. Si vous indiquez un chemin relatif, les fichiers sont créés relativement au processus Java qui démarre le serveur WebLogic.
Répertoire de téléchargement
Indiquez le répertoire de l'ordinateur local dans lequel sont téléchargés les fichiers lors de leur transfert.
Remarque : les répertoires d'archivage, de téléchargement et des erreurs représentent des chemins absolus ; ils sont créés automatiquement. Si vous indiquez un chemin relatif, les fichiers sont créés relativement au processus Java qui démarre le serveur WebLogic.
Répertoire d'erreurs
Indiquez l'emplacement dans lequel les messages sont envoyés en cas de problème.
Remarque : les répertoires d'archivage, de téléchargement et des erreurs représentent des chemins absolus ; ils sont créés automatiquement. Si vous indiquez un chemin relatif, les fichiers sont créés relativement au processus Java qui démarre le serveur WebLogic.
Encodage de la demande
Acceptez l'encodage de jeu de caractères UTF-8 par défaut pour les demandes de transport SFTP.
Mécanisme de cryptage préféré
Sélectionnez le mécanisme de cryptage à utiliser pour communiquer avec le serveur.
Si vous utilisez la valeur par défaut (Utiliser la valeur par défaut d'exécution), la liste des mécanismes de cryptage pris en charge est envoyée au serveur et chacun d'entre eux est testé jusqu'à ce qu'une correspondance soit trouvée.
Algorithme d'intégrité des données préféré
Sélectionnez l'algorithme de hachage en masse pour les vérifications de l'intégrité des données.
Si vous utilisez la valeur par défaut, Utiliser la valeur par défaut d'exécution, Oracle Service Bus envoie l'algorithme préféré de hmac-sha1 au serveur. S'il n'est pas pris en charge par le serveur, la liste des algorithmes est envoyée au serveur et chacun d'entre eux est essayé jusqu'à ce qu'une correspondance soit trouvée.
Algorithme de clé publique préféré
Sélectionnez l'algorithme de clé asymétrique pour la cryptographie par clé publique.
Si vous utilisez la valeur par défaut, Utiliser la valeur par défaut d'exécution, Oracle Service Bus envoie l'algorithme préféré de ssh-dss au serveur. S'il n'est pas pris en charge par le serveur, la liste des algorithmes est envoyée au serveur et chacun d'entre eux est essayé jusqu'à ce qu'une correspondance soit trouvée.
Algorithme d'échange de clé préféré
Sélectionnez le protocole d'échange de clé par défaut pour négocier la clé de session permettant de crypter le message.
Si vous utilisez la valeur par défaut, Utiliser la valeur par défaut d'exécution, Oracle Service Bus envoie l'algorithme préféré de diffie-hellman-group1-sha1 au serveur. S'il n'est pas pris en charge par le serveur, la liste des algorithmes est envoyée et chacun d'entre eux est essayé jusqu'à ce qu'une correspondance soit trouvée.
Algorithme de compression préféré
Indiquez s'il convient de compresser les données en cours d'exécution à l'aide de zlib. Sélectionnez zlib pour compresser les données ; sinon, sélectionnez Aucun. La valeur par défaut est Aucun.
Balayer les sous-répertoires
Cochez cette case pour balayer tous les répertoires de façon récursive.
Trier par ordre d'arrivée
Cochez cette case pour indiquer que les événements sont livrés dans leur ordre d'arrivée.
Temporisation
Entrez la temporisation du socket, en secondes, avant la suppression de la connexion. Si vous entrez 0, il n'y a pas de temporisation. La valeur par défaut est 60.
Nombre de tentatives
Indiquez le nombre de tentatives pour les échecs de connexion SFTP.

Après avoir fini
