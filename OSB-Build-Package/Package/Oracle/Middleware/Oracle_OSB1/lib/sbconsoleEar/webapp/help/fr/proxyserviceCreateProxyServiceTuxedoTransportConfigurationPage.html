Page Tuxedo - Configuration du transport

La page Créer/Modifier un service proxy - Tuxedo - Configuration du transport est l'une des pages destinées à la création et à la modification de services proxy, décrites dans les rubriques Création et configuration de services proxy et Modification des configurations des services proxy.

Pour plus d'informations, reportez-vous à Tuxedo Transport dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Cette page permet de configurer les paramètres de transport pour un service proxy utilisant le protocole de transport Tuxedo. Le tableau 19-14 explique comment l'utiliser.

Tableau 19-14 Page Créer/Modifier un service proxy - Tuxedo - Configuration du transport
Option
Procédure
Classes de table de champs
Entrez le nom des classes décrivant le tampon FML/FML32 reçu. Ces classes sont utilisées pour que les sous-programmes de conversion FML/FML32-to-XML mettent en correspondance les noms de champ et d'élément. Il s'agit d'une liste des noms qualifiés complets des classes, séparés par des espaces.
Classes de vues
Entrez le nom des classes décrivant le tampon VIEW/VIEW32 reçu ou envoyé. Ces classes sont utilisées pour que les sous-programmes de conversion VIEW-to-XML ou VIEW32-to-XML mettent en correspondance les noms de champ et d'élément. Il s'agit d'une liste des noms qualifiés complets des classes, séparés par des espaces.
Les types de tampons Tuxedo X_C_TYPE et X_COMMON sont gérés de la même manière que les tampons VIEW/VIEW32.
Si une demande entrante contient un tampon VIEW, la classe VIEW correspondante devra être indiquée dans la variable d'environnement CLASSPATH d'Oracle Service Bus.
Fichier JAR de classes
Sélectionnez une ressource JAR contenant un fichier JAR appartenant aux classes FML/FML32 ou VIEW/VIEW32 nécessaires pour cette opération d'adresse.
Point d'accès local
Sélectionnez un point d'accès local dans la liste déroulante associée à l'export. Cette liste contient les points d'accès locaux configurés dans WTC. Un service proxy ne peut pas être créé sans être associé à un point d'accès local.
S'il n'existe aucun point d'accès local ou pour en créer un, sélectionnez Nouveau. Entrez le nom du point d'accès local et l'adresse réseau locale correspondants dans les champs adjacents. Après validation de l'adresse, le point d'accès est ajouté à la configuration WTC pour chaque serveur WTC. S'il n'en existe pas, un serveur WTC est créé.
Vous pouvez entrer un nom de point d'accès existant après avoir sélectionné l'option Nouveau. Les informations existantes sont alors mises à jour avec les nouveaux paramètres. Vous pouvez changer uniquement le nom d'hôte et le numéro du port.
Point d'accès distant
Ce champ apparaît uniquement lorsque vous sélectionnez Nouveau dans le champ Point d'accès local.
Dans la liste déroulante, sélectionnez le point d'accès distant à associer au point d'accès local que vous venez de créer. S'il n'en existe pas ou pour en créer un, sélectionnez Nouveau. Entrez le nom du point d'accès et l'adresse réseau correspondants dans les champs adjacents.
Vous pouvez entrer un nom de point d'accès existant après avoir sélectionné l'option Nouveau. Les informations existantes sont alors mises à jour avec les nouveaux paramètres. Vous pouvez changer uniquement le nom d'hôte et le numéro du port.
Le point d'accès distant sera également l'identité utilisateur d'authentification pour la connexion WTC pour les demandes entrantes. Vous pouvez éventuellement créer un utilisateur avec le même ID de point d'accès dans le domaine de sécurité par défaut pour autoriser les appels entrants. Pour ce faire, sélectionnez Oui dans la liste déroulante Créer un utilisateur ?. Le mot de passe sera généré de manière aléatoire à l'aide d'une variable temporaire pour éviter les problèmes de sécurité.
Type de tampon de réponse
Cette option est disponible uniquement lorsque le champ Réponse requise ? est sélectionné.
Sélectionnez le type de tampon que le client Tuxedo distant recevra.
Sous-type de tampon de réponse
Cette option est disponible uniquement lorsque l'option Réponse requise ? est sélectionnée et que la valeur Type de tampon de réponse est définie sur VIEW ou VIEW32.
Entrez le sous-type de tampon auquel associer le tampon de réponse.
Réponse requise ?
Cochez cette case si le service doit envoyer une réponse.
Dans le statut par défaut, cette option est sélectionnée.
Cette option est effacée et non disponible si le type de service est Service de messagerie et si le type du message de réponse est Aucun.
Encodage de la demande
Indiquez un encodage de jeu de caractères pour les demandes dans des transports Tuxedo.
Encodage de la réponse
Indiquez un encodage de jeu de caractères pour les réponses dans des transports Tuxedo.
Style de transformation
Sélectionnez l'une des options suivantes :
  • Aucun : (par défaut) l'ordre des champs peut ne pas être respecté.
  • Ordonné : les champs sont présentés avec toutes leurs occurrences dans l'ordre correct.
  • Groupé et ordonné : si les champs sont structurés logiquement en tant qu'enregistrements, ils sont classés par occurrence et regroupés par enregistrement.
