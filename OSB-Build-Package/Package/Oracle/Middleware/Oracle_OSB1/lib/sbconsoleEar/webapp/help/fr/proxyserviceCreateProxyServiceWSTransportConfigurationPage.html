Page WS - Configuration du transport

La page Créer/Modifier un service proxy - WS - Configuration du transport est l'une des pages destinées à la création et à la modification de services proxy, décrites dans les rubriques Création et configuration de services proxy et Modification des configurations des services proxy.

Pour plus d'informations, reportez-vous à WS Transport dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083.

Cette page permet de configurer les paramètres de transport pour un service proxy utilisant le protocole de transport WS. Le tableau 19-15 explique comment l'utiliser.

Tableau 19-15 Page Créer/Modifier un service proxy - WS - Configuration du transport
Option
Procédure
Stratégie de répartition
Sélectionnez une stratégie de répartition pour cette adresse ou utilisez la stratégie de répartition par défaut.
La stratégie de répartition fait référence au gestionnaire de travaux WLS utilisé pour l'adresse de service. Pour plus d'informations sur les gestionnaires de travaux, reportez-vous à "Using Work Managers to Optimize Scheduled Work" dans le manuel Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Nombre de tentatives
Nombre de tentatives de redistribution d'un message au pipeline.
Si une exception non gérée survient dans le pipeline de demande d'un service proxy, le message de transport WS entrant sera redistribué au pipeline jusqu'à ce que le nombre de tentatives indiqué soit atteint. Cette valeur est importante pour traiter de manière fiable les messages de transport WS.
Délai entre deux tentatives
Durée, en secondes, pendant laquelle le système fait une pause avant de faire une autre tentative d'envoi du message au pipeline après une erreur.
