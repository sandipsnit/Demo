Création de structures de variable dans les éditeurs XQuery

La palette Structures de variable des éditeurs XQuery et XPath affiche des représentations graphiques du contenu des variables. Elle inclut par défaut les variables de contexte de message intégrées attachments, body, header, outbound et inbound.

Chaque entrée de correspondance de structure de variable dispose d'un libellé et met en correspondance une variable ou un chemin de variable avec au moins une structure. La portée de ces mises en correspondance est une phase ou un noeud de routage.

Vous pouvez également déclarer vos propres structures de variable, en fonction :

  • Des types XML, notamment :
    • Eléments de schéma
    • Eléments WSDL
    • Types de schéma
    • Types WSDL
    • Fichiers MFL
  • Interfaces de service
  • Types simples (chaîne ou tout XML)

Vous pouvez utiliser cette fonctionnalité directement pour toutes les variables définies par l'utilisateur, ainsi que $inbound, $outbound et $fault. Cependant, vous ne pouvez pas l'utiliser directement pour accéder à des pièces jointes XML dans $attachments, à des en-têtes dans $header ou à des documents et paramètres RPC dans $body, sauf si vous l'utilisez pour accéder directement aux documents et paramètres dans $body pour les messages de demande reçus par un service proxy WSDL.

Avant de commencer

Les instructions suivantes supposent que soyez en train de créer ou modifier une expression via l'éditeur d'expressions XQuery, l'éditeur de conditions XQuery ou l'éditeur d'expressions XPath, comme indiqué dans Création et modification d'expressions XPath et XQuery imbriquées.

Pour créer une structure de variable :

  1. Sélectionnez Structures de variable.
  2. Dans la palette Structures de variable, cliquez sur Ajouter une nouvelle structure.
  3. Poursuivez avec l'une des tâches énumérées dans le tableau 22-3 ci-dessous.

    4. Tableau 22-3 Création d'une structure de variable
    Opération
    Exécutez les étapes suivantes...
    Créer une structure de variable qui met en correspondance une variable et un type de schéma XML
    1. Sélectionnez Type XML en haut de la palette Structures de variable, s'il n'est pas déjà sélectionné.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Chemin de la structure, entrez le chemin de la structure de variable lors de l'exécution. Le chemin doit commencer par $.
    4. Dans le champ Type, sélectionnez le bouton d'option approprié, puis sélectionnez Type de schéma.
    5. Cliquez sur Parcourir. Le navigateur de schéma XML apparaît. Sélectionnez un schéma XML dans la liste, sélectionnez un type de schéma XML dans le panneau Définitions, puis cliquez sur Soumettre.
    6. Cliquez sur Ajouter pour créer la structure de variable.
    Créer une structure de variable qui met en correspondance une variable et un type WSDL
    1. Sélectionnez Type XML en haut de la palette Structures de variable, s'il n'est pas déjà sélectionné.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Chemin de la structure, entrez le chemin de la structure de variable lors de l'exécution.
    4. Dans le champ Type, sélectionnez le bouton d'option approprié, puis sélectionnez Type WSDL.
    5. Cliquez sur Parcourir. Le navigateur WSDL apparaît. Sélectionnez un WSDL dans la liste des WSDL, sélectionnez un type de WSDL dans le panneau Définitions, puis cliquez sur Soumettre.
    6. Cliquez sur Ajouter pour créer la structure de variable.
    Créer une structure de variable qui met en correspondance une variable et un élément de schéma XML
    1. En haut de la palette Structures de variable, sélectionnez Type XML, si cela n'est pas déjà sélectionné.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Chemin de la structure, entrez le chemin de la structure de variable lors de l'exécution.
    4. Dans le champ Type, sélectionnez le bouton d'option approprié, puis sélectionnez Elément de schéma.
    5. Cliquez sur Parcourir. Le navigateur de schéma XML apparaît. Sélectionnez un schéma XML dans la liste, sélectionnez un type de schéma XML dans le panneau Définitions, puis cliquez sur Soumettre.
    6. Cliquez sur Ajouter pour créer la structure de variable.
    Créer une structure de variable qui met en correspondance une variable et un élément WSDL
    1. En haut de la palette Structures de variable, sélectionnez Type XML, si cela n'est pas déjà sélectionné.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Chemin de la structure, entrez le chemin de la structure de variable lors de l'exécution.
    4. Dans le champ Type, sélectionnez le bouton d'option approprié, puis sélectionnez Elément WSDL.
    5. Cliquez sur Parcourir. Le navigateur WSDL apparaît. Sélectionnez un WSDL dans la liste de WSDL, sélectionnez un élément WSDL dans le panneau Définitions, puis cliquez sur Soumettre.
    6. Cliquez sur Ajouter pour créer la structure de variable.
    Créer une structure de variable qui met en correspondance une variable et un élément enfant
    1. En haut de la palette Structures de variable, sélectionnez Type XML, si cela n'est pas déjà sélectionné.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Chemin de la structure, entrez le chemin de la structure de variable lors de l'exécution.
    4. Dans le champ Type, sélectionnez le type de variable que vous souhaitez créer :

      5. Pour créer un élément de schéma XML ou une variable d'élément WSDL, sélectionnez le bouton d'option associé à cette option, puis sélectionnez Elément de schéma ou Elément WSDL.

      Pour créer une variable MFL, sélectionnez le bouton d'option associé à cette option, puis sélectionnez MFL.

    5. Pour le schéma XML, WSDL ou MFL, cliquez sur Parcourir pour sélectionner un objet dans la liste affichée par le navigateur, puis cliquez sur Soumettre. Par exemple, sélectionnez un MFL dans la liste de MFL, puis cliquez sur Soumettre.
    6. Cochez la case Définir comme enfant pour définir l'élément en tant qu'enfant de la structure créée.
    7. Cliquez sur Ajouter pour créer la structure de variable.
    Créer une structure de variable utilisant une ressource MFL
    1. En haut de la palette Structures de variable, sélectionnez Type XML, si cela n'est pas déjà sélectionné.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Chemin de la structure, entrez le chemin de la structure de variable lors de l'exécution.
    4. Dans le champ Type, sélectionnez le bouton d'option approprié, puis cliquez sur Parcourir. Le navigateur MFL apparaît.
    5. Sélectionnez un MFL dans la liste de MFL, puis cliquez sur Soumettre.
    6. Cliquez sur Ajouter pour créer la structure de variable.
    Créer une structure de variable d'interface de service
    1. En haut de la palette Structures de variable, sélectionnez Interface de service.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Chemin de la structure, l'option par défaut est déjà définie sur $body. Vous ne pouvez pas modifier ce champ.
    4. Dans le champ Service WSDL, sélectionnez l'icône Navigateur Web de services, sélectionnez un service dans la liste affichée par le navigateur Web de services, puis cliquez sur Soumettre.

      5. Le service que vous avez sélectionné apparaît dans le champ Service WSDL.

    5. Dans le champ Opération, sélectionnez une opération ou sélectionnez Aucune pour ne pas inclure d'opération.
    6. Cliquez sur Ajouter pour créer la variable.
    Créer une structure de variable simple
    1. En haut de la palette Structures de variable, sélectionnez Type simple.
    2. Dans le champ Libellé de la structure, entrez le nom d'affichage de la variable que vous souhaitez créer. Ce nom d'affichage vous permet de donner un nom pertinent à la structure pour pouvoir la reconnaître lors de la conception, mais il n'a pas d'impact lors de l'exécution.
    3. Dans le champ Nom de structure, entrez le nom de la structure de variable que vous souhaitez créer.
    4. Dans le champ Type, sélectionnez Chaîne ou Tout XML.
    5. Cliquez sur Ajouter pour créer la variable.

Après avoir fini

Poursuivez comme décrit dans Création et modification d'expressions XPath et XQuery imbriquées.
