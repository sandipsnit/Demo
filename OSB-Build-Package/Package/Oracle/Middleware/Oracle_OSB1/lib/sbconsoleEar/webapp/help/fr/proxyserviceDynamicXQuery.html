Liaison d'expressions XQuery dynamiques à des expressions XQuery imbriquées

La page Editeur d'expressions XQuery vous permet d'indiquer une expression XQuery dynamique qui renvoie le nom d'une ressource préenregistrée lors de l'exécution. Pour en savoir plus sur cet éditeur, reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.

Pour définir une expression XQuery dynamique :

  1. Sélectionnez l'option XQuery dynamique.
  2. Dans la zone Entrez une expression pour la ressource XQuery, entrez l'expression XQuery qui renverra le nom d'une ressource XQuery préenregistrée lors de l'exécution.

    3. L'exemple suivant montre la syntaxe d'une ressource XQuery (représentant le nom complet de la ressource) :

    Project/folder1/folder2/XQueryResourceName

  3. Dans la zone Lier les variables à l'aide d'un modèle XQuery ou de variables personnalisées, définissez les paramètres d'entrée pour la transformation.
    • Cliquez sur Parcourir dans le champ Sélectionner un modèle XQuery pour sélectionner la ressource inscrite existante qui servira de modèle à la forme de l'interrogation (le nombre et le nom des variables). Après avoir sélectionné un modèle, les variables apparaissent dans la zone Variables attachées (bind variable). Le modèle n'est pas enregistré avec la configuration. Il vous aide à démarrer et à définir les variables de la requête.
    • Saisissez un nom de variable dans le champ Ajouter une variable personnalisée et cliquez sur Ajouter. Pour chaque variable répertoriée sous Nom de la variable, entrez une expression XQuery avec laquelle la mettre en correspondance. Vous devez définir une correspondance pour chaque paramètre. Par exemple, si une transformation XQuery a deux paramètres d'entrée, un et deux, le champ Nom de la variable a deux libellés : un et deux. Une zone de texte, dans laquelle l'expression XQuery est entrée, est associée à chaque libellé.

      • Les expressions XQuery suivantes sont des exemples d'entrées valides dans les champs de variable :

      $body/*[1]

      $body/po:PurchaseOrder

  4. Une fois que vous avez fini, poursuivez avec l'une des tâches suivantes :
    • Cliquez sur Valider. Un message apparaît si la validation de l'expression a réussi.
    • Cliquez sur Tester. Reportez-vous à Exécution du test XQuery.
    • Enregistrez ou annulez vos modifications.
