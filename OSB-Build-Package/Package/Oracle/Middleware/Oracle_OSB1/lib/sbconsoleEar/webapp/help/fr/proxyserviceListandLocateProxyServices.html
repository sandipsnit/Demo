Localisation des services proxy

  1. Effectuez l'une des opérations suivantes :
    • Sélectionnez Explorateur de projets pour afficher la page Vue des projets ou la page Vue du projet/Vue de dossier. Ensuite, parcourez les projets et les dossiers pour trouver le service proxy.
    • Sélectionnez Navigateur Web de ressources > Services proxy. La page Récapitulatif des services proxy affiche les informations présentées dans le tableau 19-18. Pour une description plus détaillée des propriétés, reportez-vous à Modification des configurations des services proxy.
  2. Pour restreindre le nombre d'éléments dans la liste, vous pouvez filtrer par nom, par chemin ou les deux. Dans les champs Nom et Chemin, sous Rechercher, entrez le nom et/ou le chemin des cibles, puis cliquez sur le bouton Rechercher.

    3. Le chemin est constitué du·nom de projet et du·nom de dossier dans lesquels réside le service proxy.

    Les caractères génériques * et ? sont autorisés. La recherche distingue les majuscules des minuscules.

    Cliquez sur Tout visualiser pour afficher tous les services proxy du domaine. Cela efface les paramètres de la précédente recherche.

    Tableau 19-18 Page Récapitulatif des services proxy 
    Propriété
    Description
    Nom
    Nom unique du service proxy. Cliquez sur ce nom pour afficher la page Visualiser un service proxy : Détails de configuration.
    Reportez-vous à Modification des configurations des services proxy.
    Chemin
    Le chemin est constitué du·nom de projet et du·nom de dossier dans lesquels réside le service, par exemple, UDDI/Proxies/OSB_services.
    Cliquez sur le chemin d'un service proxy pour afficher la page Vue du projet/Vue de dossier qui le contient.
    Actions
    Exécutez l'une des actions suivantes :
    • Cliquez sur l'icône Modifier le flux de messages pour modifier le flux de messages du service proxy. Reportez-vous à Création, modification et visualisation de flux de messages.
    • Cliquez sur l'icône Lancer la console de test pour appeler la console de test, que vous utilisez pour valider et tester la conception de vos services et de vos transformations. Pour les services fonctionnels, vous pouvez utiliser la console de test uniquement lors de l'exécution, c'est-à-dire lorsque la session est activée. Pour les transformations, vous pouvez utiliser la console de test, que vous soyez dans une session ou non. Reportez-vous aux rubriques relatives au test des services et au test des transformations.
    • L'icône Exporter le WSDL apparaît pour les services fonctionnels de type WSDL. Cliquez dessus pour exporter une ressource WSDL que vous pouvez ensuite visualiser ou modifier dans un outil externe tel qu'un IDE.

      • Cette fonction est différente de la fonction Exporter les ressources du module Administration du système, que vous utilisez pour déplacer et préparer les ressources entre deux domaines. Reportez-vous à Export d'un WSDL.

    Options
    Cliquez sur l'icône Supprimer pour supprimer le service. Si cette ressource est référencée par d'autres, une·icône Avertissement relatif à la suppression apparaît. Vous pouvez supprimer la ressource en confirmant l'opération. Toutefois, des conflits peuvent survenir, les références à la ressource supprimée n'étant pas·résolues. Pour plus d'informations, reportez-vous à Suppression de services proxy.
