Liaison de ressources XSLT externes et d'expressions XQuery imbriquées

La page Sélectionner la ressource XQuery à exécuter permet de sélectionner une ressource XSLT à exécuter. Pour en savoir plus sur cet éditeur, reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.

Pour sélectionner une ressource XSLT à exécuter, procédez comme suit :

  1. Sélectionnez l'option Ressources XSLT.
  2. Sous le champ Sélectionner la ressource XSLT à exécuter, sélectionnez l'icône du navigateur XSLT.
  3. Dans le navigateur XSLT, sélectionnez le bouton d'option associé au XSLT que vous souhaitez exécuter, puis cliquez sur Soumettre.
  4. Sous le champ Variables attachées (bind variable), un libellé et une zone de texte correspondante apparaissent pour chaque paramètre d'entrée de la transformation. Chaque libellé représente le nom d'un paramètre et chaque zone de texte sert à définir une expression XQuery à mettre en correspondance avec le paramètre. Vous devez définir une mise en correspondance pour chaque paramètre. Par exemple, si une transformation XSL a deux paramètres d'entrée nommés un et deux, le champ Correspondance de variable a deux libellés : un et deux avec une zone de texte associée à chacun, dans laquelle une expression XQuery est entrée. Outre la mise en correspondance de toute variable d'entrée, vous devez indiquer une expression XQuery pour le document d'entrée de la transformation. La mise en correspondance est indiquée dans la zone de texte portant·le libellé Document d'entrée.

    5. Les expressions XQuery suivantes sont des exemples d'entrées valides pour ce champ :

    $body/*[1]

    $body/po:PurchaseOrder

    Remarque : Le nom de variable suivant n'est pas une entrée valide pour ce champ, ce qui entraîne une exception :
    body.
  5. Poursuivez en exécutant l'une des tâches suivantes :
    • Création d'espaces de noms à utiliser dans des expressions imbriquées.
    • Création de structures de variable dans les éditeurs XQuery
    • Construction d'expressions dans les champs de texte de l'espace de travail de l'éditeur.
    • Liaison de ressources XQuery externes et d'expressions XQuery imbriquées
    • Cliquez sur Valider. Un message apparaît si la validation de l'expression a réussi.
    • Cliquez sur Tester. Reportez-vous à Exécution du test XQuery.
