Page Stratégie de service proxy

La page Visualiser un service proxy : Stratégies fait partie d'une série de pages destinées à la modification de services proxy, comme décrit dans Création et configuration de services proxy et Modification des configurations des services proxy.

Utilisez cette page pour configurer les paramètres de stratégie des services proxy de type Service Web WSDL, Service de messagerie, N'importe quel service SOAP ou N'importe quel service XML. Le tableau 19-20 décrit comment utiliser cette page. Pour pouvoir utiliser les stratégies OWSM avec un service proxy Web WSDL, un service proxy de messagerie ou n'importe quel service proxy XML, vous devez définir le protocole sur HTTP.

Pour les services de type WSDL, les stratégies WLS 9.2 liées au service sont exposées (en mode imbriqué) dans le WSDL en vigueur. Les stratégies abstraites sont pré-traitées avant d'être imbriquées. Les stratégies OWSM ne sont pas imbriquées dans le WSDL en vigueur mais liées par référence.

Configuration des stratégies de service
Voici les options du champ Configuration des stratégies de service :
  • A partir de la banque de stratégies OWSM : les stratégies sont définies dans la banque de stratégies Oracle Web Services Manager centrale gérée par Oracle Enterprise Manager. Ces stratégies prennent en charge WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken Profile et d'autres normes du secteur.

    Si nécessaire, utilisez un filtre pour localiser les stratégies de votre choix. Pour les services proxy, seules les stratégies de service apparaissent.

    • Pour plus d'informations, reportez-vous à Securing Oracle Service Bus with Oracle Web Services Manager dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

  • A partir du langage WSDL : sélectionnez cette option si la stratégie de service est associée au langage WSDL sur lequel le service est basé. Ces stratégies prennent en charge WS-Security 1.0, SAML 1.1 et d'autres normes du secteur.

    • Avec cette option, vous pouvez visualiser (lecture seule) les stratégies de demande et de réponse de la ressource WSDL.

  • A partir d'une stratégie prédéfinie ou d'une ressource de stratégie WS : cette option permet d'ajouter, à partir de la console d'administration d'Oracle Service Bus, des stratégies de niveau service, des stratégies de niveau opération (auquel cas la stratégie s'applique aux messages de demande et de réponse), des stratégies de demande et des stratégies de réponse.

    Les stratégies sont prédéfinies dans Oracle WebLogic Server ou définies par l'utilisateur dans Oracle Service Bus avec une ressource de stratégie WS. Ces stratégies prennent en charge WS-Security 1.0, SAML 1.1 et d'autres normes du secteur.

    • Avec cette option, vous pouvez ajouter des stratégies aux niveaux service, opération, demande et réponse.

Remarque : Les modèles de liaison de stratégie sont mutuellement exclusifs. Vous devez utiliser un seul type de stratégie dans un service. Lorsque vous liez des stratégies directement au service, toutes les stratégies de type WSDL sont ignorées.

Après avoir fini

Cliquez sur Mettre à jour pour enregistrer cette configuration ou sur Réinitialiser pour annuler vos modifications.
