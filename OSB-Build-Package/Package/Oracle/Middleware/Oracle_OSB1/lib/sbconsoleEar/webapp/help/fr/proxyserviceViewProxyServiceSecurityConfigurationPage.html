Page Sécurité de service proxy

La page Visualiser un service proxy : Sécurité fait partie d'une série de pages destinées à la modification de services proxy, comme décrit dans Création et configuration de services proxy et Modification des configurations des services proxy.

Cette page permet de configurer les paramètres de sécurité d'un service proxy.

Les champs disponibles sur cette page dépendent des stratégies utilisées dans le service proxy. Par exemple, lorsque le service proxy utilise des stratégies OWSM (recommandé), l'option Remplacements de stratégie apparaît sur la page Sécurité. Pour plus d'informations, reportez-vous à "Securing Oracle Service Bus with Oracle Web Services Manager" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus sur http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Pour les stratégies WLS 9.2, la configuration du nom utilisateur/mot de passe personnalisé et du jeton personnalisé est identique. Dans les deux cas, vous indiquez des expressions XPath qui permettent à Oracle Service Bus de localiser les informations nécessaires. La racine de ces expressions XPath est la suivante :

  • Utilisez soap-env:Envelope/soap-env:Header si la liaison de service est n'importe quel SOAP ou WSDL-SOAP.
  • Utilisez soap-env:Body si la liaison de service n'est pas de type SOAP.

Le format de toutes les expressions XPath doit être valide. Les expressions XPath doivent utiliser la syntaxe XPath "declare namespace" pour déclarer l'utilisation d'espaces de noms, comme indiqué ci-dessous :

declare namespace
ns='http://webservices.example.com/MyExampleService';)

Configuration de la sécurité des services Web
Traiter l'en-tête Sécurité WS
Lorsqu'une demande client inclut un en-tête de sécurité WS, vous pouvez décider de traiter ou non cet en-tête.
  • Sélectionnez Oui : dans un scénario intermédiaire actif, le client applique la sécurité WS aux messages de demande et/ou de réponse. Le service proxy traite l'en-tête de sécurité et applique la stratégie de sécurité WS. Cette option est automatiquement définie sur Oui lorsqu'un service proxy est basé sur un WSDL possédant une stratégie de sécurité ou lorsque vous attachez une stratégie au service sur la page Configuration des stratégies de service.
  • Sélectionnez Non : dans un scénario de pass-through de sécurité WS, le client applique la sécurité WS aux messages de demande et/ou de réponse. Le service proxy ne traite pas l'en-tête de sécurité. A l'inverse, il transmet le message de demande sécurisé intact au service fonctionnel.
Contrôler l'accès
Contrôle d'accès du transport
Pour tous les services proxy, vous pouvez créer une stratégie au niveau du transport, qui exécute un contrôle de sécurité lorsqu'un client tente d'établir une connexion avec le service proxy. Seules les demandes d'utilisateurs listés dans une stratégie de niveau transport pourront être traitées.
Contrôle d'accès des messages
Une stratégie de contrôle d'accès au·niveau du message exécute un contrôle de sécurité lorsqu'un client tente d'appeler un service proxy doté d'une sécurité au·niveau du message. Vous pouvez créer une stratégie de contrôle d'accès au·niveau du message dans les cas suivants :
  • Pour les services proxy qui sont des intermédiaires de sécurité de service Web ; c'est-à-dire que vous traitez l'en-tête de sécurité WS.
  • Pour les services proxy pour lesquels vous définissez une authentification personnalisée sur cette page.
Seuls les utilisateurs listés dans la stratégie de niveau message sont autorisés à appeler l'opération.

Type d'authentification
Sélectionnez l'une des options suivantes :
  • Sélectionnez Aucune si le service proxy n'utilise pas l'authentification personnalisée.
  • Nom et mot de passe utilisateur personnalisés

    • Lorsque vous sélectionnez cette option, vous devez également choisir·un type de jeton et indiquer son chemin dans le champ prévu à cet effet.

Expression XPath de nom utilisateur
Cette option est disponible uniquement lorsque l'option Paramètres d'authentification personnalisés est définie sur Nom et mot de passe utilisateur personnalisés. Si disponible, cette option est obligatoire.
Entrez le nom utilisateur en tant qu'expression XPath.
L'expression XPath est évaluée en fonction des en-têtes de message ou des données traitées, selon le cas, ce qui permet à Oracle Service Bus d'obtenir le nom utilisateur pour l'authentification personnalisée.
Expression XPath de mot de passe utilisateur
Cette option est disponible uniquement lorsque l'option Paramètres d'authentification personnalisés est définie sur Nom et mot de passe utilisateur personnalisés. Si disponible, cette option est obligatoire.
Entrez le mot de passe en tant qu'expression XPath.
L'expression XPath est évaluée en fonction des en-têtes de message ou des données traitées, selon le cas, ce qui permet à Oracle Service Bus d'obtenir les valeurs de mot de passe pour l'authentification personnalisée.
Type de jeton
Cette option est disponible uniquement lorsque l'option Paramètres d'authentification personnalisés est définie sur Jeton personnalisé. Si disponible, cette option est obligatoire.
Sélectionnez le type de jeton dans la liste déroulante. Seuls les types de jeton actif configurés pour un fournisseur d'assertion d'identité WebLogic Server sont disponibles. Reportez-vous à "Configuring Identity Assertion Providers for Custom Tokens" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660.
Expression XPath de jeton
Cette option est disponible uniquement lorsque l'option Paramètres d'authentification personnalisés est définie sur Jeton personnalisé. Si disponible, cette option est obligatoire.
Entrez une expression XPath pour indiquer un chemin vers le jeton personnalisé. Oracle Service Bus évalue l'expression XPath de jeton en fonction des en-têtes de message ou des données traitées, selon le cas, pour obtenir le jeton d'authentification personnalisée.
Propriétés de contexte
Vous pouvez éventuellement indiquer des propriétés pour transmettre des informations de contexte supplémentaires au fournisseur d'authentification (Nom et mot de passe utilisateur personnalisés) ou au fournisseur d'assertion d'identité (Jeton personnalisé).
Les propriétés de contexte permettent (grâce à l'interface ContextHandler) de transmettre des informations supplémentaires à la structure de sécurité WebLogic, de sorte qu'un fournisseur de sécurité puisse obtenir des informations sur le contexte. Reportez-vous à "Context Properties Are Passed to Security Providers" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500.
Entrez le nom de la propriété en tant que chaîne littérale et le sélecteur de valeur en tant qu'expression XPath valide. (Les expressions XPath peuvent également·être des chaînes littérales.)
Les expressions XPath sont évaluées en fonction de la même partie du message que celle utilisée pour le jeton personnalisé ou le nom utilisateur/mot de passe personnalisé. Ainsi, les expressions XPath du sélecteur de valeur des services proxy de type SOAP sont évaluées en fonction de l'en-tête et les services proxy de type autre que SOAP, en fonction des données traitées.
L'expression XPath est évaluée lors de l'exécution pour produire la valeur de la propriété. Un élément ContextHandler est constitué d'une liste de noms et de valeurs ; le fournisseur de sécurité doit donc savoir quels noms rechercher. Par conséquent, les expressions XPath sont évaluées uniquement si un fournisseur de sécurité demande la valeur de l'une de ces propriétés définies par l'utilisateur.
Cliquez sur Ajouter une propriété pour ajouter cette propriété de contexte. Vous pouvez ajouter plusieurs propriétés de contexte.
