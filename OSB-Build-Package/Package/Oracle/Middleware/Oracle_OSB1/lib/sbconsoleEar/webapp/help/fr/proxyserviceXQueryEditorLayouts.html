Présentations·des mises en page et des tâches de l'éditeur XQuery

L'éditeur d'expressions XQuery, l'éditeur de conditions XQuery et l'éditeur d'expressions XPath sont tous composés des éléments suivants :

  • Palettes
  • Espace de travail 
  • Gestionnaire de propriétés

Palettes

Le panneau de gauche de chaque éditeur contient les palettes énumérées ci-dessous. Dans n'importe lequel de ces éditeurs, cliquez sur le nom d'une palette pour l'afficher. Chaque palette contient des entités que vous pouvez insérer dans des expressions.

  • La palette Définitions d'espace de noms liste les espaces de noms par défaut d'Oracle Service Bus, les espaces de noms de variables et les espaces de noms définis par les utilisateurs. Vous pouvez définir de nouveaux espaces de noms, qui sont ensuite ajoutés à la liste des espaces de noms définis par les utilisateurs.

    • Pour plus d'informations, reportez-vous à :

    • Construction d'expressions dans les champs de texte de l'espace de travail de l'éditeur
    • Création d'espaces de noms à utiliser dans les expressions imbriquées
  • La palette Fonctions XQuery liste un ensemble de fonctions XQuery standard. Lorsque vous insérez une fonction dans une expression, des éléments de réserve remplacent les valeurs de paramètre que vous devez fournir.

    • Pour plus d'informations, reportez-vous à :

    • Construction d'expressions dans les champs de texte de l'espace de travail de l'éditeur
    • Création de fonctions XPath personnalisées dans les éditeurs XQuery
    • Liaison de ressources XQuery externes et d'expressions XQuery imbriquées
  • La palette Structures de variable fournit un ensemble d'outils permettant d'insérer des variables et des chemins vers les variables, à l'aide d'expressions XPath.

    • Les structures de variable sont des représentations graphiques de variables ou de chemins de variable affichées par l'éditeur. En visualisant ces structures, vous pouvez les utiliser pour construire des expressions XQuery imbriquées qui référencent le contenu de la variable.

    Remarque : Les structures de variable ne créent pas de variables. Les variables sont créées lors de l'exécution en tant que cibles de l'action d'affectation dans la phase.

    Oracle Service Bus fournit plusieurs variables de contexte de message prédéfinies (attachments, body, header, outbound et inbound), dont vous pouvez afficher le contenu en tant que structures de variable. Vous pouvez également définir vos propres structures de variable.

    Pour plus d’informations, voir :

    • Construction d'expressions dans les champs de texte de l'espace de travail de l'éditeur
    • Création de structures de variable dans les éditeurs XQuery

Espace de travail

Le côté droit de la page fournit un espace de travail pour construire les expressions XQuery, les conditions XQuery ou XPath. L'espace de travail varie en fonction des trois éditeurs.

Gestionnaire de propriétés

Dans ces trois éditeurs, le gestionnaire de propriétés apparaît en bas à droite de la page. Lorsque vous sélectionnez un élément à ajouter à l'expression dans l'une des palettes, cet élément apparaît dans le gestionnaire de propriétés. Vous pouvez ensuite coller l'élément dans l'espace de travail. Reportez-vous à Construction d'expressions dans les champs de texte de l'espace de travail de l'éditeur.
