Création et modification d'expressions XPath et XQuery imbriquées

Lorsque vous ajoutez des actions à des phases ou à des noeuds de routage sur la page Modifier la configuration de la phase, une structure squelette apparaît et vous invite à configurer les détails. La figure 22-1 en donne un exemple.

Figure 22-1 Exemple de squelette de configuration d'actions

Lorsque cela est approprié au contexte, le squelette fournit des liens pour accéder aux éditeurs XQuery, dans lesquels vous pouvez construire des expressions qui seront exécutées en ligne, comme exigé par le contexte de·l'action.

Avant de commencer

Ces instructions supposent que vous créez ou modifiez une action sur·la page Modifier la configuration de la phase d'une phase de pipeline, d'une phase de gestionnaire d'erreur ou d'un noeud de routage. Reportez-vous à Création, modification et visualisation de flux de messages et Ajout et modification d'actions dans les flux de messages.

Création et modification d'une expression imbriquée

  1. Sur la page Modifier la configuration de la phase, localisez dans l'action l'emplacement où vous souhaitez ajouter ou modifier l'expression.
  2. Cliquez sur le lien de l'expression pour ouvrir l'éditeur approprié au contexte. Lorsque aucune expression n'a encore été définie, le lien indique quel type d'expression vous pouvez utiliser à cet emplacement :
    • Cliquez sur <Expression> pour créer une expression XQuery. Vous pouvez également importer une ressource XQuery ou XSLT créée en dehors d'Oracle Service Bus, puis la lier à l'expression XQuery imbriquée.
    • Cliquez sur <Condition> pour créer une expression XQuery conditionnelle (action If...Then....).
    • Cliquez sur <XPath> afin de·créer une expression XPath pour une variable de contexte de message.

      • Lorsqu'une expression a déjà été définie à un emplacement, le lien <Expression>, <Condition> ou <XPath> est remplacé par un lien qui montre l'expression elle-même, par exemple true()[icon], ou un fragment de l'expression si elle dépasse la zone de texte, par exemple, $body/urn:POSta...[icon]. Cliquez sur l'expression (ou le fragment d'expression) pour l'ouvrir dans l'éditeur approprié.

  3. Construisez l'expression, comme décrit dans les rubriques suivantes :
    • Construction d'expressions dans les champs de texte de l'espace de travail de l'éditeur
    • Création d'espaces de noms à utiliser dans les expressions imbriquées
    • Création de structures de variable dans les éditeurs XQuery
    • Création de fonctions XPath personnalisées dans les éditeurs XQuery
    • Liaison de ressources XQuery externes et d'expressions XQuery imbriquées
    • Liaison de ressources XSLT externes et d'expressions XQuery imbriquées
    • Liaison d'expressions XQuery dynamiques et d'expressions XQuery imbriquées
    • Saisie d'expressions de comparaison XQuery à l'aide de l'option Générateur
    • Saisie d'expressions unaires à l'aide de l'option Générateur
  4. Vous pouvez également exécuter une des tâches suivantes ou les deux :
    • Cliquez sur Valider pour valider l'expression.
    • Cliquez sur Tester pour tester l'expression. Reportez-vous à Exécution du test XQuery.
  5. Cliquez sur Enregistrer pour fermer l'éditeur et insérer l'expression dans l'action.

Après avoir fini

Continuez à configurer l'action, comme indiqué dans Ajout et modification d'actions dans les flux de messages.
