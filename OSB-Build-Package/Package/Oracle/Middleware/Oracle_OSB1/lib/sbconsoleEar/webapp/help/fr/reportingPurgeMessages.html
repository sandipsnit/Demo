Purge des messages

Cette page permet de purger les messages sélectionnés à partir de la banque de données des rapports.

La purge des messages est exécutée en arrière-plan. Ainsi, la page Récapitulatif des rapports sur les messages n'est pas verrouillée lors de la purge. Par conséquent, vous pouvez effectuer une longue purge ; la durée dépend du nombre de messages qui doivent être purgés. Cependant, si vous filtrez des messages de rapport, le processus de purge est ralenti ; la page Récapitulatif des rapports sur les messages peut afficher des données incorrectes dans la mesure où certaines données n'ont pas encore été purgées.

La fonctionnalité de purge des messages peut s'avérer·utile lors des phases de conception et de test de votre projet. Cependant, pour les environnements de production, Oracle recommande que la gestion des données de·votre base de données (y compris la purge) soit réalisée par l'administrateur à l'aide d'outils de gestion de base de données.

  1. Sélectionnez Opérations > Purger les messages.
  2. Exécutez l'une des opérations·suivantes, présentées·dans le tableau 27-9.

    3. Tableau 27-9 Options de purge des messages
    Opération
    Exécutez les étapes suivantes...
    Purger tous les messages
    1. Sélectionnez le bouton d'option Purger tous les messages.
    2. Cliquez sur Purger.
    Purger les messages d'une période en particulier
    1. Sélectionnez le bouton d'option Purger les messages de... A...
    2. Dans le champ Purger les messages de, entrez un mois, un jour, une année et une heure au format MM/DD/YY HH:MM:SS. Par exemple, vous pouvez entrer 10/10/07 12:45:00 AM.
    3. Dans le champ Purger les messages jusqu'à, entrez un mois, un jour, une année et une heure au format MM/DD/YY HH:MM:SS.
    4. Cliquez sur Purger.
