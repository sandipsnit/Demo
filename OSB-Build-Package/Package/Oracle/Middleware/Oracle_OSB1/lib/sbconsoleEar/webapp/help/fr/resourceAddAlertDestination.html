Ajout de destinations d'alertes

Une destination d'alertes est une adresse de destination pour les notifications d'alertes dans Oracle Service Bus. Vous pouvez créer des ressources de·destination d'alertes sur la page Créer une destination d'alertes.

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Sélectionnez Explorateur de projets, puis choisissez le projet ou le dossier auquel ajouter la destination d'alertes. La page Vue du projet/Vue de dossier apparaît.
  3. Dans la liste déroulante Créer une ressource, sélectionnez Destination de l'alerte pour afficher la page Créer une destination d'alertes.
  4. Dans le champ Nom de la ressource, entrez un nom unique pour la destination d'alertes. Ce champ est obligatoire.

    Pour obtenir des conseils sur la dénomination, suivez les restrictions concernant la dénomination des ressources.
  5. Dans le champ Description de la ressource, entrez la description de la destination d'alertes.
  6. Sélectionnez au moins une des destinations suivantes afin de l'inclure dans cette ressource de destination d'alertes.
    7. Remarque : Les alertes sont agrégées dans la console d'administration d'Oracle Service Bus lors de l'exécution, où vous pouvez les visualiser sous Opérations > Tableau de bord.
    • Interruption SNMP : si vous sélectionnez cette option, les alertes sont envoyées en tant qu'interruptions SNMP et peuvent être traitées par n'importe quel système de surveillance d'entreprise (ESM) tiers.
    • Génération de rapports : si vous sélectionnez cette option, les alertes sont envoyées au module Oracle Service Bus Reporting. Elles peuvent être capturées à l'aide d'un fournisseur de rapports personnalisé développé à l'aide du fournisseur d'API de génération de rapports via·Oracle Service Bus. Cela permet aux tiers de recevoir et de traiter des alertes en code Java personnalisé.
    • Journalisation d'alertes : si vous sélectionnez Oui, les alertes envoyées à cette destination sont consignées·dans le journal d'alertes.
  7. Ajoutez un destinataire de courrier électronique à la définition de la ressource de destination d'alertes. Reportez-vous à Ajout de destinataires de courrier électronique.
  8. Ajoutez une destination JMS à la définition de la ressource de destination d'alertes. Reportez-vous à Ajout de destinations JMS.
  9. Cliquez sur Enregistrer. La nouvelle destination d'alertes est enregistrée dans la session en cours.
  10. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
