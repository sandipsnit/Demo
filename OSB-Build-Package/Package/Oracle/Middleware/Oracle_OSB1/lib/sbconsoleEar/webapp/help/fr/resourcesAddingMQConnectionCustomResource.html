Ajout de connexions MQ

Avant de commencer

Pour connaître les versions de MQ WebSphere prises en charge dans Oracle Service Bus, reportez-vous à Oracle Fusion Middleware Supported System Configurations, à l'adresse suivante : http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html. La bibliothèque client MQ WebSphere n'est pas fournie avec le programme d'installation Oracle Service Bus. Vous pouvez l'obtenir à partir de l'installation de votre MQ WebSphere.

Pour configurer et utiliser des ressources de connexion MQ ainsi que le transport MQ dans Oracle Service Bus, vous devez ajouter la bibliothèque client MQ à la variable d'environnement CLASSPATH. Pour ce faire, copiez la bibliothèque com.ibm.mq.jar dans votre répertoire DOMAIN_NAME/lib, DOMAIN_NAME étant le nom du répertoire dans lequel vous avez placé le domaine, généralement MW_HOME\user_projects\domains\DOMAIN_NAME.

Pour ajouter une ressource de connexion MQ

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Sélectionnez Explorateur de projets, puis choisissez le projet ou le dossier auquel ajouter la ressource de connexion MQ. La page Vue du projet/Vue de dossier apparaît.
  3. Dans la liste déroulante Créer une ressource, sélectionnez Connexion MQ sous Utilitaire pour afficher la page Créer une ressource de connexion MQ.

    4. Indiquez les paramètres de connexion présentés·dans le tableau 8-2.

    Tableau 8-2 Paramètres de connexion MQ
    Propriété
    Procédure
    Nom de la ressource
    Entrez un nom unique pour cette ressource de connexion MQ.

    Pour obtenir des conseils sur la dénomination, suivez les restrictions concernant la dénomination des ressources.
    Description de la ressource
    Entrez une description pour la ressource de connexion MQ.
    Type de connexion
    Sélectionnez l'un des modes suivants pour la connexion au gestionnaire de files d'attente MQ.
    • Mode TCP : utilisez le protocole TCP/IP pour la connexion à un gestionnaire de files d'attente qui ne réside pas sur le même ordinateur qu'Oracle Service Bus.
    • Mode d'authentification : utilisez ce mode pour la connexion à un gestionnaire de files d'attente situé sur le même ordinateur qu'Oracle Service Bus.
    Nom d'hôte MQ
    Pour les connexions en·mode TCP uniquement :
    Entrez le nom d'hôte du gestionnaire de files d'attente MQ.
    Numéro de port MQ
    Pour les connexions en·mode TCP uniquement :
    Entrez le numéro de port associé au·processus d'écoute du gestionnaire de files d'attente MQ.
    Nom du gestionnaire de files d'attente MQ
    Entrez le nom du gestionnaire de files d'attente MQ auquel se connecter.
    CCSID du gestionnaire de files d'attente
    Pour les connexions en·mode TCP uniquement :
    L'identifiant de jeu de caractères codés (CCSID) doit être utilisé lors de l'établissement d'une connexion. Le CCSID sert principalement à prendre en charge l'internationalisation.
    Pour en savoir plus, consultez les bases de WebSphere MQ à l'adresse suivante : http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
    Nom de canal du gestionnaire de files d'attente MQ
    Pour les connexions en·mode TCP uniquement :
    Entrez le nom de canal du gestionnaire de files d'attente MQ.
    SSL obligatoire
    Pour les connexions en mode TCP :
    Cochez la case afin d'utiliser le protocole SSL pour l'envoi des messages. Seul le SSL côté serveur est·activé·lorsque l'option SSL bidirectionnel obligatoire n'est pas sélectionnée.
    Mécanisme de cryptage
    Cette option est disponible uniquement lorsque la case SSL obligatoire est cochée.
    Sélectionnez l'algorithme de mécanisme de cryptage que le protocole SSL doit utiliser.
    L'algorithme de mécanisme de cryptage sert à crypter et à décrypter les communications par·message entre le serveur MQ WebSphere et le client MQ WebSphere. Par conséquent, vous devez indiquer un algorithme de mécanisme de cryptage lorsque le protocole SSL est utilisé pour communiquer avec un serveur MQ WebSphere.
    SSL bidirectionnel obligatoire
    Cette option est disponible uniquement lorsque la case SSL obligatoire est cochée.
    Cochez la case pour activer l'authentification SSL côté client et côté serveur.
    Référence au fournisseur de clés de service
    Si vous avez sélectionné l'option SSL bidirectionnel obligatoire, vous devez fournir une référence au fournisseur de clés de service pour obtenir les informations appropriées sur le fichier de clés et le truststore.
    Entrez le chemin (project/folder) et le nom d'un fournisseur de clés de service, ou cliquez sur Parcourir pour en choisir·un sur la page Sélectionner un fournisseur de clés de service.
    Référence au compte de service statique
    Pour les connexions en·mode TCP uniquement :
    Obligatoire pour l'authentification par·nom utilisateur et mot de passe. Entrez le chemin (project/folder) et le nom d'un compte de service statique, ou cliquez sur Parcourir pour sélectionner des comptes de service à partir d'un navigateur Web.
    Version MQ WebSphere
    Sélectionnez la version de MQ WebSphere.
    • 5.3
    • 6.0
    • 7.0

    Remarque : les versions 5.3 et 6.0 ne sont plus prises en charge par IBM. Ces versions sont en phase d'abandon dans Oracle Service Bus.
    Taille du pool de connexions MQ
    Entrez la taille du pool de connexions MQ.
    Temporisation de connexion MQ
    Entrez l'intervalle de temps, en secondes, au bout duquel les connexions inutilisées sont détruites. Par défaut, cet intervalle est de 1 800 secondes.
    Attente maximale de connexion MQ
    Entrez la durée d'attente maximale, en millisecondes, pour qu'une connexion devienne·disponible. Si aucune connexion n'est établie dans cet intervalle de temps, Oracle Service Bus génère une exception. La valeur par défaut est de 3000 millisecondes.

  4. Cliquez sur Enregistrer. La ressource de connexion MQ est créée et enregistrée dans la session en cours.
  5. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
