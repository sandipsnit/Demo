Ajout de rôles

  1. Connectez-vous à la console d'administration d'Oracle Service Bus en tant qu'utilisateur disposant des privilèges d'administration de WebLogic Server. Seuls les utilisateurs disposant du rôle d'administration peuvent modifier les données de configuration de sécurité. Reportez-vous à "Configuring Administrative Security" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.
  2. Sélectionnez Configuration de la sécurité > Rôles pour afficher la page Rôles globaux.
  3. Cliquez sur Nouveau.

    4. Vous pouvez ajouter un rôle que vous soyez ou non connecté à une session.

  4. Dans le champ Nom du rôle, entrez un nom unique. Ce champ est obligatoire.

    5. Veillez à ne pas utiliser d'espaces ni de caractères < > dans le nom du rôle de sécurité. Les noms des rôles de sécurité font la distinction entre les majuscules et les minuscules. Selon la convention Oracle, tous les noms de rôle de sécurité sont singuliers.

  5. Pour créer le rôle, cliquez sur OK.

    6. La console d'administration d'Oracle Service Bus enregistre le rôle, qui devient immédiatement disponible dans toutes les sessions. Si vous supprimez la session dans laquelle vous vous trouviez pendant l'ajout du rôle, la console d'administration d'Oracle Service Bus ne supprime pas le nouveau rôle.

    Après avoir cliqué sur OK pour créer le rôle, vous devez définir les conditions sous lesquelles le rôle s'applique.

  6. Suivez ensuite la procédure décrite dans Définition de conditions de rôle.
