Modification des stratégies d'accès au niveau du message

La page Visualiser les détails de la stratégie permet de modifier la stratégie de contrôle d'accès au niveau du message d'un service proxy représentant un service Web et nécessitant une sécurité au niveau du message. Cette page affiche les informations répertoriées dans le tableau 24-11.

Tableau 24-11 Détails des stratégies 
Propriété
Description
Nom du service proxy
Affiche le nom du service proxy pour lequel vous avez sélectionné l'option Visualiser les stratégies sur la page Contrôle d'accès des services proxy.
Fournisseurs
Affiche les fournisseurs d'autorisation configurés pour le domaine.
Opération
Enumère les opérations du service proxy pouvant être sécurisées.
Conditions de stratégie
Affiche les conditions qui déterminent les utilisateurs étant autorisés à appeler les opérations sélectionnées sous Opérations de service.

  1. Connectez-vous à la console d'administration d'Oracle Service Bus en tant qu'utilisateur disposant des privilèges d'administration de WebLogic Server. Seuls les utilisateurs disposant du rôle d'administration peuvent modifier les données de configuration de sécurité. Reportez-vous à "Configuring Administrative Security" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.
  2. Localisez la stratégie d'accès. Reportez-vous à Localisation de stratégies de contrôle d'accès.
  3. Sur la page Sécurité, dans la colonne Contrôle d'accès des messages, cliquez sur le nom du service proxy ou de l'opération spécifique de ce service proxy. L'éditeur de stratégies apparaît.

    4. Vous pouvez modifier une stratégie de contrôle d'accès lorsque vous êtes connecté à une session.

  4. Dans le champ Fournisseurs d'autorisation, choisissez un fournisseur d'autorisation. Oracle recommande de sélectionner XACMLAuthorizer.
    5. Remarque : la prise en charge du fournisseur d'autorisation WebLogic par défaut est en phase d'abandon dans Oracle Service Bus. A la place, Oracle recommande d'utiliser le fournisseur d'autorisation XACML WebLogic.
  5. Ajoutez des conditions de stratégie. Reportez-vous à Ajout de conditions de stratégie.
  6. Après avoir terminé la saisie des conditions dans la section Conditions de stratégie, cliquez sur Enregistrer.
