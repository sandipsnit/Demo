Comptes de service

Un compte de service fournit un nom utilisateur et un mot de passe que les services proxy et fonctionnels utilisent pour l'authentification sortante, ou pour l'authentification auprès d'une ressource locale ou distante, telle qu'un serveur FTP ou JMS. Par exemple, si un service fonctionnel doit fournir un nom utilisateur et un mot de passe pour une authentification au niveau du transport avec un service Web, vous devez créer un compte de service qui spécifie le nom utilisateur et le mot de passe, puis configurer le service fonctionnel de sorte qu'il inclue les informations d'identification et de connexion du compte de service dans ses demandes entrantes.

Les noms utilisateur et les mots de passe que vous indiquez dans les comptes de service sont utilisés pour l'authentification sortante ou pour fournir des informations d'identification et de connexion à des ressources locales ou distantes. Les noms utilisateur et les mots de passe que vous indiquez dans le module de configuration de sécurité de la console d'administration d'Oracle Service Bus sont utilisés pour l'authentification entrante et pour l'authentification des demandes administratives. Reportez-vous à Spécification de comptes de service.
