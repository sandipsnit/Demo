Modification des comptes de service

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Localisez le compte de service, comme décrit dans Localisation de comptes de service.
  3. Cliquez sur le nom du compte de service. La page Visualiser les détails du compte de service affiche les informations présentées dans le tableau 16-3.

    4. Tableau 16-3 Détails du compte de service
    Propriété
    Description
    Dernière modification par
    Utilisateur ayant créé le compte de service ou l'ayant importé dans la configuration.
    Dernière modification le
    Date et heure auxquelles l'utilisateur a créé le compte de service ou l'a importé dans la configuration. Cliquez sur le lien de date et d'heure pour visualiser l'historique des modifications apportées à cette ressource. Reportez-vous à Page Visualiser l'historique des modifications.
    Références
    Nombre d'objets référencés par le compte de service. Lorsque des références existent, cliquez sur le lien numérique pour visualiser la liste des objets en question. Reportez-vous à Visualisation des références aux ressources.
    Référencé par
    Nombre d'objets référençant ce compte de service. Lorsque des références existent, cliquez sur le lien numérique pour visualiser la liste des objets en question. Par exemple, si vous avez sélectionné ce compte de service comme compte de service JMS dans un service proxy avec un protocole de transport JMS, le service proxy est énuméré en tant que référence, lorsque vous cliquez sur le lien. Reportez-vous à Visualisation des références aux ressources.
    Description
    Description du compte de service, le cas échéant.

  4. Pour modifier les champs, cliquez sur Modifier. Pour obtenir les descriptions de ces champs, reportez-vous à Ajout de comptes de service.

    5. Vous ne pouvez pas modifier le champ Nom de la ressource.

  5. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  6. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer dans·le centre des modifications.
Remarque : Si le compte de service que vous avez modifié doit être authentifié auprès d'un serveur WebLogic JMS, le serveur JMS ne reconnaîtra peut-être pas votre modification pendant un délai maximal de 60 secondes. Par défaut, le serveur WebLogic JMS vérifie les droits d'accès pour chaque destination toutes les 60 secondes. Pour changer ce comportement, modifiez la commande de démarrage du serveur WebLogic de telle façon qu'elle définit la propriété système suivante sur la fréquence (en secondes) à laquelle le serveur WebLogic JMS doit vérifier les droits d'accès : weblogic.jms.securityCheckInterval.
Si la propriété a la valeur 0 (zéro), cela garantit qu'une vérification des droits d'accès est effectuée pour chaque action send, receive et getEnumeration sur une ressource JMS.
