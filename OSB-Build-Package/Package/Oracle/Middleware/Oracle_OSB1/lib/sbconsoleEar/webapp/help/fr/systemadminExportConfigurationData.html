Export de ressources

Utilisez la page Exporter les ressources pour exporter des projets ou exporter des ressources. Si vous choisissez d'exporter un projet complet, vous pouvez sélectionner plusieurs projets à exporter. Vous pouvez toujours développer les projets pour visualiser les dossiers et ressources, mais la sélection ne se fait qu'au niveau du projet. En outre, vous ne pouvez pas sélectionner l'option Inclure les dépendances.

Vous pouvez exporter des projets et des ressources que vous travailliez dans la session ou en dehors de celle-ci. Si vous procédez à l'export dans une session, les ressources sont celles de la session et la configuration peut être incomplète ou faire l'objet de conflits. Si vous procédez à l'export en dehors d'une session, les ressources ne sont autres que les ressources centrales activées.

Oracle Service Bus ne peut pas exporter les utilisateurs, groupes ou rôles que vous créez dans le module Configuration de la sécurité de la console d'administration d'Oracle Service Bus. Il ne peut pas non plus exporter les correspondances d'informations d'identification et de connexion ou les autres données de fournisseur de sécurité que vous créez dans la console d'administration WebLogic Server. Utilisez plutôt cette console pour exporter les données en question. Reportez-vous à "Migrating Security Data" dans le manuel Oracle Fusion Middleware Securing Oracle WebLogic Server, à l'adresse http://www.oracle.com/pls/as111170/lookup?id=SECMG319.

  1. Sélectionnez Administration du système > Exporter les ressources.

    2. La page Exporter les ressources affiche la liste des objets compris dans·votre configuration. Pour chaque objet, le nom, le type, la date de dernière modification et les références sont affichés.

    Pour les services, l'horodatage de dernière modification reflète les modifications apportées à la définition de service, ainsi qu'à la configuration du transport de service ou du pipeline. Toutefois, il ne reflète pas les modifications apportées aux alertes définies pour le service.

  2. Sélectionnez Exporter les projets ou Exporter les ressources.
    3. Remarque : L'export de projets peut entraîner la suppression de ressource à l'import du fichier JAR du projet complet. Pour plus d'informations, reportez-vous à Import de ressources.
  3. Sélectionnez les projets ou ressources à exporter. Si vous avez choisi d'exporter des projets, vous ne pouvez pas désélectionner les cases des différentes ressources.
    1. Développez les dossiers de projet. Le nom, le type, la date de dernière modification et les références de chaque ressource contenue dans le projet apparaissent.
    2. Cochez les cases associées aux projets ou ressources à exporter.
    3. Désélectionnez les cases associées aux projets ou ressources à ne pas exporter.
  4. Si vous avez choisi d'exporter des ressources, cochez ou désélectionnez la case Inclure les dépendances. Utilisez cette option pour exporter d'autres ressources référencées par cette ressource.
  5. Cliquez sur Exporter pour créer un fichier JAR et l'exporter.

Après avoir fini

Le cas échéant, continuez en passant à Définition·des paramètres de sécurité de l'export.
