Import de services fonctionnels à partir d'un registre UDDI

Vous pouvez importer dans Oracle Service Bus les types de service fonctionnel suivants à partir d'un registre UDDI :

  • Services WSDL sur un transport HTTP.
  • Services proxy Oracle Service Bus publiés dans un registre UDDI. Cette fonctionnalité est principalement utilisée dans les déploiements Oracle Service Bus à plusieurs domaines où les services proxy d'un domaine doivent être repérés et acheminés vers un autre domaine.

Si vous ne disposez que d'un registre configuré, Oracle Service Bus se connecte automatiquement à ce registre, extrait tous les objets fonctionnels et remplit le formulaire de recherche. Si vous disposez de plusieurs registres configurés, l'assistant d'import vous invite tout d'abord à sélectionner le registre à partir duquel vous souhaitez importer les services.

Lors de l'import, l'URL d'interrogation permet de localiser un service spécifique, dans la mesure où un registre dispose de plusieurs types de service différents.

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Sélectionnez Administration du système > Importer à partir d'UDDI.
  3. Dans la liste déroulante Nom du registre d'import, sélectionnez le nom du registre à partir duquel importer le service, puis cliquez sur Suivant.

    4. Lorsqu'un seul registre est configuré dans Oracle Service Bus, la page de sélection du registre initial n'est pas affichée, puisqu'il s'agit du registre par défaut. La page de sélection du registre n'apparaît que lorsque au moins deux registres sont définis.

    Utilisez la page Importer à partir d'UDDI pour rechercher des services fonctionnels spécifiques et les importer. Vous pouvez rechercher un service par objet fonctionnel, par modèle ou par une combinaison des deux.

  4. Sélectionnez le nom de l'objet fonctionnel (par exemple, document Services) dans la liste déroulante Nom d'objet pour effectuer une recherche par objet fonctionnel.
  5. Dans le champ Nom du service, entrez le modèle (par exemple, a%) à rechercher. Une liste de services fonctionnels correspondant aux critères de recherche saisis apparaît.
  6. Sélectionnez les services à importer, puis cliquez sur Suivant.

    7. Si vous ne parvenez pas à trouver un service souhaité, cela peut être dû au fait que vous ne faites·pas partie du groupe de sécurité disposant des droits d'accès permettant de visualiser les enregistrements.

  7. Sélectionnez les modèles de liaison à partir desquels vous souhaitez créer des services fonctionnels, puis cliquez sur Suivant.

    8. Si un service sélectionné possède plusieurs modèles de liaison, chacun d'entre eux génère un service fonctionnel. Dans ce cas, le système vous invitera à affiner votre sélection parmi les modèles de liaison à importer.

  8. Sur la page Import UDDI : Sélectionner un emplacement d'import, sélectionnez un projet dans la liste déroulante Projet. Sélectionnez ensuite un dossier dans lequel vous souhaitez importer vos services et ressources associées, puis cliquez sur Suivant.
  9. La page Import UDDI : Vérifier et importer les services affiche une liste de toutes les ressources (services fonctionnels, MFL, schémas, WSDL, etc.) qui seront créées dans le système. Un message d'avertissement apparaît pour les ressources impossibles à importer.
    Par défaut, tous les éléments de la liste sont marqués pour l'import.
    1. Désélectionnez la case située en regard d'une ressource spécifique dont·vous disposez déjà dans votre système et dont·vous souhaitez résoudre les dépendances manuellement.
    2. Vous pouvez résoudre les dépendances sur la page Visualiser les conflits dans le centre des modifications, une fois le processus d'import terminé. Pour plus d'informations sur la visualisation et la résolution des conflits, reportez-vous à Visualisation et résolution des conflits.
  10. Cliquez sur Importer pour démarrer le processus d'import.

    11. Une fenêtre instantanée apparaît et indique la progression de l'import. Lorsque l'import est terminé, la page Récapitulatif de l'import affiche le résultat de l'import. Un message de succès apparaît en haut de la page si l'import a réussi. Si au moins une ressource n'a pas pu être importée, un message d'erreur apparaît.

    • Accédez à la colonne Statut de la ressource pour visualiser la cause de l'erreur.
    • Pour visualiser et résoudre d'autres conflits, cliquez sur Visualiser les conflits.
  11. Pour importer plus de services, cliquez sur Importer un autre élément.
  12. Pour apporter des modifications à vos sélections, cliquez sur Précédent.
