Import de comptes de service ou de fournisseurs de clés de service

Remarque : Dans les versions précédentes d'Oracle Service Bus, les fournisseurs de clés de service étaient appelés fournisseurs de service proxy.

Si le fichier JAR a été créé par AquaLogic Service Bus 3.0 ou une version ultérieure, et qu'il contient des comptes de service ou des fournisseurs de clés de service, vous pouvez importer ces ressources et les éléments qu'elles contiennent, à savoir les noms utilisateurs, les mots de passe, les correspondances utilisateur local/utilisateur distant et les liaisons d'alias et de paires de clés.

Pour chaque fournisseur de clés de service, Oracle Service Bus importe la liaison d'alias et de paires de clés dans le fournisseur de correspondances d'informations d'identification et de connexion PKI. Si les données étaient cryptées lors de l'export, vous devez fournir le mot de passe utilisé pour crypter les données. Même si·vous ne connaissez pas ce mot de passe, vous pouvez importer toutes les autres ressources non cryptées.

Si vous importez un compte de service ou un fournisseur de clés de service et qu'une ressource correspondante du même nom existe déjà dans votre domaine, la ressource importée écrasera celle existant dans votre domaine, même si cette dernière a été modifiée pendant la session en cours, sauf si vous spécifiez la conservation des paramètres de sécurité lors de l'import. Reportez-vous à l'étape 5 dans Sélection de projets et de ressources à importer.
