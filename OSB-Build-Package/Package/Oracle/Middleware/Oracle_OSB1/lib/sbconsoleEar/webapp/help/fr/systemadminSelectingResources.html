Sélection de projets et de ressources à importer

  1. Si vous importez un fichier JAR de ressource et que vous souhaitez vous assurer que toutes ses ressources associées seront importées, sélectionnez Inclure les dépendances.
    2. Remarque : Vous ne pouvez pas sélectionner l'option Inclure les dépendances si vous importez un fichier JAR de projet complet ou un fichier de configuration antérieur·à la version 2.6.
  2. Cliquez sur l'icône Ouvrir pour afficher les paramètres avancés afin de conserver les valeurs du système d'import.
  3. Lorsque vous réimportez une ressource mais souhaitez protéger les personnalisations apportées à la ressource existante, sélectionnez Conserver les variables d'environnement.
    4. Remarque : La personnalisation des références dans une ressource existante ne peut pas être conservée. Reportez-vous à Création de fichiers de personnalisation.
  4. Si vous importez la ressource Paramètres opérationnels globaux ou d'autres ressources, telles que les services fonctionnels ou proxy qui contiennent des paramètres opérationnels de niveau service associés, et que vous souhaitez conserver les valeurs opérationnelles du système d'import, sélectionnez Conserver les valeurs opérationnelles.
  5. Pour éviter d'écraser la configuration de sécurité existante dans les ressources mises à jour, sélectionnez l'une des options suivantes :
    1. Conserver la configuration des stratégies et de la sécurité : conserve la configuration de sécurité (excepté les stratégies·de contrôle d'accès) et les références aux stratégies WS directement liées au service, au lieu d'être liées au langage WSDL (Web Service Definition Language).

      2. Pour plus d'informations, reportez-vous à "Security" dans le manuel Oracle Fusion Middleware Developer's Guide for Oracle Service Bus, à l'adresse http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472.

    2. Conserver les informations d'identification et de connexion : conserve les informations d'identification et de connexion PKI dans les fournisseurs de clé de service, le nom utilisateur et le mot de passe dans les comptes de service, ainsi que les informations d'identification et de connexion (nom utilisateur et mot de passe) dans les serveurs SMTP, les serveurs proxy, les fournisseurs JNDI et les registres UDDI.
    3. Conserver les stratégies de contrôle d'accès : conserve les stratégies de contrôle d'accès.

      4. Les informations récapitulatives de la ressource sont présentées dans le tableau 28-1.

      Tableau 28-1 Informations relatives à la ressource
      Propriété
      Description
      Nom
      Nom du projet.
      Les ressources globales, telles que les informations relatives au registre UDDI, les serveurs SMTP, les serveurs proxy, les fournisseurs JNDI et les paramètres opérationnels globaux apparaissent sous le projet Système dans leur dossier respectif.
      Type
      Type de ressource : Projet.
      Développez le dossier Projet pour afficher et désélectionner les ressources.
      Opérations
      Affiche --- ou !Suppressions!
      Développez le dossier Projet pour afficher les opérations (création, mise à jour, suppression) qui seront exécutées sur les ressources.
      Pour éviter la suppression des ressources, désélectionnez-les. Les ressources ne sont programmées pour la suppression que lorsque le fichier JAR en cours d'import est un fichier JAR de projet complet et que le système d'import contient des ressources de projet ne figurant pas dans le fichier JAR importé.
      Références
      Développez le dossier Projet pour afficher le nombre d'objets référencés par·chaque ressource.

  6. Sélectionnez les objets sur lesquels vous souhaitez effectuer une opération (création, mise à jour ou suppression).
    1. Pour afficher chaque objet contenu dans le fichier JAR, développez le dossier du fichier. La case associée à chaque objet est cochée.

      2. Les détails du fichier de ressource suivants, affichés pour chaque objet, sont présentés dans le tableau 28-2.

      Tableau 28-2 Détails du fichier de ressource 
      Propriété
      Description
      Nom
      Nom et chemin de la ressource. Le chemin correspond au nom de projet et au nom du dossier dans lequel réside la ressource.
      Type
      Types de ressource d'Oracle Service Bus.
      Remarque : Si une ressource contient des données de sécurité sensibles, une icône Cryptage apparaît.
      Attention : Si vous sélectionnez un compte de service ou un fournisseur de clés de service, vous pouvez écraser les données de sécurité existantes, sauf si vous spécifiez la conservation des paramètres de sécurité lors de l'import. Reportez-vous à Import de comptes de service ou de fournisseurs de clés de service.
      Opérations
      Création, mise à jour ou suppression.
      Les ressources ne sont programmées pour la suppression que lorsque le fichier JAR en cours d'import est un fichier JAR de projet complet et que le système d'import contient des ressources de projet ne figurant pas dans le fichier JAR importé.
      Références
      Nombre d'objets référencés par·la ressource.

    2. Désélectionnez les cases situées en regard des ressources à ne pas importer.
  7. Cliquez sur Importer.

Après avoir fini

Le cas échéant, continuez en passant à Indication des paramètres de sécurité de l'import. Vous pouvez également passer à Consultation du récapitulatif d'import.
