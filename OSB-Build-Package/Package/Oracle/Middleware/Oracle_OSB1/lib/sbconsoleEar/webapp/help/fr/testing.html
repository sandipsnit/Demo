Console de test

La console de test Oracle Service Bus constitue un environnement de test basé sur un navigateur Web permettant de valider et de tester la conception de votre système. Vous pouvez configurer l'objet de votre test (service proxy, service fonctionnel ou ressource), exécuter le test et visualiser les résultats dans la console de test. Dans certains cas, vous pouvez également générer une trace dans le code et examiner l'état du message à des points de trace spécifiques. Pour plus d'informations sur la console de test, reportez-vous à "Using the Test Console" dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG357.

Seuls les utilisateurs disposant des rôles IntegrationAdmin et IntegrationDeployer peuvent utiliser la console de test. Pour plus d'informations sur les rôles, reportez-vous à "Roles in Oracle Service Bus" dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG463.

Remarque : si vous recevez un message d'erreur selon lequel le service de console de test n'est pas en cours d'exécution, essayez de définir l'adresse d'écoute du serveur d'administration sur une adresse valide spécifique, telle que localhost. Dans la console Oracle WebLogic Server, accédez à Environnement > Serveurs > admin_server_name > Configuration > Généralités pour définir l'adresse d'écoute.
