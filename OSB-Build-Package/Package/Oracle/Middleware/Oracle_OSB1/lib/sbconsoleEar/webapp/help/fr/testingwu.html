Test des services proxy

Vous pouvez tester les types de service proxy suivants : Tout XML, Tout SOAP, Messagerie, XML et SOAP. Vous pouvez tester les services proxy SOAP à l'aide des stratégies de sécurité de service Web (WSS). Reportez-vous à Web Service Security dans Configuration des données de test des services proxy.

Remarque : lorsque la console de test appelle un service proxy disposant d'une authentification de jeton personnalisé HTTP, aucune vérification de l'authentification n'est effectuée.
Attention : si vous testez les services proxy alors que l'option d'appel direct est activée, certaines étapes de sécurité importantes sont ignorées, y compris le contrôle d'accès. Oracle recommande de ne pas utiliser le service de test dans les systèmes de production.
  1. Cliquez sur Activer sous Centre des modifications pour activer la fonctionnalité de test dans la console d'administration d'Oracle Service Bus.

    2. Vous pouvez tester les services proxy à partir du navigateur Web de ressources ou de l'explorateur de projets.

  2. Sélectionnez Navigateur Web de ressources > Services proxy pour afficher la page Récapitulatif des services proxy.
  3. Sous Actions, cliquez sur l'icône Lancer la console de test correspondant au service proxy à tester.

    4. La page Test de service proxy apparaît. Cliquez, par exemple, sur l'icône correspondant au service proxy LoanGateway1 dans les exemples fournis avec le produit.

    Remarque : dans un domaine inclus dans un cluster, vous ne pouvez pas utiliser la console de test pour tester les services proxy ou fonctionnels configurés qui acheminent vers un service fonctionnel.
  4. Pour les services SOAP et XML, sélectionnez l'opération WSDL à tester.
  5. Configurez les données de test du service proxy. Ces données doivent être celles que le service proxy attend du client.

    6. Par défaut, les deux options de configuration de test, Appel direct et Inclure la fonction de trace, sont activées. Vous pouvez désélectionner l'option Appel direct, ce qui désactive également l'option Inclure la fonction de trace. Lorsque ces options sont désactivées, le test est exécuté à l'aide de la méthode d'appel indirect, selon laquelle le message est envoyé via la couche transport.

    Vous pouvez utiliser l'option Appel direct (case Appel direct cochée) et désactiver la fonction de trace d'exécution. Pour ce faire, il vous suffit de désélectionner la case Inclure la fonction de trace.

    Pour visualiser la trace de message, désélectionnez l'option Appel direct.

    Remarque : pour visualiser la trace dans le fichier journal ou de sortie standard (console du serveur), vous devez définir la journalisation d'Oracle WebLogic Server sur les niveaux de gravité suivants :

Niveau de gravité minimal à journaliser : Info
Fichier journal : Info
Sortie standard : Info

Pour plus d'informations sur la configuration des niveaux de gravité de journalisation, reportez-vous à Using Log Severity Levels dans le manuel Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server à l'adresse http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

  1. Cliquez sur Exécuter. La page Test de service proxy affiche les résultats. Pour plus d'informations sur l'interprétation des résultats de test, reportez-vous à Visualisation des résultats du test des services proxy.
  2. Pour relancer le test, cliquez sur Retour. Reprenez à partir de l'étape 5.
