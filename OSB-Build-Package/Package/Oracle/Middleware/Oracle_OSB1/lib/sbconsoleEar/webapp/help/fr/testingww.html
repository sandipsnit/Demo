Visualisation des résultats du test des services proxy

Le tableau 32-2 décrit les résultats de test des services proxy. La fonction de trace n'est activée que si vous avez sélectionné les deux options Appel direct et Inclure la fonction de trace.

Tableau 32-2 Résultats de test des services proxy 
Section
Description
Nom du service proxy
Nom du service proxy testé.
Actions de la console de test
Précédent : affiche la page précédente du navigateur Web.
Fermer : ferme la console de test.
Document de demande
Message de demande envoyé au service proxy par la console de test.
Cette section est initialement réduite si la console de test n'a pas modifié le message de demande. Elle est initialement développée pour les services SOAP configurés via l'onglet Panneau ou si la sécurité WS a été appliquée.
Si la sécurité WS a été appliquée, cette section contient deux messages SOAP : le premier correspond au message en clair et le second au message SOAP sécurisé.
Document de réponse
Réponse au message.
Dans le cas d'un service SOAP avec une réponse de sécurité WS, cette section contient deux messages SOAP. Le premier correspond au message sécurisé tel qu'il est reçu par le client. Le second est le message en clair correspondant.
Métadonnées de la réponse
Métadonnées renvoyées avec la réponse au message.
Fonction de trace des services proxy
La fonction de trace indique l'état du message lors de sa transmission dans le système. Si vous ne sélectionnez pas l'option Appel direct, la fonction de trace n'est pas exécutée. Pour plus d'informations sur cette fonction, reportez-vous à Fonction de trace des services proxy.
