Test des services fonctionnels

Vous pouvez tester les types de service fonctionnel suivants : Tout XML, Tout SOAP, Messagerie, XML et SOAP. Vous pouvez tester les services fonctionnels SOAP à l'aide des stratégies de sécurité de service Web. Pour plus d'informations, reportez-vous à "Testing Services with Web Service Security" dans le manuel Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus, à l'adresse suivante : http://www.oracle.com/pls/as111170/lookup?id=OSBAG384.

Remarque : dans un domaine inclus dans un cluster, vous ne pouvez pas utiliser la console de test pour tester les services proxy ou fonctionnels configurés qui acheminent vers un service fonctionnel.
  1. Cliquez sur Activer sous Centre des modifications pour activer la fonctionnalité de test dans la console d'administration d'Oracle Service Bus.
  2. Sélectionnez Navigateur Web de ressources > Services fonctionnels pour afficher la page Récapitulatif des services fonctionnels.
  3. Sous Actions, cliquez sur l'icône Lancer la console de test correspondant au service à tester. La page Test de service fonctionnel apparaît. Cliquez, par exemple, sur l'icône correspondant à loanSaleProcessor, dans les tutoriels fournis avec le produit.
  4. Pour les services SOAP et XML, sélectionnez l'opération WSDL à tester.
  5. Configurez les données de test du service fonctionnel (les données d'entrée doivent correspondre au message envoyé par le service proxy au service fonctionnel). Les options Appel direct et Inclure la fonction de trace qui sont disponibles lors du test des services proxy ne le sont pas pour les services fonctionnels. Par défaut, les services fonctionnels sont testés à l'aide de l'option Appel direct, ce qui signifie que les messages sont transmis via la couche transport.

    6. Pour visualiser la trace de message, désélectionnez l'option Appel direct.

    Remarque : pour visualiser la trace dans le fichier journal ou de sortie standard (console du serveur), vous devez définir la journalisation d'Oracle WebLogic Server sur les niveaux de gravité suivants :

Niveau de gravité minimal à journaliser : Info
Fichier journal : Info
Sortie standard : Info

Pour plus d'informations sur la configuration des niveaux de gravité de journalisation, reportez-vous à Using Log Severity Levels dans le manuel Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server à l'adresse http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

  1. Cliquez sur Exécuter.

    2. La page Test de service fonctionnel affiche les résultats. Pour plus d'informations, reportez-vous à Visualisation des résultats de test des services fonctionnels.

Test des pièces jointes dans les services fonctionnels

Cette section explique comment utiliser la console de test pour tester les pièces jointes dans les services fonctionnels.

A des fins d'illustration, cette section suppose qu'un service fonctionnel HTTP de type WSDL utilise une opération "submitAttachment" pour envoyer un fichier ZIP en tant que pièce jointe d'un message SOAP. Si nécessaire, modifiez les paramètres pour les différents types de pièce jointe binaire.

    1. Dans la console de test du service fonctionnel, accédez à la page XML et entrez l'enveloppe SOAP suivante pour le message de test :
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. Dans la section Pièce jointe de la console de test :
      1. Type de contenu : application/fichier ZIP (tel que défini dans le WSDL)
      2. ID de contenu : fichier ZIP (doit correspondre à la valeur de href="cid:zipFile" dans le message de test)
      3. Fichier : cliquez sur Parcourir et sélectionnez le fichier ZIP que vous avez indiqué dans l'enveloppe SOAP.
      4. Cliquez sur Ajouter.

    2. Cliquez sur Exécuter pour envoyer le message de test avec la pièce jointe.

Pour confirmer l'envoi de la pièce jointe, recherchez dans la console du serveur un message journalisé par l'opération submitAttachment, semblable au message suivant :

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
