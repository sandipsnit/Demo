Fonction de trace des services proxy

La fonction de trace est activée lorsque vous testez les services proxy à l'aide d'un appel direct. Par défaut, la case Inclure la fonction de trace est cochée lorsque l'option Appel direct l'est également. Si vous ne souhaitez pas utiliser la fonction de trace, désélectionnez la case Inclure la fonction de trace. Lorsque vous activez la fonction de trace, les résultats de test incluent les détails de la trace. Cette fonction permet d'effectuer le suivi des problèmes survenant dans le système et de les isoler afin de les corriger. Les informations de trace indiquent les chemins utilisés pour la demande et la réponse dans le code.

  1. Cliquez sur Activer sous Centre des modifications pour activer la fonctionnalité de test dans la console d'administration d'Oracle Service Bus.
  2. Sélectionnez Navigateur Web de ressources > Services proxy pour afficher la page Récapitulatif des services proxy.
  3. Sous Actions, cliquez sur l'icône Lancer la console de test correspondant au service à tester. La page Test de service proxy apparaît.
  4. Configurez les données de test du service proxy. Vous devez sélectionner les deux options Appel direct et Inclure la fonction de trace pour activer la fonction de trace. Reportez-vous à Configuration des données de test des services proxy.
  5. Cliquez sur Exécuter. La page Test de service proxy affiche les résultats de test du service ainsi que les informations relatives à la fonction de trace.
  6. Faites défiler la page jusqu'à la section Trace de l'appel.

    7. Cette section affiche une représentation du flux de messages sous forme de graphique. Vous pouvez générer la trace du message dans le service et visualiser l'état du message en des points présélectionnés de la trace. Les points de trace sont définis automatiquement.

  7. Cliquez sur le signe plus (+) pour développer le flux de messages et visualiser plus de détails.

    8. Lorsque vous visualisez la trace, vous pouvez également afficher le flux de messages dans la console d'administration d'Oracle Service Bus. Cela vous permet d'établir plus facilement un rapprochement entre la trace et les étapes/actions dans le flux de messages. Vous pouvez modifier le flux de messages et réexécuter la trace pour visualiser la sortie.
