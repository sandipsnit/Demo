Test des transformations XSLT

Les transformations XSLT (eXtensible Stylesheet Language Transformations) décrivent les correspondances de XML à XML dans Oracle Service Bus. Pour tester une ressource XSLT, vous devez fournir le document XML d'entrée et la console de test renvoie le document XML de sortie. Les transformations XSLT peuvent contenir des paramètres nommés supplémentaires. Tous les paramètres requis pour la transformation sont indiqués sur la page de configuration. Des valeurs par défaut sont disponibles mais vous pouvez les remplacer.

Le tableau 32-6 décrit les options de configuration XSLT.

Entrée XML
L'entrée XML peut être basée sur un fichier ou sur un texte. Le référencement d'un fichier pour entrée est prioritaire sur l'entrée textuelle. Sélectionnez le fichier à utiliser dans votre test. L'entrée XML est obligatoire.
<param_name> ([] au format XML)
param_name représente un paramètre XSLT nommé.
Il existe deux types d'entrée : XML et primitive (String, Integer, Float, etc.). Le type d'entrée par défaut est String. Cochez la case associée au nom du paramètre pour identifier un paramètre de type XML.

Vous pouvez tester la conception ou l'exécution.

  1. Pour tester l'exécution, cliquez sur Activer. N'activez pas la session pour tester la conception.
  2. Sélectionnez Navigateur Web de ressources > XSLT pour afficher la page Récapitulatif des ressources XSLT.
  3. Sous Actions, cliquez sur l'icône Lancer la console de test correspondant à la ressource à tester. La page Test de ressource apparaît.
  4. Configurez les données de test de la ressource dans la section Entrée et paramètres de la page. Pour plus d'informations, reportez-vous au tableau 32-6.
  5. Cliquez sur Exécuter. La page Test de ressource affiche les résultats.
  6. Pour réexécuter le test, cliquez sur Précédent. Vous pouvez fermer la console de test, puis modifier et tester à nouveau la ressource.
