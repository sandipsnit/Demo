Tests des transformations XQuery

Les correspondances XQuery peuvent décrire des correspondances de XML à XML, de XML à non-XML et de non-XML à XML. Une transformation XQuery peut utiliser plusieurs entrées, mais elle ne renvoie qu'une seule sortie. Chaque entrée correspond à une variable externe XQuery déclarée dans la ressource XQuery. La valeur d'une variable d'entrée XQuery peut être une valeur primitive (chaîne, entier, date, etc.), un document XML ou une séquence composée de ces éléments. La variable de sortie peut être une valeur primitive (chaîne, entier, date, etc.), un document XML ou une séquence composée de ces éléments.

Remarque : La console de test ne prend pas en charge les séquences sur l'entrée.

Variables
Cette section contient un champ d'entrée pour chacune des variables externes XQuery.
<param_name> ([] au format XML)
param_name représente un nom de variable XQuery dans la ressource XQuery.
Dans la console de test, une zone d'édition monoligne apparaît si les données sont de type simple. Si les données sont de type XML, c'est une zone d'édition multiligne qui est affichée.
Une entrée de combinaison (<param_name> ([] au format XML)) est utilisée lorsque aucune variable n'est saisie. Vous devez déclarer le type de variable. Cochez cette case pour identifier un paramètre de type XML.
Une entrée XML peut être basée sur un fichier ou sur un texte. Le référencement d'un fichier pour entrée est prioritaire sur l'entrée textuelle. Sélectionnez le fichier à utiliser dans votre test.
Les données entrées dans la console de test sont affichées en fonction de leur type ; cela permet à l'utilisateur de connaître plus facilement le type de données à saisir. Si aucune valeur n'est saisie, le type utilisé par défaut est XML.

Vous pouvez tester la conception ou l'exécution.

  1. Pour tester l'exécution, cliquez sur Activer. N'activez pas la session pour tester la conception.
  2. Sélectionnez Explorateur de projets > Ressources XQuery pour afficher la page Récapitulatif des ressources XQuery.
  3. Sous Actions, cliquez sur l'icône Lancer la console de test correspondant à la ressource à tester. La page Test de ressource apparaît.
  4. Configurez les données de test de la ressource dans la section Variables de la page. Pour plus d'informations, reportez-vous au tableau 32-7.
  5. Cliquez sur Exécuter. La page Test de ressource affiche les résultats.
  6. Pour réexécuter le test, cliquez sur Précédent. Vous pouvez fermer la console de test, puis modifier et tester à nouveau la ressource.
