Utilisation des éditeurs d'expressions XQuery et de conditions XQuery

Les expressions XQuery permettent de créer le contenu des données des variables de contexte de message (ou d'une partie d'une variable de contexte de message) au cours de l'exécution du flux de messages. Vous pouvez utiliser la console de test directement dans l'éditeur d'expressions XQuery pour tester la définition de l'expression.

Les conditions XQuery permettent quant à elles d'évaluer les conditions booléennes dans le flux de messages. Vous pouvez utiliser la console de test directement dans l'éditeur de conditions XQuery pour tester la définition de la condition.

Une ressource XQuery peut utiliser plusieurs entrées, mais elle ne renvoie qu'une seule sortie. Chaque entrée correspond à une variable non liée XQuery définie dans la ressource XQuery. La valeur d'une entrée XQuery peut être une valeur primitive (chaîne, entier, date, etc.), un document XML ou une séquence composée de ces éléments. La variable de sortie peut être une valeur primitive (chaîne, entier, date, etc.), un document XML ou une séquence composée de ces éléments.

Remarque : La console de test ne prend pas en charge les séquences sur l'entrée.

<param_name> ([] au format XML)
param_name représente le nom de la variable non liée XQuery correspondante.
Dans la console de test, une zone d'édition monoligne apparaît si les données sont de type simple. Si les données sont de type XML, c'est une zone d'édition multiligne qui est affichée. Une entrée de combinaison (<param_name> ([] au format XML)) est utilisée lorsque aucune variable n'est saisie. Vous devez déclarer le type de variable. Cochez cette case pour identifier un paramètre de type XML.
Une entrée XML peut être basée sur un fichier ou sur un texte. Le référencement d'un fichier pour entrée est prioritaire sur l'entrée textuelle. Sélectionnez le fichier à utiliser dans votre test.
Les données entrées dans la console de test sont affichées en fonction de leur type ; cela permet à l'utilisateur de connaître plus facilement le type de données à saisir. Le type par défaut est XML.

  1. Accédez à la console de test lors de la modification d'une action dans le flux de messages d'un pipeline.
    • Pour accéder à l'éditeur d'expressions XQuery, reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.
    • Pour accéder à l'éditeur de conditions XQuery, reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.

      • La page Expression de test XQuery apparaît. Toutes les variables d'entrée demandées sont affichées sur la page.

  2. Configurez les données de test de la ressource XQuery dans la section Variables de la page. Pour plus d'informations, reportez-vous au tableau 32-8.
  3. Cliquez sur Exécuter. La page de test affiche les résultats.
  4. Une fois le test terminé, vous pouvez cliquer sur Précédent pour en exécuter un autre. Pour exécuter un nouveau test après avoir modifié la ressource XQuery, vous devez fermer la console de test, puis la rouvrir afin que les modifications prennent effet.
