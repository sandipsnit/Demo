Utilisation de l'éditeur d'expressions XPath

Les expressions XPath permettent de sélectionner un sous-ensemble d'une variable de contexte de message XML. Vous pouvez utiliser la console de test dans l'éditeur d'expressions XPath pour tester la définition de l'expression XPath. Une expression XPath utilise un seul document XML comme entrée et génère une séquence de documents XML et/ou de types de primitive comme sortie.

Le tableau 32-9 décrit les options de configuration des expressions XPath.

Tableau 32-9 Configuration des options XPath 
Section
Description
Nom
Nom de l'expression testée.
Actions de la console de test
Exécuter
Applique la transformation.
Réinitialiser
Réinitialise le champ d'entrée.
Fermer
Annule l'opération en cours.
Variables
Cette section contient un seul champ d'entrée correspondant au document XML par rapport auquel l'expression XPath est testée.
 
L'entrée XML peut être basée sur un fichier ou sur un texte. Le référencement d'un fichier pour entrée est prioritaire sur l'entrée textuelle. Sélectionnez le fichier à utiliser dans votre test.

  1. Accédez à la console de test lors de la modification d'une action dans le flux de messages d'un pipeline. Pour accéder à l'éditeur d'expressions XPath, reportez-vous à Création et modification d'expressions XPath et XQuery imbriquées.
  2. Configurez les données de test de l'expression XPath dans la section Variables de la page. Pour plus d'informations, reportez-vous au tableau 32-9.
  3. Cliquez sur Exécuter. La page de test affiche les résultats.
  4. Une fois le test terminé, vous pouvez cliquer sur le bouton Précédent pour en exécuter un autre. Pour exécuter un nouveau test après avoir modifié l'expression XPath, vous devez fermer la console de test, puis la rouvrir afin que les modifications prennent effet.
