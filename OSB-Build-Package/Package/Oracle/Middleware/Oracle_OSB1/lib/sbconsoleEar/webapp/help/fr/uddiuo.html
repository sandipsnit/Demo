Ajout de registres UDDI

Utilisez cette page pour ajouter et configurer des registres UDDI. Une fois les registres configurés, vous pouvez y publier les services proxy Oracle Service Bus ou importer les services fonctionnels à utiliser dans un service proxy Oracle Service Bus.

Pour ajouter un registre, votre session doit être active.

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Sélectionnez Administration du système > Registres UDDI.
  3. Dans le panneau Registres UDDI, cliquez sur Ajouter un registre.
  4. Pour configurer le registre, suivez la procédure suivante. Les astérisques indiquent les champs obligatoires. Pour connaître la description des propriétés à définir lors de l'ajout d'un registre, reportez-vous au tableau 29-1.
    1. Dans le champ Nom, entrez un nom pour le registre.

      Pour obtenir des conseils sur la dénomination, suivez les restrictions concernant la dénomination des ressources.
    2. Dans le champ URL d'interrogation, saisissez une URL d'interrogation au format suivant : http://host:port/APPLICATION_SERVER_CONTEXT/uddi/inquiry.
    3. Dans le champ URL de publication, saisissez une URL de publication au format suivant : http://host:port/APPLICATION_SERVER_CONTEXT/uddi/publishing.
    4. Dans le champ URL de sécurité, saisissez une URL de sécurité au format suivant :http://host:port/APPLICATION_SERVER_CONTEXT/uddi/security.
    5. Dans le champ URL d'abonnement, saisissez une URL d'abonnement au format suivant :
      http://host:port/APPLICATION_SERVER_CONTEXT/uddi/subscription.

      6. Par défaut, la valeur de APPLICATION_SERVER_CONTEXT dans le programme d'installation est registry.

    6. Dans le champ Nom utilisateur, saisissez votre nom utilisateur pour vous connecter à la console du registre.
    7. Dans le champ Mot de passe (et Confirmer le mot de passe), saisissez votre mot de passe pour vous connecter à la console du registre.
    8. Cochez la case Charger les tModels dans le registre pour publier les tModels Oracle Service Bus dans ce registre.

      9. Ce champ n'est obligatoire que si vous publiez des services proxy dans ce registre.

    9. Cochez la case Activer l'import automatique pour synchroniser automatiquement les services importés avec le registre UDDI. Les services importés lorsque cette option est sélectionnée sont synchronisés avec le registre UDDI.
      10. Remarque : La synchronisation automatique est un processus en arrière-plan ; vous ne pouvez pas l'annuler à l'aide de la fonction Annuler de la session. L'annulation d'une modification de synchronisation automatique n'est pas définitive car le service sera resynchronisé lors du prochain cycle de synchronisation. Si vous souhaitez qu'un service importé ne soit jamais synchronisé avec le registre UDDI, vous devez détacher le service afin d'éviter d'autres mises à jour à partir du registre. Reportez-vous à Détachement de services.
  5. Cliquez sur Valider pour valider les URL fournies.
  6. Cliquez sur Enregistrer pour configurer le registre avec les paramètres fournis et l'enregistrer dans la session en cours.
  7. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
