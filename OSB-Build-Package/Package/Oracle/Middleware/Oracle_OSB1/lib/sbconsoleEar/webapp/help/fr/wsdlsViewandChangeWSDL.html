Modification de ressources WSDL

  1. Si vous ne l'avez pas déjà fait, cliquez sur Créer afin de créer une session ou cliquez sur Modifier pour accéder à une session existante. Reportez-vous à Utilisation du centre des modifications.
  2. Localisez la ressource WSDL, comme décrit dans Localisation de WSDL.
  3. Cliquez sur le nom de la ressource WSDL. La page Visualiser les détails du WSDL affiche les informations répertoriées dans les tableaux 6-2, 6-3 et 6-4.

    4. Tableau 6-2 Détails des ressources WSDL 
    Propriété
    Description
    Dernière modification par
    Utilisateur ayant créé la ressource WSDL ou l'ayant importée dans la configuration.
    Dernière modification le
    Date et heure auxquelles l'utilisateur a créé la ressource WSDL ou l'a importée dans la configuration. Cliquez sur le lien de date et d'heure pour visualiser l'historique des modifications apportées à cette ressource. Reportez-vous à Page Visualiser l'historique des modifications.
    Références
    Nombre d'objets référencés par la ressource WSDL. Lorsque des références existent, cliquez sur le lien numérique pour visualiser la liste des objets en question. Reportez-vous à Visualisation des références aux ressources.
    Référencé par
    Nombre d'objets référençant la ressource WSDL. Lorsque des références existent, cliquez sur le lien numérique pour visualiser la liste des objets en question. Par exemple, si vous sélectionnez ce port ou cette liaison de ressources WSDL comme type de service d'un service fonctionnel ou proxy spécifique, ce dernier est répertorié comme référence lorsque vous cliquez sur ce lien. Reportez-vous à Visualisation des références aux ressources.
    Description
    Description de la ressource WSDL, le cas échéant.

    La vue structurelle de la ressource WSDL est affichée par défaut. Elle comprend les informations répertoriées dans le tableau 6-3.

    Tableau 6-3 Détails de la structure WSDL 
    Propriété
    Description
    Espace de noms cible
    Espace de noms utilisé pour qualifier les définitions incluses dans la ressource WSDL.
    Définitions WSDL
    Les attributs et groupes associés à la ressource WSDL sont les suivants :
    • Types de port WSDL
    • Liaisons WSDL
    • Ports WSDL
    • Types de schéma XML
    • Eléments de schéma XML
    • Imports WSDL
    • Imports du schéma XML
    • Références de stratégie WS
    • Références de stratégie WS implicites
    Etat WSDL
    Statut de la ressource WSDL :
    • Valide

      • Tous les emplacements des schémas XML ou des ressources WSDL inclus dans la ressource WSDL en cours sont indiqués et valides. En outre, tous les emplacements des schémas XML ou des ressources WSDL imbriqués (schémas XML ou ressources WSDL inclus par la suite dans les schémas XML ou ressources WSDL inclus) sont spécifiés et valides.

    • Non valide

      • Un des emplacements des schémas XML ou des ressources WSDL inclus dans la ressource WSDL en cours n'est pas indiqué ou n'est pas valide. En outre, il se peut que l'emplacement d'un des schémas XML ou d'une des ressources WSDL imbriqués (schémas XML ou ressources WSDL inclus par la suite dans les schémas XML ou les ressources WSDL inclus) ne soit pas spécifié ou valide. Pour résoudre un schéma XML ou une ressource WSDL non valide, cliquez sur Modifier les références. Reportez-vous à Résolution des références WSDL non résolues.

  4. Cliquez sur Vue textuelle pour afficher une vue textuelle des détails de la ressource WSDL. La vue textuelle comprend les informations répertoriées dans le tableau 6-4.

    5. Tableau 6-4 Détails textuels des ressources WSDL 
    Propriété
    Description
    Espace de noms cible
    Espace de noms utilisé pour qualifier les définitions incluses dans la ressource WSDL.
    Texte
    Texte de la ressource WSDL.

  5. Pour modifier les champs, cliquez sur Modifier. Pour obtenir la description de ces champs, reportez-vous à Ajout de ressources WSDL.
  6. Effectuez les modifications souhaitées.
  7. Cliquez sur Enregistrer pour valider les mises à jour dans la session en cours.
  8. Pour terminer la session et déployer la configuration lors de l'exécution, cliquez sur Activer sous Centre des modifications.
Remarque : La page Visualiser les détails du WSDL peut également afficher le bouton Séparer les callbacks. Ce bouton n'apparaît que si la ressource WSDL contient des types de port avec des opérations de callback pouvant être réécrites. Si vous cliquez sur ce bouton, la ressource WSDL est réécrite.
