Split-Joins

Split-Joins let you send message invocations in parallel (in addition to sequentially) and to aggregate the responses. To learn more about Split-Joins, see "Improving Service Performance with Split-Join" in the Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus at http://www.oracle.com/pls/as1111/lookup?id=OSBAG291. See also, "Working with Split-Join" at http://www.oracle.com/pls/as1111/lookup?id=OSBDV193 and the "Split-Join User Interface Reference" at http://www.oracle.com/pls/as1111/lookup?id=OSBDV573 in the Oracle Fusion Middleware Developer's Guide for Oracle Service Bus.

You access Split-Joins through Flow transport-typed business services. For more information, see the Flow Transport Configuration page.
