Pagina Configurazione trasporto BPEL-10g

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio business che utilizza il protocollo di trasporto BPEL-10g (Oracle BPEL Process Manager). Per ulteriori informazioni sull'utilizzo di Oracle Service Bus con Oracle BPEL Process Manager, vedere "Oracle BPEL Process Manager Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1147.

Proprietà
Descrizione
Ruolo
Il trasporto BPEL viene utilizzato per inviare messaggi di richiesta da Oracle Service Bus a Oracle BPEL Process Manager. Il servizio business può assumere uno dei ruoli riportati di seguito.
  • Client sincrono
    Per la comunicazione sincrona con un client Oracle Service Bus, l'unica informazione di posizione necessaria è l'indirizzo BPEL. Questo indirizzo viene acquisito in modo statico come l'URI endpoint e/o in modo dinamico mediante la riscrittura dell'URI.
  • Client asincrono
    Per la comunicazione asincrona con un client Oracle Service Bus, un callback da Oracle BPEL Process Manager a Oracle Service Bus viene inviato su una connessione diversa da quella della richiesta ed è necessario che Oracle Service Bus sia configurato per fornire l'indirizzo di callback corretto. Per ulteriori informazioni, vedere "Advanced Use Cases (Asynchronous)" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
  • Callback servizio
    Se il servizio business viene concepito come callback di servizio per Oracle BPEL Process Manager (in cui Oracle BPEL Process Manager richiama un servizio esterno attraverso Oracle Service Bus), l'indirizzo di callback sarà noto solo in fase di esecuzione. Utilizzare un URI endpoint di bpel://callback.

    • Se si configura il servizio business con l'URI di contrassegno, configurare la logica della pipeline in modo da impostare dinamicamente l'URI su $outbound; ad esempio, utilizzando l'azione TransportHeader.

    Nota: un servizio business di tipo Callback servizio non supporta il bilanciamento del carico o il failover.
Proxy di callback
Questo campo facoltativo è disponibile solo per il ruolo Client asincrono e permette di selezionare il servizio proxy (deve essere un proxy SB o HTTP di tipo Qualsiasi servizio SOAP) che verrà utilizzato per instradare i callback al client Oracle Service Bus che ha effettuato la richiesta. Per ulteriori informazioni, vedere "Advanced Use Cases (Asynchronous)" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
Account di servizio
Per la sicurezza del contesto JNDI, utilizzato per accedere al servizio di consegna di Oracle BPEL Process Manager. Fare clic su Sfoglia e selezionare un account di servizio. Se non viene specificato un account di servizio, viene utilizzato un oggetto anonimo.
Non vi sono limitazioni riguardanti il tipo di account di servizio che è possibile configurare, come statico o pass-through, ma per accedere alla fase di esecuzione sono necessari un nome utente e una password.
Sospendi transazione
Selezionando Sospendi transazione il servizio business diventa non transazionale anche quando viene richiamato da una transazione.
Quando non si seleziona Sospendi transazione:
  • se il protocollo indica un protocollo supportato da WebLogic Server (t3, iiop, http), la transazione viene propagata;
  • se il protocollo indica un server OC4J (ormi, opmn), il trasporto BPEL restituisce un'eccezione perché OC4J non supporta la propagazione della transazione.
Il trasporto BPEL ignora l'opzione Sospendi transazione nelle situazioni seguenti:
  • Il servizio business viene chiamato con qualità di servizio impostata su "Miglior tentativo". Il trasporto BPEL sospende automaticamente qualsiasi transazione che non supporta la qualità di servizio.
  • Il servizio business viene chiamato con qualità di servizio impostata su "Esattamente una volta" e non vi sono transazioni.
Per una descrizione della propagazione delle transizioni, vedere "Transaction Propagation" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1151.
Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager, viene utilizzato quello predefinito.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
