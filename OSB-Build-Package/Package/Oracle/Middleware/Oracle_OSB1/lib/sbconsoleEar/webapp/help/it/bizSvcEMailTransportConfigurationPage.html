Pagina Configurazione trasporto posta elettronica

La pagina Crea/Modifica un servizio business - Configurazione trasporto posta elettronica fa parte di una serie di pagine per la creazione e la modifica di servizi business, come descritto in Creazione e configurazione dei servizi business e Modifica delle configurazioni dei servizi business.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio business che utilizza il protocollo di trasporto della posta elettronica. Nella Tabella 18-6 viene descritto come utilizzare questa pagina.

Tabella 18-6 Pagina Crea/Modifica un servizio business - Configurazione trasporto posta elettronica
Opzione
Per creare o modificare...
SSL obbligatorio
Selezionare questa opzione per comunicare mediante SSL (Secure Sockets Layer). Il numero di porta SSL predefinito per SMTP è 465.
Note:
  • affinché SSL possa funzionare, il certificato del server di posta elettronica deve essere presente nel truststore di Oracle Service Bus;
  • se si utilizza TLS, è necessario collocare il certificato protetto nel truststore predefinito ($JAVA_HOME/jre/lib/security/cacerts) oppure in un truststore qualsiasi e aggiungere il percorso nel file $DOMAIN_HOME/bin/setDomainEnv.*, come indicato di seguito:

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<percorso del truststore>

Server SMTP
Selezionare il server SMTP predefinito da utilizzare per le voci dell'URI endpoint di name@domain_name.com. Se si forniscono i parametri del server SMTP nell'URI endpoint, come descritto nella Tabella 18-3, le risorse di questo server verranno utilizzate al posto di questa impostazione Server SMTP.
Non selezionare un server SMTP se si utilizza l'opzione Sessione posta elettronica.
Per ulteriori informazioni sulla creazione delle risorse SMTP, vedere Aggiunta di server SMTP.
Sessione di posta elettronica
Immettere il nome JNDI della sessione di posta elettronica configurata da utilizzare per le voci dell'URI endpoint di name@domain_name.com. Se si forniscono i parametri della sessione di posta elettronica JNDI nell'URI endpoint, come descritto nella Tabella 18-3, tali sessioni di posta elettronica verranno utilizzate al posto di questa impostazione Sessione di posta elettronica.
Non immettere una sessione di posta elettronica se si utilizza l'opzione Server SMTP.
Nome mittente
Immettere un nome di visualizzazione per l'account di posta elettronica di origine per questo servizio.
Indirizzo del mittente
Immettere l'account di posta elettronica di origine per questo servizio.
Nome indirizzo di risposta
Immettere un nome di visualizzazione per l'account di posta elettronica della risposta.
Indirizzo di risposta
Immettere un indirizzo di posta elettronica al quale rispondere.
Timeout connessione
Immettere l'intervallo di timeout, in secondi, prima che la connessione venga interrotta. Se si immette 0 non c'è timeout.
Codifica richiesta
Accettare l'impostazione predefinita ISO-8859-1 come codifica del set di caratteri per le richieste nei trasporti di posta elettronica oppure immettere un'altra codifica del set di caratteri.

Al termine della procedura

Fare clic su Avanti per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare questa configurazione nella pagina Crea/Modifica un servizio business - Riepilogo.
