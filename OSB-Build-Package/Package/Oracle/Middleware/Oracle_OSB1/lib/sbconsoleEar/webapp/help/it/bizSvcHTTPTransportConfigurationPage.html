Pagina Configurazione trasporto HTTP

Il trasporto HTTP supporta entrambi gli endpoint HTTP e HTTPS.

La pagina Crea/Modifica un servizio business - Configurazione trasporto HTTP fa parte di una serie di pagine per la creazione e la modifica di servizi business, come descritto in Creazione e configurazione dei servizi business e Modifica delle configurazioni dei servizi business.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio business che utilizza il protocollo di trasporto HTTP. Nella Tabella 18-10 viene descritto come utilizzare questa pagina.

Metodo di richiesta HTTP
Questo parametro permette di utilizzare in una richiesta uno dei metodi HTTP riportati di seguito.
  • POST: passa tutti i relativi dati, di lunghezza illimitata, direttamente sulla connessione socket come parte del relativo corpo della richiesta HTTP. Lo scambio non è visibile al client e l'URL non viene modificato. Per le richieste basate su REST, POST indica al trasporto di eseguire un'operazione di creazione/sostituzione o di eseguire un'azione con la richiesta.
  • GET: è possibile includere come parte della richiesta alcune delle relative informazioni per descrivere meglio cosa acquisire. Queste informazioni vengono passate come una sequenza di caratteri aggiunti all'URL della richiesta in una stringa di query. È possibile utilizzare GET in un servizio business con Tipo di servizio impostato su "Qualsiasi servizio XML" oppure con Tipo di servizio impostato su "Servizio messaggistica" quando Tipo di messaggio di richiesta è impostato su "Nessuno". Per le richieste basate su REST, GET recupera una rappresentazione di una risorsa remota.
  • PUT: è possibile utilizzare PUT in un servizio business con Tipo di servizio impostato su "Qualsiasi servizio XML" o "Servizio messaggistica". PUT indica al trasporto di eseguire un'operazione di creazione/sostituzione con una richiesta basata su REST, come il caricamento di un file in una posizione conosciuta.
  • HEAD: è possibile utilizzare HEAD in un servizio business con Tipo di servizio impostato su "Qualsiasi servizio XML" o con Tipo di servizio impostato su "Servizio messaggistica" quando Tipo di messaggio di risposta è impostato su "Nessuno". HEAD indica al trasporto di acquisire le informazioni dell'intestazione da una risorsa remota piuttosto che acquisire una rappresentazione completa della risorsa in una richiesta basata su REST.
  • DELETE: è possibile utilizzare PUT in un servizio business in cui Tipo di servizio è impostato su "Qualsiasi servizio XML" o "Servizio messaggistica". DELETE indica al trasporto di eseguire un'operazione di eliminazione con una richiesta basata su REST.
Nota: se un metodo è già impostato nella variabile $outbound/transport/request/http:http-method, tale valore ha la precedenza su qualsiasi altro metodo selezionato per Metodo di richiesta HTTP.
Autenticazione
Selezionare una delle opzioni riportate di seguito.
  • Nessuna: specifica che per accedere a questo servizio l'autenticazione non è necessaria.
  • Base: specifica che per accedere a questo servizio è richiesta l'autenticazione base.

    • L'autenticazione di base indica a WebLogic Server di autenticare il client utilizzando un nome utente e password in base ai provider di autenticazione configurati nel realm di sicurezza, ad esempio il servizio di directory LDAP (Lightweight Directory Access Protocol) e Windows Active Directory. Il client deve inviare il nome utente e la password corrispondenti nell'intestazione della richiesta HTTP.

    L'autenticazione di base è sconsigliata su HTTP poiché la password viene inviata con testo in chiaro. Tuttavia, l'invio delle password su HTTPS è sicuro poiché HTTPS fornisce un canale crittografato.

    Avvertenza: per impostazione predefinita, tutti gli utenti (autorizzati e anonimi) possono accedere a un servizio business. Per limitare gli utenti che possono accedere a un servizio business, creare un criterio di autorizzazione a livello di trasporto. Vedere Modifica dei criteri di accesso a livello di trasporto.

  • Certificato client: specifica comunicazione cifrata e autenticazione client robusta (SSL bidirezionale). Per ulteriori informazioni, vedere "Configuring Transport-Level Security" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
Account di servizio
Immettere un account di servizio, ovvero una risorsa alias per un nome utente e password. Questo campo è obbligatorio se è stato selezionato il campo Richiesta autenticazione base.

Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager, viene utilizzato quello predefinito. Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codifica richiesta
Accettare l'impostazione predefinita iso-8859-1 come codifica del set di caratteri per le richieste nei trasporti HTTP oppure immettere un'altra codifica del set di caratteri.
Codifica risposta
Accettare l'impostazione predefinita iso-8859-1 come codifica del set di caratteri per le risposte nei trasporti HTTP oppure immettere un'altra codifica del set di caratteri.
Server proxy
Immettere una risorsa di server proxy oppure fare clic su Sfoglia per scegliere una voce dalla lista delle risorse di server proxy configurate.
Segui reindirizzamenti HTTP
Selezionare questa casella di controllo per specificare che è necessario seguire automaticamente i reindirizzamenti HTTP (ovvero richieste con un codice di risposta 3xx). Un reindirizzamento si verifica quando si invia una richiesta in uscita all'URL di un servizio business e questo servizio restituisce un codice di risposta (ad esempio, 302) per indicare che l'URL non è più valido ed è necessario inviare la richiesta a un altro URL.
Quando si seleziona la casella di controllo Segui reindirizzamenti HTTP, Oracle Service Bus automaticamente invia di nuovo la richiesta al nuovo URL senza alcuna azione da parte dell'utente. Deselezionare questa casella di controllo se non si desidera che i reindirizzamenti HTTP vengano seguiti automaticamente.
Utilizza modalità di streaming in chunk
Selezionare questa opzione se si desidera utilizzare la codifica di trasferimento in chunk HTTP per l'invio dei messaggi.
Nota: non utilizzare lo streaming in chunk se si utilizza l'opzione Segui reindirizzamenti HTTP. Il reindirizzamento e l'autenticazione non possono essere gestiti automaticamente in modalità in chunk.
Persistenza sessione
Selezionare questa opzione se si desidera utilizzare la persistenza della sessione (nota anche affinità tra sessioni) per le richieste HTTP per il servizio business.
Rendi persistente nome SessionID
Se l'opzione Persistenza sessione è abilitata, immettere un nome identificativo per la sessione.
