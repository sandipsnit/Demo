Pagina Configurazione trasporto JCA

Utilizzare questa pagina per configurare le impostazioni di trasporto mediante il protocollo di trasporto JCA. Per ulteriori informazioni sull'uso del trasporto JCA, vedere "JCA Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV910 e Creazione di un servizio business da una risorsa di associazione JCA.

Tabella 18-11 Configurazione trasporto JCA
Opzione
Descrizione
File JCA
Fare clic su Sfoglia per selezionare un'associazione JCA. L'associazione JCA definisce aspetti diversi del servizio, come i dettagli sull'adattatore utilizzato, un'associazione al WSDL e al file di mapping TopLink o EclipseLink, nonché le proprietà delle specifiche di attivazione/interazione richieste dal servizio.
Dopo avere selezionato un'associazione JCA valida, i restanti campi di configurazione del trasporto diventano disponibili.
Per ulteriori informazioni sulle associazioni JCA, vedere il Capitolo 12, Associazioni JCA.
Nome adattatore
Valore di sola lettura che indica il nome dell'adattatore che verrà utilizzato dal servizio JCA.
Tipo di adattatore
Valore di sola lettura che indica il tipo di adattatore.
Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager viene utilizzato quello predefinito.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Account di servizio JNDI
L'account di servizio JNDI è relativo alla sicurezza del contesto JNDI e viene utilizzato per accedere al Connection Factory gestito dall'adattatore EIS. Fare clic su Sfoglia e selezionare un account di servizio. Se non viene specificato alcun account di servizio, viene utilizzato un oggetto anonimo.
Per i servizi business JCA non vi sono limitazioni riguardanti il tipo di account di servizio JNDI che è possibile configurare, come statico o pass-through, ma per accedere alla fase di esecuzione sono necessari un nome utente e una password. I servizi proxy JCA possono utilizzare solo account di servizio JNDI statici.
Per ulteriori informazioni sugli account di servizio JNDI, vedere "Security" nel capitolo "JCA Transport" di Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV916.
Proprietà endpoint
In questo campo è possibile assegnare valori alle proprietà di endpoint, come i nuovi tentativi per il tipo di adattatore utilizzato dal servizio.
Per una lista delle proprietà dell'endpoint JCA supportate, vedere "Endpoint Properties" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1841.
Proprietà dinamiche endpoint
Questa opzione permette di passare i parametri richiesti ai servizi conformi a JCA. Ad esempio, è possibile utilizzare una proprietà dinamica di endpoint per passare i parametri di query del database a Oracle JCA Adapter for Database.
Per ulteriori informazioni sull'esecuzione di query con parametri, vedere "Oracle JCA Adapter for Database" in Oracle Fusion Middleware User's Guide for Technology Adapters all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=TKADP2117.
Immettere una coppia nome/valore per ciascuna proprietà dinamica dell'endpoint che si desidera fornire. La chiave della proprietà dell'endpoint corrisponde al nome del parametro della query.
Utilizzare sempre la configurazione dal file JCA
Questa opzione determina se le proprietà delle specifiche di attivazione (servizi proxy) e le proprietà delle specifiche di interazione (servizi business) vengono sempre utilizzate dal file JCA o meno.
Se questa opzione è selezionata (impostazione predefinita), il trasporto JCA interagisce con la struttura JCA utilizzando le proprietà delle specifiche di attivazione/interazione nel file JCA.
Se questa opzione è deselezionata, è possibile sostituire le proprietà delle specifiche di attivazione/interazione.
Per l'impatto sulla ridistribuzione derivato dall'utilizzo di questa opzione, vedere "Endpoint Redeployment" nel capitolo "JCA Transport" di Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV940.
Nome operazione
Visualizza un nome di sola lettura dell'operazione WSDL selezionata. Un'operazione può avere proprietà delle specifiche di attivazione/interazione personalizzate, visualizzate nel campo Proprietà delle specifiche di attivazione/interazione.
Proprietà delle specifiche di attivazione/interazione
Proprietà delle specifiche di attivazione è il nome del campo per i servizi proxy; Proprietà delle specifiche di interazione è il nome del campo per i servizi business.
Se il servizio è un servizio in entrata richiamato da un'applicazione EIS, in questo campo vengono visualizzate le proprietà delle specifiche di attivazione per l'operazione in entrata JCA mostrata nel campo Nome operazione.
È possibile sostituire le proprietà delle specifiche di attivazione/interazione se si deseleziona Utilizzare sempre la configurazione dal file JCA.
Nota: per gli adattatori di Oracle Adapter Suite le proprietà delle specifiche di attivazione/interazione vengono visualizzate come proprietà di sola lettura. Questo tipo di adattatori memorizzano la propria configurazione, che deve essere modificata negli strumenti di gestione di Oracle Adapter Suite.
Proprietà di connessione (precedenti)
Per i servizi JCA di versioni precedenti che utilizzano proprietà delle connessioni in modalità non gestita (non valide in questa versione), vedere le opzioni di configurazione delle connessioni all'indirizzo http://download.oracle.com/docs/cd/E13159_01/osb/docs10gr3/jcatransport/transport.html#wp1105451.

Per ulteriori informazioni sulle proprietà dell'endpoint e delle specifiche di attivazione/interazione, vedere Oracle Fusion Middleware User's Guide for Technology Adapters all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=TKADP.
