Pagina Configurazione trasporto JEJB

Utilizzare questa pagina per configurare le impostazioni di trasporto con il protocollo di trasporto JEJB. Per ulteriori informazioni sull'utilizzo del trasporto JEJB, vedere "JEJB Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036.

Tabella 18-12 Configurazione trasporto JEJB
Opzione
Descrizione
Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager viene utilizzato quello predefinito.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Versione specifica EJB
Selezionare la versione EJB dell'interfaccia EJB remota.
Passa XMLBean per valore
Selezionare questa opzione se si desidera che il trasporto generi una rappresentazione XML "inserita in linea" degli argomenti POJO (un XMLObject) i cui parametri è possibile accedere e manipolare con le espressioni XQuery.
Nota: le informazioni sul tipo non sono disponibili in linea per XMLObjects passato per valore. Se si utilizza questa opzione, non è possibile passare XMLObject digitato come argomento in un Java Callout in una pipeline di servizio proxy.
Non selezionare questa opzione se si desidera passare l'argomento POJO per riferimento, metodo che permette inoltre di ottenere prestazioni migliori.
Passa oggetto del chiamante
In alternativa alla selezione di un account di servizio, selezionare questa opzione per far sì che Oracle Service Bus passi l'oggetto autenticato dal servizio proxy quando viene richiamato l'EJB.
Account di servizio
Fare clic su Sfoglia e selezionare un account di servizio JNDI dalla lista visualizzata. Se non viene specificato un account di servizio, viene utilizzato un oggetto anonimo. Per ulteriori informazioni, vedere il Capitolo 16, Account di servizio.
JAR client
Fare clic su Sfoglia e selezionare una risorsa JAR client EJB dalla lista visualizzata. Il JAR client contiene l'interfaccia remota o business per il servizio remoto. Il JAR client è registrato come una risorsa di archivio generica.
Interfaccia home
Solo EJB 2.1: selezionare l'interfaccia EJBHome richiesta dalle opzioni inserite dal JAR.
Interfaccia remota
Solo EJB 2.1: questo campo viene popolato automaticamente in base alla configurazione dell'interfaccia home.
Interfaccia business
Solo EJB 3.0: selezionare l'interfaccia business dal file JAR client che si desidera richiamare.
Spazio di nomi di destinazione
Questo campo viene popolato con le informazioni selezionate dal file JAR.
Metodi
Selezionare i metodi richiesti. Fare clic su + per espandere il metodo; che consente di modificare i valori dei parametri predefiniti.
È possibile modificare il nome dell'operazione predefinito per un determinato metodo. Per impostazione predefinita, il nome dell'operazione è il nome del metodo. Se un EJB contiene metodi con lo stesso nome (sovraccarico), è necessario cambiare i nomi delle operazioni in modo che siano univoci. I WSDL richiedono nomi delle operazioni univoci.

Al termine della procedura

Fare clic su Avanti per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare questa configurazione nella pagina Crea/Modifica un servizio business - Riepilogo.
