Pagina Configurazione trasporto JMS

La pagina Crea/Modifica un servizio business - Configurazione trasporto JMS fa parte di una serie di pagine per la creazione e la modifica di servizi business, come descritto in Creazione e configurazione dei servizi business e Modifica delle configurazioni dei servizi business.

Per ulteriori informazioni, vedere "JMS Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio business che utilizza il protocollo di trasporto JMS. Nella Tabella 18-13 viene descritto come utilizzare questa pagina.

Tabella 18-13 Pagina Crea/Modifica un servizio business - Configurazione trasporto JMS (continua)
Opzione
Per creare o modificare...
Tipo di destinazione
Selezionare un tipo per la destinazione bridge JMS.
  • Coda (per point-to-point)
  • Argomento (per pubblicazione/sottoscrizione)
Tipo di messaggio
Selezionare una delle opzioni riportate di seguito.
  • Byte (per un flusso di byte senza interpretazione)
  • Testo (per i messaggi di testo)
Questa opzione è disabilitata se si seleziona un tipo di messaggio Java per la risposta.
Code risposta
Questa opzione è disponibile solo quando nel campo Tipo di destinazione è selezionata l'opzione Coda.
Selezionare una delle opzioni di risposta riportate di seguito.
  • Nessuna: la risposta non è prevista. Impostare questa opzione per le operazioni unidirezionali.
  • Una per tutti gli URI di richiesta: permette di immettere un singolo URI per gestire la risposta, nonché impostare altri dettagli di configurazione della risposta come la codifica e il timeout e facoltativamente selezionare un account di servizio JMS per il passaggio delle credenziali JMS/JNDI.
  • Una per ogni URI di richiesta: questa opzione fornisce il failover della risposta, permettendo di immettere un URI o una destinazione di risposta per ogni URI di richiesta.
Facoltativamente, è possibile selezionare un account di servizio per le credenziali JMS/JNDI su ciascuna coppia richiesta/risposta.
Pattern di risposta
Questa opzione è disponibile solo quando nel campo Code risposta è selezionata un'opzione di risposta.
Selezionare una delle opzioni riportate di seguito.
  • Selezionare JMSCorrelationID per tutti i servizi tranne che per i servizi JAX-RPC in esecuzione su Oracle WebLogic Server.
  • Selezionare JMSMessageID per i servizi JAX-RPC in esecuzione su Oracle WebLogic Server.
Per ulteriori informazioni, vedere "Message ID and Correlation ID Patterns for JMS Request/Response" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051.
Criterio di spedizione
Selezionare l'istanza del Work Manager di Oracle WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager, viene utilizzato quello predefinito.
Ad esempio, se il servizio business dispone di un protocollo di trasporto JMS, l'endpoint del servizio business è un file JAR MDB (Message-Driven Bean, bean basato su messaggi) che è possibile associare al criterio di spedizione specifico.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codifica richiesta
Immettere il set di caratteri per la codifica delle richieste. L'impostazione predefinita è UTF-8.
Codifica risposta
Questa opzione è disponibile solo quando nel campo Code risposta è selezionata una delle opzioni di risposta.
Immettere il set di caratteri per la codifica delle risposte. L'impostazione predefinita è UTF-8.
Timeout risposta
Questa opzione è disponibile solo quando nel campo Code risposta è selezionata una delle opzioni di risposta.
Immettere il periodo di tempo, in secondi, di attesa della risposta prima dell'interruzione della connessione. Il valore predefinito è zero (0) e indica che la risposta è senza timeout.
JAR client
Questa opzione è disponibile quando il servizio è un servizio di messaggistica con un tipo di risposta Java. Selezionare il file JAR client da utilizzare per la rimozione dalla coda dei messaggi che contengono oggetti Java. La selezione del file JAR client assicura che si trovi nel classpath.

Per ulteriori informazioni, vedere "Sending and Receiving Java Objects in Messages" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
URI risposta
Questa opzione è disponibile quando si seleziona l'opzione di risposta Una per tutti gli URI di richiesta e il pattern di risposta JMSCorrelationID.
Immettere un URI di risposta nel formato:
jms://host:port/connection_factory/jndi_destination
Per specificare la destinazione di più server, utilizzare il formato seguente:
jms://host1:port,host2:port/connection_factory/jndi_destination
Nell'URI di risposta è inoltre possibile omettere l'host e la porta. Ad esempio:
jms:///connection_factory/jndi_destination

Quando si omettono l'host e la porta si verifica la ricerca di Connection Factory/destinazione sul server locale. Ciò è utile, ad esempio, se l'URI di richiesta è diretto a un Connection Factory esterno o a una destinazione esterna, ma si desidera che la risposta sia inviata al server locale.

Nota: anche se Oracle WebLogic Server consente l'utilizzo delle barre nei nomi JNDI, ad esempio "myqueues/myqueue", i nomi JNDI contenenti barre interferiscono con il formato URI richiesto da Oracle Service Bus e non sono pertanto utilizzabili. Una soluzione alternativa a questo problema consiste nel definire un server JMS esterno e farvi riferimento nell'URI. Per ulteriori informazioni, vedere "Configure foreign servers" in Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Risposte richiesta
Questa opzione è disponibile quando si seleziona l'opzione di risposta Una per ogni URI di richiesta per il pattern JMSCorrelationID per fornire il failover della risposta.
Per ogni URI di richiesta immesso nella pagina di trasporto generica immettere un URI di risposta, come descritto per il campo URI risposta precedente.
È possibile selezionare un account di servizio facoltativo per le credenziali JMS/JNDI che il servizio utilizza per entrambe le code di richiesta e di risposta.
Destinazione - Risposte
Questa opzione è disponibile quando si seleziona l'opzione di risposta Una per tutti gli URI di richiesta per il pattern JMSMessageID.
Immettere il nome del server di destinazione che riceverà le risposte e immettere un URI di risposta, come descritto per il campo URI risposta.
Connessioni richiesta
Questa opzione è disponibile quando si seleziona l'opzione di risposta Una per ogni URI di richiesta per il pattern JMSMessageID per fornire il failover della risposta.
Per ogni URI di richiesta, identificato sequenzialmente mediante numero nel campo N. di sequenza, inserire facoltativamente un nome di Connection Factory JMS. Se non si immette un nome, il trasporto JMS utilizza il Connection Factory dall'URI di richiesta.
È possibile selezionare un account di servizio facoltativo per le credenziali JMS/JNDI che il servizio utilizza per entrambe le code di richiesta e di risposta.
Destinazione - Destinazioni
Questa opzione è disponibile quando si seleziona l'opzione di risposta Una per ogni URI di richiesta per il pattern JMSMessageID per fornire il failover della risposta. Utilizzare questo campo insieme a Connessioni richiesta.
Ogni server di destinazione elencato (determinato dai server nel dominio corrente), come i server gestiti in un cluster, elenca gli URI di richiesta in base al numero di sequenza, che corrispondono a quelli presenti nel campo Connessioni richiesta. Per ogni URI di richiesta su ciascuna destinazione, immettere la coda di destinazione sul server di destinazione che riceve le risposte.
Nota: l'ambiente di sviluppo di Oracle Service Bus supporta solo un ambiente con un unico server, dunque viene elencata una sola destinazione. Per configurare questo campo in un ambiente con più server, distribuire questo servizio all'ambiente in fase di esecuzione e completare la configurazione del servizio nella console di amministrazione di Oracle Service Bus.
Account di servizio JMS
Questa opzione è disponibile quando si seleziona l'opzione Nessuna o Una per tutti gli URI di richiesta nel campo Code risposta.
Selezionare un account di servizio da utilizzare per la risorsa JMS gestita dal server JMS. Un account di servizio è una risorsa alias per un ID utente e una password, utilizzata sia per la richiesta, sia per la risposta. Lo stesso account di servizio viene utilizzato per entrambi gli scopi JMS e JNDI.
Questo campo è ad esclusione reciproca con l'opzione Passa oggetto del chiamante. Utilizzare uno o l'altra per l'autenticazione JMS/JNDI.
Per ulteriori informazioni, vedere Account di servizio.
Usa SSL
Selezionare questa opzione solo se le richieste vengono effettuate su una connessione TLS/SSL.
TLS/SSL (Secure Sockets Layer) fornisce connessioni sicure mediante l'autenticazione reciproca di due applicazioni che si connettono su una rete e la codifica dei dati scambiati tra le applicazioni. L'autenticazione permette a un server, e facoltativamente a un client, di verificare l'identità dell'applicazione all'altra estremità di una connessione di rete. Se inoltre l'amministratore ha limitato l'accesso a singole destinazioni JMS (code o argomenti) mediante l'impostazione del controllo degli accessi sulla voce JNDI per la destinazione, è necessario che il servizio effettui l'autenticazione quando ricerca la voce nella struttura JNDI. Effettuare l'autenticazione utilizzando un account di servizio o l'opzione Passa soggetto del chiamante.
Scadenza
L'intervallo di tempo in millisecondi dopo il quale si verifica la scadenza del messaggio. Il valore predefinito è 0 e indica che il messaggio non ha scadenza.
Abilita persistenza messaggio
Modalità di consegna JMS utilizza dal servizio, che permette di bilanciare affidabilità e throughput. Selezionare questa opzione (impostazione predefinita) per una consegna garantita dei messaggi. Deselezionare questa opzione per migliore il throughput se la perdita occasionale di un messaggio è tollerata.
Unità di ordine
Immettere un'unità di ordine dei messaggi. L'unità di ordine dei messaggi permette ai producer di messaggi di raggruppare i messaggi in una unità singola rispetto all'ordine di elaborazione. Questa unità di ordine singola richiede che tutti i relativi messaggi siano elaborati sequenzialmente nell'ordine in cui sono stati creati.
Passa oggetto del chiamante
Selezionare questa casella di controllo per far sì che Oracle Service Bus passi l'oggetto autenticato quando viene inviato un messaggio.
Quando si abilita questo campo e il servizio business destina le risorse JMS a un dominio diverso, abilitare l'accettazione sicura globale su entrambi i domini. Vedere Configuring Security for a WebLogic Domain in Securing Oracle WebLogic Server.
Questo campo è ad esclusione reciproca con l'opzione Account di servizio. Utilizzare uno o l'altra per l'autenticazione JMS/JNDI.
Quando si abilita questo campo e il servizio business destina le risorse JMS a un dominio diverso, abilitare l'accettazione sicura globale su entrambi i domini. Vedere "Configuring Security for a WebLogic Domain" in Oracle Fusion Middleware Securing Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=SECMG402.
Timeout JNDI
Timeout (in secondi) della connessione JNDI utilizzato durante la ricerca della destinazione o del Connection Factory nella struttura JNDI.
Il valore predefinito è zero (0) e indica che la connessione non ha scadenza.
