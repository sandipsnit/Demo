Pagina Configurazione trasporto MQ

La pagina Crea/Modifica un servizio business - Configurazione trasporto MQ fa parte di una serie di pagine per la creazione e la modifica di servizi business, come descritto in Creazione e configurazione dei servizi business e Modifica delle configurazioni dei servizi business.

Per ulteriori informazioni, vedere "MQ Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Prima di iniziare

Configurare una risorsa di connessione MQ. Vedere Connessioni MQ.

Per configurare il trasporto MQ effettuare le operazioni riportate di seguito.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio business che utilizza il protocollo di trasporto MQ nativo. Nella Tabella 18-14 viene descritto come utilizzare questa pagina.

Tabella 18-14 Pagina Crea/Modifica un servizio business - Configurazione trasporto MQ
Opzione
Per creare o modificare...
Tipo di messaggio
Selezionare una delle opzioni riportate di seguito.
  • Byte (per un flusso di byte senza interpretazione)
  • Testo (per i messaggi di testo)
Risposta richiesta
Selezionare questa opzione per specificare che è prevista una risposta dopo l'invio di un messaggio in uscita.
Intervallo di polling
Questa opzione è disponibile solo quando la casella di controllo Risposta richiesta è selezionata.
Immettere un intervallo di polling, in millisecondi. L'impostazione predefinita è 1000.
Pattern di correlazione risposte
Questa opzione è disponibile solo quando la casella di controllo Risposta richiesta è selezionata.
Specificare se il pattern di correlazione delle risposte deve essere basato su:
  • MessageID
  • CorrelationID
  • Coda dinamica: selezionare questa opzione se l'implementazione MQ WebSphere utilizza le code dinamiche per la correlazione delle risposte. Il trasporto MQ supporta solo le code dinamiche temporanee.
Genera automaticamente valore di correlazione
Questa opzione è disponibile solo quando la casella di controllo Risposta richiesta è selezionata.
Selezionare questa casella di controllo per generare in modo automatico un CorrelationID o un MessageID.
Coda modelli
Solo per il pattern di correlazione delle risposte delle code dinamiche. Immettere il nome della coda dei modelli per generare la coda dinamica.
URI risposta MQ
Questa opzione è disponibile solo quando la casella di controllo Risposta richiesta è selezionata.
Permette di specificare la destinazione in cui la risposta verrà pubblicata. Immettere una risposta URI nello stesso formato dell'URI endpoint.
mq://local_queue_name?conn=mq_connection_resource
Oppure in questo formato se si stanno utilizzando le code dinamiche.
mq://dynamic_queue_prefix?conn=mq_connection_resource
dynamic_queue_prefix, limitato a 32 caratteri, viene utilizzato per creare la coda dinamica sul server MQ. Il nome della coda è determinato dal prefisso e da un ID univoco. Ad esempio, se dynamic_queue_prefix è esempio, il nome coda dinamica potrebbe essere esempio123129083821.
È inoltre possibile utilizzare un asterisco (*) come carattere jolly nell'URI di risposta della coda dinamica. Ad esempio:
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
Se nell'URI non viene fornito un dynamic_queue_prefix, il trasporto utilizza il nome della coda dinamica generato dal server MQ. Se nell'URI non viene fornita una mq_connection_resource esplicita (consigliato), il trasporto utilizza la mq_connection_resource dall'URI endpoint.
Per ulteriori informazioni, vedere "MQ Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.
Timeout risposta
Questa opzione è disponibile solo quando la casella di controllo Risposta richiesta è selezionata.
Immettere il numero di secondi di attesa della risposta prima dell'interruzione della connessione.
Criterio di spedizione
Il criterio di spedizione indica all'istanza il WLS Work Manager che si desidera utilizzare per l'endpoint del servizio.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Polling coda dinamica
Solo per il pattern di correlazione delle risposte delle code dinamiche. Selezionare questa opzione se si desidera che il servizio utilizzi le connessioni incluse nel pool nelle code dinamiche.
Se si desidera utilizzare una pool di connessioni distinto per le code dinamiche, considerare la configurazione di una risorsa Connessione MQ dedicata per le code dinamiche.
Non selezionare questa opzione se si desidera creare una nuova istanza di coda dinamica su ciascuna richiesta (ed eliminare la coda dopo la risposta).
Opzioni 'PUT' per l'URI dell'endpoint
Immettere le opzioni di messaggio PUT MQ tra quelle riportate di seguito.
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
È possibile utilizzare "|" o "+" per separare più opzioni. Ad esempio, è possibile specificare quanto segue.
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
Le opzioni di messaggio PUT MQ vengono applicate quando il messaggio viene posto nella coda in uscita.
URI risposta non riconosciuta MQ
Immettere l'URI che rappresenta la coda alla quale inviare i messaggi di risposta non riconosciuti. Notare che questa impostazione è abilitata solo quando la casella di controllo Genera automaticamente valore di correlazione è selezionata.
Se non si specifica un valore per questo campo, i messaggi di risposta non riconosciuti vengono eliminati.
Elabora intestazioni RFH2
Selezionare questa opzione per analizzare le intestazioni RFH2 MQ WebSphere del payload di un messaggio e generare in modo automatico un'intestazione di trasporto RFH2Headers contenente i dati RFH2.
Se non si seleziona questa opzione, il payload viene passato così come viene ricevuto.

Per informazioni sul modo in cui il trasporto MQ gestisce le intestazioni RFH2, vedere "About RFH2 Headers" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
