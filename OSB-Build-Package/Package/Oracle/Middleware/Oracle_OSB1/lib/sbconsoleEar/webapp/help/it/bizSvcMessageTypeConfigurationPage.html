Pagina Configurazione tipo di messaggio

La pagina Crea/Modifica un servizio business - Configurazione tipo di messaggio fa parte di una serie di pagine per la creazione e la modifica di servizi business, come descritto in Creazione e configurazione dei servizi business e Modifica delle configurazioni dei servizi business.

Utilizzare questa pagina per configurare i tipi di messaggio per un servizio business di tipo Servizio messaggistica.

La definizione di associazione per i servizi di messaggistica è costituita dalla configurazione del tipo di contesto dei messaggi scambiati. Non è necessario che il tipo di contesto per la risposta sia lo stesso per la richiesta; pertanto la risposta viene configurata separatamente (ad esempio, il servizio potrebbe accettare un messaggio MFL e restituire una ricevuta di attestazione XML).

Nota: i servizi business di trasporto posta elettronica, file, FTP o SFTP di tipo Servizio messaggistica supportano solo la messaggistica unidirezionale; Tipo di messaggio di risposta dovrebbe essere impostato su Nessuno. Se si seleziona un'opzione diversa da Nessuno, il protocollo file, ftp o sftp non sarà disponibile nella pagina Configurazione trasporto.

Nella Tabella 18-2 viene descritto come utilizzare la pagina Crea/Modifica un servizio business - Configurazione tipo di messaggio.

Tabella 18-2 Pagina Crea/Modifica un servizio business - Configurazione tipo di messaggio
Opzione
Per creare o modificare...
Tipo di messaggio di richiesta
Selezionare un tipo di messaggio per il messaggio di richiesta.
  • Nessuno: selezionare questa opzione quando non vi sono messaggi di richiesta (esempio di GET HTTP).
  • Binario: selezionare questa opzione se il tipo di contenuto del messaggio è sconosciuto o non è importante.
  • Testo: selezionare questa opzione se il messaggio può essere limitato al testo.
  • MFL: selezionare questa opzione se il messaggio è un documento binario conforme a una definizione MFL. È possibile configurare solo un file MFL.

    • Per gli MFL, è possibile fare clic su Sfoglia per selezionare un MFL dal Browser MFL, quindi fare clic su Sottometti.

  • XML: selezionare questa opzione se il messaggio è un documento XML. Per fornire alcune informazioni sul tipo, è possibile scegliere di dichiarare il tipo di schema XML del documento XML scambiato.
  • Java: selezionare questa opzione se si invia un oggetto Java nella richiesta. Per i messaggi contenenti oggetti Java viene utilizzato il trasporto JMS. Per ulteriori informazioni, vedere "Sending and Receiving Java Objects in Messages" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Tipo di messaggio di risposta
Selezionare un tipo di messaggio per il messaggio di risposta.
  • Nessuno: selezionare questa opzione quando non vi sono messaggi di risposta.
  • Binario: selezionare questa opzione se il tipo di contenuto del messaggio è sconosciuto o non importante.
  • Testo: selezionare questa opzione se il messaggio può essere limitato al testo.
  • MFL: selezionare questa opzione se il messaggio è un documento binario conforme a una definizione MFL. È possibile configurare solo un file MFL.

    • Per gli MFL, è possibile fare clic su Sfoglia per selezionare un MFL dal Browser MFL, quindi fare clic su Sottometti.

  • XML: selezionare questa opzione se il messaggio è un documento XML. Per fornire alcune informazioni sul tipo, è possibile scegliere di dichiarare il tipo di schema XML del documento XML scambiato.
  • Java: selezionare questa opzione se si riceve un oggetto Java nella risposta. Per i messaggi contenenti oggetti Java viene utilizzato il trasporto JMS. Per ulteriori informazioni, vedere "Sending and Receiving Java Objects in Messages" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.

Al termine della procedura

Fare clic su Avanti per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare questa configurazione nella pagina Crea/Modifica un servizio business - Riepilogo.
