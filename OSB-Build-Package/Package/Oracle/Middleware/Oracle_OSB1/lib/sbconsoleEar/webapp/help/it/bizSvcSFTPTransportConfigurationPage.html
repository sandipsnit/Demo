Pagina Configurazione trasporto SFTP

La pagina Crea/Modifica un servizio business - Configurazione trasporto SFTP fa parte di una serie di pagine per la creazione e la modifica di servizi business, come descritto in Creazione e configurazione dei servizi business e Modifica delle configurazioni dei servizi business.

Per ulteriori informazioni, vedere "HTTP and Poller Transports" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV947.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio business che utilizza il protocollo di trasporto SFTP. Nella Tabella 18-16 viene descritto come utilizzare questa pagina.

Tabella 18-16 Pagina Crea/Modifica un servizio business - Configurazione trasporto SFTP
Opzione
Per creare o modificare...
Autenticazione utente
Selezionare una delle opzioni riportate di seguito.
  • Autenticazione con nome utente e password: specifica che un account di servizio statico è associato a questo metodo di autenticazione e che il client viene autenticato utilizzando le credenziali fornite.
  • Autenticazione basata su host: specifica che per utilizzare questo metodo di autenticazione è necessario un nome utente e un provider di chiavi servizio. Gli utenti che si connettono da un host conosciuto vengono autenticati utilizzando la chiave privata dell'host.
  • Autenticazione con chiave pubblica: specifica che per utilizzare questo metodo di autenticazione è necessario un nome utente e un provider di chiavi servizio. Ciascun utente dispone della propria chiave privata.
Account di servizio
Immettere l'account di servizio per l'utente oppure fare clic su Sfoglia per selezionare gli account di servizio da un browser.
Provider di chiavi servizio
Questa opzione è disponibile solo quando è selezionato il metodo Autenticazione basata su host o Autenticazione con chiave pubblica.
Immettere un provider di chiavi servizio nel campo Provider di chiavi servizio. È possibile fare clic su Sfoglia per selezionare i provider di chiavi servizio da un browser. Questo campo è obbligatorio.
Nome utente
Questa opzione è disponibile solo quando è selezionato il metodo Autenticazione basata su host o Autenticazione con chiave pubblica.
Immettere il nome utente.
Timeout
Immettere l'intervallo di timeout del socket, in secondi, prima che la connessione venga interrotta. Se si immette 0 non c'è timeout. Il valore predefinito è 60.
Prefisso per il nome file di destinazione
Immettere il prefisso per il nome file con cui il file viene memorizzato sul server remoto.
Suffisso per il nome file di destinazione
Immettere il suffisso per il nome file con cui il file viene memorizzato sul server remoto.
Codifica richiesta
Accettare l'impostazione predefinita UTF-8 come codifica del set di caratteri per le richieste nei trasporti SFTP.
Suite di cifratura preferita
Selezionare la suite di cifratura da utilizzare durante la comunicazione con il server.
Se si utilizza il valore predefinito Usa valore predefinito fase di esecuzione, la lista di suite di cifratura supportate viene inviata al server e vengono provate tutte finché non viene trovata una corrispondenza.
Algoritmo di integrità dei dati preferito
Selezionare l'algoritmo hash di grandi quantità di dati per i controlli di integrità dei dati.
Se si utilizza il valore predefinito, che prevede l'uso dell'impostazione predefinita di runtime, Oracle Service Bus invia l'algoritmo preferito hmac-sha1 al server. Se tale algoritmo non è supportato dal server, viene inviata la lista degli algoritmi supportati e verrà provato ogni algoritmo finché non verrà trovata una corrispondenza.
Algoritmo di chiave pubblica preferito
Selezionare l'algoritmo di chiave asimmetrica per la crittografia con chiave pubblica.
Se si utilizza il valore predefinito, che prevede l'uso dell'impostazione predefinita di runtime, Oracle Service Bus invia l'algoritmo preferito ssh-dss al server. Se tale algoritmo non è supportato dal server, viene inviata la lista degli algoritmi supportati e verrà provato ogni algoritmo finché non verrà trovata una corrispondenza.
Algoritmo di scambio chiavi preferito
Selezionare il protocollo di scambio predefinito per negoziare la chiave di sessione per la cifratura del messaggio.
Se si utilizza il valore predefinito, che prevede l'uso dell'impostazione predefinita di runtime, Oracle Service Bus invia l'algoritmo preferito diffie-hellman-group1-sha1 al server. Se tale algoritmo non è supportato dal server, viene inviata la lista degli algoritmi supportati e verrà provato ogni algoritmo finché non verrà trovata una corrispondenza.
Algoritmo di compressione preferito
Specificare se è necessario comprimere i dati di esecuzione utilizzando ZLIB. Selezionare zlib per comprimere i dati, altrimenti selezionare Nessuno. Il valore predefinito è Nessuno.

Al termine della procedura

Fare clic su Avanti per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare questa configurazione nella pagina Crea/Modifica un servizio business - Riepilogo.
