Pagina Configurazione trasporto SOA-DIRECT

Nella Tabella 18-17 sono descritte le opzioni di configurazione specifiche per il trasporto SOA-DIRECT.

Per ulteriori informazioni sul trasporto SOA-DIRECT, vedere "SOA-DIRECT Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV909.

Tabella 18-17 Pagina Crea/Modifica un servizio business - Configurazione trasporto SOA-DIRECT
Proprietà
Descrizione
Account di servizio JNDI
Facoltativo. Specifica le credenziali di sicurezza per la ricerca JNDI del servizio SOA di destinazione. È necessario che l'account di servizio sia statico. Fare clic su Sfoglia e selezionare un account di servizio. Se non si specifica un account di servizio, viene utilizzato un oggetto anonimo.
Ruolo
Obbligatorio. Identifica il pattern di comunicazione utilizzato dal servizio. Selezionare una delle opzioni riportate di seguito.
  • Client sincrono: (impostazione predefinita) in questo ruolo l'opzione Proxy di callback è disabilita perché non vi sono callback asincroni. Anche l'opzione Versione WS-Addressing è disabilitata.
  • Client asincrono: in questo ruolo è possibile identificare un proxy di callback perché il callback asincrono è in genere richiesto ed è necessario selezionare una versione WS-Addressing. L'opzione asincrona è abilitata solo quando il servizio WSDL è di tipo SOAP.
  • Callback servizio: questo ruolo permette di restituire il callback asincrono a un servizio SOA dopo un richiamo del servizio esterno.

    • Il bilanciamento del carico o il failover non sono disponibili per i servizi nel ruolo Callback servizio.

Proxy di callback
Facoltativo. Abilitato solo per il ruolo Client asincrono.
Questa opzione permette di specificare il servizio proxy che riceve i callback. Quando si seleziona un proxy di callback, se nessun WS-Addressing viene fornito dalla richiesta o dalla pipeline del servizio proxy, Oracle Service Bus popola in modo automatico la proprietà ReplyTo nell'intestazione SOAP. È necessario selezionare un servizio proxy WSDL che utilizza il trasporto SB (per RMI) ed è necessario che il servizio proxy di callback comprenda il WS-Addressing.
Le proprietà WS-Addressing inviate nella richiesta o impostate nella pipeline del servizio proxy vengono utilizzate al posto del proxy di callback impostato in questa opzione.
Se non si specifica un proxy di callback e la richiesta non contiene proprietà ReplyTo, è necessario fornire le proprietà ReplyTo nell'intestazione SOAP mediante la pipeline del servizio proxy.
Proxy errori
Questa opzione non è attualmente supportata. È necessario configurare i propri servizi di callback per gestire gli errori in un pattern asincrono.
Versione WS-Addressing
Obbligatorio. Abilitato solo per il ruolo Client asincrono.
Specificare la versione WS-Addressing da utilizzare quando il WS-Addressing non viene fornito nella richiesta o nella pipeline del servizio proxy. Le proprietà WS-Addressing inviate nella richiesta o impostate nella pipeline del servizio proxy vengono utilizzate al posto della versione WS-Addressing impostata in questa opzione.
Per mancate corrispondenze della versione WS-Addressing tra gli ambienti, effettuare tutte le trasformazioni necessarie nella pipeline del servizio proxy. Per ulteriori informazioni, vedere "Transformation Examples" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1827.
Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager, viene utilizzato quello predefinito.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Passa oggetto del chiamante
Facoltativo. Selezionare questa opzione per far sì che Oracle Service Bus passi l'oggetto autenticato dal servizio proxy quando viene richiamato il servizio SOA. L'opzione Account di servizio per richiamo, in alternativa a Passa oggetto del chiamante, è disabilita quando si seleziona questa opzione.
Nota: verificare che l'accettazione sicura del dominio sia abilitata tra i server client e destinazione se questi si trovano in domini diversi. Per ulteriori informazioni, vedere "Important Information Regarding Cross-Domain Security Support" in Oracle Fusion Middleware Securing Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=SECMG403.
Account di servizio per richiamo
Facoltativo. In alternativa a Passa oggetto del chiamante, che permette di specificare le credenziali di sicurezza personalizzate selezionando un account di servizio per il richiamo RMI. È possibile specificare qualsiasi tipo di account di servizio (Pass Through, Statico, Mapping).
Fare clic su Sfoglia e selezionare un account di servizio. Se non viene specificato un account di servizio, viene utilizzato un oggetto anonimo.

Al termine della procedura

Fare clic su Avanti per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare questa configurazione nella pagina Crea/Modifica un servizio business - Riepilogo.
