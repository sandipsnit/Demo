Pagina Configurazione trasporto Tuxedo

La pagina Crea/Modifica un servizio business - Configurazione trasporto Tuxedo fa parte di una serie di pagine per la creazione e la modifica di servizi business, come descritto in Creazione e configurazione dei servizi business e Modifica delle configurazioni dei servizi business.

Per ulteriori informazioni, vedere "Tuxedo Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio business che utilizza il protocollo di trasporto Tuxedo. Nella Tabella 18-18 viene descritto come utilizzare questa pagina.

Tabella 18-18 Pagina Crea/Modifica un servizio business - Configurazione trasporto Tuxedo (continua)
Opzione
Per creare o modificare...
Classi tabelle di campi
Immettere il nome della classe o delle classi che descrivono il buffer FML/FML32 ricevuto. Tali classi sono utilizzate per le routine di conversione FML/da FML32 a XML per mappare i nomi dei campi ai nomi degli elementi. Si tratta di una lista lista di nomi di classi completamente qualificati separati da spazi.
Classi viste
Immettere il nome della classe o delle classi che descrivono il buffer VIEW/VIEW32 ricevuto o inviato. Queste classi vengono utilizzate nelle routine di conversione da VIEW a XML o da VIEW32 a XML per mappare i nomi dei campi ai nomi degli elementi. Si tratta di una lista separata da spazi di nomi di classi completamente qualificati.
File Jar classi
Selezionare una risorsa JAR che contiene un file JAR con le classi FML/FML32 o VIEW/VIEW32 necessarie per il funzionamento di questo endpoint.
Punti di accesso remoto
Dall'elenco a discesa selezionare un punto di accesso remoto associato all'importazione. Nell'elenco sono presenti i punti di accesso remoto configurati in WTC. Non è possibile creare un servizio business senza punto di accesso remoto associato.
Se non esistono punti di accesso remoto oppure per crearne uno nuovo, selezionare Nuovo. Immettere le informazioni corrispondenti nei campi Nome punto di accesso e Indirizzo di rete. Dopo la convalida dell'endpoint, il punto di accesso viene aggiunto alla configurazione WTC per ciascun server WTC. Se non esiste alcun server WTC, ne viene creato uno.
Se sono stati specificati più URI, sarà disponibile un campo di punto di accesso remoto per URI in cui l'URI è visualizzato solo a scopo informativo. Se esistono più URI, ciascuno richiede un punto di accesso remoto diverso. Se l'URI specificato corrisponde già a una risorsa WTC esistente, il punto di accesso remoto corrispondente viene visualizzato ma non può essere modificato.
Punti di accesso locale
Questo campo viene visualizzato solo quando si seleziona Nuovo nel campo Punto di accesso remoto.
Dall'elenco a discesa selezionare un punto di accesso locale da associare al punto di accesso remoto appena creato. Se non ne esiste nessuno oppure per crearne uno nuovo, selezionare Nuovo. Immettere le informazioni corrispondenti nei campi Nome punto di accesso locale e Indirizzo di rete locale.
Tipo di buffer di richiesta
Selezionare il tipo di buffer che il servizio Tuxedo remoto riceverà.
Tipo secondario di buffer di richiesta
Questa opzione è abilitata se il valore del campo Tipo di buffer di richiesta è VIEW o VIEW32. Immettere il tipo secondario di buffer al quale associare il buffer di richiesta.
È richiesta la risposta?
Selezionare questa casella di controllo per indicare una chiamata bidirezionale. Se non viene selezionata, tpcall viene richiamato con il flag TPNOREPLY e una risposta nulla viene inviata in modo asincrono.
Sospendere la transazione?
Selezionare questa casella di controllo per sospendere la transazione, se esistente. Ciò è utile quando il servizio remoto non supporta le transazioni.
Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager viene utilizzato quello predefinito.
Work Manager viene utilizzato per inviare in modo asincrono una risposta nulla in caso di richiamo unidirezionale. Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codifica richiesta
Specificare una codifica del set di caratteri per le richieste nei trasporti Tuxedo.
Codifica risposta
Specificare una codifica del set di caratteri per le risposte nei trasporti Tuxedo.
Timeout
Specificare l'intervallo di tempo massimo (in secondi) in cui la fase di esecuzione del trasporto attende le risposte; immettere un valore intero maggiore o uguale a 0. Se non specificato o impostato su zero (impostazione predefinita), il timeout delle risposte corrisponderà a BLOCKTIME, il numero massimo di secondi consentiti dal punto di accesso WTC locale per una chiamata di blocco.
Stile di trasformazione
Selezionare una delle opzioni riportate di seguito.
  • Nessuno: (impostazione predefinita) l'ordine dei campi potrebbe non essere rispettato.
  • Ordinato: i campi vengono visualizzati con tutte le relative ricorrenze nell'ordine corretto.
  • Ordinato e raggruppato: se i campi sono ordinati in modo logico come record, i campi vengono ordinati in base alla ricorrenza e raggruppati in base ai record.

Al termine della procedura

Fare clic su Avanti per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare questa configurazione nella pagina Crea/Modifica un servizio business - Riepilogo.
