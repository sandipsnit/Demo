Ordine di annullamento dei task

È possibile annullare i task in qualsiasi ordine (posto che i risultati delle singole azioni di annullamento siano dati validi). L'operazione di annullamento imposta il valore di una risorsa sul valore che questa aveva prima di essere modificata.

Nel caso in cui il task che si intende annullare è un task di creazione di un oggetto, non esiste uno stato precedente a cui poter riportare un oggetto, ovvero prima dell'esecuzione di questo task non esisteva alcun oggetto. Dunque l'operazione di annullamento elimina il nuovo oggetto dalla sessione. In questo caso, si verificano degli errori per gli oggetti che fanno riferimento all'oggetto eliminato. È possibile visualizzare tali errori nella pagina Visualizza conflitti nel Centro modifiche.

Quando non si lavora in una sessione, è possibile accedere alla pagina Visualizza modifiche alla configurazione per vedere le sessioni che sono state attivate in precedenza. Queste sessioni possono essere annullate. Il sistema non permette di annullare una sessione attivata in precedenza nel caso in cui l'azione di annullamento provoca un errore nella configurazione della fase di esecuzione. Ad esempio, se si tenta di annullare l'attivazione di una sessione e tale operazione provoca la rimozione di un oggetto a cui fa riferimento un altro oggetto, tale operazione di annullamento non viene consentita.

È possibile annullare un'azione di annullamento. Nella colonna Opzioni di Annullamento di [task] fare clic sull'icona Annulla. Oracle Service Bus supporta l'annullamento illimitato. Ciò significa che è possibile annullare l'annullamento che aveva annullato un precedente annullamento e così via.
