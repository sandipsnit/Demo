Codici di errore

In questa sezione sono descritti gli errori Oracle Service Bus che possono verificarsi durante l'esecuzione di un servizio proxy. I codici di errore associati a questi errori risultano all'interno dell'elemento della variabile di contesto fault. È possibile accedere al valore utilizzando l'istruzione XQuery seguente:

$fault/ctx:errorCode/text()

Gli errori sono accompagnati da dettagli specifici per l'errore all'interno dell'elemento fault.

Nella Tabella A-1 sono elencati i codici di errore Oracle Service Bus.

Codici di errore di sicurezza (da 386000 a 386999)
BEA-386000
Errore generale di sicurezza. Possibili cause:
  • L'autenticazione del token personalizzato o di nome utente/password personalizzati è configurata ma l'autenticazione viene già effettuata mediante WS-Security.
 
BEA-386100
Autenticazione non riuscita. Possibili cause:
  • XPath è configurato per eseguire azioni sul corpo o sull'intestazione ma tale parte di messaggio non è presente (il layer di associazione restituisce Null).
  • Il motore XPath restituisce un'eccezione nella valutazione dell'espressione XPath.
  • Il risultato del nome utente XPath o della password XPath è un nodo non di tipo TEXT o ATTR.
  • XPath restituisce più nodi.
  • Uno nome utente e una password sono stati recuperati ma l'autenticazione non è riuscita.
  • Un token personalizzato è stato recuperato ma l'asserzione dell'identità non è riuscita (il servizio di asserzione di identità CSS restituisce LoginException o qualsiasi altra eccezione).
 
BEA-386101
Token di autenticazione mancante. Possibili cause:
  • Il nome utente XPath o la password XPath restituisce un risultato vuoto o una stringa vuota.
  • Il token personalizzato XPath restituisce un risultato vuoto, un valore attributo Null o un valore testo Null.
 
BEA-386102
Autorizzazione a livello di messaggio negata. Possibili cause:
  • Il criterio di controllo dell'accesso a livello di messaggio impedisce l'accesso al servizio proxy.
 
BEA-386103
L'algoritmo di selezione dell'operazione del servizio proxy non è in grado di determinare il nome dell'operazione dalla richiesta o restituisce un'operazione non valida (ovvero non inclusa nel WSDL o Null). Possibili cause:
  • Si è verificato un errore durante l'elaborazione dell'operazione.
  • L'algoritmo di selezione dell'operazione restituisce Null.
  • L'algoritmo di selezione dell'operazione restituisce un'operazione che non è tra quelle dichiarate dal WSDL.

Schemi dei dettagli degli errori

Di seguito sono riportate le definizioni degli schemi per gli snippet XML che accompagnano i codici di errore elencati nella Tabella A-1. Tali snippet XML appaiono all'interno della variabile di contesto di errore sotto l'elemento <ctx:details>. Questi schemi non provengono tutti da un singolo spazio di nomi. Inoltre, mentre un determinato codice di errore implica in genere un particolare snippet XML, alcuni snippet XML sono utilizzati da diversi codici di errore. 

<xs:schema targetNamespace="http://www.oracle.com/wli/sb/stages/transform/config"
           xmlns:tns="http://www.oracle.com/wli/sb/stages/transform/config"
	   xmlns:xs="http://www.w3.org/2001/XMLSchema"
           xmlns:ref="http://www.oracle.com/wli/sb/reference"
           xmlns:sdk="http://www.oracle.com/wli/sb/stages/config"
	   elementFormDefault="qualified"
	   attributeFormDefault="unqualified">

            <xs:element name="faultcode" type="xs:QName"/>
            <xs:element name="faultstring" type="xs:string"/>
            <xs:element name="detail" minOccurs="0" >
                <xs:complexType mixed="true" >
