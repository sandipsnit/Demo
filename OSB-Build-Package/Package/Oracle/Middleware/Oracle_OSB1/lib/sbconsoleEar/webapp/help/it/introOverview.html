Introduzione a Oracle Service Bus

Oracle Service Bus è un bus di servizi enterprise basato su criteri e configurazioni. Fornisce integrazione orientata ai servizi altamente scalabile e affidabile, gestione dei servizi e brokering tradizionale dei messaggi in ambienti IT eterogenei. Combina in un unico prodotto software il brokering intelligente dei messaggi con instradamento e trasformazione dei messaggi e il monitoraggio e l'amministrazione dei servizi. La console di amministrazione di Oracle Service Bus permette di controllare le configurazioni dei criteri e dei servizi, nonché monitorare i task operativi e di sistema. Oracle Service Bus si basa sulle funzionalità runtime di Oracle WebLogic Server. Per ulteriori informazioni, vedere Oracle Fusion Middleware Concepts and Architecture for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBCA.

Limitazioni di denominazione delle risorse

Quando si assegna un nome a una directory o a una risorsa qualsiasi in una configurazione di Oracle Service Bus, sono consentiti i caratteri riportati di seguito.

    • Tutti i caratteri di identificativo Java, comprese le parole chiave Java, descritti nelle sezioni "Identifiers" (Identificativi) e "Keywords" (Parole chiave) della Specifica di linguaggio Java all'indirizzo http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#3.8

    • Spazi vuoti, punti e trattini all'interno dei nomi (non iniziali o finali)

I caratteri / \ * : " < > ? | non sono consentiti.
