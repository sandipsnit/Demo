Aggiunta di associazioni JCA

Il modo più facile per creare associazioni JCA in Oracle Service Bus consiste nell'importazione in blocco delle risorse JCA create (file .jca, WSDL associato e file di mapping TopLink o EclipseLink). Con l'importazione in blocco, Oracle Service Bus genera automaticamente un'associazione JCA esterna ai file .jca, risorse WSDL esterne ai WSDL JCA associati e risorse di documento XML esterne, gestendo le dipendenze tra i file.

Per ulteriori informazioni sull'importazione, vedere Caricamento di risorse da un file .zip, Caricamento di risorse da un URL e Importazione di risorse.

Dopo avere importato le risorse JCA, è possibile generare servizi proxy o business esternamente alle risorse, come descritto in Generazione di un servizio proxy da una risorsa di associazione JCA e Generazione di un servizio business da una risorsa di associazione JCA.

Se non si esegue l'importazione in blocco delle risorse JCA, è possibile creare associazioni JCA in modo manuale, come descritto nella procedura riportata di seguito.

  1. In Oracle JDeveloper creare un WSDL del servizio JCA, un file JCA (.jca) corrispondente e, se necessario, altre risorse quali file di mapping TopLink o EclipseLink e file XSLT.

    2. Per ulteriori informazioni, vedere Oracle Fusion Middleware User's Guide for Technology Adapters all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=TKADP.

  2. Se questa operazione non è già stata eseguita, fare clic su Crea per creare una nuova sessione nella console di amministratore OSB o su Modifica per accedere a una sessione esistente. Vedere Utilizzo del Centro modifiche.
  3. Nella console di amministrazione OSB creare o importare le risorse JCA create in Oracle JDeveloper, ad esempio le risorse WSDL, XML e XSLT.
  4. Creare l'associazione JCA.
    1. Selezionare Explorer progetto, quindi selezionare il progetto o la posizione della cartella per l'associazione JCA. Viene visualizzata la pagina Vista progetto/cartella.
    2. Nel campo Crea risorsa selezionare Associazione JCA per visualizzare la pagina Crea una nuova risorsa associazione JCA.
    3. Nel campo File JCA fare clic su Sfoglia per selezionare il file JCA.

      4. Il campo Nome risorsa viene popolato automaticamente con il nome del file (senza estensione di file).

      Invece di selezionare un file, è possibile copiare e incollare i contenuti di un file JCA nell'area di testo del campo File JCA.

    4. Nel campo Nome risorsa immettere o modificare il nome dell'associazione JCA. I nomi delle associazioni JCA devono essere univoci.

      Vedere Limitazioni di denominazione delle risorse per le istruzioni di denominazione.
    5. Nel campo Descrizione risorsa immettere una descrizione per questa associazione JCA.
    6. Fare clic su Salva. L'associazione JCA viene salvata nella sessione corrente.

      7. Un'avvertenza di convalida viene visualizzata se l'associazione JCA non fa ancora riferimento alle risorse richieste, come un WSDL e un file di mapping.

  5. Fare clic sul nome della nuova associazione JCA per visualizzarlo nella pagina Visualizza un'associazione JCA.
  6. Fare clic su Modifica riferimenti.
  7. Nella pagina Modifica il riferimento di una risorsa di associazione JCA scegliere Sfoglia per selezionare le risorse di riferimento dell'associazione JCA, ad esempio WSDL, un file di mapping TopLink/EclipseLink e un file XSLT.
  8. Fare clic su Salva.
  9. Per terminare la sessione e distribuire la configurazione in fase di esecuzione, fare clic su Attiva in Centro modifiche.

Dopo avere aggiunto le risorse di associazione JCA e configurato le dipendenze delle risorse (associando un WSDL), oppure se sono state importate le risorse JCA che hanno mantenuto le relative dipendenze, è possibile generare servizi proxy e business dalle associazioni JCA. Per ulteriori informazioni, vedere Generazione di un servizio proxy da una risorsa di associazione JCA e Generazione di un servizio business da una risorsa di associazione JCA.
