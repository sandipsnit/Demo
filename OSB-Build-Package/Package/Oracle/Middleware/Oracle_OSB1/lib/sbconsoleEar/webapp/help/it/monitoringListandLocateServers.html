Visualizzazione delle informazioni sui server

Nella pagina Integrità server è incluso quanto riportato di seguito.

  • Nell'intestazione della scheda Integrità server possono essere presenti un'icona Chiusura e un indicatore numerico con il numero di server chiusi.
    • Fare clic sull'icona per filtrare i server visualizzati utilizzando i criteri Shutdown Integrità e SHUTDOWN Stato.
    • Il numero indica il numero di server chiusi.
  • Un grafico a torta descrive l'integrità dei server nel dominio. È possibile fare clic sull'area del grafico che corrisponde al server per visualizzare ulteriori dettagli.
  • Riepilogo log. Per visualizzarlo, fare clic sul collegamento numerico che visualizza il numero di messaggi per una determinata severità. Ad esempio, Alert: 1, fare clic sul numero 1. Vedere Visualizzazione dei dettagli dei file di log del dominio.
  • I campi in cui sono visualizzate le informazioni sul server dipendono dalle impostazioni personalizzate per la tabella Riepilogo server. Ad esempio, il nome del server, l'integrità, il nome del cluster, il nome del computer, lo stato del server e il periodo di tempo in cui il server è in esecuzione. Per ulteriori informazioni su questi campi, vedere la Tabella 25-16.
  • Per impostazione predefinita, la frequenza di aggiornamento del dashboard è impostata su Nessun aggiornamento. Per modificare l'intervallo della frequenza di aggiornamento del dashboard, vedere Impostazione delle preferenze utente.
Attenzione: se si seleziona un intervallo di frequenza di aggiornamento del dashboard diverso da quello predefinito (Nessun aggiornamento), il browser aggiorna il dashboard a intervalli regolari. Ciò impedisce il timeout della sessione, anche in assenza di interazione con la console di amministrazione di Oracle Service Bus.
Attenzione: Oracle consiglia di utilizzare con cautela la funzione di frequenza dell'aggiornamento del dashboard. È opportuno non lasciare mai una sessione senza supervisione perché mentre la sessione è attiva altre persone possono accedervi utilizzando il computer dell'utente senza effettuare di nuovo l'autenticazione.
  • Icona Funzione di personalizzazione tabella. Vedere Personalizzazione delle viste delle tabelle.
  • Per limitare il numero di elementi nella lista o individuare server specifici, effettuare le operazioni riportate di seguito.
    • Fare clic sull'icona Apri per visualizzare ulteriori filtri di ricerca. Se si specificano più criteri di ricerca, vengono restituiti solo i risultati che soddisfano tutti i criteri.
    • Filtrare per stato di integrità. Dall'elenco a discesa Integrità selezionare per limitare i server al livello di stato di integrità specificato, quindi fare clic su Cerca.

      • Selezionare la casella di controllo o superiore per limitare la ricerca al livello di stato di integrità del server specificato (elencati a partire da quello più ampio a quello più limitato): OK, Avvertenza, Sovraccarico, Critico e Irreversibile o superiore.

    • Filtrare per server. Dall'elenco a discesa Server selezionare Tutti o il nome del server, quindi fare clic su Cerca.
    • Filtrare per nome di cluster. Dall'elenco a discesa Nome cluster selezionare Tutti o il nome del cluster associato al server, quindi fare clic su Cerca.
    • Filtrare per nome di computer. Dall'elenco a discesa Nome computer selezionare Tutti o il nome del computer associato al server, quindi fare clic su Cerca.
    • Filtrare per stato. Dall'elenco a discesa Stato selezionare Tutti o lo stato del server RUNNING, FAILED o SHUTDOWN, quindi fare clic su Cerca.
    • Fare clic su Vis. tutto per rimuovere i filtri di ricerca e visualizzare tutti i server.

Nella pagina Integrità server per ciascun server vengono visualizzate le informazioni mostrate nella Tabella 25-16, in base alle impostazioni di filtro specificate.

Tabella 25-16 Dettagli del riepilogo dei server
Proprietà
Descrizione
Server
Il nome del server. Il nome è un collegamento alla pagina Visualizza dettagli server. Vedere Visualizzazione dei dettagli dei server.
Integrità
Lo stato di integrità del server.
  • OK: il server funziona senza alcun problema.
  • OK/Non in elaborazione: il server si trova nello stato ADMIN, disponibile solo per le operazioni di amministrazione. Tutti i messaggi Oracle Service Bus della fase di esecuzione non riusciranno.
  • Avvertenza: il server potrebbe avere problemi in futuro. Per ulteriori dettagli, verificare i log del server e il valore RuntimeMBean corrispondente.
  • Sovraccarico: al server è stato assegnato più lavoro rispetto alla soglia configurata e potrebbe rifiutare ulteriore lavoro.
  • Critico: errore del server imminente; è necessario svolgere immediatamente eventuali operazioni per evitare l'errore. Per ulteriori dettagli, verificare i log del server e il valore RuntimeMBean corrispondente.
  • Chiusura: errore del server, è necessario riavviarlo.
  • Critico: errore del server imminente; è necessario svolgere immediatamente eventuali operazioni per evitare l'errore. Per ulteriori dettagli, verificare i log del server e il valore RuntimeMBean corrispondente.
Nome cluster
Il nome del cluster associato a questo server.
Nome computer
Il nome del computer associato a questo server.
Stato
Lo stato del server:
  • RUNNING
  • ADMIN
  • FAILED
  • SHUTDOWN
Tempo di attività
Il periodo di tempo in cui il server è in esecuzione.
