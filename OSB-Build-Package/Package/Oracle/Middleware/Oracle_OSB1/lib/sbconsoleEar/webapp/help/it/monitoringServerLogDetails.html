Visualizzazione dei dettagli dei file di log del dominio

Utilizzare questa pagina per visualizzare i dettagli delle voci del file di log del dominio, mostrati nella Tabella 25-18.

Tabella 25-18 Dettagli del log del dominio
Proprietà
Descrizione
Messaggio
Una descrizione dell'evento o della condizione.
Data
Visualizza l'ora e la data in cui il messaggio è stato originato, in un formato specifico per le impostazioni nazionali. La JVM (Java Virtual Machine) che esegue ciascuna istanza WebLogic Server fa riferimento al sistema operativo del computer host per le informazioni sul fuso orario locale e sul formato.
Sottosistema
Indica il sottosistema di WebLogic Server che costituisce l'origine del messaggio, ad esempio un contenitore EJB (Enterprise Java Bean) o un servizio di messaggistica JMS (Java Messaging Service).
ID messaggio
Un identificativo univoco di sei cifre.
Tutti gli ID di messaggio generati dai messaggi di sistema del server WebLogic iniziano con BEA- e sono compresi nell'intervallo numerico tra 0 e 499999.
Severità
Indica il grado di impatto o severità dell'evento segnalato dal messaggio.
  • Alert: un determinato servizio si trova in uno stato inutilizzabile mentre altre parti del sistema continuano a funzionare. Il recupero automatico non è possibile; per risolvere il problema è necessario l'intervento immediato dell'amministratore.
  • Critico: si è verificato un errore di sistema o del servizio. È possibile eseguire il recupero del sistema ma potrebbe esserci una perdita momentanea o il degrado permanente del servizio.
  • Emergenza: il server si trova in uno stato inutilizzabile. Questa severità indica un errore di sistema grave.
  • Errore: si è verificato un errore utente. Il sistema o l'applicazione è in grado di gestire l'errore senza interruzione e con degrado limitato del servizio.
  • Info: utilizzato per indicare le operazioni normali; un messaggio informativo di livello basso.
  • Avviso: un messaggio informativo con livello di importanza più alto.
  • Avvertenza: si è verificata un'operazione o una configurazione sospetta ma ciò potrebbe non influire sul normale funzionamento.
Computer
Il nome DNS del computer che ospita l'istanza di server.
Server
Il nome dell'istanza WebLogic Server su cui è stato generato il messaggio.
Thread
L'ID assegnato dalla JVM al thread in cui è stato generato il messaggio.
ID utente
L'ID utente con cui l'evento associato è stato eseguito.
ID transazione
Presente solo per i messaggi registrati nel contesto di una transazione.
ID contesto
Informazioni sul contesto per la correlazione dei messaggi provenienti da una richiesta o da un'applicazione specifica.
Dettaglio
Una descrizione dell'evento o della condizione.
Causa
La causa del messaggio.
Azione
L'azione da intraprendere.
