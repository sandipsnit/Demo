Visualizzazione delle metriche di servizio

Utilizzare questa pagina per visualizzare i dettagli del monitoraggio di servizi specifici, mostrati nella Tabella 25-10, e le metriche di servizio, mostrate nella Tabella 25-11.

Tabella 25-10 Informazioni sul monitoraggio dei servizi
Proprietà
Descrizione
Nome servizio
Il nome e il percorso del servizio.
Tipo di servizio
Il tipo di servizio padre: servizio proxy o servizio business.
Visualizza statistiche
Per visualizzare le statistiche di monitoraggio per il servizio per il periodo dell'intervallo di aggregazione corrente, selezionare Intervallo di aggregazione corrente. Nella vista Intervallo di aggregazione corrente viene mostrata una vista mobile delle statistiche relativa alle metriche del servizio.
Per visualizzare le statistiche di monitoraggio per questo servizio per il periodo a partire dalla reimpostazione più recente delle statistiche, selezionare A partire dalla reimpostazione più recente. Nella vista A partire dalla reimpostazione più recente viene mostrata una vista di conteggio dinamico delle metriche.
Server
Selezionare un server dall'elenco a discesa per visualizzare le metriche di tale server.
Se è presente un cluster, le metriche a livello di cluster vengono visualizzate per impostazione predefinita. Selezionare un singolo server gestito per visualizzare le metriche di tale server.
Se si visualizzano le metriche per un singolo server gestito in un cluster, nel campo Conteggio alert SLA viene visualizzato N/A (vengono visualizzati solo i conteggi degli alert SLA a livello di cluster).
Intervallo di aggregazione
Se nel campo Visualizza statistiche è stato selezionato Intervallo di aggregazione corrente, indica l'intervallo di aggregazione corrente impostato per il monitoraggio di questo servizio, in termini di ore e minuti. Impostare questo intervallo nella pagina Impostazioni operative. Vedere Impostazione dell'intervallo di aggregazione per un servizio.
Reimposta statistiche
Se nel campo Visualizza statistiche è stato selezionato A partire dalla reimpostazione più recente, è possibile reimpostare il monitoraggio delle statistiche per questo servizio. Fare clic su Reimposta statistiche nella parte inferiore della pagina.

Tabella 25-11 Metriche di servizio
Proprietà
Descrizione
Conteggio alert SLA
  • Il conteggio degli alert è basato sulla vista Visualizza statistiche, per l'impostazione Intervallo di aggregazione corrente o per il periodo dell'impostazione A partire dalla reimpostazione più recente.
  • La severità degli alert e l'indicatore orario vengono visualizzati solo nella vista Intervallo di aggregazione corrente. Per la vista A partire dalla reimpostazione più recente viene visualizzato solo il valore del conteggio.
  • La severità di alert per l'alert più severo entro l'intervallo di aggregazione corrente. Se sono presenti più alert, viene mostrata la severità di alert più elevata nell'intervallo di aggregazione corrente. Se non vi sono alert nell'intervallo di aggregazione corrente, viene visualizzato NORMAL.
  • L'indicatore orario visualizza l'ora dell'ultima volta in cui è stato eseguito il polling del server per le informazioni sugli alert e visualizza lo stato degli alert all'ora dell'ultimo polling del server. Se non vi erano alert l'ultima volta che è stato eseguito il polling del server, viene visualizzato NORMAL e viene mostrata l'ora del polling.
Conteggio alert pipeline
Solo per i servizi proxy.
  • Il conteggio degli alert pipeline è basato sulla vista Visualizza statistiche, per l'impostazione Intervallo di aggregazione corrente o per il periodo dell'impostazione A partire dalla reimpostazione più recente.
  • La severità degli alert e l'indicatore orario vengono visualizzati solo nella vista Intervallo di aggregazione corrente. Per la vista A partire dalla reimpostazione più recente viene visualizzato solo il valore del conteggio.
  • La severità di alert per l'alert più severo entro l'intervallo di aggregazione corrente. Se sono presenti più alert, viene mostrata la severità di alert più elevata nell'intervallo di aggregazione corrente. Se non vi sono alert pipeline nell'intervallo di aggregazione corrente, viene visualizzato NORMAL.
  • L'indicatore orario visualizza l'ora dell'ultima volta in cui è stato eseguito il polling del server per le informazioni sugli alert pipeline e visualizza lo stato degli alert all'ora dell'ultimo polling del server. Se non vi erano alert l'ultima volta che è stato eseguito il polling del server, viene visualizzato NORMAL e viene mostrata l'ora del polling.

Solo per i servizi business.

Se nel campo Visualizza statistiche è stato selezionato Intervallo di aggregazione corrente, indica il rapporto di operazioni riuscite di questo servizio entro il periodo dell'intervallo di aggregazione corrente.
Se nel campo Visualizza statistiche è stato selezionato A partire dalla reimpostazione più recente, indica il rapporto di operazioni riuscite di questo servizio entro il periodo a partire dalla reimpostazione più recente delle statistiche.
Ad esempio, se il servizio ha eseguito correttamente 9 messaggi su 10, il rapporto di operazioni riuscite è 90%.

Ad esempio, se il servizio non è riuscito a eseguire 1 messaggio su 10, il rapporto di operazioni non riuscite è 10%.
