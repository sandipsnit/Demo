Connessioni MQ

Nota: per le versioni di MQ WebSphere supportate in Oracle Service Bus, vedere Oracle Fusion Middleware Supported System Configurations all'indirizzo http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html. La libreria client MQ WebSphere non viene fornita nell'Installer di Oracle Service Bus. Acquisire la libreria client MQ WebSphere dall'installazione di MQ WebSphere.

Le connessioni MQ sono risorse condivisibili che è possibile riutilizzare in più servizi business e proxy MQ. È necessario che i servizi business e proxy MQ si connettano a un Queue Manager MQ prima di accedere a una coda MQ. Le risorse di connessione MQ forniscono i parametri di connessione necessari per la connessione a un Queue Manager MQ.

Ciascuna risorsa di connessione MQ dispone di un connection pool. Ogni servizio business o proxy che utilizza una determinata risorsa di connessione MQ per ottenere una connessione a un determinato Queue Manager utilizza lo stesso connection pool creato per tale risorsa. Pertanto, più servizi business e servizi proxy che utilizzano lo stesso Queue Manager condividono un connection pool.

Per ulteriori informazioni sulle risorse di connessione MQ e sui trasporti MQ nativi di Oracle Service Bus, vedere "MQ Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Per ulteriori informazioni sugli elementi fondamentali di MQ WebSphere, vedere http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
