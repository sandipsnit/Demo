Provider di chiavi servizio

Un provider di chiavi di servizio contiene credenziali PKI (Public Key Infrastructure) che i servizi proxy utilizzano per decifrare i messaggi SOAP in entrata e per l'autenticazione in uscita e le firme digitali. Una credenziale PKI è una chiave privata abbinata a un certificato che è possibile utilizzare per le firme digitali e la cifratura (per Web Service Security) e per l'autenticazione SSL in uscita. Il certificato contiene la chiave pubblica che corrisponde alla chiave privata.

Nota: Per utilizzare un provider di chiavi di servizio, è necessario configurare un provider di mapping delle credenziali PKI. Vedere "Configuring the WebLogic Security Framework: Main Steps" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498.

Un singolo provider di chiavi di servizio può contenere tutte le seguenti credenziali PKI:

  • Una coppia di chiavi per la cifratura digitale

    • I servizi proxy utilizzano questa coppia di chiavi per decifrare i messaggi SOAP in entrata che sono stati cifrati per la conformità a un'istruzione dei criteri dei servizi Web. Se si desidera che il provider di chiavi di servizio supporti la cifratura digitale, il keystore associato al mapper delle credenziali PKI deve contenere almeno un certificato X.509 che supporti la cifratura.

  • Una coppia di chiavi per le firme digitali

    • I servizi proxy utilizzano questa coppia di chiavi quando il relativo endpoint è un servizio Web e quest'ultimo richiede ai client di firmare una o più parti di un envelope SOAP.

  • Una coppia di chiavi per l'autenticazione client SSL (SSL bidirezionale)

    • I servizi proxy utilizzano questa coppia di chiavi per eseguire l'autenticazione quando fungono da client durante una connessione TLS/SSL (Secure Sockets Layer) in uscita, ossia durante l'instradamento di un messaggio a un servizio proxy o un servizio business HTTPS che richiede l'autenticazione con certificato del client.

È possibile utilizzare lo stesso provider di chiavi di servizio per più servizi proxy.
