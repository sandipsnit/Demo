Aggiunta di azioni If... Then

Utilizzare un'azione if...then per eseguire un'azione o un set di azioni in modo condizionale in base al risultato booleano di un'espressione XQuery.

Prima di iniziare

Nelle istruzioni riportate di seguito si presume che sia già stato modificato un flusso di messaggi nella pagina Modifica flusso di messaggi, come descritto in Creazione, modifica e visualizzazione dei flussi di messaggi.

Per aggiungere un'azione If...Then

  1. Spostarsi nella posizione in cui si desidera aggiungere l'azione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
  2. Fare clic sull'icona appropriata, quindi selezionare Aggiungi un'azione > Controllo flusso > If...Then.
  3. Fare clic su <Condizione> per visualizzare la pagina Editor condizione XQuery.

    4. La condizione creata viene utilizzata come test eseguito prima dell'inserimento della clausola then() in base alla logica if...then standard. Vedere Creazione e modifica delle espressioni XQuery e XPath in linea.

  4. Dopo aver modificato la condizione XQuery, fare clic su Aggiungi un'azione, quindi selezionare un'azione che si desidera associare alla condizione. Per ulteriori informazioni sul tipo di azione che si desidera aggiungere, vedere Aggiunta e modifica di azioni nei flussi di messaggi.

    5. Nel nodo di instradamento è possibile selezionare solo le azioni di instradamento, di instradamento dinamico o della tabella di instradamento. Tuttavia, queste azioni possono contenere azioni di richiesta e di risposta.

  5. In base a quanto richiesto dalla logica, fare clic sull'icona If...Then..., quindi fare clic su Aggiungi condizione else-if o Aggiungi condizione else per aggiungere le condizioni else-if o else. Fare clic su Aggiungi un'azione per associare le azioni a queste condizioni.

    6. Le azioni condizionali possono essere nidificate. Tuttavia, nell'editor delle fasi non è possibile nidificare più di quattro livelli cumulativi. Se si tenta di aggiungere un quinto livello, l'azione di nidificazione non viene visualizzata. I livelli cumulativi includono tutte le azioni di diramazione: condizioni if...then, tabelle di pubblicazione e tabelle di instradamento. Ad esempio è possibile avere due livelli di azioni condizionali, quindi una tabella di pubblicazione contenente una tabella di instradamento, per un totale di quattro livelli. Se si tenta di aggiungere un'altra azione condizionale (all'ultima tabella di pubblicazione), questa azione non viene visualizzata.

Al termine della procedura

Una volta completata la configurazione di questa azione, proseguire configurando altre azioni o salvando la configurazione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
