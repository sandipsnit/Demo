Aggiunta di azioni di callout Java

Utilizzare l'azione di callout Java per richiamare un metodo Java o un servizio business EJB dal flusso dei messaggi.

Prima di iniziare

Nelle istruzioni riportate di seguito si presume che sia già stato modificato un flusso di messaggi nella pagina Modifica flusso di messaggi, come descritto in Creazione, modifica e visualizzazione dei flussi di messaggi.

Per aggiungere un'azione di callout Java

  1. Spostarsi nella posizione in cui si desidera aggiungere l'azione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
  2. Fare clic sull'icona appropriata, quindi selezionare Aggiungi un'azione > Elaborazione messaggi > Callout Java.
  3. Fare clic su <Metodo>. Viene visualizzata la pagina Seleziona un JAR. Selezionare una risorsa JAR dalla lista. Viene visualizzata la pagina Seleziona una classe e un metodo.
  4. Dalla lista delle classi Java elencate, fare clic sul segno + accanto alla classe appropriata per visualizzare una lista di metodi. Selezionare un metodo e fare clic su Sottometti. L'azione di callout Java viene visualizzata nella pagina Modifica fase, come riportato di seguito.
    • <Metodo> viene sostituito dal nome del metodo Java selezionato nei passi 2 e 3. Questo nome è un collegamento alla pagina Seleziona una classe e un metodo. È possibile fare clic su questo collegamento per modificare la selezione del metodo Java.

      • Il metodo deve essere un metodo statico.

    • Parametri: per ciascun argomento richiesto dal metodo Java viene fornito un collegamento <Espressione> alla pagina Editor espressioni XQuery. Un'etichetta per ciascun collegamento indica il tipo di dati per l'argomento, che sarà uno dei seguenti:
      • Java.lang.String
      • Tipi predefiniti e tipi di classi corrispondenti (ad esempio, int e java.lang.Integer)
      • java.lang.BigDecimal e java.lang.BigInteger (questi tipi vengono usati nei calcoli finanziari in cui gli errori di arrotondamento o gli overflow non sono tollerabili)
      • solo org.apache.xbeans.XmlObject e nessun bean XML digitato.
      • byte[]
      • java.lang.String[] (SOLO INPUT)
      • XmlObject [ ] (SOLO INPUT)
      • javax.activation.DataSource
    • Restituisci parametro come riferimento: questa opzione fa in modo che il valore restituito di un richiamo di un callout Java sia <java-content ref="jcid"> un elemento di riferimento indipendentemente dal tipo effettivo, dove jcid è la chiave per l'oggetto nel repository degli oggetti pipeline. Nel campo del valore del risultato immettere il nome della variabile per contenere il riferimento java-content. Questa opzione consente di lavorare con un oggetto di riferimento nel flusso del messaggio in aggiunta all'XML pipeline per fornire passthrough, eseguire l'arricchimento del messaggio con il callout Java e le azioni in linea o eseguire la trasformazione del messaggio tra il trasporto Java e il trasporto non Java. Per ulteriori informazioni, vedere "Sending and Receiving Java Objects in Messages" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
    • Risultato: un campo Risultato in cui viene immessa la variabile alla quale deve essere assegnato il risultato. L'etichetta per il campo indica il tipo di dati del risultato.
      • Nota: se il risultato è un array di byte (l'unico possibile array restituito), viene restituito l'elemento XML di contenuto binario.
    • Allega Account di servizio: un collegamento Account di servizio consente di specificare un account di servizio facoltativo se esiste un contesto di sicurezza per questo metodo Java. Per ulteriori informazioni sui contesti di sicurezza e sugli account di servizio, vedere Account di servizio.

      • Nel caso di account di servizio fissi e mappati, l'ID utente e/o la password dell'account di servizio vengono autenticati nel sistema locale e il contesto di sicurezza viene propagato al callout Java. Nel caso di passthru, il contesto di sicurezza viene propagato al callout Java. Questo contesto è il contesto a livello di messaggio, se definito (con WS-Security). Else è il contesto a livello di trasporto.

  5. In Parametri fare clic su <Espressione>. Viene visualizzata la pagina Editor espressioni XQuery. Utilizzare l'Editor espressioni XQuery per fornire gli argomenti richiesti dal metodo Java. Vedere Creazione e modifica delle espressioni XQuery e XPath in linea.

    6. Se il tipo di valore di input immesso non corrisponde al tipo di argomento di input dichiarato, Oracle Service Bus tenta di eseguire automaticamente la conversione typecast dei valori di input nel tipo dichiarato dell'argomento di input. Ad esempio, un valore di stringa pari a "123" verrà convertito nel numero intero 123 se il tipo dichiarato dell'argomento di input è il tipo int predefinito Java.

  6. Nel campo Risultato assegnare una variabile per il risultato restituito dal metodo Java.
  7. Se esiste un contesto di sicurezza per il metodo Java, selezionare la casella di controllo e fare clic su <Account di servizio>. Viene visualizzata la pagina Seleziona account di servizio. Selezionare l'account di servizio necessario dalla lista e fare clic su Sottometti.

Al termine della procedura

Una volta completata la configurazione di questa azione, proseguire configurando altre azioni o salvando la configurazione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
