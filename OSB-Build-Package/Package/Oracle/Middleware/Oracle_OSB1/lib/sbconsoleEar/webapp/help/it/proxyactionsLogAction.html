Aggiunta di azioni di log

Utilizzare l'azione di log per creare un messaggio da registrare e per definire un set di attributi con cui viene registrato il messaggio.

Prima di iniziare

Nelle istruzioni riportate di seguito si presume che sia già stato modificato un flusso di messaggi nella pagina Modifica flusso di messaggi, come descritto in Creazione, modifica e visualizzazione dei flussi di messaggi.

Nota: per visualizzare i dati di log nel file di log o nell'output standard (console del server), è necessario impostare il log di Oracle WebLogic Server sui livelli di severità riportati di seguito.

Livello di severità minimo per il log: Informazioni
File di log: Informazioni
Output standard: Informazioni

Per informazioni sull'impostazione dei livelli di severità di log, vedere "Using Log Severity Levels" in Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

 

Per aggiungere un'azione di log

  1. Assicurarsi di abilitare il log a livello globale. Per ulteriori informazioni, vedere Abilitazione del log a livello globale.
  2. Spostarsi nella posizione in cui si desidera aggiungere l'azione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
  3. Fare clic sull'icona appropriata, quindi selezionare Aggiungi un'azione > Report > Log.
  4. Fare clic su <Espressione>. Viene visualizzata la pagina Editor espressioni XQuery. Specificare il contesto del messaggio da registrare mediante le espressioni XQuery nelle variabile di contesto. Vedere Creazione e modifica delle espressioni XQuery e XPath in linea.
  5. Nel campo Annotazione immettere le note per questa azione di log. Queste note vengono registrate insieme al risultato dell'espressione definita in precedenza.
  6. Nell'elenco a discesa Livello di severità selezionare una delle opzioni disponibili.

    7. Tabella 21-5 Livelli di severità per le azioni di log 
    Livello di severità
    Uso standard
    Informazioni
    Utilizzato per le normali operazioni di report; un messaggio informativo di basso livello.
    Avvertenza
    Si è verificata un'operazione o una configurazione sospetta, che potrebbe non influire sul normale funzionamento.
    Errore
    Si è verificato un errore utente. Il sistema o l'applicazione può gestire l'errore senza interruzioni e con un limitato rallentamento del servizio.
    Debug
    Durante lo sviluppo dell'applicazione, potrebbe risultare utile creare e utilizzare i messaggi che forniscono descrizioni approfondite dell'attività di basso livello all'interno dell'applicazione.
    Assicurarsi che il livello di severità di log delle impostazioni operative del servizio proxy sia uguale al livello di severità delle azioni di log. Per informazioni sulle impostazioni operative del servizio proxy, vedere Configurazione delle impostazioni operative per i servizi proxy.

Al termine della procedura

Una volta completata la configurazione di questa azione, proseguire configurando altre azioni o salvando la configurazione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
