Aggiunta di azioni di report

Utilizzare l'azione di report per abilitare il report dei messaggi per un servizio proxy.

Prima di iniziare

Nelle istruzioni riportate di seguito si presume che sia già stato modificato un flusso di messaggi nella pagina Modifica flusso di messaggi, come descritto in Creazione, modifica e visualizzazione dei flussi di messaggi.

Per aggiungere un'azione di report

  1. Spostarsi nella posizione in cui si desidera aggiungere l'azione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
  2. Fare clic sull'icona appropriata, quindi selezionare Aggiungi un'azione > Report > Report.
  3. Fare clic su <Espressione>. Viene visualizzata la pagina Editor espressioni XQuery Vedere Creazione e modifica delle espressioni XQuery e XPath in linea. L'espressione XQuery viene utilizzata per creare i dati che verranno riportati nel dashboard di Oracle Service Bus.
  4. Dopo aver modificato l'espressione XQuery, fare clic su Aggiungi una chiave. Vengono visualizzati due campi: un campo Nome chiave e un campo Valore chiave che include un collegamento XPath sul quale è possibile fare clic per modificare un'espressione XPath e un campo Variabile in cui è possibile immettere una variabile di contesto.

    5. Utilizzare le coppie valore-chiave per estrarre gli ID chiave da qualsiasi variabile di contesto del messaggio o da qualsiasi payload del messaggio e ignorare il resto del messaggio. Le chiavi rappresentano un metodo utile per l'identificazione di un messaggio. Vengono visualizzate come indici del report nel modulo Report. Vedere Visualizzazione dei messaggi di report e Visualizzazione dei dettagli dei messaggi.

    1. Immettere il nome di una chiave nel campo Nome chiave.
    2. Fare clic su <XPath>. Viene visualizzata la pagina Modificare un'espressione XPath. Vedere Creazione e modifica delle espressioni XQuery e XPath in linea.
    3. Immettere una variabile di contesto nel campo Variabile. Per ulteriori informazioni sulle variabili di contesto, vedere l'Appendice A relativa al contesto dei messaggi.
    4. Per aggiungere più valori chiave, fare clic sull'icona Chiave, quindi selezionare Aggiungi una chiave. Per eliminare una chiave, fare clic sull'icona Chiave, quindi selezionare Elimina questa chiave.

Ad esempio, si prenda in considerazione un'azione di report configurata in un handler errori in una fase. In caso di errore, l'azione riporta il contenuto della variabile di contesto fault. L'azione di report viene configurata come indicato di seguito.

  • Nome chiave = errorCode
  • Valore chiave = ./ctx:errorCode nella variabile fault

Ogni volta che questa azione viene eseguita in fase di esecuzione, viene generato un messaggio di report mediante Reporting Data Stream. Nella tabella riportata di seguito vengono visualizzati i risultati ottenuti dopo aver eseguito l'azione di report per due volte.

Al termine della procedura

Una volta completata la configurazione di questa azione, proseguire configurando altre azioni o salvando la configurazione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
