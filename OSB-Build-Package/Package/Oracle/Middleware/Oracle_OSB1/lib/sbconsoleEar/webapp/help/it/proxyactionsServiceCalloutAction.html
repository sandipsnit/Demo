Aggiunta di azioni di callout del servizio

Utilizzare un'azione di callout del servizio per configurare un callout sincrono (bloccante) su un servizio proxy o business registrato in Oracle Service Bus.

Per ulteriori informazioni sulle azioni di callout del servizio, vedere "Constructing Service Callout Messages" in Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBAG208.

  1. Spostarsi nella posizione in cui si desidera aggiungere l'azione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
  2. Fare clic sull'icona appropriata, quindi selezionare Aggiungi un'azione > Comunicazione > Callout servizio.
  3. Fare clic su <Servizio>. Viene visualizzato il Browser servizi.
  4. Selezionare un servizio dalla lista dei servizi proxy o business registrati, quindi fare clic su Sottometti.
  5. Se il servizio scelto al precedente passo 3 è basato su WSDL e include operazioni che possono essere richiamate sul servizio, tali operazioni vengono riportate nell'elenco a discesa di richiamo <Operazione>. Selezionare un'operazione da richiamare sul servizio.

    Nota: la selezione di un'operazione richiesta per molti motivi da Oracle Service Bus non garantisce che venga richiamata solo l'operazione selezionata. Ad esempio, se si seleziona l'Operazione A, ma un messaggio contiene anche un richiamo all'Operazione B, verrà richiamata anche l'Operazione B.
  6. Specificare il modo in cui si desidera configurare i messaggi di richiesta e di risposta selezionando una delle opzioni riportate di seguito.
    • Selezionare Configura corpo SOAP per configurare il corpo SOAP. La selezione di questa opzione consente di utilizzare direttamente $body.
      • Nota: questa opzione supporta il servizio RPC SOAP codificato, non supportato quando si configurano i parametri o il documento del payload.
    • Selezionare Configura parametri del payload oppure Configura documento del payload per configurare il payload.
  7. Le successive operazioni di configurazione dipendono dal tipo di servizio selezionato nel passo 3 e dal tipo di opzioni di configurazione scelte per tale servizio nel passo 5. Nella Tabella 21-3 vengono visualizzate le opzioni disponibili per ogni tipo di servizio.

    Per configurare queste opzioni, effettuare le operazioni riportate di seguito.
    • Nel campo Corpo della richiesta SOAP immettere il nome di una variabile che deve contenere l'XML dell'elemento Corpo della richiesta per la richiesta di callout.
    • Nel campo Corpo della risposta SOAP immettere il nome di una variabile alla quale verrà associato l'XML dell'elemento Corpo SOAP.
    Intestazione richiesta SOAP e
    Intestazione risposta SOAP
    Per configurare queste opzioni, effettuare le operazioni riportate di seguito.
    • Nel campo Intestazione richiesta SOAP immettere il nome di una variabile che deve contenere l'XML dell'elemento Intestazione SOAP per la richiesta di callout.

      • È necessario eseguire il wrapping del documento di input per l'intestazione della richiesta SOAP utilizzando <soap-env:Header>...</soap-env:Header>.

    • Nel campo Intestazione risposta SOAP immettere il nome di una variabile alla quale verrà associato l'XML dell'elemento Intestazioni SOAP.
    Parametri richiesta e
    Parametri risposta
    Per configurare queste opzioni, effettuare le operazioni riportate di seguito.
    • Nel campo Parametri richiesta immettere i nomi delle variabili che verranno valutate in fase di esecuzione per fornire i valori per i parametri di richiesta.

      • È necessario fornire solo i documenti del payload standard nella variabile di input; il package SOAP viene creato automaticamente da Oracle Service Bus. In altre parole, non eseguire il wrapping del documento di input utilizzando <soap-env:Body>...</soap-env:Body>.

      Ad esempio, durante la creazione di una variabile di input nel corpo utilizzata per questo parametro di richiesta, definire i contenuti di tale variabile utilizzando l'istruzione XPath body/* (per rimuovere il wrapper soap-env:Body) e non $body (che conserva il wrapper soap-env:Body).

    • Nei campi Parametri risposta immettere i nomi delle variabili alle quale verranno assegnate le risposte in fase di esecuzione.
    Documento richiesta e
    Documento risposta
    Per configurare queste opzioni, effettuare le operazioni riportate di seguito.
    • Nel campo Variabile documento di richiesta immettere il nome di una variabile da assegnare a un documento di richiesta.

      • Per i servizi di tipo Documento SOAP, la variabile viene valutata in fase di esecuzione per creare il corpo del messaggio SOAP inviato al servizio. Per i servizi di tipo Qualsiasi XML la variabile viene valutata in fase di esecuzione per creare il messaggio XML inviato al servizio.

      Per i servizi di tipo Documento SOAP e Qualsiasi XML, fornire solo i documenti del payload standard nella variabile di input; il package SOAP viene creato automaticamente da Oracle Service Bus. In altre parole, non eseguire il wrapping del documento di input utilizzando <soap-env:Body>...</soap-env:Body>.

      Ad esempio, durante la creazione di una variabile di input nel corpo utilizzata per questo parametro di richiesta, definire i contenuti di tale variabile utilizzando l'istruzione XPath body/* (per rimuovere il wrapper soap-env:Body) e non $body (che conserva il wrapper soap-env:Body).

      Per i servizi di tipo Messaggistica la variabile viene valutata per creare il corpo del messaggio, in base al tipo di dati previsti dal servizio. Di seguito sono riportate le limitazioni applicate alle variabili utilizzate con i servizi di tipo Messaggistica.

      • Per i servizi che prevedono dati binari, le varabili devono avere un elemento ctx:binary-content.
      • Per i servizi che prevedono dati MFL, la variabile deve avere l'equivalente XML.
      • Per i servizi che prevedono dati di testo, la variabile è una stringa.
    • Nel campo Variabile documento di risposta immettere il nome della variabile alla quale verrà assegnato un documento di risposta in fase di esecuzione.

  8. Aggiungere facoltativamente una o più intestazioni di trasporto. Per ulteriori informazioni, vedere Aggiunta di azioni per le intestazioni di trasposto.
    9. Nota: oltre alle intestazioni di trasporto specificate, vengono aggiunte intestazioni dal layer di associazione di Oracle Service Bus. Per ulteriori informazioni, vedere "Configuring Transport Headers in Message Flows" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

Al termine della procedura

Una volta completata la configurazione di questa azione, proseguire configurando altre azioni o salvando la configurazione, come descritto in Aggiunta e modifica di azioni nei flussi di messaggi.
