Pagina Configurazione trasporto FTP

La pagina Crea/Modifica un servizio proxy - Configurazione trasporto FTP fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio proxy mediante il protocollo di trasporto FTP. Nella Tabella 19-6 viene descritto come utilizzare questa pagina.

Tabella 19-6 Pagina Crea/Modifica un servizio proxy - Configurazione trasporto FTP 
Opzione
Per creare o modificare...
Autenticazione utente
Selezionare anonimo se l'utente del server FTP è anonimo oppure selezionare utente esterno se l'utente del server FTP è un account configurato esternamente.
Identità (ID di posta elettronica)
Questo campo è disponibile solo se l'opzione Autenticazione utente è impostata su anonimo.
Immettere l'ID di posta per l'utente anonimo.
Account di servizio
Questo campo è disponibile solo se l'opzione Autenticazione utente è impostata su utente esterno.
Immettere l'account di servizio per l'utente. Questo campo è obbligatorio quando l'opzione Autenticazione utente è impostata su utente esterno.
Passa in base a riferimento
Selezionare questa casella di controllo per inserire il file nella directory di archivio intermedia e passarlo come riferimento nelle intestazioni.
Streaming remoto
Selezionare questa casella di controllo per eseguire lo streaming dei file FTP direttamente dal server remoto durante l'elaborazione. Quando si seleziona questa opzione, la directory di archivio è la directory remota sul computer server FTP remoto. Pertanto, si consiglia di specificare la directory di archivio come relativa alla directory utente FTP.
Maschera di file
Immettere l'espressione regolare per i file da selezionare. L'impostazione predefinita è *.*.
Server gestito
Questo campo è disponibile solo in un dominio in cluster.
Selezionare il server gestito che deve fungere da server di polling. Tutti i server gestiti possono elaborare il messaggio, ma solo uno può eseguirne il polling.
Intervallo di polling
Immettere un intervallo di polling, in secondi. L'impostazione predefinita è 60.
Limite di lettura
Specificare il numero massimo di messaggi da leggere per cancellazione polling. Immettere 0 per specificare che non vi è alcun limite. L'impostazione predefinita è 10.
Azione successiva alla lettura
Selezionare l'azione che si verifica dopo la lettura di un messaggio.
  • Archivia: il messaggio viene archiviato.
  • Elimina: il messaggio viene eliminato.
Modalità di trasferimento
Selezionare ASCII o binario come modalità di trasferimento.
Directory di archivio
Specificare il percorso della posizione di archivio se l'opzione Azione successiva alla lettura è impostata su Archivia. Questo campo è obbligatorio se l'opzione Passa in base a riferimento è selezionata.
Nota: le directory di archivio, di scaricamento e degli errori sono percorsi assoluti e vengono create in modo automatico. Se si specifica un percorso relativo, i file vengono creati in relazione al processo Java che avvia il server WebLogic.
Directory di scaricamento
Immettere la directory sul computer locale in cui i file vengono scaricati durante il trasferimento del file.
Nota: le directory di archivio, di scaricamento e degli errori sono percorsi assoluti e vengono create in modo automatico. Se si specifica un percorso relativo, i file vengono creati in relazione al processo Java che avvia il server WebLogic.
Directory degli errori
Immettere la posizione in cui i messaggi vengono inviati se si verifica un problema.
Nota: le directory di archivio, di scaricamento e degli errori sono percorsi assoluti e vengono create in modo automatico. Se si specifica un percorso relativo, i file vengono creati in relazione al processo Java che avvia il server WebLogic.
Codifica richiesta
Accettare l'impostazione predefinita UTF-8 come codifica del set di caratteri per le richieste nei trasporti FTP.
Scansiona sottodirectory
Selezionare questa casella di controllo per eseguire la scansione di tutte le directory in modo ricorsivo.
Ordina per arrivo
Selezionare questa casella di controllo per consegnare gli eventi in ordine di arrivo.
Timeout
Immettere l'intervallo di timeout del socket, in secondi, prima che la connessione venga eliminata. Se si immette 0, non si verifica alcun timeout.
Conteggio nuovi tentativi
Specificare il numero di nuovi tentativi per gli errori di connessione FTP.

Al termine della procedura

Fare clic su Successivo per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare la configurazione nella pagina Crea/Modifica un servizio proxy - Riepilogo.
