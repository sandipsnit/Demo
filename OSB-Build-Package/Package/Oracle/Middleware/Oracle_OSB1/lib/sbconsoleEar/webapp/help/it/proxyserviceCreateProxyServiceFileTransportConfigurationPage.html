Pagina Configurazione trasporto file

La pagina Crea/Modifica un servizio proxy - Configurazione trasporto file fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio proxy mediante il protocollo di trasporto dei file. Nella Tabella 19-5 viene descritto come utilizzare questa pagina.

Tabella 19-5 Pagina Crea/Modifica un servizio proxy - Configurazione trasporto file 
Opzione
Per creare o modificare...
Maschera di file
Specifica i file di cui il servizio proxy deve eseguire il polling. Se l'URI è una directory e si specifica *.*, il servizio eseguirà il polling di tutti i file nella directory. Nella maschera di file sono consentiti solo i caratteri jolly * e ?. Le espressioni regolari non sono supportate.
Server gestito
Questo campo è disponibile solo in un dominio in cluster.
Selezionare il server gestito che deve fungere da server di polling. Tutti i server gestiti possono elaborare il messaggio, ma solo uno può eseguirne il polling.
Intervallo di polling
Immettere un intervallo di polling, in secondi. L'impostazione predefinita è 60.
Limite di lettura
Specificare il numero massimo di messaggi da leggere per cancellazione polling. Immettere 0 per specificare che non vi è alcun limite. L'impostazione predefinita è 10.
Ordina per arrivo
Selezionare questa casella di controllo per specificare che gli eventi vengono consegnati in ordine di arrivo.
Si noti che quando questa opzione viene selezionata per un servizio proxy eseguito in un ambiente in cluster, i messaggi vengono sempre inviati allo stesso server. In altre parole, il bilanciamento del carico tra i server viene ignorato quando questa opzione è selezionata.
Scansiona sottodirectory
Selezionare questa casella di controllo per eseguire la scansione di tutte le directory in modo ricorsivo.
Passa in base a riferimento
Selezionare questa casella di controllo per inserire il file nella directory di archivio intermedia e passarlo come riferimento nelle intestazioni.
Azione successiva alla lettura
Selezionare l'azione che si verifica dopo la lettura di un messaggio.
  • Archivia: il messaggio viene archiviato.
  • Elimina: il messaggio viene eliminato.
Directory intermedia
Immettere una directory intermedia in cui inserire temporaneamente i file durante l'elaborazione.
Non inserire la directory intermedia nella directory di polling, ovvero la directory identificata nell'URL del servizio proxy di trasporto file; ad esempio, file:///c:/dir1/dir2.
Directory di archivio
Specificare il percorso della posizione di archivio se l'opzione Azione successiva alla lettura è impostata su Archivia. Anche il campo Directory di archivio è un campo obbligatorio se è stato selezionato il campo Passa in base a riferimento.
Non inserire la directory di archivio nella directory di polling.
Directory degli errori
Immettere la posizione in cui i messaggi e gli allegati vengono inviati se si verifica un problema.
Non inserire la directory degli errori nella directory di polling.
Codifica richiesta
Accettare l'impostazione predefinita UTF-8 come codifica del set di caratteri per le richieste nei trasporti di file oppure immettere un'altra codifica del set di caratteri.

Al termine della procedura

Fare clic su Successivo per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare la configurazione nella pagina Crea/Modifica un servizio proxy - Riepilogo.
