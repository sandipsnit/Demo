Pagina Configurazione trasporto HTTP

Il trasporto HTTP ora supporta entrambi gli endpoint HTTP e HTTPS.

La pagina Crea/Modifica un servizio proxy - Configurazione trasporto HTTP fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio proxy mediante il protocollo di trasporto HTTP. Nella Tabella 19-7 viene descritto come utilizzare la pagina.

Tabella 19-7 Pagina Crea/Modifica un servizio proxy - Configurazione trasporto HTTP

HTTPS obbligatorio
Selezionare questa casella di controllo per gli endpoint HTTP in entrata.
Per ulteriori informazioni, vedere "Configuring Transport-Level Security" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
Autenticazione
Selezionare una delle opzioni riportate di seguito.
  • Nessuna: specifica che l'autenticazione non è richiesta.
  • Base: specifica che per accedere a questo servizio è richiesta l'autenticazione di base.

    • L'autenticazione di base indica a WebLogic Server di autenticare il client utilizzando un nome utente e password in base ai provider di autenticazione configurati nel realm di sicurezza, ad esempio il servizio di directory LDAP (Lightweight Directory Access Protocol) e Windows Active Directory. Il client deve inviare il nome utente e la password corrispondenti nell'intestazione della richiesta HTTP.

    L'autenticazione di base è sconsigliata su HTTP poiché la password viene inviata con testo in chiaro. Tuttavia, l'invio delle password su HTTPS è sicuro poiché HTTPS fornisce un canale crittografato.

    Avvertenza: per impostazione predefinita, tutti gli utenti (autorizzati e anonimi) possono accedere a un servizio proxy. Per limitare gli utenti che possono accedere a un servizio proxy, creare un criterio di autorizzazione a livello di trasporto. Vedere Modifica dei criteri di accesso a livello di trasporto.

  • Certificato client: specifica comunicazione cifrata e autenticazione client robusta (SSL bidirezionale). Per ulteriori informazioni, vedere "Configuring Transport-Level Security" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
  • Autenticazione personalizzata: specifica che un token di autenticazione è contenuto nell'intestazione HTTP. L'identità del client viene determinata mediante l'uso di questo token fornito dal client. È necessario configurare un provider di asserzione delle identità che esegua il mapping del token a un utente di Oracle Service Bus.

    • Il token di autenticazione personalizzato può essere qualsiasi tipo di token attivo supportato da un provider di asserzione delle identità di WebLogic Server configurato.

Criterio di spedizione
Selezionare un criterio di spedizione per questo endpoint. Lasciare vuoto questo campo per utilizzare il criterio di spedizione predefinito.
Il criterio di spedizione fa riferimento all'istanza di Work Manager di WLS che si desidera utilizzare per l'endpoint del servizio. Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codifica richiesta
  • Per i trasporti HTTP in entrata, tenere presente quanto riportato di seguito.

    • Se il parametro di codifica del set di caratteri dell'intestazione Content-Type non viene specificato nella richiesta client, immettere un parametro di codifica del set di caratteri. Se non si immette un valore, il campo viene impostato automaticamente su ISO-8859-1.

  • Per i trasporti HTTP in uscita, tenere presente quanto riportato di seguito.

    • Se non è stata configurata una codifica di richiesta, in fase di esecuzione Oracle Service Bus determina la codifica più appropriata mentre viene inviata una richiesta al servizio business. Nel caso di uno scenario non pass-through, la codifica di caratteri predefinita è UTF-8 in fase di esecuzione. Tuttavia, nel caso di uno scenario pass-through, in fase di esecuzione verrà eseguito il pass-through della codifica ricevuta con la risposta in uscita.

Codifica risposta
Accettare l'impostazione predefinita ISO-8859-1 come codifica del set di caratteri per le risposte nei trasporti HTTP oppure immettere un'altra codifica del set di caratteri.
Intestazione di autenticazione
Immettere qualsiasi intestazione HTTP (ad eccezione di Authorization) dalla quale Oracle Service Bus deve estrarre il token. Questo campo è disponibile solo se è stata selezionata la casella di controllo Autenticazione personalizzata.
Ad esempio, client-xyz-token.
Tipo di token di autenticazione
Selezionare un tipo di token di autenticazione. Sono disponibili solo i tipi di token attivi configurati per il provider di asserzione delle identità. Per ulteriori informazioni, vedere Configurazione dei provider di asserzione delle identità per i token personalizzati. Questo campo è disponibile solo se è stata selezionata la casella di controllo Autenticazione personalizzata.
