Pagina Configurazione trasporto JCA

Utilizzare questa pagina per configurare le impostazioni di trasporto mediante il protocollo di trasporto JCA. Per ulteriori informazioni sull'uso del protocollo di trasporto JCA, vedere "JCA Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV910 e Generazione di un servizio proxy da una risorsa di associazione JCA.

Tabella 19-8 Configurazione del trasporto JCA 
Opzione
Descrizione
File JCA
Fare clic su Sfoglia per selezionare un'associazione JCA. L'associazione JCA definisce diversi aspetti del servizio, quali i dettagli sull'adattatore utilizzato, un'associazione al file di mapping WSDL e TopLink o EclipseLink e le proprietà delle specifiche di attivazione/interazione richieste dal servizio.
Dopo aver selezionato un'associazione JCA valida, gli altri campi di configurazione del trasporto diventano disponibili.
Per ulteriori informazioni sulle associazioni JCA, vedere il Capitolo 12 relativo alle associazioni JCA.
Nome adattatore
Un valore di sola lettura che indica il nome dell'adattatore che verrà utilizzato dal servizio JCA.
Tipo di adattatore
Un valore di sola lettura che indica il tipo di adattatore.
Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager viene utilizzato quello predefinito.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Account di servizio JNDI
L'account di servizio JNDI è relativo alla sicurezza del contesto JNDI e viene utilizzato per accedere al Connection Factory gestito dall'adattatore EIS. Fare clic su Sfoglia e selezionare un account di servizio. Se non viene specificato alcun account di servizio, viene utilizzato un oggetto anonimo.
Per i servizi business JCA, non vi è alcuna limitazione sul tipo di account di servizio JNDI che è possibile configurare, ad esempio statico o pass-through, ma la fase di esecuzione deve essere in grado di accedere a un nome utente e password. I servizi proxy JCA possono utilizzare solo account di servizio JNDI statici.
Per ulteriori informazioni sugli account di servizio JNDI, vedere "Security" nel capitolo "JCA Transport" di Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV916.
Proprietà endpoint
Questo campo consente di assegnare valori alle proprietà dell'endpoint, quali nuovi tentativi per il tipo di adattatore utilizzato dal servizio.
Per una lista delle proprietà dell'endpoint JCA supportate, vedere "Endpoint Properties" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1841.
Proprietà dinamiche endpoint
Questa opzione consente di passare i parametri di richiesta ai servizi conformi a JCA. Ad esempio, è possibile utilizzare una proprietà dinamica dell'endpoint per passare i parametri delle query di database a Oracle JCA Adapter for Database.
Per ulteriori informazioni sull'esecuzione di query con parametri, vedere "Oracle JCA Adapter for Database" in Oracle Fusion Middleware User's Guide for Technology Adapters all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=TKADP2117.
Immettere una coppia nome/valore per ciascuna proprietà dinamica dell'endpoint che si desidera fornire. La chiave della proprietà dell'endpoint corrisponde al nome del parametro della query.
Utilizzare sempre la configurazione dal file JCA
Questa opzione determina se le proprietà delle specifiche di attivazione (servizi proxy) e le proprietà delle specifiche di interazione (servizi business) verranno sempre utilizzate dal file JCA.
Se questa opzione è selezionata (impostazione predefinita), il trasporto JCA interagisce con la struttura JCA mediante le proprietà della specifica di attivazione/interazione nel file JCA.
Se questa opzione è deselezionata, è possibile sostituire le proprietà delle specifiche di attivazione/interazione.
Per informazioni sull'impatto di ridistribuzione dell'uso di questa opzione, vedere "Endpoint Redeployment" nel capitolo "JCA Transport" di Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV940.
Nome operazione
Visualizza un nome di sola lettura dell'operazione WSDL selezionata. Un'operazione può avere proprietà delle specifiche di attivazione/interazione personalizzate, visualizzate nel campo Proprietà delle specifiche di attivazione/interazione.
Proprietà delle specifiche di attivazione/interazione
Proprietà delle specifiche di attivazione è il nome del campo per i servizi proxy; Proprietà delle specifiche di interazione è il nome del campo per i servizi business.
Se questo servizio è un servizio in entrata richiamato da un'applicazione EIS, questo campo visualizza le proprietà delle specifiche di attivazione per l'operazione in entrata JCA visualizzata nel campo Nome operazione.
È possibile sostituire le proprietà delle specifiche di attivazione/interazione se l'opzione Utilizzare sempre la configurazione dal file JCA è deselezionata.
Nota: per gli adattatori di Oracle Adapter Suite le proprietà delle specifiche di attivazione/interazione vengono visualizzate come proprietà di sola lettura. Questo tipo di adattatori memorizzano la propria configurazione, che deve essere modificata negli strumenti di gestione di Oracle Adapter Suite.
Proprietà di connessione (precedenti)
Per i servizi JCA precedenti che utilizzano le proprietà di connessione della modalità non gestita (non più valida in questa release), vedere le opzioni di configurazione della connessione all'indirizzo http://download.oracle.com/docs/cd/E13159_01/osb/docs10gr3/jcatransport/transport.html#wp1105451.

Per ulteriori informazioni sulle proprietà di endpoint e delle specifiche di attivazione/interazione, vedere Oracle Fusion Middleware User's Guide for Technology Adapters all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=TKADP.
