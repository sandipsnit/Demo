Pagina Configurazione trasporto JEJB

Utilizzare questa pagina per configurare le impostazioni di trasporto mediante il protocollo di trasporto JEJB. Per ulteriori informazioni sull'uso del trasporto JEJB, vedere "JEJB Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036.

Tabella 19-9 Configurazione del trasporto JEJB 
Opzione
Descrizione
Criterio di spedizione
Selezionare l'istanza del Work Manager di WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager viene utilizzato quello predefinito.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Versione specifica EJB
Selezionare la versione EJB dell'interfaccia EJB remota.
Passa XMLBean per valore
Selezionare questa opzione se si desidera che l'operazione di trasporto generi una rappresentazione XML "inserita in linea" degli argomenti POJO (un XMLObject). È possibile accedere e manipolare i parametri di questi argomenti con le espressioni XQuery.
Nota: le informazioni relative al tipo non sono disponibili in linea per gli XMLObject passati in base al valore. Se si utilizza questa opzione, non è possibile passare gli XMLObject digitati come argomento di un callout Java in una pipeline dei servizi proxy.
Non selezionare questa opzione se si desidera passare gli argomenti POJO in base al riferimento. Questa operazione determina anche prestazioni migliori.
Attributo transazione
Selezionare una delle opzioni riportate di seguito per la gestione delle transazioni.
Supporta: il trasporto accetta una transazione in entrata. La qualità del servizio è impostata su Esattamente una volta se l'operazione viene richiamata in una transazione e su Miglior tentativo se l'operazione viene richiamata all'esterno di una transazione.
Obbligatorio: il trasporto accetta una transazione in entrata. Se non esistono transazioni in corso, il trasporto ne avvia una. La qualità del servizio è impostata su Esattamente una volta.
RequiresNew: il trasporto avvia sempre una nuova transazione, sospendendone una in corso. La qualità del servizio è impostata su Esattamente una volta.
Obbligatorio: il trasporto richiama il metodo nella transazione esistente. La qualità del servizio è impostata su Esattamente una volta.
NotSupported: il trasporto sospende una transazione esistente e la riprende non appena viene richiamata. La qualità del servizio è impostata su Miglior tentativo.
Mai: il trasporto non richiama il metodo in una transazione. La qualità del servizio è impostata su Miglior tentativo.
JAR client
Fare clic su Sfoglia e selezionare una risorsa JAR client EJB dalla lista visualizzata. Il file JAR client contiene l'interfaccia remota o business per l'EJB remoto. Il file JAR client viene registrato come risorsa di archivio generica.
Interfaccia home
Solo EJB 2.1: selezionare l'interfaccia EJBHome necessaria dalle opzioni popolate dal file JAR client.
Interfaccia remota
Solo EJB 2.1: questo campo viene popolato automaticamente in base alla configurazione dell'interfaccia home.
Interfaccia business
Solo EJB 3.0: selezionare l'interfaccia business dal file JAR client che si desidera richiamare.
Spazio di nomi di destinazione
Questo campo viene popolato con le informazioni selezionate dal file JAR.
Metodi
Selezionare i metodi richiesti. Fare clic su + per espandere il metodo, che consente di modificare i valori dei parametri predefiniti.
È possibile modificare il nome dell'operazione predefinita per un determinato metodo. Per impostazione predefinita, il nome dell'operazione è il nome del metodo. Se un EJB contiene metodi con lo stesso nome (sovraccarico), è necessario modificare i nomi delle operazioni in modo da renderli univoci. I WSDL richiedono nomi di operazioni univoci.

Al termine della procedura

Fare clic su Successivo per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare la configurazione nella pagina Crea/Modifica un servizio proxy - Riepilogo.
