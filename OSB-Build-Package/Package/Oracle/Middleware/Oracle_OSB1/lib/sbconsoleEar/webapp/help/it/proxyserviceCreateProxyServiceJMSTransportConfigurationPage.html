Pagina Configurazione trasporto JMS

La pagina Crea/Modifica un servizio proxy - Configurazione trasporto JMS fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Per ulteriori informazioni, vedere "JMS Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio proxy mediante il protocollo di trasporto JMS. Nella Tabella 19-10 viene descritto come utilizzare la pagina.

Tabella 19-10 Pagina Crea/Modifica un servizio proxy - Configurazione trasporto JMS 
Opzione
Per creare o modificare...
Tipo di destinazione
Selezionare una delle opzioni riportate di seguito.
  • Coda (per un tipo di destinazione point-to-point)
  • Argomento (per una destinazione di pubblicazione/sottoscrizione)
Risposta richiesta
Questa opzione è disponibile solo se è selezionata l'opzione Coda per Tipo di destinazione.
Selezionare questa opzione per specificare che è prevista una risposta dopo l'invio di un messaggio in uscita.
Pattern risposta
Questa opzione è disponibile solo se la casella di controllo Risposta richiesta è selezionata.
Selezionare una delle opzioni riportate di seguito.
  • Selezionare JMSMessageID per i servizi JAX-RPC in esecuzione su Oracle WebLogic Server.
  • Selezionare JMSCorrelationID per tutti gli altri servizi. Quando si seleziona questa opzione, è anche necessario immettere un URI risposta.
Tipo di messaggio di risposta
Questa opzione è disponibile solo se la casella di controllo Risposta richiesta è selezionata.
Selezionare una delle opzioni riportate di seguito.
  • Byte (per un flusso di byte senza interpretazione)
  • Testo (per i messaggi di testo)
Questa opzione è disabilitata se si seleziona un tipo di messaggio Java per la risposta.
JAR client
Questa opzione è disponibile quando il servizio è un servizio di messaggistica con un tipo di richiesta Java. Selezionare il file JAR client da utilizzare per rimuovere dalla coda i messaggi che contengono oggetti Java. La selezione del file JAR client assicura che si trovi nel classpath.

Per ulteriori informazioni, vedere "Sending and Receiving Java Objects in Messages" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Criterio di spedizione
Selezionare l'istanza del Work Manager di Oracle WebLogic Server che si desidera utilizzare per il criterio di spedizione per questo endpoint. Se non esistono altri Work Manager viene utilizzato quello predefinito.
Ad esempio, se il servizio business ha un protocollo di trasporto JMS, l'endpoint del servizio business è un file JAR MDB (Message-Driven Bean, bean basati su messaggi) che è possibile associare al criterio di spedizione specifico.
Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codifica richiesta
Immettere il set di caratteri per la codifica delle richieste. L'impostazione predefinita è UTF-8.
Codifica risposta
Questa opzione è disponibile solo se la casella di controllo Risposta richiesta è selezionata.
Immettere il set di caratteri per la codifica delle risposte. L'impostazione predefinita è UTF-8.
Timeout risposta client
Questa opzione è disponibile solo se la casella di controllo Risposta richiesta è selezionata.
Immettere il numero di secondi di attesa per la risposta del server prima di interrompere la connessione. Questa opzione è valida solo se il client è un altro servizio proxy nello stesso dominio.
URI risposta
Questa opzione è disponibile solo se l'opzione JMSCorrelationID è selezionata per il pattern di correlazione della risposta.
Immettere un URI di risposta nel formato:
jms://host:port/connection_factory/jndi_destination
Per specificare la destinazione di più server, utilizzare il formato seguente:
jms://host1:port,host2:port/connection_factory/jndi_destination
È possibile omettere anche l'host e la porta nell'URI di risposta. Ad esempio:
jms:///connection_factory/jndi_destination

Se si omette l'host e la porta, la ricerca del Connection Factory e/o della destinazione viene eseguita sul server locale. Ciò risulta utile, ad esempio, se l'URI di richiesta accede a un Connection Factory o a una destinazione esterna, ma si desidera inviare la risposta al server locale.

Nota: anche se Oracle WebLogic Server consente l'utilizzo delle barre nei nomi JNDI, ad esempio "myqueues/myqueue", i nomi JNDI contenenti barre interferiscono con il formato URI richiesto da Oracle Service Bus e non sono pertanto utilizzabili. Una soluzione alternativa a questo problema consiste nel definire un server JMS esterno e farvi riferimento nell'URI. Per ulteriori informazioni, vedere "Configure foreign servers" in Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Connection Factory risposta
Questa opzione è disponibile solo se l'opzione JMSMessageID è selezionata per il pattern di correlazione della risposta.
Immettere un URI del Connection Factory di risposta.
Se un Connection Factory non è specificato, per la risposta viene utilizzato il Connection Factory della richiesta.
Account di servizio JMS
Selezionare un account di servizio da utilizzare per la risorsa JMS gestita dal server JMS. Un account di servizio è una risorsa alias per un ID utente e password, utilizzata sia per la richiesta che per la risposta. Lo stesso account di servizio viene utilizzato per entrambi gli scopi JMS e JNDI.
Per ulteriori informazioni, vedere Account di servizio.
Usa SSL
Selezionare questa opzione solo se le richieste vengono effettuate su una connessione TLS/SSL.
TLS/SSL (Secure Sockets Layer) fornisce connessioni protette consentendo a due applicazioni connesse a una rete di eseguire l'autenticazione dell'identità dell'altra e crittografando i dati scambiati tra le applicazioni. L'autenticazione consente a un server, e facoltativamente a un client, di verificare l'identità dell'applicazione sull'altra estremità di una connessione di rete. Inoltre, se l'amministrazione ha limitato l'accesso alle singole destinazioni JMS (code o argomenti) impostando il controllo dell'accesso sulla voce JNDI per la destinazione, il servizio deve eseguire l'autenticazione quando esegue la ricerca della voce nella struttura JNDI.
Selettore messaggi
Immettere un'espressione del selettore dei messaggi.
Vengono elaborati solo i messaggi con le proprietà corrispondenti all'espressione.
ID client JMS

Solo per gli argomenti, immettere l'ID client da utilizzare per il sottoscrittore. Se non viene immesso alcun valore, viene generato automaticamente un ID client. L'assegnazione di un ID personalizzato consente di identificare questo componente durante il monitoraggio del servizio. Tenere presente che questo particolare funzionamento riguarda esclusivamente i sottoscrittori permanenti.

L'ID client JMS è uno dei numerosi descrittori MDB utilizzati per generare i nomi dei sottoscrittori e il valore ID client per i sottoscrittori dell'argomento. La definizione di un ID client JMS facilita l'identificazione e la visualizzazione dei sottoscrittori di un argomento specifico mediante l'individuazione del nome dell'ID client del sottoscrittore.

Per ulteriori informazioni, vedere "JMS Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.
Sottoscrizione permanente
Questa opzione è disponibile solo se è selezionata l'opzione Argomento per Tipo di destinazione.
Selezionare questa casella di controllo se la sottoscrizione è permanente oppure lasciare la casella di controllo vuota se la sottoscrizione non è permanente.
Conteggio nuovi tentativi
Immettere il numero di nuovi tentativi di consegna che è possibile assegnare a un messaggio prima che venga spostato nella destinazione degli errori. Questo campo è valido solo per le destinazioni JMS di Oracle WebLogic Server.
Intervallo nuovi tentativi
Immettere la quantità di tempo, in millisecondi, prima di eseguire di nuovo la consegna dei messaggi sottoposti a rollback o recuperati. Questo campo è valido solo per le destinazioni JMS di Oracle WebLogic Server.
Destinazione errore
Immettere il nome della destinazione per i messaggi che hanno raggiunto il limite di riconsegna definito. Questo campo è valido solo per le destinazioni JMS di Oracle WebLogic Server.
Criterio di scadenza
Selezionare un criterio di scadenza da utilizzare quando viene individuato un messaggio scaduto su una destinazione. Questo campo è valido solo per le destinazioni JMS di Oracle WebLogic Server.
XA richiesto
Selezionare questa casella di controllo se il Connection Factory è XA.
Questo valore viene preso in considerazione quando il Connection Factory remoto non è disponibile. Se il Connection Factory è disponibile e questo valore è impostato su true, assicurarsi che il Connection Factory sia definito come transazionale.
Timeout transazione
Immettere il periodo di tempo in secondi per cui il servizio proxy JMS deve attendere l'elaborazione di una transazione. Questa opzione è valida solo per i servizi che utilizzano un Connection Factory XA. Il valore predefinito è 600 secondi.
Distribuzione messaggi argomento
Questa opzione è disponibile quando si seleziona Argomento per Tipo destinazione. Distribuzione messaggi argomento offre varie opzioni che consentono di gestire i messaggi JMS in entrata e fornire High Availability e failover.

Selezionare una delle proprietà riportate di seguito per determinare le modalità di gestione dei messaggi JMS in entrata da parte dei bean basati su messaggi.
  • Una copia per applicazione (impostazione predefinita) – Se il servizio proxy JMS viene distribuito a un cluster, questa opzione fornisce elevata disponibilità e scalabilità assicurando che un messaggio JMS in entrata venga elaborato soltanto su uno dei server disponibili in un cluster.
  • Una copia per server – Selezionare questa opzione se si desidera che i messaggi JMS in entrata pubblicati in un argomento vengano ricevuti dal servizio proxy su ciascun membro del cluster.
  • Compatibilità – Selezionare questa opzione se si desidera che i messaggi JMS in entrata vengano elaborati su un server gestito specifico oppure su tutti i server gestiti di un cluster. Quando si seleziona questa proprietà è necessario selezionare un server di destinazione.
Nota: per le destinazioni di tipo Argomento distribuite in Oracle WebLogic Server prima della versione 10.3.4, l'unica opzione valida è Compatibilità.
Le opzioni "Una copia" selezionate per Distribuzione messaggi argomento sostituiscono il criterio di condivisione della sottoscrizione e/o il criterio ID client configurati nel Connection Factory JMS.
Oggetto
Questa opzione è disponibile solo in un cluster Oracle Service Bus quando si seleziona Compatibilità per l'opzione Distribuzione messaggi argomento. Selezionare il server di destinazione che dovrà gestire i messaggi JMS in entrata. Se si seleziona una delle opzioni "Una copia" per Distribuzione messaggi argomento, questo campo visualizzerà il nome del cluster.

Se non si imposta una destinazione, l'istanza del servizio proxy JMS (per la lettura dei messaggi dall'argomento) di ogni server gestito del cluster riceverà una copia del messaggio.
Timeout JNDI
Il timeout di connessione JNDI (in secondi) utilizzato durante la ricerca della destinazione o del Connection Factory nella struttura JNDI.
