Pagina Configurazione trasporto MQ

La pagina Crea/Modifica un servizio proxy - Configurazione trasporto MQ fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Per ulteriori informazioni, vedere "MQ Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Prima di iniziare

Configurare una risorsa di connessione MQ. Vedere Connessioni MQ.

Per configurare il trasporto MQ, effettuare le operazioni riportate di seguito.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio proxy utilizzando il protocollo di trasporto MQ. Nella tabella 19-11 viene descritto come utilizzare la pagina.

Tabella 19-11 Crea/Modifica un servizio proxy - Pagina Configurazione trasporto MQ
Opzione
Per creare o modificare...
Intervallo di polling
Immettere un intervallo di polling in millisecondi. L'impostazione predefinita è 1000.
Risposta richiesta
Selezionare questa opzione per specificare che è prevista una risposta dopo l'invio di un messaggio in uscita.
Pattern di correlazione risposte
Questa opzione è disponibile solo se è selezionata la casella di controllo Risposta richiesta.
Specificare se il pattern di correlazione risposte deve essere basato su MessageIDo su CorrelationID.
URI risposta MQ
Questa opzione è disponibile solo se è selezionata la casella di controllo Risposta richiesta.
La destinazione in cui pubblicare la risposta. Immettere un URI di risposta nello stesso formato dell'URI endpoint: mq://<local-queue-name>?conn=<mq-connection-resource-ref>
Tipo di messaggio di risposta
Questa opzione è disponibile solo se è selezionata la casella di controllo Risposta richiesta.
Selezionare una delle opzioni riportate di seguito.
  • Byte (per un flusso di byte non interpretati)
  • Testo (per i messaggi di testo)
Criterio di spedizione
Selezionare un criterio di spedizione per l'endpoint.
Il criterio di spedizione fa riferimento all'istanza di WLS Work Manager che si desidera utilizzare per l'endpoint del servizio. Per informazioni sui Work Manager, vedere "Using Work Managers to Optimize Scheduled Work" in Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Soglia di mancata consegna
Immettere un valore indicante il numero di volte che la pipeline deve ritentare l'invio di un messaggio prima di reindirizzarlo alla coda specificata nel campo URI messaggi non recapitabili.
Se non si specifica un valore per questo campo, il messaggio viene reindirizzato alla coda dei messaggi non recapitabili senza effettuare nuovi tentativi.
URI messaggi non recapitabili MQ
Immettere l'URI della coda di messaggi non recapitabili alla quale i messaggi di richiesta devono essere reindirizzati dopo avere effettuato il numero di tentativi specificato nel campo Soglia di mancata consegna.
Se non si specifica un valore per questo campo, il messaggio viene restituito alla coda e ignorato dal trasporto MQ per ciascuna operazione di polling dopo l'esecuzione del numero di tentativi specificato nel campo Soglia di mancata consegna. L'URI messaggi non recapitabili utilizza lo stesso formato dell'URI endpoint.
Opzioni 'GET' dell'URI endpoint
Immettere una delle seguenti opzioni dei messaggi GET MQ:
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
Per separare più opzioni, utilizzare "|" o "+". Ad esempio, è possibile specificare quanto segue:
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
Le opzioni dei messaggi GET MQ vengono applicate durante la lettura di un messaggio dalla coda in entrata.
Elabora intestazioni RFH2
Selezionare questa opzione per analizzare le intestazioni RFH2 MQ WebSphere di un payload del messaggio e generare in modo automatico un'intestazione di trasporto RFH2Headers contenente i dati RFH2.
Se non si seleziona questa opzione, il payload viene passato così come viene ricevuto.

Per informazioni sul modo in cui il trasporto MQ gestisce le intestazioni RFH2, vedere "About RFH2 Headers" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.

Al termine della procedura

Fare clic su Successivo per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare la configurazione nella pagina Crea/Modifica un servizio proxy - Riepilogo.
