Pagina Configurazione tipo di messaggio

La pagina Crea/Modifica un servizio proxy - Configurazione tipo di messaggio fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Utilizzare questa pagina per configurare i tipi di messaggio per un servizio proxy di tipo Servizio messaggistica.

La definizione dell'associazione per i servizi di messaggistica consistono nella configurazione del content-type dei messaggi scambiati. Il content-type per la risposta non deve essere necessariamente uguale a quello della richiesta; pertanto, la risposta viene configurata separatamente (ad esempio, il servizio può accettare un messaggio MFL e restituire una conferma di ricezione XML).

Nota: i servizi proxy di trasporto posta elettronica, file, FTP o SFTP di tipo Servizio di messaggistica supportano solo la messaggistica unidirezionale; Tipo di messaggio di risposta deve essere impostato su nessuno. Se si seleziona un'opzione diversa da nessuno, i protocolli di posta elettronica, file, FTP o SFTP non saranno disponibili nella pagina Configurazione trasporto.

Nella tabella 19-2 viene descritto come utilizzare la pagina Crea/Modifica un servizio proxy - Configurazione tipo di messaggio.

Tabella 19-2 Crea/Modifica un servizio proxy - Pagina Configurazione tipo di messaggio
Opzione
Per creare o modificare...
Tipo di messaggio di richiesta
Selezionare un tipo di messaggio per il messaggio di richiesta.
  • Nessuno: selezionare questa opzione se non c'è alcun messaggio di richiesta.
  • Binario: selezionare questa opzione se il tipo di contenuto del messaggio è sconosciuto o non importante.
  • Testo: selezionare questa opzione se il messaggio può essere limitato al testo.
  • MFL: selezionare questa opzione se il messaggio è un documento binario conforme a una definizione MFL. Immettere il nome del file MFL (con il relativo percorso) o fare clic su Sfoglia per selezionare un file.

    • È possibile configurare solo un file MFL.

    Nota: per supportare più file MFL, definire il contenuto come binario o testo e utilizzare l'azione MFL nel flusso di messaggi per eseguire la conversione in XML.

  • XML: selezionare questa opzione se il messaggio è un documento XML. Immettere il nome del file XML (con il relativo percorso) o fare clic su Sfoglia per selezionare un file.

    • Facoltativamente fornire alcune informazioni sul tipo dichiarando (nel campo elemento o tipo) il tipo di schema XML del documento XML scambiato.

  • Java: selezionare questa opzione se si invia un oggetto Java nella richiesta. Per i messaggi contenenti oggetti Java viene utilizzato il trasporto JMS. Per ulteriori informazioni, vedere "Sending and Receiving Java Objects in Messages" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Tipo di messaggio di risposta
Selezionare un tipo di messaggio per il messaggio di risposta.
  • Nessuno: selezionare questa opzione se non vi sono messaggi di risposta.
  • Binario: selezionare questa opzione se il tipo di contenuto del messaggio è sconosciuto o non importante.
  • Testo: selezionare questa opzione se il messaggio può essere limitato al testo.
  • MFL: selezionare questa opzione se il messaggio è un documento binario conforme a una definizione MFL. Immettere il nome del file MFL (con il relativo percorso) o fare clic su Sfoglia per selezionare un file.

    • È possibile configurare solo un file MFL.

    Nota: per supportare più file MFL, definire il contenuto come binario o testo e utilizzare l'azione MFL nel flusso di messaggi per eseguire la conversione in XML.

  • XML: selezionare questa opzione se il messaggio è un documento XML. Immettere il nome del file XML (con il relativo percorso) o fare clic su Sfoglia per selezionare un file.

    • Facoltativamente fornire alcune informazioni sul tipo dichiarando (nel campo elemento o tipo) il tipo di schema XML del documento XML scambiato.

  • Java: selezionare questa opzione si riceve un oggetto Java nella risposta. Per i messaggi contenenti oggetti Java viene utilizzato il trasporto JMS. Per ulteriori informazioni, vedere "Sending and Receiving Java Objects in Messages" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.

Al termine della procedura

Fare clic su Successivo per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare la configurazione nella pagina Crea/Modifica un servizio proxy - Riepilogo.
