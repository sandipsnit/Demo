Pagina Configurazione trasporto SFTP

La pagina Crea/Modifica un servizio proxy - Configurazione trasporto SFTP fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Per ulteriori informazioni, vedere "HTTP and Poller Transports" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV947.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio proxy utilizzando il protocollo di trasporto SFTP. Nella tabella 19-13 viene descritto come utilizzare questa pagina.

Tabella 19-13 Crea/Modifica un servizio proxy - Pagina Configurazione trasporto SFTP
Opzione
Per creare o modificare...
Autenticazione utente
Selezionare una delle opzioni riportate di seguito.
  • Autenticazione nome utente/password - Specifica che un account di servizio statico è associato al metodo di autenticazione e il client viene autenticato utilizzando le credenziali fornite.
  • Autenticazione basata su host - Specifica che sono richiesti un nome utente e un provider di chiavi di servizio per utilizzare il metodo di autenticazione. Qualsiasi utente che si connette da un host noto viene autenticato utilizzando la chiave privata dell'host.
  • Autenticazione con chiave pubblica - Specifica che sono richiesti un nome utente e un provider di chiavi di servizio per utilizzare il metodo di autenticazione. Ogni utente ha la propria chiave privata.
Account di servizio
Immettere l'account di servizio per l'utente o fare clic su Sfoglia per selezionare gli account di servizio da un browser.
Provider di chiavi servizio
Questa opzione è disponibile solo quando è selezionata l'autenticazione basata su host o con chiave pubblica.
Immettere un provider di chiavi di servizio nel campo Provider di chiavi servizio. È possibile fare clic su Sfoglia per selezionare i provider di chiavi di servizio da un browser. Questo campo è obbligatorio.
Nome utente
Questa opzione è disponibile solo quando è selezionata l'autenticazione basata su host o con chiave pubblica.
Immettere il nome utente.
Passa in base a riferimento
Selezionare questa casella di controllo per inserire il file nella directory di archivio intermedia e passarlo come riferimento nelle intestazioni.
Streaming remoto
Selezionare questa casella di controllo per eseguire lo streaming dei file SFTP direttamente dal server remoto al momento dell'elaborazione. Quando si seleziona questa opzione, la directory di archivio è la directory remota sul computer del server SFTP remoto. Pertanto, è necessario specificare la directory di archivio in relazione alla directory utenti SFTP.
Maschera di file
Immettere l'espressione regolare per i file da selezionare. L'impostazione predefinita è *.*.
Server gestito
Questo campo è disponibile solo in un dominio in cluster.
Selezionare il server gestito che deve fungere da server di polling. Tutti i server gestiti possono elaborare il messaggio, ma solo uno può eseguirne il polling.
Intervallo di polling
Immettere l'intervallo in secondi con il quale viene eseguito il polling del file dalla posizione specificata. Il valore predefinito è 60.
Limite di lettura
Specificare il numero massimo di messaggi da leggere per cancellazione polling. Immettere 0 per specificare che non vi è alcun limite. L'impostazione predefinita è 10.
Azione successiva alla lettura
Selezionare l'operazione da eseguire sul messaggio dopo la lettura dello stesso.
  • Archivia: il messaggio viene archiviato.
  • Elimina: il messaggio viene eliminato.
Directory di archivio
Specificare il percorso dell'archivio se l'opzione Azione successiva alla lettura è impostata su Archivia. Questo campo è obbligatorio se è selezionata l'opzione Passa in base a riferimento.
Nota: le directory di archivio, di scaricamento e degli errori sono percorsi assoluti e vengono creati automaticamente. Se si specifica un percorso relativo, i file vengono creati in relazione al processo Java che avvia WebLogic Server.
Directory di scaricamento
Immettere la directory sul computer locale in cui i file vengono scaricati durante il trasferimento.
Nota: le directory di archivio, di scaricamento e degli errori sono percorsi assoluti e vengono creati automaticamente. Se si specifica un percorso relativo, i file vengono creati in relazione al processo Java che avvia WebLogic Server.
Directory degli errori
Immettere la posizione in cui i messaggi vengono inviati in caso di problemi.
Nota: le directory di archivio, di scaricamento e degli errori sono percorsi assoluti e vengono creati automaticamente. Se si specifica un percorso relativo, i file vengono creati in relazione al processo Java che avvia WebLogic Server.
Codifica richiesta
Accettare l'impostazione predefinita UTF-8 come codifica del set di caratteri per le richieste nei trasporti SFTP.
Suite di cifratura preferita
Selezionare la suite di cifratura da utilizzare durante la comunicazione con il server.
Se si utilizza il valore predefinito Usa valore predefinito fase di esecuzione, la lista di suite di cifratura supportate viene inviata al server e vengono provate tutte finché non viene trovata una corrispondenza.
Algoritmo di integrità dei dati preferito
Selezionare l'algoritmo hash di grandi quantità di dati per i controlli di integrità dei dati.
Se si utilizza il valore predefinito, che prevede l'uso dell'impostazione predefinita di runtime, Oracle Service Bus invia l'algoritmo preferito hmac-sha1 al server. Se tale algoritmo non è supportato dal server, viene inviata la lista degli algoritmi supportati e verrà provato ogni algoritmo finché non verrà trovata una corrispondenza.
Algoritmo di chiave pubblica preferito
Selezionare l'algoritmo di chiave asimmetrica per la crittografia con chiave pubblica.
Se si utilizza il valore predefinito, che prevede l'uso dell'impostazione predefinita di runtime, Oracle Service Bus invia l'algoritmo preferito ssh-dss al server. Se tale algoritmo non è supportato dal server, viene inviata la lista degli algoritmi supportati e verrà provato ogni algoritmo finché non verrà trovata una corrispondenza.
Algoritmo di scambio chiavi preferito
Selezionare il protocollo di scambio predefinito per negoziare la chiave di sessione per la cifratura del messaggio.
Se si utilizza il valore predefinito, che prevede l'uso dell'impostazione predefinita di runtime, Oracle Service Bus invia l'algoritmo preferito diffie-hellman-group1-sha1 al server. Se tale algoritmo non è supportato dal server, viene inviata la lista degli algoritmi supportati e verrà provato ogni algoritmo finché non verrà trovata una corrispondenza.
Algoritmo di compressione preferito
Specificare se è necessario comprimere i dati di esecuzione utilizzando ZLIB. Selezionare zlib per comprimere i dati, altrimenti selezionare Nessuno. Il valore predefinito è Nessuno.
Scansiona sottodirectory
Selezionare questa casella di controllo per eseguire la scansione di tutte le directory in modo ricorsivo.
Ordina per arrivo
Selezionare questa casella di controllo per consegnare gli eventi in ordine di arrivo.
Timeout
Immettere l'intervallo di timeout del socket, in secondi, prima che la connessione venga rilasciata. Se si immette 0, non viene impostato alcun timeout. Il valore predefinito è 60.
Conteggio nuovi tentativi
Specificare il numero di nuovi tentativi per gli errori di connessione SFTP.

Al termine della procedura

Fare clic su Successivo per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare la configurazione nella pagina Crea/Modifica un servizio proxy - Riepilogo.
