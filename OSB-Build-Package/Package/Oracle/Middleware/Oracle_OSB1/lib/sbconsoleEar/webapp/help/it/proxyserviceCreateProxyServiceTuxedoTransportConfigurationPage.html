Pagina Configurazione trasporto Tuxedo

La pagina Crea/Modifica un servizio proxy - Configurazione trasporto Tuxedo fa parte di una serie di pagine per la creazione e la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Per ulteriori informazioni, vedere "Tuxedo Transport" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Utilizzare questa pagina per configurare le impostazioni di trasporto per un servizio proxy utilizzando il protocollo di trasporto Tuxedo. Nella tabella 19-14 viene descritto come utilizzare la pagina.

Tabella 19-14 Pagina Crea/Modifica un servizio proxy - Configurazione trasporto Tuxedo
Opzione
Per creare o modificare...
Classi tabelle di campi
Immettere il nome della classe o delle classi che descrivono il buffer FML/FML32 ricevuto. Tali classi sono utilizzate per le routine di conversione FML/da FML32 a XML per mappare i nomi dei campi ai nomi degli elementi. Si tratta di una lista di nomi di classi completamente qualificati separati da spazi.
Classi viste
Immettere il nome della classe o delle classi che descrivono il buffer VIEW/VIEW32 ricevuto o inviato. Tali classi sono utilizzate per le routine di conversione VIEW-to-XML o VIEW32-to-XML per mappare i nomi di campo ai nomi di elemento. Si tratta di una lista di nomi di classi completamente qualificati separati da spazi.
I tipi di buffer Tuxedo X_C_TYPE e X_COMMON sono gestiti allo stesso modo dei buffer VIEW/VIEW32.
Se una richiesta in entrata contiene VIEW, è necessario specificare la classe VIEW corrispondente nel CLASSPATH di Oracle Service Bus.
File JAR classi
Selezionare una risorsa JAR contenente un file JAR con le classi FML/FML32 o VIEW/VIEW32 necessarie per l'operazione dell'endpoint.
Punto di accesso locale
Selezionare dall'elenco a discesa un punto di accesso locale associato all'esportazione. L'elenco a discesa contiene punti di accesso locali configurati in WTC. Non è possibile creare un servizio proxy se non è disponibile un punto di accesso locale associato.
Se non esiste un punto di accesso locale o per crearne uno nuovo, selezionare Nuovo. Immettere i valori corrispondenti nei campi Nome punto di accesso locale e Indirizzo di rete locale. In seguito alla convalida dell'endpoint, il punto di accesso viene aggiunto alla configurazione WTC per ciascun server WTC. Se non esiste alcun server WTC, ne viene creato uno.
È possibile immettere il nome di un punto di accesso esistente dopo aver selezionato l'opzione Nuovo. In questo modo, le informazioni esistenti vengono aggiornate con i nuovi parametri. È possibile modificare solo il nome host e il numero di porta.
Punto di accesso remoto
Questo campo è disponibile solo se si seleziona Nuovo nel campo Punto di accesso locale.
Dall'elenco a discesa selezionare un punto di accesso remoto da associare al punto di accesso locale appena creato. Se non ne esiste alcuno o per crearne uno nuovo, selezionare Nuovo. Immettere i valori corrispondenti nei campi Nome punto di accesso e Indirizzo di rete.
È possibile immettere il nome di un punto di accesso esistente dopo aver selezionato l'opzione Nuovo. In questo modo, le informazioni esistenti vengono aggiornate con i nuovi parametri. È possibile modificare solo il nome host e il numero di porta.
Il punto di accesso remoto sarà anche il principal di autenticazione per la connessione WTC per le richieste in entrata. Facoltativamente è possibile creare un utente con lo stesso ID punto di accesso nel realm di sicurezza predefinito per consentire le chiamate in entrata. A tale scopo, selezionare Sì dall'elenco a discesa Creare l'utente?. La password verrà generata a caso utilizzando una variabile temporanea per evitare problemi di sicurezza.
Tipo di buffer di risposta
Questa opzione è disponibile solo se è selezionato il campo È richiesta la risposta?.
Selezionare il tipo di buffer che il client Tuxedo remoto riceverà.
Tipo secondario di buffer di risposta
Questa opzione è disponibile solo se è selezionata l'opzione È richiesta la risposta? e il valore di Tipo di buffer di risposta è VIEW o VIEW32.
Immettere il tipo secondario di buffer al quale associare il buffer di risposta.
È richiesta la risposta?
Selezionare questa casella di controllo se è previsto l'invio di una risposta da parte del servizio.
Per impostazione predefinita, questa opzione è selezionata.
Questa opzione è deselezionata e non disponibile se il tipo di servizio è Servizio di messaggistica e il tipo di messaggio di risposta è Nessuno.
Codifica richiesta
Specificare una codifica del set di caratteri per le richieste nei trasporti Tuxedo.
Codifica risposta
Specificare una codifica del set di caratteri per le risposte nei trasporti Tuxedo.
Stile di trasformazione
Selezionare una delle opzioni riportate di seguito.
  • Nessuno: (impostazione predefinita) l'ordine dei campi potrebbe non essere rispettato.
  • Ordinato: i campi vengono visualizzati con tutte le relative ricorrenze nell'ordine corretto.
  • Ordinato e raggruppato: se sono ordinati in modo logico come record, i campi vengono ordinati in base alla ricorrenza e raggruppati in base ai record.

Al termine della procedura

Fare clic su Successivo per proseguire con la configurazione di questo servizio nella pagina successiva oppure fare clic su Ultimo per esaminare e salvare la configurazione nella pagina Crea/Modifica un servizio proxy - Riepilogo.
