Creazione di strutture di variabili negli editor XQuery

La tavolozza Strutture variabili negli editor XQuery e XPath visualizza rappresentazioni grafiche del contenuto delle variabili. Include per impostazione predefinita le variabili di contesto del messaggio incorporate attachments, body, header, outbound e inbound.

Ogni voce di mapping di una struttura di variabile ha un'etichetta e mappa una variabile o un percorso di variabile a una o più strutture. L'ambito di questi mapping è una fase o un nodo di instradamento.

È possibile anche dichiarare strutture di variabili personalizzate, in base a quanto segue:

  • Tipi XML, inclusi quelli riportati di seguito:
    • Elementi dello schema
    • Elementi WSDL
    • Tipi di schema
    • Tipi WSDL
    • MFL
  • Interfacce di servizio
  • Tipi semplici (Stringa o Qualsiasi XML)

È possibile utilizzare questa funzione direttamente per tutte le variabili definite dall'utente, nonché per $inbound, $outbound e $fault. Tuttavia, non è possibile utilizzarla direttamente per accedere agli allegati XML in $attachments, alle intestazioni in $header o ai documenti e ai parametri RPC in $body, con un'unica eccezione: è possibile utilizzarla direttamente per accedere ai documenti e ai parametri in $body per i messaggi di richiesta ricevuti da un servizio proxy WSDL.

Prima di iniziare

Queste istruzioni presuppongono che si stia creando o modificando un'espressione nell'Editor espressioni XQuery, nell'Editor condizione XQuery o nell'Editor espressioni XPath, come descritto in Creazione e modifica delle espressioni XQuery e XPath in linea.

Per creare una struttura di variabile, effettuare le operazioni riportate di seguito.

  1. Selezionare Strutture variabili.
  2. Nella tavolozza Strutture variabili fare clic su Aggiungi nuova struttura.
  3. Continuare con uno dei task elencati nella tabella 22-3 riportata di seguito.

    4. Tabella 22-3 Creazione di una nuova struttura di variabile
    Per...
    Procedura da eseguire
    Creare una struttura di variabile che mappa una variabile a un tipo di schema XML
    1. Selezionare Tipo XML nella parte superiore della tavolozza Strutture variabili, se questa opzione non è già selezionata.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Percorso struttura immettere il percorso della struttura di variabile in fase di esecuzione. Il percorso deve iniziare con $.
    4. Sotto il campo Tipo selezionare il pulsante di scelta appropriato, quindi selezionare Tipo di schema.
    5. Fare clic su Sfoglia. Viene visualizzato il browser degli schemi XML. Selezionare uno schema XML dalla lista, selezionare un tipo di schema XML nel riquadro Definizioni, quindi fare clic su Sottometti.
    6. Fare clic su Aggiungi per creare la struttura della variabile.
    Creare una struttura di variabile che mappa una variabile a un tipo WSDL
    1. Selezionare Tipo XML nella parte superiore della tavolozza Strutture variabili, se questa opzione non è già selezionata.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Percorso struttura immettere il percorso della struttura della variabile in fase di esecuzione.
    4. Sotto il campo Tipo selezionare il pulsante di scelta appropriato, quindi selezionare Tipo di WSDL.
    5. Fare clic su Sfoglia. Viene visualizzato il browser WSDL. Selezionare un WSDL dalla lista di WSDL, selezionare un tipo WSDL dal riquadro Definizioni, quindi fare clic su Sottometti.
    6. Fare clic su Aggiungi per creare la struttura della variabile.
    Creare una struttura di variabile che mappa una variabile a un elemento di schema XML
    1. Nella parte superiore della tavolozza Strutture variabili selezionare Tipo XML, se questa opzione non è già selezionata.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Percorso struttura immettere il percorso della struttura della variabile in fase di esecuzione.
    4. Sotto il campo Tipo selezionare il pulsante di scelta appropriato, quindi selezionare Elemento schema.
    5. Fare clic su Sfoglia. Viene visualizzato il browser degli schemi XML. Selezionare uno schema XML dalla lista, selezionare un tipo di schema XML nel riquadro Definizioni, quindi fare clic su Sottometti.
    6. Fare clic su Aggiungi per creare la struttura della variabile.
    Creare una struttura di variabile che mappa una variabile a un elemento WSDL
    1. Nella parte superiore della tavolozza Strutture variabili selezionare Tipo XML, se questa opzione non è già selezionata.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Percorso struttura immettere il percorso della struttura della variabile in fase di esecuzione.
    4. Sotto il campo Tipo, selezionare il pulsante di scelta appropriato, quindi selezionare Elemento WSDL.
    5. Fare clic su Sfoglia. Viene visualizzato il browser WSDL. Selezionare un WSDL dalla lista di WSDL, selezionare un elemento WSDL dal riquadro Definizioni, quindi fare clic su Sottometti.
    6. Fare clic su Aggiungi per creare la struttura della variabile.
    Creare una struttura di variabile che mappa una variabile a un elemento figlio
    1. Nella parte superiore della tavolozza Strutture variabili selezionare Tipo XML, se questa opzione non è già selezionata.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Percorso struttura immettere il percorso della struttura della variabile in fase di esecuzione.
    4. Nel campo Tipo selezionare il tipo di variabile che si desidera creare.

      5. Per creare una variabile Elemento schema XML o Elemento WSDL, selezionare il pulsante di scelta associato a questa opzione, quindi selezionare Elemento schema o Elemento WSDL.

      Per creare una variabile MFL, selezionare il pulsante di scelta associato a questa opzione, quindi selezionare MFL.

    5. Per lo schema XML, WSDL o MFL, fare clic su Sfoglia per selezionare un oggetto dalla lista visualizzata nel browser, quindi fare clic su Sottometti. Ad esempio, selezionare un MFL da una lista di MFL, quindi fare clic su Sottometti.
    6. Selezionare la casella di controllo Imposta come figlio per impostare l'elemento come figlio della struttura creata.
    7. Fare clic su Aggiungi per creare la struttura della variabile.
    Creare una struttura di variabile che utilizza una risorsa MFL
    1. Nella parte superiore della tavolozza Strutture variabili selezionare Tipo XML, se questa opzione non è già selezionata.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Percorso struttura immettere il percorso della struttura della variabile in fase di esecuzione.
    4. Sotto il campo Tipo, selezionare il pulsante di scelta appropriato, quindi fare clic su Sfoglia. Viene visualizzato il Browser MFL.
    5. Selezionare un MFL dalla lista di MFL, quindi fare clic su Sottometti.
    6. Fare clic su Aggiungi per creare la struttura della variabile.
    Creare una struttura di variabile Interfaccia servizio
    1. Nella parte superiore della tavolozza Strutture variabili selezionare Interfaccia servizio.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Percorso struttura è presente $body per impostazione predefinita. Non è possibile modificare questo campo.
    4. Nel campo Servizio basato su WSDL selezionare l'icona Browser servizi, selezionare un servizio dalla lista di servizi visualizzata nel Browser servizi, quindi fare clic su Sottometti.

      5. Il servizio selezionato viene visualizzato nel campo Servizio basato su WSDL.

    5. Nel campo Operazione selezionare un'operazione oppure Nessuna per non includere un'operazione.
    6. Fare clic su Aggiungi per creare la variabile.
    Creare una struttura di variabile semplice
    1. Nella parte superiore della tavolozza Strutture variabili selezionare Tipo semplice.
    2. Nel campo Etichetta struttura immettere un nome visualizzato per la variabile che si desidera creare. Tale campo consente di assegnare un nome significativo alla struttura in modo da poterla riconoscere in fase di progettazione senza influire sulla fase di esecuzione.
    3. Nel campo Nome struttura immettere un nome per la struttura di variabile che si desidera creare.
    4. Sotto il campo Tipo selezionare Stringa o Qualsiasi XML.
    5. Fare clic su Aggiungi per creare la variabile.
