Pagina Sicurezza servizio proxy

La pagina Visualizza un servizio proxy - Sicurezza fa parte di una serie di pagine per la modifica dei servizi proxy, come descritto in Creazione e configurazione dei servizi proxy e Modifica delle configurazioni dei servizi proxy.

Utilizzare questa pagina per configurare le impostazioni di sicurezza per un servizio proxy.

I campi disponibili in questa pagina dipendono dall'uso dei criteri nel servizio proxy. Ad esempio, se il servizio proxy utilizza i criteri OWSM (consigliati), nella pagina Sicurezza compare Sostituzioni criteri. Per ulteriori informazioni, vedere "Securing Oracle Service Bus with Oracle Web Services Manager" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Per i criteri WLS 9.2, la configurazione per il nome utente e la password personalizzati e il token personalizzato sono simili. In entrambi i casi, si specificano le espressioni XPath che consentono a Oracle Service Bus di individuare le informazioni necessarie. La radice di queste espressioni XPath è la seguente:

  • Utilizzare soap-env:Envelope/soap-env:Header se l'associazione del servizio è AnySOAP o WSDL-SOAP.
  • Utilizzare soap-env:Body se l'associazione del servizio non è basata su SOAP.

Tutte le espressioni XPath devono essere in un formato XPath valido. Le espressioni XPath devono utilizzare la sintassi XPath "declare namespace" per dichiarare gli spazi di nomi utilizzati, come indicato di seguito:

declare namespace
ns='http://webservices.example.com/MyExampleService';)

Configurazione sicurezza dei servizi Web
Elabora intestazione WS-Security
Se una richiesta client include un'intestazione di sicurezza WS-Security, decidere se elaborare l'intestazione.
  • Selezionare Sì. In uno scenario intermediario attivo, il client applica WS-Security ai messaggi di richiesta e/o risposta. Il servizio proxy elabora l'intestazione di sicurezza e applica i criteri WS-Security. Questa opzione viene impostata in modo automatico su Sì quando un servizio proxy è basato su un WSDL che contiene un criterio di sicurezza oppure quando si associa un criterio al servizio nella pagina Configurazione criterio servizio.
  • Selezionare No. In uno scenario pass-through WS-Security, il client applica WS-Security ai messaggi di richiesta e/o risposta. Il servizio proxy non elabora l'intestazione di sicurezza, ma passa il messaggio di richiesta protetto non modificato a un servizio business.
Controllo accesso
Controllo accesso trasporto
Per tutti i servizi proxy, è possibile creare un criterio a livello di trasporto che applica un controllo di sicurezza quando un client tenta di stabilire una connessione al servizio proxy. Vengono consentite solo le richieste degli utenti elencati nel criterio a livello di trasporto.
Controllo accesso messaggio
Un criterio di controllo dell'accesso a livello di messaggio applica un controllo di sicurezza quando un client tenta di richiamare un servizio proxy con la sicurezza a livello di messaggio. È possibile creare un criterio di questo tipo nei seguenti casi:
  • per i servizi proxy che sono intermediari Web Service Security, ossia se si elabora l'intestazione WS-Security;
  • per i servizi proxy per i quali si imposta l'autenticazione personalizzata in questa pagina.
Solo agli utenti elencati nel criterio a livello di messaggio è consentito richiamare l'operazione.
Autenticazione personalizzata
(a livello di messaggio)
Tipo di autenticazione
Selezionare una delle opzioni riportate di seguito.
  • Selezionare Nessuna se il servizio proxy non utilizzerà l'autenticazione personalizzata.
  • Nome utente e password personalizzati

    • Se si seleziona questa opzione, è necessario anche immettere i valori nel campo XPath nome utente e nel campo XPath password utente.

  • Token personalizzato

    • Se si seleziona questa opzione, è necessario selezionare anche un Tipo di token e immettere i valori per il campo XPath token.

XPath nome utente
Questa opzione è disponibile solo quando l'opzione Impostazioni autenticazione personalizzata è impostata su Nome utente e password personalizzati. Se disponibile, questa opzione è obbligatoria.
Immettere il nome utente come espressione XPath.
L'espressione XPath viene valutata rispetto alle intestazioni o al payload del messaggio, a seconda dei casi; ciò consente a Oracle Service Bus di ottenere il nome utente per l'autenticazione personalizzata.
XPath password utente
Questa opzione è disponibile solo quando l'opzione Impostazioni autenticazione personalizzata è impostata su Nome utente e password personalizzati. Se disponibile, questa opzione è obbligatoria.
Immettere la password come espressione XPath.
L'espressione XPath viene valutata rispetto alle intestazioni o al payload del messaggio, a seconda dei casi; ciò consente a Oracle Service Bus di ottenere i valori della password per l'autenticazione personalizzata.
Tipo di token
Questa opzione è disponibile solo quando l'opzione Impostazioni autenticazione personalizzata è impostata su Token personalizzato. Se disponibile, questa opzione è obbligatoria.
Selezionare il tipo di token dall'elenco a discesa. Sono disponibili solo i tipi di token attivi configurati per un provider di asserzione delle identità di WebLogic Server. Vedere "Configuring Identity Assertion Providers for Custom Tokens" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660.
XPath token
Questa opzione è disponibile solo quando l'opzione Impostazioni autenticazione personalizzata è impostata su Token personalizzato. Se disponibile, questa opzione è obbligatoria.
Immettere un'espressione XPath per specificare il percorso del token personalizzato. Oracle Service Bus valuta l'espressione XPath del token rispetto alle intestazioni o al payload del messaggio, a seconda dei casi, per ottenere il token per l'autenticazione personalizzata.
Proprietà contesto
Facoltativamente, specificare una o più proprietà del contesto per passare ulteriori informazioni al provider della sicurezza di autenticazione (nome utente e password personalizzati) o di asserzione delle identità (token personalizzato).
Le proprietà del contesto forniscono un sistema (l'interfaccia ContextHandler) per passare informazioni aggiuntive alla struttura di sicurezza WebLogic, in modo che un provider di sicurezza possa ottenere informazioni contestuali. Vedere "Context Properties Are Passed to Security Providers" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500.
In Nome proprietà immettere una stringa di valori e in Selettore valori un'espressione XPath valida (le espressioni XPath possono essere anche stringhe di valori).
Le espressioni XPath vengono valutate rispetto alla stessa parte di messaggio utilizzata per il token personalizzato o il nome utente e la password personalizzati. In altre parole, le espressioni XPath immesse in Selettore valori per i servizi proxy basati su SOAP vengono valutate rispetto all'intestazione e al payload per i servizi proxy non basati su SOAP.
L'espressione XPath viene valutata in fase di esecuzione per produrre il valore della proprietà. Un ContextHandler è essenzialmente una lista di nomi e valori e, come tale, richiede che un provider di sicurezza sappia quali nomi cercare. Pertanto, le espressioni XPath vengono valutate solo se un provider di sicurezza richiede il valore di una di queste proprietà definite dall'utente.
Fare clic su Aggiungi proprietà per aggiungere la proprietà del contesto. È possibile aggiungere più proprietà del contesto.

Fare clic su Aggiorna per salvare la configurazione o su Reimposta per annullare le modifiche.
