Visualizzazione dei messaggi di report

Utilizzare la pagina Riepilogo report messaggi per visualizzare le informazioni sui messaggi di report mostrati nella tabella 27-2. Nella pagina vengono visualizzati fino a 100 messaggi.

  1. Per limitare il numero di elementi nella lista o individuare messaggi specifici, effettuare le operazioni riportate di seguito.
    • Fare clic sull'icona Apri per visualizzare ulteriori filtri di ricerca. Se si specificano più criteri di ricerca, vengono restituiti solo i risultati che soddisfano tutti i criteri.
    • Per impostare i parametri di filtro, completare uno o più passi mostrati nella tabella 27-1.

      • Tabella 27-1 Parametri di filtro dei messaggi 
      Per...
      Procedura da eseguire
      Applicare un filtro in base ai parametri di tempo
      Per impostare un periodo:
      1. Selezionare il pulsante di scelta Messaggi da... A.
      2. Nel campo Messaggi da immettere una data e un'ora nel formato MM/DD/YY HH:MM:SS AM|PM. Ad esempio, è possibile immettere 10/10/07 12:45:00 AM.
      3. Nel campo Messaggi a immettere una data e un'ora nel formato MM/DD/YY HH:MM:SS AM|PM.
      In alternativa, è possibile selezionare i parametri per i messaggi più recenti:
      1. Selezionare il pulsante di scelta Per l'ultimo.
      2. Nel campo Per l'ultimo immettere il numero di giorni, quindi specificare le ore e i minuti.
      Nota: per impostazione predefinita, vengono visualizzati i report dei messaggi per gli ultimi 30 minuti.
      Applicare un filtro in base al nome del servizio
      Nel campo Nome servizio in entrata immettere il nome del servizio.
      Applicare un filtro in base al codice di errore
      Nel campo Codice di errore immettere un codice di errore.
      Applicare un filtro in base all'indice di report
      Nel campo Indice report immettere gli indici di report. Si tratta delle coppie chiave-valore definite al momento della configurazione dell'azione di report per il servizio proxy. Questa è una stringa di ricerca.

    • È possibile utilizzare i caratteri jolly nelle ricerche per i campi del nome del servizio, del codice di errore e dell'indice di report.
      • Se si immette solo l'asterisco (*) vengono restituiti tutti i messaggi. Ad esempio, se si immette * nel campo del codice di errore, vengono visualizzati i messaggi con e senza codici di errore.
      • Solo per il campo Codice di errore si applicano rigide limitazioni di ricerca. Ad esempio, per trovare i messaggi con il codice di errore BEA12345, è possibile immettere *, BE* o BEA12345.
  2. Fare clic su Cerca. Nella pagina vengono visualizzati i messaggi che soddisfano i parametri di filtro impostati. Quando si applica il filtro si possono visualizzare fino a un massimo di 1000 messaggi.
  3. Fare clic su Vis. tutto per rimuovere i filtri di ricerca e visualizzare tutti i messaggi.
  4. Per individuare un messaggio, è inoltre possibile effettuare le operazioni riportate di seguito.
    • Riordinare la lista di messaggi. Fare clic su un nome di colonna sottolineato. La freccia crescente e la freccia decrescente indicano il criterio di ordinamento. Fare clic sul nome della colonna per modificare il criterio di ordinamento.
    • Scorrere le pagine. Utilizzare i controlli della pagina sopra o sotto la tabella. Andare alla pagina selezionando il numero della pagina o utilizzando i pulsanti freccia per passare alla pagina successiva, alla pagina precedente, alla prima pagina o all'ultima pagina.

      • Tabella 27-2 Dettagli del riepilogo dei messaggi (report)
      Proprietà
      Descrizione
      Indice report
      Un collegamento alla pagina Visualizza dettagli messaggio. Vedere Visualizzazione dei dettagli dei messaggi.
      Gli indici di report sono coppie chiave-valore utilizzate per estrarre gli identificativi chiave dalla variabile di contesto di un messaggio. Le chiavi sono un modo pratico per filtrare i messaggi. Si utilizza l'azione report per aggiungere coppie chiave-valore. Vedere Aggiunta e modifica di azioni nei flussi di messaggi.
      Indicatore orario DB
      La data e l'ora in cui il messaggio è stato aggiunto al database.
      Servizio in entrata
      Il servizio in entrata associato al messaggio. Si tratta di un collegamento alla pagina Visualizza un servizio proxy - Dettagli configurazione. Vedere Modifica delle configurazioni dei servizi proxy.
      Codice di errore
      Il codice di errore associato al messaggio, se disponibile.
      I codici di errore vengono specificati quando si configura l'azione di richiamo dell'errore. Vedere "Handling Errors in Message Flows" in Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBAG232 e Aggiunta di handler degli errori dei servizi proxy.
