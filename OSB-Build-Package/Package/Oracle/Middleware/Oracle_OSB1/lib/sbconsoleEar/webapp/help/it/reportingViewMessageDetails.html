Visualizzazione dei dettagli dei messaggi

Utilizzare questa pagina per visualizzare i dettagli di un messaggio specifico.

  1. Individuare il messaggio. Vedere Visualizzazione dei messaggi di report.
  2. Fare clic sull'indice di report. Nella pagina Visualizza dettagli messaggio vengono visualizzate le informazioni sulla Configurazione generale mostrate nella tabella 27-3.

    3. Tabella 27-3 Parametri di configurazione dei messaggi 
    Proprietà
    Descrizione
    ID messaggio
    L'identificazione univoca per il messaggio.
    Indicatore orario database
    La data e l'ora in cui il messaggio è stato registrato nel database.
    Ora al punto di log
    La data e l'ora in cui il messaggio è stato registrato.
    Nome server
    Il nome del server in cui è stato generato il messaggio.
    Stato
    Lo stato della pipeline in cui è stato generato il messaggio:
    REQUEST - Indica che l'azione di report viene eseguita in una pipeline richieste.
    RESPONSE - Indica che l'azione di report viene eseguita in una pipeline risposte.
    ERROR - L'azione viene eseguita nell'handler degli errori a livello di servizio.
    Nome nodo
    Il nome del nodo della pipeline in cui è stato generato il messaggio.
    Nome pipeline
    Il nome della pipeline in cui è stato generato il messaggio.
    Nome fase
    Il nome della fase in cui è stato generato il messaggio.

    La pagina visualizza le informazioni sul Servizio in entrata mostrate nella tabella 27-4.

    Tabella 27-4 Dettagli del servizio in entrata
    Proprietà
    Descrizione
    Nome
    Il servizio proxy in entrata associato al messaggio. Si tratta di un collegamento alla pagina Visualizza un servizio proxy - Dettagli configurazione. Vedere Modifica delle configurazioni dei servizi proxy.
    URI
    L'URI associato al servizio proxy.
    Operazione
    L'operazione in entrata associata al messaggio.

    La pagina visualizza le informazioni sul Servizio in uscita mostrate nella tabella 27-5.

    Tabella 27-5 Dettagli del servizio in uscita
    Proprietà
    Descrizione
    Nome
    Il servizio business in uscita associato al messaggio. Si tratta di un collegamento alla pagina Visualizza un servizio business - Dettagli configurazione. Vedere Modifica delle configurazioni dei servizi business.
    URI
    L'URI dell'endpoint del servizio business in uscita.
    Operazione
    L'operazione richiamata sul servizio in uscita.

    La pagina visualizza le informazioni sull'Indice di report mostrate nella tabella 27-6.

    Tabella 27-6 Dettagli dell'indice di report
    Proprietà
    Descrizione
    Testo indice report
    Gli indici di report sono coppie chiave-valore utilizzate per estrarre gli identificativi chiave dalla variabile di contesto di un messaggio. Le chiavi sono un modo pratico per filtrare i messaggi. Si utilizza l'azione report per aggiungere coppie chiave-valore. Vedere Aggiunta e modifica di azioni nei flussi di messaggi.

    La pagina visualizza le informazioni sull'Errore mostrate nella tabella 27-7.

    Tabella 27-7 Dettagli delle informazioni sull'errore
    Proprietà
    Descrizione
    Codice di errore
    Il codice di errore associato al messaggio, se disponibile.
    I codici di errore vengono specificati quando si configura l'azione di richiamo dell'errore. Vedere "Handling Errors in Message Flows" in Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBAG232 e Aggiunta di handler degli errori dei servizi proxy.
    Motivo
    Il motivo del codice di errore.
    Dettaglio
    I dettagli dell'errore associati al codice di errore. Questi dettagli, se presenti, sono in genere uno stack trace del punto in cui si è verificato un particolare errore e possono essere troncati a causa di limitazioni relative alle dimensioni nel database. Il limite è di 2048 caratteri.

    La pagina visualizza le informazioni sul Corpo report mostrate nella tabella 27-8.

    Tabella 27-8 Dettagli sul corpo del report
    Proprietà
    Descrizione
    Testo corpo report
    Questo collegamento apre un altro browser che visualizza i dettagli del corpo del report. Per acquisire il testo del corpo del report si utilizza un'espressione nell'azione di report. Vedere la procedura per l'azione di report in Aggiunta e modifica di azioni nei flussi di messaggi.

  3. Dopo aver visualizzato i dettagli di un messaggio specifico, fare clic su OK.
