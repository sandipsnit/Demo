Aggiunta di connessioni MQ

Prima di iniziare

Vedere Oracle Fusion Middleware Supported System Configurations all'indirizzo http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html per informazioni sulle versioni supportate di WebSphere MQ su Oracle Service Bus. La libreria client WebSphere MQ non viene fornita insieme all'Installer di Oracle Service Bus. Ottenere tale libreria dall'installazione di WebSphere MQ.

Per configurare e utilizzare risorse di connessione MQ e il trasporto MQ in Oracle Service Bus, è necessario aggiungere la libreria client MQ al classpath. Un'opzione consiste nel copiare la libreria com.ibm.mq.jar nella directory DOMAIN_NAME/lib, dove DOMAIN_NAME è il nome della directory in cui si trova il dominio, in genere MW_HOME\user_projects\domains\DOMAIN_NAME.

Per aggiungere una risorsa di connessione MQ

  1. Se questa operazione non è stata già eseguita, fare clic su Crea per creare una nuova sessione o su Modifica per accedere a una sessione esistente. Vedere Utilizzo del Centro modifiche.
  2. Selezionare Explorer progetto, quindi selezionare un progetto o una cartella in cui aggiungere la risorsa di connessione MQ. Viene visualizzata la pagina Vista progetto/cartella.
  3. Dall'elenco a discesa Crea risorsa selezionare Connessione MQ da Utility per visualizzare la pagina Crea una nuova risorsa connessione MQ.

    4. Specificare i parametri di connessione mostrati nella tabella 8-2.

    Tabella 8-2 Parametri della connessione MQ
    Proprietà
    Per creare o modificare...
    Nome risorsa
    Immettere un nome univoco per questa risorsa di connessione MQ.

    Vedere Limitazioni di denominazione delle risorse per le istruzioni di denominazione.
    Descrizione risorsa
    Immettere una descrizione per la risorsa di connessione MQ.
    Tipo di connessione
    Selezionare una delle modalità di connessione indicate di seguito per connettersi al Queue Manager MQ.
    • modalità tcp: utilizzare TCP/IP per connettersi a un Queue Manager che non risiede sullo stesso computer di Oracle Service Bus.
    • modalità associazione: utilizzare la modalità di associazione per connettersi a un Queue Manager che si trova sullo stesso computer di Oracle Service Bus.
    Nome host MQ
    Solo per le connessioni modalità tcp:
    Immettere il nome host del Queue Manager MQ.
    Numero di porta MQ
    Solo per le connessioni modalità tcp:
    Immettere il numero di porta del listener del Queue Manager MQ.
    Nome Queue Manager MQ
    Immettere il nome del Queue Manager MQ al quale connettersi.
    CCSID Queue Manager
    Solo per le connessioni modalità tcp:
    Il CCSID (Coded Character Set Identifier) da utilizzare quando si stabilisce una connessione. Il CCSID è utilizzato soprattutto per il supporto dell'internazionalizzazione.
    Per ulteriori informazioni, vedere WebSphere MQ Fundamentals all'indirizzo http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
    Nome canale Queue Manager MQ
    Solo per le connessioni modalità tcp:
    Immettere il nome del canale di connessione al server per Queue Manager.
    SSL obbligatorio
    Per le connessioni modalità tcp:
    Selezionare la casella di controllo per utilizzare SSL per l'invio dei messaggi. È supportato solo il protocollo SSL sul lato server se non si seleziona l'opzione SSL bidirezionale obbligatorio.
    Suite di cifratura
    Questa opzione è disponibile solo quando è selezionata la casella di controllo SSL obbligatorio.
    Selezionare l'algoritmo Cipher Suite che dovrà essere utilizzato dal protocollo SSL.
    L'algoritmo della suite di cifratura è utilizzato per cifrare e decifrare le comunicazione dei messaggi tra il server WebSphere MQ e il client WebSphere MQ. Pertanto, è necessario specificare un algoritmo della suite di cifratura quando si utilizza il protocollo SSL per comunicare con un server WebSphere MQ.
    SSL bidirezionale obbligatorio
    Questa opzione è disponibile solo quando è selezionata la casella di controllo SSL obbligatorio.
    Selezionare la casella di controllo per abilitare sia l'autenticazione SSL lato client sia quella lato server.
    Riferimento al provider di chiavi di servizio
    Se si seleziona SSL bidirezionale obbligatorio è necessario fornire un riferimento al provider di chiavi di servizio per l'ottenimento delle informazioni appropriate sul keystore e sull'area di memorizzazione sicura.
    Immettere il percorso (progetto/cartella) e il nome di un provider di chiavi di servizio o fare clic su Sfoglia per selezionarne uno nella pagina Seleziona provider di chiavi servizio.
    Riferimento all'account di servizio statico
    Solo per le connessioni modalità tcp:
    Richiesto per l'autenticazione del nome utente e della password. Immettere il percorso (progetto/cartella) e il nome di un account di servizio statico o fare clic su Sfoglia per selezionare gli account di servizio da un browser.
    Versione MQ WebSphere
    Selezionare la versione di WebSphere MQ:
    • 5.3
    • 6.0
    • 7.0

    Nota: le versioni 5.3 e 6.0 non sono più supportate da IBM. Queste versioni non sono più valide in Oracle Service Bus.
    Dimensioni connection pool MQ
    Immettere le dimensioni del connection pool MQ.
    Timeout connessione MQ
    Immettere l'intervallo di tempo in secondi trascorso il quale le connessioni inutilizzate vengono distrutte. L'impostazione predefinita è 1800 secondi.
    Attesa massima connessione MQ
    Immettere l'attesa massima in millisecondi prima una connessione diventi disponibile. Se non viene stabilita una connessione entro l'intervallo di tempo specificato, Oracle Service Bus restituisce un'eccezione. L'impostazione predefinita è 3000 millisecondi.

  4. Fare clic su Salva. La risorsa di connessione MQ viene creata e salvata nella sessione corrente.
  5. Per terminare la sessione e distribuire la configurazione in fase di esecuzione, fare clic su Attiva in Centro modifiche.
