Modifica degli account di servizio

  1. Se questa operazione non è stata già eseguita, fare clic su Crea per creare una nuova sessione o su Modifica per accedere a una sessione esistente. Vedere Utilizzo del Centro modifiche.
  2. Individuare l'account di servizio, come descritto in Individuazione degli account di servizio.
  3. Fare clic sul nome dell'account di servizio. Nella pagina Visualizza dettagli account di servizio vengono visualizzate le informazioni mostrate nella tabella 16-3.

    4. Tabella 16-3 Dettagli dell'account di servizio
    Proprietà
    Descrizione
    Autore ultima modifica
    L'utente che ha creato l'account di servizio o lo ha importato nella configurazione.
    Data ultima modifica
    La data e l'ora in cui l'utente ha creato l'account di servizio o lo ha importato nella configurazione. Fare clic sul collegamento relativo alla data e all'ora per visualizzare la cronologia delle modifiche della risorsa. Vedere la pagina Visualizza cronologia modifiche.
    Riferimenti
    Il numero di oggetti a cui fa riferimento l'account di servizio. Se tali riferimenti esistono, fare clic sul collegamento numerico per visualizzare una lista di oggetti. Vedere Visualizzazione dei riferimenti alle risorse.
    Riferimento di
    Il numero di oggetti che fanno riferimento all'account di servizio. Se tali riferimenti esistono, fare clic sul collegamento numerico per visualizzare una lista degli oggetti. Ad esempio, se si seleziona l'account di servizio come account di servizio JMS in un servizio proxy con un protocollo di trasporto JMS, il servizio proxy è elencato come riferimento quando si fa clic sul collegamento. Vedere Visualizzazione dei riferimenti alle risorse.
    Descrizione
    Una descrizione dell'account di servizio, se disponibile.

  4. Per apportare una modifica ai campi, fare clic su Modifica. Per le descrizioni dei campi, vedere Aggiunta di account di servizio.

    5. Non è possibile modificare il campo Nome risorsa.

  5. Fare clic su Salva per eseguire il commit degli aggiornamenti nella sessione corrente.
  6. Per terminare la sessione e distribuire la configurazione in fase di esecuzione, fare clic su Attiva in Centro modifiche.
Nota: se l'account di servizio modificato viene utilizzato per eseguire l'autenticazione con un server JMS WebLogic, il server JMS potrebbe non riconoscere la modifica per un massimo di 60 secondi. Per impostazione predefinita, il server JMS WebLogic controlla le autorizzazioni per ciascuna destinazione ogni 60 secondi. Per cambiare questo funzionamento, modificare il comando di avvio di WebLogic Server in modo che imposti la seguente proprietà di sistema sulla frequenza (in secondi) con la quale si desidera che vengano controllare le autorizzazioni: weblogic.jms.securityCheckInterval
Un valore pari a 0 (zero) per questa proprietà assicura che venga eseguito un controllo delle autorizzazioni per ogni azione send, receive e getEnumeration su una risorsa JMS.
