Aggiunta di provider JNDI

Per aggiungere e configurare una risorsa provider JNDI e renderla disponibile in Oracle Service Bus come risorsa di sistema, è necessario essere in una sessione attiva.

  1. Se questa operazione non è stata già eseguita, fare clic su Crea per creare una nuova sessione o su Modifica per accedere a una sessione esistente. Vedere Utilizzo del Centro modifiche.
  2. Selezionare Amministrazione sistema > Provider JNDI.
  3. Fare clic su Aggiungi.
  4. Nel campo Nome immettere un nome per la risorsa provider JNDI. Questo campo è obbligatorio.

    Vedere Limitazioni di denominazione delle risorse per le istruzioni di denominazione.
  5. Nel campo Descrizione immettere una breve descrizione per il provider JNDI.
  6. Lasciare abilitata l'opzione Cache JNDI (impostazione predefinita) o selezionare Disabilitato.

    7. Se questa opzione è abilitata, il contesto JNDI e gli oggetti JNDI vengono inseriti nella cache localmente. Ciò migliora le prestazioni durante la ricerca di un oggetto. Oracle consiglia di abilitare la cache JNDI.

  7. Nel campo URL provider immettere l'URL per il provider JNDI nel formato protocol://host:port

    8. È possibile utilizzare qualsiasi protocollo, ad esempio http, https, t3, t3s, iiop, iiops.

    Questo campo è obbligatorio.

  8. Nel caso dei cluster, l'URL del provider JNDI deve essere configurato con una lista di server gestiti separati da virgole, nel seguente formato:

    9. protocol://<hostname>:<ms1port>, <hostname>:<ms2port>

  9. Se l'accesso al provider JNDI di destinazione richiede un nome utente e una password, immettere un nome utente nel campo Nome utente e la password associata nei campi Password e Conferma password.

    10. Questi campi sono facoltativi. Sono obbligatori solo se la struttura JNDI è protetta.

  10. Fare clic su Salva per creare e salvare la risorsa provider JNDI nella sessione corrente.
  11. Per terminare la sessione e distribuire la configurazione in fase di esecuzione, fare clic su Attiva in Centro modifiche.
