Importazione degli account di servizio o dei provider di chiavi di servizio

Nota: nelle versioni precedenti di Oracle Service Bus, i provider di chiavi di servizio erano denominati provider di servizi proxy.

Se il file JAR è stato creato da AquaLogic Service Bus 3.0 o versione successiva e contiene account di servizio o provider di chiavi di servizio, è possibile importare queste risorse insieme ai nomi utente, alle password, ai mapping tra utente locale e utente remoto e le associazioni coppia di chiavi con alias.

Per ogni provider di chiavi di servizio, Oracle Service Bus importa l'associazione coppia di chiavi con alias nel provider di mapping delle credenziali PKI. Se questi dati sono stati cifrati durante l'esportazione, è necessario fornire la password utilizzata per cifrare i dati. Se non si conosce la password, è possibile importare tutte le altre risorse non cifrate.

Se si importa un account di servizio o un provider di chiavi di servizio e nel dominio esiste già una risorsa con lo stesso nome, la risorsa importata sovrascrive quella già presente nel dominio, anche se quest'ultima è stata modificata durante la sessione corrente, a meno che non si specifichi di conservare le impostazioni di sicurezza durante l'importazione. Vedere il passo 5 in Selezione di progetti e risorse da importare.
