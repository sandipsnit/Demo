Selezione di progetti e risorse da importare

  1. Se si importa un file JAR di risorse e si desidera fare in modo che vengano importate tutte le risorse associate, selezionare Includi dipendenze.
    2. Nota: non è possibile selezionare Includi dipendenze se si importa un file JAR di progetto completo o un file di configurazione precedente alla versione 2.6.
  2. Fare clic sull'icona Apri per visualizzare le Impostazioni avanzate per mantenere i valori nel sistema di importazione.
  3. Quando si reimposta una risorsa ma si desidera proteggere le eventuali personalizzazioni apportate alla risorsa esistente, selezionare Conserva variabili di ambiente.
    4. Nota: la personalizzazione dei riferimenti in una risorsa esistente non può essere mantenuta. Vedere Creazione dei file di personalizzazione.
  4. Se si importa la risorsa delle impostazioni operative globali o altre risorse, come i servizi business o proxy che contengono le impostazioni operative a livello di servizio associate, e si desidera mantenere i valori operativi nel sistema di importazione, selezionare Conserva valori operativi.
  5. Per impedire la sovrascrittura della configurazione relativa alla sicurezza esistente nelle risorse da aggiornare, selezionare una delle opzioni riportate di seguito.
    1. Conserva configurazione sicurezza e criteri: conserva la configurazione della sicurezza, esclusi i criteri di controllo dell'accesso, e i riferimenti ai WS-policy associati direttamente al servizio, piuttosto che associati al WSDL.

      2. Per ulteriori informazioni, vedere "Security" in Oracle Fusion Middleware Developer's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472.

    2. Conserva credenziali: conserva le credenziali PKI nei provider di chiavi di servizio, le credenziali nome utente e password negli account di servizio e nome utente e password nei server SMTP, nei server proxy, nei provider JNDI e nei registri UDDI.
    3. Conserva criteri di controllo accesso: conserva i criteri di controllo dell'accesso.

      4. Nella tabella 28-1 vengono mostrate le informazioni di riepilogo delle risorse.

      Tabella 28-1 Informazioni sulle risorse
      Proprietà
      Descrizione
      Nome
      Il nome del progetto.
      Le risorse globali, come le informazioni sul registro UDDI, i server SMTP, i server proxy, i provider JNDI e le impostazioni operative globali, compaiono sotto il progetto System nelle rispettive cartelle.
      Tipo
      Il tipo di risorsa, ossia Progetto.
      Espandere la cartella Progetto per visualizzare e deselezionare le risorse.
      Operazioni
      Visualizza --- o !Deletions!
      Espandere la cartella Progetto per visualizzare le operazioni (creazione, aggiornamento, eliminazione) che verranno eseguite sulle risorse.
      Per impedire che le risorse vengano eliminate, deselezionarle. Le risorse vengono pianificate per l'eliminazione solo se il file JAR da importare è un file JAR di progetto completo e nello stesso progetto nel sistema di importazione sono presenti risorse che non si trovano nel file JAR importato.
      Riferimenti
      Espandere la cartella Progetto per visualizzare il numero di oggetti a cui ogni risorsa fa riferimento.

  6. Selezionare gli oggetti su cui si desidera eseguire l'operazione (creazione, aggiornamento o eliminazione).
    1. Per visualizzare ciascun oggetto contenuto nel file JAR, espandere la cartella per il file. La casella di controllo associata a ciascun oggetto è selezionata.

      2. Nella tabella 28-2 vengono mostrati i dettagli dei file delle risorse per ciascun oggetto.

      Tabella 28-2 Dettagli dei file delle risorse 
      Proprietà
      Descrizione
      Nome
      Il nome e il percorso della risorsa. Il percorso è costituito dal nome del progetto e dal nome della cartella in cui risiede la risorsa.
      Tipo
      I tipi di risorsa Oracle Service Bus.
      Nota: se una risorsa ha dati di sicurezza riservati, viene visualizzata l'icona Cifratura.
      Attenzione: Se si seleziona un account di servizio o un provider di chiavi di servizio, è possibile che vengano sovrascritti i dati di sicurezza esistenti, a meno che non si specifichi di conservare le impostazioni di sicurezza durante l'importazione. Vedere Importazione degli account di servizio o dei provider di chiavi di servizio.
      Operazioni
      Creazione, aggiornamento o eliminazione.
      Le risorse vengono pianificate per l'eliminazione solo se il file JAR da importare è un file JAR di progetto completo e nello stesso progetto nel sistema di importazione sono presenti risorse che non si trovano nel file JAR importato.
      Riferimenti
      Il numero di oggetti a cui fa riferimento la risorsa.

    2. Deselezionare le caselle di controllo accanto alle risorse che non si desidera importare.
  7. Fare clic su Importa.

Al termine della procedura

Se applicabile, continuare con le operazioni descritte in Specifica delle impostazioni di sicurezza dell'importazione. In alternativa, continuare con le operazioni descritte in Controllo del riepilogo dell'importazione.
