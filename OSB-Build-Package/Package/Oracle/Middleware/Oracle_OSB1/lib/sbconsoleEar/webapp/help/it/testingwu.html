Test dei servizi proxy

È possibile eseguire il test dei seguenti tipi di servizi proxy: Qualsiasi XML, Qualsiasi SOAP, Messaggistica, XML e SOAP. Il test dei servizi proxy SOAP può essere eseguito con i criteri Web Service Security (WSS). Vedere Web Service Security in Configurazione dei dati di test dei servizi proxy.

Nota: quando la console di test richiama un proxy con l'autenticazione del token personalizzato HTTP, il controllo dell'autenticazione non viene eseguito.
Attenzione: Se si esegue il test dei servizi proxy con l'opzione Chiamata diretta abilitata, vengono ignorati alcuni importanti passi di sicurezza, incluso il controllo dell'accesso. Oracle consiglia di non utilizzare il servizio di test nei sistemi di produzione.
  1. Fare clic su Attiva in Centro modifiche per abilitare la funzione di test nella console di amministrazione di Oracle Service Bus.

    2. È possibile eseguire il test dei servizi proxy dal Browser risorse o da Explorer progetto.

  2. Selezionare Browser risorse > Servizi proxy per visualizzare la pagina Riepilogo dei servizi proxy.
  3. Sotto Azioni fare clic sull'icona Avvia console di test associata al servizio proxy di cui eseguire il test.

    4. La console di test apre la pagina Test del servizio proxy. Ad esempio, utilizzando gli esempi forniti con il prodotto, fare clic sull'icona associata al servizio proxy LoanGateway1.

    Nota: in un dominio in cluster, non è possibile utilizzare la console di test per eseguire il test di un servizio business o proxy configurato che esegue l'instradamento a un servizio business.
  4. Per i servizi SOAP e XML, selezionare l'operazione WSDL di cui si desidera eseguire il test.
  5. Configurare i dati di test per il servizio proxy. Deve trattarsi dei dati che il servizio proxy si aspetta dal client.

    6. Per impostazione predefinita, entrambe le opzioni di configurazione, Chiamata diretta e Includi trace, sono abilitate. È possibile deselezionare l'opzione Chiamata diretta, determinando così anche la deselezione dell'opzione Includi trace. In questo modo, il test viene eseguito con il metodo della chiamata indiretta, in cui il messaggio viene inviato attraverso il livello di trasporto.

    È possibile utilizzare l'opzione Chiamata diretta (lasciandola selezionata) e disabilitare il trace di esecuzione. A tale scopo, è sufficiente deselezionare la casella di controllo Includi trace.

    Per visualizzare il trace dei messaggi, deselezionare Chiamata diretta.

    Nota: per visualizzare il trace nel file di log o nell'output standard (console del server), è necessario impostare il log di Oracle WebLogic Server sui livelli di severità riportati di seguito.

Per informazioni sull'impostazione dei livelli di severità di log, vedere "Using Log Severity Levels" in Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

  1. Fare clic su Esegui. Nella pagina Test del servizio proxy vengono visualizzati i risultati. Per informazioni sull'interpretazione dei risultati del test, vedere Visualizzazione dei risultati del test dei servizi proxy.
  2. Per eseguire di nuovo il test, fare clic su Indietro. Ripetere i passi da 5 in poi.
