Test dei servizi business

È possibile eseguire il test dei seguenti tipi di servizi business: Qualsiasi XML, Qualsiasi SOAP, Messaggistica, XML e SOAP. Il test dei servizi business SOAP può essere eseguito con i criteri Web Service Security. Per ulteriori informazioni, vedere "Testing Services with Web Service Security" in Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=OSBAG384.

Nota: in un dominio in cluster, non è possibile utilizzare la console di test per eseguire il test di un servizio business o proxy configurato che esegue l'instradamento a un servizio business.
  1. Fare clic su Attiva in Centro modifiche per abilitare la funzione di test nella console di amministrazione di Oracle Service Bus.
  2. Selezionare Browser risorse > Servizi business per visualizzare la pagina Riepilogo dei servizi business.
  3. Sotto Azioni fare clic sull'icona Avvia console di test associata al servizio di cui si desidera eseguire il test. La console di test apre la pagina Test del servizio business. Ad esempio, utilizzando le esercitazioni fornite con il prodotto, fare clic sull'icona associata a loanSaleProcessor.
  4. Per i servizi SOAP e XML, selezionare l'operazione WSDL di cui si desidera eseguire il test.
  5. Configurare i dati di test per il servizio business (i dati di input devono essere costituiti dal messaggio inviato dal servizio proxy al servizio business). Le opzioni Chiamata diretta e Includi trace disponibili quando si esegue il test dei servizi proxy non sono disponibili per i servizi business. Per impostazione predefinita, il test dei servizi business viene eseguito utilizzando l'opzione Chiamata diretta; questo significa che i messaggi passano attraverso il layer di trasporto.

    6. Per visualizzare il trace dei messaggi, deselezionare Chiamata diretta.

    Nota: per visualizzare il trace nel file di log o nell'output standard (console del server), è necessario impostare il log di Oracle WebLogic Server sui livelli di severità riportati di seguito.

Livello di severità minimo per il log: Informazioni
File di log: Informazioni
Output standard: Informazioni

Per informazioni sull'impostazione dei livelli di severità di log, vedere "Using Log Severity Levels" in Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server all'indirizzo http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

  1. Fare clic su Esegui.

    2. La pagina Test del servizio business visualizza i risultati. Per ulteriori informazioni, vedere Visualizzazione dei risultati del test dei servizi business.

Test degli allegati nei servizi business

In questa sezione viene descritto come utilizzare la console di test per eseguire il test degli allegati nei servizi business.

A scopo illustrativo, questa sezione presuppone che un servizio business HTTP basato su WSDL utilizzi un'operazione "submitAttachment" per inviare un file ZIP come allegato in un messaggio SOAP. Modificare le impostazioni in base alle necessità per i diversi tipi di allegati binari.

    1. Nella console di test per il servizio business, andare alla pagina XML e immettere il seguente envelope SOAP per il messaggio di test:
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. Nella sezione Allegato della console di test:
      1. Content-Type: applicazione/zip (in base alla definizione nel WSDL)
      2. Content-ID: zipFile (deve corrispondere al valore di href="cid:zipFile" nel messaggio di test)
      3. File: fare clic su Sfoglia e selezionare il file ZIP specificato nell'envelope SOAP.
      4. Fare clic su Aggiungi.

    2. Fare clic su Esegui per inviare il messaggio di test con l'allegato.

Per verificare l'invio dell'allegato, controllare che sulla console del server sia presente un messaggio simile al seguente, registrato dall'operazione submitAttachment:

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
