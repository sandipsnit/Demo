Trace dei servizi proxy

Il trace è abilitato quando si esegue il test dei servizi proxy utilizzando una chiamata diretta. La casella di controllo Includi trace è selezionata per impostazione predefinita con l'opzione Chiamata diretta. Se non si desidera il trace, deselezionare la casella di controllo Includi trace. Quando si abilita il trace, i risultati del test includono i dettagli del trace. Utilizzare il trace per tenere traccia dei problemi nel sistema e isolarli per la correzione. Le informazioni di trace mostrano il percorso attraverso la richiesta e i percorsi delle risposte nel codice.

  1. Fare clic su Attiva in Centro modifiche per abilitare la funzione di test nella console di amministrazione di Oracle Service Bus.
  2. Selezionare Browser risorse > Servizi proxy per visualizzare la pagina Riepilogo dei servizi proxy.
  3. Sotto Azioni fare clic sull'icona Avvia console di test associata al servizio di cui si desidera eseguire il test. La console di test apre la pagina Test del servizio proxy.
  4. Configurare i dati di test per il servizio proxy. Per abilitare il trace, è necessario che le opzioni Chiamata diretta e Includi trace siano selezionate. Vedere Configurazione dei dati di test dei servizi proxy.
  5. Fare clic su Esegui. Nella pagina Test del servizio proxy vengono visualizzati i risultati del test per il servizio e le informazioni di trace.
  6. Scorrere in basso fino alla sezione Trace richiamo.

    7. Questa sezione contiene una visualizzazione grafica del flusso di messaggi. È possibile tracciare il messaggio mediante il servizio e visualizzare lo stato del messaggio in punti preselezionati del trace. I punti del trace vengono impostati automaticamente.

  7. Fare clic sul segno più (+ ) per espandere il flusso di messaggi e visualizzare ulteriori dettagli.

    8. Quando si visualizza il trace è possibile visualizzare anche il flusso di messaggi nella console di amministrazione di Oracle Service Bus. Ciò consente di correlare il trace alle fasi e alle azioni nel flusso di messaggi. È possibile modificare il flusso di messaggi ed eseguire di nuovo il trace per visualizzare l'output.
