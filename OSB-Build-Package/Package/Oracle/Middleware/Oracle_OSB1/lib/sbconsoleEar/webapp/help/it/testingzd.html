Test delle trasformazioni XSLT

eXtensible Stylesheet Language Transformations (XSLT) descrive i mapping da XML a XML in Oracle Service Bus. Per eseguire il test di una risorsa XSLT è necessario fornire il documento XML di input e la console di test restituisce il documento XML di output. Le trasformazioni XSLT possono avere parametri denominati aggiuntivi. Tutti i parametri richiesti dalla trasformazione vengono visualizzati nella pagina di configurazione. Sono disponibili valori predefiniti, ma è possibile sostituirli.

Nella tabella 32-6 vengono descritte le opzioni di configurazione XSLT.

Input XML
L'input XML può essere basato su file o su testo. Il riferimento a un file per l'input ha precedenza sull'input testuale. Cercare e selezionare il file da utilizzare nel test. L'input XML è richiesto.
<param_name> (come XML)
param_name è un parametro XSLT denominato.
Ci sono due tipi di input: XML e primitivo (String, Integer, Float e così via). Il tipo di input predefinito è String. Selezionare la casella di controllo associata al nome del parametro per identificare un parametro di tipo XML.

È possibile eseguire il test della fase di progettazione o della fase di esecuzione.

  1. Fare clic su Attiva per eseguire il test della fase di esecuzione. Non attivare la sessione per eseguire il test della fase di progettazione.
  2. Selezionare Explorer progetto > XSLT per visualizzare la pagina Riepilogo degli XSLT.
  3. Sotto Azioni fare clic sull'icona Avvia console di test associata alla risorsa di cui si desidera eseguire il test. La console di test apre la pagina Test della risorsa.
  4. Configurare i dati di test per la risorsa nella sezione Input e parametri della pagina. Per ulteriori informazioni, vedere la tabella 32-6.
  5. Fare clic su Esegui. La pagina Test della risorsa visualizza i risultati.
  6. Per eseguire di nuovo il test, fare clic su Indietro. È possibile chiudere la console di test, modificare la risorsa e sottoporla nuovamente a test.
