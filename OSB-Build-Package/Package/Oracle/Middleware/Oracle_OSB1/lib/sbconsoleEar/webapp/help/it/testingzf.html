Utilizzo dell'Editor di espressioni e dell'Editor condizione XQuery

Le espressioni XQuery consentono di creare il contenuto di dati per le variabili di contesto del messaggio (o parte di una variabile di contesto del messaggio) durante l'esecuzione del flusso di messaggi. È possibile utilizzare la console di test direttamente nell'Editor espressioni XQuery per eseguire il test della definizione dell'espressione.

In modo analogo, le condizioni XQuery vengono utilizzate per valutare le condizioni booleane nel flusso di messaggi. È possibile utilizzare la console di test direttamente nell'Editor condizione XQuery per eseguire il test della definizione della condizione.

Una XQuery può accettare più input e restituire un unico output. Ogni input corrisponde a una variabile non associata XQuery definita nella XQuery. Il valore di un input XQuery può essere un valore primitivo (String, Integer, Date e così via), un documento XML o una sequenza di questi tipi. Il valore di output può essere un valore primitivo (String, Integer, Date e così via), un documento XML o una sequenza di questi tipi.

Nota: la console di test non supporta le sequenze per l'input.

<param_name> (come XML)
param_name è il nome della variabile non associata XQuery corrispondente.
Nella console di test viene visualizzata una casella di modifica con una singola riga se il tipo è un tipo semplice. Se i dati sono XML viene visualizzata una casella di modifica con più righe. Se non si definisce un tipo per la variabile, viene utilizzato un input combinato (<param_name> ([] come XML)). È necessario dichiarare il tipo di variabile. Selezionare la casella di controllo per identificare un parametro di tipo XML.
L'input XML può essere basato su file o su testo. Il riferimento a un file per l'input ha precedenza sull'input testuale. Cercare e selezionare il file da utilizzare nel test.
L'input nella console di test viene visualizzato in base al tipo per rendere più semplice l'individuazione del tipo di dati da immettere. Il tipo predefinito è XML.

  1. Accedere alla console di test quando si modifica un'azione nel flusso di messaggi di una pipeline.
    • Per accedere all'Editor espressioni XQuery, vedere Creazione e modifica delle espressioni XQuery e XPath in linea.
    • Per accedere all'Editor condizione XQuery, vedere Creazione e modifica delle espressioni XQuery e XPath in linea.

      • Viene visualizzata la pagina Espressione test XQuery, in cui vengono mostrate tutte le variabili di input richieste.

  2. Configurare i dati di test per l'XQuery nella sezione Variabili della pagina. Per ulteriori informazioni, vedere la tabella 32-8.
  3. Fare clic su Esegui. La pagina di test visualizza i risultati.
  4. Dopo aver completato un test, è possibile fare clic su Indietro per eseguire un nuovo test. Per eseguire un nuovo test dopo avere apportato modifiche all'XQuery, è necessario chiudere e riaprire la console di test affinché le modifiche diventino effettive.
