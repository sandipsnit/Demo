Utilizzo dell'Editor espressioni XPath

Le espressioni XPath consentono di selezionare un subset di una variabile di contesto di messaggio XML. È possibile utilizzare la console di test nell'Editor espressioni XPath per eseguire il test della definizione dell'espressione XPath. Un'espressione XPath accetta un singolo documento XML come input e genera una sequenza di documenti XML, tipi primitivi o entrambi come output.

Nella tabella 32-9 vengono descritte le opzioni di configurazione delle espressioni XPath.

Tabella 32-9 Opzioni di configurazione delle espressioni XPath
Sezione
Descrizione
Nome
Il tipo di espressione di cui si esegue il test.
Azioni della console di test
Esegui
Applica la trasformazione.
Reimposta
Reimposta il campo di input.

Annulla l'operazione corrente.
Variabili
Questa sezione contiene un singolo campo di input corrispondente al documento XML rispetto al quale viene eseguito il test dell'espressione XPath.
 
L'input XML può essere basato su file o su testo. Il riferimento a un file per l'input ha precedenza sull'input testuale. Cercare e selezionare il file da utilizzare nel test.

  1. Accedere alla console di test quando si modifica un'azione nel flusso di messaggi di una pipeline. Per accedere all'Editor espressioni XPath, vedere Creazione e modifica delle espressioni XQuery e XPath in linea.
  2. Configurare i dati di test per l'espressione XPath nella sezione Variabili della pagina. Per ulteriori informazioni, vedere la tabella 32-9.
  3. Fare clic su Esegui. La pagina di test visualizza i risultati.
  4. Dopo aver completato un test, è possibile fare clic su Indietro per eseguire un nuovo test. Per eseguire un nuovo test dopo avere apportato modifiche all'espressione XPath, è necessario chiudere e riaprire la console di test affinché le modifiche diventino effettive.
