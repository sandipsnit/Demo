DSPトランスポート構成ページ

ビジネス・サービスの作成/編集 - DSPトランスポート構成ページは、ビジネス・サービスの作成および編集(ビジネス・サービスの作成と構成およびビジネス・サービス構成の編集を参照)に関する一連のページの1つです。

詳細は、Oracle Fusion Middleware Oracle Oracle Service Busのための開発者ガイドのDSPおよびOracle Data Service Integratorのトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1229)を参照してください。

このページでは、DSP (Oracle Data Service Integrator)トランスポート・プロトコルを使用するビジネス・サービスのトランスポート設定を構成できます。表18-4に、このページの使用方法を示します。

表18-4 ビジネス・サービスの作成/編集 - DSPトランスポート構成ページ
オプション
作成または編集の方法
デバッグ・レベル
次のいずれか1つを指定します。
  • 0 - デバッグ情報なし
  • 1 - 要求メッセージに情報を出力する
  • 3 - 要求メッセージと応答メッセージに情報を出力する
サービス・アカウント
「参照」をクリックし、表示されるリストでサービス・アカウントを選択します。サービス・アカウントを指定しない場合は、匿名サブジェクトが使用されます。

詳細は次を参照してください:

  • サービス・アカウント
  • サービス・アカウントの追加
ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーに使用するWebLogic Serverワーク・マネージャのインスタンスを選択します。デフォルトのワーク・マネージャは、他にワーク・マネージャがない場合に使用されます。
ワーク・マネージャは、レスポンス処理のためのレスポンス・メッセージをポストするために使用されます。ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。

終了後

「次」をクリックして次のページでこのサービスの構成を続行するか、「最後」をクリックしてビジネス・サービスの作成/編集 - サマリー・ページでこの構成を確認して保存します。
