「DSPトランスポート構成」ページ

「ビジネス・サービスの作成/編集 - ファイル・トランスポート構成」ページは、ビジネス・サービスの作成および編集(「ビジネス・サービスの作成と構成」および「ビジネス・サービス構成の編集」を参照)に関する一連のページの1つです。

詳細については、「Oracle Service Busを介したデータ・サービスへのアクセス」を参照してください。

このページでは、DSP（Oracle Data Service Integrator）トランスポート・プロトコルを使用するビジネス・サービスのトランスポート設定を構成できます。表17-4に、このページの使用方法を示します。

表17-4 ビジネス・サービスの作成/編集 - DSPトランスポート構成ページ
オプション
作成または編集の方法
「デバッグ・レベル」
以下のいずれか1つを指定します。
  • 0 - デバッグ情報なし
  • 1 - 要求メッセージに情報を出力する
  • 3 - 要求メッセージと応答メッセージに情報を出力する
サービス・アカウント
「参照」をクリックし、表示されるリストでサービス・アカウントを選択する。サービス・アカウントを指定しない場合は、匿名サブジェクトが使用される。

関連トピック:

  • サービス・アカウント
  • サービス・アカウントの追加
ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーに使用するWebLogic Serverワーク・マネージャのインスタンスを選択します。デフォルトのワーク・マネージャは、他にワーク・マネージャがない場合に使用されます。
ワーク・マネージャは、応答処理のための応答メッセージを書き出すために使用される。ワーク・マネージャの詳細については、「ワーク・マネージャを使用したスケジューリング済み作業の最適化」およびWebLogic Serverの『Administration Consoleオンライン・ヘルプ』の「グローバル・ワーク・マネージャの作成」を参照してください。

Oracle Service BusからOracle Data Services Integratorへのアクセスの詳細については、「Oracle Service Busを介したデータ・サービスへのアクセス」を参照してください。

終わった後で

「次へ」をクリックして次のページでこのサービスの構成を続行するか、「最後」をクリックして「ビジネス・サービスの作成/編集 - 要約」ページでこの構成を確認して保存します。
