電子メール・トランスポート構成ページ

ビジネス・サービスの作成/編集 - 電子メール・トランスポート構成ページは、ビジネス・サービスの作成および編集(ビジネス・サービスの作成と構成およびビジネス・サービス構成の編集を参照)に関する一連のページの1つです。

このページでは、電子メール・トランスポート・プロトコルを使用するビジネス・サービスのトランスポート設定を構成します。表18-6に、このページの使用方法を示します。

表18-6 ビジネス・サービスの作成/編集 - 電子メール・トランスポート構成ページ
オプション
作成または編集の方法
SSLが必要
Secure Sockets Layerを使用して通信するには、このオプションを選択します。SMTPのデフォルトのSSLポート番号は465です。
注意:
  • SSLが機能するように、Oracle Service Bus信頼ストアで電子メール・サーバーの証明書を提示する必要があります。
  • TLSを使用している場合、デフォルトの信頼ストア($JAVA_HOME/jre/lib/security/cacerts)に信頼証明書を置くか、または、任意の信頼ストアに信頼証明書を置いて次に示すようにパスを$DOMAIN_HOME/bin/setDomainEnv.*ファイルに追加する必要があります。

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<path to trust store>

SMTPサーバー
name@domain_name.comのエンドポイントURIエントリに使用するデフォルトSMTPサーバーを選択します。表18-3で説明されているように、エンドポイントURIにSMTPサーバー・パラメータを指定した場合は、それらのサーバー・リソースがこのSMTPサーバー設定のかわりに使用されます。
「メール・セッション」オプションを使用する場合は、SMTPサーバーを選択しないでください。
SMTPリソース作成の詳細は、SMTPサーバーの追加を参照してください。
メール・セッション
name@domain_name.comのエンドポイントURIエントリに使用する構成済メール・セッションのJNDI名を入力します。表18-3で説明されているように、エンドポイントURIにJNDIメール・セッション・パラメータを指定した場合は、それらのメール・セッションがこのメール・セッション設定のかわりに使用されます。
「SMTPサーバー」オプションを使用する場合は、メール・セッションを入力しないでください。
送信元の名前
このサービスでの送信元の電子メール・アカウントの表示名を入力します。
送信元アドレス
このサービスでの送信元の電子メール・アカウントを入力します。
返信先の名前
電子メール・アカウントの返信先の表示名を入力します。
返信先アドレス
返信先の電子メール・アドレスを入力します。
接続タイムアウト
接続を切断するまでのタイムアウト時間を秒単位で入力します。0と入力した場合、タイムアウトは発生しません。
要求エンコーディング
電子メール・トランスポートにおける要求の文字セット・エンコーディングとして、デフォルトのISO-8859-1を受け入れるか、別の文字セット・エンコーディングを入力します。

終了後

「次」をクリックして次のページでこのサービスの構成を続行するか、「最後」をクリックしてビジネス・サービスの作成/編集 - サマリー・ページでこの構成を確認して保存します。
