HTTPトランスポート構成ページ

HTTPトランスポートはHTTPおよびHTTPSのエンドポイントをサポートします。

ビジネス・サービスの作成/編集 - HTTPトランスポート構成ページは、ビジネス・サービスの作成および編集(ビジネス・サービスの作成と構成およびビジネス・サービス構成の編集を参照)に関する一連のページの1つです。

このページでは、HTTPトランスポート・プロトコルを使用するビジネス・サービスのトランスポート設定を構成します。表18-10に、このページの使用方法を示します。

オプション
作成または編集の方法
読取りタイムアウト
読取りタイムアウト間隔を秒で入力します。
値ゼロ(0)はタイムアウトが発生しないことを示します。

接続タイムアウト間隔を秒で入力します。接続が確立する前にタイムアウトの期限が切れると、Oracle Service Busで接続エラーが発生します。
値ゼロ(0)はタイムアウトが発生しないことを示します。
HTTP要求メソッド
このパラメータにより、要求で次のいずれかのHTTPメソッドを使用できます。
  • POST - そのHTTP要求の本文の一部として、長さにかかわらずすべてのデータをソケット接続で直接渡します。この処理はクライアントから認識できず、URLも変わりません。RESTベースの要求の場合、POSTでは作成/置換操作を実行するか、または要求の操作を実行するようにトランスポートに指示します。
  • GET - 要求の一部として、取得対象を詳しく表した独自の情報を組み込むことができます。この情報は、問合せ文字列内の要求URLに追加された文字のシーケンスとして渡されます。ビジネス・サービスでGETを使用する場合、サービスのタイプに"任意のXMLサービス"を指定するか、または要求メッセージのタイプが"なし"に設定されている場合にサービスのタイプに"メッセージング・サービス"を指定できます。RESTベースの要求では、GETでリモート・リソースの表現が取得されます。
  • PUT - ビジネス・サービスでPUTを使用する場合、サービスのタイプに「任意のXMLサービス」または「メッセージ・サービス」を指定できます。PUTでは、既知の場所へのファイルのアップロードなど、RESTベースの要求で作成/置換操作を実行するようにトランスポートに指示します。
  • HEAD - ビジネス・サービスでHEADを使用する場合、サービスのタイプに任意のXMLサービスを指定するか、または応答メッセージのタイプがなしに設定されている場合に、サービスのタイプにメッセージング・サービスを指定できます。HEADはRESTベースの要求でリソースの完全な表現を取得するかわりに、リモート・リソースのヘッダー情報を取得するようトランスポートに指示します。
  • DELETE - ビジネス・サービスでPUTを使用する場合、サービスのタイプに「任意のXMLサービス」または「メッセージ・サービス」を指定できます。DELETEでは、RESTベースの要求で、削除操作を実行するようトランスポートに指示します。
注意: $outbound/transport/request/http:http-method変数でメソッドがすでに設定されている場合は、HTTP要求メソッドに選択したどのメソッドよりもその値が優先されます。
認証
次のいずれかを選択します:
  • 「なし」 - このサービスへのアクセスに認証を必要としないことを指定します。
  • 「基本」 - このサービスにアクセスするには、基本認証が必要であることを指定します。

    • 基本認証では、WebLogic Serverでユーザー名とパスワードを使用し、セキュリティ・レルム(Lightweight Directory Access Protocol (LDAP)ディレクトリ・サービスやWindows Active Directoryなど)で構成された認証プロバイダに対してクライアントの認証が行われます。クライアントは、HTTP要求ヘッダーでユーザー名とパスワードを送信する必要があります。

    HTTPでの基本認証は、パスワードがクリア・テキストで送信されるためお薦めしません。HTTPSでは暗号化されたチャネルが提供されるため、パスワードはHTTPSでの送信が安全です。

    警告:デフォルトでは、すべてのユーザー(認可ユーザーおよび匿名ユーザー)がビジネス・サービスにアクセス可能です。ビジネス・サービスにアクセスできるユーザーを制限するには、トランスポート・レベルの認可ポリシーを作成する必要があります。トランスポート・レベルのアクセス・ポリシーの編集を参照してください。

  • 「クライアント証明書」 - 暗号化通信および強力なクライアント認証(双方向SSL)を指定します。詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのトランスポート・レベルのセキュリティの構成(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557)を参照してください。
サービス・アカウント
サービス・アカウントを入力します。サービス・アカウントとは、ユーザー名とパスワードの別名リソースです。基本認証が必要フィールドを選択した場合、このフィールドは必須です。

ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーに使用するWebLogic Serverワーク・マネージャのインスタンスを選択します。デフォルトのワーク・マネージャは、他にワーク・マネージャがない場合に使用されます。ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
要求エンコーディング
HTTPトランスポートにおける要求の文字セット・エンコーディングとして、デフォルトのISO-8859-1を受け入れるか、別の文字セット・エンコーディングを入力します。
応答エンコーディング
HTTPトランスポートにおける応答の文字セット・エンコーディングとして、デフォルトのISO-8859-1を受け入れるか、別の文字セット・エンコーディングを入力します。
プロキシ・サーバー
プロキシ・サーバー・リソースを入力するか、「参照」をクリックして構成されたプロキシ・サーバー・リソースの一覧からエントリを選択します。
HTTPリダイレクトの追跡
HTTPリダイレクト(応答コード3xxによる要求)を自動的に追跡することを指定します。リダイレクトは、ビジネス・サービスのURLにアウトバウンド要求を送信し、そのサービスから、URLが無効になったためこの要求を別のURLに送信する必要があることを示す応答コード(たとえば302)が戻された場合に発生します。
「HTTPリダイレクトの追跡」チェック・ボックスが選択されている場合、要求は自動的に新しいURLに再送信され、ユーザー側でのアクションは発生しません。HTTPリダイレクトを自動的に追跡しない場合は、このチェック・ボックスの選択を解除します。
チャンク・ストリーミング・モードの使用
HTTPチャンク転送エンコーディングを使用してメッセージを送信する場合は、このオプションを選択します。
注意: 「HTTPリダイレクトの追跡」オプションを使用する場合は、チャンク・ストリーミングを同時に使用しないでください。チャンク・モードでは、リダイレクトおよび認証を自動的に処理できません。
セッション固定性
ビジネス・サービスのHTTPリクエストにセッション固定性(セッション・アフィニティとも呼ばれます)を使用する場合は、このオプションを選択します。
固定セッションID名
「セッション固定性」が有効になっている場合に、セッションの識別名を入力します。
