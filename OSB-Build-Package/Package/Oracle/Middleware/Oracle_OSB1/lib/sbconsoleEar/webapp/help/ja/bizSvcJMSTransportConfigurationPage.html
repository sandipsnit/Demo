JMSトランスポート構成ページ

ビジネス・サービスの作成/編集 - JMSトランスポート構成ページは、ビジネス・サービスの作成および編集(ビジネス・サービスの作成と構成およびビジネス・サービス構成の編集を参照)に関する一連のページの1つです。

詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのJMSトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037)を参照してください。

このページでは、JMSトランスポート・プロトコルを使用するビジネス・サービスのトランスポート設定を構成します。表18-13に、このページの使用方法を示します。

表18-13 ビジネス・サービスの作成/編集 - JMSトランスポート構成ページ(続き)
オプション
作成または編集の方法
宛先タイプ
JMSブリッジ宛先のタイプを選択します。
  • 「キュー」(ポイント・ツー・ポイントの場合)
  • 「トピック」(パブリッシュ/サブスクライブの場合)
メッセージ・タイプ
次のいずれかを選択します:
  • 「バイト」(解釈されていないバイト・ストリームの場合)
  • 「テキスト」(テキスト・メッセージの場合)
このオプションは、レスポンスのメッセージ・タイプとしてJavaを選択した場合は、無効になります。
レスポンス・キュー
このオプションは、「宛先タイプ」で「キュー」が選択されている場合にのみ使用できます。
次のいずれかのレスポンス・オプションを選択します。
  • なし - レスポンスはありません。このオプションは、一方向の操作に対して設定します。
  • すべてのリクエストURIに対して1つ - レスポンスを処理する単一のURIを指定し、エンコーディングやタイムアウトなどのその他のレスポンス構成の詳細を設定できます。オプションで、JMS/JNDI資格証明を渡すためのJMSサービス・アカウントを選択することもできます。
  • リクエストURIごとに1つ - このオプションでは、リクエストURIごとに1つのレスポンスURIまたは宛先を指定することにより、レスポンスのフェイルオーバーを実現します。
オプションで、リクエストとレスポンスの組合せごとにJMS/JNDI資格証明のサービス・アカウントを選択できます。
レスポンス・パターン
このオプションは、「レスポンス・キュー」フィールドでレスポンス・オプションを選択した場合にのみ使用できます。
次のいずれかを選択します:
  • Oracle WebLogic Server上で実行しているすべてのサービス(JAX-RPCサービスを除く)の場合は、「JMSCorrelationID」を選択します。
  • Oracle WebLogic ServerでJAX-RPCサービスを実行している場合、JMSMessageIDを選択します。
詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのJMSリクエスト/レスポンスのメッセージIDおよび相関IDのパターン(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051)を参照してください。
ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーに使用するOracle WebLogic Serverワーク・マネージャのインスタンスを選択します。デフォルトのワーク・マネージャは、他にワーク・マネージャがない場合に使用されます。
たとえば、ビジネス・サービスがJMSトランスポート・プロトコルに対応している場合、ビジネス・サービスのエンドポイントは、そのディスパッチ・ポリシーに関連付けることのできるMDB (メッセージドリブンBean)のJARファイルになります。
ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
要求エンコーディング
リクエストをエンコーディングする文字セットを入力します。デフォルトはUTF-8です。
応答エンコーディング
このオプションは、「レスポンス・キュー」フィールドでいずれかのレスポンス・オプションが選択されている場合にのみ使用できます。
レスポンスをエンコーディングする文字セットを入力します。デフォルトはUTF-8です。
応答のタイムアウト
このオプションは、「レスポンス・キュー」フィールドでいずれかのレスポンス・オプションが選択されている場合にのみ使用できます。
接続を切断するまでにレスポンスを待機する時間を秒単位で入力します。デフォルトはゼロ(0)です。この場合、レスポンスはタイムアウトしません。
クライアントJar
このオプションは、メッセージ・サービスのレスポンス・タイプがJavaであるサービスの場合に使用できます。Javaオブジェクトを含むメッセージをデキューするために使用されるクライアントJARを選択します。クライアントJARを選択すると、クライアントJARがクラスパスに設定されます。

詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのメッセージでのJavaオブジェクトの送信および受信(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874)を参照してください。
応答URI
このオプションは、「すべてのリクエストURIに対して1つ」レスポンス・オプションと「JMSCorrelationID」レスポンス・パターンを選択した場合にのみ使用できます。
応答URIを、次の書式で入力します。
jms://host:port/connection_factory/jndi_destination
複数のサーバーを対象とする場合は、次の書式を使用します。
jms://host1:port,host2:port/connection_factory/jndi_destination
レスポンスURIではホストとポートを省略することもできます。たとえば、次のように入力します。
jms:///connection_factory/jndi_destination

ホストとポートを省略した場合は、接続ファクトリ/宛先のルックアップがローカル・サーバー上で行われます。これは、たとえばリクエストURIが外部の接続ファクトリ/宛先であっても、レスポンスをローカル・サーバーに送信する場合に役立ちます。

注意: Oracle WebLogic Serverでは、"myqueues/myqueue"のようにJNDI名にフォワード・スラッシュを使用できますが、フォワード・スラッシュを使用したJNDI名はOracle Service Busで必要なURI書式の妨げになるため、その名前を使用できません。この問題を解決するには、JMS外部サーバーを定義し、URIでその外部サーバーを参照します。詳細は、Oracle Fusion Middleware Oracle WebLogic Server管理コンソール・オンライン・ヘルプ(http://www.oracle.com/pls/as111170/lookup?id=WLACH)の外部サーバーの構成を参照してください。

リクエストのレスポンス
このオプションは、「JMSCorrelationID」パターンに「リクエストURIごとに1つ」レスポンス・オプションを選択し、レスポンスのフェイルオーバーを実現している場合に使用できます。
前の「レスポンスURI」フィールドの説明に従って、汎用のトランスポート・ページで入力したリクエストURIごとに1つのレスポンスURIを入力します。
JMS/JNDI資格証明のオプションのサービス・アカウントを選択できます。このアカウントは、サービスによってリクエスト・キューとレスポンス・キューの両方に使用されます。
対象 - レスポンス
このオプションは、「JMSMessageID」パターンに「すべてのリクエストURIに対して1つ」レスポンス・オプションを選択した場合に使用できます。
レスポンスを受信する対象サーバーの名前を入力し、「レスポンスURI」フィールドの説明に従ってレスポンスURIを入力します。
リクエストの接続
このオプションは、「JMSMessageID」パターンに「リクエストURIごとに1つ」レスポンス・オプションを選択し、レスポンスのフェイルオーバーを実現している場合に使用できます。
「順序番号」フィールドの番号順に識別される各リクエストURIに対して、オプションでJMS接続ファクトリ名を入力します。名前を入力しない場合、JMSトランスポートはリクエストURIからの接続ファクトリを使用します。
JMS/JNDI資格証明のオプションのサービス・アカウントを選択できます。このアカウントは、サービスによってリクエスト・キューとレスポンス・キューの両方に使用されます。
対象 - 宛先
このオプションは、「JMSMessageID」パターンに「リクエストURIごとに1つ」レスポンス・オプションを選択し、レスポンスのフェイルオーバーを実現している場合に使用できます。このフィールドは、「リクエストの接続」とともに使用します。
(現在のドメインのサーバーから判断されて)表示された各対象サーバー(クラスタ内の管理対象サーバーなど)のリクエストURIが、「リクエストの接続」フィールドに対応する番号順に一覧表示されます。各対象のリクエストURIごとに、レスポンスを受信する対象サーバーの宛先キューを入力します。
注意: Oracle Service Busの開発環境では単一サーバー環境のみがサポートされているため、1つの対象のみが表示されます。複数サーバー環境でこのフィールドを構成する場合は、このサービスをランタイム環境にデプロイし、Oracle Service Bus管理コンソールでサービス構成を完了してください。
JMSサービス・アカウント
このオプションは、「レスポンス・キュー」フィールドで「なし」オプションまたは「すべてのリクエストURIに対して1つ」オプションを選択した場合に使用できます。
JMSサーバーによって管理されるJMSリソースに使用するサービス・アカウントを選択します。サービス・アカウントは、ユーザーIDとパスワードの別名リソースで、リクエストとレスポンスの両方で使用されます。JNDIとJMSの両方で同じサービス・アカウントが使用されます。
このフィールドは、「呼出し元のサブジェクトを渡す」オプションとは相互に排他的です。JMS/JNDI認証にはいずれか一方を使用してください。
詳細は、サービス・アカウントを参照してください。
SSLを使用
このオプションはリクエストをTLS/SSL接続で行う場合のみ選択します。
TLS/SSL (Secure Sockets Layer)では、ネットワークで接続される2つのアプリケーションが互いのIDを認証し、アプリケーション間で交換されるデータを暗号化できるようにすることによって、安全な接続が可能になります。認証により、サーバーおよび場合によってはクライアントで、ネットワークで接続しているアプリケーションのIDを確認できます。また、宛先のJNDIエントリに対してアクセス制御が設定されていることにより、管理者から個々のJMS宛先(キューまたはトピック)へのアクセスが制限されている場合、JNDIツリー内でのルックアップ時にサービスで認証を行う必要があります。「サービス・アカウント」または「呼出し元のサブジェクトを渡す」オプションを使用して認証します。
有効期限
メッセージの有効期限が切れる時間間隔(ミリ秒単位)。デフォルトは0です。メッセージの有効期限は無期限です。
メッセージ永続性を有効にする
サービスが使用するJMS配信モード。スループットの信頼性のバランスを調整できます。メッセージを確実に配信するには、このオプションを選択します(デフォルト)。メッセージが失われることがあっても許される場合は、このオプションの選択を解除するとスループットが向上します。
順序単位
メッセージ順序単位を入力します。メッセージ順序単位により、メッセージ・プロデューサは処理順序に関して複数のメッセージを1つの単位にグループ化できるようになります。1つの順序単位は、単位内のすべてのメッセージが作成された順に処理されることを必要とします。
呼出し元のサブジェクトを渡す
メッセージを送信する場合に、Oracle Service Busが認証されたサブジェクトを渡すようにするには、このチェック・ボックスを選択します。
このフィールドを有効にし、異なるドメインのJMSリソースをビジネス・サービスのターゲットにする場合は、両方のドメインでグローバルな信頼を有効にします。Oracle WebLogic Serverの保護のWebLogicドメインのセキュリティの構成を参照してください。
このフィールドは、「サービス・アカウント」オプションとは相互に排他的です。JMS/JNDI認証にはいずれか一方を使用してください。
このフィールドを有効にし、異なるドメインのJMSリソースをビジネス・サービスのターゲットにする場合は、両方のドメインでグローバルな信頼を有効にします。Oracle Fusion Middleware Oracle WebLogic Serverの保護のWebLogicドメインのセキュリティの構成( http://www.oracle.com/pls/as111170/lookup?id=SECMG402)を参照してください。
JNDIタイムアウト
JNDIツリーで宛先または接続ファクトリをルックアップするときに使用するJNDI接続タイムアウト(秒単位)。
デフォルトはゼロ(0)です。この場合、接続はタイムアウトしません。
