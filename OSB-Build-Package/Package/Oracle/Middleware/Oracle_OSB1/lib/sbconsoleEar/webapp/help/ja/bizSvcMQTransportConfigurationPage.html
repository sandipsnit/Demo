MQトランスポート構成ページ

ビジネス・サービスの作成/編集 - MQトランスポート構成ページは、ビジネス・サービスの作成および編集(ビジネス・サービスの作成と構成およびビジネス・サービス構成の編集)を参照)に関する一連のページの1つです。

詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのMQトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117)を参照してください。

開始前

MQ接続リソースを構成します。MQ接続を参照してください。

MQトランスポートを構成するには

このページでは、MQトランスポート・プロトコルを使用するビジネス・サービスのトランスポート設定を構成します。表18-14に、このページの使用方法を示します。

表18-14 ビジネス・サービスの作成/編集 - MQトランスポート構成ページ
オプション
作成または編集の方法
メッセージ・タイプ
次のいずれかを選択します:
  • 「バイト」(解釈されていないバイト・ストリームの場合)
  • 「テキスト」(テキスト・メッセージの場合)
応答が必要
アウトバウンド・メッセージの送信後に応答を受け取ることを指定する場合は、このオプションを選択します。
ポーリング間隔
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
ポーリング間隔をミリ秒単位で入力します。デフォルト値は1000です。
応答相関パターン
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
レスポンス相関パターンが次のいずれに基づく必要があるかを指定します。
  • MessageID
  • CorrelationID
  • 動的キュー - WebSphere MQ実装がレスポンス相関に動的キューを使用している場合は、このオプションを選択します。MQトランスポートでは、一時的な動的キューのみがサポートされます。
相関値の自動生成
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
CorrelationIDまたはMessageIDを自動的に生成します。
モデル・キュー
「レスポンス相関パターン」が「動的キュー」の場合のみです。動的キューを生成するために使用するモデル・キューの名前を入力します。
MQ応答URI
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
レスポンスの宛先。エンドポイントURIと同じ書式のレスポンスURIを入力します。
mq://local_queue_name?conn=mq_connection_resource
または、動的キューを使用している場合は次のように入力します。
mq://dynamic_queue_prefix?conn=mq_connection_resource
dynamic_queue_prefix (32文字に制限)は、MQサーバー上に動的キューを作成するために使用されます。キュー名は、この接頭辞に一意のIDが追加された名前になります。たとえば、dynamic_queue_prefixがexampleの場合、動的キューにはexample123129083821という名前が付けられます。
動的キューのレスポンスURIには、ワイルドカードとしてアスタリスク(*)を使用することもできます。たとえば、次のように入力します。
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
URIにdynamic_queue_prefixを指定しない場合、トランスポートは、MQサーバーによって生成される動的キュー名を使用します。URIに明示的なmq_connection_resourceを指定しない場合(ベスト・プラクティス)、トランスポートは、エンドポイントURIからのmq_connection_resourceを使用します。
詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのMQトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117)を参照してください。
応答のタイムアウト
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
接続を切断するまでの応答の待機時間を秒単位で入力します。
ディスパッチ・ポリシー
ディスパッチ・ポリシーは、サービス・エンドポイントに使用するWLSワーク・マネージャのインスタンスを参照します。
ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
動的キュー・プーリング
「レスポンス相関パターン」が「動的キュー」の場合のみです。プールされた接続を使用して動的キューに接続する場合は、このオプションを選択します。
動的キューに対して個別の接続プールを使用する場合は、動的キュー専用のMQ接続リソースを構成することを検討してください。
リクエストごとに新しい動的キュー・インスタンスを作成する場合(およびレスポンス後にキューを破棄する場合)は、このオプションを選択しないでください。
エンドポイントURI 'PUT'オプション
次の中からMQ PUTメッセージのオプションを入力します。
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
複数のオプションを区切るには、|または+を使用できます。たとえば、次を指定できます。
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
MQ PUTメッセージのオプションはメッセージがアウトバウンド・キューに配置された場合に適用されます。
MQ認識できない応答URI
認識できない応答メッセージを送信するキューを表すURIを入力します。この設定は「相関値の自動生成」チェック・ボックスが選択されている場合にのみ有効です。
このフィールドに対して値を指定しない場合、認識できない応答メッセージは削除されます。
RFH2ヘッダーのプロセス
メッセージ・ペイロードからのWebSphere MQ RFH2ヘッダーを解析し、RFH2データを含むRFH2Headersトランスポート・ヘッダーを自動生成するには、このオプションを選択します。
このオプションを選択しない場合、ペイロードは受信時の状態のまま渡されます。

MQトランスポートのRFH2ヘッダーの処理方法に関する詳細は、Oracle Fusion Middleware Oracle Service Bus開発者ガイドのRFH2ヘッダーについて(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136)を参照してください。

終了後

「次」をクリックして次のページでこのサービスの構成を続行するか、「最後」をクリックしてビジネス・サービスの作成/編集 - サマリー・ページでこの構成を確認して保存します。
