メッセージ・タイプの構成ページ

ビジネス・サービスの作成/編集 - メッセージ・タイプの構成ページは、ビジネス・サービスの作成および編集(ビジネス・サービスの作成と構成およびビジネス・サービス構成の編集を参照)に関する一連のページの1つです。

このページでは、ビジネス・サービスのサービス・タイプがメッセージング・サービスである場合、ビジネス・サービスのタイプを構成できます。

メッセージング・サービス用のバインド定義は、交換されるメッセージのコンテンツ・タイプの構成で構成されています。応答のコンテンツ・タイプは、要求のコンテンツ・タイプと同じである必要はありません。そのため、応答は個別に構成されます(たとえば、サービスでMFLメッセージを受信し、XMLの受信確認を戻すことも可能です)。

注意: タイプが「メッセージ・サービス」である、電子メール、ファイル、FTPまたはSFTPトランスポート・ビジネス・サービスがサポートするのは、一方向のメッセージングのみであり、「応答メッセージ・タイプ」は「なし」である必要があります。「なし」以外を選択した場合、ファイル、FTPまたはSFTPプロトコルはトランスポート構成ページで使用できません。

表18-2に、ビジネス・サービスの作成/編集 - メッセージ・タイプの構成ページの使用方法を示します。

表18-2 ビジネス・サービスの作成/編集 - メッセージ・タイプの構成ページ
オプション
作成または編集の方法
要求メッセージ・タイプ
要求メッセージのメッセージのタイプを選択します。
  • 「なし」 - 要求メッセージ(HTTP GETなど)がない場合は、このオプションを選択します。
  • 「バイナリ」 - メッセージのコンテンツ・タイプが不明か、重要でない場合は、このオプションを選択します。
  • 「テキスト」 - メッセージをテキストに限定できる場合は、このオプションを選択します。
  • 「MFL」 - メッセージがMFL定義に準拠したバイナリ・ドキュメントの場合は、このオプションを選択します。MFLファイルは、1つのみ構成できます。

    • MFLの場合は、「参照」をクリックし、MFLブラウザでMFLを選択してから、「発行」をクリックできます。

  • 「XML」 - メッセージがXMLドキュメントの場合は、このオプションを選択します。一部の型情報を指定するために、交換されるXMLドキュメントのXMLスキーマ型を宣言できます。
  • 「Java」 - Javaオブジェクトがリクエストで送信されている場合は、このオプションを選択します。JMSトランスポートはJavaオブジェクト・メッセージに使用されます。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのメッセージでのJavaオブジェクトの送信および受信(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874)を参照してください。
応答メッセージ・タイプ
応答メッセージのメッセージのタイプを選択します。
  • 「なし」 - 応答メッセージがない場合は、このオプションを選択します。
  • 「バイナリ」 - メッセージのコンテンツ・タイプが不明か、重要でない場合は、このオプションを選択します。
  • 「テキスト」 - メッセージをテキストに限定できる場合は、このオプションを選択します。
  • 「MFL」 - メッセージがMFL定義に準拠したバイナリ・ドキュメントの場合は、このオプションを選択します。MFLファイルは、1つのみ構成できます。

    • MFLの場合は、「参照」をクリックし、MFLブラウザでMFLを選択してから、「発行」をクリックできます。

  • 「XML」 - メッセージがXMLドキュメントの場合は、このオプションを選択します。一部の型情報を指定するために、交換されるXMLドキュメントのXMLスキーマ型を宣言できます。
  • 「Java」 - Javaオブジェクトがレスポンスで受信されている場合は、このオプションを選択します。JMSトランスポートはJavaオブジェクト・メッセージに使用されます。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのメッセージでのJavaオブジェクトの送信および受信(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874)を参照してください。
