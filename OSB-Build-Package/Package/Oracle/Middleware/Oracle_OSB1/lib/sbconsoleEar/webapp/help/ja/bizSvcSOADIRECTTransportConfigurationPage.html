SOA-DIRECTトランスポート構成ページ

表18-17では、SOA-DIRECTトランスポート固有の構成オプションについて説明します。

SOA-DIRECTトランスポートの詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのSOA-DIRECTトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV909)を参照してください。

表18-17 ビジネス・サービスの作成/編集 - SOA-DIRECTトランスポート構成ページ
プロパティ
説明
JNDIサービス・アカウント
オプション。ターゲットSOAサービスのJNDIルックアップに使用するセキュリティ資格証明を指定します。サービス・アカウントは静的である必要があります。「参照」をクリックしてサービス・アカウントを選択します。サービス・アカウントを指定しない場合は、匿名サブジェクトが使用されます。
ロール
必須。サービスが使用する通信パターンを指定します。次のいずれかのオプションを選択します。
  • 同期クライアント(デフォルト) - このロールでは、非同期コールバックが発生しないため、「コールバック・プロキシ」オプションおよび「WS-Addressingのバージョン」オプションは無効になります。
  • 非同期クライアント - このロールでは、非同期コールバックが通常必要であるため、コールバック・プロキシを指定できます。また、WS-Addressingのバージョンを選択する必要があります。非同期オプションは、WSDLサービスがSOAPタイプの場合にのみ有効になります。
  • サービス・コールバック - このロールは、外部サービスの呼出し後に非同期コールバックをSOAサービスに戻すために使用します。

    • 「サービス・コールバック」ロールのサービスでは、ロード・バランシングまたはフェイルオーバーは使用できません。

コールバック・プロキシ
オプション。「非同期クライアント」ロールを選択した場合にのみ有効になります。
このオプションでは、コールバックを受信するプロキシ・サービスを指定できます。コールバック・プロキシを選択し、リクエストまたはプロキシ・サービス・パイプラインにWS-Addressingが指定されていない場合は、Oracle Service BusによってReplyToプロパティがSOAPヘッダーに自動的に移入されます。SBトランスポートを使用するWSDLプロキシ・サービスを選択する必要があります(RMIの場合)。また、コールバック・プロキシ・サービスがWS-Addressingを解釈する必要があります。
WS-Addressingのプロパティがリクエストで送信されるか、プロキシ・サービス・パイプラインに設定されている場合は、このオプションで設定するコールバック・プロキシのかわりに、それらのプロパティが使用されます。
コールバック・プロキシを指定せず、リクエストにもReplyToプロパティが含まれていない場合は、プロキシ・サービス・パイプラインを通じてSOAPヘッダーにReplyToプロパティを指定する必要があります。
フォルト・プロキシ
このオプションは現在サポートされていません。非同期パターンでフォルトを処理するには、コールバック・サービスを構成する必要があります。
WS-Addressingのバージョン
必須。「非同期クライアント」ロールを選択した場合にのみ有効になります。
リクエストまたはプロキシ・サービス・パイプラインにWS-Addressingが指定されていない場合に使用するデフォルトのWS-Addressingのバージョンを指定します。WS-Addressingのプロパティがリクエストで送信されるか、プロキシ・サービス・パイプラインに設定されている場合は、このオプションで設定するWS-Addressingのバージョンのかわりに、それらのプロパティが使用されます。
環境間でWS-Addressingのバージョンが一致しない場合は、プロキシ・サービス・パイプラインで必要な変換を行います。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドの変換例(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1827)を参照してください。
ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーに使用するWebLogic Serverワーク・マネージャのインスタンスを選択します。デフォルトのワーク・マネージャは、他にワーク・マネージャがない場合に使用されます。
ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
呼出し元のサブジェクトを渡す
オプション。SOAサービスの呼出しでOracle Service Busがプロキシ・サービスから認証済サブジェクトを渡すようにする場合は、このオプションを選択します。このオプションを選択した場合、「呼出し元のサブジェクトを渡す」の代替オプションである「呼出しサービス・アカウント」は無効になります。
注意: クライアントと対象サーバーが別のドメインにある場合は、双方のドメインの信頼関係を有効にしてください。詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのセキュリティのクロスドメイン・セキュリティのサポートに関する重要な情報(http://www.oracle.com/pls/as111170/lookup?id=SECMG403)を参照してください。
呼出しサービス・アカウント
オプション。「呼出し元のサブジェクトを渡す」のかわりに使用できます。RMI呼出しに使用するサービス・アカウントを選択し、カスタムのセキュリティ資格証明を指定します。任意のタイプ(「パス・スルー」、「静的」、「マッピング」)のサービス・アカウントを選択できます。
「参照」をクリックしてサービス・アカウントを選択します。サービス・アカウントを指定しない場合は、匿名サブジェクトが使用されます。
