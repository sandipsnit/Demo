ビジネス・サービスの作成と構成

この項では、ビジネス・サービスの作成方法および構成方法について説明します。

JCAサービスの場合、ビジネス・サービスをJCAバインド(JCAバインド・リソースからのビジネス・サービスの生成を参照)またはOracle Enterprise Repository (Oracle Enterprise Repositoryからのビジネス・サービスの生成を参照)から生成できます。

  1. セッションの作成または編集をまだ行っていない場合は、「作成」をクリックして新しいセッションを作成するか、「編集」をクリックして既存のセッションに入ります。チェンジ・センターの使用を参照してください。
  2. 「プロジェクト・エクスプローラ」を選択し、ビジネス・サービスに追加するプロジェクトまたはフォルダを選択します。
  3. プロジェクト/フォルダ・ビュー・ページで、「リソースの作成」ドロップダウン・リストから「ビジネス・サービス」を選択します。
  4. ビジネス・サービスの作成/編集 - 全般的な構成ページで、サービスの名前を指定し、作成するサービスのタイプを選択します。

    命名については、リソースの命名制約のガイドに従ってください。
  5. 「次」をクリックします。この後のページは、最初のページでの選択によって異なります。サマリー・ページにたどり着くまで、以降の各ページで適切な情報を入力し、「保存」をクリックして、サービスを現在のセッションで保存します。

    6. 各ページを完了するための詳細な手順については、次を参照してください。

    • 全般的な構成ページ
    • メッセージ・タイプの構成ページ
    • トランスポート構成ページ
    • (プロトコル)トランスポート構成ページ
    • SOAPバインド構成ページ
    • メッセージ処理構成ページ
    • サマリー・ページ
    • ビジネス・サービスのポリシー・ページ
    • ビジネス・サービスのセキュリティ・ページ

WS-PolicyがアタッチされたWSDLからビジネス・サービスを作成すると、そのポリシー(読取り専用)が(プロトコル)トランスポート構成ページに表示されます。サービスのWS-Policyのいずれかで認証を指定している場合は、サービス・アカウントを選択する必要があります。このビジネス・サービスにルーティングされるプロキシ・サービスは、このサービス・アカウントを使用してビジネス・サービスへの認証を行います。

JCAバインド・リソースからのビジネス・サービスの生成

Oracle Service BusのアウトバウンドJCAバインド・リソースからJCAビジネス・サービスを生成できます。JCAバインド・リソースの詳細は、JCAバインドを参照してください。

JCAバインドからJCAビジネス・サービスを生成するには:

  1. Oracle Service Bus管理コンソールの「チェンジ・センター」で「作成」または「編集」をクリックします(まだ作成モードまたは編集モードに入っていない場合)。
  2. 「リソース・ブラウザ」で「JCAバインド」をクリックします。
  3. サービスの生成元になるJCAバインドを指定して、「アクション」アイコンをクリックします。
  4. 表示されたウィンドウで、生成するWSDLとサービスの名前を確認し、これらの新規リソースの場所を選択して、「生成」をクリックします。

    5. サービスおよびサービスに対応するWSDLがOracle Service Busによって生成されます。

  5. エンドポイントURIなど、生成されたサービスのその他の構成を必要に応じて変更します。

Oracle Enterprise Repositoryからのビジネス・サービスの生成

Oracle Service Busの開発環境を使用して、Oracle Enterprise Repositoryからビジネス・サービスを生成できます。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのOracle Enterprise Repositoryからのビジネス・サービスの生成(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1797)を参照してください。
