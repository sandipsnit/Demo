ビジネス・サービスのポリシー・ページ

ビジネス・サービスの表示 - ポリシー・ページは、ビジネス・サービスの編集(ビジネス・サービスの作成と構成およびビジネス・サービスの表示 - 構成の詳細ページを参照)に関する一連のページの1つです。

このページでは、サービス・タイプが「WSDLベースのXML Webサービス」、「メッセージ・サービス」、「任意のSOAPサービス」または「任意のXMLサービス」のビジネス・サービスのポリシー設定を構成します。表18-23に、このページの使用方法を示します。OWSMポリシーを「WSDLベースのXML Webサービス」、「メッセージ・サービス」または「任意のXMLサービス」プロキシ・サービスで使用するには、プロトコルをHTTPに設定する必要があります。

WSDLベースのサービスについて、サービスにバインドされているWLS 9.2のポリシーは、有効なWSDL内に置かれます(インライン化)。抽象ポリシーは、インライン化される前に前処理されます。OWSMポリシーは参照によってバインドされ、有効なWSDL内にインライン化されません。

サービス・ポリシー構成
「サービス・ポリシー構成」フィールドには次のオプションがあります。

OWSMポリシー

  • OWSMポリシー・ストアから - ポリシーは、Oracle Enterprise Managerで管理される中央のOracle Web Services Managerポリシー・ストアに定義されています。これらのポリシーは、WS-Security 1.0/1.1、SAML 1.1/2.0、KerberosToken Profileおよびその他の業界標準をサポートしています。

    このオプションを使用すると、事前定義リストからポリシーを追加できます。「サービス・レベル・ポリシー」の下の「追加」をクリックして、リストを表示します。目的のポリシーを検索する必要がある場合は、フィルタを使用します。ビジネス・サービスでは、クライアントサイド・ポリシーのみが表示されます。

    ビジネス・サービスを作成するために使用するWSDLに埋込みの標準WS-Securityポリシーが含まれていると、Oracle Service Busはエラーをスローして競合を表示します。この競合を解決するには、「OWSMポリシー・ストアから」を選択して「互換性の追加」をクリックし、埋込みWSDLポリシーを互換性のあるOracle Web Services Managerポリシーで置換します。Oracle Service Busは、WSDLに埋め込まれた標準ポリシーに基づいて、Oracle Web Services Managerポリシー・ストアから最も似ているポリシーを見つけます。アルゴリズムによって、一致するポリシーが1つまたは複数返されるか、または1つも返されない場合もあります。

    「互換性の追加」をクリックしても、Oracle Service Busで互換性のあるポリシーが見つからない場合は、「追加」をクリックして、適切なOracle Web Services Managerポリシーを選択します。Oracle Service Busで互換性のあるポリシーが複数返された場合は、使用しないポリシーを削除します。

    注意: このオプションは、ドメインでOracle Web Services Managerを使用するように構成している場合にのみアクティブ化されます。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのOracle Web Services ManagerによるOracle Service Busの保護( http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681)を参照してください。

WLS 9ポリシー

  • WSDLから - サービス・ポリシーがサービスの基になるWSDLに関連付けられている場合は、このオプションを選択します。これらのポリシーは、WS-Security 1.0、SAML 1.1およびその他の業界標準をサポートしています。

    このオプションを使用すると、WSDLからのリクエスト・ポリシーおよびレスポンス・ポリシーを表示できます(読取り専用)。

    注意: Oracle Service BusでサポートされないWS-Securityポリシーに関するビジネス・サービス構成のエラーを受信した場合は、「OWSMポリシー・ストアから」オプションを使用して、埋込みWSDLポリシーを互換性のあるOWSMポリシーで置換してください。

  • 事前定義ポリシーまたはWS-Policyリソースから - Oracle Service Bus管理コンソールから、サービス・レベル・ポリシー、操作レベル・ポリシー(リクエスト・メッセージとレスポンス・メッセージの両方に適用するポリシー)、リクエスト・ポリシー、およびレスポンス・ポリシーを追加するには、このオプションを選択します。

    ポリシーは、Oracle WebLogic Serverで事前定義されているか、WS-Policyリソースを持つOracle Service Busでユーザー定義されているかのいずれかです。これらのポリシーは、WS-Security 1.0、SAML 1.1およびその他の業界標準をサポートしています。

    このオプションを使用すると、サービス、操作、リクエストおよびレスポンスの各レベルでポリシーを追加できます。該当するセクションの下の「追加」をクリックして、リストからポリシーを選択します。

注意: ポリシー・バインド・モデルは相互に排他的です。サービス内には1つのポリシー・タイプのみ使用できます。サービスに直接ポリシーをバインドすると、すべてのWSDLベースのポリシーは無視されます。

ポリシーを追加した後は、セキュリティ・ページでオーバーライドを指定できます。
