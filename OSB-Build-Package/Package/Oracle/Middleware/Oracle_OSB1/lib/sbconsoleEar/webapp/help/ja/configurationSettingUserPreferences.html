ユーザー・プリファレンスの設定

このページでは、Oracle Service Bus管理コンソールの表示オプションとプリファレンスをカスタマイズし、保持できます。「ユーザー・プリファレンス」を更新(変更)する場合、変更セッションをアクティブ化する必要はありません。

表26-7では、ページのオプションおよびコントロールについて説明します。

表26-7 ユーザー・プリファレンス - オプションおよびコントロール
操作
手順
Oracle Service Bus管理コンソールを起動すると表示される「ホーム・ページ」の変更
最初のドロップダウン・リストから、メイン・メニューの項目(「操作」、「リソース・ブラウザ」、「プロジェクト・エクスプローラ」、「セキュリティ構成」または「システム管理」を選択します。
2番目のドロップダウン・リストから、サブ・メニューの項目の1つを選択します。(サブメニューの項目内容は、最初のメニューで選択した項目内容によって動的に変わります。)
検索フィルタの表示
「はい」または「いいえ」を選択します。
デフォルトでは、検索フィルタは表示されません(閉じている状態)。「はい」を選択すると、検索フィルタが常に表示されます(開いている状態)。
ステージ・コメントの表示
「はい」または「いいえ」を選択します。
デフォルトでは、パイプラインのステージ注釈は表示されません(閉じている状態)。「はい」を選択すると、ステージ注釈が常に表示されます(開いている状態)。
リソース・メタデータの表示
「はい」または「いいえ」を選択します。
デフォルトでは、リソース・メタデータ・セクションは表示されています(開いている状態)。「いいえ」を選択すると、メタデータ・セクションは表示されません(閉じている状態)。
Oracle Service Bus管理コンソールが「ダッシュボード」のデータの表示を更新する回数の変更
「ダッシュボード・リフレッシュ間隔」ドロップダウン・リストで、「ダッシュボード」のリフレッシュ間隔(「リフレッシュなし」、「1分」、「2分」、「3分」、「5分」、「10分」、「20分」、「30分」または「60分」)を選択します。
デフォルトの間隔は、「リフレッシュなし」ですが、定義済の他の時間を選択できます。たとえば、「5分」を選択すると、「ダッシュボード」の表示は5分ごとにリフレッシュされます。
注意: ダッシュボードのリフレッシュ間隔の値でデフォルト(「リフレッシュなし」)以外を選択すると、ブラウザは、一定間隔で定期的にダッシュボードの表示をリフレッシュします。これにより、Oracle Service Bus管理コンソールを操作していない場合でも、セッションのタイムアウトを防ぐことができます。
注意: ダッシュボードのリフレッシュ間隔の機能を使用するときには十分注意してください。Oracle Service Bus管理コンソール・セッション中に席を離れてコンピュータを放置しないでください。セッションがアクティブになっている間に、他の人がそのコンピュータを使用して再認証なしでセッションにアクセスするおそれがあります。
「ダッシュボード」にアラートの履歴データを表示する間隔の変更
「アラート履歴期間」ドロップダウン・リストで、履歴データを表示する間隔(「30分」、「1時間」、「2時間」、「3時間」または「6時間」)を選択します。デフォルト値は30分(30分前までに受信したアラートを表示)ですが、事前定義された期間を選択することもできます。たとえば、「1時間」を選択すると、1時間前までに受信したアラートが「ダッシュボード」に表示されます。
