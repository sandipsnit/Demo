Oracle Service Busについて

Oracle Service Busは、構成ベースでポリシー駆動のEnterprise Service Busです。スケーラビリティと信頼性の高いサービス志向の統合を行ったり、サービス管理、異なるIT環境に対して従来のメッセージ・ブローカリングを提供したりできます。また、インテリジェントなメッセージ・ブローカリング、メッセージのルーティングと変換、サービス監視および管理が、統一したソフトウェア製品で融合されます。Oracle Service Bus管理コンソールを使用すると、サービスやポリシー構成を管理したり、システム・タスクや操作タスクを監視したりできます。Oracle Service Busは、Oracle WebLogic Serverランタイム機能に基づきます。詳細は、Oracle Fusion Middleware Oracle Service Bus概念とアーキテクチャ(http://www.oracle.com/pls/as111170/lookup?id=OSBCA)を参照してください。

リソースの命名制約

Oracle Service Bus構成でディレクトリまたはリソースに名前を付ける場合、次の文字が使用可能です:

    • Javaキーワードを含むすべてのJava識別子文字。Java Language Specification (http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#3.8
      )の「Identifiers」と「Keywords」のセクションを参照してください。
    • 空白、ピリオドおよびハイフン(先頭と末尾以外)。

/ \ * : " < > ? | などの文字は使用できません。
