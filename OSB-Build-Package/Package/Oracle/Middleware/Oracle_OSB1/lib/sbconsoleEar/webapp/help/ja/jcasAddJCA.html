JCAバインドの追加

Oracle Service BusでJCAバインドを作成する最も簡単な方法は、作成したJCAリソース(.jcaファイル、関連付けられたWSDL、TopLinkまたはEclipseLinkマッピング・ファイル)を一括インポートすることです。一括インポート時に、.jcaファイルからJCAバインド、関連付けられたJCA WSDLからWSDLリソース、マッピング・ファイルからXMLドキュメント・リソースがそれぞれ自動的に生成され、ファイル間の依存関係が維持されます。

インポートの詳細は、Zipファイルからのリソースのロード、URLからのリソースのロードおよびリソースのインポートを参照してください。

JCAリソースをインポートすると、JCAバインド・リソースからのプロキシ・サービスの生成およびJCAバインド・リソースからのビジネス・サービスの生成の説明に従って、それらのリソースからプロキシ・サービスまたはビジネス・サービスを生成できます。

JCAリソースを一括インポートしない場合は、次に示す手順で、JCAバインドを手動で作成できます。

  1. Oracle JDeveloperでは、JCAサービスWSDLおよび対応するJCA (.jca)ファイルや、必要に応じてTopLinkまたはEclipseLinkマッピング・ファイルおよびXSLTファイルなどの他のリソースが作成されます。

    2. 詳細は、Oracle Fusion Middleware Technology Adaptersのユーザーズ・ガイド(http://www.oracle.com/pls/as111170/lookup?id=TKADP)を参照してください。

  2. まだ実行していない場合は、OSB管理コンソールで「作成」をクリックして新しいセッションを作成するか、「編集」をクリックして既存のセッションに入ってください。チェンジ・センターの使用を参照してください。
  3. OSB管理コンソールでは、Oracle JDeveloperで作成したWSDL、XMLおよびXSLTリソースなどのJCAリソースが作成またはインポートされます。
  4. JCAバインドを作成します。
    1. 「プロジェクト・エクスプローラ」を選択し、JCAバインドのプロジェクトまたはフォルダの場所を選択します。プロジェクト/フォルダ・ビュー・ページが表示されます。
    2. 「リソースの作成」フィールドで「JCAバインド」を選択すると、新しいJCAバインド・リソースの作成ページが表示されます。
    3. 「JCAファイル」フィールドで、「参照」をクリックしてJCAファイルを選択します。

      4. 「リソース名」フィールドに、ファイル名が自動的に設定されます(ファイル拡張子を除く)。

      ファイルを選択するかわりに、「JCAファイル」フィールドのテキスト領域にJCAファイルのコンテンツをコピーして貼り付けることができます。

    4. 「リソース名」フィールドで、JCAバインドの名前を入力するか変更します。JCAバインド名は一意である必要があります。

      命名については、リソースの命名制約のガイドに従ってください。
    5. 「リソースの説明」フィールドに、このJCAバインドの説明を入力します。
    6. 「保存」をクリックします。JCAバインドは現在のセッションに保存されます。

      7. JCAバインドがWSDLやマッピング・ファイルなどの必要なリソースをまだ参照していない場合は、確認の警告が表示されます。

  5. 新しいJCAバインドの名前をクリックすると、JCAバインドの表示ページにJCAバインドが表示されます。
  6. 「参照の編集」をクリックします。
  7. JCAバインド・リソースの参照を編集ページで、WSDL、TopLink/EclipseLinkマッピング・ファイルおよびXSLTファイルなどのJCAバインドの参照リソースを選択するには、「参照」をクリックします。
  8. 「保存」をクリックします。
  9. セッションを終了して構成をランタイムにデプロイするには、「チェンジ・センター」の「アクティブ化」をクリックします。

JCAバインド・リソースを追加してリソースの依存関係(WSDLの関連付け)を構成した後(または、依存関係が維持されるJCAリソースをインポートした場合)、JCAバインドからプロキシ・サービスおよびビジネス・サービスを生成できます。詳細は、JCAバインド・リソースからのプロキシ・サービスの生成およびJCAバインド・リソースからのビジネス・サービスの生成を参照してください。
