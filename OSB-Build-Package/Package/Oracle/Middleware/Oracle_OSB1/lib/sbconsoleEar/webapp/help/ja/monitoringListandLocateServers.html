サーバー情報の表示

サーバー・ヘルス・ページには次の項目が含まれています。

  • 「サーバー・ヘルス」タブ見出しには、「シャットダウン」アイコンと、シャットダウンしているサーバーの数を示す数値インジケータが表示されます。
    • アイコンをクリックすると、シャットダウン・ヘルス条件とシャットダウン状態条件で、表示されるサーバーがフィルタされます。
    • 数値は、シャットダウン中のサーバーの数を示します。
  • ドメイン内のサーバーの状態を示す円グラフ。サーバーに対応するグラフの領域をクリックして、詳細を表示できます。
  • ログ・サマリー。ログを表示するには、所定の重要度を示すメッセージの数を示す数値リンクをクリックします。たとえば、アラート: 1 の場合は、1の番号をクリックします。ドメイン・ログ・ファイルの詳細の表示を参照してください。
  • サーバー・サマリーの表に対してカスタマイズした設定に応じてサーバー情報を表示するフィールド。たとえば、サーバー名、ヘルス、クラスタ名、マシン名、サーバーの状態、サーバーの稼働時間などです。これらのフィールドの詳細は、表25-16を参照してください。
  • デフォルトでは、ダッシュボードのリフレッシュ間隔の値は、「リフレッシュなし」です。ダッシュボードのリフレッシュ間隔を変更するには、ユーザー・プリファレンスの設定を参照してください。
注意: ダッシュボードのリフレッシュ間隔の値でデフォルト(「リフレッシュなし」)以外を選択すると、ブラウザは、一定間隔で定期的にダッシュボードの表示をリフレッシュします。これにより、Oracle Service Bus管理コンソールを操作していない場合でも、セッションのタイムアウトを防ぐことができます。
注意: ダッシュボードのリフレッシュ間隔の機能を使用するときには十分注意してください。セッション中に席を離れてコンピュータを放置しないでください。セッションがアクティブになっている間に、他の人がそのコンピュータを使用して再認証なしでセッションにアクセスするおそれがあります。
  • 「表カスタマイザ」アイコン。表の表示のカスタマイズを参照してください。
  • リストに表示される項目の数を絞り、特定のサービスのみを検索するには、次の操作を行います。
    • 開くアイコンをクリックして、追加の検索フィルタを表示します。複数の検索条件を指定すると、すべての条件に一致する結果のみが戻されます。
    • ヘルス・ステータスでフィルタします。「ヘルス」ドロップダウン・リストで選択して、サーバーを指定されたヘルス・ステータス・レベルに制限し、「検索」をクリックします。

      • 「またはそれ以上」チェック・ボックスを選択すると、検索は指定した一定の値以上のサーバー・ヘルス・ステータス・レベルに限定されます。限定の度合いが低いものから順に並べると、「OK」、「警告」、「過負荷」、「クリティカル」、「致命的」となります。

    • サーバーでフィルタします。「サーバー」ドロップダウン・リストで「すべて」を選択するか、サーバー名を選択し、「検索」をクリックします。
    • クラスタ名でフィルタします。「クラスタ名」ドロップダウン・リストで「すべて」を選択するか、対象サーバーに関連付けられたクラスタの名前を選択し、「検索」をクリックします。
    • マシン名でフィルタします。「マシン名」ドロップダウン・リストで「すべて」を選択するか、対象サーバーに関連付けられたマシン名を選択し、「検索」をクリックします。
    • 状態でフィルタします。「状態」ドロップダウン・リストで「すべて」またはサーバーの状態(RUNNING、FAILEDまたはSHUTDOWN)を選択し、「検索」をクリックします。
    • 「すべて表示」をクリックして、検索フィルタを解除し、サーバーをすべて表示します。

サーバー・ヘルス・ページには、表25-16に示した情報がサーバーごとに表示されますが、それらは指定したフィルタ設定によって異なります。

表25-16 サーバー・サマリーの詳細
プロパティ
説明
サーバー
サーバーの名前。この名前はサーバーの詳細の表示ページにリンクされています。サーバーの詳細の表示を参照してください。
ヘルス
サーバーのヘルス・ステータス:
  • OK - サーバーは障害なしで稼働しています。
  • OK/処理なし - サーバーはADMIN状態です。管理操作にのみ使用可能です。すべての実行時Oracle Service Busメッセージは失敗します。
  • 警告 - 将来的にサーバーに障害が発生する可能性があります。詳細は、サーバー・ログと対応するランタイムMBeanを確認します。
  • 過負荷 - 構成されたしきい値よりも多い作業がサーバーに割り当てられています。これ以上の作業は拒否される可能性があります。
  • クリティカル - サーバーに障害が発生しようとしています。障害を防ぐためになんらかの措置をすぐに講じる必要があります。詳細は、サーバー・ログと対応するランタイムMBeanを確認します。
  • シャットダウン - サーバーに障害が発生し、サーバーを再起動する必要があります。
  • クリティカル - サーバーに障害が発生しようとしています。障害を防ぐためになんらかの措置をすぐに講じる必要があります。詳細は、サーバー・ログと対応するランタイムMBeanを確認します。
クラスタ名
このサーバーに関連付けられたクラスタの名前。
マシン名
このサーバーに関連付けられたマシンの名前。
状態
サーバーのステータス:
  • RUNNING
  • ADMIN
  • FAILED
  • SHUTDOWN
アップタイム
このサーバーの稼働時間。
