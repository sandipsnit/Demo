カスタムWS-Policy

注意: Oracle Service Busの将来のリリースでは、WLS 9.2ポリシーのかわりにOracle Web Services Managerポリシーが使用されます。このバージョンのOracle Service BusではWLS 9.2ポリシーが引き続きサポートされますが、最終的にWLS 9.2ポリシーがサポートされなくなるのに備えて、新しいサービスの作成ではOracle Web Services Managerの使用を検討する必要があります。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのOracle Web Services ManagerによるOracle Service Busの保護(http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681)を参照してください。

Web Services Policy Framework (WS-Policy)は、XMLベースの拡張フレームワークです。Webサービスの構成にドメイン固有のセキュリティ・アサーションを拡張し、Webサービスのセキュリティ要件、期待される条件および機能を指定します。Oracle Service BusにおけるWS-Policyの主な使用方法の1つとして、プロキシ・サービスおよびビジネス・サービスでの、メッセージレベル・セキュリティの構成があります。

Oracle Service Bus WS-Policyの使用

Oracle Service Busには、Auth.xml、Encrypt.xml、Sign.xmlというクライアントの認証、デジタル暗号化またはSOAPメッセージへのデジタル署名に必要な3種類のシンプルなWS-Policyファイルがあります。具体的なセキュリティのニーズがないかぎり、これらのあらかじめパッケージ化されたファイルをできるかぎり使用するようお薦めします。

これらのポリシーの使用の詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのOracle Service Busのプロキシ・サービスおよびビジネス・サービスでのWS-Policyの使用(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599)を参照してください。

カスタムWS-Policyの使用

Oracle Service Bus WS-Policy文がセキュリティのニーズを満たしていない場合、自分で書いたWS-Policy (カスタムWS-Policy)をOracle Service Busにインポートして、WSDLから参照することもできます。(Oracle Service Bus WS-Policy文は読取り専用です)。

カスタムWS-Policyの作成および参照の詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのOracle Service Busのプロキシ・サービスおよびビジネス・サービスでのWS-Policyの使用(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599)を参照してください。
