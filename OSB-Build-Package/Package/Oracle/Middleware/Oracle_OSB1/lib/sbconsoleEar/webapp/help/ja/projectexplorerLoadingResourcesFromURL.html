URLからのリソースのロード

URLまたはファイル・システムにある使用可能なリソースの一括インポートを実行できます。一括インポートでは、WSDLなどのルート・リソースが他のWSDLやスキーマなどの依存関係とともにインポートされます。依存関係マップは自動的に解決されます。

  1. セッションの作成または編集をまだ行っていない場合は、「作成」をクリックして新しいセッションを作成するか、「編集」をクリックして既存のセッションに入ります。チェンジ・センターの使用を参照してください。
  2. 「プロジェクト・エクスプローラ」を選択して、リソースを追加するプロジェクトまたはフォルダを選択します。プロジェクト/フォルダ・ビュー・ページが表示されます。
  3. 「リソースの作成」ドロップダウン・リストから、「URLからのリソース」を選択します。リソースのロード - URLからリソースをロード・ページが表示されます。
  4. 「URL/パス」フィールドに、リソースのURLまたはパスを入力します。

    5. ローカル・リソースを指定するには、fileプロトコルでパスで完全修飾されたファイル名を使用します。例: file:///c:/osbresources/ForeachAction.jar

  5. 「リソース名」フィールドに、リソースの名前を入力します。
  6. 「リソース・タイプ」ドロップダウン・リストから、リソースのタイプを選択します。
  7. 「次」をクリックすると、リソースのロード - ロードされたリソースの確認・ページが表示されます。任意のリソースをこのインポートに含めない場合は、横にあるチェック・ボックスをクリアします。
  8. 選択したリソースをインポートするには、「インポート」をクリックします。
  9. リソースのロード - インポート結果ページに、インポートされるリソースが表示されます。「別のロードを行う」をクリックして別のファイルからリソースをインポートするか、または、左側のナビゲーション・ペインから別のページを選択して、他のタスクに進みます。
  10. セッションを終了して構成をランタイムにデプロイするには、「チェンジ・センター」の「アクティブ化」をクリックします。
