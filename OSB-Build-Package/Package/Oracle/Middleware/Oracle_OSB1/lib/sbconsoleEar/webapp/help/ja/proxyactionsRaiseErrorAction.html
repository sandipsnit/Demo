エラーを発生させるアクションの追加

指定したエラー・コード(文字列)と説明を使用して例外を発生させるには、エラーを発生させるアクションを使用します。

開始前

次の手順は、メッセージ・フローの作成、編集および表示の説明に従って、メッセージ・フローの編集ページで、すでにメッセージ・フローを編集していることを前提としています。

エラーを発生させるアクションを追加するには

  1. メッセージ・フローでのアクションの追加と編集の説明に従って、アクションを追加する場所に移動します。
  2. 該当するアイコンをクリックして、「アクションの追加」→「フロー制御」→「エラーの生成」を選択します。
  3. 「エラー・コード」フィールドに、発生させるエラー・コードを入力します。
  4. 「エラー・メッセージ」フィールドに、エラー・コードの説明を入力します。

終了後

このアクションの構成が完了した後、メッセージ・フローでのアクションの追加と編集の説明に従って、その他のアクションを構成するか、または構成を保存して、続行します。

トランザクション

サービスがトランザクションに対応している場合、トリガーされたエラー発生アクションは、要求のトランザクション(非同期)または要求か応答(同期)のトランザクションのどちらかを中断します。たとえば、メッセージを内部検査して、SOAPエラーが発生していない場合でもエラー発生アクションが発生するか、およびエラー発生が原因となりトランザクションを中断する条件を決めます。
