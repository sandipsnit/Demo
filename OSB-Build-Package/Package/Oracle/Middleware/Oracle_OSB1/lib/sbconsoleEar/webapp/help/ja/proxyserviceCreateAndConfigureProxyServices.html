プロキシ・サービスの作成と構成

この項では、プロキシ・サービスを作成して構成する方法について説明します。

JCAサービスでは、JCAバインドからプロキシ・サービスを生成できます。JCAバインド・リソースからのプロキシ・サービスの生成を参照してください。

  1. セッションの作成または編集をまだ行っていない場合は、「作成」をクリックして新しいセッションを作成するか、「編集」をクリックして既存のセッションに入ります。チェンジ・センターの使用を参照してください。
  2. 「プロジェクト・エクスプローラ」を選択して、プロキシ・サービスを追加するプロジェクトまたはフォルダを選択します。
  3. プロジェクト/フォルダ・ビュー・ページで、「リソースの作成」ドロップダウン・リストから「プロキシ・サービス」を選択します。
  4. 全般的な構成ページで、サービスの名前を指定し、作成するサービスのタイプを選択します。

    命名については、リソースの命名制約のガイドに従ってください。
  5. 「次」をクリックします。この後のページは、最初のページでの選択によって異なります。サマリー・ページにたどり着くまで、以降の各ページで適切な情報を入力し、「保存」をクリックして、サービスを現在のセッションで保存します。

    6. 各ページを完了するための詳細な手順については、次を参照してください。

    • 全般的な構成ページ
    • メッセージ・タイプの構成ページ
    • トランスポート構成ページ
    • (プロトコル)トランスポート構成ページ
    • 操作選択構成ページ
    • メッセージ処理ページ
    • サマリー・ページ
    • プロキシ・サービスのポリシー・ページ
    • プロキシ・サービスのセキュリティ・ページ
  6. セッションを終了して構成をランタイムにデプロイするには、「チェンジ・センター」の「アクティブ化」をクリックします。
  7. プロキシ・サービスの作成が終了した後、次の手順でそのメッセージ・フローを構成します。メッセージ・フローによって、プロキシ・サービスの実装が決定されます。詳細は、メッセージ・フローの作成、編集および表示を参照してください。

JCAバインド・リソースからのプロキシ・サービスの生成

Oracle Service BusのインバウンドJCAバインド・リソースからJCAプロキシ・サービスを生成できます。JCAバインド・リソースの詳細は、JCAバインドを参照してください。

JCAバインドからJCAプロキシ・サービスを生成するには:

  1. Oracle Service Bus管理コンソールの「チェンジ・センター」で「作成」または「編集」をクリックします(まだ作成モードまたは編集モードに入っていない場合)。
  2. 「リソース・ブラウザ」で「JCAバインド」をクリックします。
  3. サービスの生成元になるJCAバインドを指定して、「アクション」アイコンをクリックします。
  4. 表示されたウィンドウで、生成するWSDLとサービスの名前を確認し、これらの新規リソースの場所を選択して、「生成」をクリックします。

    5. サービスおよびサービスに対応するWSDLがOracle Service Busによって生成されます。

  5. エンドポイントURIなど、生成されたサービスのその他の構成を必要に応じて変更します。
