JEJBトランスポート構成ページ

このページでは、JEJBトランスポート・プロトコルを使用してトランスポート設定を構成します。JEJBトランスポートの使用方法については、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのJEJBトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036)を参照してください。

ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーに使用するWebLogic Serverワーク・マネージャのインスタンスを選択します。デフォルトのワーク・マネージャは、他にワーク・マネージャがない場合に使用されます。
ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
EJB仕様バージョン
リモートEJBインタフェースのEJBバージョンを選択します。
値でXMLBeansを渡す
このオプションは、POJOの引数を表すインラインのXML表現(XMLObject)をトランスポートで生成する場合に選択します。このXML表現のパラメータは、XQuery式でアクセスおよび操作できます。
注意: 値で渡されるXMLObjectの型情報はインラインでは使用できません。このオプションを使用する場合、型付きのXMLObjectをプロキシ・サービス・パイプラインのJavaコールアウトの引数として渡すことはできません。
POJOを参照で渡す場合は、このオプションを選択しないでください。この場合もパフォーマンスが向上します。
トランザクション属性
トランザクションを処理する次のオプションのうちの1つを選択します。
Supports - トランスポートでは受信トランザクションが許可されます。サービス品質は、操作がトランザクション内で呼び出された場合はexactly-once、操作がトランザクション外部で呼び出された場合はbest-effortです。
Required - トランスポートでは受信トランザクションが許可されます。進行中のトランザクションが存在しない場合は、トランスポートでトランザクションが開始されます。サービス品質はexactly-onceです。
RequiresNew - トランスポートでは進行中のトランザクションが中断され、常に新しいトランザクションが開始されます。サービス品質はexactly-onceです。
Mandatory - トランスポートでは既存のトランザクション内のメソッドが呼び出されます。サービス品質はexactly-onceです。
NotSupported - トランスポートでは既存のトランザクションが中断され、呼出し時に再開されます。サービス品質はbest-effortです。
Never - トランスポートではトランザクション内のメソッドは呼び出されません。サービス品質はbest-effortです。
クライアントJar
「参照」をクリックし、表示されたリストからEJBクライアントJARリソースを選択します。クライアントJARには、リモートEJBのリモートまたはビジネス・インタフェースが含まれています。クライアントJARは汎用アーカイブ・リソースとして登録されています。
ホーム・インタフェース
EJB 2.1のみ - クライアントJARによって設定されたオプションから必要なEJBHomeインタフェースを選択します。
リモート・インタフェース
EJB 2.1のみ - このフィールドは、Homeインタフェースの構成に基づいて、自動的に情報が表示されます。
ビジネス・インタフェース
EJB 3.0のみ - クライアントJARから呼び出すビジネス・インタフェースを選択します。
対象ネームスペース
このフィールドにはJARからのピックアップされた情報が表示されます。
メソッド
必要なメソッドを選択します。+記号をクリックしてメソッドを展開すると、デフォルト・パラメータ値を編集できます。
特定のメソッドのデフォルトの操作名を変更できます。デフォルトでは、操作名はメソッド名です。EJBに同一名の(オーバーロードされた)メソッドが含まれている場合は、操作名を変更して一意の名前にする必要があります。WSDLでは一意の操作名が必要です。
