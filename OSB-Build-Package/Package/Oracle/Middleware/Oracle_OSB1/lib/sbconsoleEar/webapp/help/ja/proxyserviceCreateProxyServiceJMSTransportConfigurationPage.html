JMSトランスポート構成ページ

プロキシ・サービスの作成/編集 - JMSトランスポート構成ページは、プロキシ・サービスの作成および編集(プロキシ・サービスの作成と構成およびプロキシ・サービス構成の編集を参照)に使用する一連のページのうちの1つです。

詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのJMSトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037)を参照してください。

このページでは、JMSトランスポート・プロトコルを使用してプロキシ・サービスのトランスポート設定を構成します。表19-10に、このページの使用方法を示します。

表19-10 プロキシ・サービスの作成/編集 - JMSトランスポート構成ページ
オプション
作成または編集の方法
宛先タイプ
次のいずれかを選択します:
  • 「キュー」(ポイント・ツー・ポイントの宛先のタイプの場合)
  • 「トピック」(パブリッシュ/サブスクライブの宛先の場合)
応答が必要
このオプションは、「宛先タイプ」で「キュー」が選択されている場合にのみ使用できます。
アウトバウンド・メッセージの送信後に応答を受け取ることを指定する場合は、このオプションを選択します。
レスポンス・パターン
このオプションは、「レスポンスが必要」チェック・ボックスが選択されている場合にのみ使用できます。
次のいずれかを選択します:
  • Oracle WebLogic ServerでJAX-RPCサービスを実行している場合、JMSMessageIDを選択します。
  • 「JMSCorrelationID」: 上記以外のサービスを実行している場合。このオプションを選択した場合、次の「レスポンスURI」に入力する必要があります。
応答メッセージ・タイプ
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
次のいずれかを選択します:
  • 「バイト」(解釈されていないバイト・ストリームの場合)
  • 「テキスト」(テキスト・メッセージの場合)
このオプションは、レスポンスのメッセージ・タイプとしてJavaを選択した場合は、無効になります。
クライアントJar
このオプションは、メッセージ・サービスのリクエスト・タイプがJavaであるサービスの場合に使用できます。Javaオブジェクトを含むメッセージをデキューするために使用されるクライアントJARを選択します。クライアントJARを選択すると、クライアントJARがクラスパスに設定されます。

詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのメッセージでのJavaオブジェクトの送信および受信(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874)を参照してください。
ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーに使用するOracle WebLogic Serverワーク・マネージャのインスタンスを選択します。デフォルトのワーク・マネージャは、他にワーク・マネージャがない場合に使用されます。
たとえば、ビジネス・サービスがJMSトランスポート・プロトコルに対応している場合、ビジネス・サービスのエンドポイントは、そのディスパッチ・ポリシーに関連付けることのできるMDB (メッセージドリブンBean)のJARファイルになります。
ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
要求エンコーディング
リクエストをエンコーディングする文字セットを入力します。デフォルトはUTF-8です。
応答エンコーディング
このオプションは、「レスポンスが必要」チェック・ボックスが選択されている場合にのみ使用できます。
レスポンスをエンコーディングする文字セットを入力します。デフォルトはUTF-8です。
クライアント応答のタイムアウト
このオプションは、「レスポンスが必要」チェック・ボックスが選択されている場合にのみ使用できます。
接続を切断するまでのサーバー応答の待機時間を秒単位で入力します。このフィールドの値は、クライアントが同一ドメインの別のプロキシ・サービスである場合にのみ適用されます。
応答URI
このオプションは、「レスポンス相関パターン」でJMSCorrelationIDが選択されている場合にのみ使用できます。
応答URIを、次の書式で入力します。
jms://host:port/connection_factory/jndi_destination
複数のサーバーを対象とする場合は、次の書式を使用します。
jms://host1:port,host2:port/connection_factory/jndi_destination
レスポンスURIではホストとポートを省略することもできます。たとえば、次のように入力します。
jms:///connection_factory/jndi_destination

ホストとポートを省略した場合は、接続ファクトリ/宛先のルックアップがローカル・サーバー上で行われます。これは、たとえばリクエストURIが外部の接続ファクトリ/宛先であっても、レスポンスをローカル・サーバーに送信する場合に役立ちます。

注意: Oracle WebLogic Serverでは、"myqueues/myqueue"のようにJNDI名にフォワード・スラッシュを使用できますが、フォワード・スラッシュを使用したJNDI名はOracle Service Busで必要なURI書式の妨げになるため、その名前を使用できません。この問題を解決するには、JMS外部サーバーを定義し、URIでその外部サーバーを参照します。詳細は、Oracle Fusion Middleware Oracle WebLogic Server管理コンソール・オンライン・ヘルプ(http://www.oracle.com/pls/as111170/lookup?id=WLACH)の外部サーバーの構成を参照してください。

応答接続ファクトリ
このオプションは、「レスポンス相関パターン」でJMSMessageIDが選択されている場合にのみ使用できます。
応答接続ファクトリURIを入力します。
接続ファクトリが指定されていない場合、要求の接続ファクトリが応答に使用されます。
JMSサービス・アカウント
JMSサーバーによって管理されるJMSリソースに使用するサービス・アカウントを選択します。サービス・アカウントは、ユーザーIDとパスワードの別名リソースで、リクエストとレスポンスの両方で使用されます。JNDIとJMSの両方で同じサービス・アカウントが使用されます。
詳細は、サービス・アカウントを参照してください。
SSLを使用
このオプションはリクエストをTLS/SSL接続で行う場合のみ選択します。
TLS/SSL (Secure Sockets Layer)では、ネットワークで接続される2つのアプリケーションが互いのIDを認証し、アプリケーション間で交換されるデータを暗号化できるようにすることによって、安全な接続が可能になります。認証により、サーバーおよび場合によってはクライアントで、ネットワークで接続しているアプリケーションのIDを確認できます。また、宛先のJNDIエントリに対してアクセス制御が設定されていることにより、管理者から個々のJMS宛先(キューまたはトピック)へのアクセスが制限されている場合、JNDIツリー内でのルックアップ時にサービスで認証を行う必要があります。
メッセージ・セレクタ
メッセージ・セレクタ式を入力します。
式に一致するプロパティを持つメッセージのみが処理されます。
JMSクライアントID

トピックの場合のみ、サブスクライバに使用するクライアントIDを入力します。値が入力されていない場合は、クライアントIDが自動的に生成されます。カスタムIDを割り当てると、サービスを監視するときにこのコンポーネントの識別に役立ちます。これは恒久サブスクライバにのみ有効なことに注意してください。

JMSクライアントIDは、トピックのサブスクライバの名前およびクライアントIDの値の生成に使用する、複数のMDBディスクリプタの1つです。JMSクライアントIDを定義すると、サブスクライバ名とクライアントIDを参照することで、特定のトピックのサブスクライバをより簡単に識別および表示できます。

詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのJMSトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037)を参照してください。
恒久サブスクリプション
このオプションは、「宛先タイプ」で「トピック」が選択されている場合にのみ使用できます。
サブスクリプションが恒久である場合はこのチェック・ボックスを選択し、サブスクリプションが恒久でない場合は、このチェック・ボックスをブランクのままにします。
再試行回数
メッセージがエラー宛先に移されるまでに許可される配信の再試行回数を入力します。このフィールドは、Oracle WebLogic Server JMS宛先にのみ適用されます。
再試行間隔
ロールバックまたは復元されたメッセージが再配信されるまでの時間をミリ秒単位で入力します。このフィールドは、Oracle WebLogic Server JMS宛先にのみ適用されます。
エラー宛先
再配信制限に達したメッセージの宛先の名前を入力します。このフィールドは、Oracle WebLogic Server JMS宛先にのみ適用されます。
有効期限ポリシー
有効期限の経過したメッセージが宛先で検出されたときに使用する有効期限ポリシーを選択します。このフィールドは、Oracle WebLogic Server JMS宛先にのみ適用されます。
XAが必要
接続ファクトリがXAの場合に選択します。
この値は、リモート接続ファクトリを使用できない場合に考慮されます。接続ファクトリが使用可能で、この値がTrueの場合は、必ず、接続ファクトリをトランザクションとして定義してください。
トランザクション・タイムアウト
トランザクションが処理されるまでのJMSプロキシ・サービスの待機時間を秒単位で入力します。このオプションは、XA接続ファクトリを使用するサービスにのみ適用されます。デフォルトは600秒です。
トピック・メッセージ配信
このオプションは、「宛先タイプ」で「トピック」を選択した場合に使用できます。トピック・メッセージ配信は、インバウンドJMSメッセージを処理して高い可用性とフェイルオーバーを提供するための様々なオプションを備えています。

次のいずれかのプロパティを選択して、メッセージドリブンBeanが受信JMSメッセージをどう処理するかを決定します。
  • アプリケーションごとに1コピー (デフォルト) - JMSプロキシ・サービスがクラスタにデプロイされている場合、このオプションではインバウンドJMSメッセージはクラスタ内の使用可能なサーバーの1つのみで処理されるため、高可用性とスケーラビリティが提供されます。
  • サーバーごとに1コピー - このオプションは、トピックにパブリッシュされたインバウンドJMSメッセージが、クラスタのすべてのメンバーのプロキシ・サービスによって受信されるようにする場合に選択します。
  • 互換性 - このオプションは、インバウンドJMSメッセージがクラスタ内の特定の管理対象サーバーまたはすべての管理対象サーバーで処理されるようにする場合に選択します。このプロパティを選択する場合は、ターゲット・サーバーを選択します。
注意: 10.3.4より前のバージョンのOracle WebLogic Serverでデプロイされたトピック宛先の場合、有効なオプションは「互換性」のみです。
トピック・メッセージ配信で1コピーのオプションを選択すると、JMS接続ファクトリで構成されたサブスクリプション共有ポリシーまたはクライアントIDポリシー(あるいはその両方)がオーバーライドされます。
対象
このオプションは、「トピック・メッセージ配信」オプションで「互換性」を選択した場合にOracle Service Busクラスタのみで使用できます。受信JMSメッセージを処理するターゲット・サーバーを選択します。トピック・メッセージ配信で1コピーのオプションの1つを選択すると、このフィールドにクラスタの名前が表示されます。

ターゲットを設定しない場合、クラスタ内の各管理対象サーバーで(トピックからメッセージを読み取っている)JMSプロキシ・サービス・インスタンスにメッセージのコピーが渡されます。
JNDIタイムアウト
JNDIツリーで宛先または接続ファクトリをルックアップするときに使用するJNDI接続タイムアウト(秒単位)。
