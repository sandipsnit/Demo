MQトランスポート構成ページ

プロキシ・サービスの作成/編集 - MQトランスポート構成ページは、プロキシ・サービスの作成および編集(プロキシ・サービスの作成と構成およびプロキシ・サービス構成の編集を参照)に使用する一連のページのうちの1つです。

詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのMQトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117)を参照してください。

開始前

MQ接続リソースを構成します。MQ接続を参照してください。

MQトランスポートを構成するには

このページでは、MQトランスポート・プロトコルを使用してプロキシ・サービスのトランスポート設定を構成します。表19-11に、このページの使用方法を示します。

表19-11 プロキシ・サービスの作成/編集 - MQトランスポート構成ページ
オプション
作成または編集の方法
ポーリング間隔
ポーリング間隔をミリ秒単位で入力します。デフォルト値は1000です。
応答が必要
アウトバウンド・メッセージの送信後に応答を受け取ることを指定する場合は、このオプションを選択します。
応答相関パターン
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
応答相関パターンがMessageIDまたはCorrelationIDのいずれに基づく必要があるかを指定します。
MQ応答URI
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
応答をパブリッシュする必要がある宛先です。エンドポイントURIと同じ書式の応答URI (mq://<local-queue-name>?conn=<mq-connection-resource-ref>)を入力します。
応答メッセージ・タイプ
このオプションは、「応答が必要」チェック・ボックスが選択されている場合にのみ使用できます。
次のいずれかを選択します:
  • 「バイト」(解釈されていないバイト・ストリームの場合)
  • 「テキスト」(テキスト・メッセージの場合)
ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーを選択します。
ディスパッチ・ポリシーは、サービス・エンドポイントに使用するWLSワーク・マネージャのインスタンスを参照します。ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
バックアウトしきい値
配信失敗メッセージURIフィールドで指定されたキューにメッセージをリダイレクトする前に、パイプラインがメッセージを再試行する回数を表す値を入力します。
このフィールドに値を指定しない場合は、メッセージは再試行しないまま無効なレター・キューにリダイレクトされます。
MQ配信失敗メッセージURI
「バックアウトしきい値」フィールドで指定された再試行回数を試みた後に、要求メッセージがリダイレクトされる無効なレター・キューのURIを入力します。
このフィールドに値を指定しない場合は、「バックアウトしきい値」フィールドで指定された再試行回数を試みた後、メッセージがキューに返され、MQトランスポートでは各ポーリングが無視されます。配信失敗メッセージURIは「エンドポイントURI」と同じフォーマットを使用します。
エンドポイントURI 'GET'オプション
次の中からMQ GET messageオプションを入力してください。
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
複数のオプションを区切るには、|または+を使用できます。たとえば、次を指定できます。
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
MQ GET messageオプションは、インバウンド・キューからメッセージを読み取る合に適用されます。
RFH2ヘッダーのプロセス
メッセージ・ペイロードからのWebSphere MQ RFH2ヘッダーを解析し、RFH2データを含むRFH2Headersトランスポート・ヘッダーを自動生成するには、このオプションを選択します。
このオプションを選択しない場合、ペイロードは受信時の状態のまま渡されます。

MQトランスポートのRFH2ヘッダーの処理方法に関する詳細は、Oracle Fusion Middleware Oracle Service Bus開発者ガイドのRFH2ヘッダーについて(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136)を参照してください。

終了後

「次」をクリックして次のページでこのサービスの構成を続行するか、「最後」をクリックしてプロキシ・サービスの作成/編集 - サマリー・ページでこの構成を確認および保存します。
