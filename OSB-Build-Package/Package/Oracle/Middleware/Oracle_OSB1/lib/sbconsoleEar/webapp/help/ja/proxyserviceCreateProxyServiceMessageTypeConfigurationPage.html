メッセージ・タイプの構成ページ

プロキシ・サービスの作成/編集 - メッセージ・タイプの構成ページは、プロキシ・サービスの作成および編集(プロキシ・サービスの作成と構成およびプロキシ・サービス構成の編集を参照)に使用する一連のページのうちの1つです。

このページでは、タイプが「メッセージ・サービス」であるプロキシ・サービスのメッセージのタイプを構成します。

メッセージング・サービス用のバインド定義は、交換されるメッセージのcontent-typeの構成で構成されます。応答のcontent-typeは、要求のコンテンツ・タイプと同じである必要はありません。そのため、応答は個別に構成されます(たとえば、サービスでMFLメッセージを受信し、XMLの受信確認を戻すことができます)。

注意: タイプが「メッセージ・サービス」である、電子メール、ファイル、FTPまたはSFTPトランスポート・プロキシ・サービスがサポートするのは、一方向のメッセージングのみであり、「応答メッセージ・タイプ」は「なし」である必要があります。「なし」以外のオプションを選択すると、トランスポート構成ページで電子メール、ファイル、FTPまたはSFTPなどのプロトコルが使用できなくなります。

表19-2に、プロキシ・サービスの作成/編集 - メッセージ・タイプの構成ページの使用方法を示します。

表19-2 プロキシ・サービスの作成/編集 - メッセージ・タイプの構成ページ
オプション
作成または編集の方法
要求メッセージ・タイプ
要求メッセージのメッセージのタイプを選択します。
  • 「なし」 - 要求メッセージがない場合は、このオプションを選択します。
  • 「バイナリ」 - メッセージのコンテンツ・タイプが不明か、重要でない場合は、このオプションを選択します。
  • 「テキスト」 - メッセージをテキストに限定できる場合は、このオプションを選択します。
  • 「MFL」 - メッセージがMFL定義に準拠したバイナリ・ドキュメントの場合は、このオプションを選択します。MFLファイル名(そのパスで修飾されている)を入力するか、または「参照」をクリックして、ファイルを選択します。

    • MFLファイルは、1つのみ構成できます。

    注意: 複数のMFLファイルをサポートするには、コンテンツをバイナリまたはテキストとして定義し、メッセージ・フローのMFLアクションを使用してXMLに変換します。

  • 「XML」 - メッセージがXMLドキュメントの場合は、このオプションを選択します。XMLファイル名(そのパスで修飾されている)を入力するか、または「参照」をクリックして、ファイルを選択します。

    • (オプション)交換されるXMLドキュメントのXMLスキーマ型を(要素または型フィールドで)宣言して、一部の型情報を指定します。

  • 「Java」 - Javaオブジェクトがレスポンスで送信されている場合は、このオプションを選択します。JMSトランスポートはJavaオブジェクト・メッセージに使用されます。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのメッセージでのJavaオブジェクトの送信および受信(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874)を参照してください。
応答メッセージ・タイプ
応答メッセージのメッセージのタイプを選択します。
  • 「なし」 - 応答メッセージがない場合は、このオプションを選択します。
  • 「バイナリ」 - メッセージのコンテンツ・タイプが不明か、重要でない場合は、このオプションを選択します。
  • 「テキスト」 - メッセージをテキストに限定できる場合は、このオプションを選択します。
  • 「MFL」 - メッセージがMFL定義に準拠したバイナリ・ドキュメントの場合は、このオプションを選択します。MFLファイル名(そのパスで修飾されている)を入力するか、または「参照」をクリックして、ファイルを選択します。

    • MFLファイルは、1つのみ構成できます。

    注意: 複数のMFLファイルをサポートするには、コンテンツをバイナリまたはテキストとして定義し、メッセージ・フローのMFLアクションを使用してXMLに変換します。

  • 「XML」 - メッセージがXMLドキュメントの場合は、このオプションを選択します。XMLファイル名(そのパスで修飾されている)を入力するか、または「参照」をクリックして、ファイルを選択します。

    • (オプション)交換されるXMLドキュメントのXMLスキーマ型を(要素または型フィールドで)宣言して、一部の型情報を指定します。

  • 「Java」 - Javaオブジェクトがレスポンスで受信されている場合は、このオプションを選択します。JMSトランスポートはJavaオブジェクト・メッセージに使用されます。詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのメッセージでのJavaオブジェクトの送信および受信(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874)を参照してください。

終了後
