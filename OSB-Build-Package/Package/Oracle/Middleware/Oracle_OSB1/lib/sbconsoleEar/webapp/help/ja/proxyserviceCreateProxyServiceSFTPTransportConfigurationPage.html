SFTPトランスポート構成ページ

プロキシ・サービスの作成/編集 - SFTPトランスポート構成ページは、プロキシ・サービスの作成および編集(プロキシ・サービスの作成と構成およびプロキシ・サービス構成の編集を参照)に使用する一連のページのうちの1つです。

詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのHTTPトランスポートおよびポーラー・トランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV947)を参照してください。

このページでは、SFTPトランスポート・プロトコルを使用してプロキシ・サービスのトランスポート設定を構成します。表19-13に、このページの使用方法を示します。

表19-13 プロキシ・サービスの作成/編集 - SFTPトランスポート構成ページ
オプション
作成または編集の方法
ユーザー認証
次のいずれかを選択します:
  • 「ユーザー名/パスワード認証」 - 静的なサービス・アカウントがこの認証メソッドに関連付けられていること、およびクライアントが提供された資格証明を使用して認証されることを指定します。
  • 「ホスト・ベース認証」 - この認証メソッドを使用するには、ユーザー名およびサービス・キー・プロバイダが必要であることを指定します。既知のホストから接続しているユーザーは、ホストのプライベート・キーを使用して認証されます。
  • 「公開鍵認証」 - この認証メソッドを使用するには、ユーザー名およびサービス・キー・プロバイダが必要であることを指定します。すべてのユーザーには独自のプライベート・キーがあります。
サービス・アカウント
ユーザーのサービス・アカウントを入力するか、「参照」をクリックして、サービス・アカウントをブラウザから選択します。
サービス・キー・プロバイダ
このオプションは、「ホスト・ベース認証」または「公開鍵認証」が選択されている場合のみ有効です。
「サービス・キー・プロバイダ」フィールドに、サービス・キー・プロバイダを入力します。「参照」をクリックして、ブラウザからサービス・キー・プロバイダを選択できます。このフィールドは必須です。
ユーザー名
このオプションは、「ホスト・ベース認証」または「公開鍵認証」が選択されている場合のみ有効です。
ユーザー名を入力します。
参照渡し
このチェック・ボックスを選択すると、ファイルはアーカイブ・ディレクトリにステージングされ、ヘッダーに参照として渡されます。
リモート・ストリーミング
処理時にリモート・サーバーからSFTPファイルを直接ストリーミングする場合は、このチェック・ボックスを選択します。このオプションを選択した場合、アーカイブ・ディレクトリはリモートのSFTPサーバー・マシン上のリモート・ディレクトリになります。そのため、SFTPユーザー・ディレクトリを基準にした相対パスでアーカイブ・ディレクトリを指定する必要があります。
ファイル・マスク
選択されるファイルの正規表現を入力します。デフォルト値は*.*です。
管理対象サーバー
このフィールドが有効になるのは、クラスタ・ドメインの場合のみです。
ポーリング・サーバーとして使用する「管理対象サーバー」を選択します。メッセージはすべての管理対象サーバーで処理できますが、メッセージをポーリングできるのは1台のサーバーのみです。
ポーリング間隔
指定された場所からファイルがポーリングされる間隔を秒単位で入力します。デフォルト値は60です。
読取り制限
ポーリング・スイープ当たりの読取りメッセージ最大数を指定します。無制限にするには0と入力します。デフォルト値は10です。
読取り後のアクション
メッセージの読取り後の動作を選択します。
  • 「アーカイブ」 - メッセージがアーカイブ化されます。
  • 「削除」 - メッセージが削除されます。
アーカイブ・ディレクトリ
「読取り後のアクション」フィールドが「アーカイブ」に設定されている場合は、アーカイブの場所へのパスを入力します。「参照渡し」オプションが選択されている場合、このフィールドは必須です。
注意: アーカイブ・ディレクトリ、ダウンロード・ディレクトリおよびエラー・ディレクトリは絶対パスであり、自動的に作成されます。相対パスを指定すると、ファイルはWebLogic Serverを起動するJavaプロセスを基準にした相対的な位置に作成されます。
ダウンロード・ディレクトリ
ファイル転送中にファイルをダウンロードするローカル・マシン上のディレクトリを入力します。
注意: アーカイブ・ディレクトリ、ダウンロード・ディレクトリおよびエラー・ディレクトリは絶対パスであり、自動的に作成されます。相対パスを指定すると、ファイルはWebLogic Serverを起動するJavaプロセスを基準にした相対的な位置に作成されます。
エラー・ディレクトリ
問題が生じたときにメッセージと添付ファイルをポストする場所を入力します。
注意: アーカイブ・ディレクトリ、ダウンロード・ディレクトリおよびエラー・ディレクトリは絶対パスであり、自動的に作成されます。相対パスを指定すると、ファイルはWebLogic Serverを起動するJavaプロセスを基準にした相対的な位置に作成されます。
要求エンコーディング
SFTPトランスポートにおける要求の文字セット・エンコーディングとして、デフォルトのUTF-8を受け入れます。
優先暗号スイート
サーバーとの通信時に使用する暗号スイートを選択します。
デフォルト値「ランタイム・デフォルトの使用」を使用する場合は、サポートされている暗号スイートのリストがサーバーに送信され、一致が見つかるまで各暗号スイートが試行されます。
優先データ整合性アルゴリズム
データ整合性チェックに使用するバルク・ハッシュ・アルゴリズムを選択します。
デフォルト値ランタイム・デフォルトの使用を使用する場合、Oracle Service Busでは優先的にhmac-sha1アルゴリズムがサーバーに送信されます。サーバーがサポートしていない場合は、サポートされているアルゴリズムのリストがサーバーに送信され、一致が見つかるまで各アルゴリズムが試行されます。
優先公開鍵アルゴリズム
公開鍵の暗号化の非対称鍵アルゴリズムを選択します。
デフォルト値ランタイム・デフォルトの使用を使用する場合、Oracle Service Busではssh-dssアルゴリズムが優先的にサーバーに送信されます。サーバーがサポートしていない場合は、サポートされているアルゴリズムのリストがサーバーに送信され、一致が見つかるまで各アルゴリズムが試行されます。
優先鍵交換アルゴリズム
メッセージ暗号化のセッション鍵のネゴシエーション用に、デフォルトの鍵交換プロトコルを選択します。
デフォルト値、ランタイム・デフォルトの使用を使用する場合、Oracle Service Busでは、diffie-hellman-group1-sha1アルゴリズムが優先的にサーバーに送信されます。サーバーがサポートしていない場合は、サポートされているアルゴリズムのリストが送信され、一致が見つかるまで各アルゴリズムが試行されます。
優先圧縮アルゴリズム
zlibを使用して、処理中のデータを圧縮するかどうか指定します。zlibを選択してデータを圧縮するか、「なし」を選択します。デフォルトは「なし」です。
サブディレクトリのスキャン
すべてのディレクトリを再帰的にスキャンする場合は、このチェック・ボックスを選択します。
到着順にソート
イベントを受信順に配信する場合は、このチェック・ボックスを選択します。
タイムアウト
接続が切断されるまでのソケット・タイムアウト間隔を秒単位で入力します。0と入力した場合、タイムアウトは発生しません。デフォルト値は60です。
再試行回数
SFTP接続失敗時の再試行回数を指定します。

終了後

「次」をクリックして次のページでこのサービスの構成を続行するか、「最後」をクリックしてプロキシ・サービスの作成/編集 - サマリー・ページでこの構成を確認および保存します。
