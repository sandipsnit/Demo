WSトランスポート構成ページ

プロキシ・サービスの作成/編集 - WSトランスポート構成ページは、プロキシ・サービスの作成および編集(プロキシ・サービスの作成と構成およびプロキシ・サービス構成の編集を参照)に使用する一連のページのうちの1つです。

詳細は、Oracle Fusion Middleware Oracle Service Busのための開発者ガイドのWSトランスポート(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083)を参照してください。

このページでは、WSトランスポート・プロトコルを使用してプロキシ・サービスのトランスポート設定を構成します。表19-15に、このページの使用方法を示します。

表19-15 プロキシ・サービスの作成/編集 - WSトランスポート構成ページ
オプション
作成または編集の方法
ディスパッチ・ポリシー
このエンドポイントのディスパッチ・ポリシーを選択するか、またはデフォルトのディスパッチ・ポリシーを使用します。
ディスパッチ・ポリシーは、サービス・エンドポイントに使用するWLSワーク・マネージャのインスタンスを参照します。ワーク・マネージャの詳細は、Oracle Fusion Middleware Oracle WebLogic Serverのサーバー環境の構成のワーク・マネージャを使用したスケジューリング済作業の最適化(http://www.oracle.com/pls/as111170/lookup?id=CNFGD112)を参照してください。
再試行回数
パイプラインへのメッセージの配信の再試行回数を入力します。
プロキシ・サービスの要求パイプラインで処理されない例外が発生した場合、受信WSトランスポート・メッセージは、再試行回数で指定された回数まで、パイプラインに再配信されます。この値は、WSトランスポート・メッセージの信頼性の高い処理を実現するために重要です。
再試行の遅延
エラーの発生後に、パイプラインにメッセージを再配信するまでシステムが一時停止する秒数。
