プロキシ・サービスのポリシー・ページ

プロキシ・サービスの表示 - ポリシー・ページは、プロキシ・サービスの編集(プロキシ・サービスの作成と構成およびプロキシ・サービス構成の編集を参照)に使用する一連のページの1つです。

このページでは、サービス・タイプが「WSDL Webサービス」、「メッセージ・サービス」、「任意のSOAPサービス」または「任意のXMLサービス」のプロキシ・サービスのポリシー設定を構成します。 表19-20に、このページの使用方法を示します。OWSMポリシーを「WSDL Webサービス」、「メッセージ・サービス」または「任意のXMLサービス」プロキシ・サービスで使用するには、プロトコルをHTTPに設定する必要があります。

WSDLベースのサービスでは、サービスにバインドされているWLS 9.2ポリシーが有効なWSDLに内に置かれます(インライン化)。抽象ポリシーは、インライン化される前に前処理されます。OWSMポリシーは参照によってバインドされ、有効なWSDL内にインライン化されません。

サービス・ポリシー構成
「サービス・ポリシー構成」フィールドには次のオプションがあります。
  • OWSMポリシー・ストアから – ポリシーは、Oracle Enterprise Managerで管理される中央のOracle Web Services Managerポリシー・ストアに定義されています。これらのポリシーは、WS-Security 1.0/1.1、SAML 1.1/2.0、KerberosToken Profileおよびその他の業界標準をサポートしています。

    必要に応じてフィルタリングを使用して、目的のポリシーを探すことができます。プロキシ・サービスの場合、サービス・ポリシーのみが表示されます。

    • 詳細は、Oracle Fusion Middleware Oracle Service Busの開発者ガイドのOracle Web Services ManagerによるOracle Service Busの保護(http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681)を参照してください。

  • WSDLから – サービス・ポリシーがサービスの基になるWSDLに関連付けられている場合は、このオプションを選択します。これらのポリシーは、WS-Security 1.0、SAML 1.1およびその他の業界標準をサポートしています。

    • このオプションを使用すると、WSDLからのリクエストおよびレスポンスのポリシーを表示できます(読取り専用)。

  • 事前定義ポリシーまたはWS-Policyリソースからでは、Oracle Service Bus管理コンソールから、サービス・レベル・ポリシー、操作レベル・ポリシー(リクエスト・メッセージとレスポンス・メッセージの両方に適用するポリシー)、リクエスト・ポリシー、およびレスポンス・ポリシーを追加します。

    ポリシーは、Oracle WebLogic Serverで事前定義されているか、WS-Policyリソースを持つOracle Service Busでユーザー定義されているかのいずれかです。これらのポリシーは、WS-Security 1.0、SAML 1.1およびその他の業界標準をサポートしています。

    • このオプションを使用すると、サービス、操作、リクエストおよびレスポンスの各レベルでポリシーを追加できます。

注意: ポリシー・バインド・モデルは排他的です。サービス内には1つのポリシー・タイプのみ使用できます。サービスに直接ポリシーをバインドすると、すべてのWSDLベースのポリシーは無視されます。
