メッセージの詳細の表示

このページは、特定のメッセージの詳細を表示するために使用します。

  1. 対象のメッセージを見つけます。レポート・メッセージの表示を参照してください。
  2. レポート・インデックスをクリックします。メッセージの詳細を表示ページに、表27-3に示す「全般的な構成」の情報が表示されます。

    3. 表27-3 メッセージの構成パラメータ
    プロパティ
    説明
    メッセージID
    このメッセージの一意のID。
    データベース・タイムスタンプ
    このメッセージがデータベースに登録された日時。
    ロギングの時刻
    このメッセージのログが記録された日時。
    サーバー名
    このメッセージが生成されたサーバー名。
    状態
    このメッセージが生成されたパイプラインの状態。
    REQUEST - レポート・アクションが要求パイプラインで実行されることを示します。
    RESPONSE - レポート・アクションが応答パイプラインで実行されることを示します。
    ERROR - アクションがサービスレベルのエラー・ハンドラで実行中です。
    ノード名
    このメッセージが生成されたパイプラインのノード名。
    パイプライン名
    このメッセージが生成されたパイプラインの名前。
    ステージ名
    このメッセージが生成されたステージの名前。

    インバウンド・サービス・ページに表示される情報を表27-4に示します。

    表27-4 インバウンド・サービスの詳細
    プロパティ
    説明
    名前
    このメッセージに関連付けられたインバウンド・プロキシ・サービス。これはプロキシ・サービスの表示 - 構成の詳細ページへのリンクです。プロキシ・サービス構成の編集を参照してください。
    URI
    プロキシ・サービスに関連付けられたURI。
    操作
    このメッセージに関連付けられたインバウンド処理。

    アウトバウンド・サービス・ページに表示される情報を表27-5に示します。

    表27-5 アウトバウンド・サービスの詳細
    プロパティ
    説明
    名前
    このメッセージに関連付けられたアウトバウンド・ビジネス・サービス。これはビジネス・サービスの表示 - 構成の詳細ページへのリンクです。ビジネス・サービス構成の編集を参照してください。
    URI
    アウトバウンド・ビジネス・サービスのエンド・ポイントのURI。
    操作
    アウトバウンド・サービスで呼び出される処理の名前。

    レポート・インデックスページに表示される情報を表27-6に示します。

    表27-6 レポート・インデックスの詳細
    プロパティ
    説明
    レポート・インデックス・テキスト
    レポート・インデックスは、メッセージ・コンテキスト変数からキー識別子を抽出する場合に使用するキー値ペアです。キーはメッセージをフィルタする場合に使用すると便利です。キー値ペアを追加するには、レポート・アクションを使用します。メッセージ・フローでのアクションの追加と編集を参照してください。

    フォルト・ページに表示される情報を表27-7に示します。

    表27-7 フォルト情報の詳細
    プロパティ
    説明
    エラー・コード
    このメッセージに関連付けられたエラー・コード(ある場合)。
    エラー・コードは、エラーを発生させるアクションを構成するときに指定します。Oracle Fusion Middleware Oracle Service Busのための管理者ガイドのメッセージ・フローでのエラー処理(http://www.oracle.com/pls/as111170/lookup?id=OSBAG232)およびプロキシ・サービス・エラー・ハンドラの追加を参照してください。
    理由
    エラー・コードの理由。
    詳細
    エラー・コードに関連付けられたエラーの詳細。このような詳細情報(存在する場合)は通常、特定のエラーが発生した場所のスタック・トレースであり、データベースのサイズ制限に合わせて切り捨てられる場合があります。上限は2,048文字です。

    レポート本文ページに表示される情報を表27-8に示します。

    表27-8 レポート本文の詳細
    プロパティ
    説明
    レポート本文テキスト
    このリンクをクリックすると、別のブラウザが開き、レポート本文の詳細が表示されます。レポートの本文テキストは、レポート・アクションで式を使用して取り込みます。レポート・アクションの手順は、メッセージ・フローでのアクションの追加と編集を参照してください。

  3. 特定のメッセージの詳細を見終えた後、「OK」をクリックします。
