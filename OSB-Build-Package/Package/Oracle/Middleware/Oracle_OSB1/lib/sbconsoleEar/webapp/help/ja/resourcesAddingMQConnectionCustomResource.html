MQ接続の追加

開始前

Oracle Service BusでサポートされているバージョンのWebSphere MQについては、Oracle Fusion Middlewareでサポートされているシステム構成(http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html)を参照してください。WebSphere MQクライアント・ライブラリはOracle Service Busインストーラにバンドルされていません。WebSphere MQのインストールからWebSphere MQクライアント・ライブラリを取得してください。

MQ接続リソースとMQトランスポートをOracle Service Bus内で構成して使用するには、MQクライアント・ライブラリをクラスパスに追加する必要があります。1つの選択肢として、com.ibm.mq.jarライブラリをDOMAIN_NAME/libディレクトリにコピーします(DOMAIN_NAMEはドメインを配置したディレクトリの名前で、通常はMW_HOME\user_projects\domains\DOMAIN_NAMEです)。

MQ接続リソースを追加するには

  1. セッションの作成または編集をまだ行っていない場合は、「作成」をクリックして新しいセッションを作成するか、「編集」をクリックして既存のセッションに入ります。チェンジ・センターの使用を参照してください。
  2. 「プロジェクト・エクスプローラ」を選択して、MQ接続リソースを追加するプロジェクトまたはフォルダを選択します。プロジェクト/フォルダ・ビュー・ページが表示されます。
  3. 「リソースの作成」ドロップダウン・リストで、「ユーティリティ」の下にある「MQ接続」を選択して新しいMQ接続リソースの作成ページを表示します。

    4. 表8-2に示される接続パラメータを指定します。

    表8-2 MQ接続パラメータ
    プロパティ
    作成または編集の方法
    リソース名
    このMQ接続リソースの一意の名前を入力します。

    命名については、リソースの命名制約のガイドに従ってください。
    リソースの説明
    MQ接続リソースの説明を入力します。
    接続タイプ
    MQキュー・マネージャに接続するために、次のいずれかのモードを選択します。
    • TCPモード - Oracle Service Busと同じマシンに存在しないキュー・マネージャに接続するには、TCP/IPを使用します。
    • バインド・モード - Oracle Service Busと同じマシンに存在するキュー・マネージャに接続するには、バインド・モードを使用します。
    MQホスト名
    TCP モード接続の場合のみ:
    MQキュー・マネージャのホスト名を入力します。
    MQポート番号
    TCP モード接続の場合のみ:
    MQキュー・マネージャ・リスナーのポート番号を入力します。
    MQキュー・マネージャ名
    接続するMQキュー・マネージャ名を入力します。
    キュー・マネージャCCSID
    TCP モード接続の場合のみ:
    接続を確立する場合に使用するコード化された文字セット識別子(CCSID)。CCSIDは主にインターナショナライゼーション・サポートに使用されます。
    詳細は、WebSphere MQの基礎概念(http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm)を参照してください。
    MQキュー・マネージャのチャネル名
    TCP モード接続の場合のみ:
    キュー・マネージャのサーバー接続のチャネル名を入力します。
    SSLが必要
    TCP モード接続の場合:
    SSLを使用してメッセージ送信を行う場合は、このチェック・ボックスを選択します。「双方向SSLが必要」オプションが選択されていない場合、サーバーサイドSSLのみが有効になります。
    暗号セット
    このオプションは、「SSLが必要」チェック・ボックスが選択されている場合のみ有効です。
    SSLで使用する暗号セット・アルゴリズムを選択します。
    暗号セット・アルゴリズムは、WebSphere MQサーバーとWebSphere MQクライアントのメッセージ通信の暗号化と復号化に使用されます。そのため、SSLを使用してWebSphere MQサーバーと通信する場合は、暗号セット・アルゴリズムを指定する必要があります。
    双方向SSLが必要
    このオプションは、「SSLが必要」チェック・ボックスが選択されている場合のみ有効です。
    クライアントサイドおよびサーバーサイドSSL認証を有効にする場合は、このチェック・ボックスを選択します。
    サービス・キー・プロバイダへの参照
    「双方向SSLが必要」を選択した場合は、適切なキー・ストアと信頼ストアの情報を取得するために、サービス・キー・プロバイダへの参照を提供する必要があります。
    サービス・キー・プロバイダのパス(project/folder)と名前を入力するか、「参照」をクリックして、サービス・キー・プロバイダの選択ページから選択します。
    静的なサービス・アカウントへの参照
    TCP モード接続の場合のみ:
    ユーザー名/パスワード認証に必要。パス(project/folder)および静的サービス・アカウント名を入力するか、または「参照」をクリックしてブラウザからサービス・アカウントを選択します。
    WebSphere MQバージョン
    WebSphere MQバージョンを選択します。
    • 5.3
    • 6.0
    • 7.0

    注意: バージョン5.3および6.0は、IBMではサポートされなくなりました。これらのバージョンは、Oracle Service Busでは非推奨です。
    MQ接続プール・サイズ
    MQ接続プール・サイズを入力します。
    MQ接続タイムアウト
    未使用の接続が破棄される時間間隔を秒単位で入力します。デフォルトは1800秒です。
    MQ接続の最大待機時間
    接続が使用できるようになるまでの最大待機時間をミリ秒単位で入力します。この時間内に接続が確立されない場合、Oracle Service Busは例外をスローします。デフォルトは3000ミリ秒です。

  4. 「保存」をクリックします。MQ接続リソースは現在のセッションで作成および保存されます。
  5. セッションを終了して構成をランタイムにデプロイするには、「チェンジ・センター」の「アクティブ化」をクリックします。
