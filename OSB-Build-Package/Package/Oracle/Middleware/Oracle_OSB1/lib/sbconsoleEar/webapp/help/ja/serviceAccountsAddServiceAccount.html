サービス・アカウントの追加

  1. セッションの作成または編集をまだ行っていない場合は、「作成」をクリックして新しいセッションを作成するか、「編集」をクリックして既存のセッションに入ります。チェンジ・センターの使用を参照してください。
  2. 「プロジェクト・エクスプローラ」を選択して、サービス・アカウントを追加するプロジェクトまたはフォルダを選択します。プロジェクト/フォルダ・ビュー・ページが表示されます。
  3. 「リソースの作成」ドロップダウン・リストから「サービス・アカウント」を選択して、新しいサービス・アカウントの作成ページを表示します。
  4. 「リソース名」フィールドに、このサービス・アカウントの一意の名前を入力します。

    命名については、リソースの命名制約のガイドに従ってください。
  5. 「リソースの説明」フィールドにサービス・アカウントの説明を入力します。
  6. 「リソース・タイプ」で、次のいずれかの操作を実行します。
    • 受信クライアント要求で受け取ったユーザー名およびパスワードを提供するサービス・アカウントを作成するには、パス・スルーを選択します。
    • サービス・アカウントの構成で保存するユーザー名およびパスワードを提供するサービス・アカウントを作成するには、「静的」を選択します。
    • 1つ以上の認証されたクライアントから指定するユーザー名およびパスワードにユーザー名をマップするサービス・アカウントを作成するには、「マッピング」を選択します。
  7. 手順6で選択したリソースのタイプに応じて、表16-2に示すいずれかの手順を実行します。

    8. 表16-2 リソースのタイプのオプション 
    選択したリソースのタイプ
    手順
    パス・スルー
    「最後」をクリックします。
    静的な方法
    1. 「次」をクリックします。
    2. 「ユーザー名」、「パスワード」および「パスワードの確認」フィールドにユーザー名およびパスワードを入力します。
    3. 「最後」をクリックします。
    マッピング
    1つ以上のクライアントから指定するユーザー名およびパスワードおよびユーザー名をマップするサービス・アカウントを作成するには、次のいずれかを実行します。
    1. 「次」をクリックします。
    2. 認可リモート・ユーザーの入力表で、次の操作を実行します。
      1. 「リモート・ユーザー名」、「パスワード」および「パスワードの確認」フィールドに、アウトバウンド要求で送信するユーザー名とパスワードを入力します。
      2. 「追加」をクリックします。

        3. ユーザー・マッピングがリモート・ユーザー表に追加されます。

      3. (オプション)認可リモート・ユーザーの入力表で、他のリモート・ユーザーを追加します。
    3. 「次」をクリックします。
    4. 認証されたクライアントをリモートのユーザー名およびパスワードにマップするには、認可ローカル・ユーザーの入力表で次の操作を実行します。
      1. 「ローカル・ユーザー名」フィールドに、インバウンド要求で認証されたクライアントを識別する名前を入力します。

        2. このユーザーをOracle Service Bus管理コンソールの「セキュリティ構成」モジュールに追加していない場合は、このマッピングを実行時環境で使用する前にユーザーを追加します。ユーザーの追加を参照してください。Oracle Service Busでは、存在しないローカル・ユーザーのマッピングを作成できますが、このマッピングは認証されたユーザーに一致しないため使用されません。

      2. 「リモート・ユーザー名」リストから、「ローカル・ユーザー名」フィールドで指定した認証ユーザーに関するアウトバウンド要求で送信するユーザー名を選択します。
      3. 「追加」をクリックします。
    5. 匿名のクライアントをリモートのユーザー名にマップするには、次の操作を実行します。
      1. 「匿名要求のマッピング」チェック・ボックスを選択します。
      2. 「リモート・ユーザーの選択」リストから、すべての匿名ユーザーに関するアウトバウンド要求で送信するユーザー名を選択します。
    6. 「最後」をクリックします。

  8. 「保存」をクリックします。サービス・アカウントは現在のセッションで作成および保存されます。
  9. セッションを終了して構成をランタイムにデプロイするには、「チェンジ・センター」の「アクティブ化」をクリックします。
