ビジネス・サービスのテスト

任意のXML、任意のSOAP、メッセージング、XMLおよびSOAPのタイプのビジネス・サービスをテストできます。Webサービス・セキュリティ・ポリシーを含むSOAPビジネス・サービスをテストできます。詳細は、Oracle Fusion Middleware Oracle Service Busの管理者ガイドのWebサービス・セキュリティを含むサービスのテスト(http://www.oracle.com/pls/as111170/lookup?id=OSBAG384)を参照してください。

注意: クラスタ化されたドメインでは、テスト・コンソールを利用して、ビジネス・サービスにルーティングされる構成済みのビジネス・サービスまたはプロキシ・サービスをテストすることはできません。
  1. 「チェンジ・センター」の「アクティブ化」をクリックして、Oracle Service Bus管理コンソールのテスト機能を有効にします。
  2. 「ビジネス・サービスの要約」ページを表示するには、「リソース・ブラウザ > ビジネス・サービス」を選択します。
  3. 「アクション」の下で、テストするサービスに対応する「テスト・コンソールの起動」アイコンをクリックします。テスト・コンソールによって、ビジネス・サービス・テスト・ページが開かれます。たとえば、製品に含まれるチュートリアルを使用する場合は、loanSaleProcessorに対応するアイコンをクリックします。
  4. SOAPおよびXMLサービスの場合は、テストするWSDL操作を選択します。
  5. ビジネス・サービスのテスト・データを構成します(入力データは、プロキシ・サービスからビジネス・サービスに送信されるメッセージであることが必要です)。テストするプロキシ・サービスがビジネス・サービスで使用できない場合、「直接呼出し」オプションおよび「トレースを含む」オプションが、使用できます。デフォルトでビジネス・サービスは、「直接呼出し」オプションでテストされます。つまり、メッセージはトランスポート・レイヤーを通ります。

    6. メッセージ・トレースを表示するには、「直接呼出し」の選択を解除します。

    注意: ログ・ファイルまたは標準出力(サーバー・コンソール)にトレースを表示するには、Oracle WebLogic Serverロギングを次の重大度レベルに設定する必要があります:

ログに記録する最小重大度: 情報
ログ・ファイル: 情報
標準出力: 情報

ログ重大度レベルの設定の詳細は、Oracle Fusion Middleware Oracle WebLogic Serverログ・ファイルの構成とログ・メッセージのフィルタ処理のログ重大度レベルの使用に関する項(http://www.oracle.com/pls/as111170/lookup?id=WLLOG141)を参照してください。

  1. 「実行」をクリックします。

    2. 「ビジネス・サービス・テスト」ページに結果が表示されます。詳細については、「ビジネス・サービスのテスト結果の表示」を参照してください。

ビジネス・サービスの添付ファイルのテスト

このセクションでは、ビジネス・サービスの添付ファイルをテストするためのテスト・コンソールの使用方法について説明します。

このセクションでは説明のため、ZIPファイルをSOAPメッセージの添付ファイルとして送信するためにWSDLベースのHTTPビジネス・サービスが"submitAttachment"操作を使用するものとします。バイナリ添付ファイルの各タイプに応じて設定を変更してください。

    1. ビジネス・サービスのテスト・コンソールでXMLページに移動し、テスト・メッセージ用に次のSOAPエンベロープを入力します。
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. テスト・コンソールの「添付ファイル」セクションで、次を選択します。
      1. Content-Type: application/zip (WSDLの定義に従ってください)
      2. Content-ID: zipFile (テスト・メッセージ内のhref="cid:zipFile"の値に一致する必要があります)
      3. File: 「参照」をクリックし、SOAPエンベロープで指定したZIPファイルを選択します。
      4. 「追加」をクリックします。

    2. 「実行」をクリックし、添付ファイル付きのテスト・メッセージを送信します。

添付ファイルの送信が正常に実行されたことを確認するには、サーバー・コンソールで次のようなメッセージを確認します。これは、submitAttachment操作によって記録されます。

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
