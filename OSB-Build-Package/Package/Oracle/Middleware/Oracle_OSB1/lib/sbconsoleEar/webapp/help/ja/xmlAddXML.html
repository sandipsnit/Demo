XMLドキュメントの追加

Oracle Service BusでXMLドキュメントを作成する最も簡単な方法は、それらを一括インポートすることです。たとえば、作成したJCAリソース(.jcaファイル、関連付けられたWSDL、TopLinkまたはEclipseLinkマッピング・ファイル)を一括インポートすると、XMLドキュメント・リソースがマッピング・ファイルから自動的に生成され、リソース・ファイル間の依存関係が維持されます。

インポートの詳細は、Zipファイルからのリソースのロード、URLからのリソースのロードおよびリソースのインポートを参照してください。

一括インポートしない場合は、次に示す手順で、XMLドキュメントを手動で作成できます。

  1. セッションの作成または編集をまだ行っていない場合は、「作成」をクリックして新しいセッションを作成するか、「編集」をクリックして既存のセッションに入ってください。チェンジ・センターの使用を参照してください。
  2. 「プロジェクト・エクスプローラ」を選択して、新しいXMLドキュメントを追加するプロジェクトまたはフォルダを選択します。プロジェクト/フォルダ・ビュー・ページが表示されます。
  3. 「リソースの作成」フィールドで「XMLドキュメント」を選択して、新しいXMLドキュメントの作成ページを表示します。
  4. 「XMLファイル」フィールドで「参照」をクリックして、使用するXMLファイルを選択します。

    5. 「リソース名」フィールドに、ファイル名が自動的に設定されます(ファイル拡張子を除く)。

    ファイルを選択するかわりに、「XMLファイル」フィールドのテキスト領域にXMLファイルのコンテンツをコピーして貼り付けることができます。

  5. 「リソース名」フィールドで、XMLドキュメントの名前を入力するかまたは変更します。XMLドキュメント名は一意である必要があります。

    命名については、リソースの命名制約のガイドに従ってください。
  6. 「リソースの説明」フィールドにXMLドキュメントの説明を入力します。
  7. 「保存」をクリックします。新しいXMLドキュメントは現在のセッションに保存されます。
  8. セッションを終了して構成をランタイムにデプロイするには、「チェンジ・センター」の「アクティブ化」をクリックします。
