BPEL-10g 전송 구성 페이지

이 페이지에서는 BPEL-10g(Oracle BPEL 프로세스 관리자) 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. Oracle BPEL 프로세스 관리자와 함께 Oracle Service Bus를 사용하는 방법은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1147에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Oracle BPEL Process Manager Transport"를 참조하십시오.

속성
설명
역할
BPEL 전송은 Oracle Service Bus에서 Oracle BPEL 프로세스 관리자로 요청 메시지를 보내는 데 사용됩니다. 비즈니스 서비스는 다음 역할 중 하나를 수행할 수 있습니다.
  • 동기 클라이언트
    Oracle Service Bus 클라이언트와의 동기 통신을 위해 필요한 유일한 위치 정보는 BPEL 주소입니다. 이 주소는 끝점 URI로 정적으로 캡처되거나 URI 다시 쓰기를 통해 동적으로 캡처됩니다.
  • 비동기 클라이언트
    Oracle Service Bus 클라이언트와의 비동기 통신에서는 요청 이외의 다른 접속 방식을 통해 Oracle BPEL 프로세스 관리자에서 Oracle Service Bus로 콜백이 전송되므로 올바른 콜백 주소를 제공하도록 Oracle Service Bus를 구성해야 합니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Advanced Use Cases (Asynchronous)"를 참조하십시오.
  • 서비스 콜백
    비즈니스 서비스가 Oracle BPEL 프로세스 관리자의 서비스 콜백으로 디자인되어 Oracle BPEL 프로세스 관리자에서 Oracle Service Bus를 통해 외부 서비스를 호출하는 경우 콜백 주소는 런타임 시에만 알려집니다. 이 경우 bpel://callback의 끝점 URI를 사용하십시오.

    • 표시자 URI를 사용하여 비즈니스 서비스를 구성하는 경우에는 TransportHeader 작업을 사용하는 경우처럼 $outbound;에서 URI를 동적으로 설정하도록 파이프 라인 논리를 구성하십시오.

    주: 서비스 콜백 비즈니스 서비스에서는 로드 밸런싱이나 장애 조치를 지원하지 않습니다.
콜백 프록시
이 선택 사항 필드는 비동기 클라이언트 롤에 대해서만 제공됩니다. 이 필드에서는 요청을 제출한 Oracle Service Bus 클라이언트로 콜백을 경로 지정하는 데 사용될 프록시 서비스를 선택할 수 있습니다. 이때 [모든 SOAP] 유형의 SB 또는 HTTP 프록시를 선택해야 합니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Advanced Use Cases (Asynchronous)"를 참조하십시오.
서비스 계정
JNDI 컨텍스트 보안을 위해 Oracle BPEL 프로세스 관리자 전달 서비스에 액세스하는 데 사용됩니다. 찾아보기를 누르고 서비스 계정을 선택합니다. 서비스 계정을 지정하지 않을 경우 익명 제목이 사용됩니다.
정적 또는 통과와 같이 구성할 수 있는 서비스 계정 유형에는 제한이 없지만 런타임 시 사용자 이름 및 비밀번호에 액세스할 수 있어야 합니다.
트랜잭션 일시 중지
트랜잭션 일시 중지를 선택하면 트랜잭션에서 비즈니스 서비스를 호출한 경우에도 비즈니스 서비스는 비트랜잭션이 됩니다.
트랜잭션 일시 중지를 선택하지 않는 경우:
  • 프로토콜이 WebLogic Serve 지원 프로토콜(t3, iiop, http)이면 트랜잭션이 전달됩니다.
  • 프로토콜이 OC4J 서버(ormi, opmn)이면 OC4J에서 트랜잭션 전달을 지원하지 않으므로 BPEL 전송에서 예외 사항이 발생합니다.
다음과 같은 경우 BPEL 전송에서 트랜잭션 일시 중지 옵션을 무시합니다.
  • QoS(서비스 품질) "best-effort"에서 비즈니스 서비스가 호출됩니다. BPEL 전송에서 QoS를 지원하지 않는 트랜잭션을 자동으로 일시 중지합니다.
  • QoS가 "exactly-once"로 설정된 상태에서 비즈니스 서비스가 호출되어 트랜잭션이 없습니다.
트랜잭션 전달에 대한 설명은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1151에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Transaction Propagation"을 참조하십시오.
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않으면 기본 작업 관리자가 사용됩니다.
작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
