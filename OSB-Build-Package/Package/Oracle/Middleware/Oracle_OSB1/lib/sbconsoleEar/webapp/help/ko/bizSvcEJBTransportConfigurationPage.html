EJB 전송 구성 페이지

비즈니스 서비스 생성/편집 - EJB 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 EJB 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 18-5에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-5 비즈니스 서비스 생성/편집 - EJB 전송 구성 페이지
옵션
생성 또는 편집
호출자의 제목 전달
EJB를 호출할 때 서비스 계정이 구성되지 않은 경우 Oracle Service Bus가 프록시 서비스에서 인증된 제목을 전달하도록 하려면 이 체크 박스를 선택합니다. 이 옵션을 선택하면 서비스 계정 필드가 사용 안함으로 설정됩니다.
서비스 계정
찾아보기를 누르고 표시된 목록에서 서비스 계정을 선택합니다. 서비스 계정이 지정되지 않은 경우 익명 주체가 사용됩니다. [호출자의 주체 전달] 옵션을 사용할 경우에는 이 옵션을 사용할 수 없습니다.

자세한 내용은 다음을 참조하십시오.

  • 서비스 계정
  • 서비스 계정 추가
트랜잭션 지원
트랜잭션 지원을 지정하려면 이 체크 박스를 선택합니다.
클라이언트 Jar
찾아보기를 누르고 표시된 목록에서 EJB 클라이언트 JAR 리소스를 선택합니다. JAR 리소스 생성에 대한 자세한 내용은 JAR을 참조하십시오. 이 필드는 필수 필드입니다.
변환기 Jar
찾아보기를 누르고 표시된 목록에서 EJB 변환기 클래스 JAR 리소스를 선택합니다. EJB 클라이언트 JAR 리소스 및 변환기 클래스에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "EJB Transport"를 참조하십시오.
홈 인터페이스
EJB 2.1만 - JAR로 채워진 옵션 중에서 필요한 EJBHome 인터페이스를 선택합니다. 이 URI 샘플의 JNDI 이름은 여기서 선택한 EJBHome 인터페이스와 반드시 연관되어야 합니다. EJB가 필수 유형이 아니거나 클라이언트 JAR에 EJBHome 인터페이스가 지정되지 않은 경우 Oracle Service Bus에서 경고가 표시됩니다.
원격 인터페이스
EJB 2.1만 - 이 필드는 홈 인터페이스의 구성에 따라 자동으로 채워집니다.
비즈니스 인터페이스
EJB 3.0만 - 사용하려는 클라이언트 JAR에서 비즈니스 인터페이스를 선택합니다.
대상 네임스페이스
이 필드는 JAR에서 선택한 정보로 채워집니다.
스타일
요구 사항에 따라 문서 포함 또는 RPC를 선택합니다. 상태 비유지 세션 EJB의 메소드 두 개 이상에서 매개변수의 수와 데이터 유형이 같고, 문서 지향 작업을 하려는 경우 Document-Wrapped를 지정해야 합니다.
EJB로 라우팅하거나 게시하는 경우 $body의 내용과 스타일이 일치해야 하므로 스타일이 중요합니다. 또한 EJB에 호출할 경우 특히, 문서 포함 서비스와 같은 스타일은 매개변수 내용에 영향을 줍니다. 다음으로, 하나의 사용 패턴이 하나의 EJB 비즈니스 서비스를 정의한 다음, 해당 EJB에 경로를 지정하는 동일한 WSDL을 사용하여 프록시 서비스를 생성하는 것입니다. 이 경우 프록시를 호출하는 데 사용하는 클라이언트 툴에 WSDL의 스타일에 대한 제한 사항이 있을 수 있으므로 WSDL의 스타일에 주의해야 합니다.
인코딩
인코딩 또는 리터럴을 선택합니다.
메소드
표시된 메소드는 사용자가 선택한 EJB 원격 또는 비즈니스 인터페이스에 대한 메소드입니다. 필요한 메소드를 선택하십시오(여러 메소드 선택 가능). 메소드를 확장하려면 +를 누릅니다. 기본 매개변수 값을 편집하고 제공된 변환기가 있거나 필수 변환기가 있으면 선택합니다.
매개변수 또는 반환 유형이 JAX-RPC 스택에서 지원되지 않는 메소드는 제외하거나 그러한 인수를 변환기 클래스와 연관시켜야 합니다.
제공된 메소드에 대한 기본 작업 이름을 변경할 수 있습니다(기본적으로 작업 이름은 메소드 이름임). EJB에 같은 이름의 메소드가 포함되어 있는 경우 작업 이름을 고유하게 변경해야 합니다. WSDL에서는 고유한 작업 이름을 사용해야 합니다.
주: 제공된 메소드 간에 인증서 또는 트랜잭션 설정이 서로 다른 경우 제공된 비즈니스 서비스에 대해 메소드를 사용자 정의하고 메소드별로 비즈니스 서비스를 생성할 수 있습니다. 이렇게 하면 트랜잭션 및 인증서를 세부적으로 제어할 수 있습니다.
예외 사항
이 필드는 메소드에서 비즈니스 예외 사항이 발생하는 경우에 나타납니다. EJB 메소드에서 ArrayList와 같은 데이터 유형이 JWS(Java Web Services)에서 지원되지 않는 예외 사항이 발생하면 예외 사항 필드를 사용하여 JWS에서 지원되는 유형으로 예외 사항을 변환하는 변환기 클래스를 선택합니다.
변환기 클래스는 com.bea.wli.sb.transports.ejb.ITypeConverter를 구현해야 합니다. 변환기 클래스는 런타임 예외 사항이 아닌 확인된 예외 사항에 대해서만 구성할 수 있습니다.
클라이언트 또는 변환기 JAR에서 변환기 클래스 및 변환된 예외 사항 클래스를 패키징합니다.
자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "EJB Transport"를 참조하십시오.
