전자 메일 전송 구성 페이지

비즈니스 서비스 생성/편집 - 전자 메일 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 전자 메일 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 18-6에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-6 비즈니스 서비스 생성/편집 - 전자 메일 전송 구성 페이지
옵션
생성 또는 편집
SSL 필요
SSL(Secure Sockets Layer)을 사용하여 통신하려면 이 옵션을 선택합니다. SMTP의 기본 SSL 포트 번호는 465입니다.
주:
  • 전자 메일 서버 인증서는 SSL이 올바르게 작동하려면 Oracle Service Bus 보안 저장소에 있어야 합니다.
  • TLS를 사용하는 경우 보안 인증서를 기본 보안 저장소($JAVA_HOME/jre/lib/security/cacerts)에 놓거나 아무 보안 저장소에나 놓고 아래와 같이 $DOMAIN_HOME/bin/setDomainEnv.* 파일에 경로를 추가해야 합니다.

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<path to trust store>

SMTP 서버
name@domain_name.com의 끝점 URI 항목에 사용할 기본 SMTP 서버를 선택합니다. 표 18-3에 설명된 대로 끝점 URI에 SMTP 서버 매개변수를 제공하면 이 SMTP 서버 설정 대신 해당 서버 리소스가 사용됩니다.
[메일 세션] 옵션을 사용할 경우 SMTP 서버를 선택하지 마십시오.
SMTP 리소스 생성에 대한 자세한 내용은 SMTP 서버 추가를 참조하십시오.
메일 세션
name@domain_name.com의 끝점 URI 항목에 사용할 구성된 메일 세션의 JNDI 이름을 입력합니다. 표 18-3에 설명된 대로 끝점 URI에 JNDI 메일 세션 매개변수를 제공하면 이 메일 세션 설정 대신 해당 메일 세션이 사용됩니다.
[SMTP 서버] 옵션을 사용할 경우 메일 세션을 입력하지 마십시오.
보낸 사람 이름
이 서비스에 대한 원래 전자 메일 계정의 표시 이름을 입력합니다.
보낸 사람 주소
이 서비스의 원래 전자 메일 계정을 입력합니다.
회신 이름
전자 메일 계정에 대한 회신 표시 이름을 입력합니다.
회신 주소
전자 메일 회신 주소를 입력합니다.
접속 시간 초과
연결이 끊기기 전의 제한 시간 간격(초)을 입력합니다. 0을 입력하면 시간 초과가 없습니다.
요청 인코딩
기본값 ISO-8859-1을 전자 메일 전송 요청을 위한 문자 집합 인코딩으로 적용하거나 다른 문자 집합 인코딩을 입력합니다.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 비즈니스 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
