일반 구성 페이지

비즈니스 서비스 생성/편집 - 일반 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 비즈니스 서비스에 대한 일반 구성 설정을 수정할 수 있습니다.

비즈니스 서비스를 생성하면 이 페이지가 서비스 구성을 위한 일련의 페이지 중 첫번째로 표시됩니다. 이 페이지 이후 표시되는 페이지는 이 페이지에서 선택하는 사항에 따라 달라집니다.

표 18-1에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-1 비즈니스 서비스 생성/편집 - 일반 구성 페이지
옵션
생성 또는 편집
서비스 이름
비즈니스 서비스에 대한 고유 이름을 입력합니다.

리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
설명
설명을 입력합니다.
서비스 유형
서비스 유형은 서비스에서 교환하는 메시지의 유형과 패키징을 정의합니다. 생성하려는 비즈니스 서비스 유형을 선택합니다.
  • WSDL 웹 서비스 - WSDL 기반 비즈니스 서비스를 생성하려면 이 옵션을 선택합니다. 그런 다음 경로별로 정규화된 WSDL 이름을 입력합니다(예: myProject/myFolder/myWSDL). 또는 찾아보기를 눌러 WSDL 선택 페이지에서 WSDL을 선택합니다.

    • (포트 또는 바인딩) - 실제 전송 주소를 설명하려면 포트 이름(WSDL에 정의)을 입력하거나 전송 주소에 매핑하려면 바인딩 이름(WSDL에 정의)을 입력합니다. 위에 설명된 대로 찾아보기를 사용하여 WSDL을 선택한 경우 WSDL 정의 선택 페이지에 WSDL에서 정의한 포트 및 바인딩이 나열됩니다. 해당 페이지에서 포트 또는 바인딩을 선택하면 (포트 또는 바인딩) 필드에 선택한 이름이 채워집니다.

  • 전송 유형의 서비스 - EJB 또는 흐름 전송을 사용하는 서비스를 생성하려면 이 옵션을 선택합니다.
  • 메시징 서비스 - 완전히 다른 콘텐츠 유형의 메시지를 교환하는 서비스를 생성하려면 이 옵션을 선택합니다. 이러한 교환은 요청/응답이거나 단방향일 수 있습니다. 또한 HTTP 전송에 HTTP 'GET' 옵션을 사용하면 요청 없이도 응답을 받을 수 있습니다. 웹 서비스와 달리 요청 및 응답의 콘텐츠 유형이 같을 필요는 없습니다.
  • 모든 SOAP 서비스 - 명시적으로 정의된 구체적 인터페이스가 없는 SOAP 서비스를 생성하려면 이 옵션을 선택합니다.

    • 기본값 SOAP 1.1을 유지하거나 드롭다운 목록에서 SOAP 1.2를 선택합니다.

  • 모든 XML 서비스 - 명시적으로 정의된 구체적 인터페이스가 없는 XML 서비스를 생성하려면 이 옵션을 선택합니다.

    • HTTP GET은 메시징 서비스 및 이 서비스 유형에 대해서만 지원됩니다.

  • 비즈니스 서비스 - 기존 비즈니스 서비스를 복제하려면 이 옵션을 선택합니다.

    • 경로(프로젝트/폴더) 및 비즈니스 서비스 이름을 입력하거나 찾아보기를 눌러 비즈니스 서비스 요약 페이지에서 비즈니스 서비스를 선택합니다.

    Oracle Service Bus는 다중 서비스에 동일한 URI를 사용할 수 없으므로 복제된 서비스의 경우 URI를 변경해야 합니다.

  • 프록시 서비스 - 기존 프록시 서비스를 기반으로 비즈니스 서비스를 생성하려면 이 옵션을 선택합니다.
서비스 유형(계속)
주: 다른 서비스에서 서비스를 생성할 경우 경고 규칙은 다음 방법으로 유지 관리됩니다.
    • 프록시 서비스를 비즈니스 서비스에서 생성하거나 비즈니스 서비스를 프록시 서비스에서 생성할 경우 경고 규칙이 제거됩니다(해당하는 경우).
    • 프록시 서비스를 다른 프록시 서비스에 생성하거나 비즈니스 서비스를 다른 비즈니스 서비스에서 생성할 경우 경고 규칙이 유지됩니다(존재하는 경우).

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 비즈니스 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
