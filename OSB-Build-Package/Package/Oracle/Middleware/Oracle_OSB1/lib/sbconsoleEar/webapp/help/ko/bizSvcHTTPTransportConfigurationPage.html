HTTP 전송 구성 페이지

HTTP 전송은 HTTP 및 HTTPS 끝점을 모두 지원합니다.

비즈니스 서비스 생성/편집 - HTTP 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 HTTP 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 18-10에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-10 비즈니스 서비스 생성/편집 - HTTP 전송 구성 페이지(계속)
옵션
생성 또는 편집
읽기 시간 초과
읽기 시간 초과 간격(초)을 입력합니다.
0 값은 시간 초과가 발생하지 않음을 의미합니다.
접속 시간 초과
접속 시간 초과 간격(초)을 입력합니다. 접속을 설정하기 전에 시간 초과가 만료되면 Oracle Service Bus에서 접속 오류가 발생합니다.
0 값은 시간 초과가 발생하지 않음을 의미합니다.
HTTP 요청 방법
이 매개변수를 사용하면 요청에서 다음 HTTP 메소드 중 하나를 사용할 수 있습니다.
  • POST - 길이 제한 없이 모든 데이터를 HTTP 요청 본문의 일부로 소켓 접속을 통해 직접 전달합니다. 교환이 클라이언트에 표시되지 않고 URL이 변경되지 않습니다. REST 기반 요청에서 POST는 생성/바꾸기 작업을 수행하거나 요청된 작업을 수행하도록 전송에 알려줍니다.
  • GET - 임포트할 항목을 설명하는 정보를 요청의 일부로 포함할 수 있습니다. 이 정보는 질의 문자열의 요청 URL에 추가된 문자 시퀀스로 전달됩니다. GET은 요청 메시지 유형이 "없음"으로 설정된 경우 서비스 유형이 "모든 XML 서비스" 또는 "메시징 서비스"인 비즈니스 서비스에서 사용할 수 있습니다. REST 기반 요청에서 GET은 원격 리소스의 표현을 검색합니다.
  • PUT - 서비스 유형이 "모든 XML 서비스" 또는 "메시징 서비스"인 비즈니스 서비스에서 사용할 수 있습니다. PUT은 REST 기반 요청에서 알려진 위치에 파일 업로드와 같은 생성/바꾸기 작업을 수행하도록 전송에 알려줍니다.
  • HEAD - 응답 메시지 유형이 "없음"으로 설정된 경우 서비스 유형이 "모든 XML 서비스" 또는 "메시징 서비스"인 비즈니스 서비스에서 사용할 수 있습니다. HEAD는 REST 기반 요청에서 리소스의 전체 표현이 아닌 원격 리소스의 헤더 정보를 임포트하도록 전송에 알려줍니다.
  • DELETE - 서비스 유형이 "모든 XML 서비스" 또는 "메시징 서비스"인 비즈니스 서비스에서 사용할 수 있습니다. DELETE는 REST 기반 요청에서 삭제 작업을 수행하도록 전송에 알려줍니다.
주: 메소드가 $outbound/transport/request/http:http-method 변수에 이미 설정되어 있으면 해당 값이 HTTP 요청 방법에 대해 선택한 메소드보다 우선합니다.
인증
다음 중 하나를 선택합니다.
  • 없음 - 이 서비스에 액세스하기 위해 인증이 필요하지 않는 경우 지정합니다.
  • 기본 - 이 서비스에 액세스하기 위해 기본 인증이 필요한 경우 지정합니다.

    • 기본 인증을 지정하면 WebLogic Server는 LDAP(Lightweight Directory Access Protocol) 디렉토리 서비스 및 Windows Active Directory와 같은 보안 영역에 구성된 인증 제공자에 대해 사용자 이름과 비밀번호를 사용하여 클라이언트를 인증합니다. 클라이언트는 HTTP 요청 헤더를 통해 사용자 이름과 비밀번호를 보내야 합니다.

    HTTP를 통한 기본 인증은 비밀번호가 일반 텍스트로 전송되기 때문에 사용하지 않는 것이 좋습니다. 그러나 HTTPS는 암호화된 채널을 제공하므로 HTTPS를 통해 비밀번호를 전송하는 것은 안전합니다.

    경고: 기본적으로 모든 사용자(인증된 사용자 및 익명 사용자)가 비즈니스 서비스에 액세스할 수 있습니다. 비즈니스 서비스에 액세스할 수 있는 사용자를 제한하려면 전송 레벨 권한 부여 정책을 생성합니다. 전송 레벨 액세스 정책 편집을 참조하십시오.

  • 클라이언트 인증서 - 암호화된 통신 및 강력한 클라이언트 인증(양방향 SSL)을 지정합니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Configuring Transport-Level Security"를 참조하십시오.
서비스 계정
서비스 계정을 입력합니다. 서비스 계정은 사용자 이름과 비밀번호의 별칭 리소스입니다. 기본 인증 필요 필드를 선택한 경우 이 필드는 필수 필드입니다.

자세한 내용은 다음을 참조하십시오.

  • 서비스 계정
  • 서비스 계정 추가
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않을 경우 기본 작업 관리자가 사용됩니다. 작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
요청 인코딩
기본값 ISO-8859-1을 HTTP 전송 요청을 위한 문자 집합 인코딩으로 적용하거나 다른 문자 집합 인코딩을 입력합니다.
응답 인코딩
기본값 ISO-8859-1을 HTTP 전송에 대한 응답을 위한 문자 집합 인코딩으로 적용하거나 다른 문자 집합 인코딩을 입력합니다.
프록시 서버
프록시 서버 리소스를 입력하거나 찾아보기를 눌러 구성된 프록시 서버 리소스 목록에서 항목을 선택합니다.
HTTP 리디렉션 추적
HTTP 리디렉션(응답 코드 3xx를 포함한 요청)이 자동으로 추적되도록 지정하려면 이 체크 박스를 선택합니다. 비즈니스 서비스의 URL에 아웃바운드 요청을 보내면 리디렉션이 발생하고, 해당 서비스에 URL이 더 이상 적합하지 않고 이 요청을 다른 URL로 보내야 한다는 응답 코드(예: 302)가 반환됩니다.
HTTP 리디렉션 추적 체크 박스가 선택된 경우, Oracle Service Bus는 사용자의 특별한 작업 없이 새로운 URL로 요청을 다시 전송합니다. HTTP 리디렉션이 자동으로 추적되지 않도록 하려면 이 체크 박스를 선택 취소합니다.
조각화된 스트리밍 모드 사용
HTTP 조각화된 전송 인코딩을 사용하여 메시지를 전송하려면 이 옵션을 선택합니다.
주: [HTTP 재지정 추적] 옵션을 사용하는 경우 조각화된 스트리밍을 사용하지 마십시오. 조각화된 모드에서는 재지정 및 인증을 자동으로 처리할 수 없습니다.
세션 고정
비즈니스 서비스의 HTTP 요청에 대해 세션 고정(세션 선호도라고도 함)을 사용하려면 이 옵션을 선택합니다.
고정 SessionID 이름
세션 고정이 사용으로 설정된 경우 세션에 대한 식별 이름을 입력합니다.
