JEJB 전송 구성 페이지

이 페이지에서는 JEJB 전송 프로토콜을 사용하여 전송 설정을 구성할 수 있습니다. JEJB 전송 사용에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "JEJB Transport"를 참조하십시오.

표 18-12 JEJB 전송 구성
옵션
설명
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않으면 기본 작업 관리자가 사용됩니다.
작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
EJB 사양 버전
원격 EJB 인터페이스의 EJB 버전을 선택합니다.
값으로 XMLBeans 전달
액세스하여 XQuery 표현식으로 조작할 수 있는 매개변수를 포함하는 POJO 인수가 전송 시 "인라인된" XML 형식(XMLObject)으로 생성되도록 하려면 이 옵션을 선택합니다.
주: 값으로 전달된 XMLObject에 대해서는 유형 정보를 인라인으로 사용할 수 없습니다. 이 옵션을 사용할 경우 프록시 서비스 파이프 라인의 Java 콜아웃에서 유형이 지정된 XMLObject를 인수로 전달할 수 없습니다.
참조로 POJO를 전달하려면 이 옵션을 선택하지 마십시오. 그래야만 성능이 향상됩니다.
호출자의 제목 전달
서비스 계정을 선택하는 대신, EJB를 호출할 때 Oracle Service Bus가 프록시 서비스의 인증된 주체를 전달하도록 하려면 이 옵션을 선택합니다.
서비스 계정
찾아보기를 누르고 표시된 목록에서 JNDI 서비스 계정을 선택합니다. 서비스 계정을 지정하지 않으면 익명 주체가 사용됩니다. 자세한 내용은 16장, 서비스 계정을 참조하십시오.
클라이언트 JAR
찾아보기를 누르고 표시된 목록에서 EJB 클라이언트 JAR 리소스를 선택합니다. 클라이언트 JAR은 원격 서비스에 대한 원격 또는 비즈니스 인터페이스를 포함하며, 클라이언트 JAR은 일반 아카이브 리소스로 등록됩니다.
홈 인터페이스
EJB 2.1만 - JAR로 채워진 옵션 중에서 필요한 EJBHome 인터페이스를 선택합니다.
원격 인터페이스
EJB 2.1만 - 이 필드는 홈 인터페이스의 구성에 따라 자동으로 채워집니다.
비즈니스 인터페이스
EJB 3.0만 - 호출할 클라이언트 JAR에서 비즈니스 인터페이스를 선택합니다.
대상 네임스페이스
이 필드는 JAR에서 선택한 정보로 채워집니다.
메소드
필요한 메소드를 선택합니다. 기본 매개변수 값을 편집할 수 있도록 메소드를 확장하려면 +를 누릅니다.
제공된 메소드에 대한 기본 작업 이름을 변경할 수 있습니다. 기본적으로 작업 이름은 메소드 이름입니다. EJB에 동일한 이름의 메소드가 포함되어 있는 경우 작업 이름을 고유하게 변경해야 합니다. WSDL에서는 고유한 작업 이름을 사용해야 합니다.
