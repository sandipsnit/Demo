JMS 전송 구성 페이지

비즈니스 서비스 생성/편집 - JMS 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "JMS Transport"를 참조하십시오.

이 페이지에서는 JMS 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 18-13에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-13 비즈니스 서비스 생성/편집 - JMS 전송 구성 페이지(계속)
옵션
생성 또는 편집
대상 유형
JMS 브리지 대상 유형을 선택합니다.
  • 대기열(지점 간의 경우)
  • 항목(게시/가입의 경우)
메시지 유형
다음 중 하나를 선택합니다.
  • 바이트(해석되지 않은 바이트 스트림의 경우)
  • 텍스트(텍스트 메시지의 경우)
이 옵션은 응답 메시지 유형으로 Java를 선택한 경우 사용 안함으로 설정됩니다.
응답 대기열
이 옵션은 대상 유형으로 대기열을 선택한 경우에만 사용할 수 있습니다.
다음과 같은 응답 옵션 중 하나를 선택합니다.
  • 없음 - 응답이 필요하지 않습니다. 단뱡항 작업에 대해 이 옵션을 설정합니다.
  • 모든 요청 URI에 대해 하나 - 응답을 처리할 URI를 하나 입력하고, 기타 응답 구성 세부 정보(예: 인코딩 및 시간 초과)를 설정하고, 필요에 따라 JMS/JNDI 인증서 전달에 사용할 JMS 서비스 계정을 선택할 수 있습니다.
  • 요청 URI마다 하나씩 - 이 옵션은 응답 장애 조치를 제공하므로 요청 URI마다 응답 URI 또는 대상을 하나씩 입력할 수 있습니다.
필요에 따라 각 요청/응답 쌍에서 JMS/JNDI 인증서에 대한 서비스 계정을 선택할 수 있습니다.
응답 패턴
이 옵션은 [응답 대기열] 필드에서 응답 옵션을 선택한 경우에만 사용할 수 있습니다.
다음 중 하나를 선택합니다.
  • Oracle WebLogic Server에서 실행되는 JAX-RPC 서비스 이외의 다른 모든 서비스에 대해서는 JMSCorrelationID를 선택합니다.
  • Oracle WebLogic Server에서 실행되는 JAX-RPC 서비스에 대해서는 JMSMessageID를 선택합니다.
자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Message ID and Correlation ID Patterns for JMS Request/Response"를 참조하십시오.
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 Oracle WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않을 경우 기본 작업 관리자가 사용됩니다.
예를 들어, 비즈니스 서비스에 JMS 전송 프로토콜이 존재하는 경우 비즈니스 서비스 끝점은 특정 작업 할당 정책과 연관시킬 수 있는 MDB(메시지 파생 Bean) JAR 파일입니다.
작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
요청 인코딩
인코딩 요청에 대한 문자 집합을 입력합니다. 기본값은 UTF-8입니다.
응답 인코딩
이 옵션은 [응답 대기열] 필드에서 응답 옵션 중 하나를 선택한 경우에만 사용할 수 있습니다.
인코딩 응답에 대한 문자 집합을 입력합니다. 기본값은 UTF-8입니다.
응답 시간 초과
이 옵션은 [응답 대기열] 필드에서 응답 옵션 중 하나를 선택한 경우에만 사용할 수 있습니다.
접속을 삭제하기 전에 응답을 기다릴 시간(초)을 입력합니다. 기본값은 응답 시간이 초과되지 않음을 의미하는 0입니다.
클라이언트 Jar
이 옵션은 해당 서비스가 Java 응답 유형을 가진 메시징 서비스인 경우 사용할 수 있습니다. Java 객체가 포함된 메시지를 대기열에서 빼는 데 사용할 클라이언트 JAR을 선택하십시오. 클라이언트 JAR은 클래스 경로에 있는 경우 선택할 수 있습니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Sending and Receiving Java Objects in Messages"를 참조하십시오.
응답 URI
이 옵션은 [모든 요청 URI에 대해 하나] 응답 옵션과 JMSCorrelationID 응답 패턴을 선택한 경우 사용할 수 있습니다.
다음 형식으로 응답 URI를 입력합니다.
jms://host:port/connection_factory/jndi_destination
다중 서버를 대상으로 지정하려면 다음 형식을 사용합니다.
jms://host1:port,host2:port/connection_factory/jndi_destination
다음과 같이 응답 URI에서 호스트와 포트를 생략할 수도 있습니다. 다음과 같습니다.
jms:///connection_factory/jndi_destination

호스트와 포트를 생략할 경우 로컬 서버에서 접속 팩토리/대상이 조회됩니다. 이 형식은 요청 URI가 외부 접속 팩토리/대상을 가리키지만 요청이 로컬 서버로 전송되도록 하려는 경우 유용합니다.

주: Oracle WebLogic Server에서는 "myqueues/myqueue"와 같이 JNDI 이름에 슬래시를 사용할 수 있지만 JNDI 이름에 슬래시가 포함되어 있으면 Oracle Service Bus에 필요한 URI 형식과 충돌하므로 이러한 이름을 사용할 수 없습니다. 이 문제를 해결하려면 JMS 외부 서버를 정의하고 URI에서 이 외부 서버를 참조하십시오. 자세한 내용은 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help(http://www.oracle.com/pls/as111170/lookup?id=WLACH)의 "Configure foreign servers"를 참조하십시오.

응답 요청
이 옵션은 응답 장애 조치를 제공하도록 JMSCorrelationID 패턴에 대해 [요청 URI마다 하나씩] 응답 옵션을 선택한 경우 사용할 수 있습니다.
위 [응답 URI] 필드에 대한 설명에 따라 일반 전송 페이지에 입력된 각 요청 URI에 대해 응답 URI를 입력합니다.
필요에 따라 JMS/JNDI 인증서에 대한 서비스 계정을 선택할 수 있습니다. 그러면 서비스가 요청 대기열과 응답 대기열에 이 계정을 사용합니다.
대상 - 응답
이 옵션은 JMSMessageID 패턴에 대해 [모든 요청 URI에 대해 하나] 응답 옵션을 선택한 경우 사용할 수 있습니다.
응답을 수신할 대상 서버의 이름을 입력하고 [응답 URI] 필드에 대한 설명에 따라 응답 URI를 입력합니다.
접속 요청
이 옵션은 응답 장애 조치를 제공하도록 JMSMessageID 패턴에 대해 [요청 URI마다 하나씩] 응답 옵션을 선택한 경우 사용할 수 있습니다.
[시퀀스 번호] 필드의 숫자로 순차적으로 식별되는 각 요청 URI에 대해 필요에 따라 JMS 접속 팩토리 이름을 입력합니다. 이름을 입력하지 않으면 JMS 전송에 요청 URI의 접속 팩토리가 사용됩니다.
필요에 따라 JMS/JNDI 인증서에 대한 서비스 계정을 선택할 수 있습니다. 그러면 서비스가 요청 대기열과 응답 대기열에 이 계정을 사용합니다.
대상 - 대상
이 옵션은 응답 장애 조치를 제공하도록 JMSMessageID 패턴에 대해 [요청 URI마다 하나씩] 응답 옵션을 선택한 경우 사용할 수 있습니다. 이 필드는 [접속 요청] 옵션과 함께 사용하십시오.
현재 도메인의 서버에 따라 결정되는 각 대상 서버(예: 클러스터의 관리 서버) 목록에 [접속 요청] 필드 값에 해당하는 요청 URI가 시퀀스 번호별로 나열됩니다. 각 대상의 요청 URI 각각에 대해 응답을 수신하는 해당 대상 서버의 대상 대기열을 입력합니다.
주: Oracle Service Bus 개발 환경은 단일 서버 환경만 지원하므로 하나의 대상만 나열됩니다. 다중 서버 환경에서 이 필드를 구성하려면 해당 서비스를 런타임 환경에 배치하고 Oracle Service Bus 관리 콘솔에서 서비스 구성을 완료하십시오.
JMS 서비스 계정
이 옵션은 [응답 대기열] 필드에서 [없음] 또는 [모든 요청 URI에 대해 하나] 옵션을 선택한 경우 사용할 수 있습니다.
JMS 서버에서 관리하는 JMS 리소스에 사용할 서비스 계정을 선택합니다. 서비스 계정은 요청과 응답에 사용되는 사용자 ID 및 비밀번호의 별칭 리소스입니다. JNDI와 JMS용으로 동일한 서비스 계정이 사용됩니다.
이 필드는 [호출자의 주체 전달] 옵션과 함께 사용될 수 없습니다. JMS/JNDI 인증에 대해 둘 중 하나만 사용하십시오.
자세한 내용은 서비스 계정을 참조하십시오.
SSL 사용
TLS/SSL 접속을 통해 요청을 제출한 경우에만 이 옵션을 선택합니다.
TLS/SSL(Secure Sockets Layer)은 네트워크를 통해 접속하는 두 개의 응용 프로그램이 서로의 ID를 인증할 수 있게 하고 응용 프로그램 간에 교환되는 데이터를 암호화하여 보안 접속을 제공합니다. 인증을 사용하면 서버는 물론 선택적으로 클라이언트도 네트워크 접속의 반대쪽에 있는 응용 프로그램의 ID를 확인할 수 있습니다. 또한 관리자가 대상의 JNDI 항목에 대해 액세스 제어를 설정하여 개별 JMS 대상(대기열 또는 항목)에 대한 액세스를 제한한 경우 서비스는 JNDI 트리에서 항목을 조회할 때 [서비스 계정] 또는 [호출자의 주체 전달] 옵션을 사용하여 인증을 수행해야 합니다. [서비스 계정] 또는 [호출자의 주체 전달] 옵션을 사용하여 인증을 수행해야 합니다.
만료
메시지가 만료되기 전의 시간 간격(밀리초)입니다. 기본값은 메시지가 만료되지 않음을 의미하는 0입니다.
메시지 저장 사용
서비스에 사용되는 JMS 전달 모드입니다. 이 모드에서 처리량을 조정하여 안정성을 유지할 수 있습니다. 보장된 메시지 전달에 대해 이 옵션을 선택합니다(기본값). 가끔씩 발생하는 메시지 손실을 허용할 수 있을 경우 처리량을 향상시키려면 이 옵션의 선택을 해제합니다.
순서 단위
메시지 순서 단위를 입력합니다. 메시지 순서 단위는 메시지 생성자가 처리 순서와 관련해서 메시지를 하나의 단위로 그룹화할 수 있게 해 줍니다. 이러한 단일 순서 단위에서는 모든 메시지를 생성된 순서대로 순차적으로 처리해야 합니다.
호출자의 제목 전달
메시지 전송 시 Oracle Service Bus가 인증된 제목을 전달하도록 하려면 이 체크 박스를 선택합니다.
이 필드를 사용으로 설정한 상태에서 비즈니스 서비스가 다른 도메인에 있는 JMS 리소스를 대상으로 할 경우 양쪽 도메인에서 전역 신뢰를 사용으로 설정합니다. Securing Oracle WebLogic Server의 Configuring Security for a WebLogic Domain을 참조하십시오.
이 필드는 [서비스 계정] 옵션과 함께 사용될 수 없습니다. JMS/JNDI 인증에 대해 둘 중 하나만 사용하십시오.
이 필드를 사용으로 설정한 상태에서 비즈니스 서비스가 다른 도메인에 있는 JMS 리소스를 대상으로 할 경우 양쪽 도메인에서 전역 신뢰를 사용으로 설정합니다. http://www.oracle.com/pls/as111170/lookup?id=SECMG402에서 Oracle Fusion Middleware Securing Oracle WebLogic Server의 "Configuring Security for a WebLogic Domain"을 참조하십시오.
JNDI 시간 초과
JNDI 접속 시간 초과(초)는 JNDI 트리에서 대상 또는 접속 팩토리를 조회하는 동안 사용됩니다.
기본값은 접속 시간이 초과되지 않음을 의미하는 0입니다.
