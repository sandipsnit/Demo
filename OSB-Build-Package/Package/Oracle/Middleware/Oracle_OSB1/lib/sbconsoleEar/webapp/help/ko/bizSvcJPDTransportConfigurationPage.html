JPD 전송 구성 페이지

비즈니스 서비스 생성/편집 - JPD 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 JPD 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 17-14에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 17-14 비즈니스 서비스 생성/편집 - JPD 전송 구성 페이지(계속)
옵션
생성 또는 편집
시스템 서비스 계정
사용자의 서비스 계정을 입력하거나 찾아보기를 눌러 표시된 목록에서 계정을 선택합니다.
시스템 서비스 계정은 들어오는 메시지를 보내기 위해 JPD 전송이 사용하는 WLI 상태 비유지 세션 호출을 위한 사용자 인증서를 지정합니다.
서비스 계정을 지정하지 않을 경우 인바운드 요청 제목이 사용됩니다. 인바운드 요청 제목이 없을 경우 익명 제목이 사용됩니다.
프로세스 서비스 계정
사용자의 서비스 계정을 입력하거나 찾아보기를 눌러 표시된 목록에서 계정을 선택합니다.
프로세스 서비스 계정은 JPD 호출을 위한 사용자 인증서를 지정합니다.
서비스 계정을 지정하지 않을 경우 인바운드 요청 제목이 사용됩니다. 인바운드 요청 제목이 없을 경우 익명 제목이 사용됩니다.
콜백 프록시 위치
JPD에서 콜백을 수신하기 위한 프록시 서비스의 위치를 다음 형식으로 입력합니다.
jms://[host:port[,host:port]*]/MyFactory/MyQueue
각 항목에 대한 설명은 다음과 같습니다.
  • host: (선택 사항) 콜백 프록시 서비스가 구성된 Oracle Service Bus 서버의 호스트 이름
  • port: (선택 사항) Oracle Service Bus가 들어오는 요청을 수신하는 포트 번호
  • MyFactory: 대기열 연결 팩토리의 JNDI 이름
  • MyQueue: Oracle Service Bus 프록시 서비스가 콜백 요청을 기다리는 대기열의 JNDI 이름
클러스터화된 환경에서 관리 서버의 호스트 이름 및 포트 번호를 콤마(,)로 구분하여 지정합니다.
콜백 프록시 위치는 호출 중인 JPD가 콜백 없는 동기식 JPD 또는 비동기식 JPD인 경우 무시됩니다.
작업 할당 정책
아웃바운드 응답에 대해 기본 작업 할당 정책 또는 구성된 작업 관리자 인스턴스를 선택합니다. 작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111120/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
트랜잭션 전달
Oracle Service Bus 내의 기존 트랜잭션을 JPD에 전달하려면 체크 박스를 선택합니다.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 비즈니스 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
