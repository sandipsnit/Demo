MQ 전송 구성 페이지

비즈니스 서비스 생성/편집 - MQ 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위함 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "MQ Transport"를 참조하십시오.

시작하기 전에

MQ 접속 리소스를 구성합니다. MQ 접속을 참조하십시오.

MQ 전송을 구성하려면

이 페이지에서는 고유 MQ 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 18-14에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-14 비즈니스 서비스 생성/편집 - MQ 전송 구성 페이지
옵션
생성 또는 편집
메시지 유형
다음 중 하나를 선택합니다.
  • 바이트(해석되지 않은 바이트 스트림의 경우)
  • 텍스트(텍스트 메시지의 경우)
응답 필요
아웃바운드 메시지가 전송된 후 응답을 요청하도록 지정하려면 이 옵션을 선택합니다.
폴링 간격
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
폴링 간격(밀리초)을 입력합니다. 기본값은 1000입니다.
응답 상관 관계 패턴
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
응답 상관 관계 패턴이 다음을 기반으로 해야 할지 여부를 지정합니다.
  • MessageID
  • CorrelationID
  • 동적 대기열 - WebSphere MQ 구현에서 응답 상관 관계에 동적 대기열이 사용된 경우 이 옵션을 선택합니다. MQ 전송은 임시 동적 대기열만 지원합니다.
자동 생성 상관 관계 값
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
자동으로 CorrelationID 또는 MessageID를 생성하려면 이 체크 박스를 선택합니다.
모델 대기열
[동적 대기열] 응답 상관 관계 패턴에만 사용됩니다. 동적 대기열 생성에 사용되는 모델 대기열의 이름을 입력합니다.
MQ 응답 URI
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
응답이 게시될 대상입니다. 다음과 같이 끝점 URI와 동일한 형식으로 응답 URI를 입력합니다.
mq://local_queue_name?conn=mq_connection_resource
또는 다음과 같은 동적 대기열을 사용한다고 가정합니다.
mq://dynamic_queue_prefix?conn=mq_connection_resource
여기서 32자로 제한되는 dynamic_queue_prefix는 MQ 서버에 동적 대기열을 생성하는 데 사용됩니다. 접두어와 고유한 ID의 조합이 대기열 이름으로 지정됩니다. 예를 들어, dynamic_queue_prefix가 example일 경우 example123129083821이 동적 대기열 이름으로 지정됩니다.
다음과 같은 동적 대기열 응답 URI에서 별표(*)를 와일드 카드 문자로 사용할 수도 있습니다. 다음과 같습니다.
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
URI에 dynamic_queue_prefix를 제공하지 않을 경우 전송에는 MQ 서버가 생성한 동적 대기열 이름이 사용됩니다. URI에 명시적 mq_connection_resource를 제공하지 않을 경우(권장) 전송에는 끝점 URI의 mq_connection_resource가 사용됩니다.
자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "MQ Transport"를 참조하십시오.
응답 시간 초과
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
접속을 끊기 전의 서버 응답 대기 시간(초)을 입력합니다.
작업 할당 정책
작업 할당 정책은 서비스 끝점에 사용할 WLS 작업 관리자의 인스턴스를 참조합니다.
작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
동적 대기열 풀링
[동적 대기열] 응답 상관 관계 패턴에만 사용됩니다. 서비스가 동적 대기열에 대해 풀링된 접속을 사용하도록 하려면 이 옵션을 선택합니다.
동적 대기열에 대해 별도의 접속 풀을 사용하려면 동적 대기열에 대해 전용 MQ 접속 리소스를 구성하는 것이 좋습니다.
각 요청에 새 동적 대기열 인스턴스를 생성하고 응답 후 대기열을 삭제하려면 이 옵션을 선택하지 마십시오.
끝점 URI `PUT' 옵션
다음 중에서 MQ PUT 메시지 옵션을 입력합니다.
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
다중 옵션을 구분하기 위해 "|" 또는 "+"를 사용할 수 있습니다. 예를 들어 다음과 같이 지정할 수 있습니다.
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
메시지를 아웃바운드 대기열에 배치할 경우 MQ PUT 메시지 옵션이 적용됩니다.
MQ 인식되지 않는 응답 URI
인식되지 않는 응답 메시지를 전송해야 하는 대기열을 표시하는 URI를 입력합니다. 이 설정은 자동 생성 상관 관계 값 체크 박스를 선택한 경우에만 사용으로 설정됩니다.
이 필드의 값을 지정하지 않을 경우 인식되지 않는 응답 메시지가 삭제됩니다.
프로세스 RFH2 헤더
메시지 페이로드에서 WebSphere MQ RFH2 헤더의 구문을 분석하고 RFH2 데이터를 포함하는 RFH2Headers 전송 헤더를 자동으로 생성하려면 이 옵션을 선택합니다.
이 옵션을 선택하지 않으면 페이로드가 받는 즉시 전달됩니다.

MQ 전송에서 RFH2 헤더를 처리하는 방법은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "About RFH2 Headers"를 참조하십시오.
