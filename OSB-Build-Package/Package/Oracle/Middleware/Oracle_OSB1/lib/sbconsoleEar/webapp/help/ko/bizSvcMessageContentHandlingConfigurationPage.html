메시지 처리 구성 페이지

비즈니스 서비스 생성/편집 - 메시지 콘텐츠 처리 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 Oracle Service Bus의 메시지 콘텐츠 처리 방법을 지정할 수 있습니다. 표 18-20에서는 비즈니스 서비스에 대한 메시지 처리 옵션을 설명합니다. 각 전송 유형별로 서로 다른 옵션이 제공됩니다.

표 18-20 비즈니스 서비스 생성/편집 - 메시지 콘텐츠 처리 구성 페이지
옵션
생성 또는 편집
XOP/MTOM 지원
이 기능을 사용하면 아웃바운드 메시지를 MTOM/XOP 형식으로 인코딩할 수 있습니다. SOAP MTOM(Message Transmission Optimization Mechanism)은 웹 서비스에서 이진 데이터를 보내고 받는 방법입니다. MTOM에서는 XOP(XML-binary Optimized Packaging)를 사용하여 이진 데이터를 전송합니다.
Oracle Service Bus에서는 다음과 같은 전송을 사용하여 XOP/MTOM을 지원합니다.
  • HTTP/S
  • 로컬
  • SB
비즈니스 서비스에서 아웃바운드 메시지를 MTOM/XOP 형식으로 인코딩할 수 있도록 하려면 사용으로 설정됨 체크 박스를 선택합니다. 이 옵션은 이전 릴리스 구성을 기반으로 하는 임포트한 비즈니스 서비스에는 사용 안함으로 설정됩니다.
XOP/MTOM 지원을 사용으로 설정하면 $header 및 $body 메시지 컨텍스트 변수의 이진 데이터 처리 방법을 다음 옵션 중에서 선택할 수 있습니다.
  • 참조로 이진 데이터 포함: (기본값) 아웃바운드 응답 메시지에서 $body 메시지 컨텍스트 변수를 설정할 때 xop:Include 요소를 ctx:binary-content 요소로 대체합니다.
  • 값으로 이진 데이터 포함: 아웃바운드 응답 메시지에서 $body 메시지 컨텍스트 변수를 설정할 때 xop:Include 요소를 해당 이진 데이터의 Base64로 인코딩된 텍스트 버전으로 대체합니다.
비즈니스 서비스에 대해 XOP/MTOM 지원이 사용으로 설정된 경우 모든 아웃바운드 메시지가 MTOM 형식일 필요는 없습니다. 대신 이 설정에서는 비즈니스 서비스가 MTOM 페이로드를 처리할 수 있도록 지정합니다.
Oracle Service Bus에서는 MTOM과 SwA 조합을 지원하지 않으므로 Oracle Service Bus에서 아웃바운드 요청을 비즈니스 서비스로 작업을 할당할 때 비즈니스 서비스에서 MTOM/XOP를 사용으로 설정하고 $attachments 메시지 컨텍스트 변수가 널이 아니면 런타임 오류가 발생합니다.
첨부 파일
Oracle Service Bus에서는 HTTP/S 전송을 사용한 MIME 첨부 파일 스트리밍을 지원합니다.
이 기능을 사용하면 아웃바운드 응답 메시지의 첨부 파일을 디스크 파일에 저장한 다음 첨부 파일 내용을 메모리에 버퍼링하지 않고 스트리밍 형식으로 데이터를 처리할 수도 있습니다. 따라서 비즈니스 서비스에서 큰 첨부 파일을 안정적이고 효과적인 방식으로 처리할 수 있습니다.
비즈니스 서비스에서 아웃바운드 응답 메시지의 첨부 파일을 스트리밍할 수 있도록 하려면 첨부 파일을 디스크로 페이징 체크 박스를 선택합니다.
XOP/MTOM 지원을 사용으로 설정하는 경우에는 XOP/MTOM 지원에서 참조로 이진 데이터 포함 옵션을 선택하는 경우에만 첨부 파일 옵션을 사용할 수 있습니다. 또한 첨부 파일이 포함된 페이로드는 RFC 822를 준수해야 합니다. 특히 인터넷 헤더를 포함하는 라인은 CRLF(캐리지 리턴 줄 바꿈)로 끝나야 합니다.
결과 캐싱
결과가 거의 변경되지 않는 비즈니스 서비스를 호출할 경우 결과 캐싱을 사용하면 외부 서비스를 직접 호출하는 대신 캐시에 저장된 결과를 클라이언트에 반환하여 비즈니스 서비스 성능을 향상시킬 수 있습니다.
비즈니스 서비스에 대해 결과 캐싱을 사용하려면 먼저 결과 캐싱을 전체적으로 사용으로 설정에 설명된 대로 결과 캐싱을 전체적으로 사용으로 설정해야 합니다.
다음 지침에 따라 고급 설정 섹션에서 결과 캐싱을 구성합니다.
  • 지원됨 옵션을 선택합니다. 이렇게 하면 남은 필드가 사용으로 설정됩니다.
  • 표현식 네임스페이스 - 결과 캐싱을 구성하기 위해 표현식을 사용할 경우 [캐시 토큰 표현식]이나 [만료 시간]을 사용하든지 또는 두 옵션을 모두 사용하든지 간에 표현식에 사용할 네임스페이스 및 해당 접두어를 입력할 수 있습니다. 이 필드에서는 또한 기존 네임스페이스 목록을 확인할 수 있습니다.
  • 캐시 토큰 표현식 - Oracle Service Bus에서는 캐시 키를 사용하여 검색 또는 채우기 용도로 캐시에 저장된 결과를 식별합니다. 캐시 키의 캐시 토큰 부분은 고유한 식별자를 제공합니다. {1}캐시 토큰 표현식{2} - Oracle Service Bus에서는 캐시 키를 사용하여 검색 또는 채우기 용도로 캐시에 저장된 결과를 식별합니다.

    • 요청(비즈니스 서비스를 호출하는 프록시 서비스 메시지 흐름)에 포함된 값에서 캐시 토큰을 생성하려면 $body, $header, $operation 또는 $transportMetaData($outbound/ctx:transport/ctx:request 또는 $outbound/ctx:transport/ctx:response) 파이프 라인에서 값을 가져오는 표현식을 사용합니다. 예를 들어, $body 메시지에 있는 고객 ID로부터 캐시 토큰을 채울 수 있습니다.

    캐시 토큰 표현식은 하위 요소가 없는 속성 또는 요소와 같은 단순 콘텐츠의 값 또는 문자열로 계산되어야 합니다. 표현식이 널로 평가되거나 오류가 발생할 경우 결과가 캐시에 저장되지 않습니다.

    주: 또한 비즈니스 서비스 구성에서 캐시 토큰 표현식을 설정하지 않고도 요청에서 캐시 토큰을 생성할 수 있습니다. 이렇게 하려면 프록시 서비스 파이프 라인에서 $outbound/ctx:transport/ctx:request/ctx:cache-token의 값을 포함하십시오. cache-token에 있는 값은 비즈니스 서비스 구성의 캐시 토큰 표현식을 무효화합니다.

  • 만료 시간 - 만료 시간 또는 TTL(time-to-live)은 비즈니스 서비스 결과 캐시에 있는 항목이 비워지는 시간을 결정합니다. 기간이 0이면 만료 시간이 없음을 의미합니다. 기간이 음수면 캐시에 저장하지 않음을 의미합니다. 다음 중 하나를 선택합니다.
 
    • 기본값 사용 - 기본값은 DOMAIN_HOME/config/osb/coherence/osb-coherence-config.xml에 있는 expiry-delay 값입니다. 기본값은 5분입니다.
    • 기간 - 특정 기간을 설정합니다.
    • XQuery 표현식 - 표현식으로 캐시 토큰을 설정할 때와 같이 요청 또는 응답에서 만료 시간 값을 가져오는 XQuery 표현식을 입력할 수 있습니다. 요청 또는 응답을 선택하십시오.

      요청 또는 응답에서 만료 시간을 생성하려면 파이프 라인 $body, $header, $operation 또는 $transportMetaData($outbound/ctx:transport/ctx:request 또는 $outbound/ctx:transport/ctx:response)에서 값을 가져오는 표현식을 사용하십시오. 예를 들어, Cache-Control HTTP 헤더에서 설정한 값을 사용합니다.

      만료 시간은 정수(초), XQuery dayTimeDuration(XSD 유형) 또는 하위 요소가 없는 속성 또는 요소와 같은 단순 콘텐츠의 정수 값으로 분석되어야 합니다. 표현식이 널로 평가되거나 오류가 발생하면 결과가 캐시에 저장되지 않습니다.

      주: 비즈니스 서비스 구성에서 만료 시간을 설정하지 않고 요청에서 만료 시간을 생성할 수도 있습니다. 프록시 서비스 파이프 라인에 $outbound/ctx:transport/ctx:request/ctx:cache-ttl의 값을 포함시키면 됩니다. cache-ttl 요소의 값은 비즈니스 서비스 구성의 만료 시간을 무효화합니다.
결과 캐싱에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG170에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Improving Performance by Caching Business Service Results"를 참조하십시오.
