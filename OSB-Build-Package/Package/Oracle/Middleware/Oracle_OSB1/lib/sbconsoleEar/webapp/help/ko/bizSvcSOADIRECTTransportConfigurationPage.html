SOA-DIRECT 전송 구성 페이지

표 18-17에서는 SOA-DIRECT 전송의 전송 관련 구성 옵션에 대해 설명합니다.

SOA-DIRECT 전송에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV909에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "SOA-DIRECT Transport"를 참조하십시오.

표 18-17 비즈니스 서비스 생성/편집 - SOA-DIRECT 전송 구성
속성
설명
JNDI 서비스 계정
선택 사항입니다. (선택 사항) 대상 SOA 서비스의 JNDI 조회에 사용할 보안 인증서를 지정합니다. 서비스 계정은 정적이어야 합니다. 찾아보기를 누르고 서비스 계정을 선택합니다. 서비스 계정을 지정하지 않으면 익명 주체가 사용됩니다.
역할
필수 사항입니다. 서비스에 사용되는 통신 패턴을 식별합니다. 다음 옵션 중 하나를 선택하십시오.
  • 동기 클라이언트(기본값) - 이 롤에서는 비동기 콜백이 없으므로 [콜백 프록시] 옵션이 사용 안함으로 설정됩니다. [WS-Addressing 버전] 옵션도 사용 안함으로 설정됩니다.
  • 비동기 클라이언트 - 이 롤에서는 일반적으로 비동기 콜백이 필요하므로 콜백 프록시를 식별할 수 있으며 WS-Addressing 버전을 선택해야 합니다. 비동기 옵션은 WSDL 서비스가 SOAP 유형인 경우에만 사용으로 설정됩니다.
  • 서비스 콜백 - 이 롤은 외부 서비스 호출 후 비동기 콜백을 SOA 서비스로 반환하는 데 사용됩니다.

    • [서비스 콜백] 롤에서는 서비스에 대해 로드 밸런싱 또는 장애 조치를 사용할 수 없습니다.

콜백 프록시
선택 사항입니다. [비동기 클라이언트] 롤에 대해서만 사용으로 설정됩니다.
이 옵션을 통해 콜백을 수신하는 프록시 서비스를 지정할 수 있습니다. 콜백 프록시를 선택하면 요청 또는 프록시 서비스 파이프 라인에 제공된 WS-Addressing이 없을 경우 Oracle Service Bus가 자동으로 SOAP 헤더에 ReplyTo 속성을 채웁니다. SB 전송(RMI의 경우)을 사용하는 WSDL 프록시 서비스를 선택해야 하며 콜백 프록시 서비스가 WS-Addressing을 인식해야 합니다.
요청에서 전송되거나 프록시 서비스 파이프 라인에서 설정된 WS-Addressing 속성이 이 옵션에서 설정하는 콜백 프록시 대신 사용됩니다.
콜백 프록시를 지정하지 않은 상태에서 요청에 ReplyTo 속성이 포함되지 않을 경우 프록시 서비스 파이프 라인을 통해 SOAP 헤더에 ReplyTo 속성을 제공해야 합니다.
결함 프록시
현재 이 옵션은 지원되지 않습니다. 비동기 패턴의 결함을 처리하려면 콜백 서비스를 구성해야 합니다.
WS-Addressing 버전
필수 사항입니다. [비동기 클라이언트] 롤에 대해서만 사용으로 설정됩니다.
요청 또는 프록시 서비스 파이프 라인에 제공된 WS-Addressing이 없을 경우 사용할 기본 WS-Addressing 버전을 지정합니다. 요청에서 전송되거나 프록시 서비스 파이프 라인에서 설정된 WS-Addressing 속성이 이 옵션에서 설정하는 WS-Addressing 버전 대신 사용됩니다.
WS-Addressing 버전이 환경 간에 일치하지 않을 경우 프록시 서비스 파이프 라인에서 필요한 변환을 수행합니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1827에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Transformation Examples"를 참조하십시오.
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않으면 기본 작업 관리자가 사용됩니다.
작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
호출자의 제목 전달
선택 사항입니다. SOA 서비스를 호출할 때 Oracle Service Bus가 프록시 서비스의 인증된 주체를 전달하도록 하려면 이 옵션을 선택합니다. 이 옵션을 선택하면 [호출자의 주체 전달] 대신 [호출 서비스 계정] 옵션이 사용 안함으로 설정됩니다.
주: 클라이언트와 대상 서버가 다른 도메인에 있을 경우 클라이언트와 대상 서버 간의 도메인 신뢰를 사용으로 설정해야 합니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=SECMG403에서 Oracle Fusion Middleware Securing Oracle WebLogic Server의 "Important Information Regarding Cross-Domain Security Support"를 참조하십시오.
호출 서비스 계정
선택 사항입니다. RMI 호출에 대한 서비스 계정을 선택하여 사용자 정의 보안 인증서를 지정할 수 있는 [호출자의 주체 전달] 대신, 서비스 계정 유형(통과, 정적, 매핑)을 지정할 수 있습니다.
찾아보기를 누르고 서비스 계정을 선택합니다. 서비스 계정을 지정하지 않으면 익명 주체가 사용됩니다.
