Tuxedo 전송 구성 페이지

비즈니스 서비스 생성/편집 - Tuxedo 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Tuxedo Transport"를 참조하십시오.

이 페이지에서는 Tuxedo 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 18-18에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-18 비즈니스 서비스 생성/편집 - Tuxedo 전송 구성 페이지(계속)
옵션
생성 또는 편집
필드 테이블 클래스
수신된 FML/FML32 버퍼를 설명하는 클래스 이름을 입력합니다. 이러한 클래스 이름은 필드 이름을 요소 이름에 매핑하기 위해 FML/FML32-to-XML로의 변환 루틴에 사용됩니다. 공백으로 구분된 정규화된 클래스 이름 목록입니다.
뷰 클래스
수신 또는 전송된 VIEW/VIEW32 버퍼를 설명하는 클래스 이름을 입력합니다. 이러한 클래스 이름은 필드 이름을 요소 이름에 매핑하기 위해 VIEW에서 XML 또는 VIEW32에서 XML로의 변환 루틴에 사용됩니다. 공백으로 구분된 정규화된 클래스 이름 목록입니다.
클래스 Jar
이 끝점 작업에 필요한 FML/FML32 또는 VIEW/VIEW32 클래스가 있는 JAR 파일이 포함된 JAR 리소스를 선택합니다.
원격 액세스 지점
임포트와 연관된 드롭다운 목록에서 원격 액세스 지점을 선택합니다. 이 드롭다운 목록에는 WTC에서 구성된 원격 액세스 지점이 포함되어 있습니다. 연관된 원격 액세스 지점이 없으면 비즈니스 서비스를 생성할 수 없습니다.
원격 액세스 지점이 존재하지 않거나 새 지점을 생성하려면 새로 만들기를 선택합니다. 인접한 필드에 해당 액세스 지점 이름 및 네트워크 주소를 입력합니다. 끝점을 검증하면 액세스 지점이 각 WTC 서버의 WTC 구성에 추가됩니다. WTC 서버가 존재하지 않는 경우 서버가 생성됩니다.
URI를 두 개 이상 지정한 경우 URI당 하나의 원격 액세스 지점 필드가 있으며 정보용으로 해당 URI가 표시됩니다. URI가 두 개 이상 존재하면 각각에 다른 원격 액세스 지점이 필요합니다. 지정된 URI에 해당하는 WTC 리소스가 이미 있으면 해당 원격 액세스 지점이 표시되지만 이 액세스 지점을 수정할 수는 없습니다.
로컬 액세스 지점
이 필드는 원격 액세스 지점 필드에서 새로 만들기를 선택한 경우에만 나타납니다.
드롭다운 목록에서 새로 생성한 원격 액세스 지점과 연관시킬 로컬 액세스 지점을 선택합니다. 로컬 액세스 지점이 존재하지 않거나 새 로컬 액세스 지점을 생성하려면 새로 만들기를 선택합니다. 인접한 필드에 해당 로컬 액세스 지점 이름 및 로컬 네트워크 주소를 입력합니다.
요청 버퍼 유형
원격 Tuxedo 서비스가 수신할 버퍼 유형을 선택합니다.
요청 버퍼 하위 유형
이 옵션은 이전 요청 버퍼 유형 값이 VIEW 또는 VIEW32인 경우 사용으로 설정됩니다. 요청 버퍼와 연관시킬 버퍼 하위 유형을 입력합니다.
응답 필요
양방향 호출을 표시하려면 이 체크 박스를 선택합니다. 이 체크 박스를 선택하지 않으면 TPNOREPLY 플래그가 지정된 기본 tpcall이 호출되며 널 응답이 비동기식으로 게시됩니다.
트랜잭션 일시 중지
트랜잭션이 존재하는 경우 트랜잭션을 일시 중지하려면 이 체크 박스를 선택합니다. 이 옵션은 원격 서비스가 트랜잭션을 지원하지 않을 때 유용합니다.
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않으면 기본 작업 관리자가 사용됩니다.
이 작업 관리자는 단방향 호출의 경우 널 회신을 비동기적으로 게시하는 데 사용됩니다. 작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
요청 인코딩
Tuxedo 전송 요청을 위한 문자 집합 인코딩을 지정합니다.
응답 인코딩
Tuxedo 전송 응답을 위한 문자 집합 인코딩을 지정합니다.
시간 초과
전송 런타임이 응답을 기다리는 최대 시간(초)을 지정합니다. 0보다 크거나 같은 정수 값이어야 합니다. 시간 초과를 지정하지 않거나 0(기본값)으로 설정할 경우 응답은 로컬 WTC 액세스 지점이 호출 차단을 위해 허용하는 최대 시간(초)인 BLOCKTIME에서 시간 초과됩니다.
변환 스타일
다음 중 하나를 선택합니다.
  • 없음 - (기본값) 필드 순서는 반영되지 않을 수 있습니다.
  • 정렬 - 필드는 항목을 정확한 순서대로 표시합니다.
  • 그룹화 및 정렬 - 필드가 레코드와 같이 논리적으로 구성된 경우 필드는 항목별로 정렬되고 레코드별로 그룹화됩니다.

마친 후에
