WS 전송 구성 페이지

비즈니스 서비스 생성/편집 - WS 전송 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 구성 편집에 설명된 것처럼 비즈니스 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "WS Transport"를 참조하십시오.

이 페이지에서는 WS 전송 프로토콜을 사용하여 비즈니스 서비스의 전송 설정을 구성할 수 있습니다. 표 18-19에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-19 비즈니스 서비스 생성/편집 - WS 전송 구성 페이지
옵션
생성 또는 편집
응답 시간 초과
응답을 기다릴 시간(초)을 입력합니다.
이 필드를 비워 두면 응답 시간 초과가 없습니다. 비즈니스 서비스는 RM 정책에 구성된 시퀀스 시간 초과 동안 기다립니다.
0 값을 입력하면 시간 초과가 없습니다.
서비스 계정
서비스 계정을 입력하거나 찾아보기를 눌러 표시된 목록에서 계정을 선택합니다.
서비스 계정은 이 서비스에 HTTP 기본 인증 정책이 있을 경우 사용할 인증서를 지정합니다.
고급 설정
대기열 오류 메시지
구성된 오류 대기열에 오류 메시지 전송을 사용으로 설정하려면 체크 박스를 선택합니다.
오류 대기열 URI
이 옵션은 대기열 오류 메시지 체크 박스가 선택된 경우에만 사용할 수 있습니다.
오류 메시지를 저장하려면 다음 형식으로 JMS 대기열의 URI를 입력합니다.

jms://host:port/connFactoryJndiName/queueJndiName

주: Oracle WebLogic Server에서는 "myqueues/myqueue"와 같이 JNDI 이름에 슬래시를 사용할 수 있지만 JNDI 이름에 슬래시가 포함되어 있으면 Oracle Service Bus에 필요한 URI 형식과 충돌하므로 이러한 이름을 사용할 수 없습니다. 이 문제를 해결하려면 JMS 외부 서버를 정의하고 URI에서 이 외부 서버를 참조하십시오. 자세한 내용은 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help(http://www.oracle.com/pls/as111170/lookup?id=WLACH)의 "Configure foreign servers"를 참조하십시오.

JMS 오류 대기열 서비스 계정
이 옵션은 대기열 오류 메시지 체크 박스가 선택된 경우에만 사용할 수 있습니다.
서비스 계정을 입력하거나 찾아보기를 눌러 표시된 목록에서 계정을 선택합니다.
서비스 계정은 JNDI 조회 및 오류 메시지를 오류 대기열에 전송하는 데 사용할 인증서를 지정합니다.
오류 대기열에 SSL 사용
이 옵션은 대기열 오류 메시지 체크 박스가 선택된 경우에만 사용할 수 있습니다.
오류 대기열에 접속하기 위해 SSL을 사용하려면 체크 박스를 선택합니다.
