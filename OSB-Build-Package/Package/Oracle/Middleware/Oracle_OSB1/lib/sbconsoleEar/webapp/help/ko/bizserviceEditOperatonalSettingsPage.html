비즈니스 서비스의 작업 설정 구성

작업 설정 페이지에서 특정 비즈니스 서비스의 비즈니스 서비스 및 모니터링을 사용. 마찬가지로 작업 > 스마트 검색 페이지에서 이러한 동일한 서비스별 작업 설정을 지정할 수 있습니다. 단, 집계 간격을 설정하거나 경보 심각도 레벨을 지정할 수는 없습니다. 자세한 내용은 작업 설정 찾기 및 갱신을 참조하십시오.

서비스 레벨 설정의 런타임 효과는 해당 전역 설정에 따라 다릅니다. 서비스를 런타임 시 완벽하게 사용으로 설정하려면 전역 및 서비스 레벨 설정 모두를 사용으로 설정해야 합니다. 또한 서비스 상태도 사용으로 설정해야 합니다. 전역 설정 사용을 참조하십시오.

런타임 시 서비스가 사용으로 설정되지 않은 경우에도 모니터링 구성 설정을 변경하고 저장할 수 있습니다. 예를 들어, 서비스 모니터링이 사용 안함으로 설정된 경우에도 집계 간격을 변경하고 저장할 수 있습니다. 이와 같은 방법으로 설정을 편집한 후 나중에 사용으로 설정할 수 있습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 리소스 브라우저 > 비즈니스 서비스를 선택합니다.
  3. 비즈니스 서비스 이름을 누릅니다.

    4. 비즈니스 서비스 보기 페이지에 표 25-8과 같은 정보가 표시됩니다.

    표 25-8 비즈니스 서비스 정보
    속성
    설명
    마지막으로 수정한 사람
    이 서비스를 생성했거나 편집했거나 구성으로 임포트한 사용자입니다.
    마지막 수정 시간
    사용자가 이 서비스를 생성했거나 편집했거나 구성으로 임포트한 날짜와 시간입니다. 날짜 및 시간 링크를 눌러 이 리소스의 변경 내역을 봅니다. 변경 내역 보기 페이지를 참조하십시오.
    참조
    이 서비스에서 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    참조자
    이 서비스를 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    설명
    서비스(존재하는 경우)에 대한 설명입니다.

  4. 작업 설정 탭을 선택합니다.

    5. 표 25-9에 표시된 비즈니스 서비스 작업 설정을 구성하려면 활성 세션에 있어야 합니다.

    표 25-9 비즈니스 서비스 작업 설정 구성
    수행할 작업
    작업 단계
    상태를 사용 안함으로 설정
    사용으로 설정됨 체크 박스를 선택 취소합니다. 세션을 활성화할 때, 서비스가 메시지 처리를 중지합니다.
    서비스를 사용 안함으로 설정하면 시스템이 해당 서비스에 대해 이전에 수집했던 모든 모니터링 통계를 삭제할 수 있습니다.
    상태를 사용으로 설정
    사용으로 설정됨 체크 박스를 선택합니다. 세션을 활성화할 때, 서비스가 메시지 처리를 다시 시작합니다.
    비즈니스 서비스 끝점 URI를 오프라인 상태로 사용으로 설정
    오프라인 URI에 대한 재시도 간격 설정
    재시도 간격과 함께 사용으로 설정 체크 박스를 선택합니다.
    이 옵션을 선택하면 비즈니스 서비스가 런타임 시 응답하지 않는 끝점 URI를 제거하여(오프라인 상태로 전환) 응답하는 URI만 재시도 및 이후의 요청 처리에 사용되도록 합니다.
    수정 작업을 취한 다음 응답하지 않는 URI를 다시 사용으로 설정할 때까지 이 URI를 오프라인 상태로 유지하려면 재시도 간격을 입력하지 마십시오. 예를 들어, 재시도 간격 0은 끝점이 무제한으로 오프라인 상태에 있음을 나타냅니다. 온라인 상태로 돌아가려면 끝점 URI 페이지에서 끝점을 온라인으로 표시 작업을 사용합니다. 비즈니스 서비스 끝점 URI 측정 단위 보기를 참조하십시오.
    비즈니스 서비스가 응답하지 않는 URI를 오프라인 상태로 전환하여 지정된 시간 간격 후에 이후의 메시지 처리를 위해 동일한 URI 끝점에 다시 액세스를 시도하도록 구성하려면 재시도 간격 값을 지정하면 됩니다. 시간, 분 및 초 필드를 사용하여 활성 URI 중에서 응답하지 않는 URI를 루프 처리 상태로 유지할 시간을 지정하십시오.
    URI는 지정된 시간 간격 동안 오프라인 상태로 유지된 다음 재시도됩니다. 끝점이 응답하는 경우 해당 URI는 다시 온라인 상태로 전환되거나 오프라인 상태로 유지되어 프로세스가 반복됩니다.
    재시도 간격과 함께 사용으로 설정 옵션을 선택한 경우에만 재시도 간격을 지정할 수 있습니다.
    모니터링을 사용 안함으로 설정
    사용으로 설정됨 체크 박스를 선택 취소합니다. 세션을 활성화하는 경우 시스템은 서비스에 대한 모니터링 통계 수집을 중지하고 이전에 수집한 모든 통계를 삭제합니다.
    모니터링을 사용으로 설정
    사용으로 설정됨 체크 박스를 선택합니다. 세션을 활성화하는 경우 시스템은 서비스에 대한 모니터링 통계를 수집하기 시작합니다.
    서비스의 집계 간격 설정
    간격을 시간 또는 분 단위로 선택한 다음 갱신을 누릅니다. 시간 단위를 1보다 크게 선택하면 분 단위는 항상 기본적으로 0으로 선택됩니다. 그러나, 시간 단위를 0 또는 1로 선택하면 분 단위로 간격을 구성할 수 있습니다. 서비스의 집계 간격 설정을 참조하십시오.
    모니터링의 사용으로 설정 여부에 따라 수집 간격 설정을 변경 및 저장할 수 있습니다.
    런타임 시 모니터링을 사용으로 설정하려면 모니터링 사용 체크 박스가 선택되어 있는지 확인합니다.
    SLA 경보를 사용 안함으로 설정
    사용으로 설정됨 체크 박스를 선택 취소합니다. 해당 서비스에 대해 구성된 모든 경보 규칙 평가를 중지하기 때문에 더 이상 서비스와 관련된 SLA 경보를 수신하지 않습니다.
    SLA 경보를 사용으로 설정
    사용으로 설정됨 체크 박스를 선택합니다. 경보를 사용으로 설정할 레벨 드롭다운 목록에서 선택하여 SLA 경보를 지정한 레벨 이상으로 제한합니다. 해당 목록은 가장 포괄적인 레벨에서 가장 제한적인 레벨로 나열됩니다(일반(기본값), 경고, 사소, 중요, 심각 및 치명적).
    서비스 SLA 경보는 전역 및 서비스 레벨 모니터링 상태에 따라 다르며 SLA 경보를 런타임 시 사용으로 설정하려면 두 가지 상태 모두 사용으로 설정해야 합니다. SLA 경보를 전체적으로 사용으로 설정을 참조하십시오.
    비즈니스 서비스에 구성된 끝점 세트에 전송되는 메시지 흐름 제한
    스로틀 상태 옆에 있는 사용으로 설정됨 체크 박스를 선택합니다. 사용으로 설정하면 메시지가 우선순위대로 처리됩니다. 라우팅 옵션을 사용하여 선택적으로 메시지에 우선순위를 할당할 수 있습니다. 그렇지 않으면 메시지가 선입선출 방식으로 대기열에서 제거됩니다.
    자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG614에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Throttling"을 참조하십시오.
    비즈니스 서비스에 구성된 끝점 집합에 작업이 할당되는 메시지 수 제한
    스로틀 상태를 사용으로 설정하면 이 필드는 필수 필드가 됩니다.
    최대 동시성 값을 지정합니다. 이 값은 양의 정수 값이어야 하며 0이 될 수 없습니다.
    이 값을 초과하면 수신 메시지가 대기(스로틀) 대기열(존재하는 경우)에 보관됩니다. 그렇지 않을 경우 메시지가 삭제됩니다.
    대기열이 가득 차면 대기열에서 우선순위가 낮은 메시지가 제거되고 런타임 오류가 발생합니다. 오류 처리기가 구성되어 존재하는 경우 트리거됩니다. 새 수신 메시지가 대기열에 저장됩니다.
    메시지 동시성 제한을 초과하여 비즈니스 서비스에 전송된 메시지를 대기 대기열에 보관해야 할지 여부 지정
    대기 대기열에 보관되는 최대 메시지 수 설정
    스로틀 상태를 사용으로 설정하면 이 필드는 필수 필드가 됩니다.
    스로틀 대기열 값을 지정합니다. 이 값은 양의 정수 값이거나 기본값인 0을 유지해야 합니다. 기본값 0은 스로틀 대기열이 없음을 의미합니다. 스로틀 대기열 값이 0(스로틀 대기열이 없음)이고 메시지 수가 최대 동시성 값을 초과하는 경우 메시지가 삭제되고 오류가 발생합니다. 오류 처리기가 구성되어 존재하는 경우 트리거됩니다.
    런타임 시 스로틀을 사용 안함으로 설정하면 스로틀 대기열에 있는 모든 메시지가 처리됩니다.
    비즈니스 서비스를 삭제하거나 이름을 변경하면 스로틀 대기열에 있는 모든 메시지가 삭제됩니다.
    메시지가 대기 대기열에 보관되는 최대 시간을 지정합니다.
    스로틀 상태를 사용으로 설정하면 이 필드는 선택 사항입니다.
    메시지 만료 값을 밀리초 단위로 지정합니다. 이 값은 양의 정수 값이어야 합니다. 값이 0이면 메시지가 만료되지 않음을 나타냅니다.
    이 값은 대기열 레벨에서 설정됩니다. 대기열에 있는 모든 메시지의 메시지 만료 값은 동일합니다. 이 값을 초과할 경우 메시지가 처리되지 않고 대기열에서 제거됩니다.
    비즈니스 서비스에 대한 메시지 추적을 사용으로 설정

    메시지 추적 옆에 있는 사용으로 설정됨 체크 박스를 선택합니다. 메시지 추적을 사용으로 설정하면 Oracle Service Bus 파이프 라인과 비즈니스 서비스 간에 교환된 메시지를 시스템에서 기록합니다(아웃바운드 요청 및 응답 메시지).

    주: 로그 파일 또는 표준 출력(서버 콘솔)에서 추적을 보려면 Oracle WebLogic Server 로깅이 다음 심각도 레벨로 설정되어 있어야 합니다.

    최소 로그 심각도: 정보
    로그 파일: 정보
    표준 출력: 정보

    로그 심각도 레벨 설정에 대한 자세한 내용은 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server(http://www.oracle.com/pls/as111170/lookup?id=WLLOG141)의 "Using Log Severity Levels"를 참조하십시오.

    해당되는 경우 기록된 아웃바운드 메시지에 재시도 횟수, 오류 코드 및 오류 메시지도 포함될 수 있습니다.
    세부 정보 레벨 드롭다운 목록에서 다음 중 하나의 세부 정보 레벨을 선택하여 지정합니다.
    • 간단 - 날짜, 시간, 서비스 유형, 서비스 이름 및 URI 표시
    • 헤더 - 메시지 메타 데이터의 XML 표현과 함께 간단한 정보 표시
    • 전체 - 첨부 파일(존재하는 경우)을 포함하여 원시 페이로드와 함께 헤더 정보 표시
    세부 정보 레벨 드롭다운 목록에서 전체를 선택한 경우, 페이로드 추적 제한 필드를 사용하여 메시지 페이로드에 대한 최대 크기(KB)를 지정합니다. 또한, 기본 인코딩 필드를 사용하여 페이로드 기록에 대한 기본 인코딩을 지정합니다. 이것은 이진 페이로드 또는 이진 첨부 파일이 있는 SOAP 메시지를 기록할 때 유용합니다.
    기본 인코딩 값은 다음에 정의된 Base64나 Java 지원 인코딩이 될 수 있습니다. http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html
    기본 인코딩 필드를 비워 두면 Oracle Service Bus에서 페이로드에 호스트의 기본 인코딩을 사용할 수 있습니다. 기본 인코딩은 JVM, 기본 운영 체제(OS) 및 OS 레벨 로케일 설정의 조합에 따라 달라집니다.
    기본 인코딩 필드에 지정된 설정을 사용할 수 없는 경우(예: 구성에 적합한 옵션이 아닌 경우)에는 Oracle Service Bus에서 페이로드에 Base64 인코딩을 사용합니다.
    비즈니스 서비스에 대한 메시지 추적을 사용 안함으로 설정
    메시지 추적 옆에 있는 사용으로 설정됨 체크 박스를 선택 취소합니다.
    결과 캐싱 사용으로 설정/사용 안함으로 설정
    사용으로 설정됨을 선택하면 비즈니스 서비스에 대한 결과 캐싱이 활성화됩니다. 또한 결과 캐싱이 전체적으로 사용으로 설정되었는지 확인해야 합니다. 결과 캐싱을 전체적으로 사용으로 설정을 참조하십시오.
    [사용으로 설정됨] 옵션을 선택 해제하고 갱신을 눌러 비즈니스 서비스에 대한 결과 캐싱을 사용 안함으로 설정하면 해당 비즈니스 서비스에 대해 캐시에 저장된 모든 결과가 비워집니다.
