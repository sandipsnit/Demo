비즈니스 서비스 생성 및 구성

이 섹션에서는 비즈니스 서비스를 생성 및 구성하는 방법에 대해 설명합니다.

JCA 서비스의 경우 JCA 바인딩(JCA 바인딩 리소스에서 비즈니스 서비스 생성 참조) 또는 Oracle Enterprise Repository(Oracle Enterprise Repository에서 비즈니스 서비스 생성 참조)에서 비즈니스 서비스를 생성할 수 있습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프로젝트 탐색기를 선택한 다음 비즈니스 서비스에 추가할 프로젝트 또는 폴더를 선택합니다.
  3. 프로젝트/폴더 뷰 페이지의 리소스 생성 드롭다운 목록에서 비즈니스 서비스를 선택합니다.
  4. 비즈니스 서비스 생성/편집 - 일반 구성 페이지에서 서비스에 대한 이름을 제공하고 생성할 서비스의 유형을 선택합니다.

    리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
  5. 다음을 누릅니다. 다음에 오는 페이지는 첫번째 페이지에서 선택한 사항에 따라 달라집니다. 요약 페이지에 도달할 때까지 각 후속 페이지에 적절한 정보를 입력한 다음 저장을 눌러 현재 세션의 서비스를 저장합니다.

    6. 각 페이지 작성에 대한 자세한 지침은 다음을 참조하십시오.

    • 일반 구성 페이지
    • 메시지 유형 구성 페이지
    • 전송 구성 페이지
    • 프로토콜별 전송 구성 페이지
    • SOAP 바인딩 구성 페이지
    • 메시지 처리 구성 페이지
    • 요약 페이지
    • 비즈니스 서비스 정책 페이지
    • 비즈니스 서비스 보안 정책 페이지

WS 정책 첨부 파일이 포함된 WSDL에서 비즈니스 서비스를 생성한 경우 프로토콜별 전송 구성 페이지에 정책(읽기 전용)이 표시됩니다. 서비스의 WS 정책에서 인증을 지정하면 서비스 계정을 선택해야 합니다. 이 비즈니스 서비스로 경로 지정되는 프록시 서비스는 이 서비스 계정을 사용하여 비즈니스 서비스에 인증됩니다.

JCA 바인딩 리소스에서 비즈니스 서비스 생성

Oracle Service Bus의 아웃바운드 JCA 바인딩 리소스에서 JCA 비즈니스 서비스를 생성할 수 있습니다. 자세한 내용은 12장, JCA 바인딩을 참조하십시오.

JCA 바인딩에서 JCA 비즈니스 서비스를 생성하려면 다음을 수행합니다.

  1. 생성 또는 편집 모드가 아닐 경우 Oracle Service Bus 관리 콘솔의 변경 센터에서 생성 또는 편집을 누릅니다.
  2. 리소스 브라우저에서 JCA 바인딩을 누릅니다.
  3. 서비스를 생성할 JCA 바인딩을 찾아 작업 아이콘을 누릅니다.
  4. 나타나는 창에서 생성할 WSDL과 서비스 이름을 확인하고 이러한 새 리소스의 위치를 선택한 다음 생성을 누릅니다.

    5. Oracle Service Bus가 서비스와 해당 WSDL을 생성합니다.

  5. 생성된 서비스에 대한 기타 구성 세부 정보(예: 끝점 URI)를 적절히 수정합니다.

Oracle Enterprise Repository에서 비즈니스 서비스 생성

Oracle Service Bus 개발 환경을 사용하여 Oracle Enterprise Repository에서 비즈니스 서비스를 생성할 수 있습니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1797에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Generating a Business Service from Oracle Enterprise Repository"를 참조하십시오.
