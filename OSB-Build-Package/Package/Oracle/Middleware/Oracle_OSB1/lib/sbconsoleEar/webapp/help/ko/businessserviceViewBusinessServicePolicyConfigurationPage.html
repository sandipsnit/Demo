비즈니스 서비스 정책 페이지

비즈니스 서비스 보기 - 정책 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 보기: 구성 세부 정보 페이지에 설명된 것처럼 비즈니스 서비스를 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 서비스 유형이 WSDL 기반 XML 웹 서비스, 메시징 서비스, 모든 SOAP 서비스 또는 모든 XML 서비스인 비즈니스 서비스에 대한 정책 설정을 구성할 수 있습니다. 표 18-23에서는 이 페이지를 사용하는 방법에 대해 설명합니다. OWSM 정책을 WSDL 기반 XML 웹 서비스, 메시징 서비스 또는 모든 XML 서비스 프록시 서비스에서 사용하려면 프로토콜이 HTTP로 설정되어야 합니다.

WSDL 기반 서비스의 경우 서비스에 바인딩된 WLS 9.2 정책은 효과적인 WSDL에 노출(인라인)됩니다. 추상적인 정책은 인라인되기 전에 미리 처리됩니다. OWSM 정책은 효과적인 WSDL에 인라인되지 않고 참조로 바인딩됩니다.

서비스 정책 구성
[서비스 정책 구성] 필드의 옵션은 다음과 같습니다.

OWSM 정책

  • OWSM 정책 저장소에서 - 정책은 Oracle Enterprise Manager로 관리되는 중앙 Oracle Web Services Manager 정책 저장소에서 정의됩니다. 이러한 정책에는 WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken 프로파일 및 기타 산업 표준이 지원됩니다.

    이 옵션을 사용하면 미리 정의된 목록에서 정책을 추가할 수 있습니다. 목록을 표시하려면 [서비스 레벨 정책]에서 [추가]를 누릅니다. 필요한 경우 필터링을 사용하여 원하는 정책을 찾을 수도 있습니다. 비즈니스 서비스의 경우 클라이언트측 정책만 표시됩니다.

    비즈니스 서비스 생성에 사용되는 WSDL에 표준 WS 보안 정책이 포함되어 있을 경우 Oracle Service Bus에서 오류를 발생시키고 충돌을 표시합니다. 이 충돌을 해결하려면 OWSM 정책 저장소에서를 선택하고 호환되는 항목 추가를 눌러 포함된 WSDL 정책을 호환되는 Oracle Web Services Manager 정책으로 바꿉니다. Oracle Service Bus는 WSDL에 포함된 표준 정책을 기반으로 Oracle Web Services Manager 정책 저장소에서 최대한 가장 일치하는 정책을 찾으려고 시도합니다. 알고리즘은 일치하는 정책을 전혀 반환하지 않을 수도 있고, 한 개 또는 여러 개 반환할 수도 있습니다.

    호환되는 항목 추가를 누른 후 Oracle Service Bus에서 호환되는 정책을 찾지 못할 경우 추가를 누르고 적합한 Oracle Web Services Manager 정책을 선택합니다. Oracle Service Bus가 호환되는 정책을 여러 개 반환할 경우 사용하지 않을 정책을 삭제합니다.

    주: 이 옵션은 Oracle Web Services Manager를 사용하도록 도메인을 구성한 경우에만 활성화됩니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Securing Oracle Service Bus with Oracle Web Services Manager"를 참조하십시오.

WLS 9 정책

  • WSDL에서 - 서비스 정책이 서비스가 기반으로 하는 WSDL과 연결된 경우 이 옵션을 선택합니다. 이러한 정책에서는 WS-Security 1.0, SAML 1.1 및 기타 산업 표준이 지원됩니다.

    이 옵션을 사용할 경우 WSDL의 요청 및 응답 정책을 볼 수 있습니다(읽기 전용).

    주: Oracle Service Bus에서 지원되지 않는 WS 보안 정책에 대한 비즈니스 서비스 구성 오류를 수신하는 경우 [OWSM 정책 저장소에서] 옵션을 사용하여 내장된 WSDL 정책을 호환되는 OWSM 정책으로 바꾸십시오.

  • 미리 정의된 정책 또는 WS 정책 리소스에서 - 서비스 레벨 정책, 작업 레벨 정책(정책이 요청 및 응답 메시지 모두에 적용되는 경우), 요청 정책 및 Oracle Service Bus 관리 콘솔의 응답 정책을 추가하려면 이 옵션을 선택합니다.

    정책은 Oracle WebLogic Server에 미리 정의되거나 Oracle Service Bus에 WS 정책 리소스로 정의됩니다. 이러한 정책에서는 WS-Security 1.0, SAML 1.1 및 기타 산업 표준이 지원됩니다.

    이 옵션을 사용할 경우 서비스, 작업, 요청 및 응답 레벨에서 정책을 추가할 수 있습니다. 목록에서 정책을 선택하려면 해당하는 섹션에서 [추가]를 누릅니다.

주: 정책 바인딩 모델은 상호 배타적입니다. 서비스에서는 한 가지 유형의 정책만 사용해야 합니다. 정책을 서비스에 직접 바인딩하면 모든 WSDL 기반 정책이 무시됩니다.

정책을 추가한 후 보안 페이지에서 우선 적용을 제공할 수 있습니다.
