비즈니스 서비스 보안 정책 페이지

비즈니스 서비스 보기 - 보안 구성 페이지는 비즈니스 서비스 생성 및 구성과 비즈니스 서비스 보기: 구성 세부 정보 페이지에 설명된 것처럼 비즈니스 서비스를 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서 사용 가능한 필드는 비즈니스 서비스에 사용되는 정책에 따라 다릅니다. 예를 들어, 서비스에 OWSM 정책이 사용되는 경우(권장) [보안] 페이지에 [정책 우선 적용]이 나타납니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Securing Oracle Service Bus with Oracle Web Services Manager"를 참조하십시오.

서비스에 바인딩된 WLS 9.2 정책은 효과적인 WSDL에 노출(인라인)됩니다. 추상적인 정책은 인라인되기 전에 미리 처리됩니다. OWSM 정책은 효과적인 WSDL에 인라인되지 않고 참조로 바인딩됩니다.

이 페이지에서는 WSDL 기반 정책이 있거나 사용자 정의 정책 바인딩을 사용하는 비즈니스 서비스의 보안 설정을 구성할 수 있습니다. 표 18-24에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 18-24 비즈니스 서비스 보기 - 보안 페이지
옵션
편집
서비스 계정
찾아보기를 누르고 표시된 목록에서 서비스 계정을 선택합니다. 서비스 계정을 지정하지 않을 경우 익명 제목이 사용됩니다.
주: 사용자 인증서는 우선 적용 키에 의해 제공되므로 Oracle Web Services Manager 정책을 사용하는 서비스에 대해서는 이 옵션을 사용할 수 없습니다.
