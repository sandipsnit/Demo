변경 센터

변경 센터는 Oracle Service Bus에서 서비스와 메타 데이터를 구성하는 동안 팀 공동 작업을 허용합니다. 각 사용자는 작업 구성을 Bus의 코어 구성에 체크인할 준비가 될 때까지 sandbox 세션에서 작업합니다. 변경 센터는 다중 사용자가 구성 작업을 수행하도록 다중 레벨의 취소 및 충돌 표시를 제공합니다.

대부분의 변경 센터 페이지에는 세션을 사용 중인지(즉, 변경 센터에서 편집을 누른 경우) 또는 세션 외부에 있는지에 따라 다른 정보가 표시됩니다. 예를 들어, 세션을 사용 중이면 변경 센터 목록에서 변경 내용 보기를 누를 때 표시되는 구성 변경 내용 보기 페이지에 해당 세션에서의 모든 변경 내용이 나열되고 세션 외부에 존재하는 경우에는 모든 세션 활성화가 페이지에 나열됩니다.

표 2-1에는 변경 센터 작업 및 관련된 세션 정보가 표시됩니다.

충돌 보기
세션을 사용 중인 경우에만: 이 링크는 세션에서 발생하는 충돌의 수를 표시합니다. 현재 세션의 변경과 Oracle Service Bus 관리 콘솔 내 다른 모든 세션에서의 변경 간 모든 충돌을 보고 해결합니다. 이미 다른 세션에서 수정 및 활성화한 리소스를 현재 세션에서 수정하면 충돌이 발생합니다. 두 세션에서 동일한 리소스에 대해 두 가지 변경을 수행해도 세션 중 하나가 활성화되기 전에는 충돌이 발생하지 않습니다.
이 페이지에는 세션의 리소스에 대한 동작 오류도 표시됩니다.
