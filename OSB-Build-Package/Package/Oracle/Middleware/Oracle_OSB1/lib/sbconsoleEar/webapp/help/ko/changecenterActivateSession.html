세션 활성화

  1. 변경 센터에서 활성화를 누릅니다. 세션 활성화 페이지에는 세션 이름, 사용자 이름 및 설명 필드가 표시됩니다.

    2. 세션 생성과 폐기는 시스템에서의 다른 작업과 관계없이 진행됩니다. 그러나 다른 세션의 활성화 작업이 진행 중인 경우 세션 활성화 페이지에는 사용자에게 보류 중인 Weblogic Server 변경 내용이 있음을 나타내는 오류가 표시됩니다. 해당 세션 및 기타 활성화된 세션에서 적용된 변경 내용 간의 충돌 해결에 대한 내용은 충돌 보기 및 해결을 참조하십시오.

  2. 검증 오류가 있으면 이를 알리는 오류 메시지가 표시됩니다. 먼저 구성 충돌을 보고 해결한 다음 계속합니다. 충돌 해결에 대한 자세한 내용은 충돌 보기 및 해결을 참조하십시오.

    3. 모든 충돌을 확인할 때까지 세션을 활성화할 수 없습니다. 기존 충돌을 보는 동안 새로운 충돌이 발생하면 활성화하기 전에 새 충돌을 알리는 메시지가 팝업됩니다.

  3. 설명 필드에 설명을 입력합니다. 이 설명은 변경 센터의 변경 내용 보기를 눌러 세션 활성화로 인한 구성 변경 내용을 표시할 때 설명 열에 표시됩니다.
  4. 세션을 활성화하려면 제출을 누릅니다. 이 작업 동안 새로운 충돌이 발생하지 않으면 세션이 종료되고 구성이 런타임에 배치됩니다.
주: 다른 서버(작업 중인 서버 이외의 단일 서버 또는 클러스터의 관리 서버)에서 JMS 끝점 URI가 있는 세션을 활성화하는 경우 대상 서버가 사용 가능한지 확인합니다.
주: Oracle Service Bus는 JMS 끝점 URL이 접근할 수 없는 대상을 지정하는 경우 JMS 전송이 있는 프록시 서비스를 등록할 수 없습니다. 즉, JMS 서비스에서 Oracle Service Bus는 지정된 접속 팩토리가 존재하는지 여부를 확인하며, 없는 경우에는 세션 활성화 오류가 발생합니다.
