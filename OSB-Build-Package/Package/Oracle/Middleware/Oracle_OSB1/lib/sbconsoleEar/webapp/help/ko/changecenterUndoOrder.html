작업 실행 취소 순서

개별 실행 취소 작업이 적합한 데이터를 생성하는 경우 순서에 관계없이 작업을 실행 취소할 수 있습니다. 실행 취소 작업은 리소스 값을 해당 리소스의 변경 전 값으로 설정합니다.

실행 취소 중인 작업이 객체를 생성하는 작업인 경우에는 객체를 반환할 수 있는 이전 상태, 즉 이 작업이 수행되기 전에 존재한 객체가 없습니다. 실제로 실행 취소 작업은 새 객체를 세션에서 삭제합니다. 이 경우 삭제 중인 객체를 참조하는 객체에 대해 오류가 발생합니다. 변경 센터의 충돌 보기 페이지에서 이러한 오류를 볼 수 있습니다.

세션에서 작업 중이 아닌 경우 구성 변경 내용 보기 페이지에 액세스하여 이전에 활성화된 세션을 볼 수 있습니다. 이러한 세션을 실행 취소할 수 있습니다. 실행 취소 작업으로 인해 런타임 구성 오류가 발생하는 경우에는 이전에 활성화된 세션을 실행 취소할 수 없습니다. 예를 들어, 세션 활성화 실행 취소로 인해 다른 객체에서 참조하는 객체가 제거되는 경우에는 실행 취소 작업이 허용되지 않습니다.

실행 취소 작업을 실행 취소할 수 있습니다. [작업] 실행 취소의 옵션 열에서 실행 취소 아이콘을 누릅니다. Oracle Service Bus는 실행 취소를 무제한으로 지원합니다. 따라서 이전에 실행 취소한 작업도 얼마든지 계속 실행 취소할 수 있습니다.
