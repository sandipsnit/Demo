사용자 환경설정 설정

이 페이지에서는 Oracle Service Bus 관리 콘솔 표시 옵션 및 설정을 사용자 정의하고 유지할 수 있습니다. 변경 세션을 활성화하지 않고도 사용자 환경설정을 갱신(변경)할 수 있습니다.

표 26-7에서는 페이지의 옵션 및 콘트롤에 대해 설명합니다.

표 26-7 사용자 환경설정 - 옵션 및 콘트롤
수행할 작업
작업 단계
Oracle Service Bus 관리 콘솔을 시작할 때 열리는 페이지인 홈 페이지 변경
첫번째 드롭다운 목록의 다음 기본 메뉴 항목인 작업, 리소스 브라우저, 프로젝트 탐색기, 보안 구성 또는 시스템 관리 중에서 선택합니다.
두번째 드롭다운 목록에서 하위 메뉴 항목 중 하나를 선택합니다. 하위 메뉴의 콘텐츠는 첫번째 메뉴 선택에 따라 동적으로 변경됩니다.
검색 필터 표시
예 또는 아니오를 선택합니다.
기본적으로 검색 필터는 표시되지 않습니다(닫혀 있음). 예를 선택하면 검색 필터가 항상 표시됩니다(열려 있음).
단계 주석 표시
예 또는 아니오를 선택합니다.
기본적으로 파이프 라인 단계 주석은 표시되지 않습니다(닫혀 있음). 예를 선택하면 단계 주석이 항상 표시됩니다(열려 있음).
리소스 메타 데이터 표시
예 또는 아니오를 선택합니다.
기본적으로 리소스 메타 데이터 선택은 표시됩니다(열려 있음). 아니오를 선택하면 리소스 메타 데이터 선택이 표시되지 않습니다(닫혀 있음).
Oracle Service Bus 관리 콘솔이 대시보드의 데이터 표시를 갱신하는 빈도 변경
대시보드 새로 고침 속도 드롭다운 목록에서 새로 고침 없음, 1, 2, 3, 5, 10, 20, 30 또는 60분을 대시보드 페이지의 새로 고침 빈도로 선택합니다.
기본값은 새로 고침 없음이지만 미리 정의된 다른 시간을 선택할 수 있습니다. 예를 들어, 5분을 선택하면 대시보드가 5분마다 데이터를 갱신합니다.
주의: 기본값(새로 고침 없음) 이외의 다른 대시보드 새로 고침 빈도를 선택하면 브라우저가 정기적으로 대시보드를 새로 고칩니다. 이렇게 하면 Oracle Service Bus 관리 콘솔과의 상호 작용이 없는 경우에도 세션 시간 초과가 발생하지 않습니다.
주의: 대시보드 새로 고침 빈도 기능은 주의하여 사용하는 것이 좋습니다. Oracle Service Bus 관리 콘솔 세션을 무인 상태로 두지 마십시오. 세션이 활성 상태인 동안 다른 사용자가 재인증 없이 시스템에서 액세스 권한을 얻을 수 있습니다.
대시보드의 경보 내역 데이터를 표시할 시간 간격 변경
경보 내역 기간 드롭다운 목록에서 30분, 1, 2, 3 또는 6시간을 경보 내역 데이터를 표시할 시간 간격으로 선택합니다. 기본값은 30분이지만(최근 30분 내에 받은 경보 표시) 미리 정의된 다른 기간을 선택할 수 있습니다. 예를 들어, 1시간을 선택한 경우 테이블에 최근 한 시간 내에 받은 경보가 표시됩니다.
