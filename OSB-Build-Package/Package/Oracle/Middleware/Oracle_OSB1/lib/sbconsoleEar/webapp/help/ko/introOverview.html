Oracle Service Bus 소개

Oracle Service Bus는 구성을 조건으로 하는 정책 기반의 ESB(Enterprise Service Bus) 제품입니다. 이 제품은 다른 유형의 IT 환경에서 확장성 및 안정성이 높은 서비스 지향 통합, 서비스 관리 및 기존의 메시지 중개를 제공합니다. 또한 통합 소프트웨어 제품의 서비스 모니터링 및 관리와 함께 지능적인 메시지 중개를 메시지 경로 지정 및 변환과 결합합니다. Oracle Service Bus 관리 콘솔을 사용하면 서비스 및 정책 구성을 제어하고 시스템 및 운영 작업을 모니터할 수 있습니다. Oracle Service Bus는 Oracle WebLogic Server 런타임 기능을 사용합니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBCA에서 Oracle Fusion Middleware Concepts and Architecture for Oracle Service Bus를 참조하십시오.

리소스 이름 지정 제한 사항

Oracle Service Bus 구성에서 디렉토리 또는 리소스의 이름을 지정할 때는 다음 문자가 허용됩니다.

    • Java Language Specification(http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#3.8)의 "Identifiers" 및 "Keywords" 단원에서 설명되는 Java 키워드를 포함한 모든 Java 식별자

    • 이름 사이의 공백, 마침표 및 하이픈(시작 또는 끝 문자로는 사용될 수 없음)

/ \ * : " < > ? | 등의 문자는 허용되지 않습니다.
