JCA 바인딩 추가

Oracle Service Bus에서 JCA 바인딩을 생성하는 가장 간편한 방법은 이전에 생성한 JCA 리소스(.jca 파일, 연관된 WSDL 및 TopLink 또는 EclipseLink 매핑 파일)를 대량 임포트하는 것입니다. 대량 임포트 시 Oracle Service Bus는 자동으로 .jca 파일에서 JCA 바인딩을, 연관된 JCA WSDL에서 WSDL 리소스를, 매핑 파일에서 XML 문서 리소스를 생성하고 파일 간의 종속성을 유지합니다.

임포트에 대한 자세한 내용은 Zip 파일에서 리소스 로드, URL에서 리소스 로드 및 리소스 임포트를 참조하십시오.

JCA 리소스를 임포트한 후에는 JCA 바인딩 리소스에서 프록시 서비스 생성 및 JCA 바인딩 리소스에서 비즈니스 서비스 생성에 설명된 대로 리소스에서 프록시 또는 비즈니스 서비스를 생성할 수 있습니다.

JCA 리소스를 대량 임포트하지 않을 경우 다음 단계에 설명된 대로 수동으로 JCA 바인딩을 생성할 수 있습니다.

  1. Oracle에서 JDeveloper는 JCA 서비스 WSDL, 해당 JCA(.jca) 파일을 생성하고, 필요한 경우 TopLink 또는 EclipseLink 매핑 파일 및 XSLT 파일과 같은 기타 리소스를 생성합니다.

    2. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=TKADP에서 Oracle Fusion Middleware User's Guide for Technology Adapters를 참조하십시오.

  2. 아직 이 작업을 수행하지 않은 경우 생성를 눌러 OSB 관리 콘솔에서 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  3. OSB 관리 콘솔에서는 Oracle JDeveloper에서 생성한 JCA 리소스(예: WSDL, XML 및 XSLT 리소스)를 생성하거나 임포트합니다.
  4. JCA 바인딩을 생성합니다.
    1. 프로젝트 탐색기를 선택한 다음 JCA 바인딩에 대한 프로젝트 또는 폴더 위치를 선택합니다. 그러면 프로젝트/폴더 뷰 페이지가 표시됩니다.
    2. 리소스 생성 필드에서 JCA 바인딩을 선택하여 새 JCA 바인딩 리소스 생성 페이지를 표시합니다.
    3. JCA 파일 필드에서 찾아보기를 눌러 JCA 파일을 선택합니다.

      4. [리소스 이름] 필드에 자동으로 파일 이름(파일 확장자 제외)이 채워집니다.

      파일을 선택하는 대신 [JCA 파일] 필드 텍스트 영역에서 JCA 파일 내용을 복사하여 붙여 넣을 수도 있습니다.

    4. 리소스 이름 필드에서 JCA 바인딩의 이름을 입력하거나 변경합니다. JCA 바인딩 이름은 고유해야 합니다.

      리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
    5. 리소스 설명 필드에 이 JCA 바인딩에 대한 설명을 입력합니다.
    6. 저장을 누릅니다. 그러면 현재 세션에 JCA 바인딩이 저장됩니다.

      7. JCA 바인딩이 필요한 리소스(예: WSDL 및 매핑 파일)를 참조하지 않을 경우 검증 경고가 표시됩니다.

  5. 새 JCA 바인딩의 이름을 눌러 JCA 바인딩 보기 페이지에 표시합니다.
  6. 참조 편집을 누릅니다.
  7. JCA 바인딩 리소스의 참조 편집 페이지에서 찾아보기를 사용하여 WSDL, TopLink/EclipseLink 매핑 파일 및 XSLT 파일과 같은 JCA 바인딩의 참조 리소스를 선택합니다.
  8. 저장을 누릅니다.
  9. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.

JCA 바인딩 리소스를 추가하고 WSDL과 연관된 리소스 종속성을 구성한 후 또는 종속성이 유지되는 JCA 리소스를 임포트한 경우 JCA 바인딩에서 프록시 및 비즈니스 서비스를 생성할 수 있습니다. 자세한 내용은 JCA 바인딩 리소스에서 프록시 서비스 생성 및 JCA 바인딩 리소스에서 비즈니스 서비스 생성을 참조하십시오.
