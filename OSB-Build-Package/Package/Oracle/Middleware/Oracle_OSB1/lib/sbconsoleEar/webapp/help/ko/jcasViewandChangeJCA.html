JCA 바인딩 편집

JCA 바인딩 세부 정보 보기 페이지에서는 JCA 바인딩의 세부 정보를 확인하고 변경할 수 있습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. JCA 바인딩 찾기에 설명된 대로 JCA 바인딩을 찾습니다.
  3. JCA 바인딩 이름을 누릅니다. 그러면 JCA 바인딩 보기 페이지에 표 12-2와 같은 정보가 표시됩니다.

    4. 표 12-2 JCA 바인딩 세부 정보 
    속성
    설명
    마지막으로 수정한 사람
    이 JCA 바인딩을 생성하거나 구성으로 임포트한 사용자입니다.
    마지막 수정 시간
    사용자가 이 JCA 바인딩을 생성했거나 구성으로 임포트한 날짜와 시간입니다. 이 리소스의 변경 내역을 보려면 날짜 및 시간 링크를 누릅니다. 변경 내역 보기 페이지를 참조하십시오.
    참조
    JCA 바인딩이 참조하는 WSDL 리소스에 대한 링크를 제공합니다. 리소스에 대한 참조 보기를 참조하십시오.
    참조자
    이 JCA 바인딩을 참조하는 객체의 수입니다. 해당 참조가 존재할 경우 객체 목록을 보려면 숫자 링크를 누릅니다. 리소스에 대한 참조 보기를 참조하십시오.
    작업 아이콘
    편집 모드에서 변경 센터를 사용할 경우 작업 아이콘을 눌러 JCA 바인딩에서 Oracle Service Bus WSDL 및 JCA 서비스를 생성할 수 있습니다.
    JCA 파일
    JCA 바인딩을 읽기 전용 뷰로 표시합니다.
    추상적인 WSDL
    JCA 바인딩의 연관된 WSDL을 표시합니다. JCA 바인딩이 WSDL을 참조하지 않을 경우 연관된 WSDL을 참조하려면 참조 편집을 누르십시오.
    TopLink XML
    JCA 바인딩의 필요한 TopLink 또는 EclipseLink 매핑 파일을 표시합니다. JCA 바인딩 리소스에서 필요한 매핑 파일을 참조하지 않는 경우 참조 편집을 눌러서 필요한 매핑 파일을 선택합니다. 이 필드는 JCA 파일이 XML 리소스를 선언하는 경우에만 나타납니다.
    XSLT
    JCA 바인딩의 필요한 XSLT 파일을 표시합니다. JCA 바인딩 리소스에서 필요한 XSLT 리소스를 참조하지 않는 경우 참조 편집을 눌러서 필요한 XSLT 파일을 선택합니다. 이 필드는 소켓 어댑터와 같이 JCA 파일이 XSLT 리소스를 선언하는 경우에만 나타납니다.

  4. 설명 또는 JCA 파일을 수정하려면 편집을 누릅니다.

    5. 리소스 이름은 변경할 수 없습니다.

  5. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  6. JCA 바인딩의 필요한 리소스에 대한 참조를 변경하거나 확인되지 않은 참조를 수정하려면 참조 편집을 누르고 적합한 리소스를 선택합니다.
  7. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
