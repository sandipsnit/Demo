경보 규칙 조건 정의

이 페이지에서는 경보 규칙 조건을 정의할 수 있습니다. 조건을 한 개 이상 지정해야 합니다. 다중 조건을 지정하는 경우 And/Or 연산자를 사용하여 조건을 조합해야 합니다.

  1. 조건에 대한 집계 간격 선택 필드에서 시간과 분을 선택하여 이 조건에 대한 집계 간격을 설정합니다. 집계 간격은 조건이 테스트되는 빈도를 결정합니다. 조건은 모니터링 하위 시스템이 하나의 집계 간격을 구성하기에 충분한 데이터 샘플을 집계할 때마다 테스트됩니다.

    2. 예를 들어, 집계 간격으로 1시간을 선택하면 한 시간에 해당하는 데이터를 사용할 수 있을 때마다 조건이 테스트됩니다. 최초의 1시간이 끝날 때 처음으로 조건이 테스트됩니다. 집계 간격 1시간의 샘플링 간격이 10분으로 설정되었으므로 그 후에는 10분마다 조건이 테스트됩니다.

  2. 먼저 단순 표현식을 정의합니다. 둘 이상의 단순 표현식을 조합하여 복합 표현식을 구성할 수 있습니다. 복잡한 단순 표현식을 정의하려면 다음 작업을 수행합니다.
    1. 첫번째 드롭다운 필드에서 개수, 최소, 최대, 평균 또는 상태를 선택합니다.
    2. 다음 드롭다운 필드에서 피연산자를 선택합니다.

      3. 첫번째 드롭다운 필드에서 선택한 개수, 최소, 최대, 평균 또는 상태에 따라 피연산자 목록이 달라집니다. 예를 들어 최소, 최대 또는 평균을 선택한 경우 응답 시간 피연산자를 사용할 수 있습니다. 이 피연산자를 사용하면 응답 시간을 밀리초(msec) 단위로 설정할 수 있습니다. 또한 사용 가능한 피연산자는 서비스 자체의 구성에 따라 결정됩니다. 피연산자 수는 서비스에 파이프 라인, 경로 지정 노드, 작업 등이 있는지 여부에 따라 달라집니다.

      끝점 URI 상태를 기반으로 경보를 생성하는 방법에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG595에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Managing Endpoint URIs for Business Services"를 참조하십시오.

      개수를 선택할 경우 이 필드에는 표 25-21에 나열된 피연산자가 표시됩니다.

      표 25-21 개수 피연산자 세부 정보
      속성
      설명
      캐시 적중 수
      결과 캐싱을 사용하는 비즈니스 서비스의 경우 응답을 클라이언트에 반환하기 위해 캐시를 사용할 때마다 캐시 적중 수가 증분됩니다.
      오류 횟수
      메시지 처리에서 실패를 반환하면 오류 횟수가 증가합니다.
      실패율(%)
      지정한 집계 간격 동안 발생한 총 오류 수와 성공적으로 처리된 메시지 수의 비율입니다.
      메시지 수
      처리된 메시지의 총 수입니다.
      성공률(%)
      지정한 집계 간격 동안 발생한 메시지의 총 수와 성공적으로 처리된 메시지 수의 비율입니다.
      장애 조치 수
      비즈니스 서비스에만 해당: 장애 조치 수입니다.
      <요청 파이프 라인>.
      오류 횟수
      프록시 서비스에만 해당: 요청 파이프 라인에 의해 처리된 오류 메시지의 수입니다.
      <요청 파이프 라인>.
      메시지 수
      프록시 서비스에만 해당: 요청 파이프 라인에 의해 처리된 메시지의 수입니다.
      확인 오류 횟수
      프록시 서비스의 경우 파이프 라인에 확인 작업이 존재하는 경우에만: 확인 오류 수입니다.
      WSS 오류 횟수
      서비스에 대한 전송(HTTP에서 등)에 따라 사용 가능: 처리된 웹 서비스 보안(WSS) 오류 메시지의 수입니다. 이 카운터는 WSDL 기반 서비스에만 사용할 수 있으며 WSS 오류가 발생할 때 갱신됩니다.

    3. 다음 필드에서 원하는 비교 연산자(=, !=, > 또는 <)를 선택합니다.
    4. 다음 필드에 비교할 값을 입력합니다.
    5. 추가를 누릅니다.
  3. 복합 표현식을 정의하려면 다음을 수행합니다.
    1. 2단계의 a - e단계를 반복하여 단순 표현식을 정의합니다.
    2. 다시 a - e단계를 반복하여 다른 단순 표현식을 정의합니다.
    3. 두 표현식의 각각에 대해 체크 박스를 선택합니다.
    4. And 또는 Or을 눌러 두 표현식을 하나의 복합 표현식으로 조합합니다.

      5. 다중 표현식을 조합하면 분할 옵션을 사용할 수 있습니다. 이 옵션은 복합 표현식을 다시 별개의 단순 표현식으로 분할하는 데 사용됩니다.
