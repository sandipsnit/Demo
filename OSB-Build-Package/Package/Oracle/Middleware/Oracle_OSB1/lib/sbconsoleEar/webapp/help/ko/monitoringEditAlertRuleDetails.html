경보 규칙에 대한 일반 정보 구성

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 리소스 브라우저 > 프록시 서비스 또는 비즈니스 서비스를 선택합니다.
  3. 서비스 이름을 누릅니다.
  4. SLA 경보 규칙 탭을 선택하고 새 항목 추가를 누릅니다.
  5. 규칙 이름 필드에 경보 규칙의 이름을 입력합니다. 이 필드는 필수 필드입니다.
  6. 경보 요약 필드에 경보 규칙에 대한 간단한 설명을 입력합니다. 이 텍스트는 경보 규칙이 전자 메일 통지와 함께 구성된 경우 전자 메일 메시지의 제목 줄로도 사용되며 80자를 초과할 수 없습니다. 경보 요약을 제공하지 않는 경우 기본 텍스트인 Oracle Service Bus 경보가 대신 사용됩니다.
  7. 규칙 설명 필드에 경보 규칙에 대한 설명을 입력합니다.
  8. 경보 대상 필드에서 경보 대상 리소스 이름을 입력하거나 찾아보기를 눌러 경보 대상 선택 페이지를 표시합니다. 목록에서 경보 대상을 선택하고 제출을 누릅니다.

    9. 기본적으로 경보는 항상 Oracle Service Bus 관리 콘솔로 전달됩니다. 기타 경보 대상 리소스를 구성하고 선택하려면 경보 대상 추가 및 경보 대상 찾기를 참조하십시오.

  9. 시작 시간 필드에 HH:MM 형식으로 시작 시간을 입력합니다. 예를 들어, 09:00 AM을 입력합니다. 시작 시간 및 종료 시간 필드를 사용하여 만료 날짜까지 규칙이 매일 적용되는 시간 범위를 지정합니다. 예를 들어, 매일 9am에서 5pm까지로 지정할 수 있습니다.
  10. 종료 시간 필드에 HH:MM 형식으로 종료 시간을 입력합니다. 예를 들어, 05:00 PM을 입력합니다. 시작 시간 및 종료 시간 필드를 사용하여 만료 날짜까지 규칙이 매일 적용되는 시간 범위를 지정합니다. 예를 들어, 매일 9am에서 5pm까지로 지정할 수 있습니다.

    11. 경보 규칙은 규칙이 만료될 때까지 매일 지정한 시작 시간부터 지정한 종료 시간까지 적용됩니다.

  11. 규칙 만료 날짜 필드에 MM/DD/YYYY 형식으로 만료 날짜를 입력합니다. 예를 들어, 12/31/2010을 입력합니다. 규칙은 지정한 날짜의 11:59 pm에 만료됩니다. 날짜를 지정하지 않으면 규칙은 만료되지 않습니다.
  12. 규칙을 사용으로 설정 필드에서 기본값 예를 유지하여 이 규칙을 사용으로 설정하거나 아니오를 선택하여 규칙을 사용 안함으로 설정합니다.
  13. 경보 심각도 필드에서 다음 중 하나를 선택합니다.
    • 일반
    • 경고
    • 사소
    • 중요
    • 심각
    • 치명적
  14. 경보 빈도 필드에서 표 25-20과 같은 설정 중 하나를 선택합니다.

    15. 표 25-20 경보 빈도 설정
    속성
    설명
    매번
    이 옵션을 선택하면 경보 규칙이 true가 될 때마다 경보 규칙에 포함된 작업이 실행됩니다. 예를 들어, 평균 응답 시간이 300밀리초보다 크다는 조건을 설정하면 이 조건이 true가 될 때마다 경보를 받습니다.
    경보 규칙이 평가되는 횟수는 해당 규칙과 연관된 집계 간격과 샘플 간격에 따라 결정됩니다. 집계 간격을 5분으로 설정하면 샘플 간격은 1분이 됩니다. 5개 데이터 샘플을 사용할 수 있을 때마다 규칙이 평가됩니다. 규칙이 생성된 지 약 5분 후에 처음으로, 그리고 그 후에 매분 규칙이 평가됩니다.
    한 번 통지
    이 옵션을 선택하면 처음으로 규칙이 true가 될 때 규칙에 포함된 작업이 실행되고 조건이 재설정되어 다시 true가 될 때까지 더 이상 경보가 생성되지 않습니다. 예를 들어, 평균 응답 시간이 300밀리초보다 작다는 조건을 설정하면 처음으로 이 조건이 true가 될 때 경보를 받지만 조건이 false가 된 후 다시 true가 될 때까지 더 이상 경보를 받지 않습니다. 경보 시간 기록이 갱신되고 대시보드에 표시됩니다.

  15. 서비스와 연관된 규칙 중 하나가 true가 된 후 추가 규칙 실행을 중단하려면 추가 규칙 처리 중지 옵션에 대해 예를 선택합니다. 이 옵션을 사용하면 특정 서비스와 연관된 다중 규칙이 존재하는 경우 이후 규칙 평가를 중지할 수 있습니다. 규칙 처리를 계속하려면 기본값 아니오를 유지합니다.

    16. 이 옵션은 Microsoft Outlook의 규칙 마법사에 있는 더 이상 규칙 수행 중지 옵션과 같이 동작합니다.
