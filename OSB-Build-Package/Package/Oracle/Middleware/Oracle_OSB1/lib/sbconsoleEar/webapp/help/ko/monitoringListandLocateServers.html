서버 정보 보기

서버 건전성 페이지는 다음과 같습니다.

  • 건전성 상태 탭 헤더에 종료된 서버 수를 표시하는 종료 아이콘 및 숫자 표시기가 제공될 수 있습니다.
    • 아이콘을 누르면 Shutdown 건전성 조건 및 SHUTDOWN 상태 조건을 사용하여 표시된 서버를 필터링합니다.
    • 숫자는 종료 서버의 수를 나타냅니다.
  • 도메인에서 서버의 건전성을 설명하는 파이 차트입니다. 차트에서 서버에 해당하는 영역을 눌러 세부 정보를 표시할 수 있습니다.
  • 로그 요약입니다. 로그를 표시하려면 제공된 심각도의 메시지 수를 표시하는 숫자 링크를 누릅니다. 예를 들어, 경보: 1의 경우 숫자 1을 누릅니다. 도메인 로그 파일 세부 정보 보기를 참조하십시오.
  • 서버 요약 표에 대한 사용자 정의 설정에 따라 서버 정보를 표시하는 필드입니다. 예를 들어 서버 이름, 건전성, 클러스터 이름, 시스템 이름, 서버 상태 및 서버가 실행된 시간입니다. 이러한 필드에 대해 알아 보려면 표 25-16을 참조하십시오.
  • 기본적으로 대시보드 새로 고침 빈도는 새로 고침 없음입니다. 대시보드 새로 고침 빈도를 변경하려면 사용자 환경설정 설정을 참조하십시오.
주의: 기본값(새로 고침 없음) 이외의 다른 대시보드 새로 고침 빈도를 선택하면 브라우저가 정기적으로 대시보드를 새로 고칩니다. 이렇게 하면 Oracle Service Bus 관리 콘솔과의 상호 작용이 없는 경우에도 세션 시간 초과가 발생하지 않습니다.
주의: 대시보드 새로 고침 빈도 기능은 주의하여 사용하는 것이 좋습니다. 세션을 무인 상태로 두지 마십시오. 세션이 활성 상태인 동안 다른 사용자가 재인증 없이 시스템에서 액세스 권한을 얻을 수 있습니다.
  • 테이블 사용자 정의기 아이콘입니다. 테이블 뷰 사용자 정의를 참조하십시오.
  • 목록의 항목 수를 제한하거나 특정 서비스를 찾으려면 다음을 수행합니다.
    • 열기 아이콘을 눌러 검색 필터를 추가로 표시합니다. 다중 검색 조건을 지정하는 경우에는 모든 조건과 일치하는 결과만 반환됩니다.
    • 건전성 상태로 필터링합니다. 건전성 드롭다운 목록에서 서버를 지정된 상태로 제한하도록 선택한 다음 검색을 누릅니다.

      • 검색을 특정 서버 건전성 레벨 또는 그 이상(가장 포괄적인 레벨에서 가장 제한적인 레벨까지 나열됨: 일반, 경고, 오버로드됨, 심각 및 치명적)으로 제한하려면 이상 체크 박스를 선택합니다.

    • 서버로 필터링합니다. 서버 드롭다운 목록에서 모두 또는 서버 이름을 선택한 다음 검색을 누릅니다.
    • 클러스터 이름으로 필터링합니다. 클러스터 이름 드롭다운 목록에서 모두 또는 서버에 연관된 클러스터 이름을 선택한 다음 검색을 누릅니다.
    • 시스템 이름으로 필터링합니다. 시스템 이름 드롭다운 목록에서 모두 또는 서버에 연관된 시스템 이름을 선택한 다음 검색을 누릅니다.
    • 상태로 필터링합니다. 상태 드롭다운 목록에서 모두 또는 서버 상태(RUNNING, FAILED 또는 SHUTDOWN)를 선택한 다음 검색을 누릅니다.
    • 모두 보기를 눌러 검색 필터를 제거하고 모든 서버를 표시합니다.

서버 건전성 페이지에는 표 25-16에 표시된 것처럼 지정한 필터 설정에 따라 각 서버에 대해 다음 정보가 표시됩니다.

건전성
서버의 건전성 상태입니다.
  • 정상 - 서버가 문제 없이 동작 중입니다.
  • 확인/처리되지 않음 - 서버가 ADMIN 상태에 있고 관리 작업에만 사용할 수 있습니다. 모든 런타임 Oracle Service Bus 메시지가 실패합니다.
  • 경고 - 나중에 서버에 문제가 발생할 수 있습니다. 자세한 내용은 서버 로그와 해당 런타임 MBean을 확인하십시오.
  • 오버로드됨 - 구성된 임계값보다 많은 작업이 서버에 할당되었습니다. 서버에서 추가 작업을 거부할 수도 있습니다.
  • 심각 - 서버 실패가 보류 중입니다. 실패를 방지하려면 즉시 작업을 취해야 합니다. 자세한 내용은 서버 로그와 해당 런타임 MBean을 확인하십시오.
  • 종료 - 서버가 실패했으며 다시 시작해야 합니다.
  • 심각 - 서버 실패가 보류 중입니다. 실패를 방지하려면 즉시 작업을 취해야 합니다. 자세한 내용은 서버 로그와 해당 런타임 MBean을 확인하십시오.
