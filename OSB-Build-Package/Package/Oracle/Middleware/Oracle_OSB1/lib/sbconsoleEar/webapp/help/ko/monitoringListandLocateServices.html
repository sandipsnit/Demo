서비스 모니터링 정보 보기

서비스 건전성 페이지에는 모니터링을 사용으로 설정하는 서비스의 측정 단위가 표시됩니다. 서비스 모니터링을 사용으로 설정하는 방법을 알아보려면 프록시 서비스의 작업 설정 구성 및 비즈니스 서비스의 작업 설정 구성을 참조하십시오.

기본적으로 대시보드 새로 고침 빈도는 새로 고침 없음입니다. 대시보드 새로 고침 빈도를 변경하려면 사용자 환경설정 설정을 참조하십시오.

주의: 기본값(새로 고침 없음) 이외의 다른 대시보드 새로 고침 빈도를 선택하면 브라우저가 정기적으로 대시보드를 새로 고칩니다. 이렇게 하면 Oracle Service Bus 관리 콘솔과의 상호 작용이 없는 경우에도 세션 시간 초과가 발생하지 않습니다.
주의: 대시보드 새로 고침 빈도 기능은 주의하여 사용하는 것이 좋습니다. 세션을 무인 상태로 두지 마십시오. 세션이 활성 상태인 동안 다른 사용자가 재인증 없이 시스템에서 액세스 권한을 얻을 수 있습니다.

서비스 건전성 탭 헤더에 다음 아이콘이 표시될 수 있습니다.

  • 경고 아이콘 및 숫자 표시기입니다.
    • 이 아이콘을 누르면 오류 있음 조건을 사용하여 표시된 서비스가 필터링됩니다.
    • 숫자는 페이지를 마지막으로 새로 고친 이후 오류가 발생한 서비스의 수를 나타냅니다.
  • 끝점 오프라인 아이콘 및 숫자 표시기입니다.
    • 이 아이콘을 누르면 오프라인 끝점 URI 있음 조건을 사용하여 표시된 서비스가 필터링됩니다.
    • 숫자는 페이지를 마지막으로 새로 고친 이후 오프라인 상태인 끝점을 가진 서비스의 수를 나타냅니다.

서비스 건전성 페이지에는 표 25-5에 나열된 서비스 측정 단위 및 모니터링 정보가 표시됩니다.

  1. 통계 표시 필드에서 다음 작업 중 하나를 수행합니다.
    • 현재 집계 간격의 기간에 대해 각 서비스의 모니터링 통계를 표시하려면 현재 집계 간격을 선택합니다. 현재 집계 간격에는 서비스 측정 단위의 이동 통계 보기가 표시됩니다.
    • 서비스에 대한 통계를 마지막으로 재설정한 이후부터 기간에 대한 모니터링 통계를 표시하려면 마지막 재설정 이후를 선택합니다. 마지막 재설정 이후 보기에 실행 중인 측정 단위 수가 표시됩니다.
  2. 드롭다운 목록에서 서버를 선택하여 해당 서버에 대한 측정 단위를 표시할 수 있습니다.

    3. 클러스터가 존재하는 경우, 클러스터 범위 측정 단위가 기본값으로 표시됩니다. 개별 관리 서버를 선택하여 해당 서버에 대한 측정 단위를 표시합니다.

  3. 목록의 항목 수를 제한하거나 특정 서비스를 찾으려면 다음을 수행합니다.
    • 열기 아이콘을 눌러 검색 필터를 추가로 표시합니다. 다중 검색 조건을 지정하는 경우에는 모든 조건과 일치하는 결과만 반환됩니다.
    • 서비스 이름으로 필터링합니다. 이름 필드에 검색 대상 이름을 입력하거나 와일드 카드 문자(* 및 ?를 와일드 카드 문자로 사용하여 보다 일반적인 검색 수행)를 입력한 후 검색을 누릅니다.

      • 이 검색 방법은 서비스 이름이 모든 프로젝트와 경로에서 고유할 경우 사용하는 것이 좋습니다.

    • 서비스 경로를 기준으로 필터링합니다. 경로 필드에 검색 대상(서비스가 있는 폴더의 이름과 프로젝트 이름)의 경로를 입력합니다. * 및 ?를 와일드 카드 문자로 사용하여 보다 일반적인 검색을 수행할 수 있습니다. 해당 경로에 있는 모든 서비스가 표시됩니다.

      • 경로 필드의 형식은 다음과 같습니다.
      프로젝트 이름/루트 폴더/ . . ./상위 폴더

      서비스가 프로젝트 바로 아래에 존재하는 경우 형식은 다음과 같습니다. 프로젝트 이름

    • 서비스 이름과 경로로 필터링합니다. 이 검색 방법은 동일한 이름을 가진 둘 이상의 서비스가 서로 다른 경로에 존재하는 경우 사용하는 것이 좋습니다.
    • 상위 서비스로 필터링합니다. 서비스 드롭다운 목록에서 프록시 서비스 또는 비즈니스 서비스와 같은 상위 서비스 유형을 선택한 다음 검색을 누릅니다. 상위 서비스 필터 조건을 무시하려면 [모든 서비스]를 선택합니다.
    • 메시지가 있는 서비스로 필터링합니다. 메시지 있음 옵션을 선택한 다음 검색을 누릅니다.
    • 경보가 있는 서비스로 필터링합니다. 경보 있음 옵션을 선택한 다음 검색을 누릅니다.
    • 오류가 있는 서비스로 필터링합니다. 오류 있음 옵션을 선택한 다음 검색을 누릅니다.
    • 오프라인 URI를 가진 서비스로 필터링합니다. 오프라인 끝점 URI 있음 옵션을 선택한 다음 검색을 누릅니다.
    • 특정 프록시 서비스가 호출한 서비스로 필터링합니다. 제공된 필드에 프록시 서비스의 이름 및 경로를 입력하거나 찾아보기를 눌러 프록시 서비스 선택 페이지를 표시합니다. 목록에서 프록시 서비스를 선택하고 제출을 누릅니다. 검색을 누릅니다. 지정한 프록시 서비스에 의해 호출된 비즈니스 서비스가 표시됩니다.

      • 와일드카드는 프록시에 의해 호출됨 필드에서는 지원되지 않습니다.

    • 모두 보기를 눌러 검색 필터를 제거하고 모든 서비스를 표시합니다.

      • 표 25-5 서비스 모니터링 정보
      속성
      설명
      이름
      서비스에 할당된 이름입니다. 이름은 서비스 모니터링 세부 정보 페이지에 대한 링크입니다. 서비스 측정 단위 보기를 참조하십시오.
      경로
      서비스와 연관된 프로젝트입니다. 서비스가 프로젝트 폴더에 있으면 이 폴더도 나열됩니다. 경로는 다음 형식으로 표시됩니다.
      프로젝트 이름/루트 폴더/ . . ./상위 폴더
      프로젝트 탐색기의 해당 경로로 링크된 경로입니다.
      서비스 유형
      상위 서비스 유형: 프록시 서비스 또는 비즈니스 서비스입니다.
      집계 간격
      이 필드는 통계 표시 필드의 현재 집계 간격을 선택한 경우에만 표시됩니다.
      이 서비스 모니터링에 대해 설정된 시간과 분 단위의 현재 집계 간격입니다. 이 간격은 작업 설정 페이지에서 설정합니다. 서비스의 집계 간격 설정을 참조하십시오.
      평균 응답 시간
      현재 집계 간격의 기간 내에 또는 마지막 재설정 이후의 기간에 대해 메시지를 실행한 이 서비스의 평균 응답 시간(밀리초)입니다.
      메시지
      현재 집계 간격의 기간 또는 마지막 재설정 이후의 기간에 대해 이 서비스와 연관된 메시지 수입니다.
      오류
      현재 집계 간격의 기간 또는 마지막 재설정 이후의 기간에 대해 이 서비스와 연관된 오류 메시지 수입니다.
      경보
      이 필드는 기본적으로 숨겨져 있습니다.
      현재 집계 간격의 기간 또는 마지막 재설정 이후의 기간에 대해 이 서비스와 연관된 경보(SLA 및 파이프 라인) 수입니다.
      프록시 서비스의 경우, 클러스터의 개별 관리 서버에 대해 현재 집계 간격 기간 또는 마지막 재설정 이후의 기간 동안 이 서비스와 연관된 파이프 라인 경보 수만 표시됩니다.
      비즈니스 서비스의 경우 개별 관리 서버에 대해 N/A가 표시됩니다.
      SLA 경보
      현재 집계 간격의 기간 또는 마지막 재설정 이후의 기간에 대해 이 서비스와 연관된 SLA 경보 수입니다.
      클러스터의 개별 관리 서버에 대해 SLA 경보는 N/A입니다(클러스터 범위 SLA 경보만 표시).
      파이프 라인 경보
      프록시 서비스에만 해당:
      현재 집계 간격의 기간 또는 마지막 재설정 이후의 기간에 대해 이 서비스와 연관된 파이프 라인 경보 수입니다.
      끝점 URI 상태
      비즈니스 서비스에만 해당하는 끝점 URI 상태입니다. 상태는 끝점 URI 페이지에 대한 링크입니다. 비즈니스 서비스 끝점 URI 측정 단위 보기를 참조하십시오.
      단일 노드 도메인(개별 서버 인스턴스):
      • 온라인 - 비즈니스 서비스의 모든 끝점 URI가 온라인 상태입니다.
      • 오프라인 - 비즈니스 서비스의 모든 끝점 URI가 오프라인 상태입니다.
      • 부분 - 비즈니스 서비스의 하나 이상의 끝점 URI가 오프라인 상태입니다.
      클러스터화된 도메인:
      • 서버 필드가 클러스터 또는 관리 서버 중 하나로 설정되어 존재하는 경우 온라인 상태는 클러스터 또는 선택된 관리 서버에서 모든 끝점 URI가 각각 온라인 상태임을 나타냅니다.
      • 서버 필드가 클러스터 또는 관리 서버 중 하나로 설정되어 존재하는 경우 오프라인 상태는 클러스터 또는 선택된 관리 서버에서 모든 끝점 URI가 각각 오프라인 상태임을 나타냅니다.
      • 서버 필드가 클러스터로 설정되어 존재하는 경우 부분 상태는 비즈니스 서비스의 하나 이상의 끝점 URI가 하나 이상의 서버에서 오프라인 상태이지만, 동일한 비즈니스 서비스의 다른 끝점 URI는 하나 또는 모든 서버에서 사용할 수 있음을 나타냅니다.
      • 서버 필드가 관리 서버 중 하나로 설정되어 존재하는 경우 부분 상태는 선택된 관리 서버에서 비즈니스 서비스의 하나 이상의 끝점 URI가 오프라인 상태임을 나타냅니다.
      작업
      이 필드는 통계 표시 필드의 마지막 재설정 이후를 선택한 경우에만 표시됩니다.
      이 열에서 특정 서비스의 재설정 아이콘을 눌러 해당 서비스의 통계를 재설정할 수 있습니다. 이 작업을 수행할 것인지 확인하는 경우, 시스템이 통계를 마지막으로 재설정한 이후 서비스에 대해 수집된 모든 모니터링 통계를 삭제합니다. 그러나 해당 서비스의 현재 집계 간격 중에 수집된 통계는 삭제하지 않습니다. 또한 재설정 아이콘을 누르면 시스템은 해당 서비스의 모니터링 통계를 다시 즉시 수집하기 시작합니다.
