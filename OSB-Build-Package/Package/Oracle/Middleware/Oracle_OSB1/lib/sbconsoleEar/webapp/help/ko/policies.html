사용자 정의 WS 정책

주: Oracle Service Bus의 이후 버전에서는 WLS 9.2 정책 대신 Oracle Web Services Manager 정책이 사용될 예정입니다. 현재 버전의 Oracle Service Bus에서는 WLS 9.2 정책이 계속 지원되지만, 앞으로는 WLS 9.2 정책에 대한 지원이 중단될 예정으로 새 서비스를 생성할 때는 Oracle Web Services Manager 정책을 사용하는 것이 좋습니다. 자세한 내용은 http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Securing Oracle Service Bus with Oracle Web Services Manager"를 참조하십시오.

웹 서비스 정책 프레임워크(WS 정책)는 도메인별 보안 검증을 사용하여 웹 서비스 구성을 확장하고 웹 서비스의 보안 요구 사항, 필수 사항 및 기능을 지정하는 확장 가능한 XML 기반 프레임워크입니다. Oracle Service Bus에서 WS 정책은 프록시 서비스 및 비즈니스 서비스에서 메시지 레벨 보안을 구성하는 데 주로 사용됩니다.

Oracle Service Bus WS 정책 사용

Oracle Service Bus에는 클라이언트 인증, 디지털 암호화 또는 디지털 서명 SOAP 메시지를 요청하는 데 사용할 수 있는 3개의 간단한 WS 정책 파일(Auth.xml, Encrypt.xml 및 Sign.xml)이 포함됩니다. 특정한 보안이 필요하지 않으면 미리 패키지화된 이러한 정책 파일을 사용하는 것이 좋습니다.

이러한 정책 사용에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Using WS-Policy in Oracle Service Bus Proxy and Business Services"를 참조하십시오.

사용자 정의 WS 정책 사용

Oracle Service Bus WS 정책 문이 보안 요구에 맞지 않는 경우에는 자체 WS 정책(사용자 정의 WS 정책)을 작성하여 Oracle Service Bus로 임포트해서 WSDL에서 참조할 수 있습니다. Oracle Service Bus WS 정책 문은 읽기 전용입니다.

사용자 정의 WS 정책 생성 및 참조에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Using WS-Policy in Oracle Service Bus Proxy and Business Services"를 참조하십시오.
