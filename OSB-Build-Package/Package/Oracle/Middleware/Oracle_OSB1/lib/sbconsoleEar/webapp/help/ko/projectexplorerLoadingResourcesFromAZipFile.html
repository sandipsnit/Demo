Zip 파일에서 리소스 로드

로드할 리소스가 있는 Zip 또는 JAR 파일을 선택할 수 있기 때문에 Oracle Service Bus는 이 Zip 파일에서 리소스를 찾아 임포트합니다.

WSDL 또는 XML 스키마와 같은 리소스는 종속 리소스의 전체 트리를 정의할 수 있습니다. 일괄 로드 기능을 사용하면 리소스 세트를 한 번에 업로드할 수 있기 때문에 종속 리소스를 수동으로 확인할 필요가 없습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프로젝트 탐색기를 선택한 다음 리소스를 추가할 프로젝트 또는 폴더를 선택합니다. 프로젝트/폴더 뷰 페이지가 표시됩니다.
  3. 리소스 생성 드롭다운 목록에서 압축된 리소스를 선택합니다. 리소스 로드 페이지가 표시됩니다.
  4. Zip 파일 선택 필드에서 임포트할 Zip 파일을 지정합니다. 파일 이름(경로로 정규화됨)을 입력하거나 찾아보기를 눌러 파일을 찾은 다음 열기를 누릅니다.
  5. 파일 확장에 대한 리소스 유형 매핑을 검토합니다. 필요에 따라 확장을 변경하거나 추가합니다. 다중 확장자는 콤마로 구분합니다.

    6. Zip 파일에 있는 각 파일은 사용할 수 있는 리소스이고 파일 확장자는 파일 유형을 식별하기 위해 널리 사용되는 방법입니다. Oracle Service Bus는 각 리소스에 대한 기본 확장자를 정의하지만 이러한 리소스를 설명할 추가 확장자가 존재하는 경우, 이 맵을 사용하면 해당 확장자를 로더에 선언할 수 있습니다.

    Zip 파일과 확장 맵을 정의하면 Oracle Service Bus 로더는 Zip 파일의 내용을 스캔하고 리소스 유형을 각 파일에 연관시키려고 시도합니다. 확장이 없는 파일이나 맵에 정의된 확장이 없는 파일은 알려지지 않음으로 간주되고 로드에서 자동으로 제외됩니다.

    알려진 파일의 경우 리소스 이름은 확장자가 없는 파일 이름입니다. Zip 파일의 폴더 구조는 대상 Oracle Service Bus 관리 콘솔 폴더에 재생성됩니다.

    확장은 한 개의 파일 유형에만 연관될 수 있습니다.

  6. 다음을 누릅니다.
  7. 리소스 로드 - 로드된 리소스 검토 페이지에서 임포트할 리소스를 검토합니다. 이 임포트에 포함하지 않을 리소스는 옆에 있는 체크 박스의 선택을 취소합니다.
  8. 선택한 리소스를 임포트하려면 임포트를 누릅니다.
  9. 리소스 로드 - 임포트 결과 페이지에는 임포트한 리소스가 표시됩니다. 다른 항목 로드를 눌러 다른 파일에서 리소스를 임포트하거나 왼쪽 탐색 창에서 다른 페이지를 선택하여 다른 작업을 계속합니다.
  10. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
