서비스 키 제공자

서비스 키 제공자는 프록시 서비스에서 인바운드 SOAP 메시지 비밀번호 해독과 아웃바운드 인증 및 디지털 서명을 위해 사용하는 PKI(Public Key Infrastructur) 인증서를 포함합니다. PKI 인증서는 디지털 서명 및 암호화(웹 서비스 보안용)와 아웃바운드 SSL 인증을 위해 사용할 수 있는 인증서와 쌍을 이루는 개인 키입니다. 인증서에는 개인 키에 해당하는 공용 키가 포함되어 있습니다.

주: 서비스 키 제공자를 사용하려면 PKI 인증서 매핑 제공자를 구성해야 합니다. http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Configuring the WebLogic Security Framework: Main Steps"를 참조하십시오.

단일 서비스 키 제공자가 다음 PKI 인증서를 모두 포함할 수 있습니다.

  • 디지털 암호화를 위한 키 쌍

    • 프록시 서비스는 웹 서비스 정책 문에 따라 암호화된 인바운드 SOAP 메시지의 비밀번호를 이 키 쌍을 사용하여 해독합니다. 서비스 키 제공자가 디지털 암호화를 지원하게 하려면 PKI 인증서 매퍼와 연관된 키 저장소에 암호화를 지원하는 X.509 인증서가 하나 이상 있어야 합니다.

  • 디지털 서명을 위한 키 쌍

    • 프록시 서비스는 끝점이 웹 서비스이고 클라이언트가 SOAP Envelope의 일부분 중 하나 이상에 서명하는 것이 웹 서비스에 필요한 경우 이 키 쌍을 사용합니다.

  • SSL 클라이언트 인증(양방향 SSL)을 위한 키 쌍

    • 프록시 서비스는 아웃바운드 TLS/SSL(Secure Sockets Layer) 접속 중 클라이언트로 작동할 때(즉, 클라이언트 인증서 인증이 필요한 HTTPS 비즈니스 서비스 또는 프록시 서비스로 메시지를 라우팅할 때) 이 키 쌍을 사용하여 인증합니다.

다중 프록시 서비스에 동일한 서비스 키 제공자를 사용할 수 있습니다.
