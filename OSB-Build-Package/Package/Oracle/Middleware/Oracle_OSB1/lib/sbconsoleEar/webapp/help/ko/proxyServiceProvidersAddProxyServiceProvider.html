서비스 키 제공자 추가

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프로젝트 탐색기를 선택한 다음 서비스 키 제공자를 추가할 프로젝트나 폴더를 선택합니다. 프로젝트/폴더 뷰 페이지가 표시됩니다.
  3. 리소스 생성 드롭다운 목록에서 서비스 키 제공자를 선택하여 새 서비스 키 제공자 생성 페이지를 표시합니다.
  4. 서비스 키 제공자 이름 필드에 이 서비스 키 제공자에 대한 고유 이름을 입력합니다.

    리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
  5. 설명 필드에 서비스 키 제공자에 대한 설명을 입력합니다.
  6. 표 17-2과 같은 단계 중 원하는 단계를 수행합니다.

    7. 표 17-2 인증 옵션

    수행할 단계
    디지털 암호화
    1. 암호화 키 옆의 찾아보기를 누릅니다.

      2. 암호화 키의 별칭 선택 창에 사용자 영역의 PKI 인증서 매퍼가 사용하고 있는 키 저장소의 키 별칭이 표시됩니다.

    2. 암호화 키의 별칭 선택 창에서 키 저장소에 대한 액세스를 보안하는 데 사용하는 비밀번호를 입력합니다. (키 저장소를 생성할 때 비밀번호를 설정합니다.)
    3. X.509 인증서에 매핑되고 암호화를 지원하는 키 별칭을 선택합니다.
    4. 제출을 누릅니다.
    이 서비스 키 제공자를 프록시 서비스와 연관시킬 경우 Oracle Service Bus는 X.509 인증서를 프록시 서비스의 WSDL에 삽입합니다. 그런 다음 프록시 서비스는 이 인증서를 사용하여 끝점에 보내는 메시지를 암호화합니다. 프록시 서비스는 PKI 인증서의 개인 키를 사용하여 끝점이 반환하는 메시지를 해독합니다.
    디지털 서명
    1. 디지털 서명 키 옆의 찾아보기를 누릅니다.

      2. 디지털 서명 키의 별칭 선택 창에 사용자 영역의 PKI 인증서 매퍼가 사용하고 있는 키 저장소의 키 별칭이 표시됩니다.

    2. 디지털 서명 키의 별칭 선택 창에서 키 저장소에 대한 액세스를 보안하는 데 사용하는 비밀번호를 입력합니다. (키 저장소를 생성할 때 비밀번호를 설정합니다.)
    3. 키 별칭을 선택합니다.
    4. 제출을 누릅니다.
    SSL 클라이언트 인증(양방향 SSL)
    1. SSL 클라이언트 인증 키 옆의 찾아보기를 누릅니다.

      2. SSL 클라이언트 인증 키의 별칭 선택 창에 사용자 영역의 PKI 인증서 매퍼가 사용하고 있는 키 저장소의 키 별칭이 표시됩니다.

    2. SSL 클라이언트 인증 키의 별칭 선택 창에서 키 저장소에 대한 액세스를 보안하는 데 사용하는 비밀번호를 입력합니다. (키 저장소를 생성할 때 비밀번호를 설정합니다.)
    3. 키 별칭을 선택합니다.
    4. 제출을 누릅니다.

  7. 저장을 누릅니다. 현재 세션에 서비스 키 제공자가 저장됩니다.
  8. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
