메시지 흐름에서 작업 추가 및 편집

작업은 프록시 서비스를 통과할 때의 메시지 정의 방법을 정의하는 파이프 라인 스테이지, 오류 처리기 스테이지, 경로 지정 노드 및 분기 노드의 요소입니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

또한 파이프 라인 스테이지, 경로 지정 노드 및/또는 오류 처리기 스테이지를 추가했다고 가정합니다. 다음을 참조하십시오.

  • 메시지 흐름에 파이프 라인 쌍 노드 추가
  • 파이프 라인에 스테이지 추가
  • 프록시 서비스 오류 처리기 추가

메시지 흐름에 작업을 추가하려면 다음을 수행합니다.

  1. 작업을 추가할 구성 요소를 선택합니다. 예를 들어, 스테이지 아이콘을 누른 다음 스테이지 편집 또는 경로 지정 노드 아이콘을 누르고 경로 편집을 누릅니다.
  2. 스테이지 또는 경로 지정 노드에 작업이 이미 추가되었는지 여부에 따라 다음 중 하나를 수행합니다.
    • 작업이 추가되지 않은 경우 스테이지 구성 편집 페이지에는 작업 추가 아이콘만 표시됩니다. 해당 아이콘을 누른 다음 작업 유형을 선택합니다.
    • 하나 이상의 작업이 추가된 경우 스테이지 구성 편집 페이지에 이러한 작업을 표시하는 하나 이상의 아이콘이 표시됩니다(예: 게시 아이콘 또는 라우팅 아이콘 등). 해당 아이콘을 누르고 작업 추가를 누른 다음 작업 유형을 선택합니다.
    • 게시 작업의 요청 및 응답 작업과 같이 일부 작업에는 작업에 해당하는 작업 추가 링크가 포함되어 있습니다. 해당 아이콘을 누른 다음 작업 유형을 선택합니다.

      • 메시지 흐름에서 연결할 수 있는 작업에는 제한이 없습니다.

  3. 작업 추가를 마쳤으면 표 21-2에 설명된 대로 단계 또는 경로 지정 노드에서 작업을 추가로 구성할 수 있습니다.

    4. 표 21-2 단계 구성 작업 편집 
    수행할 작업
    작업 단계
    작업 삭제
    적절한 아이콘을 누른 다음 이 작업 삭제를 누릅니다.
    작업 아래로 이동(강등)
    적절한 아이콘을 누른 다음 작업 아래로 이동을 누릅니다. 해당 작업이 이 스테이지에 포함된 다음 작업 아래로 이동됩니다.
    이 옵션은 스테이지에 둘 이상의 작업이 포함된 경우에만 표시됩니다.
    작업 위로 이동(승격)
    적절한 아이콘을 누른 다음 작업 위로 이동을 누릅니다. 해당 작업이 이 스테이지에 포함된 이전 작업 위로 이동됩니다.

    이 옵션은 스테이지에 둘 이상의 작업이 포함된 경우에만 표시됩니다.

    작업 잘라내기
    해당 아이콘을 누른 다음 잘라내기를 누릅니다.
    작업 복사
    해당 아이콘을 누른 다음 복사를 누릅니다.
    잘라냈거나 복사한 작업 붙여넣기
    적절한 아이콘을 누른 다음 작업 붙여넣기를 누릅니다.
    여러 스테이지에서 작업을 복사하고 붙여 넣을 수 있습니다. 그러나 할당, 바꾸기 또는 삽입 작업의 경우 다음 사항에 주의합니다.
    • 소스 스테이지(복사된 스테이지)의 모든 변수 관련 및 사용자 정의 네임스페이스가 대상 스테이지(붙여 넣은 스테이지)에서 사용자 정의 네임스페이스로 추가됩니다.
    • 중복된 네임스페이스(소스 및 대상 스테이지에서 동일한 네임스페이스)는 복사되지 않습니다.
    • 충돌하는 네임스페이스(동일한 접두어를 사용하지만 URI가 다른 네임스페이스 선언)는 복사됩니다. 사용자는 해당 구성을 저장할 수 있지만 스테이지 B의 충돌하는 네임스페이스 선언이 제거될 때까지는 활성화할 수 없습니다.
    스테이지 확인
    스테이지 구성 편집 페이지에서 확인을 눌러 해당 스테이지에 구성된 모든 작업을 확인할 수 있습니다.

  4. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  5. 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 계속해서 생성할 수 있습니다.
  6. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  7. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
