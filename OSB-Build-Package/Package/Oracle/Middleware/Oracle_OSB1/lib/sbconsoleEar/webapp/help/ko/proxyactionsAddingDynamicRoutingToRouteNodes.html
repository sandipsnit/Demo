경로 지정 노드에 동적 경로 지정 추가

XQuery 리소스에서 사용할 수 있는 경로 지정 정보에 기초하여 메시지의 경로를 할당합니다.

이것은 터미널 작업이므로 이 작업 뒤에 다른 작업을 추가할 수 없습니다. 그러나 이 작업에는 요청 및 응답 작업이 포함될 수 있습니다.

경로 지정에 대한 자세한 내용은 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus(http://www.oracle.com/pls/as111170/lookup?id=OSBAG181)의 "Modeling Message Flow in Oracle Service Bus"를 참조하십시오.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

경로 지정 노드에 동적 라우팅을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 경로 지정 노드 아이콘을 누른 다음 경로 편집을 누릅니다. 스테이지 구성 편집 페이지가 표시됩니다.
  3. 작업 추가 아이콘을 누른 다음 통신 > 동적 경로 지정을 선택합니다.
  4. <표현식>을 누릅니다. XQuery 표현식 편집기가 표시됩니다.
  5. XQuery 표현식 편집기에서 다음과 비슷한 결과를 생성하는 Xquery 표현식을 입력합니다.

    6. </ctx:route>

    <ctx:service isProxy='true'>{$service}</ctx:service>

    <ctx:operation>{$operation}</ctx:operation>

    </ctx:route>

    주: 프록시 서비스가 호출될 경우 isProxy 속성을 true로 설정해야 합니다. 비즈니스 서비스를 호출 중인 경우 isProxy를 false로 설정합니다.
    주: - 서비스 이름은 정규화된 서비스 이름입니다.
    주: - 작업 요소는 선택 사항입니다.
  6. 저장을 누릅니다.
  7. 요청 작업 필드에서 작업 추가를 눌러 작업을 추가한 다음 서비스와 연관시킬 작업을 선택합니다. 둘 이상의 작업을 추가할 수 있습니다. 추가할 작업 유형에 대한 자세한 내용은 메시지 흐름에서 작업 추가 및 편집의 작업 테이블을 참조하십시오.
  8. 응답 작업 필드에서 작업 추가를 눌러 작업을 추가한 다음 서비스와 연관시킬 작업을 선택합니다. 둘 이상의 작업을 추가할 수 있습니다. 추가할 작업 유형에 대한 자세한 내용은 메시지 흐름에서 작업 추가 및 편집의 작업 테이블을 참조하십시오.
  9. 저장을 누릅니다.
  10. 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 계속해서 생성할 수 있습니다.
  11. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  12. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
