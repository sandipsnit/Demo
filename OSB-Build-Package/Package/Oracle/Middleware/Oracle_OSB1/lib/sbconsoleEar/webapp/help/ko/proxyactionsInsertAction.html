삽입 작업 추가

삽입 작업을 사용하여 XPath 표현식으로 선택된 노드에 상대적으로 식별된 위치에 XQuery 표현식의 결과를 삽입할 수 있습니다. 삽입 작업은 갱신 작업 세트 중 하나입니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

삽입 작업을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 메시지 처리 > 삽입을 선택합니다.
  3. <표현식>을 눌러 XQuery 표현식을 편집합니다. XQuery 표현식은 이름이 지정된 변수의 특정 위치에 삽입할 데이터를 생성하는 데 사용됩니다. XQuery 표현식 편집기 페이지가 표시됩니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
  4. 표현식 편집이 완료되면 드롭다운 목록에서 상대적 위치를 선택합니다. 상대적 위치는 XPath 표현식의 결과에 상대적으로 삽입이 수행되는 위치를 제어하는 데 사용됩니다.
    • 이전 - XPath 표현식으로 선택된 각 요소 또는 속성 앞에 형제 레벨로
    • 이후 - XPath 표현식으로 선택된 각 요소 또는 속성 뒤에 형제 레벨로
    • 첫번째 하위로 - XPath 표현식으로 식별된 각 구성 요소의 첫번째 하위로 XPath 결과에 속성이 반환되는 경우 오류가 발생합니다.
    • 마지막 하위로 - XPath 표현식으로 식별된 각 구성 요소의 마지막 하위로 XPath가 속성을 반환하는 경우 오류가 발생합니다.
  5. XPath를 누릅니다. XPath 표현식 편집기 페이지가 표시됩니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.

    6. 적합한 구성에는 다음 사항이 포함됩니다.

    • XQuery 및 XPath 표현식이 모두 구성 요소를 반환합니다.
    • XQuery 및 XPath 표현식은 모두 속성을 반환하므로 이 경우 XQuery 표현식이 속성을 반환해야 합니다.
  6. XPath 표현식 편집이 완료되면 포함 변수 필드에 컨텍스트 변수를 입력합니다. XPath는 이 변수의 내용을 평가합니다. 컨텍스트 변수에 대한 보다 자세한 내용은 부록 A, 메시지 컨텍스트를 참조하십시오.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
