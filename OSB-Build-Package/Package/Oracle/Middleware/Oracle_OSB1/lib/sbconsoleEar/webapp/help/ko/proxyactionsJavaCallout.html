Java 콜아웃 작업 추가

Java 콜아웃 작업을 사용하여 메시지 흐름 내에서 Java 메소드 또는 EJB 비즈니스 서비스를 호출합니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

Java 콜아웃 작업을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 메시지 처리 > Java 콜아웃을 선택합니다.
  3. <메소드>를 누릅니다. JAR 선택 페이지가 표시됩니다. 목록에서 JAR 리소스를 선택합니다. 클래스와 메소드 선택 페이지가 표시됩니다.
  4. 나열된 Java 클래스 목록에서 해당 클래스 옆에 있는 +를 눌러 메소드 목록을 표시합니다. 메소드를 선택하고 제출을 누릅니다. Java 콜아웃 작업이 다음과 같이 스테이지 편집 페이지에 표시됩니다.
    • <메소드>는 2단계와 3단계에서 선택한 Java 메소드의 이름으로 바뀝니다. 이 이름은 클래스와 메소드 선택 페이지에 대한 링크입니다. 이 링크를 눌러 선택한 Java 메소드를 변경할 수 있습니다.

      • 메소드는 정적 메소드이어야 합니다.

    • 매개변수: Java 메소드에 필요한 각 인수에 대해 XQuery 표현식 편집기 페이지에 대한 표현식 링크가 제공됩니다. 각 링크의 레이블은 다음 인수에 대한 데이터 유형 중 하나를 나타냅니다.
      • Java.lang.String
      • 기본 유형과 해당 클래스 유형(예: int 및 java.lang.Integer)
      • java.lang.BigDecimal 및 java.lang.BigInteger(이러한 유형은 반올림 오류가 있거나 오버플로우가 허용되지 않는 금융 계산에 사용됨)
      • org.apache.xbeans.XmlObject 전용 및 비입력 xml bean
      • byte[]
      • java.lang.String[] (INPUT ONLY)
      • XmlObject [ ] (INPUT ONLY)
      • javax.activation.DataSource
    • 참조로 매개변수 반환: 이 옵션은 반환 값을 실제 유형과 관계없이 Java 콜아웃 호출 <java-content ref="jcid"> 참조 요소로 만듭니다. 여기서 jcid는 파이프라인 객체 저장소에서 객체의 키입니다. 결과 값 필드에 java-content 참조를 포함할 변수의 이름을 입력합니다. 이 옵션을 사용하여 통과를 제공하거나, Java 콜아웃 및 인라인 작업에서 메시지 기능을 향상하거나, Java와 비Java 전송 간의 메시지 변환을 수행하기 위해 파이프라인 XML 외에 메시지 흐름에서 참조된 객체를 작업할 수 있습니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Sending and Receiving Java Objects in Messages"를 참조하십시오.
    • 결과: 결과가 할당될 변수를 입력하는 결과 필드입니다. 이 필드의 레이블은 결과의 데이터 유형을 나타냅니다.
      • 주: 결과가 바이트 배열(가능한 반환된 배열)인 경우, binary-content XML 요소가 반환됩니다.
    • 서비스 계정 연결: 이 Java 메소드에 대한 보안 컨텍스트가 있을 경우 서비스 계정 링크를 사용하여 서비스 계정을 선택적으로 지정할 수 있습니다. 보안 컨텍스트 및 서비스 계정에 대한 자세한 내용은 서비스 계정을 참조하십시오.

      • 고정되고 매핑된 서비스 계정의 경우 서비스 계정의 사용자 ID/비밀번호가 로컬 시스템에서 인증되면 보안 컨텍스트가 Java 콜아웃에 전달됩니다. Passthru의 경우 보안 컨텍스트는 Java 콜아웃에 전달됩니다. (WS 보안)으로 정의된 경우 이 컨텍스트는 메시지 레벨 컨텍스트입니다. 그렇지 않으면 전송 레벨 컨텍스트입니다.

  5. 매개변수에서 <표현식>을 누릅니다. XQuery 표현식 편집기 페이지가 표시됩니다. XQuery 표현식 편집기를 사용하여 Java 메소드에서 요구하는 인수를 제공합니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.

    6. 입력한 입력값의 유형이 선언한 입력 인수 유형과 일치하지 않는 경우, Oracle Service Bus는 입력값을 선언된 입력 인수 유형에 자동으로 할당합니다. 예를 들어, 선언된 입력 인수의 유형이 java 기본 int인 경우 문자열 값 "123"은 정수 123으로 변환됩니다.

  6. 결과 필드에 Java 메소드에서 반환한 결과에 대해 변수를 할당합니다.
  7. Java 메소드에 대한 보안 컨텍스트가 존재하는 경우 체크 박스를 선택하고 <서비스 계정>을 누릅니다. 서비스 계정 선택 페이지가 표시됩니다. 목록에서 필요한 서비스 계정을 선택하고 제출을 누릅니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
