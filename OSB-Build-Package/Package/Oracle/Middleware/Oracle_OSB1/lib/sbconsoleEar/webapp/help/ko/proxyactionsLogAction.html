로그 작업 추가

로그 작업을 사용하여 기록할 메시지를 생성하고 메시지가 기록된 속성 집합을 정의할 수 있습니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

주: 로그 파일 또는 표준 출력(서버 콘솔)에서 로그 데이터를 보려면 Oracle WebLogic Server 로깅이 다음 심각도 레벨로 설정되어 있어야 합니다.

최소 로그 심각도: 정보
로그 파일: 정보
표준 출력: 정보

로그 심각도 레벨 설정에 대한 자세한 내용은 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server(http://www.oracle.com/pls/as111170/lookup?id=WLLOG141)의 "Using Log Severity Levels"를 참조하십시오.

 

로그 작업을 추가하려면 다음을 수행합니다.

  1. 로깅이 전역으로 설정되었는지 확인합니다. 자세한 내용은 로깅을 전체적으로 사용으로 설정을 참조하십시오.
  2. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  3. 해당 아이콘을 누른 다음 작업 추가 > 보고 > 로그를 선택합니다.
  4. <표현식>을 누릅니다. XQuery 표현식 편집기 페이지가 표시됩니다. 컨텍스트 변수에 대한 XQuery 표현식을 통해 기록되는 메시지 컨텍스트를 지정합니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
  5. 주석 필드에 이 로그 작업에 대한 메모를 입력합니다. 메모는 이전에 정의한 표현식의 결과와 함께 기록됩니다.
  6. 심각도 레벨 드롭다운 목록에서 다음 옵션 중 하나를 선택합니다.

    7. 표 21-5 로그 작업 심각도 레벨
    심각도 레벨
    일반 용도
    정보
    정상적인 작업을 보고하는 데 사용되며 낮은 레벨의 정보 메시지입니다.
    경고
    의심스러운 작업이나 구성이 수행되었지만 정상적인 작업에 영향을 주지 않습니다.
    오류
    사용자 오류가 발생했습니다. 시스템 또는 응용 프로그램에서 서비스를 중단하지 않고 성능 저하를 방지하면서 오류를 처리할 수 있습니다.
    디버그
    응용 프로그램을 개발하고 있는 동안에는 응용 프로그램 내 낮은 레벨의 작업에 대한 세부 정보 표시를 설명하는 메시지를 생성하고 사용하는 것이 유용합니다.
    프록시 서비스의 작업 설정에 대한 로그 심각도 레벨이 로그 작업 심각도 레벨과 동일한지 확인합니다. 프록시 서비스 작업 설정에 대한 자세한 내용은 프록시 서비스의 작업 설정 구성을 참조하십시오.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
