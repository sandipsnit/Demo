게시 테이블 작업 추가

게시 테이블 작업을 사용하여 메시지를 0개 이상의 정적으로 지정된 서비스에 게시합니다. 게시에 사용할 서비스를 런타임 시 결정하는 데에는 전환 스타일 조건 로직이 사용됩니다.

게시 동작에 대한 자세한 내용은 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus(http://www.oracle.com/pls/as111170/lookup?id=OSBAG205)에서 "Performing Transformations in Message Flows"를 참조하십시오.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 작업 추가 > 통신 > 게시 테이블을 선택합니다.
  3. <표현식>을 누릅니다. XQuery 표현식 편집기 페이지가 표시됩니다. 런타임 시 라우팅이 결정되는 값을 반환하는 XQuery 표현식을 생성합니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
  4. 연산자 목록에서 비교 연산자를 선택합니다. 그런 다음 인접한 필드에 XQuery 표현식에서 반환된 값을 비교하여 평가할 값을 입력합니다.
  5. 표현식이 지정한 값을 true로 평가한 경우 메시지를 게시할 서비스를 선택하려면 <서비스>를 누릅니다. 서비스 선택 페이지가 표시됩니다.
  6. 목록에서 서비스를 선택하고 제출을 누릅니다. 이는 메시지의 대상 서비스입니다.
  7. 서비스에 정의된 작업이 존재하는 경우 호출 목록에서 선택하여 호출할 작업을 지정할 수 있습니다.
  8. 아웃바운드 작업을 인바운드 작업과 같게 하려면 아웃바운드에 인바운드 작업 사용 체크 박스를 선택합니다.
  9. 요청 작업 필드에서 메시지가 패키징되어 서비스로 전송되는 방법을 구성하려면 작업 추가를 누른 다음 서비스와 연관시킬 작업을 하나 이상 선택합니다. 추가할 작업 유형에 대한 자세한 내용은 메시지 흐름에서 작업 추가 및 편집을 참조하십시오.
    10. 주: 스테이지 편집기에서는 중첩 누적 레벨이 4개까지로 제한됩니다. 5번째 레벨을 추가하면 이 중첩 작업이 표시되지 않습니다. 누적 레벨은 분기 작업인 If... Then... 조건, 게시 테이블 및 경로 테이블을 모두 포함합니다. 예를 들어, 2개 레벨의 조건부가 있고 내부에 경로 테이블이 포함된 게시 테이블 1개가 있으면 총 4개 레벨이 됩니다. 마지막 게시 테이블에 다른 조건부를 추가하면 해당 조건부는 표시되지 않습니다.
  10. 새 사례를 삽입하려면 사례 아이콘을 누른 다음 새 사례 삽입을 선택합니다.
  11. 새 사례에 대해 단계 4-8을 반복합니다.
  12. 비즈니스 로직에 따라 결정된 대로 추가 사례를 추가합니다.
  13. 시퀀스에서 정의한 마지막 사례의 사례 아이콘을 누른 다음 기본 사례 삽입을 선택하여 기본 사례를 끝에 추가합니다.
  14. 기본 사례를 구성합니다. 이 사례는 선행 사례 중 만족되는 것이 없을 경우에 라우팅 동작을 지정합니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
