이름 바꾸기 작업 추가

이름 바꾸기 작업을 사용하여 구성 요소의 내용을 수정하지 않고 XPath 표현식으로 선택한 구성 요소의 이름을 변경할 수 있습니다. 이름 바꾸기 작업은 갱신 작업 세트 중 하나입니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

이름 바꾸기 작업을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 메시지 처리 > 이름 바꾸기를 선택합니다.
  3. <XPath>를 누릅니다. XPath 표현식 편집기 페이지가 표시됩니다. XPath 표현식은 이름을 변경할 데이터(이름이 지정된 변수에 있음)를 지정하는 데 사용됩니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
  4. 변수 필드에서 이름을 변경할 요소를 포함하는 컨텍스트 변수를 입력합니다. 컨텍스트 변수에 대한 보다 자세한 내용은 부록 A, 메시지 컨텍스트를 참조하십시오.
  5. 다음 작업 중 하나를 수행합니다.
    • 로컬 이름을 사용하여 선택한 요소의 이름을 변경하려면 첫번째 로컬 이름 라디오 단추를 선택한 다음 로컬 이름 필드에 로컬 이름을 입력합니다.
    • 네임스페이스를 사용하여 선택한 요소의 이름을 변경하려면 첫번째 네임스페이스 라디오 단추를 선택한 다음 네임스페이스 필드에 네임스페이스를 입력합니다.
    • 로컬 이름 및 네임스페이스를 사용하여 선택한 요소의 이름을 변경하려면 로컬 이름 및 네임스페이스 라디오 단추를 선택한 다음 로컬 이름 및 네임스페이스 필드에 로컬 이름 및 네임스페이스를 입력합니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
