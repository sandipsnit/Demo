바꾸기 작업 추가

바꾸기 작업을 사용하여 XPath 표현식으로 지정된 노드 또는 노드의 내용을 바꿀 수 있습니다. 노드 또는 해당 내용은 XQuery 표현식에서 반환된 값으로 바뀝니다.

바꾸기 작업은 단순 값이나 구성 요소뿐만 아니라 속성을 바꾸는 데에도 사용할 수 있습니다. 바꾸기 작업이 전체 노드를 바꾸는지 또는 노드 내용만 바꾸는지 여부에 따라, 아무것도 반환하지 않는 XQuery 표현식은 식별된 노드를 삭제하거나 해당 노드를 비워 두는 것과 같습니다.

바꾸기 작업은 갱신 작업 세트 중 하나입니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

바꾸기 작업을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 메시지 처리 > 바꾸기를 선택합니다.
  3. <XPath>를 누릅니다. XPath 표현식 편집기 페이지가 표시됩니다. XPath 표현식은 바꿀 데이터(이름이 지정된 변수에 있음)를 지정하는 데 사용됩니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
  4. XPath 표현식 편집이 완료되면 포함 변수 필드에 컨텍스트 변수를 입력합니다. 부록 A, 메시지 컨텍스트를 참조하십시오.
  5. <표현식>을 누릅니다. XQuery 표현식 편집기 페이지가 표시됩니다. XQuery 표현식은 이름이 지정된 변수에 있는, XPath로 지정된 데이터를 바꾸는 데이터를 생성하는 데 사용됩니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
  6. XQuery 표현식 편집이 완료되면 다음 옵션 중 하나를 선택합니다.
    • 전체 노드 바꾸기 - 정의한 XPath 표현식에서 선택한 노드가 내용과 함께 모두 바뀝니다.
    • 노드 내용 바꾸기 - 노드는 바뀌지 않고 해당 내용만 바뀝니다.
      • 주: 노드 내용 바꾸기 옵션을 선택하고 XPath 필드를 비워 두는 것이 전체 노드 바꾸기 옵션을 선택하고 XPath를 ./*로 설정하는 것보다 훨씬 효율적입니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
