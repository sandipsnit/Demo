회신 작업 추가

회신 작업을 사용하여 호출자에게 즉시 회신을 보내도록 지정할 수 있습니다.

회신 작업은 요청, 응답 또는 오류 파이프 라인에서 사용할 수 있습니다. 성공 또는 실패 회신을 생성하도록 구성할 수 있습니다. 인바운드 전송이 HTTP인 실패 회신의 경우 회신 작업은 즉각적인 회신이 호출자에게 전송되도록 지정합니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

회신 작업을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 흐름 제어 > 회신을 선택합니다.
  3. 성공을 선택하여 메시지가 성공했다고 회신하거나 실패를 선택하여 메시지에 오류가 있다고 회신합니다.

    4. Oracle Service Bus가 시작한 경우 회신이 실패하면 트랜잭션이 중단됩니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
