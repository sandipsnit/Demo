보고서 작업 추가

보고서 작업을 사용하여 프록시 서비스의 메시지 보고를 사용으로 설정할 수 있습니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

보고서 작업을 구성하려면

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 보고 > 보고서를 선택합니다.
  3. <표현식>을 누릅니다. XQuery 표현식 편집기 페이지가 표시됩니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오. XQuery 표현식은 Oracle Service Bus 대시보드에 보고할 데이터를 생성하는 데 사용됩니다.
  4. XQuery 표현식 편집이 완료되면 키 추가를 누릅니다. 키 이름 필드와 키 값 필드가 표시됩니다. 키 값 필드에는 눌러 XPath 표현식을 편집할 수 있는 XPath 링크와 컨텍스트 변수를 입력할 수 있는 포함 변수 필드가 포함됩니다.

    5. 키 값 쌍을 사용하여 메시지 컨텍스트 변수나 메시지 페이로드에서 키 ID를 추출하고 나머지 메시지는 무시됩니다. 키를 사용하면 메시지를 편리하게 식별할 수 있습니다. 키는 보고 모듈에서 보고서 인덱스로 표시됩니다. 자세한 내용은 보고 메시지 보기 및 메시지 세부 정보 보기를 참조하십시오.

    1. 키 이름 필드에 키 이름을 입력합니다.
    2. <XPath>를 누릅니다. XPath 표현식 편집 페이지가 표시됩니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
    3. 포함 변수 필드에 컨텍스트 변수를 입력합니다. 컨텍스트 변수에 대한 자세한 내용은 부록 A, 메시지 컨텍스트를 참조하십시오.
    4. 다른 키 값을 추가하려면 키 아이콘을 누른 다음 키 추가를 선택합니다. 키를 삭제하려면 키 아이콘을 누른 다음 이 키 삭제를 선택합니다.

예를 들어, 특정 스테이지에서 오류 처리기에 구성된 보고서 작업을 가정합니다. 오류가 발생할 경우 보고서 작업은 fault 컨텍스트 변수의 내용을 보고합니다. 보고서 작업은 다음과 같이 구성됩니다.

  • 키 이름 = errorCode
  • 키 값 = ./ctx:errorCode(변수 fault에 있음)

이 작업이 런타임 시 실행될 때마다 메시지가 보고 데이터 스트림을 통해 보고됩니다. 다음 표에서는 보고서 작업을 두 번 실행한 후의 결과를 보여줍니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
