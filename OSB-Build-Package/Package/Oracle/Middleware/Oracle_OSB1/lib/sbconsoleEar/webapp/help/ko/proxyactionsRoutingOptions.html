라우팅 옵션 작업 추가

라우팅 옵션 작업을 사용하면 $outbound의 아웃바운드 요청 속성인 URI, 서비스 품질, 모드, 재시도 매개변수 중 일부 또는 전체를 수정할 수 있습니다. 이러한 속성을 $outbound의 할당, 삽입, 바꾸기 또는 삭제 작업을 사용하여 수정할 수도 있지만 라우팅 옵션을 사용하면 XPath, XQuery 또는 $outbound 컨텍스트 변수의 구조에 대해 잘 몰라도 이러한 작업을 보다 편리하게 수행할 수 있습니다.

라우팅 옵션 작업은 컨텍스트 변수 $outbound가 적합한 경우에만 사용할 수 있습니다. 다음 작업에 추가할 수 있습니다.

  • 게시
  • 동적 게시
  • 게시 테이블
  • 서비스 콜아웃
  • 경로 지정
  • 동적 라우팅
  • 라우팅 테이블

경로 지정에 대한 자세한 내용은 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus(http://www.oracle.com/pls/as111170/lookup?id=OSBAG181)의 "Modeling Message Flow in Oracle Service Bus"를 참조하십시오.

 

라우팅 옵션 작업을 구성하려면

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 통신 > 라우팅 옵션을 선택합니다.
  3. 다음 단계 중 일부나 전체를 완료하십시오.
    • 아웃바운드 메시지에 대한 URI 설정: URI를 선택하고 XQuery 표현식 편집기를 누릅니다. URI를 반환하는 표현식을 입력합니다. 호출된 서비스에 대한 URI를 무효화합니다.
    • 서비스 품질 요소 설정: 서비스 품질을 선택하고 드롭다운 목록에서 서비스 품질 옵션을 선택합니다. 자동 계산되는 기본값을 무효화합니다.
    • 모드 설정: 모드를 선택하고 드롭다운 목록에서 request 또는 request-response를 선택합니다.
      • 주: 모드는 호출된 서비스의 인터페이스에 따라 보통 자동으로 설정됩니다. 하지만 Any Soap 또는 Any XML 서비스와 같은 경우는 예외 사항입니다.
    • 재시도 간격 설정: 재시도 간격을 선택하고 재시도 간격(초)을 지정합니다. 호출된 서비스로 구성된 기본값을 무효화합니다.
    • 재시도 횟수 설정: 재시도 횟수를 선택하고 작업을 중단하기 전에 시스템에서 시도해야 하는 횟수를 지정합니다. 호출된 서비스로 구성된 기본값을 무효화합니다.
    • 메시지 우선순위를 설정하려면 우선순위를 선택하고 XQuery 표현식 편집기를 누릅니다. 양의 정수를 반환하는 표현식을 입력합니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
