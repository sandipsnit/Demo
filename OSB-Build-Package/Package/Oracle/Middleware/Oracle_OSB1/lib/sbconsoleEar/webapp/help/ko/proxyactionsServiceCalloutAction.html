서비스 콜아웃 작업 추가

서비스 콜아웃 작업을 사용하면 Oracle Service Bus 등록 프록시 또는 비즈니스 서비스에 대한 동기(차단) 콜아웃을 구성할 수 있습니다.

서비스 콜아웃 작업에 대한 자세한 내용은 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus(http://www.oracle.com/pls/as111170/lookup?id=OSBAG208)의 "Constructing Service Callout Messages"를 참조하십시오.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 통신 > 서비스 콜아웃을 선택합니다.
  3. <서비스>를 누릅니다. 서비스 브라우저가 표시됩니다.
  4. 등록된 프록시 또는 비즈니스 서비스 목록에서 서비스를 선택한 다음 제출을 누릅니다.
  5. 위의 단계 3에서 선택한 서비스가 WSDL 기반이고 서비스에서 호출할 수 있는 작업이 포함된 경우 해당 작업은 <작업> 호출 중 드롭다운 목록에 나열됩니다. 서비스에서 호출할 작업을 선택합니다.

    주: 여러 가지 이유로 Oracle Service Bus에 필요한 작업을 선택해도 선택한 작업만 호출되는 것은 아닙니다. 예를 들어, 작업 A를 선택했지만 메시지에 작업 B에 대한 호출이 포함될 경우 작업 B도 함께 호출됩니다.
  6. 다음 옵션 중 하나를 선택하여 요청 및 응답 메시지를 구성하는 방법을 지정합니다.
    • SOAP 본문을 구성하려면 SOAP 본문 구성을 선택합니다. 이 옵션을 선택하면 $body를 바로 사용할 수 있습니다.
      • 주: 이 옵션은 페이로드 매개변수 또는 문서를 구성할 때 지원되지 않는 SOAP-RPC 인코딩을 지원합니다.
    • 페이로드를 구성하려면 페이로드 매개변수 구성 또는 페이로드 문서 구성을 선택합니다.
  7. 이후 구성 옵션은 단계 3에서 선택한 서비스 유형과 해당 서비스에 대해 단계 5에서 선택한 구성 옵션 유형에 따라 달라집니다. 표 21-3에는 각 서비스 유형에 대해 사용 가능한 옵션이 표시됩니다.
    표 21-3 각 서비스 유형에 대한 서비스 콜아웃 구성 옵션
    선택한 서비스 유형
    "SOAP 본문 구성" 옵션
    "페이로드 매개변수 구성" 옵션 또는
    "페이로드 문서 구성" 옵션
    • SOAP RPC
    • SOAP 요청 본문 및 SOAP 응답 본문
    • SOAP 요청 헤더 및 SOAP 응답 헤더(선택 사항)
    • 요청 매개변수 및 응답 매개변수
    • SOAP 요청 헤더 및 SOAP 응답 헤더(선택 사항)
    • SOAP 문서
    • 모든 SOAP
    • SOAP 요청 본문 및 SOAP 응답 본문
    • SOAP 요청 헤더 및 SOAP 응답 헤더(선택 사항)
    • 요청 문서 및 응답 문서
    • SOAP 요청 헤더 및 SOAP 응답 헤더(선택 사항)
    • XML
    • 모든 XML
    • 메시징
    • SOAP 요청 본문 및 SOAP 응답 본문
    • 요청 문서 및 응답 문서

    8. 아래의 표 21-4에서는 위의 표 21-3에 나열된 각 옵션에 대한 지침을 제공합니다.

    표 21-4 서비스 콜아웃 구성 옵션
    해당 옵션
    다음 단계를 준수합니다.
    SOAP 요청 본문 및
    SOAP 응답 본문
    해당 옵션을 구성하려면
    • SOAP 요청 본문 필드에서 콜아웃 요청을 위해 SOAP 본문 구성 요소의 XML을 보관할 변수 이름을 입력합니다.
    • SOAP 응답 본문 필드에서 응답이 바인딩되는 SOAP 본문 구성 요소의 XML에 대한 변수 이름을 입력합니다.
    SOAP 요청 헤더 및
    SOAP 응답 헤더
    해당 옵션을 구성하려면
    • SOAP 요청 헤더 필드에서 콜아웃 요청을 위해 SOAP 헤더 구성 요소의 XML을 보관할 변수 이름을 입력합니다.

      • <soap-env:Header>...</soap-env:Header>가 있는 SOAP 요청 헤더에 대한 입력 문서를 포함해야 합니다.

    • SOAP 응답 헤더 필드에서 응답이 바인딩되는 SOAP 헤더 구성 요소의 XML에 대한 변수 이름을 입력합니다.
    요청 매개변수 및
    응답 매개변수
    옵션을 구성하려면
    • 요청 매개변수 필드에 요청 매개변수의 값을 제공하기 위해 런타임에 평가할 변수 이름을 입력합니다.

      • 입력 변수에는 코어 페이로드 문서만 제공해야 합니다. SOAP 패키지는 Oracle Service Bus에서 생성됩니다. 즉, 입력 문서를 <soap-env:Body>...</soap-env:Body>에 포함시키지 마십시오.

      예를 들어, 이 요청 매개변수에 사용된 body 입력 변수를 생성하는 경우 $body(결과적으로 soap-env:Body 래퍼 유지)가 아닌 XPath 문 body/*(soap-env:Body 래퍼 제거)을 사용하여 변수 내용을 정의합니다.

    • 응답 매개변수 필드에 응답을 런타임에 할당할 변수 이름을 입력합니다.
    요청 문서 및
    응답 문서
    해당 옵션을 구성하려면
    • 요청 문서 변수 필드에 요청 문서를 할당할 변수 이름을 입력합니다.

      • SOAP 문서 유형 서비스의 경우, 변수는 해당 서비스로 보내진 SOAP 메시지의 본문을 구성하도록 런타임 시 평가됩니다. 모든 XML 서비스의 경우, 변수는 해당 서비스로 보내진 XML 메시지의 본문을 구성하도록 런타임 시 평가됩니다.

      SOAP 문서 유형 서비스 및 모든 XML 서비스의 경우, 입력 변수에서 코어 페이로드 문서만 제공하며, SOAP 패키지는 Oracle Service Bus에서 생성됩니다. 즉, 입력 문서를 <soap-env:Body>...</soap-env:Body>에 포함시키지 마십시오.

      예를 들어, 이 요청 매개변수에 사용된 body 입력 변수를 생성하는 경우 $body(결과적으로 soap-env:Body 래퍼 유지)가 아닌 XPath 문 body/*(soap-env:Body 래퍼 제거)을 사용하여 변수 내용을 정의합니다.

      메시징 서비스의 경우, 변수는 서비스에서 예상되는 데이터 유형을 기반으로 메시지 본문을 구성하도록 평가됩니다. 다음 제한 사항은 메시징 서비스에서 사용되는 변수에 적용됩니다.

      • 이진 데이터를 예상하는 서비스에 대해 변수는 ctx:binary-content 구성 요소를 가져야 합니다.
      • MFL 데이터를 예상하는 서비스에 대해 변수는 XML과 동등한 항목을 가져야 합니다.
      • 텍스트 데이터를 예상하는 서비스에 대해 변수는 문자열입니다.
    • 응답 문서 변수 필드에 응답 문서가 런타임 시 할당된 변수 이름을 입력합니다.

  8. 선택적으로 하나 이상의 전송 헤더를 추가합니다. 자세한 내용은 전송 헤더 작업 추가를 참조하십시오.
    9. 주: 지정한 전송 헤더 외에도 Oracle Service Bus 바인딩 층에 의해 헤더가 추가됩니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG201의 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus에서 "Configuring Transport Headers in Message Flows"를 참조하십시오.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
