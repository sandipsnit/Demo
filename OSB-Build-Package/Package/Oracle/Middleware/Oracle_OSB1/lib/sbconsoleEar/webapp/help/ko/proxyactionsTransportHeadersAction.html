전송 헤더 작업 추가

전송 헤더 작업을 사용하여 메시지에서 헤더 값을 설정할 수 있습니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 통신 > 전송 헤더를 선택합니다.
  3. 전송 헤더 설정 목록에서 다음 중 하나를 선택하여 런타임에 수정할 메시지 컨텍스트 위치를 지정합니다.
    • 아웃바운드 요청 - 아웃바운드 요청(경로, 게시 또는 콜아웃 작업에서 프록시 서비스에 의해 전송된 메시지)에 대한 헤더 값을 설정하려면 이 옵션을 선택합니다. 이 헤더 요소는 다음과 같이 메시지 컨텍스트에 위치합니다.

      • $outbound/ctx:transport/ctx:request/tp:headers

    • 인바운드 응답 - 인바운드 응답(프록시 서비스가 클라이언트에 보낸 응답 메시지)에 대한 헤더 값을 설정하려면 이 옵션을 선택합니다. 이 헤더 요소는 다음과 같이 메시지 컨텍스트에 위치합니다.

      • $inbound/ctx:transport/ctx:response/tp:headers

  4. 선택적으로 파이프 라인을 통해 모든 헤더 전달을 선택하여 모든 헤더를 인바운드 메시지에서 아웃바운드 메시지로 또는 그 반대로 전달합니다. 소스 헤더 세트에 있는 모든 헤더가 대상 헤더 세트에 복사되고, 대상 헤더 세트에 있는 모든 기존 값을 겹쳐 씁니다.

    5. 헤더별 통과 옵션과 함께 이 옵션을 사용하는 방법에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG201의 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus에서 "Configuring Transport Headers in Message Flows"를 참조하십시오.

  5. 추가할 각 헤더에 대해 다음 단계를 완료합니다.
    1. 전송 헤더 테이블에서 헤더 추가를 눌러 헤더 구성을 위한 필드를 표시합니다.
    2. 다음을 수행하여 헤더를 지정합니다.
      • 이름 열의 드롭다운 목록에서 헤더 이름을 선택합니다. 목록에는 대상 전송에 대해 미리 정의된 모든 헤더 이름이 포함됩니다(예: HTTP 전송에 대해 콘텐츠 유형, JMS 전송에 대해 JMSCorrelationID 등).
      • 기타 필드에 헤더 이름을 입력합니다. 해당 헤더 이름이 이 서비스 전송에 대해 미리 정의된 헤더 중 하나가 아닌 경우 전송 지정에 의해 정의된 대로 사용자 헤더가 됩니다.
    3. 작업 열의 옵션 중 하나를 선택하여 헤더 값 설정 방법을 지정합니다.

      4. 표현식에 헤더 설정

      이 옵션을 선택하면 XQuery 또는 XSLT 표현식을 사용하여 헤더의 값을 설정할 수 있습니다. 표현식은 단순(예: "text/xml") 또는 복합 XQuery/XSLT 표현식일 수 있습니다.

      Oracle Service Bus 전송 층은 모든 헤더의 XML 표현을 문자열 값으로 정의하기 때문에, 모든 표현식의 결과는 헤더 값이 설정되기 전에 문자열로 변환됩니다. 아무 것도 반환하지 않는 표현식의 헤더값은 빈 문자열로 설정됩니다. 표현식을 사용하여 헤더를 삭제할 수 없습니다.

      주의: 이 작업에서 지정할 수 있는 헤더 설정은 런타임에서 일부 지원되지 않을 수 있습니다. 사용자가 설정할 수 있는 제공된 전송에 대한 헤더 및 런타임에 지원되는 헤더에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG201의 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus에서 "Configuring Transport Headers in Message Flows"를 참조하십시오.

      헤더 삭제

      헤더가 요청 또는 응답 메타 데이터에서 제거되도록 지정합니다.

      인바운드 요청에서 헤더 복사(아웃바운드 요청에 대한 전송 헤더를 설정하는 경우)
      또는
      아웃바운드 응답에서 헤더 복사(인바운드 응답에 대한 전송 헤더를 설정하는 경우)

      이 헤더가 인바운드 메시지에서 아웃바운드 메시지로 또는 그 반대로 동일한 이름의 해당 헤더에서 직접 복사되도록 지정합니다. 예를 들어, 아웃바운드 요청에 대해 SOAPAction 헤더를 설정하려는 경우 인바운드 요청에서 헤더 복사를 선택하면 런타임에 $inbound의 SOAPAction 요청 헤더에서 값이 복사됩니다. 인바운드 응답 헤더의 경우, 복사할 헤더의 소스는 $outbound의 응답 헤더입니다.

      소스에 존재하지 않는 헤더에 대해 헤더 복사... 옵션을 선택한 경우, 이 옵션은 무시되고 이 헤더의 대상에서 작업이 수행되지 않습니다.

      전역 파이프 라인을 통해 모든 헤더 전달 옵션과 함께 이 옵션을 사용하는 방법에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG201의 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus에서 "Configuring Transport Headers in Message Flows"를 참조하십시오.

  6. 추가 헤더를 테이블에 추가하려면 헤더 아이콘을 누른 다음 헤더 추가를 누릅니다.

    7. 이 테이블은 추가 행을 포함시키기 위해 확장되고, 다른 전송 헤더를 구성하기 위해 사용할 수 있는 새 옵션 세트가 포함됩니다. 이 테이블에 필요한만큼 헤더를 추가할 수 있습니다. 런타임은 해당 XML을 생성할 때 네임스페이스를 선언하고 적절한 순서로 헤더 요소를 배치하기 때문에 테이블에서 헤더 순서를 지정할 필요가 없습니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.

 

아웃바운드 HTTP 전송 헤더에 쿠키 설정

HTTP 메시지 흐름에서 다음과 같은 방법으로 전송 헤더에 쿠키를 설정할 수 있습니다.

  • 복잡한 XML 표현식 사용(Oracle Service Bus 기본값)
  • 문자열 표현식 사용(서버 시작 옵션 필요)

쿠키를 복잡한 XML 표현식으로 설정

Oracle Service Bus의 기본 형식인 복잡한 XML 표현식을 사용하여 쿠키를 설정하려면 다음 표현식 구문을 사용하여 아웃바운드 요청에서 HTTP 쿠키 헤더의 값을 구성합니다.

<cookie-values xmlns="http://www.bea.com/wli/sb/transports/http">
<value>{fn:concat("cookie_name", "=", "cookie_value")}</value>
</cookie-values>

문자열 표현식으로 쿠키 설정

아웃바운드 요청에서 문자열로 쿠키 헤더를 설정하려면 도메인 startWebLogic 명령에 다음 옵션을 추가해야 합니다.

-Dcom.bea.osb.http.cookieAsNoComplexElement=true

이 옵션으로 서버를 재시작한 후에는 문자열 표현식으로 HTTP 쿠키 헤더를 설정할 수 있습니다. 예를 들면 다음과 같습니다.
$cookie_name = "cookie_value"
