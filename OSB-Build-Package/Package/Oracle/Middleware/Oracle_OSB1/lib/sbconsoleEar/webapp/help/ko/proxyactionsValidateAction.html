검증 작업 추가

검증 작업을 사용하면 XML 스키마 요소 또는 WSDL 리소스와 비교해서 XPath 표현식에서 선택한 요소를 검증할 수 있습니다. 전역 요소만 검증할 수 있으며 Oracle Service Bus는 로컬 요소에 대한 검증은 지원하지 않습니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

검증 작업을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 해당 아이콘을 누른 다음 작업 추가 > 메시지 처리 > 검증을 선택합니다.
  3. XPath를 눌러 검증할 요소를 지정하는 XPath 표현식을 생성합니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오. XPath 표현식 편집기에서 표현식 생성을 마쳤으면 저장을 누릅니다. 스테이지 생성 편집 페이지에서 표현식을 삽입합니다.
  4. 포함 변수 필드에서 검증할 요소를 포함하는 변수 이름을 입력합니다.
  5. 리소스를 누른 다음 WSDL 또는 스키마를 선택합니다.
  6. WSDL 브라우저 또는 XML 스키마 브라우저에서 다음을 수행합니다.
    1. WSDL 또는 XML 스키마 선택
    2. WSDL 또는 XML 스키마 유형 또는 요소
    3. 제출을 누릅니다.
  7. 이 검증 결과(부울 결과)를 저장하려면 변수에 검증 결과 저장을 선택하고 결과를 저장할 변수의 이름을 입력합니다.

    8. 또는 WSDL이나 XML 스키마 구성 요소와 비교해서 구성 요소 검증이 실패할 경우 오류를 생성하려면 검증 실패 시 오류 생성을 선택합니다.

마친 후에

이 작업의 구성을 완료하면 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 다른 작업을 구성하거나 구성을 저장합니다.
