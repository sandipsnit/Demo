경로 지정 노드에 라우팅 테이블 추가

라우팅 테이블은 전환 스타일 조건 테이블에 포함된 경로 지정 집합으로 단일 XQuery 표현식의 결과를 기반으로 서로 다른 경로를 선택할 수 있게 해주는 편리한 구문입니다.

스테이지 편집기에서는 중첩 누적 레벨이 4개까지로 제한됩니다. 5번째 레벨을 추가하면 이 중첩 작업이 표시되지 않습니다. 누적 수준은 분기 작업인 If... Then... 조건, 게시 테이블 및 경로 테이블을 모두 포함합니다. 예를 들어, 2개 레벨의 조건부가 있고 내부에 경로 테이블이 포함된 게시 테이블 1개가 있으면 총 4개 레벨이 됩니다. 마지막 게시 테이블에 다른 조건부를 추가하면 해당 조건부는 표시되지 않습니다.

메시지의 대상 서비스를 식별하고 메시지가 해당 서비스로 경로 지정되는 방법을 구성할 수 있습니다.

이것은 터미널 작업이므로 이 작업 뒤에 다른 작업을 추가할 수 없습니다. 그러나 이 작업에는 요청 및 응답 작업이 포함될 수 있습니다.

경로 지정에 대한 자세한 내용은 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus(http://www.oracle.com/pls/as111170/lookup?id=OSBAG181)의 "Modeling Message Flow in Oracle Service Bus"를 참조하십시오.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

경로 지정 노드에 라우팅 테이블을 추가하려면 다음을 수행합니다.

  1. 메시지 흐름에서 작업 추가 및 편집에 설명된 대로 작업을 추가할 위치로 이동합니다.
  2. 경로 지정 노드 아이콘을 누른 다음 경로 편집을 누릅니다. 스테이지 구성 편집 페이지가 표시됩니다.
  3. 작업 추가 아이콘을 누른 다음 통신 > 경로 지정 테이블을 선택합니다. 그러면 경로 지정 테이블 작업이 표시됩니다.
  4. 연산자 목록에서 비교 연산자를 선택한 다음 인접한 필드에 값 표현식을 입력합니다.
  5. <서비스>를 누릅니다. 서비스 선택 페이지가 표시됩니다.
  6. 목록에서 서비스를 선택하고 제출을 누릅니다.
  7. 서비스에서 작업을 호출하려면 작업 목록에서 작업을 선택합니다.
  8. 아웃바운드 작업을 인바운드 작업과 같게 하려면 아웃바운드에 인바운드 작업 사용 체크 박스를 선택합니다.
  9. 요청 작업 필드에서 작업 추가를 눌러 작업을 추가한 다음 서비스와 연결할 작업을 선택합니다. 둘 이상의 작업을 추가할 수 있습니다.
  10. 응답 작업 필드에서 작업 추가를 눌러 작업을 추가한 다음 서비스와 연결할 작업을 선택합니다. 둘 이상의 작업을 추가할 수 있습니다.

    11. 추가할 수 있는 요청 및 응답 작업 유형에 대한 자세한 내용은 메시지 플로우에서 작업 추가 및 편집을 참조하십시오.

  11. 새 사례를 삽입하려면 사례 아이콘을 누른 다음 새 사례 삽입을 선택합니다.
  12. 새 사례에 대해 단계 2-7을 반복합니다. 선행 사례 중 적합한 사례가 없을 경우 사례 아이콘을 누른 다음 기본 사례 삽입을 선택하여 선택한 경로의 끝에 기본 사례를 추가할 수 있습니다.
  13. 저장을 누릅니다.
  14. 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 계속해서 생성할 수 있습니다.
  15. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  16. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
