메시지 흐름에 조건부 분기 추가

분기 노드를 사용하면 여러 개의 가능한 경로 중 정확히 하나의 경로로 처리를 진행할 수 있습니다. 분기는 XPath 기반 스위치 테이블을 기반으로 합니다. 테이블의 각 분기는 단일 XPath 표현식에 대해 <500과 같이 메시지 흐름이 내림차순으로 평가되는 조건을 지정합니다(예: $body의 ./ns: PurchaseOrder/ns:totalCost). 먼저 만족되는 조건에 따라 다음에 오는 분기가 결정됩니다. 만족하는 분기 조건이 없는 경우 기본 분기가 옵니다. 분기 노드는 메시지 흐름에서 기본 분기를 포함하여 각 분기에 대해 하나씩, 여러 개의 하위 항목을 가질 수 있습니다.

프록시 서비스가 WSDL에 기반하지 않고 입력과 같은 다중 문서 유형을 수신하는 경우 조건부 분기 노드를 사용하는 것으로 간주됩니다.

조건부 분기는 단순하지만 고유한 문자열 값이 각 분기에 지정된 조회 테이블을 기반으로 합니다. 메시지 컨텍스트의 변수는 해당 노드의 조회 변수로 지정되고 런타임 시 값은 다음에 올 분기를 결정하는 데 사용됩니다. 조회 변수의 값과 일치하는 분기가 없으면 기본 분기가 옵니다. 조회 변수의 값이 분기 노드에 도달하기 전에 설정되도록 프록시 서비스를 설계해야 합니다.

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

메시지 흐름에 조건부 분기를 추가하려면 다음을 수행합니다.

  1. 파이프 라인 쌍 노드 아이콘 또는 분기 노드 아이콘을 누른 다음 조건부 분기 생성을 누릅니다. 조건부 분기 노드가 추가되고 삽입된 분기 노드 다음의 기존 노드가 새 조건부 분기 노드의 기본 분기로 이동합니다.
  2. 기본 이름을 변경하고 분기 노드에 대한 설명을 추가하려면 다음 작업을 수행합니다.
    1. 조건부 분기 아이콘을 누르고 이름 및 주석 편집을 누릅니다.
    2. 원하는 대로 이름 및 설명을 변경합니다.
    3. 저장을 누릅니다.
  3. 분기 정의를 추가하려면 조건부 분기 아이콘을 누르고 분기 편집을 누릅니다. 분기 노드 편집 페이지가 표시됩니다.
  4. 다음을 수행합니다.
    1. 선택한 경로 필드에서 편집을 눌러 경로를 지정할 XPath 표현식을 추가합니다. 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
    2. 변수 필드에 컨텍스트 변수를 입력합니다.
    3. 연산자 필드에서 비교 연산자를 선택합니다.
    4. 값 필드에 분기에 대한 값을 입력합니다.
    5. 레이블 필드에 분기에 대한 레이블을 입력합니다.
  5. 선택적으로 옵션에서 다음을 수행합니다.
    • 새 분기 정의를 이 분기 노드에 추가하려면 새 분기 추가를 누릅니다.
    • 분기 정의를 삭제하려면 이 분기 삭제를 누릅니다.
    • 분기 위로 이동 또는 분기 아래로 이동을 눌러 분기 정의 위치를 변경합니다. 이 옵션은 둘 이상의 분기 정의가 존재하는 경우에만 표시됩니다.
  6. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  7. 메시지 흐름 편집 페이지에서 메시지 흐름 생성, 편집 및 보기에 설명된 대로 메시지 흐름을 계속해서 생성할 수 있습니다.
  8. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  9. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
