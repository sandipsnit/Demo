메시지 흐름에 경로 지정 노드 추가

시작하기 전에

이 지침에서는 메시지 흐름 생성, 편집 및 보기에 설명된 것처럼 메시지 흐름 편집 페이지에서 메시지 흐름을 편집하고 있다고 가정합니다.

메시지 흐름에 경로 지정 노드를 추가하려면 다음을 수행합니다.

  1. 파이프 라인 쌍의 파이프 라인 쌍 노드 아이콘을 누른 다음 경로 추가를 누릅니다.
  2. 기본 이름을 변경하고 경로 지정 노드의 설명을 추가하려면 경로 지정 노드 아이콘을 누른 다음 이름 및 주석 편집을 누릅니다. 원하는 대로 이름 및 설명을 변경한 다음 저장을 누릅니다.
    3. 주: 파이프 라인 또는 경로 지정 노드의 이름을 변경하는 경우 모니터링 모듈의 대시보드 페이지에서 표시된 메시지 수는 파이프 라인 카운터가 0으로 재설정되었기 때문에 다른 구성 요소의 메시지와 상관 관계가 없을 수 있습니다. 이는 Oracle Service Bus에서 이름 바꾸기를 삭제 및 다시 생성 작업으로 처리하기 때문입니다. 이 숫자는 서비스의 모니터링 간격과 동일한 시간이 경과한 후에 다시 상관 관계를 가져야 합니다.
  3. 경로 지정 노드에 작업을 추가하려면 경로 지정 노드 아이콘을 누른 다음 경로 편집을 누릅니다. 메시지 흐름 편집 페이지가 표시됩니다. 경로 지정 노드에 추가할 수 있는 작업 정보는 다음 섹션을 참조하십시오.
    • If... Then... 작업 추가
    • 경로 지정 노드에 동적 경로 지정 추가
    • 경로 지정 노드에 라우팅 작업 추가
    • 경로 지정 노드에 라우팅 테이블 추가
    • 경로 지정 노드 오류 처리기 추가
  4. 메시지 흐름 편집 페이지에서 메시지 흐름 생성, 편집 및 보기에 설명된 대로 메시지 흐름을 계속해서 생성할 수 있습니다.
  5. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  6. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
