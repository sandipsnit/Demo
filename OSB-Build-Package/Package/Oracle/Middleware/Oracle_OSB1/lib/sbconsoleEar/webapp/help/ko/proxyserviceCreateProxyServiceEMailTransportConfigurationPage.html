전자 메일 전송 구성 페이지

프록시 서비스 생성/편집 - 전자 메일 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 전자 메일 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-4에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-4 프록시 서비스 생성/편집 - 전자 메일 전송 구성 페이지
옵션
생성 또는 편집
SSL 필요
SSL(Secure Sockets Layer)을 사용하여 통신하려면 이 옵션을 선택합니다. 기본 SSL 포트 번호는 다음과 같습니다.
  • POP3(S): 995
  • IMAP: 993
참고: SSL이 올바르게 작동하려면 전자 메일 서버 인증서가 Oracle Service Bus 보안 저장소에 있어야 합니다.
서비스 계정
서비스 계정 이름을 입력하거나 찾아보기를 눌러 브라우저에서 서비스 계정을 선택합니다.
관리 서버
이 필드는 클러스터화된 도메인에서만 사용할 수 있습니다.
폴링 서버 역할을 할 관리 서버를 선택합니다. 모든 관리 서버는 메시지를 처리할 수 있지만 그 중 하나만 메시지를 폴링할 수 있습니다.
폴링 간격
폴링 간격(초)을 입력합니다.
전자 메일 프로토콜
전자 메일 계정에 대한 서버 유형으로 POP3 또는 IMAP를 선택합니다.
읽기 제한
폴링 스윕당 읽을 최대 메시지 수를 지정합니다. 제한 없음을 지정하려면 0을 입력합니다.
참조로 전달
이 체크 박스를 선택하여 아카이브 디렉토리에 파일을 준비하고 헤더에 참조로 전달합니다.
기본적으로 새 서비스를 생성하면 참조로 전달 옵션이 선택되며 아카이브 디렉토리 위치를 지정해야 합니다.
참조로 첨부 파일 전달
이 체크 박스를 선택하여 아카이브 디렉토리에 참조 파일을 준비하고 헤더에 참조로 전달합니다.
기본적으로, 참조로 전달 옵션이 선택되면 참조로 첨부 파일 전달 옵션이 암시적으로 적용되며 아카이브 디렉토리 위치를 지정해야 합니다.
읽기 후 작업
읽은 다음 메시지에 대해 수행할 작업을 선택합니다.
  • 아카이브 - 메시지가 보관됩니다.
  • 삭제 - 메시지가 삭제됩니다.
  • 이동 - 메시지가 이동됩니다. 이동은 IMAP 프로토콜에서만 사용할 수 있습니다.
첨부 파일
첨부 파일을 처리하는 방법을 다음과 같이 선택합니다.
  • 아카이브 - 첨부 파일이 아카이브 디렉토리에 저장됩니다.
  • 무시 - 첨부 파일이 무시됩니다.
IMAP 이동 폴더
읽기 후 작업 필드가 이동으로 설정된 경우 메시지를 이동할 폴더를 입력합니다.
다운로드 디렉토리
전자 메일을 다운로드할 임시 위치를 입력합니다.
아카이브 디렉토리
읽기 후 작업 필드가 아카이브로 설정된 경우 아카이브 위치의 경로를 지정합니다.
이 필드는 참조로 전달 또는 참조로 첨부 파일 전달 옵션이 선택되는 경우에 필요합니다.
오류 디렉토리
문제가 있을 경우 메시지와 첨부 파일을 기록할 파일 시스템 디렉토리 경로를 입력합니다.
요청 인코딩
기본값 ISO-8859-1을 전자 메일 전송 요청을 위한 문자 집합 인코딩으로 적용하거나 다른 문자 집합 인코딩을 입력합니다.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 프록시 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
