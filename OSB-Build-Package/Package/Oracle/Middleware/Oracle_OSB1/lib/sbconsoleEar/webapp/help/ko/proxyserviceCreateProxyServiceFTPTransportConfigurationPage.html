FTP 전송 구성 페이지

프록시 서비스 생성/편집 - FTP 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 FTP 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-6에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-6 프록시 서비스 생성/편집 - FTP 전송 구성 페이지
옵션
생성 또는 편집
사용자 인증
FTP 서버 사용자가 익명인 경우 익명을 선택하고 FTP 서버 사용자가 외부적으로 구성된 계정인 경우 외부 사용자를 선택합니다.
ID(전자 메일 ID)
이 필드는 사용자 인증 옵션을 익명으로 설정한 경우에만 사용할 수 있습니다.
익명 사용자에 대한 메일 ID를 입력합니다.
서비스 계정
이 필드는 사용자 인증 옵션을 외부 사용자로 설정한 경우에만 사용할 수 있습니다.
외부 사용자에 대한 서비스 계정을 입력합니다. 이 필드는 사용자 인증 옵션을 외부 사용자로 설정한 경우 필수입니다.
참조로 전달
이 체크 박스를 선택하여 아카이브 디렉토리에 파일을 준비하고 헤더에 참조로 전달합니다.
원격 스트리밍
이 체크 박스를 선택하여 처리 시 원격 서버에서 FTP 파일을 직접 스트리밍합니다. 이 옵션을 선택한 경우 아카이브 디렉토리는 원격 FTP 서버 시스템에서 원격 디렉토리입니다. 따라서, 아카이브 디렉토리를 FTP 사용자 디렉토리와 상대적으로 지정해야 합니다.
파일 마스크
이 파일을 선택하기 위한 정규 표현식을 입력합니다. 기본값은 *.*입니다.
관리 서버
이 필드는 클러스터화된 도메인에서만 사용할 수 있습니다.
폴링 서버 역할을 할 관리 서버를 선택합니다. 모든 관리 서버는 메시지를 처리할 수 있지만 그 중 하나만 메시지를 폴링할 수 있습니다.
폴링 간격
폴링 간격(초)을 입력합니다. 기본값은 60입니다.
읽기 제한
폴링 스윕당 읽을 최대 메시지 수를 지정합니다. 제한 없음을 지정하려면 0을 입력합니다. 기본값은 10입니다.
읽기 후 작업
메시지를 읽은 후의 처리 방법을 선택합니다.
  • 아카이브 - 메시지가 보관됩니다.
  • 삭제 - 메시지가 삭제됩니다.
전송 모드
전송 모드로 ASCII 또는 이진을 선택합니다.
아카이브 디렉토리
읽기 후 작업 옵션이 아카이브로 설정된 경우 아카이브 위치의 경로를 지정합니다. 이 필드는 참조로 전달 옵션을 선택한 경우 필수입니다.
주: 아카이브, 다운로드 및 오류 디렉토리는 절대 경로이며 자동으로 생성됩니다. 상대 경로를 지정하는 경우 해당 파일은 WebLogic Server를 시작하는 Java 프로세스와 상대적으로 생성됩니다.
다운로드 디렉토리
파일을 전송하는 동안 파일을 다운로드할 로컬 시스템의 디렉토리를 입력합니다.
주: 아카이브, 다운로드 및 오류 디렉토리는 절대 경로이며 자동으로 생성됩니다. 상대 경로를 지정하는 경우 해당 파일은 WebLogic Server를 시작하는 Java 프로세스와 상대적으로 생성됩니다.
오류 디렉토리
문제가 존재하는 경우 메시지를 게시할 위치를 입력합니다.
주: 아카이브, 다운로드 및 오류 디렉토리는 절대 경로이며 자동으로 생성됩니다. 상대 경로를 지정하는 경우 해당 파일은 WebLogic Server를 시작하는 Java 프로세스와 상대적으로 생성됩니다.
요청 인코딩
FTP 전송 요청 시 기본 UTF-8을 문자 집합 인코딩으로 적용합니다.
하위 디렉토리 스캔
이 체크 박스를 선택하면 모든 디렉토리를 반복해서 스캔합니다.
도착순으로 정렬
이 체크 박스를 선택하면 도착한 순서대로 이벤트를 전달합니다.
시간 초과
연결이 끊기기 전의 소켓 시간 제한(초)을 입력합니다. 0을 입력하면 시간 초과가 없습니다.
재시도 횟수
FTP 접속 실패에 대한 재시도 횟수를 지정합니다.

마친 후에
