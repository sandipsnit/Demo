파일 전송 구성 페이지

프록시 서비스 생성/편집 - 전자 메일 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 파일 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-5에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-5 프록시 서비스 생성/편집 - 파일 전송 구성 페이지
옵션
생성 또는 편집
파일 마스크
프록시 서비스가 폴링해야 하는 파일을 지정합니다. URI가 디렉토리이고 *.*가 지정된 경우 서비스는 디렉토리의 모든 파일을 폴링합니다. 파일 마스크에는 와일드 카드 문자 * 및 ?만 허용됩니다. 정규 표현식은 지원되지 않습니다.
관리 서버
이 필드는 클러스터화된 도메인에서만 사용할 수 있습니다.
폴링 서버 역할을 할 관리 서버를 선택합니다. 모든 관리 서버는 메시지를 처리할 수 있지만 그 중 하나만 메시지를 폴링할 수 있습니다.
폴링 간격
폴링 간격(초)을 입력합니다. 기본값은 60입니다.
읽기 제한
폴링 스윕당 읽을 최대 메시지 수를 지정합니다. 제한 없음을 지정하려면 0을 입력합니다. 기본값은 10입니다.
도착순으로 정렬
이벤트가 도착 순서대로 전달되도록 지정하려면 이 체크 박스를 선택합니다.
이 옵션이 클러스터화된 환경에서 실행된 프록시 서비스에 대해 선택된 경우 메시지는 항상 동일한 서버로 전송됩니다. 즉, 이 옵션이 선택된 경우 서버 전체의 로드 밸런싱은 무시됩니다.
하위 디렉토리 스캔
모든 디렉토리를 반복해서 스캔하려면 이 체크 박스를 선택합니다.
참조로 전달
이 체크 박스를 선택하여 아카이브 디렉토리에 파일을 준비하고 헤더에 참조로 전달합니다.
읽기 후 작업
읽은 다음 메시지에 대해 수행할 작업을 선택합니다.
  • 아카이브 - 메시지가 보관됩니다.
  • 삭제 - 메시지가 삭제됩니다.
스테이지 디렉토리
파일을 처리하는 동안 임시로 파일을 보관할 중간 디렉토리를 입력합니다.
폴링 디렉토리(file:///c:/dir1/dir2와 같은 파일 전송 프록시 서비스의 URI로 식별되는 디렉토리) 내에 스테이지 디렉토리를 추가하지 마십시오.
아카이브 디렉토리
읽기 후 작업 옵션이 아카이브로 설정된 경우 아카이브 위치의 경로를 지정합니다. 아카이브 디렉토리 필드는 참조로 전달 필드를 선택한 경우에도 필수 필드입니다.
폴링 디렉토리 내에 아카이브 디렉토리를 추가하지 마십시오.
오류 디렉토리
문제가 존재하는 경우 메시지 및 첨부 파일이 게시되는 위치를 입력합니다.
폴링 디렉토리 내에 오류 디렉토리를 추가하지 마십시오.
요청 인코딩
기본값 UTF-8을 파일 전송 요청을 위한 문자 집합 인코딩으로 적용하거나 다른 문자 집합 인코딩을 입력합니다.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 프록시 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
