일반 구성 페이지

프록시 서비스 생성/편집 - 일반 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 프록시 서비스에 대한 일반 구성 설정을 수정할 수 있습니다.

프록시 서비스를 생성하면 이 페이지가 서비스 구성을 위한 일련의 페이지 중 첫번째 페이지로 표시됩니다. 이 페이지 이후 표시되는 페이지는 이 페이지에서 선택하는 사항에 따라 달라집니다.

표 19-1에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-1 프록시 서비스 생성/편집 - 일반 구성 페이지
옵션
생성 또는 편집
서비스 이름
프록시 서비스에 대한 고유 이름을 입력합니다.

리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
설명
설명을 입력합니다.
서비스 유형
생성할 프록시 서비스 유형을 선택합니다.
  • WSDL 웹 서비스 - WSDL 기반 프록시 서비스를 생성하려면 이 옵션을 선택합니다. 그런 다음 경로별로 정규화된 WSDL 이름을 입력합니다(예: myProject/myFolder/myWSDL). 또는 찾아보기를 눌러 WSDL 정의 선택 페이지에서 WSDL을 선택합니다.

    • (포트 또는 바인딩) - 실제 전송 주소를 설명하려면 포트 이름(WSDL에 정의)을 입력하거나 전송 주소에 매핑하려면 바인딩 이름(WSDL에 정의)을 입력합니다. 찾아보기를 사용하여 WSDL을 선택하면 WSDL 정의 선택 페이지에 정의된 포트 및 바인딩이 나열됩니다.

주: 작업에 SOAP 본문 유형을 사용하려는 경우 동일한 입력 메시지를 가진 두 개의 작업이 WSDL에 없는지 확인합니다. 동일한 입력 메시지를 검사하여 SOAP 본문 유형 작업을 고유하게 식별할 수 없습니다.
  • 전송 유형의 서비스 - JEJB 프록시 서비스를 생성하려면 이 옵션을 선택합니다.
  • 메시징 서비스 - 한 데이터 유형의 메시지를 받아서 다른 데이터 유형의 메시지로 응답할 수 있는 서비스를 생성하려면 이 옵션을 선택합니다. 이러한 교환은 요청/응답이거나 단방향일 수 있습니다.

    • HTTP GET은 XML 서비스 및 메시징 서비스 유형에서만 지원됩니다.

  • 모든 SOAP 서비스 - 명시적으로 정의된 구체적 인터페이스가 없는 SOAP 서비스를 생성하려면 이 옵션을 선택합니다.

    • 드롭다운 목록에서 SOAP 1.1 또는 SOAP 1.2를 선택하여 사용할 SOAP 버전을 지정합니다.

  • 모든 XML 서비스 - 명시적으로 정의된 구체적 인터페이스가 없는 XML 서비스를 생성하려면 이 옵션을 선택합니다.

    • HTTP GET은 XML 서비스 및 메시징 서비스 유형에서만 지원됩니다.

  • 비즈니스 서비스 - 선택한 비즈니스 서비스로 경로 지정되는 경로 지정 노드를 사용하여 프록시 서비스를 생성하려면 이 옵션을 선택합니다. 전송 입력 비즈니스 서비스에서는 프록시 서비스를 생성할 수 없습니다. DSP 전송 비즈니스 서비스에서 프록시 서비스를 생성하는 경우 Oracle Service Bus는 프록시 서비스에 대해 DSP 전송을 사용할 수 없기 때문에 프록시 서비스의 전송 유형을 HTTP로 전환합니다. 프록시 서비스의 전송 유형은 사용 가능한 다른 전송으로 변경할 수 있습니다.

    • 경로(프로젝트/폴더) 및 비즈니스 서비스 이름을 입력하거나 찾아보기를 눌러 비즈니스 서비스 선택 페이지에서 비즈니스 서비스를 선택합니다.

서비스 유형(계속)
  • 프록시 서비스 - 기존 프록시 서비스를 복제하려면 이 옵션을 선택합니다. Oracle Service Bus는 다중 서비스에 동일한 URI를 사용할 수 없으므로 복제된 서비스의 경우 URI를 변경해야 합니다.
주: 다른 서비스에서 서비스를 생성할 경우 경고 규칙은 다음 방법으로 유지 관리됩니다.
    • 프록시 서비스를 비즈니스 서비스에서 생성하거나 비즈니스 서비스를 프록시 서비스에서 생성할 경우 경고 규칙이 제거됩니다(해당하는 경우).
    • 프록시 서비스를 다른 프록시 서비스에 생성하거나 비즈니스 서비스를 다른 비즈니스 서비스에서 생성할 경우 경고 규칙이 유지됩니다(존재하는 경우).
서비스 키 제공자
필요한 경우 서비스 키 공급자의 경로(프로젝트/폴더) 및 이름을 입력하거나 찾아보기를 눌러 서비스 키 공급자 선택 페이지에서 선택합니다.
서비스 키 제공자는 다음과 같이 특정한 경우에만 필요합니다.
  • 프록시 서비스가 클라이언트 인증서 인증이 필요한 HTTPS 서비스에 메시지를 경로 지정하는 아웃바운드 양방향 TLS/SSL
  • 일부 웹 서비스 보안 시나리오(예: 프록시 서비스가 메시지를 암호화해야 하는 경우)
웹 서비스 보안 가능 프록시 서비스를 추가하려면 WS 정책 첨부 파일이 있는 WSDL(포트 또는 바인딩)에서 프록시 서비스를 생성해야 합니다.
자세한 내용은 서비스 키 제공자를 참조하십시오. 서비스 키 제공자를 생성하는 방법에 대해 알아보려면 서비스 키 제공자 추가를 참조하십시오.
레지스트리에 게시
이 옵션은 기본 UDDI 레지스트리가 존재하는 경우에만 나타납니다.
프록시 서비스를 기본 레지스트리에 자동으로 게시하려면 이 옵션을 선택합니다. 자세한 내용은 자동 게시 사용을 참조하십시오.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 프록시 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
