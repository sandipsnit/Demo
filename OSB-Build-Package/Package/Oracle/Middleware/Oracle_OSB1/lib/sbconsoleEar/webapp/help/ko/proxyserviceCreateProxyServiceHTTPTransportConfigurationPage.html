HTTP 전송 구성 페이지

HTTP 전송은 HTTP 및 HTTPS 끝점을 모두 지원합니다.

프록시 서비스 생성/편집 - HTTP 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 HTTP 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-7에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-7 프록시 서비스 생성/편집 - HTTP 전송 구성 페이지
옵션
생성 또는 편집
HTTPS 필요
인바운드 HTTPS 끝점에 대해 이 체크 박스를 선택합니다.
자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Configuring Transport-Level Security"를 참조하십시오.
인증
다음 중 하나를 선택합니다.
  • 없음 - 인증이 필요 없도록 지정합니다.
  • 기본 - 이 서비스에 액세스하기 위해 기본 인증이 필요한 경우 지정합니다.

    • 기본 인증을 지정하면 WebLogic Server는 LDAP(Lightweight Directory Access Protocol) 디렉토리 서비스 및 Windows Active Directory와 같은 보안 영역에 구성된 인증 제공자에 대해 사용자 이름과 비밀번호를 사용하여 클라이언트를 인증합니다. 클라이언트는 HTTP 요청 헤더를 통해 사용자 이름과 비밀번호를 보내야 합니다.

    HTTP를 통한 기본 인증은 비밀번호가 일반 텍스트로 전송되기 때문에 사용하지 않는 것이 좋습니다. 그러나 HTTPS는 암호화된 채널을 제공하므로 HTTPS를 통해 비밀번호를 전송하는 것은 안전합니다.

    경고: 기본적으로 모든 사용자(인증된 사용자 및 익명 사용자)가 프록시 서비스에 액세스할 수 있습니다. 프록시 서비스에 액세스할 수 있는 사용자를 제한하려면 전송 레벨 권한 부여 정책을 생성합니다. 전송 레벨 액세스 정책 편집을 참조하십시오.

  • 클라이언트 인증서 - 암호화된 통신 및 강력한 클라이언트 인증(양방향 SSL)을 지정합니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Configuring Transport-Level Security"를 참조하십시오.
  • 사용자 정의 인증 - 인증 토큰이 HTTP 헤더에 포함되도록 지정합니다. 클라이언트 ID는 이 클라이언트가 제공한 토큰을 사용하여 설정됩니다. 해당 토큰을 Oracle Service Bus 사용자에게 매핑하는 ID 검증 제공자를 구성해야 합니다.

    • 사용자 정의 인증 토큰은 구성된 WebLogic Server ID 검증 제공자가 지원하는 활성 토큰 유형 중 하나일 수 있습니다.

작업 할당 정책
이 끝점의 작업 할당 정책을 선택합니다. 기본 작업 할당 정책을 사용하려면 공백으로 둡니다.
작업 할당 정책은 서비스 끝점에 사용할 WLS 작업 관리자의 인스턴스를 참조합니다. 작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
요청 인코딩
  • HTTP 인바운드 전송:

    • 콘텐츠 유형 헤더의 문자 집합 인코딩 매개변수가 클라이언트 요청에 지정되지 않은 경우 문자 집합 인코딩 매개변수를 입력합니다. 값을 입력하지 않으면 ISO-8859-1이 필드의 기본값으로 설정됩니다.

  • HTTP 아웃바운드 전송:

    • 요청 인코딩을 구성하지 않은 경우 Oracle Service Bus 런타임은 비즈니스 서비스를 요청하는 동안 가장 적합한 인코딩을 결정합니다. 비통과 시나리오의 경우 기본 문자 인코딩은 런타임 시 UTF-8입니다. 그러나, 통과 시나리오의 경우 런타임은 아웃바운드 응답으로 수신된 인코딩을 통과합니다.

응답 인코딩
기본값 ISO-8859-1을 HTTP 전송 응답에 대한 문자 집합 인코딩으로 적용하거나 다른 문자 집합 인코딩을 입력합니다.
인증 헤더
Oracle Service Bus에서 토큰을 추출할 HTTP 헤더(권한 부여 모두 제외)을 입력합니다. 이 필드는 사용자 정의 권한 부여 체크 박스를 선택한 경우에만 사용할 수 있습니다.
예를 들어, client-xyz-token과 같습니다.
인증 토큰 유형
인증 토큰 유형을 선택합니다. ID 검증 제공자에 구성된 활성 토큰 유형만 사용할 수 있습니다. 자세한 내용은 사용자 정의 토큰의 ID 검증 제공자 구성을 참조하십시오. 이 필드는 사용자 정의 인증 체크 박스를 선택한 경우에만 사용할 수 있습니다.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 프록시 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
