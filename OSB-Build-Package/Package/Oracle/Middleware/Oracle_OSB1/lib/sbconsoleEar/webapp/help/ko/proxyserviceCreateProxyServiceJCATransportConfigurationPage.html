JCA 전송 구성 페이지

이 페이지에서는 JCA 전송 프로토콜을 사용하여 전송 설정을 구성할 수 있습니다. JCA 전송 사용에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV910에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "JCA Transport"를 참조하거나 JCA 바인딩 리소스에서 프록시 서비스 생성을 참조하십시오.

표 19-8 JCA 전송 구성
옵션
설명
JCA 파일
JCA 바인딩을 선택하려면 찾아보기를 누릅니다. JCA 바인딩에 따라 서비스의 다양한 요소(예: 사용되는 어댑터에 대한 세부 정보, WSDL과 TopLink 또는 EclipseLink 매핑 파일에 대한 바인딩, 서비스에 필요한 활성화/상호 작용 사양 속성)가 정의됩니다.
적합한 JCA 바인딩을 선택하면 나머지 전송 구성 필드가 사용 가능한 상태로 설정됩니다.
JCA 바인딩에 대한 자세한 내용은 12장, JCA 바인딩을 참조하십시오.
어댑터 이름
JCA 서비스가 사용할 어댑터 이름을 표시하는 읽기 전용 값입니다.
어댑터 유형
어댑터 유형을 표시하는 읽기 전용 값입니다.
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않으면 기본 작업 관리자가 사용됩니다.
작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
JNDI 서비스 계정
JNDI 서비스 계정은 JNDI 컨텍스트 보안을 위한 계정이며 EIS 어댑터에서 관리하는 접속 팩토리에 액세스하는 데 사용됩니다. 찾아보기를 누르고 서비스 계정을 선택합니다. 서비스 계정을 지정하지 않을 경우 익명 제목이 사용됩니다.
JCA 비즈니스 서비스의 경우 정적 또는 통과 등 구성 가능한 JNDI 서비스 계정의 유형에 대한 제한이 없지만 런타임에서 사용자 이름 및 비밀번호에 액세스할 수 있어야 합니다. JCA 프록시 서비스는 정적 JNDI 서비스 계정만 사용할 수 있습니다.
JNDI 서비스 계정에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV916에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "JCA Transport" 장, "Security"를 참조하십시오.
끝점 속성
이 필드에서 끝점 속성에 값(예: 서비스에 사용되는 어댑터 유형에 대한 재시도 횟수)을 지정할 수 있습니다.
지원되는 JCA 끝점 속성 목록은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1841에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Endpoint Properties"를 참조하십시오.
동적 끝점 속성
이 옵션을 통해 JCA 호환 서비스로 요청 매개변수를 전달할 수 있습니다. 예를 들어, 동적 끝점 속성을 사용하여 Oracle JCA Adapter for Database로 데이터베이스 질의 매개변수를 전달할 수 있습니다.
매개변수를 사용하는 질의에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=TKADP2117에서 Oracle Fusion Middleware User's Guide for Technology Adapters의 "Oracle JCA Adapter for Database"를 참조하십시오.
제공할 동적 끝점 속성 각각에 대한 이름/값 쌍을 입력합니다. 그러면 끝점 속성 키가 질의 매개변수 이름과 일치됩니다.
항상 JCA 파일의 구성 사용
이 옵션은 활성화 사양 속성(프록시 서비스)과 상호 작용 사양 속성(비즈니스 서비스)이 항상 JCA 파일에서 사용되는지 여부를 결정합니다.
이 옵션이 선택되면(기본값) JCA 전송에서 JCA 파일의 활성화/상호 작용 사양 속성을 사용하여 JCA 프레임워크와 상호 작용합니다.
이 옵션이 선택 해제되면 활성화/상호 작용 사양 속성을 무효화할 수 있습니다.
이 옵션 사용이 재배치에 미치는 영향은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV940에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "JCA Transport" 장, "Endpoint Redeployment"를 참조하십시오.
작업 이름
선택한 WSDL 작업의 읽기 전용 이름을 표시합니다. 작업에 자체 활성화/상호 작용 사양 속성이 포함될 수 있으며 활성화/상호 작용 사양 속성 필드에 표시됩니다.
활성화/상호 작용 사양 속성
활성화 사양 속성은 프록시 서비스의 필드 이름입니다. 상호 작용 사양 속성은 비즈니스 서비스의 필드 이름입니다.
이 서비스가 EIS 응용 프로그램에서 호출된 인바운드 서비스인 경우 이 필드에는 [작업 이름] 필드에 표시된 JCA 인바운드 작업에 대한 활성화 사양 속성이 표시됩니다.
항상 JCA 파일의 구성 사용을 선택 해제한 경우 활성화/상호 작용 사양 속성을 무효화할 수 있습니다.
주: Oracle Adapter Suite 어댑터의 경우 활성화/상호 작용 사양 속성이 읽기 전용으로 표시됩니다. Oracle Adapter Suite 어댑터는 구성을 저장하고 있으며 Oracle Adapter Suite 관리 툴에서 변경해야 합니다.
접속 속성(레거시)
비관리 모드 접속 속성(이번 릴리스에서는 사용되지 않음)을 사용하는 레거시 JCA 서비스에 대한 자세한 내용은 http://download.oracle.com/docs/cd/E13159_01/osb/docs10gr3/jcatransport/transport.html#wp1105451에서 접속 구성 옵션을 참조하십시오.

끝점 및 활성화/상호 작용 사양 속성에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=TKADP에서 Oracle Fusion Middleware User's Guide for Technology Adapters를 참조하십시오.
