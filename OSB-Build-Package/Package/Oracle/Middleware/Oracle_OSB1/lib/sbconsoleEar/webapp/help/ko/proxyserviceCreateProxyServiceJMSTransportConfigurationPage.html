JMS 전송 구성 페이지

프록시 서비스 생성/편집 - JMS 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "JMS Transport"를 참조하십시오.

이 페이지에서는 JMS 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-10에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-10 프록시 서비스 생성/편집 - JMS 전송 구성 페이지
옵션
생성 또는 편집
대상 유형
다음 중 하나를 선택합니다.
  • 대기열(지점 간 대상 유형의 경우)
  • 항목(게시/가입 대상의 경우)
응답 필요
이 옵션은 대상 유형으로 대기열을 선택한 경우에만 사용할 수 있습니다.
아웃바운드 메시지가 전송된 후 응답을 요청하도록 지정하려면 이 옵션을 선택합니다.
응답 패턴
이 옵션은 [응답 필요] 체크 박스가 선택된 경우에만 사용할 수 있습니다.
다음 중 하나를 선택합니다.
  • Oracle WebLogic Server에서 실행되는 JAX-RPC 서비스에 대해서는 JMSMessageID를 선택합니다.
  • 다른 모든 서비스에 대해 JMSCorrelationID를 선택합니다. 이 옵션을 선택할 경우 응답 URI도 입력해야 합니다.
응답 메시지 유형
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
다음 중 하나를 선택합니다.
  • 바이트(해석되지 않은 바이트 스트림의 경우)
  • 텍스트(텍스트 메시지의 경우)
이 옵션은 응답 메시지 유형으로 Java를 선택한 경우 사용 안함으로 설정됩니다.
클라이언트 Jar
이 옵션은 해당 서비스가 Java 요청 유형을 가진 메시징 서비스인 경우 사용할 수 있습니다. Java 객체가 포함된 메시지를 대기열에서 빼는 데 사용할 클라이언트 JAR을 선택하십시오. 클라이언트 JAR은 클래스 경로에 있는 경우 선택할 수 있습니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Sending and Receiving Java Objects in Messages"를 참조하십시오.
작업 할당 정책
이 끝점에 대해 작업 할당 정책에 사용할 Oracle WebLogic Server 작업 관리자의 인스턴스를 선택합니다. 다른 작업 관리자가 존재하지 않을 경우 기본 작업 관리자가 사용됩니다.
예를 들어, 비즈니스 서비스에 JMS 전송 프로토콜이 존재하는 경우 비즈니스 서비스 끝점은 특정 작업 할당 정책과 연결할 수 있는 MDB(메시지 파생 Bean) JAR 파일입니다.
작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
요청 인코딩
인코딩 요청에 대한 문자 집합을 입력합니다. 기본값은 UTF-8입니다.
응답 인코딩
이 옵션은 [응답 필요] 체크 박스가 선택된 경우에만 사용할 수 있습니다.
인코딩 응답에 대한 문자 집합을 입력합니다. 기본값은 UTF-8입니다.
클라이언트 응답 시간 초과
이 옵션은 [응답 필요] 체크 박스가 선택된 경우에만 사용할 수 있습니다.
접속을 끊기 전에 서버 응답 대기 시간(초)을 입력합니다. 이 필드는 클라이언트가 동일한 도메인의 다른 프록시 서비스인 경우에만 적용됩니다.
응답 URI
이 옵션은 응답 상관 관계 패턴에 대해 JMSCorrelationID가 선택된 경우에만 사용할 수 있습니다.
다음 형식으로 응답 URI를 입력합니다.
jms://host:port/connection_factory/jndi_destination
다중 서버를 대상으로 지정하려면 다음 형식을 사용합니다.
jms://host1:port,host2:port/connection_factory/jndi_destination
다음과 같이 응답 URI에서 호스트와 포트를 생략할 수도 있습니다. 다음과 같습니다.
jms:///connection_factory/jndi_destination

호스트와 포트를 생략할 경우 로컬 서버에서 접속 팩토리/대상이 조회됩니다. 이 형식은 요청 URI가 외부 접속 팩토리/대상을 가리키지만 요청이 로컬 서버로 전송되도록 하려는 경우 유용합니다.

주: Oracle WebLogic Server에서는 "myqueues/myqueue"와 같이 JNDI 이름에 슬래시를 사용할 수 있지만 JNDI 이름에 슬래시가 포함되어 있으면 Oracle Service Bus에 필요한 URI 형식과 충돌하므로 이러한 이름을 사용할 수 없습니다. 이 문제를 해결하려면 JMS 외부 서버를 정의하고 URI에서 이 외부 서버를 참조하십시오. 자세한 내용은 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help(http://www.oracle.com/pls/as111170/lookup?id=WLACH)의 "Configure foreign servers"를 참조하십시오.

응답 접속 팩토리
이 옵션은 응답 상관 관계 패턴에 대해 JMSMessageID가 선택된 경우에만 사용할 수 있습니다.
응답 접속 팩토리 URI를 입력합니다.
접속 팩토리를 지정하지 않을 경우 요청의 접속 팩토리가 응답에 사용됩니다.
JMS 서비스 계정
JMS 서버에서 관리하는 JMS 리소스에 사용할 서비스 계정을 선택합니다. 서비스 계정은 요청과 응답에 사용되는 사용자 ID 및 비밀번호의 별칭 리소스입니다. JNDI와 JMS용으로 동일한 서비스 계정이 사용됩니다.
자세한 내용은 서비스 계정을 참조하십시오.
SSL 사용
TLS/SSL 접속을 통해 요청을 제출한 경우에만 이 옵션을 선택합니다.
TLS/SSL(Secure Sockets Layer)은 네트워크를 통해 접속하는 두 개의 응용 프로그램이 상호 간의 ID를 인증할 수 있게 하고 응용 프로그램 간에 교환되는 데이터를 암호화하여 보안 접속을 제공합니다. 인증을 사용하면 서버는 물론 필요에 따라 클라이언트도 네트워크 접속의 반대쪽에 있는 응용 프로그램의 ID를 확인할 수 있습니다. 또한 관리자가 대상의 JNDI 항목에 대해 액세스 제어를 설정하여 개별 JMS 대상(대기열 또는 항목)에 대한 액세스를 제한한 경우 서비스는 JNDI 트리에서 항목을 조회할 때 인증을 수행해야 합니다.
메시지 선택기
메시지 선택기 표현식을 입력합니다.
표현식과 일치하는 속성이 있는 메시지만 처리됩니다.
JMS 클라이언트 ID

항목의 경우에만 가입자에 대해 사용할 클라이언트 ID를 입력합니다. 입력된 값이 없으면 클라이언트 ID가 자동으로 생성됩니다. 사용자 정의 ID를 지정하면 서비스를 모니터할 때 이 구성 요소를 식별하는 데 도움이 됩니다. 영구 가입자에게만 유효합니다.

JMS 클라이언트 ID는 항목의 가입자에 대한 가입자 이름과 ClientID 값을 생성하는 데 사용되는 여러 MDB 기술자 중 하나입니다. JMS 클라이언트 ID를 정의하면 가입자 이름이나 클라이언트 ID를 확인하여 특정 항목의 가입자를 쉽게 식별하고 확인할 수 있습니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "JMS Transport"를 참조하십시오.
영속 가입
이 옵션은 대상 유형으로 항목을 선택한 경우에만 사용할 수 있습니다.
영속 가입인 경우 이 체크 박스를 선택하고 영속 가입이 아닌 경우에는 선택하지 않습니다.
재시도 횟수
메시지가 오류 대상으로 이동되기 전의 메시지 전달 재시도 횟수를 입력합니다. 이 필드는 Oracle WebLogic Server JMS 대상에만 적용됩니다.
재시도 간격
롤백 또는 복구된 메시지를 재전달하기 전의 시간(밀리초)을 입력합니다. 이 필드는 Oracle WebLogic Server JMS 대상에만 적용됩니다.
오류 대상
재전달 제한 값에 도달한 메시지에 대한 대상 브리지 대상 이름을 입력합니다. 이 필드는 Oracle WebLogic Server JMS 대상에만 적용됩니다.
만료 정책
대상에서 만료된 메시지가 발생할 때 사용할 만료 정책을 선택합니다. 이 필드는 Oracle WebLogic Server JMS 대상에만 적용됩니다.
XA 필요
접속 팩토리가 XA인 경우 이 체크 박스를 선택합니다.
원격 접속 팩토리를 사용할 수 없는 경우 이 값이 계정에 적용됩니다. 접속 팩토리를 사용할 수 있고 이 값이 True인 경우 접속 팩토리가 트랜잭션으로 정의되어 있는지 확인하십시오.
트랜잭션 시간 초과
JMS 프록시 서비스에서 트랜잭션이 처리될 때까지 대기해야 하는 시간(초)을 입력합니다. 이 옵션은 XA 접속 팩토리를 사용하는 서비스에만 적용됩니다. 기본값은 600초입니다.
토픽 메시지 분배
[대상 유형]으로 [항목]을 선택할 경우 이 옵션을 사용할 수 있습니다. 토픽 메시지 분배에서는 인바운드 JMS 메시지를 처리하고 고가용성 및 장애 조치를 제공할 수 있는 다양한 옵션을 제공합니다.

메시지 방식 Bean의 수신 JMS 메시지 처리 방식을 결정하려면 다음 속성 중 하나를 선택합니다.
  • 응용 프로그램당 한 부(기본값) – JMS 프록시 서비스가 클러스터에 배치된 경우 이 옵션을 선택하면 인바운드 JMS 메시지가 클러스터의 사용 가능한 서버 중 하나에서만 처리됨으로써 고가용성과 확장성을 제공합니다.
  • 서버당 한 부 - 클러스터에 있는 모든 멤버의 프록시 서비스가 항목에 게시된 인바운드 JMS 메시지를 수신하도록 하려면 이 옵션을 선택합니다.
  • 호환성 - 클러스터에 있는 특정 관리 서버 또는 전체 관리 서버에서 인바운드 JMS 메시지가 처리되도록 하려면 이 옵션을 선택합니다. 이 속성을 선택할 경우 대상 서버를 선택합니다.
주: 항목 대상이 10.3.4 이전 버전의 Oracle WebLogic Server에 배치된 경우 [호환성] 옵션만 사용할 수 있습니다.
[토픽 메시지 분배]로 "한 부" 옵션을 선택하면 JMS 접속 팩토리에서 구성된 가입 공유 정책 및/또는 클라이언트 ID 정책이 무효화됩니다.
대상
이 옵션은 토픽 메시지 분배 옵션에 호환성을 선택한 경우 Oracle Service Bus 클러스터에서만 사용할 수 있습니다. 수신 JMS 메시지를 처리할 대상 서버를 선택합니다. [토픽 메시지 분배]로 "한 부" 옵션을 선택할 경우 이 필드가 클러스터 이름에 표시됩니다.

대상을 설정하지 않을 경우 클러스터의 각 관리 서버에 있는 JMS 프록시 서비스 인스턴스(항목을 기반으로 메시지를 읽음)로 메시지 한 부가 전송됩니다.
JNDI 시간 초과
JNDI 접속 시간 초과(초)는 JNDI 트리에서 대상 또는 접속 팩토리를 조회하는 동안 사용됩니다.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 프록시 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
