MQ 전송 구성 페이지

프록시 서비스 생성/편집 - MQ 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "MQ Transport"를 참조하십시오.

시작하기 전에

MQ 접속 리소스를 구성합니다. MQ 접속을 참조하십시오.

MQ 전송을 구성하려면

이 페이지에서는 고유 MQ 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-11에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-11 프록시 서비스 생성/편집 - MQ 전송 구성 페이지
옵션
생성 또는 편집
폴링 간격
폴링 간격(밀리초)을 입력합니다. 기본값은 1000입니다.
응답 필요
아웃바운드 메시지가 전송된 후 응답을 요청하도록 지정하려면 이 옵션을 선택합니다.
응답 상관 관계 패턴
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
응답 상관 관계 패턴을 MessageID 또는 CorrelationID 기준으로 할지 여부를 지정합니다.
MQ 응답 URI
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
응답이 게시되어야 하는 대상입니다. 끝점 URI와 동일한 형식의 응답 URI를 입력합니다. mq://<local-queue-name>?conn=<mq-connection-resource-ref>
응답 메시지 유형
이 옵션은 응답 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
다음 중 하나를 선택합니다.
  • 바이트(해석되지 않은 바이트 스트림의 경우)
  • 텍스트(텍스트 메시지의 경우)
작업 할당 정책
이 끝점에 대한 작업 할당 정책을 선택합니다.
작업 할당 정책은 서비스 끝점에 사용할 WLS 작업 관리자의 인스턴스를 참조합니다. 작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
백아웃 임계값
메시지를 전달하지 못한 편지 URI 필드에서 지정한 대기열로 재지정하기 전에 파이프 라인이 메시지를 재시도해야 하는 횟수를 표시하는 값을 입력합니다.
이 필드에 값을 지정하지 않을 경우 재시도 없이 메시지가 전달하지 못한 편지 대기열로 재지정됩니다.
MQ 전달하지 못한 편지 URI
백아웃 임계값 필드에 지정된 재시도 횟수를 시도한 후 요청 메시지가 재지정되어야 하는 전달하지 못한 편지 대기열의 URI를 입력합니다.
이 필드에 값을 지정하지 않을 경우 백아웃 임계값 필드에 지정된 횟수를 재시도한 후 메시지가 대기열로 반환되고 각 폴에 대한 MQ 전송에서 무시됩니다. 전달하지 못한 편지 URI는 끝점 URI와 동일한 형식을 사용합니다.
끝점 URI `GET' 옵션
다음 중에서 MQ GET 메시지 옵션을 입력합니다.
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
다중 옵션을 구분하기 위해 "|" 또는 "+"를 사용할 수 있습니다. 예를 들어 다음과 같이 지정할 수 있습니다.
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
MQ GET 메시지 옵션은 인바운드 대기열에서 메시지를 읽을 때 적용됩니다.
프로세스 RFH2 헤더
메시지 페이로드에서 WebSphere MQ RFH2 헤더의 구문을 분석하고 RFH2 데이터를 포함하는 RFH2Headers 전송 헤더를 자동으로 생성하려면 이 옵션을 선택합니다.
이 옵션을 선택하지 않으면 페이로드가 받는 즉시 전달됩니다.

MQ 전송에서 RFH2 헤더를 처리하는 방법은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "About RFH2 Headers"를 참조하십시오.

마친 후에

다음을 눌러 다음 페이지에서 이 서비스를 계속 구성하거나 마지막을 눌러 프록시 서비스 생성/편집 - 요약 페이지에서 이 구성을 검토 및 저장합니다.
