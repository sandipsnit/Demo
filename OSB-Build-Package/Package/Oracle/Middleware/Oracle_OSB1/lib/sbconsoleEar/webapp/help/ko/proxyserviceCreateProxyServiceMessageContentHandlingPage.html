메시지 처리 페이지

프록시 서비스 생성/편집 - 메시지 처리 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 프록시 서비스에서 메시지 내용을 스트리밍할지 여부, 서비스에서 수신한 요청 메시지를 디코딩하고 보낸 응답 메시지를 인코딩할 방법 및 프록시 서비스에서 첨부 파일 내용을 메모리에 버퍼링하지 않고 MIME 첨부 파일을 스트리밍할지 여부를 지정할 수 있습니다.

이 페이지를 사용하면 프록시 서비스를 사용으로 설정하여 메시지 내용을 메모리에 저장하지 않고 스트리밍할 수 있습니다. 또한 프록시 서비스를 사용으로 설정하여 MTOM/XOP 형식으로 인바운드 메시지를 디코딩하고 구문 분석하며 적합할 경우 MTOM/XOP 형식을 사용하여 응답을 보낼 수 있습니다. SOAP MTOM(Message Transmission Optimization Mechanism)은 웹 서비스에서 이진 데이터를 보내고 받는 방법입니다. MTOM에서는 XOP(XML-binary Optimized Packaging)를 사용하여 이진 데이터를 전송합니다.

이 페이지를 사용하면 프록시 서비스를 사용으로 설정하여 MIME 첨부 파일 내용을 디스크 파일에 저장한 다음 첨부 파일 내용을 메모리에 버퍼링하지 않고 스트리밍 형식으로 데이터를 처리할 수도 있습니다. 따라서 프록시 서비스에서 큰 첨부 파일을 안정적이고 효과적인 방식으로 처리할 수 있습니다.

표 19-17에서는 프록시 서비스 생성/편집 - 메시지 처리 페이지를 사용하는 방법에 대해 설명합니다.

표 19-17 프록시 서비스 생성/편집 - 메시지 처리 페이지
옵션
생성 또는 편집
트랜잭션 필요
Oracle Service Bus가 트랜잭션 컨텍스트에서 프록시 서비스 메시지 흐름을 실행하도록 하려면 이 옵션을 선택합니다. 전역 트랜잭션이 존재할 경우 이 옵션을 선택하지 않더라도 전송 제공자가 요청에서 이 트랜잭션을 전달합니다. 전역 트랜잭션이 존재하지 않을 경우 메시지 흐름 런타임이 트랜잭션을 시작합니다. 메시지 흐름 런타임이 트랜잭션을 시작하면 서비스 구성이 검증되거나 실행(예: 보안 확인 또는 WS-I 검증)되기 전에, 그리고 메시지 흐름이 실행되기 전에 트랜잭션 컨텍스트가 시작되어 모든 처리 및 실행이 트랜잭션 컨텍스트에서 진행됩니다.
메시지 흐름 런타임이 트랜잭션을 시작하면 서비스 품질은 [정확히 1회]입니다. 하지만 서비스 콜아웃 또는 게시 작업에 기본값인 [최선형]으로 설정된 아웃바운드 서비스 품질 매개변수가 있을 경우 Oracle Service Bus는 트랜잭션 컨텍스트 외부에서 해당 작업을 실행합니다. Oracle Service Bus가 동일한 요청 트랜잭션 컨텍스트에서 해당 작업을 실행하도록 하려면 해당 작업에 대한 서비스 품질을 [정확히 1회]로 설정하십시오.
서비스는 이 옵션 설정에 관계없이 메시지 패턴(동기, 비동기, 단방향)을 유지합니다.
트랜잭션 시간 초과의 경우 Oracle WebLogic Server 콘솔에서 구성된 전역 트랜잭션 시간 초과 값이 적용됩니다.
트랜잭션 예외 사항
Oracle Service Bus는 실패한 트랜잭션에 대해 시스템 오류 처리기를 호출합니다. 사용자가 구성한 오류 처리기에서는 실패한 트랜잭션 예외 사항을 확인할 수 없습니다. 동기 패턴의 경우 트랜잭션 예외 사항은 응답을 통해 반환됩니다. 트랜잭션이 요청에서 커밋되는 비동기 패턴의 경우 예외 사항은 요청 스레드에서 다시 전송됩니다.
비동기 패턴의 경우 트랜잭션이 요청에서 커밋된 후 발생한 응답 오류는 트랜잭션에 영향을 끼치지 않습니다.
응답에 동일한 트랜잭션 사용
이 옵션은 단방향 및 비동기 메시징 패턴에만 적용됩니다.

이 옵션을 선택할 경우 Oracle Service Bus는 요청 스레드에서 응답 스레드로 트랜잭션 컨텍스트를 전달합니다.
이 옵션을 선택할 경우 초기 메시지 패턴 설정(예: 비동기 또는 단방향)에 관계없이 메시지 패턴이 [동기]로 자동 설정됩니다.
단방향 패턴에서와 같이 응답을 전송하기 전에 요청에 포함된 비즈니스 서비스의 트랜잭션을 커밋해야 할 경우 등에는 이 옵션을 사용하지 마십시오.
트랜잭션 시간 초과의 경우 Oracle WebLogic Server 콘솔에서 구성된 전역 트랜잭션 시간 초과 값이 적용됩니다.
내용 스트리밍
메시지 내용을 메모리에 저장하지 않고 스트리밍하려면 이 옵션을 선택합니다.
사용으로 설정됨 체크 박스를 선택하고 다음을 선택합니다.
  • 중간 내용을 메모리에 버퍼링할지(메모리 버퍼) 디스크 파일에 버퍼링할지(디스크 버퍼) 여부
  • 압축을 사용으로 설정할 지 여부
자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG314에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Streaming body Content"를 참조하십시오.
XOP/MTOM 지원
Oracle Service Bus에서는 다음과 같은 전송을 사용하여 XOP/MTOM을 지원합니다.
  • HTTP/S
  • 로컬
  • SB
프록시 서비스에서 MTOM/XOP 형식으로 인바운드 메시지를 디코딩하고 구문 분석하며 적합할 경우 MTOM/XOP 형식을 사용하여 응답을 보낼 수 있도록 하려면 사용으로 설정됨 체크 박스를 선택합니다. 이 옵션은 이전 릴리스 구성을 기반으로 하는 임포트한 프록시 서비스에는 사용 안함으로 설정됩니다.
XOP/MTOM 지원을 사용으로 설정하면 $header 및 $body 메시지 컨텍스트 변수의 이진 데이터 처리 방법을 다음 옵션 중에서 선택할 수 있습니다.
  • 참조로 이진 데이터 포함: (기본값) 인바운드 응답 메시지에서 $header 및 $body 메시지 컨텍스트 변수를 설정할 때 xop:Include 요소를 ctx:binary-content 요소로 대체합니다.
  • 값으로 이진 데이터 포함: 인바운드 응답 메시지에서 $header 및 $body 메시지 컨텍스트 변수를 설정할 때 xop:Include 요소를 해당 이진 데이터의 Base64로 인코딩된 텍스트 버전으로 대체합니다.
Java 콜아웃이나 MFL(Message Format Language) 변환에 데이터를 전달하는 경우처럼 이진 데이터에 직접 액세스해야 하는 경우에 참조로 이진 데이터 포함을 사용합니다.
다음과 같은 경우에 값으로 이진 데이터 포함을 사용합니다.
  • MTOM과 비MTOM 서비스 간에 브리징하려는 경우. 예를 들어 요청을 수신한 다음 비MTOM이 사용으로 설정된 서비스로 경로 지정되는 MTOM이 사용으로 설정된 프록시 서비스를 가정합니다. 이 옵션을 사용하면 Base64로 인코딩된 형식으로 XML의 이진 데이터를 보내는 기존 표준을 준수할 수 있습니다.
  • 이진 데이터 대신 사용할 base64binary가 필요한 XML 스키마에 대해 메시지 내용을 검증하려는 경우
프록시 서비스에 대해 XOP/MTOM 지원을 사용으로 설정하는 경우 모든 인바운드 메시지가 MTOM 형식일 필요는 없습니다. 대신 이 설정에서는 MTOM 형식의 메시지가 도착하면 프록시 서버에서 적절히 처리하도록 지정합니다. 또한 XOP/MTOM 지원에 대해 사용으로 설정되지 않은 프록시 서비스가 MTOM 형식의 메시지를 받으면 서비스에서 메시지를 거부하며 런타임 오류가 발생합니다.
첨부 파일
Oracle Service Bus에서는 다음 전송을 사용한 MIME 첨부 파일 스트리밍을 지원합니다.
  • HTTP/S
  • 로컬(첨부 파일 스트리밍이 사용으로 설정된 HTTP 프록시를 통해 연결된 경우)
프록시 서비스에서 MIME 첨부 파일을 스트리밍할 수 있도록 하려면 첨부 파일을 디스크로 페이징 체크 박스를 선택합니다. HTTP 프록시 서비스가 사용으로 설정되면 옵션이 프록시 서비스 인바운드 요청 메시지에 적용됩니다.
XOP/MTOM 지원을 선택하는 경우에는 XOP/MTOM 지원에서 참조로 이진 데이터 포함 옵션을 선택하는 경우에만 첨부 파일 옵션을 사용할 수 있습니다. 또한 첨부 파일이 포함된 페이로드는 RFC 822를 준수해야 합니다. 특히 인터넷 헤더를 포함하는 라인은 CRLF(캐리지 리턴 줄 바꿈)로 끝나야 합니다.
