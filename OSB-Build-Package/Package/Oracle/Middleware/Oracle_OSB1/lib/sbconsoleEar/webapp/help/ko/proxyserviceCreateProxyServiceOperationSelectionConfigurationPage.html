작업 선택 구성 페이지

프록시 서비스 생성/편집 - 작업 선택 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다. 생성한 서비스가 작업인 경우에만 이 페이지가 표시됩니다.

이 페이지에서는 WS-I 규격(SOAP 1.1 서비스 전용)을 강제 적용하고 이 프록시 서비스에서 호출할 작업을 결정하는 데 사용되는 선택 알고리즘을 선택할 수 있습니다. 이 옵션은 WSDL에서 정의된 SOAP 또는 XML 서비스에만 사용할 수 있습니다.

WSDL 사양은 받은 SOAP 메시지의 유형을 기반으로 호출할 작업을 계산하는 기본 알고리즘을 정의합니다. 그러나 다른 수단을 기반으로 작업을 선택해야 하는 경우(예: 성능 문제, 서명/암호화 문제 또는 기본 알고리즘이 해당되지 않는 경우)도 있습니다.

Oracle Service Bus는 추가 알고리즘을 제공합니다. 각 알고리즘은 동일한 패턴을 따르며 표현식 평가를 기반으로 정적 테이블에서 해당 작업을 조회하는 데 사용되는 값을 임포트합니다.

Oracle Service Bus는 일반적으로 인바운드 메시지에 데이터가 누락되어 작업을 결정할 수 없거나, 적합한 작업에 해당하지 않는 데이터가 존재하는 경우에 매우 관대합니다. 이러한 두 조건 모두 $operation이 비어 있게 됩니다. Oracle Service Bus는 이러한 메시지를 모두 거부하기보다, 컨텍스트에서 작업 변수를 초기화하지 않고 오히려 메시지를 계속 처리합니다.

그러나 보안 요구 사항은 프록시 서비스가 WSDL 기반이고 다음 조건 중 하나 이상이 참인 경우 강제 적용됩니다.

  • WSDL에는 WS 보안 정책이 있고 프록시는 즉시 활성 상태가 됩니다.
  • 프록시에는 메시지 레벨 사용자 정의 인증(사용자 정의 토큰 또는 사용자 이름/비밀번호)이 있습니다.

해당 조건이 충족되면, 런타임 시 작업 선택 알고리즘에 적합한 작업 이름이 반환되는지 확인합니다. 작업 선택에 널이 반환되거나 작업이 WSDL에 없으면 메시지가 거부되고 오류가 발생합니다.

표 19-16 프록시 서비스 생성/편집 - 작업 선택 구성 페이지
옵션
생성 또는 편집
WS-I 규격 강제 적용
SOAP 1.1 서비스 전용:
웹 서비스 상호 운용성 기구에서 정의한 기본 프로파일을 서비스에서 준수할지 여부를 지정하려면 이 체크 박스를 선택합니다.
서비스가 WS-I 규격인 경우 해당 서비스에서 보내고 받는 메시지에 대해 검사가 수행됩니다. 프록시의 경우 프록시에 의해 수신되는 요청 메시지에 대해 검사가 수행됩니다. 호출된 서비스(예: 서비스 콜아웃 작업 또는 경로 지정 노드를 통해 프록시에서 호출한 서비스)의 경우 해당 서비스에서 수신된 응답 메시지에 대해 검사가 수행됩니다. 이 서비스는 호출된 서비스의 WS-I 규격 속성이고, 호출된 서비스에서 수신된 메시지에 대해 확인이 수행되는지 여부를 결정하는 프록시가 없습니다. 호출된 서비스에 대해 WS-I 규격 테스트를 지정하는 경우 메시지 흐름이 응답 오류에 대해 fault를 생성합니다.
선택 알고리즘
다음 중 하나를 선택하고 추가 필수 단계를 수행합니다.
  • 전송 헤더 - 조회 값을 포함한 전송 헤더를 정의하려면 이 알고리즘을 선택합니다. 그런 다음 다음을 수행합니다.
    • 헤더 이름 필드에 호출할 작업을 선택하기 위한 키로 사용되는 값을 추출할 전송 헤더를 입력합니다.
    • 작업 매핑 필드의 값 필드에서 각 작업에 대한 값을 지정합니다. 이 값은 작업의 키로 사용됩니다.
  • SOAPAction 헤더 - 이 프록시 서비스와 관련된 WSDL에서 작업 매핑이 자동으로 수행되도록 지정하려면 이 알고리즘을 선택합니다.
  • WS-주소 지정 - SOAP 메시지의 SOAP 헤더에 있는 WS 주소 지정 작업 태그에 조회 값이 포함되도록 지정하려면 이 알고리즘을 선택합니다. 작업 매핑 필드의 값 필드에서 각 작업에 대한 값을 지정합니다. 이 값은 작업의 키로 사용됩니다.
  • SOAP 헤더 - SOAP 헤더에 대해 평가되는 XPath 표현식을 정의하려면 이 알고리즘을 선택합니다. 이렇게 하면 조회 값을 얻을 수 있습니다. 그런 다음 다음을 수행합니다.
    • XPath 표현식 필드에서 호출할 작업을 선택하기 위한 키로 사용되는 값을 추출할 XPath 표현식을 지정합니다.
    • 작업 매핑 필드의 값 필드에서 각 작업에 대한 값을 지정합니다. 이 값은 작업의 키로 사용됩니다.
  • SOAP 본문 유형 - 받은 SOAP 메시지 유형을 기반으로 호출할 작업을 계산하는 WSDL 사양으로 정의되는 기본 알고리즘입니다.

    • 프록시 서비스가 암호화된 본문을 사용하는 웹 서비스 보안 통과 시나리오에 대해 구성된 경우에는 SOAP 본문 유형 알고리즘을 선택할 수 없습니다. 이와 유사한 경고 사항이 암호화된 SOAP 헤더 통과에도 적용됩니다.

    입력 메시지를 검사하여 작업을 고유하게 식별할 수 없기 때문에 동일한 입력 메시지가 포함된 작업이 두 개인 WSDL이 존재하는 경우 작업에 대해 이 알고리즘을 선택하지 마십시오.

  • 페이로드 유형 - WSDL 포트 또는 WSDL 바인딩을 기반으로 하는 XML 서비스의 경우에만 사용할 수 있습니다.

호출할 작업을 선택하기 위한 키로 사용되는 값을 추출하는 전송 헤더를 입력합니다.

작업 매핑
이 옵션은 선택 알고리즘 옵션이 전송 헤더, WS 주소 지정 또는 SOAP 본문 유형으로 설정된 경우에만 사용할 수 있습니다.
값 필드에서 각 작업의 값을 지정합니다. 이 값은 작업의 키로 사용됩니다.
