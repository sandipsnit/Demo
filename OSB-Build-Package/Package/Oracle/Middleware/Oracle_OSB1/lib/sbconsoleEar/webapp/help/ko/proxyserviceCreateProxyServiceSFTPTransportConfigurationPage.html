SFTP 전송 구성 페이지

프록시 서비스 생성/편집 - SFTP 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV947에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "HTTP and Poller Transports"를 참조하십시오.

이 페이지에서는 SFTP 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-13에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-13 프록시 서비스 생성/편집 - SFTP 전송 구성 페이지
옵션
생성 또는 편집
사용자 인증
다음 중 하나를 선택합니다.
  • Username Password Authentication - 정적 서비스 계정을 이 인증 방법과 연관시키고 제공된 인증서를 사용하여 클라이언트를 인증하도록 지정합니다.
  • Host Based Authentication - 이 인증 방법을 사용할 때 사용자 이름과 서비스 키 제공자를 요청하도록 지정합니다. 알려진 호스트에서 접속하는 사용자는 호스트의 전용 키를 사용하여 인증합니다.
  • Public Key Authentication - 이 인증 방법을 사용할 때 사용자 이름과 서비스 키 제공자를 요청하도록 지정합니다. 모든 사용자는 고유한 전용 키를 가집니다.
서비스 계정
사용자의 서비스 계정을 입력하거나 찾아보기를 눌러 브라우저에서 서비스 계정을 선택합니다.
서비스 키 제공자
이 옵션은 Host Based 또는 Public Key Authentication을 선택한 경우에만 사용할 수 있습니다.
서비스 키 제공자 필드에 서비스 키 제공자를 입력합니다. 찾아보기를 눌러 브라우저에서 서비스 키 제공자를 선택할 수 있습니다. 이 필드는 필수 필드입니다.
사용자 이름
이 옵션은 Host Based 또는 Public Key Authentication을 선택한 경우에만 사용할 수 있습니다.
사용자 이름을 입력합니다.
참조로 전달
이 체크 박스를 선택하여 아카이브 디렉토리에 파일을 준비하고 헤더에 참조로 전달합니다.
원격 스트리밍
이 체크 박스를 선택하면 처리 시 SFTP 파일이 원격 서버에서 직접 스트리밍합니다. 이 옵션을 선택한 경우 아카이브 디렉토리는 원격 SFTP 서버 시스템에서 원격 디렉토리입니다. 따라서 아카이브 디렉토리를 SFTP 사용자 디렉토리와 상대적으로 지정해야 합니다.
파일 마스크
이 파일을 선택하기 위한 정규 표현식을 입력합니다. 기본값은 *.*입니다.
관리 서버
이 필드는 클러스터화된 도메인에서만 사용할 수 있습니다.
폴링 서버 역할을 할 관리 서버를 선택합니다. 모든 관리 서버는 메시지를 처리할 수 있지만 그 중 하나만 메시지를 폴링할 수 있습니다.
폴링 간격
특정 위치에서 파일이 폴링되는 간격(초)을 입력합니다. 기본값은 60입니다.
읽기 제한
폴링 스윕당 읽을 최대 메시지 수를 지정합니다. 제한 없음을 지정하려면 0을 입력합니다. 기본값은 10입니다.
읽기 후 작업
메시지를 읽은 후의 처리 방법을 선택합니다.
  • 아카이브 - 메시지가 보관됩니다.
  • 삭제 - 메시지가 삭제됩니다.
아카이브 디렉토리
읽기 후 작업 옵션이 아카이브로 설정된 경우 아카이브 위치의 경로를 지정합니다. 이 필드는 참조로 전달 옵션을 선택한 경우 필수입니다.
주: 아카이브, 다운로드 및 오류 디렉토리는 절대 경로이며 자동으로 생성됩니다. 상대 경로를 지정하는 경우 해당 파일은 WebLogic Server를 시작하는 Java 프로세스와 상대적으로 생성됩니다.
다운로드 디렉토리
파일을 전송하는 동안 파일을 다운로드할 로컬 시스템의 디렉토리를 입력합니다.
주: 아카이브, 다운로드 및 오류 디렉토리는 절대 경로이며 자동으로 생성됩니다. 상대 경로를 지정하는 경우 해당 파일은 WebLogic Server를 시작하는 Java 프로세스와 상대적으로 생성됩니다.
오류 디렉토리
문제가 존재하는 경우 메시지를 게시할 위치를 입력합니다.
주: 아카이브, 다운로드 및 오류 디렉토리는 절대 경로이며 자동으로 생성됩니다. 상대 경로를 지정하는 경우 해당 파일은 WebLogic Server를 시작하는 Java 프로세스와 상대적으로 생성됩니다.
요청 인코딩
기본값인 UTF-8을 SFTP 전송 요청을 위한 문자 집합 인코딩으로 적용합니다.
선호 암호화 모음
서버와 통신할 때 사용할 암호화 모음을 선택합니다.
기본값인 런타임 기본값 사용을 사용할 경우 지원되는 암호화 모음 목록이 서버로 전송되고 일치 항목을 찾을 때까지 각 암호화가 시도됩니다.
선호 데이터 무결성 알고리즘
데이터 무결성 검사를 위한 대량 해싱 알고리즘을 선택합니다.
기본값인 런타임 기본값 사용을 사용할 경우 Oracle Service Bus가 hmac-sha1 선호 알고리즘을 서버로 보냅니다. 서버에서 지원하지 않을 경우 지원되는 알고리즘 목록이 서버로 전송되고 일치하는 알고리즘을 찾을 때까지 각 암호화가 시도됩니다.
선호 공용 키 알고리즘
공용 키 암호화를 위한 비대칭 키 알고리즘을 선택합니다.
기본값인 런타임 기본값 사용을 사용할 경우 Oracle Service Bus가 ssh-dss 선호 알고리즘을 서버로 보냅니다. 서버에서 지원하지 않을 경우 지원되는 알고리즘 목록이 서버로 전송되고 일치하는 알고리즘을 찾을 때까지 각 암호화가 시도됩니다.
선호 키 교환 알고리즘
메시지 암호화를 위한 세션 키를 협상하기 위해 기본 키 교환 프로토콜을 선택합니다.
기본값인 런타임 기본값 사용을 사용할 경우 Oracle Service Bus가 diffie-hellman-group1-sha1 선호 알고리즘을 서버로 보냅니다. 서버에서 지원하지 않을 경우 지원되는 알고리즘 목록이 전송되고 일치하는 알고리즘을 찾을 때까지 각 암호화가 시도됩니다.
선호 압축 알고리즘
zlib을 사용하여 실행 중인 데이터를 압축할지 여부를 선택합니다. 데이터를 압축하려면 zlib을 선택하고, 그렇지 않으면 없음을 선택합니다. 기본값은 없음입니다.
하위 디렉토리 스캔
이 체크 박스를 선택하면 모든 디렉토리를 반복해서 스캔합니다.
도착순으로 정렬
이 체크 박스를 선택하면 도착한 순서대로 이벤트를 전달합니다.
시간 초과
연결이 끊기기 전의 소켓 시간 제한(초)을 입력합니다. 0을 입력하면 시간 초과가 없습니다. 기본값은 60입니다.
재시도 횟수
SFTP 접속 실패에 대한 재시도 횟수를 지정합니다.
