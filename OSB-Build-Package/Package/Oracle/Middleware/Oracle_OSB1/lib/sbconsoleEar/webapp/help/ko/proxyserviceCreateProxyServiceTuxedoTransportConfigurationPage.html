Tuxedo 전송 구성 페이지

프록시 서비스 생성/편집 - Tuxedo 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Tuxedo Transport"를 참조하십시오.

이 페이지에서는 Tuxedo 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-14에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-14 프록시 서비스 생성/편집 - Tuxedo 전송 구성 페이지
옵션
생성 또는 편집
필드 테이블 클래스
수신된 FML/FML32 버퍼를 설명하는 클래스 이름을 입력합니다. 이러한 클래스 이름은 필드 이름을 구성 요소 이름에 매핑하기 위해 FML/FML32-to-XML 변환 루틴에 사용됩니다. 공백으로 구분된 정규화된 클래스 이름 목록입니다.
뷰 클래스
수신 또는 송신된 VIEW/VIEW32 버퍼를 설명하는 클래스 이름을 입력합니다. 이러한 클래스 이름은 필드 이름을 요소 이름에 매핑하기 위해 VIEW-to-XML 또는 VIEW32-to-XML 변환 루틴에 사용됩니다. 공백으로 구분된 정규화된 클래스 이름 목록입니다.
X_C_TYPE 및 X_COMMON Tuxedo 버퍼 유형은 VIEW/VIEW32 버퍼와 같은 방식으로 처리됩니다.
들어오는 요청에 VIEW가 포함되어 존재하는 경우 해당 VIEW 클래스를 Oracle Service Bus CLASSPATH에서 지정해야 합니다.
클래스 Jar
끝점 작업에 필요한 FML/FML32 또는 VIEW/VIEW32 클래스가 있는 JAR 파일이 포함된 JAR 리소스를 선택합니다.
로컬 액세스 지점
드롭다운 목록에서 익스포트와 연관된 로컬 액세스 지점을 선택합니다. 드롭다운 목록에는 WTC에 구성된 로컬 액세스 지점이 포함되어 있습니다. 연관된 로컬 액세스 지점이 없을 경우 프록시 서비스를 생성할 수 없습니다.
로컬 액세스 지점이 존재하지 않거나 로컬 액세스 지점을 새로 생성하려면 새로 만들기를 선택합니다. 인접한 필드에 해당 로컬 액세스 지점 이름 및 로컬 네트워크 주소를 입력합니다. 끝점을 검증하면 액세스 지점이 각 WTC 서버의 WTC 구성에 추가됩니다. WTC 서버가 존재하지 않는 경우 서버가 생성됩니다.
새로 만들기 옵션을 선택한 다음 기존 액세스 지점 이름을 입력할 수 있습니다. 이렇게 하면 새 매개변수를 사용하여 기존 정보가 갱신됩니다. 호스트 이름 및 포트 번호만 변경할 수 있습니다.
원격 액세스 지점
이 필드는 로컬 액세스 지점 필드에서 새로 만들기를 선택한 경우에만 표시됩니다.
드롭다운 목록에서 새로 생성된 로컬 액세스 지점과 관련된 원격 액세스 지점을 선택합니다. 원격 액세스 지점이 존재하지 않거나 로컬 액세스 지점을 새로 생성하려는 경우 새로 만들기를 선택합니다. 인접한 필드에 해당 액세스 지점 이름 및 네트워크 주소를 입력합니다.
새로 만들기 옵션을 선택한 다음 기존 액세스 지점 이름을 입력할 수 있습니다. 이렇게 하면 새 매개변수를 사용하여 기존 정보가 갱신됩니다. 호스트 이름 및 포트 번호만 변경할 수 있습니다.
또한 원격 액세스 지점은 인바운드 요청의 WTC 접속에 대한 기본 인증이 됩니다. 들어오는 호출을 허용하는 기본 보안 영역에서 액세스 지점 ID가 동일한 사용자를 선택적으로 생성할 수 있습니다. 이렇게 하려면 사용자를 생성하시겠습니까? 드롭다운 목록에서 예를 선택합니다. 보안 문제를 방지하기 위해 비밀번호는 임시 변수를 사용하여 임의로 생성됩니다.
응답 버퍼 유형
이 옵션은 응답 필요 필드가 선택된 경우에만 사용할 수 있습니다.
원격 tuxedo 클라이언트가 수신하는 버퍼 유형을 선택합니다.
응답 버퍼 하위 유형
이 옵션은 응답 필요 옵션을 선택하고 응답 버퍼 유형 값이 VIEW 또는 VIEW32인 경우에만 사용할 수 있습니다.
응답 버퍼와 연관할 버퍼 하위 유형을 입력합니다.
응답 필요
서비스에서 응답을 전송해야 하는 경우 이 체크 박스를 선택합니다.
기본적으로 이 옵션이 선택되어 있습니다.
서비스 유형이 메시징 서비스이고 응답 메시지 유형이 없음인 경우 이 옵션의 선택이 취소되고 사용할 수 없게 됩니다.
요청 인코딩
Tuxedo 전송 요청을 위한 문자 집합 인코딩을 지정합니다.
응답 인코딩
Tuxedo 전송 응답을 위한 문자 집합 인코딩을 지정합니다.
변환 스타일
다음 중 하나를 선택합니다.
  • 없음 - (기본값) 필드 순서는 반영되지 않을 수 있습니다.
  • 정렬 - 필드는 항목을 정확한 순서대로 표시합니다.
  • 그룹화 및 정렬 - 필드가 레코드와 같이 논리적으로 구성된 경우 필드는 항목별로 정렬되고 레코드별로 그룹화됩니다.
