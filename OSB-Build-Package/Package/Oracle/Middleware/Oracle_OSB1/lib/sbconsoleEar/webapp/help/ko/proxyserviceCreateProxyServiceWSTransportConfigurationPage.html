WS 전송 구성 페이지

프록시 서비스 생성/편집 - WS 전송 구성 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 생성하고 편집하기 위한 일련의 페이지 중 하나입니다.

자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "WS Transport"를 참조하십시오.

이 페이지에서는 WS 전송 프로토콜을 사용하여 프록시 서비스의 전송 설정을 구성할 수 있습니다. 표 19-15에서는 이 페이지를 사용하는 방법에 대해 설명합니다.

표 19-15 프록시 서비스 생성/편집 - WS 전송 구성 페이지
옵션
생성 또는 편집
작업 할당 정책
이 끝점의 작업 할당 정책을 선택하거나 기본 작업 할당 정책을 사용합니다.
작업 할당 정책은 서비스 끝점에 사용할 WLS 작업 관리자의 인스턴스를 참조합니다. 작업 관리자에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=CNFGD112에서 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server의 "Using Work Managers to Optimize Scheduled Work"를 참조하십시오.
재시도 횟수
메시지를 파이프 라인으로 전달하려고 재시도한 횟수입니다.
프록시 서비스의 요청 파이프 라인에서 처리되지 않은 예외 사항이 발생한 경우 재시도 횟수에서 지정한 횟수만큼 들어오는 WS 전송 메시지가 파이프 라인으로 재전달됩니다. 이 값은 WS 전송 메시지의 안정적인 처리를 위해 중요합니다.
재시도 지연
오류 발생 후 메시지를 파이프 라인으로 다시 보내려고 시도하기 전에 시스템이 일시 중지되는 시간(초)입니다.

마친 후에
