XQuery 편집기에서 변수 구조 생성

XQuery 및 XPath 편집기의 변수 구조 팔레트에는 변수의 내용이 그림으로 표시됩니다. 기본적으로 이 팔레트에는 attachments, body, header, outbound 및 inbound와 같은 내장 메시지 컨텍스트 변수가 있습니다.

각 변수 구조 매핑 항목에는 레이블이 있으며 변수 또는 변수 경로를 하나 이상의 구조에 매핑합니다. 이러한 매핑의 범위는 스테이지 또는 경로 지정 노드입니다.

다음을 기반으로 고유한 변수 구조를 선언할 수도 있습니다.

  • 다음을 포함한 XML 유형:
    • 스키마 요소
    • WSDL 요소
    • 스키마 유형
    • WSDL 유형
    • MFL
  • 서비스 인터페이스
  • 단순 유형(문자열 또는 모든 XML)

모든 사용자 정의 변수와 $inbound, $outbound 및 $fault에 대해 직접 이 기능을 사용할 수 있습니다. 그러나 이 기능을 사용하여 $attachments의 XML 첨부 파일, $header의 헤더, $body의 문서 및 RPC 매개변수에 직접 액세스할 수는 없습니다. 단, WSDL 프록시 서비스에서 받은 요청 메시지에 대한 $body의 문서 및 매개변수는 이 기능을 사용하여 직접 액세스할 수 있습니다.

시작하기 전에

이 지침에서는 인라인 XQuery 및 XPath 표현식 생성 및 편집에 설명된 것처럼 XQuery 표현식 편집기, XQuery 조건 편집기 또는 XPath 표현식 편집기에서 표현식을 생성하거나 편집하고 있다고 가정합니다.

변수 구조를 생성하려면

  1. 변수 구조를 선택합니다.
  2. 변수 구조 팔레트에서 새 구조 추가를 누릅니다.
  3. 아래의 표 22-3에 나열된 모든 작업을 계속 수행합니다.

    4. 표 22-3 새 변수 구조 생성
    수행할 작업
    수행할 단계
    변수를 XML 스키마 유형에 매핑하는 변수 구조 생성
    1. 선택되어 있지 않은 경우 변수 구조 팔레트의 맨 위에 있는 XML 유형을 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 런타임 시 구조 경로 필드에 변수 구조의 경로를 입력합니다. 경로는 $ 문자로 시작해야 합니다.
    4. 유형 필드에서 적절한 라디오 단추를 선택한 다음 스키마 유형을 선택합니다.
    5. 찾아보기를 누릅니다. XML 스키마 브라우저가 표시됩니다. 목록에서 XML 스키마를 선택하고 정의 창에서 XML 스키마 유형을 선택한 다음 제출을 누릅니다.
    6. 추가를 눌러 변수 구조를 생성합니다.
    변수를 WSDL 유형에 매핑하는 변수 구조 생성
    1. 선택되어 있지 않은 경우 변수 구조 팔레트의 맨 위에 있는 XML 유형을 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 런타임 시 구조 경로 필드에 변수 구조의 경로를 입력합니다.
    4. 유형 필드에서 적절한 라디오 단추를 선택한 다음 WSDL 유형을 선택합니다.
    5. 찾아보기를 누릅니다. WSDL 브라우저가 표시됩니다. WSDL 목록에서 WSDL을 선택하고 정의 창에서 WSDL 유형을 선택한 다음 제출을 누릅니다.
    6. 추가를 눌러 변수 구조를 생성합니다.
    변수를 XML 스키마 구성 요소에 매핑하는 변수 구조 생성
    1. 선택되어 있지 않은 경우 변수 구조 팔레트의 맨 위에서 XML 유형을 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 런타임 시 구조 경로 필드에 변수 구조의 경로를 입력합니다.
    4. 유형 필드에서 적절한 라디오 단추를 선택한 다음 스키마 구성 요소를 선택합니다.
    5. 찾아보기를 누릅니다. XML 스키마 브라우저가 표시됩니다. 목록에서 XML 스키마를 선택하고 정의 창에서 XML 스키마 유형을 선택한 다음 제출을 누릅니다.
    6. 추가를 눌러 변수 구조를 생성합니다.
    변수를 WSDL 구성 요소에 매핑하는 변수 구조 생성
    1. 선택되어 있지 않은 경우 변수 구조 팔레트의 맨 위에서 XML 유형을 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 런타임 시 구조 경로 필드에 변수 구조의 경로를 입력합니다.
    4. 유형 필드에서 적절한 라디오 단추를 선택한 다음 WSDL 구성 요소를 선택합니다.
    5. 찾아보기를 누릅니다. WSDL 브라우저가 표시됩니다. WSDL 목록에서 WSDL을 선택하고 정의 창에서 WSDL 구성 요소를 선택한 다음 제출을 누릅니다.
    6. 추가를 눌러 변수 구조를 생성합니다.
    변수를 하위 요소에 매핑하는 변수 구조 생성
    1. 선택되어 있지 않은 경우 변수 구조 팔레트의 맨 위에서 XML 유형을 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 런타임 시 구조 경로 필드에 변수 구조의 경로를 입력합니다.
    4. 유형 필드에서 생성할 변수 유형을 선택합니다.

      5. XML 스키마 구성 요소 또는 WSDL 구성 요소 변수를 생성하려면 이 옵션과 연관된 라디오 단추를 선택한 다음 스키마 구성 요소 또는 WSDL 구성 요소를 선택합니다.

      MFL 변수를 생성하려면 이 옵션과 연관된 라디오 단추를 선택한 다음 MFL을 선택합니다.

    5. XML 스키마, WSDL 또는 MFL에 대해 찾아보기를 눌러 브라우저에 표시되는 목록에서 객체를 선택한 다음 제출을 누릅니다. 예를 들어 MFL 목록에서 MFL을 선택한 다음 제출을 누릅니다.
    6. 생성하고 있는 구조의 하위로 구성 요소를 설정하려면 하위로 설정 체크 박스를 선택합니다.
    7. 추가를 눌러 변수 구조를 생성합니다.
    MFL 리소스를 사용하는 변수 구조 생성
    1. 선택되어 있지 않은 경우 변수 구조 팔레트의 맨 위에서 XML 유형을 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 런타임 시 구조 경로 필드에 변수 구조의 경로를 입력합니다.
    4. 유형 필드에서 적절한 라디오 단추를 선택한 다음 찾아보기를 누릅니다. MFL 브라우저가 표시됩니다.
    5. MFL 목록에서 MFL을 선택하고 제출을 누릅니다.
    6. 추가를 눌러 변수 구조를 생성합니다.
    서비스 인터페이스 변수 구조 생성
    1. 변수 구조 팔레트의 맨 위에서 서비스 인터페이스를 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 구조 경로 필드에 기본값이 이미 $body로 설정되어 있습니다. 이 필드는 변경할 수 없습니다.
    4. WSDL 기반 서비스 필드에서 서비스 브라우저 아이콘을 선택하고, 서비스 브라우저에 표시되는 서비스 목록에서 서비스를 선택한 다음 제출을 누릅니다.

      5. 선택한 서비스가 WSDL 기반 서비스 필드에 표시됩니다.

    5. 작업 필드에서 작업을 선택하거나 작업을 포함하지 않도록 없음을 선택합니다.
    6. 추가를 눌러 변수 구조를 생성합니다.
    단순 변수 구조 생성
    1. 변수 구조 팔레트의 맨 위에서 단순 유형을 선택합니다.
    2. 구조 레이블 필드에 생성할 변수에 대한 표시 이름을 입력합니다. 이 표시 이름을 통해 구조에 의미있는 이름을 지정하여 해당 구조를 디자인 시간에 인식할 수 있으며 구조는 런타임에는 영향을 미치지 않습니다.
    3. 구조 이름 필드에 생성할 변수 구조에 대한 이름을 입력합니다.
    4. 유형 필드에서 문자열 또는 모든 XML을 선택합니다.
    5. 추가를 눌러 변수 구조를 생성합니다.
