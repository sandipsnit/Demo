동적 XQuery 표현식을 인라인 XQueries에 바인딩

XQuery 표현식 편집기 페이지에서 런타임에 사전 등록된 XQuery 리소스의 이름으로 평가되는 동적 XQuery 표현식을 지정할 수 있습니다. 이 편집기에 대한 보다 자세한 내용은 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.

동적 XQuery 표현식을 정의하려면

  1. 동적 XQuery 옵션을 선택합니다.
  2. XQuery 리소스에 대한 표현식을 입력합니다. 영역에서 런타임에 사전 등록된 XQuery 리소스 이름으로 평가될 XQuery 표현식을 입력합니다.

    3. 다음은 XQuery 리소스 구문을 표시합니다(리소스의 전체 이름 표시):

    Project/folder1/folder2/XQueryResourceName

  3. 사용자 정의 변수의 XQuery 템플리트를 사용하여 변수 바인드 영역에서 변환을 위해 입력 매개변수를 정의합니다.
    • 질의 형식(변수의 개수 및 이름)에 대한 템플리트 역할을 수행할 기존의 등록된 리소스를 선택하려면 XQuery 템플리트 선택 필드에서 찾아보기를 누릅니다. 템플리트를 선택하면 변수가 바인드 변수 영역에 나타납니다. 템플리트는 구성에서 유지되지 않습니다. 대신 템플리트는 질의의 변수를 지정하는 데 도움을 주기 위한 빠른 시작 역할을 합니다.
    • 사용자 정의 변수 추가 필드에 변수 이름을 입력하고 추가를 누릅니다. 변수 이름 아래에 나열된 각 변수에 대해 해당 변수로 매핑할 XQuery 표현식을 입력합니다. 각 매개변수에 대해 매핑을 정의해야 합니다. 예를 들어, XQuery 변환에 one과 two라는 두 개의 입력 매개변수가 있으면 변수 이름 필드에 one과 two라는 두 개의 레이블이 생성됩니다. XQuery 표현식이 입력되는 텍스트 상자는 각 레이블과 연관됩니다.

      • 다음 XQuery 표현식은 변수 필드에 대한 적합한 입력의 예입니다.

      $body/*[1]

      $body/po:PurchaseOrder

  4. 마친 후에 다음 작업 중 하나를 계속 진행합니다.
    • 검증을 누릅니다. 표현식이 성공적으로 검증되면 메시지가 표시됩니다.
    • 테스트를 누릅니다. XQuery 테스트 수행을 참조하십시오.
    • 변경 사항을 저장하거나 무시합니다.
