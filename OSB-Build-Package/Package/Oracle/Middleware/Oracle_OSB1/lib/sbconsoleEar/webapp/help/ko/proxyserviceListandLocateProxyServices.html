프록시 서비스 찾기

  1. 다음 작업을 수행합니다.
    • 프로젝트 탐색기를 선택하여 프로젝트 뷰 페이지 또는 프로젝트/폴더 뷰 페이지를 표시합니다. 그런 다음 프로젝트와 폴더를 탐색하여 프록시 서비스를 찾습니다.
    • 리소스 브라우저 > 프록시 서비스를 선택합니다. 그러면 프록시 서비스 요약 페이지에 표 19-18과 같은 정보가 표시됩니다. 속성에 대한 자세한 설명은 프록시 서비스 구성 편집을 참조하십시오.
  2. 목록에서 항목 수를 제한하기 위해 이름, 경로 또는 이 모두를 사용하여 필터링할 수 있습니다. 이름 및 경로 필드의 검색 아래에서 대상의 이름 및/또는 경로를 입력한 다음 검색 단추를 누릅니다.

    3. 프록시 서비스가 있는 폴더 이름과 프로젝트 이름입니다.

    와일드 카드 문자 * 및 ?를 사용할 수 있습니다. 검색 결과는 대소문자를 구분합니다.

    모두 보기를 눌러 도메인의 모든 프록시 서비스를 표시합니다. 이렇게 하면 이전 검색의 검색 매개변수가 삭제됩니다.

    표 19-18 프록시 서비스 요약 페이지 
    속성
    설명
    이름
    프록시 서비스의 고유 이름입니다. 이름을 눌러 프록시 서비스 보기 - 구성 세부 정보 페이지를 표시합니다.
    프록시 서비스 구성 편집을 참조하십시오.
    경로
    경로는 프로젝트 이름 및 프록시 서비스가 있는 폴더의 이름입니다(예: UDDI/Proxies/OSB_services).
    프록시 서비스의 경로를 눌러 프록시 서비스를 포함하는 프로젝트/폴더 뷰 페이지를 표시합니다.
    작업
    다음 작업 중 하나를 수행합니다.
    • 메시지 흐름 편집 아이콘을 눌러 프록시 서비스의 메시지 흐름을 편집합니다. 메시지 흐름 생성, 편집 및 보기를 참조하십시오.
    • 서비스 및 변환 설계 검증 및 테스트에 사용할 테스트 콘솔을 호출하려면 테스트 콘솔 시작 아이콘을 누릅니다. 비즈니스 서비스에 대해 런타임 시, 즉 세션이 활성화될 때 테스트 콘솔만 사용할 수 있습니다. 변환에 대해 세션의 사용 여부와 관계없이 테스트 콘솔을 사용할 수 있습니다. 자세한 내용은 서비스 테스트 및 변환 테스트를 참조하십시오.
    • WSDL 기반 비즈니스 서비스에 대해 WSDL 익스포트 아이콘이 표시됩니다. 이 아이콘을 눌러 WSDL을 익스포트하면 IDE와 같은 외부 툴에서 WSDL을 보거나 수정할 수 있습니다.

      • 이 기능은 두 개의 도메인 간 리소스 이동 및 준비에 사용하는 시스템 관리 모듈의 리소스 익스포트 기능과는 다릅니다. WSDL 익스포트를 참조하십시오.

    옵션
    서비스를 삭제하려면 삭제 아이콘을 누릅니다. 다른 리소스에서 이 리소스를 참조하는 경우 삭제 경고 아이콘이 표시됩니다. 경고 확인을 통해 리소스를 삭제할 수 있습니다. 이 경우 삭제된 리소스에 대한 해결되지 않은 참조로 인해 충돌이 발생할 수 있습니다. 자세한 내용은 프록시 서비스 삭제를 참조하십시오.
