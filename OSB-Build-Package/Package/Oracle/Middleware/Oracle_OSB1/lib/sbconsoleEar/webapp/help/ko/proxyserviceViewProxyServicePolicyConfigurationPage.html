프록시 서비스 정책 페이지

프록시 서비스 보기 - 정책 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 서비스 유형이 WSDL 웹 서비스, 메시징 서비스, 모든 SOAP 서비스 또는 모든 XML 서비스인 프록시 서비스에 대한 정책 설정을 구성할 수 있습니다. 표 19-20에서는 이 페이지를 사용하는 방법에 대해 설명합니다. OWSM 정책을 WSDL 웹 서비스, 메시징 서비스 또는 모든 XML 서비스 프록시 서비스에서 사용하려면 프로토콜이 HTTP로 설정되어야 합니다.

WSDL 기반 서비스의 경우 서비스에 바인딩된 WLS 9.2 정책이 효과적인 WSDL에 노출(인라인)됩니다. 추상 정책은 인라인되기 전에 미리 처리됩니다. OWSM 정책은 효과적인 WSDL에 인라인되지 않고 참조로 바인딩됩니다.

서비스 정책 구성
[서비스 정책 구성] 필드의 옵션은 다음과 같습니다.
  • OWSM 정책 저장소에서 - 정책은 Oracle Enterprise Manager로 관리되는 중앙 Oracle Web Services Manager 정책 저장소에서 정의됩니다. 이러한 정책에는 WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken 프로파일 및 기타 산업 표준이 지원됩니다.

    필요한 경우 필터링을 사용하여 원하는 정책을 찾을 수도 있습니다. 프록시 서비스의 경우 서비스 정책만 표시됩니다.

    • 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Securing Oracle Service Bus with Oracle Web Services Manager"를 참조하십시오.

  • WSDL에서 - 서비스 정책이 서비스가 기반으로 하는 WSDL과 연결된 경우 이 옵션을 선택합니다. 이러한 정책에서는 WS-Security 1.0, SAML 1.1 및 기타 산업 표준이 지원됩니다.

    • 이 옵션을 사용할 경우 WSDL의 요청 및 응답 정책을 볼 수 있습니다(읽기 전용).

  • 미리 정의된 정책 또는 WS 정책 리소스에서 - 서비스 레벨 정책, 작업 레벨 정책(정책이 요청 및 응답 메시지 모두에 적용되는 경우), 요청 정책 및 Oracle Service Bus 관리 콘솔의 응답 정책을 추가합니다.

    정책은 Oracle WebLogic Server에 미리 정의되거나 Oracle Service Bus에 WS 정책 리소스로 정의됩니다. 이러한 정책에서는 WS-Security 1.0, SAML 1.1 및 기타 산업 표준이 지원됩니다.

    • 이 옵션을 사용할 경우 서비스, 작업, 요청 및 응답 레벨에서 정책을 추가할 수 있습니다.

주: 정책 바인딩 모델은 상호 배타적입니다. 서비스에서는 한 가지 유형의 정책만 사용해야 합니다. 정책을 서비스에 직접 바인딩하면 모든 WSDL 기반 정책이 무시됩니다.
