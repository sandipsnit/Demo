프록시 서비스 보안 페이지

프록시 서비스 보기 - 보안 페이지는 프록시 서비스 생성 및 구성과 프록시 서비스 구성 편집에 설명된 것처럼 프록시 서비스를 편집하기 위한 일련의 페이지 중 하나입니다.

이 페이지에서는 프록시 서비스의 보안 설정을 구성할 수 있습니다.

이 페이지에서 사용 가능한 필드는 프록시 서비스에 사용되는 정책에 따라 다릅니다. 예를 들어, 프록시 서비스에 OWSM 정책이 사용되는 경우(권장) [보안] 페이지에 [정책 우선 적용]이 나타납니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Securing Oracle Service Bus with Oracle Web Services Manager"를 참조하십시오.

WLS 9.2 정책에서 사용자 정의 사용자 이름/비밀번호와 사용자 정의 토큰의 구성은 둘 다 비슷합니다. 이 두 경우, Oracle Service Bus를 사용으로 설정하여 필요한 정보를 찾을 수 있도록 XPath 표현식을 지정합니다. 이 XPath 표현식의 루트는 다음과 같습니다.

  • 서비스 바인딩이 AnySOAP 또는 WSDL-SOAP인 경우 soap-env:Envelope/soap-env:Header를 사용합니다.
  • 서비스 바인딩이 SOAP 기반이 아닌 경우 soap-env:Body를 사용합니다.

모든 XPath 표현식은 적합한 XPath 형식이어야 합니다. XPath 표현식은 XPath "declare namespace" 구문을 사용하여 다음과 같이 사용된 네임스페이스를 선언해야 합니다.

declare namespace
ns='http://webservices.example.com/MyExampleService';)

표 19-21은 프록시 서비스 보기 - 보안 구성 페이지를 설명합니다.

서비스 키 제공자
필요한 경우 서비스 키 공급자의 경로(프로젝트/폴더) 및 이름을 입력하거나 찾아보기를 눌러 서비스 키 공급자 선택 페이지에서 선택합니다.
자세한 내용은 서비스 키 제공자를 참조하십시오. 서비스 키 제공자를 생성하는 방법에 대해 알아보려면 서비스 키 제공자 추가를 참조하십시오.

자세한 내용은 표 19-20을 참조하거나 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Securing Oracle Service Bus with Oracle Web Services Manager"를 참조하십시오.
웹 서비스 보안 구성
WS 보안 헤더 처리
클라이언트에 WS 보안 보안 헤더가 존재하는 경우 이 헤더 처리 여부를 결정합니다.
  • 예를 선택하는 경우: 활성 중개자 시나리오에서 클라이언트가 요청 및/또는 응답 메시지에 WS 보안을 적용합니다. 프록시 서비스가 보안 헤더를 처리하고 WS 보안 정책을 적용합니다. 프록시 서비스가 보안 정책을 사용하는 WSDL을 기반으로 하거나 [서비스 정책 구성] 페이지에서 서비스에 정책을 추가한 경우 자동으로 이 옵션이 [예]로 설정됩니다.
  • 아니오를 선택하는 경우: WS 보안 통과 시나리오에서 클라이언트가 WS 보안을 요청 및/또는 응답 메시지에 적용합니다. 프록시 서비스가 보안 헤더를 처리하지 않습니다. 대신, 적용되지 않은 보안 요청 메시지를 비즈니스 서비스에 전달합니다.
액세스 제어
전송 액세스 제어
모든 프록시 서비스의 경우 클라이언트가 프록시 서비스와 접속하려고 시도할 때 보안 점검을 적용하는 전송 레벨 정책을 생성할 수 있습니다. 전송 레벨 정책에 나열된 사용자의 요구만 처리할 수 있습니다.
메시지 액세스 제어
메시지 레벨 액세스 제어 정책은 클라이언트가 메시지 레벨 보안을 사용하여 프록시 서비스를 호출하려고 시도할 때 보안 점검을 적용합니다. 다음의 경우에 메시지 레벨 액세스 제어 정책을 생성할 수 있습니다.
  • WS 보안 헤더를 처리하는 활성 웹 서비스 보안 중간 단계인 프록시 서비스의 경우
  • 이 페이지에 사용자 정의 인증을 설정하는 프록시 서비스의 경우
메시지 레벨 정책에 나열된 사용자만 작업을 호출할 수 있습니다.
사용자 정의 인증
(메시지 레벨)
인증 유형
다음 중 하나를 선택합니다.
  • 프록시 서비스에서 사용자 정의 인증을 사용하지 않는 경우 없음을 선택합니다.
  • 사용자 정의 사용자 이름 및 비밀번호

    • 이 옵션을 선택하면 사용자 이름 XPath 필드 및 사용자 비밀번호 XPath 필드에도 값을 입력해야 합니다.

  • 사용자 정의 토큰

    • 이 옵션을 선택하면 토큰 유형을 선택한 다음 토큰 XPath 필드에도 값을 입력해야 합니다.

사용자 이름 XPath
사용자 정의 인증 설정 옵션이 사용자 정의 사용자 이름 및 비밀번호로 설정된 경우에만 이 옵션을 사용할 수 있습니다. 이 옵션을 사용할 수 존재하는 경우 이 옵션은 필수입니다.
XPath 표현식으로 사용자 이름을 입력합니다.
Xpath 표현식은 선택한 옵션에 따라 메시지 헤더 또는 페이로드에 대해 평가되며, 이를 통해 Oracle Service Bus에서 사용자 정의 인증에 필요한 사용자 이름을 임포트할 수 있습니다.
사용자 비밀번호 Xpath
사용자 정의 인증 설정 옵션이 사용자 정의 사용자 이름 및 비밀번호로 설정된 경우에만 이 옵션을 사용할 수 있습니다. 이 옵션을 사용할 수 존재하는 경우 이 옵션은 필수입니다.
Xpath 표현식으로 비밀번호를 입력합니다.
Xpath 표현식은 선택한 옵션에 따라 메시지 헤더 또는 페이로드에 대해 평가되며, 이를 통해 Oracle Service Bus에서 사용자 정의 인증에 필요한 비밀번호 값을 임포트할 수 있습니다.
토큰 유형
사용자 정의 인증 설정 옵션이 사용자 정의 토큰으로 설정된 경우에만 이 옵션을 사용할 수 있습니다. 이 옵션을 사용할 수 존재하는 경우 이 옵션은 필수입니다.
드롭다운 목록에서 토큰 유형을 선택합니다. WebLogic Server ID 검증 제공자용으로 구성된 활성 토큰 유형만 사용할 수 있습니다. http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Configuring Identity Assertion Providers for Custom Tokens"를 참조하십시오.
토큰 XPath
사용자 정의 인증 설정 옵션이 사용자 정의 토큰으로 설정된 경우에만 이 옵션을 사용할 수 있습니다. 이 옵션을 사용할 수 존재하는 경우 이 옵션은 필수입니다.
XPath 표현식을 입력하여 사용자 정의 토큰의 경로를 지정합니다. Oracle Service Bus는 선택한 옵션에 따라 메시지 헤더 또는 페이로드에 대해 토큰 XPath 표현식을 평가하여 사용자 정의 인증에 필요한 토큰을 임포트합니다.
컨텍스트 속성
선택적으로 하나 이상의 컨텍스트 속성을 지정하여 추가 컨텍스트 정보를 인증(사용자 정의 사용자 이름 및 비밀번호) 또는 ID 검증(사용자 정의 토큰) 보안 제공자에게 전달합니다.
컨텍스트 속성은 보안 제공자가 컨텍스트 정보를 얻을 수 있도록 WebLogic 보안 프레임워크에 추가 정보를 전달하는 방법(ContextHandler 인터페이스)을 제공합니다. http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Context Properties Are Passed to Security Providers"를 참조하십시오.
리터럴 문자열로 속성 이름을, 적합한 XPath 표현식으로 값 선택기를 입력합니다. (XPath 표현식은 리터럴 문자열일 수도 있습니다.)
Xpath 표현식은 사용자 정의 토큰 또는 사용자 정의 사용자 이름/비밀번호에 사용되는 동일한 메시지 부분에 대해 평가됩니다. 즉, SOAP 기반 프록시 서비스에 대한 값 선택기 XPath 표현식은 헤더와 비SOAP 기반 프록시 서비스의 페이로드에 대해 평가됩니다.
XPath 표현식은 런타임 시 평가되어 속성 값을 생성합니다. ContextHandler는 기본적으로 이름/값 목록이며, 그렇기 때문에 보안 제공자는 검색할 이름이 무엇인지를 알고 있어야 합니다. 따라서 XPath 표현식은 보안 제공자가 이러한 사용자 정의 속성 중 값 하나에 대해 요청하는 경우에만 평가됩니다.
이 컨텍스트 속성을 추가하려면 속성 추가를 누릅니다. 다중 컨텍스트 속성을 추가할 수 있습니다.
