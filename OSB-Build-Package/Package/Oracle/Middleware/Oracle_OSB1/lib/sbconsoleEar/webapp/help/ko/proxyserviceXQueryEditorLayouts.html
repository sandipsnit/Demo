XQuery 편집기 레이아웃 및 작업 이해

XQuery 표현식 편집기, XQuery 조건 편집기 및 XPath 표현식 편집기는 각각 다음 구성 요소로 구성됩니다.

  • 팔레트
  • 작업 영역
  • 속성 검사기

팔레트

각 편집기의 왼쪽 창에는 아래 나열된 팔레트가 있습니다. 편집기에서 팔레트 이름을 눌러 표시합니다. 각 팔레트에는 편집기에서 표현식에 삽입할 수 있는 엔티티가 있습니다.

  • 네임스페이스 정의 팔레트에는 기본 Oracle Service Bus 네임스페이스, 변수 네임스페이스 및 사용자 정의 네임스페이스가 나열됩니다. 사용자 정의 네임스페이스 목록에 추가되는 새 네임스페이스를 정의할 수 있습니다.

    • 자세한 내용은 다음을 참조하십시오.

    • 편집기 작업 영역 텍스트 필드에서 표현식 생성
    • 인라인 표현식에서 사용할 네임스페이스 생성
  • XQuery 함수 팔레트에는 표준 XQuery 함수 세트가 나열됩니다. 표현식에 함수를 삽입하면 사용자가 입력해야 하는 매개변수 값에 위치 표시자가 사용됩니다.

    • 자세한 내용은 다음을 참조하십시오.

    • 편집기 작업 영역 텍스트 필드에서 표현식 생성
    • XQuery 편집기에서 사용자 정의 XPath 함수 생성
    • 외부 XQuery 리소스를 인라인 XQuery에 바인딩
  • 변수 구조 팔레트는 XPath 표현식을 사용하여 변수 및 변수 경로를 삽입하기 위한 툴 세트를 제공합니다.

    • 변수 구조는 편집기에 표시되는 변수 또는 변수 경로의 그래픽 표현입니다. 이러한 그래픽 표현을 사용하여 변수 구조를 시각화하고 변수 내용을 참조하는 인라인 XQuery 표현식을 생성할 수 있습니다.

    주: 변수 구조에서 변수를 생성하지 않습니다. 변수는 런타임 시 스테이지의 할당 작업 대상으로 생성됩니다.

    Oracle Service Bus는 미리 정의된 여러 메시지 컨텍스트 변수(예: 첨부 파일, 본문, 헤더, 아웃바운드 및 인바운드)를 제공하며 이러한 변수의 내용은 변수 구조로 표시할 수 있습니다. 또한 고유한 변수 구조를 정의할 수 있습니다.

    자세한 내용은 다음을 참조하십시오.

    • 편집기 작업 영역 텍스트 필드에서 표현식 생성
    • XQuery 편집기에서 변수 구조 생성

작업 영역

페이지의 오른쪽에는 XQuery 표현식, XQuery 조건 또는 XPath를 생성하기 위한 작업 영역이 있습니다. 작업 영역은 세 가지 편집기에서 각각 다르게 표시됩니다.

속성 검사기

세 가지 편집기 모두에서 속성 검사기는 페이지의 오른쪽 아래에 표시됩니다. 팔레트 중 하나에서 표현식을 추가할 항목을 선택한 경우 해당 항목이 속성 검사기에 표시됩니다. 그러면 해당 항목을 작업 영역에 붙여 넣을 수 있습니다. 편집기 작업 영역 텍스트 필드에서 표현식 생성를 참조하십시오.
