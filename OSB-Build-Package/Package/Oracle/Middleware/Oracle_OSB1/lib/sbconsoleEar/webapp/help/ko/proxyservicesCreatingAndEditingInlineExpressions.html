인라인 XQuery 및 XPath 표현식 생성 및 편집

단계 구성 편집 페이지에서 단계 또는 경로 지정 노드에 작업을 추가할 경우 구성 세부 정보를 확인하는 페이지에 스켈레톤 구조가 표시됩니다. 그림 22-1은 예를 보여 줍니다.

그림 22-1 작업 구성 스켈레톤의 예

해당 스켈레톤이 컨텍스트에 적합한 경우 XQuery 편집기에 액세스할 수 있는 링크를 제공합니다. 사용자는 이 편집기에서 작업 컨텍스트의 필요에 따라 인라인을 실행할 표현식을 생성할 수 있습니다.

시작하기 전에

이러한 지침은 사용자가 파이프 라인 스테이지, 오류 처리기 스테이지 또는 경로 지정 노드의 스테이지 구성 편집 페이지에서 작업을 생성하거나 편집한다고 가정합니다. 메시지 흐름 생성, 편집 및 보기와 메시지 흐름에서 작업 추가 및 편집을 참조하십시오.

인라인 표현식을 생성하거나 수정하려면 다음을 수행합니다.

  1. 스테이지 구성 편집 페이지에서 표현식을 추가하거나 편집할 작업 내의 위치를 찾습니다.
  2. 표현식 링크를 눌러 컨텍스트에 적합한 편집기를 엽니다. 표현식이 아직 정의되지 않은 경우 링크는 다음과 같이 해당 위치에서 사용할 수 있는 표현식 종류를 표시합니다.
    • <Expression>을 눌러 XQuery 표현식을 생성합니다. 또한 Oracle Service Bus 외부에서 생성한 XQuery 또는 XSLT 리소스를 임포트한 다음 인라인 XQuery에 바인드할 수도 있습니다.
    • <Condition>을 눌러 if..then... 작업의 XQuery 조건식을 생성합니다.
    • <XPath>를 눌러 메시지 컨텍스트 변수의 XPath 표현식을 생성합니다.

      • 위치에서 표현식이 이미 정의된 경우 <Expression>, <Condition> 또는 <XPath> 링크는 표현식 자체를 표시하는 링크(예: true()[icon]) 또는 표현식이 너무 긴 경우에는 표현식 부분(예: $body/urn:POSta...[icon])으로 바뀝니다. 표현식 또는 표현식 부분을 눌러 적절한 편집기에서 표현식을 엽니다.

  3. 다음 항목에서 설명하는 것처럼 표현식 작성:
    • 편집기 작업 영역 텍스트 필드에서 표현식 생성
    • 인라인 표현식에서 사용할 네임스페이스 생성
    • XQuery 편집기에서 변수 구조 생성
    • XQuery 편집기에서 사용자 정의 XPath 함수 생성
    • 외부 XQuery 리소스를 인라인 XQuery에 바인딩
    • 외부 XSLT 리소스를 인라인 Xquery에 바인딩
    • 동적 XQuery 표현식을 인라인 XQueries에 바인딩
    • 빌더 옵션을 사용하여 XQuery 비교 표현식 입력
    • 빌더 옵션을 사용하여 단항 표현식 입력
  4. 선택 사항으로 다음 작업 중 하나 또는 모두를 수행합니다.
    • 검증을 눌러 표현식을 검증합니다.
    • 테스트를 눌러 표현식을 테스트합니다. XQuery 테스트 수행을 참조하십시오.
  5. 저장을 눌러 편집기를 닫고 작업의 표현식을 삽입합니다.

마친 후에

메시지 흐름에서 작업 추가 및 편집에 설명된 대로 계속해서 작업을 구성합니다.
