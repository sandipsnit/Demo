보고 메시지 보기

메시지 보고서 요약 페이지에서는 표 27-2와 같은 보고 메시지 정보를 볼 수 있습니다. 한 페이지에 최대 100개의 메시지가 표시됩니다.

  1. 목록에 표시되는 항목 수를 제한하거나 특정 메시지를 찾으려면:
    • 열기 아이콘을 눌러 검색 필터를 추가로 표시합니다. 다중 검색 조건을 지정하는 경우에는 모든 조건과 일치하는 결과만 반환됩니다.
    • 필터 매개변수를 설정하려면 표 27-1과 같은 단계 중 하나 이상을 수행합니다.

      • 표 27-1 메시지 필터 매개변수 
      수행할 작업
      수행할 단계
      시간 매개변수를 기준으로 필터링
      시간 기간을 설정합니다.
      1. 메시지 보낸 사람... 받는 사람... 라디오 단추를 선택합니다.
      2. 메시지 보낸 사람 필드에서 MM/DD/YY HH:MM:SS AM|PM 형식으로 날짜 및 시간을 입력합니다. 예를 들어 10/10/07 12:45:00 AM을 입력할 수 있습니다.
      3. 메시지 받는 사람 필드에서 MM/DD/YY HH:MM:SS AM|PM 형식으로 날짜 및 시간을 입력합니다.
      또는 가장 최근 메시지에 대한 매개변수를 선택할 수 있습니다.
      1. 마지막의 경우 라디오 단추를 선택합니다.
      2. 마지막의 경우 필드에 일 수를 입력한 다음 시간과 분을 지정합니다.
      주: 기본적으로 최근 30분 동안의 메시지 보고서가 표시됩니다.
      서비스 이름을 기준으로 필터링
      인바운드 서비스 이름 필드에 서비스 이름을 입력합니다.
      오류 코드를 기준으로 필터링
      오류 코드 필드에 오류 코드를 입력합니다.
      보고서 인덱스를 기준으로 필터링
      보고서 인덱스 필드에 보고서 인덱스를 입력합니다. 보고서 인덱스는 이 프록시 서비스에 대한 보고서 작업을 구성할 때 정의한 키 값 쌍입니다. 이것은 문자열 검색입니다.

    • 서비스 이름, 오류 코드 및 보고서 인덱스 필드 검색에 와일드 카드 문자를 사용할 수 있습니다.
      • 별표(*)만 입력하면 모든 메시지가 반환됩니다. 예를 들어, 오류 코드 필드에 *를 입력하면 오류 코드가 있는 메시지와 없는 메시지가 모두 표시됩니다.
      • 오류 코드 필드에서만 엄격한 검색 제한이 적용됩니다. 예를 들어, 오류 코드가 BEA12345인 메시지를 찾으려면 *, BE*, 또는 BEA12345를 입력할 수 있습니다.
  2. 검색을 누릅니다. 설정한 필터 매개변수에 해당하는 메시지가 페이지에 표시됩니다. 메시지를 필터링할 때 표시할 수 있는 최대 메시지 수는 1000개입니다.
  3. 모두 보기를 눌러 검색 필터를 제거하고 모든 메시지를 표시합니다.
  4. 메시지를 찾기 위해 다음 작업을 수행할 수도 있습니다.
    • 메시지 목록을 재정렬합니다. 밑줄로 표시된 열 이름을 누릅니다. 오름차순 및 내림차순 화살표는 정렬 순서를 나타냅니다. 열 이름을 누르면 정렬 순서가 변경됩니다.
    • 페이지를 스크롤합니다. 테이블의 위 또는 아래에 있는 페이지 콘트롤을 사용합니다. 페이지 번호를 선택하여 특정 페이지로 이동하거나 화살표 단추를 사용하여 다음, 이전, 첫 페이지 또는 마지막 페이지로 이동할 수 있습니다.

      • 표 27-2 메시지 (보고) 요약 세부 정보
      속성
      설명
      보고서 인덱스
      메시지 세부 정보 보기 페이지에 대한 링크입니다. 메시지 세부 정보 보기를 참조하십시오.
      보고서 인덱스는 메시지 컨텍스트 변수에서 키 ID를 추출할 때 사용하는 키 값 쌍입니다. 키를 사용하면 메시지를 편리하게 필터링할 수 있습니다. 보고서 작업을 사용하여 키 값 쌍을 추가합니다. 메시지 흐름에서 작업 추가 및 편집을 참조하십시오.
      DB 시간 기록
      메시지가 데이터베이스에 추가된 날짜와 시간입니다.
      인바운드 서비스
      메시지와 관련된 인바운드 서비스입니다. 서비스는 프록시 서비스 보기 - 구성 세부 정보 페이지에 대한 링크입니다. 프록시 서비스 구성 편집을 참조하십시오.
      오류 코드
      이 메시지와 관련된 오류 코드입니다(존재하는 경우).
      오류 생성 작업을 구성할 때 오류 코드를 지정합니다. http://www.oracle.com/pls/as111170/lookup?id=OSBAG232에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Handling Errors in Message Flows"를 참조하거나 프록시 서비스 오류 처리기 추가를 참조하십시오.

관련 주제

http://www.oracle.com/pls/as111170/lookup?id=OSBAG569에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "How to Enable Message Reporting"을 참조하십시오.
