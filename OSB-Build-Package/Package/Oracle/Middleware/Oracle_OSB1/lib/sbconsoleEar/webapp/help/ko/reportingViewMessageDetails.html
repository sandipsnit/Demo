메시지 세부 정보 보기

이 페이지에서는 특정 메시지의 세부 정보를 볼 수 있습니다.

  1. 메시지를 찾습니다. 보고 메시지 보기를 참조하십시오.
  2. 보고서 인덱스를 누릅니다. 그러면 메시지 세부 정보 보기 페이지에 표 27-3과 같은 일반 구성 정보가 표시됩니다.

    3. 표 27-3 메시지 구성 매개변수
    속성
    설명
    메시지 ID
    이 메시지의 고유 ID입니다.
    데이터베이스 시간 기록
    메시지가 데이터베이스에 등록된 날짜와 시간입니다.
    로깅 시간
    메시지가 기록된 날짜와 시간입니다.
    서버 이름
    이 메시지가 생성된 서버의 이름입니다.
    상태
    이 메시지가 생성된 파이프 라인의 상태입니다.
    요청 - 보고 작업이 요청 파이프 라인에서 실행됨을 나타냅니다.
    응답 - 보고 작업이 응답 파이프 라인에서 실행됨을 나타냅니다.
    오류 - 서비스 레벨 오류 처리기에서 작업을 실행 중입니다.
    노드 이름
    이 메시지가 생성된 파이프 라인의 노드 이름입니다.
    파이프 라인 이름
    이 메시지가 생성된 파이프 라인의 이름입니다.
    스테이지 이름
    이 메시지가 생성된 스테이지의 이름입니다.

    이 페이지에는 표 27-4와 같은 인바운드 서비스 정보가 표시됩니다.

    표 27-4 인바운드 서비스 세부 정보
    속성
    설명
    이름
    이 메시지와 관련된 인바운드 프록시 서비스입니다. 서비스는 프록시 서비스 보기 - 구성 세부 정보 페이지에 대한 링크입니다. 프록시 서비스 구성 편집을 참조하십시오.
    URI
    프록시 서비스와 관련된 URI입니다.
    작업
    이 메시지와 관련된 인바운드 작업입니다.

    이 페이지에는 표 27-5와 같은 아웃바운드 서비스 정보가 표시됩니다.

    표 27-5 아웃바운드 서비스 세부 정보
    속성
    설명
    이름
    이 메시지와 관련된 아웃바운드 비즈니스 서비스입니다. 서비스는 프록시 서비스 보기 - 구성 세부 정보 페이지에 대한 링크입니다. 비즈니스 서비스 구성 편집을 참조하십시오.
    URI
    아웃바운드 비즈니스 서비스 끝점에 대한 URI입니다.
    작업
    아웃바운드 서비스에서 호출되는 작업의 이름입니다.

    이 페이지에는 표 27-6과 같은 보고서 인덱스 정보가 표시됩니다.

    표 27-6 보고서 인덱스 세부 정보
    속성
    설명
    보고서 인덱스 텍스트
    보고서 인덱스는 메시지 컨텍스트 변수에서 키 ID를 추출할 때 사용하는 키 값 쌍입니다. 키를 사용하면 메시지를 편리하게 필터링할 수 있습니다. 보고서 작업을 사용하여 키 값 쌍을 추가합니다. 메시지 흐름에서 작업 추가 및 편집을 참조하십시오.

    이 페이지에는 표 27-7에 나열된 오류 정보가 표시됩니다.

    표 27-7 결함 정보 세부 정보
    속성
    설명
    오류 코드
    메시지와 관련된 오류 코드입니다(존재하는 경우).
    오류 생성 작업을 구성할 때 오류 코드를 지정합니다. http://www.oracle.com/pls/as111170/lookup?id=OSBAG232에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Handling Errors in Message Flows"를 참조하거나 프록시 서비스 오류 처리기 추가를 참조하십시오.
    원인
    오류 코드의 원인입니다.
    세부 정보
    오류 코드와 관련된 오류 세부 정보입니다. 이러한 세부 정보(존재하는 경우)는 일반적으로 특정 오류가 발생한 위치의 스택 추적이며 데이터베이스의 크기 제한으로 인해 잘릴 수 있습니다. 2048자로 제한됩니다.

    이 페이지에는 표 27-8과 같은 보고서 본문 정보가 표시됩니다.

    표 27-8 보고서 본문 세부 정보
    속성
    설명
    보고서 본문 텍스트
    이 링크는 보고서 본문의 세부 정보를 나타내는 다른 브라우저를 엽니다. 보고서 작업의 표현식을 사용하여 보고서 본문 텍스트를 캡처할 수 있습니다. 메시지 흐름에서 작업 추가 및 편집에서 보고서 작업 절차를 참조하십시오.

  3. 특정 메시지의 세부 정보를 모두 확인했으면 확인을 누릅니다.
