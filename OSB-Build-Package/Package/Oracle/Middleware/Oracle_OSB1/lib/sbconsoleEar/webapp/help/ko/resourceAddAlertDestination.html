경보 대상 추가

경보 대상이란 Oracle Service Bus에서의 경보 통지를 위한 대상 주소입니다. 경보 대상 생성 페이지에서 경보 대상 리소스를 생성할 수 있습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프로젝트 탐색기를 선택한 다음 경보 대상을 추가할 프로젝트나 폴더를 선택합니다. 프로젝트/폴더 뷰 페이지가 표시됩니다.
  3. 생성 리소스 드롭다운 목록에서 경보 대상을 선택하여 경보 대상 생성 페이지를 표시합니다.
  4. 리소스 이름 필드에 이 경보 대상에 대한 고유 이름을 입력합니다. 이 필드는 필수입니다.

    리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
  5. 리소스 설명 필드에 경보 대상에 대한 설명을 입력합니다.
  6. 이 경보 대상 리소스에 포함할 대상으로 다음 대상 중 일부 또는 전부를 선택합니다.
    7. 주: 경보는 사용자가 작업 > 대시보드에서 볼 수 있는 Oracle Service Bus 관리 콘솔에 런타임으로 집계됩니다.
    • SNMP 트랩: SNMP 트랩을 지정하면 경보가 SNMP 트랩으로 보내지므로 타사 엔터프라이즈 모니터링 시스템(EMS)에서도 처리할 수 있습니다.
    • 보고: 보고를 지정하면 경보는 Oracle Service Bus 보고 모듈로 전송되며 Oracle Service Bus에서 보고 API 제공자를 사용하여 개발할 수 있는 사용자 정의 보고 제공자를 사용하여 캡처할 수 있습니다. 이를 통해 타사가 사용자 정의 Java 코드로 경보를 받고 처리할 수 있습니다.
    • 경보 로깅: 예를 선택하면 이 경보 대상으로 전송된 경보가 경보 로그에 기록됩니다.
  7. 전자 메일 받는 사람을 경보 대상 리소스 정의에 추가합니다. 전자 메일 받는 사람 추가를 참조하십시오.
  8. JMS 대상을 경보 대상 리소스 정의에 추가합니다. JMS 대상 추가를 참조하십시오.
  9. 저장을 누릅니다. 현재 세션에 새 경보 대상이 저장됩니다.
  10. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
