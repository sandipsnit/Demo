MQ 접속 추가

시작하기 전에

Oracle Service Bus에서 지원되는 WebSphere MQ 버전은 http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html에서 Oracle Fusion Middleware Supported System Configurations를 참조하십시오. WebSphere MQ 클라이언트 라이브러리는 Oracle Service Bus 설치 프로그램과 함께 제공되지 않습니다. WebSphere MQ를 설치하여 WebSphere MQ 클라이언트 라이브러리를 얻으십시오.

Oracle Service Bus에서 MQ 접속 리소스 및 MQ 전송을 구성하고 사용하려면 클래스 경로에 MQ 클라이언트 라이브러리를 추가해야 합니다. 한 가지 방법은 DOMAIN_NAME/lib 디렉토리에 com.ibm.mq.jar 라이브러리를 복사하는 것입니다. 여기서 DOMAIN_NAME은 도메인이 있는 디렉토리 이름(일반적으로 MW_HOME\user_projects\domains\DOMAIN_NAME)입니다.

MQ 접속 리소스를 추가하려면 다음을 수행합니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프로젝트 탐색기를 선택한 다음 MQ 접속 리소스를 추가할 프로젝트나 폴더를 선택합니다. 프로젝트/폴더 뷰 페이지가 표시됩니다.
  3. 리소스 생성 드롭다운 목록에서 유틸리티의 MQ 접속을 선택하여 새 MQ 접속 리소스 생성 페이지를 표시합니다.

    4. 표 8-2에 표시된 접속 매개변수를 지정합니다.

    표 8-2 MQ 접속 매개변수
    속성
    생성 또는 편집
    리소스 이름
    이 MQ 접속 리소스에 대한 고유 이름을 입력합니다.

    리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
    리소스 설명
    MQ 접속 리소스에 대한 설명을 입력합니다.
    접속 유형
    MQ 대기열 관리자에 접속하기 위해 다음 모드 중 하나를 선택합니다.
    • tcp mode - TCP/IP를 사용하여 Oracle Service Bus와 다른 시스템에 있는 대기열 관리자에 연결합니다.
    • binding mode - 바인딩 모드를 사용하여 Oracle Service Bus와 동일한 시스템에 있는 대기열 관리자에 접속합니다.
    MQ 호스트 이름
    tcp 모드 접속의 경우에만:
    MQ 대기열 관리자의 호스트 이름을 입력합니다.
    MQ 포트 번호
    tcp 모드 접속의 경우에만:
    MQ 대기열 관리자 리스너의 포트 번호를 입력합니다.
    MQ 대기열 관리자 이름
    접속할 대상 MQ 대기열 관리자의 이름을 입력합니다.
    대기열 관리자 CCSID
    tcp 모드 접속의 경우에만:
    CCSID(coded character set identifier)는 접속 설정 중에 사용됩니다. CCSID는 주로 국제화 지원에 사용됩니다.
    자세한 내용은 http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm에서 WebSphere MQ 기본 개념을 참조하십시오.
    MQ 대기열 관리자 채널 이름
    tcp 모드 접속의 경우에만:
    대기열 관리자 서버 접속 채널 이름을 입력합니다.
    SSL 필요
    tcp 모드 접속의 경우:
    메시지 전송에 SSL을 사용하려면 체크 박스를 선택합니다. 양방향 SSL 필요 옵션이 선택되지 않으면 서버측 SSL만 사용으로 설정됩니다.
    암호화 제품군
    이 옵션은 SSL 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
    SSL에서 사용할 암호화 제품군 알고리즘을 선택합니다.
    암호화 제품군 알고리즘은 WebSphere MQ 서버와 WebSphere MQ 클라이언트간 메시지 통신을 암호화 및 해독하는 데 사용됩니다. 따라서 암호화 제품군 알고리즘은 WebSphere MQ 서버와 통신할 SSL을 사용할 때 지정되어야 합니다.
    양방향 SSL 필요
    이 옵션은 SSL 필요 체크 박스가 선택된 경우에만 사용할 수 있습니다.
    클라이언트측 및 서버측 SSL 인증을 모두 사용으로 설정하려면 체크 박스를 선택합니다.
    서비스 키 제공자에 대한 참조
    양방향 SSL 필요를 선택한 경우 적절한 키 저장소 및 트러스트 저장소 정보를 임포트 위해 서비스 키 제공자에 대한 참조를 제공해야 합니다.
    서비스 키 제공자의 이름 및 경로(project/folder)를 입력하거나 찾아보기를 눌러 서비스 키 제공자 선택 페이지에서 선택합니다.
    정적 서비스 계정에 대한 참조
    tcp 모드 접속의 경우에만:
    사용자 이름 및 비밀번호 인증에 필요합니다. 정적 서비스 계정의 경로(프로젝트/폴더) 및 이름을 입력하거나 찾아보기를 눌러 브라우저에서 선택합니다.
    WebSphere MQ 버전
    WebSphere MQ 버전을 선택합니다.
    • 5.3
    • 6.0
    • 7.0

    주: 버전 5.3 및 6.0은 IBM에서는 더 이상 지원되지 않습니다. 이러한 버전은 Oracle Service Bus에서 더 이상 사용되지 않습니다.
    MQ 접속 풀 크기
    MQ 접속 풀 크기를 입력합니다.
    MQ 접속 시간 초과
    사용되지 않은 접속이 삭제되는 시간 간격(초)을 입력합니다. 기본값은 1800초입니다.
    MQ 접속 최대 대기 시간
    접속을 사용할 수 있을 때까지 대기할 최대 시간(밀리초)을 입력합니다. 해당 시간 간격 내에 접속되지 않으면 Oracle Service Bus에서 예외 사항이 발생합니다. 기본값은 3000밀리초입니다.

  4. 저장을 누릅니다. 현재 세션에서 해당 MQ 접속 리소스가 생성되고 저장됩니다.
  5. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
