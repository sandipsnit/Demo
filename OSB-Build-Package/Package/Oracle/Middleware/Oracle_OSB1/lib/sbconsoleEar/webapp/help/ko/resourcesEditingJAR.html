JAR 편집

JAR 리소스 보기 - 구성 페이지를 사용하여 JAR 파일을 갱신합니다. Oracle Service Bus를 사용하여 JAR을 편집할 수 없으므로 JAR 파일을 갱신하면 기본적으로 리소스가 새 버전의 JAR을 가리키게 됩니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. JAR 찾기에 설명된 대로 JAR을 찾습니다.
  3. JAR 이름을 누릅니다. JAR 리소스 보기 - JAR 구성페이지에는 표 7-2에 나열된 정보가 표시됩니다.

    4. 표 7-2 JAR 정보 
    속성
    설명
    마지막으로 수정한 사람
    이 JAR을 구성으로 임포트한 사용자의 이름입니다.
    마지막 수정 시간
    사용자가 이 JAR을 구성으로 임포트한 날짜와 시간입니다. 날짜 및 시간 링크를 눌러 이 리소스의 변경 내역을 봅니다. 변경 내역 보기 페이지를 참조하십시오.
    참조
    이 JAR에서 참조하는 JAR의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 JAR 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    참조자
    이 JAR을 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    설명
    이 JAR에 대한 설명입니다(존재하는 경우).

    기본적으로 JAR 구성이 표시되며 이러한 구성에는 표 7-3에 표시된 정보가 포함되어 있습니다.

    표 7-3 JAR 구성 세부 정보
    속성
    설명
    JAR 구성
    이 JAR의 인터페이스와 클래스의 목록입니다.
    Java 클래스에 포함된 메소드 목록을 표시하려면 클래스 이름 옆의 (+)를 누릅니다.

  4. 필드를 변경하려면 갱신을 누릅니다.

    5. 대상 필드만 편집할 수 있습니다. Oracle Service Bus에서 JAR을 편집할 수 없습니다. JAR을 갱신하려면 사용할 새 JAR을 선택해야 합니다. 갱신하는 방법은 JAR 추가를 참조하십시오.

  5. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  6. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
주: EJB 비즈니스 서비스에서 사용한 JAR를 변경한 경우(JAR 리소스가 새 버전의 JAR를 가리키도록 지정), EJB를 재배치해야 합니다. 그런 다음 이 JAR를 사용하는 EJB 서비스를 편집하고 JAR 리소스를 다시 선택하고 저장, 활성화해야 합니다. 그렇게 하면 새 JAR을 사용하도록 EJB 비즈니스 서비스가 다시 패키지화됩니다.
주: 하지만 Java 콜아웃 작업과 Tuxedo 기반 서비스는 자동으로 새 JAR을 선택합니다.
