사용자 추가

  1. WebLogic Server 관리 권한을 가진 사용자로 Oracle Service Bus 관리 콘솔에 로그인합니다. 관리 롤을 가진 사용자만 보안 구성 데이터를 수정할 수 있습니다. http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Configuring Administrative Security"를 참조하십시오.
  2. 보안 구성을 선택하여 사용자 요약 페이지를 표시합니다.
  3. 새 항목 추가를 눌러 새 사용자 생성 - 일반 구성 페이지를 표시합니다.

    4. 세션 내부나 외부에서 사용자를 추가할 수 있습니다.

  4. 사용자 이름 필드에 고유 이름을 입력합니다. 이 필드는 필수 필드입니다.
  5. 비밀번호 필드에 비밀번호를 입력합니다. 이 필드는 필수 필드입니다.
    6. 주: 인증 제공자는 최소 비밀번호 길이를 지정할 수 있습니다. WebLogic 인증 제공자에 정의된 사용자의 경우 기본 최소 비밀번호 길이는 8자입니다. WebLogic Server 관리 콘솔을 사용하여 이러한 설정을 사용자 정의할 수 있습니다. WebLogic 인증 제공자는 DefaultAuthenticator라는 이름으로 기본 보안 영역에서 구성됩니다.
  6. 비밀번호 필드에 입력한 비밀번호와 같은 비밀번호를 비밀번호 확인 필드에 입력합니다. 이 필드는 필수 필드입니다.
  7. 인증 제공자 필드에서 이 사용자의 인증 제공자를 선택합니다.

    8. 여러 인증 제공자가 보안 영역에 구성된 경우 해당 제공자가 목록에 나타납니다. 새 사용자에 대한 정보를 저장할 인증 제공자 데이터베이스를 선택합니다.

  8. 그룹 멤버쉽 필드에서 이 사용자의 그룹을 선택합니다.
    1. 사용 가능한 그룹 필드에서 그룹을 선택합니다.
    2. 화살표를 눌러 해당 그룹을 현재 그룹 필드로 이동합니다.
  9. 저장을 눌러 사용자를 생성합니다.

    10. Oracle Service Bus 관리 콘솔은 사용자를 저장하며 저장된 사용자는 모든 세션에서 즉시 사용할 수 있게 됩니다. 사용자를 선택할 때 한 세션을 사용한 다음 해당 세션을 폐기하면 Oracle Service Bus 관리 콘솔은 새 사용자를 삭제하지 않습니다.
