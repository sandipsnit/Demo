메시지 레벨 액세스 정책 편집

정책 세부 정보 보기 페이지에서는 웹 서비스이며 메시지 레벨 보안이 필요하도록 구성된 프록시 서비스의 메시지 레벨 액세스 제어 정책을 편집할 수 있습니다. 이 페이지에는 표 24-11과 같은 정보가 표시됩니다.

표 24-11 정책 세부 정보
속성
설명
프록시 서비스 이름
프록시 서비스에 대한 액세스 제어 페이지에서 정책 보기를 선택한 프록시 서비스 이름을 표시합니다.
제공자
영역에 대해 구성된 권한 부여 제공자를 표시합니다.
작업
보안 프록시 서비스의 작업을 나열합니다.
정책 조건
서비스 작업에서 선택한 작업을 호출할 수 있는 사용자를 결정할 조건을 표시합니다.

  1. WebLogic Server 관리 권한을 가진 사용자로 Oracle Service Bus 관리 콘솔에 로그인합니다. 관리 롤을 가진 사용자만 보안 구성 데이터를 수정할 수 있습니다. http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Configuring Administrative Security"를 참조하십시오.
  2. 액세스 정책을 찾습니다. 액세스 제어 정책 찾기를 참조하십시오.
  3. 메시지 액세스 제어 열의 보안 페이지에서 프록시 서비스 이름이나 프록시 서비스의 특정 작업을 누릅니다. 정책 편집기가 표시됩니다.

    4. 세션 내부에서 액세스 제어 정책을 편집할 수 있습니다.

  4. 권한 부여 제공자 필드에서 권한 부여 제공자를 선택합니다. XACMLAuthorizer를 선택하는 것이 좋습니다.
    5. 주: Oracle Service Bus에서는 WebLogic 기본 권한 부여 제공자가 더 이상 지원되지 않습니다. 대신, WebLogic XACML 권한 부여 제공자를 사용할 것을 권장합니다.
  5. 정책 조건을 추가합니다. 정책 조건 추가를 참조하십시오.
  6. 정책 조건 섹션에 조건을 입력한 다음 저장을 누릅니다.
