역할 조건 정의

  1. 전역 역할 페이지에서 전역 역할 조건 페이지를 표시하려면 새 전역 역할의 이름을 누릅니다.
  2. 역할 조건에서 조건 추가를 누릅니다.

    3. 다음 메시지가 표시됩니다.

    새 조건으로 사용할 술어 선택

  3. 목록 상자에서 술어를 선택합니다. 일반적으로 그룹을 선택합니다. 그룹을 사용하여 보안 역할을 생성하면 그룹의 모든 멤버, 즉 다중 사용자에게 보안 역할을 부여할 수 있습니다.
  4. 다음을 누릅니다. 그런 다음 조건 술어에 대해 선택한 내용에 따라 표 24-6과 같은 단계 중 하나를 수행합니다.

    5. 표 24-6 조건 술어 옵션
    조건 술어
    수행할 단계
    그룹을 선택하여 이 역할이 할당되어야 하는 그룹을 정의하는 인수를 한 개 이상 입력해야 하는 경우
    1. 그룹 인수 이름 필드에 그룹을 정의하는 인수를 입력합니다.
    2. 추가를 누릅니다.
    3. 필요한 경우 인수 추가가 완료될 때까지 단계 1과 단계 2를 반복합니다. 제거를 눌러 목록에서 인수를 제거할 수 있습니다.
    4. 마침을 누릅니다.
    사용자를 선택하여 이 역할이 할당되어야 하는 사용자를 정의하는 인수를 한 개 이상 입력해야 하는 경우
    1. 사용자 인수 이름 필드에 사용자를 정의하는 인수를 입력합니다.
    2. 추가를 누릅니다.
    3. 필요한 경우 인수 추가가 완료될 때까지 단계 1과 단계 2를 반복합니다. 제거를 눌러 목록에서 인수를 제거할 수 있습니다.
    4. 마침을 누릅니다.
    서버가 개발 모드입니다, 모든 사용자의 액세스 허용 또는 모든 사용자의 액세스 거부를 선택한 경우
    마침을 누릅니다.
    Access occurs between specified hours 등의 시간 제한이 있는 술어를 선택하여 시작 및 종료 시간과 GMT 오프셋을 선택하는 경우
    1. 시작 시간 필드에 hh:mm:ss AM|PM 형식으로 허용 가능한 가장 빠른 시간을 입력합니다. 예를 들어, 12:45:00 AM을 입력합니다.
    2. 종료 시간 필드에 hh:mm:ss AM|PM 형식으로 허용 가능한 가장 최근 시간을 입력합니다. 예를 들어, 12:45:00 AM을 입력합니다.
    3. GMT offset 필드에 GMT+hh:mm 형식으로 GMT 이전 시간을 입력하거나 GMT-hh:mm 형식으로 GMT 이후 시간을 입력합니다. 예를 들어, 미국의 동부 표준시는 GMT-5:00입니다.
    4. 마침을 누릅니다.
    컨텍스트 요소 정의됨을 선택하여 컨텍스트 요소 이름을 입력하는 경우
    1. Context element name 필드에 컨텍스트 요소의 이름을 입력합니다.
    2. 마침을 누릅니다.
    Context element's value equals a numeric constant, Context element's value is greater than a numeric constant 또는 Context element's value is less than a numeric constant를 선택하여 컨텍스트 요소 이름 및 비교할 숫자 값을 입력하는 경우
    1. Context element name 필드에 값을 평가할 컨텍스트 요소의 이름을 입력합니다.
    2. 숫자 값 필드에 숫자 값을 입력합니다.
    3. 마침을 누릅니다.
    Context element's value equals a string value를 선택하여 컨텍스트 요소 이름 및 비교할 문자열 값을 입력하는 경우
    1. Context element name 필드에 값을 평가할 컨텍스트 요소의 이름을 입력합니다.
    2. 문자열 값 필드에 비교할 문자열 값을 입력합니다.
    3. 마침을 누릅니다.
    이전에 액세스 또는 이후에 액세스 등의 시간 제한이 있는 술어를 선택한 경우
    1. 날짜 필드에 mm/dd/yy 형식으로 날짜를 입력합니다. 예를 들어, 1/1/04를 입력합니다. hh:mm:ss AM|PM 형식으로 시간 옵션을 추가할 수 있습니다. 예를 들어 1/1/04 12:45:00 AM을 입력할 수 있습니다.
    2. 마침을 누릅니다.
    Access occurs on specified days of the week 등의 시간 제한이 있는 술어를 선택하여 요일과 GMT 오프셋을 선택하는 경우
    1. 요일 필드에 요일을 입력합니다.
    2. GMT offset 필드에 GMT+hh:mm 형식으로 GMT 이전 시간을 입력하거나 GMT-hh:mm 형식으로 GMT 이후 시간을 입력합니다. 예를 들어, 미국의 동부 표준시는 GMT-5:00입니다.
    3. 마침을 누릅니다.
    지정된 날짜에 액세스, 지정된 날짜 이전에 액세스 또는 지정된 날짜 이후에 액세스 등의 시간 제한이 있는 술어를 선택한 경우
    1. Day of the Month 필드에 -31에서 31 사이 범위의 값을 사용하여 현재 월의 서수 날짜를 입력합니다. 음수 값은 월의 마지막 날짜로부터 거꾸로 세므로 월의 마지막 날짜는 -1로 지정됩니다. 0은 월의 첫 번째 날짜 이전 날짜를 나타냅니다.
    2. GMT offset 필드에 GMT+hh:mm 형식으로 GMT 이전 시간을 입력하거나 GMT-hh:mm 형식으로 GMT 이후 시간을 입력합니다. 예를 들어, 미국의 동부 표준시는 GMT-5:00입니다.
    3. 마침을 누릅니다.

  5. 필요한 경우 단계를 반복하여 다른 롤 조건을 기반으로 표현식을 추가합니다. 롤 조건 섹션에서 표 24-7과 같은 단계 중 하나를 수행하여 표현식을 수정할 수 있습니다.

    6. 표 24-7 롤 조건 옵션
    수행할 작업
    수행할 단계
    선택한 표현식의 순서를 변경합니다.
    위로 이동 및 아래로 이동을 누릅니다.
    역할 조건을 병합하거나 병합을 취소하고 강조 표시된 and 및 or 문을 표현식 간에 전환합니다.
    결합 및 결합 취소를 누릅니다.
    조건을 부정으로 생성합니다. 예를 들어, 연산자 그룹 아님은 역할에서 연산자 그룹을 제외합니다.
    무효화를 누릅니다.
    선택한 표현식을 삭제합니다.
    제거를 누릅니다.

  6. 역할 조건 섹션에 있는 모든 표현식이 올바르면 저장을 누릅니다.
  7. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
