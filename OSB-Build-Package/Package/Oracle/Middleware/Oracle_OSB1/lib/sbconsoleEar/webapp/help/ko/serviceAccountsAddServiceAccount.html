서비스 계정 추가

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프로젝트 탐색기를 선택한 다음 서비스 계정을 추가할 프로젝트나 폴더를 선택합니다. 프로젝트/폴더 뷰 페이지가 표시됩니다.
  3. 리소스 생성 드롭다운 목록에서 서비스 계정을 선택하여 새 서비스 계정 생성 페이지를 표시합니다.
  4. 리소스 이름 필드에 이 서비스 계정에 대한 고유 이름을 입력합니다.

    리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
  5. 리소스 설명 필드에 서비스 계정에 대한 설명을 입력합니다.
  6. 리소스 유형에서 다음 중 하나를 수행합니다.
    • 들어오는 클라이언트 요청에서 받는 사용자 이름과 비밀번호를 제공하는 서비스 계정을 생성하려면 통과를 선택합니다.
    • 서비스 계정 구성과 함께 저장하는 사용자 이름과 비밀번호를 제공하는 서비스 계정을 생성하려면 정적을 선택합니다.
    • 하나 이상의 인증된 클라이언트의 사용자 이름을 지정한 사용자 이름과 비밀번호로 매핑하는 서비스 계정을 생성하려면 매핑을 선택합니다.
  7. 단계 6에서 선택한 리소스 유형에 따라 표 16-2와 같은 단계 중 하나를 수행합니다.

    8. 표 16-2 리소스 유형 옵션 
    선택한 리소스 유형
    작업 단계
    통과
    마지막을 누릅니다.
    정적
    1. 다음을 누릅니다.
    2. 사용자 이름 필드, 비밀번호 및 비밀번호 확인 필드에 사용자 이름과 비밀번호를 입력합니다.
    3. 마지막을 누릅니다.
    매핑
    하나 이상의 클라이언트 사용자 이름을 지정하는 사용자 이름과 비밀번호에 매핑하는 서비스 계정을 생성하려면 다음을 수행합니다.
    1. 다음을 누릅니다.
    2. 권한 있는 원격 사용자 입력 테이블에서 다음 작업을 수행합니다.
      1. 원격 사용자 이름, 비밀번호와 비밀번호 확인 필드에 아웃바운드 요청으로 보낼 사용자 이름과 비밀번호를 입력합니다.
      2. 추가를 누릅니다.

        3. 사용자 매핑이 원격 사용자 테이블에 추가됩니다.

      3. (선택 사항) 권한 있는 원격 사용자 입력 테이블에 추가 원격 사용자를 추가합니다.
    3. 다음을 누릅니다.
    4. 인증된 클라이언트를 원격 사용자 이름과 비밀번호에 매핑하려면 권한 있는 로컬 사용자 입력 테이블에서 다음을 수행합니다.
      1. 로컬 사용자 이름 필드에 인바운드 요청에 대해 인증된 클라이언트를 식별하는 이름을 입력합니다.

        2. 이 사용자를 Oracle Service Bus 관리 콘솔의 보안 구성 모듈에 아직 추가하지 않은 경우 런타임 환경에서 이 매핑을 사용하기 전에 추가해야 합니다. 사용자 추가를 참조하십시오. Oracle Service Bus를 사용하면 존재하지 않는 로컬 사용자를 위한 매핑을 생성할 수 있지만 매핑은 인증된 사용자와 일치하지 않으며 사용되지도 않습니다.

      2. 원격 사용자 이름 목록에서 로컬 사용자 이름 필드에 지정한 인증된 사용자에 대해 아웃바운드 요청으로 보낼 사용자 이름을 선택합니다.
      3. 추가를 누릅니다.
    5. anonymous 클라이언트를 원격 사용자 이름으로 매핑하려면 다음 작업을 수행합니다.
      1. 익명 요청 매핑 체크 박스를 선택합니다.
      2. 원격 사용자 선택 목록에서 모든 익명 사용자에 대해 아웃바운드 요청으로 보낼 사용자 이름을 선택합니다.
    6. 마지막을 누릅니다.

  8. 저장을 누릅니다. 현재 세션에서 해당 서비스 계정이 생성되고 저장됩니다.
  9. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
