서비스 계정 편집

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 서비스 계정 찾기에 설명된 대로 서비스 계정을 찾습니다.
  3. 서비스 계정 이름을 누릅니다. 그러면 서비스 계정 세부 정보 보기 페이지에 표 16-3과 같은 정보가 표시됩니다.

    4. 표 16-3 서비스 계정 세부 정보
    속성
    설명
    마지막으로 수정한 사람
    이 서비스 계정을 생성하거나 서비스 계정을 구성으로 임포트한 사용자입니다.
    마지막 수정 시간
    사용자가 이 비즈니스 서비스를 생성했거나 구성으로 임포트한 날짜와 시간입니다. 날짜 및 시간 링크를 눌러 이 리소스의 변경 내역을 봅니다. 변경 내역 보기 페이지를 참조하십시오.
    참조
    이 서비스 계정에서 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    참조자
    이 서비스 계정을 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 예를 들어, JMS 전송 프로토콜을 사용하는 프록시 서비스의 JMS 서비스 계정으로 이 서비스 계정을 선택한 경우 링크를 누르면 해당 프록시 서비스가 참조로 나열됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    설명
    이 서비스 계정에 대한 설명입니다(존재하는 경우).

  4. 필드를 변경하려면 편집을 누릅니다. 필드에 대한 설명은 서비스 계정 추가를 참조하십시오.

    5. 리소스 이름 필드는 변경할 수 없습니다.

  5. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  6. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
주: 수정한 서비스 계정이 WebLogic JMS 서버로 인증하는 데 사용되면 JMS 서버는 최대 60초까지 수정한 내용을 인식하지 못할 수도 있습니다. 기본적으로 WebLogic Server JMS는 60초마다 각 대상에 대해 권한을 확인합니다. 이 동작을 변경하려면 다음 시스템 속성에 Weblogic Server JMS에서 사용 권한을 확인할 빈도(초)를 설정하도록 WebLogic Server 시작 명령을 수정합니다.weblogic.jms.securitycheckinterval
이 속성 값이 0이면 JMS 리소스의 모든 send, receive 및 getEnumeration 작업에 대해 사용 권한을 확인합니다.
