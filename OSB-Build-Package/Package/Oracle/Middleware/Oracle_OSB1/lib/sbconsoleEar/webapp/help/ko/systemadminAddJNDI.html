JNDI 제공자 추가

JNDI 제공자 리소스를 추가 및 구성하여 Oracle Service Bus에서 시스템 리소스로 사용할 수 있게 합니다. 사용자는 활성 세션에 있어야 합니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 시스템 관리 > JNDI 제공자를 선택합니다.
  3. 추가를 누릅니다.
  4. 이름 필드에 JNDI 제공자 리소스에 대한 이름을 입력합니다. 이 필드는 필수입니다.

    리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
  5. 설명 필드에 JNDI 제공자에 대한 간략한 설명을 입력합니다.
  6. JNDI 캐시 옵션을 사용으로 설정(기본값)으로 유지하거나 사용 안함으로 설정을 선택합니다.

    7. 사용으로 설정된 경우, JNDI 컨텍스트 및 JNDI 객체가 로컬로 캐시되고 객체 조회를 수행할 때 성능이 향상됩니다. JNDI 캐시를 사용으로 설정으로 유지하는 것이 좋습니다.

  7. 제공자 URL 필드에 protocol://host:port 형식으로 JNDI 제공자의 URL을 입력합니다.

    8. 모든 프로토콜을 사용할 수 있습니다(예: http, https, t3, t3s, iiop, iiops)

    이 필드는 필수 필드입니다.

  8. 클러스터의 경우 JNDI 제공자 URL은 다음 형식과 같이 관리 서버 목록을 콤마로 구분하여 구성되어야 합니다.

    9. protocol://<hostname>:<ms1port>, <hostname>:<ms2port>

  9. 사용자 이름과 비밀번호를 입력해야 대상 JNDI 제공자에 액세스할 수 존재하는 경우 사용자 이름 필드에 사용자 이름을 입력하고 비밀번호와 비밀번호 확인 필드에 해당 비밀번호를 입력합니다.

    10. 이러한 필드는 선택 사항으로, JNDI 트리에 보안이 유지될 때만 필요합니다.

  10. 저장를 눌러 현재 세션의 JNDI 제공자 리소스를 저장합니다.
  11. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
