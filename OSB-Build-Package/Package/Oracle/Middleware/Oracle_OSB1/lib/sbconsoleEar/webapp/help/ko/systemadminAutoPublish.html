자동 게시 사용

자동 게시를 사용하여 프록시 서비스를 레지스트리에 자동으로 게시할 수 있습니다. 이렇게 하려면 프록시 서비스 생성/편집 - 일반 구성 페이지에서 레지스트리에 게시 옵션을 사용으로 설정해야 합니다. 일반 구성 페이지를 참조하십시오. 레지스트리에 게시 옵션은 기본 레지스트리가 존재하는 경우에만 표시됩니다. 기본 UDDI 구성 설정을 참조하십시오.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프로젝트 탐색기를 선택합니다.
  3. 프로젝트 폴더를 확장하고 편집할 프록시 서비스를 선택합니다.
  4. 프록시 서비스 보기 페이지의 아래쪽에서 편집을 누릅니다.
  5. 레지스트리에 게시를 선택한 다음 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  6. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.

    7. 서비스가 기본 레지스트리에 게시되도록 구성됩니다.

주:
주:
  • 서비스가 레지스트리에 성공적으로 게시되면 서비스를 레지스트리에 자동 게시하도록 선택한 옵션을 더 이상 사용할 수 없습니다.
주:
  • 서비스가 성공적으로 게시되지 않은 경우 서비스를 다시 게시할 수 있습니다. 서비스를 다시 게시하려면 자동 게시 상태 페이지에서 서비스를 선택하고 게시를 누릅니다.
주:
  • 레지스트리에 게시 옵션을 사용으로 설정하면 프록시 서비스가 생성되거나 편집되는 즉시 게시되고 세션이 활성화됩니다. 로컬 전송을 사용한 서비스를 제외하고 모든 프록시 서비스에 레지스트리에 게시 옵션을 사용할 수 있습니다.
