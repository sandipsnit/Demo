JNDI 제공자 편집

이 페이지를 사용하여 필요한 경우 JNDI 제공자의 세부 정보를 보고 구성을 편집할 수 있습니다. 또한 JNDI 제공자 캐시를 재설정하고 JNDI 제공자에 대한 접속을 테스트할 수 있습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. JNDI 제공자를 찾습니다. JNDI 제공자 보기를 참조하십시오.
  3. JNDI 제공자 이름을 누릅니다.

    4. JNDI 제공자 보기- 구성 페이지에 표 30-2와 같은 JNDI 제공자 정보가 표시됩니다.

    표 30-2 JNDI 제공자 정보
    속성
    설명
    마지막으로 수정한 사람
    이 JNDI 제공자 리소스를 생성하거나 수정하거나 구성으로 임포트한 사용자입니다.
    마지막 수정 시간
    이 JNDI 제공자를 작성 또는 수정했거나 구성으로 임포트한 날짜와 시간입니다. 날짜 및 시간 링크를 눌러 이 리소스의 변경 내역을 봅니다. 변경 내역 보기 페이지를 참조하십시오.
    참조
    이 JNDI 제공자에서 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 참조 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    참조자
    이 JNDI 제공자를 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    설명
    이 JNDI 제공자에 대한 설명입니다.
    JNDI 캐시
    사용 또는 사용 안함으로 설정 상태입니다. 사용으로 설정된 경우, JNDI 컨텍스트 및 JNDI 객체가 로컬로 캐시되고 객체 조회를 수행할 때 성능이 향상됩니다. JNDI 캐시를 사용으로 설정으로 유지하는 것이 좋습니다.
    제공자 URL
    JNDI 제공자 URL입니다.
    JNDI 요청 시간 초과
    JNDI 요청 시간 초과(밀리초)입니다. 기본값 0은 시간 초과가 없음을 의미합니다.
    사용자 이름 및 비밀번호
    보안 JNDI 제공자에 액세스하는 데 사용되는 사용자 이름으로, 비밀번호를 변경할 수 있습니다.

  4. 편집을 눌러 JNDI 제공자 편집- [provider-name] 페이지를 표시합니다.

    5. 이름 필드는 편집할 수 없습니다.

  5. 설명 필드에서 JNDI 제공자에 대한 간략한 설명을 편집할 수 있습니다.
  6. 제공자 URL 필드에서 JNDI 제공자의 URL을 편집할 수 있습니다.
  7. 사용자 이름 및 비밀번호 필드를 편집할 수 있습니다.

    8. JNDI 제공자에 사용자 이름과 비밀번호가 구성된 경우에는 사용자 이름을 편집할 수 없습니다. 새 비밀번호 및 비밀번호 확인 필드를 편집할 수 있습니다. JNDI에 사용자 이름과 비밀번호가 구성되지 않은 경우에는 사용자 이름 필드를 편집할 수 있게 되고 새 사용자 이름을 입력할 수 있습니다.

  8. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  9. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
