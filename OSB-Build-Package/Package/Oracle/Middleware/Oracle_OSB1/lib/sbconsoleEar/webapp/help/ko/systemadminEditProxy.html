프록시 서버 편집

이 페이지를 사용하여 필요한 경우 프록시 서버의 세부 정보를 보고 구성을 편집할 수 있습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 프록시 서버를 찾습니다. 프록시 서버 찾기를 참조하십시오.
  3. 프록시 서버 이름을 누릅니다.

    4. 프록시 서버 보기 - 구성 페이지에 표 30-6과 같은 정보가 표시됩니다.

    표 30-6 프록시 서버 정보
    속성
    설명
    마지막으로 수정한 사람
    프록시 서버 리소스를 생성하거나 수정한 사용자입니다.
    마지막 수정 시간
    사용자가 프록시 서버 리소스를 생성하거나 수정한 날짜 및 시간입니다. 날짜 및 시간 링크를 눌러 이 리소스의 변경 내역을 봅니다. 변경 내역 보기 페이지를 참조하십시오.
    참조
    이 프록시 서버가 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 참조 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    참조자
    이 프록시 서버를 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    설명
    이 프록시 서버에 대한 설명(제공된 경우)입니다.
    호스트-포트 매개변수
    다음 정보를 포함하여 프록시 서버 리소스의 각 프록시 서버에 대한 세부 정보입니다.
    • 서버 호스트: 프록시 서버의 호스트 이름 또는 IP 주소입니다.

      • Oracle Service Bus 프록시 서버의 서버 호스트 이름은 실제 프록시 서버의 서버 호스트 이름과 동일해야 합니다.

    • 일반 텍스트 포트: 프록시 서버 일반 텍스트 포트 번호입니다.
    • SSL 포트: 프록시 서버 SSL 포트 번호입니다.
    사용자 이름
    프록시 인증에 사용된 사용자 이름입니다.

  4. 편집을 눌러 프록시 서버 편집 - [server-name] 페이지를 표시합니다.

    5. 이름 필드는 편집할 수 없습니다.

  5. 설명 필드에서 프록시 서버에 대한 간략한 설명을 편집할 수 있습니다.
  6. 호스트-포트 매개변수 섹션에서 다음 정보를 입력하여 새 프록시 서버를 추가할 수 있습니다.
    1. 서버 호스트 필드에 프록시 서버의 호스트 이름이나 IP 주소를 입력합니다. 이 필드는 필수 필드입니다.

      2. Oracle Service Bus 프록시 서버의 서버 호스트 이름은 실제 프록시 서버의 서버 호스트 이름과 동일해야 합니다.

    2. 일반 텍스트 포트 필드에 프록시 서버 일반 텍스트 포트 번호를 입력합니다.
    3. SSL 포트 필드에 프록시 서버 SSL 포트 번호를 입력합니다. 일반 텍스트 또는 SSL 포트 번호를 입력해야 합니다.
    4. 추가를 누릅니다.

      5. 각 프록시 서버 리소스에 대해 다중 프록시 서버를 구성할 수 있습니다. 따라서 Oracle Service Bus에서 로드 밸런싱을 수행하고 프록시 서버 리소스에 내결함성 기능을 제공할 수 있습니다.

      다음을 수행하여 기존 프록시 서버 구성을 편집하거나 삭제할 수도 있습니다.

    • 편집할 프록시 서버의 옵션 필드에서 편집 아이콘을 누릅니다. 서버 호스트 필드, 일반 텍스트 포트 필드 및 SSL 포트 필드를 편집하고 갱신을 누릅니다.
    • 삭제할 프록시 서버의 옵션 필드에서 삭제 아이콘을 누릅니다.
  7. 프록시 인증에 사용된 사용자 이름 및 비밀번호 필드를 편집할 수 있습니다.
  8. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  9. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
