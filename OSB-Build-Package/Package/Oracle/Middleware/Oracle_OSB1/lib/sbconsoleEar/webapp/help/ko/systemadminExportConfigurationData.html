리소스 익스포트

리소스 익스포트 페이지를 사용하여 프로젝트 익스포트 또는 리소스 익스포트 작업을 수행할 수 있습니다. 전체 프로젝트를 익스포트하도록 선택하는 경우 익스포트할 프로젝트를 하나 이상 선택할 수 있습니다. 프로젝트를 확장하여 폴더와 리소스를 볼 수는 있지만 프로젝트 레벨에서만 선택할 수 있습니다. 또한 종속성 포함은 선택할 수 없습니다.

세션 내부에서 작업을 하든지, 외부에서 작업을 하든지 프로젝트 및 리소스를 익스포트할 수 있습니다. 세션 내부에서 익스포트하면 리소스는 세션 리소스가 되며 구성이 불완전하거나 충돌이 있을 수 있습니다. 세션 외부에서 익스포트하면 리소스는 활성화된 핵심 리소스가 됩니다.

Oracle Service Bus는 사용자가 Oracle Service Bus 관리 콘솔의 보안 구성 모듈에서 생성하는 사용자, 그룹 또는 롤을 익스포트할 수 없습니다. WebLogic Server 관리 콘솔에서 생성하는 인증서 맵과 기타 보안 제공자 데이터도 익스포트할 수 없습니다. 대신 WebLogic Server 관리 콘솔을 사용하여 이 데이터를 익스포트합니다. http://www.oracle.com/pls/as111170/lookup?id=SECMG319에서 Oracle Fusion Middleware Securing Oracle WebLogic Server의 "Migrating Security Data"를 참조하십시오.

  1. 시스템 관리 > 리소스 익스포트를 선택합니다.

    2. 리소스 익스포트 페이지에 구성의 객체 목록이 표시됩니다. 각 객체의 이름, 유형, 마지막 수정 시간 및 참조가 표시됩니다.

    서비스의 경우, 최종 수정된 시간 기록은 서비스 전달 또는 파이프 라인 구성뿐만 아니라 서비스 정의에 적용된 수정 사항도 반영합니다. 하지만 반영하지 못하는 경우, 해당 서비스에 대해 정의된 경보에 적용된 변경 사항을 반영합니다.

  2. 프로젝트 익스포트 작업을 수행할지 리소스 익스포트 작업을 수행할지 여부를 선택합니다.
    3. 주: 프로젝트를 익스포트하면 전체 프로젝트 JAR 파일을 임포트할 때 리소스가 삭제될 수도 있습니다. 자세한 내용은 리소스 임포트를 참조하십시오.
  3. 익스포트할 프로젝트 또는 리소스를 선택합니다. 프로젝트 익스포트를 선택한 경우, 개별 리소스에 대한 체크 박스를 선택 해제할 수 없습니다.
    1. 프로젝트 폴더를 확장합니다. 프로젝트에 포함된 각 객체의 이름, 유형, 마지막 수정 시간 및 참조가 표시됩니다.
    2. 익스포트할 프로젝트 또는 리소스와 연관된 체크 박스를 선택합니다.
    3. 익스포트할 프로젝트 또는 리소스와 연관된 체크 박스의 선택을 취소합니다.
  4. 리소스 익스포트를 선택한 경우 종속성 포함 체크 박스를 선택하거나 선택을 취소합니다. 이 옵션을 사용하면 이 리소스가 참조하는 다른 모든 리소스를 익스포트할 수 있습니다.
  5. 익스포트를 눌러 JAR 파일을 생성하여 익스포트합니다.

마친 후에

해당하는 경우 보안 설정 익스포트 지정에서 계속 진행합니다.
