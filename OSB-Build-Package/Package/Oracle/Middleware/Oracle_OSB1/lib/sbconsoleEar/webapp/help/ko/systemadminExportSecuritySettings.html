보안 설정 익스포트 지정

익스포트에 서비스 계정 또는 서비스 키 제공자가 포함되면 리소스 익스포트 - 보안 설정 페이지가 표시됩니다.

각 서비스 계정에 대해 Oracle Service Bus는 사용자 이름과 비밀번호 또는 로컬 사용자를 원격 사용자 맵으로 익스포트합니다(서비스 계정에 저장된 데이터에 따라). 각 서비스 키 제공자에 대해 Oracle Service Bus는 PKI 인증서 매핑 제공자에서 키 쌍 바인딩으로 별칭을 익스포트합니다. 키 저장소에서 개인 키, 인증서 또는 다른 데이터는 익스포트하지 않습니다. 키 저장소 데이터는 키 저장소 공급업체에서 제공하는 툴을 사용하여 익스포트해야 합니다.

Oracle Service Bus에서 서비스 계정, 서비스 키 제공자, UDDI 레지스트리, JNDI 제공자 또는 SMTP 제공자 리소스에 있는 사용자 이름과 비밀번호 데이터를 JAR 파일로 익스포트 전에 암호화하려면 다음을 수행합니다.

  1. 민감한 데이터 보호 체크 박스를 선택합니다.
  2. 비밀번호를 입력하고 확인합니다.

    3. 이 JAR을 임포트할 때 이 비밀번호를 지정하지 않으면 Oracle Service Bus는 암호화된 사용자 이름과 비밀번호 데이터가 포함된 리소스를 임포트하지 않습니다. 비밀번호를 지정하지 않고도 JAR에서 암호화되지 않은 다른 모든 리소스를 임포트할 수 있습니다.

  3. 익스포트 완료 단추를 누릅니다.
  4. 익스포트 또는 익스포트 완료를 누르면 구성 데이터 처리 팝업 창이 표시됩니다. Oracle Service Bus 관리 콘솔에서 다른 작업을 수행하기 전에 이 창이 닫힐 때까지 기다려야 합니다. 파일 다운로드 대화상자가 표시되면 창이 자동으로 닫힙니다.
  5. 파일 다운로드 대화상자에서 열기를 눌러 익스포트한 JAR 파일을 열거나 저장을 눌러 JAR 파일을 데스크톱에 저장합니다.
