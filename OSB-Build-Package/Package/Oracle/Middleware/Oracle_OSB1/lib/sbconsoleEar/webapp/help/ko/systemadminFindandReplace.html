환경 값 찾기 및 바꾸기

찾기 및 바꾸기 페이지를 사용하여 도메인마다 다르게 설정되는 환경 값을 검색할 수 있습니다. 환경 값은 한 도메인에서 다른 도메인으로(예: 테스트에서 운영으로) 구성을 이동할 때 값이 변경될 가능성이 큰 구성 데이터의 미리 정의된 특정 필드입니다. 환경 값은 URL, URI, 파일 및 디렉토리 이름, 전자 메일 등과 같은 엔티티를 뜻하며 경보 대상, 프록시 서비스, 비즈니스 서비스, SMTP 서버, JNDI 제공자 리소스 및 UDDI 레지스트리 항목에서 환경 값을 찾을 수 있습니다.

이 페이지는 세션을 사용 중인지 여부에 따라 다르게 동작합니다. 세션을 사용 중인 경우 환경 값을 찾아 바꿀 수 있습니다. 하지만 세션 외부에 존재하는 경우에는 환경 값을 찾을 수만 있고 모두 바꾸기는 사용 안함으로 설정합니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 시스템 관리 > 찾기 및 바꾸기를 선택합니다.
  3. 값 찾기 필드에 찾으려는 환경 값을 입력합니다.
  4. 입력한 값이 포함되는 구성의 환경 값 목록을 표시하려면 찾기를 누릅니다.

    5. 소유자, 유형 및 환경 값 열을 포함하는 관련 값 목록이 표시됩니다.

  5. 현재 세션에서 변경된 항목만 찾으려면 현재 세션에서 변경된 항목만 체크 박스를 선택합니다.
  6. 표 31-1에 설명된 대로 변수 유형에서 찾기 드롭다운 목록에서 검색할 변수 유형을 선택합니다.

  9. 원래 환경 값을 새 값으로 바꾸려면 모두 바꾸기를 누릅니다.

    10. 값 찾기 필드에 입력한 환경 값의 모든 항목이 현재 세션에서 바꿀 내용 필드에 입력한 환경 값으로 바뀝니다.

    주: 특정 환경 값은 Oracle Service Bus 관리 콘솔로 찾고 바꿀 수 없는 복잡한 XML 객체입니다. 그러나 스크립트에서 ALSBConfigurationMBean을 사용하여 이러한 환경 변수를 직접 설정할 수도 있습니다. ALSBConfigurationMBean에 대한 자세한 내용은 Oracle Service Bus API Reference를 참조하십시오. API를 통해 설정하는 것 외에도, 사용자 정의 파일을 사용하여 복잡한 유형의 환경 값을 설정할 수 있습니다. 사용자 정의 파일 실행을 참조하십시오.
  10. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.

관련 주제

EnvValueTypes에 대한 자세한 내용은 Oracle Service Bus API Reference를 참조하십시오. 이 환경 값의 Java 유형 및 위치 값이 API Reference에 지정되어 있습니다.
