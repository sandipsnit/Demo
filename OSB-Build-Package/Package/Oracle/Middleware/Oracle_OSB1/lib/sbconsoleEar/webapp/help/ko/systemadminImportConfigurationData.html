리소스 임포트

리소스 임포트 페이지를 사용하여 다른 Oracle Service Bus 도메인에서 이전에 익스포트한 전체 프로젝트 JAR 파일 또는 리소스 JAR 파일을 임포트할 수 있습니다. 익스포트한 데이터의 하위 집합만 임포트하도록 선택할 수 있습니다. 리소스가 임포트하는 시스템에 이미 존재하는 경우 해당 리소스가 갱신됩니다. 그렇지 않으면 리소스가 생성됩니다. 임포트한 JAR이 전체 프로젝트 JAR이고 임포트한 JAR 파일에 없는 리소스가 임포트하는 시스템의 동일한 프로젝트에 존재하는 경우 리소스는 삭제에 대해서만 예약됩니다. 다른 프로젝트에 있는 리소스는 삭제하지 않습니다.

프로젝트 레벨에서 익스포트했더라도 리소스 레벨에서 리소스를 임포트할 수 있습니다. 예를 들어, 시스템 리소스가 전체 프로젝트 JAR 파일로 익스포트한 경우라도 임포트할 때 선택을 취소할 수 있습니다.

구성을 익스포트할 때 사용자, 그룹, 롤 또는 인증서를 익스포트할 수 없습니다. 따라서 익스포트한 구성을 임포트할 때 해당 객체를 다시 생성해야 합니다.

주: Oracle Service Bus 구성에 속하지 않는 리소스를 임포트하는 방법은 Zip 파일에서 리소스 로드 및 URL에서 리소스 로드를 참조하십시오.
  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 시스템 관리 > 리소스 임포트를 선택합니다.
  3. 익스포트한 프로젝트 또는 리소스 JAR 파일을 선택합니다.
    1. 파일 이름 필드에서 찾아보기를 눌러 JAR 파일이 저장된 디렉토리를 찾습니다.
    2. JAR 파일을 선택하고 열기를 누릅니다.
  4. 다음을 누릅니다. 익스포트할 프로젝트 및 리소스 선택에서 계속 진행합니다.
