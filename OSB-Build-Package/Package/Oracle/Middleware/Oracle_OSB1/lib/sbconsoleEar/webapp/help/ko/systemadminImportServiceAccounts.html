서비스 계정 또는 서비스 키 제공자 임포트

주: 이전 Oracle Service Bus 버전에서는 서비스 키 제공자를 프록시 서비스 제공자라고 했습니다.

JAR 파일이 Oracle Service Bus 3.0 이상에서 생성되어 서비스 계정이나 서비스 키 제공자를 포함하고 존재하는 경우 이 리소스와 함께 이 리소스에 포함된 사용자 이름, 비밀번호, 로컬 사용자 대 원격 사용자 매핑 및 키 쌍 바인딩에 대한 별칭을 함께 임포트할 수 있습니다.

각 서비스 키 제공자에 대해 Oracle Service Bus는 키 쌍 바인딩에 대한 별칭을 PKI 인증서 매핑 제공자로 임포트합니다. 이 데이터가 익스포트하는 동안 암호화된 경우에는 데이터를 암호화하는 데 사용된 비밀번호를 입력해야 합니다. 비밀번호를 모르는 경우에는 암호화되지 않은 다른 모든 리소스를 임포트할 수 있습니다.

서비스 계정이나 서비스 키 제공자를 임포트할 때 동일한 이름의 리소스가 이미 현재 도메인에 존재하면 임포트하는 동안 보안 설정을 유지하도록 지정하지 않을 경우 이 리소스가 현재 세션 사용 중에 수정된 경우라도 임포트한 리소스가 이미 도메인에 있는 리소스를 겹쳐 씁니다. 임포트할 프로젝트 및 리소스 선택의 단계 5를 참조하십시오.
