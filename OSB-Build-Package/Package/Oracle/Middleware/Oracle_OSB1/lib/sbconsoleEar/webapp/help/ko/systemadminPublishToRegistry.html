프록시 서비스를 UDDI 레지스트리에 게시

서비스를 레지스트리에 게시하여 다른 조직에서 이 서비스를 발견하고 사용 가능하게 할 수 있습니다. Oracle Service Bus에서 개발한 모든 프록시 서비스는 UDDI 레지스트리에 게시할 수 있습니다. 서비스를 게시하고자 하는 비즈니스 엔티티를 선택하고 여러 서비스를 동시에 게시할 수 있습니다.

레지스트리에 게시하려면 먼저 해당 레지스트리에 계정이 있어야 합니다. Oracle Service Bus에서는 버전 3.0 호환 UDDI 레지스트리와의 상호 운용성을 지원합니다. 이 섹션에 설명된 구성에서는 Oracle Service Registry를 사용한다고 가정합니다.

Oracle Service Registry에서 사용자 계정을 설정하는 방법은 http://www.oracle.com/technology/products/soa/registry/index.html에서 Oracle Service Registry 제품 설명서를 참조하십시오.

레지스트리에서의 서비스 게시 취소는 Oracle Service Registry 설치에서 수행됩니다.

  1. 시스템 관리 > UDDI에 게시를 선택합니다.

    2. 세션을 사용하는 중에는 게시할 수 없습니다. 레지스트리 목록에 액세스하려면 세션을 종료하십시오. 액세스 권한이 있는 모든 레지스트리가 선택한 레지스트리의 드롭다운 목록에 표시됩니다.

  2. 게시 레지스트리 이름에서 게시할 레지스트리 이름을 선택하고 다음을 누릅니다.

    3. 단일 레지스트리가 Oracle Service Bus에서 구성되면 이 레지스트리가 기본 레지스트리가 되므로 초기 레지스트리 선택 페이지가 표시되지 않습니다. 두 개 이상의 레지스트리가 정의된 경우에만 레지스트리 선택 페이지가 표시됩니다.

    UDDI에 게시: 개별 서비스 선택 및 게시 페이지가 표시되며 Oracle Service Bus 프록시 서비스의 목록 및 각 서비스의 프로젝트 폴더 내 위치를 보여 줍니다. 최상위 프로젝트 레벨 폴더에서 시작하여 모든 서비스에 대한 이름과 유형, 설명 등에 대한 세부 정보가 표시됩니다.

  3. 게시할 서비스를 선택합니다.
    1. 프로젝트 폴더를 확장하여 정의된 프록시 서비스를 봅니다. 기본적으로 프로젝트 폴더와 폴더 내용이 게시용으로 표시됩니다.
    2. 게시할 항목을 개별적으로 선택할 수 있습니다.
  4. 비즈니스 엔티티에 서비스 게시 드롭다운 목록에서 레지스트리에서 서비스를 추가할 비즈니스 엔티티를 선택합니다. 이 비즈니스 엔티티 아래에 해당 서비스가 레지스트리에 분류됩니다.
  5. 게시를 선택하여 서비스를 게시합니다.

    6. 서비스가 성공적으로 게시되었음을 알리는 확인 메시지가 표시됩니다.

  6. 다른 항목 게시를 눌러 UDDI에 게시: 레지스트리 선택 페이지로 돌아갑니다.
