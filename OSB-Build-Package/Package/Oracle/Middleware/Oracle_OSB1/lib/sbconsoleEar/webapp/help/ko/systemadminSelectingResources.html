임포트할 프로젝트 및 리소스 선택

  1. 리소스 JAR 파일을 임포트하고 연관된 모든 리소스도 함께 임포트했는지 확인하려면 종속성 포함을 선택합니다.
    2. 주: 전체 프로젝트 JAR 파일 또는 이전 버전인 2.6 구성 파일을 임포트하는 경우에는 종속성 포함을 선택할 수 없습니다.
  2. 임포트하는 시스템에서 값을 유지하려면 열기 아이콘을 눌러 고급 설정을 표시합니다.
  3. 리소스를 다시 임포트할 때 기존 리소스의 사용자 정의를 그대로 유지하려면 환경 변수 유지를 선택합니다.
    4. 주: 기존 리소스 내의 모든 참조에 대한 사용자 정의를 유지할 수 없습니다. 사용자 정의 파일 생성을 참조하십시오.
  4. 전역 작업 설정 리소스 또는 비즈니스 또는 서비스 레벨 관련 작업 설정을 포함한 프록시 서비스와 같은 기타 리소스를 임포트하는 경우 및 임포트하는 시스템에서 작업 값을 유지하려는 경우 작업 값 유지를 선택합니다.
  5. 갱신 중인 리소스의 기존 보안 관련 구성을 겹쳐 쓰지 않으려면 다음 옵션 중 하나를 선택합니다.
    1. 보안 및 정책 구성 유지 - 보안 구성(액세스 제어 정책 제외) 및 WSDL이 아니라 서비스에 직접 바인딩된 WS 정책에 대한 참조를 유지합니다.

      2. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472에서 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus의 "Security"를 참조하십시오.

    2. 인증서 유지 - 서비스 키 제공자, 서비스 계정의 사용자 이름 및 비밀번호와 SMTP 서버, 프록시 서버, JNDI 제공자 및 UDDI 레지스트리의 사용자 이름 및 비밀번호 인증서의 PKI 인증서를 유지합니다.
    3. 액세스 제어 정책 유지 - 액세스 제어 정책을 유지합니다.

      4. 리소스 요약 정보는 표 28-1에 표시되어 있습니다.

      표 28-1 리소스 정보
      속성
      설명
      이름
      프로젝트 이름입니다.
      UDDI 레지스트리 정보, SMTP 서버, 프록시 서버, JNDI 제공자 및 전역 작업 설정과 같은 전역 리소스가 각 폴더의 시스템 폴더 아래에 나타납니다.
      유형
      리소스 유형으로, 프로젝트입니다.
      리소스를 표시하고 선택 취소하려면 프로젝트 폴더를 확장합니다.
      작업
      --- 또는 !Deletions!을 표시합니다.
      리소스에서 수행할 작업(생성, 갱신, 삭제)을 표시하려면 프로젝트 폴더를 확장합니다.
      리소스를 삭제하지 않으려면 리소스 선택을 취소합니다. 임포트한 JAR이 전체 프로젝트 JAR이고 임포트한 JAR 파일에 없는 프로젝트 리소스가 임포트하는 시스템에 존재하는 경우 리소스는 삭제에 대해서만 예약됩니다.
      참조
      각 리소스에서 참조하는 객체 수를 표시하려면 프로젝트 폴더를 확장합니다.

  6. 작업(추가, 갱신 또는 삭제)할 객체를 선택합니다.
    1. JAR 파일에 포함된 각 객체를 표시하려면 파일의 폴더를 확장합니다. 각 객체와 연관된 체크 박스가 선택됩니다.

      2. 각 객체에 대해 표시된 다음 리소스 파일 세부 정보는 표 28-2에 표시되어 있습니다.

      표 28-2 리소스 파일 세부 정보 
      속성
      설명
      이름
      리소스의 이름과 경로입니다. 리소스가 있는 폴더 이름과 프로젝트 이름입니다.
      유형
      Oracle Service Bus 리소스 유형입니다.
      주: 리소스에 중요한 보안 데이터가 존재하는 경우 암호화 아이콘이 표시됩니다.
      주의: 서비스 계정 또는 서비스 키 제공자를 선택하면 임포트하는 동안 보안 설정을 유지하도록 지정하지 않을 경우 기존 보안 데이터를 겹쳐 쓸 수 있습니다. 서비스 계정 또는 서비스 키 제공자 임포트를 참조하십시오.
      작업
      생성, 갱신 또는 삭제합니다.
      임포트한 JAR이 전체 프로젝트 JAR이고 임포트한 JAR 파일에 없는 프로젝트 리소스가 임포트하는 시스템에 존재하는 경우 리소스는 삭제에 대해서만 예약됩니다.
      참조
      이 리소스에서 참조하는 객체의 수입니다.

    2. 임포트하지 않으려는 리소스 옆에 있는 체크 박스의 선택을 취소합니다.
  7. 임포트를 누릅니다.

마친 후에

해당하는 경우 보안 설정 임포트 지정에서 계속 진행합니다. 또는 임포트 요약 검토에서 계속 진행합니다.
