테스트 콘솔

Oracle Service Bus 테스트 콘솔은 시스템 설계를 검증하고 테스트하는 데 사용되는 브라우저 기반의 테스트 환경입니다. 테스트 콘솔에서 테스트 객체(프록시 서비스, 비즈니스 서비스 또는 리소스)를 구성하고 테스트를 실행하여 결과를 볼 수 있습니다. 일부 인스턴스에서 코드를 추적할 수 있으며 특정 추적 지점에서 메시지 상태를 검사할 수 있습니다. 테스트 콘솔에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG357에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Using the Test Console"을 참조하십시오.

IntegrationAdmin 및 IntegrationDeployer 롤을 가진 사용자만 테스트 콘솔을 사용할 수 있습니다. 롤에 대한 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG463에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Roles in Oracle Service Bus"를 참조하십시오.

주: 테스트 콘솔 서비스가 실행 중이 아님을 나타내는 오류가 표시되면 관리 서버 수신 주소를 적합한 특정 주소(예: localhost)로 설정해 보십시오. 수신 주소를 설정하려면 Oracle WebLogic Server 콘솔에서 환경 > 서버 > admin_server_name > 구성 > 일반 사항으로 이동하십시오.
