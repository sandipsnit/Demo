프록시 서비스 테스트

모든 XML, 모든 SOAP, 메시징, XML 및 SOAP 등의 프록시 서비스 유형을 테스트할 수 있습니다. 웹 서비스 보안(WSS) 정책을 사용하여 SOAP 프록시 서비스를 테스트할 수 있습니다. 프록시 서비스 테스트 데이터 구성의 웹 서비스 보안을 참조하십시오.

주: 테스트 콘솔이 HTTP 사용자 정의 토큰 인증을 사용한 프록시를 호출한 경우, 인증 점검이 완료되지 않습니다.
주의: 직접 호출 옵션을 사용으로 설정하여 프록시 서비스를 테스트하면 액세스 제어를 비롯하여 일부 중요한 보안 단계가 무시됩니다. 생산 시스템에서는 테스트 서비스를 사용하지 않는 것이 좋습니다.
  1. 변경 센터에서 활성화를 눌러 Oracle Service Bus 관리 콘솔에서 테스트 기능을 사용으로 설정합니다.

    2. 리소스 브라우저 또는 프로젝트 탐색기에서 프록시 서비스를 테스트할 수 있습니다.

  2. 리소스 브라우저 > 프록시 서비스를 선택하여 프록시 서비스 요약 페이지를 표시합니다.
  3. 작업에서 테스트할 프록시 서비스와 연관된 테스트 콘솔 시작 아이콘을 누릅니다.

    4. 테스트 콘솔에서 프록시 서비스 테스트 페이지가 열립니다. 예를 들어, 제품과 함께 제공된 예를 사용하여 LoanGateway1 프록시 서비스와 연관된 아이콘을 누릅니다.

    주: 클러스터화된 도메인에서는 구성된 비즈니스 서비스나 비즈니스 서비스로 경로 지정되는 프록시 서비스를 테스트하는 테스트 콘솔을 사용할 수 없습니다.
  4. SOAP 및 XML 서비스에 대해 테스트할 WSDL 작업을 선택합니다.
  5. 프록시 서비스에 대한 테스트 데이터를 구성합니다. 이 데이터는 프록시 서비스가 클라이언트로부터 예상할 수 있는 데이터여야 합니다.

    6. 기본적으로 직접 호출 및 추적 포함이라는 두 개의 테스트 구성 옵션을 모두 사용으로 설정합니다. 직접 호출 옵션을 선택 취소할 수 있으며, 이때 추적 포함 옵션도 선택 취소됩니다. 이렇게 하면 메시지가 전송 계층을 통해 전송되는 간접 호출 방법을 사용하여 테스트를 수행할 수 있습니다.

    직접 호출 옵션(직접 호출이 선택된 상태로 유지)을 사용하고 실행 추적을 사용 안함으로 설정하며 추적 포함 체크 박스를 선택 해제할 수 있습니다.

    메시지 추적을 보려면 [직접 호출]을 선택 해제합니다.

    주: 로그 파일 또는 표준 출력(서버 콘솔)에서 추적을 보려면 Oracle WebLogic Server 로깅이 다음 심각도 레벨로 설정되어 있어야 합니다.

로그 심각도 레벨 설정에 대한 자세한 내용은 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server(http://www.oracle.com/pls/as111170/lookup?id=WLLOG141)의 "Using Log Severity Levels"를 참조하십시오.

     

  1. 실행을 누릅니다. 프록시 서비스 테스트 페이지에 결과가 표시됩니다. 테스트 결과 해석 방법에 대한 자세한 내용은 프록시 서비스 테스트 결과 보기를 참조하십시오.
  2. 테스트를 다시 실행하려면 뒤로를 누릅니다. 단계 5를 반복하고 계속 진행합니다.
