비즈니스 서비스 테스트

모든 XML, 모든 SOAP, 메시지, XML, SOAP 유형의 비즈니스 서비스를 테스트할 수 있습니다. SOAP 비즈니스 서비스는 웹 서비스 보안 정책을 사용하여 테스트할 수 있습니다. 자세한 내용은 http://www.oracle.com/pls/as111170/lookup?id=OSBAG384에서 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus의 "Testing Services with Web Service Security"를 참조하십시오.

주: 클러스터화된 도메인에서는 구성된 비즈니스 서비스나 비즈니스 서비스로 경로 지정되는 프록시 서비스를 테스트하는 테스트 콘솔을 사용할 수 없습니다.
  1. 변경 센터에서 활성화를 눌러 Oracle Service Bus 관리 콘솔에서 테스트 기능을 사용으로 설정합니다.
  2. 리소스 브라우저 > 비즈니스 서비스를 선택하여 비즈니스 서비스 요약 페이지를 표시합니다.
  3. 작업에서 테스트할 서비스와 연관된 테스트 콘솔 시작 아이콘을 누릅니다. 그러면 테스트 콘솔에서 비즈니스 서비스 테스트 페이지가 열립니다. 예를 들어, 제품과 함께 제공된 자습서를 사용하여 loanSaleProcessor에 해당하는 아이콘을 누릅니다.
  4. SOAP 및 XML 서비스에 대해 테스트할 WSDL 작업을 선택합니다.
  5. 비즈니스 서비스에 대한 테스트 데이터를 구성합니다(입력 데이터는 프록시 서비스에서 비즈니스 서비스로 전송된 메시지여야 함). 프록시 서비스를 테스트할 때 사용할 수 있는 직접 호출 및 추적 포함 옵션은 비즈니스 서비스의 경우에는 사용할 수 없습니다. 기본적으로 비즈니스 서비스는 직접 호출 옵션을 사용하여 테스트되는데, 이는 메시지가 전송 계층을 통해 전달됨을 의미합니다.

    6. 메시지 추적을 보려면 [직접 호출]을 선택 해제합니다.

    주: 로그 파일 또는 표준 출력(서버 콘솔)에서 추적을 보려면 Oracle WebLogic Server 로깅이 다음 심각도 레벨로 설정되어 있어야 합니다.

최소 로그 심각도: 정보
로그 파일: 정보
표준 출력: 정보

로그 심각도 레벨 설정에 대한 자세한 내용은 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server(http://www.oracle.com/pls/as111170/lookup?id=WLLOG141)의 "Using Log Severity Levels"를 참조하십시오.

  1. 실행을 누릅니다.

    2. 비즈니스 서비스 테스트 페이지에 결과가 표시됩니다. 자세한 내용은 비즈니스 서비스 테스트 결과 보기를 참조하십시오.

비즈니스 서비스의 첨부 테스트

이 섹션에서는 테스트 콘솔을 사용하여 비즈니스 서비스에서 첨부를 테스트하는 방법을 설명합니다.

예시 목적을 위해, 이 섹션에서는 WSDL 기반 HTTP 비즈니스 서비스가 "submitAttachment" 작업을 사용하여 SOAP 메시지에 ZIP 파일을 첨부로 보낸다고 가정합니다. 다른 유형의 이진 첨부의 경우 설정을 적절히 수정하십시오.

    1. 비즈니스 서비스의 테스트 콘솔에서 XML 페이지로 이동하여 다음과 같은 테스트 메시지용 SOAP Envelope를 입력합니다.
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. 테스트 콘솔의 첨부 섹션에서:
      1. 콘텐츠 유형: application/zip (WSDL에 정의된 대로 지정)
      2. 콘텐츠 ID: zipFile (테스트 메시지의 href="cid:zipFile" 값과 일치해야 함)
      3. 파일: 찾아보기를 누르고 SOAP Envelope에 지정된 ZIP 파일을 선택합니다.
      4. 추가를 누릅니다.

    2. 실행을 눌러 첨부 파일이 포함된 테스트 메시지를 보냅니다.

첨부 파일 전송을 성공했는지 확인하려면 서버 콘솔에서 다음과 비슷한 메시지가 submitAttachment 작업으로 로깅되는지 확인하십시오.
