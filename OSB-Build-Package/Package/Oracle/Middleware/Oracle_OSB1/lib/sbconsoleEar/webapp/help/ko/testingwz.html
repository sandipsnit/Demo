프록시 서비스 추적

직접 호출을 사용하여 프록시 서비스를 테스트하는 경우 추적이 사용으로 설정됩니다. 기본적으로 직접 호출 옵션과 함께 추적 포함 체크 박스가 선택되어 있습니다. 추적을 수행하지 않으려면 추적 포함 체크 박스를 선택 취소합니다. 추적을 사용하면 테스트 결과에 추적 세부 정보가 포함됩니다. 추적을 사용으로 설정하면 시스템에서 발생한 문제를 추적한 다음 수정을 위해 격리시킵니다. 추적 정보는 요청이 발생한 경로와 응답 경로를 코드로 표시합니다.

  1. 변경 센터에서 활성화를 눌러 Oracle Service Bus 관리 콘솔에서 테스트 기능을 사용으로 설정합니다.
  2. 리소스 브라우저 > 프록시 서비스를 선택하여 프록시 서비스 요약 페이지를 표시합니다.
  3. 작업에서 테스트할 서비스와 연관된 테스트 콘솔 시작 아이콘을 누릅니다. 그러면 테스트 콘솔에서 프록시 서비스 테스트 페이지가 열립니다.
  4. 프록시 서비스에 대한 테스트 데이터를 구성합니다. 추적을 사용으로 설정하려면 직접 호출 및 추적 포함 옵션을 선택해야 합니다. 프록시 서비스 테스트 데이터 구성을 참조하십시오.
  5. 실행을 누릅니다. 프록시 서비스 테스트 페이지에 서비스에 대한 테스트 결과와 추적 정보가 표시됩니다.
  6. 호출 추적 섹션으로 스크롤합니다.

    7. 이 섹션에는 메시지 플로우의 그래픽 표현이 표시됩니다. 서비스를 통해 메시지를 추적하고 미리 선택한 지점에서 추적에 있는 메시지의 상태를 볼 수 있습니다. 추적 지점이 자동으로 설정됩니다.

  7. 메시지 플로우를 확장하여 자세한 내용을 보려면 + 플러스 기호를 누릅니다.

    8. 추적을 보는 동안 Oracle Service Bus 관리 콘솔에서 메시지 흐름을 볼 수도 있습니다. 이러한 기능을 사용하면 메시지 흐름의 단계 및 작업에 추적을 연결할 수 있습니다. 출력을 보려면 메시지 흐름을 수정하고 추적을 다시 실행할 수 있습니다.
