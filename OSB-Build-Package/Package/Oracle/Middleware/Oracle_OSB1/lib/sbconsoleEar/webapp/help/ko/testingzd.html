XSLT 변환 테스트

XSLT(eXtensible Stylesheet Language Transformation)는 Oracle Service Bus의 XML에서 XML로의 매핑을 설명합니다. XSLT 리소스를 테스트하려면 입력 XML 문서를 제공해야 하며 테스트 콘솔은 출력 XML 문서를 반환합니다. XSLT 변환은 추가 명명된 매개변수를 가질 수 있습니다. 변환에 필요한 모든 매개변수는 구성 페이지에 표시됩니다. 기본값이 제공되지만 해당 값을 우선 적용할 수 있습니다.

표 32-6에서는 XSLT 구성 옵션에 대해 설명합니다.

표 32-6 XSLT 구성 옵션
섹션
설명
이름
테스트되는 리소스의 이름입니다.
테스트 콘솔 작업
실행
변환을 적용합니다.
 
재설정
입력 필드를 재설정합니다.
 
닫기
현재 작업을 취소합니다.
입력 및 매개변수
XSLT 리소스를 테스트하기 위한 입력 문서 및 매개변수입니다.
XML 입력
XML 입력은 파일 기반 또는 텍스트 기반일 수 있습니다. 입력에 대한 파일 참조는 텍스트 입력보다 우선합니다. 테스트에 사용할 파일을 찾아서 선택합니다. XML 입력은 필수입니다.
<param_name> ([]는 XML용)
param_name은 명명된 XSLT 매개변수입니다.
XML과 기본(문자열, integer, float 등)의 두 가지 입력 유형이 있습니다. 기본 입력 유형은 문자열입니다. XML 유형의 매개변수를 구분하려면 매개변수 이름과 관련된 체크 박스를 선택합니다.

디자인 시간 또는 런타임을 테스트할 수 있습니다.

  1. 런타임을 테스트하려면 활성화를 누릅니다. 디자인 시간을 테스트하기 위해 세션을 활성화하지 마십시오.
  2. 프로젝트 탐색기 > XSLTXSLT를 선택하여 XSLT 요약 페이지를 표시합니다.
  3. 작업에서 테스트할 리소스와 연관된 테스트 콘솔 시작 아이콘을 누릅니다. 그러면 테스트 콘솔에서 리소스 테스트 페이지가 열립니다.
  4. 페이지의 입력 및 매개변수 섹션에서 리소스에 대한 테스트 데이터를 구성합니다. 자세한 내용은 표 32-6을 참조하십시오.
  5. 실행을 누릅니다. 리소스 테스트 페이지에 결과가 표시됩니다.
  6. 다시 테스트하려면 뒤로를 누릅니다. 테스트 콘솔을 닫고 리소스를 수정하여 다시 테스트할 수 있습니다.
