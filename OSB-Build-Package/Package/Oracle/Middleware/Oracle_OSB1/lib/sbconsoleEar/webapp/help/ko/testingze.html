XQuery 변환 테스트

XQuery 맵은 XML에서 XML, XML에서 비XML, 비XML에서 XML로의 매핑을 설명할 수 있습니다. XQuery 변환은 다중 입력을 사용하여 하나의 출력을 반환할 수 있습니다. 각 입력은 XQuery 리소스에서 선언된 XQuery 외부 변수에 해당합니다. Xquery 입력 변수의 값은 기본 값(문자열, 정수, 날짜 등), XML 문서 또는 이러한 유형의 시퀀스일 수 있습니다. 출력 값은 기본 값(문자열, 정수, 날짜 등), XML 문서 또는 이러한 유형의 시퀀스일 수 있습니다.

주: 테스트 콘솔은 입력 시 시퀀스를 지원하지 않습니다.

표 32-7에서는 XQuery 구성 옵션에 대해 설명합니다.

표 32-7 XQuery 구성 옵션
섹션
설명
이름
테스트되는 리소스의 이름입니다.
테스트 콘솔 작업
실행
변환을 실행합니다.
재설정
입력 필드를 재설정합니다.
닫기
테스트 콘솔을 닫습니다.
변수
이 섹션은 XQuery 외부 변수 각각에 대한 하나의 입력 필드를 포함합니다.
<param_name> ([]는 XML용)
param_name은 XQuery 리소스의 XQuery 변수 이름입니다.
테스트 콘솔에서 유형이 간단하면 단일 행 편집 상자가 표시됩니다. 데이터가 XML이면 다중 행 편집 상자가 표시됩니다.
변수를 입력하지 않는 경우 조합 입력(<param_name> ([]는 XML용))이 사용됩니다. 변수 유형을 선언해야 합니다. XML 유형의 매개변수를 구분하려면 체크 박스를 선택합니다.
XML 입력은 파일 기반 또는 텍스트 기반일 수 있습니다. 입력에 대한 파일 참조는 텍스트 입력보다 우선합니다. 테스트에 사용할 파일을 찾아서 선택합니다.
테스트 콘솔을 통한 입력은 입력해야 하는 데이터 유형을 좀 더 쉽게 이해할 수 있도록 유형을 기반으로 렌더링됩니다. 입력되지 않은 경우 기본 유형은 XML입니다.

디자인 시간 또는 런타임을 테스트할 수 있습니다.

  1. 런타임을 테스트하려면 활성화를 누릅니다. 디자인 시간을 테스트하기 위해 세션을 활성화하지 마십시오.
  2. 프로젝트 탐색기 > XQuery를 선택하여 XQuery 요약 페이지를 표시합니다.
  3. 작업에서 테스트할 리소스와 연관된 테스트 콘솔 시작 아이콘을 누릅니다. 그러면 테스트 콘솔에서 리소스 테스트 페이지가 열립니다.
  4. 페이지의 변수 섹션에서 리소스에 대한 테스트 데이터를 구성합니다. 자세한 내용은 표 32-7을 참조하십시오.
  5. 실행을 누릅니다. 리소스 테스트 페이지에 결과가 표시됩니다.
  6. 다시 테스트하려면 뒤로를 누릅니다. 테스트 콘솔을 닫고 리소스를 수정하여 다시 테스트할 수 있습니다.
