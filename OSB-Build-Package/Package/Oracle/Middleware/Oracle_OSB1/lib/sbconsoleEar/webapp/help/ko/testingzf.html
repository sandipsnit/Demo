XQuery 표현식 편집기 및 XQuery 조건 편집기 사용

XQuery 표현식은 메시지 흐름 실행 중 메시지 컨텍스트 변수 또는 메시지 컨텍스트 변수의 일부에 대한 데이터 콘텐츠를 생성하는 데 사용됩니다. XQuery 표현식 편집기에서 테스트 콘솔을 직접 사용하여 표현식 정의를 테스트할 수 있습니다.

이와 유사하게 XQuery 조건을 사용하여 메시지 흐름에서 부울 조건을 평가할 수 있습니다. XQuery 조건 편집기에서 테스트 콘솔을 직접 사용하여 조건 정의를 테스트할 수 있습니다.

Xquery는 다중 입력을 사용하여 하나의 출력을 반환할 수 있습니다. 각 입력은 XQuery에 바인딩되지 않은 변수(XQuery에서 정의됨)에 해당합니다. Xquery 입력 값은 기본 값(문자열, 정수, 날짜 등), XML 문서 또는 이러한 유형의 시퀀스일 수 있습니다. 출력 값은 기본 값(문자열, 정수, 날짜 등), XML 문서 또는 이러한 유형의 시퀀스일 수 있습니다.

주: 테스트 콘솔은 입력 시 시퀀스를 지원하지 않습니다.

표 32-8에서는 XQuery 구성 옵션에 대해 설명합니다.

표 32-8 XQuery 구성 테스트
섹션
설명
이름
테스트할 표현식 유형입니다.
테스트 콘솔 작업
실행
변환을 적용합니다.
재설정
입력 필드를 재설정합니다.
닫기
현재 작업을 취소합니다.
변수
이 섹션은 XQuery에 바인딩되지 않은 변수 각각에 대한 하나의 입력 필드를 포함합니다.
<param_name> ([]는 XML용)
param_name은 XQuery에 바인딩되지 않은 해당 변수의 이름입니다.
테스트 콘솔에서 유형이 간단하면 단일 행 편집 상자가 표시됩니다. 데이터가 XML이면 다중 행 편집 상자가 표시됩니다. 조합 입력(<param_name>([]는 xml용))은 변수가 입력되지 않은 경우에 사용되며 변수 유형을 선언해야 합니다. XML 유형의 매개변수를 구분하려면 체크 박스를 선택합니다.
XML 입력은 파일 기반 또는 텍스트 기반일 수 있습니다. 입력에 대한 파일 참조는 텍스트 입력보다 우선합니다. 테스트에 사용할 파일을 찾아서 선택합니다.
테스트 콘솔을 통한 입력은 입력해야 하는 데이터 유형을 좀 더 쉽게 이해할 수 있도록 유형을 기반으로 렌더링됩니다. 기본 유형은 XML입니다.

  1. 파이프 라인의 메시지 흐름에 있는 작업을 편집할 때 테스트 콘솔에 액세스합니다.
    • XQuery 표현식 편집기에 액세스하려면 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.
    • XQuery 조건 편집기에 액세스하려면 인라인 XQuery 및 XPath 표현식 생성 및 편집을 참조하십시오.

      • XQuery 테스트 표현식 페이지가 표시됩니다. 요청된 모든 입력 변수가 페이지에 표시됩니다.

  2. 페이지의 변수 섹션에서 XQuery에 대한 테스트 데이터를 구성합니다. 자세한 내용은 표 32-8을 참조하십시오.
  3. 실행을 누릅니다. 테스트 페이지에 결과가 표시됩니다.
  4. 테스트를 완료하면 뒤로를 눌러 새 테스트를 실행할 수 있습니다. XQuery를 변경한 후에 새 테스트를 실행하려면 변경 내용을 적용하기 위해 테스트 콘솔을 닫았다가 다시 열어야 합니다.
