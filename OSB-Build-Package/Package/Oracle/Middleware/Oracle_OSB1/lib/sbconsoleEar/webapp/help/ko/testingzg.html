XPath 표현식 편집기 사용

XPath 표현식을 사용하여 XML 메시지 컨텍스트 변수의 하위 집합을 선택할 수 있습니다. XPath 표현식 편집기에서 테스트 콘솔을 사용하여 XPath 표현식 정의를 테스트할 수 있습니다. Xpath 표현식은 하나의 XML 문서를 입력으로 사용하며 XML 문서, 기본 유형 또는 이 두 가지 모두의 시퀀스를 결과로 생성합니다.

표 32-9에서는 XPath 표현식 구성 옵션에 대해 설명합니다.

표 32-9 XPath 구성 옵션
섹션
설명
이름
테스트할 표현식 유형입니다.
테스트 콘솔 작업
실행
변환을 적용합니다.
재설정
입력 필드를 재설정합니다.
닫기
현재 작업을 취소합니다.
변수
이 섹션은 XML 문서에 대해 이 XPath 표현식이 테스트되는 XML 문서에 해당하는 단일 입력 필드를 포함합니다.
 
XML 입력은 파일 기반 또는 텍스트 기반일 수 있습니다. 입력에 대한 파일 참조는 텍스트 입력보다 우선합니다. 테스트에 사용할 파일을 찾아서 선택합니다.

  1. 파이프 라인 메시지 흐름의 작업을 편집할 때 테스트 콘솔에 액세스합니다. XPath 표현식 편집기에 액세스하려면 인라인 XQuery 및 Xpath 표현식 생성 및 편집을 참조하십시오.
  2. 페이지의 변수 섹션에서 XPath 표현식에 대한 테스트 데이터를 구성합니다. 자세한 내용은 표 32-9를 참조하십시오.
  3. 실행을 누릅니다. 테스트 페이지에 결과가 표시됩니다.
  4. 테스트를 완료하면 뒤로 단추를 눌러 새 테스트를 실행할 수 있습니다. XPath 표현식을 변경한 후에 새 테스트를 실행하려면 변경 내용을 적용하기 위해 테스트 콘솔을 닫았다가 다시 열어야 합니다.
