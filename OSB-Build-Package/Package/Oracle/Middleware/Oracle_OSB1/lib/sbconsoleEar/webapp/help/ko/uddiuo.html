UDDI 레지스트리 추가

이 페이지에서는 UDDI 레지스트리를 추가하고 구성할 수 있습니다. 레지스트리를 구성한 다음 해당 레지스트리에 Oracle Service Bus 프록시 서비스를 게시하거나 Oracle Service Bus 프록시 서비스에서 사용하기 위해 해당 레지스트리에서 비즈니스 서비스를 임포트할 수 있습니다.

동작 중인 세션에서만 레지스트리를 구성할 수 있습니다.

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 시스템 관리 > UDDI 레지스트리를 선택합니다.
  3. UDDI 레지스트리 패널에서 레지스트리 추가를 누릅니다.
  4. 레지스트리를 구성하려면 다음 단계를 완료합니다. 별표는 필수 필드를 나타냅니다. 레지스트리를 추가할 때 설정해야 할 속성에 대한 설명은 표 29-1을 참조하십시오.
    1. 이름 필드에 레지스트리에 대한 이름을 입력합니다.

      리소스 이름 지정 제한 사항의 이름 지정 지침을 따르십시오.
    2. 조회 URL 필드에 http://host:port/APPLICATION_SERVER_CONTEXT/uddi/inquiry 형식으로 조회 URL을 입력합니다.
    3. 게시 URL 필드에 http://host:port/APPLICATION_SERVER_CONTEXT/uddi/publishing 형식으로 게시 URL을 입력합니다.
    4. 보안 URL 필드에 http://host:port/APPLICATION_SERVER_CONTEXT/uddi/security 형식으로 보안 URL을 입력합니다.
    5. 가입 URL 필드에
      http://host:port/APPLICATION_SERVER_CONTEXT/uddi/subscription 형식으로 가입 URL을 입력합니다.

      6. 기본적으로 설치 프로그램의 APPLICATION_SERVER_CONTEXT 값은 registry입니다.

    6. 사용자 이름 필드에 레지스트리 콘솔에 로그인하기 위한 사용자 이름을 입력합니다.
    7. 비밀번호 /(비밀번호 확인) 필드에 레지스트리 콘솔에 로그인하기 위한 비밀번호를 입력합니다.
    8. 이 레지스트리에 Oracle Service Bus tModel을 게시하려면 레지스트리로 tModel 로드 체크 박스를 선택합니다.

      9. 이 레지스트리에 프록시 서비스를 게시할 경우에만 이 필드가 필요합니다.

    9. 임포트한 서비스와 UDDI 레지스트리를 자동으로 동기화하려면 자동 임포트를 사용으로 설정 체크 박스를 선택합니다. 이 옵션을 선택하여 임포트한 서비스는 UDDI 레지스트리와 지속적으로 동기화됩니다.
      10. 주: 자동 동기화는 백그라운드 프로세스로, 세션의 실행 취소 기능을 사용하여 실행 취소할 수 없습니다. 자동 동기화 변경 실행 취소는 다음 동기화 주기에서 서비스가 재동기화되므로 영구적이지 않습니다. 임포트한 서비스와 UDDI 레지스트리가 동기화되지 않도록 하려면 레지스트리에서 이후 갱신을 하지 않도록 서비스를 분리해야 합니다. 서비스 분리를 참조하십시오.
  5. 확인을 눌러 제공한 URL을 확인합니다.
  6. 제공된 설정으로 레지스트리를 구성하여 현재 세션에서 해당 구성을 저장하려면 저장을 누릅니다.
  7. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
