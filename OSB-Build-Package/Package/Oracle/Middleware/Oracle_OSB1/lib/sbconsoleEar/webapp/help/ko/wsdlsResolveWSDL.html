해결되지 않은 WSDL 참조 해결

WSDL 리소스의 참조 편집 페이지를 사용하여 WSDL 임포트 및 XML 스키마 임포트 등의 WSDL 참조에 대한 매핑을 구성하여 해결되지 않은 WSDL 참조를 해결할 수 있습니다. WSDL에 WS 정책에 대한 해결되지 않은 참조가 있을 수도 있습니다.

해결되지 않은 WSDL 또는 XML 스키마 참조 해결
  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 리소스 브라우저 > WSDL을 선택합니다.
  3. WSDL 요약 페이지의 WSDL 이름 열에서 해당 WSDL 이름을 눌러 해결되지 않은 참조의 세부 정보를 표시합니다. WSDL 세부 정보 보기 페이지가 표시됩니다.
  4. WSDL 리소스의 참조 편집 페이지를 표시하려면 참조 편집을 누릅니다.
  5. 리소스 유형 필드에서 WSDL 또는 XML 스키마를 선택합니다.
  6. 찾아보기를 누릅니다. 리소스 유형에 따라 WSDL 브라우저 또는 XML 스키마 브라우저가 표시됩니다.
  7. WSDL 브라우저 또는 XML 스키마 브라우저에서 WSDL이나 XML 스키마를 선택한 다음 정의 창에서 정의를 선택합니다.
  8. 제출을 누릅니다. 선택한 WSDL이나 XML 스키마가 리소스 이름 필드에 표시됩니다.
  9. 저장을 눌러 참조를 해결합니다.
  10. WSDL 세부 정보 보기 페이지에서 확인을 누릅니다. 해당 WSDL이 적합 상태로 표시됩니다.
해결되지 않은 WS 정책 참조 해결
  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. 리소스 브라우저 > WSDL을 선택합니다.
  3. WSDL 요약 페이지의 WSDL 이름 열에서 해당 WSDL 이름을 눌러 해결되지 않은 참조의 세부 정보를 표시합니다. WSDL 세부 정보 보기 페이지가 표시됩니다.
  4. WSDL 리소스의 참조 편집 페이지를 표시하려면 참조 편집을 누릅니다.
  5. WS 정책 이름 필드에서 찾아보기를 누릅니다. 정책 브라우저가 표시됩니다.
  6. 정책 브라우저에서 WS 정책을 선택합니다.
  7. 제출을 누릅니다. 선택한 WS 정책이 WS 정책 이름 필드에 표시됩니다.
  8. 저장을 눌러 참조를 해결합니다.
  9. WSDL 세부 정보 보기 페이지에서 확인을 누릅니다. 해당 WSDL이 적합 상태로 표시됩니다.
