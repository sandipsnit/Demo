WSDL 편집

  1. 아직 이 작업을 수행하지 않은 경우 생성을 눌러 새 세션을 생성하거나 편집을 눌러 기존 세션을 입력합니다. 변경 센터 사용을 참조하십시오.
  2. WSDL 찾기에 설명된 대로 WSDL을 찾습니다.
  3. WSDL 이름을 누릅니다. WSDL 세부 정보 보기 페이지에는 표 6-2, 표 6-3 및 표 6-4에 나열된 정보가 표시됩니다.

    4. 표 6-2 WSDL 세부 정보 
    속성
    설명
    마지막으로 수정한 사람
    이 WSDL을 생성하거나 WSDL을 구성으로 임포트한 사용자입니다.
    마지막 수정 시간
    사용자가 이 WSDL을 생성하거나 WSDL을 구성으로 임포트한 날짜와 시간입니다. 날짜 및 시간 링크를 눌러 이 리소스의 변경 내역을 봅니다. 변경 내역 보기 페이지를 참조하십시오.
    참조
    이 WSDL에서 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    참조자
    이 WSDL을 참조하는 객체의 수입니다. 해당 참조가 존재하는 경우 숫자 링크를 누르면 객체 목록이 표시됩니다. 예를 들어, 이 WSDL의 포트 또는 바인딩을 특정 비즈니스 서비스나 프록시 서비스의 서비스 유형으로 선택한 경우 해당 링크를 누르면 비즈니스 서비스나 프록시 서비스가 참조로 나열됩니다. 리소스에 대한 참조 보기를 참조하십시오.
    설명
    이 WSDL에 대한 설명입니다(존재하는 경우).

    기본적으로 WSDL의 구조적 뷰가 표시되며 이 뷰에는 표 6-3에 표시된 정보가 포함됩니다.

    표 6-3 WSDL 구조 세부 정보
    속성
    설명
    대상 네임스페이스
    WSDL에 포함된 정의를 식별하는 데 사용되는 네임스페이스입니다.
    WSDL 정의
    WSDL과 관련된 속성 및 그룹입니다.
    • WSDL 포트 유형
    • WSDL 바인딩
    • WSDL 포트
    • XML 스키마 유형
    • XML 스키마 요소
    • WSDL 임포트
    • XML 스키마 임포트
    • WS 정책 참조
    • 암시적 WS 정책 참조
    WSDL 상태
    WSDL의 상태입니다.
    • 적합

      • 현재 WSDL에 포함된 XML 스키마나 WSDL에 대한 모든 위치가 지정되고 적합합니다. 또한 중첩된 XML 스키마 또는 WSDL(포함된 WSDL 또는 XML 스키마에 포함된 XML 스키마 또는 WSDL)의 모든 위치가 지정되고 적합합니다.

    • 부적합

      • 현재 WSDL에 포함된 XML 스키마나 WSDL에 대한 한 개 이상의 위치가 지정되지 않았거나 적합하지 않습니다. 또한 한 개 이상의 중첩된 XML 스키마 또는 WSDL(포함된 WSDL 또는 XML 스키마에 포함된 XML 스키마 또는 WSDL)의 위치가 지정되지 않았거나 적합하지 않을 수 있습니다. 부적합한 XML 스키마나 WSDL을 확인하려면 참조 편집을 누릅니다. 해결되지 않은 WSDL 참조 해결을 참조하십시오.

  4. 텍스트 뷰를 눌러 WSDL 세부 정보의 텍스트 뷰를 표시합니다. 텍스트 뷰에는 표 6-4에 나열된 정보가 표시됩니다.

    5. 표 6-4 WSDL 텍스트 세부 정보
    속성
    설명
    대상 네임스페이스
    WSDL에 포함된 정의를 식별하는 데 사용되는 네임스페이스입니다.
    텍스트
    이 WSDL에 사용할 텍스트입니다.

  5. 필드를 변경하려면 편집을 누릅니다. 필드에 대한 설명은 WSDL 추가를 참조하십시오.
  6. 필드를 적절하게 편집합니다.
  7. 저장을 눌러 현재 세션에서 갱신을 커밋합니다.
  8. 세션을 종료하고 구성을 런타임에 배치하려면 변경 센터에서 활성화를 누릅니다.
주: WSDL 세부 정보 보기 페이지에 개별 콜백 단추가 포함될 수도 있습니다. 이 단추는 다시 작성할 수 있는 콜백 작업을 가진 포트 유형이 WSDL에 존재하는 경우에만 표시됩니다. 이 단추를 누르면 WSDL이 다시 작성됩니다.
