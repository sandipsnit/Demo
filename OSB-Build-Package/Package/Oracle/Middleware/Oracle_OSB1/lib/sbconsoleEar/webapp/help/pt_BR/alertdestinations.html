Destinos de Alerta

Um recurso do destino de alerta captura uma lista de destinatários que podem receber notificações de alerta do Oracle Service Bus. Em contextos de monitoramento do sistema típicos, alertas gerados pelo Oracle Service Bus possuem significância para um conjunto finito de usuários.

No Oracle Service Bus, é possível configurar cada recurso do destino de alerta para incluir um conjunto de destinatários de acordo com um determinado contexto. Destinos de alerta são usados por ações de Alerta configurados no fluxo de mensagens e também por regras de alerta de SLA. Um destino de alerta pode incluir um ou mais dos seguintes tipos de destinos: console (default), stream de Dados do Relatório, trap do SNMP, e-mail, fila JMS ou tópico JMS.

No caso de destinos de e-mail e JMS, um recurso de destino pode incluir uma lista de endereços de e-mail ou URIs JMS, respectivamente.

É possível reutilizar os destinos de alerta entre as configurações de alerta para serviços. Consulte Adicionando Ações de Alerta.
