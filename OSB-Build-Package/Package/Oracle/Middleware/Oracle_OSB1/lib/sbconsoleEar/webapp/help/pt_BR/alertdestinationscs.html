Adicionando Destinatários de E-mail

Para adicionar um destino de e-mail, você deve configurar um Servidor SMTP (consulte Adicionando Servidores SMTP) ou uma sessão JavaMail no WebLogic Server (consulte "Configure access to JavaMail" no Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help em http://download.oracle.com/docs/cd/E15523_01/apirefs.1111/e13952/taskhelp/mail/CreateMailSessions.html). Se não houver recursos do Servidor SMTP ou sessões JavaMail disponíveis configuradas, não será possível configurar um destinatário de e-mail.

  1. Selecione Enviar um alerta via e-mail na lista drop-down e clique em Adicionar para exibir a página Adicionar Destinatário de E-mail.
  2. No campo Destinatários de E-mail, informe um destinatário de e-mail no formato: mailto:username@hostname

    3. Esse campo é obrigatório.

    É possível especificar vários destinatários de e-mail, informando os nomes de usuário e nomes de host em uma lista separada por vírgula. Por exemplo, mailto:username@hostname [,username_1@hostname_1]...[,username_n@hostname_n]

    Apenas o primeiro destinatário de e-mail precisa ser prefixado com o texto "mailto:".

  3. No campo Servidor SMTP, selecione o nome do Servidor SMTP para o e-mail de saída. Esse campo não é obrigatório se uma sessão de e-mail for selecionada na próxima etapa.
  4. No campo Sessão de E-mail, selecione uma sessão de e-mail disponível. Esse campo não é obrigatório se uma sessão de e-mail for selecionada na etapa anterior.
  5. No campo Nome de Origem, forneça o nome do remetente da notificação de alerta. Esse campo é opcional.
  6. No campo Endereço de Origem, forneça um endereço de e-mail válido. Este campo é obrigatório se um valor para o campo "Nome de Origem" for especificado.
  7. No campo Nome para Resposta, forneça um nome para o qual uma resposta pode ser endereçada. Esse campo é opcional.
  8. No campo Endereço para Resposta, forneça um endereço de e-mail para o qual uma resposta pode ser enviada. Este campo é obrigatório se um valor para o campo "Nome para Resposta" for especificado.
  9. No campo Timeout da Conexão, informe o número de segundos que uma conexão deve aguardar uma resposta do servidor antes de atingir o timeout. O valor default é 0.
  10. No campo Codificação da Solicitação, informe um valor de codificação de conjunto de caracteres. O valor de codificação default é iso-8859-1.
  11. Clique em Salvar. O destinatário do e-mail será adicionado à lista de destinatários deste destino de alerta e salvo na sessão atual.
