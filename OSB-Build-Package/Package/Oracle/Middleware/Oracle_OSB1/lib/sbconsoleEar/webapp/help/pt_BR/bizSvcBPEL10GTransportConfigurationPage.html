Página Configuração do Transporte BPEL-10g

Use esta página para configurar as definições de transporte para um serviço de negócios usando o protocolo de transporte BPEL-10g (Oracle BPEL Process Manager). Para obter mais informações sobre o uso do Oracle Service Bus com o Oracle BPEL Process Manager, consulte "Oracle BPEL Process Manager Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1147.

Descrição
Atribuição
O transporte BPEL é usado para enviar mensagens de solicitação do Oracle Service Bus para o Oracle BPEL Process Manager. O serviço de negócios pode atender a uma das seguintes atribuições:
  • Cliente Síncrono
    Para comunicação síncrona com um cliente do Oracle Service Bus, as únicas informações do local necessárias são o endereço BPEL. Esse endereço é capturado estaticamente como o URI do ponto final e/ou dinamicamente por meio da regravação do URI.
  • Cliente Assíncrono
    Para comunicação assíncrona com um cliente do Oracle Service Bus, um callback do Oracle BPEL Process Manager para o Oracle Service Bus é enviado em uma conexão diferente da solicitação, e você deve configurar o Oracle Service Bus para fornecer o endereço de callback correto. Para obter mais informações, consulte "Advanced Use Cases (Asynchronous)" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
  • Callback de Serviço
    Se um serviço de negócios for desenvolvido para ser um callback de serviço para o Oracle BPEL Process Manager (onde o Oracle BPEL Process Manager está chamando um serviço externo por meio do Oracle Service Bus), o endereço de callback é conhecido apenas no run-time. Use um URI do Ponto Final como bpel://callback.

    • Se você configurar o serviço de negócios com o URI do marcador, configure a lógica do pipeline para definir dinamicamente o URI em $outbound; por exemplo, usando a ação TransportHeader.

    Observação: Um serviço de negócios Callback de Serviço não suporta balanceamento de carga ou failover.
Proxy de Callback
Este campo opcional só está disponível para a atribuição Cliente Assíncrono. Esse campo permite selecionar o serviço de proxy (deve ser um proxy SB ou HTTP do tipo Qualquer SOAP) que será usado para encaminhar callbacks para o cliente do Oracle Service Bus que fez a solicitação. Para obter mais informações, consulte "Advanced Use Cases (Asynchronous)" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163.
Conta do Serviço
Para segurança do contexto de JNDI, usado para acessar o serviço de entrega do Oracle BPEL Process Manager. Clique em Procurar e selecione uma conta de serviço. Se nenhuma conta de serviço for especificada, será usado um assunto anônimo.
Não há restrição quanto ao tipo de conta de serviço que pode ser configurada, como estática ou de passagem, mas o run-time deve conseguir acessar um nome de usuário e senha.
Suspender Transação
A seleção de Suspender Transação torna o serviço de negócios não-transacional, mesmo que o serviço de negócios seja chamado por uma transação.
Se você não selecionar Suspender Transação:
  • Se o protocolo indicar um protocolo suportado pelo WebLogic Server (t3, iiop, http), a transação será propagada.
  • Se o protocolo indicar um servidor OC4J (ormi, opmn), o transporte BPEL emitirá uma exceção, pois OC4J não suporta propagação da transação.
O transporte BPEL ignora a opção Suspender Transação nas seguintes situações:
  • O serviço de negócios é chamado com a qualidade de serviço (QoS) "melhor esforço". O transporte BPEL suspende automaticamente qualquer transação que não suporte QoS.
  • O serviço de negócios é chamado com a QoS definida como "exatamente uma vez" e não há transação.
Para obter uma descrição da propagação de transações, consulte "Transaction Propagation" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1151.
Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir.
Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
