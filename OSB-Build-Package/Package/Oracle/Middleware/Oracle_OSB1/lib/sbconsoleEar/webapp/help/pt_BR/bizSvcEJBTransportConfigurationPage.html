Página Configuração do Transporte EJB

A página Criar/Editar um Serviço de Negócios - Configuração do Transporte EJB pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Use esta página para configurar as definições de transporte para um serviço de negócios usando o protocolo de transporte EJB. Tabela 18-5 descreve como usar a página.

Tabela 18-5 - Página Criar/Editar um Serviço de Negócios - Configuração do Transporte EJB
Opção
Para criar ou editar...
Passar Assunto do Chamador
Marque esta caixa de seleção para que o Oracle Service Bus transmita o assunto autenticado do serviço de proxy ao chamar o EJB e nenhuma conta de serviço esteja configurada. Observe que o campo Conta de Serviço está desativado quando essa opção é selecionada.
Conta do Serviço
Clique em Procurar e selecione uma conta de serviço na lista exibida. Se nenhuma conta de serviço for especificada, será usado um assunto anônimo. Esta opção não estará disponível se você usar a opção Passar Assunto do Chamador.

Para obter mais informações:

  • Contas de Serviço
  • Adicionando Contas de Serviço
Suporta Transação
Marque esta caixa de seleção para especificar suporte para a transação.
Jar do Cliente
Clique em Procurar e selecione um recurso JAR do cliente EJB na lista exibida. Para saber sobre a criação de recursos JAR, consulte JARs. Esse campo é obrigatório.
Jar do Conversor
Clique em Procurar e selecione um recurso JAR de classe do conversor de EJB na lista exibida. Para saber mais sobre recursos JAR do cliente EJB e classes do conversor, consulte "EJB Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.
Interface Local
Apenas EJB 2.1 - Selecione a interface EJBHome necessária das opções preenchidas pelo JAR. O nome da JNDI nesta amostra de URI deve estar associado à interface EJBHome selecionada aqui. Se o EJB não for do tipo necessário ou uma interface EJBHome não for especificada no JAR do cliente, o Oracle Service Bus exibirá uma advertência.
Interface Remota
Apenas EJB 2.1 - Este campo é preenchido automaticamente com base na configuração da Interface Inicial.
Interface de Negócios
Apenas EJB 3.0 - Selecione a interface de negócios no JAR do cliente que você deseja usar.
Namespace de Destino
Este campo é preenchido com informações selecionadas do JAR.
Estilo
Selecione Documento Agrupado ou RPC, de acordo com os requisitos. Se dois ou mais métodos do EJB de sessão sem preservação do estado tiverem o mesmo número e tipo de dados de parâmetros, e você quiser que as operações sejam orientadas por documento, você deverá especificar que elas sejam agrupadas por documento.
O estilo é importante porque, durante o roteamento ou publicação no EJB, $body deve conter conteúdo que corresponda ao estilo. Além disso, durante a chamada para um EJB, o estilo afeta o conteúdo do parâmetro, especialmente para documento agrupado. Em segundo lugar, um padrão de uso é definir um serviço de negócios de EJB e depois criar um serviço de proxy com o mesmo WSDL que encaminhe para o EJB. Nesse caso raro, deve ser considerado o estilo do WSDL, porque a ferramenta do cliente usada para chamar o proxy pode ter limitações quanto ao estilo do WSDL.
Codificação
Selecione Codificado ou Literal.
Métodos
Os métodos exibidos são aqueles da interface remota ou de negócios do EJB selecionada. Selecione os métodos necessários (é possível selecionar vários métodos). Clique em + para expandir o método. Edite os valores de parâmetro default e selecione um conversor, se fornecido (ou necessário).
É necessário excluir os métodos com parâmetros ou tipos de retorno que não sejam suportados pela pilha JAX-RPC ou é necessário associar tais argumentos com classes do conversor.
É possível alterar o nome da operação default para um determinado método. (Por default, o nome da operação é o nome do método.) Se um EJB contiver métodos com o mesmo nome, você deverá alterar os nomes das operações para que sejam nomes de operações exclusivos que exigem WSDLs exclusivos.
Observação: Se as credenciais ou definições de transação forem diferentes entre os métodos de um determinado EJB, você poderá personalizar os métodos para um determinado serviço de negócios e criar um serviço de negócios por método. Isso oferece um controle refinado sobre transações e credenciais.
Exceções
Este campo aparece se um método emitir uma exceção de negócios. Se um método de EJB emitir uma exceção que possua tipos de dados não suportados pelo JWS (Java Web Services), como um ArrayList, use o campo Exceções para selecionar uma classe de conversor que converta a exceção em um tipo suportado pelo JWS.
Sua classe do conversor deve implementar com.bea.wli.sb.transports.ejb.ITypeConverter. Classes do conversor só podem ser configuradas para exceções verificadas e não para exceções do run-time.
Compacte a classe do conversor e a classe de exceção convertida no JAR do cliente ou do conversor.
Para obter mais informações, consulte "EJB Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Negócios - Resumo.
