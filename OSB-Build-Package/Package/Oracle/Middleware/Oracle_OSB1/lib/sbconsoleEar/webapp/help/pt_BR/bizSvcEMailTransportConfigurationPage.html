Página Configuração do Transporte de E-mail

A página Criar/Editar um Serviço de Negócios - Configuração do Transporte de E-Mail pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Use esta página para configurar as definições de transporte para um serviço de negócios usando o protocolo de transporte de e-mail. Tabela 18-6 descreve como usar a página.

Tabela 18-6 - Página Criar/Editar um Serviço de Negócios - Configuração do Transporte de E-mail
Opção
Para criar ou editar...
Camada SSL Obrigatória
Selecione essa opção para se comunicar usando o Secure Sockets Layer. O número da porta SSL default para SMTP é 465.
Observações:
  • O certificado do servidor de e-mail deverá estar presente na área de armazenamento confiável do Oracle Service Bus para que a SSL funcione.
  • Se estiver usando TLS, você deverá colocar o certificado de confiabilidade na área de armazenamento confiável default ($JAVA_HOME/jre/lib/security/cacerts) ou colocá-lo em qualquer área de armazenamento confiável e adicionar o caminho no arquivo $DOMAIN_HOME/bin/setDomainEnv.*, conforme mostrado abaixo:

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<caminho da área de armazenamento confiável>

Servidor SMTP
Selecione o Servidor SMTP default a ser usado para as entradas do URI do ponto final de name@domain_name.com. Se você fornecer os parâmetros do servidor SMTP no URI do ponto final, conforme descrito na Tabela 18-3, esses recursos do servidor serão usados, em vez desta definição do Servidor SMTP.
Não selecione um servidor SMTP caso use a opção Sessão de E-mail.
Para obter mais informações sobre a criação de recursos SMTP, consulte Adicionando Servidores SMTP.
Sessão de E-mail
Informe o nome da JNDI da sessão de e-mail configurada a ser usado para entradas de URI do ponto final de name@domain_name.com. Se você fornecer parâmetros da sessão de e-mail de JNDI no URI do ponto final, conforme descrito na Tabela 18-3, essas sessões de e-mail serão usadas no lugar desta definição da Sessão de E-mail.
Não informe uma Sessão de E-mail caso use a opção Servidor SMTP.
Nome de Origem
Informe um nome para exibição da conta de e-mail de origem para esse serviço.
Endereço de Origem
Informe a conta de e-mail de origem para esse serviço.
Nome para Resposta
Informe um nome de exibição para a conta de e-mail para resposta.
Endereço para Resposta
Informe um endereço de e-mail para resposta.
Timeout de Conexão
Informe o intervalo de timeout, em segundos, antes que a conexão seja eliminada. Se você informar 0, não haverá timeout.
Codificação da Solicitação
Aceite o default ISO-8859-1 como a codificação de conjunto de caracteres para solicitações em transportes de e-mail ou informe outra codificação de conjunto de caracteres.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Negócios - Resumo.
