Página Configuração Geral

A página Criar/Editar um Serviço de Negócios - Configuração Geral pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Use esta página para modificar as definições de configuração gerais para um serviço de negócios.

Quando você cria um serviço de negócios essa é a primeira página exibida em uma série de páginas para configuração do serviço. As páginas exibidas após essa diferem de acordo com as opções feitas nesta página.

Tabela 18-1 descreve como usar a página.

Tabela 18-1 - Página Criar/Editar um Serviço de Negócios - Configuração Geral
Opção
Para criar ou editar...
Nome do Serviço
Informe um nome exclusivo para o serviço de negócios.

Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
Descrição
Informe uma descrição.
Tipo de Serviço
Um tipo de serviço define os tipos e compactação das mensagens trocadas pelo serviço. Selecione o tipo de serviço de negócios para criar:
  • Web Service de WSDL - Selecione esta opção para criar um serviço de negócios com base em um WSDL. Em seguida, informe o nome WSDL, qualificado por seu caminho (por exemplo, myProject/myFolder/myWSDL). Como alternativa, clique em Procurar para selecionar um WSDL na página Selecionar um WSDL.

    • (porta ou binding) - Informe o nome de uma porta (definido no WSDL) para descrever um endereço de transporte real ou informe o nome de um binding (definido no WSDL) para mapear para um endereço de transporte. Se você usar Procurar para selecionar um WSDL, a página Selecionar uma Definição WSDL listará as portas e os bindings definidos no WSDL. Quando você escolher uma porta ou um binding nessa página, o campo (porta ou binding) será preenchido com o nome selecionado.

  • Serviço do Tipo de Transporte - Selecione esta opção para criar um serviço que use o transporte EJB ou de Fluxo.
  • Serviço de Mensagens - Selecione esta opção para criar um serviço que troque mensagens de cada tipo de conteúdo diferente. Essas trocas podem ser de solicitação/resposta ou unilaterais. Também pode ter apenas uma resposta sem solicitação quando usado com a opção HTTP `GET' para o transporte HTTP. Ao contrário de Web Services, o tipo de conteúdo da solicitação e a resposta não precisam ser iguais.
  • Qualquer Serviço SOAP - Selecione esta opção para criar um serviço SOAP que não possua uma interface concreta definida explicitamente.

    • Mantenha o SOAP 1.1 default ou selecione SOAP 1.2 na lista drop-down.

  • Qualquer Serviço XML - Selecione esta opção para criar um serviço XML que não possua uma interface concreta definida explicitamente.

    • HTTP GET só é suportado para serviços de mensagens e para este tipo de serviço.

  • Serviço de Negócios - Selecione esta opção para clonar um serviço de negócios existente.

    • Informe o caminho (projeto/pasta) e o nome do serviço de negócios; ou clique em Procurar para selecionar o serviço de negócios na página Resumo de Serviços de Negócios.

    Como o Oracle Service Bus não aceita o mesmo URI para vários serviços, você deve alterar o URI para o serviço clonado.

  • Serviço de Proxy - Selecione esta opção para criar um serviço de negócios com base em um serviço de proxy existente.
Tipo de Serviço (continuação)
Observação: Quando um serviço é criado com base em outro serviço, as regras de alerta são mantidas da seguinte maneira:
    • Quando um serviço de proxy é criado a partir de um serviço de negócios ou um serviço de negócios é criado a partir de um serviço de proxy, as regras de alerta, se houver, são removidas.
    • Quando um serviço de proxy é criado a partir de outro serviço de proxy ou um serviço de negócios é criado a partir de outro serviço de negócios, as regras de alerta, se houver, são mantidas.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Negócios - Resumo.
