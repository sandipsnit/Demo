Página Configuração do Transporte HTTP

O transporte HTTP suporta pontos finais HTTP e HTTPS.

A página Criar/Editar um Serviço de Negócios - Configuração do Transporte HTTP pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Use esta página para configurar as definições de transporte para um serviço de negócios usando o protocolo de transporte HTTP. Tabela 18-10 descreve como usar a página.

Método de Solicitação HTTP
Este parâmetro permite que você use um dos seguintes métodos HTTP em uma solicitação:
  • POST - Transmite todos os dados, de comprimento ilimitado, diretamente sobre a conexão do soquete como parte do corpo da solicitação HTTP. A troca é invisível ao cliente e o URL não é alterado. Para solicitações baseadas em REST, POST informa o transporte para executar uma operação de criação/substituição ou executar uma ação com a solicitação.
  • GET - Você pode incluir, como parte da solicitação, algumas de suas próprias informações que descrevam melhor o que obter. Essas informações são transmitidas como uma sequência de caracteres anexada ao URL de solicitação em uma string de consulta. Você pode usar GET em um serviço de negócios com um Tipo de Serviço "Qualquer Serviço XML " ou com um Tipo de Serviço "Serviço de Mensagens" quando o Tipo de Mensagem da Solicitação for definido como "Nenhum". Para solicitações baseadas no REST, GET recuperará uma representação de um recurso remoto.
  • PUT - Você pode usar PUT em um serviço de negócios com um Tipo de Serviço "Qualquer Serviço XML" ou "Serviço de Mensagens". PUT informa ao transporte para executar uma operação de criação/substituição com uma solicitação baseada em REST, como upload de um arquivo para um local conhecido.
  • HEAD - Você pode usar HEAD em um serviço de negócios com um Tipo de Serviço "Qualquer Serviço XML" ou "Serviço de Mensagens" quando o Tipo de Mensagem de Resposta estiver definido como "Nenhum". HEAD informa ao transporte para obter informações do cabeçalho para um recurso remoto, em vez de obter uma representação completa do recurso em uma solicitação baseada em REST.
  • DELETE - Você pode usar PUT em um serviço de negócios com um Tipo de Serviço "Qualquer Serviço XML" ou "Serviço de Mensagens". DELETE informa ao transporte para executar uma operação de deleção com uma solicitação baseada em REST.
Observação: Se um método já estiver definido na variável $outbound/transport/request/http:http-method, esse valor terá precedência sobre qualquer método selecionado para o Método de Solicitação HTTP.
Autenticação
Selecione uma das seguintes opções:
  • Nenhum - Especifica que não é necessária autenticação para acessar esse serviço.
  • Básico - Especifica se é necessária autenticação básica para acessar esse serviço.

    • A autenticação básica instrui o WebLogic Server a autenticar o cliente usando um nome de usuário e senha nos provedores de autenticação configurados no realm de segurança, como um serviço de diretório LDAP (Lightweight Directory Access Protocol) e o Windows Active Directory. O cliente deve enviar seu nome de usuário e senha no cabeçalho da solicitação HTTP.

    Autenticação básica é altamente não recomendável sobre HTTP, porque a senha é enviada em texto sem formatação. No entanto, é seguro enviar senhas sobre HTTPS, porque HTTPS oferece um canal criptografado.

    Advertência: Por default, todos os usuários (autorizados e anônimos) podem acessar um serviço de negócios. Para limitar os usuários que podem acessar um serviço de negócios, crie uma política de autorização de transporte. Consulte Editando Políticas de Acesso de Transporte.

  • Certificado Cliente - Especifica comunicação criptografada e autenticação de cliente forte (SSL bilateral). Para saber mais, consulte "Configuring Transport-Level Security" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
Conta do Serviço
Informe uma conta de serviço. Uma conta de serviço é um recurso de alias para um nome de usuário e senha. Este é um campo obrigatório caso você tenha selecionado o campo Autenticação Básica Obrigatória.

Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir. Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificação da Solicitação
Aceite o default iso-8859-1 como a codificação de conjunto de caracteres para solicitações em transportes HTTP ou informe outra codificação de conjunto de caracteres.
Codificação da Resposta
Aceite o default iso-8859-1 como a codificação de conjunto de caracteres para respostas em transportes HTTP ou informe outra codificação de conjunto de caracteres.
Servidor Proxy
Informe um recurso do servidor proxy ou clique em Procurar para escolher uma entrada na lista de recursos configurados do servidor proxy.
Seguir redirecionamentos de HTTP
Marque esta caixa de seleção para especificar que os redirecionamentos HTTP (que são solicitados com um código de resposta 3xx) devem ser seguidos automaticamente. A mensagem é redirecionada ando você envia uma mensagem de saída para o URL de um serviço de negócios e esse serviço retorna um código de resposta (por exemplo, 302) que informe que o URL não é mais válido e essa solicitação precise ser enviada para outro URL.
Se você marcar a caixa de seleção Seguir Redirecionamentos de HTTP, o Oracle Service Bus reenviará automaticamente a solicitação para o novo URL sem nenhuma ação de sua parte. Desmarque essa caixa de seleção se não quiser que os redirecionamentos de HTTP sejam seguidos automaticamente.
Usar Modo de Streaming em Partes
Selecione esta opção se quiser usar codificação de transferência em partes HTTP para enviar mensagens.
Observação: não use parte do stream se usar a opção Seguir Redirecionamentos de HTTP. O modo de partes não pode ser automaticamente redirecionado e autenticado.
Permanência na sessão
Selecione essa opção se você quiser usar a permanência na sessão (também conhecido como afinidade de sessão) para solicitações HTTP do serviço de negócios.
Nome de ID da Sessão de Permanência
Se a opção Permanência na Sessão estiver ativada, informe um nome identificador para a sessão.
