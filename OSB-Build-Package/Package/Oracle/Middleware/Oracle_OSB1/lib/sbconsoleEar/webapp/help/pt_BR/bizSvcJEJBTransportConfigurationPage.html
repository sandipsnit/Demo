Página Configuração do Transporte JEJB

Use esta página para configurar definições de transporte usando o protocolo de transporte JEJB. Para obter mais informações sobre o uso do transporte JEJB, consulte "JEJB Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036.

Tabela 18-12 - configuração do transporte JEJB
Opção
Descrição
Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir.
Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Versão da Especificação do EJB
Selecione a versão do EJB da interface remota do EJB.
Passar XMLBeans por valor
Selecione esta opção se quiser que o transporte gere uma representação XML "em linha" de argumentos POJO (um XMLObject) cujos parâmetros você pode acessar e manipular com expressões XQuery.
Observação: as informações do tipo não estão disponíveis em linha para XMLObjects transmitidos por valor. Se você usar essa opção, não poderá transmitir o XMLObject digitado como o argumento em uma Chamada Java em um pipeline de serviço de proxy.
Não selecione esta opção se quiser transmitir o POJO por referência, o que também resulta em melhor desempenho.
Passar Assunto do Chamador
Como uma alternativa para a seleção de uma Conta de Serviço, selecione esta opção para que o Oracle Service Bus passe o assunto autenticado do serviço de proxy ao chamar o EJB.
Conta do Serviço
Clique em Procurar e selecione uma conta de serviço de JNDI na lista exibida. Se nenhuma conta de serviço for especificada, será usado um assunto anônimo. Para obter mais informações, consulte o Capítulo 16, Contas de Serviço.
JAR do Cliente
Clique em Procurar e selecione um recurso JAR do cliente EJB na lista exibida. O JAR do cliente contém a interface remota ou de negócios do serviço remoto. O JAR do Cliente é registrado como um Recurso de Archive genérico.
Interface Local
Apenas EJB 2.1 - Selecione a interface EJBHome necessária das opções preenchidas pelo JAR.
Interface Remota
Apenas EJB 2.1 - Este campo é preenchido automaticamente com base na configuração da Interface Inicial.
Interface de Negócios
Apenas EJB 3.0 - Selecione a interface de negócios do JAR do cliente que você deseja chamar.
Namespace de Destino
Este campo é preenchido com informações selecionadas do JAR.
Métodos
Selecione os métodos necessários. Clique em + para expandir o método, o que permite editar os valores de parâmetro default.
É possível alterar o nome da operação default para um determinado método. Por default, o nome da operação é o nome do método. Se um EJB contiver métodos com o mesmo nome (sobrecarregado), você deverá alterar os nomes das operações para que sejam exclusivos. WSDLs requerem nomes de operações exclusivos.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Negócios - Resumo.
