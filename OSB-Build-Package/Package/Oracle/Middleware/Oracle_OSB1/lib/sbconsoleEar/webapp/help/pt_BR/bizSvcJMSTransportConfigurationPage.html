Página Configuração do Transporte JMS

A página Criar/Editar um Serviço de Negócios - Configuração do Transporte JMS pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Para obter mais informações, consulte "JMS Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Use esta página para configurar as definições de transporte para um serviço de negócios usando o protocolo de transporte JMS. Tabela 18-13 descreve como usar a página.

Tabela 18-13 - Página Criar/Editar um Serviço de Negócios - Configuração do Transporte JMS (Continuação)
Opção
Para criar ou editar...
Tipo de Destino
Selecione um tipo para o destino de ponte JMS:
  • Fila (para ponto-a-ponto)
  • Tópico (para publicação/inscrição)
Tipo de Mensagem
Selecione uma das seguintes opções:
  • Bytes (para um fluxo de bytes não-interpretados)
  • Texto (para mensagens de texto)
Esta opção estará desativada se você selecionar um Tipo de Mensagem de Java para a resposta.
Filas de Respostas
Esta opção só está disponível quando Fila é selecionado para o Tipo de Destino.
Selecione uma das seguintes opções de resposta:
  • Nenhum - Nenhuma resposta é esperada. Defina esta opção para operações unilaterais.
  • Um para todos os URIs da Solicitação - Permite informar um único URI para tratar a resposta, além de definir outros detalhes de configuração da resposta, como codificação e timeout, e como opção selecionar uma Conta de Serviço JMS para transmitir as credenciais JMS/JNDI.
  • Um por URI da Solicitação - Esta opção oferece failover de resposta, permitindo informar um URI ou destino de resposta para cada URI de solicitação.
Como opção, é possível selecionar uma conta de serviço para credenciais JMS/JNDI em cada par de solicitação/resposta.
Padrão de Resposta
Esta opção só está disponível quando você seleciona uma opção de resposta no campo Fila de Respostas.
Selecione uma das seguintes opções:
  • Selecione JMSCorrelationID para todos os serviços JAX-RPC em execução no Oracle WebLogic Server.
  • Selecione JMSMessageID para serviços JAX-RPC em execução no Oracle WebLogic Server.
Para obter mais informações, consulte "Message ID and Correlation ID Patterns for JMS Request/Response" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051.
Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do Oracle WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir.
Por exemplo, se o serviço de negócios tiver um protocolo de transporte JMS, o ponto final do serviço de negócios será um arquivo JAR MDB (bean baseado em mensagens) que pode ser associado à política de despacho específica.
Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificação da Solicitação
Informe o conjunto de caracteres para solicitações de codificação. O default é UTF-8.
Codificação da Resposta
Esta opção só está disponível quando uma das opções de resposta é selecionada no campo Filas de Respostas.
Informe o conjunto de caracteres para respostas de codificação. O default é UTF-8.
Timeout da Resposta
Esta opção só está disponível quando uma das opções de resposta é selecionada no campo Filas de Respostas.
Informe a quantidade de tempo, em segundos, para aguardar a resposta antes de eliminar a conexão. O default, zero (0), significa que a resposta nunca atinge o timeout.
Jar do Cliente
Esta opção estará desativada quando o serviço for um Serviço de Mensagens com um tipo de resposta de Java. Selecione o cliente JAR a ser usado para retirar da fila mensagens que contenham Objetos Java. Selecionar o cliente JAR garante que ele esteja no classpath.

Para obter mais informações, consulte "Sending and Receiving Java Objects in Messages" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
URI da Resposta
Esta opção só está disponível quando você seleciona a opção de resposta Um para todos os URIs da Solicitação e o padrão de resposta JMSCorrelationID.
Informe um URI de resposta no formato:
jms://host:port/connection_factory/jndi_destination
Para destinar vários servidores, use o seguinte formato:
jms://host1:port,host2:port/connection_factory/jndi_destination
Também é possível omitir o host e a porta no URI de resposta. Por exemplo:
jms:///connection_factory/jndi_destination

Quando você omite o host e a porta, a pesquisa de factory de conexão/destino ocorre no servidor local. Isso é útil, por exemplo, se o URI da solicitação for para um factory de conexão/destino estrangeiro, mas você quiser que a resposta seja enviada para o servidor local.

Observação: Embora o Oracle WebLogic Server permita o uso de barras em nomes JNDI, como "myqueues/myqueue", os nomes JNDI com barras interferem no formato de URI exigido pelo Oracle Service Bus e você não pode usá-los. Para resolver esse problema, defina e referencie um servidor externo JMS no URI. Para obter mais informações, consulte "Configure foreign servers" em Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help em http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Respostas de Solicitação
Esta opção só está disponível quando você seleciona a opção de resposta Um por URI da Solicitação para o padrão JMSCorrelationID para oferecer failover de resposta.
Para cada URI de solicitação informado na página genérica de Transporte, informe um URI de Resposta, conforme descrito no campo URI de Resposta anterior.
Você pode selecionar uma Conta de Serviço opcional para credenciais JMS/JNDI usadas pelo serviço para as filas de solicitação e de resposta.
Alvo - Respostas
Esta opção só está disponível quando você seleciona a opção de resposta Um para todos os URIs da Solicitação para o padrão de resposta JMSMessageID.
Informe o nome do servidor de Destino que deve receber respostas e informe um URI de Resposta, conforme descrito no campo URI de Resposta.
Conexões da Solicitação
Esta opção só está disponível quando você seleciona a opção de resposta Um por URI da Solicitação para o padrão JMSMessageID para oferecer failover de resposta.
Para cada URI de solicitação, identificado sequencialmente pelo número no campo No. de Seq., como opção informe um nome de Factory de Conexão JMS. Se você não informar um nome, o transporte JMS usará o factory de conexão do URI de solicitação.
Você pode selecionar uma Conta de Serviço opcional para credenciais JMS/JNDI usadas pelo serviço para as filas de solicitação e de resposta.
Alvo - Destinos
Esta opção só está disponível quando você seleciona a opção de resposta Um por URI da Solicitação para o padrão JMSMessageID para oferecer failover de resposta. Use este campo em conjunto com Conexões da Solicitação.
Cada servidor de Destino listado (determinado pelos servidores no domínio atual), como Servidores Gerenciados em um cluster, lista os URIs pelo No. de Seq., que correspondem aos do campo Conexões da Solicitação. Para cada URI de solicitação em cada destino, informe a fila de Destino nesse servidor de destino que recebe respostas.
Observação: Como o ambiente de desenvolvimento do Oracle Service Bus suporta apenas um ambiente de um servidor, só um Destino será listado. Para configurar esse campo em um ambiente de vários servidores, implante esse serviço no ambiente do runtime e conclua a configuração de serviço na Console do Oracle Service Bus Administration.
Conta de Serviço JMS
Esta opção só está disponível quando você usa as opções Nenhum ou Um para todos os URIs da Solicitação no campo Filas de Respostas.
Selecione uma conta de serviço a ser usada para o recurso JMS gerenciado pelo servidor JMS. Uma conta de serviço é um recurso de alias para um ID de usuário e senha, usado para a solicitação e resposta. A mesma conta de serviço é usada para JMS e JNDI.
Este campo é mutuamente exclusivo com a opção Passar Assunto do Chamador. Use um ou o outro para autenticação JMS/JNDI.
Para obter mais informações, consulte Contas de Serviço.
Usar SSL
Selecione esta opção apenas se as solicitações forem feitas em uma conexão TLS/SSL.
O TLS/SSL (Secure Sockets Layer) oferece conexões seguras permitindo que duas aplicações se conectem em uma rede para autenticar a identidade da outra e criptografando os dados trocados entre as aplicações. A autenticação permite que um servidor e, como opção, um cliente, verifique a identidade da aplicação na outra extremidade de uma conexão de rede. Além disso, se o administrador tiver acesso restrito a destinos JMS individuais (filas ou tópicos) definindo o controle de acesso na entrada de JNDI para o destino, o serviço deverá ser autenticado ao pesquisar a entrada na árvore de JNDI. Autentique usando uma Conta de Serviço ou a opção Passar Assunto do Chamador.
Expiração
O intervalo de tempo, em milissegundos, após o qual a mensagem expira. O valor default é 0, que significa que a mensagem nunca expira.
Ativar a Persistência de Mensagens
O modo de entrega JMS usado pelo serviço, que permite equilibrar confiabilidade com rendimento. Selecione esta opção (default) para garantia de entrega da mensagem. Cancele a seleção desta opção para aprimorar o rendimento se uma perda de mensagem ocasional for tolerável.
Unidade de Ordem
Informe uma unidade de ordem de mensagens. A unidade de ordem de mensagens permite que os produtores de mensagens agrupem as mensagens em uma única unidade relacionada à ordem de processamento. Essa única unidade de ordem requer que todas as mensagens nessa unidade sejam processadas sequencialmente na ordem em que são criadas
Passar Assunto do Chamador
Marque esta caixa de seleção para fazer com que o Oracle Service Bus transmita o assunto autenticado ao enviar uma mensagem.
Quando você ativar este campo e o serviço de negócios destinar-se a recursos JMS em um domínio diferente, ative a confiança global em ambos os domínios. Consulte Configuring Security for a WebLogic Domain no Securing Oracle WebLogic Server.
Este campo é mutuamente exclusivo com a opção Conta de Serviço. Use um ou o outro para autenticação JMS/JNDI.
Quando você ativar este campo e o serviço de negócios destinar-se a recursos JMS em um domínio diferente, ative a confiança global em ambos os domínios. Consulte "Configuring Security for a WebLogic Domain" no Oracle Fusion Middleware Securing Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=SECMG402.
Timeout de JNDI
O timeout da conexão de JNDI (em segundos) usado ao pesquisar o destino ou o factory de conexão na árvore JNDI.
O default, zero (0), significa que a conexão nunca atinge o timeout.
