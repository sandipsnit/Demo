Página Configuração do Transporte MQ

A página Criar/Editar um Serviço de Negócios - Configuração do Transporte MQ pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Para obter mais informações, consulte "MQ Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Antes de começar

Configure um recurso de Conexão MQ. Consulte Conexões MQ.

Para configurar o transporte de MQ

Use esta página para configurar as definições de transporte para um serviço de negócios usando o protocolo de transporte MQ nativo. Tabela 18-14 descreve como usar a página.

Tabela 18-14 - Página Criar/Editar um Serviço de Negócios - Configuração do Transporte MQ
Opção
Para criar ou editar...
Tipo de Mensagem
Selecione uma das seguintes opções:
  • Bytes (para um fluxo de bytes não-interpretados)
  • Texto (para mensagens de texto)
Resposta Obrigatória?
Selecione esta opção para especificar que uma resposta é esperada após o envio de uma mensagem de saída.
Intervalo de Sondagem
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Informe um intervalo de sondagem, em milissegundos. O default é 1000.
Padrão de Correlação de Resposta
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Especifique se o padrão de correlação da resposta deve ser baseado em:
  • MessageID
  • CorrelationID
  • Fila Dinâmica - Selecione esta opção se a implementação do WebSphere MQ usar filas dinâmicas para correlação de resposta. O transporte MQ suporta apenas filas dinâmicas temporárias.
Gerar Automaticamente Valor de Correlação
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Marque esta caixa de seleção para gerar automaticamente um CorrelationID ou MessageID.
Fila de Modelos
Apenas para o Padrão de Correlação de Resposta da Fila Dinâmica. Informe o nome da fila de modelo usada para gerar a fila dinâmica.
URI da Resposta MQ
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
O destino no qual a resposta deve ser publicada. Informe um URI de resposta no mesmo formato do URI do ponto final:
mq://local_queue_name?conn=mq_connection_resource
ou se você estiver usando filas dinâmicas:
mq://dynamic_queue_prefix?conn=mq_connection_resource
O dynamic_queue_prefix, que é limitado a 32 caracteres, é usado para criar a fila dinâmica no servidor MQ. O nome da fila torna-se o prefixo mais um ID exclusivo. Por exemplo, considerando o dynamic_queue_prefix, a fila dinâmica seria nomeada com algo semelhante a example123129083821.
Também é possível usar um asterisco (*) como um curinga no URI de resposta da fila dinâmica. Por exemplo:
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
Se você não fornecer um dynamic_queue_prefix no URI, o transporte usará o nome da fila dinâmica gerado pelo servidor MQ. Se você não fornecer um mq_connection_resource explícito no URI (melhor prática), o transporte usará o mq_connection_resource do URI do ponto final.
Para obter mais informações, consulte "MQ Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.
Timeout da Resposta
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Informe o número de segundos para aguardar uma resposta antes de eliminar a conexão.
Política de Distribuição
Política de despacho refere-se à instância do Gerenciador de Trabalho do WLS que você deseja usar para o ponto final do serviço.
Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Polling de Fila Dinâmico
Apenas para o Padrão de Correlação de Resposta da Fila Dinâmica. Selecione esta opção se quiser que o serviço use conexões agrupadas para filas dinâmicas.
Se deseja usar um pool de conexões separado para filas dinâmicas, considere configurar um recurso de Conexão MQ dedicado para as filas dinâmicas.
Não selecione esta opção se quiser criar uma nova instância de fila dinâmica em cada solicitação (e destruir a fila após a resposta).
Opções `PUT' do URI do ponto final
Informe as opções de mensagens PUT MQ dentre o seguinte:
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
É possível usar "|" ou "+" para separar várias opções. Por exemplo, você pode especificar o seguinte:
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
As opções de mensagem PUT MQ são aplicadas quando a mensagem é colocada na fila de saída.
URI de Resposta Não Reconhecida MQ
Informe o URI que representa a fila para a qual as mensagens de resposta não reconhecidas devem ser enviadas. Observe que essa definição só é ativada quando a caixa de seleção Gerar Automaticamente Valor de Correlação está marcada.
Se você não especificar um valor para este campo, mensagens de resposta não reconhecidas serão deletadas.
Processar Cabeçalhos RFH2
Selecione esta opção para fazer parse de cabeçalhos RFH2 do WebSphere MQ de um payload de mensagem e gerar automaticamente um cabeçalho de transporte RFH2Headers contendo os dados RFH2.
Se você não selecionar esta opção, o payload será transmitido conforme tiver sido recebido.

Para obter informações sobre como o transporte MQ lida com cabeçalhos RFH2, consulte "About RFH2 Headers" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
