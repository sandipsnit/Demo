Página Configuração do Transporte SOA-DIRECT

A Tabela 18-17 descreve as opções de configuração específicas ao transporte para o transporte SOA-DIRECT.

Para obter mais informações sobre o transporte SOA-DIRECT, consulte "SOA-DIRECT Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV909.

Tabela 18-17 - Criar/Editar um Serviço de Negócios - Configuração do Transporte SOA-DIRECT
Propriedade
Descrição
Conta de Serviço de JNDI
Opcional. Especifica as credenciais de segurança para a consulta de JNDI do serviço SOA de destino. A conta de serviço deve ser estática. Clique em Procurar e selecione uma conta de serviço. Se você não especificar uma conta de serviço, será usado um assunto anônimo.
Atribuição
Obrigatório. Identifica o padrão de comunicação usado pelo serviço. Selecione uma das seguintes opções:
  • Cliente Síncrono (default) - Nesta atribuição, como não há callback assíncrono, a opção Proxy de Callback Proxy está desativada. A opção Endereçamento WS também está desativada.
  • Cliente Assíncrono - Nesta atribuição, como callback assíncrono geralmente é usado, é possível identificar um Proxy de Callback, e você deve selecionar uma Versão de Endereçamento WS. A opção assíncrona só está ativada quando o serviço WSDL é do tipo SOAP.
  • Callback de Serviço - Esta atribuição serve para retorno do callback assíncrono para um serviço SOA após uma chamada de serviço externa.

    • Balanceamento de carga ou failover não estão disponíveis para serviços na atribuição Callback de Serviço.

Proxy de Callback
Opcional. Ativado apenas para a atribuição de Cliente Assíncrono.
Esta opção permite especificar o serviço de proxy que recebe callbacks. Quando você seleciona um proxy de callback, se nenhum Endereçamento WS for fornecido pelo pipeline de solicitação ou do serviço de proxy, o Oracle Service Bus preenche automaticamente a propriedade ReplyTo no cabeçalho SOAP. Você deve selecionar um serviço de proxy WSDL que use o transporte SB (para RMI) e o serviço de proxy de callback deve entender o Endereçamento WS.
Propriedades do Endereçamento WS que são enviadas na solicitação ou definidas no pipeline do serviço de proxy são usadas no lugar do proxy de callback definido nesta opção.
Se não especificar um Proxy de Callback, e a solicitação não contiver propriedades ReplyTo, você deverá fornecer propriedades ReplyTo no cabeçalho SOAP por meio do pipeline do serviço de proxy.
Proxy com Falha
Esta opção não é suportada no momento. É necessário configurar seus serviços de callback para tratar falhas em um padrão assíncrono.
Versão do Endereçamento WS
Obrigatório. Ativado apenas para a atribuição de Cliente Assíncrono.
Especifique a versão do Endereçamento WS default a ser usada quando nenhum use Endereçamento WS for fornecido no pipeline de solicitação ou do serviço de proxy. Propriedades do Endereçamento WS que são enviadas na solicitação ou definidas no pipeline do serviço de proxy são usadas no lugar da versão do Endereçamento WS definido nesta opção.
Para incompatibilidades de versão do Endereçamento WS entre ambientes, execute as transformações necessárias no pipeline do serviço de proxy. Para obter mais informações, consulte "Transformation Examples" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1827.
Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir.
Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Passar Assunto do Chamador
Opcional. Selecione esta opção para que o Oracle Service Bus transmita o assunto autenticado do serviço de proxy ao chamar o serviço SOA. A opção Conta do Serviço de Chamada, uma alternativa para Passar Assunto do Chamador, é desativada quando você seleciona esta opção.
Observação: Certifique-se de que a confiança do domínio esteja ativada entre o servidor cliente e o servidor de destino caso estejam em domínios diferentes. Para obter mais informações, consulte "Important Information Regarding Cross-Domain Security Support" no Oracle Fusion Middleware Securing Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=SECMG403.
Conta do Serviço de Chamada
Opcional. Alternativa para o Passar Assunto do Chamador, que permite especificar credenciais de segurança personalizadas selecionando uma conta de serviço para chamada RMI. É possível especificar qualquer tipo de conta de serviço (Passagem, Estática, Mapeamento).
Clique em Procurar e selecione uma conta de serviço. Se você não especificar uma conta de serviço, será usado um assunto anônimo.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Negócios - Resumo.
