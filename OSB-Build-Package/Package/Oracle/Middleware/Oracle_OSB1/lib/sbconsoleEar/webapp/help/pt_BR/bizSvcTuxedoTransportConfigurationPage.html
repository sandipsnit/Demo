Página Configuração do Transporte de Tuxedo

A página Criar/Editar um Serviço de Negócios - Configuração do Transporte de Tuxedo pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Para obter mais informações, consulte "Tuxedo Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Use esta página para configurar as definições de transporte para um serviço de negócios usando o protocolo de transporte de Tuxedo. Tabela 18-18 descreve como usar a página.

Tabela 18-18 - Página Criar/Editar um Serviço de Negócios - Configuração do Transporte de Tuxedo (Continuação)
Opção
Para criar ou editar...
Classes da Tabela de Campo
Informe o nome da classe ou classes que descrevem o buffer FML/FML32 recebido. São usadas nas rotinas de conversão FML/FML32 para XML para mapear os nomes de campo para nomes de elemento. Essa é uma lista separada por espaços de nomes de classe totalmente qualificados.
Exibindo Classes
Informe o nome da classe ou classes que descrevem o buffer VIEW/VIEW32 recebido ou enviado. São usadas nas rotinas de conversão VIEW para XML ou VIEW32 para XML para mapear os nomes de campo para nomes de elemento. Essa é uma lista separada por espaços de nomes de classe totalmente qualificados.
Jar de Classes
Selecione um recurso JAR que contenha um arquivo JAR com as classes FML/FML32 ou VIEW/VIEW32 necessárias para esta operação do ponto final.
Ponto(s) de Acesso Remoto
Selecione um ponto de acesso remoto na lista drop-down associado à Importação. A lista contém pontos de acesso remoto configurados no WTC. Um serviço de negócios não pode ser criado se não houver um ponto de acesso remoto associado.
Se nenhum ponto de acesso remoto existir ou para criar um novo, selecione Novo. Informe o Nome do Ponto de Acesso Local e o Endereço de Rede correspondentes nos campos adjacentes. Na validação do ponto final, o ponto de acesso é adicionado à configuração do WTC para cada servidor WTC. Se nenhum servidor WTC existir, um será criado.
Se mais de um URI for especificado, haverá um campo do ponto de acesso remoto por URI e o URI será exibido para fins informativos. Se existir mais de um URI, cada um exigirá um ponto de acesso remoto diferente. Se o URI especificado já corresponder um recurso do WTC existente, o ponto de acesso de remoto correspondente será exibido, mas não poderá ser modificado.
Ponto(s) de Acesso Local
Este campo só aparece quando você seleciona Novo no campo Ponto de Acesso Remoto.
Na lista drop-down, selecione um ponto de acesso local a ser associado ao ponto de acesso remoto recém-criado. Se nenhum existir ou para criar um novo, selecione Novo. Informe o Nome do Ponto de Acesso Local e o Endereço de Rede Local correspondentes nos campos adjacentes.
Tipo de Buffer da Solicitação
Selecione o tipo de buffer que o serviço remoto de Tuxedo receberá.
Subtipo de Buffer da Solicitação
Esta opção está ativada se o valor anterior de Tipo de Buffer da Solicitação for VIEW ou VIEW32. Informe o subtipo do buffer ao qual associar o buffer da solicitação.
Resposta Obrigatória?
Marque esta caixa de seleção para indicar uma chamada bidirecional. Se não estiver marcada, o tpcall subjacente será chamado com o flag TPNOREPLY e uma resposta nula será postada assincronamente.
Suspender Transação?
Marque esta caixa de seleção para suspender a transação, se existir. Isso é útil quando o serviço remoto não suporta transações.
Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir.
O Gerenciador de Trabalho é usado para postar assincronamente uma resposta nula no caso de uma chamada unilateral. Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificação da Solicitação
Especifique uma codificação de conjunto de caracteres para solicitações em transportes do Tuxedo.
Codificação da Resposta
Especifique uma codificação de conjunto de caracteres para respostas em transportes do Tuxedo.
Timeout
Especifica a quantidade máxima de tempo (em segundos) que o run-time de transporte aguarda respostas; um valor inteiro que seja maior ou igual a 0. Se não for especificado ou for definido como zero (default), as respostas atingirão o timeout no BLOCKTIME, o número máximo de segundos que um ponto de acesso ao WTC local permite para uma chamada de bloqueio.
Estilo de Transformação
Selecione uma das seguintes opções:
  • Nenhum - (default) A ordem dos campos pode não ser respeitada.
  • Ordenado - Os campos são apresentados com todas as suas ocorrências na ordem correta.
  • Ordenado e Agrupado - Se os campos forem estruturados logicamente como registros, os campos serão ordenados por ocorrência e agrupados por registro.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Negócios - Resumo.
