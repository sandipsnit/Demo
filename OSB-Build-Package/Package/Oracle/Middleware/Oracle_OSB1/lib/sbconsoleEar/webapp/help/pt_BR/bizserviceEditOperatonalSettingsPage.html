Configurando Definições Operacionais para Serviços de Negócios

Use a página Definições Operacionais para ativar e desativar um serviço de negócios e monitoramento para serviços de negócios específicos. Da mesma forma, você pode especificar essas mesmas definições operacionais específicas ao serviço na página Operações > Pesquisa Inteligente, com as seguintes exceções: não é possível definir um intervalo de agregação ou especificar um nível de severidade de alerta. Para obter mais informações, consulte Localizando e Atualizando Definições Operacionais.

Os efeitos do run-time das definições no nível do serviço dependem das definições globais correspondentes. Você deve ativar as definições globais e no nível do serviço para que um serviço seja completamente ativado no run-time. Além disso, o Estado do Serviço também deve ser ativado. Consulte Ativando Definições Globais.

É possível alterar e salvar as definições de configuração do monitoramento mesmo que o serviço não seja ativado no run-time. Por exemplo, é possível alterar e salvar o Intervalo de Agregação mesmo que o Monitoramento do Serviço esteja desativado. Dessa maneira, você pode editar as definições e ativá-las posteriormente.

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione Resource Browser > Serviços de Negócios.
  3. Clique no nome do serviço de negócios.

    4. A página Exibir um Serviço de Negócios exibe as informações mostradas na Tabela 25-8.

    Permitir que URIs de ponto final do serviço de negócios sejam colocados off-line
    Definir um intervalo de repetição para URIs off-line
    Marque a caixa de seleção Ativar com Intervalo de Repetição.

    Ativar Monitoramento
    Marque a caixa de seleção Ativado. Quando você ativa a sessão, o sistema começa a coletar estatísticas de monitoramento do serviço.
    Definir um Intervalo de Agregação para um serviço
    Selecione o intervalo em termos de horas ou minutos, depois clique em Atualizar. Se sua seleção para horas exceder 1, então a seleção default para minutos será sempre zero. No entanto, se sua seleção para horas for 0 ou 1, então você poderá configurar intervalos em termos de minutos. Consulte Definindo o Intervalo de Agregação para um Serviço.
    É possível alterar e salvar a definição do Intervalo de Agregação independentemente se o monitoramento está ou não ativado.
    Para ativar o monitoramento no run-time, certifique-se de que a caixa de seleção Ativar Monitoramento esteja marcada.
    Desativar Alertas de SLA
    Desmarque a caixa de seleção Ativado. O sistema para de avaliar as regras de alerta configuradas para o serviço; no entanto, você não recebe mais alertas de SLA associados ao serviço.
    Ativar Alertas de SLA
    Marque a caixa de seleção Ativado. Na lista drop-down Ativar Alerta no nível, selecione para restringir os alertas de SLA ao nível especificado ou acima (listado do nível mais inclusivo para o nível mais restritivo): Normal (default), Advertência, Menor, Maior, Crítico e Fatal.
    O alerta de SLA de serviço depende dos estados de monitoramento global e de serviço; ambos devem ser ativados para que o alerta de SLA seja ativado no run-time. Consulte Ativando Alertas de SLA Globalmente.
    Restringir o fluxo de mensagens enviadas para o conjunto de pontos finais configurados no serviço de negócios
    Marque a caixa de seleção Ativado ao lado de Estado de Throttling. Quando a opção está ativada, as mensagens são processadas por prioridade. Como opção, é possível designar uma prioridade para as mensagens usando opções de roteamento; caso contrário, as mensagens serão desenfileiradas em uma base primeiro a entrar, primeiro a sair.
    Para obter mais informações, consulte "Throttling" no Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG614.
    Restringir o número de mensagens sendo despachadas para um conjunto de pontos finais configurados no serviço de negócios
    Se Estado de Throttling estiver ativado, este campo será obrigatório.
    Especifique um valor de Concorrência Máxima; deve ser um valor inteiro positivo, não pode ser zero.
    Se excedido, as mensagens de entrada serão colocadas em uma fila (throttling) de espera, se uma existir. Se não, as mensagens serão descartadas.
    Se a fila estiver cheia, as mensagens na fila com uma prioridade mais baixa serão removidas e um erro de run-time será emitido. Se um handler de erro estiver configurado, ele será acionado. As novas mensagens de entrada serão enfileiradas.
    Especifica se as mensagens enviadas para serviços de negócios que excederam o limite de concorrência da mensagem devem ser mantidas em uma fila de espera
    Definir o número máximo de mensagens que são mantidas na fila de espera
    Se Estado de Throttling estiver ativado, este campo será obrigatório.
    Especifique um valor para Fila de Throttling; deve ser um valor inteiro positivo ou mantenha o valor default, zero, significando que não há fila de throttling. Quando o valor da fila de throttling for zero (nenhuma fila de throttling) e o número de mensagens exceder o valor de Concorrência Máxima, as mensagens serão descartadas e um erro será emitido. Se um handler de erro estiver configurado, ele será acionado.
    Se você desativar o throttling no run-time, todas as mensagens na fila de throttling serão processadas.
    Quando você deleta ou renomeia um serviço de negócios, todas as mensagens na fila de throttling são descartadas.
    Especifique a duração máxima de tempo que as mensagens serão mantidas em uma fila de espera
    Se Estado de Throttling estiver ativado, este campo será opcional.
    Especifique um valor de Expiração da Mensagem em milissegundos; deve ser um valor inteiro positivo. Um valor zero significa que a mensagem nunca expirará.
    Esse valor é definido na fila; todas as mensagens colocadas na fila possuem o mesmo valor de Expiração da Mensagem. Quando excedidas, as mensagens são removidas da fila sem serem processadas.
    Ativar rastreamento da mensagem para um serviço de negócios

    Marque a caixa de seleção Ativado ao lado de Rastreamento da Mensagem. Depois que você ativa o rastreamento da mensagem, o sistema registra as mensagens trocadas entre o pipeline e o serviço de negócios do Oracle Service Bus (mensagens de solicitação e de resposta de saída).

    Observação: Para ver o rastreamento no arquivo de log ou na saída padrão (console do servidor), o log do Oracle WebLogic Server deve ser definido com os seguintes níveis de severidade:

    Severidade mínima para registro: Info
    Arquivo de log: Info
    Saída padrão: Info

    Para obter informações sobre a definição de níveis de severidade do log, consulte "Using Log Severity Levels" em Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

    Observe que, quando aplicável, as mensagens de saída registradas também podem incluir o número de repetições, o código do erro e a mensagem de erro.
    Na lista drop-down Nível de Detalhe, escolha para especificar o nível de detalhe entre as opções a seguir:
    • Conciso - Exibe a data, a hora, o tipo de serviço, o nome do serviço e o URI
    • Cabeçalhos - Exibem informações concisas junto com a representação XML dos metadados da mensagem
    • Completo - Exibe as informações dos cabeçalhos junto com o payload bruto, incluindo anexos, se houver
    Se você escolher Completo na lista drop-down Nível de Detalhe, especifique o tamanho máximo (em kilobytes) para o payload da mensagem usando o campo Limite de Rastreamento de Payload. Além disso, especifique a codificação default para log do payload usando o campo Codificação Default. Isso pode ser útil ao registrar payloads binários ou mensagens SOAP com anexos binários.
    O valor de codificação default pode ser Base64 ou qualquer codificação suportada por Java, conforme definido em: http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html.
    Deixar o campo Codificação Default vazio faz com que o Oracle Service Bus use a codificação default do host para o payload. A codificação default depende de uma combinação da JVM, do sistema operacional (SO) subjacente e das definições de configuração regional no nível do SO.
    Se a definição especificada no campo Codificação Default não puder ser usada (por exemplo, não for uma opção válida para a configuração), então o Oracle Service Bus usará a codificação Base64 para o payload.
    Desativar rastreamento da mensagem para um serviço de negócios
    Desmarque a caixa de seleção Ativado ao lado de Rastreamento da Mensagem.
    Ativando e Desativando o Armazenamento dos Resultados no Cache
    A seleção de Ativado ativa o armazenamento no cache dos resultados do serviço de negócios. Você também deve garantir que o armazenamento no cache dos resultados esteja ativado globalmente. Consulte Ativando Caches de Resultados Globalmente.
    Quando você desativa o armazenamento no cache do resultado para um serviço de negócios desmarcando a opção Ativado e clicando em Atualizar, todos os resultados armazenados no cache para esse serviços de negócios serão descarregados.
