Página Políticas do Serviço de Negócios

A página Exibir um Serviço de Negócios - Configuração de Políticas pertence a uma série de páginas para edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e na página Exibir um Serviço de Negócios - Detalhes da Configuração.

Use esta página para configurar definições de política para um serviço de negócios com um dos seguintes tipos de serviço: Web Service XML baseado em WSDL, Serviço de Mensagem, Qualquer Serviço SOAP ou Qualquer Serviço XML. A Tabela 18-23 descreve como usar a página. Para que políticas de OWSM sejam utilizadas com as opções Web Service XML Baseado em WSDL, Serviço de Mensagem ou Qualquer Serviço Proxy de Serviço XML, o protocolo deverá ser definido como HTTP.

Observe que, para serviços baseados no WSDL, as políticas do WLS 9.2 vinculadas ao serviço são exportas (alinhadas) no WSDL efetivo. Políticas abstratas são pré-processadas antes de serem alinhadas. As políticas do OWSM são vinculadas por referência, e não alinhadas no WSDL efetivo.

Configuração da Política de Serviço
A seguir estão as opções para o campo Configuração da Política de Serviço:

Políticas OWSM

  • Da Área de Armazenamento de Política do OWSM - As políticas são definidas na Área de Armazenamento de Política central do Oracle Web Services Manager gerenciado pelo Oracle Enterprise Manager. Elas suportam WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken Profile e outros padrões da indústria.

    Com essa opção, Você pode adicionar políticas a partir de uma lista predefinida. Clique em Adicionar nas Políticas do Nível de Serviço para exibir a lista. Use filtro, se necessário, para localizar as políticas desejadas. Em serviços de negócios, só são exibidas políticas do lado do cliente.

    Se um WSDL usado para criar um serviço de negócios contiver políticas padrão de Segurança de Web Services incorporadas, o Oracle Service Bus gerará um erro e exibirá um conflito. Para solucioná-lo, substitua as políticas WSDL incorporadas por políticas Oracle Web Services Manager compatíveis selecionando Da Área de Armazenamento de Política do OWSM e clicando em Adicionar Compatível. O Oracle Service Bus tenta localizar da melhor forma a política correspondente mais próxima na área de armazenamento de políticas Oracle Web Services Manager, com base na política padrão incorporada no WSDL. O algoritmo pode apresentar nenhuma, uma ou várias políticas correspondentes.

    Se depois de clicar em Adicionar Compatível, o Oracle Service Bus não localizar políticas compatíveis, clique em Adicionar e selecione as políticas Oracle Web Services Manager apropriadas. Se o Oracle Service Bus apresentar mais de uma política compatível, delete aquelas que não desejar usar.

    Observação: Essa opção só é ativada somente se você tiver configurado seu domínio para usar o Oracle Web Services Manager. Para obter mais informações, consulte "Securing Oracle Service Bus with Oracle Web Services Manager" em Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Políticas do WLS 9

  • Do WSDL - Selecione esta opção se a política de serviço for associada ao WSDL no qual o serviço se baseia. Essas políticas suportam WS-Security 1.0, SAML 1.1 e outros padrões da indústria.

    Com essa opção, você pode exibir (somente leitura) as políticas de solicitação e resposta do WSDL.

    Observação: Se você receber um erro na configuração do serviço de negócios sobre Políticas de Segurança de Web Services não suportadas pelo Oracle Service Bus, use a opção Da Área de Armazenamento de Política do OWSM para substituir as políticas WSDL incorporadas por outras compatíveis do OWSM.

  • Na Política Pré-definida ou no Recurso de Políticas de Web Services Selecione essa opção para adicionar políticas de nível de serviço, de nível de operação (em cujo caso a política se aplica às mensagem de solicitação e de resposta), de solicitação e de resposta da Console do Oracle Service Bus Administration.

    As políticas são predefinidas no Oracle WebLogic Server ou definidas pelo usuário no Oracle Service Bus com um recurso de Política de Web Services. Essas políticas suportam WS-Security 1.0, SAML 1.1 e outros padrões da indústria.

    Com esta opção você pode adicionar políticas nos níveis de serviço, operação, solicitação e resposta. Clique em Adicionar na seção apropriada para selecionar políticas de uma lista.

Observação: Os modelos de binding de política são mutuamente exclusivos. Você deve usar apenas um tipo de política em um serviço. Se você efetuar bind de políticas diretamente para o serviço, todas as políticas baseadas no WSDL serão ignoradas.

Depois de adicionar as políticas, você pode proporcionar substituições na página Segurança.

 

Depois de finalizar

Clique em Atualizar para salvar esta configuração, ou clique em Redefinir para desfazer suas alterações.
