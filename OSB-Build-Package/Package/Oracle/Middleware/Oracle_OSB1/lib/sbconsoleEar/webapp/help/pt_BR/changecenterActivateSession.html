Ativando Sessões

  1. Clique em Ativar no Centro de Alterações. A página Ativar Sessão mostra o nome da sessão, o nome de usuário e um campo de descrição.

    2. A criação de uma sessão e seu descarte prosseguem independente de outra atividade no sistema. No entanto, se outra sessão estiver no processo de ser ativada, a página Ativar Sessão exibirá um erro indicando que o usuário possui as alterações do WebLogic Server pendentes. Para obter informações sobre a solução de conflitos entre alterações feitas na sessão e outras sessões ativadas, consulte Exibindo e Resolvendo Conflitos.

  2. Se ocorrerem erros de validação, será exibida uma mensagem de erro, indicando que a sessão apresenta erros de validação. Exiba e corrija os conflitos de configuração antes de prosseguir. Para obter informações sobre a solução de conflitos, consulte Exibindo e Resolvendo Conflitos.

    3. Você não poderá ativar a sessão até que tenha exibido todos os conflitos. Se novos conflitos surgirem enquanto você exibir os conflitos existentes, antes de ativar, será exibida uma mensagem informando sobre novos conflitos.

  3. Informe uma descrição no campo Descrição. Essa descrição é exibida na coluna Descrição quando você clica em Exibir Alterações no Centro de Alterações para exibir as alterações de configuração causadas por ativações de sessão.
  4. Para ativar a sessão, clique em Submeter. Se não ocorrer nenhum conflito novo nesse meio tempo, a sessão será encerrada e a configuração será implantada no run-time.
Observação: Quando você tentar ativar uma sessão com um URI do ponto final JMS em outro servidor (um único servidor diferente do servidor no qual você está trabalhando, ou um Servidor Gerenciado em um cluster), certifique-se de que o servidor de destino esteja disponível.
Observação: O Oracle Service Bus não permite o registro de serviços de proxy com transporte JMS se o URL do ponto final JMS especificar um destino que seja inatingível. Em outras palavras, para serviços JMS, o Oracle Service Bus verifica se o factory de conexão especificado existe; se não existir, ocorrerá um erro de ativação de sessão.
