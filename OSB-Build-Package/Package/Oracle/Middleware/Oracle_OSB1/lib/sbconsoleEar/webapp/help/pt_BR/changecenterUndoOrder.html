Ordem de Tarefas Desfazer

É possível desfazer tarefas em qualquer ordem (desde que ações de reversão individuais resultem em dados válidos). A operação de reversão define o valor de um recurso como o valor que tinha antes da alteração feita nesse recurso.

Caso a tarefa que está sendo desfeita seja uma que criou um objeto, não haverá estado anterior para o qual um objeto poderá ser retornado; em outras palavras, nenhum objeto existia antes da execução desta tarefa. Efetivamente, a operação de reversão deleta o novo objeto da sessão. Nesse caso, ocorrerão erros para os objetos que referenciam o que está sendo deletado. É possível exibir tais erros na página Exibir Conflitos no Centro de Alterações.

Quando você não está trabalhando em uma sessão, pode acessar a página Exibir Alterações de Configuração para ver as sessões que foram ativadas anteriormente. É possível desfazer essas sessões. O sistema não permite desfazer uma sessão que foi ativada anteriormente se um erro na configuração do run-time tiver resultado de uma ação de reversão. Por exemplo, se você tentar desfazer uma ativação de sessão que resulte na remoção de um objeto que está sendo referenciado por outro objeto, essa ação de reversão não será permitida.

É possível reverter uma ação de reversão. Na coluna Opções de Desfazer [tarefa], clique no ícone Desfazer. O Oracle Service Bus suporta ações de reversão ilimitadas. Isso significa que você pode desfazer a reversão que desfez uma reversão anterior, etc.
