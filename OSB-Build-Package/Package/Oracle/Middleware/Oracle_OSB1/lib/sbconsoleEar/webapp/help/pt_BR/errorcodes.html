Códigos de Erro

Esta seção descreve os erros do Oracle Service Bus que podem ocorrer quando um serviço de proxy está sendo executado. Os códigos de erro associados a esses erros aparecem dentro do elemento da variável de contexto de falha. É possível acessar o valor usando a seguinte instrução XQuery:

$fault/ctx:errorCode/text()

Erros são acompanhados por detalhes específicos ao erro dentro do elemento fault.

A Tabela A-1 lista os códigos de erro do Oracle Service Bus.

Códigos de erro de segurança (386000...386999)
BEA-386000
Erro geral de segurança. Os possíveis motivos incluem o seguinte:
  • A autenticação de token personalizado ou de nome de usuário/senha personalizado está configurada, mas a autenticação ocorreu via Segurança de Web Services.
 
BEA-386100
Falha na autenticação. Os possíveis motivos incluem o seguinte:
  • O XPath está configurado para atuar no corpo ou no cabeçalho, mas não existe tal parte da mensagem (a camada de binding retorna nulo).
  • O mecanismo XPath emite uma exceção ao avaliar a expressão XPath.
  • O resultado do XPath do nome de usuário ou da senha é um nó que não é do tipo TEXT nem ATTR.
  • O XPath retorna mais de um nó.
  • Um nome de usuário/senha foi recuperado, mas a autenticação falhou.
  • Um token personalizado foi recuperado, mas a asserção de identidade falhou (o serviço de asserção de identidade CSS emite LoginException ou qualquer outra exceção).
 
BEA-386101
Token de autenticação ausente. Os possíveis motivos incluem o seguinte:
  • O XPath do nome de usuário ou o XPath da senha retorna um resultado ou uma string vazia.
  • O XPath de token personalizado retorna um resultado vazio, valor de atributo nulo ou um valor de texto nulo.
 
BEA-386102
Autorização ' mensagem negada. Os possíveis motivos incluem o seguinte:
  • A política de controle de acesso de mensagem nega acesso ao serviço de proxy.
 
BEA-386103
O algoritmo de seleção da operação do serviço de proxy não pode determinar o nome da operação da solicitação ou retorna uma operação inválida (uma que não está no WSDL ou que é nula). Os possíveis motivos incluem o seguinte:
  • Um erro ocorreu durante o cálculo da operação.
  • O algoritmo de seleção da operação retorna nulo.
  • O algoritmo de seleção da operação retorna uma operação que não faz parte das operações declaradas pelo WSDL.

Esquemas de Detalhes do Erro

A seguir, há definições de esquema para snippets XML que acompanham os códigos de erro listados na Tabela A-1. Esses snippets XML aparecem na variável de contexto de falha no elemento <ctx:details>. Nem todos esses esquemas vêm de um único namespace. Além disso, embora um determinado código de erro geralmente implique um snippet XML específico, alguns snippets XML são usados por diversos códigos de erro. 

<xs:schema targetNamespace="http://www.oracle.com/wli/sb/stages/transform/config"
           xmlns:tns="http://www.oracle.com/wli/sb/stages/transform/config"
	   xmlns:xs="http://www.w3.org/2001/XMLSchema"
           xmlns:ref="http://www.oracle.com/wli/sb/reference"
           xmlns:sdk="http://www.oracle.com/wli/sb/stages/config"
	   elementFormDefault="qualified"
	   attributeFormDefault="unqualified">

Uma Resposta de Falha SOAP Foi Recebida 

    <xs:element name="ReceivedFaultDetail" type="tns:ReceivedFaultDetail"/>
    <xs:complexType name="ReceivedFaultDetail">
        <xs:sequence>
            <xs:element name="faultcode" type="xs:QName"/>
            <xs:element name="faultstring" type="xs:string"/>
            <xs:element name="detail" minOccurs="0" >
                <xs:complexType mixed="true" >
                    <xs:sequence>
                        <xs:any namespace="##any" minOccurs="0" maxOccurs="unbounded" processContents="lax" />
                    </xs:sequence>
                    <xs:anyAttribute namespace="##any" processContents="lax" />
                </xs:complexType>
            </xs:element>
            <!-- HTTP-specific status/response code -->
            <xs:element name="http-response-code" type="xs:int" minOccurs="0"/>
        </xs:sequence>
    </xs:complexType>

Uma Resposta Não Reconhecida Foi Recebida 

    <xs:element name="UnrecognizedResponseDetail" type="tns:UnrecognizedResponseDetail"/>
    <xs:complexType name="UnrecognizedResponseDetail">
        <xs:sequence>
            <xs:element name="detail" minOccurs="0" type="xs:string" />
            <!-- HTTP-specific status/response code -->
            <xs:element name="http-response-code" type="xs:int" minOccurs="0"/>
        </xs:sequence>
    </xs:complexType>

Uma Resposta de Erro Desconhecido Foi Recebida 

    <xs:element name="ErrorResponseDetail" type="tns:ErrorResponseDetail"/>
    <xs:complexType name="ErrorResponseDetail">
        <xs:sequence>
            <xs:element name="detail" minOccurs="0" type="xs:string" />
            <!-- HTTP-specific status/response code -->
            <xs:element name="http-response-code" type="xs:int" minOccurs="0"/>
        </xs:sequence>
    </xs:complexType>
