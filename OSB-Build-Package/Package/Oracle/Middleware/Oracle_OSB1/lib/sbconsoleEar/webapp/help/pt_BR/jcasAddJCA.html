Adicionando Bindings de JCA

A forma mais fácil de criar Bindings de JCA no Oracle Service Bus é importando em alto volume dos recursos de JCA criados (arquivo .jca, WSDL associado e arquivo de mapeamento TopLink ou EclipseLink). Em importação em alto volume, o Oracle Service Bus gera automaticamente um Binding de JCA entre arquivos .jca, recursos WSDL dos WSDLs de JCA associados e recursos de Documento XML de arquivos de mapeamento, e mantém as dependências entre os arquivos.

Para obter mais informações sobre importação, consulte Carregando Recursos de um Arquivo Zip, Carregando Recursos de um URL e Importando Recursos.

Ao importar seus recursos JCA, você pode gerar serviços de proxy ou de negócios com base neles, conforme descrito em Gerando um Serviço de Proxy a partir de um Recurso de Binding de JCA e em Gerando um Serviço de Negócios a partir de um Recurso de Binding de JCA.

Se você não importar em alto volume seus recursos de JCA, poderá criar Bindings de JCA manualmente, conforme descrito nas etapas a seguir:

  1. No Oracle JDeveloper, crie um WSDL de serviço JCA, um arquivo JCA (.jca) correspondente e, se necessário, outros recursos como arquivos de mapeamento TopLink ou EclipseLink e arquivos XSLT.

    2. Para obter mais informações, consulte o Oracle Fusion Middleware User's Guide for Technology Adapters em http://www.oracle.com/pls/as111170/lookup?id=TKADP.

  2. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão na Console do OSB Administration ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  3. Na Console do OSB Administration, crie ou importe os recursos JCA que você criou no Oracle JDeveloper, como os recursos WSDL, XML e XSLT.
  4. Crie o Binding de JCA.
    1. Selecione o Project Explorer, depois selecione o local do projeto ou pasta do Binding de JCA. A página View de Projeto/Pasta é mostrada.
    2. No campo Criar Recurso, selecione Binding de JCA para exibir a página Criar um Novo Recurso de Binding de JCA.
    3. No campo Arquivo JCA, clique em Procurar para selecionar o arquivo JCA.

      4. O campo Nome do Recurso é preenchido automaticamente com o nome do arquivo (menos a extensão do arquivo).

      Em vez de selecionar um arquivo, é possível copiar e colar o conteúdo de um arquivo JCA na área de texto do campo Arquivo JCA.

    4. No campo Nome do Recurso, informe ou altere o nome do JCA Binding. Esses nomes devem ser exclusivos.

      Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
    5. No campo Descrição do Recurso, informe uma descrição deste Binding de JCA.
    6. Clique em Salvar. O Binding de JCA será salvo na sessão atual.

      7. Uma advertência de validação é exibida se o Binding de JCA ainda não referenciar os recursos necessários, como um WSDL e um arquivo de mapeamento.

  5. Clique no nome do novo Binding de JCA para exibi-lo na página Exibir um Binding de JCA.
  6. Clique em Editar Referências.
  7. Na página Editar a Referência de um Recurso de Binding de JCA, Pesquise para selecionar os recursos de referência do Binding de JCA, como o WSDL, um arquivo de mapeamento TopLink/EclipseLink e um arquivo XSLT.
  8. Clique em Salvar.
  9. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.

Depois de adicionar recursos de Binding de JCA e configurar as dependências de recurso (associando um WSDL) - ou se você tiver importado recursos de JCA que mantenham suas dependências - você pode gerar serviços de proxy e de negócios a partir de Bindings de JCA. Para obter mais informações, consulte Gerando um Serviço de Proxy a partir de um Recurso de Binding de JCA e Gerando um Serviço de Negócios a partir de um Recurso de Binding de JCA.
