Definindo Condições da Regra de Alerta

Use esta página para definir condições para a regra de alerta. Você deve especificar pelo menos uma condição. Se você especificar várias condições, deverá usar os operadores E/Ou para combiná-los.

  1. No campo Selecionar Intervalo de Agregação para a Condição, selecione o número de horas e minutos para definir o intervalo de agregação para esta condição. O intervalo de agregação determina a frequência com a qual a condição é testada. A condição é testada toda vez que o subsistema de monitoramento agrega amostras de dados suficientes para constituir 1 intervalo de agregação.

    2. Por exemplo, se você selecionar um intervalo de agregação de 1 hora, a condição sempre testada toda vez que o valor de dados de uma hora estiver disponível. A primeira vez em que a condição é testada é no final da primeira hora. Depois disso, a condição é testada a cada 10 minutos, porque o intervalo de amostra para um intervalo de agregação de 1 hora é definido como 10 minutos.

  2. Comece definindo uma expressão simples. Duas ou mais expressões simples podem ser combinadas para formar uma expressão complexa. Para definir uma expressão simples complexa, execute o seguinte procedimento:
    1. No primeiro campo drop-down, selecione Contagem, Mínimo, Máximo, Médio ou Status.
    2. No próximo campo drop-down, selecione um operando.

      3. Dependendo se você seleciona Contagem, Mínimo, Máximo, Média ou Status no primeiro campo drop-down, a lista dos operandos varia. Por exemplo, se você selecionar Mínimo, Máximo ou Média, o operando Tempo de Resposta estará disponível. Use este operando para definir o tempo de resposta em milissegundos (mseg). Os operandos disponíveis também dependem da configuração do serviço em si. O número de operandos varia dependendo se um serviço possui pipelines, nós de rota, operações, etc.

      Para obter informações sobre como gerar alertas com base no status do URI do ponto final, consulte "Managing Endpoint URIs for Business Services" no Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG595.

      Quando você seleciona Contagem, esse campo exibe os operandos mostrados na Tabela 25-21.

      Tabela 25-21 - Detalhes do Operando de Contagem

      Para serviços de negócios que usam armazenamento no cache do resultado, a Contagem de Acertos do Cache é incrementada toda vez que o cache é usado para retornar uma resposta para um cliente.

      A contagem de erros é incrementada quando o processamento da mensagem retorna uma falha.

      O número total de mensagens processadas.
      Relação com Sucesso (%)
      A relação de mensagens processadas com sucesso para o número total de mensagens encontradas durante o intervalo de agregação especificado.
      Contagem de failover
      Apenas para serviços de negócios: a contagem de failovers.
      <Request Pipeline>.
      Contagem de Erros
      Apenas para serviços de proxy: O número de mensagens com erro processadas pelo pipeline da solicitação.
      <Request Pipeline>.
      Contagem de Mensagens
      Apenas para serviços de proxy: O número de mensagens processadas pelo pipeline da solicitação.
      Contagem de Erros de Validação

    3. No próximo campo, selecione o operador de comparação desejado: =, != > ou <.
    4. No próximo campo, informe o valor para comparação.
    5. Clique em Adicionar.
  3. Para definir uma expressão complexa:
    1. Repita as etapas a - e na etapa 2 para definir uma expressão simples.
    2. Repita as etapas a - e novamente para definir outra expressão simples.
    3. Marque a caixa de seleção para cada uma das duas expressões.
    4. Clique em E ou em Ou para combinar as expressões em uma expressão complexa.

      5. A opção Dividir está disponível depois que você combina várias expressões. Essa opção é usada para dividir expressões complexas de volta em expressões simples separadas.
