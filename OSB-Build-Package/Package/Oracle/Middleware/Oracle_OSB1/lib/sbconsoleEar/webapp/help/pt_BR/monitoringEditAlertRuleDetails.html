Configurando Informações Gerais para Regras de Alerta

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione Resource Browser > Serviços de Proxy ou Serviços de Negócios.
  3. Clique no nome do serviço.
  4. Selecione a guia Regras de Alerta de SLA e clique em Adicionar Novo.
  5. No campo Nome da Regra, informe um nome para a regra de alerta. Esse campo é obrigatório.
  6. No campo Resumo do Alerta, informe uma breve descrição da regra de alerta. Esse texto também é usado como a linha de assunto de uma mensagem de e-mail se esta regra de alerta for configurada com uma notificação de alerta de e-mail, e pode conter no máximo 80 caracteres. Se você não fornecer um resumo do alerta, o texto default, Alerta do Oracle Service Bus, será usado.
  7. No campo Descrição da Regra, informe uma descrição para a regra de alerta.
  8. No campo Destino de Alerta, informe o nome do recurso do destino de alerta ou clique em Procurar para exibir a página Selecionar Destino de Alerta. Selecione um destino de alerta na lista e clique em Submeter.

    9. Por default, o alerta irá sempre para a Console do Oracle Service Bus Administration. Para configurar e selecionar outros recursos do destino de alerta, consulte Adicionando Destinos de Alerta e Localizando Destinos de Alerta.

  9. No campo Hora de Início, informe uma hora de início no formato HH:MM. Por exemplo, informe 09:00 AM. Use os campos Hora de Início e Hora de Término para especificar um intervalo de tempo durante o qual a regra está ativa em cada dia antes da data de expiração. Por exemplo, entre 9 h e 17 h de todos os dias.
  10. No campo Hora de Término, informe uma hora de término no formato HH:MM. Por exemplo, informe 05:00 PM. Use os campos Hora de Início e Hora de Término para especificar um intervalo de tempo durante o qual a regra está ativa em cada dia antes da data de expiração. Por exemplo, entre 9 h e 17 h de todos os dias.

    11. A regra de alerta está ativa diariamente durante a hora de início especificada até a hora de término especificada, até que a regra expire.

  11. No campo Data de Expiração da Regra, informe uma data de expiração no formato MM/DD/AAAA. Por exemplo, informe 12/31/2010. A regra expira às 23 h 59 min da data especificada. Se você não especificar uma data, a regra nunca expirará.
  12. No campo Regra Ativada, mantenha Sim como o default para garantir que esta regra seja ativada ou selecione Não para desativar esta regra.
  13. No campo Severidade do Alerta, selecione um dos seguintes:
    • Normal
    • Advertência
    • Menor
    • Maior
    • Crítico
    • Fatal
  14. No campo Frequência do Alerta, selecione uma das seguintes definições, mostradas na Tabela 25-20.

    15. Tabela 25-20 - Definição da Frequência de Alerta
    Propriedade
    Descrição
    Sempre
    Se você selecionar esta opção, as ações incluídas na regra de alerta serão executadas toda vez que a regra de alerta for avaliada como verdadeira. Por exemplo, se definir a condição de que o tempo médio de resposta é superior a 300 milissegundos, você receberá um alerta toda vez que essa condição for avaliada como verdadeira.
    O número de vezes que uma regra de alerta é avaliada depende do intervalo de agregação e do intervalo de amostra associado a essa regra. Se o intervalo de agregação for definido como 5 minutos, o intervalo de amostra será de 1 minuto. As regras são avaliadas toda vez que 5 amostras de dados estão disponíveis. Portanto, a regra é avaliada pela primeira vez aproximadamente 5 minutos depois de criada e a cada minuto dali em diante.
    Notificar Uma Vez
    Se você selecionar esta opção, as ações incluídas na regra serão executadas da primeira vez que a regra for avaliada como verdadeira, e nenhum outro alerta será gerado até que a condição seja redefinida e avaliada como verdadeira novamente. Por exemplo, se definir a condição de que o tempo médio de resposta é inferior a 300 milissegundos, você receberá um alerta da primeira vez que essa condição for avaliada como verdadeira, mas não receberá nenhum outro alerta até que a condição seja avaliada como falso e depois como verdadeira novamente. O timestamp do alerta é atualizado e exibido no Painel de Controle.

  15. Selecione Sim para a opção Parar de Processar Mais Regras se quiser abortar a execução de regras adicionais depois que uma das regras associadas a um serviço é avaliada como verdadeira. Use esta opção para parar de avaliar regras subsequentes quando houver várias regras associadas a um serviço específico. Mantenha Não como o default para continuar processando regras.

    16. Esta opção comporta-se como a opção Parar de Processar Mais Regras no Assistente de Regras no Microsoft Outlook.

Depois de finalizar

Clique em Próximo para exibir a página Nova Regra de Alerta - Configuração de Condições. Continue em Definindo Condições da Regra de Alerta.
