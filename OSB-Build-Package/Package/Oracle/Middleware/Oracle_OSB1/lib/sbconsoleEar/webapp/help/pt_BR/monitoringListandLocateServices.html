Exibindo Informações sobre Monitoramento do Serviço

A página Condição do Serviço exibe métricas para serviços que possuem monitoramento ativado. Para saber como ativar o monitoramento para serviços, consulte Configurando Definições Operacionais para Serviços de Proxy e Configurando Definições Operacionais para Serviços de Negócios.

Por default, a taxa de atualização do Painel de Controle é Sem Atualização. Para alterar o intervalo da taxa de atualização do Painel de Controle, consulte Definindo Preferências do Usuário.

Cuidado: Selecionando-se um intervalo de taxa de atualização do painel de controle diferente do default (Sem Atualização), o browser atualiza o Painel de Controle em intervalos regulares. Isso impede que a sessão atinja o timeout, mesmo sem que você interaja com a Console do Oracle Service Bus Administration.
Cuidado: A Oracle recomenda que você use o recurso de taxa de atualização do Painel de Controle com cuidado. Você nunca deve deixar uma sessão sem assistência. Enquanto a sessão está ativa, outras pessoas podem obter acesso a ela em sua máquina sem nova autenticação.

No cabeçalho da guia Condição do Serviço, estes ícones podem estar presentes:

  • Um ícone de Advertência e indicador numérico.
    • Clicar no ícone filtra os serviços exibidos usando os critérios Possui Erros.
    • O número indica o número de serviços com erros desde que a página foi atualizada pela última vez.
  • Um ícone de Pontos Finais Off-line e um indicador numérico.
    • Clicar no ícone filtra os serviços exibidos usando os critérios Possui URIs de Ponto Final Off-line.
    • O número indica o número de serviços com pontos finais que ficaram off-line desde que a página foi atualizada pela última vez.

A página Condição do Serviço exibe métricas de serviço e informações de monitoramento, mostradas na Tabela 25-5.

  1. No campo Exibir Estatísticas, execute um dos seguintes procedimentos:
    • Para exibir estatísticas de monitoramento para cada serviço do período de agregação atual, selecione Intervalo de Agregação Atual. A view Intervalo de Agregação Atual mostra uma exibição estatística móvel das métricas de serviço.
    • Para exibir estatísticas de monitoramento para o período desde que você redefiniu as estatísticas para um serviço, selecione Desde a Última Redefinição. A view Desde a Última Redefinição mostra uma contagem atual das métricas.
  2. Selecione um Servidor na lista drop-down para exibir métricas para esse servidor.

    3. Se um cluster existir, métricas de todo o cluster serão exibidas por default. Selecione um Servidor Gerenciado individual para exibir métricas para esse servidor.

  3. Para restringir o número de itens na lista ou para localizar serviços específicos:
    • Clique no ícone Abrir para exibir filtros de pesquisa adicionais. Se você especificar vários critérios de pesquisa, apenas os resultados que correspondem a todos os critérios serão retornados.
    • Filtrar por nome do serviço. No campo Nome, informe o nome do destino de pesquisa ou informe caracteres curingas (use * e ? como caracteres curingas para executar uma pesquisa mais generalizada), em seguida, clique em Pesquisar.

      • Este método de pesquisa é preferencial se o nome do serviço for exclusivo entre todos os projetos e caminhos.

    • Filtrar por caminho do serviço. No campo Caminho, informe o caminho do destino de pesquisa, que é o nome do projeto e o nome da pasta em que os serviços residem. É possível usar * e ? como caracteres curingas para executar uma pesquisa mais generalizada. Todos os serviços que residam nesse caminho serão exibidos.

      • O formato do campo Caminho é o seguinte:
      project-name/root-folder/ . . ./parent-folder

      Se um serviço estiver diretamente abaixo do projeto, o formato será o seguinte: project-name

    • Filtrar por nome e caminho do serviço. Esse método de pesquisa é preferencial se houver mais de um serviço com o mesmo nome que resida em diferentes caminhos.
    • Filtrar por serviço pai. Na lista drop-down Serviço, selecione o tipo de serviço pai: Serviços de Proxy ou de Negócios, depois clique em Pesquisar. Selecione Todos os Serviços para ignorar os critérios do filtro do serviço pai.
    • Filtrar por serviços com mensagens. Selecione a opção Possui Mensagens, depois clique em Pesquisar.
    • Filtrar por serviços com alertas. Selecione a opção Possui Alertas, depois clique em Pesquisar.
    • Filtrar por serviços com erros. Selecione a opção Possui Erros, depois clique em Pesquisar.
    • Filtrar por serviços com URIs off-line. Selecione a opção Possui URIs de Ponto Final Off-line, depois clique em Pesquisar.
    • Filtrar por serviços chamados por um serviço de proxy específico. Informe o nome e o caminho de um serviço de proxy no campo fornecido ou clique em Procurar para exibir a página Selecionar Serviço de Proxy. Selecione o serviço de proxy na lista e clique em Submeter. Clique em Pesquisar. Os serviços de negócios chamados pelo serviço de proxy especificado serão exibidos.

      • Caracteres curingas não são permitidos no campo Chamado pelo Proxy.

    • Clique em Exibir Tudo para remover os filtros de pesquisa e exibir todos os serviços.

      • Tabela 25-5 - Informações sobre Monitoramento do Serviço
      Propriedade
      Descrição
      Nome
      O nome designado ao serviço. O nome é um link para a página Detalhes de Monitoramento do Serviço. Consulte Exibindo Métricas de Serviço.
      Caminho
      O projeto associado ao serviço. Se o serviço residir em uma pasta do projeto, essa pasta também será listada. O caminho é exibido no formato:
      project-name/root-folder/ . . ./parent-folder
      O caminho é um link para o caminho correspondente no Project Explorer.
      Tipo de Serviço
      O tipo de serviço pai: serviço de proxy ou serviço de negócios.
      Intervalo de Agregação
      Este campo só é exibido quando você seleciona Intervalo de Agregação Atual no campo Exibir Estatísticas.
      O intervalo de agregação atual definido para monitoramento deste serviço, em termos de horas e minutos. Defina este intervalo na página Definições Operacionais. Consulte Definindo o Intervalo de Agregação para um Serviço.
      Tempo Méd. de Resposta
      O tempo médio de resposta (em milissegundos) que este serviço gastou para executar mensagens dentro do período do intervalo de agregação atual ou durante o período desde a última redefinição.
      Mensagens
      O número de mensagens associadas a este serviço durante o período do intervalo de agregação atual ou durante o período desde a última redefinição.
      Erros
      O número de mensagens de erro associadas a este serviço durante o período do intervalo de agregação atual ou para o período desde a última redefinição.
      Alertas
      Este campo está oculto por default.
      O número de alertas (SLA e pipeline) associados a este serviço durante o período do intervalo de agregação atual ou durante o período desde a última redefinição.
      Em um Servidor Gerenciado individual em um cluster, para serviços de proxy, mostra o número de alertas do pipeline apenas associados a este serviço durante o período do intervalo de agregação atual ou durante o período desde a última redefinição.
      Para serviços de negócios, para Servidores Gerenciados individuais, exibe N/D.
      Alertas de SLA
      O número de alertas de SLA associados a este serviço durante o período do intervalo de agregação atual ou durante o período desde a última redefinição.
      Para um Servidor Gerenciado individual em um cluster, Alertas de SLA serão N/D (apenas alertas de SLA de todo o cluster são exibidos).
      Alertas do Pipeline
      Apenas para serviços de proxy:
      O número de alertas do pipeline associados a este serviço durante o período do intervalo de agregação atual ou durante o período desde a última redefinição.
      Status do URI do ponto final
      Apenas para serviços de negócios, o status de URIs de ponto final. O status é um link para a página URIs de Ponto Final. Consulte Exibindo Métricas de URIs de Ponto Final de Serviços de Negócios.
      Para um único domínio de nó (instâncias individuais do servidor):
      • On-line - todos os URIs de ponto final para o serviço de negócios estão on-line.
      • Off-line - todos os URIs de ponto final para o serviço de negócios estão off-line.
      • Parcial - Pelo menos um dos URIs de ponto final para o serviço de negócios está off-line.
      Para um domínio em cluster:
      • Quando o campo Servidor é definido como Cluster ou como um dos Servidores Gerenciados, o status On-line denota que todos os URIs de ponto final estão on-line no cluster ou no Servidor Gerenciado selecionado, respectivamente.
      • Quando o campo Servidor é definido como Cluster ou como um dos Servidores Gerenciados, o status Off-line denota que todos os URIs de ponto final estão off-line no cluster ou no Servidor Gerenciado selecionado, respectivamente.
      • Quando o campo Servidor é definido como Cluster, o status Parcial denota que pelo menos um dos URIs do ponto final para o serviço de negócios está off-line em pelo menos um dos servidores ou que um dos URIs do ponto final está off-line em todos os servidores, mas os outros URIs do ponto final para o mesmo serviço de negócios ainda estão disponíveis em um ou em todos os servidores.
      • Quando o campo Servidor é definido como um dos Servidores Gerenciados, o status Parcial denota que pelo menos um dos URIs de ponto final para o serviço de negócios está off-line no Servidor Gerenciado selecionado.
      Ação
      Este campo só é exibido quando você seleciona Desde a Última Redefinição no campo Exibir Estatísticas.
      Nesta coluna, você pode clicar no ícone Redefinir de um serviço específico para redefinir as estatísticas desse serviço. Quando você confirma que deseja fazer isso, o sistema deleta todas as estatísticas de monitoramento que foram coletadas para o serviço desde que você redefiniu as estatísticas pela última vez. No entanto, o sistema não deleta as estatísticas sendo coletadas durante o Intervalo de Agregação Atual para o serviço. Além disso, depois que você clica no ícone Redefinir, o sistema começa imediatamente a coletar estatísticas de monitoramento para o serviço de novo.
