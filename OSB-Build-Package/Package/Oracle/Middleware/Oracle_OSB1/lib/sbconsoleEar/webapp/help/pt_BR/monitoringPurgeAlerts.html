Expurgando Alertas

Use esta página para deletar permanentemente os alertas de SLA ou de pipeline especificados.

Observação: Esta ação não pode ser desfeita.
  1. Selecione Operações > Painel de Controle.
  2. Selecione a guia Alertas de SLA ou Alertas do Pipeline, depois clique no link Histórico de Alertas Estendidos.
  3. No lado direito superior da página Histórico de Alertas Estendidos, clique em Expurgar Histórico de Alertas de SLA ou em Expurgar Histórico de Alertas do Pipeline.
  4. Execute uma das etapas a seguir, mostradas na Tabela 25-3.

    5. Tabela 25-3 - Opções de Expurgação do Alerta
    Para...
    Execute Estas Etapas...
    • Expurgar todos os alertas de SLA
    • Expurgar todos os alertas do pipeline
    1. Selecione Expurgar Todos os Alertas.
    2. Clique em Expurgar.
    • Expurgar alertas de SLA dentro de um período especificado
    • Expurgar alertas do pipeline dentro de um período especificado
    1. Selecione Expurgar Alertas de... para...
    2. No campo Expurgar de, informe um mês, um dia, um ano e uma hora.
    3. No campo Expurgar para, informe um mês, um dia, um ano e uma hora.
    4. Clique em Expurgar.

Observação: Se o servidor no qual os alertas estão sendo expurgados estiver indisponível, a estrutura de log persistirá um registro e sondará o servidor a cada 30 minutos para verificar a disponibilidade. Quando a estrutura de log encontrar o servidor disponível, ela expurgará o(s) alerta(s).
