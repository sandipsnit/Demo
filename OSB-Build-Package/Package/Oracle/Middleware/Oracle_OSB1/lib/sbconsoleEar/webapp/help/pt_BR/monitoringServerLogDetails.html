Exibindo Detalhes de Arquivos de Log do Domínio

Use esta página para exibir detalhes de entradas do arquivo de log do domínio, mostrados na Tabela 25-18.

Tabela 25-18 - Detalhes de Log do Domínio
Propriedade
Descrição
Mensagem
Uma descrição do evento ou condição.
Data
Exibe a hora e data em que a mensagem foi originada, em um formato específico à configuração regional. A JVM (Java Virtual Machine) que executa cada instância do WebLogic Server refere-se ao sistema operacional do computador host para informações sobre o fuso horário e o formato locais.
Subsistema
Indica o subsistema do WebLogic Server que foi a origem da mensagem; por exemplo, container de EJB (Enterprise Java Beans) ou JMS (Java Messaging Service).
ID da Mensagem
Um identificador exclusivo de seis dígitos.
Todos os IDs de mensagens geradas pelo sistema WebLogic Server começam com BEA- e ficam dentro de um intervalo numérico de 0 a 499999.
Gravidade
Indica o grau de impacto ou seriedade do evento relatado pela mensagem:
  • Alerta - Um serviço específico está em um estado inútil enquanto outras partes do sistema continuam funcionando. A recuperação automática não é possível; atenção imediata do administrador é necessária para resolver o problema.
  • Crítico - Ocorreu um erro de sistema ou de serviço. O sistema pode se recuperar, mas pode ser uma perda momentânea ou degradação permanente do serviço.
  • Emergência - O servidor está em um estado inútil. Essa severidade indica uma falha severa ou pânico no sistema.
  • Erro - Ocorreu um erro do usuário. O sistema ou aplicação pode tratar o erro sem interrupção e degradação limitada do serviço.
  • Info - Usado para operações normais de relatórios; uma mensagem informativa no nível inferior.
  • Aviso - Uma mensagem informativa com um nível superior de importância.
  • Advertência - Uma operação ou configuração suspeita ocorreu, mas não pode afetar a operação normal.
Máquina
O nome DNS do computador que hospeda a instância do servidor.
Servidor
O nome da instância do WebLogic Server em que a mensagem foi gerada.
Thread
O ID que a JVM atribui ao thread em que a mensagem foi originada.
ID do Usuário
O ID do usuário em que o evento associado foi executado.
ID da Transação
Apresenta apenas as mensagens registradas no contexto de uma transação.
ID do Contexto
Informações do contexto para correlacionar mensagens vindas de uma solicitação ou aplicação específica.
Detalhe
Uma descrição do evento ou condição.
Causa
A causa da mensagem.
Ação
A ação que deve ser executada.
