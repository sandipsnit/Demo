Políticas de Web Services Personalizadas

Observação: Em releases futuras do Oracle Service Bus, as políticas do Oracle Web Services Manager substituirão as políticas do WLS 9.2. Embora esta versão do Oracle Service Bus continue oferecendo suporte para políticas do WLS 9.2, você deve considerar usar políticas do Oracle Web Services Manager para criação de novo serviço para se preparar para a eventual reprovação de suporte para políticas do WLS 9.2. Para obter mais informações, consulte "Securing Oracle Service Bus with Oracle Web Services Manager" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681.

A Estrutura da Política de Web Services (WS-Policy) é uma estrutura extensível baseada em XML que amplia a configuração de um Web Service com asserções de segurança específicas ao domínio e especifica os requisitos de segurança, as expectativas e os recursos do Web Service. No Oracle Service Bus, um dos principais usos da Política de Web Services é configurar a segurança de mensagem em serviços de proxy e em serviços de negócios.

Usando Políticas de Web Services do Oracle Service Bus

O Oracle Service Bus inclui três arquivos de Política de Web Services simples que podem ser usados para exigir que os clientes autorizem, criptografem digitalmente ou assinem digitalmente mensagens SOAP: Auth.xml, Encrypt.xml e Sign.xml. A Oracle recomenda que, a menos que haja necessidades de segurança específicas, você use esses arquivos pré-compactados o mais frequentemente possível.

Para obter mais informações sobre o uso destas políticas, consulte "Using WS-Policy in Oracle Service Bus Proxy and Business Services" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.

Usando Políticas de Web Services Personalizadas

Se as instruções da Política de Web Services do Oracle Service Bus não atenderem às suas necessidades de segurança, você poderá gravar suas próprias Políticas de Web Services (Políticas de Web Services personalizadas), importá-las para o Oracle Service Bus e consultá-las no WSDL. (As instruções da Política de Web Services do Oracle Service Bus são somente para leitura.)

Para obter informações sobre a criação e consulta a Políticas de Web Services personalizadas, consulte "Using WS-Policy in Oracle Service Bus Proxy and Business Services" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599.
