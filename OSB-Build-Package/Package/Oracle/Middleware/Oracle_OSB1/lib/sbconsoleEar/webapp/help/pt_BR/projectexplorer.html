Trabalhando com Projetos, Pastas e Recursos

O Project Explorer organiza configurações e recursos em projetos e pastas. No Project Explorer, é possível exibir a página View de Projetos, que lista todos os projetos no domínio e as páginas View de Projeto/Pasta, que listam as pastas e recursos de cada projeto.

Projetos são construções de agrupamento não-hierárquicos, não-sobrepostas e de nível superior.

Todos os recursos do Oracle Service Bus (por exemplo, serviços, Políticas de Web Services, WSDLs, transformações XQuery, etc.) residem em exatamente um projeto. Projetos não são sobrepostos. Recursos podem ser criados diretamente em um projeto ou podem ser melhor organizados em pastas. No entanto, é possível referenciar qualquer recurso, independente do projeto em que resida.

Quando você cria um domínio no Oracle Service Bus, um projeto default é criado.

Nomeando Projetos e Pastas

Os nomes de projetos e pastas são limitados a 64 caracteres e devem seguir as Restrições para Nomeação de Recursos.

Os nomes, comprimento e níveis de aninhamento de projetos e pastas são ultimamente afetados pelos limites do sistema operacional. A criação de pastas ou projetos com nomes muito longos ou pastas amplamente aninhadas pode falhar devido às limitações do sistema operacional.

Qualificando Nomes de Recursos Usando Projetos e Pastas

Projetos e pastas qualificam os nomes de recursos do Oracle Service Bus. Uma referência a um recurso é construída da seguinte maneira:

project-name/root-folder/.../parent-folder/resource-name

Se um recurso estiver localizado diretamente abaixo de um projeto, a referência será construída conforme a seguir:

project-name/resource-name
