Exibindo Recursos em um Web Browser

Exibir recursos do Oracle Service Bus em um Web browser padrão, usando os seguintes URLs

    • Caminho para recursos de WSDL:
      http://host:port/sbresource?WSDL/project_path/wsdlname
      ou
      http://host:port/proxy_service_endpoint_URI?WSDL (para serviços de proxy de HTTP baseados em WSDL)
    • Caminho para recursos do Serviço de proxy:
      http://host:port/sbresource?PROXY/project_path/proxyname
      http://host:port/sbresource?ORAPROXY/project_path/proxyname (para serviços de proxy com políticas do Oracle Web Services Manager anexadas)
    • Caminho para recursos do Serviço de Negócios:
      http://host:port/sbresource?BIZ/project_path/bizname
      http://host:port/sbresource?ORABIZ/project_path/bizname (para serviços de negócios com as políticas do Oracle Web Services Manager anexadas)
    • Caminho para recursos do WSS:
      http://host:port/sbresource?POLICY/project_path/policyname
    • Caminho para recursos do MFL:
      http://host:port/sbresource?MFL/project_path/mflname
    • Caminho para recursos do SCHEMA:
      http://host:port/sbresource?SCHEMA/project_path/schemaname

    Observação: se você usar caracteres especiais nos nomes do recurso, os URLs usados para expor os recursos no Oracle Service Bus deverão ser codificados em UTF-8 na ordem para escapar os caracteres especiais.

    Você também pode recuperar WSDLs contendo políticas do Oracle Web Services Manager de forma que as políticas estejam em conformidade com os padrões suportados de Política de WS e Política de segurança de WS. Para obter mais informações, consulte "Advertising WSDLs to Support WS Standards" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1877.
