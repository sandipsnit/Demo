Provedores de Chaves de Serviço

Um provedor de chaves de serviço contém credenciais de PKI (Public Key Infrastructure) que os serviços de proxy usam para decriptografar mensagens SOAP de entrada e para autenticação de saída e assinaturas digitais. Uma credencial de PKI é um par de chaves privadas com um certificado que pode ser usado para assinaturas digitais e criptografia (para Segurança do Web Service) e para autenticação SSL de saída. O certificado contém a chave pública que corresponde à chave privada.

Observação: Para usar um provedor de chaves de serviço, você deve configurar um provedor de mapeamento de credencial PKI. Consulte "Configuring the WebLogic Security Framework: Main Steps" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498.

Um único provedor de chaves de serviço pode conter todas as seguintes credenciais de PKI:

  • Um par de chaves para criptografia digital

    • Serviços de proxy usam este par de chaves para decriptografar mensagens SOAP de entrada que foram criptografadas de acordo com uma instrução de Política de Web Services. Se quiser que o provedor de chaves de serviço suporte criptografia digital, a área de armazenamento de chaves associada ao mapeador de credencial de PKI deverá conter pelo menos um certificado X.509 que suporte criptografia.

  • Um par de chaves para assinaturas digitais

    • Serviços de proxy usam este par de chaves quando seu ponto final é um Web Service e o Web Service exige que os clientes assinem uma ou mais partes de um envelope SOAP.

  • Um par de chaves para autenticação de cliente SSL (SSL bilateral)

    • Serviços de proxy usam este par de chaves para autenticação ao atuar como um cliente durante uma conexão TLS/SSL (Secure Sockets Layer) de saída; ou seja, ao encaminhar uma mensagem para um serviço de negócios ou um serviço de proxy HTTPS que requer autenticação de certificado cliente.

É possível usar o mesmo provedor de chaves de serviço para vários serviços de proxy.
