Adicionando Provedores de Chaves de Serviço

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione o Project Explorer, depois selecione um projeto ou pasta no qual adicionar o provedor de chaves de serviço. A página View de Projeto/Pasta é mostrada.
  3. Na lista drop-down Criar Recurso, selecione Provedor de Chaves de Serviço para exibir a página Criar um Novo Provedor de Chaves de Serviço.
  4. No campo Nome do Provedor da Chave de Serviço, informe o nome exclusivo do provedor da chave de serviço.

    Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
  5. No campo Descrição, informe uma descrição para o provedor de chaves de serviço.
  6. Execute uma das etapas a seguir, mostradas na Tabela 17-2.

    7. Tabela 17-2 - Opções de Autenticação
    Para Adicionar um Par de Chaves para...
    Execute Estas Etapas...
    Criptografia digital
    1. Ao lado de Chave de Criptografia, clique em Procurar.

      2. A janela Selecionar um alias para Chave de Criptografia exibe os apelidos de chave da área de armazenamento de chaves que o mapeador de credencial do realm está usando.

    2. Na janela Selecionar um alias para Chave de Criptografia, informe a senha usada para proteger o acesso à área de armazenamento de chaves. (Defina a senha ao criar a área de armazenamento de chaves.)
    3. Selecione um alias de chave que mapeie para um certificado X.509 e que suporte criptografia.
    4. Clique em Submeter.
    Quando você associa este provedor de chaves de serviço a um serviço de proxy, o Oracle Service Bus incorpora o certificado X.509 ao WSDL do serviço de proxy. O serviço de proxy então usa esse certificado para criptografar as mensagens enviadas para este ponto final. O serviço de proxy usa a chave privada na credencial de PKI para decriptografar as mensagens retornadas pelo ponto final.
    Assinaturas digitais
    1. Ao lado de Chave de Assinatura Digital, clique em Procurar.

      2. A janela Selecionar um alias para Chave de Assinatura Digital exibe os apelidos de chave da área de armazenamento de chaves que o mapeador de credencial do realm está usando.

    2. Na janela Selecionar um alias para Chave de Assinatura Digital, informe a senha usada para proteger o acesso à área de armazenamento de chaves. (Defina a senha ao criar a área de armazenamento de chaves.)
    3. Selecione um alias de chave.
    4. Clique em Submeter.
    Autenticação do cliente SSL (SSL bilateral)
    1. Ao lado de Chave de Autenticação de Cliente SSL, clique em Procurar.

      2. A janela Selecionar um alias para Chave de Autenticação do Cliente SSL exibe os apelidos de chave da área de armazenamento de chaves que o mapeador de credencial do realm está usando.

    2. Na janela Selecionar um alias para Chave de Autenticação de Cliente SSL, informe a senha usada para proteger o acesso à área de armazenamento de chaves. (Defina a senha ao criar a área de armazenamento de chaves.)
    3. Selecione um alias de chave.
    4. Clique em Submeter.

  7. Clique em Salvar. O provedor e chaves de serviço será salvo na sessão atual.
  8. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
