Adicionando Ações de Chamada Java

Use a ação de chamada Java para chamar um método Java, ou um serviço de negócios EJB, no fluxo de mensagens.

Antes de começar

Estas instruções consideram que você já esteja editando um fluxo de mensagens na página Editar Fluxo de Mensagens, conforme explicado em Criando, Editando e Exibindo Fluxos de Mensagens.

Para adicionar uma ação de Chamada Java

  1. Navegue até o local onde você deseja adicionar a ação, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
  2. Clique no ícone apropriado, depois selecione Adicionar uma Ação > Processamento da Mensagem > Chamada Java.
  3. Clique em <Method>. A página Selecionar um JAR será exibida. Selecione um recurso JAR na lista. A página Selecionar uma Classe e um Método será exibida.
  4. Na lista de classes Java listadas, clique no + ao lado da classe apropriada para exibir uma lista de métodos. Selecione um método e clique em Submeter. A ação de chamada Java será exibida na página Editar Estágio, conforme a seguir:
    • <Method> é substituído pelo nome do método Java selecionado nas etapas 2 e 3. Esse nome é um link para a página Selecionar uma Classe e Método. Você pode clicar neste link para alterar sua seleção de método Java.

      • O método deve ser estático.

    • Parâmetros: Um link <Expression> para a página Editor de Expressão XQuery é fornecida para cada argumento exigido pelo método Java. Um label para cada link indica o tipo de dados do argumento, que será um dos seguintes:
      • Java.lang.String
      • Tipos primitivos e seus tipos de classe correspondentes (por exemplo, int vs. java.lang.Integer)
      • java.lang.BigDecimal e java.lang.BigInteger (esses tipos são usados em cálculos financeiros quando erros de arredondamento ou overflows não são toleráveis)
      • apenas org.apache.xbeans.XmlObject e beans xml sem tipo.
      • byte[]
      • java.lang.String[] (INPUT ONLY)
      • XmlObject [ ] (INPUT ONLY)
      • javax.activation.DataSource
    • Retornar parâmetro como referência: esta opção torna o valor de retorno de uma invocação de Java Callout um elemento <java-content ref="jcid"> independentemente de seu tipo real, onde jcid é a chave para o objeto no repositório de objetos do pipeline. No campo de valor Resultado, digite o nome da variável que contém a referência de conteúdo java. Esta opção permite que você trabalhe com um objeto referenciado no fluxo de mensagens e também no XML do pipeline para oferecer passthrough, executando enriquecimento da mensagem com Java Callout e ações inline ou executando transformação da mensagem entre transportes Java e não Java. Para obter mais informações, consulte "Sending and Receiving Java Objects in Messages" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
    • Resultado: Um campo Resultado em que você informa a variável para a qual o resultado deve ser designado. O label do campo indica o tipo de dados do resultado.
      • Observação: Se o resultado for uma matriz de bytes (a única matriz possível retornada), o elemento XML de conteúdo binário será retornado.
    • Separar uma Conta de Serviço: Um link Conta de Serviço permite especificar uma Conta de Serviço opcional se houver um contexto de segurança para este método Java. Para saber mais sobre contextos de segurança e contas de serviço, consulte Contas de Serviço.

      • No caso de contas de serviço fixa e mapeadas, o ID do usuário/senha da conta de serviço será autenticado no sistema local e o contexto de segurança será propagado para a chamada Java. No caso de passagem, o contexto de segurança será propagado para a chamada Java. Esse contexto será o contexto de mensagem se definido (com Segurança de Web Services). Senão, será o contexto do transporte.

  5. Em Parâmetros, clique em <Expression>. A página Editor de Expressão XQuery será exibida. Use o Editor de Expressão XQuery para fornecer os argumentos necessários pelo método Java. Consulte Criando e Editando Expressões XQuery e XPath em Linha

    6. Se o tipo do valor de entrada informado não corresponder ao tipo de argumento de entrada declarado, o Oracle Service Bus tentará escalar automaticamente os valores de entrada no tipo declarado do argumento de entrada. Por exemplo, um valor de string "123" será convertido no número inteiro 123 caso o tipo declarado do argumento de entrada seja a primitiva java int.

  6. No campo Resultado, designe uma variável para o resultado retornado pelo método Java.
  7. se houver um contexto de segurança para o método Java, marque a caixa de seleção e clique em <Service Account>. A página Selecionar Conta de Serviço será exibida. Selecione a conta de serviço necessária na lista e clique em Submeter.

Depois de finalizar

Ao concluir a configuração desta ação, continue configurando outras ações ou salvando sua configuração, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
