Adicionando Ações de Log

Use a ação de log para construir uma mensagem a ser registrada e para definir um conjunto de atributos com os quais a mensagem é registrada.

Antes de começar

Estas instruções consideram que você já esteja editando um fluxo de mensagens na página Editar Fluxo de Mensagens, conforme explicado em Criando, Editando e Exibindo Fluxos de Mensagens.

Observação: Para ver dados de log no arquivo de log ou na saída padrão (console do servidor), o log do Oracle WebLogic Server deve ser definido com os seguintes níveis de severidade:

Severidade mínima para registro: Info
Arquivo de log: Info
Saída padrão: Info

Para obter informações sobre a definição de níveis de severidade do log, consulte "Using Log Severity Levels" em Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

 

Para adicionar uma ação de Log

  1. Certifique-se de que os Logs estejam ativados globalmente. Para obter mais informações, consulte Ativando Logs Globalmente.
  2. Navegue até o local onde você deseja adicionar a ação, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
  3. Clique no ícone apropriado, depois selecione Adicionar uma Ação > Relatórios > Log.
  4. Clique em <Expression>. A página Editor de Expressão XQuery será exibida. Especifique o contexto da mensagem a ser registrado por meio de expressões XQuery em variáveis de contexto. Consulte Criando e Editando Expressões XQuery e XPath em Linha.
  5. No campo Anotação, informe observações para esta ação de log. Essas observações serão registradas junto com o resultado a expressão definida anteriormente.
  6. Na lista drop-down nível de severidade, selecione uma das opções.

    7. Tabela 21-5 - Níveis de Severidade de Ação do Log
    Nível de Gravidade
    Uso Típico
    Info
    Usado para operações normais de relatórios; uma mensagem informativa no nível inferior.
    Advertência
    Uma operação ou configuração suspeita ocorreu, mas não pode afetar a operação normal.
    Erro
    Ocorreu um erro do usuário. O sistema ou aplicação pode tratar o erro sem interrupção e degradação limitada do serviço.
    Depurar
    Embora sua aplicação esteja em desenvolvimento, pode ser útil criar e usar mensagens que forneçam descrições detalhadas de atividade de baixo nível na aplicação.
    Certifique-se de que o nível de severidade do Log nas definições operacionais do serviço de proxy seja igual àquele da ação de log. Para obter informações sobre definições operacionais de serviços de proxy, consulte Configurando Definições Operacionais para Serviços de Proxy.

Depois de finalizar

Ao concluir a configuração desta ação, continue configurando outras ações ou salvando sua configuração, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
