Adicionando Ações da Tabela de Publicação

Use uma ação da tabela de publicação para publicar uma mensagem para zero ou mais serviços especificados estaticamente. A lógica da condição de estilo de alternância é usada para determinar no run-time quais serviços serão usados para a publicação.

Para obter mais informações sobre comportamento de publicação, consulte "Performing Transformations in Message Flows" em Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG205.

  1. Navegue até o local onde você deseja adicionar a ação, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
  2. Selecione Adicionar uma Ação > Comunicação > Publicar Tabela.
  3. Clique em <Expression>. A página Editor de Expressão XQuery será exibida. Crie uma expressão XQuery, que, no run-time, retorna o valor sobre o qual será tomada a decisão de roteamento. Consulte Criando e Editando Expressões XQuery e XPath em Linha.
  4. Na lista Operador, selecione um operador de comparação. Em seguida, no campo adjacente, informe um valor com o qual o valor retornado da expressão XQuery será avaliado.
  5. Clique em <Service> para selecionar um serviço no qual as mensagens devem ser publicadas caso a expressão seja avaliada como verdadeira para o valor especificado. A página Selecionar Serviço será exibida.
  6. Selecione um serviço na lista e, em seguida, clique em Submeter. Este é o serviço de destino para a mensagem.
  7. Se o serviço tiver operações definidas, você poderá especificar a operação a ser chamada selecionando-a na lista chamando.
  8. Se você deseja que a operação de saída seja igual à operação de entrada, marque a caixa de seleção Usar operação de entrada para saída.
  9. No campo Ações de Solicitação, para configura como a mensagem é compactada e enviada para o serviço, clique em Adicionar uma Ação, depois selecione uma ou mais ações que você deseja associar ao serviço. Para saber mais sobre o tipo de ação que você deseja adicionar, consulte Adicionando e Editando Ações em Fluxos de Mensagens.
    10. Observação: Existe um limite de aninhamento de quatro níveis cumulativos no editor de estágio. Se você tentar adicionar um quinto nível, a ação de aninhamento não será exibida. Níveis cumulativos incluem todas as ações de ramificação: condições se... então..., tabelas de publicação e tabelas de rota. Por exemplo, você pode ter 2 níveis de condicionais, depois uma tabela de publicação com uma tabela de rota dentro dela, totalizando 4 níveis. Se você tentar adicionar outra condicional (na última tabela de publicação), a condicional não será exibida.
  10. Para inserir um novo caso, clique no ícone Caso, depois selecione Inserir Novo Caso.
  11. Repita as etapas 4-8 para o novo caso.
  12. Adicionar casos extras conforme determinado pela lógica de negócios.
  13. Clique no ícone Caso do último caso definido na sequência, depois selecione Inserir Caso Default para adicionar um caso default no final.
  14. Configurar o caso default - a configuração desse caso especifica o comportamento de roteamento caso nenhum dos casos anteriores seja satisfeito.

Depois de finalizar

Ao concluir a configuração desta ação, continue configurando outras ações ou salvando sua configuração, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
