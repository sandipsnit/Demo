Adicionando Ações de Chamada de Serviço

Use uma ação de chamada de serviço para configurar uma chamada síncrona (bloqueio) para um serviço de proxy ou de negócios registrado para o Oracle Service Bus.

Para obter mais informações sobre ações de chamada de serviço, consulte "Constructing Service Callout Messages" em Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG208.

  1. Navegue até o local onde você deseja adicionar a ação, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
  2. Clique no ícone apropriado, depois selecione Adicionar uma Ação > Comunicação > Chamada de Serviço.
  3. Clique em <Service>. O Browser do Serviço será exibido.
  4. Selecione um serviço na lista de serviços de proxy ou de negócios registrados, depois clique em Submeter.
  5. Se o serviço escolhido na etapa 3, acima, for baseado em WSDL e tiver operações que possam ser chamadas no serviço, essas operações são listadas na lista drop-down de chamada de <Operação>. Selecione uma operação a ser chamada no serviço.

    Observação: A seleção de uma operação, que o Oracle Service Bus exige por vários motivos, não garante que somente a operação selecionada seja chamada. Por exemplo, se você selecionar OperationA, mas uma mensagem também contiver uma chamada para Operation B, OperationB também será chamada.
  6. Escolha como você deseja configurar as mensagens de solicitação e de resposta selecionando uma das seguintes opções:
    • Selecione Configurar Corpo SOAP para configurar o Corpo SOAP. A seleção desta opção permite usar $body diretamente.
      • Observação: Esta opção suporta SOAP-RPC codificado, que não é suportado durante a configuração de parâmetros ou do documento do payload.
    • Selecione Configurar Parâmetros de Payload ou ou Configurar Documento de Payload para configurar o payload.
  7. Opções de configuração subsequentes dependem do tipo de serviço selecionado na etapa 3 e no tipo de opções de configuração escolhidas para esses serviço na etapa 5. A Tabela 21-3 mostra as opções disponíveis para cada tipo de serviço.

    8. A Tabela 21-4, a seguir, fornece instruções para cada uma das opções listadas na Tabela 21-3, acima.

    Tabela 21-4 - Opções de Configuração de Chamada do Serviço
    Para Estas Opções...
    Siga Estas Etapas...
    Corpo da Solicitação SOAP e
    Corpo da Resposta SOAP
    Para configurar estas opções,
    • No campo Corpo da Solicitação SOAP, informe o nome de uma variável que conterá o XML do elemento Corpo SOAP para a solicitação de chamada.
    • No campo Corpo da Resposta SOAP, informe o nome de uma variável à qual o XML do elemento Corpo SOAP na resposta será ligado.
    Cabeçalho da Solicitação SOAP e
    Cabeçalho da Resposta SOAP
    Para configurar estas opções,
    • No campo Cabeçalho da Solicitação SOAP, informe o nome de uma variável que conterá o XML do elemento Cabeçalho SOAP para a solicitação de chamada

      • Você deve agrupar o documento de entrada para o Cabeçalho da Solicitação SOAP com <soap-env:Header>...</soap-env:Header>.

    • No campo Cabeçalho da Resposta SOAP, informe o nome de uma variável à qual o XML dos Cabeçalhos SOAP na resposta, se houver, será ligado.
    Parâmetros de Solicitação e
    Parâmetros de Resposta
    Para configurar as opções,
    • Nos campos Parâmetros da Solicitação, informe nomes para as variáveis que serão avaliadas no run-time para fornecer valores para os parâmetros da solicitação.

      • É necessário fornecer apenas os documentos principais do payload na variável de entrada; o pacote SOAP é criado para você pelo Oracle Service Bus. Em outras palavras, não agrupe o documento de entrada com <soap-env:Body>...</soap-env:Body>.

      Por exemplo, ao criar uma variável de entrada do corpo que seja usada para este parâmetro de solicitação, defina o conteúdo dessa variável usando a instrução XPath body/* (para remover o wrapper soap-env:Body), não $body (que resulta na manutenção do wrapper soap-env:Body).

    • Nos campos Parâmetros da Resposta, informe os nomes das variáveis para as quais as respostas serão designadas no run-time.
    Documento de Solicitação e
    Documento de Resposta
    Para configurar estas opções,
    • No campo Variável do Documento de Solicitação, informe o nome de uma variável a ser atribuída a um documento de solicitação.

      • Para os serviços do tipo Documento SOAP, a variável é avaliada no run-time para formar o corpo da mensagem SOAP enviada para o serviço. Para os serviços Qualquer XML, a variável é avaliada no run-time para formar o corpo da mensagem XML enviada para o serviço.

      Para os serviços do tipo Documento SOAP e para os serviços Qualquer XML, forneça apenas os documentos principais do payload na variável de entrada; o pacote SOAP é criado para você pelo Oracle Service Bus. Em outras palavras, não agrupe o documento de entrada com <soap-env:Body>...</soap-env:Body>.

      Por exemplo, ao criar uma variável de entrada do corpo que seja usada para este parâmetro de solicitação, defina o conteúdo dessa variável usando a instrução XPath body/* (para remover o wrapper soap-env:Body), não $body (que resulta na manutenção do wrapper soap-env:Body).

      Para serviços de Mensagens, a variável é avaliada para formar o corpo da mensagem, com base no tipo de dados esperados pelo serviço. As seguintes restrições aplicam-se às variáveis usadas com serviços de Mensagens:

      • Para serviços que esperam dados binários, as variáveis devem ter um elemento ctx:binary-content .
      • Para serviços que esperam dados MFL, a variável deve ter o equivalente XML.
      • Para serviços que esperam dados de texto, a variável é uma string.
    • No campo Variável do Documento de Resposta, informe o nome da variável para a qual um documento de resposta será designado no run-time.

  8. Como opção, adicione um ou mais cabeçalhos de transporte. Para obter mais informações, consulte Adicionando Ações do Cabeçalho de Transporte.
    9. Observação: Além dos cabeçalhos de transporte que você especifica, cabeçalhos são adicionados à camada de binding do Oracle Service Bus. Para obter mais informações, consulte "Configuring Transport Headers in Message Flows" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG201.

Depois de finalizar

Ao concluir a configuração desta ação, continue configurando outras ações ou salvando sua configuração, conforme descrito em Adicionando e Editando Ações em Fluxos de Mensagens.
