Adicionando Ramificações Condicionais a Fluxos de Mensagens

Um nó de ramificação permite que o processamento prossiga exatamente em um dos diversos caminhos possíveis. A ramificação é orientada por uma tabela de alternância baseada no XPath. Cada ramificação na tabela especifica uma condição (por exemplo, <500) que é avaliada em ordem decrescente no fluxo de mensagens com uma única expressão XPath (por exemplo, ./ns: PurchaseOrder/ns:totalCost em $body). Qual condição for satisfeita primeiro determinará qual ramificação será seguida. Se nenhuma condição de ramificação for satisfeita, então a ramificação default será seguida. Um nó de ramificação pode ter diversos descendentes no fluxo de mensagens: um para cada ramificação, incluindo a ramificação default.

Se o serviço de proxy não for baseado em um WSDL e receber vários tipos de documento como entrada, considere um nó de ramificação condicional.

Uma ramificação condicional é orientada por uma tabela de pesquisa com cada ramificação marcada com um valor de string simples, mas exclusivo. Uma variável no contexto da mensagem é designada como a variável de pesquisa para esse nó e, no run-time, seu valor é usado para determinar qual ramificação seguir. Se nenhuma ramificação corresponder ao valor da variável de pesquisa, a ramificação default será seguida. Você deve designar o serviço de proxy de tal forma que o valor da variável de pesquisa seja definido antes de atingir o nó de ramificação.

Antes de começar

Estas instruções consideram que você já esteja editando um fluxo de mensagens na página Editar Fluxo de Mensagens, conforme explicado em Criando, Editando e Exibindo Fluxos de Mensagens.

Para adicionar uma ramificação condicional a um fluxo de mensagens

  1. Clique em um ícone Nó do Par de Pipelines ou em um ícone Nó de Ramificação e, em seguida, clique em Criar Ramificação Condicional. O nó de ramificação condicional é adicionado e os nós existentes após o nó de ramificação inserido são movidos para a ramificação default do novo nó de ramificação condicional.
  2. Para alterar o nome default e adicionar uma descrição para o nó de ramificação, faça o seguinte:
    1. Clique no ícone Ramificação Condicional, depois clique em Editar Nome e Anotação.
    2. Altere o nome e a descrição, conforme o desejado.
    3. Clique em Salvar.
  3. Para adicionar definições de ramificação, clique no ícone Ramificação Condicional, depois clique em Editar Ramificação. A página Editar Nó de Ramificação será exibida.
  4. Faça o seguinte:
    1. No campo Caminho Selecionado, clique em Editar para adicionar uma expressão XPath para especificação do caminho. Consulte Criando e Editando Expressões XQuery e XPath em Linha.
    2. No campo Variável, informe uma variável de contexto.
    3. No campo Operador, selecione um operador de comparação.
    4. No campo Valor, informe um valor para a ramificação.
    5. Na coluna Label, informe um label para a ramificação.
  5. Como alternativa, em Opções:
    • Clique em Adicionar uma Nova Ramificação para adicionar uma nova definição de ramificação a este nó de ramificação.
    • Clique em Deletar esta Ramificação para deletar uma definição de ramificação.
    • Clique em Mover Ramificação para Cima ou em Mover Ramificação para Baixo para alterar as posições de definições de ramificação. Essa opção é exibida apenas quando existe mais de uma definição de ramificação.
  6. Clique em Salvar para confirmar as atualizações na sessão atual.
  7. Na página Editar Fluxo de Mensagens, continue para construir o fluxo de mensagens, conforme descrito em Criando, Editando e Exibindo Fluxos de Mensagens.
  8. Clique em Salvar para confirmar as atualizações na sessão atual.
  9. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
