Página Configuração do Transporte de E-mail

A página Criar/Editar um Serviço de Proxy - Configuração do Transporte WS pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Use esta página para configurar as definições de transporte para um serviço de proxy usando o protocolo de transporte de e-mail. Tabela 19-4 descreve como usar a página.

Tabela 19-4 - Página Criar/Editar um Serviço de Proxy - Configuração do Transporte de E-mail
Opção
Para criar ou editar...
Camada SSL Obrigatória
Selecione essa opção para se comunicar usando o Secure Sockets Layer. Os números de porta SSL default são:
  • POP3(S): 995
  • IMAP: 993
Observação: O certificado do servidor de e-mail deverá estar presente na área de armazenamento confiável do Oracle Service Bus para que a SSL funcione.
Conta do Serviço
Informe um nome de conta de serviço ou clique em Procurar para selecionar contas de serviço a partir de um browser.
Servidor Gerenciado
Este campo só é exibido em um domínio em cluster.
Selecione o Servidor Gerenciado para atuar como o servidor de sondagem. Todos os Servidores Gerenciados podem processar a mensagem, mas apenas um pode sondar a mensagem.
Intervalo de Sondagem
Informe um intervalo de sondagem, em segundos.
Protocolo de E-mail
Selecione POP3 ou IMAP como o tipo de servidor para a conta de e-mail.
Limite de Leitura
Especifique o número máximo de mensagens a serem lidas por sweep de polling. Informe 0 para especificar nenhum limite.
Passar por Referência
Marque esta caixa de seleção para separar o arquivo no diretório de archive e transmiti-lo como uma referência nos cabeçalhos.
Por default, quando você cria um novo serviço, a opção Passar por Referência é selecionada e você deve especificar o local do diretório de archive.
Passar Anexos por Referência
Marque esta caixa de seleção para separar os anexos no diretório de archive e transmiti-los como uma referência nos cabeçalhos.
Por default, quando a opção Passar por Referência é selecionada, a opção Passar Anexos por é implicitamente verdadeira e você deve especificar o local do diretório de archive.
Ação Pós-Leitura
Selecione o que acontece a uma mensagem depois que é lida:
  • Arquivar - A mensagem é arquivada.
  • Deletar - A mensagem é deletada.
  • Mover - A mensagem será movida. A movimentação só estará disponível com o protocolo IMAP.
Anexos
Selecione como os anexos são tratados:
  • Archive - os anexos são salvos no diretório de archive.
  • Ignorar - os anexos são ignorados.
Pasta de Movimentação IMAP
Informe a pasta para a qual a mensagem será movida se o campo Ação Pós-Leitura for definido como Mover.
Diretório de Download
Informe um local temporário para download de e-mails.
Diretório de Archive
Especifique o caminho para o local de archive se o campo Ação Pós-Leitura for definido como Archive.
Este campo é obrigatório se a opção Passar por Referência ou Passar Anexos por Referência for selecionada.
Diretório de Erros
Informe o caminho do diretório do sistema de arquivos para gravar a mensagem e os anexos se houver um problema.
Codificação da Solicitação
Aceite o default ISO-8859-1 como a codificação de conjunto de caracteres para solicitações em transportes de e-mail ou informe outra codificação de conjunto de caracteres.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Proxy - Resumo.
