Página Configuração do Transporte de Arquivos

A página Criar/Editar um Serviço de Negócios - Configuração do Transporte de Arquivos pertence a uma série de páginas para criação e edição de serviços de negócios, conforme descrito em Criando e Configurando Serviços de Negócios e Editando Configurações do Serviço de Negócios.

Use esta página para configurar as definições de transporte para um serviço de proxy usando o protocolo de transporte de arquivos. Tabela 19-5 descreve como usar esta página.

Tabela 19-5 - Página Criar/Editar um Serviço de Proxy - Configuração do Transporte de Arquivos
Opção
Para criar ou editar...
Máscara do Arquivo
Especifica os arquivos que devem ser agrupados pelo serviço de proxy. Se o URI for um diretório e *.* for especificado, então o serviço agrupará todos os arquivos no diretório. Apenas os caracteres curingas * e ? são permitidos na Máscara do Arquivo. Expressões regulares não são suportadas.
Servidor Gerenciado
Este campo só é exibido em um domínio em cluster.
Selecione o Servidor Gerenciado para atuar como o servidor de sondagem. Todos os Servidores Gerenciados podem processar a mensagem, mas apenas um pode sondar a mensagem.
Intervalo de Sondagem
Informe um intervalo de sondagem, em segundos. O default é 60.
Limite de Leitura
Especifique o número máximo de mensagens a serem lidas por sweep de polling. Informe 0 para especificar nenhum limite. O default é 10.
Classificar por Chegada
Marque esta caixa de seleção para selecionar quais eventos são entregues na ordem de chegada.
Observe que, quando essa opção é selecionada para um serviço de proxy executado em um ambiente em cluster, as mensagens são sempre enviadas para o mesmo servidor. Em outras palavras, o balanceamento de carga entre os servidores é ignorado quando essa opção é selecionada.
Verificar Subdiretórios
Marque esta caixa de seleção para verificar recursivamente todos os diretórios.
Passar por Referência
Marque esta caixa de seleção para separar o arquivo no diretório de archive e transmiti-lo como uma referência nos cabeçalhos.
Ação Pós-Leitura
Selecione o que acontece a uma mensagem depois que é lida:
  • Arquivar - A mensagem é arquivada.
  • Deletar - A mensagem é deletada.
Diretório de Estágio
Informe um diretório intermediário para separar temporariamente os arquivos durante seu processamento.
Não coloque o diretório de estágio dentro do diretório de polling (o diretório identificado no URL do serviço de proxy de transporte; por exemplo, arquivo:///c:/dir1/dir2).
Diretório de Archive
Especifique o caminho para o local de archive se a opção Ação Pós-Leitura for definida como Archive. O campo Diretório de Archive também é obrigatório se você tiver selecionado o campo Passar por Referência.
Não colocar o diretório de archive dentro do diretório de polling.
Diretório de Erros
Informe o local em que as mensagens e anexos serão postados se houver um problema.
Não colocar o diretório de erros dentro do diretório de polling.
Codificação da Solicitação
Aceite o default UTF-8 como a codificação de conjunto de caracteres para solicitações em transportes de arquivo ou informe outra codificação de conjunto de caracteres.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Proxy - Resumo.
