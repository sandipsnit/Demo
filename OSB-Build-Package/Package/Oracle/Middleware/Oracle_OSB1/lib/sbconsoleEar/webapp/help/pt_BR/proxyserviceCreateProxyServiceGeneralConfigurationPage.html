Página Configuração Geral

A página Criar/Editar um Serviço de Proxy - Configuração Geral pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Use esta página para modificar as definições de configuração gerais para um serviço de proxy.

Quando você cria um serviço de proxy, essa é a primeira página exibida em uma série de páginas para configuração do serviço. As páginas exibidas após essa diferem de acordo com as opções feitas nesta página.

A Tabela 19-1 descreve como usar a página.

Tabela 19-1 - Página Criar/Editar um Serviço de Proxy - Configuração Geral
Opção
Para criar ou editar...
Nome do Serviço
Informe um nome exclusivo para o serviço proxy.

Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
Descrição
Informe uma descrição.
Tipo de Serviço
Selecione o tipo de serviço de proxy a ser criado:
  • Web Service de WSDL - Selecione esta opção para criar um serviço de proxy com base em um WSDL. Em seguida, informe o nome WSDL, qualificado por seu caminho (por exemplo, myProject/myFolder/myWSDL). Como alternativa, clique em Procurar para selecionar um WSDL na página Selecionar uma Definição WSDL.

    • (porta ou binding) - Informe o nome de uma porta (definido no WSDL) para descrever um endereço de transporte real ou informe o nome de um binding (definido no WSDL) para mapear para um endereço de transporte. Se você usar Procurar para selecionar um WSDL, a página Selecionar uma Definição WSDL listará as portas e os bindings definidos.

Observação: Se você for usar o Tipo de Corpo SOAP para operações, certifique-se de que o WSDL não possua duas operações com a mesma mensagem de entrada. A operação Tipo de Corpo SOAP não pode ser identificada exclusivamente com a inspeção da mensagem de entrada.
  • Serviço do Tipo de Transporte - Selecione esta opção para criar um serviço de proxy JEJB.
  • Serviço de Mensagens - Selecione esta opção para criar um serviço que possa receber mensagens de um tipo de dados e responder com mensagens de um tipo de dados diferente. Essas trocas podem ser de solicitação/resposta ou unilaterais.

    • (HTTP GET é suportado apenas nos tipos de serviço Qualquer Serviço XML e Serviço de Mensagens.)

  • Qualquer Serviço SOAP - Selecione esta opção para criar um serviço SOAP que não possua uma interface concreta definida explicitamente.

    • Selecione SOAP 1.1 ou SOAP 1.2 na lista drop-down para especificar a versão SOAP a ser usada.

  • Qualquer Serviço XML - Selecione esta opção para criar um serviço XML que não possua uma interface concreta definida explicitamente.

    • (HTTP GET é suportado apenas nos tipos de serviço Qualquer Serviço XML e Serviço de Mensagens.)

  • Serviço de Negócios - Selecione esta opção para criar um serviço de proxy com um nó de rota que encaminhe para o serviço de negócios selecionado. Não é possível criar um serviço de proxy de um serviço de negócios do tipo de transporte. Se você criar um serviço de proxy com base em um serviço de negócios de transporte DSP, o Oracle Service Bus alternará o tipo de transporte do serviço de proxy para o HTTP, porque o transporte DSP não poderá ser usado para serviços de proxy. É possível alterar o tipo de transporte do serviço de proxy para qualquer outro transporte disponível.

    • Informe o caminho (projeto/pasta) e o nome do serviço de negócios; ou clique em Procurar para selecionar o serviço de negócios na página Selecionar Serviço de Negócios.

Tipo de Serviço (continuação)
  • Serviço de Proxy - Selecione esta opção para clonar um serviço de proxy existente. Como o Oracle Service Bus não aceita o mesmo URI para vários serviços, você deve alterar o URI para o serviço clonado.
Observação: Quando um serviço é criado com base em outro serviço, as regras de alerta são mantidas da seguinte maneira:
    • Quando um serviço de proxy é criado a partir de um serviço de negócios ou um serviço de negócios é criado a partir de um serviço de proxy, as regras de alerta, se houver, são removidas.
    • Quando um serviço de proxy é criado a partir de outro serviço de proxy ou um serviço de negócios é criado a partir de outro serviço de negócios, as regras de alerta, se houver, são mantidas.
Provedor de Chaves de Serviço
Se necessário, informe o caminho (projeto/pasta) e o nome de um provedor de chaves de serviço ou clique em Procurar para selecionar um na página Selecionar Provedor de Chaves de Serviço.
Um provedor de chaves de serviço só é obrigatório em determinados casos:
  • TLS/SSL bilateral de saída, em que o serviço de proxy encaminha mensagens para serviços HTTPS que requerem autenticação de certificado cliente.
  • Em alguns cenários de segurança do Web Service, por exemplo, se o serviço de proxy exigir que as mensagens sejam criptografadas.
Para adicionar um serviço de proxy ativado para segurança do Web service, é necessário criar o serviço de proxy a partir de um WSDL (porta ou binding) com anexos de Política de Web Services.
Para obter mais informações, consulte Provedores de Chaves de Serviço. Para saber como criar um provedor de chaves de serviço, consulte Adicionando Provedores de Chaves de Serviço.
Publicar no Registro
Esta opção só aparece quando existe um registro UDDI default.
Selecione esta opção para publicar o serviço de proxy no registro default automaticamente. Para obter mais informações, consulte Usando Publicação Automática.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Proxy - Resumo.
