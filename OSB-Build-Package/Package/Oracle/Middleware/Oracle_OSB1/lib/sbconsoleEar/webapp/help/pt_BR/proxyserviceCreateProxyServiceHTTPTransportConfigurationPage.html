Página Configuração do Transporte HTTP

O transporte HTTP agora suporta pontos finais HTTP e HTTPS.

A página Criar/Editar um Serviço de Proxy - Configuração do Transporte HTTP pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Use esta página para configurar as definições de transporte para um serviço de proxy usando o protocolo de transporte HTTP. Tabela 19-7 descreve como usar a página.

Tabela 19-7 - Página Criar/Editar um Serviço de Proxy - Configuração do Transporte HTTP
Opção
Para criar ou editar...
HTTPS obrigatório
Marque esta caixa de seleção para pontos finais HTTPS de entrada.
Para saber mais, consulte "Configuring Transport-Level Security" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
Autenticação
Selecione uma das seguintes opções:
  • Nenhum - Especifica que não é necessária autenticação
  • Básico - Especifica se é necessária autenticação básica para acessar esse serviço.

    • A autenticação básica instrui o WebLogic Server a autenticar o cliente usando um nome de usuário e senha nos provedores de autenticação configurados no realm de segurança, como um serviço de diretório LDAP (Lightweight Directory Access Protocol) e o Windows Active Directory. O cliente deve enviar seu nome de usuário e senha no cabeçalho da solicitação HTTP.

    Autenticação básica é altamente não recomendável sobre HTTP, porque a senha é enviada em texto sem formatação. No entanto, é seguro enviar senhas sobre HTTPS, porque HTTPS oferece um canal criptografado.

    Advertência: Por default, todos os usuários (autorizados e anônimos) podem acessar um serviço de proxy. Para limitar os usuários que podem acessar um serviço de proxy, crie uma política de autorização de transporte. Consulte Editando Políticas de Acesso de Transporte.

  • Certificado Cliente - Especifica comunicação criptografada e autenticação de cliente forte (SSL bilateral). Para saber mais, consulte "Configuring Transport-Level Security" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557.
  • Autenticação Personalizada - Especifique que um token de autenticação está contido em um cabeçalho HTTP. A identidade do cliente é estabelecida por meio do uso deste token fornecido pelo cliente. Você deve configurar um provedor de Asserção de Identidade que mapeia o token para um usuário do Oracle Service Bus.

    • O token de autenticação personalizado pode ser qualquer tipo de token ativo suportado por um provedor de Asserção de Identidade do WebLogic configurado.

Política de Distribuição
Selecione uma política de despacho para esse ponto final. Deixe em branco para usar a política de despacho default.
Política de despacho refere-se à instância do Gerenciador de Trabalho do WLS que você deseja usar para o ponto final do serviço. Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificação da Solicitação
  • Para transportes de entrada HTTP:

    • Se o parâmetro de codificação do conjunto de caracteres do cabeçalho Tipo de Conteúdo não for especificado na Solicitação do Cliente, informe um. Se você não informar um valor, o campo assumirá ISO-8859-1 como default.

  • Para transportes de saída HTTP:

    • Se você não tiver configurado uma codificação de solicitação, o run-time do Oracle Service Bus decidirá a codificação mais apropriada ao fazer uma solicitação no serviço de negócios. No caso de um cenário não de passagem, a codificação de caractere default será UTF-8 no run-time. No entanto, se for um cenário de passagem, o run-time transmitirá a codificação recebida com a resposta de saída.

Codificação da Resposta
Aceite o default ISO-8859-1 como a codificação de conjunto de caracteres para respostas em transportes HTTP ou informe outra codificação de conjunto de caracteres.
Cabeçalho de Autenticação
Informe o cabeçalho HTTP (qualquer um, exceto Autorização) do qual o Oracle Service Bus deve extrair o token. Esse campo só estará disponível se você tiver marcado a caixa de seleção Autenticação Personalizada.
Por exemplo, client-xyz-token.
Tipo de Token de Autenticação
Selecione um tipo de token de autenticação. Apenas os tipos de token ativos configurados para um provedor de Asserção de Identidade estão disponíveis. (Consulte Configuring Identity Assertion Providers para Tokens Personalizados para obter mais informações.) Esse campo só estará disponível se você tiver marcado a caixa de seleção Autenticação Personalizada.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Proxy - Resumo.
