Página Configuração do Transporte JCA

Use esta página para configurar definições de transporte usando o protocolo de transporte JCA. Para obter mais informações sobre o uso do transporte JCA, consulte "JCA Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV910 e Gerando um Serviço de Proxy a partir de um Recurso de Binding de JCA.

Tabela 19-8 - configuração do transporte JCA
Opção
Descrição
Arquivo JCA
Clique em Procurar para selecionar um Binding de JCA. O Binding de JCA define diferentes aspectos do serviço, como detalhes sobre o adaptador usado, um binding para o arquivo de mapeamento WSDL e TopLink ou EclipseLink e as propriedades de especificação de ativação/interação necessária pelo serviço.
Quando você seleciona um Binding de JCA válido, os campos da configuração de transporte restantes permanecem disponíveis.
Para obter mais informações sobre Bindings de JCA, consulte o Capítulo 12, Bindings de JCA.
Nome do Adaptador
Um valor somente para leitura mostrando o nome do adaptador que o serviço JCA usará.
Tipo de Adaptador
Um valor somente para leitura mostrando o tipo de adaptador.
Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir.
Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Conta de Serviço de JNDI
A Conta de Serviço de JNDI serve para segurança do contexto de JNDI, usada para acessar o factory de conexão gerenciado pelo adaptador EIS. Clique em Procurar e selecione uma conta de serviço. Se nenhuma conta de serviço for especificada, será usado um assunto anônimo.
Para serviços de negócios JCA, não há restrição quanto ao tipo de conta de serviço de JNDI que pode ser configurada, como estática ou de passagem, mas o run-time deve conseguir acessar um nome de usuário e senha. Os serviços de proxy JCA podem usar apenas contas de serviço de JNDI estáticas.
Para obter mais informações sobre contas de serviço de JNDI, consulte "Security" no capítulo "JCA Transport" do Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV916
Propriedades do Ponto Final
Este campo permite que você atribua valores para propriedades do ponto final, como repetições para o tipo de adaptador usado pelo serviço.
Para obter uma lista de propriedades suportadas do ponto final JCA, consulte "Endpoint Properties" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1841.
Propriedades Dinâmicas do Ponto Final
Esta opção permite transmitir parâmetros da solicitação para serviços compatíveis com JCA. Por exemplo, você pode usar uma propriedade dinâmica do ponto final para transmitir parâmetros de consulta do banco de dados para o Oracle JCA Adapter for Database.
Para obter mais informações sobre consulta com parâmetros, consulte "Oracle JCA Adapter for Database" no Oracle Fusion Middleware User's Guide for Technology Adapters em http://www.oracle.com/pls/as111170/lookup?id=TKADP2117.
Informe um par de nome/valor para cada propriedade dinâmica do ponto final que você deseja fornecer. A chave da propriedade do ponto final corresponde ao nome do parâmetro de consulta.
Sempre usar configuração do arquivo JCA
Esta opção determina se ou não as Propriedades da Especificação de Ativação (serviços de proxy) e as Propriedades da Especificação de Interação (serviços de negócios) são sempre usadas do arquivo JCA.
Se esta opção for selecionada (default), o transporte JCA interagirá com a estrutura JCA usando as propriedades da especificação de ativação/interação no arquivo JCA.
Se essa opção for desmarcada, você poderá sobrepor as Propriedades da Especificação de Ativação/Interação.
Para saber o impacto de nova implantação do uso desta opção, consulte "Endpoint Redeployment" no capítulo "JCA Transport" do Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV940.
Nome da Operação
Exibe um nome somente para leitura da operação WSDL selecionada. Uma operação pode ter suas próprias propriedades de especificação de ativação/interação, mostradas no campo Propriedades de Especificação de Ativação/Interação.
Propriedades da Especificação de Ativação/Interação
Propriedades da Especificação de Ativação é o nome do campo para serviços de proxy; Propriedades da Especificação de Interação é o nome do campo para serviços de negócios.
Se este serviço for um serviço de entrada chamado por uma aplicação EIS, esse campo exibirá as propriedades da especificação de ativação para a operação de entrada JCA mostrada no campo Nome da Operação.
É possível substituir as propriedades da especificação de ativação/interação se você desmarcar Sempre usar configuração do arquivo JCA.
Observação: Para os adaptadores do Oracle Adapter Suite, propriedades de especificação de ativação/interação são exibidas como somente para leitura. Os adaptadores do Oracle Adapter Suite armazenam suas próprias configurações, que você deve alterar nas ferramentas de gerenciamento do Oracle Adapter Suite.
Propriedades de conexão (legado)
Para serviços JCA de legado que usam propriedades de conexão de modo não-gerenciado (obsoleto nesta release), consulte as opções de configuração da conexão em http://download.oracle.com/docs/cd/E13159_01/osb/docs10gr3/jcatransport/transport.html#wp1105451.

Para obter mais informações sobre o ponto final e as propriedades de especificação de ativação/interação, consulte o Oracle Fusion Middleware User's Guide for Technology Adapters em http://www.oracle.com/pls/as111170/lookup?id=TKADP.
