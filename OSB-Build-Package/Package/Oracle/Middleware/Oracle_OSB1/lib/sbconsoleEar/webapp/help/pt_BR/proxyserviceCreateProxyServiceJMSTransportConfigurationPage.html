Página Configuração do Transporte JMS

A página Criar/Editar um Serviço de Proxy - Configuração do Transporte JMS pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Para obter mais informações, consulte "JMS Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.

Use esta página para configurar as definições de transporte para um serviço de proxy usando o protocolo de transporte JMS. Tabela 19-10 descreve como usar a página.

Tabela 19-10 - Página Criar/Editar um Serviço de Proxy - Configuração do Transporte JMS
Opção
Para criar ou editar...
Tipo de Destino
Selecione uma das seguintes opções:
  • Fila (para um tipo de destino de ponto-a-ponto)
  • Tópico (para um destino de publicação/inscrição)
Resposta Obrigatória?
Esta opção só está disponível quando Fila é selecionado para o Tipo de Destino.
Selecione esta opção para especificar que uma resposta é esperada após o envio de uma mensagem de saída.
Padrão de Resposta
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Selecione uma das seguintes opções:
  • Selecione JMSMessageID para serviços JAX-RPC em execução no Oracle WebLogic Server.
  • Selecione JMSCorrelationID para todos os outros serviços. Ao selecionar esta opção, você também deve informar um URI de Resposta.
Tipo de Mensagem de Resposta
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Selecione uma das seguintes opções:
  • Bytes (para um fluxo de bytes não-interpretados)
  • Texto (para mensagens de texto)
Esta opção estará desativada se você selecionar um Tipo de Mensagem de Java para a resposta.
Jar do Cliente
Esta opção estará disponível quando o serviço for um Serviço de mensagens com um tipo de solicitação de Java. Selecione o cliente JAR a ser usado para retirar da fila mensagens que contenham Objetos Java. Selecionar o cliente JAR garante que ele esteja no classpath.

Para obter mais informações, consulte "Sending and Receiving Java Objects in Messages" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874.
Política de Distribuição
Selecione a instância do Gerenciador de Trabalho do Oracle WebLogic Server que você deseja usar para a política de despacho deste ponto final. O Gerenciador de Trabalho default será usado se nenhum outro existir.
Por exemplo, se o serviço de negócios tiver um protocolo de transporte JMS, o ponto final do serviço de negócios será um arquivo JAR MDB (bean baseado em mensagens) que pode ser associado à política de despacho específica.
Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Codificação da Solicitação
Informe o conjunto de caracteres para solicitações de codificação. O default é UTF-8.
Codificação da Resposta
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Informe o conjunto de caracteres para respostas de codificação. O default é UTF-8.
Timeout da Resposta do Cliente
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Informe o número de segundos para aguardar uma resposta do servidor antes de eliminar a conexão. Isso só se aplica se o cliente for outro serviço de proxy no mesmo domínio.
URI da Resposta
Esta opção só está disponível quando JMSCorrelationID é selecionado para o Padrão de Correlação da Resposta.
Informe um URI de resposta no formato:
jms://host:port/connection_factory/jndi_destination
Para destinar vários servidores, use o seguinte formato:
jms://host1:port,host2:port/connection_factory/jndi_destination
Também é possível omitir o host e a porta no URI de resposta. Por exemplo:
jms:///connection_factory/jndi_destination

Quando você omite o host e a porta, a pesquisa de factory de conexão/destino ocorre no servidor local. Isso é útil, por exemplo, se o URI da solicitação for para um factory de conexão/destino estrangeiro, mas você quiser que a resposta seja enviada para o servidor local.

Observação: Embora o Oracle WebLogic Server permita o uso de barras em nomes JNDI, como "myqueues/myqueue", os nomes JNDI com barras interferem no formato de URI exigido pelo Oracle Service Bus e você não pode usá-los. Para resolver esse problema, defina e referencie um servidor externo JMS no URI. Para obter mais informações, consulte "Configure foreign servers" em Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help em http://www.oracle.com/pls/as111170/lookup?id=WLACH.

Factory de Conexão de Resposta
Esta opção só está disponível quando JMSMessageID é selecionado para o Padrão de Correlação da Resposta.
Informe um URI do factory de conexão de resposta.
Se um factory de conexão não for especificado, o factory de conexão da solicitação será usado para a resposta.
Conta de Serviço JMS
Selecione uma conta de serviço a ser usada para o recurso JMS gerenciado pelo servidor JMS. Uma conta de serviço é um recurso de alias para um ID de usuário e senha, usado para a solicitação e resposta. A mesma conta de serviço é usada para JMS e JNDI.
Para obter mais informações, consulte Contas de Serviço.
Usar SSL
Selecione esta opção apenas se as solicitações forem feitas em uma conexão TLS/SSL.
O TLS/SSL (Secure Sockets Layer) oferece conexões seguras permitindo que duas aplicações se conectem em uma rede para autenticar a identidade da outra e criptografando os dados trocados entre as aplicações. A autenticação permite que um servidor e, como opção, um cliente, verifique a identidade da aplicação na outra extremidade de uma conexão de rede. Além disso, se o administrador tiver acesso restrito a destinos JMS individuais (filas ou tópicos) definindo o controle de acesso na entrada de JNDI para o destino, o serviço deverá ser autenticado ao pesquisar a entrada na árvore de JNDI.
Seletor de Mensagens
Informe uma expressão do seletor de mensagem.
Apenas mensagens com propriedade correspondentes à expressão são processadas
ID do Cliente JMS

Para tópicos apenas, informe o ID do cliente a ser utilizado para o assinante. Se nenhum valor for informado, um ID de cliente será gerado automaticamente. Designar um ID personalizado ajuda a identificar esse componente durante o monitoraramento do serviço. Observe que essa opção só está efetiva para assinantes duráveis.

O ID do cliente JMS é um dos vários descritores de MDB usados para gerar nomes de assinantes e o valor de ClientID para os assinantes do tópico. A definição de um ID do cliente JMS facilita identificar e exibir os assinantes para um tópico específico por meio da verificação do nome do assinante ou do ID do cliente.

Para obter mais informações, consulte "JMS Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037.
Assinatura Durável
Esta opção só está disponível se Tópico for selecionado para o Tipo de Destino.
Marque esta caixa de seleção se a assinatura for durável ou deixe em branco se a assinatura não for durável
Contagem de Repetições
Informe o número de novas tentativas de entrega que uma mensagem pode fazer antes de ser movida para o destino de erro. Esse campo aplica-se apenas aos destinos JMS do Oracle WebLogic Server.
Intervalo de Repetição
Informe a quantidade de tempo, em milissegundos, antes que as mensagens recuperadas ou que sofreram rollback sejam reentregues. Esse campo aplica-se apenas aos destinos JMS do Oracle WebLogic Server.
Destino de Erro
Informe o nome do destino alvo de mensagens que atingiram o limite de nova entrega. Esse campo aplica-se apenas aos destinos JMS do Oracle WebLogic Server.
Política de Expiração
Selecione uma Política de Expiração a ser usada quando uma mensagem expirada é encontrada em um destino. Esse campo aplica-se apenas aos destinos JMS do Oracle WebLogic Server.
XA Obrigatório?
Marque esta caixa de seleção caso o factory de conexão seja XA.
O valor é levado em consideração quando o factory de conexão remoto está indisponível. Se o factory de conexão estiver disponível e esse valor for verdadeiro, certifique-se de ele ela esteja definido como transacional.
Timeout da Transação
Informe o tempo em segundos que o serviço de proxy JMS deve para aguardar até que uma transação seja processada. Esta opção aplica-se somente aos serviços que utilizam uma factory de conexão XA. O default é 600 segundos.
Distribuição de mensagens do tópico
Esta opção está disponível ao selecionar Tópico para o Tipo de Destino. A Distribuição de Mensagens de Tópico fornece opções diferentes para manipular mensagens JMS de entrada e fornecer alta disponibilidade e falha.

Selecione uma das seguintes propriedades para determinar como os beans acionados por mensagens manipulam mensagens JMS de entrada:
  • Uma cópia por aplicação (padrão) – Se o serviço de proxy do JMS estiver disponibilizado em um cluster, esta opção oferecerá alta disponibilidade e escalabilidade, garantindo que uma mensagem de JMS de entrada seja processada somente em um dos servidores disponíveis em um cluster.
  • Uma Cópia por Servidor – Selecione essa opção se desejar que as mensagens JMS de entrada sejam publicadas em um tópico a ser recebido pelo serviço proxy em cada membro do cluster.
  • Compatibilidade – Selecione essa opção se desejar que as mensagens JMS de entrada sejam processadas em um Servidor Gerenciado específico ou em todos os Servidores Gerenciados em um cluster. Se você selecionar essa propriedade, selecione um servidor de Destino.
Observação: Para destinos de tópico disponibilizados em um Servidor Oracle WebLogic Server anterior à versão 10.3.4, a única opção válida é Compatibilidade.
As opções "Uma Cópia" que forem selecionadas para Distribuição de Mensagens de Tópico substituirão a Política de Compartilhamento de Assinatura e/ou a Política de ID de Cliente configurada no JMS Connection Factory.
Destino
Essa opção só está disponível em um cluster do Oracle Service Bus quando você seleciona Compatibilidade para a opção Distribuição de Mensagens de Tópico. Selecione o servidor de destino que irá manipular mensagens JMS de entrada. Se selecionar uma das opções "Uma Cópia" para a Distribuição de Mensagens de Tópico, esse campo exibe o nome do cluster.

Se você não definir um destino, a instância do serviço de proxy JMS (leitura de mensagens fora do Tópico) em cada Servidor Gerenciado no cluster obterá uma cópia da mensagem.
Timeout de JNDI
O timeout da conexão de JNDI (em segundos) usado ao pesquisar o destino ou o factory de conexão na árvore JNDI.
