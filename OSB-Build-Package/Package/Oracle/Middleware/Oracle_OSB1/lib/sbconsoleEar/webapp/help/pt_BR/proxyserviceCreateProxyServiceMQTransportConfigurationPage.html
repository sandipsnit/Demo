Página Configuração do Transporte MQ

A página Criar/Editar um Serviço de Proxy - Configuração do Transporte MQ pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Para obter mais informações, consulte "MQ Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117.

Antes de começar

Configure um recurso de Conexão MQ. Consulte Conexões MQ.

Para configurar o transporte de MQ

Use esta página para configurar as definições de transporte para um serviço de proxy usando o protocolo de transporte MQ nativo. Tabela 19-11 descreve como usar a página.

Tabela 19-11 - Página Criar/Editar um Serviço de Proxy - Configuração do Transporte MQ
Opção
Para criar ou editar...
Intervalo de Sondagem
Informe um intervalo de sondagem, em milissegundos. O default é 1000.
Resposta Obrigatória?
Selecione esta opção para especificar que uma resposta é esperada após o envio de uma mensagem de saída.
Padrão de Correlação de Resposta
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Especifique se o padrão de correlação da resposta deve ser baseado em MessageID ou em CorrelationID.
URI da Resposta MQ
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
O destino no qual a resposta deve ser publicada. Informe um URI de resposta no mesmo formato do URI do ponto final: mq://<local-queue-name>?conn=<mq-connection-resource-ref>
Tipo de Mensagem de Resposta
Esta opção está disponível apenas se a caixa de seleção Resposta Obrigatória? estiver marcada.
Selecione uma das seguintes opções:
  • Bytes (para um fluxo de bytes não-interpretados)
  • Texto (para mensagens de texto)
Política de Distribuição
Selecione uma política de despacho para esse ponto final.
Política de despacho refere-se à instância do Gerenciador de Trabalho do WLS que você deseja usar para o ponto final do serviço. Para obter informações sobre Gerenciadores de Trabalho, consulte "Using Work Managers to Optimize Scheduled Work" no Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=CNFGD112.
Limite de Backout
Informe um valor que represente o número de vezes que o pipeline deve repetir uma mensagem antes de redirecioná-la para a fila especificada no campo URI do Dead Letter.
Se você não especificar um valor para este campo, a mensagem será redirecionada para a fila de dead letter sem nenhuma nova tentativa.
URI do Dead Letter MQ
Informe o URI da fila de dead letter para a qual as mensagens de solicitação devem ser redirecionadas depois de tentar o número especificado de repetições no campo Limite de Backout.
Se você não especificar um valor para este campo, a mensagem será retornada para a fila e ignorada pelo transporte MQ de cada polling após novas tentativas representadas pelo número de vezes especificado no campo Limite de Backout. O URI do Dead Letter usa o mesmo formato do URI do Ponto Final.
Opções `GET' do URI do ponto final
Informe as opções de mensagens GET MQ dentre o seguinte:
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
É possível usar "|" ou "+" para separar várias opções. Por exemplo, você pode especificar o seguinte:
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
As opções de mensagem GET MQ são aplicadas durante a leitura de uma mensagem da fila de entrada.
Processar Cabeçalhos RFH2
Selecione esta opção para fazer parse de cabeçalhos RFH2 do WebSphere MQ de um payload de mensagem e gerar automaticamente um cabeçalho de transporte RFH2Headers contendo os dados RFH2.
Se você não selecionar esta opção, o payload será transmitido conforme tiver sido recebido.

Para obter informações sobre como o transporte MQ lida com cabeçalhos RFH2, consulte "About RFH2 Headers" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136.
