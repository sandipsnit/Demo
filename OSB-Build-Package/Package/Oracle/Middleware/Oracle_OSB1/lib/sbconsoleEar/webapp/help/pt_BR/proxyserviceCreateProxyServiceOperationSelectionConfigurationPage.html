Página Configuração da Seleção de Operação

A página Criar/Editar um Serviço de Proxy - Configuração da Seleção de Operação pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy. Esta página só será exibida se o serviço que você está criando tiver operações.

Use esta página para impor a conformidade WS-I (apenas para serviços SOAP 1.1) e escolha o algoritmo de seleção a ser usado para determinar a operação chamada por este serviço de proxy. Essa opção só está disponível para os serviços SOAP ou XML definidos com base em um WSDL.

A especificação WSDL define um algoritmo default para calcular qual operação é chamada com base no tipo da mensagem SOAP recebida. No entanto, há casos (por exemplo, problemas de desempenho, problemas de assinatura/criptografia ou o algoritmo default não é aplicável) em que pode ser necessário selecionar a operação com base em outros meios.

O Oracle Service Bus oferece algoritmos adicionais. Cada um deles segue o mesmo padrão e é baseado na avaliação de uma expressão para obter um valor que é usado para pesquisar a operação correspondente em uma tabela estática.

Geralmente, o Oracle Service Bus é bastante flexível·se uma mensagem de entrada estiver com dados faltando de forma que a operação não possa ser determinada ou se tiver dados que não correspondam a uma operação válida. As duas condições fazem com que $operation fique vazia. Em vez de rejeitar todas essas mensagens, o Oracle Service Bus não inicializa a variável da operação no contexto, mas continua para processar a mensagem.

No entanto, os requisitos de segurança serão impostos se o serviço de proxy for baseado no WSDL e se pelo menos uma das seguintes condições for verdadeira:

  • O WSDL possui uma política de Segurança de Web Services e o proxy é um intermediário ativo.
  • O proxy possui autenticação personalizada de mensagem (token ou nome de usuário/senha personalizado).

Se estas condições forem atendidas, haverá uma verificação do run-time para garantir que o algoritmo de seleção da operação retorne um nome de operação válido. Se a seleção da operação retornar nula ou uma operação que não esteja no WSDL, a mensagem será rejeitada e um erro será emitido.

Impor Conformidade WS-I
Apenas para serviços SOAP 1.1:
Marque esta caixa de seleção se quiser especificar se ou não o serviço está de acordo com o Perfil Básico definido pela Organização de Interoperabilidade Web Services.
Quando um serviço é marcado como compatível com WS-I, verificações serão executadas nas mensagens enviadas para e desse serviço. Para proxies, verificações são executadas nas mensagens recebidas pelo proxy. Para serviços chamados (ou seja, serviços chamados por um proxy por meio de uma ação de chamada do serviço ou nó de rota), verificações são executadas nas mensagens de resposta recebidas desses serviços. Observe que é a propriedade de conformidade WS-I do serviço chamado, e não o proxy, que determina se ou não as verificações são executadas nas mensagens recebidas do serviço chamado. Se você especificar o teste de conformidade WS-I para um serviço chamado, o fluxo de mensagens gerará uma falha para erros de resposta.
Algoritmo de Seleção
Selecione uma das seguintes opções e execute as etapas adicionais necessárias:
  • Cabeçalho de Transporte - Selecione este algoritmo para definir o cabeçalho de transporte que contém o valor de pesquisa. Em seguida:
    • No campo Nome do Cabeçalho, informe o cabeçalho de transporte que extrai o valor usado como uma chave para selecionar a operação sendo chamada.
    • No campo Mapeamento da Operação, especifique o valor para cada operação no campo Valor. O valor é usado como a chave da operação.
  • SOAPAction Header - Selecione este algoritmo para especificar que o mapeamento da operação deve ser feito automaticamente a partir do WSDL associado a este serviço de proxy.
  • Endereçamento WS - Selecione este algoritmo para especificar que o valor de pesquisa é contido pela tag Ação do Endereçamento WS localizado nos cabeçalhos SOAP da mensagem SOAP. Então, no campo Mapeamento da Operação, especifique o valor para cada operação no campo Valor. O valor é usado como a chave da operação.
  • Cabeçalho SOAP - Selecione este algoritmo para definir uma expressão XPath a ser avaliada com os cabeçalhos SOAP. Isso permite obter o valor de pesquisa. Em seguida:
    • No campo Expressão XPath, especifique a expressão XPath que extrai o valor usado como uma chave para selecionar a operação sendo chamada.
    • No campo Mapeamento da Operação, especifique o valor para cada operação no campo Valor. O valor é usado como a chave da operação.
  • Tipo de Corpo SOAP - Esse é o algoritmo default definido pela especificação WSDL para calcular qual operação é chamada com base no tipo da mensagem SOAP recebida.

    • Se o serviço de proxy for configurado para um cenário de passagem de segurança do Web Service com um corpo criptografado, não será possível selecionar este algoritmo. Uma advertência semelhante aplica-se aos cabeçalhos SOAP criptografados da passagem.

    Se você tiver um WSDL que possua duas operações com a mesma mensagem de entrada, não selecione este algoritmo para operações, porque a operação não pode ser identificada exclusivamente inspecionando a mensagem de entrada.

  • Tipo de Payload - Disponível apenas para serviços XML com base em uma porta WSDL ou binding WSDL.
Nome do Cabeçalho
Esta opção só está disponível quando a opção Algoritmo de Seleção é definida como Cabeçalho de Transporte.
Informe o cabeçalho de transporte que extrai o valor usado como uma chave para selecionar a operação que está sendo chamada.
Expressão do XPath
Esta opção só está disponível quando a opção Algoritmo de Seleção estiver definida como SOAPHeader.
Especifica a expressão XPath que extrai o valor usado como uma chave para selecionar a operação sendo chamada.
Mapeamento da Operação
Esta opção só está disponível quando a opção Algoritmo de Seleção estiver definida como Cabeçalho de Transporte, Endereçamento WS ou Tipo de Corpo SOAP.
Especifique o valor para cada operação no campo Valor. O valor é usado como a chave da operação.
