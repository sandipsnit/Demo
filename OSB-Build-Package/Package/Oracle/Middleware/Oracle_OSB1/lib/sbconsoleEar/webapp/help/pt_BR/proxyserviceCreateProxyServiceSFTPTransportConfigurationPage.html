Página Configuração do Transporte SFTP

A página Criar/Editar um Serviço de Proxy - Configuração do Transporte SFTP pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Para obter mais informações, consulte "HTTP and Poller Transports" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV947.

Use esta página para configurar as definições de transporte para um serviço de proxy usando o protocolo de transporte SFTP. Tabela 19-13 descreve como usar esta página.

Tabela 19-13 - Página Criar/Editar um Serviço de Proxy - Configuração do Transporte SFTP
Opção
Para criar ou editar...
Autenticação do Usuário
Selecione uma das seguintes opções:
  • Autenticação de Nome de Usuário e Senha - Especifica se uma conta de serviço estática está associada a este método de autenticação e se o cliente é autenticado usando as credenciais fornecidas.
  • Autenticação Baseada em Host - Especifica se um nome de usuário e um provedor de chaves de serviço devem usar esse método de autenticação. Qualquer usuário conectando-se de um host conhecido será autenticado usando a chave privada do host.
  • Autenticação de Chave Pública - Especifica se um nome de usuário e um provedor de chaves de serviço são necessários para uso desse método de autenticação. Cada usuário possui sua própria chave privada.
Conta do Serviço
Informe a conta de serviço para o usuário ou clique em Procurar para selecionar contas a partir de um browser.
Provedor de Chaves de Serviço
Esta opção só está disponível quando Autenticação Baseada no Host ou Autenticação de Chave Pública é selecionado.
Informe um provedor de chaves de serviço no campo Provedor de Chaves de Serviço. Você pode clicar em Procurar para selecionar provedores de chaves de serviço a partir de um browser. Esse campo é obrigatório.
Nome do Usuário
Esta opção só está disponível quando Autenticação Baseada no Host ou Autenticação de Chave Pública é selecionado.
Informe o nome de usuário.
Passar por Referência
Marque esta caixa de seleção para separar o arquivo no diretório de archive e transmiti-lo como uma referência nos cabeçalhos.
Fluxo Remoto
Marque esta caixa de seleção para fazer stream dos arquivos SFTP diretamente do servidor remoto no momento do processamento. Quando você seleciona esta opção, o diretório de archive é o diretório remoto na máquina remota do servidor SFTP. Portanto, você deve especificar o diretório de archive em relação ao diretório de usuário SFTP.
Máscara do Arquivo
Informe a expressão regular para os arquivos a serem selecionados. O default é *.*.
Servidor Gerenciado
Este campo só é exibido em um domínio em cluster.
Selecione o Servidor Gerenciado para atuar como o servidor de sondagem. Todos os Servidores Gerenciados podem processar a mensagem, mas apenas um pode sondar a mensagem.
Intervalo de Sondagem
Informe o intervalo em segundos em que o arquivo é agrupado do local especificado. O default é 60.
Limite de Leitura
Especifique o número máximo de mensagens a serem lidas por sweep de polling. Informe 0 para especificar nenhum limite. O default é 10.
Ação Pós-Leitura
Selecione o que acontece a uma mensagem depois que é lida.
  • Arquivar - A mensagem é arquivada.
  • Deletar - A mensagem é deletada.
Diretório de Archive
Especifique o caminho para o local de archive se a opção Ação Pós-Leitura for definida como Archive. Esse campo é obrigatório se a opção Passar por Referência for selecionada.
Observação: Os diretórios de Archive, Download e Erro são caminhos absolutos e são criados automaticamente. Se você especificar um caminho relativo, os arquivos serão criados em relação ao processo Java que inicia o WebLogic Server.
Diretório de Download
Informe o diretório na máquina local para o qual os arquivos são submetidos a download durante a transferência de arquivos.
Observação: Os diretórios de Archive, Download e Erro são caminhos absolutos e são criados automaticamente. Se você especificar um caminho relativo, os arquivos serão criados em relação ao processo Java que inicia o WebLogic Server.
Diretório de Erros
Informe o local em que as mensagens são postadas se houver um problema.
Observação: Os diretórios de Archive, Download e Erro são caminhos absolutos e são criados automaticamente. Se você especificar um caminho relativo, os arquivos serão criados em relação ao processo Java que inicia o WebLogic Server.
Codificação da Solicitação
Aceite o default UTF-8 como a codificação de conjunto de caracteres para solicitações em transportes SFTP.
Conjunto de Cifras Preferencial
Selecione o conjunto de cifras a ser usado ao estabelecer comunicação com o servidor.
Se você usar o valor default, Utilizar Runtime Default, a lista de conjuntos de cifras suportados será enviada ao servidor e cada um será tentado até que haja uma correspondência.
Algoritmo de Integridade de Dados Preferencial
Selecione o algoritmo de bulk-hashing para verificações de integridade de dados.
Se você usar o valor default, Usar Default de Runtime, o Oracle Service Bus enviará o algoritmo preferencial hmac-sha1 ao servidor. Se essa opção não for suportada pelo servidor, a lista de algoritmos suportados será enviada ao servidor, e cada um deles será testado até que um valor correspondente seja encontrado.
Algoritmo de Chave Pública Preferencial
Selecione o algoritmo de chave assimétrica para a criptografia de chave pública.
Se você usar o valor default, Usar Default de Runtime, o Oracle Service Bus enviará o algoritmo preferencial ssh-dss ao servidor. Se essa opção não for suportada pelo servidor, a lista de algoritmos suportados será enviada ao servidor, e cada um deles será testado até que um valor correspondente seja encontrado.
Algoritmo de Intercâmbio de Chave Preferencial
Selecione o protocolo de intercâmbio de chave default para negociar a chave de sessão para criptografar a mensagem.
Se você usar o valor default, Usar Default de Runtime, o Oracle Service Bus enviará o algoritmo preferencial diffie-hellman-group1-sha1 ao servidor. Se essa opção não for suportada pelo servidor, a lista de algoritmos suportados será enviada, e cada um deles será testado até que um valor correspondente seja encontrado.
Algoritmo de Compactação Preferencial
Selecione se quiser compactar dados automaticamente usando zlib. Selecione zlib para compactar dados; caso contrário, selecione Nenhum. O default é Nenhum.
Verificar Subdiretórios
Marque esta caixa de seleção para verificar recursivamente todos os diretórios
Classificar por Chegada
Marque esta caixa de seleção para enviar eventos na ordem de chegada.
Timeout
Informe o intervalo de timeout do soquete, em segundos, antes que a conexão seja eliminada. Se você informar 0, não haverá timeout. O valor default é 60.
Contagem de Repetições
Especifique o número de repetições para falhas de conexão SFTP.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Proxy - Resumo.
