Página Configuração do Transporte de Tuxedo

A página Criar/Editar um Serviço de Proxy - Configuração do Transporte de Tuxedo pertence a uma série de páginas para criação e edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Para obter mais informações, consulte "Tuxedo Transport" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200.

Use esta página para configurar as definições de transporte para um serviço de proxy usando o protocolo de transporte de Tuxedo. Tabela 19-14 descreve como usar a página.

Tabela 19-14 - Página Criar/Editar um Serviço de Proxy - Configuração do Transporte de Tuxedo
Opção
Para criar ou editar...
Classes da Tabela de Campo
Informe o nome da classe ou classes que descrevem o buffer FML/FML32 recebido. São usadas nas rotinas de conversão FML/FML32 para XML para mapear os nomes de campo para nomes de elemento. Essa é uma lista separada por espaços de nomes de classe totalmente qualificados.
Exibindo Classes
Informe o nome da classe ou classes que descrevem o buffer VIEW/VIEW32 recebido ou enviado. São usadas nas rotinas de conversão VIEW para XML ou VIEW32 para XML para mapear os nomes de campo para nomes de elemento. Essa é uma lista separada por espaços de nomes de classe totalmente qualificados.
Os tipos de buffer de Tuxedo X_C_TYPE e X_COMMON são controlados da mesma maneira como os buffers VIEW/VIEW32.
Se uma solicitação de entrada contiver um VIEW, então a classe VIEW correspondente deverá ser especificada no CLASSPATH do Oracle Service Bus.
Jar de Classes
Selecione um recurso JAR que contenha um arquivo JAR com as classes FML/FML32 ou VIEW/VIEW32 necessárias para esta operação do ponto final.
Ponto de Acesso Local
Selecione um ponto de aceso local na lista drop-down associada à exportação. A lista drop-down contém pontos de acesso local configurados no WTC. Um serviço de proxy não pode ser criado se não houver um ponto de acesso local associado.
Se nenhum ponto de acesso local existir ou para criar um novo, selecione Novo. Informe o Nome do Ponto de Acesso Local e o Endereço de Rede Local correspondentes nos campos adjacentes. Na validação do ponto final, o ponto de acesso é adicionado à configuração do WTC para cada servidor WTC. Se nenhum servidor WTC existir, um será criado.
É possível informar um nome de ponto de acesso existente após a seleção da opção Novo. Isso faz com que as informações existentes sejam atualizadas com os novos parâmetros. É possível alterar apenas o nome do host e o número da porta.
Ponto de Acesso Remoto
Este campo só aparece quando você seleciona Novo no campo Ponto de Acesso Local.
Na lista drop-down, selecione um ponto de acesso remoto a ser associado ao ponto de acesso local recém-criado. Se nenhum existir ou para criar um novo, selecione Novo. Informe o Nome do Ponto de Acesso Local e o Endereço de Rede correspondentes nos campos adjacentes.
É possível informar um nome de ponto de acesso existente após a seleção da opção Novo. Isso faz com que as informações existentes sejam atualizadas com os novos parâmetros. É possível alterar apenas o nome do host e o número da porta.
O ponto de acesso remoto também será a autenticação principal para a conexão WTC para solicitações de entrada. Como opção, é possível criar um usuário com o mesmo ID do ponto de acesso no realm de segurança default para permitir chamadas de entrada. Para isso, selecione Sim na lista drop-down Criar Usuário?. A senha será gerada aleatoriamente usando uma variável temporária para evitar problemas de segurança.
Tipo de Buffer da Resposta
Esta opção está disponível apenas se o campo Resposta Obrigatória? for selecionado.
Selecione o tipo de buffer que o cliente remoto de Tuxedo receberá.
Subtipo de Buffer da Resposta
Esta opção só está disponível quando a opção Resposta Obrigatória? é selecionada e o valor de Tipo de Buffer de Resposta é VIEW ou VIEW32.
Informe o subtipo do buffer ao qual associar o buffer de resposta.
Resposta Obrigatória?
Marque esta caixa se for esperado que este serviço envie uma resposta.
O status default é que essa opção esteja selecionada.
Esta opção está desmarcada e indisponível se o tipo de serviço for Serviço de Mensagens e o tipo de mensagem de resposta for Nenhum.
Codificação da Solicitação
Especifique uma codificação de conjunto de caracteres para solicitações em transportes do Tuxedo.
Codificação da Resposta
Especifique uma codificação de conjunto de caracteres para respostas em transportes do Tuxedo.
Estilo de Transformação
Selecione uma das seguintes opções:
  • Nenhum - (default) A ordem dos campos pode não ser respeitada.
  • Ordenado - Os campos são apresentados com todas as suas ocorrências na ordem correta.
  • Ordenado e Agrupado - Se os campos forem estruturados logicamente como registros, os campos serão ordenados por ocorrência e agrupados por registro.

Depois de finalizar

Clique em Próximo para continuar configurando este serviço na próxima página; ou clique em Último para rever e salvar essa configuração na página Criar/Editar um Serviço de Proxy - Resumo.
