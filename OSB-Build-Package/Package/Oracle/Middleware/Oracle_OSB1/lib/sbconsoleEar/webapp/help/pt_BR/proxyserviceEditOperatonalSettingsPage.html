Configurando Definições Operacionais para Serviços de Proxy

Use a página Definições Operacionais para ativar e desativar um serviço de proxy e monitoramento para serviços de proxy específicos. Da mesma forma, você pode especificar essas mesmas definições operacionais específicas ao serviço na página Operações > Pesquisa Inteligente, com as seguintes exceções: não é possível definir um intervalo de agregação ou especificar um nível de severidade de alerta ou de log. Para obter mais informações, consulte Localizando e Atualizando Definições Operacionais.

Os efeitos do run-time das definições no nível do serviço dependem das definições globais correspondentes. Você deve ativar as definições globais e no nível do serviço para que um serviço seja completamente ativado no run-time. Além disso, o Estado do Serviço também deve ser ativado. Consulte Ativando Definições Globais.

É possível alterar e salvar as definições de configuração do monitoramento mesmo que o serviço não seja ativado no run-time. Por exemplo, é possível alterar e salvar o Intervalo de Agregação mesmo que o Monitoramento do Serviço esteja desativado. Dessa maneira, você pode editar as definições e ativá-las posteriormente.

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione Resource Browser > Serviços de Proxy.
  3. Clique no nome do serviço de proxy.

    4. A página Exibir um Serviço de Proxy exibe as informações mostradas na Tabela 25-6.

    Definir um Intervalo de Agregação para um serviço
    Selecione o intervalo em termos de horas ou minutos, depois clique em Atualizar. Se sua seleção para horas exceder 1, então a seleção default para minutos será sempre zero. No entanto, se sua seleção para horas for 0 ou 1, então você poderá configurar intervalos em termos de minutos. Consulte Definindo o Intervalo de Agregação para um Serviço.

    Marque a caixa de seleção Ativado. Na lista drop-down Ativar Alerta no nível, selecione para restringir os alertas de SLA ao nível especificado ou acima (listado do nível mais inclusivo para o nível mais restritivo): Normal (default), Advertência, Menor, Maior, Crítico e Fatal.
    O alerta de SLA de serviço depende dos estados de monitoramento global e de serviço; ambos devem ser ativados para que o alerta de SLA seja ativado no run-time. Consulte Ativando Alertas de SLA Globalmente.
    Desativar Alertas do Pipeline

    Marque a caixa de seleção Ativado.     Ativar a ação de relatório de um serviço de proxy
    Marque a caixa de seleção Ativado ao lado de Relatórios de Mensagens.
    Desativar a saída de log para uma ação de log do pipeline
    Desmarque a caixa de seleção Ativado ao lado de Relatórios. Quando você ativa a sessão, o sistema para de executar ações de log; portanto, você não recebe mais saída de log associada ao serviço.
    Ativar a saída de log para uma ação de log do pipeline
    Marque a caixa de seleção Ativado ao lado de Log. Na lista drop-down Ativar Alerta no nível, escolha para restringir a saída de log ao nível especificado ou acima (listado do nível mais inclusivo para o nível mais restritivo): Depuração (default), Info, Advertência e Erro.
    Ativar rastreamento de execução de runtime para um serviço de proxy

    Marque a caixa de seleção Ativado ao lado de Rastreamento da Execução. Depois que você ativa o rastreamento da execução, o sistema registra diversos detalhes coletados do Contexto do Fluxo de Mensagens e do Contexto da Mensagem. Esses detalhes incluem: nome do estágio; nome do pipeline ou do nó de rota e o Contexto da Mensagem atual.

    Observação: Para ver o rastreamento no arquivo de log ou na saída padrão (console do servidor), o log do Oracle WebLogic Server deve ser definido com os seguintes níveis de severidade:

    Severidade mínima para registro: Info
    Arquivo de log: Info
    Saída padrão: Info

    Para obter informações sobre a definição de níveis de severidade do log, consulte "Using Log Severity Levels" em Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

    Ativar rastreamento de run-time para um serviço de proxy
    Desmarque a caixa de seleção Ativado ao lado de Rastreamento da Execução.
    Ativar rastreamento da mensagem para um serviço de proxy

    Marque a caixa de seleção Ativado ao lado de Rastreamento da Mensagem. Depois que você ativa o rastreamento da mensagem, o sistema registra as mensagens trocadas entre o pipeline e o serviço de negócios do Oracle Service Bus (mensagens de solicitação e de resposta de entrada e de saída).

    Observação: Para ver o rastreamento no arquivo de log ou na saída padrão (console do servidor), o log do Oracle WebLogic Server deve ser definido com os seguintes níveis de severidade:

    Severidade mínima para registro: Info
    Arquivo de log: Info
    Saída padrão: Info

    Para obter informações sobre a definição de níveis de severidade do log, consulte "Using Log Severity Levels" em Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

    Observe que, quando aplicável, as mensagens de saída registradas também podem incluir o número de repetições, o código do erro e a mensagem de erro.
    Na lista drop-down Nível de Detalhe, escolha para especificar o nível de detalhe entre as opções a seguir:
    • Conciso - Exibe a data, a hora, o tipo de serviço, o nome do serviço e o URI
    • Cabeçalhos - Exibem informações concisas junto com a representação XML dos metadados da mensagem
    • Completo - Exibe as informações dos cabeçalhos junto com o payload bruto, incluindo anexos, se houver
    Se você escolher Completo na lista drop-down Nível de Detalhe, especifique o tamanho máximo (em kilobytes) para o payload da mensagem usando o campo Limite de Rastreamento de Payload. Além disso, especifique a codificação default para log do payload usando o campo Codificação Default. Isso pode ser útil ao registrar payloads binários ou mensagens SOAP com anexos binários.
    O valor de codificação default pode ser Base64 ou qualquer codificação suportada por Java, conforme definido em: http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html.
    Deixar o campo Codificação Default vazio faz com que o Oracle Service Bus use a codificação default do host para o payload. A codificação default depende de uma combinação da JVM, do sistema operacional (SO) subjacente e das definições de configuração regional no nível do SO.
    Se a definição especificada no campo Codificação Default não puder ser usada (por exemplo, não for uma opção válida para a configuração), então o Oracle Service Bus usará a codificação Base64 para o payload.
