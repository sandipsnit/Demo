Página Política do Serviço de Proxy

A página Exibir um Serviço de Proxy - Políticas pertence a uma série de páginas para edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Use esta página para configurar definições de política para um serviço proxy com um tipo de serviço Web Service WSDL, Serviço de Mensagem, Qualquer Serviço SOAP ou Qualquer Serviço XML.A Tabela 19-20 descreve como usar a página. Para políticas de OWSM a serem utilizadas com Web Service de WSDL, Serviço de Mensagens ou qualquer serviço de proxy do Serviço XML, o protocolo deve ser definido como HTTP.

Para serviços baseados no WSDL, as políticas do WLS 9.2 vinculadas ao serviço são expostas (alinhadas) no WSDL efetivo. Políticas abstratas são pré-processadas antes de serem alinhadas. As políticas do OWSM são vinculadas por referência, e não alinhadas no WSDL efetivo.

Configuração da Política de Serviço
A seguir estão as opções para o campo Configuração da Política de Serviço:
  • Da Área de Armazenamento de Política do OWSM – As políticas são definidas na Área de Armazenamento de Política central do Oracle Web Services Manager gerenciado pelo Oracle Enterprise Manager. Elas suportam WS-Security 1.0/1.1, SAML 1.1/2.0, KerberosToken Profile e outros padrões da indústria.

    Use filtro, se necessário, para localizar as políticas desejadas. Em serviços de proxy, só são exibidas políticas de serviço.

    • Para obter mais informações, consulte "Securing Oracle Service Bus with Oracle Web Services Manager" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

  • Do WSDL – Selecione esta opção se a política de serviço for associada ao WSDL em que o serviço se baseia. Essas políticas suportam WS-Security 1.0, SAML 1.1 e outros padrões da indústria.

    • Com esta opção, é possível exibir (somente para leitura) políticas de solicitação e de resposta do WSDL.

  • Da Política Pré-definida ou do Recurso de Web Services em que você adiciona políticas de nível de serviço, de nível de operação (em cujo caso a política se aplica às mensagem de solicitação e de resposta), de solicitação e de resposta da Console do Oracle Service Bus Administration.

    As políticas são predefinidas no Oracle WebLogic Server ou definidas pelo usuário no Oracle Service Bus com um recurso de Política de Web Services. Essas políticas suportam WS-Security 1.0, SAML 1.1 e outros padrões da indústria.

    • Com esta opção, é possível adicionar políticas nos níveis de serviço, operação, solicitação e resposta.

Observação: Os modelos de binding de política são mutuamente exclusivos. Você deve usar apenas um tipo de política em um serviço. Se você efetuar bind de políticas diretamente para o serviço, todas as políticas baseadas no WSDL serão ignoradas.

Depois de finalizar

Clique em Atualizar para salvar esta configuração, ou clique em Redefinir para desfazer suas alterações.
