Página Segurança do Serviço de Proxy

A página Exibir um Serviço de Proxy - Segurança pertence a uma série de páginas para edição de serviços de proxy, conforme descrito em Criando e Configurando Serviços de Proxy e Editando Configurações do Serviço de Proxy.

Use esta página para configurar as definições de segurança para um serviço de proxy.

Os campos disponíveis nesta página dependem do uso de políticas no serviço de proxy. Por exemplo, se o serviço de proxy usar políticas do OWSM (recomendado), Substituições de Política aparecerão na página Segurança. Para obter mais informações, consulte "Securing Oracle Service Bus with Oracle Web Services Manager" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681.

Para políticas do WLS 9.2, a configuração para o nome de usuário/senha personalizado e para o token personalizado é semelhante. Em ambos os casos, você especifica expressões XPath que permitem que o Oracle Service Bus localize as informações necessárias. A raiz destas expressões XPath é a seguinte:

  • Use soap-env:Envelope/soap-env:Header se o binding de serviço for AnySOAP ou WSDL-SOAP.
  • Use soap-env:Body se o binding de serviço não for baseado em SOAP.

Todas as expressões XPath devem estar em um formato XPath válido. As expressões XPath devem usar a sintaxe "declare namespace" do XPath para declarar os namespaces usados, conforme a seguir:

declare namespace
ns='http://webservices.example.com/MyExampleService';)

Configuração da Segurança de Web Services
Processar Cabeçalho de Segurança de Web Services
Se uma solicitação de cliente incluir um cabeçalho de Segurança de Web Services, decida se ou não processar este cabeçalho:
  • Selecionar Sim: Em um cenário intermediário ativo, o cliente aplica a Segurança de Web Services às mensagens de solicitação e/ou de resposta. O serviço de proxy processa o cabeçalho de segurança e aplica a política de Segurança de Web Services. Essa opção é definida automaticamente em Sim quando um serviço proxy é baseado em um WSDL que possua uma política de segurança ou quando anexar uma política ao serviço na página Configuração da Política do Serviço.
  • Selecionar Não: Em um cenário de passagem de Segurança de Web Services, o cliente aplica a Segurança de Web Services às mensagens de solicitação e/ou de resposta. O serviço de proxy não processa o cabeçalho de segurança. Em vez disso, transmite a mensagem de solicitação protegida inalterada para um serviço de negócios.
Controle de Acesso
Controle de Acesso do Transporte
Para todos os serviços de proxy, você pode criar uma política de transporte, que aplica uma verificação de segurança quando um cliente tenta estabelecer uma conexão com o serviço de proxy. Apenas solicitações de usuários que estejam listados na política de transporte têm permissão para prosseguir.
Controle de Acesso à Mensagem
Uma política de controle de acesso no nível da mensagem aplica uma verificação de segurança quando um cliente tenta chamar um serviço de proxy com segurança no nível da mensagem. Você pode criar uma política de controle de acesso no nível da mensagem nos seguintes casos:
  • Para serviços de proxy que são intermediários de segurança de Web Services ativos; ou seja, você processa o cabeçalho de Segurança de Web Services.
  • Para serviços de proxy aos quais você pode definir autenticação personalizada nesta página.
Apenas usuários listados na política de mensagem têm permissão para chamar a operação.
Autenticação Personalizada
(Nível de Mensagem)
Tipo de Autenticação
Selecione uma das seguintes opções:
  • Selecione Nenhum se o serviço de proxy não for usar autenticação personalizada.
  • Nome do Usuário e Senha Personalizados

    • Ao selecionar esta opção, também é necessário informar valores no campo XPath do Nome do Usuário e no campo XPath da Senha do Usuário.

  • Token Personalizado

    • Ao selecionar esta opção, também é necessário selecionar um Tipo de Token e informar valores para o campo XPath do Token.

XPath do Nome do Usuário
Esta opção só está disponível quando a opção Definições de Autenticação Personalizadas está definida como Nome do Usuário e Senha Personalizados. Quando disponível, essa opção é obrigatória.
Informe o nome de usuário como uma expressão XPath.
A expressão XPath é avaliada com os cabeçalhos ou payload da mensagem, conforme apropriado, que permitem que o Oracle Service Bus obtenha o nome de usuário e para autenticação personalizada.
XPath da Senha do Usuário
Esta opção só está disponível quando a opção Definições de Autenticação Personalizadas está definida como Nome do Usuário e Senha Personalizados. Quando disponível, essa opção é obrigatória.
Informe a senha como uma expressão XPath.
A expressão XPath é avaliada com os cabeçalhos ou payload da mensagem, conforme apropriado, que permitem que o Oracle Service Bus obtenha os valores de senha para autenticação personalizada.
Tipo de Token
Esta opção só está disponível quando a opção Definições de Autenticação Personalizadas está definida como Token Personalizado. Quando disponível, essa opção é obrigatória.
Selecione o tipo de token na lista drop-down. Apenas os tipos de token ativos configurados para um provedor de Asserção de Identidade do WebLogic estão disponíveis. Consulte "Configuring Identity Assertion Providers for Custom Tokens" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660.
XPath do Token
Esta opção só está disponível quando a opção Definições de Autenticação Personalizadas está definida como Token Personalizado. Quando disponível, essa opção é obrigatória.
Informe uma expressão XPath para especificar um caminho para o token personalizado. O Oracle Service Bus avaliará a expressão XPath do Token com os cabeçalhos ou payload da mensagem, conforme apropriado, para obter o token para autenticação personalizada.
Propriedades do Contexto
Como opção, especifique uma ou mais propriedades de contexto para transmitir informações adicionais de contexto para o provedor de segurança Autenticação (Nome de Usuário e Senha Personalizado) ou Asserção de Identidade (Token Personalizado).
Propriedades de Contexto fornecem uma forma (a interface ContextHandler) de transmitir informações adicionais para o WebLogic Security Framework de forma que um provedor de segurança possa obter informações contextuais. Consulte "Context Properties Are Passed to Security Providers" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500.
Informe o Nome da Propriedade como uma string literal, e o Seletor de Valor como uma expressão XPath válida. (Expressões XPath também podem ser strings literais.)
As expressões XPath são avaliadas com a mesma parte da mensagem usadas para o token personalizado ou nome de usuário/senha personalizado. Ou seja, as expressões XPath do Seletor de Valor para serviços de proxy baseados no SOAP são avaliadas com o cabeçalho e com o payload para serviços de proxy baseados em não-SOAP.
A expressão XPath é avaliada no run-time para produzir o valor da propriedade. Um ContextHandler é essencialmente uma lista de nome/valor e, portanto, exige que um provedor de segurança saiba quais nomes procurar. Portanto, as expressões XPath são só avaliadas se um provedor de segurança solicitar o valor de uma dessas propriedades definidas pelo usuário.
Clique em Adicionar Propriedade para adicionar essa propriedade de contexto. É possível adicionar várias propriedades de contexto.
