Exibindo Mensagens de Relatórios

Use a página Resumo do Relatório de Mensagens para exibir informações de mensagens de relatórios mostradas na Tabela 27-2. A página exibe até 100 mensagens.

  1. Para restringir o número de itens na lista ou para localizar mensagens específicas:
    • Clique no ícone Abrir para exibir filtros de pesquisa adicionais. Se você especificar vários critérios de pesquisa, apenas os resultados que correspondem a todos os critérios serão retornados.
    • Para definir os parâmetros de filtro, execute uma ou mais das seguintes etapas mostradas na Tabela 27-1.

      • Tabela 27-1 Parâmetros do Filtro de Mensagens 
      Para...
      Execute Estas Etapas...
      Filtrar por parâmetros de tempo
      Defina um período:
      1. Selecione o botão de rádio Mensagens de... para...
      2. No campo Mensagens de, informe uma data e hora no formato MM/DD/AA HH:MM:SS AM|PM. Por exemplo, é possível informar 10/10/07 12:45:00 AM.
      3. No campo Mensagens para, informe uma data e hora no formato MM/DD/YY HH:MM:SS AM|PM.
      Como alternativa, é possível selecionar parâmetros para as mensagens mais recentes:
      1. Selecione o botão de rádio Para o Último
      2. No campo Para o Último, informe o número de dias, depois especifique as horas e os minutos.
      Observação: Por default, os relatórios de mensagens dos últimos 30 minutos são exibidos.
      Filtrar por nome do serviço
      No campo Nome do Serviço de Entrada, informe o nome do serviço.
      Filtrar por código do erro
      No campo Código do Erro, informe um código de erro.
      Filtrar por índice do relatório
      No campo Índice do Relatório, informe os índices do relatório. São os pares de valor de chave definidos durante a configuração da ação de relatório para este serviço de proxy. Esta é uma pesquisa de string.

    • É possível usar caracteres curingas nas pesquisas dos campos nome do serviço, código do erro e índice do relatório.
      • Informar apenas asterisco (*) retorna todas as mensagens. Por exemplo, se você informar * no campo do código de erro, mensagens com e sem códigos de erro serão exibidas.
      • No campo Código do Erro apenas, limitações de pesquisa estritas são aplicáveis. Por exemplo, para localizar mensagens com o código do erro BEA12345, é possível informar *, BE* ou BEA12345.
  2. Clique em Pesquisar. A página exibirá as mensagens que se enquadram aos parâmetros do filtro definidos. O número máximo de mensagens que podem ser exibidas ao filtrar mensagens é 1000.
  3. Clique em Exibir Tudo para remover os filtros de pesquisa e exibir todas as mensagens.
  4. Para localizar uma mensagem, você também pode fazer o seguinte:
    • Classificar a lista de mensagens novamente. Clique em um nome de coluna sublinhado. Setas crescentes e decrescentes indicam a ordem de classificação. Clique no nome da coluna para alterar a ordem de classificação.
    • Role pelas páginas. Use os controles de página acima ou abaixo da tabela. Para ir para uma página, selecione o número da página ou use os botões de seta para ir para a próxima página, página anterior, primeira ou última página.

      • Tabela 27-2 - Detalhes do Resumo de Mensagens (Relatórios)
      Propriedade
      Descrição
      Índice do Relatório
      Um link para a página Exibir Detalhes da Mensagem. Consulte Exibindo Detalhes da Mensagem.
      Índices de relatório são pares de valor de chave usados para extrair identificadores-chave de uma variável de contexto da mensagem. Chaves são uma forma conveniente de filtrar mensagens. Use a ação do relatório para adicionar pares de valor de chave. Consulte Adicionando e Editando Ações em Fluxos de Mensagens.
      Timestamp do BD
      A data e hora em que a mensagem foi adicionada ao banco de dados.
      Serviço de Entrada
      O serviço de entrada associado à mensagem. O serviço é um link para a página Exibir um Serviço de Proxy - Detalhes da Configuração. Consulte Editando Configurações de Serviço de Proxy.
      Código do Erro
      O código do erro associado a esta mensagem, se houver.
      Especifique códigos de erro ao configurar a ação de emissão de erro. Consulte "Handling Errors in Message Flows" no Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG232 e em Adicionando Handlers de Erro do Serviço de Proxy.
