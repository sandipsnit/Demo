Adicionando Destinos de Alerta

Um destino de alerta é um endereço de destino para notificações de alerta no Oracle Service Bus. É possível criar recursos do destino de alerta na página Criar Destino de Alerta.

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione o Project Explorer, depois selecione um projeto ou pasta no qual adicionar o destino de alerta. A página View de Projeto/Pasta é mostrada.
  3. Na lista drop-down Criar Recurso, selecione Destino de Alerta para exibir a página Criar Destino de Alerta.
  4. No campo Nome do Recurso, informe um nome exclusivo para esse destino de alerta. Esse campo é obrigatório.

    Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
  5. No campo Descrição do Recurso, informe uma descrição para o destino de alerta.
  6. Selecione qualquer ou todos os destinos a seguir a serem incluídos neste recurso de destino do alerta.
    7. Observação: São agregados alertas na Console do Oracle Service Bus Administration no runtime, onde você pode exibi-los em Operações > Painel de Controle.
    • Trap do SNMP: Se você especificar Trap do SNMP, os alertas serão enviados como traps de SNMP e poderão ser processados por qualquer sistemas de monitoramento corporativo (ESM, Enterprise Monitoring Systems) de terceiros.
    • Relatórios: Se você especificar Relatórios, os alertas serão enviados para o módulo Relatórios do Oracle Service Bus e poderão ser capturados usando um Provedor de Relatórios personalizado que pode ser desenvolvido usando as APIs de Relatórios fornecidas pelo Oracle Service Bus. Isso permite que terceiros recebam e processem alertas em código Java personalizado Java.
    • Log de Alerta: Se você selecionar Sim, os alertas enviados para este destino de alerta serão registrados no log de alerta.
  7. Adicione um destinatário de e-mail à definição de recurso do destino de alerta. Consulte Adicionando Destinatários de E-mail.
  8. Adicione um Destino JMS para a definição de recurso do destino de alerta. Consulte Adicionando Destinos JMS.
  9. Clique em Salvar. O novo destino de alerta é salvo na sessão atual.
  10. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
