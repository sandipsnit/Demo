Adicionando Split-Joins

Antes de começar

Split-Joins requerem um arquivo .flow que é criado durante a criação de um Split-Join usando os plug-ins do Oracle Service Bus para Eclipse ou durante a importação de um JAR de configuração do Split-Join para o plug-in. O arquivo .flow pode ter referências a serviços de proxy, serviços de negócios ou a recursos WSDL externos. Os recursos dos quais o Split-Join depende devem estar presentes no servidor antes da ativação do Split-Join.

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione o Project Explorer, depois selecione um projeto ou pasta no qual adicionar o Split-Join. A página View de Projeto/Pasta é mostrada.
  3. Na lista drop-down Criar Recurso, selecione Split-Join para exibir a página Criar um Novo Recurso de Split-Join.

    4. Especifique os parâmetros mostrados na Tabela 9-2.

    Tabela 9-2 - Parâmetros do Split-Join
    Propriedade
    Para criar ou editar...
    Nome do Recurso
    Informe um nome exclusivo para esse Split-Join.

    Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
    Descrição do Recurso
    Informe uma descrição para o Split-Join.
    Split-Join
    Clique em Procurar para localizar um arquivo .flow no sistema, geralmente localizado em MW_HOME\user_projects\workspaces\your_workspace.
    Política de Distribuição
    Selecione a instância do Gerenciador de Trabalho do WebLogic Server usada para criar threads (ramificações paralelas) para executar o Split-Join. O Gerenciador de Trabalho default será usado se nenhum outro existir.

  4. Clique em Salvar. O Split-Join será criado e salvo na sessão atual.
    5. Observação: Recursos dos quais depende o Split-Join devem estar presentes antes da ativação do Split-Join.
  5. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
