Adicionando Conexões MQ

Antes de começar

Consulte Oracle Fusion Middleware Supported System Configurations em http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html para obter as versões suportadas do WebSphere MQ no Oracle Service Bus. A biblioteca do cliente do WebSphere MQ não é compactada com o instalador do Oracle Service Bus. Obtenha a biblioteca do cliente do WebSphere MQ de sua instalação do WebSphere MQ.

Para configurar e usar os recursos de Conexão MQ e o transporte MQ no Oracle Service Bus, é necessário adicionar a biblioteca de clientes do MQ ao classpath. Uma opção é copiar a biblioteca com.ibm.mq.jar para o diretório DOMAIN_NAME/lib, em que DOMAIN_NAME é o nome do diretório em que você localizou o domínio, geralmente MW_HOME\user_projects\domains\DOMAIN_NAME.

Para adicionar um recurso de Conexão MQ

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione o Project Explorer, depois selecione um projeto ou pasta no qual adicionar o recurso de Conexão MQ. A página View de Projeto/Pasta é mostrada.
  3. Na lista drop-down Criar Recurso, selecione Conexão MQ em Utilitário para exibir a página Criar um Novo Recurso de Conexão MQ.

    4. Especifique os parâmetros de conexão mostrados na Tabela 8-2.

    Tabela 8-2 - Parâmetros da Conexão MQ
    Propriedade
    Para criar ou editar...
    Nome do Recurso
    Informe um nome exclusivo para esse recurso MQ Connection.

    Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
    Descrição do Recurso
    Informe uma descrição para o recurso de Conexão MQ.
    Tipo de Conexão
    Selecione um dos modos a seguir para estabelecer conexão com o gerenciador de filas MQ:
    • modo tcp - Use o TCP/IP para estabelecer conexão com um gerenciador de filas que não resida na mesma máquina do Oracle Service Bus.
    • modo de binding - Use o modo de bindings para estabelecer conexão com um gerenciador de filas que esteja localizado na mesma máquina do Oracle Service Bus.
    Nome do Host MQ
    Apenas para conexões tcp mode:
    Informe o nome do host do gerenciador de filas MQ.
    Número da Porta MQ
    Apenas para conexões tcp mode:
    Informe o número da porta do listener do gerenciador de filas MQ.
    Nome do Gerenciador de Filas MQ
    Informe o nome do gerenciador de filas MQ ao qual se conectar.
    Gerenciador de Filas CCSID
    Apenas para conexões tcp mode:
    O identificador do conjunto de caracteres codificado (CCSID) a ser usado ao estabelecer uma conexão. O CCSID é usado para suporte de internacionalização.
    Para saber mais, consulte Fundamentos do WebSphere MQ em http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm.
    Nome do Canal do Gerenciador de Filas MQ
    Apenas para conexões tcp mode:
    Informe o nome do canal de conexão do servidor do gerenciador de filas.
    Camada SSL Obrigatória
    Para conexões tcp mode:
    Marque a caixa de seleção para usar o SSL para enviar mensagens. Apenas o SSL do lado do servidor será ativado quando a opção SSL Bilateral Obrigatório não estiver selecionada.
    Conjunto de Cifras
    Esta opção está disponível apenas quando a caixa de seleção SSL Obrigatório está marcada.
    Selecione o algoritmo de Conjunto de Cifras a ser usado pelo SSL.
    O algoritmo de Conjunto de Cifras é usado para criptografar e decriptografar comunicações de mensagens entre o servidor e o cliente WebSphere MQ. Assim, um algoritmo de Conjunto de Cifras deve ser especificado ao usar SSL para comunicação com um servidor do WebSphere MQ.
    SSL Bilateral Obrigatório
    Esta opção está disponível apenas quando a caixa de seleção SSL Obrigatório está marcada.
    Marque a caixa de seleção para ativar autenticação SSL no lado do cliente e no lado do servidor.
    Referência ao Provedor de Chaves de Serviço
    Se você selecionou SSL Bidirecional Obrigatório, deverá fornecer uma referência ao provedor de chaves de serviço para obter informações apropriadas da área de armazenamento de chaves e do armazenamento confiável.
    Informe o caminho (projeto/pasta) e o nome de um provedor de chaves de serviço ou clique em Procurar para selecionar um na página Selecionar Provedor de Chaves de Serviço.
    Referência à Conta de Serviço Estática
    Apenas para conexões tcp mode:
    Obrigatório para autenticação do nome de usuário e senha. Informe o caminho (projeto/pasta) e o nome de uma conta de serviço estática ou clique em Procurar para selecionar contas de serviço usando um browser.
    Versão do WebSphere MQ
    Selecione a versão do WebSphere MQ:
    • 5.3
    • 6.0
    • 7.0

    Observação: as versões 5.3 e 6.0 não são mais suportadas pela IBM. Essas versões estão obsoletas no Oracle Service Bus.
    Tamanho do Pool de Conexões MQ
    Informe o tamanho do pool de conexões MQ.
    Timeout da Conexão MQ
    Informe o intervalo de tempo em segundos após o qual as conexões não utilizadas serão destruídas. O default são 1800 segundos.
    Espera Máxima da Conexão MQ
    Informe a Espera Máxima em milissegundos para o tempo de espera até que uma conexão esteja disponível. Se uma conexão não for estabelecida dentro desse intervalo de tempo, o Oracle Service Bus emitirá uma exceção. O default é 3000 milissegundos.

  4. Clique em Salvar. O recurso de Conexão MQ é criado e salvo na sessão atual.
  5. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
