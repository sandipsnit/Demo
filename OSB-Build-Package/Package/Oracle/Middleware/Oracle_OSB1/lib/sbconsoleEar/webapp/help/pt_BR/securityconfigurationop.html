Usuários

Usuários são entidades que podem ser autenticadas. Um usuário pode ser uma pessoa ou uma entidade de software, como um cliente de Web Services. Você deve conceder a cada usuário uma identidade exclusiva (nome) em um realm de segurança.

Geralmente, os usuários que você cria se encaixam em duas categorias:

  • Usuários clientes que podem acessar seus serviços de proxy ou serviços de negócios.

    • Se você criar um grande número de usuários clientes, considere organizá-los em grupos de segurança.

  • Os usuários administrativos que podem usar a Console do Oracle Service Bus Administration para criar ou modificar serviços de proxy, serviços de negócios e outros recursos do Oracle Service Bus.

    • O Oracle Service Bus usa segurança baseada em atribuição para suas funções administrativas. Em vez de conceder privilégios de acesso diretamente aos usuários, o Oracle Service Bus concede privilégios administrativos apenas a atribuições de segurança. Para conceder privilégios administrativos a um usuário, coloque o usuário em um dos grupos de segurança default, que é uma das atribuições de segurança predefinidas.

Grupos

Para facilitar a administração de um grande número de usuários, é possível organizar os usuários em grupos nomeados. Depois, em vez de conceder privilégios de acesso ou identidades de atribuição a usuários individuais, conceda privilégios ou identidades para grupos.

Grupos Administrativos de Segurança

O Oracle Service Bus oferece grupos de segurança default para facilitar a concessão aos usuários de acesso a funções administrativas, como a criação de serviços de proxy. Cada grupo faz parte de uma das atribuições de segurança predefinidas do Oracle Service Bus às quais foram concedidos privilégios administrativos.

Para obter mais informações, consulte "Configuring Administrative Security" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.

Atribuições

Uma atribuição de segurança é uma identidade que pode ser concedida a um usuário ou grupo com base nas condições do ambiente do run-time. Ao criar políticas de controle de acesso, você pode conceder acesso a uma atribuição, grupo ou usuário.

Por exemplo, é possível criar dois grupos de usuários: MyCustomersEast e MyCustomersWest. Crie uma atribuição de segurança nomeada PrivilegedCustomer e crie condições para que o grupo MyCustomersWest esteja na atribuição de 8h a 20h Horário Padrão do Leste e o grupo MyCustomersEast esteja na atribuição de 20h a 8h Horário Padrão do Leste. Em seguida, crie uma política de controle de acesso para um serviço de proxy que conceda à atribuição PrivilegedCustomer acesso ao serviço. Usuários diferentes terão acesso em momentos diferentes, dependendo se estão no grupo MyCustomersEast ou MyCustomersWest.

Atribuições Administrativas de Segurança

O Oracle Service Bus oferece quatro atribuições de segurança predefinidas (mais quatro atribuições predefinidas do WebLogic Server) que concedem privilégios administrativos. Não é possível alterar os privilégios de acesso para as atribuições de segurança administrativa do Oracle Service Bus, mas você pode alterar as condições em que um usuário ou grupo está em uma das atribuições.

Para obter mais informações sobre estas atribuições e os privilégios disponíveis para cada atribuição, consulte "Configuring Administrative Security" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539.

Políticas de Controle de Acesso

Uma política de controle de acesso especifica as condições em que usuários, grupos ou atribuições podem acessar um serviço de proxy. Por exemplo, é possível criar uma política que permita que os usuários com a atribuição GoldCustomer acessem um serviço de proxy e que permita que os usuários com a atribuição SilverCustomer acessem o serviço de proxy apenas após a meia-noite em noites durante a semana.

Para todos os serviços de proxy, você pode criar uma política de transporte, que aplica uma verificação de segurança quando um cliente tenta estabelecer uma conexão com o serviço de proxy. Apenas solicitações de usuários que estejam listados na política de transporte têm permissão para prosseguir.

Uma política de controle de acesso no nível da mensagem aplica uma verificação de segurança quando um cliente tenta chamar um serviço de proxy com segurança no nível da mensagem. Você pode criar uma política de controle de acesso no nível da mensagem nos seguintes casos:

  • Para serviços de proxy que são intermediários de segurança de Web Services ativos
  • Para serviços de proxy que possuem autenticação personalizada de mensagem

Apenas usuários listados na política de mensagem têm permissão para chamar a operação.

Dados e Sessões da Configuração da Segurança

Usuários, grupos e atribuições são persistidos em provedores de segurança, que não são controlados pelas sessões do Oracle Service Bus. Portanto, é possível criar ou modificar esses dados estando dentro ou fora de uma sessão. Qualquer inclusão ou modificação feita nesses dados é efetivada imediatamente e disponibilizada para todas as sessões. Se você descartar uma sessão em que adicionou ou modificou os dados, os dados de segurança não serão descartados.

Políticas de controle de acesso são persistidas em provedores de autorização. E agora existe uma referência a eles no repositório do Oracle Service Bus.

Políticas de controle de acesso são gerenciadas em uma sessão de design do Oracle Service Bus, e não fora da sessão. Como as alterações são feitas em uma sessão, não é possível confirmar nem descartar as alterações, como acontece com outros recursos.

Embora ACLs possam ser gerenciadas na Console do Oracle Service Bus Administration, é possível alterar políticas fora do Oracle Service Bus. No entanto, a alteração de políticas fora do Oracle Service Bus pode fazer com que a referência no Oracle Service Bus fique desatualizada e inválida.

Portanto, para gerenciamento consistente, gerencie completamente ACLs fora das sessões do Oracle Service Bus (usando os MBeans do provedor de autorização ou ferramentas do provedor de autorização de terceiros) ou gerencie-as completamente dentro das sessões do Oracle Service Bus. Qualquer combinação das duas abordagens pode resultar em uma exibição inconsistente de políticas.
