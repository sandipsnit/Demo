Adicionando Contas de Serviço

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione o Project Explorer, depois selecione um projeto ou pasta no qual adicionar a conta de serviço. A página View de Projeto/Pasta é mostrada.
  3. Na lista drop-down Criar Recurso, selecione Conta de Serviço para exibir a página Criar uma Nova Conta de Serviço.
  4. No campo Nome do Recurso, informe um nome exclusivo para essa conta de serviço.

    Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
  5. No campo Descrição do Recurso, informe uma descrição para a conta de serviço.
  6. Em Tipo de Recurso, execute um destes procedimentos:
    • Para criar uma conta de serviço que forneça os nomes de usuário e senhas recebidos de solicitações do cliente de entrada, selecione Passagem.
    • Para criar uma conta de serviço que forneça um nome de usuário e senha que você salva com a configuração da conta de serviço, selecione Estático.
    • Para criar uma conta de serviço que mapeia o nome de usuário de um ou mais clientes autenticados para nomes de usuário e senhas que você especifica, selecione Mapeamento.
  7. Dependendo do tipo de recurso selecionado na etapa 6, execute uma das seguintes etapas descritas na Tabela 16-2.

    8. Tabela 16-2 Opções do Tipo de Recurso 
    Tipo de Recurso Selecionado
    Execute Estas Etapas
    Passagem
    Clique em Último.
    Estático
    1. Clique em Próximo.
    2. Informe o nome do usuário e a senha nos campos Nome do Usuário, Senha e Confirmar Senha.
    3. Clique em Último.
    Mapeamento
    Para criar uma conta de serviço que mapeia o nome de usuário de um ou mais clientes para nomes de usuário e senhas que você especifica, faça o seguinte:
    1. Clique em Próximo.
    2. Na tabela Informar Usuário Remoto Autorizado, faça o seguinte:
      1. Nos campos Nome do Usuário Remoto, Senha e Confirmar Senha, informe o nome de usuário e a senha que você deseja enviar em solicitações de saída.
      2. Clique em Adicionar.

        3. O mapeamento do usuário é adicionado à tabela Usuários Remotos.

      3. (Opcional) Inclua usuários remotos adicionais na tabela Informar Usuário Remoto Autorizado.
    3. Clique em Próximo.
    4. Para mapear clientes autorizados para nomes de usuário e senhas remotos, faça o seguinte na tabela Informar Usuário Local Autorizado:
      1. No campo Nome do Usuário Local, informe o nome que identifica um cliente que foi autenticado na solicitação de entrada.

        2. Se você ainda não tiver adicionado esse usuário ao módulo Configuração da Segurança da Console do Oracle Service Bus Administration, faça isso para poder usar este mapeamento em um ambiente de runtime. Consulte Adicionando Usuários. O Oracle Service Bus permite a criação de um mapeamento para um usuário local inexistente, mas o mapeamento nunca corresponderá a um usuário autenticado e nunca será usado.

      2. Na lista Nome do Usuário Remoto, selecione o nome do usuário que você deseja enviar em solicitações de saída para o usuário autenticado especificado no campo Nome do Usuário Local.
      3. Clique em Adicionar.
    5. Para mapear clientes anônimos para nomes de usuário remotos, faça o seguinte:
      1. Marque a caixa de seleção Mapear Solicitações Anônimas.
      2. Na lista Selecionar Usuário Remoto, selecione o nome do usuário que você deseja enviar em solicitações de saída para todos os usuários anônimos.
    6. Clique em Último.

  8. Clique em Salvar. A conta de serviço é criada e salva na sessão atual.
  9. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
