Editando Contas de Serviço

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Localize a conta de serviço, conforme descrito em Localizando Contas de Serviço.
  3. Clique no nome da conta de serviço. A página Exibir Detalhes da Conta de Serviço exibe as informações mostradas na Tabela 16-3.

    4. Tabela 16-3 - Detalhes da Conta de Serviço
    Propriedade
    Descrição
    Última Modificação
    O usuário que criou esta conta de serviço ou a importou para a configuração.
    Última Modificação em
    A data e hora em que o usuário criou esta conta de serviço ou a importou para a configuração. Clique no link da data e hora para exibir o histórico de alterações deste recurso. Consulte a página Exibir Histórico de Alteração.
    Referências
    O número de objetos referenciados por esta conta de serviço. Se tais referências existirem, clique no link numérico para exibir uma lista dos objetos. Consulte Exibindo Referências a Recursos.
    Referenciado por
    O número de objetos que referenciam esta conta de serviço. Se tais referências existirem, clique no link numérico para exibir uma lista dos objetos. Por exemplo, se você selecionou esta conta de serviço como a conta de serviço JMS em um serviço de proxy com um protocolo de transporte JMS, o serviço de proxy será listado como uma referência quando você clicar no link. Consulte Exibindo Referências a Recursos.
    Descrição
    Uma descrição desta conta de serviço, se existir.

  4. Para fazer uma alteração nos campos, clique em Editar. Consulte Adicionando Contas de Serviço para obter descrições dos campos.

    5. Não é possível alterar o campo Nome do Recurso.

  5. Clique em Salvar para confirmar as atualizações na sessão atual.
  6. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
Observação: Se a conta de serviço modificada for usada para autenticação em um servidor JMS do WebLogic, o servidor JMS pode não reconhecer sua modificação durante até 60 segundos. Por default, o Servidor JMS do WebLogic verifica as permissões para cada destino a cada 60 segundos. Para alterar esse comportamento, modifique o comando de inicialização do WebLogic Server para que defina a seguinte propriedade do sistema com a frequência (em segundos) com a qual você deseja que o Servidor JMS do WebLogic verifique as permissões: weblogic.jms.securityCheckInterval
Um valor 0 (zero) para essa propriedade garante que uma verificação de permissões seja executada para cada ação de envio, recebimento e getEnumeration em um recurso JMS.
