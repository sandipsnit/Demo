Especificando Contas de Serviço

Você pode usar a mesma conta de serviço para vários serviços de negócios e de proxy. Para especificar o nome de usuário e senha fornecidos por uma conta de serviço, é possível usar qualquer uma das técnicas a seguir:

  • Estático

    • Requer que você salve um nome de usuário e senha com a configuração da conta de serviço. A conta de serviço decodifica esse nome de usuário e senha na solicitação de saída.

  • Passagem do nome do usuário e senha

    • Faz com que a conta de serviço forneça o nome de usuário e senha recebidos de uma solicitação de cliente de entrada. Por exemplo, se uma solicitação HTTP BÁSICA de entrada contiver "pat" e "patspassword" como o nome de usuário e senha, a conta de serviço decodificará "pat" e "patspassword" na solicitação de saída.

    Como essa técnica requer que o cliente solicite nomes de usuário e senhas de texto sem formatação, ela só é aplicável para solicitações de cliente que usem o protocolo HTTP BÁSICO, uma autenticação de Token do Nome de Usuário de Segurança de Web Services com uma senha de texto sem formatação ou um nome token de nome de usuário e senha personalizado.

    A Oracle recomenda que você use essa técnica somente quando o Oracle Service Bus e o ponto final pertencerem ao mesmo domínio de autenticação. Por exemplo, use essa técnica ao encaminhar mensagens de uma única organização e quando tanto o Oracle Service Bus quanto o consumidor da mensagem forem autenticados em um servidor LDAP comum.

    As seguintes restrições se aplicam a esta técnica:

    • Não pode ser usado em solicitações de saída que autenticam o Oracle Service Bus para um servidor local ou remoto ou um recurso do sistema, como um servidor FTP ou JMS.
    • Não pode ser usado com a função fn-bea:lookupBasicCredentials do XQuery. Para obter mais informações, consulte "XQuery Implementation" no Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG444.
      • Observação: Se o serviço for um intermediário de WSS ativo, você poderá usar a Segurança de Web Services para criptografar um Token do Nome de Usuário de Segurança de Web Services ou um nome de usuário/senha personalizado. Nesta instância, a passagem do nome de usuário/senha funciona, porque o proxy primeiro decriptografará a solicitação e depois terá acesso ao nome de usuário/senha do usuário de texto sem formatação.
  • Mapeamento de usuários

    • Requer que você correlacione (mapeie) o nome de usuário que é o resultado da autenticação de uma solicitação de entrada de um cliente (o nome de usuário local) em um nome de usuário e senha especificados (o nome de usuário e a senha remotos). Quando a conta de serviço recebe uma solicitação de um cliente autenticado que foi mapeada, ela fornece o nome de usuário e a senha remotos apropriados para a solicitação de saída do serviço de negócios ou do serviço de proxy.

    Se o cliente for autenticação no transporte e na mensagem, a conta de serviço mapeará o nome de usuário de mensagem para o nome de usuário e senha remotos.

    Também é possível mapear um nome de usuário anônimo para um nome de usuário e senha remotos.

    As seguintes restrições se aplicam a esta técnica:

    • Não pode ser usado em solicitações de saída que autenticam o Oracle Service Bus para um servidor local ou remoto ou um recurso do sistema, como um servidor FTP ou JMS.
    • Não pode ser usado com a função fn-bea:lookupBasicCredentials do XQuery. Para obter mais informações, consulte "XQuery Implementation" no Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG444.
