Adicionando Provedores de JNDI

Adicione e configure um recurso do Provedor de JDNI e disponibilizá-lo no Oracle Service Bus como um recurso do sistema. Você deve estar em uma sessão ativa.

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione Administração do Sistema > Provedores de JNDI.
  3. Clique em Adicionar.
  4. No campo Nome, informe um nome exclusivo para esse recurso Provedor de JNDI. Esse campo é obrigatório.

    Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
  5. No campo Descrição, informe uma breve descrição para o Provedor de JNDI.
  6. Mantenha a opção Cache de JNDI como Ativado (default) ou selecione Desativado.

    7. Quando ativado, o contexto e os objetos de JDNI são armazenados no cache localmente, o que melhora o desempenho durante uma pesquisa de objeto. A Oracle recomenda que você mantenha o cache de JNDI ativado.

  7. No campo URL do Provedor, informe o URL do Provedor de JNDI no formato: protocol://host:port

    8. É possível usar qualquer protocolo, por exemplo, http, https, t3, t3s, iiop, iiops.

    Esse campo é obrigatório.

  8. No caso de clusters, o URL do Provedor de JNDI deverá ser configurado com uma lista separada por vírgula de Servidores Gerenciados, no formato:

    9. protocol://<hostname>:<ms1port>, <hostname>:<ms2port>

  9. Se o acesso ao Provedor de JNDI de destino exigir um nome de usuário e senha, informe um nome de usuário no campo Nome do Usuário e a senha associada nos campos Senha e Confirmar Senha.

    10. Estes campos são opcionais e obrigatórios apenas se a árvore de JDNI estiver protegida.

  10. Clique em Salvar para criar e salvar o recurso do Provedor de JNDI na sessão atual.
  11. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
