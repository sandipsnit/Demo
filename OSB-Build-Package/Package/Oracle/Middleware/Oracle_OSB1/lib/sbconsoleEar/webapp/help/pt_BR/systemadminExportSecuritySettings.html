Especificando as Definições de Segurança de Exportação

Se a exportação incluir uma conta de serviço ou um provedor de chaves de serviço, a página Exportar Recursos - Definições de Segurança será exibida.

Para cada conta de serviço, o Oracle Service Bus exporta o nome de usuário e senha ou o mapeamento de usuário local para usuário remoto (dependendo de quais dados foram armazenados na conta de serviço). Para cada provedor de chaves de serviço, o Oracle Service Bus exporta o binding de alias ao par de chaves do provedor de mapeamento de credencial de PKI; ele não exporta chaves privadas, certificados nem outros dados das áreas de armazenamento de chaves. Os dados das áreas de armazenamento de chaves devem ser exportados usando as ferramentas fornecidas pelo fornecedor da área de armazenamento de chaves.

Se você quiser que o Oracle Service Bus criptografe os dados do nome de usuário e senha que estão nos recursos da conta de serviço, provedor de chaves de serviço, Registro UDDI, Provedor de JNDI ou provedor SMTP antes de exportá-lo para o arquivo JAR, faça o seguinte:

  1. Marque a caixa de seleção Proteger Dados Importantes.
  2. Informe e confirme uma senha.

    3. Quando você ou outra pessoa tentar importar esse JAR, o Oracle Service Bus não importará os recursos com dados criptografados do nome de usuário e senha, a menos que você especifique esta senha. É possível importar todos os outros recursos não-criptografados no JAR sem especificar a senha.

  3. Clique em Concluir Exportação.
  4. Quando você clica em Exportar ou Concluir Exportação, a janela pop-up Processando Dados da Configuração é exibida. Aguarde o fechamento da janela antes de fazer qualquer coisa na Console do Oracle Service Bus Administration. A janela será fechada automaticamente quando a caixa de diálogo Download de Arquivo for exibida.
  5. Na caixa de diálogo, clique em Download de Arquivo, clique em Abrir para abrir o arquivo JAR exportado ou clique em Salvar para salvar o arquivo JAR na área de trabalho.
