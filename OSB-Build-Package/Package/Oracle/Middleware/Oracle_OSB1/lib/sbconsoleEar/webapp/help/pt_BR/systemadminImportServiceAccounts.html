Importando Contas de Serviço ou Provedores de Chaves de Serviço

Observação: Em versões anteriores do Oracle Service Bus, os provedores de chaves de serviço eram chamados de provedores de serviços de proxy.

Se o arquivo JAR foi criado pelo AquaLogic Service Bus 3.0 ou posterior e contiver contas de serviço ou provedores de chaves de serviço, você poderá importar esses recursos junto com os nomes de usuário, senhas, mapeamentos de usuário local para usuário remoto e bindings de alias ao par de chaves que contêm.

Para cada provedor de chaves de serviço, o Oracle Service Bus importa o alias para o binding do par de chaves no provedor de mapeamento de credencial de PKI. Se esses dados foram criptografados durante a exportação, você deverá fornecer a senha que foi usada para criptografar os dados. Se você não souber a senha, poderá importar todos os outros recursos não-criptografados.

Se você importar uma conta de serviço ou um provedor de chaves de serviço e um recurso correspondente com o mesmo nome já existir no domínio, o recurso importado substituirá o que já está no domínio, mesmo que o recurso que já está no domínio tenha sido modificado durante a sessão atual, a menos que você especifique para preservar definições de segurança durante a importação. Consulte a etapa 5 em Selecionando Projetos e Recursos para Importar.
