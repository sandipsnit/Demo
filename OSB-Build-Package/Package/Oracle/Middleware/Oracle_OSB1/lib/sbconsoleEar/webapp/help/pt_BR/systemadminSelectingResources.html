Selecionando Projetos e Recursos para Importar

  1. Se estiver importando um arquivo JAR do recurso e quiser garantir que todos os seus recursos associados sejam importados, selecione Incluir Dependências.
    2. Observação: Incluir Dependências não pode ser selecionado se você estiver importando um arquivo JAR do projeto inteiro ou um arquivo de configuração anterior à versão 2.6.
  2. Clique no ícone Abrir para exibir Configurações Avançadas para preservar valores no sistema de importação.
  3. Quando você está importando novamente um recurso, mas não deseja proteger nenhuma personalização feita no recurso existente, selecione Preservar Variáveis de Ambiente.
    4. Observação: A personalização de referências em um recurso existente não pode ser preservada. Consulte Criando Arquivos de Personalização.
  4. Quando você estiver importando o recurso de Definições Operacionais Globais ou outros recursos como serviços de negócios ou de proxy que contenham definições operacionais de serviço associadas, e quiser preservar os valores operacionais no sistema de importação, selecione Preservar Valores Operacionais.
  5. Para impedir a que uma configuração relacionada à segurança existente em recursos que estão sendo atualizados seja sobrescrita, selecione entre as seguintes opções:
    1. Preservar Configuração de Segurança e de Política - preserva a configuração de segurança (excluindo políticas de controle de acesso) e as referências para as Políticas de Web Services ligadas diretamente ao serviço (em vez de ligadas ao WSDL).

      2. Para obter mais informações, consulte "Security" no Oracle Fusion Middleware Developer's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472.

    2. Preservar Credenciais - preserva as credenciais do PKI nos provedores de chaves de serviço, nome do usuário e senhas em contas de serviço, e credenciais do nome do usuário e senha em Servidores SMTP, Servidores Proxy, Provedores JNDI e Registros UDDI.
    3. Preservar Políticas de Controle de Acesso - preserva políticas de controle de acesso.

      4. As informações resumidas do recurso são mostradas na Tabela 28-1.

      Tabela 28-1 - Informações sobre Recursos
      Propriedade
      Descrição
      Nome
      O nome do projeto.
      Recursos globais, como informações do Registro UDDI, Servidores SMTP, Servidores Proxy, Provedores de JNDI e Definições Operacionais Globais aparecem no projeto Sistema em suas respectivas pastas.
      Tipo
      O tipo de recurso, que é Projeto.
      Expanda a pasta do Projeto a ser exibida e cancele a seleção de recursos.
      Operações
      Exibe --- ou !Deletions!
      Expanda a pasta Projeto para exibir as operações (criar, atualizar, deletar) que serão executadas nos recursos.
      Para impedir que recursos sejam deletados, desmarque sua seleção. Recursos só são programados para deleção quando o JAR sendo importado é um JAR do projeto inteiro e há recursos do projeto no sistema de importação que não estão presentes no arquivo JAR importado.
      Referências
      Expanda a pasta do Projeto para exibir o número de objetos referenciados por cada recurso.

  6. Selecione os objetos em que você deseja operar (criar, atualizar ou deletar).
    1. Para exibir cada objeto contido no arquivo JAR, expanda a pasta do arquivo. A caixa de seleção associada a cada objeto é selecionada.

      2. Os detalhes do arquivo de recursos a seguir, exibidos para cada objeto, são mostrados na Tabela 28-2.

      Tabela 28-2 Detalhes do Arquivo de Recursos 
      Propriedade
      Descrição
      Nome
      O nome e o caminho do recurso. O caminho é o nome do projeto e o nome da pasta em que reside o recurso.
      Tipo
      Os tipos de recurso do Oracle Service Bus.
      Observação: Se um recurso tiver dados de segurança importantes, um ícone Criptografia será exibido.
      Cuidado: Se você selecionar uma conta de serviço ou um provedor de chaves de serviço, poderá substituir os dados de segurança existentes a menos que você especifique para preservar as definições de segurança durante a importação. Consulte Importando Contas de Serviço ou Provedores de Chaves de Serviço.
      Operações
      Crie, atualize ou delete.
      Recursos só são programados para deleção quando o JAR sendo importado é um JAR do projeto inteiro e há recursos do projeto no sistema de importação que não estão presentes no arquivo JAR importado.
      Referências
      O número de objetos referenciados por este recurso.

    2. Desmarque as caixas de seleção ao lado dos recursos que você não deseja importar.
  7. Clique em Importar.

Depois de finalizar

Se aplicável, continue em Especificando as Definições de Segurança de Importação. Ou continue em Revisando o Resumo de Importação.
