Testando Serviços de Proxy

Você pode testar os seguintes tipos de serviços de proxy: qualquer XML, qualquer SOAP, Mensagens, XML e SOAP. É possível testar serviços de proxy SOAP com políticas de Segurança de Web Services (WSS). Consulte Segurança do Web Service em Configurando Dados do Teste de Serviços de Proxy.

Observação: Quando a Console de Teste chama um proxy com autenticação personalizada de token HTTP, a verificação de autenticação não é executada.
Cuidado: O teste de serviços de proxy com a opção de chamada direta ativada ignora algumas etapas de segurança importantes, incluindo controle de acesso. A Oracle recomenda que você não use o serviço de teste em sistemas de produção.
  1. Clique em Ativar em Centro de Alterações para ativar o recurso de teste na Console do Oracle Service Bus Administration.

    2. É possível testar os serviços de proxy no Resource Browser ou no Project Explorer.

  2. Selecione Browser do Recurso > Serviços de Proxy para exibir a página Resumo de Serviços de Proxy.
  3. Em Ações, clique no ícone Acionar Console de Teste associado ao serviço de proxy que você deseja testar.

    4. A Console de Teste abre a página Teste do Serviço de Proxy. Por exemplo, usando os exemplos fornecidos com o produto, clique no ícone associado ao serviço de proxy LoanGateway1.

    Observação: Em um domínio em cluster, não é possível usar a Console de Teste para testar nenhum serviço de negócios ou serviço de proxy configurado que encaminhe para um serviço de negócios.
  4. Para os serviços SOAP e XML, selecione a operação WSDL que você deseja testar.
  5. Configure os dados de teste para o serviço de proxy. Devem ser os dados que o serviço de proxy espera do cliente.

    6. Por default, ambas as opções de configuração de teste, Chamada Direta e Incluir Rastreamento, estão ativadas. É possível desmarcar a opção Chamada Direta, que também desmarca a opção Incluir Rastreamento. Fazendo isso, o teste é executado usando o método de chamada indireta em que a mensagem é enviada por meio da camada de transporte.

    É possível usar a opção Chamada Direta (deixe Chamada Direta selecionado) e desativar o rastreamento da Execução; basta desmarcar a caixa de seleção Incluir Rastreamento.

    Para ver o rastreamento da mensagem, desmarque Chamada Direta.

    Observação: Para ver o rastreamento no arquivo de log ou na saída padrão (console do servidor), o log do Oracle WebLogic Server deve ser definido com os seguintes níveis de severidade:

Severidade mínima para registro: Info
Arquivo de log: Info
Saída padrão: Info

Para obter informações sobre a definição de níveis de severidade do log, consulte "Using Log Severity Levels" em Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

     

  1. Clique em Executar. A página Teste do Serviço de Proxy exibirá os resultados. Para obter informações sobre os resultados do teste, consulte Exibindo Resultados do Teste de Serviços de Proxy.
  2. Para executar o teste novamente, clique em Voltar. Repita as etapas de 5 em diante.
