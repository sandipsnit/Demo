Testando Serviços de Negócios

Você pode testar os seguintes tipos de serviços de negócios: qualquer XML, qualquer SOAP, Mensagens, XML e SOAP. É possível testar serviços de negócios SOAP com políticas de Segurança de Web Services. Para obter mais informações, consulte "Testing Services with Web Service Security" no Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus em http://www.oracle.com/pls/as111170/lookup?id=OSBAG384.

Observação: Em um domínio em cluster, não é possível usar a Console de Teste para testar nenhum serviço de negócios ou serviço de proxy configurado que encaminhe para um serviço de negócios.
  1. Clique em Ativar em Centro de Alterações para ativar o recurso de teste na Console do Oracle Service Bus Administration.
  2. Selecione Browser do Recurso > Serviços de Negócios para exibir a página Resumo de Serviços de Negócios.
  3. Em Ações, clique no ícone Acionar Console de Teste associado ao serviço que você deseja testar. A Console de Teste abre a página Teste do Serviço de Negócios. Por exemplo, usando os tutoriais fornecidos com o produto, clique no ícone associado a loanSaleProcessor.
  4. Para os serviços SOAP e XML, selecione a operação WSDL que você deseja testar.
  5. Configure os dados de teste para o serviço de negócios (os dados de entrada devem ser a mensagem sendo enviada pelo serviço de proxy para o serviço de negócios). As opçõesChamada Direta e Incluir Rastreamento disponíveis quando serviços de proxy de teste não estão disponíveis para serviços de negócios. Por default, os serviços de negócios são testados usando a opção Chamada Direta, significando que as mensagens passam pela camada de transporte.

    6. Para ver o rastreamento da mensagem, desmarque Chamada Direta.

    Observação: Para ver o rastreamento no arquivo de log ou na saída padrão (console do servidor), o log do Oracle WebLogic Server deve ser definido com os seguintes níveis de severidade:

Severidade mínima para registro: Info
Arquivo de log: Info
Saída padrão: Info

Para obter informações sobre a definição de níveis de severidade do log, consulte "Using Log Severity Levels" em Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server em http://www.oracle.com/pls/as111170/lookup?id=WLLOG141.

  1. Clique em Executar.

    2. A página Teste do Serviço de Negócios exibe os resultados. Para obter mais informações, consulte Exibindo Resultados do Teste de Serviços de Negócios.

Testando anexos no Serviços de Negócios

Esta seção descreve como usar a Console de Teste para testar anexos nos serviços de negócios.

Para fins de ilustração, esta seção assume que um serviço de negócios de HTTP baseado em WSDL usa uma operação "submitAttachment" para enviar um arquivo ZIP como anexo em uma mensagem SOAP. Modifique as configurações conforme adequado para os diferentes tipos de anexos binários.

    1. Na Console de Teste para o serviço de negócios, vá para a página XML e digite o seguinte Envelope SOAP para a mensagem de teste:
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. Na seção Anexo da Console de Teste:
      1. Tipo de conteúdo: aplicação/zip (conforme definido no WSDL)
      2. ID do conteúdo: zipFile (deve ser equivalente ao valor de href="cid:zipFile" na mensagem de teste)
      3. Arquivo: Clique em Procurar e selecione o arquivo ZIP que você especificou no Envelope SOAP.
      4. Clique em Adicionar.

    2. Clique em Executar para enviar a mensagem de texto com o anexo.

Para confirmar o êxito de envio do anexo, verifique se na console do servidor há uma mensagem semelhante à seguinte, que é conectada pela operação submitAttachment:

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
