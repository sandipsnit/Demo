Testando Transformações XSLT

XSLTs (eXtensible Stylesheet Language Transformations) descrevem mapeamentos de XML para XML no Oracle Service Bus. Para testar um recurso XSLT, você deve fornecer o documento XML de entrada e a Console de Teste retornará o documento XML de saída. Transformações XSLT podem ter parâmetros com nomes adicionais. Todos os parâmetros exigidos pela transformação são exibidos na página de configuração. Valores default são disponibilizados, mas é possível substituí-los.

Tabela 32-6 descreve as opções de configuração XSLT.

Tabela 32-6 - Configurando Opções XSLT

Entrada e Parâmetros
O documento e parâmetros de entrada para teste do recurso XSLT.
Entrada XML
A entrada XML pode ser baseada em arquivo ou em texto. A referência de um arquivo para entrada tem precedência sobre entrada textual. Pesquise e selecione o arquivo que você deseja usar no teste. A entrada XML é obrigatória.
<param_name> ([] as XML)
param_name é um parâmetro XSLT nomeado.
Existem dois tipos de entrada: XML e primitivo (String, número inteiro, flutuação, etc.). O tipo de entrada default é String. Marque a caixa de seleção associada ao nome do parâmetro para identificar um parâmetro do tipo XML.

É possível testar o tempo de design ou o run-time.

  1. Clique em Ativar se quiser testar o run-time. Não ative a sessão para testar o tempo de design.
  2. Selecione Project Explorer > XSLTs para exibir a página Resumo de XSLTs.
  3. Em Ações, clique no ícone Acionar Console de Teste associado ao recurso que você deseja testar. A Console de Teste abre a página Teste do Recurso.
  4. Configure os dados de teste para o recurso na seção Entrada e Parâmetros da página. Para obter mais informações, consulte a Tabela 32-6.
  5. Clique em Executar. A página Teste do Recurso exibe os resultados.
  6. Para testar novamente, clique em Voltar. É possível fechar a Console de Teste, modificar e testar o recurso novamente.
