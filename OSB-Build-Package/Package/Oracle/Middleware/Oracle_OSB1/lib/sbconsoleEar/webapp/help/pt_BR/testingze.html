Testando Transformações XQuery

Mapas XQuery podem descrever mapeamentos XML para XML, XML para não-XML e não-XML para XML. Uma transformação XQuery pode utilizar várias entradas e retornar uma saída. Cada entrada corresponde a uma variável externa XQuery declarada no recurso XQuery. O valor de uma variável de entrada XQuery pode ser um valor primitivo (String, inteiro, data, etc.), um documento XML ou uma sequência desses tipos. O valor de saída pode ser um valor primitivo (String, inteiro, data, etc.), um documento XML ou uma sequência desses tipos.

Observação: A Console de Teste não suporta sequências na entrada.

Tabela 32-7 descreve as opções de configuração XQuery.

Tabela 32-7 - Configurando Opções XQuery

Variáveis
Esta seção contém um campo de entrada para cada uma das variáveis externas XQuery.
<param_name> ([] as XML)
param_name é um nome de variável XQuery no recurso XQuery.
Na Console de Teste, uma caixa de edição de linha única é exibida se o tipo for simples. Uma caixa de seleção de várias linhas é exibida se os dados forem XML.
Uma entrada de combinação (<param_name> ([] como XML)) é usada quando a variável não é digitada. Você deve declarar o tipo de variável. Marque a caixa de seleção para identificar um parâmetro do tipo XML.
Uma entrada XML pode ser baseada em arquivo ou em texto. A referência de um arquivo para entrada tem precedência sobre entrada textual. Pesquise e selecione o arquivo que você deseja usar no teste.
A entrada na Console de Teste é convertida com base no tipo para facilitar o entendimento do tipo de dados que você deve informar. Quando não digitado, o tipo default é XML.

É possível testar o tempo de design ou o run-time.

  1. Clique em Ativar se quiser testar o run-time. Não ative a sessão para testar o tempo de design.
  2. Selecione Project Explorer > XQueries para exibir a página Resumo de XQueries.
  3. Em Ações, clique no ícone Acionar Console de Teste associado ao recurso que você deseja testar. A Console de Teste abre a página Teste do Recurso.
  4. Configure os dados de teste para o recurso na seção Variáveis da página. Para obter mais informações, consulte a Tabela 32-7.
  5. Clique em Executar. A página Teste do Recurso exibe os resultados.
  6. Para testar novamente, clique em Voltar. É possível fechar a Console de Teste, modificar e testar o recurso novamente.
