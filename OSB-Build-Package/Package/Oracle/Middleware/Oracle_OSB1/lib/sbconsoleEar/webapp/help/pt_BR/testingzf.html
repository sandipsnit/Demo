Usando os Editores de Expressão e de Condição XQuery

Use expressões XQuery para criar o conteúdo de dados para as variáveis de contexto da mensagem (ou parte de uma variável de contexto da mensagem) durante a execução do fluxo de mensagens. É possível usar a Console de Teste diretamente no Editor de Expressão XQuery para testar a definição da expressão.

Da mesma forma, use condições XQuery para avaliar condições Booleanas no fluxo de mensagens. É possível usar a Console de Teste diretamente no Editor de Condição XQuery para testar a definição da condição.

Um XQuery pode utilizar várias entradas e retornar uma saída. Cada entrada corresponde a uma variável não associada XQuery definida no XQuery. O valor de uma entrada XQuery pode ser um valor primitivo (String, inteiro, data, etc.), um documento XML ou uma sequência desses tipos. O valor de saída pode ser um valor primitivo (String, inteiro, data, etc.), um documento XML ou uma sequência desses tipos.

Observação: A Console de Teste não suporta sequências na entrada.

Variáveis
Esta seção contém um campo de entrada para cada uma das variáveis não associadas XQuery.
<param_name> ([] as XML)
param_name é o nome da variável não associada XQuery correspondente.
Na Console de Teste, uma caixa de edição de linha única é exibida se o tipo for simples. Uma caixa de seleção de várias linhas é exibida se os dados forem XML. Uma entrada de combinação (<param_name> ([] como XML)) é usada quando a variável não estiver digitada. Você deve declarar o tipo de variável. Marque a caixa de seleção para identificar um parâmetro do tipo XML.
Uma entrada XML pode ser baseada em arquivo ou em texto. A referência de um arquivo para entrada tem precedência sobre entrada textual. Pesquise e selecione o arquivo que você deseja usar no teste.
A entrada na Console de Teste é convertida com base no tipo para facilitar o entendimento do tipo de dados que você deve informar. O tipo default é XML.

  1. Acesse a Console de Teste ao editar uma ação no fluxo de mensagens de um pipeline.
    • Para acessar o Editor de Expressão XQuery, consulte Criando e Editando Expressões XQuery e XPath em Linha.
    • Para acessar o Editor de Condição XQuery, consulte Criando e Editando Expressões XQuery e XPath em Linha.

      • A página Expressão de Teste XQuery é exibida. Todas as variáveis de entrada solicitadas são exibidas na página.

  2. Configure os dados de teste para o XQuery na seção Variáveis da página. Para obter mais informações, consulte a Tabe 32-8.
  3. Clique em Executar. A página de teste exibe os resultados.
  4. Ao concluir um teste, você pode clicar em Voltar para executar um teste novo. Para executar um teste novo depois de fazer alterações no XQuery, você deve fechar e reabrir a Console de Teste para que as alterações tenham efeito.
