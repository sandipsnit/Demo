Adicionando Registros UDDI

Use esta página para adicionar e configurar registros UDDI. Após a configuração dos registros, você pode publicar os serviços de proxy do Oracle Service Bus neles ou importar serviços de negócios deles a serem usados em um serviço de proxy do Oracle Service Bus.

Você deve estar em uma sessão ativa para adicionar um registro.

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione Administração do Sistema > Registros UDDI.
  3. No painel Registros UDDI, clique em Adicionar Registro.
  4. Para configurar o registro, execute as seguintes etapas. Asteriscos denotam campos obrigatórios. Consulte a Tabela 29-1 para obter descrições das propriedades que devem ser definidas ao adicionar um registro.
    1. No campo Nome, informe um nome para o registro.

      Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
    2. No campo URL de Consulta, informe um URL de consulta no formato: http://host:port/APPLICATION_SERVER_CONTEXT/uddi/inquiry
    3. No campo URL de Publicação, informe um URL de publicação no formato http://host:port/APPLICATION_SERVER_CONTEXT/uddi/publishing
    4. No campo URL de Segurança, informe um URL de segurança no formato http://host:port/APPLICATION_SERVER_CONTEXT/uddi/security
    5. No campo URL de Assinatura, informe um URL de assinatura no formato
      http://host:port/APPLICATION_SERVER_CONTEXT/uddi/subscription

      6. Por default, o valor de APPLICATION_SERVER_CONTEXT no instalador é registry.

    6. No campo Nome do Usuário, informe o nome do usuário para efetuar log-in na console de registro.
    7. No campo Senha /(Confirmar Senha), informe a senha para efetuar log-in na console de registro.
    8. Marque a caixa de seleção Carregar tModel no registro para publicar os tModels do Oracle Service Bus neste registro.

      9. Este campo só é obrigatório ao publicar serviços de proxy neste registro.

    9. Marque a caixa de seleção Ativar Importação Automática para sincronizar automaticamente os serviços importados com o registro UDDI. Qualquer serviço importado com esta opção selecionada será mantido em sincronia com o registro UDDI.
      10. Observação: A autossincronização é um processo em segundo plano; não é possível revertê-la usando a função Desfazer da sessão. A reversão de uma alteração de autossincronização não é permanente, pois o serviço será ressincronizado no próximo ciclo de sincronização. Se deseja que um serviço importado permaneça fora de sincronia com o registro UDDI, você precisa separar o serviço para evitar mais atualizações do registro. Consulte Separando Serviços.
  5. Clique em Validar para validar os URLs fornecidos.
  6. Clique em Salvar para configurar o registro com as definições fornecidas e salvá-lo na sessão atual.
  7. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
