Editando WSDLs

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Localize o WSDL, conforme descrito em Localizando WSDLs.
  3. Clique no nome do WSDL. A página Exibir Detalhes WSDL exibe as informações mostradas na Tabela 6-2, Tabela 6-3 e Tabela 6-4.

    4. Tabela 6-2 Detalhes do WSDL 
    Propriedade
    Descrição
    Última Modificação
    O usuário que criou este WSDL ou o importou para a configuração.
    Última Modificação em
    A data e hora em que o usuário criou este WSDL serviço ou o importou para a configuração. Clique no link da data e hora para exibir o histórico de alterações deste recurso. Consulte a página Exibir Histórico de Alteração.
    Referências
    O número de objetos referenciados por este WSDL. Se tais referências existirem, clique no link numérico para exibir uma lista dos objetos. Consulte Exibindo Referências a Recursos.
    Referenciado por
    O número de objetos que referenciam este WSDL. Se tais referências existirem, clique no link numérico para exibir uma lista dos objetos. Por exemplo, se você selecionou esta porta ou binding de WSDL como o tipo de serviço para um serviço de negócios ou de proxy específico, o serviço de negócios ou de proxy será listado como uma referência quando você clicar no link. Consulte Exibindo Referências a Recursos.
    Descrição
    Uma descrição deste WSDL, se existir.

    A view estrutural do WSDL é exibida por default, que inclui as informações mostradas na Tabela 6-3.

    Tabela 6-3 - Detalhes da Estrutura WSDL
    Propriedade
    Descrição
    Namespace de Destino
    O namespace usado para qualificar qualquer uma das definições incluídas no WSDL.
    Definições de WSDL
    Os atributos e grupos associados ao WSDL:
    • Tipos de Porta de WSDL
    • Bindings de WSDL
    • Portas de WSDL
    • Tipos de Esquema XML
    • Elementos do Esquema XML
    • Importações de WSDL
    • Importações do Esquema XML
    • Referências de Política de Web Services
    • Referências Implícitas de Política de Web Services
    Estado de WSDL
    O status do WSDL:
    • Válido

      • Todos os locais para esquemas XML ou WSDLs incluídos pelo WSDL atual são especificados e válidos. Além disso, todos os locais para esquemas XML ou WSDLs aninhados (esquemas XML ou WSDLs incluídos subsequentemente pelos WSDLs ou esquemas XML incluídos) são especificados e válidos.

    • Inválida

      • Um ou mais locais para esquemas XML ou WSDLs incluídos pelo WSDL atual não foram especificados ou são inválidos. Além disso, o local para um ou mais esquemas XML ou WSDLs aninhados (esquemas XML ou WSDLs incluídos subsequentemente pelos WSDLs ou esquemas XML incluídos) talvez não esteja especificado ou seja inválido. Para resolver um esquema XML ou WSDL inválido, clique em Editar Referências. Consulte Resolvendo Referências WSDL Não Resolvidas.

  4. Clique em View de texto para exibir uma view de texto dos detalhes do WSDL. A view de texto inclui as informações mostradas na Tabela 6-4.

    5. Tabela 6-4 - Detalhes do Texto WSDL
    Propriedade
    Descrição
    Namespace de Destino
    O namespace usado para qualificar qualquer uma das definições incluídas no WSDL.
    Texto
    O texto deste WSDL.

  5. Para fazer uma alteração nos campos, clique em Editar. Consulte Adicionando WSDLs para obter descrições dos campos.
  6. Faça as edições apropriadas.
  7. Clique em Salvar para confirmar as atualizações na sessão atual.
  8. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
Observação: A página Exibir Detalhes do WSDL também pode incluir um botão Separar Callbacks. Esse botão só é exibido se o WSDL tiver tipos de porta com operações de callback que possam ser regravados. Se você clicar neste botão, o WSDL será regravado.
