Adicionando Documentos XML

A forma mais fácil de criar Documentos XML no Oracle Service Bus é importando os mesmos. Por exemplo, se você importar em alto volume recursos de JCA criados (arquivo .jca, WSDL associado e arquivo de mapeamento TopLink ou EclipseLink), o Oracle Service Bus gerará automaticamente recursos de Documento XML de arquivos de mapeamento e manterá as dependências entre os arquivos de recursos.

Para obter mais informações sobre importação, consulte Carregando Recursos de um Arquivo Zip, Carregando Recursos de um URL e Importando Recursos.

Se você não importar em alto volume, poderá criar Documentos XML manualmente, conforme descrito nas etapas a seguir:

  1. Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
  2. Selecione o Project Explorer, depois selecione um projeto ou pasta no qual adicionar o Documento XML. A página View de Projeto/Pasta é mostrada.
  3. No campo Criar Recurso, selecione Documento XML para exibir a página Criar um Novo Documento XML.
  4. No campo Arquivo XML, clique em Procurar e selecione o arquivo XML que deseja usar.

    5. O campo Nome do Recurso é preenchido automaticamente com o nome do arquivo (menos a extensão do arquivo).

    Em vez de selecionar um arquivo, é possível copiar e colar o conteúdo de um arquivo XML na área de texto do campo Arquivo XML.

  5. No campo Nome do Recurso, informe ou altere o nome do documento XML. Os nomes de Documento XML devem ser exclusivos.

    Siga as Restrições para Nomeação de Recursos para obter orientações em relação a nomes.
  6. No campo Descrição do Recurso, informe uma descrição para o Documento XML.
  7. Clique em Salvar. O novo Documento XML é salvo na sessão atual.
  8. Para encerrar a sessão e implantar a configuração no run-time, clique em Ativar em Centro de Alterações.
