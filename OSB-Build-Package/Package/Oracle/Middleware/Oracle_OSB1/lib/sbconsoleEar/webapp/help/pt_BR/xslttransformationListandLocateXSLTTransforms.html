Localizando Transformações XSL

  1. Execute um dos seguintes procedimentos:
    • Selecione o Project Explorer para exibir a página View de Projetos ou a página View de Projeto/Pasta. Em seguida, navegue pelos projetos e pastas para localizar a transformação XSL.
    • Selecione Resource Browser > XSLTs. A página Resumo de XSLTs exibe as informações mostradas na Tabela 14-1.
  2. Para restringir o número de itens na lista, você pode filtrar por nome, caminho ou ambos. Nos campos Nome e Caminho, em Pesquisar, informe o nome e/ou o caminho do(s) destino(s) e, em seguida, clique no botão Pesquisar.

    3. O caminho é o nome do projeto e o nome da pasta em que reside o XSLT.

    Caracteres curingas * e ? são permitidos. A pesquisa faz distinção entre maiúsculas e minúsculas.

    Clique em Exibir Tudo para exibir todos os XSLTs no domínio. Isso limpa os parâmetros de pesquisa da pesquisa anterior.

    Tabela 14-1 - Detalhes da Transformação XSL
    Propriedade
    Descrição
    Nome XSLT
    O nome designado à transformação XSL. Clique no nome para exibir a página Exibir Detalhes da Transformação XSL.
    Consulte Editando Transformações XSL.
    Caminho
    O caminho é o nome do projeto e o nome da pasta em que reside a transformação XSL, por exemplo, MyProject/MyFolder/MyResource.
    Clique no caminho de um XSLT para exibir a página View de Projeto/Pasta que o contém.
    Ações
    Clique no ícone Acionar Console de Teste para chamar a Console de Teste, que você usa para validar e testar o design de seus serviços e transformações. Para transformações XSL, você pode usar a Console de Teste estando dentro ou fora de uma sessão. Para saber mais, consulte Testando Transformações.
    Opções
    Se ainda não tiver feito isso, clique em Criar para criar uma nova sessão ou clique em Editar para informar uma sessão existente. Consulte Usando o Centro de Alterações.
    Clique no ícone Deletar para deletar o XSLT. Um ícone Advertência de Deleção é exibido quando outros recursos referenciam esse recurso. É possível deletar o recurso com uma confirmação de advertência. Isso pode resultar em conflitos devido a referências não resolvidas referentes ao recurso deletado. Para obter mais informações, consulte Deletando uma Transformação XSLT.
