添加电子邮件收件人

在添加电子邮件目标之前，必须配置 SMTP 服务器 (请参阅添加 SMTP 服务器)，或者配置 WebLogic Server 中的 JavaMail 会话 (请参阅 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的“Configure access to JavaMail”，网址为： http://download.oracle.com/docs/cd/E15523_01/apirefs.1111/e13952/taskhelp/mail/CreateMailSessions.html)。如果没有已配置的可用 SMTP 服务器资源或 JavaMail 会话，则不能配置电子邮件收件人。

  1. 从下拉列表中选择“通过电子邮件发送预警”，然后单击“添加”以便显示“添加电子邮件收件人”页。
  2. 在“邮件收件人”字段中，以 mailto:username@hostname 格式输入电子邮件收件人。

    3. 这是必填字段。

    通过在逗号分隔的列表中输入用户名和主机名，可指定多个电子邮件收件人。例如，mailto:username@hostname [,username_1@hostname_1]...[,username_n@hostname_n]

    仅第一个邮件收件人需要加前缀文本“mailto:”。

  3. 在“SMTP 服务器”字段中，选择传出电子邮件的 SMTP 服务器的名称。如果要在下一步选择邮件会话，则不必填写此字段。
  4. 在“邮件会话”字段中，选择一个可用的邮件会话。如果已在上一步选择了 SMTP 服务器，则不必填写此字段。
  5. 在“发件人姓名”字段中，提供用于预警通知的发件人姓名。此字段是可选的。
  6. 在“发件人地址”字段中，提供有效的电子邮件地址。如果指定了“发件人姓名”字段的值，则需要填写此字段。
  7. 在“回复姓名”字段中，提供可将回复发送到的对象的姓名。此字段是可选的。
  8. 在“回复地址”字段中，提供可将回复发送到的电子邮件地址。如果指定了“回复姓名”字段的值，则需要填写此字段。
  9. 在“连接超时”字段中，输入超时前连接必须等待来自服务器的响应的秒数。默认值为 0。
  10. 在“请求编码”字段中，输入字符集编码值。默认的编码值是 iso-8859-1。
  11. 单击保存。电子邮件收件人将添加到此预警目标的电子邮件收件人列表中，并保存在当前会话中。
