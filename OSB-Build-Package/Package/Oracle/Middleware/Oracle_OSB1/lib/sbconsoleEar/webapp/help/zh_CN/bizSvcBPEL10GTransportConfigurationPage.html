BPEL-10g 传输配置页

使用此页可为使用 BPEL-10g (Oracle BPEL Process Manager) 传输协议的业务服务配置传输设置。有关将 Oracle Service Bus 与 Oracle BPEL Process Manager 配合使用的详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Oracle BPEL Process Manager Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1147。

属性
说明
角色
BPEL 传输用于从 Oracle Service Bus 向 Oracle BPEL Process Manager 发送请求消息。业务服务可服务下列角色之一：
  • 同步客户机
    用于与 Oracle Service Bus 客户机的同步通信，需要的唯一位置信息是 BPEL 地址。此地址静态捕获为端点 URI 和/或通过 URI 重新写入动态捕获。
  • 异步客户机
    用于与 Oracle Service Bus 客户机的异步通信，从 Oracle BPEL Process Manager 向 Oracle Service Bus 的回调通过不同于请求的连接发送，您必须配置 Oracle Service Bus 提供正确的回调地址。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Advanced Use Cases (Asynchronous)”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163。
  • 服务回调
    如果业务服务旨在作为 Oracle BPEL Process Manager 的服务回调（此情况下，Oracle BPEL Process Manager 通过 Oracle Service Bus 调用外部服务），则回调地址仅在运行时可知。使用端点 URI bpel://callback。

    • 如果为业务服务配置标记 URI，请配置管道逻辑以动态设置 $outbound 的 URI；例如，使用 TransportHeader 操作。

    注： 服务回调业务服务不支持负载平衡或故障转移。
回调代理
此可选字段仅适用于异步客户机角色。通过此字段可选择用于将回调路由到发出请求的 Oracle Service Bus 客户机的代理服务 (必须为 SB 代理或任何 SOAP 类型的 HTTP 代理)。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Advanced Use Cases (Asynchronous)”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163。
服务帐户
对于 JNDI 上下文安全性，用于访问 Oracle BPEL Process Manager 传送服务。单击“浏览”并选择服务帐户。如果未指定任何服务帐户，则将使用匿名主题。
对可配置的服务帐户的类型（如静态或通过）没有限制，但是运行时必须能够访问用户名和口令。
挂起事务处理
选择“挂起事务处理”可使业务服务成为非事务处理性服务，即使业务服务是由事务处理调用的。
如果不选择“挂起事务处理”：
  • 如果协议表示支持 WebLogic Server 的协议（t3、iiop 和 http），则传播事务处理。
  • 如果协议表示 OC4J 服务器（ormi 和 opmn），则 BPEL 传输将引发异常错误，因为 OC4J 不支持事务处理传播。
在下列情况下，BPEL 传输将忽略“挂起事务处理”选项：
  • 以“最佳效果”的服务质量 (QoS) 调用业务服务。BPEL 传输自动挂起不支持 QoS 的所有事务处理。
  • 以“恰好一次”的 QoS 调用业务服务且没有事务处理。
有关事务处理传播的说明，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Transaction Propagation”，网址为： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1151。
分派策略
选择要对此端点的分派策略使用的 WebLogic Server 工作管理器实例。如果不存在其他工作管理器，则使用默认工作管理器。
有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
