“EJB 传输配置”页

创建/编辑业务服务 - EJB 传输配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

使用此页可为使用 EJB 传输协议的业务服务配置传输设置。表 18-5 介绍了如何使用此页。

表 18-5“创建/编辑业务服务 - EJB 传输配置”页
选项
创建或编辑...
传递调用者的主题
选中此复选框，可在调用 EJB 和未配置服务帐户时让 Oracle Service Bus 传递代理服务的经过验证的主题。请注意，选择此选项时，将禁用服务帐户字段。
服务帐户
单击浏览，然后从显示的列表中选择服务帐户。如果未指定任何服务帐户，则将使用匿名主题。如果使用了“传递调用者的主题”选项，则此选项不可用。

详细信息：

  • 服务帐户
  • 添加服务帐户
支持事务处理
选中此复选框可指定事务处理支持。
客户机 Jar
单击浏览，然后从显示的列表中选择 EJB 客户机 JAR 资源。要了解有关创建 JAR 资源的信息，请参阅 JAR。这是必填字段。
转换程序 JAR
单击浏览，然后从显示的列表中选择 EJB 转换程序类 JAR 资源。要了解有关 EJB 客户机 JAR 资源和转换程序类的详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“EJB Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009。
主接口
仅限 EJB 2.1 - 从由 JAR 填充的选项中选择所需的 EJBHome 接口。此 URI 示例中的 JNDI 名称必须与您在此处选择的 EJBHome 接口相关联。如果 EJB 不是所需的类型，或者未在客户机 JAR 中指定 EJBHome 接口，则 Oracle Service Bus 将显示警告。
远程接口
仅限 EJB 2.1 - 此字段是根据主接口的配置自动填充的。
业务接口
仅限 EJB 3.0 - 在要使用的客户机 JAR 中选择业务接口。
目标名称空间
此字段是由从 JAR 中选取的信息填充的。
样式
根据需要，选择包装文档或 RPC。如果无状态会话 EJB 的两个或多个方法具有相同的参数个数和参数数据类型，并且您希望操作是面向文档的，则必须指定它们为包装文档。
由于在路由或发布到 EJB 时，$body 的内容必须与样式相匹配，因此样式很重要。此外，在调用到 EJB 时，样式将会影响参数内容，尤其是包装文档。其次，有一种使用模式是定义 EJB 业务服务，然后使用路由到 EJB 的同一 WSDL 创建代理服务。在这种情况下，鉴于用来调用代理的客户机工具可能对 WSDL 的样式有所限制，因此在使用 WSDL 的样式时必须谨慎。
编码
选择已编码或文字。
方法
显示的方法是所选 EJB 远程接口或业务接口的方法。选择所需方法 (可以选择多个方法)。单击“+”可展开方法。编辑默认参数值并选择转换程序 (如果已提供或需要的话)。
必须排除带有 JAX-RPC 堆栈不支持的参数或返回类型的方法，或者必须将此类变量与转换程序类关联。
可以更改指定方法的默认操作名。(默认情况下，操作名是方法名。)如果 EJB 包含同名的方法，必须更改操作名以使其唯一。WSDL 需要唯一的操作名。
注：如果在指定的 EJB 的方法之间，身份证明或事务处理设置不相同，则可以为指定业务服务定制方法并为每个方法创建业务服务。通过此功能可以对事务处理和身份证明进行细粒度控制。
异常错误
方法引发业务异常错误时显示此字段。如果 EJB 方法引发的异常错误具有 Java Web Services (JWS) 不支持的数据类型 (如 ArrayList)，则使用“异常错误”字段选择将该异常错误转换为 JWS 支持的类型的转换程序类。
转换程序类必须实施 com.bea.wli.sb.transports.ejb.ITypeConverter。只能为已检查异常错误配置转换程序类，不能为运行时异常错误配置转换程序类。
将客户机或转换程序 JAR 中的转换程序类和已转换异常错误类打包。
有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“EJB Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑业务服务 - 概要”页上查看并保存此配置。
