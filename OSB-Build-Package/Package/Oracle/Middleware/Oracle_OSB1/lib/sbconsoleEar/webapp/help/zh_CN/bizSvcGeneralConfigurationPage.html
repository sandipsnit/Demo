“一般配置”页

创建/编辑业务服务 - 一般配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

使用此页可修改业务服务的一般配置设置。

创建业务服务时，这是一系列用于配置该服务的页中显示的第一页。在该页之后显示的页面因在此页上做出的选择而异。

表 18-1 介绍了如何使用此页。

表 18-1“创建/编辑业务服务 - 一般配置”页
选项
创建或编辑...
服务名
输入业务服务的唯一名称。

如需命名指导，请参阅资源命名限制。
说明
输入说明。
服务类型
服务类型定义了服务所交换的消息的类型和打包。选择要创建的业务服务的类型：
  • WSDL Web 服务 - 选择此选项可创建基于 WSDL 的业务服务。然后，输入 WSDL 名称。该名称由其路径限定，例如 myProject/myFolder/myWSDL。或者，单击浏览从选择 WSDL 页中选择 WSDL。

    • (端口或绑定) - 输入端口的名称 (在 WSDL 中定义) 以描述实际传输地址，或输入绑定的名称 (在 WSDL 中定义) 以映射到传输地址。如果按上述方式使用浏览来选择 WSDL，则选择 WSDL 定义页将列出在 WSDL 中定义的所有端口和绑定。在该页上选择端口或绑定后，将以所选名称填充 (端口或绑定) 字段。

  • 传输类型服务 - 使用此选项可创建使用 EJB 或 Flow 传输的服务。
  • 消息传送服务 - 选择此选项可创建用来交换内容类型大相径庭的消息的服务。这些交换可以是“请求/响应”式的或是单向的。就 HTTP 传输而言，当以 HTTP 'GET' 选项使用时，也可以只有响应而无请求。与 Web 服务不同的是，这里请求和响应的内容类型无须相同。
  • 任何 SOAP 服务 - 选择此选项可创建没有明确定义的具体接口的 SOAP 服务。

    • 保留默认值 SOAP 1.1，或者从下拉列表中选择 SOAP 1.2。

  • 任何 XML 服务 - 选择此选项可创建没有明确定义的具体接口的 XML 服务。

    • 仅对消息传送服务和此服务类型支持 HTTP GET。

  • 业务服务 - 选择此选项可克隆现有业务服务。

    • 输入业务服务的路径 (项目/文件夹) 和名称；或单击浏览从业务服务概要页选择业务服务。

    由于 Oracle Service Bus 不接受多项服务具有相同的 URI，因此，必须更改克隆服务的 URI。

  • 代理服务 - 选择此选项可基于现有代理服务创建业务服务。
服务类型 (续)
注： 如果服务是基于其他服务创建的，将按以下方式维护预警规则：
    • 如果代理服务是基于业务服务创建的，或者业务服务是基于代理服务创建的，则会删除预警规则 (如果有)。
    • 如果代理服务是基于其他代理服务创建的，或者业务服务是基于其他业务服务创建的，则会保留预警规则 (如果有)。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑业务服务 - 概要”页上查看并保存此配置。
