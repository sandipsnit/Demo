“HTTP 传输配置”页

HTTP 传输支持 HTTP 端点和 HTTPS 端点。

创建/编辑业务服务 - HTTP 传输配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

HTTP 请求方法
使用此参数可以在请求中使用以下 HTTP 方法之一：
  • POST - 将其所有数据 (长度不受限制) 作为 HTTP 请求正文的一部分直接通过套接字连接进行传递。交换对于客户机是不可见的，且 URL 不会更改。对于基于 REST 的请求，POST 指示传输执行创建/替换操作或者使用请求执行操作。
  • GET - 可将其自身某些信息 (更好地描述获取内容) 包含在请求中。此信息是作为查询字符串中附加到请求 URL 后面的一系列字符进行传递的。当“请求消息类型”设置为“无”时，可以在服务类型为“任何 XML 服务”或“消息传送服务”的业务服务中使用 GET。对于基于 REST 的请求，GET 将检索远程资源的表示形式。
  • PUT - 可以在服务类型为“任何 XML 服务”或“消息传送服务”的业务服务中使用 PUT。PUT 指示传输使用基于 REST 的请求执行创建/替换操作，例如将文件上传到一个已知位置。
  • HEAD - 当“响应消息类型”设置为“无”时，可以在服务类型为“任何 XML 服务”或“消息传送服务”的业务服务中使用 HEAD。HEAD 指示传输获取远程资源的标头信息，而非基于 REST 的请求中资源的完整表示形式。
  • DELETE - 可以在服务类型为“任何 XML 服务”或“消息传送服务”的业务服务中使用 PUT。DELETE 指示传输使用基于 REST 的请求执行删除操作。
注： 如果已在 $outbound/transport/request/http:http-method 变量中设置了方法，则该值优先于您为“HTTP 请求方法”选择的任何方法。
验证
选择以下选项之一：
  • 无 - 指定无须通过验证即可访问此服务。
  • 基本 - 指定访问该服务需要进行基本验证。

    • 基本验证可指示 WebLogic 服务器根据安全领域中所配置的验证提供程序 (如轻型目录访问协议 (LDAP) 目录服务和 Windows Active Directory)，对使用用户名和口令的客户机进行验证。客户机必须在 HTTP 请求标头中发送其用户名和口令。

    强烈建议不要通过 HTTP 进行基本验证，因为口令是以明文形式发送的。但是，通过 HTTPS 发送口令则是安全的，因为 HTTPS 提供了一个加密通道。

    警告：默认情况下，所有用户 (授权用户和匿名用户) 都能访问业务服务。要限制可访问业务服务的用户，请创建传输级授权策略。请参阅编辑传输级访问策略。

  • 客户机证书 - 指定加密的通信和强客户机验证 (双向 SSL)。要了解详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Configuring Transport-Level Security”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557。
服务帐户
输入服务帐户。服务帐户是用户名和口令的别名资源。如果您选择了需要基本验证字段，则这是一个必填字段。

分派策略
选择要对此端点的分派策略使用的 WebLogic Server 工作管理器实例。如果不存在其他工作管理器，则使用默认工作管理器。有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
请求编码
接受默认值 iso-8859-1 作为 HTTP 传输中请求的字符集编码，或者输入其他字符集编码。
响应编码
接受默认值 iso-8859-1 作为 HTTP 传输中响应的字符集编码，或者输入其他字符集编码。
代理服务器
输入代理服务器资源或单击“浏览”从已配置代理服务器资源列表中选择一个条目。
执行 HTTP 重定向
选中此复选框可指定应自动执行 HTTP 重定向 (即带有响应代码 3xx 的请求)。如果向业务服务的 URL 发送出站请求，且该服务返回一个响应代码 (如 302)，表明此 URL 不再有效，需要将此请求发送到另一个 URL，则将发生重定向。
如果选中执行 HTTP 重定向复选框，则 Oracle Service Bus 自动将请求重新发送到新的 URL，而不用您执行任何操作。如果不需要自动执行 HTTP 重定向，请取消选中此复选框。
使用块流模式
如果要使用 HTTP 块传输编码来发送消息，请选择此选项。
注：如果使用“执行 HTTP 重定向”选项，请勿使用块流。在块模式中，无法自动处理重定向和验证。
会话粘连
如果希望对业务服务的 HTTP 请求使用会话粘连 (也称为会话关联性)，请选择此选项。
粘连会话 ID 名称
如果启用了会话粘连，则输入会话的标识名称。
