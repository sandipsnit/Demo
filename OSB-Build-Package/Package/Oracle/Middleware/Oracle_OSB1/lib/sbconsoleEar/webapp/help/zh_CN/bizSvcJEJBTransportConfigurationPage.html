“JEJB 传输配置”页

使用此页可使用 JEJB 传输协议配置传输设置。有关使用 JEJB 传输的详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“JEJB Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036。

表 18-12 JEJB 传输配置
选项
说明
分派策略
选择要对此端点的分派策略使用的 WebLogic Server 工作管理器实例。如果不存在其他工作管理器，则使用默认工作管理器。
有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
EJB 规范版本
选择远程 EJB 接口的 EJB 版本。
按值传递 XMLBean
如果希望传输生成 POJO 变量 (XMLObject) 的“内嵌”XML 表示方式 (您可以使用 XQuery 表达式访问和处理该变量的参数)，请选择此选项。
注：按值传递的 XMLObject 中没有内嵌的类型信息。如果使用此选项，则无法将类型化 XMLObject 在代理服务管道的 Java 标注中作为参数进行传递。
如果要按引用传递 POJO (还会提高性能)，请勿选择此选项。
传递调用者的主题
作为选择“服务帐户”的替代选项，选择此选项可以使 Oracle Service Bus 在调用 EJB 时传递代理服务的已通过验证的主题。
服务帐户
单击浏览，然后从显示的列表中选择 JNDI 服务帐户。如果未指定任何服务帐户，将使用匿名主题。有关详细信息，请参阅第 16 章“服务帐户”。
客户机 JAR
单击浏览并从显示的列表中选择 EJB 客户机 JAR 资源。客户机 JAR 包含远程服务的远程接口或业务接口。客户机 JAR 注册为常规档案资源。
主接口
仅限 EJB 2.1 - 从 JAR 填充的选项中选择所需的 EJBHome 接口。
远程接口
仅限 EJB 2.1 - 此字段是根据主接口的配置自动填充的。
业务接口
仅限 EJB 3.0 - 在要调用的客户机 JAR 中选择业务接口。
目标名称空间
此字段是由从 JAR 中选取的信息填充的。
方法
选择所需的方法。单击 + 以展开该方法，从而可以编辑默认参数值。
可以更改指定方法的默认操作名。默认情况下，操作名是方法名。如果 EJB 包含同名的方法 (超载)，必须更改操作名以使其唯一。WSDL 需要唯一的操作名。

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑业务服务 - 概要”页上查看并保存此配置。
