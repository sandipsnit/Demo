“JMS 传输配置”页

创建/编辑业务服务 - JMS 传输配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“JMS Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037。

使用此页可为使用 JMS 传输协议的业务服务配置传输设置。表 18-13 介绍了如何使用此页。

表 18-13“创建/编辑业务服务 - JMS 传输配置”页 (续)
选项
创建或编辑...
目标类型
选择 JMS 桥接程序目标的类型：
  • 队列 (用于点对点)
  • 主题 (用于发布/订阅)
消息类型
选择以下选项之一：
  • 字节 (用于未解析的字节流)
  • 文本 (用于文本消息)
如果您为响应选择 Java 消息类型，则此选项将禁用。
响应队列
只有在对“目标类型”选择了“队列”后，该选项才可用。
选择以下响应选项之一：
  • 无 - 不希望获得响应。请对单向操作设置此选项。
  • 所有请求 URI 使用一个 - 使用此选项可输入一个 URI 来处理响应、设置其他响应配置详细资料 (如编码和超时)，以及选择用于传递 JMS/JNDI 身份证明的 JMS 服务帐户 (可选)。
  • 每个请求 URI 都有一个 - 此选项提供响应故障转移，并允许您为每个请求 URI 输入响应 URI 或目标。
(可选) 可以为每个请求/响应对中的 JMS/JNDI 身份证明选择一个服务帐户。
响应模式
只有在“响应队列”字段中选择响应选项后，该选项才可用。
选择以下选项之一：
  • 为运行在 Oracle WebLogic Server 上除 JAX-RPC 服务外的所有服务选择 JMSCorrelationID。
  • 为运行在 Oracle WebLogic Server 上的 JAX-RPC 服务选择 JMSMessageID。
有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Message ID and Correlation ID Patterns for JMS Request/Response”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051。
分派策略
选择要对此端点的分派策略使用的 Oracle WebLogic Server 工作管理器实例。如果不存在其他工作管理器，则使用默认工作管理器。
例如，如果业务服务具有 JMS 传输协议，则该业务服务端点是一个 MDB (消息驱动 Bean) JAR 文件，您可以将其与特定的分派策略进行关联。
有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
请求编码
输入编码请求的字符集。默认值为 UTF-8。
响应编码
只有在“响应队列”字段中选择其中一个响应选项后，该选项才可用。
输入编码响应的字符集。默认值为 UTF-8。
响应超时
只有在“响应队列”字段中选择其中一个响应选项后，该选项才可用。
输入在断开连接之前等待响应的时间 (秒)。默认值零 (0) 表示响应永不超时。
客户机 Jar
当服务是具有 Java 响应类型的消息传送服务时，此选项可用。请选择要用于将包含 Java 对象的消息出队的客户机 JAR。选择客户机 JAR 时请确保它位于类路径中。

有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Sending and Receiving Java Objects in Messages”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。
响应 URI
当选择“所有请求 URI 使用一个”响应选项和 JMSCorrelationID 响应模式后，此选项可用。
按照以下格式输入响应 URI：
jms://host:port/connection_factory/jndi_destination
要定位多个服务器，请使用以下格式：
jms://host1:port,host2:port/connection_factory/jndi_destination
您也可以省略响应 URI 中的主机和端口。例如：
jms:///connection_factory/jndi_destination

省略主机和端口时，会在本地服务器上进行连接工厂/目标查找。在以下情况下，这很有用：例如，如果请求 URI 转至外部连接工厂/目标，但您希望响应发送到本地服务器。

注：虽然 Oracle WebLogic Server 允许在 JNDI 名称中使用正斜杠 (例如“myqueues/myqueue”)，但是带正斜杠的 JNDI 名称会对 Oracle Service Bus 所需的 URI 格式产生干扰，因此您不能使用这些名称。要解决此问题，请定义一个 JMS 外部服务器，然后在 URI 中引用该外部服务器。有关详细信息，请参阅 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的“Configure foreign servers”，网址为：http://www.oracle.com/pls/as111170/lookup?id=WLACH。

请求响应
当对 JMSCorrelationID 模式选择“每个请求 URI 都有一个”响应选项以提供响应故障转移后，此选项可用。
对于在一般“传输”页上输入的每个请求 URI，输入一个响应 URI，如先前的“响应 URI”字段所述。
对于服务用于请求和响应队列的 JMS/JNDI 身份证明，可以选择一个可选服务帐户。
目标 - 响应
当对 JMSMessageID 模式选择“所有请求 URI 使用一个”响应选项后，此选项可用。
输入要接收响应的目标服务器的名称，并输入响应 URI，如“响应 URI”字段所述。
请求连接
当对 JMSMessageID 模式选择“每个请求 URI 都有一个”响应选项以提供响应故障转移后，此选项可用。
对于“序列号”字段中按数字顺序标识的每个请求 URI，可选择输入一个 JMS 连接工厂名称。如果不输入名称，则 JMS 传输将使用请求 URI 中的连接工厂。
对于服务用于请求和响应队列的 JMS/JNDI 身份证明，可以选择一个可选服务帐户。
目标 - 目的地
当对 JMSMessageID 模式选择“每个请求 URI 都有一个”响应选项后，此选项可用。将此字段与“请求连接”一起使用。
列出的每个目标服务器 (由当前域中的服务器确定，如集群中的托管服务器) 都按序列号列出请求 URI，序列号与“请求连接”字段中的编号相对应。对于每个目标上的每个请求 URI，输入接收响应的那个目标服务器上的目标队列。
注：由于 Oracle Service Bus 开发环境只支持单服务器环境，因此只列出一个目标。要在多服务器环境中配置此字段，请将此服务部署到运行时环境，然后在 Oracle Service Bus 管理控制台中完成服务配置。
JMS 服务帐户
在“响应队列”字段中使用“无”或“所有请求 URI 使用一个”选项后，此选项可用。
选择要用于 JMS 服务器管理的 JMS 资源的服务帐户。服务帐户是用于请求和响应的用户 ID 和口令的别名资源。对于 JMS 和 JNDI 使用相同的服务帐户。
此字段与“传递调用者的主题”选项互斥。请使用两者之一进行 JMS/JNDI 验证。
有关详细信息，请参阅服务帐户。
使用 SSL
只有通过 TLS/SSL 连接创建请求后，才能选择此选项。
TLS/SSL (安全套接字层) 通过允许网络上连接的两个应用程序对对方的标识进行验证，并通过对应用程序之间交换的数据进行加密，来提供安全连接。验证操作使服务器 (还可以使客户机) 能够验证网络连接另一端应用程序的标识。另外，如果管理员通过对目标的 JNDI 条目设置访问控制，限制了对各个 JMS 目标 (队列或主题) 的访问权限，则当此业务服务在 JNDI 树中查找此条目时，必须进行服务验证。使用“服务帐户”或“传递调用者的主题”选项进行验证。
失效
消息失效之前的时间间隔 (毫秒)。默认值为 0，这表示消息永不失效。
启用消息持久性
服务使用的 JMS 传送模式，您可以使用此模式通过吞吐量平衡可靠性。对可靠消息传送选择此选项 (默认值)。如果消息偶尔丢失是可接受的，则取消选择此选项可提高吞吐量。
顺序单元
输入消息顺序单元。使用消息顺序单元，消息生成器可以根据处理顺序将消息归组到一个单元中。此单个顺序单元要求此单元中的所有消息都按照其创建顺序进行处理。
传递调用者的主题
选中此复选框，可在发送消息时让 Oracle Service Bus 传递经过验证的主题。
在不同域中启用此字段和业务服务目标 JMS 资源时，请在这两个域中启用全局信任。请参阅 Securing Oracle WebLogic Server 中的“Configuring Security for a WebLogic Domain”。
此字段与“服务帐户”选项互斥。请使用两者之一进行 JMS/JNDI 验证。
在不同域中启用此字段和业务服务目标 JMS 资源时，请在这两个域中启用全局信任。请参阅 Oracle Fusion Middleware Securing Oracle WebLogic Server 中的“Configuring Security for a WebLogic Domain”，网址为： http://www.oracle.com/pls/as111170/lookup?id=SECMG402。
JNDI 超时
在 JNDI 树中查找目标或连接工厂时使用的 JNDI 连接超时 (秒)。
默认值零 (0) 表示连接永不超时。
