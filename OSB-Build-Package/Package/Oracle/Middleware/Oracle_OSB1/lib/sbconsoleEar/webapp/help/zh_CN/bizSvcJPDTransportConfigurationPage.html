JPD 传输配置页

“创建/编辑业务服务 - JPD 传输配置”页是一系列用于创建和编辑业务服务的页面中的一个页面，如创建和配置业务服务和编辑业务服务配置所述。

使用此页可为使用 JPD 传输协议的业务服务配置传输设置。表 17-14 介绍了如何使用此页。

表 17-14“创建/编辑业务服务 - JPD 传输配置”页 (续)
选项
创建或编辑...
系统服务帐户
输入用户的服务帐户，或单击“浏览”从显示的列表中选择一个服务帐户。
系统服务帐户指定用于调用 WLI 无状态会话 Bean（JPD 用来发送传入消息）的用户身份证明。
如果未指定任何服务帐户，则将使用入站请求主题。如果没有任何入站请求主题，则将使用匿名主题。
流程服务帐户
输入用户的服务帐户，或单击“浏览”从显示的列表中选择一个服务帐户。
流程服务帐户指定用于调用 JPD 的用户身份证明。
如果未指定任何服务帐户，则将使用入站请求主题。如果没有任何入站请求主题，则将使用匿名主题。
回调代理位置
输入用于从 JPD 接收回调的代理服务位置，以下列格式指定：
jms://[host:port[,host:port]*]/MyFactory/MyQueue
其中：
  • host：（可选）配置了回调代理服务的 Oracle Service Bus 服务器的主机名
  • port：（可选）Oracle Service Bus 用于监听传入请求的端口号
  • MyFactory：队列连接工厂的 JNDI 名称
  • MyQueue：Oracle Service Bus 代理服务等待回调请求时所在队列的 JNDI 名称
在集群环境中，指定受管服务器的主机名和端口号，以逗号 (,) 分隔。
如果要调用的 JPD 是同步 JPD 或没有回调的异步 JPD，则将忽略回调代理位置。
分派策略
为出站响应选择默认分派策略或已配置的工作管理器实例。有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111120/lookup?id=CNFGD112。
传播事务处理
选中该复选框可将 Oracle Service Bus 中的现有事务处理传播到 JPD。

完成之后

单击“下一步”继续在下一页上配置此服务；或单击“最后一步”在创建/编辑业务服务 - 概要页上查看并保存此配置。
