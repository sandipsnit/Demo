“MQ 传输配置”页

创建/编辑业务服务 - MQ 传输配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“MQ Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117。

开始之前

配置 MQ 连接资源。请参阅 MQ 连接。

配置 MQ 传输

使用此页可为使用本机 MQ 传输协议的业务服务配置传输设置。表 18-14 介绍了如何使用此页。

表 18-14“创建/编辑业务服务 - MQ 传输配置”页
选项
创建或编辑...
消息类型
选择以下选项之一：
  • 字节 (用于未解析的字节流)
  • 文本 (用于文本消息)
是否需要响应
选择此选项可指定希望在发送出站消息之后获得响应。
轮询间隔
只有在选中是否需要响应复选框后，该选项才可用。
输入轮询间隔 (毫秒)。默认值为 1000。
响应相关性模式
只有在选中是否需要响应复选框后，该选项才可用。
指定响应相关性模式是否应基于以下项目：
  • MessageID
  • CorrelationID
  • 动态队列 - 如果 WebSphere MQ 实施对响应相关性使用动态队列，请选择此选项。MQ 传输只支持临时的动态队列。
自动生成相关性值
只有在选中是否需要响应复选框后，该选项才可用。
选中此复选框可自动生成 CorrelationID 或 MessageID。
模型队列
仅适用于动态队列响应相关性模式。输入用于生成动态队列的模型队列的名称。
MQ 响应 URI
只有在选中是否需要响应复选框后，该选项才可用。
应将响应发布到的目标。输入响应 URI，其格式应与端点 URI 的格式相同：
mq://local_queue_name?conn=mq_connection_resource
或者，如果使用的是动态队列：
mq://dynamic_queue_prefix?conn=mq_connection_resource
dynamic_queue_prefix (限制为最多 32 个字符) 用于在 MQ 服务器上创建动态队列。队列名称将为前缀加唯一 ID。例如，如果 dynamic_queue_prefix 是 example，则动态队列的名称将是类似于 example123129083821 这样的名称。
您也可以在动态队列响应 URI 中使用星号 (*) 作为通配符。例如：
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
如果未在 URI 中提供 dynamic_queue_prefix，传输会使用 MQ 服务器生成的动态队列名。如果未在 URI 中提供 mq_connection_resource (最佳实践)，传输会使用端点 URI 的 mq_connection_resource。
有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“MQ Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117。
响应超时
只有在选中是否需要响应复选框后，该选项才可用。
输入在删除连接之前要等待的秒数。
分派策略
分派策略是指您希望用于此服务端点的 WLS 工作管理器实例。
有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
动态队列共享
仅适用于动态队列响应相关性模式。如果要使服务使用动态队列的池式连接，请选择此选项。
如果要对动态队列使用单独的连接池，请考虑针对动态队列配置专用 MQ 连接资源。
如果要在每个请求上创建新的动态队列实例 (并在响应后销毁该队列)，请勿选择此选项。
端点 URI 'PUT' 选项
从以下选项中进行选择以输入 MQ PUT 消息选项：
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
可以使用“|”或“+”分隔多个选项。例如，可以指定以下内容：
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
当消息被置于出站队列中时，将应用 MQ PUT 消息选项。
MQ 无法识别的响应 URI
输入代表无法识别的响应消息应发送到的队列的 URI。请注意，仅当选中自动生成相关性值复选框时，才会启用此设置。
如果未为此字段指定值，将删除无法识别的响应消息。
处理 RFH2 标头
选择此选项可对消息有效负载的 WebSphere MQ RFH2 标头进行语法分析，并自动生成包含 RFH2 数据的 RFH2Headers 传输标头。
如果未选择此选项，则会传递收到的有效负载。

有关 MQ 传输如何处理 RFH2 标头的信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“About RFH2 Headers”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑业务服务 - 概要”页上查看并保存此配置。
