“消息类型配置”页

创建/编辑业务服务 - 消息类型配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

使用此页可为类型为消息传送服务的业务服务配置消息类型。

消息传送服务的绑定定义包括配置要交换的消息的内容类型。响应的内容类型无须与请求的内容类型相同；因此响应是单独配置的 (例如，服务可能会接受 MFL 消息，而返回 XML 接收确认)。

注： 类型为消息传送服务的电子邮件、File、FTP 或 SFTP 传输业务服务只支持单向消息传送；响应消息类型应为无。如果选择无以外的其他选项，则“传输配置”页上便不提供文件、FTP 或 SFTP 协议。

表 18-2 介绍了如何使用创建/编辑业务服务 - 消息类型配置页。

表 18-2“创建/编辑业务服务 - 消息类型配置”页
选项
创建或编辑...
请求消息类型
选择请求消息的消息类型：
  • 无 - 如果没有任何请求消息 (例如 HTTP GET)，请选择此选项。
  • 二进制 - 如果消息的内容类型未知或者不重要，请选择此选项。
  • 文本 - 如果可将消息限制为文本，请选择此选项。
  • MFL - 如果消息是一个符合 MFL 定义的二进制文档，请选择此选项。您只能配置一个 MFL 文件。

    • 对于 MFL，可以单击浏览从 MFL 浏览器中选择一个 MFL，然后单击提交。

  • XML - 如果消息是 XML 文档，请选择此选项。要提供一些类型信息，您可以选择声明所交换的 XML 文档的 XML 方案类型。
  • Java - 如果 Java 对象是通过请求发送的，请选择此选项。JMS 传输用于 Java 对象消息。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Sending and Receiving Java Objects in Messages”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。
响应消息类型
选择响应消息的消息类型：
  • 无 - 如果没有任何响应消息，请选择此选项。
  • 二进制 - 如果消息的内容类型未知或者不重要，请选择此选项。
  • 文本 - 如果可将消息限制为文本，请选择此选项。
  • MFL - 如果消息是一个符合 MFL 定义的二进制文档，请选择此选项。您只能配置一个 MFL 文件。

    • 对于 MFL，可以单击浏览从 MFL 浏览器中选择一个 MFL，然后单击提交。

  • XML - 如果消息是 XML 文档，请选择此选项。要提供一些类型信息，您可以选择声明所交换的 XML 文档的 XML 方案类型。
  • Java - 如果 Java 对象是通过响应接收的，请选择此选项。JMS 传输用于 Java 对象消息。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Sending and Receiving Java Objects in Messages”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑业务服务 - 概要”页上查看并保存此配置。
