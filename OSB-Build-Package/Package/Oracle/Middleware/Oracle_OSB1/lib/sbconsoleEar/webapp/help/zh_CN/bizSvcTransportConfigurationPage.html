“传输配置”页

创建/编辑业务服务 - 传输配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

使用此页可为业务服务选择传输协议，并可设置其他传输配置设置。

出站传输级安全性应用于 Oracle Service Bus 代理服务和业务服务之间的连接。有关传输级安全性的详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Configuring Transport-Level Security”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557。

选项
创建或编辑...
协议
从列表中选择传输协议。可用的协议因要创建的服务类型而异：
  • WSDL Web 服务：soa-direct、bpel-10g、dsp、http、jca、jms、sb、ws
  • 传输类型服务：ejb、flow、jejb
  • 消息传送服务：电子邮件、File、ftp、http、jms、mq (如果可用)、sftp、tuxedo
  • 任何 SOAP 服务：dsp、http、jms、sb
  • 任何 XML 服务：dsp、电子邮件、file、ftp、http、jms、mq (如果可用)、sb、sftp、tuxedo
负载平衡算法
选择以下负载平衡算法之一：
  • 循环法 - 此算法针对该业务服务，对端点 URI 字段中输入的 URL 进行动态排序。如果第一个 URL 失败，则会尝试下一个 URL，依此类推，直到尝试了指定的重试计数为止。

    • 对于每个新消息，都有一个新的 URL 顺序。

  • 随机 - 此算法针对该业务服务，对端点 URI 字段中输入的 URL 列表进行随机排序。如果第一个 URL 失败，则会尝试下一个 URL，依此类推，直到尝试了指定的重试计数为止。
  • 随机加权 - 此算法针对该业务服务，对端点 URI 字段中输入的 URL 列表进行随机排序，但是某些 URL 的重试次数要比其他 URL 多，具体取决于您在权重字段中输入的值。
  • 无 - 此算法针对该业务服务，对端点 URI 字段中输入的 URL 列表进行从上到下的排序。
端点 URI
根据在协议字段中选择的传输协议的格式，输入端点 URL。以下为各个传输的 URI 格式的说明。可选的 URI 元素显示在方括号 [] 中。有关传输 URI 的详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的各个传输章节，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV。
  • bpel-10g - protocol://host[:port] [/protocol-path]/domain/process[/version[/partnerlink/role]]
  • dsp - t3://dsp-ip-address:port/dsp-app-name
  • ejb - ejb:jndi_provider_name:ejb_jndi_name

    • 在 URI 中，jndi_provider_name 是 JNDI 提供程序资源的名称，ejb_jndi_name 是 EJB 的 JNDI 服务器中的 JNDI 名称。

    如果 JNDI 提供程序位于同一台服务器上，则无需指定 JNDI 提供程序。URI 将是 ejb::ejb_jndi_name。

    注：如果 EJB 运行于 IBM WebSphere 上，则 ejb_jndi_name 的格式必须为：

    cell/nodes/node_name/servers/server_name/ejb_jndi_name

    或

    cell/clusters/cluster_name/ejb_jndi_name

    如 http://publib.boulder.ibm.com/infocenter/wasinfo/v6r1/index.jsp?topic=/com.ibm.websphere.nd.iseries.doc/info/iseriesnd/ae/rnam_example_prop3.html 上的 IBM WebSphere 文档中所述。

  • 电子邮件 - 可以使用以下格式的任意组合输入多个端点 URI：
    • mailto:name@domain_name.com
    • mailto:name@domain_name.com?smtp=smtp_server_resource
    • mailto:name@domain_name.com?mailsession=jndi_mail_session
  • file - file:///drivename:/somename
  • flow - flow:<拆分-联接资源的引用路径>

    • 拆分-联接资源的引用路径是拆分-联接的路径 (项目/文件夹) 和名称，例如 batchorderProcessing/batchorder。

  • ftp - ftp://host:port/directory
  • http - http://host:port/someService

    • HTTP 传输支持 HTTP 端点和 HTTPS 端点。

  • jca - jca://<resource_adapter_jndi>
  • jejb - jejb:jndi_provider_name:ejb_jndi_name

    • EJB 2.1 示例：jejb:myProvider:osb.jejb.myJejbBiz21

    EJB 3.0 示例：jejb:myProvider:myBiz31#osb.jejb.myJejbBiz

    注：如果 EJB 运行于 IBM WebSphere 上，则 ejb_jndi_name 的格式必须为：

    cell/nodes/node_name/servers/server_name/ejb_jndi_name

    或

    cell/clusters/cluster_name/ejb_jndi_name

    如 http://publib.boulder.ibm.com/infocenter/wasinfo/v6r1/index.jsp?topic=/com.ibm.websphere.nd.iseries.doc/info/iseriesnd/ae/rnam_example_prop3.html 上的 IBM WebSphere 文档中所述。

  • jms - jms://host:port/factoryJndiName/destJndiName

    • 要将一个 JMS 目标定位到多个服务器，请使用以下 URI 格式：
    jms://host1:port,host2:port/QueueConnectionFactory/DestName

    注：虽然 Oracle WebLogic Server 允许在 JNDI 名称中使用正斜杠 (例如“myqueues/myqueue”)，但是带正斜杠的 JNDI 名称会对 Oracle Service Bus 所需的 URI 格式产生干扰，因此您不能使用这些名称。要解决此问题，请定义一个 JMS 外部服务器，然后在 URI 中引用该外部服务器。有关详细信息，请参阅 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的“Configure foreign servers”，网址为：http://www.oracle.com/pls/as111170/lookup?id=WLACH。

    在集群中：JMS URI 中的主机名必须与在 WebLogic 服务器中配置的集群服务器主机名完全匹配。

端点 URI (续)
  • mq - mq://<local-queue-name>?conn=<mq-connection-resource-ref>

    • local-queue-name 是业务服务从中读取消息的 MQ 队列的名称。

    mq-connection-resource-ref 是 MQ 连接资源的路径 (项目/文件夹) 和名称；例如，default/my_MQconnection。

    注： 端点 URI 不能包含空格，因此不要创建名称中带有空格的 MQ 连接资源或项目/文件夹。

    要使 MQ 传输可在 Oracle Service Bus 中使用，请参阅 MQ 连接。

  • sb - sb://[<jndi_provider_name/>]service_name

    • jndi_provider_name (可选) 是 Oracle Service Bus JNDI 提供程序资源的名称。如果忽略，则会使用默认上下文。

    service_name 是目标服务，与远程代理服务 URI 相对应。

  • sftp - sftp://hostname:port/directory
  • soa-direct - protocol://authority/default/compositeName[!versionNumber[*label]]/serviceName

    • 当 SOA 服务与 Oracle Service Bus 都位于同一服务器上时，protocol 和 authority 是可选的。

  • tuxedo - tuxedo:resourcename[/remotename]

    • 在 URI 中，resourcename 对应于 WTC 导入名，remotename 对应于由远程 Tuxedo 域导出的服务名。URI resourcename 是必需的，remotename 是可选的。

    如果指定了多个 URI，则必须使各端点具有唯一的资源名。如果未指定远程名，则其值为资源名的值。如果未输入远程名，或者远程名与资源名相同，则只允许指定一个 URI。在这种情况下，资源名和远程名将具有相同的值。这将允许已定义的 WTC 导入使用 WTC 负载平衡和故障转移。

  • ws - http://host:port/somename or https://host:port/somename
单击添加可添加一个或多个附加 URI。在运行时，URL 是根据您在负载平衡算法字段中选择的负载平衡算法来选择的。
如果您在负载平衡算法字段中选择了随机加权，则还可以在端点 URI 字段中输入一个权重。默认值为 1。
如果定义了多个端点，并在负载平衡算法字段中选择了无，则端点的顺序就会非常重要。您可以使用选项列中的上箭头和下箭头对端点进行重新排序。
Oracle Service Bus 不支持在同一业务服务中使用重复的端点 URI。
重试计数
指定在发送出站请求时发生传送失败的情况下要重试各个 URL 端点的次数；换言之，即故障转移尝试次数。
例如，业务服务具有一个已配置的 URI (U1)，重试次数设置为 3。如果第一次尝试时 U1 失败，则系统将再重试 U1 端点 3 次。
如果业务服务具有 2 个已配置的 URI (U1 和 U2) 且重试计数为 3，则在第一次尝试 (例如，对 U1 的尝试) 失败的情况下，系统将尝试 (故障转移到) 下一个 URI (U2)。如果该尝试也失败，则系统会再进行两次尝试：一次对 U1 的尝试和一次对 U2 的尝试。
重试迭代间隔
指定系统对列表中的所有端点 URI 进行新一轮迭代之前暂停的秒数。
例如，业务服务配置了两个 URI (U1 和 U2)，重试计数为 2，重试迭代间隔为 5 秒。如果第一次尝试 (针对 U1) 失败，则系统将立即尝试 U2。但如果 U2 也失败，则系统将等待 5 秒，然后再次从 U1 开始新一轮重试。
重试应用程序错误
选择是或否。
指定在发送出站请求时发生传送失败的情况下，是否根据应用程序错误 (例如 SOAP 故障) 重试端点 URI。
此字段仅对以下传输可用：HTTP、EJB、JMS、DSP、Tuxedo、SB 和 WS。
