“Tuxedo 传输配置”页

创建/编辑业务服务 - Tuxedo 传输配置页是一系列用于创建和编辑业务服务的页中的一个页面，如创建和配置业务服务和编辑业务服务配置中所述。

有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Tuxedo Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200。

使用此页可为使用 Tuxedo 传输协议的业务服务配置传输设置。表 18-18 介绍了如何使用此页。

表 18-18“创建/编辑业务服务 - Tuxedo 传输配置”页 (续)
选项
创建或编辑...
字段表类
输入描述收到的 FML/FML32 缓冲区的类的名称。FML/FML32-to-XML 转换例行程序使用它们将字段名映射到元素名。这是一个由空格分隔的全限定类名列表。
视图类
输入描述收到或发送的 VIEW/VIEW32 缓冲区的类的名称。VIEW-to-XML 或 VIEW32-to-XML 转换例行程序使用它们将字段名映射到元素名。这是一个由空格分隔的全限定类名列表。
类 Jar
选择一个 JAR 资源，该资源应包含一个 JAR 文件，以及此端点操作所需的 FML/FML32 或 VIEW/VIEW32 类。
远程访问点
从下拉列表中选择一个与导入相关联的远程访问点。该列表中包含在 WTC 中配置的远程访问点。如果没有相关联的远程访问点，则无法创建业务服务。
如果没有任何远程访问点存在，或者要新建远程访问点，请选择新建。在相邻字段中输入相应的访问点名称和网络地址。在验证端点时，会将该访问点添加到每个 WTC 服务器的 WTC 配置中。如果没有任何 WTC 服务器存在，则会创建一个 WTC 服务器。
如果指定了多个 URI，则每个 URI 都将有一个远程访问点字段，且会显示 URI 以进行提示。如果存在多个 URI，则每个 URI 都需要不同的远程访问点。如果指定的 URI 已与现有的 WTC 资源相对应，则会显示对应的远程访问点，但不能对其进行修改。
本地访问点
只有在选择远程访问点字段中的新建后，才会显示此字段。
在下拉列表中，选择要与新建的远程访问点关联的本地访问点。如果不存在任何本地访问点，或者要新建本地访问点，请选择新建。在相邻字段中输入相应的本地访问点名称和本地网络地址。
请求缓冲区类型
选择远程 Tuxedo 服务将接收的缓冲区的类型。
请求缓冲区子类型
如果以前的请求缓冲区类型值为“VIEW”或“VIEW32”，则会启用此选项。输入与请求缓冲区相关联的缓冲区子类型。
是否需要响应？
选中此复选框可指示双向调用。如果未选中，则使用 TPNOREPLY 标记调用底层 tpcall，并异步发布空响应。
是否挂起事务处理？
选中此复选框可挂起事务处理 (如果存在)。这在远程服务不支持事务处理时非常有用。
分派策略
选择要对此端点的分派策略使用的 WebLogic Server 工作管理器实例。如果不存在其他工作管理器，则使用默认工作管理器。
此工作管理器用于在单向调用中异步发布空回复。有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
请求编码
指定 Tuxedo 传输中请求的字符集编码。
响应编码
指定 Tuxedo 传输中响应的字符集编码。
超时
指定传输运行时等待回复的最长时间 (秒)；大于或等于 0 的整数值。如果未指定或设置为 0 (默认值)，则回复将在 BLOCKTIME (本地 WTC 访问点允许用于一个阻塞调用的最大秒数) 后超时。
转换样式
选择以下选项之一：
  • 无 - (默认值) 可以不考虑字段顺序。
  • 有序 - 将按照字段出现的正确顺序显示它们。
  • 有序和分组 - 如果字段作为记录进行逻辑组织，则这些字段将按出现的先后进行排序，并且按记录进行分组。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑业务服务 - 概要”页上查看并保存此配置。
