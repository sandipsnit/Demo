为业务服务配置操作设置

使用操作设置页可以启用和禁用业务服务以及对特定服务的监视。同样，可以在操作 > 智能搜索页上指定这些相同的特定于服务的操作设置，但以下情况例外：不能设置聚集间隔或者指定预警严重性级别。有关详细信息，请参阅查找和更新操作设置。

服务级设置的运行时效果取决于相应的全局设置。必须同时启用全局和服务级设置，才能在运行时完全启用服务。此外，还必须启用服务状态。请参阅启用全局设置。

即使在运行时将不启用该服务，您也可以更改和保存监视配置设置。例如，即使禁用了服务监视，也可以更改和保存聚集间隔。这样，您可以编辑设置并在稍后启用它们。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择资源浏览器 > 业务服务。
  3. 单击业务服务名。

    4. 查看业务服务页显示表 25-8 中所示的信息。

    表 25-8 业务服务信息
    属性
    说明
    上次修改者
    创建或编辑该服务，或将其导入到配置中的用户。
    上次修改时间
    用户创建或编辑该服务，或将其导入到配置中的日期和时间。单击日期和时间链接可查看此资源的更改历史记录。请参阅“查看更改历史记录”页。
    引用
    此服务引用的对象数。如果存在此类引用，可单击数字链接以查看对象列表。请参阅查看对资源的引用。
    引用项
    引用此服务的对象数。如果存在此类引用，可单击数字链接以查看对象列表。请参阅查看对资源的引用。
    说明
    此服务的说明 (如果存在)。

  4. 选择操作设置选项卡。

    5. 您必须处于活动会话中才能配置表 25-9 中所示的业务服务操作设置。

    表 25-9 配置业务服务操作设置
    操作
    步骤
    禁用状态
    清除已启用复选框。激活会话后，服务将停止处理消息。
    禁用某个服务不再会导致系统删除以前为该服务收集的所有监视统计信息。
    启用状态
    选中已启用复选框。激活会话后，服务将恢复处理消息。
    启用要置于脱机状态的业务服务端点 URI
    为脱机 URI 设置重试间隔
    选中启用并设置重试间隔复选框。
    选中此选项之后，业务服务将在运行时删除非响应端点 URI (使其脱机)，以便只使用响应 URI 进行重试以及处理后续请求。
    如果您希望使非响应 URI 一直保持脱机状态，直到采取更正操作并重新启用 URI，请勿提供重试间隔。例如，重试间隔为零表示端点无限期保持脱机状态。要使其重新处于联机状态，请使用端点 URI 页上的使端点联机操作。请参阅查看业务服务端点 URI 度量。
    如果您希望将业务服务配置为使非响应 URI 处于脱机状态，并在指定的时间间隔之后重新尝试访问同一 URI 端点以处理后续消息，请通过指定重试间隔值执行此操作。使用小时、分钟和秒字段指定将非响应 URI 保持处于活动 URI 处理循环之外的时间。
    URI 将在指定的时间间隔内保持脱机状态，然后对其进行重试。如果端点做出响应，则 URI 将再次变为联机状态，否则，它将保持脱机状态，并且此过程将重复执行。
    只有在选中启用并设置重试间隔选项之后，才能指定重试间隔。
    禁用监视
    清除已启用复选框。激活会话后，系统将停止为该服务收集监视统计信息，并删除以前收集的所有统计信息。
    启用监视
    选中已启用复选框。激活会话后，系统将开始为该服务收集监视统计信息。
    设置服务的聚集间隔
    以小时或分钟数选择间隔，然后单击更新。如果选择的小时数超过 1，则默认选择的分钟数始终为零。但如果选择的小时数为 0 或 1，则可以根据分钟数配置间隔。请参阅设置服务的聚集间隔。
    无论是否已启用监视，您都可以更改和保存聚集间隔设置。
    要在运行时启用监视，请确保已选中启用监视复选框。
    禁用 SLA 预警
    清除已启用复选框。系统将停止评估为该服务配置的任何预警规则；因此您不会再收到与该服务关联的 SLA 预警。
    启用 SLA 预警
    选中已启用复选框。在启用预警级别下拉列表中选择条目，将 SLA 预警限制到指定级别或更高级别 (按限制性最低级别到最高级别列出)：“普通”(默认值)、“警告”、“次要”、“主要”、“严重”以及“致命”。
    服务 SLA 预警依赖于全局监视状态和服务级监视状态，必须启用这两者才能在运行时启用 SLA 预警。请参阅全局启用 SLA 预警。
    限制发送到业务服务中配置的端点集的消息流
    选中限制状态旁的已启用复选框。如果已启用，则会按优先级处理消息。您还可以选择使用路由选项为消息分配优先级；否则，消息将按照先进先出的顺序出列。
    有关详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Throttling”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG614。
    限制分派到业务服务中配置的一组端点的消息数
    如果已启用限制状态，则此字段是必需的。
    指定最大并发数值；它必须为正整数值，不能为零。
    如果已超出，则传入消息会放入到等待 (限制) 队列中 (如果存在)。如果未超出，则会放弃消息。
    如果队列已满，将删除其中优先级较低的消息，并会出现运行时错误。如果配置了错误处理程序，便会触发此程序。新的传入消息将入列。
    指定是否应将发送至已超出其消息并发限制的业务服务的消息保留在等待队列中
    设置等待队列中保留的最大消息数
    如果已启用限制状态，则此字段是必需的。
    指定限制队列值；它必须为正整数值，或保留默认值零 (表示没有限制队列)。当限制队列值为零 (没有限制队列) 且消息数超出最大并发数值时，将放弃这些消息并引发错误。如果配置了错误处理程序，便会触发此程序。
    如果在运行时禁用限制，将处理限制队列中的所有消息。
    当删除或重命名业务服务时，将放弃限制队列中的所有消息。
    指定消息将保留在等待队列中的最长时间
    如果已启用限制状态，则此字段是可选的。
    指定消息失效值 (毫秒)；它必须为正整数值。值为零表示消息永远不会失效。
    此值是在队列级别设置的；放入该队列的所有消息都具有相同的消息失效值。如果已超出，则会在不对消息进行处理的情况下从队列中删除消息。
    对业务服务启用消息跟踪

    选中消息跟踪旁边的已启用复选框。启用消息跟踪之后，系统将记录在 Oracle Service Bus 管道和业务服务之间交换的消息 (出站请求和响应消息)。

    注：要查看日志文件或标准输出 (服务器控制台) 中的跟踪信息，必须将 Oracle WebLogic Server 日志记录设置为以下严重性级别：

    要记录的最低严重性：信息
    日志文件：信息
    标准输出：信息

    有关设置日志严重性级别的信息，请参阅 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的“Using Log Severity Levels”，网址为：http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

    请注意，如果适用，记录的出站消息还可能包含重试次数、错误代码和错误消息。
    在详细级别下拉列表中，选择从以下各项中指定详细资料的级别：
    • 简洁 - 显示日期、时间、服务类型、服务名和 URI
    • 标头 - 显示简要信息以及消息元数据的 XML 表示形式
    • 完整 - 显示标头信息以及原始有效负载，包括附件 (如果有)
    如果从详细级别下拉列表中选择“完整”，请使用有效负载跟踪限制字段指定消息有效负载的最大大小 (KB)。此外，还应使用默认编码字段指定用于记录有效负载的默认编码。这在记录二进制有效负载或带有二进制附件的 SOAP 消息时可能十分有用。
    默认的编码值可以为 Base64 或任何支持 Java 的编码，如以下位置中的定义：http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html。
    如果将默认编码字段留空，Oracle Service Bus 将对有效负载使用主机的默认编码。默认编码取决于 JVM、底层操作系统 (OS) 和操作系统级区域设置的组合设置。
    如果无法使用在默认编码字段中指定的设置 (例如，不是有效的配置选项)，则 Oracle Service Bus 将对有效负载使用 Base64 编码。
    为业务服务禁用消息跟踪
    清除消息跟踪旁边的已启用复选框。
    启用和禁用结果高速缓存
    选择已启用可以为业务服务激活结果高速缓存。还必须确保已全局启用结果高速缓存。请参阅全局启用结果高速缓存。
    通过取消选中“已启用”选项并单击更新来禁用业务服务的结果高速缓存时，将刷新为该业务服务高速缓存的所有结果。
