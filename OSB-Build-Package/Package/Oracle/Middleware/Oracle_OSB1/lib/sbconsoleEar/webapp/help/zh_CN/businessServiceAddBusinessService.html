创建和配置业务服务

此部分介绍如何创建和配置业务服务。

对于 JCA 服务，可以从 JCA 绑定生成业务服务 (如从 JCA 绑定资源生成业务服务中所述)，也可以从 Oracle Enterprise Repository 生成业务服务 (如从 Oracle Enterprise Repository 生成业务服务中所述)。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择项目浏览器，然后选择要将业务服务添加到的项目或文件夹。
  3. 在“项目/文件夹视图”页上，选择创建资源下拉列表中的业务服务。
  4. 在创建/编辑业务服务 - “一般配置”页上，提供服务的名称并选择要创建的服务的类型。

    如需命名指导，请参阅资源命名限制。
  5. 单击下一步。后续页取决于在第一页上做出的选择。在每一个后续页上输入相应的信息，直到到达概要页，然后单击保存以保存当前会话中的服务。

    6. 有关填写各个页的详细说明，请参阅以下内容：

    • “一般配置”页
    • “消息类型配置”页
    • “传输配置”页
    • “协议特定的传输配置”页
    • “SOAP 绑定配置”页
    • “消息处理配置”页
    • “概要”页
    • “业务服务策略”页
    • “业务服务安全”页

如果某个业务服务是基于包括 WS-Policy 附件的 WSDL 创建的，则这些策略 (只读) 会显示在“协议特定的传输配置”页中。如果此服务的任何 WS-Policy 指定了验证，则必须选择服务帐户。路由到该业务服务的代理服务将使用此服务帐户来对业务服务进行验证。

从 JCA 绑定资源生成业务服务

可以从 Oracle Service Bus 中的出站 JCA 绑定资源生成 JCA 业务服务。有关 JCA 绑定资源的详细信息，请参阅第 12 章“JCA 绑定”。

要从 JCA 绑定生成 JCA 业务服务，请执行以下操作：

  1. 在 Oracle Service Bus 管理控制台中，如果尚未处于创建或编辑模式，请在“更改中心”中单击创建或编辑。
  2. 在资源浏览器中，单击 JCA 绑定。
  3. 找到要用于生成服务的 JCA 绑定，然后单击操作图标。
  4. 在显示的窗口中，确认 WSDL 的名称以及要生成的服务，选择这些新资源的位置，然后单击生成。

    5. Oracle Service Bus 将生成服务及其对应的 WSDL。

  5. 适当时修改有关生成的服务的任何其他配置详细资料，如端点 URI。

从 Oracle Enterprise Repository 生成业务服务

可以使用 Oracle Service Bus 开发环境从 Oracle Enterprise Repository 生成业务服务。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Generating a Business Service from Oracle Enterprise Repository”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1797。
