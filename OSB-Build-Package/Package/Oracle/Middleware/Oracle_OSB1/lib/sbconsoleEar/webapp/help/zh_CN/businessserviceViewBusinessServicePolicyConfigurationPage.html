“业务服务策略”页

查看业务服务 - 策略配置页是一系列用于编辑业务服务的页中的一个页面，如创建和配置业务服务和查看业务服务 - 配置详细资料页中所述。

使用此页可以为具有“基于 WSDL 的 XML Web 服务”、“消息传送服务”、“任何 SOAP 服务”或“任何 XML 服务”服务类型的业务服务配置策略设置。表 18-23 介绍了如何使用此页。为了将 OWSM 策略用于“基于 WSDL 的 XML Web 服务”、“消息传送服务”或“任何 XML 服务”代理服务，协议必须设置为 HTTP。

请注意，对于基于 WSDL 的服务，绑定到该服务的 WLS 9.2 策略是在有效 WSDL 中公开 (内嵌) 的。抽象策略在内嵌之前会进行预处理。OWSM 策略通过引用绑定的，而不是内嵌在有效 WSDL 中。

服务策略配置
以下是“服务策略配置”字段中的选项：

OWSM 策略

  • 来自 OWSM 策略存储 - 策略是在 Oracle Enterprise Manager 管理的中央 Oracle Web Services Manager 策略存储中定义的。这些策略支持 WS-Security 1.0/1.1、SAML 1.1/2.0、KerberosToken Profile 和其他行业标准。

    使用此选项，可以从预定义列表中添加策略。单击“服务级别策略”下方的“添加”可显示列表。如有必要，可使用筛选功能来找到所需的策略。对于业务服务，只显示客户端策略。

    如果用于创建业务服务的 WSDL 包含嵌入式标准 WS-Security 策略，则 Oracle Service Bus 会引发错误并显示冲突。要解决此冲突，请通过选择来自 OWSM 策略存储并单击添加兼容项将嵌入式 WSDL 策略替换为兼容的 Oracle Web Services Manager 策略。Oracle Service Bus 将尽最大努力尝试基于 WSDL 中嵌入的标准策略从 Oracle Web Services Manager 策略存储中查找最匹配的策略。算法可能会返回零个、一个或多个匹配的策略。

    如果单击添加兼容项后 Oracle Service Bus 没有找到兼容的策略，请单击添加并选择相应的 Oracle Web Services Manager 策略。如果 Oracle Service Bus 返回多个兼容策略，请删除您不需要使用的策略。

    注：仅当您已将域配置为使用 Oracle Web Services Manager 时，才会激活此选项。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Securing Oracle Service Bus with Oracle Web Services Manager”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681。

WLS 9 策略

  • 来自 WSDL - 如果服务策略与服务所基于的 WSDL 相关联，请选择此选项。这些策略支持 WS-Security 1.0、SAML 1.1 和其他行业标准。

    使用此选项，您可以查看来自 WSDL 的 (只读) 请求策略和响应策略。

    注：如果您在业务服务配置中收到有关 Oracle Service Bus 不支持的 WS-Security 策略的错误，请使用“来自 OWSM 策略存储”选项将嵌入式 WSDL 策略替换为兼容的 OWSM 策略。

  • 来自预定义策略或 WS-Policy 资源 - 选择此选项时，可以从 Oracle Service Bus 管理控制台添加服务级策略、操作级策略 (在这种情况下，策略同时应用于请求消息和响应消息)、请求策略以及响应策略。

    策略是在 Oracle WebLogic Server 中预定义的，或者是在具有 WS-Policy 资源的 Oracle Service Bus 中由用户定义的。这些策略支持 WS-Security 1.0、SAML 1.1 和其他行业标准。

    使用此选项，可以添加服务级策略、操作级策略、请求级策略以及响应级策略。单击相应部分下方的“添加”可从列表中选择策略。

注：策略绑定模型是相互排斥的。在一个服务中只能使用一种类型的策略。如果将策略直接绑定到服务，则会忽略所有基于 WSDL 的策略。

添加策略后，您可以在“安全”页上提供覆盖。
