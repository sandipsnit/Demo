更改中心

在 Oracle Service Bus 中配置服务和元数据时，更改中心允许团队协作。每个用户均在一个沙盒会话中工作，直到用户准备将工作配置签入到总线的核心配置中。由于多个用户使用该配置，因此更改中心提供多个级别的撤消以及对冲突的可见性。

大多数更改中心页都会根据您是在会话中 (例如，在更改中心中单击了编辑) 还是在会话外而显示不同的信息。例如，如果在会话中，则单击“更改中心”中的查看更改时所显示的查看配置更改页将列出您在该会话中所做的全部更改；在会话外时，该页则会列出所有会话激活。

查看冲突
仅在会话内：此链接显示会话中出现的冲突数。查看和解决当前会话中所做的更改与 Oracle Service Bus 管理控制台中所有其他会话中所做的更改之间的所有冲突。如果在当前会话中修改了某个资源，而该资源已被另一个会话修改并激活，则会发生冲突。由两个会话对同一资源所做的两个更改在其中一个会话被激活之前不会导致冲突。
