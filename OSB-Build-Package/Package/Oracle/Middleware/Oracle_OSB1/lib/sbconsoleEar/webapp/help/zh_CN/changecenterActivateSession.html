激活会话

  1. 在更改中心中单击激活。激活会话页会显示会话名称、用户名和说明字段。

    2. 无论系统中的其他活动如何，创建会话和放弃会话的过程都将继续。但是，如果正在激活另一个会话，则激活会话页将显示一个错误，指出具有待处理 WebLogic 服务器更改的用户。有关解决在您的会话中和其他已激活会话中所做的更改之间的冲突的信息，请参阅查看和解决冲突。

  2. 如果有验证错误，则会显示错误消息，说明会话中存在验证错误。继续之前，请查看并解决配置冲突。有关解决冲突的信息，请参阅查看并解决冲突。

    3. 直到查看了所有冲突之后，您才能激活会话。如果在查看现有的冲突时出现新的冲突，则在激活前，会弹出一个消息，说明存在新冲突。

  3. 在说明字段中输入说明。当单击更改中心下的查看更改来显示由会话激活造成的配置更改时，此说明将显示在说明列中。
  4. 要激活该会话，请单击提交。如果在此期间内未出现新冲突，则会话将结束，而配置将部署到运行时。
注： 当尝试使用 JMS 端点 URI 在另一台服务器 (当前使用的服务器之外的服务器，或者集群中的托管服务器) 上激活会话时，请确保目标服务器可用。
注： 如果 JMS 端点 URL 指定的目标服务器不可访问，则 Oracle Service Bus 不允许使用 JMS 传输注册代理服务。换句话说，对于 JMS 服务，Oracle Service Bus 检查指定的连接工厂是否存在；如果不存在，则发生会话激活错误。
