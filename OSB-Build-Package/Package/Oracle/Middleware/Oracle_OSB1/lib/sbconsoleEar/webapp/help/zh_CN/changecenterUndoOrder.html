撤消任务的顺序

可以任意顺序撤消任务 (如果单个撤消操作能生成有效数据)。撤消操作会将资源的值设置为对该资源进行更改之前的值。

如果要撤消的任务是创建了对象的任务，则对象没有可返回到的以前状态 - 换句话说，在执行此任务之前不存在任何对象。这种撤消操作可以有效地将新对象从会话中删除。在这种情况下，引用了这个将删除的对象的那些对象会出错。可以在“更改中心”的查看冲突页中查看此类错误。

当您不在会话中工作时，可以访问查看配置更改页来查看以前激活的会话。可以撤消这些会话。如果撤消以前激活的会话的某个操作会导致运行时配置错误，则系统不允许您执行此撤消操作。例如，如果您尝试撤消一个会话激活，而这样会导致由其他对象引用的对象被删除，则不会允许该撤消操作。

可以撤消一个撤消操作。在撤消 [任务] 的选项列中，单击撤消图标。Oracle Service Bus 支持无限的撤消操作。这意味着您可以撤消对上一次撤消的撤消操作，依此类推。
