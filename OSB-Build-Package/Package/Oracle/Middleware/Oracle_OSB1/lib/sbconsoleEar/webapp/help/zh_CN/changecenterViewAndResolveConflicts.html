查看和解决冲突

使用此页可以查看有关配置中的错误的诊断消息，并可查看和解决您的会话中及其他激活的会话中所做更改之间的冲突。查看冲突链接还可以显示会话中的活动冲突数。

查看冲突
  1. 单击更改中心下的查看冲突。查看冲突页可以显示以下任何组或所有组的信息，具体取决于冲突的性质：
    • 错误 (错误图标表示不可提交的严重冲突) - 通知您配置中存在严重冲突。如果没有解决冲突，则无法提交更改。请参阅表 2-6 和解决冲突。
    • 并发更新 (警告图标表示可提交的非严重冲突) - 警告您存在与其他已激活的会话不兼容的更改。请参阅表 2-7。
    • 信息消息 (警告图标表示可提交的非严重冲突) - 通知您配置中存在非严重冲突。

      • 表 2-6 错误
      属性
      说明
      名称
      错误消息所指的资源。此名称是指向资源的链接。
      路径
      资源所在的项目和文件夹。
      资源类型
      资源的类型。
      消息
      冲突的说明。要解决此冲突，请参阅解决冲突。

  2. 要显示某个特定冲突的详细资料，请单击名称列中该冲突的链接。

    3. 如果自开始编辑当前会话以来，您在会话中对其做出更改的对象在运行时已更改，则查看冲突页将显示表 2-7 中所示的信息。

    表 2-7 并发更新
    属性
    说明
    名称
    冲突的资源。此名称是指向资源的链接。
    您的更改
    您在会话中对此对象所做更改的说明。
    他人的更改
    其他用户在其会话中对此对象所做更改的说明。
    同步
    单击同步图标，使此对象恢复到在运行时保存它的状态。

解决冲突

要解决冲突，请使用消息列中提供的信息 (如表 2-6 中所述) 了解问题，然后编辑导致冲突的对象，以解决该问题。

对于有并发更新冲突的情况 (即在当前会话中修改了某个资源，而该资源已被另一个会话修改并激活，在该情况下发生的冲突 (如表 2-7 中所述))，您可以按照以下两种方法之一解决该冲突：

  • 单击更改中心下的激活。这样会将您所做的更改保存到运行时，这意味着将覆盖由上一个会话激活部署到核心数据的更改。换句话说，将您在此会话中的更改会覆盖其他用户在另一个会话中保存的更改。
  • 单击此表的同步列中的同步图标 (如表 2-7 中所述)。此操作会在该会话中将此对象恢复到在运行时保存它的状态。
