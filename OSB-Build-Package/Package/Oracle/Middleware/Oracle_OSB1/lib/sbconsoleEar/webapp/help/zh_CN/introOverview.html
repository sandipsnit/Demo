Oracle Service Bus 简介

Oracle Service Bus 是基于配置的、策略驱动的企业服务总线。它可在异构 IT 环境中提供高度可扩展且可靠的、面向服务的集成、服务管理和传统的消息代理。它将具有消息路由和转换功能的智能消息代理、服务监视和管理组合到了一个统一的软件产品中。通过 Oracle Service Bus 管理控制台，您可以控制服务和策略配置，以及监视系统和操作任务。Oracle Service Bus 依赖于 Oracle WebLogic Server 运行时工具。有关详细信息，请参阅 Oracle Fusion Middleware Concepts and Architecture for Oracle Service Bus，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBCA。

资源命名限制

命名 Oracle Service Bus 配置中的任何目录或资源时，允许以下字符：

    • 所有 Java 标识符字符，包括 Java 关键字，如 Java 语言规范的“标识符”和“关键字”部分 (网址为：http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#3.8) 中所述

    • 名称中的空格、句点和连字符 (非前导或尾随)

不允许诸如 / \ * : " < > ? | 等字符。
