添加 JCA 绑定

在 Oracle Service Bus 中创建 JCA 绑定的最简单方法是批量导入已创建的 JCA 资源 (.jca 文件、相关联的 WSDL 和 TopLink 或 EclipseLink 映射文件)。批量导入时，Oracle Service Bus 会自动根据 .jca 文件生成 JCA 绑定，根据相关联的 JCA WSDL 生成 WSDL 资源，并根据映射文件生成 XML 文档资源，以及维护文件之间的相关性。

有关导入的详细信息，请参阅从 Zip 文件加载资源、从 URL 加载资源和导入资源。

在导入 JCA 资源之后，您可以根据这些资源生成代理或业务服务，如从 JCA 绑定资源生成代理服务和从 JCA 绑定资源生成业务服务中所述。

如果您未批量导入 JCA 资源，则可以手动创建 JCA 绑定，如以下步骤所述：

  1. 在 Oracle JDeveloper 中创建 JCA 服务 WSDL、对应的 JCA (.jca) 文件，如果需要，还可创建其他资源，例如，TopLink、EclipseLink 映射文件和 XSLT 文件。

    2. 有关详细信息，请参阅 Oracle Fusion Middleware User's Guide for Technology Adapters，网址为：http://www.oracle.com/pls/as111170/lookup?id=TKADP。

  2. 如果尚未执行此操作，请单击创建在 OSB 管理控制台中新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  3. 在 OSB 管理控制台中，创建或导入您在 Oracle JDeveloper 中创建的 JCA 资源，例如，WSDL、XML 和 XSLT 资源。
  4. 创建 JCA 绑定。
    1. 选择项目浏览器，然后选择 JCA 绑定的项目或文件夹位置。此时将显示项目视图/文件夹视图页。
    2. 从创建资源字段中选择 JCA 绑定，以显示创建新的 JCA 绑定资源页。
    3. 在JCA 文件字段中，单击浏览以选择 JCA 文件。

      4. “资源名”字段会自动填充文件名 (无文件扩展名)。

      如果不选择文件，请复制 JCA 文件的内容并粘贴在“JCA 文件”字段文本区域。

    4. 在资源名字段中，输入或更改 JCA 绑定的名称。JCA 绑定名必须是唯一的。

      如需命名指导，请参阅资源命名限制。
    5. 在资源说明字段中，输入此 JCA 绑定的说明。
    6. 单击保存。JCA 绑定会保存在当前会话中。

      7. 如果 JCA 绑定尚未引用所需的资源 (如 WSDL 和映射文件)，将会显示一条验证警告。

  5. 单击新 JCA 绑定的名称，可在查看 JCA 绑定页中显示它。
  6. 单击编辑引用。
  7. 在编辑 JCA 绑定资源的引用页中，浏览并选择 JCA 绑定引用资源，例如，WSDL、TopLink/EclipseLink 映射文件和 XSLT 文件。
  8. 单击保存。
  9. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。

在添加 JCA 绑定资源和配置资源相关性 (关联 WSDL) 之后，或者如果已导入维护相关性的 JCA 资源，可从 JCA 绑定生成代理和业务服务。有关详细信息，请参阅从 JCA 绑定资源生成代理服务和从 JCA 绑定资源生成业务服务。
