定义预警规则条件

使用此页可定义预警规则的条件。您至少必须指定一个条件。如果要指定多个条件，则必须使用 And/Or 运算符组合这些条件。

  1. 在选择条件的聚集间隔字段中，选择小时数和分钟数以设置此条件的聚集间隔。此聚集间隔决定了条件的测试频率。每当监视子系统聚集的数据样本足以构成一个聚集间隔时，都会测试此条件。

    2. 例如，如果选择的聚集间隔为 1 小时，则每当有一小时的数据可用时都会测试此条件。第一次测试此条件是在第一个小时结束时。之后，则每 10 分钟测试一次条件，因为 1 小时的聚集间隔的采样间隔被设置为 10 分钟。

  2. 开始先定义一个简单表达式。可以将两个或多个简单表达式组合在一起以形成一个复合表达式。要定义一个复合简单表达式，请执行以下操作：
    1. 在第一个下拉字段中，选择计数、最小值、最大值、平均值或状态。
    2. 在下一个下拉字段中，选择一个操作数。

      3. 根据您在第一个下拉字段中选择的是计数、最小值、最大值、平均值还是状态，操作数的列表也随之变化。例如，如果选择“最小值”、“最大值”或“平均值”，则“响应时间”操作数可用。使用此操作数可设置响应时间 (毫秒)。可用的操作数还取决于服务本身的配置。操作数的数量根据服务是否有管道、路由节点、操作等而有所不同。

      有关如何根据端点 URI 状态生成预警的信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Managing Endpoint URIs for Business Services”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG595。

      选择计数后，此字段将显示表 25-21 中所示的操作数。

      表 25-21 计数操作数详细资料
      属性

    3. 在下一个字段中，选择所需的比较运算符：=、!=、> 或 <。
    4. 在下一个字段中，输入要与其进行比较的值。
    5. 单击添加。
  3. 要定义复合表达式，请执行以下操作：
    1. 重复第 2 步中的步骤 a 至 e，定义一个简单表达式。
    2. 再次重复步骤 a 至 e，定义另一个简单表达式。
    3. 分别选中这两个表达式的复选框。
    4. 单击 And 或 Or 将这两个表达式组合为一个复合表达式。

      5. 组合了多个表达式后，就可以使用拆分选项了。此选项用于将复合表达式拆分回单独的简单表达式。

完成之后

单击上一步返回到一般配置页。或者，单击最后一步显示新预警规则 - [服务名] 页。继续复查预警规则配置。
