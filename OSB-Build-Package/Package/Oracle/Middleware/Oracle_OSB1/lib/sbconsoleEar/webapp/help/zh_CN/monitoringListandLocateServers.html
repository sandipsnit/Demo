查看服务器信息

服务器健康状况页中包含以下内容：

  • 在服务器健康状况选项卡标题上，可能会显示关闭图标和表示已关闭的服务器数的数字指示符：
    • 单击该图标可使用关闭健康状况条件和 SHUTDOWN 状态条件来筛选显示的服务器。
    • 该数字表示已关闭的服务器数。
  • 用于描述域中服务器健康状况的饼图。可以单击该饼图中与服务器相对应的区域以便显示详细信息。
  • 日志概要。要显示日志，请单击用于显示给定严重性的消息数的数字链接。例如，要查看警告 1，请单击数字 1。请参阅查看域日志文件详细资料。
  • 根据服务器概要表的定制设置显示服务器信息的字段。例如，服务器名、健康状况、集群名、计算机名、服务器状态和服务器已运行的时间长度。要了解有关这些字段的详细信息，请参阅表 25-16。
  • 默认情况下，面板刷新率为不刷新。要更改面板刷新率间隔，请参阅设置用户首选项。
注意： 可通过选择默认值 (不刷新) 之外的面板刷新率间隔，使浏览器定期刷新面板。这可防止会话超时，即使在不与 Oracle Service Bus 管理控制台交互的情况下，也是如此。
注意： Oracle 建议您谨慎使用面板刷新率功能。绝不应将会话置于无人值守模式。当会话处于活动状态时，其他人可从您的计算机访问该会话而无须重新进行验证。
  • 表定制程序图标。请参阅定制表视图。
  • 要限制列表中的项目个数或查找特定服务器，请执行以下操作：
    • 单击打开图标可显示其他搜索筛选器。如果指定多个搜索条件，则仅返回与所有条件均匹配的结果。
    • 按健康状况进行筛选。从健康状况下拉列表中选择条目，将服务器限制到指定的健康状况级别，然后单击搜索。

      • 选中或更高级别复选框，将搜索限制到指定的服务器健康状况级别或更高级别 (按限制性最低级别到最高级别列出)：“正常”、“警告”、“超载”、“严重”和“致命”。

    • 按服务器进行筛选。从服务器下拉列表中，选择“全部”或服务器名称，然后单击搜索。
    • 按集群名进行筛选。从集群名下拉列表中，选择“全部”或与服务器关联的集群的名称，然后单击搜索。
    • 按计算机名进行筛选。从计算机名下拉列表中，选择“全部”或与服务器关联的计算机的名称，然后单击搜索。
    • 按状态进行筛选。从状态下拉列表中，选择“全部”或服务器的状态：“RUNNING”、“FAILED”或“SHUTDOWN”，然后单击搜索。
    • 单击查看全部可删除搜索筛选器，并显示所有服务器。

服务器健康状况页显示各个服务器的以下信息 (如表 25-16 中所示)，具体取决于您指定的筛选器设置。

表 25-16 服务器概要详细资料
属性
说明
服务器
服务器的名称。此名称是一个指向查看服务器详细资料页的链接。请参阅查看服务器详细资料。
健康状况
服务器的健康状况：
  • 正常 - 服务器正常运行，没有任何问题。
  • 确定/未处理 - 服务器处于 ADMIN 状态，仅可用于管理操作。所有运行时 Oracle Service Bus 消息都将失败。
  • 警告 - 服务器将来可能会出现问题。有关详细信息，请查看服务器日志和相应的 RuntimeMBean。
  • 超载 - 为服务器分配的工作超过了配置的阈值；它可能会拒绝其他任何工作。
  • 严重 - 服务器失败待处理；必须立即执行某些操作以防止失败。有关详细信息，请查看服务器日志和相应的 RuntimeMBean。
  • 关闭 - 服务器已失败，必须重新启动。
  • 严重 - 服务器失败待处理；必须立即执行某些操作以防止失败。有关详细信息，请查看服务器日志和相应的 RuntimeMBean。
集群名
与此服务器相关联的集群的名称。
计算机名
与此服务器相关联的计算机的名称。
状态
服务器的状态：
  • RUNNING
  • ADMIN
  • FAILED
  • SHUTDOWN
正常运行时间
服务器已经运行的时间长度。
