查看服务器监视信息

服务健康状况页显示已启用了监视的服务的度量。要了解如何启用对服务的监视，请参阅为代理服务配置操作设置和为业务服务配置操作设置。

默认情况下，面板刷新率为不刷新。要更改面板刷新率间隔，请参阅设置用户首选项。

注意： 可通过选择默认值 (不刷新) 之外的面板刷新率间隔，使浏览器定期刷新面板。这可防止会话超时，即使在不与 Oracle Service Bus 管理控制台交互的情况下，也是如此。
注意： Oracle 建议您谨慎使用面板刷新率功能。绝不应将会话置于无人值守模式。当会话处于活动状态时，其他人可从您的计算机访问该会话而无须重新进行验证。

在服务健康状况选项卡标题上，可能会显示这些图标：

  • 警告图标和数字指示符。
    • 单击该图标可使用有错误条件筛选显示的服务。
    • 该数字表示自从上次刷新该页以来发生错误的服务数量。
  • 脱机端点图标和数字指示符。
    • 单击该图标可使用包含脱机端点 URI 条件筛选显示的服务。
    • 该数字表示自上次刷新该页以来带有处于脱机状态的端点的服务数量。

服务健康状况页显示表 25-5 中所示的服务度量和监视信息。

  1. 在显示统计信息字段中，执行以下操作之一：
    • 要显示当前聚集间隔的时段内每个服务的监视统计信息，请选择当前聚集间隔。“当前聚集间隔”视图显示服务度量的动态统计信息视图。
    • 要显示自上次重置服务统计信息以来的时段内的监视统计信息，请选择自从上次重置。“自从上次重置”视图显示度量的累计动态计数。
  2. 从下拉列表中选择服务器以便显示此服务器的度量。

    3. 如果存在集群，则在默认情况下显示集群范围的度量。选择单独的托管服务器以便显示此服务器的度量。

  3. 要限制列表中的项目个数或查找特定服务，请执行以下操作：
    • 单击打开图标可显示其他搜索筛选器。如果指定多个搜索条件，则仅返回与所有条件均匹配的结果。
    • 按服务名筛选。在名称中输入搜索目标的名称，或输入通配符（将 * 和 ? 用作通配符可执行更全面的搜索），然后单击搜索。

      • 如果服务的名称在所有项目和路径中是唯一的，则最好使用此搜索方法。

    • 按服务名筛选。在路径字段中，输入搜索目标的路径，该路径由项目名和服务所在的文件夹名称组成。可将 * 和 ? 用作通配符以执行更全面的搜索。将显示该路径中的所有服务。

      • 路径字段的格式如下：
      project-name/root-folder/ . . ./parent-folder

      如果某个服务直接位于项目下，则此格式如下：project-name

    • 按服务名称和路径进行筛选。如果多个同名服务存在于不同的路径中，则最好使用此搜索方法。
    • 按父服务进行筛选。从服务下拉列表中，选择父服务类型：“代理服务”或“业务服务”，然后单击搜索。选择“所有服务”可忽略父服务筛选条件。
    • 按带有消息的服务进行筛选。选择有消息选项，然后单击搜索。
    • 按带有预警的服务进行筛选。选择有预警选项，然后单击搜索。
    • 按带有错误的服务进行筛选。选择有错误选项，然后单击搜索。
    • 按带有脱机 URI 的服务进行筛选。选择包含脱机端点 URI 选项，然后单击搜索。
    • 按特定代理服务调用的服务进行筛选。在所提供的字段中输入代理服务的名称和路径，或者单击浏览显示选择代理服务页。从列表中选择代理服务，然后单击提交。单击搜索。此时将显示由您指定的代理服务调用的业务服务。

      • 由代理调用字段中不支持通配符。

    • 单击查看全部可删除搜索筛选器，并显示所有服务。

      • 表 25-5 服务监视信息
      属性
      说明
      名称
      分配给服务的名称。此名称是指向服务监视详细资料页的链接。请参阅查看服务度量。
      路径
      与服务相关联的项目。如果服务位于项目文件夹中，则也会列出此文件夹。此路径按以下格式显示：
      project-name/root-folder/ . . ./parent-folder
      此路径是指向项目浏览器中相应路径的链接。
      服务类型
      父服务类型：代理服务或业务服务。
      聚集间隔
      只有在显示统计信息字段选择了当前聚集间隔后，才显示该字段。
      监视此服务的当前聚集间隔设置，以小时和分钟表示。可以在操作设置页上设置此间隔。请参阅设置服务的聚集间隔。
      平均响应时间
      在当前聚集间隔的时段内或自上次重置以来的时段内，此服务执行消息所用的平均响应时间 (毫秒)。
      消息
      在当前聚集间隔的时段内或自上次重置以来的时段内，与该服务关联的消息数。
      错误
      在当前聚集间隔的时段内或自上次重置以来的时段内，与该服务关联的错误消息数。
      预警
      默认情况下，将隐藏此字段。
      在当前聚集间隔的时段内或自上次重置以来的时段内，与此服务关联的预警 (SLA 和管道) 数。
      对于集群中的单个托管服务器的代理服务，它将显示当前聚集间隔的时段内或自上次重置以来的时段内，仅与此服务关联的管道预警数。
      对于单个托管服务器的业务服务，则显示 N/A。
      SLA 预警
      在当前聚集间隔的时段内或自上次重置以来的时段内，与此服务关联的 SLA 预警数。
      对于集群中的单个托管服务器，SLA 预警将为 N/A (仅显示集群范围内的 SLA 预警)。
      管道预警
      仅适用于代理服务：
      在当前聚集间隔的时段内或自上次重置以来的时段内，与此服务关联的管道预警数。
      端点 URI 状态
      端点 URI 的状态 (仅适用于业务服务)。该状态是指向端点 URI 页的链接。请参阅查看业务服务端点 URI 度量。
      对于单个节点域 (单独服务器实例)：
      • 联机 - 业务服务的所有端点 URI 均处于联机状态。
      • 脱机 - 业务服务的所有端点 URI 均处于脱机状态。
      • 部分 - 业务服务的至少一个端点 URI 处于脱机状态。
      对于集群域：
      • 将服务器字段设置为集群或其中一个托管服务器之后，联机状态表示所有端点 URI 分别在整个集群或在选定托管服务器上处于联机状态。
      • 将服务器字段设置为集群或其中一个托管服务器之后，脱机状态表示所有端点 URI 分别在整个集群或在选定托管服务器上处于脱机状态。
      • 将服务器字段设置为集群之后，部分状态表示业务服务的至少一个端点 URI 在至少一个服务器上处于脱机状态，或者一个端点 URI 在所有服务器上处于脱机状态，但同一业务服务的其他端点 URI 在一个或所有服务器上仍可用。
      • 将服务器字段设置为其中一个托管服务器之后，部分状态表示业务服务的至少一个端点 URI 在选定托管服务器上处于脱机状态。
      操作
      只有在显示统计信息字段中选择了自从上次重置后，才会显示该字段。
      在此列中，可以单击特定服务的重置图标，重置该服务的统计信息。如果确认要重置统计信息，则系统会删除从上次重置统计信息以来为该服务收集的所有监视统计信息。但系统不会删除当前聚集间隔内为服务收集的统计信息。此外，单击重置图标后，系统将立即再次开始为服务重新收集监视统计信息。
