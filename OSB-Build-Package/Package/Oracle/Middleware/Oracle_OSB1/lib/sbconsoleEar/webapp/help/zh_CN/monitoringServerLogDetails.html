表 25-18 域日志详细资料
属性
说明
消息
事件或条件的说明。
日期
以区域设置特有的格式显示消息的生成时间和日期。运行每个 WebLogic 服务器实例的 Java 虚拟机 (JVM) 会引用主机操作系统，以获得有关本地时区和格式的信息。
子系统
指示作为消息源的 WebLogic 服务器子系统；例如，企业 Java Bean (EJB) 容器或 Java 消息传送服务 (JMS)。
消息 ID
一个唯一的六位标识符。
WebLogic 服务器系统消息生成的所有消息 ID 均以 BEA- 开始，并且都在 0-499999 数字范围内。
严重性
表示消息所报告的事件的影响或严重性：
  • 预警 - 某个特定服务处于不可使用的状态，而系统的其他部分可以继续运行。无法自动恢复；需要管理员立即对此引起重视，以解决问题。
  • 严重 - 出现了系统或服务错误。系统可以恢复，但是可能造成服务的暂时丢失或导致永久性的性能下降。
  • 紧急情况 - 服务器处于不可使用状态。此严重性表示出现了严重的系统失败或紧急情况。
  • 错误 - 出现了用户错误。系统或应用程序不必中断就可以处理此类错误，服务性能的下降也有限。
  • 信息 - 用于报告正常的操作；是一种低级别的信息消息。
  • 通知 - 一种重要级别较高的信息消息。
  • 警告 - 出现了一个可疑的操作或配置，但是可能不会影响正常的操作。
计算机
承载服务器实例的计算机的 DNS 名称。
服务器
生成消息的 WebLogic 服务器实例的名称。
线程
JVM 分配给在其中生成消息的线程 ID。
用户 ID
在其下执行关联事件的用户 ID。
事务处理 ID
仅在事务处理上下文中记录的消息才具有此 ID。
上下文 ID
用于关联来自某个特定请求或应用程序的消息的上下文信息。
详细资料
事件或条件的说明。
原因
消息的原因。
操作
应该采取的操作。
