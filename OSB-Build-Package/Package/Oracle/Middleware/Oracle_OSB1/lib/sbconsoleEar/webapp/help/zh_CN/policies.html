定制 WS-Policy

注： 在将来的 Oracle Service Bus 发行版中，Oracle Web Services Manager 策略将替换 WLS 9.2 策略。由于此版本的 Oracle Service Bus 继续支持 WLS 9.2 策略，因此您应在创建新服务时考虑使用 Oracle Web Services Manager 策略，以准备最终废弃对 WLS 9.2 策略的支持。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Securing Oracle Service Bus with Oracle Web Services Manager”，网址为 http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681。

Web 服务策略框架 (WS-Policy) 是一个基于 XML 的可扩展框架，它使用域特定安全断言扩展 Web 服务的配置，并指定了 Web 服务的安全要求、期望和功能。在 Oracle Service Bus 中，WS-Policy 的主要用途之一是配置代理服务和业务服务中的消息级安全。

使用 Oracle Service Bus WS-Policy

Oracle Service Bus 中包括三个简单的 WS-Policy 文件，可以使用这些文件要求客户机授权、数字加密或数字签署 SOAP 消息：Auth.xml、Encrypt.xml 和 Sign.xml。Oracle 建议：除非有特定的安全需求，否则应尽可能使用这些预打包的文件。

有关使用这些策略的详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Using WS-Policy in Oracle Service Bus Proxy and Business Services”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599。

使用定制 WS-Policy

如果 Oracle Service Bus WS-Policy 语句不能满足您的安全需要，则可以编写您自己的 WS-Policy (定制 WS-Policy)，将其导入到 Oracle Service Bus，并从 WSDL 对其进行引用。(Oracle Service Bus WS-Policy 语句是只读的。)

有关创建和引用定制 WS-Policy 的信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Using WS-Policy in Oracle Service Bus Proxy and Business Services”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599。
