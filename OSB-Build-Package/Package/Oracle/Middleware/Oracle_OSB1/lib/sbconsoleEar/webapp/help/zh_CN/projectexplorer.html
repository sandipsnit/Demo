使用项目、文件夹和资源

项目浏览器可将配置和资源组织为项目和文件夹。在项目浏览器中，可以显示“项目视图”页(该页列出域中的所有项目) 和“项目/文件夹视图”页 (该页列出每个项目中的文件夹和资源)。

项目是不分层、不重叠的顶级分组结构。

所有 Oracle Service Bus 资源 (例如，服务、WS-Policy、WSDL、XQuery 转换等) 都仅位于一个项目中。项目不会重叠。资源可在项目下直接创建，也可进一步组织为文件夹。但您可以引用任何资源，而不管它位于哪个项目中。

在 Oracle Service Bus 中创建域时，将会创建默认项目。

命名项目和文件夹

项目名和文件夹名被限制为 64 个字符，并且必须符合资源命名限制。

项目和文件夹的名称、长度和嵌套层级最终会受操作系统限制的影响。由于操作系统的限制，使用过长的名称或深度嵌套的文件夹创建文件夹或项目可能会失败。

使用项目和文件夹限定资源名

项目和文件夹限定了 Oracle Service Bus 资源的名称。以如下方式构造对资源的引用：

project-name/root-folder/. . ./parent-folder/resource-name

如果资源直接位于项目之下，则以如下方式构造该引用：

project-name/resource-name
