从 Zip 文件加载资源

可以选择包含要加载资源的 Zip 或 JAR 文件；Oracle Service Bus 可从该 Zip 文件查找并导入资源。

WSDL 或 XML 方案这类资源可以定义相关资源的完整树。批量加载功能允许您一次上传一组资源；而无须手动解析相关性。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择项目浏览器，然后选择要向其中添加资源的项目或文件夹。此时将显示“项目/文件夹视图”页。
  3. 从创建资源下拉列表中，选择压缩的资源。此时将显示加载资源页。
  4. 在选择 Zip 文件字段中，指定要导入的 Zip 文件。输入该文件的名称 (由其路径限定)；或者，单击浏览，找到该文件，然后单击打开。
  5. 复查资源类型到文件扩展名的映射。根据需要更改或添加扩展名。使用逗号分隔多个扩展名。

    6. Zip 文件中的每个文件都是可能利用的资源，通常使用文件扩展名来标识文件类型。Oracle Service Bus 为每个资源定义了默认扩展名，但是，如果有其他扩展名可用于描述这些资源，此映射允许您向加载器声明这些扩展名。

    定义 Zip 文件和扩展名映射后，Oracle Service Bus 加载器会扫描 Zip 文件的内容，并尝试将资源类型与每个文件进行关联。没有扩展名的文件或没有映射中定义的扩展名的文件将被视为未知，并自动从加载操作中排除。

    对于已知文件，资源名就是不包含扩展名的文件名。Zip 文件的文件夹结构将在目标 Oracle Service Bus 管理控制台文件夹中重新创建。

    扩展名只能与一种文件类型相关联。

  6. 单击下一步。
  7. 在“加载资源 - 复查已加载的资源”页上，复查将导入的资源。如果您不希望在此导入中包括某些资源，则可清除这些资源旁的复选框。
  8. 要导入所选资源，请单击导入。
  9. “加载资源 - 导入结果”页显示了导入的资源。单击加载另一个从其他文件导入资源，或者从左侧导航窗格中选择另一页以继续执行其他任务。
  10. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
