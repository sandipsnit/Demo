服务密钥提供程序

服务密钥提供程序包含公共密钥基础结构 (PKI) 身份证明，代理服务使用这些身份证明解密入站 SOAP 消息、进行出站验证和数字签名。PKI 身份证明是与证书成对的私有密钥，可用于数字签名和加密 (以便提供 Web 服务安全)，并可用于出站 SSL 验证。该证书包含与此私有密钥相对应的公共密钥。

注： 要使用服务密钥提供程序，必须配置 PKI 身份证明映射提供程序。请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Configuring the WebLogic Security Framework: Main Steps”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498。

一个服务密钥提供程序可以包含以下所有 PKI 身份证明：

  • 数字加密密钥对

    • 代理服务使用此密钥对对已加密 (以遵守 Web 服务策略声明) 的入站 SOAP 消息进行解密。如果希望服务密钥提供程序支持数字加密，则与 PKI 身份证明映射器关联的密钥库至少必须包含一个支持加密的 X.509 证书。

  • 数字签名密钥对

    • 如果代理服务的端点是 Web 服务，并且该 Web 服务需要客户机对 SOAP 信封的一个或多个部分进行签名，则代理服务可使用此密钥对。

  • SSL 客户机验证密钥对 (双向 SSL)

    • 代理服务在出站 TLS/SSL (安全套接字层) 连接期间充当客户机时，可使用此密钥对进行验证；换言之，将消息路由到需要客户机证书验证的 HTTPS 业务服务或代理服务时，可使用该密钥对。

可以对多个代理服务使用同一个服务密钥提供程序。
