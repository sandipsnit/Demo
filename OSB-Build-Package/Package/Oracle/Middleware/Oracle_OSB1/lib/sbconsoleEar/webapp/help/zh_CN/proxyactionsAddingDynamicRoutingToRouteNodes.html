向路由节点添加动态路由

根据 XQuery 资源中可用的路由信息为消息分配路由。

这是一项最终操作，表示在此操作后不能添加其他操作。但是，此操作可以包含请求和响应操作。

有关路由的详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Modeling Message Flow in Oracle Service Bus”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG181。

开始之前

这些说明假定您已在编辑消息流页中编辑消息流，如创建、编辑和查看消息流中所述。

向路由节点添加动态路由

  1. 导航至要添加操作的位置，如向消息流添加操作和编辑消息流中的操作中所述。
  2. 单击路由节点图标，然后单击编辑路由。此时将显示编辑阶段配置页。
  3. 单击添加操作图标，然后选择通信 > 动态路由。
  4. 单击 <表达式>。此时将显示 XQuery 表达式编辑器页。
  5. 在 XQuery 表达式编辑器中输入 XQuery 表达式，其结果类似于：

    6. <ctx:route>

    <ctx:service isProxy='true'>{$service}</ctx:service>

    <ctx:operation>{$operation}</ctx:operation>

    </ctx:route>

    注： 调用代理服务时，应将 isProxy 属性设置为“真”。调用业务服务时，请将 isProxy 设置为“假”。
    注： - 服务名是全限定的服务名。
    注： - 操作元素是可选的
  6. 单击保存。
  7. 在请求操作字段中，单击添加操作以添加操作，然后选择要与此服务相关联的操作。可以添加多个操作。要了解有关希望添加的操作类型的详细信息，请参阅向消息流添加操作和编辑消息流中的操作中的操作表。
  8. 在响应操作字段中，单击添加操作以添加操作，然后选择要与此服务相关联的操作。可以添加多个操作。要了解有关希望添加的操作类型的详细信息，请参阅向消息流添加操作和编辑消息流中的操作中的操作表。
  9. 单击保存。
  10. 在编辑消息流页上，继续构造消息流，如创建、编辑和查看消息流中所述。
  11. 单击保存可提交当前会话中的更新。
  12. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
