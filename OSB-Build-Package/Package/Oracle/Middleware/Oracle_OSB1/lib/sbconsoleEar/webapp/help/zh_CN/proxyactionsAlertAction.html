添加预警操作

根据管道中的消息上下文，使用预警操作生成预警，以将其发送到预警目标。与 SLA 预警不同，由预警操作生成的通知主要用于业务目的或报告错误，而不是用于监视系统健康状况。配置和选择预警目标时应从此方面考虑。要了解预警目标的详细信息，请参阅预警目标。

如果没有为服务或在域级别启用管道预警，会在消息处理期间绕过配置的预警操作。

开始之前

这些说明假定您已在编辑消息流页中编辑消息流，如创建、编辑和查看消息流中所述。

添加预警操作

  1. 导航至要添加操作的位置，如向消息流添加操作和编辑消息流中的操作中所述。
  2. 单击相应的图标，然后选择添加操作 > 报告 > 预警。
  3. 单击 <目标>。此时将显示选择预警目标页。从列表中选择所需的预警目标，然后单击提交。

    4. 默认情况下，预警始终会发送到 Oracle Service Bus 管理控制台。

  4. 单击 <表达式>。此时将显示 XQuery 表达式编辑器页。可通过基于上下文变量的 XQuery 表达式指定要添加至预警消息的消息上下文。请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。
  5. 在预警概要字段中，输入该预警的简要说明。这将是电子邮件通知的主题行，可包含的字符不能超过 80 个。如果没有提供说明，则改用预定义的主题行“Oracle Service Bus 预警”。
  6. 在严重性级别下拉列表中，为此预警选择下面一种严重性级别：普通、警告、次要、主要、严重和致命。

完成之后

配置完此操作后，继续配置其他操作或保存您的配置，如向消息流添加操作和编辑消息流中的操作中所述。
