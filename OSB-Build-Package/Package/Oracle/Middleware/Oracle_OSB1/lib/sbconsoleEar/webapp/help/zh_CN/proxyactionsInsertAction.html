添加插入操作

使用插入操作可将 XQuery 表达式的结果插入到已识别的位置 (相对于 XPath 表达式所选节点来说)。插入操作是一组更新操作中的一个操作。

开始之前

这些说明假定您已在编辑消息流页中编辑消息流，如创建、编辑和查看消息流中所述。

添加插入操作

  1. 导航至要添加操作的位置，如向消息流添加操作和编辑消息流中的操作中所述。
  2. 单击相应的图标，然后选择添加操作 > 消息处理 > 插入。
  3. 单击 <表达式>，以编辑 XQuery 表达式。XQuery 表达式用来创建数据，该数据将插入到命名变量中的指定位置。此时将显示 XQuery 表达式编辑器页。请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。
  4. 完成此表达式的编辑后，从下拉列表中选择相对位置。相对位置用于控制相对于 XPath 表达式的结果来执行插入的位置：
    • 之前 - 作为 XPath 表达式选定的每个元素或属性之前的同级
    • 之后 - 作为 XPath 表达式选定的每个元素或属性之后的同级
    • 作为以下项的第一个子级 - 作为 XPath 表达式识别的每个元素的第一个子级。如果 XPath 的结果返回属性，则会发生错误。
    • 作为以下项的最后一个子级 - 作为 XPath 表达式识别的每个元素的最后一个子级。如果 XPath 返回属性，则会发生错误。
  5. 单击 XPath。此时将显示 XPath 表达式编辑器页。请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。

    6. 有效配置包括满足以下条件的配置：

    • XQuery 和 XPath 表达式都返回元素。
    • XQuery 表达式和 XPath 表达式两者均返回属性，在这种情况下，XQuery 表达式必须返回属性。
  6. 完成 XPath 表达式的编辑后，在“变量”字段中输入一个上下文变量。XPath 会计算此变量的内容。要了解上下文变量的详细信息，请参阅附录 A“消息上下文”。

完成之后

配置完此操作后，继续配置其他操作或保存您的配置，如向消息流添加操作和编辑消息流中的操作中所述。
