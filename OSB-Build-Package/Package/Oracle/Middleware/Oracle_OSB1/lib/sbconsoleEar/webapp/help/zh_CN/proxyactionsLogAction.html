添加记录操作

使用记录操作可构造要记录的消息，并定义用于记录消息的一组属性。

开始之前

这些说明假定您已在编辑消息流页中编辑消息流，如创建、编辑和查看消息流中所述。

注：要查看日志文件或标准输出 (服务器控制台) 中的日志数据，必须将 Oracle WebLogic Server 日志记录设置为以下严重性级别：

要记录的最低严重性：信息
日志文件：信息
标准输出：信息

有关设置日志严重性级别的信息，请参阅 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的“Using Log Severity Levels”，网址为：http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

 

添加记录操作

  1. 请确保已全局启用日志记录。有关详细信息，请参阅全局启用日志记录。
  2. 导航至要添加操作的位置，如向消息流添加操作和编辑消息流中的操作中所述。
  3. 单击相应的图标，然后选择添加操作 > 报告 > 日志。
  4. 单击 <表达式>。此时将显示 XQuery 表达式编辑器页。可指定通过上下文变量中的 XQuery 表达式记录消息上下文。请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。
  5. 在批注字段中，输入此记录操作的注释。这些注释会随以前定义的表达式的结果一起记录。
  6. 在严重性级别下拉列表中，选择其中的一个选项。

    7. 表 21-5 日志操作严重性级别
    严重性级别
    典型用法
    信息
    用于报告正常的操作；是一种低级别的信息性消息。
    警告
    出现了一个可疑的操作或配置，但是可能不会影响正常的操作。
    错误
    出现了用户错误。系统或应用程序不必中断就可以处理此类错误，服务性能的下降也有限。
    调试
    在开发应用程序的过程中，可能会发现创建和使用在应用程序中提供低级别活动的详细说明的消息很有用。
    请确保代理服务操作设置中的日志严重性级别与日志操作严重性级别相同。有关代理服务操作设置的信息，请参阅为代理服务配置操作设置。

完成之后

配置完此操作后，继续配置其他操作或保存您的配置，如向消息流添加操作和编辑消息流中的操作中所述。
