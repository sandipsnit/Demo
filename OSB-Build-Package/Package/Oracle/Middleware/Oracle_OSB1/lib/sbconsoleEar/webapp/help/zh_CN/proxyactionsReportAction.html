添加报告操作

使用报告操作可为代理服务启用消息报告。

开始之前

这些说明假定您已在编辑消息流页中编辑消息流，如创建、编辑和查看消息流中所述。

添加报告操作

  1. 导航至要添加操作的位置，如向消息流添加操作和编辑消息流中的操作中所述。
  2. 单击相应的图标，然后选择添加操作 > 报告 > 报告。
  3. 单击 <表达式>。此时将显示 XQuery 表达式编辑器页。请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。XQuery 表达式用于创建将在 Oracle Service Bus 面板中报告的数据。
  4. 完成 XQuery 表达式的编辑后，单击添加密钥。此时将显示两个字段：密钥名字段和密钥值字段，字段中包括可单击以编辑 XPath 表达式的 XPath 链接和在其中可输入上下文变量的“在变量中”字段。

    5. 使用密钥值对可从任何消息上下文变量或消息有效负载提取密钥标识符，并可以忽略消息的其余部分。密钥是识别消息的便利方法。它们在“报告”模块中以报告索引的形式显示。请参阅查看报告消息和查看消息详细资料。

    1. 在密钥名字段中输入密钥名。
    2. 单击 <XPath>。此时将显示编辑 XPath 表达式页。请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。
    3. 在变量字段中输入上下文变量。要了解上下文变量的详细信息，请参阅附录 A“消息上下文”。
    4. 要添加更多密钥值，请单击密钥图标，然后选择添加密钥。要删除密钥，请单击密钥图标，然后选择删除此密钥。

例如，考虑在某个阶段执行对错误处理程序配置的报告操作。该操作可以报告出错时 fault 上下文变量的内容。报告操作的配置如下：

  • 密钥名 = errorCode
  • 密钥值 = /ctx:errorCode (位于变量 fault 中)。

只要在运行时执行该操作，就会通过报告数据流报告消息。下表显示了执行两次报告操作后的结果。

完成之后

配置完此操作后，继续配置其他操作或保存您的配置，如向消息流添加操作和编辑消息流中的操作中所述。
