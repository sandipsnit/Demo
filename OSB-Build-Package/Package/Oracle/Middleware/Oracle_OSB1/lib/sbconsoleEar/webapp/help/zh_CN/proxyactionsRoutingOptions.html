添加路由选项操作

使用“路由选项”操作可以修改 $outbound 中出站请求的以下任何或所有属性：URI、服务质量、模式、重试参数。虽然这些属性可以使用对 $outbound 执行的“赋值”、“插入”、“替换”或“删除”操作修改，但使用“路由选项”操作执行此任务更简单，不需要了解 XPath、XQuery 或 $outbound 上下文变量的结构。

“路由选项”操作只能在上下文变量 $outbound 有效时使用。不能将其添加到以下操作：

  • 发布
  • 动态发布
  • 发布表
  • 服务标注
  • 路由
  • 动态路由
  • 路由表

有关路由的详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Modeling Message Flow in Oracle Service Bus”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG181。

 

要配置路由选项操作，请执行以下操作：

  1. 导航至要添加操作的位置，如向消息流添加操作和编辑消息流中的操作中所述。
  2. 单击相应的图标，然后选择添加操作 > 通信 > 路由选项。
  3. 完成以下任一或全部步骤：
    • 设置出站消息的 URI：选择 URI，然后单击 XQuery 表达式编辑器。输入返回 URI 的表达式。这将替换已调用服务的 URI。
    • 要设置服务质量元素，请选择服务质量，然后从下拉列表中选择“服务质量”选项。这将替换自动计算的默认值。
    • 要设置模式，请选择模式，然后从下拉列表中选择“请求”或“请求 - 响应”。
      • 注： 此项通常已根据调用的服务的接口自动进行设置。但是，在诸如“任何 Soap 服务”或“任何 XML 服务”之类的一些情况中，并非如此。
    • 设置“重试间隔”：选择重试间隔，然后指定两次重试之间的秒数。这将替换使用已调用服务配置的默认值。
    • 设置“重试计数”：选择重试计数，然后指定终止操作之前系统必须尝试的重试次数。这将替换使用已调用服务配置的默认值。
    • 设置“消息优先级”：选择优先级，然后单击 XQuery 表达式编辑器。输入返回正整数的表达式。

完成之后

配置完此操作后，继续配置其他操作或保存您的配置，如向消息流添加操作和编辑消息流中的操作中所述。
