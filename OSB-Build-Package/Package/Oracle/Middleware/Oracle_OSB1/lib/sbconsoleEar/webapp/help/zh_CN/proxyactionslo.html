向路由节点添加路由表

路由表是包装在一个切换式条件表中的一组路由。它是一种简化的构造，通过它可以根据单个 XQuery 表达式的结果选择不同的路由。

在阶段编辑器中，存在累积级别最多为 4 的嵌套限制。如果尝试添加第 5 层，将不显示此嵌套操作。累积级别包括所有分支操作：If...Then... 条件、发布表和路由表。例如，如果您有 2 层的条件操作，还有一个其中包含路由表的发布表，则累积级别总计为 4 层。如果尝试添加其他条件操作（到最后一个发布表），将不显示该条件。

这是一项最终操作，表示在此操作后不能添加其他操作。但是，此操作可以包含请求和响应操作。

有关路由的详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Modeling Message Flow in Oracle Service Bus”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG181。

开始之前

这些说明假定您已在编辑消息流页中编辑消息流，如创建、编辑和查看消息流中所述。

向路由节点添加路由表

  1. 导航至要添加操作的位置，如向消息流添加操作和编辑消息流中的操作中所述。
  2. 单击路由节点图标，然后单击编辑路由。此时将显示编辑阶段配置页。
  3. 单击添加操作图标，然后选择通信 > 路由表。此时将显示路由表操作。
  4. 在“运算符”列表中，选择比较运算符，然后在相邻字段中输入一个值表达式。
  5. 单击“<服务>”。此时将显示“选择服务”页。
  6. 从列表中选择一个服务，然后单击“提交”。
  7. 如果要对服务调用某个操作，请从“操作”列表中选择相应的操作
  8. 如果希望使出站操作与入站操作相同，请选中为出站使用入站操作复选框。
  9. 在“请求操作”字段中，单击“添加操作”以添加操作，然后选择要与此服务相关联的操作。可以添加多个操作。
  10. 在“响应操作”字段中，单击“添加操作”以添加操作，然后选择要与此服务相关联的操作。可以添加多个操作。

    11. 要了解可以添加的请求和响应操作类型的详细信息，请参阅向消息流添加操作和编辑消息流中的操作。

  11. 要插入新案例，请单击案例图标，然后选择插入新案例。
  12. 对新案例重复步骤 2 至步骤 7。如果先前所有的案例都不满足要求，可以单击案例图标，然后选择插入默认案例，以将默认案例插入其路由被选中的案例末端。
  13. 单击“保存”。
  14. 在编辑消息流页上，继续构造消息流，如创建、编辑和查看消息流中所述。
  15. 单击保存可提交当前会话中的更新。
  16. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
