向消息流添加操作分支

消息流定义基于 Web 服务描述语言 (WSDL) 的代理服务时，需要进行特定于操作的处理。Oracle Service Bus 提供了最小配置的分支节点 (可以根据操作自动进行分支处理)，无须根据操作手动配置分支节点。换而言之，在消息流中创建操作分支节点时，可以根据 WSDL 中定义的操作快速构建分支处理逻辑，因为 Oracle Service Bus 管理控制台会在分支节点配置页中显示这些操作。

分支节点使处理过程只能沿着几个可能路径中的一个路径继续进行。分支处理是由基于 XPath 的切换式表驱动的。该表中的每个分支都指定了一个条件 (例如，<500)，该条件是根据单个 XPath 表达式以消息流的顺序计算的 (例如，基于 $body 的 ./ns: PurchaseOrder/ns:totalCost)。首先满足的条件确定后面要沿哪一个分支继续。如果不满足任何分支条件，则后面将沿默认分支继续。一个分支节点在消息流中可能具有几个子节点：每个分支一个，包括默认分支。

开始之前

这些说明假定您已在编辑消息流页中编辑了消息流，如创建、编辑和查看消息流中所述。

要向消息流添加操作分支，请执行以下操作：

  1. 单击管道对节点图标或分支节点图标，然后单击创建操作分支。将添加操作分支节点，并将已插入的分支节点后的任何现有节点都移至新操作分支节点的默认分支。
  2. 要更改默认名称并添加分支节点的说明，请执行以下操作：
    1. 单击操作分支图标，然后单击编辑名称和批注。
    2. 根据需要更改名称和说明。
    3. 单击保存。
  3. 要添加分支定义，请单击操作分支图标，然后单击编辑分支。此时将显示编辑分支节点页。
  4. 在操作分支定义面板中，选择一个服务操作。
  5. 或者，在选项下，执行以下操作：
    • 单击添加新分支，将新的分支定义添加至此分支节点。
    • 单击删除此分支，删除分支定义。
    • 单击上移分支或下移分支，更改分支定义的位置。此选项仅在有多个分支定义时才会显示。
  6. 单击保存。
  7. 在编辑消息流页上，继续构造消息流，如创建、编辑和查看消息流中所述。
  8. 单击保存可提交当前会话中的更新。
  9. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
