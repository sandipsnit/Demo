向消息流添加管道对节点

消息流可以不包含管道对节点，也可以包含多个管道对节点：代理服务 (或对服务的操作) 的请求和响应管道，以及可为阶段、管道和代理服务定义的错误处理程序管道。管道可以包括一个或多个阶段，阶段又可以包括操作。

开始之前

这些说明假定您已在编辑消息流页中编辑了消息流，如创建、编辑和查看消息流中所述。

要向消息流添加管道对节点，请执行以下操作：

  1. 单击代理服务图标，然后单击添加管道对。
  2. 要更改默认名称并添加管道对节点的说明，请执行以下操作：
    1. 单击管道对节点图标，然后单击编辑名称和批注。
    2. 根据需要更改名称和说明。
    3. 单击保存。
      4. 注： 重命名管道或路由节点时，由于管道计数器重置为零，监视模块中面板页上显示的消息数可能与其他组件的消息数不关联。这是因为 Oracle Service Bus 将重命名视为删除和重新创建操作。经过与服务的监视间隔一样长的时间后，这些数目会再次关联。
  3. 要将阶段添加至管道，请参阅向管道添加阶段。
  4. 将操作添加至管道中的阶段。请参阅向消息流添加操作和编辑消息流中的操作。
  5. 在编辑消息流页上，继续构造消息流，如创建、编辑和查看消息流中所述。
  6. 单击保存可提交当前会话中的更新。
  7. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
