在编辑器工作区的文本字段中构建表达式

XQuery 表达式编辑器、XQuery 条件编辑器和 XPath 表达式编辑器都提供了文本字段，您可以通过直接键入或粘贴选项板中的项，在这些文本字段中构建表达式。

开始之前

这些说明假定您将在 XQuery 表达式编辑器、XQuery 条件编辑器或 XPath 表达式编辑器中创建或编辑表达式，如创建和编辑内嵌 XQuery 和 XPath 表达式中所述。

要在文本字段中构建表达式，请执行以下操作：

  1. 显示包含文本字段的面板。根据编辑器，执行以下操作之一：
    • 在 XQuery 表达式编辑器中，如果 XQuery 文本 (位于工作区按钮栏下) 尚未处于选定状态，请选择该选项。
    • 在 XQuery 条件编辑器中，如果文本 (位于工作区按钮栏下) 尚未处于选定状态，请选择该选项。
    • 在 XPath 表达式编辑器中，不必选择任何内容，因为没有可供选择其他工具的选项。
      • 注： 选择以上任何链接都将显示一个文本字段，可在该字段中创建适用于上下文的完整表达式。但是，只要编辑器中提供了文本字段，就可使用本主题中描述的工具和技术，例如，在将来自导入的资源的变量绑定到内嵌表达式的情况下 (如将外部 XQuery 资源绑定到内嵌 XQuery 和将外部 XSLT 资源绑定到内嵌 XQuery 中所述)。
  2. 如果需要，请将表达式或表达式片断键入或粘贴到字段中。如果要以此方式创建完整表达式，请跳至下面的步骤 7。否则，请继续下一步。
  3. 选择包含要添加到表达式的项的选项板，并找到要添加的项，如下面的表 22-1 中所述。

    4. 表 22-1 选项板

    名称空间定义
    列出默认的 Oracle Service Bus 名称空间、变量名称空间和用户定义的名称空间。列出名称空间缩写 (如果已定义)。

    包含一组标准的 XQuery 函数，这些函数按字母顺序和类型组织，其中包含您开发的定制函数。通过单击加号 (+) 或减号 (-) 展开或折叠树中的节点。
    另请参阅：
    • XQuery 转换
    • Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“XQuery Implementation”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG444。
    变量结构
    以树的形式显示变量及其内容，这样有助于可视化。
    显示变量结构及其内容
    • 从位于选项板顶部的下拉列表中选择结构的名称。列表显示内置消息上下文变量 (attachments、body、header、outbound 和 inbound)，以及用户定义的任何结构，这些结构是按类型 (“XML 类型”、“服务接口”和“简单类型”) 组织的。
    • 通过单击加号 (+) 或减号 (-) 展开或折叠树中的节点。
    也可以定义自己的变量结构。请参阅在 XQuery 编辑器中创建变量结构。
    变量结构不创建变量。变量是在运行时作为阶段中“分配”操作的目标创建的。
    如果从变量结构树将项目插入到文本字段中，则该项目是作为用于描述路径的 XPath 表达式插入的。
    另请参阅 XQuery 转换。

    1. 在选项板中单击一个项目。该项目将显示在属性检查器窗格中：
      • 函数会随您必须提供的任何值的占位符一起显示。
      • 变量及其属性会作为 XPath 表达式显示。

    XQuery 函数选项板或变量结构选项板
    1. 在选项板中单击一个项目。该项目将显示在属性检查器窗格中。
    2. 使用标准键盘或鼠标操作，在属性检查器中选择并复制项目。
    3. 在文本字段中选择一个位置，使用标准键盘或鼠标操作将项目粘贴到该文本字段中。

  5. 继续拖放函数以构建所需表达式。
  6. 根据需要，在文本字段中编辑表达式。
  7. 还可以执行下面的任一或全部操作：
    • 单击验证。如果表达式验证成功，则会显示消息。
    • 单击测试可测试表达式。请参阅执行 XQuery 测试。
  8. 单击保存关闭编辑器并将表达式插入操作中。

完成之后

继续配置该操作，如向消息流添加操作和编辑消息流中的操作中所述。
