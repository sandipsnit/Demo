创建和配置代理服务

此部分介绍如何创建和配置代理服务。

对于 JCA 服务，可以从 JCA 绑定生成代理服务，如从 JCA 绑定资源生成代理服务中所述。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择项目浏览器，然后选择要将代理服务添加到的项目或文件夹。
  3. 在“项目/文件夹视图”页上，选择创建资源下拉列表中的代理服务。
  4. 在“一般配置”页上，提供服务的名称并选择要创建的服务的类型。

    如需命名指导，请参阅资源命名限制。
  5. 单击下一步。后续页取决于在第一页上做出的选择。在每一个后续页上输入相应的信息，直到到达概要页，然后单击保存以保存当前会话中的服务。

    6. 有关填写各个页的详细说明，请参阅以下内容：

    • “一般配置”页
    • “消息类型配置”页
    • “传输配置”页
    • “协议特定的传输配置”页
    • “操作选择配置”页
    • “消息处理”页
    • “概要”页
    • “代理服务策略”页
    • “代理服务安全”页
  6. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
  7. 创建代理服务之后，下一步是配置其消息流。消息流定义了代理服务的实施。要了解详细信息，请参阅创建、编辑和查看消息流。

从 JCA 绑定资源生成代理服务

可以从 Oracle Service Bus 中的入站 JCA 绑定资源生成 JCA 代理服务。有关 JCA 绑定资源的详细信息，请参阅第 12 章“JCA 绑定”。

要从 JCA 绑定生成 JCA 代理服务，请执行以下操作：

  1. 在 Oracle Service Bus 管理控制台中，如果尚未处于创建或编辑模式，请在“更改中心”中单击创建或编辑。
  2. 在资源浏览器中，单击 JCA 绑定。
  3. 找到要用于生成服务的 JCA 绑定，然后单击操作图标。
  4. 在显示的窗口中，确认 WSDL 的名称以及要生成的服务，选择这些新资源的位置，然后单击生成。

    5. Oracle Service Bus 将生成服务及其对应的 WSDL。

  5. 适当时修改有关生成的服务的任何其他配置详细资料，如端点 URI。
