“FTP 传输配置”页

创建/编辑代理服务 - FTP 传输配置页是一系列用于创建和编辑代理服务的页面中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

使用此页可为使用 FTP 传输协议的代理服务配置传输设置。表 19-6 介绍了如何使用此页。

表 19-6“创建/编辑代理服务 - FTP 传输配置”页
选项
创建或编辑...
用户验证
如果 FTP 服务器的用户是匿名用户，则选择匿名；如果 FTP 服务器的用户是外部配置的帐户，则选择外部用户。
标识 (电子邮件 ID)
仅当用户验证选项设置为匿名时，此字段才可用。
输入匿名用户的邮件 ID。
服务帐户
仅当用户验证选项设置为外部用户时，此字段才可用。
输入用户的服务帐户。当用户验证选项设置为外部用户时，此字段为必填字段。
引用传递
选中此复选框可将文件临时保存在归档目录中，并将其作为标头中的引用进行传递。
远程传输
选中该复选框可在进行处理时直接从远程服务器传输 FTP 文件。如果选择此选项，则归档目录即是远程 FTP 服务器上的远程目录。因此，应以 FTP 用户目录的相对路径的形式指定归档目录。
文件掩码
输入要选取的文件的正则表达式。默认值为 *.*。
托管服务器
此字段仅在集群域中可用。
选择托管服务器作为轮询服务器。所有托管服务器均可处理消息，但是只有一个托管服务器可以轮询消息。
轮询间隔
输入轮询间隔 (秒)。默认值为 60。
读取限制
指定每次轮询扫描读取的最大消息数。输入 0 可指定没有限制。默认值为 10。
读后操作
选择如何处理阅读后的消息。
  • 归档 - 将消息归档。
  • 删除 - 将消息删除。
传输模式
选择 ASCII 或二进制作为传输模式。
归档目录
如果读后操作字段设置为归档，请指定归档位置的路径。如果选择了引用传递选项，则此字段是必填字段。
注： 归档、下载和错误目录都是绝对路径，并且是自动创建的。如果指定了相对路径，则将相对于启动 WebLogic 服务器的 Java 进程创建文件。
下载目录
输入在文件传输期间将文件下载到的本地计算机上的目录。
注： 归档、下载和错误目录都是绝对路径，并且是自动创建的。如果指定了相对路径，则将相对于启动 WebLogic 服务器的 Java 进程创建文件。
错误目录
输入在出现问题的情况下要将消息发布到的位置。
注： 归档、下载和错误目录都是绝对路径，并且是自动创建的。如果指定了相对路径，则将相对于启动 WebLogic 服务器的 Java 进程创建文件。
请求编码
接受默认值 UTF-8 作为 FTP 传输中请求的字符集编码。
扫描子目录
选中该复选框可递归扫描所有目录
按到达时间排序
选中该复选框可按到达时间顺序传送事件。
超时
输入在删除连接之前所需经过的套接字超时间隔 (秒)。如果输入 0，则没有超时。
重试计数
指定 FTP 连接失败时重试的次数。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑代理服务 - 概要”页上查看并保存此配置。
