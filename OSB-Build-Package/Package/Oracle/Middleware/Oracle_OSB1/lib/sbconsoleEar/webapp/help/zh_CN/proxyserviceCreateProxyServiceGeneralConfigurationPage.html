“一般配置”页

创建/编辑代理服务 - 一般配置页是一系列用于创建和编辑代理服务的页面中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

使用此页可修改代理服务的一般配置设置。

创建代理服务时，这是一系列用于配置该服务的页中显示的第一页。在该页之后显示的页随在此页上做出的选择而有所不同。

表 19-1 介绍了如何使用此页。

表 19-1“创建/编辑代理服务 - 一般配置”页
选项
创建或编辑...
服务名
输入代理服务的唯一名称。

如需命名指导，请参阅资源命名限制。
说明
输入说明。
服务类型
选择要创建的代理服务的类型：
  • WSDL Web 服务 - 选择此选项可创建基于 WSDL 的代理服务。然后，输入 WSDL 名称。该名称由其路径限定，例如 myProject/myFolder/myWSDL。或者，单击浏览从选择 WSDL 定义 页中选择 WSDL。

    • (端口或绑定) - 输入端口的名称 (在 WSDL 中定义) 以描述实际的传输地址，或者输入绑定的名称 (在 WSDL 中定义) 以映射到传输地址。如果使用浏览选择 WSDL，则选择 WSDL 定义页将列出所有已定义的端口和绑定。

注： 如果要将 SOAP 正文的类型用于操作，则要确保 WSDL 的任何两项操作都不使用同一输入消息。无法通过检查输入消息来唯一地确定 SOAP 正文类型操作。
  • 传输类型服务 - 选择此选项可创建 JEJB 代理服务。
  • 消息传送服务 - 选择此选项创建的服务，能够接收一种数据类型的消息而用另一种数据类型的消息进行响应。这些交换可以是“请求/响应”式的或是单向的。

    • (HTTP GET 仅在“任何 XML 服务”和“消息传送服务”服务类型中受支持。)

  • 任何 SOAP 服务 - 选择此选项可创建没有明确定义的具体接口的 SOAP 服务。

    • 在下拉列表中选择 SOAP 1.1 或 SOAP 1.2，以便指定要使用的 SOAP 版本。

  • 任何 XML 服务 - 选择此选项可创建没有明确定义的具体接口的 XML 服务。

    • (HTTP GET 仅在“任何 XML 服务”和“消息传送服务”服务类型中受支持。)

  • 业务服务 - 选择此选项可创建使用路由节点 (用于路由到您选择的业务服务) 的代理服务。不能基于传输类型的业务服务创建代理服务。如果您基于 DSP 传输业务服务创建代理服务，则 Oracle Service Bus 会将代理服务的传输类型切换为 HTTP，因为 DSP 传输类型不能用于代理服务。可以将代理服务的传输类型更改为任何其他可用的传输。

    • 输入业务服务的路径 (项目/文件夹) 和名称；或者单击浏览可从选择业务服务页中选择业务服务。

服务类型 (续)
  • 代理服务 - 选择此选项可克隆现有代理服务。由于 Oracle Service Bus 不接受多项服务具有相同的 URI，因此，必须更改克隆服务的 URI。
注： 如果服务是基于其他服务创建的，将按以下方式维护预警规则：
    • 如果代理服务是基于业务服务创建的，或者业务服务是基于代理服务创建的，则会删除预警规则 (如果有)。
    • 如果代理服务是基于其他代理服务创建的，或者业务服务是基于其他业务服务创建的，则会保留预警规则 (如果有)。
服务密钥提供程序
如有必要，请输入路径 (项目/文件夹) 和服务密钥提供程序的名称，或者单击浏览从选择服务密钥提供程序页中选择一个服务密钥提供程序。
只有在特定情况下才需要服务密钥提供程序：
  • 出站双向 TLS/SSL，代理服务使用它将消息路由到需要客户机证书验证的 HTTPS 服务。
  • 在某些提供 Web 服务安全的情况下，例如，在代理服务要求对消息进行加密的情况下。
要添加已启用 Web 服务安全的代理服务，必须基于具有 WS-Policy 附件的 WSDL (端口或绑定) 创建代理服务。
有关详细信息，请参阅服务密钥提供程序。要了解如何创建服务密钥提供程序，请参阅添加服务密钥提供程序。
发布到注册表
此选项只有在默认 UDDI 注册表存在时才显示。
选中该选项可自动将代理服务发布到默认注册表。有关详细信息，请参阅使用自动发布。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑代理服务 - 概要”页上查看并保存此配置。
