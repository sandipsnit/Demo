“HTTP 传输配置”页

现在，HTTP 传输既支持 HTTP 端点，又支持 HTTPS 端点。

创建/编辑代理服务 - HTTP 传输配置页是一系列用于创建和编辑代理服务的页面中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

使用此页可为使用 HTTP 传输协议的代理服务配置传输设置。表 19-7 介绍了如何使用此页。

表 19-7“创建/编辑代理服务 - HTTP 传输配置”页
选项
创建或编辑...
HTTPS 是必需的
为入站 HTTPS 端点选中此复选框。
要了解详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Configuring Transport-Level Security”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557。
验证
选择以下选项之一：
  • 无 - 指定验证不是必需的。
  • 基本 - 指定访问该服务需要进行基本验证。

    • 基本验证可指示 WebLogic 服务器根据安全领域中所配置的验证提供程序 (如轻型目录访问协议 (LDAP) 目录服务和 Windows Active Directory)，对使用用户名和口令的客户机进行验证。客户机必须在 HTTP 请求标头中发送其用户名和口令。

    强烈建议不要通过 HTTP 进行基本验证，因为口令是以明文形式发送的。但是，通过 HTTPS 发送口令则是安全的，因为 HTTPS 提供了一个加密通道。

    警告：默认情况下，所有用户 (授权用户和匿名用户) 都可以访问代理服务。要限制可访问代理服务的用户，请创建传输级授权策略。请参阅编辑传输级访问策略。

  • 客户机证书 - 指定加密的通信和强客户机验证 (双向 SSL)。要了解详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Configuring Transport-Level Security”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557。
  • 定制验证 - 指定包含在 HTTP 标头中的验证标记。通过使用此客户机提供的标记建立客户机的标识。必须配置将标记映射到 Oracle Service Bus 用户的标识断言提供程序。

    • 定制验证标记可以是所配置的 WebLogic 服务器标识断言提供程序支持的任何活动标记类型。

分派策略
为此端点选择分派策略。将此字段保持为空则可使用默认的分派策略。
分派策略是指您希望用于此服务端点的 WLS 工作管理器实例。有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
请求编码
  • 对于 HTTP 入站传输：

    • 如果未在客户机请求中指定 Content-Type 标头的字符集编码参数，请输入一个字符集编码参数。如果未输入值，该字段将默认为 ISO-8859-1。

  • 对于 HTTP 出站传输：

    • 如果尚未配置请求编码，Oracle Service Bus 请求业务服务时，将在运行时确定最合适的编码。对于非传递方案，运行时默认的字符编码是 UTF-8。但是，如果是传递方案，运行时将传递出站响应时接收的编码。

响应编码
接受默认值 ISO-8859-1 作为 HTTP 传输中响应的字符集编码，或者输入其他字符集编码。
验证标头
输入 Oracle Service Bus 将要从中提取标记的 HTTP 标头 (除 Authorization 之外的任何一个)。仅当选中定制验证复选框时，此字段才可用。
例如，client-xyz-token。
验证标记类型
选择验证标记类型。只有为标识断言提供程序配置的活动标记类型才可用。（有关定制标识的详细信息，请参阅配置标识断言提供程序。）仅当选中定制验证复选框时，此字段才可用。
