“MQ 传输配置”页

创建/编辑代理服务 - MQ 传输配置页是一系列用于创建和编辑代理服务的页面中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“MQ Transport”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117。

开始之前

配置 MQ 连接资源。请参阅 MQ 连接。

配置 MQ 传输

使用此页可为使用本机 MQ 传输协议的代理服务配置传输设置。表 19-11 介绍了如何使用此页。

表 19-11“创建/编辑代理服务 - MQ 传输配置”页
选项
创建或编辑...
轮询间隔
输入轮询间隔 (毫秒)。默认值为 1000。
是否需要响应
选择此选项可指定希望在发送出站消息之后获得响应。
响应相关性模式
只有在选中是否需要响应复选框后，该选项才可用。
指定响应相关性模式是否应基于 MessageID 或 CorrelationID。
MQ 响应 URI
只有在选中是否需要响应复选框后，该选项才可用。
应该将响应发布到的目标。输入响应 URI，其格式应该与端点 URI 的格式相同：mq://<local-queue-name>?conn=<mq-connection-resource-ref>
响应消息类型
只有在选中是否需要响应复选框后，该选项才可用。
选择以下选项之一：
  • 字节 (用于未解析的字节流)
  • 文本 (用于文本消息)
分派策略
选择此端点的分派策略。
分派策略是指您希望用于此服务端点的 WLS 工作管理器实例。有关工作管理器的信息，请参阅 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的“Using Work Managers to Optimize Scheduled Work”，网址为：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
退回阈值
输入一个值，该值代表管道在将消息重定向到死信 URI 字段中指定的队列之前，应对消息执行重试操作的次数。
如果未为此字段指定值，则不进行任何重试操作就将消息重定向到死信队列。
MQ 死信 URI
输入死信队列的 URI，管道对请求消息尝试了退回阈值字段中指定的重试次数之后，应将请求消息重定向到该 URI。
如果没有为此字段指定值，则对于每次轮询，在重试了退回阈值字段中指定的次数之后，消息将返回队列，并且 MQ 传输将放弃消息。死信 URI 与端点 URI 使用相同的格式。
端点 URI 'GET' 选项
从以下选项中进行选择以输入 MQ GET 消息选项：
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
可以使用“|”或“+”分隔多个选项。例如，可以指定以下内容：
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
从入站队列中读取消息时，将应用 MQ GET 消息选项。
处理 RFH2 标头
选择此选项可对消息有效负载的 WebSphere MQ RFH2 标头进行语法分析，并自动生成包含 RFH2 数据的 RFH2Headers 传输标头。
如果未选择此选项，则会传递收到的有效负载。

有关 MQ 传输如何处理 RFH2 标头的信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“About RFH2 Headers”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑代理服务 - 概要”页上查看并保存此配置。
