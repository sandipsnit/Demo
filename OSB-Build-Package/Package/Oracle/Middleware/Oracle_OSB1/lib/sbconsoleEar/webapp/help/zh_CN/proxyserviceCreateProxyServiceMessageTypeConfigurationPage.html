“消息类型配置”页

创建/编辑代理服务 - 消息类型配置页是一系列用于创建和编辑代理服务的页中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

使用此页可为类型为消息传送服务的代理服务配置消息类型。

消息传送服务的绑定定义包括配置所交换消息的 content-type。响应的 content-type 无须与请求的 content-type 相同；因此响应是单独配置的 (例如，服务可能会接受 MFL 消息，而返回 XML 接收确认)。

注： 类型为消息传送服务的电子邮件、File、FTP 或 SFTP 传输代理服务只支持单向消息传送；响应消息类型应为 none。如果选择除 none 以外的选项，“传输配置”页上将不提供电子邮件、File、FTP 或 SFTP 协议。

表 19-2 介绍了如何使用创建/编辑代理服务 - 消息类型配置页。

表 19-2“创建/编辑代理服务 - 消息类型配置”页
选项
创建或编辑...
请求消息类型
选择请求消息的消息类型：
  • 无 - 如果没有任何请求消息，请选择此选项。
  • 二进制 - 如果消息的内容类型未知或者不重要，请选择此选项。
  • 文本 - 如果可将消息限制为文本，请选择此选项。
  • MFL - 如果消息是一个符合 MFL 定义的二进制文档，请选择此选项。输入 MFL 文件名 (由其路径限定)，或者单击浏览以选择文件。

    • 您只能配置一个 MFL 文件。

    注：要支持多个 MFL 文件，应将内容定义为二进制或文本类型，并使用消息流中的 MFL 操作来转换为 XML。

  • XML - 如果消息是 XML 文档，请选择此选项。输入 XML 文件名 (由其路径限定)，或者单击浏览以选择文件。

    • 还可以通过声明 (在元素或类型字段中) 所交换的 XML 文档的 XML 方案类型，提供某些类型信息。

  • Java - 如果 Java 对象是通过请求发送的，请选择此选项。JMS 传输用于 Java 对象消息。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Sending and Receiving Java Objects in Messages”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。
响应消息类型
选择响应消息的消息类型：
  • 无 - 如果没有任何响应消息，请选择此选项。
  • 二进制 - 如果消息的内容类型未知或者不重要，请选择此选项。
  • 文本 - 如果可将消息限制为文本，请选择此选项。
  • MFL - 如果消息是一个符合 MFL 定义的二进制文档，请选择此选项。输入 MFL 文件名 (由其路径限定)，或者单击浏览以选择文件。

    • 您只能配置一个 MFL 文件。

    注：要支持多个 MFL 文件，应将内容定义为二进制或文本类型，并使用消息流中的 MFL 操作来转换为 XML。

  • XML - 如果消息是 XML 文档，请选择此选项。输入 XML 文件名 (由其路径限定)，或者单击浏览以选择文件。

    • 还可以通过声明 (在元素或类型字段中) 所交换的 XML 文档的 XML 方案类型，提供某些类型信息。

  • Java - 如果 Java 对象是通过响应接收的，请选择此选项。JMS 传输用于 Java 对象消息。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Sending and Receiving Java Objects in Messages”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。
