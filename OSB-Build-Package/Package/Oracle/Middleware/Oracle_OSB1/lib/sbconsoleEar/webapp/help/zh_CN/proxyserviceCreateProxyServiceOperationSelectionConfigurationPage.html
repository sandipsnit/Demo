“操作选择配置”页

创建/编辑代理服务 - 操作选择配置页是一系列用于创建和编辑代理服务的页中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。仅当所创建的服务具有操作时，才会显示此页。

使用此页可强制符合 WS-I 标准 (仅适用于 SOAP 1.1 服务)，并可选择用于确定由此代理服务调用的操作的选择算法。此选项仅对于从 WSDL 中定义的 SOAP 或 XML 服务可用。

WSDL 规范定义了一个默认算法，以基于接收的 SOAP 消息的类型计算要调用哪一个操作。但是，有些情况下 (例如，性能问题、签名/加密问题，或默认算法不适用)，您可能需要基于其他方法选择操作。

Oracle Service Bus 提供了其他算法。每个算法都沿用相同的模式并基于表达式的计算，来获得用于在静态表中查找相应操作的值。

通常，Oracle Service Bus 并不在意入站消息丢失数据从而导致无法确定操作，也不在意入站消息具有与有效操作不对应的数据。这些条件将导致 $operation 为空。Oracle Service Bus 并不拒绝所有此类消息，而不会在上下文中初始化操作变量，但是会继续处理消息。

但是，如果代理服务基于 WSDL，并且以下条件中至少有一个为真，则会强制符合安全要求：

  • WSDL 具有 WS-Security 策略，且代理是主动中介。
  • 代理具有消息级定制验证 (定制标记或用户名/口令)。

如果满足这些条件，则会进行运行时检查以确保操作选择算法返回有效的操作名。如果操作选择返回空值或不属于 WSDL 的操作，则会拒绝消息并引发错误。

选择算法
选择以下选项之一，然后执行所需的其他步骤：
  • 传输标头 - 选择此算法可定义包含查找值的传输标头。然后，执行以下操作：
    • 在标头名称字段中输入传输标头，该传输标头用来提取用作选择被调用操作的密钥的值。
    • 在操作映射字段下的值字段中，为各个操作指定值。此值用作操作的密钥。
  • SOAPAction 标头 - 选择此算法可指定自动从与此代理服务关联的 WSDL 执行操作映射。
  • WS-Addressing - 选择此算法可指定查找值由位于 SOAP 消息的 SOAP 标头中的 WS-Addressing Action 标记包含。然后，在操作映射字段下的值字段中，指定各个操作的值。此值用作操作的密钥。
  • SOAP 标头 - 选择此算法可定义要依据 SOAP 标头计算的 XPath 表达式。这使您可以获取查找值。然后，执行以下操作：
    • 在 XPath 表达式字段中指定 XPath 表达式，该表达式用来提取用作选择被调用操作的密钥的值。
    • 在操作映射字段下的值字段中，为各个操作指定值。此值用作操作的密钥。
  • SOAP 正文类型 - 这是由 WSDL 规范定义的默认算法，用来基于收到的 SOAP 消息的类型计算调用哪一个操作。

    • 如果针对使用加密正文的 Web 服务安全传递场景配置了代理服务，则不能选择此算法。类似的警告适用于传递加密的 SOAP 标头。

    如果具有的 WSDL 有两个带有相同输入消息的操作，请不要为操作选择此算法，因为无法通过检查输入消息来唯一标识操作。

  • 有效负载类型 - 仅可用于基于 WSDL 端口或 WSDL 绑定的 XML 服务。
