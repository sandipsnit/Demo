“传输配置”页

创建/编辑代理服务 - 传输配置页是一系列用于创建和编辑代理服务的页面中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

使用此页可为代理服务选择传输协议，并可设置其他一般传输配置设置。表 19-3 介绍了如何使用此页。

注： 入站传输级安全性应用于客户机应用程序和 Oracle Service Bus 代理服务。出站传输级安全性应用于 Oracle Service Bus 代理服务和业务服务之间的连接。要了解有关传输级安全性的详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Configuring Transport-Level Security”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557。

表 19-3“创建/编辑代理服务 - 传输配置”页
选项
创建或编辑...
协议
从列表中选择传输协议。可用的协议因要创建的服务类型而异：
  • WSDL Web 服务：http、jca、jms、本地、sb、ws
  • 传输类型：jejb
  • 消息传送服务：电子邮件、file、ftp、http、jms、本地、mq (如果可用)、sftp、tuxedo
  • 任何 SOAP 服务：http、jms、本地、sb
  • 任何 XML 服务：电子邮件、File、ftp、http、jms、本地、mq (如果可用)、sb、sftp、tuxedo
端点 URI
以基于您在以上协议字段中选择的传输协议的格式，输入端点 URI。这些格式如下所示：
  • 电子邮件 - mailfrom:mail-server-hostname:mail-server-port
  • file - file:///drivename:/somename
  • ftp - ftp://hostname:port/directory
  • http - someName

    • HTTP 传输支持 HTTP 端点和 HTTPS 端点。

  • jca - jca://<resource_adapter_jndi>
  • jejb - URL 格式为 ejb_jndi_name。为 JEJB 代理服务配置的 URI 将成为全局 JNDI 名称，用于查找客户机 JAR 中远程/业务接口的 JEJB 传输所生成的无状态会话 bean。

    • 注：对于 EJB 3.0，ejb_jndi_name 是生成的 Bean 中 @javax.ejb.Stateless 注释的 mappedName 属性。生成的 EJB 服务的查找 JNDI 名称具有 #interface_class 后缀，这是业务接口的全限定名称。

    您可以访问以下 JEJB 代理服务：

    EJB 2.1：protocol://host:port/ejb_jndi_name

    EJB 3.0：protocol://host:port/ejb_jndi_name#interface_class

    注：如果 EJB 运行于 IBM WebSphere 上，则 ejb_jndi_name 的格式必须为：

    cell/nodes/node_name/servers/server_name/ejb_jndi_name

    或

    cell/clusters/cluster_name/ejb_jndi_name

    如 http://publib.boulder.ibm.com/infocenter/wasinfo/v6r1/index.jsp?topic=/com.ibm.websphere.nd.iseries.doc/info/iseriesnd/ae/rnam_example_prop3.html 上的 IBM WebSphere 文档中所述。

  • jms - jms://host:port/factoryJndiName/destJndiName

    • 要将 JMS 目标定位到多个服务器，请使用以下 URI 格式：

    jms://host1:port,host2:port/QueueConnectionFactory/DestName

    请注意，在创建代理服务时，可以配置 JMS 端点 URI，即使位于该端点的服务器不可用时也是如此。但是，对于 JMS，激活会话后，该端点必须可用。要了解详细信息，请参阅激活会话。

    注：虽然 Oracle WebLogic Server 允许在 JNDI 名称中使用正斜杠 (例如“myqueues/myqueue”)，但是带正斜杠的 JNDI 名称会对 Oracle Service Bus 所需的 URI 格式产生干扰，因此您不能使用这些名称。要解决此问题，请定义一个 JMS 外部服务器，然后在 URI 中引用该外部服务器。有关详细信息，请参阅 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的“Configure foreign servers”，网址为：http://www.oracle.com/pls/as111170/lookup?id=WLACH。

    在集群中：JMS URI 中的主机名必须与在 WebLogic 服务器中配置的集群服务器主机名完全匹配。

  • 本地

    • 此传输不需要端点 URI。

  • mq - mq://<local-queue-name>?conn=<mq-connection-resource-ref>

    • local-queue-name 是代理服务从中读取消息的 MQ 队列的名称。

    mq-connection-resource-ref 是 MQ 连接资源的路径 (项目/文件夹) 和名称；例如，default/my_MQconnection。

    注： 端点 URI 不能包含空格，因此不要创建名称中带有空格的 MQ 连接资源或项目/文件夹。

    要使 MQ 传输可在 Oracle Service Bus 中使用，请参阅 MQ 连接。

端点 URI (续)
  • sb - service_name

    • service_name 是代理服务的唯一标识符。默认情况下，此名称将是代理服务名称。

    service_name 只能包含 URI 中允许的字符 (如 RFC2396 中所述，网址为：http://www.ietf.org/rfc/rfc2396.txt)，但它不能包含正斜杠 (/) 或冒号 (:) 字符。

  • sftp - sftp://hostname:port/directory
  • tuxedo - exportname

    • URI exportname 对应于一个 WTC 导出，远程 Tuxedo 域将其标识为 Tuxedo 服务。

    如果指定了多个 URI，则必须使各端点具有唯一的资源名。如果未指定远程名，则其值为资源名的值。如果未输入远程名，或者远程名与资源名相同，则只允许指定一个 URI。在这种情况下，资源名和远程名将具有相同的值。这将允许用户使用已定义的 WTC 导入来利用 WTC 负载平衡和故障转移。

    注：如果配置了两个相同的 URI，则会出现错误，表明该服务器名已存在。

  • ws - /contextPath

    • 对于使用 HTTP 或 WS 传输的代理服务，contextPath 必须唯一。

获取所有标头
选择是可检索传输中的所有标头。
选择否可检索定义的一组标头。如果选择否，请在标头字段中输入一组标头，然后单击添加。(此步骤不适用于本地传输。)
注： Oracle Service Bus 不会将请求中的 HTTP 授权标头传递到管道，因为这样会打开安全漏洞。您可能会在无意中创建将用户名和未加密的口令写入日志文件的日志操作。如果您的设计方案要求 HTTP 授权标头位于管道中，请执行下列操作：

a.在 Oracle Service Bus 的启动命令中，将以下系统属性设置为 true：com.bea.wli.sb.transports.http.GetHttpAuthorizationHeaderAllowed

b.在 Oracle Service Bus 管理控制台中，选择“传输配置”页上的获取所有标头或用户指定的标头，然后在该页上指定授权。
c.重新启动 Oracle Service Bus.

Oracle Service Bus 会将授权标头传递到管道。

完成之后

单击下一步继续在下一页上配置此服务；或单击最后一步在“创建/编辑代理服务 - 概要”页上查看并保存此配置。
