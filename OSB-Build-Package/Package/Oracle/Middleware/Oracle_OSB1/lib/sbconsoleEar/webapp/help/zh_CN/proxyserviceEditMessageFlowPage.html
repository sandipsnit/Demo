“编辑消息流”页

使用编辑消息流页可为代理服务构造消息流。

编辑消息流页上左侧的导航窗格显示消息流中的节点和对象的树视图。如果在另外的页上定义了对象的详细资料，则可以单击对象的名称以便显示关联的页。

右侧窗格提供了用于构造消息流的字段。如果尚未定义消息流，则该窗格中包括一个用于表明该服务的启动节点的代理服务图标。单击该图标可为服务添加管道对节点、路由节点、条件分支、操作分支和错误处理。

向该页添加对象后，该页上会显示图标以代表这些对象。对象之间的关系是通过线条和边界框来表示的。单击编辑消息流页上的图标可显示您能够对该对象执行的操作的菜单。菜单上的可用选项因上下文而异。有关图标和选项的完整列表，请参阅表 20-1。

图标
说明
菜单选项

代理服务
消息流的启动节点。
  • 添加管道对 - 请参阅向消息流添加管道对节点。
  • 添加路由 - 请参阅向消息流添加路由节点。
  • 添加条件分支 - 请参阅向消息流添加条件分支。
  • 添加操作分支 - 请参阅向消息流添加操作分支。
  • 添加服务错误处理程序 - 请参阅添加代理服务错误处理程序。

管道对节点
管道对节点包含一个请求管道和一个响应管道。
  • 编辑名称和批注
  • 添加管道对 - 请参阅向消息流添加管道对节点。
  • 添加条件分支 - 请参阅向消息流添加条件分支。
  • 添加操作分支 - 请参阅向消息流添加操作分支。
  • 粘贴路由 - 只有在已剪切或复制了路由节点且所剪切或复制的内容位于剪贴板中的情况下，此选项才可用。

响应管道
请参阅上面的管道对节点。
  • 添加阶段 - 请参阅向管道添加阶段。
  • 添加管道错误处理程序 - 请参阅添加管道错误处理程序。

请求管道
请参阅上面的管道对节点。
  • 添加阶段 - 请参阅向管道添加阶段。
  • 添加管道错误处理程序 - 请参阅添加管道错误处理程序。

具有错误处理程序的管道
带有为其定义的错误处理程序的管道。
  • 编辑管道错误处理程序 - 请参阅添加管道错误处理程序。
  • 删除管道错误处理程序

路由节点
路由节点操作定义了在消息经过路由节点时对消息的处理。
  • 编辑路由 - 请参阅向消息流添加路由节点。
  • 编辑名称和批注
  • 添加路由错误处理程序 - 请参阅添加路由节点错误处理程序。

带有错误处理程序的路由节点
带有为其定义的错误处理程序的路由节点。
  • 编辑路由错误处理程序 - 请参阅添加路由节点错误处理程序。
  • 删除路由错误处理程序
阶段节点
阶段节点是操作的容器。
  • 编辑阶段 - 请参阅向管道添加阶段。
  • 编辑名称和批注
  • 添加阶段 - 请参阅向管道添加阶段。
  • 添加阶段错误处理程序 - 请参阅添加阶段服务错误处理程序。
带有错误处理程序的阶段节点
带有为其定义的错误处理程序的阶段节点。
  • 编辑阶段错误处理程序 - 请参阅添加阶段错误处理程序。
  • 删除阶段错误处理程序
条件分支节点
条件分支节点使处理过程只能沿几个可能路径中的一个路径向下进行。
  • 编辑分支 - 请参阅向消息流添加条件分支。
  • 编辑名称和批注
操作分支节点
操作分支节点可根据指定的操作确定后面要执行的分支。
  • 编辑分支 - 请参阅向消息流添加操作分支。
  • 编辑名称和批注

分支节点
分支节点是由条件分支节点或操作分支节点定义的备用节点之一。
  • 添加管道对 - 请参阅向消息流添加管道对节点。
  • 添加路由 - 请参阅向消息流添加路由节点。
  • 添加条件分支 - 请参阅向消息流添加条件分支。
  • 添加操作分支 - 请参阅向消息流添加操作分支。
  • 粘贴路由 - 只有在已剪切或复制了路由节点且所剪切或复制的内容位于剪贴板中的情况下，此选项才可用。
错误处理程序
错误处理程序提供了重新发送消息流中的错误的逻辑。
  • 添加服务错误处理程序 - 请参阅添加代理服务错误处理程序。
