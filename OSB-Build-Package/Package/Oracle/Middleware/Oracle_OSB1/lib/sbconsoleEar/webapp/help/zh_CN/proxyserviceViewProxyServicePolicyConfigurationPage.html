“代理服务策略”页

查看代理服务 - 策略页是一系列用于编辑代理服务的页中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

使用此页可以为具有“WSDL Web 服务”、“消息传送服务”、“任何 SOAP 服务”或“任何 XML 服务”服务类型的代理服务配置策略设置。 表 19-20 介绍了如何使用此页。为了将 OWSM 策略用于“WSDL Web 服务”、“消息传送服务”或“任何 XML 服务”代理服务，协议必须设置为 HTTP。

对于基于 WSDL 的服务，绑定到该服务的 WLS 9.2 策略是在有效 WSDL 中公开 (内嵌) 的。抽象策略在内嵌之前会进行预处理。OWSM 策略是通过引用绑定的，而不是内嵌在有效 WSDL 中。

服务策略配置
以下是“服务策略配置”字段中的选项：
  • 来自 OWSM 策略存储 – 策略是在 Oracle Enterprise Manager 管理的中央 Oracle Web Services Manager 策略存储中定义的。这些策略支持 WS-Security 1.0/1.1、SAML 1.1/2.0、KerberosToken Profile 和其他行业标准。

    如有必要，请使用筛选功能找到想要的策略。对于代理服务，只显示服务策略。

    • 有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Securing Oracle Service Bus with Oracle Web Services Manager”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681。

  • 来自 WSDL – 如果服务策略与服务所基于的 WSDL 相关联，请选择此选项。这些策略支持 WS-Security 1.0、SAML 1.1 和其他行业标准。

    • 使用此选项可以查看 (只读) WSDL 中的请求和响应策略。

  • 来自预定义策略或 WS-Policy 资源 - 在选中此选项的情况下，可以从 Oracle Service Bus 管理控制台添加服务级策略、操作级策略 (在这种情况下，策略同时应用于请求消息和响应消息)、请求策略以及响应策略。

    策略是在 Oracle WebLogic Server 中预定义的，或者是在具有 WS-Policy 资源的 Oracle Service Bus 中由用户定义的。这些策略支持 WS-Security 1.0、SAML 1.1 和其他行业标准。

    • 使用此选项可以在服务、操作、请求和响应级别添加策略。

注： 策略绑定模型是相互排斥的。在一个服务中只能使用一种类型的策略。如果直接将策略绑定到服务，则会忽略所有基于 WSDL 的策略。
