“代理服务安全”页

查看代理服务 - 安全页是一系列用于编辑代理服务的页中的一个页面，如创建和配置代理服务和编辑代理服务配置中所述。

使用此页可为代理服务配置安全设置。

此页上可用的字段取决于代理服务中使用的策略。例如，如果代理服务使用 OWSM 策略 (推荐)，“策略覆盖”会显示在“安全”页上。有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Securing Oracle Service Bus with Oracle Web Services Manager”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681。

对于 WLS 9.2 策略，定制用户名/口令和定制标记的配置是类似的。在这两种情况下，应指定 XPath 表达式，使 Oracle Service Bus 能够找到必要的信息。这些 XPath 表达式的根如下：

  • 如果服务绑定为 AnySOAP 或 WSDL-SOAP，则使用 soap-env:Envelope/soap-env:Header。
  • 如果服务绑定并不基于 SOAP，则使用 soap-env:Body。

所有 XPath 表达式必须采用有效的 XPath 格式。XPath 表达式必须使用 XPath“declare namespace”语法来声明所用的任何名称空间，如下所示：

declare namespace
ns='http://webservices.example.com/MyExampleService';)

Web 服务安全配置
处理 WS-Security 标头
如果客户机请求包括 WS-Security 安全标头，请确定是否处理此标头：
  • 选择“是”：在使用主动中介的情况下，客户机将 WS-Security 应用于请求和/或响应消息。代理服务将处理此安全标头，并强制实施 WS-Security 策略。当代理策略基于包含安全策略的 WSDL 时，或者您在“服务策略配置”页上将策略附加到服务时，此选项将自动设置为“是”。
  • 选择“否”：在 WS-Security 通过场景中，客户机将 WS-Security 应用于请求和/或响应消息。代理服务不处理此安全标头。相反，它会将受保护的请求消息原封不动地传递到业务服务。
访问控制
传输访问控制
对于所有代理服务，可以创建传输级别策略，该策略可在客户机尝试建立与代理服务的连接时应用安全检查。仅允许传输级别策略中列出的用户发出的请求继续下去。
消息访问控制
消息级访问控制策略在客户机尝试调用具有消息级安全的代理服务时应用安全检查。可以在下列情况下创建消息级访问控制策略：
  • 对于作为主动 Web 服务安全中间者的代理服务；就是说，您处理 WS-Security 标头。
  • 对于您在此页上为其设置定制验证的代理服务。
仅允许消息级别策略中列出的用户调用该操作。
定制验证
 (消息级)
验证类型
选择以下选项之一：
  • 如果代理服务不使用定制验证，请选择无。
  • 定制用户名和口令

    • 如果选择此选项，则还必须在用户名 XPath 字段和用户口令 XPath 字段中输入值。

  • 定制标记

    • 如果选择此选项，则还必须选择标记类型，并在标记 XPath 字段中输入值。

用户名 XPath
仅当定制验证设置选项设置为定制用户名和口令时，此选项才可用。在可用时，此选项是必需的。
输入用户名作为 XPath 表达式。
根据需要，将以消息标头或有效负载为依据计算 XPath 表达式，从而使 Oracle Service Bus 可以获取用于定制验证的用户名。
用户口令 XPath
仅当定制验证设置选项设置为定制用户名和口令时，此选项才可用。在可用时，此选项是必需的。
输入口令作为 XPath 表达式。
根据需要，将以消息标头或有效负载为依据计算 XPath 表达式，从而使 Oracle Service Bus 可以获取用于定制验证的口令值。
标记类型
仅当定制验证设置选项设置为定制标记时，此选项才可用。在可用时，此选项是必需的。
从下拉列表中选择标记类型。只有为 WebLogic 服务器身份断言提供程序配置的活动标记类型才可用。请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Configuring Identity Assertion Providers for Custom Tokens”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660。
标记 XPath
仅当定制验证设置选项设置为定制标记时，此选项才可用。在可用时，此选项是必需的。
输入 XPath 表达式可指定定制标记的路径。根据需要，Oracle Service Bus 依据消息标头或有效负载对标记 XPath 表达式求值，以获取用于定制验证的标记。
上下文属性
可选择指定一个或多个上下文属性，以便将其他上下文信息传递给验证 (定制用户名和口令) 或标识断言 (定制标记) 安全提供程序。
上下文属性提供了一种将附加信息传递到 WebLogic 安全框架的方式 (ContextHandler 接口)，这样使安全提供程序能够获取上下文信息。请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Context Properties Are Passed to Security Providers”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500。
输入属性名作为文字字符串，输入值选择器作为有效的 XPath 表达式。(XPath 表达式也可以是文字字符串。)
XPath 表达式是根据用于定制标记或定制用户名/口令的相同消息部分来计算的。就是说，根据非基于 SOAP 的代理服务的标头和有效负载，对基于 SOAP 的代理服务的值选择器 XPath 表达式进行求值。
在运行时对 XPath 表达式进行求值以生成属性值。ContextHandler 实际上是名称/值列表，因此，需要安全提供程序了解要查找的名称。从而，仅当安全提供程序要求为这些用户定义的属性之一提供值时，才对 XPath 表达式进行求值。
单击添加属性以添加此上下文属性。可以添加多个上下文属性。
