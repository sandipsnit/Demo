了解 XQuery 编辑器的布局和任务

XQuery 表达式编辑器、XQuery 条件编辑器和 XPath 表达式编辑器均由以下组件构成：

  • 选项板
  • 工作区
  • 属性检查器

选项板

每个编辑器的左面板都包含以下选项板。在任何编辑器中，单击选项板的名称都可以显示该选项板。在编辑器中，每个选项板都包含可插入表达式中的实体。

  • 名称空间定义选项板列出了默认的 Oracle Service Bus 名称空间、变量名称空间和用户定义的名称空间。可以定义新的名称空间，然后可将其添加至用户定义的名称空间列表。

    • 有关详细信息，请参阅：

    • 在编辑器工作区的文本字段中构建表达式
    • 创建要在内嵌表达式中使用的名称空间
  • XQuery 函数选项板列出了一组标准 XQuery 函数。如果将函数插入表达式，则会对您必须提供的参数值使用占位符。

    • 有关详细信息，请参阅：

    • 在编辑器工作区的文本字段中构建表达式
    • 在 XQuery 编辑器中创建定制 XPath 函数
    • 将外部 XQuery 资源绑定到内嵌 XQuery
  • 变量结构选项板提供了一组工具，用于通过 XPath 表达式插入变量及这些变量的路径。

    • 变量结构是该编辑器中显示的变量或变量路径的图形表示形式。这些图形表示形式使得变量结构一目了然。使用它们，可以构造引用变量内容的内嵌 XQuery 表达式。

    注： 变量结构不创建变量。变量是在运行时作为阶段中“分配”操作的目标创建的。

    Oracle Service Bus 提供了多个预定义的消息上下文变量 (attachments、body、header、outbound 和 inbound)，这些变量的内容能够以变量结构的形式显示。也可以定义自己的变量结构。

    有关详细信息，请参阅：

    • 在编辑器工作区的文本字段中构建表达式
    • 在 XQuery 编辑器中创建变量结构

工作区

该页的右侧提供了用于构造 XQuery 表达式、XQuery 条件或 XPath 的工作区。在这三个编辑器中，工作区有所不同。

属性检查器

在这三个编辑器中，属性检查器显示在该页的右下方。从其中一个选项板中选择要添加至表达式的项目后，该项目将显示在“属性检查器”中。然后，可将该项目粘贴到工作区中。请参阅在编辑器工作区的文本字段中构建表达式。
