创建和编辑内嵌 XQuery 和 XPath 表达式

在“编辑阶段配置”页上向阶段或路由节点添加操作时，该页上将会显示一个主干结构，提示输入配置详细资料。图 22-1 显示了一个示例。

图 22-1 操作配置主干示例

只要适用于上下文，该骨架结构就会提供用于访问 XQuery 编辑器的链接。您可以使用这些编辑器，按照操作中上下文的要求，构造将以内嵌方式执行的表达式。

开始之前

这些说明假定您将在管道阶段、错误处理程序阶段或路由节点的“编辑阶段配置”页中创建或编辑操作。请参阅创建、编辑和查看消息流和向消息流添加操作和编辑消息流中的操作。

要创建或修改内嵌表达式，请执行以下操作：

  1. 在编辑阶段配置页上，找到操作中要添加或编辑表达式的位置。
  2. 单击表达式链接，以打开适用于该上下文的编辑器。如果尚未定义表达式，该链接会表明您可在该位置使用哪种表达式：
    • 单击 <Expression> 可创建 XQuery 表达式。此外，还可以导入在 Oracle Service Bus 之外创建的 XQuery 或 XSLT 资源，然后将其绑定到内嵌 XQuery。
    • 单击 <Condition> 可创建 if...then 操作的 XQuery 条件表达式。
    • 单击 <XPath> 可创建消息上下文变量的 XPath 表达式。

      • 如果已在某个位置定义了表达式，则 <Expression>、<Condition> 或 <XPath> 链接将由表示表达式本身的链接 (例如 true()[icon]) 或由于太长而无法全部显示的表达式的片段 (例如 $body/urn:POSta...[icon]) 替代。单击该表达式 (或表达式片段) 可在相应编辑器中将其打开。

  3. 构建表达式，如以下主题所述：
    • 在编辑器工作区的文本字段中构建表达式
    • 创建要在内嵌表达式中使用的名称空间
    • 在 XQuery 编辑器中创建变量结构
    • 在 XQuery 编辑器中创建定制 XPath 函数
    • 将外部 XQuery 资源绑定到内嵌 XQuery
    • 将外部 XSLT 资源绑定到内嵌 XQuery
    • 将动态 XQuery 表达式绑定到内嵌 XQuery
    • 使用构建器选项输入 XQuery 比较表达式
    • 使用构建器选项输入一元表达式
  4. 还可以执行下面的任一或全部操作：
    • 单击验证对表达式进行验证。
    • 单击测试可测试表达式。请参阅执行 XQuery 测试。
  5. 单击保存关闭编辑器并将表达式插入操作中。

完成之后

继续配置该操作，如向消息流添加操作和编辑消息流中的操作中所述。
