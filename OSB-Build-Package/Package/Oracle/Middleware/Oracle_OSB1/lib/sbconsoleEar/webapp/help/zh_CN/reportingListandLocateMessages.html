查看报告消息

使用消息报告概要页可查看表 27-2 中所示的报告消息信息。此页最多显示 100 条消息。

  1. 要限制该列表中项的数量或查找特定消息，请执行以下操作：
    • 单击打开图标可显示其他搜索筛选器。如果指定多个搜索条件，则仅返回与所有条件均匹配的结果。
    • 要设置筛选器参数，请执行下面的一个或多个步骤，如 表 27-1 中所示。

      • 表 27-1 消息筛选器参数 
      操作
      步骤
      按时间参数筛选
      设置时段：
      1. 选择消息来自... 到... 单选按钮。
      2. 在消息来自字段中，以 MM/DD/YY HH:MM:SS AM|PM 格式输入日期和时间。例如，可以输入 10/10/07 12:45:00 AM。
      3. 在到字段中，以 MM/DD/YY HH:MM:SS AM|PM 格式输入日期和时间。
      另外，也可以针对最近消息选择参数：
      1. 选择最后一次单选按钮。
      2. 在最后一次字段中，输入天数，然后指定小时和分钟数。
      注： 默认情况下，将会显示最后 30 分钟的消息报告。
      按服务名筛选
      在入站服务名字段中，输入服务名。
      按错误代码筛选
      在错误代码字段中，输入错误代码。
      按报告索引筛选
      在报告索引字段中，输入报告索引。这些是您为该代理服务配置报告操作时定义的键值对。这是字符串搜索。

    • 可在服务名、错误代码和报告索引字段搜索中使用通配符。
      • 仅输入星号 (*) 将返回所有消息。例如，如果在错误代码字段中输入 *，则会显示包含错误代码的消息和不包含错误代码的消息。
      • 严格的搜索限制仅适用于“错误代码”字段。例如，要查找包含错误代码 BEA12345 的消息，可以输入 *、BE* 或 BEA12345。
  2. 单击搜索。此页中显示了匹配您设置的筛选参数的消息。在筛选消息时可显示的最大消息数为 1000。
  3. 单击查看全部可删除搜索筛选器，并显示所有消息。
  4. 要查找一条消息，也可以执行以下操作：
    • 对消息列表进行重新排序。单击带下划线的列名。升序箭头和降序箭头指示排序顺序。单击列名可更改排序顺序。
    • 滚动查看各页。使用表上面的页控件，或表下面的页控件。通过选择页码转到某一页，或通过使用箭头按钮转到下一页、上一页、第一页或最后一页。

      • 表 27-2 消息 (报告) 概要详细资料
      属性
      说明
      报告索引
      指向查看消息详细资料页的链接。请参阅查看消息详细资料。
      报告索引是用于从消息上下文变量中提取键标识符的键值对。键是筛选消息的便利方法。可以使用报告操作添加键值对。请参阅向消息流添加操作和编辑消息流中的操作。
      数据库时间戳
      消息添加到数据库中的日期和时间。
      入站服务
      与消息关联的入站服务。此服务是指向查看代理服务 - 配置详细资料页的链接。请参阅编辑代理服务配置。
      错误代码
      与此消息关联的错误代码 (如果有)。
      配置引发错误操作时可以指定错误代码。请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Handling Errors in Message Flows”(网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG232) 和添加代理服务错误处理程序。
