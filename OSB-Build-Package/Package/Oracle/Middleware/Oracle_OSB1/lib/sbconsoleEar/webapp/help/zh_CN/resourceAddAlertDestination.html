添加预警目标

预警目标是 Oracle Service Bus 中预警通知的目标地址。可以在创建预警目标页中创建预警目标资源。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择项目浏览器，然后选择要在其中添加预警目标的项目或文件夹。此时会显示项目视图/文件夹视图页。
  3. 从创建资源下拉列表中选择预警目标以便显示创建预警目标页。
  4. 在资源名字段中，为此预警目标输入唯一名称。这是必填字段。

    如需命名指导，请参阅资源命名限制。
  5. 在资源说明字段中，输入此预警目标的说明。
  6. 选择以下目标中的一个或全部，使其包含在此预警目标资源中。
    7. 注： 预警在运行时聚集在 Oracle Service Bus 管理控制台中，您可以在该管理控制台的操作 > 面板下查看它们。
    • SNMP 陷阱：如果指定“SNMP 陷阱”，则预警会以 SNMP 陷阱形式发送，并可由任何第三方企业监视系统 (ESM) 处理。
    • 报告：如果指定报告，会将预警发送到 Oracle Service Bus 报告模块，并使用定制的报告提供程序 (该提供程序可通过 Oracle Service Bus 使用报告 API 提供程序开发) 捕获此预警。它允许第三方在定制的 Java 代码中接收和处理预警。
    • 预警日志记录：如果选择是，则发送到此预警目标的预警将记录到预警日志中。
  7. 向预警目标资源定义中添加一个电子邮件收件人。请参阅添加电子邮件收件人。
  8. 向预警目标资源定义中添加 JMS 目标。请参阅添加 JMS 目标。
  9. 单击保存。新的预警目标会保存在当前会话中。
  10. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
