添加拆分-联接

开始之前

拆分-联接需要一个 .flow 文件，此文件的创建时间是您在使用适用于 Eclipse 的 Oracle Service Bus 插件创建拆分-联接时或在将拆分-联接配置 JAR 导入此插件时。此 .flow 文件可能具有对代理服务、业务服务或外部 WSDL 资源的引用。在激活拆分-联接之前，服务器上必须存在拆分-联接所依赖的资源。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择项目浏览器，然后选择要在其中添加拆分-联接的项目或文件夹。此时会显示项目视图/文件夹视图页。
  3. 从创建资源下拉列表中选择拆分-联接，以便显示创建新的拆分-联接资源页。

    4. 指定表 9-2 中显示的参数。

    表 9-2 拆分-联接参数
    属性
    创建或编辑...
    资源名
    输入此拆分-联接的唯一名称。

    如需命名指导，请参阅资源命名限制。
    资源说明
    输入拆分-联接的说明。
    拆分-联接
    单击浏览以在系统上查找 .flow 文件，它通常位于 MW_HOME\user_projects\workspaces\your_workspace 下面。
    分派策略
    选择用于创建线程 (并行分支) 的 WebLogic 服务器工作管理器的实例，以便执行拆分-联接。如果不存在其他工作管理器，则使用默认工作管理器。

  4. 单击保存。将创建拆分-联接并将其保存在当前会话中。
    5. 注： 在激活拆分-联接之前，拆分-联接所依赖的资源必须存在。
  5. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
