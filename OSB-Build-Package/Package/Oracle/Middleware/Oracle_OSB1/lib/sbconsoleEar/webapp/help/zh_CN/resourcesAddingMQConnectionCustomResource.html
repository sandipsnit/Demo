添加 MQ 连接

开始之前

有关 Oracle Service Bus 上受支持的 WebSphere MQ 版本，请参阅 Oracle Fusion Middleware Supported System Configurations，网址为：http://www.oracle.com/technology/software/products/ias/files/fusion_certification.html。WebSphere MQ 客户机库未与 Oracle Service Bus 安装程序绑定在一起。请从 WebSphere MQ 安装中获取 WebSphere MQ 客户机库。

要在 Oracle Service Bus 中配置和使用 MQ 连接资源和 MQ 传输，必须将 MQ 客户机库添加到类路径中。一种选择是将 com.ibm.mq.jar 库复制到 DOMAIN_NAME/lib 目录，其中 DOMAIN_NAME 是您存放域的目录的名称，通常为 MW_HOME\user_projects\domains\DOMAIN_NAME。

要添加 MQ 连接资源，请执行以下操作：

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择项目浏览器，然后选择要在其中添加 MQ 连接资源的项目或文件夹。此时会显示项目视图/文件夹视图页。
  3. 从创建资源下拉列表中选择实用程序下的 MQ 连接，以便显示创建新的 MQ 连接资源页。

    4. 指定表 8-2 中所示的连接参数。

    表 8-2 MQ 连接参数
    属性
    创建或编辑...
    资源名
    输入此 MQ 连接资源的唯一名称。

    如需命名指导，请参阅资源命名限制。
    资源说明
    为 MQ 连接资源输入说明。
    连接类型
    为连接到 MQ 队列管理器选择以下模式之一：
    • tcp 模式 - 使用 TCP/IP 可连接到不与 Oracle Service Bus 驻留在同一计算机上的队列管理器。
    • 绑定模式 - 使用绑定模式可连接到与 Oracle Service Bus 位于同一台计算机上的队列管理器。
    MQ 主机名称
    只用于 tcp 模式连接：
    输入 MQ 队列管理器的主机名称。
    MQ 端口号
    只用于 tcp 模式连接：
    输入 MQ 队列管理器监听程序的端口号。
    MQ 队列管理器名称
    输入要连接的 MQ 队列管理器的名称。
    队列管理器 CCSID
    只用于 tcp 模式连接：
    建立连接时使用的编码字符集标识符 (CCSID)。CCSID 主要用于国际化支持。
    要了解详细信息，请参阅“WebSphere MQ Fundamentals”，网址为：http://www.redbooks.ibm.com/redbooks/SG247128/wwhelp/wwhimpl/java/html/wwhelp.htm。
    MQ 队列管理器通道名称
    只用于 tcp 模式连接：
    输入队列管理器服务器连接通道名称。
    需要使用 SSL
    用于 tcp 模式连接：
    选中此复选框可使用 SSL 发送消息。如果未选中需要使用双向 SSL 选项，则将仅启用服务器端 SSL。
    密码套件
    只有在选中需要使用 SSL 复选框后，该选项才可用。
    选择要由 SSL 使用的密码套件算法。
    密码套件算法用来加密和解密 WebSphere MQ 服务器和 WebSphere MQ 客户机之间的消息通信。因此，在使用 SSL 与 WebSphere MQ 服务器通信时，必须指定密码套件算法。
    需要使用双向 SSL
    只有在选中需要使用 SSL 复选框后，该选项才可用。
    选中此复选框可同时启用客户端 SSL 验证和服务器端 SSL 验证。
    对服务密钥提供程序的引用
    如果选择需要使用双向 SSL，则必须提供对服务密钥提供程序的引用，以便获得适当的密钥库和信任存储信息。
    输入服务密钥提供程序的路径 (项目/文件夹) 和名称，或者单击浏览从选择服务密钥提供程序页中选择一个服务密钥提供程序。
    对静态服务帐户的引用
    只用于 tcp 模式连接：
    要求用户名和口令验证。输入静态服务帐户的路径 (项目/文件夹) 和名称，或者单击浏览从浏览器中选择一个服务帐户。
    WebSphere MQ 版本
    选择 WebSphere MQ 版本：
    • 5.3
    • 6.0
    • 7.0

    注：IBM 不再支持版本 5.3 和 6.0。这些版本在 Oracle Service Bus 中已过时。
    MQ 连接池大小
    输入 MQ 连接池的大小。
    MQ 连接超时
    输入时间间隔 (秒)，在此间隔之后，将销毁未使用的连接。默认值为 1800 秒。
    MQ 连接最长等待时间
    输入等待连接可用的最长等待时间 (毫秒)。如果在该时间间隔内未建立连接，Oracle Service Bus 将引发异常错误。默认值为 3000 毫秒。

  4. 单击保存。将创建 MQ 连接资源并将其存储在当前会话中。
  5. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
