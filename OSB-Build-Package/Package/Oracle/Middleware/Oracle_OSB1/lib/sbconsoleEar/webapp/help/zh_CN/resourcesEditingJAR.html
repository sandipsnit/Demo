编辑 JAR

使用查看 JAR 资源 - 配置页可更新 JAR 文件。从本质上讲，更新 JAR 文件就是将资源指向一个新版本的 JAR，因为您无法使用 Oracle Service Bus 编辑 JAR。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或者单击编辑进入现有会话。请参阅使用更改中心。
  2. 查找 JAR，如查找 JAR 中所述。
  3. 单击 JAR 名称。查看 JAR 资源 - JAR 配置页显示了表 7-2 中所示的信息。

    4. 表 7-2 JAR 信息 
    属性
    说明
    上次修改者
    将此 JAR 导入到配置中的用户。
    上次修改时间
    用户将此 JAR 导入到配置中的日期和时间。单击日期和时间链接可查看此资源的更改历史记录。请参阅“查看更改历史记录”页。
    引用
    此 JAR 引用的 JAR 数。如果存在此类引用，可单击数字链接以查看 JAR 列表。请参阅查看对资源的引用。
    引用项
    引用此 JAR 的对象数。如果存在此类引用，可单击数字链接以查看对象列表。请参阅查看对资源的引用。
    说明
    此 JAR 的说明 (如果存在)。

    默认情况下，会显示 JAR 配置，其中包括表 7-3 中所示的信息。

    表 7-3 JAR 配置详细资料
    属性
    说明
    JAR 配置
    此 JAR 中的类和接口列表。
    在 Java 类名旁边单击 (+) 以便显示此类中包含的方法的列表。

  4. 要对这些字段进行更改，请单击更新。

    5. 只能编辑说明字段。您无法在 Oracle Service Bus 中编辑某个 JAR。要更新 JAR，必须选择一个要使用的新 JAR。要了解如何执行此操作，请参阅添加 JAR。

  5. 单击保存可提交当前会话中的更新。
  6. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
注： 如果更改 EJB 业务服务使用的 JAR (将 JAR 资源指向新版本的 JAR)，则需要重新部署 EJB。您将需要编辑所有使用此 JAR 的 EJB 服务，然后重新选择、保存和激活此 JAR 资源。这将重新打包 EJB 业务服务以使用新的 JAR。
注： Java 标注操作和基于 Tuxedo 的服务将自动选取新的 JAR。
