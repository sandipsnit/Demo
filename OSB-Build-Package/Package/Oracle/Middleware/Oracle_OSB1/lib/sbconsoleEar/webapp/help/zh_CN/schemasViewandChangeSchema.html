编辑 XML 方案

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 查找 XML 方案，如查找 XML 方案中所述。
  3. 单击 XML 方案名称。查看 XML 方案详细资料页显示了表 4-2、表 4-3 和表 4-4 中所示的信息。

    4. 表 4-2 XML 方案详细资料 
    属性
    说明
    上次修改者
    创建此 XML 方案或将其导入到配置中的用户。
    上次修改时间
    用户创建此 XML 方案或将其导入到配置中的日期和时间。单击日期和时间链接可查看此资源的更改历史记录。请参阅“查看更改历史记录”页。
    引用
    此 XML 方案引用的对象数。如果存在此类引用，可单击数字链接以查看对象列表。请参阅查看对资源的引用。
    引用项
    引用此 XML 方案的对象数。如果存在此类引用，可单击数字链接以查看对象列表。例如，如果此 XML 方案由特定 WSDL 引用，则在单击该链接后，此 WSDL 将作为引用列出。请参阅查看对资源的引用。
    说明
    此 XML 方案的说明 (如果存在)。

    默认情况下，将显示 XML 方案的结构视图，其中包括表 4-3 中所示的信息。

    表 4-3 XML 结构详细资料
    属性
    说明
    目标名称空间
    用来限定 XML 方案中所包括的任何定义的名称空间。
    XML 方案定义
    与此 XML 方案关联的属性和组：
    • XML 方案类型
    • XML 方案元素
    • XML 方案导入
    • XML 方案添加/重新定义
    方案状态
    XML 方案的状态：
    • 有效
      如果当前 XML 方案添加的所有 XML 方案的位置都已指定，则 XML 方案将显示为有效。而且，所有嵌套 XML 方案 (由添加的 XML 方案接着添加的 XML 方案) 的位置都已指定。
    • 无效
      如果 XML 方案的一个或多个位置没有指定，则 XML 方案将显示为无效。此外，可能有一个或多个嵌套 XML 方案 (由添加的 XML 方案接着添加的 XML 方案) 的位置没有指定。要解析无效的 XML 方案，请单击编辑引用。请参阅解析未解析的 XML 方案引用。

  4. 单击文本视图将显示 XML 方案详细资料的文本视图。文本视图包括表 4-4 中所示的信息。

    5. 表 4-4 XML 方案文本详细资料
    属性
    说明
    目标名称空间
    用来限定 XML 方案中所包括的任何定义的名称空间。
    XML 方案
    此 XML 方案的文本。

  5. 要对这些字段进行更改，请单击编辑。有关这些字段的说明，请参阅添加 XML 方案。
  6. 进行适当的编辑。
  7. 单击保存可提交当前会话中的更新。
  8. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
