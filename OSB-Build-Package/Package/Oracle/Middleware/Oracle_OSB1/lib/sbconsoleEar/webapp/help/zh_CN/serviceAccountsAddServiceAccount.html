添加服务帐户

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择项目浏览器，然后选择要在其中添加服务帐户的项目或文件夹。此时会显示项目视图/文件夹视图页。
  3. 从创建资源下拉列表中选择服务帐户，以便显示创建新服务帐户页。
  4. 在资源名字段中，输入此服务帐户的唯一名称。

    如需命名指导，请参阅资源命名限制。
  5. 在资源说明字段中，输入此服务帐户的说明。
  6. 在资源类型下，执行以下操作之一：
    • 要创建可提供从传入客户机请求接收的用户名和口令的服务帐户，请选择通过。
    • 要创建可提供随服务帐户配置一起保存的用户名和口令的服务帐户，请选择静态。
    • 要创建可将用户名从一个或多个经过验证的客户机映射到指定的用户名和口令的服务帐户，请选择映射。
  7. 根据在步骤 6 中选择的资源类型，执行下列步骤之一，如表 16-2 中所述。

    8. 表 16-2 资源类型选项 
    选定的资源类型
    步骤
    通过
    单击上一步。
    静态
    1. 单击下一步。
    2. 在用户名、口令和确认口令字段中输入用户名和口令。
    3. 单击上一步。
    映射
    要创建服务帐户以将用户名从一个或多个客户机映射到指定的用户名和口令，请执行以下操作：
    1. 单击下一步。
    2. 在“输入授权远程用户”表中，执行以下操作：
      1. 在远程用户名、口令和确认口令字段中，输入要在出站请求中发送的用户名和口令。
      2. 单击添加。

        3. 用户映射将添加到“远程用户”表中。

      3. (可选) 在“输入授权远程用户”表中添加其他远程用户。
    3. 单击下一步。
    4. 要将授权客户机映射到远程用户名和口令，请在“输入授权本地用户”表中执行以下操作：
      1. 在本地用户名字段中，输入用于标识已在入站请求中经过验证的客户机的名称。

        2. 如果尚未将此用户添加到 Oracle Service Bus 管理控制台的“安全配置”模块中，请先添加此用户，然后在运行时环境中使用此映射。请参阅添加用户。使用 Oracle Service Bus 可以为不存在的本地用户创建映射，但是该映射将永远无法与已通过验证的用户匹配，因而将永远无法使用。

      2. 在远程用户名列表中，为在“本地用户名”字段中指定的已通过验证的用户选择要在出站请求中发送的用户名。
      3. 单击添加。
    5. 要将匿名客户机映射到远程用户名，请执行以下操作：
      1. 选中映射匿名请求复选框。
      2. 在“选择远程用户”列表中，为所有匿名用户选择在出站请求中发送的用户名。
    6. 单击上一步。

  8. 单击保存。将创建服务帐户并将其保存在当前会话中。
  9. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
