编辑服务帐户

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 查找服务帐户，如查找服务帐户中所述。
  3. 单击服务帐户名。查看服务帐户详细资料页显示表 16-3 中所示的信息。

    4. 表 16-3 服务帐户详细资料
    属性
    说明
    上次修改者
    创建该服务帐户或将其导入配置的用户。
    上次修改时间
    用户创建此服务帐户或将其导入到配置中的日期和时间。单击日期和时间链接可查看此资源的更改历史记录。请参阅“查看更改历史记录”页。
    引用
    此服务帐户引用的对象数。如果存在此类引用，可单击数字链接以查看对象列表。请参阅查看对资源的引用。
    引用项
    引用此服务帐户的对象数。如果存在此类引用，可单击数字链接以查看对象列表。例如，如果选定此服务帐户作为使用 JMS 传输协议的代理服务中的 JMS 服务帐户，则在单击该链接后，此代理服务将作为引用列出。请参阅查看对资源的引用。
    说明
    此服务帐户的说明 (如果存在)。

  4. 要对这些字段进行更改，请单击编辑。有关这些字段的说明，请参阅添加服务帐户。

    5. 您不能更改资源名字段。

  5. 单击保存可提交当前会话中的更新。
  6. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
注： 如果修改的服务帐户用于使用 WebLogic JMS 服务器来进行的验证，则 JMS 服务器可能在最多 60 秒的时间内无法识别修改。默认情况下，WebLogic Server JMS 每隔 60 秒检查一次每个目标的权限。要更改此行为，请修改 WebLogic 服务器启动命令，以便它将以下系统属性设置为您希望 WebLogic 服务器 JMS 检查权限的频率 (秒)：weblogic.jms.securityCheckInterval。
此属性的值为 0 (零) 可确保为对 JMS 资源执行的每个 send、receive 和 getEnumeration 操作执行权限检查。
