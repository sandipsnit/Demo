指定服务帐户

可将同一服务帐户用于多个业务服务和代理服务。要指定服务帐户提供的用户名和口令，可使用下列方法之一：

  • 静态

    • 要求用户名和口令随服务帐户配置一起保存。服务帐户会在出站请求中对此用户名和口令进行编码。

  • 用户名和口令传递

    • 使服务帐户可提供它从传入客户机请求接收的用户名和口令。例如，如果入站 HTTP 基本请求包含作为用户名和口令的“pat”和“patspassword”，则服务帐户会在出站请求中对“pat”和“patspassword”进行编码。

    因为此方法要求客户机请求包含明文用户名和口令，所以该方法仅适用于使用 HTTP 基本协议的客户机请求、使用带有明文口令的 Web 服务安全用户名标记验证的客户机请求，或者使用定制用户名和口令标记的客户机请求。

    Oracle 建议仅在 Oracle Service Bus 和端点属于同一验证域时使用此技术。例如，如果要在单个组织中路由消息，并且 Oracle Service Bus 和消息使用者都针对公用 LDAP 服务器进行验证，则可使用此技术。

    下列限制适用于此方法：

    • 该技术不能在验证 Oracle Service Bus 对本地或远程服务器或系统资源（如 FTP 服务器或 JMS 服务器）的访问权限的出站请求中使用。
    • 它不能与 fn-bea:lookupBasicCredentials XQuery 函数一起使用。有关详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“XQuery Implementation”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG444。
      • 注： 如果代理为活动的 WSS 中间者，您可以使用 WS-Security 来加密 WS-Security 用户名标记或定制用户名/口令。在这种情况下，因为代理将首先对请求进行解密，然后才具有对明文用户名/口令的访问权，所以用户名/口令传递有效。
  • 用户映射

    • 要求您将通过对客户机的入站请求进行验证获得的用户名（本地用户名）关联（映射）到指定的用户名和口令（远程用户名和口令）。服务帐户接收到已映射的经过验证的客户机的请求时，它会为业务服务或代理服务出站请求提供正确的远程用户名和口令。

    如果客户机在传输级和消息级上都进行验证，则服务帐户会将消息级用户名映射到远程用户名和口令。

    还可以将匿名用户名映射到远程用户名和口令。

    下列限制适用于此方法：

    • 该技术不能在验证 Oracle Service Bus 对本地或远程服务器或系统资源（如 FTP 服务器或 JMS 服务器）的访问权限的出站请求中使用。
    • 它不能与 fn-bea:lookupBasicCredentials XQuery 函数一起使用。有关详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“XQuery Implementation”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBAG444。
