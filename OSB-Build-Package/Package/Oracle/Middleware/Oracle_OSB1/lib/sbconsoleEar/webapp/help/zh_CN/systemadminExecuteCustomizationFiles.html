执行定制设置文件

使用此页可执行以前保存在系统中的定制设置 XML 文件。定制设置文件为您在部署期间修改环境 (从开发环境更改到临时环境、从临时环境更改到生产环境) 或在设计时修改环境提供了一种便捷的方式。

只有处于某个会话之中才能执行定制设置文件。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择系统管理 > 执行定制设置文件。
  3. 选择要执行的定制设置文件。
    1. 在文件名字段中，单击浏览以查找存储此定制设置文件的目录。
    2. 选择此 XML 文件，然后单击打开。
  4. 单击下一步。将显示以下定制设置详细资料，如表 31-2 中所示。

    5. 表 31-2 定制设置详细资料
    属性
    说明
    操作
    • 查找和替换 - 通过字符串替换来修改环境值。
    • 映射引用 - 将在资源中找到的现有引用映射到其他引用。
    • 设置环境值 - 修改资源的环境值。
    适用于
    要定制的服务、项目、文件夹或资源的名称。
    此名称是一个指向服务或资源的配置详细资料页的链接，或者是一个指向项目浏览器中项目或文件夹的链接。
    说明
    每个定制设置操作的说明。

  5. 要仅定制在当前会话中更改的资源、服务、项目或文件夹，请选中仅在当前会话中更改的项复选框。

    6. 如果将定制设置限制为仅针对在当前会话中修改的资源，则应用到列会随当前会话中修改的资源一起更新。如果限制项目或文件夹的定制设置，则应用到列中会显示该项目或文件夹内部包含的在当前会话中修改的资源。

  6. 单击执行可提交当前会话中的更新。

    7. 要查看定制设置详细资料，请在会话中选择更改中心下的查看更改，然后单击定制设置任务。请参阅查看配置更改。

  7. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
