查找和替换环境值

使用查找和替换页可以搜索因域而异的环境值。环境值是配置数据中的一些预定义字段，当您的配置从一个域移动到另一个域时 (例如从测试到生产)，这些字段的值很可能会发生变化。环境值代表实体，如 URL、URI、文件和目录名、服务器名、电子邮件等。另外，也可以在预警目标、代理服务、业务服务、SMTP 服务器和 JNDI 提供程序资源以及 UDDI 注册表项中找到环境值。

此页的行为因您是否位于会话中而异。如果位于会话中，则可以查找和替换环境值。但是，如果位于会话外，则只能查找环境值；全部替换被禁用。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择系统管理 > 查找和替换。
  3. 在查找值字段中，输入要查找的环境值。
  4. 要显示配置中包含您所输入值的环境值列表，请单击查找。

    5. 此时会显示一个相关值列表，其中包括所有者、类型和环境值列。

  5. 要仅查找在当前会话中更改过的项目，请选中仅在当前会话中更改的项复选框。
  6. 从按变量类型查找下拉列表中选择要针对其进行搜索的变量类型，如表 31-1 中所述。

  7. 要查找位于特定项目中的环境值，请从在项目中定位下拉列表中选择该项目名。
  8. 在替换为字段中，输入新的环境值。
  9. 要使用新值替换原始的环境值，请单击全部替换。

    10. 在查找值字段中输入的环境值的所有匹配项都将替换为当前会话中在替换为字段内输入的环境值。

    注： 某些环境值是复杂的 XML 对象，无法使用 Oracle Service Bus 管理控制台查找和替换这些对象。但是，仍可以使用 ALSBConfigurationMBean 在脚本中直接设置这些环境值。有关 ALSBConfigurationMBean 的详细信息，请参阅 Oracle Service Bus API 引用。除了通过 API 设置环境值之外，还可以使用定制设置文件设置复杂类型环境值。请参阅执行定制设置文件。
  10. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
