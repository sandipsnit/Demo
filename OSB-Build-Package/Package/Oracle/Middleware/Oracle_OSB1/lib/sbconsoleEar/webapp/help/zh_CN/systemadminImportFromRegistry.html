从 UDDI 注册表导入业务服务

可以将以下业务服务类型从 UDDI 注册表导入到 Oracle Service Bus 中：

  • 通过 HTTP 传输的 WSDL 服务。
  • 发布到 UDDI 注册表的 Oracle Service Bus 代理服务。此功能主要用于多域 Oracle Service Bus 部署，在这些部署中，一个域中的代理服务需要发现并路由到其他域的代理服务。

如果仅配置了一个注册表，则 Oracle Service Bus 会自动连接到该注册表，然后检索所有业务实体并填充搜索表单。如果配置了多个注册表，则导入向导会提示您首先选择要从其中导入服务的注册表。

在导入时，如果注册表有几个不同类型的服务，则可使用查询 URL 查找特定服务。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择系统管理 > 从 UDDI 导入。
  3. 在导入注册表名称下拉列表中，选择您要从其中导入服务的注册表的名称，然后单击下一步。

    4. 如果 Oracle Service Bus 中仅配置了一个注册表，则不会显示初始注册表选择页，因为这是默认注册表。只有在定义了两个或多个注册表的情况下，才会显示注册表选择页。

    使用从 UDDI 导入页可以搜索并导入特定的业务服务。可以按业务实体、按模式或使用这两者的组合来搜索服务。

  4. 在实体名称下拉列表中选择业务实体名称 (例如，文档服务)，以便按业务实体进行搜索。
  5. 在服务名字段中，输入要搜索的模式 (如 a%)。此时会显示与输入的搜索条件相匹配的业务服务的列表。
  6. 选择要导入的服务，然后单击下一步。

    7. 如果找不到需要的服务，则原因可能是，您不属于有权可查看记录的安全组。

  7. 选择创建业务服务所要根据的绑定模板，然后单击下一步。

    8. 如果选定的服务具有多个绑定模板，则每个模板都将生成一个业务服务。在这种情况下，系统会提示您进一步缩小要导入的绑定模板的选择范围。

  8. 在导入 UDDI：选择导入位置页的项目下拉列表中选择一个项目，并选择要将服务及其相关联的资源导入的文件夹，然后单击下一步。
  9. 此时导入 UDDI：复查并导入服务页会显示将在系统中创建的所有资源 (业务服务、MFL、方案、WSDL 等) 的列表。此时还会显示不能导入的任何资源的警告消息。
    默认情况下，列表中的所有项目都会标记为导入的项目。
    1. 如果系统中已存在某一特定资源，并要手动解析相关性，请清除该资源旁边的复选框。
    2. 导入过程完成后，您可以在更改中心的查看冲突页中解析相关性。有关查看和解决冲突的详细信息，请参阅查看和解决冲突。
  10. 单击导入以开始导入过程。

    11. 显示一个弹出式窗口，说明导入过程。导入完成后，导入概要页会显示导入的结果。如果导入成功，则会在页顶部显示成功消息。如果有一个或多个资源无法导入，则会显示错误消息。

    • 转到资源的状态列可查看出错原因。
    • 要查看和解决其他冲突，请单击查看冲突。
  11. 要导入更多服务，请单击导入另一个。
  12. 要更改您的选择，请单击上一步。
