导入服务帐户或服务密钥提供程序

注： 在较早的 Oracle Service Bus 版本中，将服务密钥提供程序称为代理服务提供程序。

如果 JAR 文件由 AquaLogic Service Bus 3.0 或更高版本创建，且该 JAR 文件包含服务帐户或服务密钥提供程序，则可将这些资源随它们包含的用户名、口令、本地用户到远程用户的映射以及别名到密钥对的绑定一起导入。

对于每个服务密钥提供程序，Oracle Service Bus 会将别名到密钥对的绑定导入到 PKI 身份证明映射提供程序中。如果在导出期间对此数据进行了加密，则必须提供用于对该数据进行加密的口令。如果不知道该口令，则可以导出所有其他非加密资源。

如果要导入某个服务帐户或服务密钥提供程序，并且域中已存在对应的同名资源，则导入的资源将覆盖域中已有的同名资源，即使在当前会话期间已修改域中已有的该资源，也是如此，除非在导入期间指定保留安全设置。请参阅选择要导入的项目和资源中的步骤 5。
