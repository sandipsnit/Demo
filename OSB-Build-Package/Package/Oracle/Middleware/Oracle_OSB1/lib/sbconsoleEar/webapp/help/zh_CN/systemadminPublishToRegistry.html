将代理服务发布到 UDDI 注册表

可以将您的服务发布到注册表供其他组织发现和使用。在 Oracle Service Bus 中开发的所有代理服务都能够发布到 UDDI 注册表。您可以选择要在其下面发布服务的业务实体，并可以一次发布多个服务。

在发布到注册表之前，您必须拥有具有该注册表的帐户。Oracle Service Bus 支持与 3.0 版本兼容的 UDDI 注册表进行互操作。本部分中描述的配置假设您使用的是 Oracle Service Registry。

有关在 Oracle Service Registry 中设置用户帐户的信息，请参阅可从以下位置访问的 Oracle Service Registry 产品文档：http://www.oracle.com/technology/products/soa/registry/index.html。

从注册表中取消发布服务需通过 Oracle Service Registry 安装完成。

  1. 选择系统管理 > 发布到 UDDI。

    2. 只有不在会话中时，才能进行发布。退出会话便可访问注册表列表。所有可以访问的注册表都将显示在选定注册表的下拉列表中。

  2. 从发布注册表名称中选择您要发布到的注册表的名称，然后单击下一步。

    3. 如果 Oracle Service Bus 中仅配置了一个注册表，则不会显示初始注册表选择页，因为这是默认注册表。只有在定义了两个或多个注册表的情况下，才会显示注册表选择页。

    此时会显示发布到 UDDI：选择单独服务并发布页，其中显示了 Oracle Service Bus 代理服务及其在项目文件夹中的位置的列表。从最顶端的项目级别文件夹开始，显示了所有服务的名称、类型和说明的详细资料。

  3. 选择要发布的服务。
    1. 展开项目文件夹可以查看定义的代理服务。默认情况下，该文件夹及其内容标记为发布。
    2. 可以选择要发布的单独项。
  4. 在将服务发布到业务实体下拉列表中，选择您要将注册表中的服务添加到的业务实体。这是注册表中，服务将在其下进行分类的业务实体。
  5. 单击发布以发布服务。

    6. 此时会显示确认消息，表示服务已成功发布。

  6. 单击发布另一个返回到发布到 UDDI：选择注册表页。
