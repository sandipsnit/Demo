选择要导入的项目和资源

  1. 如果您将导入资源 JAR 文件并要确保导入所有与之相关联的资源，请选择包括相关性。
    2. 注： 如果将导入完整的项目 JAR 文件或早于版本 2.6 的配置文件，则不能选择包括相关性。
  2. 单击打开图标可显示用于保留该导入系统中值的高级设置。
  3. 如果您要重新导入一个资源，但希望保护对现有资源所做的任何定制设置，请选中保留环境变量。
    4. 注： 不能保留对现有资源中任何引用的定制。请参阅创建定制设置文件。
  4. 在导入“全局操作设置”资源或其他资源 (如包含相关联的服务级操作设置的业务服务或代理服务) 时，如果希望保留该导入系统中的操作值，则选择保留操作值。
  5. 为防止覆盖将要更新的资源中与安全相关的现有配置，请在以下选项中进行选择：
    1. 保留安全和策略配置 - 保留安全配置 (不包括访问控制策略) 和对直接绑定到服务 (而不是绑定到 WSDL) 的 WS-Policy 的引用。

      2. 有关详细信息，请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Security”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472。

    2. 保留身份证明 - 保留服务密钥提供程序中的 PKI 身份证明、服务帐户中的用户名和口令，以及 SMTP 服务器、代理服务器、JNDI 提供程序和 UDDI 注册表中的用户名和口令身份证明。
    3. 保留访问控制策略 - 保留访问控制策略。

      4. 表 28-1 显示资源概要信息。

      表 28-1 资源信息
      属性
      说明
      名称
      项目的名称。
      诸如 UDDI 注册表信息、SMTP 服务器、代理服务器、JNDI 提供程序，以及全局“操作设置”之类的全局资源都在其各自文件夹的系统项目下显示。
      类型
      资源类型 (为“项目”)。
      展开“项目”文件夹可显示和取消选择资源。
      操作
      显示 --- 或 !Deletions!
      展开“项目”文件夹可显示将对资源执行的操作 (创建、更新、删除)。
      为了防止删除资源，请取消选择它们。只有在要导入的 JAR 是完整的项目 JAR，并且该导入系统中存在导入的 JAR 文件中不存在的项目资源的情况下，才应调度资源以便进行删除。
      引用
      展开“项目”文件夹以便显示每个资源引用的对象数。

  6. 选择要对其执行操作 (创建、更新或删除) 的对象。
    1. 要显示 JAR 文件中包含的每个对象，可展开该文件的文件夹。与每个对象关联的复选框均处于选中状态。

      2. 将针对每个对象显示以下资源文件详细资料，如表 28-2 中所示。

      表 28-2 资源文件详细资料 
      属性
      说明
      名称
      资源的名称和路径。该路径由项目名和资源所在的文件夹名称组成。
      类型
      Oracle Service Bus 资源类型。
      注： 如果某个资源具有敏感的安全数据，则会显示一个加密图标。
      注意： 如果您选择了服务帐户或服务密钥提供程序，则可能覆盖现有的安全数据，除非您在导入期间指定保留安全设置。请参阅导入服务帐户或服务密钥提供程序。
      操作
      创建、更新或删除。
      只有在要导入的 JAR 是完整的项目 JAR，并且该导入系统中存在导入的 JAR 文件中不存在的项目资源的情况下，才应调度资源以便进行删除。
      引用
      此资源引用的对象数。

    2. 清除您不想导入的资源旁边的复选框。
  7. 单击导入。

完成之后

如果适用，继续指定导入安全设置。或者，继续复查导入概要。
