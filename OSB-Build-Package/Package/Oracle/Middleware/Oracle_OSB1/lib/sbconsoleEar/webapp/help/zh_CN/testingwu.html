测试代理服务

可以测试下列类型的代理服务：“任何 XML”、“任何 SOAP”、“消息传送”、“XML”和“SOAP”。可以测试具有 Web 服务安全 (WSS) 策略的 SOAP 代理服务。请参阅配置代理服务测试数据中的“Web 服务安全”。

注： 当测试控制台调用具有 HTTP 定制标记验证的代理时，此验证检查尚未完成。
注意： 启用了“直接调用”选项的测试代理服务将忽略某些重要的安全步骤，其中包括访问控制。Oracle 建议不要在生产系统中使用测试服务。
  1. 单击更改中心下面的激活以启用 Oracle Service Bus 管理控制台中的测试功能。

    2. 可以在资源浏览器或项目浏览器中测试代理服务。

  2. 选择资源浏览器 > 代理服务以便显示代理服务概要页。
  3. 在操作下，单击与要测试的代理服务相关联的 启动测试控制台图标。

    4. 此时，测试控制台会打开代理服务测试页。例如，使用随产品提供的示例时，单击与 LoanGateway1 代理服务关联的图标。

    注： 在集群域中，无法使用测试控制台测试任何配置的业务服务或路由到某个业务服务的代理服务。
  4. 对于 SOAP 与 XML 服务，选择要测试的 WSDL 操作。
  5. 为代理服务配置测试数据。此数据必须是代理服务希望从客户机得到的数据。

    6. 默认情况下，会启用直接调用和包括跟踪这两个测试配置选项。可以清除直接调用选项，这样也就会清除包括跟踪选项。这样做会使用间接调用方法执行测试，在此情况下，消息是通过传输层发送的。

    可以使用直接调用选项 (保持选中直接调用) 并禁用执行跟踪；仅仅清除包括跟踪复选框。

    要查看消息跟踪，请取消选中“直接调用”。

    注：要查看日志文件或标准输出 (服务器控制台) 中的跟踪信息，必须将 Oracle WebLogic Server 日志记录设置为以下严重性级别：

要记录的最低严重性：信息
日志文件：信息
标准输出：信息

有关设置日志严重性级别的信息，请参阅 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的“Using Log Severity Levels”，网址为：http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

     

  1. 单击“执行”。“代理服务测试”页会显示测试结果。有关解释测试结果的信息，请参阅查看代理服务测试结果。
  2. 要再次运行测试，请单击上一步。重复步骤 5 以及其后的步骤。
