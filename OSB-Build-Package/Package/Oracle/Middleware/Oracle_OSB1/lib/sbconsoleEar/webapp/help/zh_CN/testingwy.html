测试业务服务

可以测试下列类型的业务服务：“任何 XML”、“任何 SOAP”、“消息传送”、“XML”和“SOAP”。可以测试具有 Web 服务安全策略的 SOAP 业务服务。有关详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Testing Services with Web Service Security”，网址为： http://www.oracle.com/pls/as111170/lookup?id=OSBAG384。

注： 在集群域中，无法使用测试控制台测试任何配置的业务服务或路由到某个业务服务的代理服务。
  1. 单击更改中心下面的激活以启用 Oracle Service Bus 管理控制台中的测试功能。
  2. 选择“资源浏览器”>“业务服务”以便显示“业务服务概要”页。
  3. 在操作下，单击与要测试的服务关联的启动测试控制台图标。此时，测试控制台将打开业务服务测试页。例如，使用随产品提供的教程时，可单击与 loanSaleProcessor 关联的图标。
  4. 对于 SOAP 与 XML 服务，选择要测试的 WSDL 操作。
  5. 配置业务服务的测试数据（输入数据应为将由代理服务向业务服务发送的消息）。当测试代理服务不可用于业务服务时，“直接调用”和“包括跟踪”选项是可用的。默认情况下，使用直接调用选项可测试业务服务，即消息通过传输层进行传递。

    6. 要查看消息跟踪，请取消选中“直接调用”。

    注：要查看日志文件或标准输出 (服务器控制台) 中的跟踪信息，必须将 Oracle WebLogic Server 日志记录设置为以下严重性级别：

要记录的最低严重性：信息
日志文件：信息
标准输出：信息

有关设置日志严重性级别的信息，请参阅 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的“Using Log Severity Levels”，网址为：http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

  1. 单击“执行”。

    2. “业务服务测试”页会显示结果。有关详细信息，请参阅查看业务服务测试结果。

测试业务服务中的附件

此部分介绍如何使用测试控制台来测试业务服务中的附件。

为了进行说明，此部分假定一个基于 WSDL 的 HTTP 业务服务使用“submitAttachment”操作将 ZIP 文件作为 SOAP 消息中的附件进行发送。修改设置，使其适合于不同类型的二进制附件。

    1. 在业务服务的测试控制台中，转至 XML 页并为测试消息输入以下 SOAP 信封：
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. 在测试控制台的“附件”部分：
      1. Content-Type：application/zip (如 WSDL 中所定义)
      2. Content-ID：zipFile (必须与测试消息中 href="cid:zipFile" 的值匹配)
      3. 文件：单击浏览，然后选择您在 SOAP 信封中指定的 ZIP 文件。
      4. 单击添加。

    2. 单击执行以发送带有附件的测试消息。

要确认附件是否已成功发送，请查看服务器控制台中是否存在类似于以下内容的消息，该消息由 submitAttachment 操作记录：

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
