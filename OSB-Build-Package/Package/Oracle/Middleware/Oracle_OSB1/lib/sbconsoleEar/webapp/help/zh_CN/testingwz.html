跟踪代理服务

使用“直接调用”测试代理服务时，会启用跟踪。默认情况下，“包括跟踪”复选框是随“直接调用”选项一起选中的。如果不希望进行跟踪，请清除“包括跟踪”复选框。启用跟踪后，测试结果会包含跟踪的详细信息。使用跟踪可跟踪系统中的问题，并可将其隔离，以便进行更正。跟踪信息显示了在代码中的请求和响应路径中采用的路径。

  1. 单击更改中心下面的激活以启用 Oracle Service Bus 管理控制台中的测试功能。
  2. 选择“资源浏览器”>“代理服务”以便显示“代理服务概要”页。
  3. 在操作下，单击与要测试的服务关联的启动测试控制台图标。此时，测试控制台将打开代理服务测试页。
  4. 为代理服务配置测试数据。要启用跟踪，必须选择“直接调用”和“包括跟踪”选项。请参阅配置代理服务测试数据。
  5. 单击“执行”。“代理服务测试”页显示了服务的测试结果和跟踪信息。
  6. 向下滚动到“调用跟踪”部分。

    7. 此部分显示图形化表示形式的消息流。可以通过服务来跟踪消息并在预选跟踪点查看消息的状态。跟踪点是自动设置的。

  7. 单击“+”加号可展开消息流以查看更多详细信息。

    8. 查看跟踪时，还可以在 Oracle Service Bus 管理控制台中查看消息流。这有助于将跟踪与消息流中的阶段和操作进行关联。可以修改消息流并再次运行跟踪以查看输出。
