请求文档
输入字段将生成发送到业务服务的请求消息。单击执行可使用输入的值运行测试。此时，测试控制台会显示请求消息和服务的响应消息。
“请求文档”部分中的输入是特定于服务类型的。以下部分列出了服务类型和对各个服务类型所需的输入的描述。
任何 XML
这种服务要求输入为有效负载形式。有效负载就是即将发送的消息的内容。该内容应为 XML 文档。可以浏览到某个文件，或在文本框内输入消息内容。
任何 SOAP
这种服务要求输入为有效负载形式。有效负载就是即将发送的消息的内容。该内容应为 SOAP 信封。可以浏览到某个文件，或在文本框内输入消息内容。
请求文档（续）
消息传送
消息传送服务定义了 4 种可能的输入类型：none、XML、Binary 或 Text。这种服务要求单一的输入形式（基于文件或基于文本），除非是不需要任何输入的 none 类型。
Oracle 建议从文件中输入二进制输入。会使用系统编码将文本区中输入的数据转换为二进制输入。
对于文本服务，从文件中输入的数据必须转换为文本。编码输入字段用于指定转换期间要应用的编码。如果未配置此字段，则会使用系统编码。默认情况下，会使用在代理服务端点上配置的编码值对编码字段进行初始化。
请求文档（续）
XML
这种服务要求输入为有效负载形式。有效负载就是即将发送的消息的内容。该内容应为 XML 文档。可以浏览到某个文件，或在文本框内输入消息内容。
这是已定义多种操作的基于 WSDL 的服务。Oracle 提供了一个用于测试此服务的示例文档。在页的顶部列出了所有操作，并使用箭头突出显示已选定的操作。
SOAP 文档
对于 SOAP 文档，SOAP 信封通常由零个或多个标头和一个正文有效负载组成。
“表单”和“XML”选项卡提供了用来指定内容的备用方法。
表单选项卡包含“SOAP 标头”字段和“SOAP 正文”字段。“SOAP 标头”字段的内容应为 SOAP 标头标记 (这样可以定义多个标头)。“SOAP 正文”字段包含作为消息的一部分实际被发送的数据。该内容应为 XML 文档。标头和正文都用于生成 SOAP 信封。
XML 选项卡包含一个输入字段。此字段的内容应为即将发送的 SOAP 信封。
请求文档（续）
SOAP RPC
对于 SOAP RPC，SOAP 信封由零个或多个标头以及零个或多个参数组成。
“表单”和“XML”选项卡提供了用来指定内容的备用方法。
表单选项卡包含 SOAP 标头的单一输入，以及分别与各个参数相对应的一个输入字段 (输入字段的名称与参数的名称相对应)。“SOAP 标头”字段的内容应为 SOAP 标头标记 (这样可以定义多个标头)。
WSDL 用于检测每个参数的类型。单行输入字段用于基元类型，多行输入字段用于 XML 类型。示例文档是为了方便测试而自动生成的。
测试控制台使用标头和参数生成 SOAP 信封。
XML 选项卡包含一个输入字段。此字段的内容应为即将发送的 SOAP 信封。有效负载 (XML 输入) 可以是基于文件的，也可以是基于文本的。通过引用某个文件来提供输入优先于文本输入。浏览并选择要在测试中使用的文件。
Web 服务安全
此部分只适用于选定的操作具有 Web 服务安全 (WSS) 策略的情况下的 SOAP 服务。
服务提供程序
测试服务可从此服务提供程序获取用于 Web 服务安全操作 (数字签名和加密) 的所有客户端 PKI (密钥对) 身份证明。此字段是可选的。某些情况下需要使用服务提供程序，具体取决于此操作的请求/响应策略。有关详细信息，请参阅 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的“Testing Services with Web Service Security”，网址为 http://www.oracle.com/pls/as111170/lookup?id=OSBAG384。
Web 服务安全（续）
用户名
测试服务生成的 Web 服务安全用户名标记中的用户名。此字段是可选的。只有在某些情况下（操作的请求策略指定了标识断言）才需要此用户名。
请勿将此字段与传输安全上下文用户名字段相混淆。该字段必须是本地安全领域内有效的用户名和口令。无效的用户名或口令会在测试服务中导致客户端出错。
在某些情况下，测试服务生成 SAML 断言时可能也会使用此用户名和口令。
口令
测试服务生成的 Web 服务安全用户名标记中的口令。
传输
默认情况下，“传输”部分处于折叠状态。字段和值取决于测试配置。
验证
用户名
调用业务服务时，测试服务用于设置安全上下文的用户名。如果业务服务将收到 SAML 标记，此标识可能会在 SAML 标记中传播。请参阅 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的“Using SAML for Authentication”，网址为：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1647。该字段必须是本地安全领域内有效的用户名和口令。无效的用户名或口令会在测试服务中导致客户端出错。
口令
关联的口令。有关详细信息，请参阅用户名。
服务提供程序
当测试具有 CLIENT-CERT 验证的 HTTPS 业务服务时，会使用该字段。服务提供程序必须有相关联的 SSL 客户机身份证明。测试服务将在 SSL 握手期间使用该身份证明
调用模式
请求\响应
