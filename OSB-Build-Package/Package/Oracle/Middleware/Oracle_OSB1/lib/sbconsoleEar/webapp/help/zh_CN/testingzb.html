表 32-4 业务服务的测试结果
部分
说明
业务服务名称
业务服务的名称。
测试控制台操作
单击“返回”可显示上一个浏览页。
单击关闭可关闭测试控制台窗口。
请求文档
通过测试控制台发送到业务服务的请求消息。
如果测试控制台没有修改此请求消息，则此部分最初会处于折叠状态。对于使用表单选项卡配置的 SOAP 服务，或者在应用了 WS-Security 的情况下，则此部分最初会处于展开状态。
如果已应用了 WS-Security，则此部分将包含两个 SOAP 消息。第一个消息是明文消息。第二个消息是受保护的 SOAP 消息。
响应文档
消息响应。
对于带有 WS-Security 响应的 SOAP 服务，此部分将包含两个 SOAP 消息。第一个 SOAP 消息是客户机接收的受保护的消息。第二个 SOAP 消息是相应的明文消息。
响应元数据
随消息响应返回的元数据。

注： 显示受保护的 SOAP 消息时会进行美化，例如，添加额外的空格。因为空格会影响文档的语义，所以此 SOAP 消息不能始终用作字面值数据。例如，数字签名区分空格，因此可能会由于空格问题而导致签名无效。
