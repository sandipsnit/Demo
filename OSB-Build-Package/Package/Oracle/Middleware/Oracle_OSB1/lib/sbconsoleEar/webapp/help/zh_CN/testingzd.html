测试 XSLT 转换

可扩展样式表语言转换 (XSLT) 用于描述 Oracle Service Bus 中的 XML 到 XML 映射。要测试 XSLT 资源，必须提供输入 XML 文档，然后测试控制台将返回输出 XML 文档。XSLT 转换可以有其他指定的参数。转换所需的所有参数均显示在配置页上。默认值可用，但您可以覆盖这些值。

表 32-6 描述 XSLT 配置选项。

输入和参数
用于测试 XSLT 资源的输入文档和参数。
XML 输入
XML 输入可以是基于文件的，也可以是基于文本的。通过引用某个文件来提供输入优先于文本输入。浏览并选择要在测试中使用的文件。XML 输入是必填字段。
<param_name> ([] as XML)
param_name 是指定的 XSLT 参数。
有两种类型的输入：XML 和基元型（字符串型、整型、浮点型等）。默认输入类型为字符串型。选中与参数名相关联的复选框可标识 XML 类型的参数。

可以针对设计时或运行时进行测试。

  1. 如果要针对运行时进行测试，请单击“激活”。如果要针对设计时进行测试，请不要激活会话。
  2. 选择“项目浏览器”>“XSLT”以便显示“XSLT 概要”页。
  3. 在操作下，单击与要测试的资源关联的启动测试控制台图标。此时，测试控制台将打开资源测试页。
  4. 在此页面的输入和参数部分中，为资源配置测试数据。有关详细信息，请参阅表 32-6。
  5. 单击“执行”。“资源测试”页会显示结果。
  6. 要重新测试，请单击返回。您可以关闭测试控制台，然后修改并重新测试资源。
