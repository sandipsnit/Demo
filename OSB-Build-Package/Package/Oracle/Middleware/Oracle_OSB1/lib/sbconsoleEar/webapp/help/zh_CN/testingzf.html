使用 XQuery 表达式编辑器和 XQuery 条件编辑器

可以使用 XQuery 表达式在执行消息流期间针对消息上下文变量 (或部分消息上下文变量) 创建数据内容。可以在 XQuery 表达式编辑器中直接使用测试控制台测试表达式的定义。

同样，也可以使用 XQuery 条件来评估消息流中的布尔条件。可以在 XQuery 条件编辑器中直接使用测试控制台测试条件的定义。

XQuery 可接受多个输入并返回一个输出结果。每个输入都分别与 XQuery 中定义的一个 XQuery 未绑定变量相对应。XQuery 输入的值可以是基元值（字符串、整数、日期等等）、XML 文档或这些类型的序列。输出值可以是基元值（字符串、整数、日期等等）、XML 文档或这些类型的序列。

注： 测试控制台不支持输入序列。

param_name 是相应 XQuery 非绑定变量的名称。
在测试控制台中，如果类型为简单类型，将显示单行编辑框。如果数据为 XML，则将显示多行编辑框。如果变量是非类型的，则将使用组合输入 (<param_name> ([] as XML))。选中该复选框可标识类型为 XML 的参数。
XML 输入可以是基于文件的，也可以是基于文本的。通过引用某个文件来提供输入优先于文本输入。浏览并选择要在测试中使用的文件。
测试控制台中的输入基于类型来呈现，以便于了解必须输入的数据类型。默认类型为 XML。

  1. 在编辑管道消息流中的操作时访问测试控制台。
    • 要访问 XQuery 表达式编辑器，请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。
    • 要访问 XQuery 条件编辑器，请参阅创建和编辑内嵌 XQuery 和 XPath 表达式。

      • 显示“XQuery 测试表达式”页。所有需要的输入变量都显示在页面上。

  2. 在此页的变量部分中，为 XQuery 配置测试数据。有关详细信息，请参阅表 32-8。
  3. 单击“执行”。测试页会显示结果。
  4. 完成测试后，可单击返回以执行新的测试。要在更改 XQuery 后执行新的测试，必须先关闭测试控制台，然后再将其打开，以使更改生效。
