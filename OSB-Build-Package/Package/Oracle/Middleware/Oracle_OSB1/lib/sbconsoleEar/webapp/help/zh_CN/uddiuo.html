添加 UDDI 注册表

使用此页可添加和配置 UDDI 注册表。配置了注册表之后，可以将 Oracle Service Bus 代理服务发布到这些注册表，或从这些注册表导入要在 Oracle Service Bus 代理服务中使用的业务服务。

必须处于活动会话中才可以添加注册表。

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 选择系统管理 > UDDI 注册表。
  3. 在 UDDI 注册表面板中，单击“添加注册表”。
  4. 要配置注册表，请完成下列步骤。星号表示必填字段。有关必须在添加注册表时设置的属性的说明，请参阅表 29-1。
    1. 在名称字段中，输入注册表的名称。

      如需命名指导，请参阅资源命名限制。
    2. 在“查询 URL”字段中，以 http://host:port/APPLICATION_SERVER_CONTEXT/uddi/inquiry 格式输入查询 URL。
    3. 在“发布 URL”字段中，以 http://host:port/APPLICATION_SERVER_CONTEXT/uddi/publishing 格式输入发布 URL。
    4. 在“安全 URL”字段中，以 http://host:port/APPLICATION_SERVER_CONTEXT/uddi/security 格式输入安全 URL。
    5. 在“订阅 URL”字段中，以
      http://host:port/APPLICATION_SERVER_CONTEXT/uddi/subscription 格式输入订阅 URL。

      6. 默认情况下，安装程序中的 APPLICATION_SERVER_CONTEXT 的值为 registry。

    6. 在“用户名”字段中输入登录到注册表控制台所需的用户名。
    7. 在“口令”/（“确认口令”）字段中输入登录到注册表控制台所需的口令。
    8. 选中“将 tModels 加载到注册表”复选框将 Oracle Service Bus tModels 发布到此注册表。

      9. 该字段仅在向此注册表发布代理服务时为必填字段。

    9. 选中“启用自动导入”复选框可将导入服务与 UDDI 注册表自动同步。在选定此选项的情况下导入的任何服务都将与 UDDI 注册表保持同步。
      10. 注： “自动同步”是后台进程；不能使用会话“撤消”功能取消此进程。由于在下一个同步循环中将重新同步服务，因此撤消自动同步更改不是永久性操作。如果希望一个导入的服务与注册表不同步，必须分离该服务以避免将来基于注册表进行更新。请参阅分离服务。
  5. 单击“验证”可验证已提供的 URL。
  6. 单击“保存”用提供的设置配置注册表并将其保存在当前会话中。
  7. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
