编辑 WSDL

  1. 如果尚未执行此操作，请单击创建新建一个会话，或单击编辑进入现有会话。请参阅使用更改中心。
  2. 查找 WSDL，如查找 WSDL 中所述。
  3. 单击 WSDL 名称。查看 WSDL 详细资料页显示表 6-2、表 6-3 和表 6-4 中所示的信息。

    4. 表 6-2 WSDL 详细资料 
    属性
    说明
    上次修改者
    创建此 WSDL 或将其导入到配置中的用户。
    上次修改时间
    用户创建此 WSDL 或将其导入到配置中的日期和时间。单击日期和时间链接可查看此资源的更改历史记录。请参阅“查看更改历史记录”页。
    引用
    此 WSDL 引用的对象数。如果存在此类引用，可单击数字链接以查看对象列表。请参阅查看对资源的引用。
    引用项
    引用此 WSDL 的对象数。如果存在此类引用，可单击数字链接以查看对象列表。例如，如果选择此 WSDL 的端口或绑定作为特定业务服务或代理服务的服务类型，则在单击该链接后，此业务服务或代理服务将作为引用列出。请参阅查看对资源的引用。
    说明
    此 WSDL 的说明 (如果存在)。

    默认情况下，将显示 WSDL 的结构视图，其中包括表 6-3 中所示的信息。

    表 6-3 WSDL 结构详细资料
    属性
    说明
    目标名称空间
    用来限定 WSDL 中所包括的任何定义的名称空间。
    WSDL 定义
    与此 WSDL 关联的属性和组：
    • WSDL 端口类型
    • WSDL 绑定
    • WSDL 端口
    • XML 方案类型
    • XML 方案元素
    • WSDL 导入
    • XML 方案导入
    • WS-Policy 引用
    • 隐式 WS-Policy 引用
    WSDL 状态
    WSDL 的状态：
    • 有效

      • 当前 WSDL 包括的所有 XML 方案或 WSDL 的位置都已指定并且有效。而且，所有嵌套 XML 方案或 WSDL (由包括的 WSDL 或 XML 方案随后包括的 XML 方案或 WSDL) 的位置都已指定并且有效。

    • 无效

      • 当前 WSDL 所包括的一个或多个 XML 方案或 WSDL 的位置没有指定或者无效。此外，一个或多个嵌套 XML 方案或 WSDL（由包括的 WSDL 或 XML 方案接着包括的 XML 方案或 WSDL）的位置可能尚未指定或者无效。要解析无效的 XML 方案或 WSDL，请单击“编辑引用”。请参阅解析未解析的 WSDL 引用。

  4. 单击文本视图将显示 WSDL 详细资料的文本视图。文本视图包括表 6-4 中所示的信息。

    5. 表 6-4 WSDL 文本详细资料
    属性
    说明
    目标名称空间
    用来限定 WSDL 中所包括的任何定义的名称空间。
    文本
    此 WSDL 的文本。

  5. 要对这些字段进行更改，请单击编辑。有关这些字段的说明，请参阅添加 WSDL。
  6. 进行适当的编辑。
  7. 单击保存可提交当前会话中的更新。
  8. 要结束该会话并将配置部署到运行时，请单击更改中心下的激活。
注： 查看 WSDL 详细资料页可能也包括一个单独的回调按钮。只有在 WSDL 具有回调操作可重写的端口类型时，才会显示此按钮。如果单击此按钮，则将重写此 WSDL。
