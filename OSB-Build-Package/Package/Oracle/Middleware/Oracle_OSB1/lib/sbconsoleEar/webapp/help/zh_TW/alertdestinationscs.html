新增電子郵件收件者

在您新增電子郵件目標前，您必須設定 SMTP 伺服器 (請參閱新增 SMTP 伺服器)，或設定 WebLogic Server 中的 JavaMail 階段作業 (請參閱 http://download.oracle.com/docs/cd/E15523_01/apirefs.1111/e13952/taskhelp/mail/CreateMailSessions.html的 Oracle Fusion Middleware Oracle WebLogic Server 管理主控台線上說明中的「設定 JavaMail 的存取」)。如果沒有可用的 SMTP 伺服器資源或 JavaMail 階段作業及設定，您將無法設定電子郵件收件者。

  1. 選擇下拉清單中的「透過電子郵件傳送警示」，然後按一下「新增」以顯示「新增電子郵件收件者」頁面。
  2. 在「郵件收件者」欄位中，以下列格式輸入電子郵件收件者：mailto:username@hostname

    3. 這是必要欄位。

    指定多個電子郵件收件者時，可以輸入以逗號分隔的使用者名稱和主機名稱的清單：例如，mailto:username@hostname [,username_1@hostname_1]...[,username_n@hostname_n]

    只有第一個郵件收件者才需要在字首加上文字 "mailto:"。

  3. 在 SMTP 伺服器欄位中，選擇傳出電子郵件的 SMTP 伺服器名稱。如果在下一步中選擇郵件階段作業，則不需要此欄位。
  4. 在郵件階段作業欄位中，選擇可用的郵件階段作業。如果在上一步中選擇 SMTP 伺服器，則不需要此欄位。
  5. 在寄件者名稱欄位中，提供警示通知的寄件者名稱。這是可選欄位。
  6. 在寄件者地址欄位中，提供有效的電子郵件地址。如果指定「寄件者名稱」欄位的值，則需要此欄位。
  7. 在回覆名稱欄位中，提供接收回覆的收件者名稱。這是可選欄位。
  8. 在回覆地址欄位中，提供接收回覆的電子郵件地址。如果指定「回覆名稱」欄位的值，則需要此欄位。
  9. 在連線逾時欄位中，輸入連線在逾時之前必須等待伺服器回應的秒數。預設值為 0。
  10. 在要求編碼欄位中，輸入字元集編碼值。預設編碼值為 iso-8859-1。
  11. 按一下儲存。電子郵件收件者會新增至此警示目標的電子郵件收件者清單，並儲存在目前的階段作業中。
