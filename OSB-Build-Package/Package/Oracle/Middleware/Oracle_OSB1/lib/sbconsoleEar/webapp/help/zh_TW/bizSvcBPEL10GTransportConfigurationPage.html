BPEL-10g 傳輸組態頁面

使用這個頁面設定使用 BPEL-10g (Oracle BPEL Process Manager) 傳輸協定的業務服務的傳輸設定。如需使用 Oracle Service Bus 及 Oracle BPEL Process Manager 的詳細資訊，請參閱在 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Oracle BPEL Process Manager Transport"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1147。

描述
角色
BPEL 傳輸可讓您將要求訊息從 Oracle Service Bus 傳送至 Oracle BPEL Process Manager。業務服務可作為下列角色之一：
  • 同步從屬端
    可與 Oracle Service Bus 從屬端進行同步通訊，唯一必要的位置資訊為 BPEL 位址。此位址可以端點 URI 的形式靜態，及 (或) 透過 URI 重新寫入動態捕獲。
  • 非同步從屬端
    若要與 Oracle Service Bus 從屬端進行非同步通訊，Oracle BPEL Process Manager 的回傳在傳送至 Oracle Service Bus 時會使用與要求不同的連線，而您必須設定 Oracle Service Bus 以提供正確的回傳位址。如需更多詳細訊，請參閱在 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Advanced Use Cases (Asynchronous)"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163。
  • 服務回傳
    如果業務服務設計為 Oracle BPEL Process Manager 的服務回傳 (此時 Oracle BPEL Process Manager 會透過 Oracle Service Bus 呼叫外部服務)，則只有在執行時期能夠知悉回傳位址。請使用端點 URI bpel://callback。

    • 若您以標記 URI 設定業務服務，請設定管線邏輯，以動態設定 $outbound 上的 URI；例如，請使用 TransportHeader 動作。

    注意： 服務回傳業務服務不支援負載平衡或容錯移轉。
回傳代理
此選擇性欄位僅適用於非同步從屬端角色。此欄位可讓您選擇代理主機服務 (必須是任何 SOAP 類型的 SB 或 HTTP 代理)，用以將回傳路由至提出要求的 Oracle Service Bus 從屬端。如需更多詳細訊，請參閱在 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Advanced Use Cases (Asynchronous)"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1163。
服務帳號
針對 JNDI 相關資訊環境安全性，用以存取 Oracle BPEL Process Manager 傳遞服務。按一下「瀏覽」，然後選擇服務帳號。若未指定服務帳號，則會使用匿名主題。
服務帳號在可設定的類型方面並無限制 (如靜態或透通)，但執行時期必須可存取使用者名稱與密碼。
暫停交易
選擇「暫停交易」會使業務服務成為非交易服務，即使業務服務是由交易所叫用亦然。
若未選擇「暫停交易」：
  • 如果通訊協定指出 WebLogic Server 支援的通訊協定 (t3、iiop、http)，則會傳播交易。
  • 如果通訊協定指出 OC4J 伺服器 (ormi、opmn)，BPEL 傳輸將會引發異常狀況，因為 OC4J 不支援交易傳播。
在下列情況下，BPEL 傳輸會忽略「暫停交易」選項：
  • 在服務品質 (QoS) 設為「最佳效果」的情況下呼叫業務服務。BPEL 傳輸會自動暫停任何不支援 QoS 的交易。
  • 在 QoS 設為「正好一次」的情況下呼叫業務服務，此時不會有交易。
如需交易傳播描述，請參閱在 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Transaction Propagation"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1151。
分配原則
選取要針對此端點的分配原則而使用的 WebLogic Server 工作管理程式執行處理。如果不存在其他工作管理程式，則會使用預設的工作管理程式。
如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
