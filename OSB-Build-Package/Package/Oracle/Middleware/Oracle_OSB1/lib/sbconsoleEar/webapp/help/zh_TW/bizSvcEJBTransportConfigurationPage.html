EJB 傳輸組態頁面

「建立/編輯業務服務 - EJB 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

使用此頁面，可讓您透過 EJB 傳輸通訊協定來設定業務服務的傳輸設定。表 18-5 說明如何使用此頁面。

表 18-5 建立/編輯 商業 服務 - EJB 傳輸組態頁面 
選項
建立或編輯...
傳送呼叫者的主題
選取此核取方塊，讓 Oracle Service Bus 在叫用 EJB 而未設定服務帳號時，從代理主機服務傳送已驗證身份的主題。請注意，選取此選項時，會停用「服務帳號」欄位。
服務帳號
按一下瀏覽，然後從顯示的清單中選取服務帳號。如果未指定服務帳號，就會使用匿名主題。使用「傳送呼叫者的主題」選項時無法使用此選項。

支援交易
選取此核取方塊以指定交易支援。
從屬端 Jar
按一下瀏覽並從顯示的清單選取 EJB 從屬端 JAR 資源。若要瞭解建立 JAR 資源的相關資訊，請參閱 JAR。這是必要欄位。
轉換器 Jar
按一下瀏覽並從顯示的清單選取 EJB 轉換器類別 JAR 資源。如需瞭解 EJB 從屬端 JAR 資源及轉換器類別，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "EJB Transport"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009。
本位目錄介面
僅限 EJB 2.1 - 從 JAR 所填寫的選項選取必要的 EJBHome 介面。此 URI 範例的 JNDI 名稱必須與您在此所選取的 EJBHome 介面相關聯。如果 EJB 不是必要的類型，或未在 client-jar 中指定 EJBHome 介面，Oracle Service Bus 就會顯示警告。
遠端介面
僅限 EJB 2.1 - 此欄位會根據「本位目錄介面」的組態自動填寫。
商業介面
僅限 EJB 3.0 - 在從屬端 JAR 中選取要使用的商業介面。
目標命名空間
此欄位由選自 JAR 的資訊填充。
樣式
根據您的需求選取文件換行或 RPC。如果兩個以上的無狀態階段作業 EJB 方法具有相同的參數數目及資料類型，且您想要的作業是文件式作業，則必須將無狀態階段作業 EJB 指定為文件換行。
由於路由或發佈到 EJB 時，$body 的內容必須與樣式相符，因此樣式很重要。呼叫 EJB 時，樣式還會影響參數內容，特別是文件換行的參數內容。再者，會使用一個使用情況模式來定義 EJB 業務服務，然後使用路由到 EJB 的同一個 WSDL 來建立代理主機服務。在此情況下必須注意 WSDL 的樣式，因為用來叫用代理的從屬端工具可能會限制 WSDL 的樣式。
編碼
選取「已編碼」或「常值」。
方法
顯示的方法是所選 EJB 遠端或商業介面的方法。請選取需要的方法 (您可以選取多個方法)。請按一下 + 來展開方法。編輯預設參數值並選取轉換器 (如有提供或需要)。
若 JAX-RPC 堆疊不支援方法的參數或傳回類型，請排除這些方法，否則您必須建立引數與轉換類別的關聯。
您可以為指定的方法變更預設作業名稱。(根據預設值，作業名稱是方法名稱)。如果 EJB 包含相同名稱的方法，您必須變更作業名稱，讓它們具有唯一名稱。WSDL 要求作業名稱必須是唯一名稱。
注意：在指定的 EJB 中，如果方法之間的證明資料或交易設定不同，您可以自訂指定業務服務的方法，並為每個方法建立業務服務。這樣可讓您精確控制交易和證明資料。
異常狀況
若方法引發商業異常狀況時，即會出現此欄位。如果 EJB 方法引發資料類型不受 Java Web Services (JWS) 支援的異常狀況時 (如 ArrayList)，請使用「異常狀況」欄位選取適當的轉換器類別，將異常狀況轉換成 JWS 支援的類型。
您的轉換器類別必須實行 com.bea.wli.sb.transports.ejb.ITypeConverter。轉換器類別只能針對已檢查的異常狀況進行設定，無法對執行時期異常狀況進行設定。
請將轉換器類別與轉換的異常狀況類別包覆在從屬端或轉換器 JAR 中。
如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「EJB Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1009。
