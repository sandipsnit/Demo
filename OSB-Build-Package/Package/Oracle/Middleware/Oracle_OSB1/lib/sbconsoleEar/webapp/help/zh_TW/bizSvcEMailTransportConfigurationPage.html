電子郵件傳輸組態頁面

「建立/編輯業務服務 - 電子郵件傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

使用此頁面，可讓您透過電子郵件傳輸通訊協定來設定業務服務的傳輸設定。表 18-6 說明如何使用此頁面。

表 18-6 建立/編輯業務服務 - 電子郵件傳輸組態頁面 
選項
建立或編輯...
需要 SSL
選取此選項即可使用「安全通訊端層」進行通訊。SMTP 的預設 SSL 連接埠號碼是 465。
附註：
  • Oracle Service Bus 信任存放區中必須要有電子郵件伺服器憑證，才能使 SSL 順利運作。
  • 若您目前使用 TLS，您必須將信任憑證放置在預設的信任存放區 ($JAVA_HOME/jre/lib/security/cacerts) 中，或將信任憑證放置在任何信任存放區，然後在 $DOMAIN_HOME/bin/setDomainEnv.* 檔案中新增路徑，如下所示：

    set EXTRA_JAVA_PROPERTIES=%EXTRA_JAVA_PROPERTIES%
    -Djavax.net.ssl.trustStore=<信任存放區路徑>

SMTP 伺服器
對於 name@domain_name.com 的端點 URI 項目，請選取要使用的預設 SMTP 伺服器。如果您在端點 URI 中提供如表 18-3 描述中的 SMTP 伺服器參數，將會使用這些伺服器資源代替這個 SMTP 伺服器設定。
如果您使用「郵件階段作業」選項，請不要選取 SMTP 伺服器。
如需更多建立 SMTP 資源的資訊，請參閱新增 SMTP 伺服器。
郵件階段作業
對於 name@domain_name.com 的端點 URI 項目，請輸入設定郵件階段作業要使用的 JNDI 名稱。如果您在端點 URI 中提供如表 18-3 描述中的 JNDI 郵件階段作業參數，將會使用這些郵件階段作業代替這個郵件階段作業設定。
如果您使用 SMTP 伺服器選項，請不要選取郵件階段作業。
寄件者名稱
輸入此項服務的原始電子郵件帳號顯示名稱。
寄件者地址
輸入此項服務的原始電子郵件帳號。
回覆名稱
輸入回覆電子郵件帳號的顯示名稱。
回覆地址
輸入回覆的電子郵件地址。
連線逾時
輸入捨棄連線之前的逾時間隔 (以秒為單位)。如果輸入 0，則沒有逾時。
要求編碼
接受預設值 iso-8859-1 做為電子郵件傳輸中要求的字元集編碼，或輸入其他字元集編碼。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
