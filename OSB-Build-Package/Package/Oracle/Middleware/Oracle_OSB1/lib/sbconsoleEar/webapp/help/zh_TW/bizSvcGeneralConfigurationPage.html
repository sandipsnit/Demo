一般組態頁面

「建立/編輯業務服務 - 一般組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

使用此頁面可修改業務服務的一般組態。

當您建立業務服務時，這將是一系列用以設定服務的頁面中所顯示的第一頁。此頁面之後所顯示的頁面視您在此頁面上所做的選擇而有所不同。

表 18-1 說明如何使用此頁面。

表 18-1 建立/編輯業務服務 - 一般組態頁面
選項
建立或編輯...
服務名稱
輸入業務服務的唯一名稱。

請遵循資源命名限制的命名指南。
描述
輸入描述。
服務類型
服務類型會定義服務所交換之訊息的類型和封裝。選取要建立的業務服務類型：
  • WSDL Web 服務 - 選取此選項，建立以 WSDL 為基礎的業務服務。然後輸入 WSDL 名稱，並以其路徑限定 (例如 myProject/myFolder/myWSDL)。或者也可以按一下「瀏覽」，從「選取 WSDL」頁面中選取 WSDL。

    • (連接埠或連結) - 輸入連接埠名稱 (定義於 WSDL 中) 以說明實際的傳輸位址，或輸入連結名稱 (定義於 WSDL 中) 以對應到傳輸位址。若您如上述使用瀏覽選取 WSDL，選取 WSDL 定義頁面即會列出 WSDL 中所定義的所有連接埠與連結。當您選擇該頁面上的連接埠或連結時，(連接埠或連結) 欄位即會填入選定名稱。

  • 傳輸型服務 - 選取此選項，可建立使用 EJB 或流程傳輸的服務。
  • 訊息傳遞服務 - 選取此選項所建立的服務可交換內容類型完全不同的訊息。這些交換可以是要求/回應交換或單向交換。當搭配 HTTP 傳輸的 HTTP `GET' 選項使用時，也可以只有回應，不要有要求。與 Web 服務不同，要求和回應的內容類型不需要相同。
  • 任何 SOAP 服務 - 選取此選項以建立沒有明確定義之具體介面的 SOAP 服務。

    • 保留預設值 SOAP 1.1，或從下拉清單中選取 SOAP 1.2。

  • 任何 XML 服務 - 選取此選項以建立沒有明確定義之具體介面的 XML 服務

    • 只有訊息傳遞服務及此服務類型支援 HTTP GET。

  • 業務服務 - 選取此選項可複製現有的業務服務。

    • 輸入路徑 (project/folder) 與業務服務的名稱，或按一下「瀏覽」，從「業務服務的摘要」頁面選取業務服務。

    因為 Oracle Service Bus 不接受多個服務具有相同的 URI，所以必須變更所複製服務的 URI。

  • 代理主機服務 - 選取此選項可建立以現有代理主機服務為基礎的業務服務。
服務類型 (續)
注意： 從另一個服務建立服務時，會以下列方式維護警示規則：
    • 從業務服務建立代理主機服務或從代理主機服務建立業務服務時，會移除警示規則 (如果有的話)。
    • 從另一個代理主機服務建立代理主機服務或從另一個業務服務建立業務服務時，會保留警示規則 (如果有的話)。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
