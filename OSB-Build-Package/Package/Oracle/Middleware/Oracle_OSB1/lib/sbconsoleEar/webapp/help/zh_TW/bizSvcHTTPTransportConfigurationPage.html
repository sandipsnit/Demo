HTTP 傳輸組態頁面

HTTP 傳輸支援 HTTP 與 HTTPS 端點。

「建立/編輯業務服務 - HTTP 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

使用此頁面，可讓您透過 HTTP 傳輸通訊協定來設定業務服務的傳輸設定。表 18-10 說明如何使用此頁面。

HTTP 要求方法
此參數可讓您在要求中使用下列其中一個 HTTP 方法：
  • POST - 將所有不限長度的資料直接在通訊端連線上傳遞，做為一部分的 HTTP 要求主體。從屬端不會看見交換的過程，且 URL 不會變更。對於以 REST 為基礎的要求，POST 會指示傳輸執行建立/取代作業，或執行附有要求的動作。
  • GET - 您可以包含資訊做為要求的一部分，而該資訊可進一步說明要取得的項目。此項資訊會做為附加至要求 URL 的一連串字元，以查詢字串形式進行傳遞。當「要求訊息類型」設為「無」時，您可以在具有服務類型「任何 XML 服務」的業務服務中使用 GET。對於以 REST 為基礎的要求，GET 會擷取遠端資源的形式。
  • PUT - 您可以在具有服務類型「任何 XML 服務」或「訊息傳遞服務」的業務服務中使用 PUT。PUT 會指示傳輸以 REST 為基礎的要求執行建立/取代作業，如上載檔案至已知位置。
  • HEAD - 當「回應訊息類型」設為「無」時，您可以在具有服務類型「任何 XML 服務」或「訊息傳遞服務」的業務服務中使用 HEAD。HEAD 會指示傳輸取得遠端資源的標頭資訊，而不會以 REST 為基礎的要求取得完整形式的資源。
  • DELETE - 您可以在具有服務類型「任何 XML 服務」或「訊息傳遞服務」的業務服務中使用 PUT。DELETE 會指示傳輸以 REST 為基礎的要求執行刪除作業。
注意： 若已在 $outbound/transport/request/http:http-method 變數中設定方法，該值將會優先於您針對 HTTP 要求方法所選取的任何方法。
認證
選取下列其中一項：
  • 無 - 指定在存取此服務時不需進行認證。
  • 基本 - 指定需要進行基本認證，才能存取此服務。

    • 基本認證會指示 WebLogic Server 使用使用者名稱和密碼，向安全範圍中所設定的認證提供者 (如輕量型目錄存取通訊協定 (LDAP) 目錄服務和 Windows Active Directory) 進行從屬端的認證。從屬端必須在 HTTP 要求標頭上傳送它的使用者名稱和密碼。

    因為密碼會以純文字格式傳送，所以強烈建議不要透過 HTTP 進行基本認證。不過，透過 HTTPS 傳送密碼是安全的，因為 HTTPS 可提供加密通道。

    警告：所有使用者 (授權和匿名) 預設都可以存取業務服務。若要限制可以存取業務服務的使用者，請建立傳輸層級授權原則。請參閱編輯傳輸層級存取原則。

  • 從屬端憑證 - 指定加密通訊和嚴密的從屬端認證 (雙向 SSL)。若要進一步瞭解，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「Configuring Transport-Level Security」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557。
服務帳號
輸入服務帳號。服務帳號是使用者名稱和密碼的別名資源。如果您選取「需要基本認證」欄位，就需要「服務帳號」欄位。

分配原則
選取要針對此端點的分配原則所使用的 WebLogic Server 工作管理程式執行處理。如果沒有其他的工作管理程式，就會使用預設的工作管理程式。如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
要求編碼
接受預設值 iso-8859-1 做為 HTTP 傳輸中要求的字元集編碼，或輸入其他字元集編碼。
回應編碼
接受預設值 iso-8859-1 做為 HTTP 傳輸中回應的字元集編碼，或輸入其他字元集編碼。
代理伺服器
輸入代理伺服器資源，或按一下「瀏覽」，從已設定的代理伺服器資源清單中選擇項目。
遵循 HTTP 重新導向
選取此核取方塊可指定應自動遵循 HTTP 重新導向 (附有回應碼 3xx 的要求)。當您將外送要求傳送到業務服務的 URL 時，會發生重新導向，且該服務會傳回回應碼 (如 302)，說明 URL 無效且此要求需被傳送到其他 URL。
如果選取遵循 HTTP 重新導向核取方塊，Oracle Service Bus 會自動將要求重新傳送到新的 URL，您不需採取任何動作。如果您不想自動遵循重新導向 HTTP，請取消選取此核取方塊。
使用區塊串流模式
如果想使用 HTTP 區塊傳輸編碼來傳送訊息，請選取此選項。
注意：如果使用「遵循 HTTP 重新導向」選項，請勿使用區塊串流。無法在區塊模式自動處理重新導向和認證。
階段作業黏性
若想要針對業務服務的 HTTP 要求使用階段作業黏性 (也稱為階段作業相關性)，請選取此選項。
自黏階段作業 ID 名稱
如果啟用階段作業黏性，請輸入階段作業的識別名稱。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
