JEJB 傳輸組態頁面

使用此頁面利用 JCA 傳輸通訊協定來設定傳輸設定。如需使用 JCA 傳輸的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "JCA Transport"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1036。

分配原則
選取要針對此端點的分配原則而使用的 WebLogic Server 工作管理程式執行處理。如果不存在其他工作管理程式，則會使用預設的工作管理程式。
如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
EJB 規格版本
選取遠端 EJB 介面的 EJB 版本。
透過值傳送 XMLBeans
如果您想要傳輸產生以「內置」XML 呈現的 POJO 引數 (MLObject) (這是您可以用 XQuery 表示式存取及管理的參數)，請選取這個選項。
請注意：透過值傳送的 XMLObjects 沒有內置的類型資訊。如果使用這個選項，您在代理主機服務管線的 Java Callout 中將無法傳送類型為 XMLObject 的引數。
如果想要透過參照傳送 POJO (會有較佳的效能)，請不要選取這個選項。
傳送呼叫者的主題
選取服務帳號的另一種方式，即是選取此選項，讓 Oracle Service Bus 於叫用 EJB 時從代理主機服務傳送已驗證身份的主題。
服務帳號
按一下瀏覽並從顯示的清單中選取 JNDI 服務帳號。如果沒有指定服務帳號，則使用匿名主題。如需更多資訊，請參閱第 16 章，服務帳號。
從屬端 JAR
按一下瀏覽並從顯示的清單選取 EJB 從屬端 JAR 資源。從屬端 JAR 包含遠端服務的遠端或業務介面。從屬端 JAR 註冊為一般的封存檔資源。
本位目錄介面
僅限 EJB 2.1 - 從 JAR 填充的選項選取需要的 EJBHome 介面。
遠端介面
僅限 EJB 2.1 - 此欄位會根據「本位目錄介面」的組態自動填寫。
業務介面
僅限 EJB 3.0 - 從您要啟動的從屬端 JAR 中選取要使用的業務介面。
目標命名空間
此欄位由選自 JAR 的資訊填充。
方法
選取需要的方法。按一下 + 可展開方法，這可讓您編輯預設的參數值。
您可以為指定的方法變更預設作業名稱。(根據預設值，作業名稱是方法名稱)。如果 EJB 包含相同名稱 (超載) 的方法，您必須變更作業名稱，讓它們具有唯一名稱。WSDL 要求作業名稱必須是唯一名稱。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
