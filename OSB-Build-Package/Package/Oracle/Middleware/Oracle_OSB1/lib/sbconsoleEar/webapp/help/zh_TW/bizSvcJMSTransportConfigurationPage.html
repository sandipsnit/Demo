JMS 傳輸組態頁面

「建立/編輯業務服務 - JMS 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「JMS Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037。

使用此頁面，可讓您透過 JMS 傳輸通訊協定來設定業務服務的傳輸設定。表 18-13 說明如何使用此頁面。

表 18-13 建立/編輯業務服務 - JMS 傳輸組態頁面 (續)
選項
建立或編輯...
目標類型
選取 JMS 橋目標的類型：
  • 佇列 (適用於點對點)
  • 主題 (適用於發佈/訂閱)
訊息類型
選取下列其中一項：
  • 位元組 (針對無法解譯位元組的串流)
  • 文字 (適用於文字訊息)
如果為回應選取了 Java 訊息類型，則會停用此選項。
回應佇例
只有在選擇「佇列」做為「目標類型」時，才能使用此選項。
選擇下列其中一個回應選項：
  • 無 - 預期沒有回應。請為單向作業設定這個選項。
  • 所有要求 URI 使用一個 - 讓您輸入單一 URI 以處理回應，以及設定回應組態詳細資料，例如編碼及逾時，及選擇性地選取 JMS 服務帳號以傳輸 JMS/JNDI 證明資料。
  • 每個要求 URI 使用一個 - 這個選項提供回應容錯移轉，讓您對每個要求的 URI 輸入一個回應 URI 或目標。
您可以在每個要求/回應的 JMS/JNDI 證明資料，選擇性地選取服務帳號。
回應模式
只有當您在「回應佇列」欄位中選取回應選項時，才可以使用此選項。
選取下列其中一項：
  • 針對在 Oracle WebLogic Server 上不是執行 JAX-RPC 服務的所有服務，選取 JMSCorrelationID。
  • 針對在 Oracle WebLogic Server 上執行的 JAX-RPC 服務選取 JMSMessageID。
如需更多資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Message ID and Correlation ID Patterns for JMS Request/Response"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1051。
分配原則
選取要針對此端點的分配原則而使用的 Oracle WebLogic Server 工作管理程式執行處理。如果不存在其他工作管理程式，則會使用預設的工作管理程式。
例如，如果業務服務具有 JMS 傳輸通訊協定，則業務服務端點是可以與特定分配原則產生關聯的 MDB (訊息驅動 Bean) JAR 檔案。
如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
要求編碼
輸入編碼要求的字元集。預設為 UTF-8。
回應編碼
只有當選取「回應佇列」欄位中的其中一個回應選項時，才可以使用這個選項。
輸入編碼回應的字元集。預設為 UTF-8。
回應逾時
只有當選取「回應佇列」欄位中的其中一個回應選項時，才可以使用這個選項。
輸入在捨棄連線之前等待回應的時間量 (秒)。預設值零 (0) 表示回應永遠不逾時。
從屬端 Jar
當服務為 Java 回應類型的「訊息傳遞服務」時即可使用此選項。選取要用來取消佇列含 Java 物件之訊息的從屬端 JAR。選取從屬端 JAR 可確保其位於類別路徑中。

有關詳細資料，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Sending and Receiving Java Objects in Messages"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。
回應 URI
當您選取「所有要求 URI 使用一個」回應選項及 JMSCorrelationID 回應模式時，才可以使用這個選項。
請以下列格式輸入回應 URI：
jms://host:port/connection_factory/jndi_destination
若要將多部伺服器設定為目標，請使用下列格式：
jms://host1:port,host2:port/connection_factory/jndi_destination
您也可以在回應 URI 中省略主機及連接埠。例如：
jms:///connection_factory/jndi_destination

當省略主機及連接埠，會在本機伺服器上查找連線工廠/目的地。這非常有用，例如，要求的 URI 會送到外部連線工廠/目的地，但是您想要回應傳送至本機伺服器。

注意：雖然 Oracle WebLogic Server 允許在 JNDI 名稱中使用正斜線 (例如 myqueues/myqueue)，但是含有正斜線的 JNDI 名稱會干擾 Oracle Service Bus 所需的 URI 格式，因此您無法使用那些名稱。若要解決此問題，請定義一個 JMS 外部伺服器，並在 URI 中參照該外部伺服器。如需詳細資訊，請參閱 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的 Configure foreign servers，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLACH。

要求回應
當您對 JMSCorrelationID 模式選取「每個要求 URI 使用一個」回應選項以提供回應容錯移轉時，可以使用這個選項。
對於在一般「傳輸」頁面上輸入的每個要求 URI，請輸入回應 URI，如同先前「回應 URI」欄位所描述。
您可以選取 JMS/JNDI 證明資料的選擇性服務帳號，服務將使用此證明資料於要求及回應佇列中。
目標 - 回應
當您選取 JMSCorrelationID 回應樣式之所有要求 URI 回應選項的其中一個時，才可以使用這個選項。
輸入要接收回應的目標伺服器名稱，並且輸入回應 URI，如「回應 URI」欄位所描述。
要求連線
當您選取一個 JMSMessageID 樣式的每個 URI 回應選項以提供回應容錯移轉時，可以使用這個選項。
對每個要求 URI，根據在「序號」欄位的數字循序識別，選擇性輸入 JMS 連線工廠名稱。如果您沒有輸入名稱，JMS 傳輸會使用來自要求 URI 的連線工廠。
您可以選取 JMS/JNDI 證明資料的選擇性「服務帳號」，服務將使用此證明資料於要求及回應佇列中。
目標 - 目的地
當您選取 JMSMessageID 樣式的每個要求 URI 回應選項的其中之一以提供回應容錯移轉時，才可以使用這個選項。與要求連線一起使用這個欄位。
每個列出的目標伺服器 (由目前網域中的伺服器決定)，例如在叢集中的「受管理伺服器」，會根據「序號」列出在「要求連線」欄位中對應於這些序號的要求 URI。請為每個目標上的要求 URI，輸入要接收回應之目標伺服器的目的地佇列。
注意：因為 Oracle Service Bus 開發環境只支援單一伺服器環境，因此只會列出一個目標。若要在多個伺服器環境中設定這個欄位，請在「Oracle Service Bus 管理主控台」中將此這個服務建置到執行時期環境中並完成服務組態。
JMS 服務帳號
當您在「回應佇列」欄位中使用「無」或「所有要求 URI 使用一個」選項時，才可以使用這個選項。
選取要用於 JMS 伺服器所管理之 JMS 資源的服務帳號。服務帳號是用於要求及回應的使用者 ID 及密碼的別名資源。同一個服務帳號可同時用於 JMS 和 JNDI。
這個欄位與「傳送呼叫者主題」選項互斥。使用其中一個於 JMS/JNDI 認證。
如需更多資訊，請參閱服務帳號。
使用 SSL
只有在透過 TLS/SSL 連線提出要求時，才可以選取這個選項。
TLS/SSL (安全通訊端層) 可以提供安全連線，方式是允許兩個透過網路連線的應用程式對對方的識別進行認證，並加密應用程式之間所交換的資料。認證允許伺服器和從屬端 (選擇性) 認證網路連線另一端上應用程式的識別。此外，如果管理員已經在目標的 JNDI 項目上設定存取控制，進而限制了對個別 JMS 目標 (佇列或主題) 的存取，則服務在 JNDI 樹狀圖中尋找該項目時必須認證。使用「服務帳號」或「傳送呼叫者的主題」選項以進行認證。
到期
訊息到期前的時間間隔 (毫秒)。預設值為 0，表示訊息永不到期。
啟用訊息持續性
服務所使用的 JMS 傳遞模式，讓您在傳輸量與可靠度之間取捨。請選取這個選項 (預設) 確保訊息完成傳遞。如果可以承受偶爾流失訊息，請取消選取這個項以改善傳輸量。
順序單元
輸入訊息的順序單元。訊息順序單元可讓訊息產生器根據訊息的處理順序分類為單一單元。此單一順序單元要求該單元內的所有訊息，按照其建立的順序依次處理。
傳送呼叫者的主題
選取此核取方塊，讓 Oracle Service Bus 在傳送訊息時傳送已驗證身份的主題。
當您在不同網域啟用此欄位及業務服務目標 JMS 資源時，請同時在這兩個網域中啟用全域信任。請參閱 Securing Oracle WebLogic Server 中的 Configuring Security for a WebLogic Domain。
這個欄位與「服務帳號」選項互斥。請使用其中一個於 JMS/JNDI 認證。
當您在不同網域啟用此欄位及業務服務目標 JMS 資源時，請同時在這兩個網域中啟用全域信任。請參閱 Oracle Fusion Middleware Securing Oracle WebLogic Server 中的 "Configuring Security for a WebLogic Domain"，網址為： http://www.oracle.com/pls/as111170/lookup?id=SECMG402。
JNDI 逾時
在 JNDI 目錄樹中查詢目標/連線工廠時，使用的 JNDI 連線逾時 (以秒為單位)。
預設值零 (0) 表示連線永遠不逾時。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
