JPD 傳輸設定頁面

「建立/編輯商業服務 - JPD 傳輸設定」頁面是一系列頁面中的其中一頁，供您建立和編輯商業服務 (如建立和設定商業服務和編輯商業服務設定中所述)。

使用此頁面，可讓您透過 JPD 傳輸通訊協定來設定商業服務的傳輸設定。表 17-14 說明如何使用此頁面。

表 17-14 建立/編輯商業服務 - JPD 傳輸組態頁面 (續)
選項
建立或編輯...
系統服務帳戶
輸入使用者的服務帳戶，或按一下「瀏覽」，從顯示的清單中選擇服務帳戶。
系統服務帳戶會指定 JPD 傳輸用以傳送傳入訊息的 WLI 無狀態階段作業 Bean 叫用的使用者認證。
若未指定服務帳戶，則會使用內傳要求主題。若沒有內傳要求主題，則會使用匿名主題。
流程服務帳戶
輸入使用者的服務帳戶，或按一下「瀏覽」，從顯示的清單中選擇服務帳戶。
流程服務帳戶會指定 JPD 叫用的使用者認證。
若未指定服務帳戶，則會使用內傳要求主題。若沒有內傳要求主題，則會使用匿名主題。
回傳代理位置
輸入代理服務接收來自 JPD 回傳的位置，以下列格式指定：
jms://[host:port[,host:port]*]/MyFactory/MyQueue
其中：
  • host：(可選) 回傳代理服務設定所在之 Oracle Service Bus 伺服器的主機名稱
  • port：(可選) Oracle Service Bus 監聽傳入要求的連接埠號
  • MyFactory：佇列連接工廠的 JNDI 名稱
  • MyQueue：Oracle Service Bus 代理服務等待回傳要求之佇列的 JNDI 名稱
在叢集環境中，指定被管伺服器的主機名稱與連接埠號 (以逗號分隔)。
如果所叫用的 JPD 是同步 JPD 或不具回傳的非同步 JPD，則會忽略回傳代理位置。
分派原則
為外送回應選取預設的分配原則或設定的工作管理程式執行處理。如需工作管理程式的相關資訊，請參閱「Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server」中的「Using Work Managers to Optimize Scheduled Work」，網址為： http://www.oracle.com/pls/as111120/lookup?id=CNFGD112。
傳播交易
選擇此核取方塊可將 Oracle Service Bus 內的現有交易傳播至 JPD。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯商業服務 - 摘要頁面上檢查並儲存這項設定。
