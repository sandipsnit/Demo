MQ 傳輸組態頁面

「建立/編輯業務服務 - MQ 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「MQ Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117。

開始之前

設定 MQ 連線資源。請參閱 MQ 連線。

設定 MQ 傳輸

使用此頁面，可讓您透過原生 MQ 傳輸通訊協定來設定業務服務的傳輸設定。表 18-14 說明如何使用此頁面。

表 18-14 建立/編輯業務服務 - MQ 傳輸組態頁面 
選項
建立或編輯...
訊息類型
選取下列其中一項：
  • 位元組 (針對無法解譯位元組的串流)
  • 文字 (適用於文字訊息)
是否需要回應
選取此選項可指定在傳送外送訊息後需要回應。
輪詢間隔
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
輸入輪詢間隔 (毫秒)。預設值為 1000。
回應關聯模式
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
指定回應關聯模式是否應該根據下列項目：
  • MessageID
  • CorrelationID
  • 動態佇列 - 如果 WebSphere MQ 的實行使用回應關聯的動態佇列，請選取這個選項。MQ 傳輸只支援暫存動態佇列。
自動產生關聯值
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
選取此核取方塊以自動產生 CorrelationID 或 MessageID。
模型佇列
僅適用動態佇列回應關聯模式。輸入使用的模型佇列名稱以產生動態佇列。
MQ 回應 URI
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
回應應發佈的目的地。請輸入與端點 URI 相同格式的回應 URI：
mq://local_queue_name?conn=mq_connection_resource
或者，若要使用動態佇列：
mq://dynamic_queue_prefix?conn=mq_connection_resource
dynamic_queue_prefix 限制 32 字元，可用於在 MQ 伺服器上建立動態佇例。佇列名稱變為字首加上唯一 ID。例如，假設 dynamic_queue_prefix 為 example，動態佇列會命名為類似 example123129083821 的名稱。
您也可以在動態佇列回應 URI 中使用星號 (*) 作為萬用字元。例如：
mq://dynamic_queue_prefix*?conn=mq_connection_resource
mq://dynamic_queue_prefix*
mq://*
如果您在 URI 中沒有提供 dynamic_queue_prefix，傳輸將會使用 MQ 伺服器產生的動態佇列名稱。如果您在 URI 沒有提供明確的 mq_connection_resource (最佳作法)，傳輸會使用來自端點 URI 的 mq_connection_resource。
如需詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "MQ Transport"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117。
回應逾時
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
輸入在捨棄連線之前等待回應的秒數。
分配原則
分配原則是您想為此服務端點使用的 WLS 工作管理程式的執行處理。
如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
動態佇列集區
僅適用動態佇列回應關聯模式。如果您想要讓服務使用動態佇列的集區連線，請選取這個選項。
如果您想要使用動態佇列的個別連線集區，考慮設定動態佇列的專用 MQ 連線資源。
如果您想要在每個要求上建立新的動態佇列執行處理 (以及在回應後損毀佇列)，請不要選取這個選項。
端點 URI 'PUT' 選項
輸入下列 MQ PUT 訊息選項：
  • MQC.MQPMO_ALTERNATE_USER_AUTHORITY
  • MQC.MQPMO_DEFAULT_CONTEXT
  • MQC.MQPMO_FAIL_IF_QUIESCING
  • MQC.MQPMO_LOGICAL_ORDER
  • MQC.MQPMO_NEW_CORREL_ID
  • MQC.MQPMO_NEW_MSG_ID
  • MQC.MQPMO_NO_CONTEXT
  • MQC.MQPMO_NO_SYNCPOINT
  • MQC.MQPMO_NONE
  • MQC.MQPMO_PASS_ALL_CONTEXT
  • MQC.MQPMO_PASS_IDENTITY_CONTEXT
  • MQC.MQPMO_RESOLVE_LOCAL_Q
  • MQC.MQPMO_SET_ALL_CONTEXT
  • MQC.MQPMO_SET_IDENTITY_CONTEXT
  • MQC.MQPMO_SYNCPOINT
  • MQC.MQPMO_VERSION_1
  • MQC.MQPMO_VERSION_2
您可以使用 "|" 或 "+" 隔開多個選項。例如，您可以指定下列項目：
MQC.MQPMO_LOGICAL_ORDER | MQC.MQPMO_NEW_MSG_ID
將訊息置於外送佇列時，會套用 MQ PUT 訊息選項。
MQ 無法辨識的回應 URI
輸入 URI，此 URI 代表應該傳送無法辨識之回應訊息的佇列。請注意，只有在選取「自動產生關聯值」核取方塊時，才會啟用此設定。
如果您未指定此欄位的值，則會刪除無法辨識的回應訊息。
處理 RFH2 標頭
選取此選項，可從訊息有效負載剖析 WebSphere MQ RFH2 標頭，並自動產生內含 RFH2 資料的 RFH2Headers 傳輸標頭。
如果沒有選取此選項，則會直接傳遞收到的有效負載。

有關 MQ 傳輸如何處理 RFH2 標頭的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "About RFH2 Headers"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
