訊息處理組態頁面

「建立/編輯業務服務 - 訊息處理組態」頁面是一系列建立和編輯業務服務頁面中的一頁，如建立和設定業務服務及編輯業務服務組態中所述。

使用此頁面可指定 Oracle Service Bus 如何處理訊息內容。表 18-20 說明業務服務的訊息處理選項。不同的選項可用於不同的傳輸類型。

XOP/MTOM 支援
此功能可讓您使用 MTOM/XOP 格式編碼外送訊息。SOAP 訊息傳輸最佳化機制 (MTOM) 是從 Web 服務接收及傳送二進位資料的方法。MTOM 會使用 XML 二進位最佳化封裝 (XOP) 傳輸二進位資料。
Oracle Service Bus 支援使用下列傳輸的 XOP/MTOM：
  • HTTP/S
  • 本機
  • SB
選取已啟用核取方塊，讓業務服務以 MTOM/XOP 格式編碼外送訊息。請注意，對於以舊版組態為基礎的已匯入業務服務，將會停用此選項。
如果已啟用「XOP/MTOM 支援」，請從下列選項中，選取要如何處理 $header 與 $body 訊息相關資訊環境變數中的二進位資料：
  • 透過參照包含二進位資料：(預設值) 在設定 $body 訊息相關資訊環境變數時，將外送回應訊息中的 xop:Include 元素取代為 ctx:binary-content 元素。
  • 透過值包含二進位資料：在設定 $body 訊息相關資訊環境變數時，將外送回應訊息中的 xop:Include 元素取代為 Base64 編碼文字版的對應二進位資料。
請注意，如果已為業務服務啟用「XOP/MTOM 支援」，外送訊息則不一定全都必須使用 MTOM 格式。此設定會指定業務服務將可處理 MTOM 有效負載。
由於 Oracle Service Bus 不支援 MTOM 與 SwA 的組合，因此若在 Oracle Service Bus 嘗試分配外送要求至業務服務時，業務服務已啟用 MTOM/XOP，且 $attachments 訊息相關資訊環境變數並非空值，系統即會發出執行時期錯誤。
附件
Oracle Service Bus 支援以 HTTP/S 傳輸以串流傳送 MIME 附件。
此功能可讓您將外送回應訊息中的附件儲存在磁碟檔案中，然後以串流的方式處理資料，不需在記憶體中緩衝處理附件內容。此功能可讓業務服務穩定而有效地處理大型附件。
選取「將附件分頁至磁碟」核取方塊，讓業務服務以串流傳送外送回應訊息中的附件。
請注意，如果您啟用「XOP/MTOM 支援」，您必須在「XOP/MTOM 支援」下選擇透過參照包含二進位資料選項，才可使用「附件」選項。另請注意，包含附件的有效負載必須符合 RFC 822。明確而言，包含網際網路標頭的字行，必須以 CRLF 終止 (換行字元)。
結果快取
如果您叫用結果不常變更的業務服務，則結果快取會將快取的結果傳回至從屬端，而不用直接叫用外部服務，如此便可改善業務服務效能。
若要使用業務服務的結果快取，您必須在全域啟用它，如全域啟用結果快取中所述。
請根據以下準則，在「進階設定」區段中設定結果快取。
  • 選取支援選項。選取後，即會啟用其餘的欄位。
  • 表示式命名空間 - 使用表示式設定結果快取時 (無論使用的是「快取記號表示式」、「到期時間」或二者)，您可以輸入要用於表示式的命名空間和對應字首。此欄位也可讓您檢視現有的命名空間清單。
  • 快取記號表示式 - Oracle Service Bus 使用快取鍵來識別要取出或填入的快取結果。快取鍵的快取記號部分會提供唯一的 ID。您可以使用表示式 (「快取記號表示式」) 產生快取鍵的快取記號部分，該部分專用於識別唯一的業務服務快取結果。

    • 若要從要求中的值產生快取記號 (在叫用業務服務的代理主機服務訊息流程中)，請使用可從管線 $body、$header、$operation 或 $transportMetaData ($outbound/ctx:transport/ctx:request 或 $outbound/ctx:transport/ctx:response) 取得該值的表示式。例如，您可以從訊息 $body 的用戶 ID 中產生快取記號。

    「快取記號表示式」必須解析成一個「字串」或簡單的內容值 (例如，屬性或不含子元素的元素)。如果表示式評估為空值或發生錯誤，則不會快取結果。

    注意：您也可以從要求產生快取記號，不需在業務服務組態中設定「快取記號表示式」。若要這樣做，請在代理主機服務管線的 $outbound/ctx:transport/ctx:request/ctx:cache-token 中加上一個值。快取記號中的任何值都可覆寫業務服務組態中的「快取記號表示式」。

  • 到期時間 -「到期時間」或存留時間 (TTL) 可決定何時為業務服務的結果快取清除某筆資料。持續時間為零 (0) 表示永不到期。負值的持續時間表示永不快取。請選取下列其中一項：
 
    • 使用預設值 - 預設值是 DOMAIN_HOME/config/osb/coherence/osb-coherence-config.xml 中的到期延遲值。預設值為 5 分鐘。
    • 持續時間 - 設定特定的時間長度。
    • XQuery 表示式 - 使用表示式來設定快取記號時，您可以使用能從要求或回應中取得「到期時間」值的 XQuery 表示式。請選取「要求」或「回應」。

      若要從要求或回應中的值來產生到期時間，請使用可從管線 $body、$header、$operation 或 $transportMetaData ($outbound/ctx:transport/ctx:request 或 $outbound/ctx:transport/ctx:response) 取得該值的表示式。例如，使用您設定在 Cache-Control HTTP 標題中的值。

      「到期時間」必須解析成「整數」(代表秒數)、XQuery dayTimeDuration (XSD 類型) 或代表秒數之簡單內容的整數值 (例如，屬性或不含子元素的元素)。如果表示式評估為空值或發生錯誤，則不會快取結果。

      注意：您也可以從要求中產生到期時間，不需在業務服務組態中設定「到期時間」。若要這樣做，請在代理主機服務管線的 $outbound/ctx:transport/ctx:request/ctx:cache-ttl 中加上一個值。cache-ttl 元素的任何值都可覆寫業務服務組態中的「到期時間」。
如需結果快取的詳細描述，請參閱「Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus」中的「Improving Performance by Caching Business Service Results」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBAG170。
