表 18-17 描述 SOA-DIRECT 傳輸的傳輸特定組態選項。

如需 SOA-DIRECT 傳輸的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "SOA-DIRECT Transport"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV909。

表 18-17 建立/編輯業務服務 - SOA-DIRECT 傳輸組態
特性
描述
JNDI 服務帳號
選擇性。指定目標 SOA 服務的 JNDI 查找的安全性證明資料。服務帳號必須是靜態的。按一下瀏覽，然後選取服務帳號。如果您沒有指定服務帳號，會使用匿名主題。
角色
必要。定義服務使用的通訊模式。選取下列其中一個選項：
  • 同步從屬端 (預設) - 在這個角色中，因為沒有非同步回傳，因此「回傳代理主機」選項停用。「WS-Addressing 版本」選項也是停用。
  • 非同步從屬端 - 在這個角色中，因為非同步回傳通常是必要的，您可以定義「回傳代理主機」，而且您必須選取「WS-Addressing 版本」。只有在 WSDL 服務為 SOAP 類型時，才會啟用非同步選項。
  • 服務回傳 - 這個角色是在外部服務呼叫後，傳回非同步回傳到 SOA 服務。

    • 「服務回傳」角色中的服務，無法使用負載平衡或容錯移轉。

回傳代理
選擇性。只有對非同步從屬端角色啟用。
這個選項讓您指定接收回傳的代理主機服務。當您選取回傳代理主機時，如果要求或代理主機服務管線沒有提供 WS-Addressing，Oracle Service Bus 會自動在 SOAP 標頭中填入 ReplyTo 特性。您必須選取使用 SB 傳輸 (對 RMI) 的 WSDL 代理主機服務，並且回傳代理主機服務必須瞭解 WS-Addressing。
在要求中傳送或在代理主機服務管線中設定的 WS-Addressing 特性會用於取代您在這個選項中設定的回傳代理主機。
如果您不指定回傳代理主機，而且在要求中不包含 ReplyTo 特性，您必須透過代理主機服務管線在 SOAP 標頭中提供 ReplyTo 特性。
錯誤主機
目前不支援這個選項。您必須設定您的回傳服務以處理在非同步模式中的錯誤。
WS-Addressing 版本
必要。只對非同步從屬端角色啟用。
指定預設 WS-Addressing 版本以便當要求或代理主機管線中沒有提供 WS-Addressing 時使用。在要求中傳送或在代理主機管線中設定的 WS-Addressing 特性用於取代您在這個選項中設定的 WS-Addressing 版本。
對於在不同環境之間不相符的 WS-Addressing 版本，請在代理主機服務管線中執行任何需要的轉換。如需更多資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Transformation Examples"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1827。
分配原則
選取要針對此端點的分配原則而使用的 WebLogic Server 工作管理程式執行處理。如果不存在其他工作管理程式，則會使用預設的工作管理程式。
如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
傳送呼叫者的主題
選擇性。選取這個選項，讓 Oracle Service Bus 在叫用 SOA 服務時，從代理主機服務傳送已驗證的主題。「呼叫服務帳號」選項是「傳送呼叫者的主題」的替代方法，當您選取這個選項時將會停用。
注意：如果在不同的網域中，確認在從屬端及目標伺服器間的網域信任已啟用。如需更多資訊，請參閱 Oracle Fusion Middleware Securing Oracle WebLogic Server 中的 "Important Information Regarding Cross-Domain Security Support"，網址為： http://www.oracle.com/pls/as111170/lookup?id=SECMG403。
呼叫服務帳號
選擇性。「傳送呼叫者的主題」的替代方法，讓您選取 RMI 呼叫服務帳號以指定自訂安全性證明資料。您可以指定任何類型的服務帳號 (傳遞通道、靜態、對應)。
按一下瀏覽，然後選取服務帳號。如果您不指定服務帳號，則將使用非同步主題。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
