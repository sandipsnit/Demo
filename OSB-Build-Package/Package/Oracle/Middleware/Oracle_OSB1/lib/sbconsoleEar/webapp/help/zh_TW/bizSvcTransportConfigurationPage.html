傳輸組態頁面

「建立/編輯業務服務 - 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

使用此頁面可選取業務服務的傳輸通訊協定，以及設定其他一般傳輸組態設定。

外送傳輸層級安全適用於 Oracle Service Bus 代理主機服務與業務服務之間的連線。如需傳輸層級安全的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Configuring Transport-Level Security"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1557。

通訊協定
從清單中選取傳輸通訊協定。可用的通訊協定會根據所建立的服務類型而不同：
  • WSDL Web 服務：soa-direct、bpel-10g、dsp、http、jca、jms、sb、ws
  • 傳輸型服務：ejb、flow、jejb
  • 訊息傳遞服務：email、file、ftp、http、jms、mq (如果可用)、sftp、tuxedo
  • 任何 SOAP 服務： dsp、http、jms、sb
  • 任何 XML 服務： dsp、email、file、ftp、http、jms、mq (如果可用)、sb、sftp、tuxedo
負載平衡演算法
選取下列其中一項負載平衡演算法：
  • 循環法 - 此演算法會動態排序您在端點 URI 欄位中為此業務服務輸入的 URL。如果第一個排序失敗，則會嘗試下一個，直到重試計數完全耗盡為止。

    • 針對每個新訊息，都會有新的 URL 順序。

  • 隨機 - 此演算法會隨機排序您在端點 URI 欄位中為此業務服務輸入的 URL 清單。如果第一個排序失敗，則會嘗試下一個，直到重試計數完全耗盡為止。
  • 隨機加權 - 此演算法會隨機排序您在「端點 URI」欄位中為此業務服務輸入的 URL 清單，但是根據您在「加權」欄位中輸入的值，對某些 URL 嘗試較多次。
  • 無 - 此演算法會排序您在「端點 URI」欄位中為此業務服務輸入的 URL 清單，從頭到尾。
端點 URI
請根據您在通訊協定欄位中選取的傳輸通訊協定格式，輸入端點 URL。以下是每種傳輸 URI 格式的描述。選擇性 URI 元素會以方括號 [] 顯示。如需傳輸 URI 的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的相關傳輸章節，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV。
  • bpel-10g - protocol://host[:port] [/protocol-path]/domain/process[/version[/partnerlink/role]]
  • dsp - t3://dsp-ip-address:port/dsp-app-name
  • ejb - ejb:jndi_provider_name:ejb_jndi_name

    • 在 URI 中，jndi_provider_name 是 JNDI 提供者資源的名稱，而 ejb_jndi_name 是 JNDI 伺服器中 EJB 的 JNDI 名稱。

    如果 JNDI 提供者位於同一部伺服器上，則不需指定 JNDI 提供者。URI 將會是 ejb::ejb_jndi_name。

    注意：如果您的 EJB 在 IBM WebSphere 上執行，則 ejb_jndi_name 必須為下列格式：

    cell/nodes/node_name/servers/server_name/ejb_jndi_name

    或

    cell/clusters/cluster_name/ejb_jndi_name

    如 IBM WebSphere 文件 (http://publib.boulder.ibm.com/infocenter/wasinfo/v6r1/index.jsp?topic=/com.ibm.websphere.nd.iseries.doc/info/iseriesnd/ae/rnam_example_prop3.html) 所述。

  • 電子郵件 - 您可以在下列格式的任意組合中輸入多個端點 URI：
    • mailto:name@domain_name.com
    • mailto:name@domain_name.com?smtp=smtp_server_resource
    • mailto:name@domain_name.com?mailsession=jndi_mail_session
  • file - file:///drivename:/somename
  • flow - flow:<reference path to split-join resource>

    • 分割連接資源的參照路徑是分割連接的路徑 (project/folder) 和名稱，例如 batchorderProcessing/batchorder。

  • ftp - ftp://host:port/directory
  • http - http://host:port/someService

    • HTTP 傳輸支援 HTTP 與 HTTPS 端點。

  • jca - jca://<resource_adapter_jndi>
  • jejb - jejb:jndi_provider_name:ejb_jndi_name

    • EJB 2.1 範例：jejb:myProvider:osb.jejb.myJejbBiz21

    EJB 3.0 範例：jejb:myProvider:myBiz31#osb.jejb.myJejbBiz

    注意：如果您的 EJB 在 IBM WebSphere 上執行，則 ejb_jndi_name 必須為下列格式：

    cell/nodes/node_name/servers/server_name/ejb_jndi_name

    或

    cell/clusters/cluster_name/ejb_jndi_name

    如 IBM WebSphere 文件 (http://publib.boulder.ibm.com/infocenter/wasinfo/v6r1/index.jsp?topic=/com.ibm.websphere.nd.iseries.doc/info/iseriesnd/ae/rnam_example_prop3.html) 所述。

  • jms - jms://host:port/factoryJndiName/destJndiName

    • 若要將 JMS 目標鎖定到多部伺服器，請使用下列的 URI 格式：
    jms://host1:port,host2:port/QueueConnectionFactory/DestName

    注意：雖然 Oracle WebLogic Server 允許在 JNDI 名稱中使用正斜線 (例如 myqueues/myqueue)，但是含有正斜線的 JNDI 名稱會干擾 Oracle Service Bus 所需的 URI 格式，因此您無法使用那些名稱。若要解決此問題，請定義一個 JMS 外部伺服器，並在 URI 中參照該外部伺服器。如需詳細資訊，請參閱 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的 Configure foreign servers，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLACH。

    在叢集中：JMS URI 的主機名稱必須與設定在 WebLogic Server 的叢集伺服器名稱完全一致。

端點 URI (續)
  • mq - mq://<local-queue-name>?conn=<mq-connection-resource-ref>

    • local-queue-name 是 MQ 佇列的名稱，業務服務從這個佇列讀取訊息。

    mq-connection-resource-ref 是 MQ 連線資源的路徑 (project/folder) 和名稱。例如，default/my_MQconnection。

    注意： 「端點 URI」不可包含空格，因此建立 MQ 連線資源或專案/資料夾時，請勿在名稱中包含空格。

    若要在 Oracle Service Bus 中使用 MQ 傳輸，請參閱 MQ 連線。

  • sb - sb://[<jndi_provider_name/>]service_name

    • jndi_provider_name (選擇性) 是 Oracle Service Bus JNDI 提供者資源的名稱。若加以忽略，則會使用預設相關資訊環境。

    service_name 是目標服務，對應至遠端代理主機服務 URI。

  • sftp - sftp://hostname:port/directory
  • soa-direct - [protocol://authority]/default/compositeName[!versionNumber[*label]]/serviceName

    • 當 SOA 服務與 Oracle Service Bus 放置在同一台伺服器上時，通訊協定及授權是選擇性的。

  • tuxedo - tuxedo:resourcename[/remotename]

    • 在 URI 中，resourcename 會對應到 WTC 匯入名稱，而 remotename 會對應到遠端 Tuxedo 網域所匯出的服務名稱。URI resourcename 為必要，而 remotename 則是選擇性的。

    如果指定多個 URI，則端點必須有唯一的資源名稱。如果未指定遠端名稱，則它的值必須是資源名稱的值。如果未輸入任何遠端名稱，或遠端名稱與資源名稱相同，則只會允許一個 URI。在此情況下，資源名稱和遠端名稱的值會相同。這可讓已定義的「WTC 匯入」使用 WTC 負載平衡和容錯移轉。

  • ws - http://host:port/somename 或 https://host:port/somename
按一下新增，可新增一或多個其他的 URI。在執行時期，會根據您在負載平衡演算法欄位中選取的負載平衡演算法選取 URL。
如果您在負載平衡演算法欄位中選取隨機加權，也可以在端點 URI 欄位中輸入加權。預設值為 1。
如果您定義了多個端點，並在負載平衡演算法欄位中選取無，端點的順序就十分重要。您可以使用選項欄中的向上及向下箭頭，重新排序端點。
Oracle Service Bus 不支援相同業務服務內的重複端點 URI。
重試計數
為避免傳送外送要求時發生傳遞失敗，請指定重試個別 URL 端點的次數，亦即容錯移轉的嘗試次數。
例如，假設某個業務服務設定了一個 URI (U1)，而重試次數設為 3。如果 U1 在第一次嘗試時失敗，系統將會再重試 U1 端點 3 次。
如果某個業務服務設定了 2 個 URI (U1 與 U2)，而重試計數為 3，則在第一次嘗試 (以 U1 為例) 失敗時，系統將會嘗試 (故障移轉至) 下一個 URI (U2)。如果該嘗試也失敗，系統會再嘗試兩次，對 U1 與 U2 各試一次。
重試迭代間隔
指定系統再次對清單中所有的端點 URI 迭代之前所暫停的秒數。
例如，某業務服務設定了兩個 URI (U1 與 U 2)、重試計數 2 和重試迭代間隔 5 秒。如果第一次嘗試 (對 U1) 失敗，系統將會立即嘗試 U2。但如果 U2 也失敗，系統則會等待 5 秒，然後再次重試 U1。
重試應用程式錯誤
選取「是」或「否」。
為避免傳送外送要求時發生傳遞失敗，請指定是否要根據應用程式錯誤 (例如 SOAP 錯誤) 重試端點 URI。
此欄位僅供下列傳輸使用：HTTP、EJB、JMS、DSP、Tuxedo、SB 以及 WS。
