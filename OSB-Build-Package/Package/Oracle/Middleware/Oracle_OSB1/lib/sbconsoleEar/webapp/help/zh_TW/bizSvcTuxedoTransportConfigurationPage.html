Tuxedo 傳輸組態頁面

「建立/編輯業務服務 - Tuxedo 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「Tuxedo Transport」，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200。

使用此頁面，可以使用 Tuxedo 傳輸通訊協定設定業務服務的傳輸設定。 表 18-18 會描述如何使用此頁面。

表 18-18 建立/編輯業務服務 - Tuxedo 傳輸組態頁面 (續)
選項
建立或編輯...
欄位表格類別
輸入一或多個類別的名稱，用以說明已接收的 FML/FML32 緩衝區。這些名稱是用於 FML/FML32-to-XML 轉換常式，以將欄位名稱對應到元素名稱。這是完整類別名稱的清單 (以空格來分隔各個名稱)。
檢視畫面類別
輸入一或多個類別的名稱，用以說明已接收或傳送的 VIEW/VIEW32 緩衝區。這些名稱是用於 VIEW 至 XML 或 VIEW32 至 XML 轉換常式中，以將欄位名稱對應到元素名稱。這是完整類別名稱的清單 (以空格來分隔各個名稱)。
類別 Jar
選取適當的 JAR 資源，其中包含的 JAR 檔案必須具有此端點作業所需的 FML/FML32 或 VIEW/VIEW32 類別。
遠端存取點
從與「匯入」關聯的下拉清單中選取「遠端存取點」。此清單包含在 WTC 中設定的遠端存取點。如果沒有關聯的遠端存取點，就無法建立業務服務。
如果沒有遠端存取點存在，或要建立新的存取點，請選取新增。在相鄰的欄位中，輸入對應的存取點名稱與網路位址。端點通過驗證後，存取點即會新增至每部 WTC 伺服器的 WTC 組態中。如果不存在 WTC 伺服器，則會建立一部 WTC 伺服器。
如果已指定一個以上的 URI，則每個 URL會有一個遠端存取點，並會顯示 URI 以供參考。如果有多個 URI 存在，則每個 URI 都需要不同的遠端存取點。如果指定的 URI 已對應到現有的 WTC 資源，則會顯示對應的遠端存取點，但無法修改。
本機存取點
只有在選取「遠端存取點」欄位中的「新增」時，才會出現此欄位。
從下拉清單中，選擇要與新建的遠端存取點關聯的本機存取點。如果不存在存取點，或要建立新的存取點，請選取新增。在相鄰的欄位中，輸入對應的本機存取點名稱與本機網路位址。
要求緩衝區類型
選取遠端 Tuxedo 服務將接收的緩衝區類型。
要求緩衝區子類型
如果先前的要求緩衝區類型值為 VIEW 或 VIEW32，則會啟用此選項。請輸入緩衝區子類型，使用該子類型與要求緩衝區產生關聯。
是否需要回應?
選取此核取方塊以指定雙向叫用。如果未核取，底層的 tpcall 會與 TPNOREPLY 旗標一起叫用，且會非同步發佈空值回應。
是否暫停交易？
選取此核取方塊以暫停現有的交易 (如果有的話)。當遠端服務不支援交易時，此核取方塊會很有用。
分配原則
選取要針對此端點的分配原則而使用的 WebLogic Server 工作管理程式執行處理。如果不存在其他工作管理程式，則會使用預設的工作管理程式。
在單向叫用時，工作管理程式可非同步張貼空值回覆。如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
要求編碼
指定 Tuxedo 傳輸中要求的字元集編碼。
回應編碼
指定 Tuxedo 傳輸中回應的字元集編碼。
逾時
指定傳輸執行時期等待回覆的時間上限 (以秒為單位)，此值必須為大於或等於 0 的整數值。若未指定或設為零 (預設值)，回覆將會在 BLOCKTIME (本機 WTC 存取點允許封鎖呼叫的最大秒數) 逾時。
轉換樣式
選取下列其中一項：
  • 無 - (預設值) 可能不會顧及欄位的順序。
  • 已排序 - 會以正確的順序顯示欄位及其所有出現次數。
  • 排序並分組 - 如果欄位已邏輯結構化為記錄，則會以出現順序排序欄位，並依記錄進行分組。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
