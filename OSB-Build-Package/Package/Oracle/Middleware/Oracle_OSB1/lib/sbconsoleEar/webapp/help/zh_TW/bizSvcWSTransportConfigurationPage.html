WS 傳輸組態頁面

「建立/編輯業務服務 - WS 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯業務服務 (如建立和設定業務服務和編輯業務服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「WS Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083。

使用此頁面，可讓您透過 WS 傳輸通訊協定來設定業務服務的傳輸設定。表 18-19 說明如何使用此頁面。

表 18-19 建立/編輯業務服務 - WS 傳輸組態頁面
選項
建立或編輯...
回應逾時
輸入等待回應的秒數。
如果將此欄位保留為空白，表示沒有回應逾時。業務服務將會等待 RM 原則中所設定之順序逾時的持續時間。
如果您輸入零 (0) 的值，則沒有逾時，因此將永不逾時。
服務帳號
輸入服務帳號，或按一下「瀏覽」，從顯示的清單中選取服務帳號。
當此服務上具有 HTTP 基本認證原則時，服務帳號會指定將使用的證明資料。
進階設定
佇列錯誤訊息
選取此核取方塊，以便將錯誤訊息傳送至已設定的錯誤佇列。
錯誤佇列 URI
只有在選取「佇列錯誤訊息」核取方塊時，才能使用此選項。
以下列格式輸入用以儲存錯誤訊息之 JMS 佇列的 URI：

jms://host:port/connFactoryJndiName/queueJndiName

注意：雖然 Oracle WebLogic Server 允許在 JNDI 名稱中使用正斜線 (例如 myqueues/myqueue)，但是含有正斜線的 JNDI 名稱會干擾 Oracle Service Bus 所需的 URI 格式，因此您無法使用那些名稱。若要解決此問題，請定義一個 JMS 外部伺服器，並在 URI 中參照該外部伺服器。如需詳細資訊，請參閱 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的 Configure foreign servers，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLACH。

JMS 錯誤佇列服務帳號
只有在選取「佇列錯誤訊息」核取方塊時，才能使用此選項。
輸入服務帳號，或按一下「瀏覽」，從顯示的清單中選取服務帳號。
服務帳號會指定用於 JNDI 尋找以及傳送錯誤訊息至錯誤佇列所使用的證明資料。
對錯誤佇列使用 SSL
只有在選取「佇列錯誤訊息」核取方塊時，才能使用此選項。
選取此核取方塊，可在連線到錯誤佇列時使用 SSL。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」，在建立/編輯業務服務 - 摘要頁面上檢查並儲存這項組態。
