建立和設定業務服務

本節說明如何建立及設定業務服務。

針對 JCA 服務，您可以從 JCA 連結產生業務服務，如 「從 JCA 連結資源產生業務服務」所描述，或從 Oracle Enterprise Repository，如「從 Oracle Enterprise Repository 產生業務服務」所描述。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「專案瀏覽器」，然後選取要新增業務服務的專案或資料夾。
  3. 在專案/資料夾檢視畫面頁面上，從「建立資源」下拉清單中選取「業務服務」。
  4. 在建立/編輯業務服務 - 一般組態頁面上提供服務的名稱，並選取要建立的服務類型。

    請遵循資源命名限制的命名指南。
  5. 按一下下一步。接下來的頁面視您在第一頁上所做的選擇而有所不同。在後續每一頁輸入適當的資訊，直到進入摘要頁面為止，然後按一下「儲存」，將服務儲存在目前的階段作業中。

    6. 有關完成各個頁面的詳細指示，請參閱下列項目：

    • 一般組態頁面
    • 訊息類型組態頁面
    • 傳輸組態頁面
    • 通訊協定特定的傳輸組態頁面
    • SOAP 連結組態頁面
    • 訊息處理組態頁面
    • 摘要頁面
    • 業務服務原則頁面
    • 業務服務安全頁面

如果是從包含 WS-Policy 附件的 WSDL 建立業務服務，則原則 (唯讀) 會顯示在通訊協定特定的傳輸組態頁面上。如果任何一個服務的 WS-Policies 指定認證，您必須選取服務帳號。路由到此業務服務的代理主機服務將會使用此服務帳號來認證業務服務。

從 JCA 連結資源產生業務服務

您可以從 Oracle Service Bus 的外送 JCA 連結資源來產生 JCA 業務服務。如需 JCA 連結資源的詳細資訊，請參閱第 12 章，JCA 連結。

從 JCA 連結產生 JCA 業務服務：

  1. 在「Oracle Service Bus 管理主控台中」，如果您尚未處於「建立」或「編輯」模式，請按一下「變更中心」中的建立或編輯。
  2. 在「資源瀏覽器」中按一下「JCA 連結」。
  3. 找出您要用於產生服務的 JCA 連結，然後按一下「動作」圖示。
  4. 在顯示的視窗中，確認您要產生的 WSDL 的名稱及服務，為這些新資源選取位置後按一下「產生」。

    5. Oracle Service Bus 會產生服務及其對應 WSDL。

  5. 適當地修改其他在產生服務上的組態詳細資訊，例如端點 URI。

從 Oracle Enterprise Repository 產生業務服務

您可以透過 Oracle Service Bus 開發環境，從 Oracle Enterprise Repository 產生業務服務。如需更多資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Generating a Business Service from Oracle Enterprise Repository"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1797。
