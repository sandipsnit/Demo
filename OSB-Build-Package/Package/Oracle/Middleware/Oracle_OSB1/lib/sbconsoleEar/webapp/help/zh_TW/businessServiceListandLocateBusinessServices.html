尋找業務服務

  1. 請執行下列任一項操作：
    • 選取專案瀏覽器以顯示專案檢視畫面頁面或專案/資料夾檢視畫面頁面。然後瀏覽專案和資料夾，以尋找業務服務。
    • 選取資源瀏覽器 > 業務服務。業務服務的摘要頁面會顯示表 18-21 中所示的資訊。如需有關特性的詳細資訊，請參閱編輯業務服務組態。
  2. 若要限制清單中的項目數，則可以依據名稱和 (或) 路徑來進行篩選。在「名稱」和「路徑」欄位的「搜尋」下，輸入目標的名稱和 (或) 路徑，然後按一下「搜尋」按鈕。

    3. 路徑是專案名稱與業務服務所在資料夾的名稱。

    萬用字元 * 和 ? 皆可使用。搜尋會區分大小寫。

    按一下檢視全部可顯示網域中所有的業務服務。這樣會清除前次搜尋的搜尋參數。

    表 18-21 業務服務的摘要頁面
    特性
    描述
    名稱
    業務服務的唯一名稱。按一下名稱，以顯示檢視業務服務 - 組態詳細資訊頁面。
    請參閱編輯業務服務組態。
    路徑
    路徑是專案名稱與業務服務所在資料夾的名稱，例如 UDDI/BusServices/OSB_services。
    按一下業務服務的路徑，以顯示包含它的專案/資料夾檢視畫面頁面。
    動作
    請執行下列任何操作：
    • 按一下啟動測試主控台圖示來叫用「測試主控台」，您可以用此來驗證和測試服務及轉換的設計。在業務服務中，只能在執行期間 (也就是說，在階段作業啟動時) 使用「測試主控台」。至於轉換，則不論在階段作業內或外，都可以使用「測試主控台」。請參閱測試服務和測試轉換。
    • 將針對 WSDL 型業務服務顯示匯出 WSDL 圖示。按一下此圖示以匯出 WSDL，您接著可以使用外部工具 (例如 IDE) 來檢視或修改該 WSDL。

      • 請注意，這和「系統管理」模組中的「匯出資源」功能不同，該功能用來在兩個網域間移動及計劃資源。請參閱匯出 WSDL。

    • 將針對傳輸型業務服務 (例如 EJB 和流程) 顯示產生 WSDL 圖示 (但不會針對分割連接流程服務)。按一下此圖示以產生 WSDL，您接著可以檢視或修改該 WSDL。請參閱產生 WSDL。
    選項
    • 按一下刪除圖示刪除服務。當其他資源參照此資源時，會顯示刪除警告圖示。確認警告後即可刪除資源。因為仍有已刪除資源的未解析參照，所以可能會導致衝突。有關詳細資訊，請參閱刪除業務服務。
