業務服務原則頁面

「檢視業務服務 - 原則組態」頁面是一系列頁面中的其中一頁，供您編輯業務服務 (如建立和設定業務服務和檢視業務服務 - 組態詳細資訊頁面中所述)。

您可以在此頁面設定服務類型為「WSDL 型 XML Web 服務」、「訊息傳遞服務」、「任何 SOAP 服務」或「任何 XML 服務」之業務服務的原則設定。表 18-23 說明如何使用此頁面。為了能夠在「WSDL 型 XML Web 服務」、「訊息傳遞服務」或「任何 XML 服務」代理主機服務使用 OWSM 原則，必須將通訊協定設為 HTTP。

請注意，在 WSDL 類型服務中，連結至服務的 WLS 9.2 原則都會公開 (內置) 於有效的 WSDL 中。抽象原則在內置之前會先經過預先處理。OWSM 原則由參照連結，不會內置至有效的 WSDL 中。

服務原則組態
下列是「服務原則組態」欄位中的選項：

OWSM 原則

  • 從 OWSM 原則儲存庫 - 原則是在 Oracle Enterprise Manager 管理的中央 Oracle Web Services Manager 原則儲存庫中定義。這些原則支援 WS-Security 1.0/1.1、SAML 1.1/2.0、KerberosToken Profile 及其他業界標準。

    您可以使用此選項新增預先定義清單中的原則。按一下「服務層級原則」下的「新增」就會顯示清單。如有需要，請使用篩選來找出您想要的原則。業務服務只會顯示從屬端原則。

    如果使用 WSDL 來建立含內嵌標準 WS 安全原則的業務服務，Oracle Service Bus 會發生錯誤並顯示衝突。若要解決此衝突，請選取從 OWSM 原則儲存庫並按一下新增相容，以相容的 Oracle Web Services Manager 原則取代內嵌 WSDL 原則。Oracle Service Bus 會根據 WSDL 內嵌的標準原則，盡量嘗試從 Oracle Web Services Manager 原則儲存庫中，尋找最接近的相符原則。此演算法可能傳回零個、一個或多個相符的原則。

    如果按新增相容之後，Oracle Service Bus 找不到相容的原則，請按一下新增並選取適當的 Oracle Web Services Manager 原則。如果 Oracle Service Bus 傳回多個相容原則，請刪除不想使用的原則。

    注意：只有將網域設定為使用 Oracle Web Services Manager，才能啟動此選項。如需詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 Securing Oracle Service Bus with Oracle Web Services Manager，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681。

WLS 9 原則

  • 從 WSDL – 如果服務原則與服務所依據的 WSDL 相關聯，請選取此選項。這些原則支援 WS-Security 1.0、SAML 1.1 及其他業界標準。

    您可以使用此選項來檢視 (唯讀) 來自 WSDL 的要求和回應。

    注意：如果在業務服務組態中收到有關 Oracle Service Bus 本來不支援之 WS 安全原則的錯誤，請使用「從 OWSM 原則儲存庫」選項，以相容的 OWSM 原則取代內嵌 WSDL 原則。

  • 從預先定義的原則或 WS-Policy 資源 - 選取此選項即可從「Oracle Service Bus 管理主控台」新增服務層級原則、作業層級原則 (這類原則適用於要求和回應訊息)、要求原則以及回應原則。

    原則是在 Oracle WebLogic Server 中預先定義或是在含有 WS-Policy 資源的 Oracle Service Bus 中由使用者定義。這些原則支援 WS-Security 1.0、SAML 1.1 及其他業界標準。

    您可以使用此選項在服務、作業、要求和回應層級新增原則。按一下適當區段下的「新增」，即可從清單中選取原則。

注意：原則連結模型是互斥的。您只能在服務中使用一種原則。如果將原則直接連結至服務，就會略過所有 WSDL 型原則。

新增原則後，您可以在「安全」頁面提供覆寫。

 

完成作業後

按一下「更新」以儲存此組態，或者按一下「重設」以還原變更。
