變更中心

變更中心允許團隊在 Oracle Service Bus 中設定服務和描述資料時協同作業。除非使用者已經準備好將工作中組態簽入至匯流排的核心組態，否則每位使用者都是在沙箱階段作業中作業。變更中心可提供多位使用者處理組態的多個還原層級，而且可以檢視衝突。

大部分的變更中心頁面會根據您在階段作業中 (例如，如果您按變更中心的編輯) 或在階段作業外，顯示不同的資訊。例如，在階段作業中，當您按一下「變更中心」的檢視變更時，就會顯示檢視組態變更頁面並列示您在階段作業中所做的一切變更；在階段作業外時，頁面會列示所有的階段作業啟動。

檢視衝突
僅在階段作業中：此連結會顯示階段作業中發生的衝突數目。檢視並解決因為在目前階段作業與「Oracle Service Bus 管理主控台」中所有其他階段作業之間所做的變更而產生的所有衝突。如果在目前階段作業中修改的資源已經由其他階段作業修改及啟動，就會產生衝突。兩個階段作業對相同資源所做的兩個變更不會導致衝突，除非已啟動其中一個階段作業。
本頁也顯示階段作業中資源的語意錯誤。
檢視變更
在階段作業中：檢視您在目前階段作業所做的組態變更。
在階段作業之外：檢視前一個階段作業啟動造成的組態變更。
還原變更
在階段作業中：還原您在目前階段作業所做的變更。
在階段作業之外：還原階段作業的啟動。
檢視所有階段作業
在階段作業中和階段作業之外：檢視「Oracle Service Bus 管理主控台」中的所有現有階段作業。
啟動階段作業
僅在階段作業中：啟動階段作業。
檢視作業詳細資訊
在階段作業中：檢視您在目前階段作業所做的特定變更的詳細資訊。
在階段作業之外：檢視前一個階段作業啟動所做的特定變更的詳細資訊。
