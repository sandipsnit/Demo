啟動階段作業

  1. 在變更中心按一下啟動。啟動階段作業頁面會顯示階段作業名稱、使用者名稱和描述欄位。

    2. 不管系統中的其他活動，建立和關閉階段作業的作業均會繼續。但是，如果其他階段作業正在啟動，啟動階段作業頁面會顯示錯誤，指出具有擱置的 Weblogic Server 變更之使用者。有關解決您的階段作業與其他已啟動的階段作業所做變更產生的衝突的詳細資訊，請參閱檢視和解決衝突。

  2. 如果有驗證錯誤，會顯示錯誤訊息，指出您的階段作業有驗證錯誤。在繼續作業之前，請先檢視並修正組態衝突。有關解決衝突的詳細資訊，請參閱檢視和解決衝突。

    3. 直到您檢視所有的衝突之後，才可以啟動階段作業。如果在您檢視現有的衝突時又產生新的衝突，在啟動之前，會跳出訊息通知您有新的衝突。

  3. 在「描述」欄位中輸入描述。在您按一下「變更中心」之下的「檢視變更」時，「描述」欄中會顯示此描述，以顯示階段作業啟動所造成的組態變更。
  4. 若要啟動階段作業，請按一下「提交」。如果在其間沒有產生新的衝突，階段作業會結束並建置組態至執行期間。
注意： 當您嘗試以 JMS 端點 URI 在另一部伺服器 (正在工作的伺服器以外的單一伺服器或叢集中的受管理伺服器) 啟動階段作業時，請確認目的地伺服器可用。
注意： 如果 JMS 端點 URL 指定的目的地無法到達，則 Oracle Service Bus 不允許以 JMS 傳輸登錄代理主機服務。換句話說，Oracle Service Bus 會為 JMS 服務檢查特定連線工廠是否存在；如果不存在，便會產生階段作業啟動錯誤。
