還原工作的順序

您可依照任何順序來還原工作 (前提是個別的還原動作會產生有效資料)。還原作業會將資源的值設為該資源變更之前的值。

在此例中，要還原的是建立物件的工作，物件沒有先前的狀態可返回 - 換句話說，在執行此工作之前並沒有物件存在。還原作業有效地從階段作業刪除新的物件。在此例中，參照要刪除的物件之物件發生錯誤。您可以在「變更中心」的「檢視衝突」頁面上檢視此種錯誤。

當您不在階段作業中工作時，可以存取檢視組態變更頁面，以查看先前啟動的階段作業。您可以復原這些階段作業。如果復原動作會造成執行期間設定的錯誤，系統不會讓您復原先前啟動的階段作業。例如，如果您嘗試復原的階段作業啟動會導致移除另一個物件所參照的物件，則不會允許該復原動作。

可以將還原動作還原。在還原 [作業] 的選項欄中，按一下還原圖示。Oracle Service Bus 支援無限的還原。這表示您可以還原您還原先前還原的還原，以此類推。
