檢視和解決衝突

使用此頁面可讓您檢視組態中錯誤的診斷訊息，以及檢視與解決在您的階段作業和其他已啟動階段作業中所做的變更之間的衝突。檢視衝突連結也會顯示階段作業中即時衝突的數目。

檢視衝突
  1. 按一下變更中心下的檢視衝突。檢視衝突頁面可視衝突的性質顯示下列任何或所有資訊集合：
    • 錯誤 (錯誤圖示表示無法確認的重大衝突) - 通知您在組態中有重大衝突。若未解決這些衝突，則無法確認變更。請參閱表 2-6 並解決衝突。
    • 並行更新 (以警告圖示表示可確認的非重大衝突) - 警告您與其他已啟動的階段作業有不相容的變更。請參閱表 2-7。
    • 參考訊息 (「警告」圖示表示可確認的非重大衝突) - 通知您在組態中有非重大衝突。

    3. 如果您在某階段作業中變更的物件在您開始編輯目前的階段作業後已於執行時期中變更，「顯示衝突」頁面即會顯示表 2-7 所示的資訊。

解決衝突

若要解決衝突，請使用「訊息」欄中提供的資訊 (如表 2-6 中所述) 以瞭解問題所在，然後編輯造成衝突的物件，以修正問題。

若為發生並行更新衝突的情況，也就是說，在已由其他階段作業修改並啟動的現行階段作業中修改資源時發生衝突 (如表 2-7 中所述)，有兩種方法可供您解決衝突：

  • 按一下變更中心下的啟動。這樣會將您的變更儲存至執行期間，表示將覆寫先前階段作業啟動建置到核心資料的變更。換句話說，其他使用者在其他階段作業儲存的變更，已經被您在此階段作業的變更所覆寫。
  • 在表格按一下同步欄中的同步圖示 (如表 2-7 中所述)。此動作會將此物件 (在此階段作業中) 回復到執行時期所儲存的狀態。
