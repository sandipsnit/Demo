檢視組態變更

此頁面會根據您在階段作業中 (也就是說，您已按一下「變更中心」的「編輯」) 或在階段作業外，顯示不同的資訊。

  • 當您在階段作業中時，檢視組態變更頁面會顯示您在目前階段作業期間所做的組態變更清單。請參閱檢視階段作業中的組態變更。
  • 當您在階段作業之外時，檢視組態變更頁面會顯示先前階段作業啟動所造成的組態變更清單。請參閱檢視階段作業啟動造成的組態變更。
檢視階段作業中的組態變更
  1. 如果尚未做這樣的處理，請按一下建立來建立新的階段作業，以對目前的組態進行變更。請參閱使用變更中心。
  2. 至少對組態做一個變更。
  3. 按一下「變更中心」下方的「檢視變更」。針對您在目前的階段作業期間對組態所做的每項變更，「檢視組態變更」頁面會顯示表 2-2 所示的資訊。

    4. 表 2-2 在階段作業中的組態變更
    特性
    描述
    工作
    實作之工作的描述。此作業連結到作業詳細資訊頁面。請參閱檢視作業詳細資訊。
    執行時間
    執行工作的日期和時間。
    使用者
    實行變更的使用者名稱。
    工作狀態
    工作的狀態：
    已完成-工作已完成。
    已還原-工作已還原。
    還原已完成-已完成還原。
    還原者
    還原工作的使用者名稱。如果工作沒有還原，此欄位將包含 None。
    選項
    按一下「還原」圖示，以放棄執行工作。
    當您在階段作業中工作時，可依照任何順序來還原工作。
    還原工作的結果是工作的物件會還原至有問題的工作執行之前的狀態。但是請注意，在您還原工作之後對相同物件所執行的任何工作也會還原。請參閱還原工作。

檢視階段作業啟動造成的組態變更
  1. 確定您不在階段作業中。請參閱結束階段作業。
  2. 按一下「變更中心」下方的「檢視變更」。針對您先前啟動的每個階段作業，檢視組態變更頁面會顯示表 2-3 所示的資訊。

    3. 表 2-3 啟動的組態變更
    特性
    描述
    工作
    啟動的階段作業之描述。此作業連結到作業詳細資訊頁面，其中顯示在階段作業中執行的作業。請參閱檢視作業詳細資訊。
    描述
    如果在使用「啟動階段作業」頁面來啟動階段作業之前有輸入描述，會顯示對啟動的階段作業之額外描述。
    執行時間
    啟動階段作業的日期和時間。
    使用者
    啟動階段作業的使用者名稱。
    工作狀態
    階段作業的狀態：
    已啟動 - 已啟動階段作業。
    已部分啟動-如果無法存取叢集中的一或多台伺服器，以致無法在這些伺服器上啟動階段作業，則會在階段作業啟動期間顯示這個狀態。可以存取伺服器時，就會在伺服器上啟動任何尚未啟動的變更。
    已還原-已還原之前啟動的階段作業，並已捨棄在階段作業中執行的所有作業。
    還原已啟動-已啟動還原。
    正在進行-因為階段作業啟動可能需要很長的時間，所以正在進行階段作業啟動時會顯示此狀態。
    已失敗 - 如果階段作業啟動失敗會顯示。Oracle Service Bus 會追蹤階段作業啟動失敗，但是不會追蹤在階段作業中個別更新引起的失敗。
    還原者
    還原此工作的使用者名稱。如果階段作業啟動沒有還原，此欄位將包含 None。
    選項
    按一下還原圖示，以放棄階段作業啟動及在該階段作業中所執行的作業。您可依照任何順序還原階段作業啟動。
    按一下「還原到階段作業」圖示 ，以建立新的階段作業，並預先填入所需的工作以便還原階段作業中已啟動的變更。請參閱還原到階段作業。
