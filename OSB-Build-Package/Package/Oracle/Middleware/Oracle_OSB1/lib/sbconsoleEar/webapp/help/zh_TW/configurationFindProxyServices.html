尋找代理主機服務

  1. 選取「作業」>「智慧搜尋」。
  2. 在類型下拉清單中，選取代理主機服務，然後按一下搜尋。代理主機服務的摘要頁面會顯示表 26-2 中所示的資訊。
  3. 若要限制清單中的項目數，或尋找特定的代理主機服務：
    • 按一下開啟圖示，以顯示其他搜尋篩選。如果指定了多個搜尋條件，則只會傳回符合全部條件的結果。
    • 依代理主機服務名稱與路徑篩選。在「名稱」和「路徑」欄位中，輸入搜尋目標的名稱和路徑，然後按一下「搜尋」。此路徑是專案名稱和代理主機服務所在之資料夾的名稱。您可以使用 * 和 ? 做為萬用字元，以執行更廣泛的搜尋。
    • 依服務狀態篩選。從狀態下拉清單中選取代理主機服務狀態：「已啟用」或「已停用」，然後按一下搜尋。選擇「全部」可忽略服務狀態篩選條件。
    • 依服務監控篩選。從監控下拉清單中選取代理主機服務的監控狀態：「已啟用」或「已停用」，然後按一下搜尋。選擇「全部」可忽略服務監控篩選條件。
    • 依服務的 SLA 警示設定篩選。從 SLA 警示下拉清單中選取代理主機服務的 SLA 警示設定：「已啟用」或「已停用」，然後按一下搜尋。選取「全部」即可忽略服務 SLA 警示篩選條件。
    • 依服務的管線警示設定篩選。從管線警示下拉清單中選取代理主機服務的管線警示設定：「已啟用」或「已停用」，然後按一下搜尋。選取「全部」即可忽略服務管線警示篩選條件。
    • 依具有訊息報告的服務篩選。從訊息報告下拉清單中選取代理主機服務的訊息報告狀態：「已啟用」或「已停用」，然後按一下搜尋。選取「全部」即可忽略服務訊息報告篩選條件。
    • 依具有記錄的服務篩選。從日誌下拉清單中選取代理主機服務的記錄狀態：「已啟用」或「已停用」，然後按一下搜尋。選取「全部」即可忽略服務記錄篩選條件。
    • 依具有執行時期追蹤的服務篩選。從追蹤下拉清單中選取代理主機服務的執行時期追蹤狀態：「已啟用」或「已停用」，然後按一下搜尋。選擇「全部」可忽略服務追蹤篩選條件。

      • 表 26-2 代理主機服務摘要資訊 
      特性
      描述
      名稱
      指派給代理主機服務的名稱。此名稱是連到作業設定頁面的連結。請參閱設定代理主機服務的作業設定。
      路徑
      與代理主機服務關聯的專案。若是服務位於專案資料夾內，則也會列出該資料夾。顯示的路徑格式如下：
      project-name/root-folder/ . . ./parent-folder
      此路徑是連到「專案瀏覽器」中對應路徑的連結。
      類型
      父項服務類型：代理主機服務。
      狀態
      代理主機服務狀態：「已啟用」或「已停用」。
      監督
      代理主機服務的監督狀態：「已啟用」或「已停用」。
      SLA 警示
      SLA 警示狀態：「已啟用」或「已停用」，以及在等於和高於下列層級啟用的層級：「正常」(N)、「警告」(W)、「次要」(Mn)、「主要」(Mj)、「臨界」(C) 或「嚴重」(F)。
      管線警示
      管線警示狀態：「已啟用」或「已停用」，以及在等於和高於下列層級啟用的層級：「正常」(N)、「警告」(W)、「次要」(Mn)、「主要」(Mj)、「臨界」(C) 或「嚴重」(F)。
      報告
      服務的訊息報告狀態：「已啟用」或「已停用」。
      日誌
      記錄狀態：「已啟用」或「已停用」，且已啟用的層級為(含以上)：「除錯」(D)、「資訊」(I)、「警告」(W) 或「錯誤」(E)。
      追蹤
      代理主機服務的執行時期追蹤狀態：「已啟用」或「已停用」。
      動作
      「編輯訊息流程」圖示是連到該代理主機服務之管線的連結。

您也可以使用此頁面來進行下列操作：

  • 若要移除搜尋篩選並顯示所有代理主機服務，請按一下「檢視全部」。
  • 若要捨棄變更，並使用與目前已儲存之設定相同的搜尋條件來重新整理頁面，按一下「重設」。
  • 在階段作業中，若要啟用或停用任何服務層級的設定，請選取或清除設定旁邊的核取方塊，然後按一下「更新」。
    • 注意： 服務層級設定的執行時期效果取決於其對應的全域設定。您必須啟用服務的全域設定與服務層級設定，才能在執行時期完全啟用服務。此外，「SLA 警示」同時取決於全域與服務層級的「監督」狀態。亦即，這兩個者都必須啟用，才能在執行時期啟用 SLA 警示。請參閱啟用全域設定。
    注意： 此外，也必須啟用「服務狀態」。
  • 若要自訂您的作業設定資訊檢視畫面，請按一下表格自訂器圖示。請參閱自訂表格檢視畫面。
