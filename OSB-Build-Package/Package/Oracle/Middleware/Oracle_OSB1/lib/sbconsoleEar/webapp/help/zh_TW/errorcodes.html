錯誤碼

本節說明執行代理主機服務時可能發生的 Oracle Service Bus 錯誤。與這些錯誤關聯的錯誤碼會在 fault 相關資訊環境變數元素內表現出來。您可以使用下列 XQuery 敘述來存取這個值：

$fault/ctx:errorCode/text()

fault 元件內的錯誤碼後面附有該錯誤的詳細資訊。

表 A-1 列出 Oracle Service Bus 錯誤碼。

安全錯誤碼 (386000...386999)
BEA-386000
一般安全性錯誤。可能的原因如下：
  • 雖設定自訂記號或自訂使用者名稱/密碼認證，但已透過 WS 安全執行認證。
 
BEA-386100
認證失敗。可能的原因如下：
  • XPath 設定為用在主體或標頭上，但無此訊息區段 (連結層傳回空值)。
  • 評估 XPath 表示式時，XPath 引擎引發異常狀況。
  • 使用者名稱 XPath 或密碼 XPath 的結果不是 TEXT 或 ATTR 類型的節點。
  • XPath 傳回多個節點。
  • 已擷取使用者名稱/密碼，但認證失敗。
  • 已擷取自訂記號，但識別宣告失敗 (CSS 識別宣告服務引發 LoginException 或任何其他異常狀況)。
 
BEA-386101
遺漏認證記號。可能的原因如下：
  • 使用者名稱 XPath 或密碼 XPath 傳回空的結果或空字串。
  • 自訂記號 XPath 傳回空的結果、空值屬性值或空值文字值。
 
BEA-386102
拒絕訊息層級授權。可能的原因如下：
  • 訊息層級的存取控制原則拒絕對代理主機服務的存取。
 
BEA-386103
代理主機服務作業選取演算法無法確定要求的作業名稱或傳回無效作業 (不在 WSDL 中或空值的作業)。可能的原因如下：
  • 計算作業時發生錯誤。
  • 作業選取演算法傳回空值。
  • 作業選取演算法傳回的作業不是 WSDL 所宣告的作業。

錯誤詳細資訊綱要

以下是 XML 片段的 綱要定義，其中附有表 A-1 中所列的錯誤碼。這些 XML 片段會出現在錯誤相關資訊環境變數中的 <ctx:details> 元素底下。這些 綱要並非全都來自單一命名空間。同時，雖然已知的錯誤碼通常暗示某一特定 XML 片段，但某些 XML 片段可為數個錯誤碼所使用。
