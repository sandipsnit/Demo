Oracle Service Bus 簡介

Oracle Service Bus 是組態型原則驅動 Enterprise service bus (ESB) 的 Service Bus。它提供高度擴充性與可靠性的服務導向整合、服務管理、與異質 IT 環境間的傳統訊息代理。它在統一的軟體產品中，結合了智慧型訊息代理和訊息路由與轉換，以及服務監督與管理功能。「Oracle Service Bus 管理主控台」能讓您控制服務和原則組態，並可監督系統和作業工作。Oracle Service Bus 依賴 WebLogic Server 執行時期的功能來執行。如需詳細資訊，請參閱Oracle Fusion Middleware Concepts and Architecture for Oracle Service Bus，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBCA。

資源命名限制

在 Oracle Service Bus 組態中命名任一目錄或資源時，允許的字元如下：

    • 所有 Java 識別碼字元 (含 Java 關鍵字)，如 Java 語言規格的「識別碼」和「關鍵字」章節所述，網址為：http://java.sun.com/docs/books/jls/third_edition/html/lexical.html#3.8

    • 名稱內的空格、句號和連字號 (不為開始或結束字元)

不允許使用 / \ * : " < > ? | 等字元。
