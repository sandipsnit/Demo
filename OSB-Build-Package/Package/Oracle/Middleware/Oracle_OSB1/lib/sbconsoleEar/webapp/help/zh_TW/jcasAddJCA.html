新增 JCA 連結

在 Oracle Service Bus 中建立 JCA 連結的最簡單方法，就是批次匯入您已建立的 JCA 資源 (.jca 檔案、關聯的 WSDL，及 TopLink 或 EclipseLink 對應檔案)。在批次匯入上，Oracle Service Bus 會自動地從 .jca 檔案中產生 JCA 連結、從關聯的 JCA WSDL 產生 WSDL 資源，及從對應檔案產生 XML 文件資源，以及維護檔案之間的相依性。

如需更多有關匯入的資訊，請參閱從 Zip 檔案載入資源、從 URL 檔案載入資源以及匯入資源。

在匯入 JCA 資源後，您可以從其產生代理主機程式或商業資源，如從 JCA 連結資源產生代理主機服務及從 JCA 連結資源產生業務服務的描述。

如果沒有要批次匯入 JCA 資源，您可以手動建立 JCA 連結，如下列步驟描述：

  1. 在 Oracle JDeveloper 中建立一個 JCA 服務 WSDL、相對應的 JCA (.jca) 檔案，並視需要建立其他資源，例如 TopLink 或 EclipseLink 對應檔案和 XSLT 檔案。

    2. 如需更多資訊，請參閱 Oracle Fusion Middleware User's Guide for Technology Adapters，網址為： http://www.oracle.com/pls/as111170/lookup?id=TKADP。

  2. 如果尚未進行這樣的處理，請按一下建立在「OSB 管理主控台」中建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  3. 在「OSB 管理主控台」中，建立或匯入在 Oracle JDeveloper 中建立的 JCA 資源，例如 WSDL、XML 以及 XSLT 資源。
  4. 建立 JCA 連結。
    1. 選取專案瀏覽器，然後選取「JCA 連結」的專案或資料夾位置。隨即顯示專案/資料夾檢視畫面頁面。
    2. 從「建立資源」欄位，選取「JCA 連結」以顯示「建立新的 JCA 連結資源」頁面。
    3. 在「JCA 檔案」欄位中，按一下「瀏覽」選取 JCA 檔案。

      4. 「資源名稱」欄位會自動以檔案名稱 (除去副檔名) 填入。

      如果不選取檔案，您可以在「JCA 檔案」欄位的文字區域中複製及貼上 JCA 檔案的內容。

    4. 在資源名稱欄位，輸入或變更 JCA 連結名稱。JCA 連結名稱必須是唯一的。

      請遵循資源命名限制的命名指南。
    5. 在「資源描述」欄位，請輸入此 JCA 連結的描述。
    6. 按一下「儲存」。JCA 連結會儲存在目前階段作業中。

      7. 如果 JCA 連結還沒參照必要的資源時，例如 WSDL及對應檔案，會出現驗證警告。

  5. 按一下新的 JCA 連結名稱以便在「檢視 JCA 連結」頁面中顯示。
  6. 按一下「編輯參照」。
  7. 在編輯 JCA 連結資源的參照頁面中，瀏覽以選取「JCA 連結」的參照資源，例如 WSDL、TopLink/EclipseLink 對應檔案以及 XSLT 檔案。
  8. 按一下「儲存」。
  9. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。

在您新增 JCA 連結資源及設定資源相依性 (與 WSDL 關聯) 之後，或您已經匯入維護他們相依性的 JCA 資源之後，您可以從 JCA 連結產生代理主機及業務服務。如需更多資訊，請參閱「從 JCA 連結資源產生代理主機服務」及「從 JCA 連結資源產生業務服務」。
