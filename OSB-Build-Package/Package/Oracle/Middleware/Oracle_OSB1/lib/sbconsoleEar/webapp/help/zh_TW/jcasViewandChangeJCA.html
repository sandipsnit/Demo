編輯 JCA 連結

使用「檢視 JCA 連結詳細資料」頁面檢視及變更 JCA 連結的詳細資訊。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找 JCA 連結，如尋找 JCA 連結所描述。
  3. 按一下 JCA 連結名稱。檢視 JCA 連結頁面會顯示在表 12-2 中所示的資訊。

    4. 表 12-2 JCA 連結詳細資訊 
    特性
    描述
    上次修改者
    建立此 JCA 連結或將它匯入到組態中的使用者。
    上次修改時間
    使用者建立此 JCA 連結或將其匯入組態中的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱「檢視變更歷史記錄頁面」。
    參照
    提供 JCA 連結參照的 WSDL 資源連結。請參閱「檢視資源的參照」。
    參照者
    參照此 JCA 連結的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱「檢視資源參照」。
    動作圖示
    使用編輯模式中的「變更中心」，「動作」可讓您從 JCA 連結產生 Oracle Service Bus WSDL 及 JCA 服務。
    JCA 檔案
    顯示 JCA 連結的唯讀檢視。
    抽象 WSDL
    顯示 JCA 連結的關聯 WSDL。如果 JCA 連結沒有參照 WSDL，請按一下「編輯參照」以參照關聯的 WSDL。
    TopLink XML
    顯示「JCA 連結」的必要 TopLink 或 EclipseLink 對應檔案。如果「JCA 連結」資源未參照必要的對應檔案，請按一下編輯參照來選取必要的對應檔案。只有在 JCA 檔案宣告 XML 資源時，才會顯示此欄位。
    XSLT
    顯示「JCA 連結」的必要 XSLT 檔案。如果「JCA 連結」資源未參照必要的 XSLT 資源，請按一下編輯參照來選取必要的 XSLT 檔案。只有在 JCA 檔案宣告 XSLT 資源 (例如通訊端轉接器) 時，才會顯示此欄位。

  4. 若要修改「描述」或「JCA 檔案」，請按一下「編輯」。

    5. 您不可以變更「資源名稱」。

  5. 按一下「儲存」，確認目前階段作業中的更新。
  6. 若要變更「JCA 連結」之必要資源的參照，或修正無法解析的參照，請按一下編輯參照並選擇適當的資源。
  7. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
