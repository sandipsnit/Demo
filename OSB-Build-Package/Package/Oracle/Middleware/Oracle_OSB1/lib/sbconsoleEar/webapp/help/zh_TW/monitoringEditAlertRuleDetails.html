設定警示規則的一般資訊

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「資源瀏覽器」>「代理主機服務」或「業務服務」。
  3. 按一下服務名稱。
  4. 選取「SLA 警示規則」頁籤，再按一下「新增」。
  5. 在規則名稱欄位中，輸入警示規則的名稱。這是必要欄位。
  6. 在警示摘要欄位中，輸入警示規則的簡短描述。如果這個警示規則已設定為使用電子郵件警示通知，則這些文字也會成為電子郵件訊息的主旨行，且不能超過 80 個字元。如果未提供警示摘要，則會改用預設文字 Oracle Service Bus Alert。
  7. 在「規則描述」欄位中，輸入警示規則的描述。
  8. 在警示目標欄位中，輸入警示目標資源的名稱，或按一下瀏覽顯示選取警示目標頁面。從清單中選取警示目標，然後按一下提交。

    9. 依照預設，警示一律會傳送至「Oracle Service Bus 管理主控台」。若要設定及選取其他警示目標資源，請參閱新增警示目標及尋找警示目標。

  9. 在開始時間欄位中，以 HH:MM 格式輸入開始時間。例如，輸入 09:00 AM。請利用開始時間和結束時間欄位來指定規則在到期日之前在一天之中生效的時段。例如，每天從 9am 到 5pm。
  10. 在結束時間欄位中，以 HH:MM 格式輸入結束時間。例如，輸入 05:00 PM。請利用開始時間和結束時間欄位來指定規則在到期日之前在一天之中生效的時段。例如，每天從 9am 到 5pm。

    11. 警示規則會從您指定的開始時間到結束時間為止的每一天生效，一直到規則到期為止。

  11. 在規則到期日欄位中，以 MM/DD/YYYY 格式輸入到期日。例如，輸入 12/31/2010。規則會在所指定日期的 11:59 pm 到期。如果沒有指定日期，則規則將永遠不會到期。
  12. 在「已啟用規則」欄位中，將預設值保留為「是」，確保這個規則已啟用，或選取「否」，停用這個規則。
  13. 在「警示嚴重程度」欄位中，選取下列其中一項：
    • 正常
    • 警告
    • 次要
    • 主要
    • 臨界
    • 嚴重
  14. 在警示頻率欄位中，選擇下列其中一項設定 (如表 25-20 中所示)。

    15. 表 25-20 警示頻率設定
    特性
    描述
    每次
    如果選取這個選項，則每次警示規則評估為 true 時，就會執行警示規則中的動作。例如，若設定平均回應時間大於 300 毫秒的條件，則每次這個條件評估為 true 時，您就會收到警示。
    警示規則的評估次數是根據與此規則相關的彙總間隔和取樣間隔而定。如果彙總間隔設為 5 分鐘，則取樣間隔為 1 分鐘。每當取得 5 個資料樣本時，就會評估一次規則。因此，規則在建立之後大約 5 分鐘時會做第一次評估，此後就每分鐘評估一次。
    通知一次
    如果選取這個選項，則規則第一次評估為 true 時會執行規則中的動作，然後就不再產生警示，一直到條件本身重設並再次評估為 true 時，才會再產生警示。例如，若設定平均回應時間小於 300 毫秒的條件，則此條件第一次評估為 true 時，您會收到警示，但接下來就不會再收到任何謷示，一直到條件評估為 false 並再次評估為 true 時，才會再收到警示。警示時間戳記會更新並顯示在儀表板上。

  15. 如果您要在與某一服務相關的其中一個規則評估為 true 之後中止執行後續規則，請在停止處理其他規則選項選取是。當特定服務有多個相關的規則時，使用此選項可讓您停止評估後續的規則。將預設值保持為「否」則會繼續處理規則。

    16. 此選項就像 Microsoft Outlook「規則精靈」中的「停止處理其他規則」選項一樣。

完成作業後

按下一頁顯示新增警示規則 - 條件組態頁面。繼續進行定義警示規則條件。
