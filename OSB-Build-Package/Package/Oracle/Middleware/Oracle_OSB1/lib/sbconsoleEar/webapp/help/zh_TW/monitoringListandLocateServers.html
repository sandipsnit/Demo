檢視伺服器資訊

「伺服器狀況」頁面包含下列項目：

  • 在「伺服器狀況」頁籤標頭上，可能會出現「關閉」圖示和顯示已關閉伺服器數目的數值指示器：
    • 按一下圖示時，會使用「Shutdown 狀況」條件和「SHUTDOWN 狀態」條件來篩選顯示的伺服器。
    • 數字指出已關閉伺服器的數目。
  • 描述網域中伺服器狀況的圓餅圖。您可以按一下圖表中與伺服器對應的區域，以顯示其他詳細資訊。
  • 日誌摘要。若要顯示日誌，請按一下顯示指定嚴重程度訊息數的數字連結。例如，警示：1，按一下數字 1。請參閱檢視網域日誌檔的詳細資訊。
  • 根據伺服器摘要表格中的自訂設定，顯示伺服器資訊的欄位。例如，伺服器名稱、狀況、叢集名稱、機器名稱、伺服器狀態和伺服器已執行的時間長度。若要進一步瞭解這些欄位，請參閱表 25-16。
  • 「儀表板重新整理速率」預設為不重新整理。若要變更「儀表板重新整理速率」間隔，請參閱設定使用者偏好設定。
注意： 選取預設值 (不重新整理) 之外的儀表板重新整理速率間隔，瀏覽器就會定期重新整理「儀表板」。這可防止階段作業逾時，即使您未與「Oracle Service Bus 管理主控台」互動也一樣。
注意： Oracle 建議您小心使用「儀表板重新整理速率」功能。請不要將階段作業保留為自動。當您的階段作業在作用中時，其他人就可以從您的機器存取該階段作業，而不需要重新進行認證。
  • 表格自訂器圖示。請參閱自訂表格檢視畫面。
  • 若要限制清單中的項目數或尋找特定伺服器：
    • 按一下開啟圖示，以顯示其他搜尋篩選。如果指定了多個搜尋條件，則只會傳回符合全部條件的結果。
    • 依狀況篩選。從「狀況」下拉清單中，選取將伺服器限制為指定的狀況層級，然後按一下「搜尋」。

      • 選取「或以上」核取方塊，可將搜尋限制在指定的伺服器狀況層級或更高層級 (從最寬鬆到最受限層級依序排列)：正常、警告、超載、臨界及嚴重。

    • 依伺服器篩選。從「伺服器」下拉清單中，選取「全部」或伺服器名稱，然後按一下「搜尋」。
    • 依叢集名稱篩選。從「叢集名稱」下拉清單中，選取「全部」或與伺服器相關的叢集名稱，然後按一下「搜尋」。
    • 依機器名稱篩選。從「機器名稱」下拉清單中，選取「全部」或與伺服器相關的機器名稱，然後按一下「搜尋」。
    • 依狀態篩選。從「狀態」下拉清單中，選取「全部」或伺服器的狀態 (執行中、失敗或關閉)，然後按一下「搜尋」。
    • 按一下「檢視全部」移除搜尋篩選，並顯示所有伺服器。

「伺服器狀況」頁面會根據您所指定的篩選設定，顯示每個伺服器的下列資訊 (如表 25-16 中所示)。

表 25-16 伺服器摘要詳細資訊 
特性
描述
伺服器
伺服器的名稱。此名稱是連到檢視伺服器詳細資訊頁面的連結。請參閱檢視伺服器詳細資訊。
狀況
伺服器的狀況：
  • 正常 - 伺服器正在運作，沒有任何問題。
  • 確定/不處理 - 伺服器處於「管理」狀態，只供進行管理作業。所有執行時期 Oracle Service Bus 訊息都將失敗。
  • 警告 - 伺服器在未來可能會有問題。有關詳細資訊，請檢查伺服器日誌及對應的 RuntimeMBean。
  • 超載 - 指定給伺服器的工作超過設定的重要值；它可能會拒絕更多的工作。
  • 臨界 - 伺服器失敗擱置；必須立即進行某些操作，以防止失敗。有關詳細資訊，請檢查伺服器日誌及對應的 RuntimeMBean。
  • 關閉 - 伺服器失敗，必須重新啟動。
  • 臨界 - 伺服器失敗擱置；必須立即進行某些操作，以防止失敗。有關詳細資訊，請檢查伺服器日誌及對應的 RuntimeMBean。
叢集名稱
與此伺服器相關之叢集的名稱。
機器名稱
與此伺服器相關之機器的名稱。
狀態
伺服器的狀態：
  • 執行中
  • 管理
  • 失敗
  • 關閉
正常執行時間
此伺服器已執行的時間長度。
