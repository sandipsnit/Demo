檢視服務監督資訊

服務狀況頁面會顯示已啟用監控之服務的測量結果。若要瞭解如何啟用服務的監控，請參閱設定代理主機服務的作業設定和設定業務服務的作業設定。

「儀表板重新整理速率」預設為不重新整理。若要變更「儀表板重新整理速率」間隔，請參閱設定使用者偏好設定。

注意： 選取預設值 (不重新整理) 之外的儀表板重新整理速率間隔，瀏覽器就會定期重新整理「儀表板」。這可防止階段作業逾時，即使您未與「Oracle Service Bus 管理主控台」互動也一樣。
注意： Oracle 建議您小心使用「儀表板重新整理速率」功能。請不要將階段作業保留為自動。當您的階段作業在作用中時，其他人就可以從您的機器存取該階段作業，而不需要重新進行認證。

在「服務狀況」頁籤標頭上，可能會出現這些圖示：

  • 「警告」圖示和數值指示器。
    • 按一下圖示時，會篩選使用「含有錯誤」條件顯示的服務。
    • 數字指出自上次重新整理頁面後發生錯誤的服務數目。
  • 「端點離線」圖示和數值指示器。
    • 按一下圖示時，會篩選使用「含有離線端點 URI」條件顯示的服務。
    • 數字指出自上次重新整理頁面後端點離線的服務數目。

「服務狀況」頁面顯示服務測量結果和監督資訊 (如表 25-5 中所示)。

  1. 在「顯示統計資訊」欄位中，請執行下列其中一項操作：
    • 若要顯示每一服務在目前彙總間隔期間的監控統計資訊，請選取目前的彙總間隔。「目前彙總間隔」檢視畫面會顯示服務測量結果的動態統計檢視畫面。
    • 若要顯示自從上次重設服務統計資訊期間的監控統計資訊，請選取上次重設之後。「上次重設之後」檢視畫面顯示測量結果的執行中計數。
  2. 從下拉清單中選取「伺服器」，以顯示該伺服器的量測。

    3. 如果叢集存在，預設會顯示叢集範圍的量測。選取個別的「受管理伺服器」以顯示該伺服器的量測。

  3. 若要限制清單中的項目數或尋找特定服務：
    • 按一下開啟圖示，以顯示其他搜尋篩選。如果指定了多個搜尋條件，則只會傳回符合全部條件的結果。
    • 依服務名稱篩選。在「名稱」欄位中，輸入搜尋目標的名稱，或輸入萬用字元 (使用 * 及 ? 當成萬用字元，以執行更廣泛的搜尋)，然後按一下「搜尋」。

      • 如果服務名稱在所有專案及路徑中都是唯一的，表示此搜尋方法是慣用的方法。

    • 依服務路徑篩選。在「路徑」欄位中輸入搜尋目標的路徑，此為專案名稱以及服務所在的資料夾名稱。您可以使用 * 和 ? 當成萬用字元，以執行更廣泛的搜尋。將顯示所有位於該路徑的服務。

      • 「路徑」欄位的格式如下：
      project-name/root-folder/ . . ./parent-folder

      如果服務就在專案的下一層，則格式為：project-name

    • 依服務名稱及路徑篩選。如果有多個名稱相同的服務位於不同的路徑，表示此搜尋方法是慣用的方法。
    • 依父項服務篩選。從服務下拉清單中選取父項服務類型：「代理主機服務」或「業務服務」，然後按一下搜尋。選擇「所有服務」忽略父項服務篩選條件。
    • 依具有訊息的服務篩選。選取「含有訊息」選項，然後按一下「搜尋」。
    • 依具有警示的服務篩選。選取「含有警示」選項，然後按一下「搜尋」。
    • 依具有錯誤的服務篩選。選取「含有錯誤」選項，然後按一下「搜尋」。
    • 依具有離線 URI 的服務篩選。選取「含有離線端點 URI」選項，然後按一下「搜尋」。
    • 依特定代理主機服務所叫用的服務篩選。在提供的欄位中輸入代理主機服務的名稱和路徑，或按一下瀏覽顯示選取代理主機服務頁面。從清單中選取代理主機服務，然後按一下提交。按一下搜尋。將顯示您指定的代理主機服務所叫用的業務服務。

      • 「由代理伺服器叫用」欄位中不支援萬用字元。

    • 按一下「檢視全部」移除搜尋篩選，並顯示所有服務。

      特性
      描述
      名稱
      指派給服務的名稱。此名稱是連到服務監控詳細資訊頁面的連結。請參閱檢視服務測量結果。
      路徑
      與服務關聯的專案。若是服務位於專案資料夾內，則也會列出該資料夾。顯示的路徑格式如下：
      project-name/root-folder/ . . ./parent-folder
      此路徑是連到「專案瀏覽器」中對應路徑的連結。
      服務類型
      父項服務類型：代理主機服務或業務服務。
      彙總間隔
      只有當選取「顯示統計資訊」欄位中的「目前的彙總間隔」時，才會顯示此欄位。
      為了監控此服務而設定的目前彙總間隔，以小時及分鐘表示。您可以在作業設定頁面中設定此間隔。請參閱設定服務的彙總間隔。
      平均回應時間
      此服務在目前彙總間隔期間內或在上次重設後期間執行訊息所需的平均回應時間 (以毫秒為單位)。
      訊息
      在目前彙總間隔期間或在上次重設後期間與此服務相關的訊息數目。
      錯誤
      在目前彙總間隔期間或在上次重設後期間與此服務相關的錯誤訊息數目。
      警示
      此欄位預設為隱藏。
      在目前彙總間隔期間或上次重設之後的這段期間，與此服務相關的警示數 (SLA 和管線)。
      如果是叢集中用於代理主機服務的個別「受管理伺服器」，則「只」會顯示在目前彙總間隔或自上次重設之後的這段期間，與此服務相關的管線警示數。
      用於業務服務的個別「受管理伺服器」，則會顯示 N/A。
      SLA 警示
      在目前彙總間隔期間或在上次重設後期間與此服務相關的 SLA 警示數目。
      如果是叢集中的個別「受管理伺服器」，則「SLA 警示」會是「N/A」(只會顯示叢集範圍 SLA 警示)。
      管線警示
      僅適用於代理主機服務：
      在目前彙總間隔期間或在上次重設後期間與此服務相關的管線警示數目。
      端點 URI 狀態
      僅限業務服務，端點 URI 的狀態：此狀態是連到「端點 URI」頁面的連結。請參閱檢視業務服務端點 URI 量測。
      適用於單一節點網域 (個別伺服器執行處理)：
      • 線上 - 業務服務的所有端點 URI 都是上線狀態。
      • 離線 - 業務服務的所有端點 URI 都是離線狀態。
      • 部分 - 業務服務至少有一個端點 URI 是離線狀態。
      適用於叢集網域：
      • 「伺服器」欄位設定為「叢集」或其中一台受管理伺服器時，「線上」狀態分別表示所有端點 URI 在整個叢集中或選定受管理伺服器上都是上線狀態。
      • 「伺服器」欄位設定為「叢集」或其中一台受管理伺服器時，「離線」狀態分別表示所有端點 URI 在整個叢集中或選定受管理伺服器上都是離線狀態。
      • 「伺服器」欄位設定為「叢集」時，「部分」狀態表示業務服務至少有一個端點 URI 在至少一台伺服器上是離線狀態，或是有一個端點 URI 在所有伺服器上都是離線狀態，但是相同業務服務的其他端點 URI 在一台或所有伺服器上仍然可用。
      • 「伺服器」欄位設定為其中一台受管理伺服器時，「部分」狀態表示業務服務至少有一個端點 URI 在選定受管理伺服器上是離線狀態。
      動作
      只有當選取「顯示統計資訊」欄位中的「上次重設之後」時，才會顯示此欄位。
      在此欄中，您可以按一下特定服務的重設圖示，以重設該服務的統計資訊。確認要這樣做時，系統會刪除上次重設統計資訊後，針對服務所收集的所有監控統計資訊。不過，系統並不會刪除在服務的「目前的彙總間隔」期間所收集的統計資訊。此外，在您按一下重設圖示之後，系統會立即為服務重新開始收集監控統計資訊。
