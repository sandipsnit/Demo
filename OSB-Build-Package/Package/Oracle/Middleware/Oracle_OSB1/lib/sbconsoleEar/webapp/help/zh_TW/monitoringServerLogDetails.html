檢視網域日誌檔的詳細資訊

使用此頁面可檢視網域日誌檔項目的詳細資訊，如 表 25-18 中所示。

表 25-18 網域日誌詳細資訊
特性
描述
訊息
事件或條件的描述。
Date
顯示建立訊息的時間及日期，以語言地區特有的格式表示。執行每一個 WebLogic Server 執行處理的「Java 虛擬機器 (JVM)」會參照主機機器作業系統，以取得本機時區及格式的相關資訊。
子系統
指出屬於訊息來源的 WebLogic Server 的子系統；例如，Enterprise Java Bean (EJB) 容器或 Java Messaging Service (JMS)。
訊息 ID
一個唯一的六位數識別碼。
WebLogic Server 系統訊息產生的所有訊息 ID 都是以 BEA- 開始並落在數字範圍 0-499999 內。
嚴重程度
指出訊息所報告之事件的影響或嚴重程度：
  • 警示 - 特殊服務處於不穩定狀態，但系統的其他部分則繼續運作。無法自動恢復；需要管理員立即協助，才能解決問題。
  • 臨界 - 發生系統或服務錯誤。系統可以恢復，但是可能會暫時失去服務，或導致服務永久降級。
  • 緊急 - 伺服器處於不穩定狀態。此嚴重程度表示系統故障或不穩定。
  • 錯誤 - 發生使用者錯誤。系統或應用程式可處理該錯誤而不中斷服務，但是會有部分服務降級。
  • 資訊 - 用於報告正常作業；低階參考訊息。
  • 注意 - 具有高階重要性的參考訊息。
  • 警告 - 發生可疑的作業或組態，但可能不會影響正常作業。
機器
主控伺服器執行處理之電腦的 DNS 名稱。
伺服器
產生訊息之 WebLogic Server 執行處理的名稱。
繫線
JVM 指派給繫線 (建立訊息的來源) 的 ID。
使用者 ID
執行相關事件時所依據的使用者 ID。
交易 ID
只呈現給交易相關資訊環境內記錄的訊息。
相關資訊環境 ID
使來自特定要求或應用程式的訊息相互關聯的相關資訊環境資訊。
詳細資訊
事件或條件的描述。
原因
訊息的原因。
動作
應該採取的動作。
