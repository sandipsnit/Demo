自訂 WS 原則

注意： 在未來的 Oracle Service Bus 版本中，Oracle Web Services Manager 原則會取代 WLS 9.2 原則。雖然此版本的 Oracle Service Bus 會繼續支援 WLS 9.2 原則，不過您應該在建立新服務時，考慮使用 Oracle Web Services Manager 原則，以避免將來不再支援 WLS 9.2 原則的情況。如需詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Securing Oracle Service Bus with Oracle Web Services Manager"，網址為：http://www.oracle.com/pls/as1111/lookup?id=OSBDV1681。

「Web Service 原則架構」(WS 原則) 是可延伸的 XML 型架構，它可以利用網域特有的安全性聲明來延伸 Web 服務的組態，以及指定 Web Service 的安全性需求、預期及功能。在 Oracle Service Bus 中，「WS 原則」的其中一個主要用途，就是在代理主機服務和業務服務中設定訊息層級安全性。

使用 Oracle Service Bus WS 原則

Oracle Service Bus 包含三個簡易「WS 原則」檔案，您可以使用這些檔案來要求從屬端授權、數位加密或數位簽署 SOAP 訊息：Auth.xml、Encrypt.xml 和 Sign.xml。Oracle 建議除非您有特定的安全需求，否則儘可能使用這些預先包裝的檔案。

如需使用這些原則的詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「Using WS-Policy in Oracle Service Bus Proxy and Business Services」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599。

使用自訂 WS 原則

如果「Oracle Service Bus WS 原則」敘述不符合您的安全需求，則可以撰寫自己的「WS 原則」(自訂「WS 原則」)、將它們匯入至 Oracle Service Bus，然後從 WSDL 參照它們(「Oracle Service Bus WS 原則」敘述是唯讀敘述)。

如需建立及參考自訂 WS 原則的相關資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Using WS-Policy in Oracle Service Bus Proxy and Business Services"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1599。
