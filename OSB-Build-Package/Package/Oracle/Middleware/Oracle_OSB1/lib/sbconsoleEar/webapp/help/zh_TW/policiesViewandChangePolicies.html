編輯自訂 WS 原則

使用檢視 WS-Policy 詳細資訊頁面可以檢視及變更特定自訂「Web 服務原則」(WS-Policy) 的詳細資訊。「Oracle Service Bus 管理主控台」不會顯示 Oracle Service Bus WS-Policy，因為它們是唯讀的。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找「WS 原則」(如尋找自訂 WS 原則中所述)。
  3. 按一下 WS 原則名稱。「檢視 WS 原則詳細資訊」頁面會顯示表 11-2 中所示的資訊。

    4. 表 11-2 WS 原則詳細資訊 
    特性
    描述
    上次修改者
    建立此 WS 原則或將其匯入至組態中的使用者。
    上次修改時間
    使用者建立此 WS 原則或將其匯入組態的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱檢視變更歷史記錄頁面。
    參照
    此 WS 原則參照的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    參照者
    參照此 WS 原則的物件數。如果有這類參照，請按一下數字連結 以檢視物件清單。請參閱檢視資源參照。
    描述
    此 WS 原則的描述 (如果有的話)。
    原則
    此「WS 原則」的文字。

  4. 若要變更欄位，請按一下編輯。有關欄位的描述，請參閱新增自訂 WS 原則。

    5. 您不可以變更「資源名稱」欄位。

  5. 按一下「儲存」，確認目前階段作業中的更新。
  6. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
