使用專案、資料夾和資源

「專案瀏覽器」會將組態和資源組織至專案和資料夾中。在「專案瀏覽器」中，您可以顯示專案檢視畫面頁面 (列出網域中的所有專案) 和專案/資料夾檢視畫面頁面 (列出每個專案中的資料夾和資源)。

專案是沒有階層、沒有重疊的最上層分組建構。

所有 Oracle Service Bus 資源 (例如，服務、WS 原則、WSDL、XQuery 轉換等) 都只位於一個專案中。專案不會重疊。您可以直接在專案下建立資源，或進一步將它們組織成資料夾。不過，無論是否有任何資源位於專案中，您都可以參照該資源。

在 Oracle Service Bus 中建立網域時，會建立預設專案。

命名專案和資料夾

專案名稱和資料夾名稱的限制為 64 個字元，而且必須遵循資源命名限制。

巢狀專案及資料夾的名稱、長度及層級最終會受到您的作業系統限制的影響。由於作業系統的限制，您無法建立名稱非常長的資料夾或專案，或巢狀很深的資料夾。

使用專案和資料夾限定資源名稱

專案和資料夾限定 Oracle Service Bus 資源的名稱。資料的參照建構如下：

project-name/root-folder/. . ./parent-folder/resource-name

如果資源正好位於專案之下，則參照建構如下：

project-name/resource-name
