資源類型和副檔名
Zip 檔案中的每一個檔案都是可能的資源，而且最常使用副檔名來識別檔案類型。Oracle Service Bus 會定義每一個資源的預設副檔名，但是如果您有其他的副檔名來說明那些資源，則此對應可讓您對載入器宣告那些副檔名。
在您定義 Zip 檔案和副檔名對應之後，Oracle Service Bus 載入器會掃描 Zip 檔案的內容，然後嘗試使資源類型與每一個檔案產生關聯。沒有副檔名的檔案，或對應中沒有定義的副檔名的檔案，都會被視為「不明」，因此載入時會自動排除它。
若為已知的檔案，資源名稱即是沒有副檔名的檔案名稱。將在目標「Oracle Service Bus 管理主控台」資料夾中重新建立 Zip 檔案的資料夾結構。
