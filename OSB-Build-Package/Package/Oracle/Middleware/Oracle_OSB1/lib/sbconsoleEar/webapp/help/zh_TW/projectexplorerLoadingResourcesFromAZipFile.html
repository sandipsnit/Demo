從 Zip 檔案載入資源

您可以選取含有您要載入之資源的 Zip 或 JAR 檔案；Oracle Service Bus 會從此 Zip 檔案尋找並匯入資源。

WSDL 或 XML 綱要這類資源可以定義完整相依資源樹狀圖。大量載入功能可讓您一次上傳一組資源，不需要手動解析相依性。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取專案瀏覽器，然後選取要在其中新增資源的專案或資料夾。隨即顯示專案/資料夾檢視畫面頁面。
  3. 從建立資源下拉清單中，選取壓縮的資源。隨即顯示載入資源頁面。
  4. 在選取 Zip 檔案欄位中，指定要匯入的 Zip 檔案。輸入檔案名稱 (以其路徑限定)；或按一下瀏覽，找到檔案，然後按一下開啟。
  5. 檢查資源類型與副檔名的對應。根據需要，變更或新增副檔名。請利用逗號隔開多個副檔名。

    6. Zip 檔案中的每一個檔案都是可能的資源，而且最常使用副檔名來識別檔案類型。Oracle Service Bus 會定義每一個資源的預設副檔名，但是如果您有其他的副檔名來說明那些資源，則此對應可讓您對載入器宣告那些副檔名。

    在您定義 Zip 檔案和副檔名對應之後，Oracle Service Bus 載入器會掃描 Zip 檔案的內容，然後嘗試使資源類型與每一個檔案產生關聯。沒有副檔名的檔案，或對應中沒有定義副檔名的檔案，都會被視為「不明」，因此載入時會自動排除它。

    若為已知的檔案，資源名稱即是沒有副檔名的檔案名稱。將在目標「Oracle Service Bus 管理主控台」資料夾中重新建立 Zip 檔案的資料夾結構。

    副檔名只能與一個檔案類型相關。

  6. 按一下「下一步」。
  7. 在載入資源 - 檢查已載入的資源頁面上，檢查所匯入的資源。您可以清除任何不想要併入此匯入之資源旁邊的核取方塊。
  8. 若要匯入選取的資源，請按一下「匯入」。
  9. 載入資源 - 匯入結果頁面會顯示匯入的資源。按一下「載入另一個」，從另一個檔案匯入資源，或從左瀏覽窗格選取不同頁面，繼續進行其他工作。
  10. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
