服務金鑰提供者

服務金鑰提供者包含公用金鑰基礎結構 (PKI) 證明資料，而代理主機服務會將該證明資料用於解密內送 SOAP 訊息，以及用於外送認證和數位簽章。PKI 證明資料是與憑組配組的私密金鑰，可用於數位簽章和加密 (Web 服務安全)，以及用於內送 SSL 認證。憑證包含對應至私密金鑰的公用金鑰。

注意： 若要使用服務金鑰提供者，您必須設定 PKI 證明資料對應提供者。請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Configuring the WebLogic Security Framework: Main Steps"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1498。

單一服務金鑰提供者可以包含下列所有 PKI 證明資料：

  • 數位加密的金鑰組

    • 代理主機服務使用此金鑰組，將遵照 Web 服務原則聲明加密的內送 SOAP 訊息解密。如果您希望服務金鑰提供者支援數位加密，與 PKI 證明資料對應器相關的金鑰儲存庫至少必須包含一個支援加密的 X.509 憑證。

  • 數位簽章的金鑰組

    • 代理主機服務的端點若是 Web 服務，而且 Web Service 要求從屬端簽署 SOAP 信封的一個或多個部分，代理主機服務就會使用此金鑰組。

  • SSL 從屬端認證金鑰組 (雙向 SSL)

    • 代理主機服務在外送 TLS/SSL (安全通訊端層) 連線期間做為從屬端時，會使用此金鑰組進行認證；也就是，將訊息路由至要求從屬端憑證認證的 HTTPS 業務服務或代理主機服務時。

多個代理主機服務可以使用同一個服務金鑰提供者。
