新增服務金鑰提供者

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取專案瀏覽器，然後選取要在其中新增服務金鑰提供者的專案或資料夾。隨即顯示專案/資料夾檢視畫面頁面。
  3. 從「建立資源」下拉清單中選取「服務金鑰提供者」，以顯示「建立新的服務金鑰提供者」頁面。
  4. 在服務金鑰提供者名稱欄位中，輸入此服務金鑰提供者的唯一名稱。

    請遵循資源命名限制的命名指南。
  5. 在「描述」欄位中，輸入服務金鑰提供者的描述。
  6. 執行下列任一步驟 (如表 17-2 中所示)。

    7. 表 17-2 認證選項
    針對此新增金鑰組...
    請完成這些步驟...
    數位加密
    1. 按一下「加密金鑰」旁邊的「瀏覽」。

      2. 「選取加密金鑰的別名」視窗會顯示您範圍中 PKI 證明資料對應器使用之金鑰儲存庫中金鑰的別名。

    2. 在「選取加密金鑰的別名」視窗中，輸入您用於保護金鑰儲存庫存取安全的密碼 (您在建立金鑰庫時設定密碼)。
    3. 選取對應至 X.509 憑證並支援加密的金鑰別名。
    4. 按一下「提交」。
    您將此服務金鑰提供者與代理主機服務關聯時，Oracle Service Bus 會將 X.509 憑證內嵌入代理主機服務的 WSDL。然後代理主機服務使用此憑證，加密其傳送至端點的訊息。代理主機服務使用 PKI 證明資料中的私密金鑰，解密端點傳回的訊息。
    數位簽章
    1. 按一下「數位簽章金鑰」旁邊的「瀏覽」。

      2. 「選取數位簽章金鑰的別名」視窗會顯示您範圍中 PKI 證明資料對應器使用之金鑰儲存庫中的金鑰別名。

    2. 在「選取數位簽章金鑰的別名」視窗中，輸入您用於保護金鑰儲存庫存取安全的密碼 (您在建立金鑰庫時設定密碼)。
    3. 選取金鑰別名。
    4. 按一下「提交」。
    SSL 從屬端認證 (雙向 SSL)
    1. 按一下「SSL 從屬端認證金鑰」旁邊的「瀏覽」。

      2. 「選取 SSL 從屬端認證金鑰的別名」視窗會顯示您範圍中 PKI 證明資料對應器使用之金鑰儲存庫中的金鑰別名。

    2. 在「選取 SSL 從屬端認證金鑰的別名」視窗中，輸入您用於保護金鑰儲存庫存取安全的密碼 (您在建立金鑰庫時設定密碼)。
    3. 選取金鑰別名。
    4. 按一下「提交」。

  7. 按一下「儲存」。服務金鑰提供者會儲存在目前的階段作業中。
  8. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
