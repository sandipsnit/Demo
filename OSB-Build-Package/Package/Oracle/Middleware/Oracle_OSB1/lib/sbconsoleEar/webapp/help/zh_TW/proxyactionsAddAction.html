新增和編輯訊息流程的動作

動作是管線階段、錯誤處理程式階段、路由節點和分支節點的元素，可定義當訊息流經代理主機服務時要如何定義訊息。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯訊息流程 (如建立、編輯和檢視訊息流程中所述)。

也假設您已新增管線階段、路由節點和 (或) 錯誤處理程式階段。請參閱：

  • 將管線組節點新增至訊息流程
  • 將階段新增至管線
  • 新增代理主機服務錯誤處理程式

若要將動作新增至訊息流程

  1. 選取想要新增動作的元件。例如，按一下「階段」圖示，然後按一下「編輯階段」，或按一下「路由節點」圖示，然後按一下「編輯路由」。
  2. 並根據動作是新增至階段或路由節點，執行下列其中一項操作：
    • 如果尚未新增動作，編輯階段組態頁面只會顯示新增動作圖示。按一下該圖示，然後選取動作類型。
    • 如果已新增一或多個動作，則「編輯階段組態」頁面會顯示一或多個代表這些動作的圖示，例如，「發佈」圖示或「路由」圖示等。按一下適當的圖示，並按一下「新增動作」，然後選取動作類型。
    • 部分動作 (例如，發佈動作中的要求和回應動作) 包含適當動作的新增動作連結。按一下該圖示，然後選取動作類型。

      • 在訊息流程中，並未限制動作的鏈結。

  3. 當您完成動作的新增時，可以進一步設定階段或路由節點中的動作 (如表 21-2 中所述)。

    4. 表 21-2 編輯階段組態工作 
    若要...
    請完成此步驟...
    刪除動作
    按一下適當的圖示，然後按一下「刪除此動作」。
    將動作下移 (後移)
    按一下適當的圖示，然後按一下下移動作。動作會移至此階段所含的下一個動作下方。
    只有在階段含有兩個以上的動作時，才會顯示此選項。
    將動作上移 (前移)
    按一下適當的圖示，然後按一下上移動作。動作會移至此階段所含的上一個動作上方。

    只有在階段含有兩個以上的動作時，才會顯示此選項。

    剪下動作
    按一下適當的圖示，然後按一下「剪下」。
    複製動作
    按一下適當的圖示，然後按一下「複製」。
    貼上已剪下或複製的動作
    按一下適當的圖示，然後按一下「貼上動作」。
    您可以跨階段複製和貼上動作。不過，如果是「指派」、「取代」或「插入」動作，請注意下列幾點：
    • 來源 (複製) 階段中所有變數相關和使用者定義的命名空間，在目標 (貼上) 階段中都會新增為使用者定義命名空間。
    • 不會複製重複的命名空間 (在來源和目標階段中都相同的命名空間)。
    • 會複製衝突的命名空間 (字首相同但是 URI 不同的命名空間宣告)。使用者將能儲存組態，但在移除階段 B 中衝突的命名空間宣告之前，無法啟動組態。
    驗證階段
    在「編輯階段組態」頁面中，按一下「驗證」以驗證該階段中設定的所有動作。

  4. 按一下「儲存」，確認目前階段作業中的更新。
  5. 在「編輯訊息流程」頁面上，繼續建構訊息流程 (如建立、編輯和檢視訊息流程中所述)。
  6. 按一下「儲存」，確認目前階段作業中的更新。
  7. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
