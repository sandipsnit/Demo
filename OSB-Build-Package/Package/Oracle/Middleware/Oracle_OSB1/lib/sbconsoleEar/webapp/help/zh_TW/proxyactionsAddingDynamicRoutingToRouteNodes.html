將動態路由新增至路由節點

根據 XQuery 資源中的可用路由資訊，指派訊息的路由。

這是終止動作，表示您無法在此動作後新增另一個動作。然而，此動作可以包含要求和回應動作。

如需路由的詳細資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 Modeling Message Flow in Oracle Service Bus，網址為 http://www.oracle.com/pls/as111170/lookup?id=OSBAG181。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要將動態路由新增至路由節點

  1. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  2. 按一下「路由節點」圖示，然後按一下「編輯路由」，隨即顯示「編輯階段組態」頁面。
  3. 按一下「新增動作」圖示，然後選取「通訊」>「動態路由」。
  4. 按一下「<表示式>」。隨即顯示 XQuery 表示式編輯器。
  5. 在「XQuery 表示式編輯器」中，輸入 XQuery 表示式，而其結果與下列類似：

    6. <ctx:route>

        <ctx:service isProxy='true'>{$service}</ctx:service>

        <ctx:operation>{$operation}</ctx:operation>

    </ctx:route>

    注意： 如果正在呼叫代理主機服務，應將 isProxy 屬性設定為 true。如果正在呼叫業務服務，請將 isProxy 設定為 false。
    注意： - 服務名稱是完整服務名稱。
    注意： - 這是選擇性作業元素。
  6. 按一下「儲存」。
  7. 在要求動作欄位中，按一下新增動作以新增動作，然後選取想要與服務相關聯的動作。您可以新增多個動作。若要進一步瞭解想要新增之動作類型的相關資訊，請參閱新增和編輯訊息流程的動作中的動作表格。
  8. 在回應動作欄位中，按一下新增動作以新增動作，然後選取想要與服務相關聯的動作。您可以新增多個動作。若要進一步瞭解想要新增之動作類型的相關資訊，請參閱新增和編輯訊息流程的動作中的動作表格。
  9. 按一下「儲存」。
  10. 在「編輯訊息流程」頁面上，繼續建構訊息流程 (如建立、編輯和檢視訊息流程中所述)。
  11. 按一下「儲存」，確認目前階段作業中的更新。
  12. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
