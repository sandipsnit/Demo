新增警示動作

使用警示動作以根據管線中的訊息相關資訊環境產生警示，並將警示傳送至警示目標。和 SLA 警示不同，警示動作所產生的通知主要是針對商業用途，或是為了報告錯誤，而不是用來監督系統狀況。在設定和選擇警示目標時應牢記這一點。若要進一步瞭解警示目標的相關資訊，請參閱警示目標。

如果未針對服務啟用管線警示或未在網域層級啟用管線警示，則會在訊息處理期間略過設定的警示動作。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要新增警示動作

  1. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  2. 按一下適當的圖示，然後選取「新增動作」>「報告」>「警示」。
  3. 按一下 <目標>。隨即顯示選取警示目標頁面。從清單中選取需要的警示目標，然後按一下提交。

    4. 依照預設，警示一律會傳送至「Oracle Service Bus 管理主控台」。

  4. 按一下「<表示式>」。隨即顯示 XQuery 表示式編輯器頁面。您可以透過相關資訊環境變數上的 XQuery 表示式，指定要新增至警示訊息的訊息相關資訊環境。請參閱建立和編輯內置 XQuery 和 XPath 表示式。
  5. 在警示摘要欄位中，輸入警示的簡短描述。這將成為電子郵件通知的主旨，並且可包含 80 個以內的字元。如果未提供任何描述，則會改用預先定義的主旨：「Oracle Service Bus Alert 警示」。
  6. 在「嚴重度層級」下拉清單裡，從下列選項中選取此警示的嚴重度層級：「正常」、「警告」、「次要」、「主要」、「臨界」和「嚴重」。

完成作業後

完成此動作的組態時，請繼續設定其他動作，或儲存組態 (如新增和編輯訊息流程的動作中所述)。
