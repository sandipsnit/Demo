新增日誌動作

使用日誌動作可以建構要記錄的訊息，以及定義連同訊息一起記錄的屬性集。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯訊息流程 (如建立、編輯和檢視訊息流程中所述)。

注意：若要查看日誌檔或標準輸出 (伺服器主控台) 中的日誌資料，必須將 Oracle WebLogic Server 記錄設定成下列嚴重程度層級：

要記錄的最低嚴重程度：資訊
日誌檔：資訊
標準輸出：資訊

如需設定日誌檔嚴重程度層級的資訊，請參閱 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的 Using Log Severity Levels，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

 

若要新增日誌動作

  1. 確定已啟用全域記錄。如需詳細資訊，請參閱啟用全域記錄。
  2. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  3. 按一下適當的圖示，然後選取「新增動作」>「報告」>「日誌」。
  4. 按一下 <表示式>。隨即顯示 XQuery 表示式編輯器頁面。您可以指定要透過相關資訊環境變數上的 XQuery 表示式來記錄的訊息相關資訊環境。請參閱建立和編輯內置 XQuery 和 XPath 表示式。
  5. 在註解欄位中，輸入此日誌動作的註解。這些註解會連同先前定義之表示式的結果一起記錄。
  6. 選取「嚴重度層級」下拉清單中的其中一個選項。

    7. 表 21-5 日誌動作嚴重度層級
    嚴重度層級
    典型用法
    資訊
    用於報告正常作業；低層級參考訊息。
    警告
    發生可疑的作業或組態，但不影響正常作業。
    錯誤
    發生使用者錯誤。系統或應用程式可處理該錯誤而不中斷服務，但是會有部分服務降級。
    除錯
    當應用程式仍在開發階段時，您可能會發現，在應用程式內建立及使用訊息來提供低層級活動的完整描述是很有幫助的。
    確定代理主機服務作業設定中的「日誌」嚴重程度層級與日誌動作嚴重程度層級相同。如需代理主機服務作業設定詳細資訊，請參閱設定代理主機服務的作業設定。

完成作業後

完成此動作的組態時，請繼續設定其他動作，或儲存組態 (如新增和編輯訊息流程的動作中所述)。
