新增發佈表格動作

使用發佈表格動作，可以將訊息發佈至零個或多個靜態指定的服務。切換型條件邏輯可用於確定要在執行時期使用哪些服務進行發佈。

如需發佈行為的詳細資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 Performing Transformations in Message Flows，網址為 http://www.oracle.com/pls/as111170/lookup?id=OSBAG205。

  1. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  2. 選取「新增動作」>「通訊」>「發佈表格」。
  3. 按一下「<表示式>」。隨即顯示 XQuery 表示式編輯器頁面。建立 XQuery 表示式，此表示式在執行時期會傳回路由決定所依據的值。請參閱建立和編輯內置 XQuery 和 XPath 表示式。
  4. 從「運算子」清單中，選取比較運算子。然後，在鄰近欄位中，輸入用來評估 XQuery 表示式所傳回值的值。
  5. 按一下<服務>以選擇服務，如果表示式將您指定的值評估為 true，則會將訊息發佈給此服務。隨即顯示選擇服務頁面。
  6. 從清單中選擇一個服務，然後按一下「提交」。這是訊息的目標服務。
  7. 如果服務已定義作業，則可以從「叫用」清單中選取作業來指定要叫用的作業。
  8. 如果您想要外送的作業和內送的作業一樣，請選取「使用內送作業進行外送」核取方塊。
  9. 在要求動作欄位中，若要設定如何封裝訊息並傳送給服務，請按一下新增動作，然後選取一或多個要與服務產生關聯的動作。若要進一步瞭解想要新增之動作類型的相關資訊，請參閱新增和編輯訊息流程的動作。
    10. 注意： 階段編輯器中有四個累計層級的巢狀限制。如果嘗試新增第五層，則不會顯示巢狀動作。累計層級包含所有分支動作：if... then... 條件、發佈表格和路由表格。例如，您可以建立 2 層條件，然後再建立一個含有路由表格的發佈表格，這樣總共就有 4 層。如果您嘗試新增另一個條件 (至最後一個發佈表格)，將不會顯示條件。
  10. 若要插入新的 Case，請按一下「Case」圖示，然後選取「插入新的 Case」。
  11. 針對新的 Case 重複步驟 4-8。
  12. 新增您的商業邏輯所指定的其他 Case。
  13. 按一下您在序列中定義之最後一個 Case 的「Case」圖示，然後選取「插入預設 Case」，在尾端新增預設 Case。
  14. 設定預設 Case - 如果未滿足先前任一個 Case，此 Case 的組態會指定路由行為。

完成作業後

完成此動作的組態時，請繼續設定其他動作，或儲存組態 (如新增和編輯訊息流程的動作中所述)。
