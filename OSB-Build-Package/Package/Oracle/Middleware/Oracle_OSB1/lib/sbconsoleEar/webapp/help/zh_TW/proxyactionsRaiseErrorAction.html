新增發出錯誤動作

使用發出錯誤動作來發出異常狀況，其中含有指定的錯誤碼 (字串) 和描述。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要新增發出錯誤動作

  1. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  2. 按一下適當的圖示，然後選取「新增動作」>「流程控制」>「發出錯誤」。
  3. 在「錯誤碼」欄位中，輸入您要發出的錯誤碼。
  4. 在「錯誤訊息」欄位中，輸入錯誤碼的描述。

完成作業後

完成此動作的組態時，請繼續設定其他動作，或儲存組態 (如新增和編輯訊息流程的動作中所述)。

交易

如果服務是交易服務，則觸發的「發出錯誤」動作會中止要求 (非同步) 或是要求或回應 (同步) 中的交易。例如，您可以自我檢查訊息，並決定即使未發生 SOAP 錯誤還是應該發生「發出錯誤」動作的條件，而「發出錯誤」會導致交易中斷。
