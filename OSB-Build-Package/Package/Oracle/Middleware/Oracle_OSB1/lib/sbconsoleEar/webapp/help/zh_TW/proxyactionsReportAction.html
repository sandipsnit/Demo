新增報告動作

使用報告動作，可以啟用代理主機服務的訊息報告。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要新增報告動作

  1. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  2. 按一下適當的圖示，然後選取「新增動作」>「報告」>「報告」。
  3. 按一下「<表示式>」。隨即顯示 XQuery 表示式編輯器頁面。請參閱建立和編輯內置 XQuery 和 XPath 表示式。XQuery 表示式可用來建立要在 Oracle Service Bus 儀表板報告的資料。
  4. 當您完成編輯 XQuery 表示式時，請按一下新增關鍵字。即會顯示兩個欄位：關鍵字名稱欄位和關鍵字值欄位，其中包括 XPath 連結，您可以按一下此連結來編輯 XPath 表示式，並且可以在變數欄位中輸入相關資訊環境變數。

    5. 您可以使用關鍵字/值組，從訊息相關資訊環境變數或訊息有效負載中擷取關鍵字 ID，並忽略其餘訊息。關鍵字是識別訊息的捷徑。它們會在報告模組中顯示為報告索引。請參閱檢視報告訊息和檢視訊息詳細資訊。

    1. 在「關鍵字名稱」欄位中輸入關鍵字名稱。
    2. 按一下 <XPath>。隨即顯示編輯 XPath 表示式頁面。請參閱建立和編輯內置 XQuery 和 XPath 表示式。
    3. 在變數欄位中輸入相關資訊環境變數。若要進一步瞭解相關資訊環境變數的相關資訊，請參閱附錄 A 訊息相關資訊環境。
    4. 若要新增更多關鍵字值，請按一下關鍵字圖示，然後選取新增關鍵字。若要刪除關鍵字，請按一下關鍵字圖示，然後選取刪除此關鍵字。

例如，假設階段的錯誤處理程式上已設定報告動作。此動作會在發生錯誤時報告 fault 相關資訊環境變數的內容。報告動作設定如下：

  • 關鍵字名稱 = errorCode
  • 關鍵字值 = ./ctx:errorCode (於變數 fault 中)

完成作業後

完成此動作的組態時，請繼續設定其他動作，或儲存組態 (如新增和編輯訊息流程的動作中所述)。
