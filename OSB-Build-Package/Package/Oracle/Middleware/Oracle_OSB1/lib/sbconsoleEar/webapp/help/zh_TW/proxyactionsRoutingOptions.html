新增路由選項動作

「路由選項」動作可以用來修改 $outbound 中之外送要求的下列任何一項或全部特性：URI、服務品質、模式、重試參數。雖然可以在 $outbound 上使用「指派」、「插入」、「取代」或「刪除」動作來修改這些特性，但是使用「路由」選項會提供較簡單的方法來執行此工作，而不需要知道 $outbound 相關資訊環境變數的 XPath、XQuery 或結構。

「路由選項」動作只可以在相關資訊環境變數 $outbound 有效的地方使用。它可以新增至下列動作：

  • 發佈
  • 動態發佈
  • 發佈表格
  • 服務外部呼叫
  • 路由
  • 動態路由
  • 路由表格

如需路由的詳細資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 Modeling Message Flow in Oracle Service Bus，網址為 http://www.oracle.com/pls/as111170/lookup?id=OSBAG181。

 

若要設定路由選項動作：

  1. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  2. 按一下適當的圖示，然後選取「新增動作」>「通訊」>「路由選項」。
  3. 完成下列任何或全部步驟：
    • 若要設定外送訊息的 URI：選取 URI，然後按一下 XQuery 表示式編輯器。輸入傳回 URI 的表示式。這樣會覆寫所叫用之服務的 URI。
    • 若要設定「服務品質」元素：選取服務品質，然後從下拉清單中選取「服務品質」選項。這樣會覆寫自動運算的預設值。
    • 若要設定「模式」：選取「模式」，並從下拉清單中選取要求或要求-回應。
      • 注意： 這通常會根據所叫用之服務介面而自動設定。不過，在某些情況下則不會，例如 Any Soap 或 Any XML 服務。
    • 若要設定「重試間隔」：選取重試間隔，然後指定重試間的秒數。這會覆寫以所叫用之服務設定的預設值。
    • 若要設定「重試計數」：選取重試計數，然後指定系統在中斷動作之前必須嘗試的重試次數。這會覆寫以所叫用之服務設定的預設值。
    • 若要設定「訊息優先順序」：選取優先順序，然後按一下 XQuery 表示式編輯器。輸入傳回正整數的表示式。

完成作業後

完成此動作的組態時，請繼續設定其他動作，或儲存組態 (如新增和編輯訊息流程的動作中所述)。
