將路由表格新增至路由節點

路由表格是一組包覆在切換型條件表格中的路由。它是一種速記建構，允許根據單一 XQuery 表示式的結果，選擇不同的路由。

階段編輯器中有四個累計層次的巢狀限制。如果嘗試新增第五層，則不會顯示此巢狀動作。累計層次包含所有分支動作：If... Then... 條件、發佈表格和路由表格。例如，您可以建立 2 層條件，然後再建立一個含有路由表格的發佈表格，這樣總共就有 4 層。如果您嘗試新增另一個條件 (至最後一個發佈表格)，將不會顯示條件。

這是終止動作，表示您無法在此動作後新增另一個動作。然而，此動作可以包含要求和回應動作。

如需路由的詳細資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 Modeling Message Flow in Oracle Service Bus，網址為 http://www.oracle.com/pls/as111170/lookup?id=OSBAG181。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要將路由表格新增至路由節點

  1. 瀏覽至想要新增動作的位置 (如新增和編輯訊息流程的動作中所述)。
  2. 按一下「路由節點」圖示，然後按一下「編輯路由」，隨即顯示「編輯階段組態」頁面。
  3. 按一下新增動作圖示，然後選取通訊 > 路由表格。隨即顯示路由表格動作。
  4. 從「運算子」清單中，選擇比較運算子，然後在鄰近欄位中輸入值表示式。
  5. 按一下<服務>。隨即顯示選擇服務頁面。
  6. 從清單中選擇一個服務，然後按一下「提交」。
  7. 如果想要在服務上叫用作業，請從「作業」清單中選擇作業
  8. 如果想要外送作業與內送作業相同，請選取「使用內送作業進行外送」核取方塊。
  9. 在要求動作欄位中，按一下新增動作以新增動作，然後選取想要與服務相關聯的動作。您可以新增多個動作。
  10. 在回應動作欄位中，按一下新增動作以新增動作，然後選取想要與服務相關聯的動作。您可以新增多個動作。

    11. 若要進一步瞭解可以新增之要求和回應動作的類型，請參閱新增和編輯訊息流程的動作。

  11. 若要插入新的 Case，請按一下「Case」圖示，然後選取「插入新的 Case」。
  12. 針對新的 Case 重複步驟 2-7。如果先前的任何 Case 不符合您的需求，則可以按一下「Case」圖示，然後選擇「插入預設 Case」以在選定路由的那端新增預設 Case。
  13. 按一下「儲存」。
  14. 在「編輯訊息流程」頁面上，繼續建構訊息流程 (如建立、編輯和檢視訊息流程中所述)。
  15. 按一下「儲存」，確認目前階段作業中的更新。
  16. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
