將條件分支新增至訊息流程

分支節點可讓處理作業繼續轉往其中一個可能的路徑。分支處理是透過 XPath 型切換表格所驅動。表格中的每個分支都會指定條件 (例如，<500)，用以依序使用單一 XPath 表示式沿著訊息流程進行評估 (例如，./ns: PurchaseOrder/ns:totalCost (於 $body 上))。先滿足的條件就可決定要遵循的分支。如果未滿足任何分支條件，則會遵循預設分支。分支節點在訊息流程中可能會有多個子代，包括預設分支在內，每個分支都會有一個子代。

如果代理主機服務不是根據 WSDL，而且接收多種文件類型做為輸入，則請考慮使用條件分支節點。

條件分支是由查詢表格所驅動，每個分支都使用簡單但唯一的字串值標記。訊息相關資訊環境中的變數指定為該節點的查詢變數，並在執行時期使用其值決定要遵循的分支。如果沒有符合查詢變數值的分支，則會遵循預設分支。您應該用這種方式設計代理主機服務，先設定查詢變數的值，再到達分支節點。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯過訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要將條件分支新增至訊息流程

  1. 按一下管線組節點圖示或分支節點圖示，然後按一下建立條件分支。隨即新增條件分支節點，位在所插入之分支節點後方的所有現有節點，都會移至新條件分支節點的預設分支中。
  2. 若要變更分支節點的預設名稱並新增描述，請執行下列操作：
    1. 按一下「條件分支」圖示，然後按一下「編輯名稱和註解」。
    2. 依需要變更名稱和描述。
    3. 按一下「儲存」。
  3. 若要新增分支定義，請按一下條件分支圖示，然後按一下編輯分支。隨即顯示編輯分支節點頁面。
  4. 請執行下列操作：
    1. 在選取的路徑欄位中，按一下編輯以新增指定路徑的 XPath 表示式。請參閱建立和編輯內置 XQuery 和 XPath 表示式。
    2. 在「變數」欄位中輸入相關資訊環境變數。
    3. 從「運算子」欄位中，選取比較運算子。
    4. 在「值」欄位中，輸入分支的值。
    5. 在「標籤」欄位中，輸入分支的標籤。
  5. 選擇性地在「選項」下：
    • 按一下「新增分支」，將新分支定義新增至此分支節點。
    • 按一下「刪除此分支」刪除分支定義。
    • 按一下上移分支或按一下下移分支，變更分支定義的位置。只有在存在多個分支定義時，才會顯示此選項。
  6. 按一下「儲存」，確認目前階段作業中的更新。
  7. 在「編輯訊息流程」頁面上，繼續建構訊息流程 (如建立、編輯和檢視訊息流程中所述)。
  8. 按一下「儲存」，確認目前階段作業中的更新。
  9. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
