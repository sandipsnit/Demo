將作業分支新增至訊息流程

當訊息流程定義以 Web 服務描述語言 (Web Services Description Language，WSDL) 為基礎的代理主機服務時，需要進行作業特定處理。Oracle Service Bus 提供根據作業自動進行分支的基本設定分支節點，而不需要根據作業手動設定分支節點。亦即在訊息流程中建立作業分支節點時，可以根據 WSDL 中定義的作業快速建立分支邏輯，因為「Oracle Service Bus 管理主控台」會將那些作業呈現在分支節點組態頁面中。

分支節點可讓處理作業繼續轉往其中一個可能的路徑。分支處理是透過 XPath 型切換表格所驅動。表格中的每個分支都會指定條件 (例如，<500)，用以依序使用單一 XPath 表示式沿著訊息流程進行評估 (例如，./ns: PurchaseOrder/ns:totalCost (於 $body 上))。先滿足的條件就可決定要遵循的分支。如果未滿足任何分支條件，則會遵循預設分支。分支節點在訊息流程中可能會有多個子代，包括預設分支在內，每個分支都會有一個子代。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯過訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要將作業分支新增至訊息流程

  1. 按一下管線組節點圖示或分支節點圖示，然後按一下建立作業分支。隨即新增作業分支節點，位在所插入之分支節點後方的所有現有節點，都會移至新作業分支節點的預設分支中。
  2. 若要變更分支節點的預設名稱並新增描述，請執行下列操作：
    1. 按一下「作業分支」圖示，然後按一下「編輯名稱和註解」。
    2. 依需要變更名稱和描述。
    3. 按一下「儲存」。
  3. 若要新增分支定義，請按一下作業分支圖示，然後按一下編輯分支。隨即顯示編輯分支節點頁面。
  4. 在「作業分支定義」面板中，選取服務作業。
  5. 選擇性地在「選項」下：
    • 按一下「新增分支」，將新分支定義新增至此分支節點。
    • 按一下「刪除此分支」刪除分支定義。
    • 按一下上移分支或按一下下移分支，變更分支定義的位置。只有在存在多個分支定義時，才會顯示此選項。
  6. 按一下「儲存」。
  7. 在「編輯訊息流程」頁面上，繼續建構訊息流程 (如建立、編輯和檢視訊息流程中所述)。
  8. 按一下「儲存」，確認目前階段作業中的更新。
  9. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
