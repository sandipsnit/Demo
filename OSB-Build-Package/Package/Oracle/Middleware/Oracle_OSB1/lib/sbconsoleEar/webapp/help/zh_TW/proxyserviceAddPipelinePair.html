將管線組節點新增至訊息流程

訊息流程可以包含零個以上的管線組節點，也就是代理主機服務 (或服務上的作業) 的要求和回應管線，以及可以定義給階段、管線和代理主機服務的錯誤處理程式管線。管線可以包含一個或多個階段，而階段則會包含動作。

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯過訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要將管線組節點新增至訊息流程

  1. 按一下「代理主機服務」圖示，然後按一下「新增管線組」。
  2. 若要變更預設名稱以及新增管線組節點的描述，請執行下列操作：
    1. 按一下「管線組節點」圖示，然後按一下「編輯名稱和註解」。
    2. 依需要變更名稱和描述。
    3. 按一下「儲存」。
      4. 注意： 重新命名管線或路由節點時，因為會將管線計數器重設為零，所以監控模組之儀表板頁面中所顯示的訊息數可能會與其他元件的訊息數無關。這是因為 Oracle Service Bus 會將重新命名動作視為刪除再重新建立動作。而經過與服務監控間隔相等的時間間隔之後，這些數目應該會再次相關。
  3. 若要將階段新增至管線，請參閱將階段新增至管線。
  4. 若要將動作新增至管線中的階段。請參閱新增和編輯訊息流程的動作。
  5. 在「編輯訊息流程」頁面上，繼續建構訊息流程 (如建立、編輯和檢視訊息流程中所述)。
  6. 按一下「儲存」，確認目前階段作業中的更新。
  7. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
