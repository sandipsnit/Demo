將路由節點新增至訊息流程

開始之前

這些指示假設您已在「編輯訊息流程」頁面中編輯過訊息流程 (如建立、編輯和檢視訊息流程中所述)。

若要將路由節點新增至訊息流程

  1. 按一下管線組的「管線組節點」圖示，然後按一下「新增路由」。
  2. 若要變更預設名稱並新增路由節點的描述，請按一下「路由節點」圖示，然後按一下「編輯名稱和註解」。請依需要變更名稱和描述，然後按一下「儲存」。
    3. 注意： 重新命名管線或路由節點時，因為會將管線計數器重設為零，所以監控模組之儀表板頁面中所顯示的訊息數可能會與其他元件的訊息數無關。這是因為 Oracle Service Bus 會將重新命名動作視為刪除再重新建立動作。而經過與服務監控間隔相等的時間間隔之後，這些數目應該會再次相關。
  3. 若要將動作新增至路由節點，請按一下路由節點圖示，然後按一下編輯路由。隨即顯示編輯訊息流程頁面。有關可以新增至路由節點之動作的詳細資訊，請參閱下列各節：
    • 新增 If... Then... 動作
    • 將動態路由新增至路由節點
    • 將路由動作新增至路由節點
    • 將路由表格新增至路由節點
    • 新增路由節點錯誤處理程式
  4. 在「編輯訊息流程」頁面上，繼續建構訊息流程 (如建立、編輯和檢視訊息流程中所述)。
  5. 按一下「儲存」，確認目前階段作業中的更新。
  6. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
