建立和設定代理主機服務

本節說明如何建立及設定代理主機服務。

對於 JCA 服務，您可以從「JCA 連結」產生代理主機服務，如從 JCA 連結資源產生代理主機服務中所述。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「專案瀏覽器」，然後選取要在其中新增代理主機服務的專案或資料夾。
  3. 在專案/資料夾檢視畫面頁面的「建立資源」下拉清單中，選取「代理主機服務」。
  4. 在一般組態頁面上提供服務的名稱，並選取要建立的服務類型。

    請遵循資源命名限制的命名指南。
  5. 按一下下一步。接下來的頁面視您在第一頁上所做的選擇而有所不同。在後續每一頁輸入適當的資訊，直到進入摘要頁面為止，然後按一下「儲存」，將服務儲存在目前的階段作業中。

    6. 有關完成各個頁面的詳細指示，請參閱下列項目：

    • 一般組態頁面
    • 訊息類型組態頁面
    • 傳輸組態頁面
    • 通訊協定特定傳輸組態頁面
    • 作業選取組態頁面
    • 訊息處理頁面
    • 摘要頁面
    • 代理主機服務原則頁面
    • 代理主機服務安全頁面
  6. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
  7. 建立代理主機服務後，下一步是設定其訊息流程。訊息流程會定義代理主機服務的實行方式。如需詳細資訊，請參閱建立、編輯和檢視訊息流程。

從 JCA 連結資源產生代理主機服務

您可以從 Oracle Service Bus 的內送 JCA 連結資源來產生 JCA 代理主機服務。如需 JCA 連結資源的詳細資訊，請參閱第 12 章，JCA 連結。

從 JCA 連結產生 JCA 代理主機服務：

  1. 在「Oracle Service Bus 管理主控台中」，如果您尚未處於「建立」或「編輯」模式，請按一下「變更中心」中的建立或編輯。
  2. 在「資源瀏覽器」中按一下「JCA 連結」。
  3. 找出您要用於產生服務的 JCA 連結，然後按一下「動作」圖示。
  4. 在顯示的視窗中，確認您要產生的 WSDL 的名稱及服務，為這些新資源選取位置後按一下「產生」。

    5. Oracle Service Bus 會產生服務及其對應 WSDL。

  5. 適當地修改其他在產生服務上的組態詳細資訊，例如端點 URI。
