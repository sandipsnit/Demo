電子郵件傳輸組態頁面

「建立/編輯代理主機服務 - 電子郵件傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

使用此頁面，可以使用電子郵件傳輸通訊協定來設定代理主機服務的傳輸設定值。表 19-4 說明如何使用此頁面。

表 19-4 建立/編輯代理主機服務 - 電子郵件傳輸組態頁面
選項
建立或編輯...
需要 SSL
選取此選項即可使用「安全通訊端層」進行通訊。預設的 SSL 連接埠號碼包括：
  • POP3(S)：995
  • IMAP：993
注意：Oracle Service Bus 信任存放區中必須要有電子郵件伺服器憑證，才能使 SSL 順利運作。
服務帳號
輸入服務帳號名稱，或按一下「瀏覽」，從瀏覽器中選取服務帳號。
受管理伺服器
此欄位只能用於叢集網域。
選取要作為輪詢伺服器的受管理伺服器。所有受管理伺服器都可以處理訊息，但只有一個受管理伺服器可以輪詢訊息。
輪詢間隔
輸入輪詢間隔 (秒)。
電子郵件通訊協定
選取 POP3 或 IMAP 做為電子郵件帳號的伺服器類型。
讀取限制
指定每次輪詢清除所讀取的訊息數目上限。請輸入 0，以指定沒有限制。
透過參照來傳遞
選取此核取方塊，將檔案臨時放置於封存檔目錄中，並以參照形式在標頭中傳遞該檔案。
根據預設，當您建立新的服務時，會選取「透過參照來傳遞」選項，而且您必須指定封存檔目錄位置。
透過參照來傳遞附件
選取此核取方塊，將附件臨時放置於封存檔目錄中，並以參照形式在標頭中傳遞該附件。
根據預設，選取「透過參照來傳遞」選項後，「透過參照來傳遞附件」選項就會隱含為 true，而您必須指定封存檔目錄位置。
讀取後動作
選取訊息讀取後的動作：
  • 封存 - 封存訊息。
  • 刪除 - 刪除訊息。
  • 移動 - 移動訊息。「移動」只能與 IMAP 通訊協定一起使用。
附件
選取附件的處理方式：
  • 封存 - 附件會儲存至封存檔目錄。
  • 忽略 - 忽略附件。
IMAP 移動資料夾
輸入在「讀取後動作」欄位設為「移動」時，要移入訊息的資料夾。
下載目錄
輸入用來下載電子郵件的暫時位置。
封存檔目錄
指定在「讀取後動作」欄位設為「封存檔」時之封存檔位置的路徑。
如果已選定「透過參照來傳遞」或「透過參照來傳遞附件」選項，則這是必要的欄位。
錯誤目錄
輸入發生問題時要寫入訊息及任何附件的檔案系統目錄路徑。
要求編碼
接受預設值 ISO-8859-1 做為電子郵件傳輸中要求的字元集編碼，或輸入其他字元集編碼。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
