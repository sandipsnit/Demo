FTP 傳輸組態頁面

「建立/編輯代理主機服務 - FTP 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

使用此頁面，可以使用 FTP 傳輸通訊協定來設定代理主機服務的傳輸設定。表 19-6 說明如何使用此頁面。

表 19-6 建立/編輯代理主機服務 - FTP 傳輸組態頁面
選項
建立或編輯...
使用者認證
如果 FTP 伺服器的使用者是匿名使用者，請選取「匿名」，如果 FTP 伺服器的使用者是外部設定的帳號，請選取「外部使用者」。
識別 (電子郵件 ID)
只有在「使用者認證」選項設為「匿名」時，才可以使用此欄位。
輸入匿名使用者的郵件 ID。
服務帳號
只有在「使用者認證」選項設為「外部使用者」時，才可以使用此欄位。
輸入使用者的服務帳號。如果「使用者認證」選項設為「外部使用者」，則這是必要欄位。
透過參照來傳遞
選取此核取方塊，將檔案臨時放置於封存檔目錄中，並以參照形式在標頭中傳遞該檔案。
遠端串流
選取此核取方塊，以在處理時直接從遠端伺服器用串流的方式傳送 FTP 檔案。選取此選項時，封存檔目錄就是遠端 FTP 伺服器機器上的遠端目錄。因此，應該以相對於 FTP 使用者目錄的方式指定封存檔目錄。
檔案遮罩
輸入要選擇之檔案的正規表示式。預設值為 *.*。
受管理伺服器
此欄位只能用於叢集網域。
選取要作為輪詢伺服器的受管理伺服器。所有受管理伺服器都可以處理訊息，但只有一個受管理伺服器可以輪詢訊息。
輪詢間隔
輸入輪詢間隔 (秒)。預設值是 60。
讀取限制
指定每次輪詢清除所讀取的訊息數目上限。請輸入 0，以指定沒有限制。預設值為 10。
讀取後動作
選取訊息讀取後的動作。
  • 封存 - 封存訊息。
  • 刪除 - 刪除訊息。
傳輸模式
選取「ASCII」或「二進位」做為傳輸模式。
封存檔目錄
如果讀取後動作選項設為封存檔，請指定封存檔位置的路徑。如果選取透過參照來傳遞選項，則此為必要欄位。
注意： 封存檔、下載和錯誤目錄都是絕對路徑，而且會自動予以建立。如果指定相對路徑，則會在與啟動 WebLogic Server 之 Java 流程相對的位置建立檔案。
下載目錄
輸入本機機器上的目錄，這是檔案傳輸期間下載檔案的位置。
注意： 封存檔、下載和錯誤目錄都是絕對路徑，而且會自動予以建立。如果指定相對路徑，則會在與啟動 WebLogic Server 之 Java 流程相對的位置建立檔案。
錯誤目錄
輸入在發生問題時用來張貼訊息的位置。
注意： 封存檔、下載和錯誤目錄都是絕對路徑，而且會自動予以建立。如果指定相對路徑，則會在與啟動 WebLogic Server 之 Java 流程相對的位置建立檔案。
要求編碼
接受預設值 UTF-8 做為 FTP 傳輸中要求的字元集編碼。
掃描子目錄
選取此核取方塊，以遞迴方式掃描所有目錄。
依到達時間排序
選取此核取方塊，依到達的順序傳遞事件。
逾時
輸入在捨棄連線之前的通訊端逾時間隔 (秒)。如果輸入 0，則沒有逾時。
重試計數
指定 FTP 連線失敗的重試次數。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
