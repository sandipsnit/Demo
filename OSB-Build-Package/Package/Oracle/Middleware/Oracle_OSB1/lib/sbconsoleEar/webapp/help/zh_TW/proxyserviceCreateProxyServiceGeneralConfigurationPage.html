一般組態頁面

「建立/編輯代理主機服務 - 一般組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

使用此頁面可以修改代理主機服務的一般組態。

建立代理主機服務時，這是用來設定該服務之一系列頁面中所顯示的第一個頁面。此頁面之後所顯示的頁面視您在此頁面上所做的選擇而有所不同。

表 19-1 會描述如何使用此頁面。

表 19-1 建立/編輯代理主機服務 - 一般組態頁面 
選項
建立或編輯...
服務名稱
輸入代理主機服務的唯一名稱。

請遵循資源命名限制的命名指南。
描述
輸入描述。
服務類型
選取要建立的代理主機服務類型：
  • WSDL Web 服務 - 選取此選項，以根據 WSDL 建立代理主機服務。然後輸入 WSDL 名稱，並以其路徑限定 (例如 myProject/myFolder/myWSDL)。或是按一下「瀏覽」，從「選取 WSDL 定義」頁面中選取 WSDL。

    • (連接埠或連結) - 輸入連接埠名稱 (使用 WSDL 定義) 以說明實際傳輸位址，或輸入連結名稱 (使用 WSDL 定義) 以對應至傳輸位址。如果使用「瀏覽」以選取 WSDL，則「選取 WSDL 定義」頁面會列出所有定義的連接埠和連結。

注意： 如果是要使用「SOAP 主體類型」來進行作業，請確定 WSDL 沒有兩個具有相同輸入訊息的作業。檢查輸入訊息並無法唯一識別「SOAP 主體類型」作業。
  • 傳輸類型服務 - 選取這個選項建立 JEJB 代理主機服務。
  • 訊息傳遞服務 - 選取此選項以建立服務，而此服務可以接收某種資料類型的訊息，並以不同資料類型的訊息回應。這些交換可以是要求/回應交換或單向交換

    • (只有「任何 XML 服務」和「訊息傳遞服務」服務類型才支援 HTTP GET)。

  • 任何 SOAP 服務 - 選取此選項以建立沒有明確定義之具體介面的 SOAP 服務。

    • 從下拉清單中選取「SOAP 1.1」或「SOAP 1.2」以指定要使用的 SOAP 版本。

  • 任何 XML 服務 - 選取此選項以建立沒有明確定義之具體介面的 XML 服務

    • (只有「任何 XML 服務」和「訊息傳遞服務」服務類型才支援 HTTP GET)。

  • 業務服務 - 選取此選項以建立具有路由節點的代理主機服務，而此路由節點會路由至所選取的業務服務。您無法利用傳輸類型的業務服務建立代理主機服務。因為 DSP 傳輸不適用於代理主機服務，所以如果是從 DSP 傳輸業務服務建立代理主機服務，則 Oracle Service Bus 會將代理主機服務的傳輸類型切換為 HTTP。您可以將代理主機服務的傳輸類型變更為其他任何可用的傳輸。

    • 輸入路徑 (project/folder) 和業務服務名稱，或按一下「瀏覽」，從「選取業務服務」頁面選取業務服務。

服務類型 (續)
  • 代理主機服務 - 選取此選項以複製現有的代理主機服務。因為 Oracle Service Bus 不接受多個服務具有相同的 URI，所以必須變更所複製服務的 URI。
注意： 從另一個服務建立服務時，會以下列方式維護警示規則：
    • 從業務服務建立代理主機服務或從代理主機服務建立業務服務時，會移除警示規則 (如果有的話)。
    • 從另一個代理主機服務建立代理主機服務或從另一個業務服務建立業務服務時，會保留警示規則 (如果有的話)。
服務金鑰提供者
必要時，輸入路徑 (專案/資料夾) 和服務金鑰提供者名稱，或按一下「瀏覽」，從「選取服務金鑰提供者」頁面中選取服務金鑰提供者。
只有在特定情況下才需要服務金鑰提供者：
  • 外送雙向 TLS/SSL：代理主機服務會將訊息路由至需要從屬端憑證認證的 HTTPS 服務。
  • 在某些 Web 服務安全案例中，例如代理主機服務需要加密訊息時。
若要新增已啟用 Web 服務安全性的代理主機服務，則必須從具有 WS 原則附件的 WSDL (連接埠或連結) 建立代理主機服務。
如需詳細資訊，請參閱服務金鑰提供者。若要瞭解如何建立服務金鑰提供者，請參閱新增服務金鑰提供者。
發佈到登錄
只有在預設 UDDI 登錄檔存在時，才會出現此選項。
選取此選項，自動將代理主機服務發佈至預設登錄檔。如需詳細資訊，請參閱使用自動發佈。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
