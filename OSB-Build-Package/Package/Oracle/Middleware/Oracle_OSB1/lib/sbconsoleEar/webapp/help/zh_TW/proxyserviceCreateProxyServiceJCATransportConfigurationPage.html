JCA 傳輸組態頁面

使用此頁面，可透過 JCA 傳輸通訊協定來設定傳輸設定。如需使用 JCA 傳輸的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "JCA Transport" (網址為 http://www.oracle.com/pls/as111170/lookup?id=OSBDV910) 以及從 JCA 連結資源產生代理主機服務。

JCA 檔案
按一下瀏覽選取「JCA 連結」。JCA 連結定義不同服務的面向，例如關於轉接器使用的詳細資訊、WSDL 及 TopLink 的連結或 EclipseLink 對應檔案，以及服務所要求的啟動/交互作用規格特性。
一旦您選取有效的 JCA 連結，其餘的傳輸組態欄位就變成可以使用。
如需更多在 JCA 連結上的資訊，請參閱第 12 章，JCA 連結。

顯示 JCA 服務要使用之轉接器名稱的唯讀值。
轉接器類型
顯示轉接器類型的唯讀值。
分配原則
選取要針對此端點的分配原則而使用的 WebLogic Server 工作管理程式執行處理。如果不存在其他工作管理程式，則會使用預設的工作管理程式。
如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
JNDI 服務帳號
JNDI 服務帳號專供 JNDI 相關資訊環境安全使用，用來存取 EIS 轉接器管理的連線工廠。按一下瀏覽，然後選取服務帳號。若未指定服務帳號，則會使用匿名主題。
並無限制 JCA 業務服務可設定的 JNDI 服務帳號類型 (例如，靜態或透通)，不過執行時期必須可以存取使用者名稱和密碼。JCA 代理主機服務只可使用靜態 JNDI 服務帳號。
如需 JNDI 服務帳號的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中 "JCA Transport" 章節的 "Security"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV916。
端點特性
這個欄位讓您指派值到端點特性，例如服務使用的連接器類型重試次數。
如需支援 JCA 的端點特性清單，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Endpoint Properties"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1841。
動態端點特性
這個選項讓您傳送要求參數到 JCA相容服務。例如，您可以使用動態端點特性傳送資料庫查詢參數到資料庫的 Oracle JCA 轉接器。
如需更多查詢參數的資訊，請參閱 Oracle Fusion Middleware User's Guide for Technology Adapters 中的 "Oracle JCA Adapter for Database"，網址為： http://www.oracle.com/pls/as111170/lookup?id=TKADP2117。
輸入您想提供的每個動態端點特性的名稱/值。端點特性索引鍵會配對查詢參數名稱。
一律使用來自 JCA 檔案的組態
此選項可決定否一律使用來自 JCA 檔案的「啟動規格特性」(代理主機服務) 和「互動規格特性」(業務服務)。
如果選取此選項 (預設)，JCA 傳輸會使用 JCA 檔案中的啟動/互動規格特性來與 JCA 架構進行互動。
如果取消選取此選項，您可以覆寫啟動/互動規格特性。
如需使用此選項對重新建置產生之影響的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中 "JCA Transport" 章節的 "Endpoint Redeployment"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV940。
作業名稱
顯示所選 WSDL 作業的唯讀名稱。作業可以具有自己的啟動/互動規格特性 (如「啟動規格特性」或「互動規格特性」欄位所示)。
啟動/互動規格特性
「啟動規格特性」是代理主機服務的欄位名稱；「互動規格特性」則是業務服務的欄位名稱。
如果此服務是 EIS 應用程式叫用的內送服務，則此欄位會針對「作業名稱」欄位出現之 JCA 內送輸入作業顯示啟動規格特性。
如果取消選取「一律使用來自 JCA 檔案的組態」，您可以覆寫啟動/互動規格特性。
注意： 如果是 Oracle Adapter Suite 轉接器，則啟動/互動規格特性一律會顯示為唯讀。Oracle Adapter Suite 轉接器會儲存自己的組態，而您必須在 Oracle Adapter Suite 管理工具中才能變更這些組態。
連線特性 (舊型)
若要使用非管理模式連線特性的舊型 JCA 服務 (在這個版本已不再使用)，請參閱在 http://download.oracle.com/docs/cd/E13159_01/osb/docs10gr3/jcatransport/transport.html#wp1105451 的連線組態選項。

如需端點和啟動/互動規格特性的詳細資訊，請參閱 Oracle Fusion Middleware User's Guide for Technology Adapters，網址為： http://www.oracle.com/pls/as111170/lookup?id=TKADP。
