分配原則

EJB 規格版本
選取遠端 EJB 介面的 EJB 版本。
透過值傳送 XMLBeans
如果您想要傳輸產生以「內置」XML 呈現的 POJO 引數 (MLObject) (這是您可以用 XQuery 表示式存取及管理的參數)，請選取這個選項。
請注意：透過值傳送的 XMLObjects 沒有內置的類型資訊。如果使用這個選項，您在代理主機服務管線的 Java Callout 中將無法傳送類型為 XMLObject 的引數。
如果想要透過參照傳送 POJO (會有較佳的效能)，請不要選取這個選項。
交易屬性
選取下列處理交易的選項其中之一：
支援 - 傳輸接受傳入的交易。如果作業是在交易中呼叫則服務品質是「正好一次」，如果作業是在交易外呼叫，則為「最佳效果」。
必要 - 傳輸接受傳入的交易。如果沒有存在正在進行的交易，則傳輸會啟動一個交易。服務品質為「正好一次」。
RequiresNew - 傳輸永遠會啟動新的交易，暫停正在進行中的交易。服務品質為「正好一次」。
強制 - 傳輸呼叫現存交易中的方法。服務品質為「正好一次」。
NotSupported - 傳輸暫停現存的交易，並且在呼叫時繼續。服務品質為「最佳效果」。
永不 - 傳輸不會在交易中呼叫方法。服務品質為「最佳效果」。
從屬端 JAR
按一下瀏覽並從顯示的清單選取 EJB 從屬端 JAR 資源。從屬端 JAR 包含遠端 EJB 的遠端或業務介面。從屬端 JAR 註冊為一般封存檔資源。
本位目錄介面
僅限 EJB 2.1 - 在從屬端 JAR 填入的選項中選取必要的 EJBHome 介面。
遠端介面
僅限 EJB 2.1 - 此欄位會根據「本位目錄介面」的組態自動填寫。
業務介面
僅限 EJB 3.0 - 從您要啟動的從屬端 JAR 中選取要使用的業務介面。
目標命名空間
此欄位由選自 JAR 的資訊填充。
方法
選取需要的方法。按一下 + 可展開方法，這可讓您編輯預設的參數值。
您可以為指定的方法變更預設作業名稱。(根據預設值，作業名稱是方法名稱)。如果 EJB 包含相同名稱 (超載) 的方法，您必須變更作業名稱，讓它們具有唯一名稱。WSDL 要求作業名稱必須是唯一名稱。
