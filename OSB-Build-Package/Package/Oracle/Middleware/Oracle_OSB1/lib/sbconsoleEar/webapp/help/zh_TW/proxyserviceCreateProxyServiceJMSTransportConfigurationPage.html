JMS 傳輸組態頁面

「建立/編輯代理主機服務 - JMS 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「JMS Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037。

使用此頁面，可以使用 JMS 傳輸通訊協定來設定代理主機服務的傳輸設定。表 19-10 說明如何使用此頁面。

表 19-10 建立/編輯代理主機服務 - JMS 傳輸組態頁面  
選項
建立或編輯...
目標類型
選取下列其中一項：
  • 佇列 (適用於點對點目標類型)
  • 主題 (適用於發佈/訂閱目標)。
是否需要回應
只有將「目標類型」選取為「佇列」時，才可以使用此選項。
選取此選項可指定在傳送外送訊息後需要回應。
回應模式
只有在選擇「是否需要回應」核取方塊時，才可以使用此選項。
選取下列其中一項：
  • 針對在 Oracle WebLogic Server 上執行的 JAX-RPC 服務選取 JMSMessageID。
  • 如果是其他所有服務，請選取 JMSCorrelationID。如果選取此選項，必須同時輸入回應 URI。
回應訊息類型
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
選取下列其中一項：
  • 位元組 (針對無法解譯位元組的串流)
  • 文字 (適用於文字訊息)
如果為回應選取了 Java 訊息類型，則會停用此選項。
從屬端 Jar
當服務為要求類型 Java 的「訊息傳遞服務」時即可使用此選項。選取要用來取消佇列含 Java 物件之訊息的從屬端 JAR。選取從屬端 JAR 可確保其位於類別路徑中。

有關詳細資料，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Sending and Receiving Java Objects in Messages"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。
分配原則
選取要針對此端點的分配原則而使用的 Oracle WebLogic Server 工作管理程式執行處理。如果不存在其他工作管理程式，則會使用預設的工作管理程式。
例如，如果業務服務具有 JMS 傳輸通訊協定，則業務服務端點是可以與特定分配原則產生關聯的 MDB (訊息驅動 Bean) JAR 檔案。
如需工作管理程式的相關資訊，請參閱·Oracle·Fusion·Middleware·Configuring·Server·Environments·for·Oracle·WebLogic·Server·中的·"Using·Work·Managers·to·Optimize·Scheduled·Work"，網址為：http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
要求編碼
輸入編碼要求的字元集。預設為 UTF-8。
回應編碼
只有在選擇「是否需要回應」核取方塊時，才可以使用此選項。
輸入編碼回應的字元集。預設為 UTF-8。
從屬端回應逾時
只有在選擇「是否需要回應」核取方塊時，才可以使用此選項。
輸入在捨棄連線之前等待伺服器回應的秒數。這只適用於從屬端是相同網域的另一個代理主機服務時。
回應 URI
只有在選擇「JMSCorrelationID」做為「回應關聯模式」時，才能使用此選項。
請以下列格式輸入回應 URI：
jms://host:port/connection_factory/jndi_destination
若要將多部伺服器設定為目標，請使用下列格式：
jms://host1:port,host2:port/connection_factory/jndi_destination
您也可以在回應 URI 中省略主機及連接埠。例如：
jms:///connection_factory/jndi_destination

當省略主機及連接埠，會在本機伺服器上查找連線工廠/目的地。這非常有用，例如，要求的 URI 會送到外部連線工廠/目的地，但是您想要回應傳送至本機伺服器。

注意：雖然 Oracle WebLogic Server 允許在 JNDI 名稱中使用正斜線 (例如 myqueues/myqueue)，但是含有正斜線的 JNDI 名稱會干擾 Oracle Service Bus 所需的 URI 格式，因此您無法使用那些名稱。若要解決此問題，請定義一個 JMS 外部伺服器，並在 URI 中參照該外部伺服器。如需詳細資訊，請參閱 Oracle Fusion Middleware Oracle WebLogic Server Administration Console Online Help 中的 Configure foreign servers，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLACH。

回應連線工廠
只有在選擇「JMSMessageID」做為「回應關聯模式」時，才能使用此選項。
輸入回應連線工廠 URI。
如果未指定連線工廠，則會將要求的連線工廠用於回應。
JMS 服務帳號
選取要用於 JMS 伺服器所管理之 JMS 資源的服務帳號。服務帳號是用於要求及回應的使用者 ID 及密碼的別名資源。同一個服務帳號可同時用於 JMS 和 JNDI。
如需更多資訊，請參閱服務帳號。
使用 SSL
只有在透過 TLS/SSL 連線提出要求時，才可以選取這個選項。
TLS/SSL (安全通訊端層) 可以提供安全連接，方式是允許兩個透過網路連接的應用程式對對方的標識進行認證，並加密應用程式之間所交換的資料。認證允許伺服器和從屬端 (可選) 驗證網路連接另一端上應用程式的標識。此外，如果管理員已經在目標的 JNDI 項目上設定存取控制，進而限制了對個別 JMS 目標 (佇列或主題) 的存取，則服務在 JNDI 樹狀圖中尋找該項目時必須認證。
訊息選取器
輸入訊息選取器表示式。
只會處理特性與表示式相符的訊息。
JMS 從屬端 ID

僅供主題使用，請輸入要用於訂閱者的從屬端 ID。若未輸入任何值，系統會自動產生從屬端 ID。指派自訂 ID 可幫助您在監督服務時識別此元件。請注意，這只對持續訂閱者才有效。

JMS 從屬端 ID 是數個 MDB 描述元其中之一，這些描述元是用來產生主題訂閱者的訂閱者名稱和 ClientID 值。定義 JMS 從屬端 ID 即可查看訂閱者名稱或從屬端 ID，更易於識別與檢視特定主題的訂閱者。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「JMS Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1037。
持續訂閱
只有將「目標類型」選取為「主題」時，才可以使用此選項。
如果是持續訂閱，請選取此核取方塊，如果不是持續訂閱，請將它保留空白
重試計數
輸入訊息在移至錯誤目標之前可以重試傳遞的次數。此欄位只適用於 Oracle WebLogic Server JMS 目標。
重試間隔
輸入在重新傳遞復原或恢復之訊息前的時間量 (毫秒)。此欄位只適用於 Oracle WebLogic Server JMS 目標。
錯誤目標
輸入已達重新傳遞限制的訊息所要送往的目標名稱。此欄位只適用於 Oracle WebLogic Server JMS 目標。
到期原則
選取在目標上發現到期訊息時使用的「到期原則」。此欄位只適用於 Oracle WebLogic Server JMS 目標。
是否需要 XA
如果您的連線工廠為 XA，請選取此核取方塊。
如果遠端連線工廠無法使用，則會考慮此值。如果有可用的連線工廠，且此值為 true，請確定連線工廠定義為可交易。
交易逾時
輸入 JMS 代理主機服務等待交易進行處理的時間 (以秒為單位)。此選項僅適用於使用 XA 連線工廠的服務。預設為 600 秒。
主題訊息分送
選取「主題」做為「目標類型」時即可使用此選項。「主題訊息分送」提供不同的選項以處理內送 JMS 訊息，並提供高可用性與容錯移轉。

選取下列其中一個特性，以決定訊息驅動 Bean 處理傳入 JMS 訊息的方式：
  • 每個應用程式一個複本 (預設) – 如果 JMS 代理主機服務建置於叢集上，此選項可藉由確保內送 JMS 訊息只會在叢集的其中一個伺服器上處理，來提供高可用性和擴充性。
  • 每個伺服器一個複本 – 如果想要將內送 JMS 訊息發佈至主題，以便讓叢集每個成員上的代理主機服務收到，請選取此選項。
  • 相容性 – 如果想在叢集的特定「受管理伺服器」或所有「受管理伺服器」上處理內送 JMS 訊息，請選取此選項。如果選取此特性，請選取目標伺服器。
注意：Oracle WebLogic Server 10.3.4 之前的版本中建置的主題目標，唯一有效的選項為「相容性」。
您為「主題訊息分送」選取的「一個複本」選項會覆寫在 JMS 連線工廠設定的「訂閱共用原則」和/或「從屬端 ID 原則」。
目標
當您選取「相容性」做為「主題訊息分送」選項時，此選項僅可在 Oracle Service Bus 叢集中使用。選取將處理傳入 JMS 訊息的目標伺服器。如果為「主題訊息分送」選取「一個複本」選項的其中一個，此欄位便會顯示叢集的名稱。

若未設定目標，叢集中每個「受管理伺服器」上的 JMS 代理主機服務執行處理 (讀取「主題」的訊息) 就會收到訊息的複本。
JNDI 逾時
在 JNDI 目錄樹中查詢目標/連線工廠時，使用的 JNDI 連線逾時 (以秒為單位)。

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
