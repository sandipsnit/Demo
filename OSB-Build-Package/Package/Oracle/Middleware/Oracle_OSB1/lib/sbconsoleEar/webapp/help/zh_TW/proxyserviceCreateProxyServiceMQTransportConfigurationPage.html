MQ 傳輸組態頁面

「建立/編輯代理主機服務 - MQ 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「MQ Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1117。

開始之前

設定 MQ 連線資源。請參閱 MQ 連線。

設定 MQ 傳輸

使用此頁面，可透過原生 MQ 傳輸通訊協定來設定代理主機服務的傳輸設定。表 19-11 說明如何使用此頁面。

表 19-11 建立/編輯代理主機服務 - MQ 傳輸組態頁面 
選項
建立或編輯...
輪詢間隔
輸入輪詢間隔 (毫秒)。預設值為 1000。
是否需要回應
選取此選項可指定在傳送外送訊息後需要回應。
回應關聯模式
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
指定回應關聯模式應該根據「MessageID」或「CorrelationID」。
MQ 回應 URI
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
應發佈回應的目標。使用與端點 URI 相同的格式輸入回應 URI：mq://<local-queue-name>?conn=<mq-connection-resource-ref>
回應訊息類型
只有在選取「是否需要回應」核取方塊時，才可以使用此選項。
選取下列其中一項：
  • 位元組 (針對無法解譯位元組的串流)
  • 文字 (適用於文字訊息)
分配原則
選取此端點的分配原則。
分配原則指的是您想用於服務端點的 WLS 工作管理程式執行處理。如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
退出臨界值
輸入的值代表在將訊息重新導向至「寄不出的信件的 URI」欄位中所指定的佇列之前，管線應該重試訊息的次數。
如果您未指定此欄位的值，則會將訊息重新導向至「寄不出的信件」佇列，而不進行任何嘗試。
MQ 寄不出的信件 URI
輸入「寄不出的信件」佇列的 URI，在嘗試「退出臨界值」欄位中所指定的次數之後，應該將要求訊息重新導向至該佇列。
如果您未指定此欄位的值，則會在重試退出臨界值欄位中所指定的次數之後將訊息傳回佇列，且每次輪詢的 MQ 傳輸也會忽略該訊息。「寄不出的信件的 URI」使用的格式與「端點 URI」相同。
端點 URI 'GET' 選項
輸入下列 MQ GET 訊息選項：
  • MQC.MQGMO_ACCEPT_TRUNCATED_MSG
  • MQC.MQGMO_ALL_MSGS_AVAILABLE
  • MQC.MQGMO_BROWSE_FIRST
  • MQC.MQGMO_BROWSE_NEXT
  • MQC.MQGMO_COMPLETE_MSG
  • MQC.MQGMO_CONVERT
  • MQC.MQGMO_FAIL_IF_QUIESCING
  • MQC.MQGMO_LOCK
  • MQC.MQGMO_LOGICAL_ORDER
  • MQC.MQGMO_MARK_BROWSE_CO_OP
  • MQC.MQGMO_MARK_SKIP_BACKOUT
  • MQC.MQGMO_NO_SYNCPOINT
  • MQC.MQGMO_NONE
  • MQC.MQGMO_NO_WAIT
  • MQC.MQGMO_SYNCPOINT
  • MQC.MQGMO_SYNCPOINT_IF_PERSISTENT
  • MQC.MQGMO_UNLOCK
  • MQC.MQGMO_UNMARK_BROWSE_CO_OP
  • MQC.MQGMO_UNMARK_BROWSE_HANDLE
  • MQC.MQGMO_UNMARKED_BROWSE_MSG
  • MQC.MQGMO_VERSION_1
  • MQC.MQGMO_VERSION_2
  • MQC.MQGMO_VERSION_3
  • MQC.MQGMO_WAIT
您可以使用 "|" 或 "+" 隔開多個選項。例如，您可以指定下列項目：
MQC.MQGMO_ACCEPT_TRUNCATED_MSG | MQC.MQGMO_LOCK
讀取內送佇列中的訊息時，會套用 MQ GET 訊息選項。
處理 RFH2 標頭
選取此選項，可從訊息有效負載剖析 WebSphere MQ RFH2 標頭，並自動產生內含 RFH2 資料的 RFH2Headers 傳輸標頭。
如果沒有選取此選項，則會直接傳遞收到的有效負載。

有關 MQ 傳輸如何處理 RFH2 標頭的詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "About RFH2 Headers"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1136。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
