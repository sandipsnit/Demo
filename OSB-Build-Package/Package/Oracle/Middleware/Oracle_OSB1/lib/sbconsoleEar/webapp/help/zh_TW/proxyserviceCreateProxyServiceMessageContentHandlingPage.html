訊息處理頁面

「建立/編輯代理主機服務 - 訊息處理」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務及編輯代理主機服務組態所述)。

使用此頁面可指定代理主機服務是否應以串流傳送訊息內容、代理主機服務應如何解碼接收的要求訊息及編碼傳送的回應訊息以及代理主機服務是否應以串流傳送 MIME 附件，而非緩衝處理記憶體中的附件內容。

使用此頁面時，您將可讓代理主機服務以串流傳送訊息內容，而非將其儲存在記憶體中。在適當情況下，您也可以讓代理主機服務以 MTOM/XOP 格式解碼及剖析內送訊息，以及使用 MTOM/XOP 格式傳送回應。SOAP 訊息傳輸最佳化機制 (MTOM) 是可對 Web 服務收送二進位資料的方法。MTOM 會使用 XML 二進位最佳化封裝 (XOP) 傳輸二進位資料。

使用此頁面時，您也能夠讓代理主機服務將 MIME 附件內容儲存在磁碟檔案中，然後以串流的方式處理資料，而非緩衝處理記憶體中的附件內容。如此可讓代理主機服務穩定而有效地處理大型附件。

需要交易
選取這個選項以確保 Oracle Service Bus 在交易相關資訊環境內執行代理主機服務訊息流程。如果全域交易已經存在，傳輸提供者會在要求中傳播全域交易 (即使未選取這個選項)。如果全域交易不存在，訊息流程執行時間會啟動交易。如果訊息流程執行時間啟動交易，交易相關資訊環境會開始於服務組態已驗證或執行之前 (例如，安全檢查或 WS-I 驗證)，或是在訊息流程執行前，以確保所有的處理及執行都是在交易相關資訊環境內。
如果訊息流程執行時間啟動交易，服務品質為「正好一次」。然而，如果「服務外部呼叫」或「發佈」動作將外送服務品質的參數設為「最佳效果」(預設)，Oracle Service Bus 會在交易相關資訊環境之外執行這些動作。若要讓 Oracle Service Bus 在相同的要求交易相關資訊環境中執行這些動作，請在這些動作上設定服務品質為「正好一次」。
服務會忽略在這個選項上的設定，而維持它的訊息模式 (同步、非同步、單向)。
針對交易逾時，會套用在 Oracle WebLogic Server 主控台中所設定的全域交易逾時值。
交易中的異常狀況
Oracle Service Bus 會為失敗的交易呼叫系統錯誤處理程式。您無法在使用者設定的錯誤處理程式中攔截失敗的交易異常狀況。對於同步模式而言，交易異常狀況會透過回應傳回。對於非同步模式而言，其中的交易是設計在要求中確認，異常狀況會在要求繫線中傳回。
請注意在非同步模式中，在要求中交易確認後所發生的回應錯誤不會影響交易。
回應使用相同交易
此選項僅適用於單向和非同步訊息傳遞模式。

如果選取此選項，Oracle Service Bus 會從要求繫線傳播交易相關資訊環境到回應繫線。
如果您選取這個選項，訊息模式會自動變成同步，忽略初始訊息模式的設定 (例如非同步或單向)。
例如，如果在傳送回應 (例如以單向模式) 之前，在要求中的業務服務需要交易確認，則不該使用這個選項。
針對交易逾時，會套用在 Oracle WebLogic Server 主控台中所設定的全域交易逾時值。
內容串流
選取此選項以串流方式傳送訊息內容，而不是將內容儲存在記憶體中。
選取「已啟用」核取方塊，然後選擇下列項目：
  • 緩衝處理記憶體 (「記憶體緩衝區」) 中的中繼內容，還是緩衝到磁碟檔案 (「磁碟緩衝區」)
  • 是否啟用「壓縮」
如需詳細資訊，請參閱「Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus」中的「Streaming body Content」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBAG314。
XOP/MTOM 支援
Oracle Service Bus 支援使用下列傳輸的 XOP/MTOM：
  • HTTP/S
  • 本機
  • SB
在適當情況下選取已啟用核取方塊，可讓代理主機服務以 MTOM/XOP 格式解碼及剖析內送訊息，以及使用 MTOM/XOP 格式傳送回應。請注意，對於以舊版組態為基礎的已匯入代理主機服務，將會停用此選項。
如果已啟用「XOP/MTOM 支援」，請從下列選項中，選取要如何處理 $header 與 $body 訊息相關資訊環境變數中的二進位資料：
  • 透過參照包含二進位資料：(預設值) 在設定 $header 與 $body 訊息相關資訊環境變數時，將內送要求訊息中的 xop:Include 元素取代為 ctx:binary-content 元素。
  • 透過值包含二進位資料：在設定 $header 與 $body 訊息相關資訊環境變數時，將內送要求訊息中的 xop:Include 元素取代為 Base64 編碼文字版的對應二進位資料。
當您需要直接存取二進位資料時，請使用「透過參照包含二進位資料」，例如在傳遞資料至 Java 外部呼叫，或進行訊息格式語言 (MFL) 轉換時。
在下列情況下，請使用「透過值包含二進位資料」：
  • 橋接 MTOM 與非 MTOM 服務。例如，假設有已啟用 MTOM 的代理主機服務收到要求，而接著將路由至未啟用 MTOM 的服務。您可以使用此選項，以符合將 XML 中的二進位資料以 Base64 編碼格式傳送的現有標準。
  • 針對必須使用 base64binary 元素 (代替二進位資料) 的 XML 綱要驗證訊息內容
請注意，如果已為代理主機服務啟用「XOP/MTOM 支援」，內送訊息則不一定全都必須使用 MTOM 格式。此設定會指定，在 MTOM 格式的訊息送達時，代理主機服務應據以處理。另請注意，當未針對「XOP/MTOM 支援」啟用的代理主機服務收到 MTOM 格式的訊息時，服務將會拒絕訊息，並發出執行時期錯誤。
附件
Oracle Service Bus 支援以下列傳輸以串流傳送 MIME 附件。
  • HTTP/S
  • 本機 (透過已啟用串流附件的 HTTP 代理鏈結時)
選取將附件分頁至磁碟核取方塊，讓代理主機服務以串流傳送 MIME 附件。針對 HTTP 代理主機服務啟用時，此選項會套用至代理主機服務內傳要求訊息。
請注意，當您選取「XOP/MTOM 支援」時，您必須在「XOP/MTOM 支援」下選擇透過參照包含二進位資料選項後，才可使用「附件」選項。另請注意，包含附件的有效負載必須符合 RFC 822。明確而言，包含網際網路標頭的字行，必須以 CRLF 終止 (換行字元)。
