訊息類型組態頁面

「建立/編輯代理主機服務 - 訊息類型組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

使用此頁面可以設定類型為「訊息傳遞服務」之代理主機服務的訊息類型。

訊息傳遞服務的連結定義包含設定所交換之訊息的 content-type。回應和要求的 content-type 不需要相同；因此，會個別設定回應 (例如，服務可以接受 MFL 訊息，而傳回 XML 認可回條)。

注意： 類型為「訊息傳遞服務」的電子郵件、檔案、FTP 或 SFTP 傳輸代理主機服務，只支援單向訊息傳遞；回應訊息類型應該是 none。如果選取 none 以外的選項，則「傳輸組態」頁面上不會有 E-mail、File、FTP 或 SFTP 通訊協定。

表 19-2 描述如何使用「建立/編輯代理主機服務 - 訊息類型組態」頁面。

表 19-2 建立/編輯代理主機服務 - 訊息類型組態頁面 
選項
建立或編輯...
要求訊息類型
選取要求訊息的訊息類型：
  • 無 - 如果沒有要求訊息，請選取此選項。
  • 二進位 - 如果訊息的 content-type 不明或不重要，請選取此選項。
  • 文字 - 如果訊息可以限制為文字，請選取此選項。
  • MFL - 如果訊息是符合 MFL 定義的二進位文件，請選取此選項。輸入 MFL 檔案名稱 (以其路徑限定)，或按一下「瀏覽」選取檔案。

    • 您只可以設定一個 MFL 檔案。

    注意：若要支援多個 MFL 檔案，請將內容定義為二進位或文字，並在訊息流程中使用 MFL 動作以轉換為 XML。

  • XML - 如果訊息是 XML 文件，請選取此選項。輸入 XML 檔案名稱 (以其路徑限定)，或按一下「瀏覽」選取檔案。

    • 選擇性地在「element or type」欄位中宣告所交換 XML 文件的 XML 綱要類型，以提供部分類型資訊。

  • Java - 如果在要求中傳送 Java 物件，請選取此選項。JMS 傳輸專供 Java 物件訊息使用。有關詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Sending and Receiving Java Objects in Messages"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。
回應訊息類型
選取回應訊息的訊息類型：
  • 無 - 如果沒有回應訊息，請選取此選項。
  • 二進位 - 如果訊息的 content-type 不明或不重要，請選取此選項。
  • 文字 - 如果訊息可以限制為文字，請選取此選項。
  • MFL - 如果訊息是符合 MFL 定義的二進位文件，請選取此選項。輸入 MFL 檔案名稱 (以其路徑限定)，或按一下「瀏覽」選取檔案。

    • 您只可以設定一個 MFL 檔案。

    注意：若要支援多個 MFL 檔案，請將內容定義為二進位或文字，並在訊息流程中使用 MFL 動作以轉換為 XML。

  • XML - 如果訊息是 XML 文件，請選取此選項。輸入 XML 檔案名稱 (以其路徑限定)，或按一下「瀏覽」選取檔案。

    • 選擇性地在「element or type」欄位中宣告所交換 XML 文件的 XML 綱要類型，以提供部分類型資訊。

  • Java - 如果在回應中收到 Java 物件，請選取此選項。JMS 傳輸專供 Java 物件訊息使用。有關詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Sending and Receiving Java Objects in Messages"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1874。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
