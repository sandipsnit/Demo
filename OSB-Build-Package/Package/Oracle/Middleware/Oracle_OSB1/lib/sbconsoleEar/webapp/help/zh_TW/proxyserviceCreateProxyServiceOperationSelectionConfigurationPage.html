作業選取組態頁面

建立/編輯代理主機服務 - 作業選擇組態頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。只有在建立的服務具有作業時，才會顯示此頁面。

使用此頁面可以強制 WS-I 相容 (僅適用於 SOAP 1.1 服務)，以及選取用來判斷此代理主機服務所呼叫之作業的選取演算法。此選項只適用於從 WSDL 定義的 SOAP 或 XML 服務。

WSDL 規格會定義預設演算法，以根據接收到之 SOAP 訊息的類型來計算要呼叫的作業。然而，有些情況 (例如，效能問題、簽章/加密問題或預設演算法不適用) 可能會需要根據其他方法來選取作業。

Oracle Service Bus 提供其他演算法。這些演算法都遵循相同的模式，也就是先用表示式求值，然後再根據這個值查詢靜態表格中的對應作業。

如果內送訊息遺漏資料而無法判斷作業，或具有的資料未對應至有效的作業，Oracle Service Bus 通常都會容許這些狀況。上述這兩種狀況都會導致 $operation 變為空的。除非拒絕所有這類訊息，否則 Oracle Service Bus 不會初始化相關資訊環境中的 operation 變數，而是會繼續處理該訊息。

然而，如果代理主機服務是 WSDL 型，而且至少符合下列其中一個條件，則會強制安全性需求：

  • WSDL 具有 WS 安全原則，而代理是作用中媒介。
  • 代理具有訊息層級自訂認證 (自訂記號或使用者名稱/密碼)。

如果符合這些條件，則會進行執行時期檢查，以確定作業選取演算法傳回的是有效的作業名稱。如果作業選取傳回空值或不在 WSDL 中的作業，則會拒絕訊息，並引發錯誤。

強制 WS-I 相容
僅限 SOAP 1.1 服務：
如果想要指定服務是否要符合「Web 服務互操作性組織」所定義的「基本設定檔」，請選取此核取方塊。

選取演算法
選取下列其中一項，並執行任何必要額外步驟：
  • 傳輸標頭 - 選取此演算法，以定義含有查詢值的傳輸標頭。然後：
    • 在「標頭名稱」欄位中輸入傳輸標頭，以擷取該標頭中的值做為索引鍵來選取所要叫用的作業。
    • 在作業對應欄位下的值欄位中，指定每個作業的值。此值是做為作業的索引鍵。
  • SOAPAction 標頭 - 選取此演算法，指定從此代理主機服務的關聯 WSDL 自動完成作業對應。
  • WS 定址 - 選取此演算法，指定查詢值是由 WS 定址 Action 標記所包含，而此標記位在 SOAP 訊息的 SOAP 標頭中。然後，在作業對應欄位的值欄位中，指定每個作業的值。此值是做為作業的索引鍵。
  • SOAP 標頭 - 選取此演算法，定義根據 SOAP 標頭來評估 XPath 表示式。這可讓您取得查詢值。然後：
    • 在「XPath 表示式」欄位中指定 XPath 表示式，以擷取該表示式中的值做為索引鍵來選取所要叫用的作業。
    • 在作業對應欄位下的值欄位中，指定每個作業的值。此值是做為作業的索引鍵。
  • SOAP 主體類型 - 這是以 WSDL 規範定義的預設演算法，會根據接收到的 SOAP 訊息類型計算要呼叫的作業。

    • 如果代理主機服務是設定給具有加密主體的 Web 服務安全傳遞案例，則無法選取此演算法。類似的警告適用於傳遞加密 SOAP 標頭。

    如果您擁有的 WSDL 具有輸入訊息相同的兩個作業，則請不要針對這些作業選取此演算法，因為檢查輸入訊息並無法唯一識別作業。

  • 有效負載類型 - 僅適用於根據 WSDL 連接埠或 WSDL 連結的 XML 服務。
標頭名稱

輸入傳輸標頭，以解壓縮該標頭中的值做為索引鍵來選取要叫用的作業。
XPath 表示式
只有在「選取演算法」選項設為「SOAPHeader」時，才可以使用此選項。
指定 XPath 表示式，以解壓縮該表示式中的值做為索引鍵來選取所要叫用的作業。
作業對應
只有在「選取演算法」選項設為「傳輸標頭」、「WS 定位」或「SOAP 主體類型」時，才可以使用此選項。
在值欄位中，指定每個作業的值。此值是做為作業的索引鍵。
