Tuxedo 傳輸組態頁面

「建立/編輯代理主機服務 - Tuxedo 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「Tuxedo Transport」，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1200。

使用此頁面，可以使用 Tuxedo 傳輸通訊協定來設定代理主機服務的傳輸設定。表 19-14 說明如何使用此頁面。

表 19-14 建立/編輯代理主機服務 - Tuxedo 傳輸組態頁面 
選項
建立或編輯...
欄位表格類別
輸入用來說明所接收之 FML/FML32 緩衝區的類別名稱。這些是用於 FML/FML32-to-XML 轉換常式，以將欄位名稱對應到元素名稱。這是完整類別名稱的清單 (以空格來分隔各個名稱)。
檢視畫面類別
輸入用來說明所接收之 VIEW/VIEW32 緩衝區的類別名稱。這些是用於 VIEW-to-XML 或 VIEW32-to-XML 轉換常式，以將欄位名稱對應到元素名稱。這是完整類別名稱的清單 (以空格來分隔各個名稱)。
X_C_TYPE 和 X_COMMON Tuxedo 緩衝區類型的處理方式都與 VIEW/VIEW32 緩衝區相同。
如果傳入要求含有 VIEW，則應該在 Oracle Service Bus CLASSPATH 中指定對應的 VIEW 類別。
類別 Jar
包含 JAR 檔案的 JAR 資源，此 JAR 檔案具有此端點作業所需要的 FML/FML32 或 VIEW/VIEW32 類別。
本機存取點
從匯出的相關下拉清單中選取本機存取點。此下拉清單含有 WTC 中所設定的本機存取點。如果沒有相關聯的本機存取點，則不會建立代理主機服務。
如果沒有本機存取點，或要建立新的存取點，請選取新增。在相鄰的欄位中，輸入對應的本機存取點名稱與本機網路位址。端點通過驗證後，存取點即會新增至每部 WTC 伺服器的 WTC 組態中。如果不存在 WTC 伺服器，則會建立一部 WTC 伺服器。
選取新增選項之後，還是可以輸入現有的存取點名稱。這樣會使用新的參數更新現有的資訊。您只可以變更主機名稱和連接埠號。
遠端存取點
只有在「本機存取點」欄位中選取「新增」時，才會出現此欄位。
從下拉清單中選擇與新建立本機存取點相關聯的遠端存取點。如果不存在存取點，或要建立新的存取點，請選取新增。在相鄰的欄位中，輸入對應的存取點名稱與網路位址。
選取新增選項之後，還是可以輸入現有的存取點名稱。這樣會使用新的參數更新現有的資訊。您只可以變更主機名稱和連接埠號。
遠端存取點也會是內送要求之 WTC 連線的認證委派人。您可以選擇性地使用預設安全範圍中的相同存取點 ID 來建立使用者，以允許傳入呼叫。做法是「是」(位在「建立使用者?」下拉清單中)。密碼會使用暫時變數隨機產生，以避免安全問題。
回覆緩衝區類型
只有在選取「是否需要回應?」欄位時，才可以使用此選項。
選取遠端 Tuxedo 從屬端所接收的緩衝區類型。
回覆緩衝區子類型
只有在選取「是否需要回應?」選項且「回覆緩衝區類型」值是「VIEW」或「VIEW32」時，才可以使用此選項。
請輸入與回覆緩衝區相關聯的緩衝區子類型。
是否需要回應?
如果此服務需要傳送回應，請選取此核取方塊。
預設狀態是選取此選項。
如果服務類型是「訊息傳遞服務」且回應訊息類型是「無」，則會清除此選項且無法使用。
要求編碼
指定 Tuxedo 傳輸中要求的字元集編碼。
回應編碼
指定 Tuxedo 傳輸中回應的字元集編碼。
轉換樣式
選取下列其中一項：
  • 無 - (預設值) 可能不會顧及欄位的順序。
  • 已排序 - 會以正確的順序顯示欄位及其所有出現次數。
  • 排序並分組 - 如果欄位已邏輯結構化為記錄，則會以出現順序排序欄位，並依記錄進行分組。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
