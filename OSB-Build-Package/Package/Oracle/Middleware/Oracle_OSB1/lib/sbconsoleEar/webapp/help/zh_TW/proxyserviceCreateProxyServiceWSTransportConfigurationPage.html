WS 傳輸組態頁面

「建立/編輯代理主機服務 - WS 傳輸組態」頁面是一系列頁面中的其中一頁，供您建立和編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「WS Transport」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1083。

使用此頁面，可以使用 WS 傳輸通訊協定來設定代理主機服務的傳輸設定。表 19-15 說明如何使用此頁面。

表 19-15 建立/編輯代理主機服務 - WS 傳輸組態頁面 
選項
建立或編輯...
分配原則
選取此端點的分配原則，或使用預設分配原則。
分配原則指的是您想用於服務端點的 WLS 工作管理程式執行處理。如需工作管理程式的相關資訊，請參閱 Oracle Fusion Middleware Configuring Server Environments for Oracle WebLogic Server 中的 "Using Work Managers to Optimize Scheduled Work"，網址為： http://www.oracle.com/pls/as111170/lookup?id=CNFGD112。
重試計數
將訊息傳遞給管線的重試次數。
如果代理主機服務的要求管線發生未處理的異常狀況，則會將傳入 WS 傳輸訊息重新傳遞給管線，而重新傳遞的次數上限就是重試計數所指定的次數。若要可靠地處理 WS 傳輸訊息，則此值十分重要。
重試延遲
在發生錯誤之後，系統重新試著將訊息傳送給管線之前的暫停秒數。

完成作業後

按「下一頁」繼續在下一個頁面上設定此服務，或按一下「最後一頁」在建立/編輯代理主機服務 - 摘要頁面上檢查並儲存這項組態。
