在 XQuery 編輯器中建立變數結構

XQuery 編輯器和 XPath 編輯器中的變數結構面板會以圖形呈現方式顯示變數的內容。預設會包含內建訊息相關資訊環境變數 attachments、body、header、outbound 和 inbound。

每個變數結構對應項目都有一個標籤，且會將一或多個變數路徑對應至一或多個結構。這些對應的領域就是階段或路由節點。

您也可以根據下列項目來宣告自己的變數結構：

  • XML 類型，包括：
    • 綱要元素
    • WSDL 元素
    • 綱要類型
    • WSDL 類型
    • MFL
  • 服務介面
  • 簡單類型 (字串或任何 XML)

此功能可以直接用於所有使用者定義的變數，以及 $inbound、$outbound 和 $fault。但是無法直接用它來存取 $attachments 中的 XML 附件、$header 中的標頭或是 $body 中的文件和 RPC 參數，唯一的例外是，如果對象是 WSDL 代理主機服務接收到的要求訊息，則可以直接使用這項功能存取 $body 中的文件和參數。

開始之前

這些指示假設您在「XQuery 表示式編輯器」、「XQuery 條件編輯器」或「XPath 表示式編輯器」中建立或編輯表示式 (如建立和編輯內置 XQuery 和 XPath 表示式中所述)。

若要建立變數結構：

  1. 選取「變數結構」。
  2. 在「變數結構」面板中，按一下「新增結構」。
  3. 繼續進行下面表 22-3 中所列的所有工作。

    4. 表 22-3 建立新的變數結構
    若要...
    請完成這些步驟...
    建立將變數對應至 XML 綱要類型的變數結構
    1. 選取「變數結構」面板頂端的「XML 類型」(如果尚未選取)。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在「結構路徑」欄位中，輸入變數結構在執行時期時的路徑。路徑的開頭必須是 $。
    4. 在「類型」欄位下選取適當的單選按鈕，然後選取「綱要類型」。
    5. 按一下瀏覽。隨即顯示 XML 綱要瀏覽器。從清單中選取「XML 綱要」，並從「定義」窗格中選取 XML 綱要類型，然後按一下提交。
    6. 按一下「新增」，以建立變數結構。
    建立將變數對應至 WSDL 類型的變數結構
    1. 選取「變數結構」面板頂端的「XML 類型」(如果尚未選取)。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在「結構路徑」欄位中，輸入變數結構在執行時期時的路徑。
    4. 在「類型」欄位下選取適當的單選按鈕，然後選取「WSDL 類型」。
    5. 按一下瀏覽。隨即顯示 WSDL 瀏覽器。從 WSDL 清單中選取 WSDL，並從「定義」窗格中選取 WSDL 類型，然後按一下提交。
    6. 按一下「新增」，以建立變數結構。
    建立將變數對應至 XML 綱要元素的變數結構
    1. 選取「變數結構」面板頂端的「XML 類型」(如果尚未選取)。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在「結構路徑」欄位中，輸入變數結構在執行時期時的路徑。
    4. 在「類型」欄位下選取適當的單選按鈕，然後選取「綱要元素」。
    5. 按一下瀏覽。隨即顯示 XML 綱要瀏覽器。從清單中選取「XML 綱要」，並從「定義」窗格中選取 XML 綱要類型，然後按一下提交。
    6. 按一下「新增」，以建立變數結構。
    建立將變數對應至 WSDL 元素的變數結構
    1. 選取「變數結構」面板頂端的「XML 類型」(如果尚未選取)。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在「結構路徑」欄位中，輸入變數結構在執行時期時的路徑。
    4. 在「類型」欄位下選取適當的單選按鈕，然後選取「WSDL 元素」。
    5. 按一下瀏覽。隨即顯示 WSDL 瀏覽器。從 WSDL 清單中選取 WSDL，並從「定義」窗格中選取 WSDL 元素，然後按一下提交。
    6. 按一下「新增」，以建立變數結構。
    建立將變數對應至子元素的變數結構
    1. 選取「變數結構」面板頂端的「XML 類型」(如果尚未選取)。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在「結構路徑」欄位中，輸入變數結構在執行時期時的路徑。
    4. 在「類型」欄位下選取想要建立的變數類型：

      5. 若要建立「XML 綱要元素」或「WSDL 元素」變數，請選取與此選項相關聯的單選按鈕，然後選取「綱要元素」或「WSDL 元素」。

      若要建立 MFL 變數，請選取與此選項相關聯的單選按鈕，然後選取「MFL」。

    5. 如果是 XML 綱要、WSDL 或 MFL，請按一下瀏覽以從瀏覽器顯示的清單中選取物件，然後按一下提交。例如，從 MFL 清單中選取 MFL，然後按一下提交。
    6. 選取「設為子項」核取方塊以將元素設為所要建立結構的子項。
    7. 按一下「新增」，以建立變數結構。
    建立使用 MFL 資源的變數結構
    1. 選取「變數結構」面板頂端的「XML 類型」(如果尚未選取)。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在「結構路徑」欄位中，輸入變數結構在執行時期時的路徑。
    4. 在類型欄位下選取適當的單選按鈕，然後按一下瀏覽。隨即顯示 MFL 瀏覽器。
    5. 從 MFL 清單中選取 MFL，然後按一下「提交」。
    6. 按一下「新增」，以建立變數結構。
    建立「服務介面」變數結構
    1. 選取「變數結構」面板頂端的「服務介面」。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在結構路徑欄位中，已將預設值設為 $body。您無法變更此欄位。
    4. 在「WSDL 型服務」欄位中選取「服務瀏覽器」圖示，並從「服務瀏覽器」所顯示的服務清單中選取服務，然後按一下「提交」。

      5. 選取的服務會顯示在「WSDL 型服務」欄位中。

    5. 在「作業」欄位中選取作業，或選取「無」不要包含作業。
    6. 按一下「新增」，以建立變數。
    建立「簡單」變數結構
    1. 選取「變數結構」面板頂端的「簡單類型」。
    2. 在結構標籤欄位中，輸入想要建立之變數的顯示名稱。此顯示名稱可讓您提供有意義的結構名稱，讓您可以在設計時期辨識它，但是並不會影響執行時期。
    3. 在「結構名稱」欄位中，輸入想要建立之變數結構的名稱。
    4. 在「類型」欄位下選取「字串」或「任何 XML」。
    5. 按一下「新增」，以建立變數。

完成作業後

請如建立和編輯內置 XQuery 和 XPath 表示式所述繼續進行。
