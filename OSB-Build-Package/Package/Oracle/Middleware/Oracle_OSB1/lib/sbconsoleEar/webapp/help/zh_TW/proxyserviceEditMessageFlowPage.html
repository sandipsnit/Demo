編輯訊息流程頁面

使用「編輯訊息流程」頁面，以建構代理主機服務的訊息流程。

編輯訊息流程頁面的左瀏覽窗格會顯示訊息流程中之節點和物件的樹狀圖檢視畫面。物件詳細資訊定義於不同頁面時，可以按一下要顯示相關頁面的物件名稱。

右窗格則會提供做為訊息流程建構依據的欄位。尚未定義訊息流程時，此窗格會包含單一代理主機服務圖示，表示服務的開始節點。按一下此圖示以新增服務的管線組節點、路由節點、條件分支、作業分支和錯誤處理。

在頁面中新增物件時，頁面中會顯示代表物件的圖示。而物件之間的關係則會利用線條和週框方塊顯示。按一下編輯訊息流程頁面中的圖示，即可顯示可以在該物件執行的動作功能表。功能表中的可用選項可能會因相關資訊環境而不同。如需完整的圖示和選項清單，請參閱表 20-1。

管線組節點
管線組節點是由一個要求管線和一個回應管線所組成。
  • 編輯名稱和註解
  • 新增管線組 - 請參閱將管線組節點新增至訊息流程。
  • 新增條件分支 - 請參閱將條件分支新增至訊息流程。
  • 新增作業分支 - 請參閱將作業分支新增至訊息流程。
  • 貼上路由 - 只有在剪下或複製路由節點，而且該路由節點已在剪貼簿上時，才可以使用此選項。

請參閱上面的管線組節點。
  • 新增階段 - 請參閱將階段新增至管線。
  • 新增管線錯誤處理程式 - 請參閱新增管線錯誤處理程式。

路由節點
路由節點動作會定義訊息在流經路由節點時的處理方式。

條件分支節點
條件分支節點可讓處理作業在多個可能的路徑當中，挑選一個路徑繼續進行。
  • 編輯分支 - 請參閱將條件分支新增至訊息流程。
  • 編輯名稱和註解
作業分支節點
作業分支節點會根據指定的作業來判斷要遵循的分支。

  • 編輯名稱和註解

分支節點
分支節點是一種由條件分支節點或作業分支節點定義的替代節點。
  • 新增管線組 - 請參閱將管線組節點新增至訊息流程。

錯誤處理程式
錯誤處理程式提供在訊息流程中重新傳送錯誤的邏輯。
