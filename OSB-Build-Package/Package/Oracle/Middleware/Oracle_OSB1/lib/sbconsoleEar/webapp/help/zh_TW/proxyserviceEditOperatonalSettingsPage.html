設定代理主機服務的作業設定

使用作業設定頁面可讓您啟用和停用代理主機服務，並監控特定代理主機服務的監控。同樣地，您也可以在作業 > 智慧搜尋頁面中指定這些相同服務特有的作業設定，但是您無法設定彙總間隔，也無法指定警示或記錄的嚴重度層級。有關詳細資訊，請參閱尋找和更新作業設定。

服務層級設定的執行時期效果取決於其對應的全域設定。您必須同時啟用服務的全域設定與服務層級設定，才能在執行時期完全啟用服務。此外，也必須啟用服務狀態。請參閱啟用全域設定。

即使服務不會在執行時期啟用，您一樣可以變更和儲存監控組態。例如，即使服務監控已停用，您一樣可以變更和儲存「彙總間隔」。因此，您可以先編輯設定，稍後再啟用。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「資源瀏覽器」>「代理主機服務」。
  3. 按一下代理主機服務名稱。

    4. 檢視代理主機服務頁面會顯示表 25-6 中所示的資訊。

    表 25-6 代理主機服務資訊 
    特性
    描述
    上次修改者
    建立或編輯此服務的使用者，或將此服務匯入組態中的使用者。
    上次修改時間
    使用者建立或編輯此服務或將其匯入組態中的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱檢視變更歷史記錄頁面。
    參照
    此服務參照的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    參照者
    參照此服務的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    描述
    服務的描述 (如果有的話)。

  4. 選取「作業設定」頁籤。

    5. 您必須在作用中階段作業中，才能設定表 25-7 中所示的代理主機服務作業設定。

    表 25-7 設定代理主機服務作業設定
    若要...
    請...
    停用「狀態」。
    清除已啟用核取方塊。啟動階段作業時，服務會停止處理訊息。
    停用服務不會再導致系統刪除先前為服務收集的所有監控統計資訊。
    啟用「狀態」
    選取已啟用核取方塊。啟動階段作業時，服務會恢復處理訊息。
    停用「監督」
    清除已啟用核取方塊。當您啟動階段作業時，系統會停止收集服務的監控統計資訊，並刪除先前收集的任何統計資訊。
    啟用「監督」
    選取監控旁邊的已啟用核取方塊。從啟用管線監控於下拉清單中，選取將監控限制為指定的層級或更高層級 (從最寬鬆到最受限層級依序排列)：動作、管線和服務。
    當您啟動階段作業時，系統會開始收集服務的監控統計資訊。
    設定服務的「彙總間隔」。
    選取間隔的小時或分鐘數，然後按一下更新。如果選擇的時數超過 1，則預設的分鐘選擇一律為零。但如果選擇的時數為 0 或 1，則可設定以分鐘表示的間隔。請參閱設定服務的彙總間隔。
    不論是否已啟用監督，您都可以變更和儲存彙總間隔設定。
    若要在執行時期啟用監督，請確定已選取「啟用監督」核取方塊。
    停用「SLA 警示」
    清除已啟用核取方塊。系統會停止評估任何為服務設定的警示規則；因此，您不會再收到服務相關的 SLA 警示。
    啟用「SLA 警示」
    選取已啟用核取方塊。從啟用警示於下拉清單中，選取將 SLA 警示限制於指定的層級或更高層級 (從最寬鬆到最受限層級依序排列)：正常 (預設值)、警告、次要、主要、危急以及嚴重。
    服務 SLA 警示取決於全域與服務層級的監控狀態。亦即，必須啟用這兩者，才能在執行時期啟用 SLA 警示。請參閱啟用全域 SLA 警示。
    停用「管線警示」
    清除已啟用核取方塊。啟動階段作業時，系統會停止執行管線警示動作；因此，您不會再收到服務相關的管線警示。
    啟用「管線警示」
    選取已啟用核取方塊。從啟用警示於下拉清單中，選取將管線警示限制在指定的嚴重性層級或更高層級 (從最寬鬆到最受限層級依序排列)：正常 (預設值)、警告、次要、主要、危急以及嚴重。
    停用代理主機服務的報告動作
    清除訊息報告旁邊的已啟用核取方塊。啟動階段作業時，系統會停止執行報告動作；因此，您不會再收到服務相關的訊息報告。
    啟用代理主機服務的報告動作
    選取「訊息報告」旁邊的「已啟用」核取方塊。
    停用記錄管線登入動作的輸出
    清除記錄旁邊的已啟用核取方塊。啟動階段作業時，系統會停止執行記錄動作；因此，您不會再收到服務相關的記錄輸出。
    啟用記錄管線日誌動作的輸出
    選取記錄旁邊的已啟用核取方塊。從啟用記錄於下拉清單中，選擇將記錄輸出限制為指定的層級或更高層級 (從最寬鬆到最受限層級依序排列)：除錯 (預設)、資訊、警告及錯誤。
    啟用代理主機服務的執行時期執行追蹤

    選取執行追蹤旁邊的已啟用核取方塊。啟用執行追蹤之後，系統就會記錄揀選自「訊息流程相關資訊環境」和「訊息相關資訊環境」的不同詳細資訊。這些詳細資訊包括：階段名稱、管線或路由節點名稱，以及目前的「訊息相關資訊環境」。

    注意：若要查看日誌檔或標準輸出 (伺服器主控台) 中的追蹤，必須將 Oracle WebLogic Server 記錄設定成下列嚴重程度層級：

    要記錄的最低嚴重程度：資訊
    日誌檔：資訊
    標準輸出：資訊

    如需設定日誌檔嚴重程度層級的資訊，請參閱 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的 Using Log Severity Levels，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

    停用代理主機服務的執行時期追蹤
    清除「執行追蹤」旁邊的「已啟用」核取方塊。
    啟用代理主機服務的訊息追蹤

    選取訊息追蹤旁邊的已啟用核取方塊。啟用訊息追蹤之後，系統會記錄 Oracle Service Bus 管線與代理主機服務之間交換的訊息 (內傳要求與回應，以及外傳要求和回應訊息)。

    注意：若要查看日誌檔或標準輸出 (伺服器主控台) 中的追蹤，必須將 Oracle WebLogic Server 記錄設定成下列嚴重程度層級：

    要記錄的最低嚴重程度：資訊
    日誌檔：資訊
    標準輸出：資訊

    如需設定日誌檔嚴重程度層級的資訊，請參閱 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的 Using Log Severity Levels，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

     

    請注意，在適用時，已記錄的外送訊息亦可包含重試次數、錯誤碼與錯誤訊息。
    在「詳細層級」下拉清單中，從下列項目中選擇指定詳細資訊層級：
    • 精簡 - 顯示日期、時間、服務 類型、服務名稱和 URI
    • 標頭 - 顯示精簡資訊，以及以 XML 呈現的訊息描述資料
    • 完整 - 顯示標頭資訊，以及原始有效負載 (包含附件，如果有的話)
    如果從詳細層級下拉清單中選擇「完整」，請使用有效負載追蹤限制欄位指定訊息有效負載的大小上限 (KB)。同時，使用預設編碼欄位，指定用來記錄有效負載的預設編碼。記錄二進位有效負載或具有二進位附件的 SOAP 訊息，這十分有用。
    預設的編碼值可以是 Base64 或任何 Java 支援的編碼，如下所定義： http://java.sun.com/javase/6/docs/technotes/guides/intl/encoding.doc.html。
    若將預設編碼欄位保留為空白，會使得 Oracle Service Bus 使用主機的預設編碼處理有效負載。預設編碼取決於 JVM、底層作業系統 (OS) 與 OS 層級語言地區設定的組合。
    若指定於「預設編碼」欄位中的設定無法使用 (例如不是適用於組態的有效選項)，Oracle Service Bus 會使用 Base64 編碼處理有效負載。
    停用代理主機服務的訊息追蹤
    清除「訊息追蹤」旁邊的「已啟用」核取方塊。
