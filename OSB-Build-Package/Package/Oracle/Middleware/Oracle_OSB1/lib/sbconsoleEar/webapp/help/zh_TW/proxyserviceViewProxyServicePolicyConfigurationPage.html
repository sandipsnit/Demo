代理主機服務原則頁面

「檢視代理主機服務 - 原則」頁面是一系列頁面中的其中一頁，供您編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

您可以在此頁面設定服務類型為「WSDL Web 服務」、「訊息傳遞服務」、「任何 SOAP 服務」或「任何 XML 服務」之代理主機服務的原則設定。 表 19-20 說明如何使用此頁面。為了能夠在「WSDL Web 服務」、「訊息傳遞服務」或「任何 XML 服務」代理主機服務使用 OWSM 原則，必須將通訊協定設為 HTTP。

在 WSDL 類型服務中，連結至服務的 WLS 9.2 原則會都會公開 (內置) 於有效的 WSDL 中。抽象原則在內置之前會先經過預先處理。OWSM 原則由參照連結，不會內置至有效的 WSDL 中。

服務原則組態
下列是「服務原則組態」欄位中的選項：
  • 從 OWSM 原則儲存庫 – 原則是在受 Oracle Enterprise Manager 管理的中央「Oracle Web Services Manager 原則儲存庫」中定義。這些原則支援 WS-Security 1.0/1.1、SAML 1.1/2.0、KerberosToken Profile 及其他業界標準。

    如有需要，請使用篩選來找出您想要的原則。在代理主機服務中，只會顯示服務原則。

    • 如需更多資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Securing Oracle Service Bus with Oracle Web Services Manager"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681。

  • 從 WSDL – 如果服務原則是與服務所依據的 WSDL 相關聯，請選取此選項。這些原則支援 WS-Security 1.0、SAML 1.1 及其他業界標準。

    • 使用這個選項可以檢視 (唯讀) 來自 WSDL 的要求及回應原則。

  • 從預先定義的原則或 WS-Policy 資源，您可以從「Oracle Service Bus 管理主控台」新增服務層級原則、作業層級原則 (這類原則會同時套用至要求和回應訊息)、要求原則集及回應原則。

    原則是在 Oracle WebLogic Server 中預先定義或是在包含 WS-Policy 資源的 Oracle Service Bus 中由使用者定義。這些原則支援 WS-Security 1.0、SAML 1.1 及其他業界標準。

    • 使用這個選項可以新增在服務、作業、要求及回應層級的原則。

注意： 原則連結模型是互斥的。您必須只能在服務中使用一種原則。如果將原則直接連結至服務，就會略過所有 WSDL 型原則。
