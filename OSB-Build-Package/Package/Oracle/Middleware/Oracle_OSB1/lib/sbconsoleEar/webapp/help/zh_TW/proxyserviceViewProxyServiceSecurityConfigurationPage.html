代理主機服務安全頁面

「檢視代理主機服務 - 安全性」頁面是一系列頁面中的其中一頁，供您編輯代理主機服務 (如建立和設定代理主機服務和編輯代理主機服務組態中所述)。

使用此頁面可以設定代理主機服務的安全設定。

在這個頁面中的欄位，取決於代理主機服務中所使用的原則。例如，如果代理主機服務使用 OWSM 原則 (建議)，原則覆寫會出現在這個安全頁面上。如需更多詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Securing Oracle Service Bus with Oracle Web Services Manager"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1681。

對於 WLS 9.2 原則，自訂使用者名稱/密碼和自訂記號的組態類似。在這兩種情況下，請指定可讓 Oracle Service Bus 找到所需資訊的 XPath 表示式。這些 XPath 表示式的根如下：

  • 如果服務連結是 AnySOAP 或 WSDL-SOAP，請使用 soap-env:Envelope/soap-env:Header。
  • 如果服務連結不是 SOAP 型，請使用 soap-env:Body。

所有 XPath 表示式都必須為有效的 XPath 格式。XPath 表示式必須使用 XPath "declare namespace" 語法，宣告所有使用的命名空間 (如下所示)：

declare namespace
ns='http://webservices.example.com/MyExampleService';)

Web 服務安全組態
處理 WS 安全標頭
如果從屬端要求包含 WS 安全性安全標頭，請決定是否要處理此標頭：
  • 選擇「是」：在作用中媒介案例中，從屬端會將 WS 安全性套用至要求和/或回應訊息。代理主機服務會處理安全標頭，並強制執行 WS 安全原則。當代理主機服務是以有安全原則的 WSDL 為基礎時或當您附加原則至「服務原則組態」頁面上的服務時，此選項會自動設為「是」。
  • 選取「否」：在 WS 安全性傳遞通道案例中，從屬端會將 WS 安全性套用至要求和/或回應訊息。代理主機服務不會處理安全標頭，而是將有此安全保護的要求訊息原封不動地傳遞至業務服務。
存取控制
傳輸存取控制
您可以針對所有代理主機服務來建立傳輸層級原則，以在從屬端試圖建立代理主機服務的連線時，套用安全檢查。唯有列在傳輸層級原則中的使用者要求，才能繼續進行。
訊息存取控制
當從屬端嘗試叫用具有訊息層級安全性的代理主機服務時，訊息層級存取控制原則會套用安全性檢查。您可以在下列情況中建立訊息層級存取控制原則：
  • 代理主機服務是作用中的 Web 服務安全媒介，亦即您會處理 WS 安全標頭。
  • 您已為代理主機服務在此頁面上設定自訂認證。
唯有列在訊息層級原則中的使用者，才能叫用作業。
自訂認證
(訊息層級)
認證類型
選取下列其中一項：
  • 如果代理主機服務不會使用自訂認證，請選取「無」。
  • 自訂使用者名稱和密碼

    • 如果選取此選項，則也必須在「使用者名稱 XPath」欄位和「使用者密碼 XPath」欄位中輸入值。

  • 自訂記號

    • 如果選取此選項，則也必須選取「記號類型」，並在「記號 XPath」欄位中輸入值。

使用者名稱 XPath
只有在自訂認證設定選項設為自訂使用者名稱和密碼時，才可以使用此選項。可用時，這是必要選項。
請輸入使用者名稱做為 XPath 表示式。
XPath 表示式會依需要根據訊息標頭或有效負載進行評估，讓 Oracle Service Bus 可以取得使用者名稱進行自訂認證。
使用者密碼 XPath
只有在自訂認證設定選項設為自訂使用者名稱和密碼時，才可以使用此選項。可用時，這是必要選項。
請輸入密碼做為 XPath 表示式。
XPath 表示式會依需要根據訊息標頭或有效負載進行評估，讓 Oracle Service Bus 可以取得密碼值進行自訂認證。
記號類型
只有在自訂認證設定選項設為自訂記號時，才可以使用此選項。可用時，這是必要選項。
從下拉清單中選取記號類型。只可以使用設定給 WebLogic Server 識別宣告提供者的作用中記號類型。請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「Configuring Identity Assertion Providers for Custom Tokens」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1660。
記號 XPath
只有在自訂認證設定選項設為自訂記號時，才可以使用此選項。可用時，這是必要選項。
請輸入 XPath 表示式，以指定自訂記號的路徑。Oracle Service Bus 會依需要根據訊息標頭或有效負載來評估「記號 XPath」表示式，以取得供自訂認證的記號。
相關資訊環境特性
選擇性地指定一個或多個相關資訊環境特性，將其他相關資訊環境資訊傳遞給「認證」(自訂使用者名稱和密碼) 或「識別宣告」(自訂記號) 安全提供者。
「相關資訊環境特性」提供的方法 (ContextHandler 介面) 可以將其他資訊傳遞給「WebLogic 安全架構」，這樣安全提供者就可以取得相關資訊環境資訊。請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「Context Properties Are Passed to Security Providers」，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1500。
在「特性名稱」中輸入常值字串，而在「值選取器」中輸入有效的 XPath 表示式 (XPath 表示式也可以是常值字串)。
XPath 表示式是根據用於自訂記號或自訂使用者名稱/密碼的相同訊息部分來進行評估。亦即，SOAP 型代理主機服務的「值選取器」XPath 表示式是根據標頭以及根據非 SOAP 型代理主機服務的有效負載進行評估。
XPath 表示式是在執行時期進行評估，以產生特性值。ContextHandler 基本上是名稱/值清單，就這一點而言，安全提供者需要瞭解所要尋找的名稱。因此，只有在安全提供者要求其中一個使用者定義特性的值時，才會評估 XPath 表示式。
按一下新增特性，新增此相關資訊環境特性。您可以新增多個相關資訊環境特性。
