瞭解 XQuery 編輯器配置和工作

「XQuery 表示式編輯器」、「XQuery 條件編輯器」和「XPath 表示式編輯器」各由下列元件組成：

  • 面板
  • 工作區
  • 特性檢查器

面板

每個編輯器的左面板都含有下面所列的面板。在任何編輯器中，按一下面板的名稱以顯示該面板。編輯器的每個面板都含有您可以插入至表示式的實體。

  • 命名空間定義面板會列出預設 Oracle Service Bus 命名空間、變數命名空間和使用者定義的命名空間。您可以定義新命名空間，然後將新命名空間新增至使用者定義的命名空間清單中。

    • 有關詳細資訊，請參閱：

    • 在編輯器工作區文字欄位中建置表示式
    • 建立用於內置表示式的命名空間
  • 「XQuery 函數」面板會列出一組標準 XQuery 函數。當您將函數插入至表示式時，會使用預留位置來表示您必須提供的參數值。

    • 有關詳細資訊，請參閱：

    • 在編輯器工作區文字欄位中建置表示式
    • 在 XQuery 編輯器中建立自訂 XPath 函數
    • 將外部 XQuery 資源連結至內置 XQuery
  • 「變數結構」面板提供一組使用 XPath 表示式來插入變數和變數路徑的工具。

    • 變數結構是以圖形方式呈現編輯器中所顯示的變數或變數路徑。它們可協助您視覺化變數結構，而且可以用來建構內置 XQuery 表示式，以參照變數內容。

    注意： 變數結構並不會建立變數。變數是在執行時期建立，做為階段中的「指派」動作目標。

    Oracle Service Bus 提供數個預先定義的訊息相關資訊環境變數 (attachments、body、header、outbound 和 inbound)，而您可以將它們的內容顯示為變數結構。您也可以定義自己的變數結構。

    如需詳細資訊，請參閱

    • 在編輯器工作區文字欄位中建置表示式
    • 在 XQuery 編輯器中建立變數結構

工作區

頁面的右邊會提供用來建構 XQuery 表示式、XQuery 條件或 XPath 的工作區。這三種編輯器的工作區會不同。

特性檢查器

在這三個編輯器中，特性檢查器會顯示在頁面的右下方。如果您從其中一個面板選擇項目以新增至表示式，則該項目會顯示在「特性檢查器」中。然後，您可以將該項目貼入工作區中。請參閱在編輯器工作區文字欄位中建置表示式。
