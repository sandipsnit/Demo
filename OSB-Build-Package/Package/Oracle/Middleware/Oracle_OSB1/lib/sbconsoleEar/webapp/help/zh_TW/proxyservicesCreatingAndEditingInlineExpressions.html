建立和編輯內置 XQuery 和 XPath 表示式

在編輯階段組態頁面上新增階段或路由節點的動作時，會在頁面上顯示提示組態詳細資訊的樣板結構。請參考圖 22-1 中的範例。

圖  22-1 動作組態樣板範例

只要相關資訊環境適當，樣板就會提供用來存取 XQuery 編輯器的連結，而您可以在其中建構以內置方式執行的表示式 (如動作中的相關資訊環境所需要)。

開始之前

這些指示假設您在管線階段、錯誤處理程式階段或路由節點的編輯階段組態頁面中建立或編輯了動作。請參閱建立、編輯和檢視訊息流程及新增和編輯訊息流程的動作。

若要建立或修改內置表示式

  1. 在「編輯階段組態」頁面上，尋找動作內想要新增或編輯表示式的位置。
  2. 按一下表示式連結，以開啟相關資訊環境適用的編輯器。尚未定義表示式時，連結會告知您可以在該位置使用的表示式種類：
    • 按一下「<表示式>」以建立 XQuery 表示式。您也可以匯入在 Oracle Service Bus 外部建立的 XQuery或 XSLT 資源，然後將該資源連結至內置 XQuery。
    • 按一下「<條件>」以建立 if..then... 動作的 XQuery 條件表示式。
    • 按一下「<XPath>」以建立訊息相關資訊環境變數的 XPath 表示式。

      • 當表示式已在某個位置定義後，<表示式>、<條件> 或 <XPath> 連結會取代為顯示表示式本身的連結 (例如 true()[icon])，或表示式的片段 (如果表示式太長)，例如 $body/urn:POSta...[icon]。請按一下表示式 (或表示式片段)，在適當的編輯器中開啟表示式。

  3. 建置表示式 (如下列主題所述)：
    • 在編輯器工作區文字欄位中建置表示式
    • 建立用於內置表示式的命名空間
    • 在 XQuery 編輯器中建立變數結構
    • 在 XQuery 編輯器中建立自訂 XPath 函數
    • 將外部 XQuery 資源連結至內置 XQuery
    • 將外部 XSLT 資源連結至內置 XQuery
    • 將動態 XQuery 表示式連結至內置 XQuery
    • 使用製作器選項輸入 XQuery 比較表示式
    • 使用製作器選項輸入一元表示式
  4. 選擇性地執行下列任一項或兩項操作：
    • 按一下「驗證」驗證表示式。
    • 按一下測試來測試表示式。請參閱執行 XQuery 測試。
  5. 按一下「儲存」關閉編輯器，並在動作中插入表示式。

完成作業後

繼續設定動作 (如新增和編輯訊息流程的動作中所述)。
