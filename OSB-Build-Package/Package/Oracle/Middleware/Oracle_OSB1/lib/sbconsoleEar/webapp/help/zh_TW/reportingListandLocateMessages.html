檢視報告訊息

使用訊息報告摘要頁面，可檢視表 27-2 中所顯示的報告訊息資訊。此頁面最多可以顯示 100 則訊息。

  1. 若要限制清單中的項目數或尋找特定訊息：
    • 按一下開啟圖示，以顯示其他搜尋篩選。如果指定了多個搜尋條件，則只會傳回符合全部條件的結果。
    • 若要設定篩選參數，請執行表 27-1 中所示的一或多個步驟。

      • 表 27-1 訊息篩選參數 
      若要...
      請完成這些步驟...
      依時間參數篩選
      設定時間間隔：
      1. 選取「訊息自... 目標 ...」單選按鈕。
      2. 在訊息自欄位中，以 MM/DD/YY HH:MM:SS AM|PM 格式輸入日期和時間。例如，您可以輸入 10/10/07 12:45:00 AM。
      3. 在「目標」欄位中，輸入 MM/DD/YY HH:MM:SS AM|PM 格式的日期和時間。
      您也可以針對最新的訊息選取參數：
      1. 選取「針對最近」單選按鈕。
      2. 在「針對最近」欄位中輸入天數，然後指定小時和分鐘。
      注意： 預設會顯示最後 30 分鐘的訊息報告。
      依服務名稱篩選
      在「內送服務名稱」欄位中，輸入服務名稱。
      依錯誤碼篩選
      在「錯誤碼」欄位中，輸入錯誤碼。
      依報告索引篩選
      在報告索引欄位中，輸入報告索引。這些就是您在設定此代理主機服務的報告動作時所定義的關鍵字值組。這是一種字串搜尋。

    • 在服務名稱、錯誤碼和報告索引欄位搜尋中，都可以使用萬用字元。
      • 只輸入星號 (*) 會傳回所有訊息。例如，如果在錯誤碼欄位中輸入 *，則會顯示所有訊息，而不論其是否含有錯誤碼。
      • 只有在「錯誤碼」欄位中，才會套用嚴格搜尋限制。例如，若要尋找錯誤碼為 BEA12345 的訊息，則可以輸入 *、BE* 或 BEA12345。
  2. 按一下「搜尋」。此頁面會顯示符合所設定之篩選參數的訊息。篩選訊息時，可以顯示的訊息數目上限是 1000 則。
  3. 按一下「檢視全部」移除搜尋篩選，並顯示所有訊息。
  4. 若要尋找訊息，也可以執行下列操作：
    • 重新排序訊息的清單。按一下加上水平底線的欄名稱。遞增和遞減箭頭表示排序順序。按一下欄名稱以變更排序順序。
    • 捲動頁面。使用表格上下方的頁面控制項。您可以選取頁碼以前往該頁，也可以使用箭頭按鈕前往下一頁、上一頁、第一頁或最後一頁。

      • 表 27-2 訊息 (報告) 摘要詳細資訊
      特性
      描述
      報告索引
      連到檢視訊息詳細資訊頁面的連結。請參閱檢視訊息詳細資訊。
      報告索引是當您要取出訊息相關資訊環境變數中的關鍵字 ID 時，所使用的關鍵字值組。關鍵字是篩選訊息的便利方法。您可以使用報告動作來新增關鍵字值組。請參閱新增和編輯訊息流程的動作。
      DB 時間戳記
      在資料庫中新增訊息的日期和時間。
      內送服務
      與訊息相關聯的內送服務。此服務是連到檢視代理主機服務 - 組態詳細資訊頁面的連結。請參閱編輯代理主機服務組態。
      錯誤碼
      與此訊息相關聯的錯誤碼 (如果有的話)。
      當您設定發生錯誤動作時，您必須指定錯誤碼。請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 "Handling Errors in Message Flows"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBAG232，以及「新增代理主機服務錯誤處理程式」。
