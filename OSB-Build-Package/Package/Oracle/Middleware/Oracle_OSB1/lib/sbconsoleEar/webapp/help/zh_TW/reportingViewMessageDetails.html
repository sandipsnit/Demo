檢視訊息詳細資訊

使用此頁面可以檢視特定訊息的詳細資訊。

  1. 尋找訊息。請參閱檢視報告訊息。
  2. 按一下報告索引。「檢視訊息詳細資訊」頁面會顯示表 27-3 中所示的「一般組態」資訊。

    3. 表 27-3 訊息組態參數 
    特性
    描述
    訊息 ID
    此訊息的唯一 ID。
    資料庫時間戳記
    在資料庫中登錄訊息的日期和時間。
    記錄時間點
    記錄訊息的日期和時間。
    伺服器名稱
    產生此訊息之伺服器的名稱。
    狀態
    產生此訊息之管線的狀態：
    要求 - 表示報告動作是在要求管線中執行。
    回應 - 表示報告動作是在回應管線中執行。
    錯誤 - 動作是在服務層級錯誤處理程式中執行。
    節點名稱
    產生此訊息之管線的節點名稱。
    管線名稱
    產生此訊息之管線的名稱。
    階段名稱
    產生此訊息之階段的名稱。

    此頁面會顯示表 27-4 中所顯示的內送服務資訊。

    表 27-4 內送服務詳細資訊 
    特性
    描述
    名稱
    與此訊息相關聯的內送代理主機服務。這是檢視代理主機服務 - 組態詳細資訊頁面的連結。請參閱編輯代理主機服務組態。
    URI
    與代理主機服務相關聯的 URI。
    作業
    與此訊息相關聯的內送作業。

    此頁面會顯示表 27-5 中所顯示的外送服務資訊。

    表 27-5 外送服務詳細資訊
    特性
    描述
    名稱
    與此訊息相關聯的外送業務服務。這是檢視業務服務 - 組態詳細資訊頁面的連結。請參閱編輯業務服務組態。
    URI
    外送業務服務端點的 URI。
    作業
    在外送服務上叫用之作業的名稱。

    此頁面會顯示表 27-6 中所顯示的報告索引資訊。

    表 27-6 報告索引詳細資訊 
    特性
    描述
    報告索引文字
    報告索引是當您要取出訊息相關資訊環境變數中的關鍵字 ID 時，所使用的關鍵字值組。關鍵字是篩選訊息的便利方法。您可以使用報告動作來新增關鍵字值組。請參閱新增和編輯訊息流程的動作。

    此頁面會顯示表 27-7 中所示的「錯誤」資訊。

    表 27-7 錯誤資訊詳細資訊 
    特性
    描述
    錯誤碼
    與訊息相關聯的錯誤碼 (如果有的話)。
    當您設定發生錯誤動作時，您必須指定錯誤碼。請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 "Handling Errors in Message Flows"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBAG232，以及「新增代理主機服務錯誤處理程式」。
    原因
    錯誤碼的原因。
    詳細資訊
    與錯誤碼相關聯的錯誤詳細資訊。這些詳細資訊 (如果有的話) 通常會追蹤特定錯誤在堆疊中的發生位置，而且可能會因為資料庫中的大小限制而遭到截斷。限制是 2048 個字元。

    此頁面會顯示表 27-8 中所顯示的報告主體資訊。

    表 27-8 報告主體詳細資訊
    特性
    描述
    報告主體文字
    此連結會開啟另一個瀏覽器，以顯示報告主體的詳細資訊。您可以在報告動作中使用表示式來擷取報告主體文字。請參閱新增和編輯訊息流程的動作中的報告動作程序。

  3. 檢視完特定訊息的詳細資訊後，請按一下「確定」。
