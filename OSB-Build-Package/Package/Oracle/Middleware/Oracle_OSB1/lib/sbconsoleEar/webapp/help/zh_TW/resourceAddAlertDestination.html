新增警示目標

警示目標是 Oracle Service Bus 中之警示通知的目標位址。您可以在「建立警示目標」頁面中建立警示目標資源。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取專案瀏覽器，然後選取要在其中新增警示目標的專案或資料夾。隨即顯示專案/資料夾檢視畫面頁面。
  3. 從「建立資源」下拉清單中選取「警示目標」，以顯示「建立警示目標」頁面。
  4. 在資源名稱欄位中，輸入此警示目標的唯一名稱。這是必要欄位。

    請遵循資源命名限制的命名指南。
  5. 在「資源描述」欄位中，輸入警示目標的描述。
  6. 選取下列任一或全部目標，以加入至此警示目標資源。
    7. 注意： 警示是於執行時期在「Oracle Service Bus 管理主控台」中彙總，您可以在作業 > 儀表板底下檢視警示。
    • SNMP 陷阱：如果指定「SNMP 陷阱」，則警示會以 SNMP 陷阱形式傳送，而且所有協力廠商企業監督系統 (ESM) 都可以處理這些警示。
    • 報告：如果指定「報告」，則警示會傳送至「Oracle Service Bus 報告」模組，而且可以使用自訂「報告提供者」進行擷取，而這類自訂「報告提供者」可以使用 Oracle Service Bus 的「報告 API」提供者進行開發。這允許協力廠商以自訂的 Java 程式碼來接收和處理警示。
    • 警示記錄日誌：如果您選取「是」，傳送到這個警示目的地的警示會記錄警示日誌。
  7. 將電子郵件收件者新增至警示目標資源定義。請參閱新增電子郵件收件者。
  8. 將「JMS 目標」新增至警示目標資源定義中。請參閱新增 JMS 目標。
  9. 按一下「儲存」。新的警示目標會儲存在目前的階段作業中。
  10. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
