編輯 JAR 相依性

在 Oracle Service Bus 中，JAR 資源可以包含與不同 JAR 中的其他類別相依的類別。使用「編輯 JAR 相依性」頁面新增或移除參照的 JAR 資源。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找 JAR (如尋找 JAR 中所述)。
  3. 按一下 JAR 名稱。

    4. 在「檢視 JAR 資源 - JAR 組態」頁面的「相依性」頁籤上，數值指示器會顯示設定給此 JAR 資源的所參照 JAR 數目，如果沒有，則為零。

  4. 選取「相依性」頁籤，以顯示或新增參照的 JAR。

    5. 「JAR 相依性」頁面包含表 7-4 中所顯示的資訊。

    表 7-4 JAR 相依性資訊
    特性
    描述
    名稱
    參照的 JAR 的名稱。此名稱是連到檢視 JAR 資源 - JAR 組態頁面的連結。請參閱編輯 JAR。
    路徑
    路徑是專案名稱和 JAR 所在資料夾的名稱。

  5. 若要新增或移除參照的 JAR，請按一下「編輯」。

    6. 「編輯 JAR 相依性」頁面包含表 7-5 中所顯示的資訊。

    表 7-5 JAR 相依性詳細資訊
    特性
    描述
    資訊清單類別路徑
    JAR 資訊清單檔案中的類別路徑項目。
    名稱
    參照的 JAR 資源的名稱和路徑。
    選項
    • 使用上下箭頭變更參照的 JAR 的優先順序。
    • 按一下「刪除」圖示，以移除參照的 JAR。

  6. 按一下「新增 JAR」。
  7. 在 JAR 瀏覽器中，尋找或選取參照的 JAR 資源，然後按一下「提交」。

    8. 例如，如果您編輯 JAR A 並新增 JAR B 和 C，則 JAR A 會參照 (相依於) JAR B，且 JAR A 也會參照 (相依於) JAR C。

    使用「選項」欄中的箭頭來重新排序參照的 JAR 的階層，或使用「刪除」圖示來移除參照的 JAR。

  8. 按一下「儲存」，確認目前階段作業中的更新。
  9. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
