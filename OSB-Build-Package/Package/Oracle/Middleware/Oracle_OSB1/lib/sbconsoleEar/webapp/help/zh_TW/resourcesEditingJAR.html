編輯 JAR

您可以使用檢視 JAR 資源 - 組態頁面來更新 JAR 檔案。更新 JAR 檔案基本上包含將資源指向新版的 JAR，因為您無法使用 Oracle Service Bus 編輯 JAR。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找 JAR (如尋找 JAR 中所述)。
  3. 按一下 JAR 名稱。「檢視 JAR 資源 - JAR 組態」頁面會顯示表 7-2 中所顯示的資訊。

    4. 表 7-2 JAR 資訊 
    特性
    描述
    上次修改者
    將此 JAR 匯入組態中的使用者。
    上次修改時間
    使用者將此 JAR 匯入組態中的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱檢視變更歷史記錄頁面。
    參照
    此 JAR 參照的 JAR 數。如果有這類參照，請按一下數字連結以檢視 JAR 清單。請參閱檢視資源參照。
    參照者
    參照此 JAR 的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    描述
    此 JAR 的描述 (如果存在的話)。

    預設會顯示 JAR 組態，其包含表 7-3 中所顯示的資訊。

    表 7-3 JAR 組態詳細資訊
    特性
    描述
    JAR 組態
    此 JAR 中的類別和介面清單。
    按一下 Java 類別名稱旁邊的 (+)，以顯示該類別中所含方法的清單。

  4. 若要變更欄位，請按一下「更新」。

    5. 您唯一可以編輯的欄位是描述。您無法在 Oracle Service Bus 中實際編輯 JAR。若要更新 JAR，則必須選擇要使用的新 JAR。若要瞭解作法，請參閱新增 JAR。

  5. 按一下「儲存」，確認目前階段作業中的更新。
  6. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
注意： 如果您變更「EJB 業務服務」所使用的 JAR (將 JAR 資源指向新版的 JAR)，則需要重新建置 EJB。然後，您必須編輯所有使用此 JAR 的 EJB 服務、重新選取 JAR 資源，然後進行儲存和啟動。這樣會重新封裝 EJB 業務服務，以使用新的 JAR。
注意： 不過，Java 外部呼叫動作和 Tuxedo 型服務會自動接獲新的 JAR。
