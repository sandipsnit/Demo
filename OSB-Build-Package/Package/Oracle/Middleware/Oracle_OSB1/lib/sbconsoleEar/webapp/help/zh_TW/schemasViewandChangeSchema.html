編輯 XML 綱要

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找 XML 綱要(如尋找 XML 綱要中所述)。
  3. 按一下 XML 綱要名稱。「檢視 XML 綱要詳細資訊」頁面會顯示表 4-2、表 4-3 和表 4-4 中所顯示的資訊。

    4. 表 4-2 XML 綱要詳細資訊 
    特性
    描述
    上次修改者
    建立這個 XML 綱要或將其匯入組態中的使用者。
    上次修改時間
    使用者建立此 XML 綱要或將其匯入組態中的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱檢視變更歷史記錄頁面。
    參照
    此 XML 綱要參照的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    參照者
    參照此 XML 綱要的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。例如，若特定 WSDL 參照此 XML 綱要，當您按一下連結時，就會將該 WSDL 列為參照。請參閱檢視資源參照。
    描述
    此 XML 綱要的描述 (若有的話)。

    預設會顯示 XML 綱要的結構檢視畫面，其包含表 4-3 中所顯示的資訊。

    表 4-3 XML 結構詳細資訊
    特性
    描述
    目標命名空間
    用來限定 XML 綱要中所包含之任何定義的命名空間。
    XML 綱要定義
    與 XML 綱要相關聯的屬性及群組：
    • XML 綱要類型
    • XML 綱要元素
    • XML 綱要匯入
    • XML 綱要包含/重新定義
    綱要狀態
    XML 綱要狀態：
    • 有效
      指定現行 XML 綱要所包含之 XML 綱要的所有位置時，XML 綱要就會顯示為有效。而且會指定巢狀 XML 綱要(內含之 XML 綱要所連續包含的 XML 綱要) 的所有位置。
    • 無效
      未指定一或多個 XML 綱要的位置時，XML 綱要就會顯示為無效。而且可能未指定一或多個巢狀 XML 綱要 (內含之 XML 綱要所連續包含的 XML 綱要) 的位置。若要解析無效的 XML 綱要，請按一下編輯參照。請參閱解析未解析的 XML 綱要參照。

  4. 按一下文字檢視畫面，以顯示 XML 綱要詳細資訊的文字檢視畫面。此文字檢視畫面包含表 4-4 中所顯示的資訊。

    此 XML 綱要的文字。

  5. 若要變更欄位，請按一下編輯。有關欄位的描述，請參閱新增 XML 綱要。
  6. 進行適當的編輯。
  7. 按一下「儲存」，確認目前階段作業中的更新。
  8. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
