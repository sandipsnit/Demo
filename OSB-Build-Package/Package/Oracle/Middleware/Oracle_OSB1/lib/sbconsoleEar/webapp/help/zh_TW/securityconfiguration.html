安全組態

您可以使用「安全組態」模組來建立及修改 Oracle Service Bus 內送安全及管理安全中所使用的安全資料。

內送傳輸層級安全性及訊息層級安全性會使用使用者、群組及角色資料來驗證內送的從屬端要求。它會套用存取控制原則，以決定要授權哪些經驗證的使用者來使用代理主機服務及業務服務。

管理安全會利用使用者、群組及角色資料來決定要授權哪些認證使用者來建立或修改 Oracle Service Bus 組態資料，或是監督 Oracle Service Bus 效能。如需詳細資訊，請參閱「瞭解使用者、群組、安全角色和原則」。另請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Configuring Administrative Security"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539。

注意： 當您匯出組態時，不能匯出使用者、群組或角色，因為這些物件位在安全提供者儲存庫中。當您將所匯出的組態匯入時，必須重新建立這些物件，或是使用 WebLogic Server 工具 (若有的話) 來將這些物件匯出及匯入。
