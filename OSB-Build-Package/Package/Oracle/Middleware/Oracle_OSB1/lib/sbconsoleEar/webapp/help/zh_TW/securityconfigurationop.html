瞭解使用者、群組、安全角色和原則

本節包括下列主題：

  • 使用者
  • 群組
  • 角色
  • 存取控制原則
  • 安全組態資料和階段作業

使用者

使用者是可以進行身份驗證的實體。使用者可以是個人或軟體實體，例如，Web Service 從屬端。您必須在安全領域中，提供唯一標識 (名稱) 給每個使用者。

通常您所建立的使用者分為二大類：

  • 從屬端使用者：可存取您的代理主機服務或業務服務。

    • 如果您建立了大量的從屬端使用者，請考慮將其組織成安全性群組。

  • 可使用「Oracle Service Bus 管理主控台」來建立或修改代理主機服務、業務服務及其他 Oracle Service Bus 資源的管理使用者。

    • Oracle Service Bus 會使用以角色為主的安全來執行其管理功能。Oracle Service Bus 不會直接授與存取權給使用者，只會提供管理權限給安全角色。若要提供管理權限給使用者，請將使用者放在其中一個預先定義安全角色中的其中一個預設安全群組中。

群組

為了方便管理大量的使用者，您可以將使用者組織在已命名的群組中。然後授與權限或標識給群組，而不要提供存取權或角色標識給個別使用者。

管理安全群組

Oracle Service Bus 提供預設安全群組，以便提供使用者存取管理功能的權限 (如建立代理主機服務)。每個群組都是已授與管理權限的其中一個預先定義 Oracle Service Bus 安全角色。

如需詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Security"，網址為：http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539。

角色

安全角色是可以依據執行時期環境中的條件，來授與使用者或群組的標識。當您建立存取控制原則時，可以授與存取權給角色、群組或使用者。

例如，您可以建立兩個群組：MyCustomersEast 和 MyCustomersWest。建立安全角色 PrivilegedCustomer，並建立條件，讓 MyCustomersWest 群組在 8am 到 8pm EST 這段期間執行此角色，而 MyCustomersEast 群組則是在 8pm 到 8am EST 這段期間執行此角色。然後建立代理主機服務的存取控制原則，將 PrivilegedCustomer 角色存取權指定給服務。不同的使用者可以在不同的時間存取，端視是在 MyCustomersEast 或 MyCustomersWest 群組中而定。

管理安全角色

Oracle Service Bus 提供四種提供管理權限的預先定義安全角色 (外加四種 WebLogic Server 所提供的預先定義角色)。您不能變更 Oracle Service Bus 管理安全角色的存取權限，但可以變更使用者或群組位於其中一個角色的條件。

如需關於這些角色及每個角色可用的授權詳細資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Configuring Administrative Security"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1539。

存取控制原則

存取控制原則可指定在哪些條件下，使用者、群組或角色可存取代理主機服務。例如，您可以建立原則，讓 GoldCustomer 角色中的使用者永遠都可以存取代理主機服務，並且讓 SilverCustomer 角色中的使用者可以在工作日晚間 12pm 之後存取代理主機服務。

您可以針對所有代理主機服務來建立傳輸層級原則，以在從屬端試圖建立代理主機服務的連線時，套用安全檢查。唯有列在傳輸層級原則中的使用者要求，才能繼續進行。

當從屬端嘗試叫用具有訊息層級安全性的代理主機服務時，訊息層級存取控制原則會套用安全性檢查。您可以在下列情況中建立訊息層級存取控制原則：

  • 代理主機服務若是作用中的 Web Service 安全媒介
  • 代理主機服務若具有訊息層級的自訂認證

唯有列在訊息層級原則中的使用者，才能叫用作業。

安全組態資料和階段作業

使用者、群組和角色都會保存在不受 Oracle Service Bus 階段作業控制的安全提供者中。因此，無論您是在階段作業內部或外部，都可以建立或修改此資料。對此資料做的任何新增或修改都會立即生效，並可供所有階段作業使用。如果您在階段作業中新增或修改資料，然後捨棄該階段作業，並「不會」捨棄安全資料。

存取控制原則會保存在授權提供者中。Oracle Service Bus 儲存庫中目前有對該項目的參照。

存取控制原則會在 Oracle Service Bus 設計階段作業受到管理，而非於階段作業外。由於變更是在階段作業進行的，因此您可以像其他資源一般提交或捨棄變更。

雖然可從「Oracle Service Bus 管理主控台」管理 ACL，但是您也可以在 Oracle Service Bus 外部變更原則。但若在 Oracle Service Bus 外部變更原則，可能會使 Oracle Service Bus 中的參照到期並失效。

因此，為達到管理的一致性，請完全在 Oracle Service Bus 階段作業外管理 ACL (使用授權提供者 MBean 或第三方授權提供者工具)，或完全從 Oracle Service Bus 階段作業內進行管理。如果混用這兩種方法，可能會產生不一致的原則檢視畫面。
