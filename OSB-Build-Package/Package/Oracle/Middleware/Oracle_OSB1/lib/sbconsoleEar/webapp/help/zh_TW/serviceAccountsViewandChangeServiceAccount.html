編輯服務帳號

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找服務帳號 (如尋找服務帳號中所述)。
  3. 按一下服務帳號名稱。「檢視服務帳號詳細資訊」頁面會顯示表 16-3 中所示的資訊。

    4. 表 16-3 服務帳號詳細資訊
    特性
    描述
    上次修改者
    建立這個服務帳號或將其匯入組態中的使用者。
    上次修改時間
    使用者建立這個服務帳號或將其匯入組態中的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱檢視變更歷史記錄頁面。
    參照
    此服務帳號參照的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    參照者
    參照此服務帳號的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。例如，若您在使用 JMS 傳輸通訊協定的代理主機服務中，選取這個服務帳號來作為 JMS 服務帳號，當您按一下連結時，就會將該代理主機服務列為參照。請參閱檢視資源參照。
    描述
    此服務帳號的描述 (若有的話)。

  4. 若要變更欄位，請按一下編輯。有關欄位的描述，請參閱新增服務帳號。

    5. 您不可以變更「資源名稱」欄位。

  5. 按一下「儲存」，確認目前階段作業中的更新。
  6. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
注意： 如果您修改的服務帳號會用來向 WebLogic JMS 伺服器進行驗證，JMS 伺服器可能會有長達 60 秒的時間，無法辨識您修改的內容。依預設，WebLogic Server JMS 每 60 秒會檢查一次各目標的使用權限。若要變更此行為，請修改 WebLogic Server 啟動命令，以將下列系統特性設成您要 WebLogic Server JMS 檢查權限的頻率 (秒)： weblogic.jms.securityCheckInterval
此特性值若為 0 (零) 可確保 JMS 資源每次發生 send、receive 和getEnumeration 動作時，都會執行權限檢查。
