指定服務帳號

您可以將相同的服務帳號用於多個業務服務及代理主機服務。若要指定服務帳號所提供的使用者名稱及密碼，您可以使用下列任一技術：

  • 靜態

    • 需要您以服務帳號組態來儲存使用者名稱及密碼。服務帳號會在外傳要求中將此使用者名稱及密碼編碼。

  • 使用者名稱及密碼傳遞通道

    • 可讓服務帳號提供從傳入之從屬端要求接收的使用者名稱及密碼。例如，若內傳的 HTTP BASIC 要求包含 "pat" 及 "patspassword" 作為使用者名稱及密碼，服務帳號就會在外傳要求中將 "pat" 及 "patspassword" 編碼。

    因為此技術需要從屬端要求包含純文字使用者名稱及密碼，所以只適用於使用 HTTP BASIC 通訊協定、Web Service 安全使用者名稱記號認證 (具有純文字密碼) 或自訂使用者名稱和密碼記號的從屬端要求。

    Oracle 建議您，唯有當 Oracle Service Bus 及端點屬於同一認證網域時，才使用此技巧。例如，當您在單一組織中遞送訊息，而 Oracle Service Bus 和訊息使用者向一般的 LDAP 伺服器進行認證時，就可以使用此技術。

    此技術具有下列限制：

    • 不能用於向本機或遠端伺服器或系統資源 (如 FTP 伺服器或 JMS 伺服器) 驗證 Oracle Service Bus 的外傳要求。
    • 無法以 fn-bea:lookupBasicCredentials XQuery 函數使用。如需更多資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 "XQuery Implementation"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBAG444。
      • 注意： 如果您的代理伺服器是作用中的 WSS 媒介，則可以使用 WS 安全性來加密 WS 安全性使用者名稱記號或使用自訂使用者名稱/密碼。在此情況下，因為代理伺服器會先解密要求後再存取純文字使用者名稱/密碼，所以使用者名稱/密碼傳遞通道會運作。
  • 使用者對應

    • 需要您將驗證從屬端 (本機使用者名稱) 之內傳要求所使用的使用者名稱，與您指定的使用者名稱和密碼 (遠端使用者名稱和密碼) 產生關聯 (對應)。當服務帳號從所對應之經驗證的從屬端接收要求時，會針對業務服務或代理主機服務外傳要求提供適當的遠端使用者名稱及密碼。

    如果從屬端在傳輸層級和訊息層級都進行驗證，服務帳號就會將訊息層級使用者名稱對應至遠端使用者名稱和密碼。

    您也可以將匿名使用者名稱對應至遠端使用者名稱和密碼。

    此技術具有下列限制：

    • 不能用於向本機或遠端伺服器或系統資源 (如 FTP 伺服器或 JMS 伺服器) 驗證 Oracle Service Bus 的外傳要求。
    • 無法以 fn-bea:lookupBasicCredentials XQuery 函數使用。如需更多資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 "XQuery Implementation"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBAG444。
