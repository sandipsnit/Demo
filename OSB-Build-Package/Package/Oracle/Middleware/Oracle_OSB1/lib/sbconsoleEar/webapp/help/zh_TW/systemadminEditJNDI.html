編輯 JNDI 提供者

使用此頁面可以檢視 JNDI 提供者的詳細資訊，以及依需要編輯組態。您還可以重設 JNDI 提供者快取，以及測試與 JNDI 提供者的連線。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找 JNDI 提供者。請參閱檢視 JNDI 提供者。
  3. 按一下 JNDI 提供者的名稱。

    4. 檢視 JNDI 提供者 - 組態頁面會顯示表 30-2 中所示的「JNDI 提供者」資訊。

    表 30-2 JNDI 提供者資訊 
    特性
    描述
    上次修改者
    建立或編輯此 JNDI 提供者資源的使用者，或將此資源匯入組態中的使用者。
    上次修改時間
    使用者建立或修改此 JNDI 提供者資源，或是將其匯入組態中的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱檢視變更歷史記錄頁面。
    參照
    此 JNDI 提供者所參照的物件數。如果有這類參照，請按一下數字連結以檢視參照清單。請參閱檢視資源參照。
    參照者
    參照此 JNDI 提供者的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    描述
    此 JNDI 提供者的描述。
    JNDI 快取
    「已啟用」或「已停用」。啟用後，即會將 JNDI 相關資訊環境及 JNDI 物件快取至本機，以便在執行物件查詢時提高效能。Oracle 建議您將 JNDI 快取保持在已啟用狀態。
    提供者 URL
    JNDI 提供者的 URL。
    JNDI 要求逾時
    JNDI 要求逾時 (毫秒)。預設值零 (0) 代表沒有逾時。
    使用者名稱和密碼
    用以存取安全 JNDI 提供者的使用者名稱，且具備變更密碼的能力。

  4. 按一下「編輯」，以顯示「編輯 JNDI 提供者 - [提供者名稱]」頁面。

    5. 您無法編輯「名稱」欄位。

  5. 您可以在「描述」欄位中編輯 JNDI 提供者的簡短描述。
  6. 您可以在「提供者 URL」欄位中編輯 JNDI 提供者的 URL。
  7. 您可以編輯「使用者名稱」與「密碼」欄位。

    8. 如果已對「JNDI 提供者」設定使用者名稱和密碼，則無法再編輯使用者名稱。您可以編輯新密碼與確認密碼欄位。如果未對 JNDI 設定使用者名稱和密碼，則可以編輯使用者名稱欄位，並且可以輸入新的使用者名稱。

  8. 按一下「儲存」，確認目前階段作業中的更新。
  9. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
