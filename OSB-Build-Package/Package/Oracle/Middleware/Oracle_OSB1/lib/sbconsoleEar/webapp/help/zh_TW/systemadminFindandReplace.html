尋找和取代環境值

您可以在尋找及取代頁面搜尋在不同網域間有所差異的環境值。環境值是指組態資料中某些預先定義的欄位，其值經常會在您將組態移至其他網域時 (例如從測試網域移至生產網域) 有所變更。環境值代表 URL、URI、檔案與目錄名稱、伺服器名稱、電子郵件等實體。此外，可以在警示目標、代理主機服務、業務服務、SMTP 伺服器和 JNDI 提供者資源，以及 UDDI 登錄檔項目中找到環境值。

此頁面會根據您是否在階段作業中而有不同的執行方式。若您在階段作業中，即可尋找和取代環境值。但若您在階段作業外，則只能尋找環境值，並且「全部取代」會停用。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「系統管理」>「尋找及取代」。
  3. 在「尋找值」欄位中，輸入您要尋找的環境值。
  4. 若要顯示您的組態中含有您所輸入之值的環境值清單，請按一下「尋找」。

    5. 畫面中會顯示相關值的清單，其中包含「擁有者」、「類型」與「環境值」等欄。

  5. 若只要尋找目前階段作業中所變更的項目，請選取「僅限在目前階段作業中變更的項目」核取方塊。
  6. 從在變數類型中尋找下拉清單中選擇要搜尋的變數類型 (如表 31-1 中所述)。

  8. 在「取代為」欄位中輸入新的環境值。
  9. 若要將原有的環境值取代為新值，請按一下「全部取代」。

    10. 您在「尋找值」欄位中輸入的環境值所找到的所有相符項目，都會取代為您在目前階段作業的「取代為」欄位中輸入的環境值。

    注意： 某些環境值屬於無法透過「Oracle Service Bus 管理主控台」尋找及取代的複雜 XML 物件。不過，您仍然可以從命令碼使用 ALSBConfigurationMBean 直接設定這些環境變數。如需 ALSBConfigurationMBean 的詳細資訊，請參閱 Oracle Service Bus API Reference。除了透過 API 設定以外，您還可以使用自訂檔案來設定複雜類型環境值。請參閱 執行自訂檔案。
  10. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
