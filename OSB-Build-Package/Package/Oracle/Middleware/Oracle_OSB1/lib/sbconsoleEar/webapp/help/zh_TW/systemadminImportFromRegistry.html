從 UDDI 登錄檔匯入業務服務

您可以將 UDDI 登錄檔中的下列業務服務類型匯入 Oracle Service Bus：

  • 透過 HTTP 傳輸的 WSDL 服務。
  • 發佈到 UDDI 登錄檔的 Oracle Service Bus 代理主機服務。這項功能主要是運用在某個網域中的代理主機服務必須搜尋並路由至另一個網域之代理主機服務的多網域 Oracle Service Bus 建置中。

如果您只設定一個登錄檔，Oracle·Service·Bus·就會自動連線該登錄檔、擷取所有業務實體，並填入搜尋表單。若您設定了多個登錄檔，匯入精靈就會提示您先選取要從中匯入服務的登錄檔。

在匯入時，若登錄檔具有數個不同類型的服務，即會使用詢問 URL 尋找特定的服務。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「系統管理」>「從 UDDI 匯入」。
  3. 從「登錄檔名稱」下拉清單中，選取想要從中匯入服務的登錄檔名稱，然後按「下一步」。

    4. 若僅在·Oracle·Service·Bus·中設定一個登錄檔，則不會顯示初始登錄檔選取頁面，因為這是預設登錄檔。只有在定義了兩個或更多登錄檔時，才會顯示登錄檔選取頁面。

    使用從 UDDI 匯入頁面可以搜尋特定的業務服務，並予以匯入。您可以依據業務實體、模式或兩者的組合來搜尋服務。

  4. 從「實體名稱」下拉清單中選取業務實體名稱 (如文件服務)，以依據業務實體進行搜尋。
  5. 在服務名稱欄位中，輸入您要搜尋的模式 (例如 a%)。即會顯示與輸入之搜尋條件相符的業務服務清單。
  6. 選取您要匯入的服務，然後按「下一步」。

    7. 若您找不到所需的服務，可能是因為您不屬於具有相關權限而可檢視其記錄的安全群組。

  7. 選取您要據以建立業務服務的連結樣板，然後按「下一步」。

    8. 若所選的服務含有多個連結樣板，每個連結樣板都將產生一個業務服務。在此情況下，系統會提示您進一步縮小您對所要匯入之連結樣板的選取。

  8. 在「匯入 UDDI：選取匯入位置」頁面上，選取「專案」下拉清單中的專案，再選取您的服務與相關資源所要匯入的資料夾，然後按「下一步」。
  9. 「匯入 UDDI：檢查並匯入服務」頁面會顯示將在系統中建立之所有資源的清單 (業務服務、MFL、綱要、WSDL 等)。系統會針對無法匯入的資源顯示警告訊息。
    預設情況下，清單中所有要匯入的項目都會加以標示。
    1. 若您的系統中已有某個特定資源，而您想手動解除其相依性，請清除該資源旁的核取方塊。
    2. 您可以在匯入處理作業完成後，從變更中心的檢視衝突頁面中解析相依性。有關檢視和解決衝突的詳細資訊，請參閱檢視和解決衝突。
  10. 按一下「匯入」以啟動匯入流程。

    11. 即會顯示指出匯入進度的快顯視窗。當匯入完成時，匯入摘要頁面會顯示匯入的結果。若匯入作業成功，頁面頂端會顯示成功訊息。若有一或多項資源無法匯入，則會顯示錯誤訊息。

    • 請至資源的「狀態」欄檢視導致錯誤的原因。
    • 若要檢視及解決其他衝突，請按一下「檢視衝突」。
  11. 若要匯入其他服務，請按一下「匯入另一個」。
  12. 若要變更您的選取，請按一下「返回」。
