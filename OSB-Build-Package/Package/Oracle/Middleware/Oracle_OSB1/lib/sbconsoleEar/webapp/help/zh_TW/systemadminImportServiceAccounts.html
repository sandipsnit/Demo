匯入服務帳號或服務金鑰提供者

注意： 在舊的 Oracle Service Bus 版本中，服務金鑰提供者稱為代理主機服務提供者。

如果 JAR 檔案是由 AquaLogic Service Bus 3.0 或更新版本所建立，且含有服務帳號或服務金鑰提供者，則可以匯入這些資源，以及它們所含的使用者名稱、密碼、本機使用者與遠端使用者對應以及金鑰組連結別名。

對於每個服務金鑰提供者，Oracle Service Bus 會將金鑰組連結的別名匯入 PKI 證明資料對應提供者。若此資料在匯出的過程中執行了加密，您就必須提供當時用以加密資料的密碼。若您不知道密碼，可以匯出所有其他未加密的資源。

除非指定在匯入期間保留安全設定，否則如果匯入服務帳號或服務金鑰提供者和網域中已存在的同名對應資源，則匯入的資源會覆寫網域中原有的資源，即使原有的資源在目前階段作業中曾進行過修改也是一樣。請參閱選取要匯入的專案和資源中的步驟 5。
