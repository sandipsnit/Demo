將代理主機服務發佈至 UDDI 登錄檔

您可以將服務發佈至登錄檔，並使其成為可供其他組織搜尋及使用的項目。所有由 Oracle Service Bus 開發的代理主機服務均可發佈至 UDDI 登錄檔。您可以選取要在其下發佈服務的業務實體，也可以同時發佈多項服務。

若要發佈至某個登錄檔，您必須先具備含有該登錄檔的帳號。Oracle Service Bus 支援與 3.0 版相容之 UDDI 登錄檔的互操作性。本節所說明的組態會假設您使用的是 Oracle Service 登錄檔。

如需有關在 Oracle Service Registry 中設定使用者帳號的資訊，請參閱「Oracle Service Registry」產品文件 (請從 http://www.oracle.com/technology/products/soa/registry/index.html 取得)。

若要從登錄檔中取消發佈服務，必須從 Oracle Service 登錄檔安裝執行。

  1. 選取「系統管理」>「發佈至 UDDI」。

    2. 您必須在階段作業外才能進行發佈。請結束階段作業以存取登錄檔清單。您能夠存取的所有登錄檔，均會顯示於所選登錄檔的下拉清單中。

  2. 從「發佈登錄檔名稱」中，選取想要發佈之登錄檔的名稱，然後按「下一步」。

    3. 若僅在·Oracle·Service·Bus·中設定一個登錄檔，則不會顯示初始登錄檔選取頁面，因為這是預設登錄檔。只有在定義了兩個或更多登錄檔時，才會顯示登錄檔選取頁面。

    畫面中會顯示發佈至 UDDI：選取個別服務並發佈頁面，其中列出 Oracle Service Bus 代理主機服務及其在專案資料夾中所在位置的清單。它會從最上方的專案層級資料夾開始，顯示所有服務的名稱、類型與描述等詳細資訊。

  3. 選取您要發佈的服務。
    1. 展開專案資料夾以檢視已定義的代理主機服務。預設情況下，要發佈的資料夾及其內容均會加以標示。
    2. 您可以選取要發佈的個別項目。
  4. 從將服務發佈至業務實體下拉清單中，選取想要在登錄檔中新增服務的業務實體。這就是服務據以在登錄檔中分類的業務實體。
  5. 按一下「發佈」以發佈服務。

    6. 畫面上會顯示確認訊息，指出服務已順利發佈。

  6. 按一下「發佈另一個」，返回「發佈至 UDDI：選取登錄檔」頁面。
