選取要匯入的專案和資源

  1. 如果您匯入資源 JAR 檔案，而且想要確定它的所有相關資源也都已匯入，請選取「包含相依性」。
    2. 注意： 如果匯入完整專案 JAR 檔案或 2.6 版以前的組態檔，則無法選取「包含相依性」。
  2. 如果想要保留匯入系統中的值，請按一下「開啟」圖示以顯示「進階設定」。
  3. 當您重新匯入資源，但想保有對現有資源所做的自訂時，請選取「保留環境變數」。
    4. 注意： 在現有資源內所自訂的任何參照，皆無法保留。請參閱建立自訂檔案。
  4. 如果您匯入全域作業設定資源或其他資源 (如含有相關服務層級作業設定的商業或代理主機服務)，並且需要保留匯入系統中的作業值，請選取「保留作業值」。
  5. 若不要覆寫所更新資源中的現有安全相關組態，請從下列選項中進行選取：
    1. 保留安全和原則組態 - 保留安全組態 (不含存取控制原則) 和直接連結至服務之「WS 原則」的參照 (而不是連結至 WSDL)。

      2. 如需詳細資訊，請參閱「Oracle Fusion Middleware Developer's Guide for Oracle Service Bus」中的「Security」，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1472。

    2. 保留證明資料 - 保留服務金鑰提供者中的 PKI 證明資料、服務帳號中的使用者名稱和密碼，以及 SMTP 伺服器、代理伺服器、JNDI 提供者和 UDDI 登錄檔中的使用者名稱和密碼證明資料。
    3. 保留存取控制原則 - 保留存取控制原則。

  6. 選取要對其進行操作 (建立、更新或刪除) 的物件。
    1. 若要顯示 JAR 檔案中的每個物件，請展開包含該檔案的資料夾。即會勾選與每個物件相關聯的核取方塊。

      2. 表 28-2 會針對每個物件顯示下列資源檔案詳細資訊。

    2. 清除不匯入之資源旁的核取方塊。
  7. 按一下「匯入」。

完成作業後

如果適當，請繼續進行指定匯入安全設定。或是繼續進行檢查匯入摘要。
