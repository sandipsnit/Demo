檢視 JNDI 提供者

使用此頁面可以檢視 JNDI 提供者的清單，以及表 30-1 中所示的資訊。

此外，還可以搜尋先前設定的 JNDI 提供者。

  1. 若要限制清單中的項目數或尋找特定 JNDI 提供者：
    • 按一下開啟圖示，以顯示其他搜尋篩選。如果指定了多個搜尋條件，則只會傳回符合全部條件的結果。
    • 在「名稱」欄位中輸入您要尋找的 JNDI 提供者名稱，然後按一下「搜尋」以搜尋特定項目。

      • 您可以輸入萬用字元 (? 表示單一字元，而 * 表示多個字元)，以執行更廣泛的搜尋。

    • 輸入想要尋找之 JNDI 提供者的「提供者 URL」，然後按一下「搜尋」。
  2. 按一下「檢視全部」移除搜尋篩選，並顯示所有設定成使用 Oracle Service Bus 的 JNDI 提供者資源。

    3. 表 30-1 顯示 UDDI 登錄檔組態設定。

    表 30-1 JNDI 提供者資訊 
    特性
    描述
    名稱
    指派給此 JNDI 提供者的唯一名稱。此名稱是連到檢視 JNDI 提供者 - 組態頁面的連結。請參閱編輯 JNDI 提供者。
    提供者 URL
    JNDI 提供者的 URL。請參閱新增 JNDI 提供者。
    JNDI 快取
    「已啟用」或「已停用」。啟用後，即會將 JNDI 相關資訊環境及 JNDI 物件快取至本機，以便在執行物件查詢時提高效能。Oracle 建議您將 JNDI 快取保持在已啟用狀態。
    動作
    • 按一下「重設」圖示，將 JNDI 相關資訊環境重設為捨棄 JNDI 連線和本機快取的物件。
    • 按一下「啟動測試主控台」圖示，以測試 JNDI 提供者是否可以順利建立連線以及取得 JNDI 相關資訊環境。
    選項
    • 按一下「重新命名」圖示，以重新命名 JNDI 提供者。
    • 按一下刪除圖示，以刪除特定「JNDI 提供者」。請參閱刪除 JNDI 提供者。
