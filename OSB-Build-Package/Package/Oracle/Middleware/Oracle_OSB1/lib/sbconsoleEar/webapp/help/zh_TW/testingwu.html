測試代理主機服務

您可以測試下列類型的代理主機服務：任何 XML、任何 SOAP、訊息傳遞、XML 和 SOAP。您可以使用 Web Service 安全 (WSS) 原則來測試 SOAP 代理主機服務。請參閱設定代理主機服務測試資料中的「Web Service 安全」。

注意： 當「測試主控台」利用 HTTP 自訂記號認證呼叫代理主機時，不會執行認證檢查。
注意： 測試已啟用直接呼叫選項的代理主機服務，會略過一些重要的安全步驟 (含存取控制)。Oracle 建議您不要在生產系統中使用測試服務。
  1. 按一下變更中心底下的啟動，即可啟用「Oracle Service Bus 管理主控台」中的測試功能。

    2. 您可以從「資源瀏覽器」或「專案瀏覽器」測試代理主機服務。

  2. 選取「資源瀏覽器」>「代理主機服務」，以顯示「代理主機服務的摘要」頁面。
  3. 在「動作」下，按一下與想要測試之代理主機服務相關聯的「啟動測試主控台」圖示。

    4. 「測試主控台」會開啟測試代理主機服務頁面。例如，使用產品隨附的範例時，請按一下與 LoanGateway1 代理主機服務相關聯的圖示。

    注意： 在叢集網域中，無法使用「測試主控台」來測試任何設定的業務服務，或路由至業務服務的代理主機服務。
  4. 若為 SOAP 與 XML 服務，請選擇您要測試的 WSDL 作業。
  5. 設定代理主機服務的測試資料。這必須是代理主機服務預期要從從屬端取得的資料。

    6. 預設會啟用直接呼叫和包含追蹤這兩個測試組態選項。您可以清除直接呼叫選項，這樣也會一併清除包含追蹤選項。若執行此動作，測試即會以透過傳輸層傳送訊息的間接方式執行。

    您可以使用直接呼叫選項 (維持選取直接呼叫)，並停用「執行追蹤」；只需取消選取包含追蹤核取方塊即可。

    若要查看訊息追蹤，請取消選取「直接呼叫」。

    注意：若要查看日誌檔或標準輸出 (伺服器主控台) 中的追蹤，必須將 Oracle WebLogic Server 記錄設定成下列嚴重程度層級：

要記錄的最低嚴重程度：資訊
日誌檔：資訊
標準輸出：資訊

如需設定日誌檔嚴重程度層級的資訊，請參閱 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的 Using Log Severity Levels，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

     

  1. 按一下「執行」。測試代理主機服務頁面會顯示結果。有關解譯測試結果的相關資訊，請參閱檢視代理主機服務測試結果。
  2. 若要重新執行測試，請按一下返回。從步驟 5 開始重複執行。
