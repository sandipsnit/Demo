直接呼叫
將訊息傳送至代理主機服務，而不使用 Oracle Service Bus 傳輸層。「測試主控台」的輸入資料必須是從從屬端傳送至代理主機服務的資料。
「直接呼叫」的相反功能為「間接呼叫」。取消選取直接呼叫選項，就可以呼叫該功能。這是業務服務的預設執行選項。間接呼叫會透過傳輸層傳送訊息。在此情況下，「測試主控台」的輸入資料必須是從代理主機服務傳送至所呼叫之服務的資料。

要求文件
輸入欄位會產生要傳送至代理主機服務的要求訊息。按一下執行即可使用輸入的值執行測試。「測試主控台」會顯示要求訊息及服務的回應訊息與描述資料。
「要求文件」區段中的輸入是服務類型特有的輸入。下列各節會列出服務類型以及每種類型所需要之輸入的描述。
任何 XML
服務要求以有效負載的形式輸入。有效負載是所傳送之訊息的內容。此內容應屬於 XML 文件。您可以瀏覽至檔案，也可以在文字方塊中輸入訊息內容。
要求文件 (續)
任何 SOAP
服務要求以有效負載的形式輸入。有效負載是所傳送之訊息的內容。此內容應屬於 SOAP 信封。您可以瀏覽至檔案，也可以在文字方塊中輸入訊息內容。
訊息傳遞
訊息傳遞服務定義四種可能的輸入類型：none、XML、Binary 或 Text。此類型的服務會要求檔案型或文字型的單一輸入，只有類型 none 除外而不會要求任何輸入。
Oracle 建議從檔案輸入二進位輸入。在文字區域中輸入的資料，會以系統編碼轉換為二進位輸入。
從檔案輸入並用於文字服務的資料，必須轉換為文字。編碼輸入欄位可用以指定轉換期間應套用的編碼。若未設定此欄位，則會使用系統編碼。預設會以代理主機服務端點上所設定的編碼值來初始化編碼欄位。
XML
服務要求以有效負載的形式輸入。有效負載是所傳送之訊息的內容。此內容應屬於 XML 文件。您可以瀏覽至檔案，也可以在文字方塊中輸入訊息內容。
如果代理主機服務為已定義多項作業的 WSDL 型服務，「測試主控台」就會產生並提供可在測試服務時使用的範例文件。您可以直接使用此範例資料，或在編輯之後執行測試，或者也可以提供您自己的測試資料。
所有作業都會列在頁面的頂端。
要求文件 (續)
SOAP 文件
就 SOAP 文件而言，SOAP 信封通常包含零個或多個標頭，以及一個主體有效負載。「表單」和「XML」標籤提供其他指定內容的方式。
表單頁籤包含「SOAP 標頭」欄位和「SOAP 主體」欄位。「SOAP 標頭」欄位的內容應為「SOAP 標頭」標籤 (允許定義多個標頭)。「SOAP 主體」欄位中含有實際上屬於傳送訊息一部分的資料。此內容應屬於 XML 文件。標頭與主體均可用以產生 SOAP 信封。
要求文件 (續)
SOAP RPC
就 SOAP RPC 而言，SOAP 信封包含零或多個標頭，以及零或多個引數。
「表單」和「XML」標籤提供其他指定內容的方式。
表單頁籤包含 SOAP 標頭的單一輸入，以及每個引數的一個輸入欄位 (輸入欄位的名稱對應於引數名稱)。「SOAP 標頭」欄位的內容應為「SOAP 標頭」標籤 (允許定義多個標頭)。
WSDL 可用以偵測每個引數的類型。單行輸入欄位可用於基本類型；多行輸入欄位可用於 XML 類型。範例文件會自動產生，以加速測試作業。
「測試主控台」會使用標頭和引數產生 SOAP 信封。
XML 頁籤包含單一輸入欄位。此欄位的內容應為要傳送的 SOAP 信封。有效負載 (XML 輸入) 可以是檔案型或文字型。參照檔案的輸入優先於文字輸入。請瀏覽並選擇想要用於測試的檔案。
Web Service 安全
本節只有在選定作業具有 Web Service 安全 (WSS) 原則時，才適用於 SOAP 服務。
服務提供者
測試服務會由此服務提供者，取得 Web Service 安全作業 (數位簽章和加密) 所需的所有從屬端 PKI (金鑰組) 證明資料。這是可選欄位。在特定情況下必須要有服務提供者，視作業的要求/回應原則而定。有關詳細資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 "Testing Services with Web Service Security"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBAG384。
使用者名稱
從測試服務產生之 Web Service 安全使用者名稱記號中的使用者名稱。這是可選欄位。只有在作業的要求原則指定標識聲明的特定情況下，才需使用此使用者名稱。
請勿將此欄位與傳輸安全相關資訊環境使用者名稱欄位混淆。
這必須是本機安全領域中的有效使用者名稱和密碼。無效的使用者名稱或密碼都會導致測試服務產生從屬端錯誤。
密碼
從測試服務產生之 Web Service 安全使用者名稱記號中的密碼。
傳輸
預設會收合傳輸區段。而欄位和值則是取決於測試組態。
認證
使用者名稱
叫用代理主機服務時，用來設定測試服務所使用之安全相關資訊環境的使用者名稱。
如果代理主機伺服器路由訊息到預期接收 SAML 記號的業務服務，這就是記號要表示的識別。如需更多資訊，請參閱 Oracle Fusion Middleware Developer's Guide for Oracle Service Bus 中的 "Using SAML for Authentication"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBDV1647。
請勿將此欄位與 Web Service 安全 (WSS) 使用者名稱欄位混淆。這必須是本機安全領域中的有效使用者名稱和密碼。無效的使用者名稱或密碼都會導致測試服務產生從屬端錯誤。
注意： 當「測試主控台」利用 HTTP 自訂記號認證呼叫代理主機時，不會執行認證檢查。
密碼
相關聯的密碼。有關詳細資訊，請參閱使用者名稱。

要求/回應
