測試業務服務

您可以測試下列業務服務的類型：任何 XML、任何 SOAP、訊息、XML 及 SOAP。您可以使用 Web 服務安全原則來測試 SOAP 業務服務。有關詳細資訊，請參閱 Oracle Fusion Middleware Administrator's Guide for Oracle Service Bus 中的 "Testing Services with Web Service Security"，網址為： http://www.oracle.com/pls/as111170/lookup?id=OSBAG384。

注意： 在叢集網域中，無法使用「測試主控台」來測試任何設定的業務服務，或路由至業務服務的代理主機服務。
  1. 按一下變更中心底下的啟動，即可啟用「Oracle Service Bus 管理主控台」中的測試功能。
  2. 選擇「資源瀏覽器」>「業務服務」，以顯示「業務服務的摘要」頁面。
  3. 在動作底下，按一下與想要測試之服務相關聯的啟動測試主控台圖示。「測試主控台」就會開啟測試業務服務頁面。例如，使用產品隨附的教學課程，按一下與 loanSaleProcessor 相關聯的圖示。
  4. 若為 SOAP 與 XML 服務，請選擇您要測試的 WSDL 作業。
  5. 設定業務服務的測試資料 (輸入資料應為要由代理主機服務傳送至業務服務的訊息)。測試代理主機服務時可用的直接呼叫和包含追蹤選項，不適用於業務服務。業務服務預設會使用直接呼叫選項進行測試，意即訊息會透過傳輸層傳送。

    6. 若要查看訊息追蹤，請取消選取「直接呼叫」。

    注意：若要查看日誌檔或標準輸出 (伺服器主控台) 中的追蹤，必須將 Oracle WebLogic Server 記錄設定成下列嚴重程度層級：

要記錄的最低嚴重程度：資訊
日誌檔：資訊
標準輸出：資訊

如需設定日誌檔嚴重程度層級的資訊，請參閱 Oracle Fusion Middleware Configuring Log Files and Filtering Log Messages for Oracle WebLogic Server 中的 Using Log Severity Levels，網址為 http://www.oracle.com/pls/as111170/lookup?id=WLLOG141。

  1. 按一下「執行」。

    2. 測試業務服務頁面會顯示結果。有關詳細資訊，請參閱檢視業務服務測試結果。

測試業務服務中的附件

此區段描述如何使用「測試主控台」來測試業務服務中的附件。

為方便說明，此區段假設 WSDL 型 HTTP 業務服務使用 "submitAttachment" 作業來傳送 ZIP 檔案做為 SOAP 訊息的附件。請視需要針對不同類型的二進位附件修改設定。

    1. 在業務服務的「測試主控台」中，前往 XML 頁面並為測試訊息輸入下列 SOAP 信封：
    <env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
       <env:Header/>
   <env:Body>
      <m:submitAttachment xmlns:m="http://www.alsb.com/SOAPwithAttachment/">
         <submitAttachment>
            <fileName>c:\yourfile.zip</fileName>
         </submitAttachment>
         <zipFile href="cid:zipFile"/>
      </m:submitAttachment>
   </env:Body>
    </env:Envelope>

     

    1. 在「測試主控台」的「附件」區段：
      1. Content-Type：application/zip (如 WSDL 所定義)
      2. Content-ID：zipFile (必須符合測試訊息中的 href="cid:zipFile" 值)
      3. 檔案：按一下瀏覽，然後選取您在 SOAP 信封中指定的 ZIP 檔案。
      4. 按一下「新增」。

    2. 按一下執行以傳送含附件的測試訊息。

若要確認已成功傳送附件，請檢查伺服器主控台中類似下列的訊息，該訊息由 submitAttachment 作業所記錄：

WS called - received the following properties:
submitAttachment is:
       com.alsb.soapwithattachment.SubmitAttachmentRequestType@e2abb0
zipFile is: javax.activation.DataHandler@175cf0a
