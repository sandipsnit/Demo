追蹤代理主機服務

當您使用直接呼叫測試代理主機服務時，就會啟用追蹤。「包含追蹤」核取方塊預設會與「直接呼叫」選項一起選擇。如果不想要進行追蹤，請清除「包含追蹤」核取方塊啟用追蹤時，測試結果會包含追蹤的詳細資訊。使用追蹤，可以追蹤系統中的問題，並找出這些問題予以更正。追蹤資訊會顯示透過程式碼中的要求和回應路徑所取得的路徑。

  1. 按一下變更中心底下的啟動，即可啟用「Oracle Service Bus 管理主控台」中的測試功能。
  2. 選擇「資源瀏覽器」>「代理主機服務」，以顯示「代理主機服務的摘要」頁面。
  3. 在動作底下，按一下與想要測試之服務相關聯的啟動測試主控台圖示。「測試主控台」就會開啟測試代理主機服務頁面。
  4. 設定代理主機服務的測試資料。若要啟用追蹤，必須要選取直接呼叫和包含追蹤選項。請參閱設定代理主機服務測試資料。
  5. 按一下「執行」。測試代理主機服務頁面即會顯示服務的測試結果與追蹤資訊。
  6. 向下捲動至「叫用追蹤」區段。

    7. 此區段顯示以圖形方式來呈現訊息流程。您可以透過服務追蹤訊息，並在追蹤中檢視預先選擇點的訊息狀態。追蹤點將會自動設定。

  7. 按一下 + 加號展開訊息流程，以檢視詳細資訊。

    8. 檢視追蹤的同時，也可以在「Oracle Service Bus 管理主控台」中檢視訊息流程。這有助於產生追蹤與訊息流程中之階段和動作的關聯。您可以修改訊息流程並重新執行追蹤，以檢視輸出。
