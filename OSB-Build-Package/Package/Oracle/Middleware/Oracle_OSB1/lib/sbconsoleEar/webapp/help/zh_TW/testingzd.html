測試 XSLT 轉換

可延伸樣式表語言轉換 (XSLT) 說明 Oracle Service Bus 中的 XML 至 XML 對應。若要測試 XSLT 資源，您必須提供輸入 XML 文件，「測試主控台」就會傳回輸出 XML 文件。XSLT 轉換可能有其他指定的參數。轉換所需的所有參數皆會顯示在組態頁面上。您可以使用預設值，也可以覆寫這些預設值。

表 32-6 描述 XSLT 組態選項。

XML 輸入
XML 輸入可以是檔案型或文字型。參照檔案的輸入優先於文字輸入。請瀏覽並選擇想要用於測試的檔案。需要 XML 輸入。
<param_name> ([] as XML)
param_name 為具名的 XSLT 參數。
輸入有兩種類型：XML 和基本 (字串、整數、浮點等)。預設輸入類型是「字串」。選擇與參數名稱相關聯的核取方塊，可識別類型為 XML 的參數。

您可以測試設計時期或執行時期。

  1. 如果想要測試執行時期，請按一下啟動。而測試設計時期，則請不要啟動階段作業。
  2. 選擇「專案瀏覽器」>「XSLT」，以顯示「XSLT 的摘要」頁面。
  3. 在動作底下，按一下與想要測試之資源相關聯的啟動測試主控台圖示。「測試主控台」就會開啟測試資源頁面。
  4. 在頁面的輸入與參數區段中設定資源的測試資料。有關詳細資訊，請參閱表 32-6。
  5. 按一下「執行」。測試資源頁面會顯示結果。
  6. 若要重新測試，請按一下返回。您可以關閉「測試主控台」、進行修改，然後重新測試資源。
