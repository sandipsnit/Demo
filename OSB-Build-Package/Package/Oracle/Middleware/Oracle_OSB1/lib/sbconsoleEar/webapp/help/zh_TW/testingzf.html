使用 XQuery 表示式和 XQuery 條件編輯器

您可以在訊息流程執行期間使用 XQuery 表示式，以建立訊息相關資訊環境變數 (或訊息相關資訊環境變數的一部分) 的資料內容。您可以在「XQuery 表示式編輯器」中直接使用「測試主控台」測試表示式的定義。

同樣地，您也可以使用 XQuery 條件來評估訊息流程中的布林值條件。您可以在「XQuery 條件編輯器」中直接使用「測試主控台」測試條件的定義。

XQuery 可接受多項輸入並傳回一項輸出。每項輸入各對應至 XQuery中所定義的一項 XQuery 未繫結變數。XQuery 輸入的值可以是基本值 (字串、整數、日期等)、XML 文件或這些類型的順序。輸出值可以是基本值 (字串、整數、日期等)、XML 文件，或這些類型的順序。

注意： 「測試主控台」不支援在輸入中使用序列。

param_name 是對應 XQuery 未繫結變數的名稱。
在「測試主控台」中，如果類型為簡單類型，會顯示單行編輯方塊。如果資料為 XML，則會顯示多行編輯方塊。當變數不具有類型時，會使用組合輸入 (<param_name> ([] as XML))。您必須宣告變數類型。選擇此核取方塊可識別類型為 XML 的參數。
XML 輸入可以是檔案型或文字型。參照檔案的輸入優先於文字輸入。請瀏覽並選擇想要用於測試的檔案。
「測試主控台」中的輸入會根據類型來轉譯，以便讓您更容易瞭解必須輸入的資料類型。預設類型為 XML。

  1. 在編輯管線之訊息流程中的動作時存取「測試主控台」。
    • 若要存取「XQuery 表示式編輯器」，請參閱建立和編輯內置 XQuery 和 XPath 表示式。
    • 若要存取「XQuery 條件編輯器」，請參閱建立和編輯內置 XQuery 和 XPath 表示式。

      • 隨即顯示 XQuery 表示式測試頁面。所有要求的輸入變數都會顯示在頁面上。

  2. 在頁面的變數區段中設定 XQuery 的測試資料。有關詳細資訊，請參閱表 32-8。
  3. 按一下「執行」。測試頁面會顯示結果。
  4. 完成測試之後，可以按一下返回執行新的測試。若要在變更 XQuery 後執行新的測試，您必須關閉並重新開啟「測試主控台」才能使變更生效。
