新增 UDDI 登錄檔

使用此頁面可以新增和設定 UDDI 登錄檔。設定登錄檔之後，就可以將 Oracle Service Bus 代理主機服務發佈至登錄檔，或從登錄檔匯入業務服務，以用於 Oracle Service Bus 代理主機服務。

您必須在作用中階段作業中才能新增登錄檔。

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「系統管理」>「UDDI 登錄檔」。
  3. 在「UDDI 登錄檔」面板中，按一下「新增登錄檔」。
  4. 若要設定登錄檔，請完成下列步驟。星號代表必要欄位。有關新增登錄檔時必須設定之特性的描述，請參閱表 29-1。
    1. 在名稱欄位中，輸入登錄檔的名稱。

      請遵循資源命名限制的命名指南。
    2. 在「詢問 URL」欄位中，以下列格式輸入詢問 URL：http://host:port/APPLICATION_SERVER_CONTEXT/uddi/inquiry
    3. 在「發佈 URL」欄位中，以下列格式輸入發佈 URL：http://host:port/APPLICATION_SERVER_CONTEXT/uddi/publishing
    4. 在「安全性 URL」欄位中，以下列格式輸入安全性 URL：http://host:port/APPLICATION_SERVER_CONTEXT/uddi/security
    5. 在「訂閱 URL」欄位中，以下列格式輸入訂閱 URL：
      http://host:port/APPLICATION_SERVER_CONTEXT/uddi/subscription

      6. 預設情況下，安裝程式中的 APPLICATION_SERVER_CONTEXT 值為 registry。

    6. 在「使用者名稱」欄位中，輸入用以登入登錄檔主控台的使用者名稱。
    7. 在「密碼」/(「確認密碼」) 欄位中，輸入用以登入登錄檔主控台的密碼。
    8. 選擇「將 tModels 載入至登錄檔」核取方塊，將 Oracle Service Bus tModels 發佈至此登錄檔。

      9. 只有將代理主機服務發佈至此登錄檔時，才需要使用此欄位。

    9. 選取啟用自動匯入核取方塊，以自動同步匯入的服務與 UDDI 登錄檔。任何在選擇此選項的狀態下匯入的服務，都會保持與 UDDI 登錄檔之間的同步。
      10. 注意： 自動同步為背景處理程序；您無法使用階段作業還原功能加以反向。因為服務將在下一個同步週期重新同步，所以對自動同步變更進行復原只是暫時的。如果要讓匯入的服務與 UDDI 登錄檔之間不同步，則必須中斷服務，使其無法從登錄檔執行進一步的更新。請參閱中斷服務。
  5. 按一下「驗證」以驗證您提供的 URL。
  6. 按一下「儲存」，以使用提供的設定來設定登錄檔，並將它儲存在目前的階段作業中。
  7. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
