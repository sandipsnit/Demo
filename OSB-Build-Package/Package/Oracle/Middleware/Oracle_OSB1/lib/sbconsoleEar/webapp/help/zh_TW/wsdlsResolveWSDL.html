解析未解析的 WSDL 參照

您可以在編輯 WSDL 資源的參照頁面設定 WSDL 匯入和 XML 綱要匯入這類 WSDL 參照的對應，來解析未解析的 WSDL 參照。WSDL 也可能會有對 WS 原則的未解析參照。

解析未解析的 WSDL 或 XML 綱要參照
  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「資源瀏覽器」>「WSDL」。
  3. 在 WSDL 的摘要頁面的 WSDL 名稱欄中，按一下適當的 WSDL 名稱以檢視未解析之參照的詳細資訊。隨即顯示檢視 WSDL 詳細資訊頁面。
  4. 按一下「編輯參照」，以顯示「編輯 WSDL 資源的參照」頁面。
  5. 在「資源類型」欄位中，選取「WSDL」或「XML 綱要」。
  6. 按一下瀏覽。視選取的資源類型而定，隨即顯示 WSDL 瀏覽器或 XML 綱要瀏覽器。
  7. 在「WSDL 瀏覽器」或「XML 綱要瀏覽器」中，選取 WSDL 或 XML 綱要，然後從「定義」窗格中選取定義。
  8. 按一下「提交」。選取的 WSDL 或「XML 綱要」會顯示在資源名稱欄位中。
  9. 按一下「儲存」以解析參照。
  10. 在檢視 WSDL 詳細資訊頁面中，按一下確定。WSDL 的狀態會顯示為有效。
解析未解析的 WS 原則參照
  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取「資源瀏覽器」>「WSDL」。
  3. 在 WSDL 的摘要頁面的 WSDL 名稱欄中，按一下適當的 WSDL 名稱以檢視未解析之參照的詳細資訊。隨即顯示檢視 WSDL 詳細資訊頁面。
  4. 按一下「編輯參照」，以顯示「編輯 WSDL 資源的參照」頁面。
  5. 在 WS 原則名稱欄位中，按一下瀏覽。隨即顯示原則瀏覽器。
  6. 在「原則瀏覽器」中，選取 WS 原則。
  7. 按一下「提交」。選取的「WS 原則」會顯示在 WS 原則名稱欄位中。
  8. 按一下「儲存」以解析參照。
  9. 在檢視 WSDL 詳細資訊頁面中，按一下確定。WSDL 的狀態會顯示為有效。
