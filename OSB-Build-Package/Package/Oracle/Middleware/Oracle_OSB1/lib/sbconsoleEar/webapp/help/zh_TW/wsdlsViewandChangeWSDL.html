編輯 WSDL

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 尋找 WSDL (如尋找 WSDL 中所述)。
  3. 按一下 WSDL 名稱。「檢視 WSDL 詳細資訊」頁面會顯示表 6-2、表 6-3 和表 6-4 中所顯示的資訊。

    4. 表 6-2 WSDL 詳細資訊 
    特性
    描述
    上次修改者
    建立此 WSDL 或將它匯入組態中的使用者。
    上次修改時間
    使用者建立此 WSDL 或將其匯入組態中的日期和時間。按一下日期和時間連結，以檢視此資源的變更歷史記錄。請參閱檢視變更歷史記錄頁面。
    參照
    此 WSDL 所參照的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。請參閱檢視資源參照。
    參照者
    參照此 WSDL 的物件數。如果有這類參照，請按一下數字連結以檢視物件清單。例如，如果您選取此 WSDL 連接埠或連結作為特定業務服務或代理主機服務的服務類型，則業務服務或代理主機服務會在您按一下連結時列為參照。請參閱檢視資源參照。
    描述
    此 WSDL 的描述 (如果存在的話)。

    預設會顯示 WSDL 的結構檢視畫面，其包含表 6-3 中所顯示的資訊。

    表 6-3 WSDL 結構詳細資訊
    特性
    描述
    目標命名空間
    用於限定 WSDL 所含任何定義的命名空間。
    WSDL 定義
    與 WSDL 相關聯的屬性和群組：
    • WSDL 連接埠類型
    • WSDL 連結
    • WSDL 連接埠
    • XML 綱要類型
    • XML 綱要元素
    • WSDL 匯入
    • XML 綱要匯入
    • WS 原則參照
    • 隱含 WS 原則參照
    WSDL 狀態
    WSDL 的狀態：
    • 有效

      • 已有效指定目前 WSDL 所含的所有 XML 綱要或 WSDL 的位置。此外，也已有效指定所有巢狀的 XML 綱要或 WSDL (所含的 WSDL 或 XML 綱要底下包含的其他 XML 綱要或 WSDL) 位置。

    • 無效

      • 未指定目前 WSDL 所含的一或多個 XML 綱要或 WSDL 的位置，或位置無效。此外，可能未指定一或多個巢狀 XML 綱要或 WSDL (所含的 WSDL 或 XML 綱要底下包含的其他 XML 綱要或 WSDL) 的位置，或位置無效。若要解析無效的 XML 綱要或 WSDL，請按一下「編輯參照」。請參閱解析未解析的 WSDL 參照。

  4. 按一下文字檢視畫面以顯示 WSDL 詳細資訊的文字檢視畫面。此文字檢視畫面包含表 6-4 中所示的資訊。

    5. 表 6-4 WSDL 文字詳細資訊
    特性
    描述
    目標命名空間
    用於限定 WSDL 所含任何定義的命名空間。
    文字
    此 WSDL 的文字。

  5. 若要變更欄位，請按一下編輯。有關欄位的描述，請參閱新增 WSDL。
  6. 進行適當的編輯。
  7. 按一下「儲存」，確認目前階段作業中的更新。
  8. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
注意： 檢視 WSDL 詳細資訊頁面還可能包含個別回呼按鈕。只有在 WSDL 的連接埠類型具有可重新寫入的回傳作業時，才會顯示此按鈕。按一下此按鈕會重新寫入 WSDL。
