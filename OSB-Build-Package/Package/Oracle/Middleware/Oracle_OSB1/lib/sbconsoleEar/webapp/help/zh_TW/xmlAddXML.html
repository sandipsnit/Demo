新增 XML 文件

在 Oracle Service Bus 中建立 XML 文件的最簡單方法是批次匯入。例如，如果您批次匯入已經建立的 JCA 資源 (.jca 檔案、關聯的 WSDL，及 TopLink 或 EclipseLink 對應檔案)，Oracle Service Bus 自動從對應檔案產生 XML 文件資源，並維護資源檔案之間的相依性。

如需更多有關匯入的資訊，請參閱從 Zip 檔案載入資源、從 URL 檔案載入資源以及匯入資源。

如果您沒有批次匯入，可以手動建立 XML 文件，如下列步驟所述：

  1. 如果尚未進行這樣的處理，請按一下建立以建立新的階段作業，或按一下編輯以輸入現有的階段作業。請參閱使用變更中心。
  2. 選取專案瀏覽器，然後選取要在其中新增「XML 文件」的專案或資料夾。隨即顯示專案/資料夾檢視畫面頁面。
  3. 從「建立資源」欄位，選取「XML 文件」，以顯示「建立新的 XML 文件」頁面。
  4. 在「XML 檔案」欄位中，按一下「瀏覽」然後選取您要使用的 XML 檔案。

    5. 「資源名稱」欄位會自動以檔案名稱 (除去副檔名) 填入。

    您可以在 XML 檔案欄位文字區域中，複製及貼上 XML 檔案的內容，而不是選取檔案。

  5. 在資源名稱欄位中，輸入或變更 XML 文件的名稱。XML 文件的名稱必須是唯一的。

    請遵循資源命名限制的命名指南。
  6. 在「資源描述」欄位中，輸入 XML 文件的描述。
  7. 按一下「儲存」。新的 XML 文件會儲存在目前的階段作業中。
  8. 若要結束階段作業並將組態建置至執行時期，請按一下「變更中心」下的「啟動」。
