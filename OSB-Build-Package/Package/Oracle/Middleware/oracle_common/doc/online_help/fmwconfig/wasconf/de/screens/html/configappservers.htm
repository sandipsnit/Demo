Anwendungsserver konfigurieren

Auf diesem Bildschirm können Sie Anwendungsserver zu jedem Knoten in der Zelle hinzufügen, clonen oder löschen. Außerdem können Sie die Einstellungen für einen vorhandenen Anwendungsserver ändern. Pflichtfelder sind durch ein Sternchen (*) gekennzeichnet.


Hinweis:

Verwenden Sie diesen Bildschirm nur für die Erstellung von Anwendungsservern, die entweder (a) zum primären Member eines Clusters werden oder (b) zu keinem Cluster gehören. Erstellen Sie auf diesem Bildschirm keine sekundären Cluster-Member. Verwenden Sie stattdessen den Bildschirm Zusätzliche Cluster-Member konfigurieren, um einem Cluster sekundäre Cluster-Member hinzuzufügen.

Beispiel: Sie möchten ein Cluster mit den drei Servern soa_server_1, soa_server_2 und soa_server_3 erstellen. Fügen Sie dazu den Anwendungsserver soa_server_1 auf dem Bildschirm Anwendungsserver konfigurieren hinzu. Erstellen Sie das Cluster auf den Bildschirmen Cluster konfigurieren, und wählen Sie soa_server_1 als primäres Cluster-Member. Dann können Sie soa_server_2 und soa_server_3 mit dem Bildschirm Zusätzliche Cluster-Member konfigurieren zum Cluster hinzufügen.

  • Name: Der Name des Anwendungsservers. Der Standardname für einen neuen Server ist new_ManagedServer_n, wobei n bei 1 beginnt und für jeden neuen hinzugefügten Server erhöht wird. Um den Servernamen zu ändern, klicken Sie auf das Feld Name, und geben Sie den neuen Namen ein.

  • Knotenname: Eine Dropdown-Liste mit allen verfügbaren Knoten. Klicken Sie auf die Liste, um den Knoten zu wählen, auf dem dieser Anwendungsserver gehalten wird. Wenn Sie noch keine anderen Rechner mit der aktuellen Zelle verbunden haben, ist nur ein Knoten in dieser Liste verfügbar.

  • Entwicklungsmodus: Aktivieren Sie dieses Kontrollkästchen, um den Anwendungsserver im Entwicklungsmodus auszuführen. Lassen Sie das Kontrollkästchen leer, um den Anwendungsserver im Production-Modus auszuführen. Informationen zu den Unterschieden zwischen dem Entwicklungsmodus und dem Production-Modus in einer WebSphere-Umgebung finden Sie in der WebSphere-Dokumentation.

Um einen Server hinzuzufügen, klicken Sie auf Hinzufügen, und konfigurieren Sie die Einstellungen für den neuen Server.

Um einen vorhandenen Server zu clonen, wählen Sie den zu clonenden Server, und klicken Sie auf Clonen. Der Tabelle wird eine neue Serverzeile hinzugefügt. Der neue Server übernimmt alle Attribute vom vorhandenen Server. Der Standardname des neuen Servers lautet clone_source_cloned1, wobei clone_source der Name des geclonten Servers ist. Bearbeiten Sie das Feld Name, wählen Sie den Knoten, auf dem der Server gehalten wird, und aktivieren oder deaktivieren Sie die Option Entwicklungsmodus.

Um einen Server zu löschen, wählen Sie den Server, und klicken Sie auf Löschen. Sie können jeweils immer nur einen Server löschen.

Um die Anzeige vom standardmäßigen tabellarischen Format in ein Registerkartenformat zu ändern (mit einer Registerkarte pro Server), klicken Sie auf Anzeige umschalten. Um zum tabellarischen Format zurückzukehren, klicken Sie erneut auf Anzeige umschalten.

Wenn Sie keine weiteren Server hinzufügen möchten, klicken Sie auf Weiter, um fortzufahren.


Hinweis:

Alle auf diesem Bildschirm vorgenommenen Änderungen werden erst in der Zellenkonfiguration gespeichert, wenn Sie im letzten Bildschirm des Konfigurationsassistenten auf Erstellen klicken. Wenn Sie den Assistenten verlassen oder auf Zurück klicken, um zu einem vorherigen Bildschirm zurückzukehren, müssen Sie die Änderungen erneut vornehmen.
