Endpunkte konfigurieren

Auf diesem Bildschirm können Sie die folgenden Endpunkte (Listening-Adressen) auf den einzelnen Servern, einschließlich des Administrationsservers, konfigurieren. Alle Endpunkte müssen in der Zelle eindeutig sein. Standardmäßig werden einem Server eindeutige Endpunkte zugewiesen, wenn Sie den Server zur Zelle hinzufügen. Sie müssen diese nur ändern, wenn Sie einen anderen Port verwenden möchten als den Standardport, der auf einem Server konfiguriert ist.

Informationen zu den verschiedenen auf diesem Bildschirm aufgelisteten Endpunkten finden Sie in der WebSphere-Dokumentation.

So ändern Sie Endpunkte:

  1. Wählen Sie in der Dropdown-Liste Servername/Knotenname den Server, für den Sie die Endpunkte ändern möchten. Die Liste enthält alle Server in der Zelle, einschließlich des Administrationsservers, der Anwendungsserver, die Clustern zugewiesen sind, und der Standalone-Anwendungsserver, die auf allen Knoten in der Zelle vorhanden sind.

  2. Geben Sie den neuen Endpunktwert in der entsprechenden Zeile ein. Wenn Sie einen bereits verwendeten Wert eingeben, wird der folgende Validierungsfehler angezeigt. Klicken Sie auf OK, und geben Sie einen anderen Wert für den Endpunkt ein.



    Hinweise:

    Wenn Sie mehrere Deployment Manager auf demselben Rechner ausführen möchten, müssen Sie sicherstellen, dass die Endpunkte in jeder verknüpften Zelle für alle Deployment Manager-Knoten auf dem Rechner eindeutig sind.

    Wenn Sie gleichermaßen Anwendungsserver auf Remote-Rechnern haben und diese Server verschiedenen Zellen auf dem Deployment Manager-Host zugewiesen sind, müssen Sie sicherstellen, dass alle Endpunkte für alle Anwendungsserver auf einem Rechner eindeutig sind.

    Sie erhalten keine Warnungen zu in Konflikt stehenden Endpunkten, die in verschiedenen Zellen vorhanden sind.

  3. Wiederholen Sie Schritt 2 für jeden Endpunkt, den Sie ändern möchten.

Sie können Endpunkte auch auf beliebigen Servern deaktivieren, wenn Sie sie nicht benötigen. Um einen oder mehrere Endpunkte auf einem Server zu deaktivieren, wählen Sie den Server aus der Dropdown-Liste Servername/Knotenname, und deaktivieren Sie das Kontrollkästchen für jeden Endpunkt, den Sie deaktivieren möchten.

Um einen gemeinsamen Host für alle Endpunkte auf einem Server zu verwenden, wählen Sie den Server aus der Dropdown-Liste Servername/Knotenname, und aktivieren Sie das Kontrollkästchen Gemeinsamen Host für alle Endpunkte verwenden. Wenn dieses Kontrollkästchen aktiviert ist, können die Hostnamen für jeden Endpunkt auf dem derzeit ausgewählten Server nicht mehr bearbeitet werden, mit Ausnahme des Admin-Endpunkt-Hostnamens.

Wenn Sie fertig sind, klicken Sie auf Weiter. Wenn Endpunktkonflikte vorhanden sind, wird das Dialogfeld Validierungsfehler angezeigt, und Sie müssen den Konflikt beheben, bevor Sie fortfahren können.
