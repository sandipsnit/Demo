JDBC-Komponentenschema konfigurieren

Für einige Fusion Middleware-Komponenten (Beispiel: SOA und Web Center Spaces) können JDBC-Datenquellen als Teil des Datenbankschemas der Komponente definiert werden. Sie werden dann während der Installation mit dem Utility für das Erstellen des Repositorys (RCU) geladen.

Wenn Sie mit dem Konfigurationsassistenten eine Zelle für solche Komponenten erstellen, können Sie die Schemaeinstellungen für die JDBC-Komponente, wie Datenbanktreiber, Schemaeigentümer, Kennwort usw., konfigurieren.

In der unteren Bildschirmhälfte sind die JDBC-Komponentenschemas aufgelistet, die mit den Produkten verknüpft sind, für die Sie die Zelle erstellen (Beispiel: SOA).


Hinweis:

Um eines oder mehrere der Schemas als Oracle RAC-Multidatenquellen zu konfigurieren, aktivieren Sie die Kontrollkästchen neben den Schemanamen, und aktivieren Sie das Kontrollkästchen Konfigurieren Sie gewählte Komponentenschemas als RAC-Datenquellschemas im nächsten Fensterbereich. Klicken Sie anschließend auf Weiter.

Wählen Sie die Schemas, für die Sie Datenquelleinstellungen vornehmen möchten, indem Sie die Kontrollkästchen neben den Schemanamen aktivieren.


Hinweis:

Wenn Sie mehrere Komponentenschemas auswählen, wird möglicherweise der Text Variiert unter Komponentenschemas in bestimmten Feldern angezeigt. Dies weist darauf hin, dass die aktuellen Werte dieser Felder nicht in allen ausgewählten Komponentenschemas identisch sind. Wenn Sie die Werte in solchen Feldern ändern, werden die neuen Werte einheitlich auf die gewählten Komponentenschemas angewendet.

Die Standardwerte von Komponentenschemaparametern wie Hersteller, Treiber, Hostname und Portnummer sind abhängig von den Werten, die in den Anwendungsvorlagen festgelegt sind.

  • Hersteller: Wählen Sie den Datenbankhersteller.

  • Treiber: Wählen Sie den JDBC-Treiber für den Verbindungsaufbau zur Datenbank. Die Liste enthält gängige JDBC-Treiber für den gewählten Datenbankhersteller.

  • Schemaeigentümer: Geben Sie den Benutzernamen für die Datenbankanmeldung ein.

  • Schemakennwort: Geben Sie das Kennwort für den angegebenen Schemaeigentümer an.

  • DBMS/Service: Geben Sie einen Datenbank-DBMS-Namen oder einen Servicenamen ein, wenn Sie einen Treiber für einen Servicetyp ausgewählt haben.

  • Hostname: Geben Sie den Namen des Servers ein, der die Datenbank hostet.

  • Port: Geben Sie die Portnummer für die Verbindung mit dem Server ein, der die Datenbank hostet.

Die von Ihnen angegebenen Werte werden für die gewählten Schemas in den entsprechenden Spalten in der Schemaliste angezeigt.

Wenn Sie die Konfiguration der Komponentenschemas abgeschlossen haben, klicken Sie auf Weiter, um fortzufahren.
