Zellen-, Profil- und Knoteninformationen angeben

Geben Sie die entsprechenden Werte in die folgenden Felder ein. Jedes Feld ist obligatorisch. Verwenden Sie nur alphanumerische Zeichen für jedes Feld.

  • Zellenname: Geben Sie einen eindeutigen Namen für die neue Zelle ein. Dies ist der Name des Verzeichnisses, in dem die Zelle erstellt wird, unter den folgenden zwei Verzeichnissen:

    • WAS_HOME/profiles/DepMgrProfile/config/cells, wobei WAS_HOME das WebSphere-Installationsverzeichnis und DepMgrProfile der Name des Deployment Manager-Profils ist.

    • WAS_HOME/profiles/AppSrvrProfile/config/cells, wobei WAS_HOME das WebSphere-Installationsverzeichnis und AppSrvrProfile der Name des Anwendungsserverprofils ist.

    Der Standardname lautet machine_nameCelln, wobei machine_name der Name des aktuellen Rechners und n die nächste zweistellige Nummer in der Sequence auf diesem Rechner ist. Beispiel: Wenn der Rechnername machine1 lautet und Cell01 bereits vorhanden ist, lautet der Standardzellenname "machine1Cell02".

  • Deployment Manager-Profilname: Geben Sie einen eindeutigen Deployment Manager-Profilnamen für diese Zelle ein. Dies ist der Name des Verzeichnisses, in dem das Deployment Manager-Profil erstellt wird, unter WAS_HOME/profiles.

    Der Standardname lautet Dmgrn, wobei n die nächste zweistellige Nummer in der Sequence ist. Beispiel: Wenn Deployment Manager-Profil "Dmgr01" bereits vorhanden ist, lautet der Standardwert für dieses Feld "Dmgr02".

  • Deployment Manager-Knotenname: Geben Sie einen eindeutigen Deployment Manager-Knotennamen für diese Zelle ein.

    Der Standardname lautet machine_nameCellManagern, wobei machine_name der Name des aktuellen Rechners und n die nächste zweistellige Nummer in der Sequence ist. Beispiel: Wenn der Rechnername machine1 lautet und CellManager01 bereits vorhanden ist, lautet der Standardknotenname "machine1CellManager02".

  • Anwendungsserver-Profilname: Geben Sie einen eindeutigen Anwendungsserver-Profilnamen für diese Zelle ein. Dies ist der Name des Verzeichnisses, in dem das Anwendungsserverprofil erstellt wird, unter WAS_HOME/profiles.

    Der Standardname lautet Customn, wobei n die nächste zweistellige Nummer in der Sequence ist. Beispiel: Wenn Anwendungsserverprofil "Custom01" bereits vorhanden ist, lautet der Standardwert für dieses Feld "Custom02".

  • Anwendungsserver-Knotenname: Geben Sie einen eindeutigen Anwendungsserver-Knotennamen für diese Zelle ein.

    Der Standardname lautet machine_nameNoden, wobei machine_name der Name des aktuellen Rechners und n die nächste zweistellige Nummer in der Sequence auf diesem Rechner ist. Beispiel: Wenn der Rechnername machine1 lautet und Node01 bereits vorhanden ist, lautet der Standardknotenname "machine1Node02".

Wenn Sie jedes Feld aufgefüllt haben, klicken Sie auf Weiter.
