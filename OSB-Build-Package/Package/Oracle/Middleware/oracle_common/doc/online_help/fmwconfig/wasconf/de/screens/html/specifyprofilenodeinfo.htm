Geben Sie Profil- und Knotennamensinformationen an

  • Anwendungsserver-Profilname: Geben Sie einen eindeutigen Anwendungsserver-Profilnamen für diesen Knoten ein. Dies ist der Name des Verzeichnisses, in dem das Anwendungsserverprofil erstellt wird, unter WAS_HOME/profiles auf dem Rechner, auf dem Sie den Knoten erstellen, und auf dem Deployment Manager-Host.

    Der Standardname lautet Customn, wobei n die nächste zweistellige Nummer in der Sequence auf dem Deployment Manager-Knoten ist, den Sie auf dem vorherigen Bildschirm angegeben haben. Beispiel: Wenn das Anwendungsserverprofil "Custom01" bereits in der Zelle vorhanden ist, lautet der Standardwert für dieses Feld "Custom02".

    Wenn Sie den Standardprofilnamen ändern, stellen Sie sicher, dass er innerhalb der Zelle eindeutig ist.

  • Anwendungsserver-Knotenname: Geben Sie den Knotennamen für diesen Anwendungsserver ein.

    Der Standardname lautet machine_nameNoden, wobei machine_name der Name des aktuellen Rechners und n die nächste zweistellige Nummer in der Sequence auf diesem Rechner ist. Beispiel: Wenn der Rechnername machine2 lautet und Node01 bereits auf dem Rechner vorhanden ist, lautet der Standardknotenname "machine2Node02".

    Wenn Sie den Standardknotennamen ändern, stellen Sie sicher, dass er innerhalb der Zelle eindeutig ist.

Wenn Sie jedes Feld aufgefüllt haben, klicken Sie auf Weiter, um fortzufahren.
