Ziel-Deployments auf Clustern oder Servern

Hier können Sie Anwendungen und Librarys für das Deployment auf Servern oder Clustern definieren.

Anwendungen und Librarys, die mit dem Produkt verknüpft sind, für das Sie die Zelle konfigurieren (Beispiel: SOA) werden automatisch auf den Server ausgerichtet, der für dieses Produkt erstellt wurde, oder auf das Cluster, dem dieser Server zugewiesen ist. Auf diesem Bildschirm können Sie Anwendungen und Librarys auf zusätzliche Server und Cluster ausrichten.

So richten Sie ein Anwendungs-Deployment auf ein Cluster oder einen Server aus:

  1. Wählen Sie im Listenfeld Ziel das Cluster oder den Server, auf dem Sie Anwendungen oder Librarys bereitstellen möchten.

    Der Name des gewählten Ziels wird als Titel des Listenfeldes auf der rechten Seite angezeigt.

  2. Aktivieren Sie im Listenfeld target_name die Kontrollkästchen für die Anwendungen oder Librarys, die auf dem gewählten Ziel bereitgestellt werden sollen.

    Die hier angezeigten Anwendungen und Librarys hängen von den Produkten ab, die Sie auf dem Bildschirm Produkte zur Zelle hinzufügen weiter vorne im Assistenten ausgewählt haben.


    Hinweis:

    Wenn Sie einen Server im Listenfeld Ziel wählen, sind einige der Kontrollkästchen im Listenfeld target_name möglicherweise deaktiviert. Dies weist auf Anwendungen und Librarys hin, die bereits auf das Cluster ausgerichtet sind, das den gewählten Server enthält.

    Nachdem Sie Anwendungen und Librarys ausgewählt haben, werden die Namen der als Ziele definierten Cluster und Server in der Spalte Ziel im Listenfeld target_name angezeigt.

  3. Wiederholen Sie die Schritte 1 und 2 nach Bedarf für weitere Cluster und Server.

  4. Wenn Sie alle erforderlichen Einstellungen vorgenommen haben, klicken Sie auf die Schaltfläche Weiter.

Wenn Sie eine vorhandene Zelle neu konfigurieren und einen Server oder ein Cluster löschen, auf den bzw. das derzeit Anwendungen ausgerichtet sind, richtet der Konfigurationsassistent diese Anwendungen automatisch wie folgt neu aus:

  • Wenn die Anwendungen ursprünglich nur auf den Server oder das Cluster ausgerichtet waren, den bzw. das Sie jetzt löschen (das heißt, nach dem Löschen des Servers oder Clusters hätten die Anwendungen in der geänderten Zelle keine Ziele mehr), richtet der Konfigurationsassistent automatisch die Anwendungen auf alle berechtigen Ziele neu aus.

    Berechtigte Ziele sind alle Cluster oder Server, die nicht in der Konfigurationsgruppenspezifikation (config-groups.xml) einer enthaltenen Vorlage definiert sind. In der Datei config-groups.xml angegebene Server oder Cluster gehören zu der Vorlage und sind daher nicht für die automatische Zieldefinition berechtigt.

  • Falls die Anwendungen ursprünglich auf mehrere Ziele ausgerichtet waren (einschließlich Server, Cluster und Administrationsserver) und einer der ausgerichteten Server oder Cluster gelöscht wird, werden die verbleibenden Zielverknüpfungen beibehalten. Der Konfigurationsassistent nimmt keine Neuausrichtung der Anwendungen vor.
