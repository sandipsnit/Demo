Configurar Servidores de Aplicaciones

En esta pantalla, puede agregar, clonar o suprimir servidores de aplicaciones de cualquier nodo de la celda. También puede cambiar la configuración de un servidor de de aplicaciones existente. Los campos necesarios se indican con un asterisco (*).


Nota:

Utilice esta pantalla sólo para crear servidores de aplicaciones que, o bien (a) vayan a ser el miembro primario de un cluster o (b) no vayan a formar parte de ningún cluster. No cree miembros de cluster secundarios en esta pantalla. Para ello, utilice la pantalla Configurar Miembros de Cluster Adicionales para agregar miembros de cluster secundarios a un cluster.

Por ejemplo, suponga que desea crear un cluster de tres servidores con los servidores soa_server_1, soa_server_2 y soa_server_3. Para ello, agregue el servidor de aplicaciones soa_server_1 en la pantalla Configurar Servidores de Aplicaciones. Cree el cluster en las pantallas Configurar Clusters y seleccione soa_server_1 como miembro primario del cluster. A continuación, puede utilizar la pantalla Configurar Miembros de Cluster Adicionales para agregar soa_server_2 y soa_server_3 al cluster.

  • Nombre: Nombre del servidor de aplicaciones. El nombre por defecto de un servidor nuevo es new_ManagedServer_n, donde el valor n empieza en 1 y se va incrementando para cada nuevo servidor que agregue. Para cambiar el nombre del servidor, haga clic en el campo Nombre e introduzca un nombre nuevo.

  • Nombre del Nodo: Lista desplegable de todos los nodos disponibles. Haga clic en la lista para seleccionar el nodo en el que reside este servidor de aplicaciones. Si aún no ha federado otras máquinas en la celda actual, sólo habrá un nodo disponible en esta lista.

  • Modo de Desarrollo: Active esta casilla de control para ejecutar el servidor de aplicaciones en modo de desarrollo. Deje esta casilla sin marcar para ejecutar el servidor de aplicaciones en modo de producción. Para obtener información acerca de las diferencias entre el modo de producción y el modo de desarrollo, consulte la documentación de WebSphere.

Para agregar un servidor, haga clic en Agregar y establezca la configuración del servidor nuevo.

Para clonar un servidor ya existente, seleccione el servidor que desee clonar y haga clic en Clonar. Se agrega una línea de servidor nueva a la tabla. El servidor nuevo hereda todos los atributos del servidor existente. El nombre por defecto del servidor nuevo es clone_source_cloned1, donde clone_source es el nombre del servidor que ha clonado. Edite el campo Nombre, seleccione el nodo en el que reside el servidor y active o desactive la opción Modo de Desarrollo.

Para suprimir un servidor, seleccione el servidor y haga clic en Suprimir. Sólo se puede suprimir un servidor a la vez.

Para cambiar la visualización de formato tabular por defecto (con un separador por cada servidor), haga clic en Cambiar Visualización. Vuelva a hacer clic en Cambiar Visualización para volver al formato tabular.

Cuando haya terminado de agregar servidores, haga clic en Siguiente para continuar.


Nota:

Los cambios que realice en esta pantalla no se guardan en la configuración de la celda hasta que no se hace clic en Crear en la última pantalla del Asistente de Configuración. Si sale del asistente o hace clic en Anterior para volver a la pantalla anterior, deberá volver a realizar los cambios.
