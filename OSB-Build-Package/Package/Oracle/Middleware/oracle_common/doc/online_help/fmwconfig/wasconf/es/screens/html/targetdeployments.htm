Dirigir Despliegues a Clusters o Servidores

Utilice esta pantalla para dirigir aplicaciones y bibliotecas para su despliegue en servidores o clusters.

Las aplicaciones y bibliotecas asociadas al producto para el que va a configurar la celda (por ejemplo, SOA) se dirigen automáticamente al servidor que se ha creado para dicho producto o para el cluster al que se ha asignado ese servidor. En esta pantalla, puede dirigir aplicaciones y bibliotecas a servidores y clusters adicionales.

Para dirigir un despliegue de aplicación a un cluster o servidor:

  1. En el cuadro de lista Destino, seleccione el cluster o el servidor en el que desee desplegar las aplicaciones o bibliotecas.

    El nombre del destino seleccionado se muestra como el título del cuadro de lista de la derecha.

  2. En el cuadro de lista con los nombres de destino, active las casillas de control correspondientes a las aplicaciones que va a desplegar en el destino seleccionado.

    Las aplicaciones y bibliotecas que se muestran aquí varían en función de los productos seleccionados en la pantalla anterior del asistente Agregar Productos a Celda.


    Nota:

    Si selecciona un servidor en el cuadro de lista Destino, algunas de las casillas de control del cuadro de lista con los nombres de destino pueden estar desactivadas, lo que indica las aplicaciones y bibliotecas que ya se han dirigido al cluster que contiene el servidor seleccionado.

    Después de seleccionar las aplicaciones y bibliotecas, los nombres de los clusters de destino se muestran en la columna Destino del cuadro de lista con los nombres de destino.

  3. Repita los pasos 1 y 2 para los demás clusters y servidores según sea necesario.

  4. Después de realizar las selecciones requeridas, haga clic en Siguiente.

Si al volver a configurar una celda existente suprime un servidor o cluster al que se han dirigido actualmente aplicaciones, el Asistente de Configuración redirige automáticamente esas aplicaciones de la siguiente manera:

  • Si originalmente las aplicaciones estaban destinadas sólo al servidor o al cluster que está suprimiendo (es decir, después de suprimir el servidor o el cluster, las aplicaciones dejarían de estar destinadas en la celda modificada), el Asistente de Configuración redirige automáticamente las aplicaciones a todos los destinos seleccionables.

    Un destino seleccionable es cualquier cluster o servidor que no se haya definido en la especificación de grupos de configuración (archivo config-groups.xml) de una plantilla incluida. Los servidores o clusters que se especifiquen en config-groups.xml son esencialmente propiedad de la plantilla y, por lo tanto, no se pueden seleccionar como destinos automáticos.

  • Si las aplicaciones estaban destinadas originalmente a varios destinos (incluidos servidores, clusters y el servidor de administración) y se suprime uno de los servidores o clusters de destino, el Asistente de Configuración deja intactas las asociaciones de destino restantes y no intenta redirigir las aplicaciones.
