Configurer les serveurs d'applications

A partir de cet écran, vous pouvez ajouter, cloner ou supprimer des serveurs d'applications sur n'importe quel noeud de la cellule. Vous pouvez également modifier les paramètres d'un serveur d'applications existant. Les champs obligatoires sont signalés par un astérisque (*).


Remarque :

Utilisez cet écran uniquement pour créer des serveurs d'applications qui (a) constitueront le membre principal d'un cluster ou (b) qui ne feront pas partie d'un cluster. Ne créez pas des membres de cluster secondaires sur cet écran. Utilisez plutôt l'écran Configurer les membres de cluster supplémentaires pour ajouter des membres de cluster secondaires à un cluster.

Par exemple, si vous souhaitez créer un cluster à trois serveurs avec les serveurs soa_server_1, soa_server_2 et soa_server_3, vous devez ajouter le serveur d'applications soa_server_1 sur l'écran Configurer les serveurs d'applications. Ensuite, créez le cluster sur les écrans Configurer les clusters et sélectionnez soa_server_1 en tant que membre principal du cluster. Vous pouvez ensuite utiliser l'écran Configurer les membres de cluster supplémentaires pour ajouter les serveurs soa_server_2 et soa_server_3 au cluster.

  • Nom : nom du serveur d'applications. Le nom par défaut d'un nouveau serveur est new_ManagedServer_n, où n commence à 1 et est incrémenté à chaque nouveau serveur ajouté. Pour modifier le nom du serveur, cliquez sur le champ Nom et saisissez le nouveau nom.

  • Nom du noeud : liste déroulante de tous les noeuds disponibles. Cliquez sur la liste pour sélectionner le noeud sur lequel le serveur d'applications réside. Si vous n'avez pas encore fédéré d'autres ordinateurs à la cellule en cours, un seul noeud est disponible dans cette liste.

  • Mode Développement : cochez cette case pour exécuter le serveur d'applications en mode développement. Laissez-la vide si vous souhaitez qu'il soit exécuté en mode production. Pour plus d'informations sur les différences entre le mode production et le mode développement dans un environnement WebSphere, reportez-vous à la documentation WebSphere.

Pour ajouter un serveur, cliquez sur Ajouter, puis configurez les paramètres du nouveau serveur.

Pour cloner un serveur existant, sélectionnez le serveur en question, puis cliquez sur Cloner. Une nouvelle ligne de serveur est ajoutée à la table. Le nouveau serveur hérite de tous les attributs du serveur existant. Le nom par défaut du nouveau serveur est clone_source_cloned1, où clone_source correspond au nom du serveur cloné. Modifiez le champ Nom, sélectionnez le noeud sur lequel le serveur réside, puis sélectionnez ou désélectionnez l'option Mode Développement.

Pour supprimer un serveur, sélectionnez-le et cliquez sur Supprimer. Vous ne pouvez supprimer qu'un seul serveur à la fois.

Pour passer de l'affichage par défaut sous forme de table à un affichage à onglets (un onglet par serveur), cliquez sur Changer d'affichage. Cliquez de nouveau sur Changer d'affichage pour revenir au format tabulaire.

Une fois l'ajout de serveurs terminé, cliquez sur Suivant pour continuer.


Remarque :

Les modifications apportées sur cet écran ne seront enregistrées dans la configuration de cellule que lorsque vous cliquerez sur Créer sur le dernier écran de l'assistant Configuration. Si vous quittez l'assistant ou cliquez sur Précédent pour revenir à un écran précédent, vous devez effectuer à nouveau les modifications.
