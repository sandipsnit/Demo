Configurer les adresses

Utilisez cet écran pour configurer les adresses (d'écoute) suivantes sur chacun des serveurs, notamment le serveur d'administration. Toutes les adresses doivent être uniques dans la cellule. Par défaut, des adresses uniques sont affectées à un serveur lorsque vous ajoutez ce dernier à la cellule. Il n'est pas nécessaire de les modifier à moins que vous ne souhaitiez utiliser un port différent du port par défaut configuré sur un serveur donné.

Pour plus d'informations sur les diverses adresses répertoriées sur cet écran, reportez-vous à la documentation WebSphere.

Pour modifier des adresses, procédez comme suit :

  1. Dans la liste déroulante Nom du serveur/Nom du noeud, sélectionnez le serveur pour lequel vous souhaitez modifier les adresses. La liste contient tous les serveurs dans la cellule, notamment le serveur d'administration, les serveurs d'applications affectés aux clusters et les serveurs d'applications autonomes se trouvant sur tous les noeuds de la cellule.

  2. Saisissez la nouvelle valeur d'adresse sur la ligne appropriée. Si vous saisissez une valeur déjà utilisée, l'erreur de validation suivante apparaît. Cliquez alors sur OK et saisissez une nouvelle valeur pour l'adresse.



    Remarques :

    Si vous envisagez d'exécuter plusieurs gestionnaires de déploiement sur le même ordinateur, vous devez vous assurer que les adresses dans chaque cellule associée sont uniques pour tous les noeuds de gestionnaire de déploiement sur cet ordinateur.

    De même, si vous possédez des serveurs d'applications sur des ordinateurs distants et que ces serveurs sont affectés à des cellules différentes sur l'hôte du gestionnaire de déploiement, vous devez vous assurer que toutes les adresses sont uniques, pour l'intégralité des serveurs d'applications sur un ordinateur donné.

    Vous ne serez pas averti en cas de conflit d'adresses dans diverses cellules.

  3. Répétez l'étape 2 pour chaque adresse à modifier.

Vous pouvez également désactiver une adresse sur un serveur quelconque si vous n'en avez pas besoin. Pour désactiver des adresses sur un serveur, sélectionnez ce dernier dans la liste déroulante Nom du serveur/Nom du noeud et désélectionnez les cases associées aux adresses à désactiver.

Pour utiliser un hôte commun pour toutes les adresses d'un serveur donné, sélectionnez ce serveur dans la liste déroulante Nom du serveur/Nom du noeud, puis cochez la case Appliquer un hôte commun à toutes les adresses. Une fois cette case cochée, vous ne pouvez plus modifier les noms d'hôte de chaque adresse du serveur sélectionné, à l'exception du nom d'hôte Admin end point (adresse admin).

Lorsque vous avez terminé, cliquez sur Suivant. S'il existe des conflits d'adresse, la boîte de dialogue Erreur de validation apparaît et vous devez résoudre le conflit pour pouvoir continuer.
