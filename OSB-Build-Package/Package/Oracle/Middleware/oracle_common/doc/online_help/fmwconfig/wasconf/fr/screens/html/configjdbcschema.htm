Configurer le schéma de composant JDBC

Pour certains composants Fusion Middleware (par exemple, SOA et Web Center Spaces), les sources de données JDBC peuvent être définies comme éléments du schéma de base de données du composant, chargés (lors de l'installation) via l'utilitaire de création de référentiel (RCU).

Lorsque vous créez une cellule pour ces composants à l'aide de l'assistant Configuration, vous pouvez configurer les paramètres de schéma de composant JDBC, tels que le pilote de base de données, le propriétaire de schéma, le mot de passe, etc.

Les schémas de composant JDBC associés aux produits pour lesquels vous créez la cellule (par exemple, SOA) sont répertoriés dans la partie inférieure de l'écran.


Remarque :

Pour configurer des schémas en tant que sources de données multiples Oracle RAC, cochez les cases situées en regard du nom des schémas, puis cochez la case Configurer les schémas de composant sélectionnés en tant que schéma de sources de données multiples RAC dans le panneau suivant. Cliquez sur Suivant lorsque vous avez terminé.

Pour sélectionner les schémas pour lesquels indiquer des paramètres de source de données, cochez la case située en regard de leur nom.


Remarque :

Si vous sélectionnez plusieurs schémas de composant, le texte Varie selon les schémas de composant peut apparaître dans certains champs, ce qui indique que les valeurs en cours de ces champs sont différentes dans les schémas de composant sélectionnés. Si vous modifiez les valeurs de ces champs, les nouvelles valeurs sont appliquées de façon uniforme dans les schémas de composant sélectionnés.

Les valeurs par défaut des paramètres de schéma de composant tels que Fournisseur, Pilote, Nom d'hôte et Numéro de port dépendent des valeurs indiquées dans les modèles d'application.

  • Fournisseur : sélectionnez le fournisseur de base de données.

  • Pilote : sélectionnez le pilote JDBC à utiliser pour la connexion à la base de données. La liste comprend les pilotes JDBC standard pour le fournisseur de base de données sélectionné.

  • Propriétaire de schéma : saisissez le nom utilisateur à employer pour la connexion à la base de données.

  • Mot de passe de schéma : saisissez le mot de passe correspondant au propriétaire de schéma indiqué.

  • SGBD/Service : entrez un nom SGBD de base de données ou un nom de service si vous avez sélectionné un pilote de type service.

  • Nom d'hôte : saisissez le nom du serveur hébergeant la base de données.

  • Port : entrez le numéro de port à utiliser pour la connexion au serveur qui héberge la base de données.

Les valeurs que vous indiquez sont affichées dans les colonnes appropriées de la liste de schémas, pour les schémas sélectionnés.

Une fois la configuration des schémas de composant terminée, cliquez sur Suivant pour continuer.
