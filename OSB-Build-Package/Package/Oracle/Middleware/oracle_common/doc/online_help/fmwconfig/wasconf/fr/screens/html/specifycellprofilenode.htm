Indiquer le nom de la cellule, du profil et du noeud

Saisissez les valeurs appropriées dans les champs suivants. Chaque champ est obligatoire. Utilisez uniquement des caractères alphanumériques dans chaque champ.

  • Nom de la cellule : saisissez un nom unique pour la nouvelle cellule. Il s'agit du nom du répertoire dans lequel la cellule est créée, sous les deux répertoires suivants :

    • WAS_HOME/profiles/DepMgrProfile/config/cells, où WAS_HOME correspond au répertoire d'installation WebSphere et DepMgrProfile désigne le nom du profil de gestionnaire de déploiement.

    • WAS_HOME/profiles/AppSrvrProfile/config/cells, où WAS_HOME correspond au répertoire d'installation WebSphere et AppSrvrProfile désigne le nom du profil du serveur d'applications.

    Le nom par défaut est machine_nameCelln, où machine_name correspond au nom de l'ordinateur en cours et n désigne le nombre à deux chiffres suivant dans la séquence sur cet ordinateur. Par exemple, si le nom de l'ordinateur est machine1 et que Cell01 existe déjà, le nom par défaut de la cellule est machine1Cell02.

  • Nom du profil du gestionnaire de déploiement : saisissez un nom de profil de gestionnaire de déploiement unique pour cette cellule. Il s'agit du nom du répertoire dans lequel le profil de gestionnaire de déploiement est créé, sous WAS_HOME/profiles.

    Le nom par défaut est Dmgrn, où n est le nombre à deux chiffres suivant dans la séquence. Par exemple, si le profil de gestionnaire de déploiement Dmgr01 existe déjà, le nom par défaut de ce champ est Dmgr02.

  • Nom du noeud du gestionnaire de déploiement : saisissez un nom de noeud de gestionnaire de déploiement unique pour cette cellule.

    Le nom par défaut est machine_nameCellManagern, où machine_name correspond au nom de l'ordinateur en cours et n désigne le nombre à deux chiffres suivant dans la séquence. Par exemple, si le nom de l'ordinateur est machine1 et que CellManager01 existe déjà, le nom de noeud par défaut est machine1CellManager02.

  • Nom du profil du serveur d'applications : saisissez un nom de profil de serveur d'applications unique pour cette cellule. Il s'agit du nom du répertoire dans lequel le profil du serveur d'applications est créé, sous WAS_HOME/profiles.

    Le nom par défaut est Customn, où n correspond au nombre à deux chiffres suivant dans la séquence. Par exemple, si le profil du serveur d'applications Custom01 existe déjà, le nom par défaut de ce champ est Custom02.

  • Nom du noeud du serveur d'applications : saisissez un nom de noeud de serveur d'applications unique pour cette cellule.

    Le nom par défaut est machine_nameNoden, où machine_name correspond au nom de l'ordinateur en cours et n désigne le nombre à deux chiffres suivant dans la séquence sur cet ordinateur. Par exemple, si le nom de l'ordinateur est machine1 et que Node01 existe déjà, le nom de noeud par défaut est machine1Node02.

Une fois chaque champ rempli, cliquez sur Suivant.
