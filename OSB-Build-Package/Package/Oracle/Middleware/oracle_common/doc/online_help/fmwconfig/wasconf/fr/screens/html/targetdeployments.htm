Cibler les déploiements sur des clusters ou des serveurs

Utilisez cet écran pour cibler des applications et des bibliothèques afin de les déployer sur des serveurs ou des clusters.

Les applications et les bibliothèques associées au produit pour lequel vous configurez la cellule (par exemple, SOA) sont ciblées automatiquement sur le serveur créé pour ce produit ou sur le cluster auquel ce serveur géré est affecté. Dans cet écran, vous pouvez cibler les applications et les bibliothèques sur des serveurs et clusters supplémentaires.

Pour cibler un déploiement d'application sur un cluster ou serveur, procédez comme suit :

  1. Dans la zone de liste Target, sélectionnez le cluster ou le serveur sur lequel déployer les applications ou les bibliothèques.

    Le nom de la cible sélectionnée est affiché en tant que titre de la zone de liste à droite.

  2. Dans la zone de liste target_name, cochez les cases correspondant aux applications ou bibliothèques à déployer sur la cible sélectionnée.

    Les applications et bibliothèques affichées ici varient selon les produits sélectionnés précédemment sur l'écran Ajouter des produits à la cellule de l'assistant.


    Remarque :

    Lorsque vous sélectionnez un serveur dans la zone de liste Target, certaines cases à cocher de la zone de liste target_name peuvent être désactivées ; elles correspondent aux applications et bibliothèques déjà ciblées sur le cluster contenant le serveur sélectionné.

    Une fois que vous avez sélectionné les applications et les bibliothèques voulues, les noms des clusters et serveurs ciblés apparaissent dans la colonne Target de la zone de liste target_name.

  3. Répétez les étapes 1 et 2 pour les autres clusters et serveurs requis.

  4. Une fois les sélections obligatoires créées, cliquez sur Suivant.

Lorsque vous reconfigurez une cellule existante et que vous supprimez un serveur ou un cluster sur lequel les applications sont ciblées, l'assistant Configuration cible de nouveau automatiquement ces dernières comme suit :

  • Si les applications n'ont été initialement ciblées que sur le serveur ou le cluster que vous supprimez (au terme de la suppression, elles ne sont pas ciblées dans la cellule modifiée), l'assistant Configuration cible de nouveau automatiquement ces applications sur toutes les cibles admissibles.

    Une cible admissible est un cluster ou un serveur non défini dans la spécification de groupes de configuration (fichier config-groups.xml) d'un modèle inclus. Les serveurs ou les clusters indiqués dans le fichier config-groups.xml appartiennent essentiellement au modèle et ne peuvent donc pas faire l'objet d'un ciblage automatique.

  • Si les applications ont été initialement ciblées sur plusieurs cibles (y compris les serveurs, les clusters et le serveur d'administration), et que l'un des serveurs ou clusters ciblés est supprimé, l'assistant Configuration laisse les autres associations de cibles intactes et ne tente pas de cibler de nouveau ces applications.
