Configura Application Server

In questa schermata è possibile aggiungere, duplicare o eliminare Application Server in qualsiasi nodo nella cella. Inoltre, è possibile modificare le impostazioni per un Application Server esistente. I campi obbligatori sono indicati da asterisco (*).


Nota:

utilizzare questa schermata solo per creare Application Server che (a) siano membri primari di un cluster o che (b) non facciano parte di un cluster. Non creare membri secondari di un cluster da questa schermata. In alternativa, utilizzare la schermata Configura membri cluster aggiuntivi per aggiungere membri secondari a un cluster.

Ad esempio, si supponga di voler creare un cluster a tre server con server soa_server_1, soa_server_2 e soa_server_3. Per fare ciò, aggiungere l'Application Server soa_server_1 nella schermata Configura Application Server. Creare il cluster nella schermataConfigura cluster e selezionare soa_server_1 come membro cluster primario. Quindi sarà possibile utilizzare la schermata Configura membri cluster aggiuntivi per aggiungere soa_server_2 e soa_server_3 al cluster.

  • Nome: il nome dell'Application Server. Il nome predefinito per un nuovo server è new_ManagedServer_n, dove n inizia da 1 e aumenta per ogni nuovo server aggiunto. Per modificare il nome del server, fare clic nel campo Nome e digitare il nuovo nome.

  • Nome nodo: un elenco a discesa di tutti i nodi disponibili. Fare clic sulla lista per selezionare il nodo sul quale risiede l'Application Server. Se non si sono ancora federati altri computer alla cella corrente, nella lista sarà presente un solo nodo.

  • Modalità di sviluppo: selezionare questa casella se si desidera eseguire l'Application Server in modalità di sviluppo. Lasciare la casella non selezionata per eseguire l'Application Server in modalità di produzione. Per le informazioni relative alle differenze fra le due modalità in ambiente WebSphere, fare riferimento alla documentazione di WebSphere.

Per aggiungere un server, fare clic su Aggiungi e configurare le impostazione per il nuovo server.

Per duplicare un server esistente, selezionare il server da duplicare e fare clic su Duplica. Alla tabella viene aggiunto un nuovo server che eredita tutti gli attributi del server esistente. Il nome predefinito del nuovo server è clone_source_cloned1, dove clone_source è il nome del server duplicato. Modificare il campo Nome, selezionare il nodo sul quale risiede il server e selezionare o annullare l'opzione Modalità di sviluppo.

Per eliminare un server, selezionarlo e fare clic su Elimina. È possibile eliminare un solo server alla volta.

Per passare dal formato di visualizzazione tabulare predefinito al formato a schede (con una scheda per ogni server), fare clic su Cambia visualizzazione. Fare di nuovo clic su Cambia visualizzazione per tornare al formato tabulare.

Al termine, fare clic su Successivo per continuare.


Nota:

Tutte le modifiche apportate in questa schermata non sono salvate nella configurazione della cella fino a quando non si fa clic su Crea nell'ultima schermata della configurazione guidata. Se si esce dalla procedura guidata o si fa clic su Precedente per ritornare a una schermata precedente, è necessario apportare di nuovo le modifiche.
