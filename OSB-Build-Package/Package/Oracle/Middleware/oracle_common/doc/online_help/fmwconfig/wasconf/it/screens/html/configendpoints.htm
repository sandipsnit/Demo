Configura endpoint

Utilizzare questa schermata per configurare gli endpoint riportati di seguito (indirizzi di ascolto) su ciascun server, incluso il Server di amministrazione. Tutti gli endpoint devono essere univoci nella cella. Per impostazione predefinita, gli endpoint univoci sono assegnati a un server quando si aggiunge il server alla cella. Non è necessario modificarli a meno che non si desideri utilizzare una porta diversa da quella predefinita configurata per un determinato server.

Per le informazioni sui vari endpoint elencati nella schermata fare riferimento alla documentazione di WebSphere.

Per modificare gli endpoint:

  1. Nell'elenco a discesa Nome server/Nome nodo, selezionare il server per il quale si desidera modificare gli end point. L'elenco contiene tutti i server nella cella, incluso il Server di amministrazione, gli Application Server assegnati ai cluster e gli Application Server autonomi esistenti su tutti i nodi nella cella.

  2. Digitare il nuovo valore dell'endpoint nella riga appropriata. Se si immette un valore già utilizzato, viene visualizzato l'errore riportato di seguito. Fare clic su OK e immettere un valore diverso per l'endpoint.



    Note:

    Se si prevede di eseguire più Deployment Manager sullo stesso computer, è necessario assicurarsi che gli endpoint in ciascuna cella associata siano univoci su tutti i nodi Deploymenti Manager sul computer.

    Analogamente, se si dispone di Application Server su macchine remote, e tali server sono assegnati a celle diverse sull'host Deployment Manager, è necessario assicurarsi che tutti gli endoint per tutti gli Application Server su un dato computer siano univoci.

    Non si riceveranno avvertenze relative a conflitti di endpoint esistenti su celle diverse.

  3. Ripetere il passo 2 per ciascun endopoint da modificare.

È anche possibile disattivare un endpoint su un server se non è necessario. Per disattivare uno o più endpoint su un server, selezionare il server dall'elenco a discesa Nome server/Nome nodo e annullare la selezione delle caselle di controllo relative a ciascun endpoint da disattivare.

Per utilizzare un host comune per tutti gli endpoint su un determinato server, selezionare il server dall'elenco a discesa Nome server/Nome nodo e selezionare la casella di controllo Usa host comune per tutti gli endpoint. Quando la casella è selezionata, i nomi host per ciascun endopoint sul server selezionato non sono più modificabili a eccezione del nome host Admin end point.

Al termine, fare clic su Successivo. Se si verificano conflitti relativi agli endpoint, viene visualizzato il messaggio Errore di convalida ed è necessario correggere gli errori prima di continuare.
