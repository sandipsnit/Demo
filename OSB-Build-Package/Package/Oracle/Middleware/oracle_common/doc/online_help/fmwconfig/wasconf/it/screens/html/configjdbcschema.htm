Configura schema dei componenti JDBC

Per alcuni componenti di Fusion Middleware (ad esempio, SOA e WebCenter Spaces), è possibile che le origini dati JDBC siano definite come parte dello schema di database del componente e vengano caricate durante l'installazione utilizzando Repository Creation Utility (RCU).

Quando si crea una cella per tali componenti utilizzando la Configurazione guidata, è possibile configurare le impostazioni degli schemi dei componenti JDBC, come ad esempio il driver di database, il proprietario dello schema, la password e così via.

Gli schemi dei componenti JDBC associati ai prodotti per cui si sta creando la cella (ad esempio, SOA) sono elencati nella parte inferiore della schermata.


Nota:

Per configurare uno o più schemi come origini dati multiple Oracle RAC, selezionare le caselle di controllo adiacenti ai nomi degli schemi e selezionare la casella di controllo Configurare nel pannello successivo gli schemi dei componenti selezionati come schemi di origini dati multiple RAC. Fare clic su Successivo una volta completata l'operazione.

Scegliere gli schemi per cui si desidera specificare le impostazioni delle origini dati selezionando la casella di controllo accanto ai nomi degli schemi.


Nota:

quando si selezionano più schemi dei componenti, in alcuni campi potrebbe essere visualizzato il testo Varia a seconda degli schemi dei componenti, indicante che i valori correnti di questi campi non sono gli stessi per tutti gli schemi dei componenti selezionati. Se si prosegue e si modificano i valori in tali campi, i nuovi valori vengono applicati in modo uniforme agli schemi dei componenti selezionati.

I valori predefiniti dei parametri degli schemi dei componenti quali fornitore, driver, nome host e numero di porta dipendono dai valori specificati nei modelli di applicazione.

  • Fornitore: selezionare il fornitore del database.

  • Driver: selezionare il driver JDBC che si desidera utilizzare per connettersi al database. L'elenco contiene i driver JDBC comuni per il DBMS selezionato.

  • Proprietario schema: immettere il nome utente per la connessione al database.

  • Password schema : immettere la password per il proprietario schema selezionato.

  • DBMS/Servizio - Immettere un nome DBMS per il database o il nome del servizio se è stato selezionato un driver del tipo di servizio.

  • Nome host : immettere il nome del server su cui si trova il database.

  • Porta : immettere il numero di porta da usare per la connessione al server su cui si trova il database.

I valori specificati vengono visualizzati nelle colonne appropriate della lista di schemi per gli schemi selezionati.

Al termine, fare clic su Successivo poer continuare.
