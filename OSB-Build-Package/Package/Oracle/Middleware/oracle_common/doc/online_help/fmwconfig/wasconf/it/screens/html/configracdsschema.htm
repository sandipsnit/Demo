Configura schema dei componenti per origine dati RAC

Utilizzare questa schermata per configurare gli schemi dei componenti inclusi nella cella WebSphere come schemi Oracle RAC.

Gli schemi dei componenti scelti da configurare come schemi Oracle RAC nella schermata Configura schema dei componenti JDBC della procedura guidata sono elencati nella parte inferiore della schermata.

Nella lista di schemi nella parte inferiore della schermata, scegliere gli schemi che si desidera configurare come schemi Oracle RAC selezionando la casella di controllo accanto al nome dello schema.


Nota:

quando si selezionano più schemi dei componenti, in alcuni campi potrebbe essere visualizzato il testo Varia a seconda degli schemi dei componenti, indicante che i valori correnti di questi campi non sono gli stessi per tutti gli schemi dei componenti selezionati. Se si prosegue e si modificano i valori in tali campi, i nuovi valori vengono applicati in modo uniforme agli schemi dei componenti selezionati.

  • Driver : selezionare il driver JDBC da utilizzare per connettersi al database.

  • Nome servizio : immettere un nome di servizio per il database Oracle RAC.

  • Nome utente - immettere il nome utente per la connessione al database.

  • Password - immettere la password per il nome utente specificato.

  • Nome host - immettere il nome del server su cui si trovano le istanze di database Oracle RAC.

  • Nome istanza - immettere il nome di ogni istanza di database Oracle.

  • Numero di porta - immettere il numero di porta da usare per la connessione al server su cui si trova il database.


Nota:

è necessario specificare il nome host, il nome istanza e il numero di porta di almeno un'istanza di database.

Per aggiungere un'istanza di database, fare clic su Aggiungi e quindi specificare il nome host, il nome dell'istanza e il numero di porta.

I valori specificati vengono visualizzati nelle colonne appropriate della lista di schemi per gli schemi selezionati.
