Indirizza servizi personalizzati a server

Utilizzare questa schermata per indirizzare i servizi personalizzati ai server appropriati. Non è possibile indirizzare i servizi personalizzati ai cluster. I server assegnati ai cluster sono inclusi nella casella di riepilogo Destinazione.

I servizi personalizzati associati al prodotto per il quale si sta configurando la cella (ad esempio, SOA) vengono indirizzati automaticamente al server creato per tale prodotto. In questa schermata, è possibile indirizzare i servizi a server aggiuntivi.

Per indirizzare i servizi ai server, attenersi alla procedura descritta di seguito.

  1. Nella casella di riepilogo Destinazione selezionare il server in cui si desidera distribuire i servizi.

    Il nome della destinazione selezionata viene visualizzato come titolo della casella di riepilogo sulla destra.

  2. Nella casella di riepilogo nome_destinazione selezionare le caselle di controllo corrispondenti ai servizi da distribuire nella destinazione selezionata.

    I servizi visualizzati variano in base ai prodotti selezionati nella precedente schermata Aggiungi prodotti alla cella della procedura guidata.

    Dopo aver selezionato i servizi personalizzati, i nomi dei servizi indirizzati vengono visualizzati nella colonna Destinazione della casella di riepilogo nome_destinazione.

  3. Ripetere i passi 1 e 2 per ogni server, se necessario.

  4. Dopo aver effettuato le selezioni necessarie, fare clic su Successivo.

Durante la riconfigurazione di una cella esistente, se si elimina un server a cui sono attualmente indirizzati i servizi personalizzati, la Configurazione guidata automaticamente reindirizza tali servizi nel modo descritto di seguito.

  • Se i servizi personalizzati sono stati originariamente indirizzati solo al server che si sta eliminando, ovvero dopo aver eliminato il server i servizi personalizzati risultano non indirizzati nella cella modificata, la Configurazione guidata automaticamente reindirizza i servizi personalizzati a tutte le destinazioni idonee.

    Una destinazione idonea è qualsiasi server non definito nella specifica dei gruppi di configurazione (file config-groups.xml) di un modello di prodotto incluso. I server specificati nel file config-groups.xml sono essenzialmente di proprietà del modello e pertanto non sono idonei all'indirizzamento automatico.

  • Se i servizi personalizzati sono originariamente indirizzati a più destinazioni, compresi i server e il server di amministrazione, e uno dei server di destinazione viene eliminato, la Configurazione guidata lascia invariate le associazioni destinazione rimanenti e non cerca di reindirizzare i servizi.
