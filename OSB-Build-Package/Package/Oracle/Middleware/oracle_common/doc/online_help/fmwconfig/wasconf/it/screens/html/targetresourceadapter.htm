Indirizza Resource Adapter a server

Utilizzare questa schermata per indirizzare i resource adapter a singoli server. Non è possibile indirizzare i resource adapter ai cluster. I server assegnati ai cluster sono inclusi nella lista Destinazione.

I resource adapter associati al prodotto per il quale si sta configurando la cella (ad esempio, SOA) sono indirizzati automaticamente al server creato per tale prodotto. In questa schermata è possibile indirizzare i resource adapter a server aggiuntivi.

Per indirizzare i resource adapter ai server:

  1. Nella lista Destinazione, selezionare il server sul quale indirizzare i resource adapter.

    Il nome della destinazione selezionata viene visualizzato come titolo della lista sulla destra.

  2. Nella lista target_name, selezionare le caselle di controllo corrispondenti ai resource adapter da indirizzare al server selezionato.

    I resource adapter visualizzati possono variare in base ai prodotti selezionati nella schermata Aggiungi prodotti alla cella della procedura guidata.

    Dopo avere selezionato i resource adapter, i nomi dei server indirizzati sono visualizzati nella colonna Destinazione nella lista target_name.

  3. Ripetere i passi 1 e 2 per gli altri server come richiesto.

  4. Dopo aver effettuato le selezioni necessarie, fare clic su Successivo.

Quando si sta riconfigurando una cella esistente, se si elimina un server al quale sono indirizzati i resource adapter, la procedura guidata di configurazione reindirizza automaticamente tali resource adapter come segue:

  • Se i resource adapter sono stati indirizzati inizialmente solo al server che si sta eliminando (ovvero, dopo l'eliminazione del server, i resource adapter non saranno più indirizzati nella cella modificata), la Configurazione guidata reindirizza in modo automatico i resource adapter a tutte le destinazioni idonee.

    Una destinazione idonea è un server non definito nella specifica dei gruppi di configurazione (file config-groups.xml) di un modello di prodotto incluso. I server specificati nel file config-groups.xml sono essenzialmente di proprietà del modello e, pertanto, non sono idonei per l'indirizzamento automatico.

  • Se i resource adapter sono stati indirizzati inizialmente a destinazioni multiple (inclusi i server e l'Administration Server) e uno dei server indirizzati viene eliminato, la Configurazione guidata lascia intatte le associazioni di destinazione restanti e non tenta di reindirizzare i resource adapter.
