Indirizza servizi a cluster o server

Utilizzare questa schermata per indirizzare i servizi ai cluster o ai servizi appropriati.

I servizi associati al prodotto per cui si sta configurando la cella, ad esempio SOA, vengono indirizzati automaticamente al server creato per tale prodotto o al cluster a cui è assegnato il server specificato. In questa schermata è possibile indirizzare i servizi a server e cluster aggiuntivi.

Per indirizzare i servizi ai server o ai cluster, attenersi alla procedura descritta di seguito.

  1. Nella casella di riepilogo Destinazione selezionare il cluster o il server su cui si desidera distribuire i servizi.

    Il nome della destinazione selezionata viene visualizzato come titolo della casella di riepilogo a destra.

  2. Nella casella di riepilogo nome_destinazione selezionare le caselle di controllo corrispondenti ai servizi da distribuire sulla destinazione selezionata.

    I servizi visualizzati variano in base ai prodotti selezionati nella schermata Aggiungi prodotti alla cella precedentemente visualizzata nella procedura guidata.


    Nota:

    Quando si seleziona un server nella casella di riepilogo Destinazione è possibile che alcune caselle di controllo nell'elenco nome_destinazione siano disabilitate, a indicare i servizi già indirizzati al cluster contenente il server selezionato.

    Dopo aver selezionato i servizi, i nomi dei cluster e dei server indirizzati vengono visualizzati nella colonna Destinazione della casella di riepilogo nome_destinazione.

  3. Ripetere i passi 1 e 2 per gli altri cluster e server come richiesto.

  4. Dopo aver effettuato le selezioni necessarie, fare clic su Successivo.

Durante la riconfigurazione di una cella esistente, se si elimina un server o un cluster su cui sono attualmente indirizzati i servizi, la Configurazione guidata reindirizza automaticamente tali servizi nel modo descritto di seguito.

  • Se i servizi sono stati originariamente indirizzati solo al server o al cluster che si sta eliminando, ovvero dopo aver eliminato il server o il cluster, i servizi risultano non indirizzati nella cella modificata, la Configurazione guidata automaticamente reindirizza i servizi a tutte le destinazioni idonee.

    Una destinazione idonea è qualsiasi cluster o server non definito nella specifica dei gruppi di configurazione (file config-groups.xml) di un modello di prodotto incluso. I server o i cluster specificati nel file config-groups.xml sono essenzialmente di proprietà del modello e pertanto non sono idonei all'indirizzamento automatico.

  • Se i servizi sono originariamente indirizzati a più destinazioni, compresi i server, i cluster e il server di amministrazione, e uno dei server o cluster di destinazione viene eliminato, la Configurazione guidata lascia invariate le associazioni destinazione rimanenti e non cerca di reindirizzare i servizi.
