アプリケーション・サーバーの構成

この画面で、セルの任意のノードへのアプリケーション・サーバーの追加、クローン作成または削除を行うことができます。既存のアプリケーション・サーバーの設定も変更できます。必須フィールドはアスタリスク（*）で示されています。


注意:

この画面では、（a）クラスタのプライマリ・メンバーとなる、または（b）クラスタの一部にはならないアプリケーション・サーバーのみ作成します。この画面でセカンダリ・クラスタ・メンバーを作成しないでください。かわりに、「追加のクラスタ・メンバーの構成」画面を使用してセカンダリ・クラスタ・メンバーをクラスタに追加します。

たとえば、サーバーsoa_server_1、soa_server_2およびsoa_server_3がある3サーバー・クラスタを作成するとします。そのためには、「アプリケーション・サーバーの構成」画面でsoa_server_1アプリケーション・サーバーを追加します。「クラスタの構成」画面でクラスタを作成し、soa_server_1をプライマリ・クラスタ・メンバーとして選択します。次に、「追加のクラスタ・メンバーの構成」画面を使用して、soa_server_2とsoa_server_3をクラスタに追加します。

  • 名前: アプリケーション・サーバーの名前。新しいサーバーのデフォルト名はnew_ManagedServer_nです。nは1から始まり、新しいサーバーを追加するたびに増加します。サーバー名を変更するには、「名前」フィールドをクリックし、新しい名前を入力します。

  • ノード名: 使用可能なすべてのノードのドロップダウン・リスト。リストをクリックして、このアプリケーション・サーバーが置かれているノードを選択します。現在のセルに他のマシンをまだフェデレートしていない場合、このリストでは1つのノードのみ使用可能です。

  • 開発モード: アプリケーション・サーバーを開発モードで実行するには、このチェック・ボックスを選択します。アプリケーション・サーバーを本番モードで実行する場合は、このチェック・ボックスの選択を解除したままにします。WebSphere環境での本番モードと開発モードの違いについては、WebSphereのドキュメントを参照してください。

サーバーを追加するには、「追加」をクリックし、新しいサーバーの設定を構成します。

既存のサーバーのクローンを作成するには、クローンを作成するサーバーを選択し、「クローン」をクリックします。新しいサーバー行が表に追加されます。新しいサーバーは、既存のサーバーからすべての属性を継承します。新しいサーバーのデフォルト名はclone_source_cloned1です。clone_sourceは、クローンを作成したサーバーの名前です。「名前」フィールドを編集し、サーバーが置かれているノードを選択し、「開発モード」オプションを選択または選択を解除します。

サーバーを削除するには、サーバーを選択し、「削除」をクリックします。一度に1つのサーバーのみ削除できます。

デフォルトの表形式からタブ化形式（サーバーごとに1つのタブを使用）に表示を切り替えるには、「表示の切替え」をクリックします。表形式に戻るには、「表示の切替え」をもう一度クリックします。

サーバーの追加が終了したら、「次へ」をクリックして続行します。


注意:

この画面で行った変更は、構成ウィザードの最後の画面で「作成」をクリックするまでセル構成に保存されません。ウィザードを終了するか、「前へ」をクリックして前の画面に戻った場合は、変更をもう一度行う必要があります。
