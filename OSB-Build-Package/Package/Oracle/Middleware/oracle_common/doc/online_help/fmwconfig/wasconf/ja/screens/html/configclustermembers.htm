追加のクラスタ・メンバーの構成

この画面では、クラスタにアプリケーション・サーバーを追加します。クラスタに新しいアプリケーション・サーバーを追加すると、クラスタのプライマリ・メンバーからその属性のクローンが作成されます。

  • 名前: アプリケーション・サーバーの名前。新しいサーバーのデフォルト名はnew_ClusterMember_nです。nは1から始まり、新しいサーバーを追加するたびに増加します。サーバー名を変更するには、「名前」フィールドをクリックし、新しい名前を入力します。

  • ノード名: 使用可能なすべてのノードのドロップダウン・リスト。リストをクリックして、このアプリケーション・サーバーが置かれているノードを選択します。現在のセルに他のマシンをまだフェデレートしていない場合、このリストでは1つのノードのみ使用可能です。

  • クラスタ名: 使用可能なすべてのクラスタのドロップダウン・リスト。このサーバーを割り当てるクラスタを選択します。

  • 開発モード: アプリケーション・サーバーを開発モードで実行するには、このチェック・ボックスを選択します。アプリケーション・サーバーを本番モードで実行する場合は、このチェック・ボックスの選択を解除したままにします。WebSphere環境での本番モードと開発モードの違いについては、WebSphereのドキュメントを参照してください。

追加のクラスタ・メンバーを追加するには、次の手順を実行します。

  1. 「追加」をクリックします。新しい行が表に追加されます。

  2. サーバーの名前を入力し、適切なノード、クラスタおよび開発モード設定を選択します。

  3. 追加するクラスタ・メンバーごとに、手順1と2を繰り返します。

クラスタ・メンバーを削除するには、クラスタ・メンバーを選択し、「削除」をクリックします。一度に1つのクラスタ・メンバーのみ削除できます。

デフォルトの表形式からタブ化形式（各クラスタごとに1つのタブを使用）に表示を切り替えるには、「表示の切替え」をクリックします。表形式に戻るには、「表示の切替え」をもう一度クリックします。

終了したら、「次へ」をクリックして続行します。


注意:

この画面で行った変更は、構成ウィザードの最後の画面で「作成」をクリックするまでセル構成に保存されません。ウィザードを終了するか、「前へ」をクリックして前の画面に戻った場合は、変更をもう一度行う必要があります。
