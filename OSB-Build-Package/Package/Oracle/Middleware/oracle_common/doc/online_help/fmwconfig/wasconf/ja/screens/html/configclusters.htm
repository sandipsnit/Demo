クラスタの構成

クラスタはアプリケーション・サーバーのグループであり、互いに連携してアプリケーションのスケーラビリティと高可用性を実現します。クラスタを作成することで、アプリケーションとリソースをホストする1つの単位として動作するようにサーバーをグループ化できます。

この画面では、クラスタを追加し、クラスタのプライマリ・クラスタ・メンバーとなるアプリケーション・サーバーを選択します。すべての追加クラスタ・メンバーは、このプライマリ・クラスタ・メンバーのクローンとして作成されます。クラスタを作成した後で、次の画面でアプリケーション・サーバーをクラスタに追加できます。

この画面で、クラスタの削除または既存のクラスタの設定の変更を行うこともできます。

クラスタを作成するには、次の手順を実行します。

  1. 「追加」をクリックします。新しい行が表に表示されます。新しいクラスタのデフォルト名はnew_Cluster_nです。nは1から始まり、新しいクラスタを追加するたびに増加します。デフォルトを変更するには、「名前」列に目的の名前を入力します。クラスタの名前は、セル内のすべてのコンポーネント名の中で一意である必要があります。

  2. 最初のクラスタ・メンバードロップダウン・リストからサーバーを選択することで、既存のアプリケーション・サーバーをクラスタのプライマリ・メンバーとして選択します。まだクラスタに割り当てられていないアプリケーション・サーバーのみがこのリストに表示されます。

  3. 作成する追加のクラスタごとに、手順1と2を繰り返します。

  4. 終了したら、「次へ」をクリックします。「追加のクラスタ・メンバーの構成」画面で追加のサーバーをクラスタに追加できます。この画面は「次へ」をクリックしたときに表示されます。


    注意:

    この画面で行った変更は、構成ウィザードの最後の画面で「作成」をクリックするまでセル構成に保存されません。ウィザードを終了するか、「前へ」をクリックして前の画面に戻った場合は、変更をもう一度行う必要があります。

クラスタを削除するには、クラスタを選択し、「削除」をクリックします。クラスタの削除では、プライマリ・クラスタ・メンバーであるアプリケーション・サーバーは削除されません。クラスタから解除されるだけです。一度に1つのクラスタのみ削除できます。

クラスタの設定を変更するには、クラスタを選択し、適切な変更を加えます。

デフォルトの表形式からタブ化形式（各クラスタごとに1つのタブを使用）に表示を切り替えるには、「表示の切替え」をクリックします。表形式に戻るには、「表示の切替え」をもう一度クリックします。

クラスタの構成が終了したら、「次へ」をクリックして続行します。

WebSphere環境のクラスタの詳細は、WebSphereのドキュメントを参照してください。
