エンド・ポイントの構成

この画面では、管理サーバーなどの各サーバーで次のエンド・ポイント（リスニング・アドレス）を構成します。すべてのエンド・ポイントはセル内で一意である必要があります。デフォルトでは、セルにサーバーを追加したときに、一意のエンド・ポイントがサーバーに割り当てられます。特定のサーバーで構成されたデフォルト以外のポートを使用しないかぎり、エンド・ポイントを変更する必要はありません。

この画面にリストされる各種エンド・ポイントの詳細は、WebSphereのドキュメントを参照してください。

エンド・ポイントを変更するには、次の手順を実行します。

  1. サーバー名/ノード名ドロップダウン・リストから、エンド・ポイントを変更するサーバーを選択します。リストには、管理サーバー、クラスタに割り当てられているアプリケーション・サーバーおよびセルのすべてのノードに存在するスタンドアロン・アプリケーション・サーバーを含め、セル内のすべてのサーバーが含まれます。

  2. 適切な行に新しいエンド・ポイント値を入力します。すでに使用されている値を入力した場合は、次の検証エラーが表示されます。「OK」をクリックし、エンド・ポイントに対して別の値を入力します。



    注意:

    同じマシンで複数のデプロイメント・マネージャの実行を計画している場合は、関連付けられている各セルのエンド・ポイントが、マシン上のすべてのデプロイメント・マネージャ・ノード間で一意であることを確認する必要があります。

    同様に、リモート・マシン上にアプリケーション・サーバーがあり、それらのサーバーがデプロイメント・マネージャ・ホスト上の異なるセルに割り当てられている場合は、特定のマシン上のすべてのアプリケーション・サーバーのすべてのエンド・ポイントが一意であることを確認する必要があります。

    異なるセルに存在するエンド・ポイントの競合については警告されません。

  3. 変更する各エンド・ポイントについて手順2を繰り返します。

不要な場合は、任意のサーバー上のエンド・ポイントを無効にすることもできます。サーバー上の1つ以上のエンド・ポイントを無効にするには、サーバー名/ノード名ドロップダウン・リストからサーバーを選択し、無効にする各エンド・ポイントのチェック・ボックスの選択を解除します。

特定のサーバー上のすべてのエンド・ポイントに共通ホストを使用するには、サーバー名/ノード名ドロップダウン・リストからサーバーを選択し、「すべてのエンド・ポイントに共通ホストを使用」チェック・ボックスを選択します。このチェック・ボックスが選択されている場合は、Admin end pointというホスト名を除き、現在選択されているサーバー上の各エンド・ポイントのホスト名が編集不能になります。

終了したら、「次へ」をクリックします。エンド・ポイントが競合している場合は、「検証エラー」ダイアログ・ボックスが表示され、続行する前に競合を解決する必要があります。
