カスタム・サービスのサーバーへのターゲット設定

この画面では、カスタム・サービスを適切なサーバーにターゲット指定します。カスタム・サービスをクラスタにターゲット指定することはできません。クラスタに割り当てられているサーバーは、「ターゲット」リスト・ボックスに含まれています。

セルを構成する製品（SOAなど）に関連付けられたカスタム・サービスは、その製品に作成されたサーバーに、自動的にターゲットとして指定されます。この画面では、サービスを追加のサーバーに割り当てることができます。

サービスをサーバーにターゲット指定するには、次の手順を実行します。

  1. 「ターゲット」リスト・ボックスで、サービスをデプロイするサーバーを選択します。

    選択したターゲットの名前は右側のリスト・ボックスのタイトルとして表示されます。

  2. target_nameリスト・ボックスで、選択したターゲットにデプロイするサービスに対応するチェック・ボックスを選択します。

    ウィザードの「セルへの製品追加」画面で選択した製品によって表示されるサービスが異なります。

    カスタム・サービスを選択すると、ターゲットに指定したサーバーの名前がtarget_nameリスト・ボックスの「ターゲット」列に表示されます。

  3. 必要に応じて、他のサーバーで手順1と2を繰り返します。

  4. 必要な選択が完了した後、「次へ」をクリックします。

既存のセルを再構成するときに、カスタム・サービスが現在割り当てられているサーバーを削除する場合、構成ウィザードによってそれらのサービスのターゲットが次のように自動的に再設定されます。

  • 元のカスタム・サービスが削除対象のサーバーのみに割り当てられていた（つまり、サーバーを削除した後、カスタム・サービスは変更されたセル内で未割当ての状態になる）場合は、構成ウィザードによって、資格のあるすべてのターゲットに対し、カスタム・サービスのターゲットの再設定が自動的に行われます。

    資格のあるターゲットとは、包含されている製品テンプレートの構成グループの指定（config-groups.xmlファイル）内に定義されていない任意のサーバーです。config-groups.xmlに指定されているサーバーは本質的にテンプレートによって所有されるため、自動ターゲット設定には適していません。

  • 元のカスタム・サービスが複数のターゲット（サーバー、管理サーバーなど）に割り当てられていて、ターゲット・サーバーのいずれかが削除された場合、構成ウィザードは残りのターゲットの関連付けをそのまま残し、サービスのターゲットの再設定は行われません。
