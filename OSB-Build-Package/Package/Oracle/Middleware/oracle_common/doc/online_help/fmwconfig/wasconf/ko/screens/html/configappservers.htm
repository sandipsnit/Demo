애플리케이션 서버 구성

이 화면에서는 셀의 특정 노드에 애플리케이션 서버를 추가, 복제 또는 삭제할 수 있습니다. 기존 애플리케이션 서버에 대한 설정을 변경할 수도 있습니다. 별표(*)가 표시된 필드는 필수 필드입니다.


주:

(a) 클러스터의 기본 멤버가 되거나 (b) 클러스터의 일부가 되지 않을 애플리케이션 서버를 생성할 때만 이 화면을 사용하십시오. 이 화면에서는 보조 클러스터 멤버를 생성하지 마십시오. 클러스터에 보조 클러스터 멤버를 추가하려면 추가 클러스터 멤버 구성 화면을 사용하십시오.

예를 들어, soa_server_1, soa_server_2, soa_server_3의 세 서버로 구성된 클러스터를 생성한다고 가정해 보겠습니다. 먼저 애플리케이션 서버 구성 화면에서 soa_server_1 애플리케이션 서버를 추가합니다. 클러스터 구성 화면에서 클러스터를 생성하고 soa_server_1을 기본 클러스터 멤버로 선택합니다. 그런 다음 추가 클러스터 멤버 구성 화면에서 soa_server_2 및 soa_server_3을 클러스터에 추가합니다.

  • 이름: 애플리케이션 서버의 이름입니다. 새 서버의 기본 이름은 new_ManagedServer_n입니다. 여기서 n은 1부터 시작하고 새 서버가 추가될 때마다 1씩 증가합니다. 서버 이름을 변경하려면 이름 필드를 누르고 새 이름을 입력합니다.

  • 노드 이름: 사용 가능한 모든 노드의 드롭다운 목록입니다. 목록을 눌러 이 애플리케이션 서버가 상주할 노드를 선택합니다. 아직 다른 시스템이 현재 셀에 통합되지 않았다면 이 목록에서 하나의 노드만 사용할 수 있습니다.

  • 개발 모드: 애플리케이션 서버를 개발 모드에서 실행하려면 이 체크 박스를 선택합니다. 애플리케이션 서버를 운용 모드에서 실행하려면 이 체크 박스를 선택하지 마십시오. WebSphere 환경에서 개발 모드와 운용 모드 간의 차이점에 대한 자세한 내용은 WebSphere 설명서를 참조하십시오.

서버를 추가하려면 추가를 누르고 새 서버에 대한 설정을 구성합니다.

기존 서버를 복제하려면 복제할 서버를 선택하고 복제를 누릅니다. 새 서버 행이 테이블에 추가됩니다. 새 서버는 기존 서버의 모든 속성을 상속합니다. 새 서버의 기본 이름은 clone_source_cloned1입니다. 여기서 clone_source는 복제한 서버의 이름입니다. 이름 필드를 편집하고 서버가 상주할 노드를 선택한 후 개발 모드 옵션을 선택하거나 지웁니다.

서버를 삭제하려면 서버를 선택하고 삭제를 누릅니다. 한 번에 하나의 서버만 삭제할 수 있습니다.

표시를 기본 표 형식에서 탭 형식(각 서버에 대해 하나의 탭)으로 전환하려면 표시 전환을 누릅니다. 표시 전환을 다시 누르면 표 형식으로 돌아갑니다.

서버를 추가했으면 다음을 눌러 계속 진행하십시오.


주:

이 화면에서 수행하는 변경 작업은 마지막 구성 마법사 화면에서 생성을 누를 때까지 셀 구성에 저장되지 않습니다. 마법사를 종료하거나 이전을 눌러 이전 화면으로 돌아갈 경우 변경한 작업을 다시 수행해야 합니다.
