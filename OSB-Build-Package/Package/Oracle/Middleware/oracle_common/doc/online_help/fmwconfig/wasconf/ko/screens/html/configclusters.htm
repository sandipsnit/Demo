클러스터 구성

클러스터는 애플리케이션 서버의 그룹으로, 여러 서버가 함께 작동하여 애플리케이션에 확장성 및 고가용성을 제공합니다. 클러스터를 생성하여 애플리케이션과 리소스를 호스트하는 단일 단위로 작동할 수 있도록 서버를 그룹화할 수 있습니다.

이 화면에서는 클러스터를 추가하고 클러스터의 기본 클러스터 멤버가 될 애플리케이션 서버를 선택할 수 있습니다. 모든 추가 클러스터 멤버는 이 기본 클러스터 멤버로부터 복제됩니다. 클러스터를 생성한 후 다음 화면에서 추가 애플리케이션 서버를 클러스터에 추가할 수 있습니다.

그리고 이 화면에서 클러스터를 삭제하거나 기존 클러스터에 대한 설정을 수정할 수도 있습니다.

클러스터를 생성하려면 다음과 같이 하십시오.

  1. 추가를 누릅니다. 새 행이 테이블에 나타납니다. 새 클러스터에 대한 기본 이름은 new_Cluster_n입니다. 여기서 n은 1부터 시작하고 새 클러스터가 추가될 때마다 1씩 증가합니다. 기본값을 변경하려면 이름 열에 원하는 이름을 입력합니다. 클러스터 이름은 같은 셀 내에 존재하는 다른 구성 요소 이름과 중복되면 안됩니다.

  2. 첫번째 클러스터 멤버 드롭다운 목록에서 서버를 선택하여 기존 애플리케이션 서버를 클러스터의 기본 멤버로 선택합니다. 아직 클러스터에 지정되지 않은 애플리케이션 서버만 이 목록에 표시됩니다.

  3. 생성하려는 각 추가 클러스터에 대해 1단계와 2단계를 반복합니다.

  4. 작업을 마쳤으면 다음을 누릅니다. 다음을 누를 때 표시되는 추가 클러스터 멤버 구성 화면에서 추가 서버를 클러스터에 추가할 수 있습니다.


    주:

    이 화면에서 수행하는 변경 작업은 마지막 구성 마법사 화면에서 생성을 누를 때까지 셀 구성에 저장되지 않습니다. 마법사를 종료하거나 이전을 눌러 이전 화면으로 돌아갈 경우 변경한 작업을 다시 수행해야 합니다.

클러스터를 삭제하려면 클러스터를 선택하고 삭제를 누릅니다. 클러스터를 삭제할 때 기본 클러스터 멤버인 애플리케이션 서버는 삭제되지 않고 단지 클러스터에서만 제거됩니다. 한 번에 하나의 클러스터만 삭제할 수 있습니다.

클러스터 설정을 변경하려면 클러스터를 선택하고 변경 작업을 수행합니다.

표시를 기본 표 형식에서 탭 형식(각 클러스터에 대해 하나의 탭)으로 전환하려면 표시 전환을 누릅니다. 표시 전환을 다시 누르면 표 형식으로 돌아갑니다.

클러스터를 구성했으면 다음을 눌러 계속하십시오.

WebSphere 환경에서 클러스터에 대한 자세한 내용은 WebSphere 설명서를 참조하십시오.
