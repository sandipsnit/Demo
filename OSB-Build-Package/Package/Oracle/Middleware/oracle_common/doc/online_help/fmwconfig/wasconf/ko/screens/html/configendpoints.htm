끝점 구성

이 화면에서는 애플리케이션 서버를 비롯하여 각 서버에서 다음 끝점(수신 주소)을 구성할 수 있습니다. 모든 끝점은 셀에서 고유해야 합니다. 서버를 셀에 추가할 때 기본적으로 고유한 끝점이 서버에 지정됩니다. 제공된 서버에서 구성된 기본값 포트를 그대로 사용하려면 수정할 필요가 없습니다.

이 화면에 나열되는 다양한 끝점에 대한 자세한 내용은 WebSphere 설명서를 참조하십시오.

끝점을 변경하려면 다음과 같이 하십시오.

  1. 서버 이름/노드 이름 드롭다운 목록에서 끝점을 변경할 서버를 선택합니다. 목록에는 관리 서버, 클러스터에 지정된 애플리케이션 서버 및 셀의 모든 노드에 존재하는 독립형 애플리케이션 서버를 비롯하여 셀의 모든 서버가 포함됩니다.

  2. 해당하는 행에 새 끝점 값을 입력합니다. 이미 사용 중인 값을 입력할 경우 다음과 같은 검증 오류가 표시됩니다. 확인을 누르고 끝점에 대한 다른 값을 입력합니다.



    주:

    동일 시스템에서 여러 배치 관리자를 실행하려는 경우 각 연관된 셀의 끝점은 시스템의 모든 배치 관리자 노드에서 고유해야 합니다.

    마찬가지로, 원격 시스템에 애플리케이션 서버가 있고 이러한 서버가 배치 관리자 호스트의 서로 다른 셀에 지정된 경우 제공된 시스템의 모든 애플리케이션 서버에 대한 모든 끝점이 고유해야 합니다.

    서로 다른 셀에 존재하는 끝점이 충돌할 경우에는 경고가 표시되지 않습니다.

  3. 변경하려는 각 끝점에 대해 2단계를 반복합니다.

서버에서 끝점이 필요하지 않은 경우 사용 안함으로 설정할 수도 있습니다. 서버에서 하나 이상의 끝점을 사용 안함으로 설정하려면 서버 이름/노드 이름 드롭다운 목록에서 서버를 선택하고 사용 안함으로 설정할 각 끝점에 대한 체크 박스를 지웁니다.

제공된 서버에서 모든 끝점에 대해 공통 호스트를 사용하려면 서버 이름/노드 이름 드롭다운 목록에서 서버를 선택한 후 모든 끝점에 공통 호스트 사용 체크 박스를 선택합니다. 이 체크 박스가 선택되면 관리 끝점 호스트 이름을 제외하고 현재 선택된 서버의 각 끝점에 대한 호스트 이름을 편집할 수 없게 됩니다.

작업을 마쳤으면 다음을 누릅니다. 끝점 충돌이 있을 경우 검증 오류 대화상자가 나타나며 계속 진행하기 전에 충돌을 해결해야 합니다.
