RAC 데이터 소스 구성 요소 스키마 구성

이 화면에서는 WebSphere 셀에 포함된 구성 요소 스키마를 Oracle RAC 스키마로 구성할 수 있습니다.

마법사의 JDBC 구성 요소 스키마 구성 화면에서 Oracle RAC 스키마로 구성하도록 선택한 구성 요소 스키마는 화면 아래쪽에 표시됩니다.

화면 아래쪽의 스키마 목록에서 각 스키마 이름 옆에 있는 체크 박스를 선택하여 Oracle RAC 스키마로 구성할 스키마를 선택합니다.


주:

다중 스키마를 선택하면 "구성 요소 스키마에 따라 다릅니다"라는 텍스트가 특정 필드에 표시되며, 이는 이러한 필드의 현재 값이 선택된 스키마 간에 다르다는 뜻입니다. 계속 진행하여 이러한 필드의 값을 변경하면 새 값이 선택된 스키마 간에 동일하게 적용됩니다.

  • 드라이버: 데이터베이스에 접속하는 데 사용할 JDBC 드라이버를 선택합니다.

  • 서비스 이름: 데이터베이스 Oracle RAC 서비스 이름을 입력합니다.

  • 사용자 이름: 데이터베이스에 접속하기 위한 사용자 이름을 입력합니다.

  • 비밀번호: 지정된 사용자 이름의 비밀번호를 입력합니다.

  • 호스트 이름: Oracle RAC 데이터베이스 인스턴스를 호스트하는 서버 이름을 입력합니다.

  • 인스턴스 이름: 각 Oracle 데이터베이스 인스턴스의 이름을 입력합니다.

  • 포트: 데이터베이스를 호스트하는 서버에 접속하는 데 사용되는 포트 번호를 입력합니다.


주:

최소한 데이터베이스 인스턴스 하나의 호스트 이름, 인스턴스 이름 및 포트 번호를 입력해야 합니다.

데이터베이스 인스턴스를 추가하려면 추가를 누른 다음 호스트 이름, 인스턴스 이름 및 포트를 지정합니다.

선택된 스키마에 대해 지정한 값은 스키마 목록의 해당 열에 표시됩니다.
