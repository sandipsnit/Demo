셀, 프로파일 및 노드 정보 지정

다음 필드에 알맞은 값을 입력합니다. 각 필드는 필수 필드입니다. 각 필드에는 영숫자만 사용하십시오.

  • 셀 이름: 새 셀에 대한 고유한 이름을 입력합니다. 다음 두 디렉토리 아래에 셀이 생성되는 디렉토리의 이름입니다.

    • WAS_HOME/profiles/DepMgrProfile/config/cells, 여기서 WAS_HOME은 WebSphere 설치 디렉토리이고 DepMgrProfile은 배치 관리자 프로파일의 이름입니다.

    • WAS_HOME/profiles/AppSrvrProfile/config/cells, 여기서 WAS_HOME은 WebSphere 설치 디렉토리이고 AppSrvrProfile은 애플리케이션 서버 프로파일의 이름입니다.

    기본 이름은 machine_nameCelln입니다. 여기서 machine_name은 현재 시스템의 이름이고, n은 해당 시스템에서 다음 두 자리 숫자입니다. 예를 들어, 시스템 이름이 machine1이고 Cell01이 존재하는 경우 기본 셀 이름은 machine1Cell02입니다.

  • 배치 관리자 프로파일 이름: 이 셀에 대한 고유한 배치 관리자 프로파일 이름을 입력합니다. WAS_HOME/profiles 아래에 배치 관리자 프로파일이 생성되는 디렉토리의 이름입니다.

    기본 이름은 Dmgrn입니다. 여기서 n은 다음 두 자리 숫자입니다. 예를 들어, 배치 관리자 프로파일 Dmgr01이 존재하는 경우 이 필드에 대한 기본값은 Dmgr02입니다.

  • 배치 관리자 노드 이름: 이 셀에 대한 고유한 배치 관리자 노드 이름을 입력합니다.

    기본 이름은 machine_nameCellManagern입니다. 여기서 machine_name은 현재 시스템의 이름이고, n은 다음 두 자리 숫자입니다. 예를 들어, 시스템 이름이 machine1이고 CellManager01이 존재하는 경우 기본 노드 이름은 machine1CellManager02입니다.

  • 애플리케이션 서버 프로파일 이름: 이 셀에 대한 고유한 애플리케이션 서버 프로파일 이름을 입력합니다. WAS_HOME/profiles 아래에 애플리케이션 서버 프로파일이 생성되는 디렉토리의 이름입니다.

    기본 이름은 Customn입니다. 여기서 n은 다음 두 자리 숫자입니다. 예를 들어, 애플리케이션 서버 프로파일 Custom01이 존재하는 경우 이 필드에 대한 기본값은 Custom02입니다.

  • 애플리케이션 서버 노드 이름: 이 셀에 대한 고유한 애플리케이션 서버 노드 이름을 입력합니다.

    기본 이름은 machine_nameNoden입니다. 여기서 machine_name은 현재 시스템의 이름이고, n은 해당 시스템에서 다음 두 자리 숫자입니다. 예를 들어, 시스템 이름이 machine1이고 Node01이 존재하는 경우 기본 노드 이름은 machine1Node02입니다.

각 필드에 입력을 마쳤으면 다음을 누릅니다.
