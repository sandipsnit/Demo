Configurar Servidores de Aplicações

Nesta tela, é possível adicionar, clonar ou deletar Servidores de Aplicações em qualquer nó na célula. Você também pode alterar as definições de um Servidor de Aplicações existente. Os campos obrigatórios são indicados por um asterisco (*).


Observação:

Use esta tela para criar Servidores de Aplicações que (a) serão o membro principal de um cluster ou (b) não farão parte de um cluster. Não crie membros do cluster secundários nesta tela. Em vez disso, use a tela Configurar Membros do Cluster Adicionais para adicionar membros do cluster secundários a um cluster.

Por exemplo, suponha que você queira criar um cluster de três servidores com os servidores soa_server_1, soa_server_2 e soa_server_3. Para isso, adicione o servidor soa_server_1 Application Server na tela Configurar Servidores de Aplicações. Crie o cluster nas telas Configurar Clusters selecione soa_server_1 como o membro do cluster principal. Então, você pode usa a tela Configurar Membros do Cluster Adicionais para adicionar soa_server_2 e soa_server_3 ao cluster.

  • Nome — O nome do Servidor de Aplicações. O nome default para um novo servidor é new_ManagedServer_n, em que n começa em 1 e aumenta conforme cada novo servidor é adicionado. Para alterar o nome do servidor, clique no campo Nome e digite o novo nome.

  • Nome do nó — Uma lista drop-down de todos os nós disponíveis. Clique na lista para selecionar o nó em que este Servidor de Aplicações reside. Caso você ainda não tenha federado outras máquinas na célula atual, apenas um nó estará disponível nesta lista.

  • Modo de desenvolvimento — Marque esta caixa de seleção para executar o Servidor de Aplicações no modo de desenvolvimento. Deixe esta caixa de seleção desmarcada para executar o Servidor de Aplicações no modo de produção. Para obter informações sobre as diferenças entre o modo de produção e o modo de desenvolvimento em um ambiente do WebSphere, consulte a documentação do WebSphere.

Para adicionar um servidor, clique em Adicionar e configure as definições para o novo servidor.

Para clonar um servidor existente, selecione o servidor que você deseja clonar e clique em Clonar. Uma nova linha do servidor será adicionada à tabela. O novo servidor herdará todos os atributos do servidor existente. O nome default do novo servidor é clone_source_cloned1, em que clone_source é o nome do servidor clonado. Edite o campo Nome, selecione o nó em que o servidor reside e selecione ou desmarque a opção Modo de desenvolvimento.

Para deletar um servidor, selecione o servidor e clique em Deletar. Você pode deletar apenas um servidor por vez.

Para trocar a exibição do formato tabular default para um formato com guias (com uma guia para cada servidor), clique em Trocar Exibição. Clique em Trocar Exibição novamente para retornar ao formato tabular.

Quando terminar de adicionar servidores, clique em Próximo para continuar.


Observação:

As alterações feitas nesta tela não são salvas na configuração da célula até você clicar em Criar na última tela do Assistente de Configuração. Se sair do assistente ou clicar em Anterior para retornar a uma tela anterior, você deverá fazer as alterações novamente.
