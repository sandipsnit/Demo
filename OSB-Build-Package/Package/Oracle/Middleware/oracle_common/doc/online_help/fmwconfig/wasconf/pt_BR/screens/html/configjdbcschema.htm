Configurar Esquema do Componente JDBC

Para alguns componentes do Fusion Middleware (por exemplo, SOA e Web Center Spaces), as origens de dados JDBC podem ser definidas como parte do esquema do banco de dados do componente, que são carregadas (durante a instalação) usando o Repository Creation Utility (RCU).

Quando você cria uma célula para tais componentes usando o Assistente de Configuração, é possível configurar as definições do esquema do componente JDBC, como driver do banco de dados, proprietário do esquema, senha e assim por diante.

Os esquemas do componente JDBC associados aos produtos para os quais você está criando a célula (por exemplo, SOA) estão listados na metade inferior da tela.


Observação:

Para configurar um ou mais dos esquemas como várias origens de dados do Oracle RAC, marque as caixas de seleção ao lado dos nomes de esquemas e marque a caixa de seleção Configurar esquemas de componentes selecionados como esquemas de várias origens de dados do RAC no painel a seguir. Em seguida, clique no botão Próximo.

Selecione os esquemas cujas definições de origem de dados você deseja especificar, marcando a caixa de seleção adjacente a cada nome de esquema.


Observação:

Quando você seleciona vários esquemas de componente, o texto Varia entre os esquemas de componentes pode ser exibido em alguns campos, indicando que os valores atuais desses campos são diferentes entre os esquemas de componente selecionados. Se você alterar os valores nesses campos, os novos valores serão aplicados uniformemente entre os esquemas de componente selecionados.

Os valores default dos parâmetros do esquema do componente, como fornecedor, driver, nome do host e número da porta, dependem dos valores especificados nos modelos da aplicação.

  • Fornecedor — Selecione o fornecedor do banco de dados.

  • Driver — Selecione o driver JDBC a ser usado para conexão com o banco de dados. A lista inclui drivers JDBC comuns para o fornecedor de banco de dados selecionado.

  • Proprietário do Esquema — Informe o nome do usuário para conexão com o banco de dados.

  • Senha do Esquema — Informe a senha relativa ao proprietário do esquema especificado.

  • DBMS/Serviço — Informe um nome do DBMS do banco de dados ou um nome de serviço, caso selecione um driver do tipo de serviço.

  • Nome do Host — Informe o nome do servidor que está hospedando o banco de dados.

  • Porta — Informe o número da porta a ser usado para conexão com o servidor que hospeda o banco de dados.

Os valores especificados são exibidos nas colunas apropriadas na lista de esquemas, para os esquemas selecionados.

Quando terminar de configurar os esquemas de componentes, clique em Próximo para continuar.
