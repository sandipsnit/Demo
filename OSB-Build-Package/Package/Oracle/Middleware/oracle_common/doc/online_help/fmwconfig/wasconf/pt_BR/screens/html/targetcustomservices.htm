Direcionar Serviços Personalizados para Servidores

Use esta tela para direcionar serviços personalizados para os servidores apropriados. Você não pode direcionar serviços personalizados para clusters. Os servidores designados para clusters estão incluídos na caixa de listagem Destino.

Os serviços personalizados que estão associados ao produto para o qual você está configurando a célula (por exemplo, SOA) são direcionados automaticamente para o servidor criado para aquele produto. Nesta tela, é possível direcionar serviços para servidores adicionais.

Para direcionar serviços para servidores:

  1. No painel Destino, selecione o servidor em que você deseja implantar serviços.

    O nome do destino selecionado é exibido como o título da caixa de listagem à direita.

  2. Nas caixas de listagem target_name, marque as caixas de seleção correspondentes aos serviços a serem implantados no destino selecionado.

    Os serviços exibidos aqui variam, dependendo dos produtos selecionados na tela Adicionar Produtos à Célula anteriormente no assistente.

    Depois que você seleciona serviços personalizados, os nomes dos servidores direcionados são exibidos na coluna Destino na caixa de listagem target_name.

  3. Repita as etapas 1 e 2 para os outros servidores, conforme necessário.

  4. Após efetuar as seleções necessárias, clique em Próximo.

Ao reconfigurar uma célula existente, se você deletar um servidor para o qual os serviços personalizados estão direcionados no momento, o Assistente de Configuração redirecionará automaticamente estes serviços conforme a seguir:

  • Se os serviços personalizados foram direcionados originalmente somente ao servidor que você está deletando agora (isto é, após você deletar o servidor, os serviços personalizados se tornariam não direcionados na célula modificada), então o Assistente de Configuração redireciona automaticamente os serviços personalizados a todos os alvos apropriados.

    Um destino apropriado é qualquer servidor que não é definido na especificação dos grupos de configuração (arquivo config-groups.xml) de um modelo de produto incluído. Os servidores que são especificados em config-groups.xml pertencem basicamente ao modelo e, por isso, não são apropriados para direcionamento automático.

  • Se os serviços personalizados foram direcionados originalmente a vários alvos (incluindo servidores e o Servidor de Administração) e um dos servidores direcionados for deletado, então o Assistente de Configuração deixará as associações de destino restantes intactas e não tentará redirecionar os serviços.
