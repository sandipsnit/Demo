Direcionar Implantações para Clusters ou Servidores

Use esta tela para direcionar aplicações e bibliotecas para implantação em servidores ou clusters.

Aplicações e bibliotecas associadas ao produto para o qual você está configurando a célula (por exemplo, SOA) são direcionadas automaticamente para o servidor criado para aquele produto ou para o cluster ao qual aquele servidor é designado. Nesta tela, é possível direcionar aplicações e bibliotecas para servidores e clusters adicionais.

Para direcionar uma implantação de aplicação para um cluster ou servidor:

  1. Na caixa de listagem Destino, selecione o cluster ou servidor em que você deseja implantar as aplicações ou bibliotecas.

    O nome do destino selecionado é exibido como o título da caixa de listagem à direita.

  2. Nas caixas de listagem target_name, marque as caixas de seleção correspondentes às aplicações ou bibliotecas a serem implantadas no destino selecionado.

    As aplicações e bibliotecas exibidas aqui variam, dependendo dos produtos selecionados na tela Adicionar Produtos à Célula anteriormente no assistente.


    Observação:

    Quando você seleciona um servidor na caixa de listagem Destino, algumas das caixas de seleção na caixa de listagem target_name podem ser desativadas, indicando aplicações e bibliotecas que já estão direcionadas para o cluster que contém o servidor selecionado.

    Após a seleção das aplicações e bibliotecas, os nomes dos clusters e servidores direcionados são exibidos na coluna Destino na caixa de listagem target_name.

  3. Repita as etapas 1 e 2 para os outros clusters e servidores, conforme necessário.

  4. Após efetuar as seleções e entradas necessárias, clique em Próximo.

Ao reconfigurar uma célula existente, se você deletar um servidor ou cluster para o qual as aplicações estão direcionadas no momento, o Assistente de Configuração redirecionará automaticamente estas aplicações conforme a seguir:

  • Se as aplicações foram direcionadas originalmente somente ao servidor ou cluster que você está deletando agora (isto é, após você deletar o servidor ou cluster, as aplicações se tornariam não direcionadas na célula modificada), então o Assistente de Configuração redireciona automaticamente as aplicações a todos os destinos apropriados.

    Um destino apropriado é qualquer cluster ou servidor que não é definido na especificação dos grupos de configuração (arquivo config-groups.xml) de um modelo incluído. Os servidores ou clusters que são especificados em config-groups.xml pertencem basicamente ao modelo e, por isso, não são apropriados para direcionamento automático.

  • Se as aplicações foram direcionadas originalmente a vários destinos (incluindo servidores, clusters e o Servidor de Administração) e um dos servidores ou clusters direcionados for deletado, então o Assistente de Configuração deixará as associações de destino restantes intactas e não tentará redirecionar as aplicações.
