配置应用程序服务器

在此屏幕中，您可以将应用程序服务器添加、克隆到单元中的任何节点，也可以删除单元中任何节点上的应用程序服务器。您还可以更改现有应用程序服务器的设置。星号 (*) 指示必填字段。


注：

使用此屏幕只能创建 (a) 作为集群的主要成员的应用程序服务器或 (b) 不属于集群的一部分的应用程序服务器。不要在此屏幕上创建辅助集群成员。改为使用配置其他集群成员屏幕将辅助集群成员添加到集群。

例如，假设您要使用服务器 soa_server_1、soa_server_2 和 soa_server_3 来创建三服务器集群。要执行此操作，请在配置应用程序服务器屏幕上添加 soa_server_1 应用程序服务器。在配置集群屏幕上创建集群，并将 soa_server_1 选为主要集群成员。然后，您可以使用配置其他集群成员屏幕将 soa_server_2 和 soa_server_3 添加到集群。

  • 名称 - 应用程序服务器的名称。新服务器的默认名称为 new_ManagedServer_n，其中 n 从 1 开始，并且每次添加一个新服务器都会递增。要更改服务器名称，请单击名称字段并键入新名称。

  • 节点名 - 所有可用节点的下拉列表。单击此列表可选择此应用程序服务器驻留的节点。如果您尚未将其他计算机与当前单元联合，则此列表上将只有一个节点可用。

  • 开发模式 - 选中此复选框可在开发模式下运行应用程序服务器。要在生产模式下运行应用程序服务器，请保留此复选框为清除状态。有关 WebSphere 环境中生产模式和开发模式之间的差异的信息，请参阅 WebSphere 文档。

要添加服务器，请单击添加并配置新服务器的设置。

要克隆现有服务器，请选择要克隆的服务器，然后单击克隆。一个新服务器行将添中到表中。新服务器会继承该现有服务器的所有属性。新服务器的默认名称为 clone_source_cloned1，其中 clone_source 为所克隆的服务器的名称。编辑名称字段，选择服务器驻留的节点，选中或清除开发模式选项。

要删除某个服务器，请选择该服务器，然后单击删除。您一次只能删除一个服务器。

要将显示方式从默认的表格格式切换到选项卡格式 (每个服务器对应一个选项卡)，请单击切换显示。再次单击切换显示则恢复到表格格式。

完成添加服务器后，单击下一步可继续。


注：

在您单击最后一个配置向导屏幕上的创建前，在此屏幕上所做的任何更改都不会保存到单元中。如果您退出了向导，或者单击上一步返回到前面的屏幕，则必须再次进行更改。
