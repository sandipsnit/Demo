配置其他集群成员

使用此屏幕可将其他应用程序服务器添加到集群。将新的应用程序服务器添加到集群时，其属性将从集群的主要成员中克隆。

  • 名称 - 应用程序服务器的名称。新服务器的默认名称为 new_ClusterMember_n，其中 n 从 1 开始，并且每次添加一个新服务器都会递增。要更改服务器名称，请单击名称字段并键入新名称。

  • 节点名 - 所有可用节点的下拉列表。单击此列表可选择此应用程序服务器驻留的节点。如果您尚未将其他计算机与当前单元联合，则此列表上将只有一个节点可用。

  • 集群名称 - 所有可用集群的下拉列表。选择要将此服务器分配到的集群。

  • 开发模式 - 选中此复选框可在开发模式下运行应用程序服务器。要在生产模式下运行应用程序服务器，请保留此复选框为清除状态。有关 WebSphere 环境中生产模式和开发模式之间的差异的信息，请参阅 WebSphere 文档。

要添加其他集群成员，请执行以下操作：

  1. 单击添加。一个新行将添加到表中。

  2. 输入服务器的名称，然后选择相应的节点、集群和开发模式设置。

  3. 对每个要添加的集群成员重复步骤 1 和 2。

要删除某个集群成员，请选择该集群成员，然后单击删除。您一次只能删除一个集群成员。

要将显示方式从默认的表格格式切换到选项卡格式 (每个集群对应一个选项卡)，请单击切换显示。再次单击切换显示则恢复到表格格式。

完成后，单击下一步可继续。


注：

在您单击最后一个配置向导屏幕上的创建前，在此屏幕上所做的任何更改都不会保存到单元中。如果您退出了向导，或者单击上一步返回到前面的屏幕，则必须再次进行更改。
