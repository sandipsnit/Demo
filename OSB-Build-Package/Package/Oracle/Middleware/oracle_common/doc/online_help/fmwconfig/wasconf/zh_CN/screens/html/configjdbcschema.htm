配置 JDBC 组件方案

对于某些 Fusion Middleware 组件 (例如，SOA 和 Web Center Spaces)，可能会将 JDBC 数据源定义为组件的数据库方案的一部分，这些数据源是在安装期间使用资料档案库创建实用程序 (RCU) 加载的。

使用配置向导为此类组件创建单元时，可以配置 JDBC 组件方案设置，例如数据库驱动程序、方案所有者、口令等。

此屏幕的下半部分列出了与要为其创建单元的产品 (例如 SOA) 关联的 JDBC 组件方案。


注：

要将一个或多个方案配置为 Oracle RAC 多数据源，请选中这些方案名称旁的复选框，然后选中在下一个面板中将所选组件方案配置为 RAC 多数据源方案复选框。完成后，单击下一步按钮。

通过选中每个方案名称旁的复选框，来选择要为其指定数据源设置的方案。


注：

选择多个组件方案时，某些字段中可能显示各个组件方案之间不同文本，指示在所选组件方案中这些字段的当前值是不同的。如果更改此类字段中的值，则将在所选组件方案中统一应用新值。

供应商、驱动程序、主机名和端口号等组件方案参数的默认值取决于在应用程序模板中指定的值。

  • 供应商 - 选择数据库供应商。

  • 驱动程序 - 选择要用于连接到数据库的 JDBC 驱动程序。该列表中包括适用于所选数据库供应商的公用 JDBC 驱动程序。

  • 方案所有者 - 输入用于连接到数据库的用户名。

  • 方案口令 - 输入指定的方案所有者的口令。

  • DBMS/服务 - 输入数据库 DBMS 名或服务名 (如果选定了服务类型驱动程序)。

  • 主机名 - 输入承载数据库的服务器的名称。

  • 端口 - 输入用于连接到承载数据库的服务器的端口号。

在方案列表中，为选定方案指定的值将显示在相应列中。

完成配置组件方案后，单击下一步可继续。
