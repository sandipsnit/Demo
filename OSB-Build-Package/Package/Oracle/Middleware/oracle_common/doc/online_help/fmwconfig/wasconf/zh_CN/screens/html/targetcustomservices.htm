将定制服务定位到服务器

使用此屏幕可以将定制服务定位到相应的服务器。您无法将定制服务定位到集群。已分配给集群的服务器将包括在目标列表框中。

与要配置单元的产品 (例如 SOA) 关联的定制服务将自动定位到为该产品创建的服务器。在此屏幕中，您可以将服务定位到其他服务器。

要将服务定位到服务器，请执行以下操作：

  1. 在目标列表框中，选择要在其中部署服务的服务器。

    所选目标的名称将作为右侧的列表框的标题显示。

  2. 在 target_name 列表框中，选中与要在所选目标上部署的服务对应的复选框。

    此处显示的服务因您在该向导中前面的将产品添加到单元屏幕中选择的产品而异。

    选择定制服务后，所定位的服务器的名称将显示在 target_name 列表框的目标列中。

  3. 如有必要，对其他服务器重复执行步骤 1 和 2。

  4. 做出所需选择之后，单击下一步。

重新配置现有单元时，如果删除了定制服务当前定位到的服务器，则配置向导将按如下所示自动重新定位这些服务：

  • 如果定制服务最初仅定位到当前正在删除的服务器 (即，删除该服务器之后，将取消定制服务在所修改单元中的定位)，则配置向导会自动将定制服务重新定位到所有符合条件的目标。

    符合条件的目标是指未在已包含的产品模板的配置组规范 (config-groups.xml 文件) 中定义的任何服务器。在 config-groups.xml 中指定的服务器实质上由模板拥有，因此不符合自动定位的条件。

  • 如果定制服务最初定位到多个目标 (包括服务器和管理服务器)，并且定位到的其中一个服务器被删除，则配置向导将保持其余的目标关联不变，并且不会尝试重新定位这些服务。
