将资源适配器定位到服务器

使用此屏幕可以将资源适配器定位到各个服务器。您无法将资源适配器定位到集群。已分配给集群的服务器将包括在目标列表框中。

与要配置单元的产品 (例如 SOA) 关联的资源适配器将自动定位到为该产品创建的服务器。在此屏幕中，您可以将资源适配器定位到其他服务器。

要将资源适配器定位到服务器，请执行以下操作：

  1. 在目标列表框中，选择要在其上定位资源适配器的服务器。

    所选目标的名称将作为右侧的列表框的标题显示。

  2. 在 target_name 列表框中，选中与要定位到所选服务器的资源适配器对应的复选框。

    此处显示的资源适配器因您在该向导中前面的将产品添加到单元屏幕中选择的产品而异。

    选择资源适配器后，所定位的服务器的名称将显示在 target_name 列表框的目标列中。

  3. 如有必要，对其他服务器重复执行步骤 1 和 2。

  4. 做出所需选择之后，单击下一步。

重新配置现有单元时，如果删除了资源适配器当前定位到的服务器，则配置向导将按如下所示自动重新定位这些资源适配器：

  • 如果资源适配器最初仅定位到当前正在删除的服务器 (即，删除该服务器之后，将取消资源适配器在所修改单元中的定位)，则配置向导会自动将资源适配器重新定位到所有符合条件的目标。

    符合条件的目标是指未在已包含的产品模板的配置组规范 (config-groups.xml 文件) 中定义的任何服务器。在 config-groups.xml 中指定的服务器实质上由模板拥有，因此不符合自动定位的条件。

  • 如果资源适配器最初定位到多个目标 (包括服务器和管理服务器)，并且所定位的其中一个服务器被删除，则配置向导将保持其余的目标关联不变，并且不会尝试重新定位这些资源适配器。
