設定應用程式伺服器

您可以在此畫面新增、複製或刪除「應用程式伺服器」到封包中的任一節點。也可以變更現有的「應用程式伺服器」設定。星號 (*) 表示必要欄位。


注意：

只能使用此畫面來建立 (a) 不是叢集主要成員，或 (b) 不會是叢集一部分的「應用程式伺服器」。不要在此畫面上建立次要叢集成員。請改用設定其他叢集成員畫面來新增次要叢集成員至叢集。

例如，假如您要建立有 soa_server_1、soa_server_2 以及 soa_server_3 伺服器的 3 個伺服器叢集。作法是在設定應用程式伺服器畫面上新增 soa_server_1 應用程式伺服器。在設定叢集畫面上建立叢集，然後選取 soa_server_1 作為主要叢集成員。然後您可以使用設定其他叢集成員畫面來新增 soa_server_2 和 soa_server_3 至叢集。

  • 名稱—「應用程式伺服器」的名稱。新伺服器的預設名稱為 new_ManagedServer_n，其中 n 從 1 開始，隨著您新增的每個新伺服器遞增。若要變更伺服器名稱，請按一下名稱欄位，鍵入新名稱。

  • 節點名稱—所有可用節點的下拉式清單。按一下清單以選取此「應用程式伺服器」所駐留的節點。如果您尚未將其他機器與目前的封包聯合，則此清單上只會有一個可用的節點。

  • 開發模式—要在開發模式中執行「應用程式伺服器」，請選取此核取方塊。要在生產模式中執行「應用程式伺服器」，則請清除此核取方塊。有關 WebSphere 環境中生產模式與開發模式之間差異的詳細資訊，請參閱 WebSphere 文件。

若要新增伺服器，請按一下新增，並設定新伺服器的設定。

若要複製現有的伺服器，請選取您想要複製的伺服器，按一下複製。新伺服器行就會新增到表格。新伺服器繼承現有伺服器的所有屬性。新伺服器的預設名稱為 clone_source_cloned1，其中 clone_source 是您複製之伺服器的名稱。若要編輯名稱欄位，請選取伺服器所駐留的節點，然後選取或清除開發模式選項。

若要刪除伺服器，請選取伺服器，然後按一下刪除。您一次只能刪除一個伺服器。

若要將預設的顯示從表格格式切換為頁籤式格式 (一個頁籤代表一個伺服器)，請按一下切換顯示。再按一下切換顯示，就會回到表格格式。

完成新增伺服器後，請按下一步，繼續作業。


注意：

在上一個「組態精靈」畫面按一下建立之前，您在此畫面上所做的任何變更都不會儲存。如果您結束精靈，或按上一步返回先前的畫面，就必須重新進行變更。
