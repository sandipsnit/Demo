設定其他叢集成員

使用此畫面可以新增其他「應用程式伺服器」到您的叢集。新增新的「應用程式伺服器」到叢集時，其屬性會從叢集的主要成員複製。

  • 名稱—「應用程式伺服器」的名稱。新伺服器的預設名稱為 new_ClusterMember_n，其中 n 從 1 開始，隨著您新增的每個新伺服器遞增。若要變更伺服器名稱，請按一下名稱欄位，並鍵入新名稱。

  • 節點名稱—所有可用節點的下拉式清單。按一下清單以選取此「應用程式伺服器」所駐留的節點。如果您尚未將其他機器與目前的封包聯合，則此清單上只會有一個可用的節點。

  • 叢集名稱—此下拉式清單包含所有可用的叢集。選取此伺服器要指派到的叢集。

  • 開發模式—如果要在開發模式中執行「應用程式伺服器」，請選取此核取方塊。要在生產模式中執行應用程式伺服器，則請清除此核取方塊。有關 WebSphere 環境中生產模式與開發模式之間差異的詳細資訊，請參閱 WebSphere 文件。

新增其他叢集成員：

  1. 按一下新增，會在表格新增新的資料列。

  2. 請輸入伺服器的名稱，然後選取適當的節點、叢集以及開發模式設定。

  3. 請針對每位要新增的叢集成員，重複 1 到 2 的步驟。

若要刪除叢集成員，請選取叢集成員，然後按一下刪除。您一次只能刪除一位叢集成員。

若要將預設的顯示從表格格式切換為頁籤式格式 (一個頁籤代表一個叢集)，請按一下切換顯示。再按一下切換顯示，會回到表格格式。

完成後，請按下一步，繼續作業。


注意：

在上一個「組態精靈」畫面按一下建立之前，您在此畫面上所做的任何變更都不會儲存。如果您結束精靈，或按上一步返回先前的畫面，就必須重新進行變更。
