設定叢集

叢集是共同運作的一組「應用程式伺服器」，為應用程式提供擴充性和高可用性。您可以建立叢集來群組伺服器，讓它們可以當成主控應用程式和資源的單一單元來操作。

使用此畫面可以新增叢集，並選取「應用程式伺服器」作為叢集的主要叢集成員。所有的其他叢集成員都是從此一主要叢集成員複製。建立叢集後，您可以新增其他「應用程式伺服器」到下一個畫面上的叢集。

您也可以使用此畫面來刪除叢集或修改現有叢集的設定。

建立叢集：

  1. 按一下新增。表格上就會出現新的一行。新叢集的預設名稱為 new_Cluster_n，其中 n 從 1 開始，隨著您新增的每個新叢集遞增。若要變更預設，請在名稱欄位中鍵入想要的名稱。叢集的名稱在封包內所有元件名稱中必須是唯一的。

  2. 請從第一個叢集成員下拉式清單中的伺服器，選取現有的「應用程式伺服器」作為叢集的主要成員。清單中只顯示尚未指派到叢集的「應用程式伺服器」。

  3. 請針對每一個要建立的其他叢集，重複 1 到 2 的步驟。

  4. 完成後，請按下一步。按下一步之後，會顯示設定其他叢集成員畫面，您可以新增其他伺服器到此畫面上的叢集。


    注意：

    在上一個「組態精靈」畫面按一下建立之前，您在此畫面上所做的任何變更都不會儲存。如果您結束精靈，或按上一步返回先前的畫面，就必須重新進行變更。

若要刪除叢集，請選取叢集，然後按一下刪除。刪除叢集時，並不會刪除主要叢集成員的「應用程式伺服器」，僅會將其由叢集移除，之後可新增至其他叢集。您一次只能刪除一個叢集。

若要變更叢集的設定，請選取叢集，然後進行適當的變更。

若要將預設的顯示從表格格式切換為頁籤式格式 (一個頁籤代表一個叢集)，請按一下切換顯示。再按一下切換顯示，會回到表格格式。

完成叢集設定後，請按下一步，繼續作業。

有關 WebSphere 環境中叢集的詳細資訊，請參閱 WebSphere 文件。
