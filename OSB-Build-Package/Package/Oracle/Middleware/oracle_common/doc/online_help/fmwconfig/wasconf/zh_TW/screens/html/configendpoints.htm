設定端點

使用此畫面可以在每個伺服器 (包含「管理伺服器」) 上設定下列端點 (監聽位址)。所有的端點在封包中必須是唯一的。在預設狀況下，新增伺服器到封包時，唯一的端點就會指派到伺服器。除非您要使用預設 (指定伺服器上所設定) 以外的連接埠，否則無需修改它們。

有關此畫面所列出各種端點的詳細資訊，請參閱 WebSphere 文件。

若要變更端點：

  1. 從伺服器名稱/節點名稱下拉式清單中，選取您想要為其變更端點的伺服器。此清單包含封包中所有伺服器，包括指派到叢集的「管理伺服器」、「應用程式伺服器」，以及存在於封包中所有節點上的獨立「應用程式伺服器」。

  2. 請在適當的資料列鍵入新端點值。如果您輸入的值已在使用中，就會顯示下列驗證錯誤。請按一下確定，然後輸入端點的其他值。



    注意事項：

    如果您打算在同一台機器上執行多個「建置管理程式」，就必須確定每個相關聯封包中的端點，在該機器上的所有「建置管理程式」節點中是唯一的。

    同樣地，如果您在遠端機器上有「應用程式伺服器」，並且指派到「建置管理程式」主機上的不同封包，則您必須確定指定機器上所有「應用程式伺服器」的所有端點都是唯一的。

    若不同封包上有衝突的端點，您並不會收到警告。

  3. 請針對每一個要變更的端點，重複步驟 2。

如果不需要端點，您也可停用任一伺服器上的端點。若要停用伺服器上一或多個端點，請從伺服器名稱/節點名稱下拉式清單中選取伺服器，然後清除要停用之每個端點的核取方塊。

若要指定伺服器上的所有端點都使用一般主機，請從伺服器名稱/節點名稱下拉式清單中選取伺服器，然後選取所有端點都使用一般主機核取方塊。選取此核取方塊時，目前選取之伺服器上每個端點的主機名稱會變成無法編輯，管理端點主機名稱是例外情況。

完成後，請按下一步。如果有任何端點發生衝突，就會出現驗證錯誤對話方塊，您必須先解決衝突才能繼續。
