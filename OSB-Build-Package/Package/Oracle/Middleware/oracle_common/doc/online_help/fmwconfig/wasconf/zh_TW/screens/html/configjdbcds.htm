設定 JDBC 資料來源

JDBC 資料來源包含資料庫連線集區，而這些資料庫連線集區是在建立資料來源執行處理 (建置或定位資料來源執行處理) 時，或伺服器啟動時建立的。應用程式會在 JNDI 樹狀圖上尋找資料來源，然後請求連線。應用程式不再需要連線時，會將連線傳回到資料來源的連線集區中。

使用此畫面可以設定封包中定義的 JDBC 資料來源。

與正在建立封包之產品相關聯的 JDBC 資料來源，會列在畫面的下半部。


注意：

若要設定一個或多個資料來源作為 Oracle RAC 多重資料來源，請選取與資料來源名稱相鄰的核取方塊，並選取在下一個面板中，將選取資料來源設定為 RAC 多重資料來源核取方塊，然後按下一步按鈕。

選取資料來源名稱旁的核取方塊，以選取想要指定設定的每個資料來源。


注意事項：

當您選取多個資料來源時，可能會在特定欄位中顯示因資料來源而異文字，指出在選取之資料來源中，那些欄位的目前值會不同。如果您變更這類欄位中的值，則會將新值統一套用至選取之資料來源。

資料來源參數 (如供應商、驅動程式、主機名稱和連接埠號碼) 的預設值取決於應用程式樣板中指定的值。

  • 供應商—選取資料庫供應商。

  • 驅動程式—選取要用來連線至資料庫的 JDBC 驅動程式。此清單包含選取之資料庫供應商的一般 JDBC 驅動程式。

  • 使用者名稱—輸入用來連線資料庫的使用者名稱。

  • 密碼—輸入指定之使用者名稱的密碼。

  • DBMS/服務—輸入 DBMS SID 或服務名稱。您輸入的值取決於您選取的驅動程式。

    • 如果您選取的 Oracle 驅動程式名稱包含 "for Instance connections" 字樣，就必須輸入 SID。

    • 如果 Oracle 驅動程式名稱包含 "for Service connections" 字樣，就必須輸入服務名稱。

    • 有關設定 DataDirect 驅動程式的詳細資訊，請參閱 DataDirect 文件。

  • 主機名稱—輸入主控資料庫之伺服器的名稱。

  • 連接埠—輸入用來連線至伺服器的連接埠號碼。

所指定的值會顯示在選取資料來源之資料來源清單中的適當資料欄。

完成 JDBC 資料來源設定後，請按下一步，繼續作業。
