設定 JDBC 元件綱要

對於某些 Fusion Middleware 元件 (例如，SOA 和 Web Center Spaces) 而言，可能會將使用「建立儲存區域公用程式」(RCU) (安裝時) 載入的 JDBC 資料來源，定義為該元件資料庫綱要的一部分。

當您使用「組態精靈」為這類元件建立封包時，您可以設定 JDBC 元件綱要設定，如資料庫驅動程式、綱要所有者、密碼等。

與正在建立封包之產品 (例如，SOA) 相關聯的 JDBC 元件綱要，會列在畫面的下半部。


注意：

若要設定一或多個綱要作為 Oracle RAC 多重資料來源，請選取綱要名稱旁的核取方塊，並選取在下一個面板中，將選取元件綱要設定為 RAC 多重資料來源綱要核取方塊。完成後，按下一步。

選取綱要名稱旁的核取方塊，以選取想要為其指定資料來源設定的綱要。


注意：

當您選取多個元件綱要時，可能會在特定欄位中顯示因元件綱要而異文字，指出在選取之元件綱要中，那些欄位的目前值會不同。如果您變更這類欄位中的值，則會將新值統一套用至選取的元件綱要。

元件綱要參數 (如供應商、驅動程式、主機名稱和連接埠號碼) 的預設值，取決於應用程式樣板中指定的值。

  • 供應商—選取資料庫供應商。

  • 驅動程式—選取要用來連線至資料庫的 JDBC 驅動程式。此清單包含選取之資料庫供應商的一般 JDBC 驅動程式。

  • 綱要所有者—輸入用來連線資料庫的使用者名稱。

  • 綱要密碼—輸入指定之綱要所有者的密碼。

  • DBMS/服務—輸入資料庫 DBMS 名稱或服務名稱 (如果您選取服務類型驅動程式)。

  • 主機名稱—輸入主控資料庫之伺服器的名稱。

  • 連接埠—輸入用來連線至主控資料庫之伺服器的連接埠號碼。

對於選取的綱要，您指定的值會顯示在綱要清單的適當資料欄。

完成元件綱要設定後，請按下一步，繼續作業。
