設定 RAC 資料來源元件綱要

使用此畫面將 WebSphere 封包中所含的元件綱要設為 Oracle RAC 綱要。

在精靈的設定 JDBC 元件綱要畫面中，您選擇將其設定為 Oracle RAC 綱要的元件綱要，會列在畫面的下半部。

在畫面下半部的綱要清單中，選取綱要名稱旁的核取方塊，以選取要設定為 Oracle RAC 綱要的每個綱要。


注意：

當您選取多個綱要時，可能會在特定欄位中顯示因元件綱要而異文字，指出在選取之綱要中，那些欄位的目前值會不同。如果您繼續進行並變更這類欄位中的值，則會將新值統一套用至選取的綱要。

  • 驅動程式—選取用來連線資料庫的 JDBC 驅動程式。

  • 服務名稱—輸入資料庫 Oracle RAC 服務名稱。

  • 使用者名稱—輸入用來連線資料庫的使用者名稱。

  • 密碼—輸入指定之使用者名稱的密碼。

  • 主機名稱—輸入主控 Oracle RAC 資料庫執行處理之伺服器的名稱。

  • 執行處理名稱—輸入每個 Oracle 資料庫執行處理的名稱。

  • 連接埠—輸入用來連線至主控資料庫之伺服器的連接埠號碼。


注意：

您至少必須指定一個資料庫執行處理的主機名稱、執行處理名稱和連接埠號碼。

若要新增資料庫執行處理，請按一下新增。指定主機名稱、執行處理名稱和連接埠。

對於選取的綱要，您指定的值會顯示在綱要清單的適當資料欄。
