指定封包、設定檔和節點資訊

請在下列欄位中輸入適當的值。每一個都是必要欄位。每一個欄位中只能使用英數字元。

  • 封包名稱—為新封包輸入唯一的名稱。這是在下列兩個目錄底下，建立封包所在的目錄名稱：

    • WAS_HOME/profiles/DepMgrProfile/config/cells，其中 WAS_HOME 是 WebSphere 安裝目錄，而 DepMgrProfile 是「建置管理程式」設定檔的名稱。

    • WAS_HOME/profiles/AppSrvrProfile/config/cells，其中 WAS_HOME 是 WebSphere 安裝目錄，而 AppSrvrProfile 是「應用程式伺服器」設定檔的名稱。

    預設名稱為 machine_nameCelln，其中 machine_name 是目前機器的名稱，而 n 是該機器依序的下一個二位數數字。例如，如果機器名稱是 machine1，而 Cell01 已經存在，則預設的封包名稱為 machine1Cell02。

  • 建置管理程式設定檔名稱—為此封包輸入唯一的「建置管理程式」設定檔名稱。這是在 WAS_HOME/profiles 底下建立「建置管理程式」設定檔所在的目錄名稱。

    預設的名稱為 Dmgrn，其中 n 是依序的下一個二位數數字。例如，如果「建置管理程式」設定檔 Dmgr01 已經存在，則此欄位的預設值為 Dmgr02。

  • 建置管理程式節點名稱—為此封包輸入唯一的「建置管理程式」節點名稱。

    預設名稱為 machine_nameCellManagern，其中 machine_name 是目前機器的名稱，而 n 是依序的下一個二位數數字。例如，如果機器名稱是 machine1，而 CellManager01 已經存在，則預設的節點名稱為 machine1CellManager02。

  • 應用程式伺服器設定檔名稱—為此封包輸入唯一的「應用程式伺服器」設定檔名稱。這是在 WAS_HOME/profiles 底下建立「應用程式伺服器」設定檔所在的目錄名稱。

    預設的名稱為 Customn，其中 n 是序列中下一個二位數的數字。例如，如果「應用程式伺服器」設定檔 Custom01 已經存在，則此欄位的預設值為 Custom02。

  • 應用程式伺服器節點名稱—為此封包輸入唯一的「應用程式伺服器」節點名稱。

    預設名稱為 machine_nameNoden，其中 machine_name 是目前機器的名稱，而 n 是該機器依序的下一個二位數數字。例如，如果機器名稱是 machine1，而 Node01 已經存在，則預設的節點名稱為 machine1Node02。

每個欄位都輸入完成後，請按下一步。
