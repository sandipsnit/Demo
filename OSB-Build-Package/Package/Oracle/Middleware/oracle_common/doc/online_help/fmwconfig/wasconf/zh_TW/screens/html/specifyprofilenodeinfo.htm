指定設定檔和節點名稱資訊

  • 應用程式伺服器設定檔名稱—為此節點輸入唯一的「應用程式伺服器」設定檔名稱。此名稱是在 WAS_HOME/profiles 底下，於建立節點所在的機器上與「建置管理程式」主機上，建立「應用程式伺服器」設定檔所在的目錄名稱。

    預設的名稱為 Customn，其中 n 是「建置管理程式」節點 (您在上一個畫面所指定) 依序的下一個二位數數字。例如，如果封包中已經有「應用程式伺服器」設定檔 Custom01，則此欄位的預設值為 Custom02。

    如果您變更預設設定檔名稱，請確定該名稱在封包中是唯一的。

  • 應用程式伺服器節點名稱—輸入此「應用程式伺服器」要使用的節點名稱。

    預設名稱為 machine_nameNoden，其中 machine_name 是目前機器的名稱，而 n 是目前機器上依序的下一個二位數數字。例如，如果機器名稱是 machine2，而 Node01 已經存在，則預設的節點名稱為 machine2Node02。

    如果您變更預設節點名稱，請確定該名稱在封包中是唯一的。

每個欄位都輸入完成後，請按下一步，繼續作業。
