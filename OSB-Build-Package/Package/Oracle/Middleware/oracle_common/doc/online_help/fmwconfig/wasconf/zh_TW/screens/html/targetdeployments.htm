將建置定位至叢集或伺服器

使用此畫面以針對伺服器或叢集的建置，定位應用程式和程式庫。

會將與設定封包之產品 (例如，SOA) 相關聯的應用程式和程式庫，自動定位至為該產品所建立的伺服器，或自動定位至已指派該伺服器的叢集。在此畫面中，您可以將應用程式和程式庫定位至其他伺服器和叢集。

若要將應用程式建置定位至叢集或伺服器：

  1. 在目標清單方塊中，選取想要在其上建置應用程式或程式庫的叢集或伺服器。

    選取之目標的名稱會顯示為右側清單方塊的標題。

  2. 在 target_name 清單方塊中，選取對應至要建置於選取目標之應用程式或程式庫的核取方塊。

    這裡顯示的應用程式和程式庫，會根據您先前在精靈的將產品新增至封包畫面中選取的產品而不同。


    注意：

    當您在目標清單方塊中選取伺服器時，可能會停用 target_name 清單方塊中的部分核取方塊，以指出已定位至含有選取伺服器之叢集的應用程式和程式庫。

    選取應用程式和程式庫之後，已定位叢集和伺服器的名稱會顯示在 target_name 清單方塊中的目標資料欄內。

  3. 依需要針對其他叢集和伺服器，重複步驟 1 和 2。

  4. 進行必要的選取之後，請按一下「下一步」。

當您正在重新設定現有的封包時，如果您刪除應用程式目前所定位的伺服器或叢集，「組態精靈」會依下列方式自動重新定位應用程式：

  • 如果應用程式之前只定位到您刪除的伺服器或叢集 (即在您刪除伺服器或叢集之後，應用程式在修改後的封包中會變成未定位)，「組態精靈」會自動重新定位應用程式至所有合適的目標。

    合適的目標是指未在包含樣板之組態群組規格 (config-groups.xml 檔案) 中定義的任何叢集或伺服器。config-groups.xml 檔案中指定的伺服器或叢集基本上是樣板所擁有，因此不適合用來自動定位。

  • 如果應用程式之前定位到多個目標 (包括伺服器、叢集和「管理伺服器」)，刪除其中一個所定位的伺服器或叢集時，「組態精靈」會維持其他的目標關聯不變，而且不會嘗試重新定位應用程式。
