Administrationsserver konfigurieren

Der Administrationsserver ist der zentrale Punkt, von dem aus Sie die Domain verwalten. Sie können über den URL protocol://listen-address:listen-port auf den Administrationsserver zugreifen. Eines der folgenden Protokolle kann verwendet werden: t3, t3s, http oder https.

Hier können Sie die Einstellungen für den folgenden Administrationsserver konfigurieren oder ändern. Pflichtfelder sind mit einem Sternchen (*) markiert.

Feld Beschreibung
*Name Der Standardname ist "AdminServer". Gültige Servernamen setzen sich aus einer Folge von alphabetischen und numerischen Zeichen zusammen.

Jede Serverinstanz in der Produktionsumgebung muss einen eindeutigen Namen haben, unabhängig von der Domain oder dem Cluster, der bzw. dem sie angehört, und unabhängig davon, ob es sich um einen Administrationsserver oder einen Managed Server handelt. Darüber hinaus muss der Name des Administrationsservers unter allen Komponentennamen innerhalb der Domain eindeutig sein.

Hinweis: Dieser Wert wird nur zu Identifizierungszwecken angegeben. Er wird nicht als Teil des URL für Anwendungen, die auf dem Server bereitgestellt werden, verwendet. Der Servername wird in der Administrationskonsole von WebLogic Server angezeigt. Außerdem müssen Sie diesen Namen zur Identifizierung des Servers angeben, wenn Sie Befehlszeilen-Utilitys oder APIs von WebLogic Server verwenden.

*Listening-Adresse Wählen Sie aus der Dropdown-Liste einen Wert für die Listening-Adresse aus.

Wenn Sie localhost als Listening-Adresse für eine Serverinstanz auswählen, können remote ausgeführte Prozesse keine Verbindung mit der Serverinstanz aufnehmen. Dann können nur solche Prozesse Verbindung mit der Serverinstanz aufnehmen, die sich auf demselben Rechner wie die Serverinstanz befinden. Wenn die Serverinstanz als lokaler Host zur Verfügung stehen muss (Beispiel: Sie erstellen administrative Skripte, die mit dem lokalen Host verbunden sind) und auch für remote ausgeführte Prozesse zugänglich sein soll, wählen Sie Alle lokalen Adressen. Die Serverinstanz ermittelt dann die Adresse des Rechners und überwacht diesen.

Weitere Informationen dazu finden Sie unter Listening-Adresse angeben.
Listening-Port Geben Sie einen gültigen Wert für den Listening-Port ein, der für reguläre, ungesicherte Anforderungen (über Protokolle wie HTTP und T3) verwendet werden soll.

Der Standardwert ist 7001 für den Administrationsserver. Wenn Sie dieses Feld leer lassen, wird der Standardwert verwendet.

Der Bereich gültiger Listening-Ports liegt zwischen 1 und 65534.

Weitere Informationen finden Sie unter Listening Port angeben.
SSL aktiviert Aktivieren Sie dieses Kontrollkästchen, um den SSL-Listening-Port zu aktivieren. Standardmäßig ist SSL für alle neuen Server deaktiviert.
SSL-Listening-Port Dieses Feld ist nur dann aktiviert, wenn das Kontrollkästchen SSL aktiviert aktiviert ist.

Geben Sie einen gültigen Wert ein, der für gesicherte Anforderungen (über Protokolle wie HTTPS und T3S) verwendet werden soll. Der Standardwert ist 7002. Wenn Sie dieses Feld leer lassen, wird der Standardwert verwendet.

Der Bereich gültiger Listening-Ports liegt zwischen 1 und 65535.

Hinweis: Standardmäßig verwendet eine Serverinstanz Demonstrationszertifikate, um Anforderungen von einem gesicherten Port zu authentifizieren. In einer Produktionsumgebung müssen Sie SSL für die Verwendung von Zertifikaten von einer Zertifizierungsstelle konfigurieren.

Wenn Sie alle erforderlichen Einstellungen und Einträge vorgenommen haben, klicken Sie auf die Schaltfläche Weiter.

Weitere Informationen finden Sie in der Dokumentations-Library für Oracle Fusion Middleware im Oracle Technology Network.

Listening-Adresse angeben

Dieser Abschnitt enthält Richtlinien zur Angabe einer Listening-Adresse für den Administrationsserver. Diese Richtlinien gelten auch für alle Managed Server.

Listening-Adresstyp Verhalten
Alle lokalen Adressen oder DNS-Name Auf mehrfach vernetzten Rechnern ist eine Serverinstanz mit allen verfügbaren IP-Adressen verbunden.
IP-Adresse oder DNS-Name
  • Um eine Verbindung mit einer Serverinstanz herzustellen, kann für Prozesse entweder die IP-Adresse oder der entsprechende DNS-Namen angegeben werden.
  • Bei Angabe von LocalHost ist die Verbindung von Prozessen nicht erfolgreich.
  • Sie müssen die vorhandenen Prozesse aktualisieren, die LocalHost für die Verbindung mit einer Serverinstanz verwenden.
  • Für Verbindungen, bei denen als Listening-Adresse die IP-Adresse und als Listening Port ein gesicherter Port angegeben werden, muss die Hostnamenüberprüfung deaktiviert werden.

Hinweis:Um einen DNS-Namen zu einer IP-Adresse aufzulösen, muss WebLogic Server Kontakt zum entsprechenden DNS-Server herstellen können oder die IP-Adresszuordnung lokal erreichen. Daher müssen Sie bei Angabe eines DNS-Namens für die Listening-Adresse entweder einen Port lange genug offen lassen, damit die WebLogic Server-Instanz eine Verbindung zum DNS-Server herstellen und die Zuordnungen cachen kann. Oder Sie müssen die IP-Adresszuordnung in einer lokalen Datei angeben. Wenn Sie eine IP-Adresse für die Listening-Adresse angeben und eine Clientanforderung dann einen DNS-Namen angibt, versucht WebLogic Server, den DNS-Namen aufzulösen. Falls er jedoch nicht auf die DNS-Namenszuordnung zugreifen kann, ist die Anforderung nicht erfolgreich.
localhost
  • Um eine Verbindung zwischen Prozessen und der Serverinstanz herzustellen, muss LocalHost angegeben werden.
  • Eine Verbindung mit der Serverinstanz kann nur für Prozesse hergestellt werden, die sich auf dem gleichen Rechner wie die Serverinstanz befinden (lokale Prozesse).

Listening Port angeben

Beachten Sie bei der Angabe des Listening Ports für den Administrationsserver die folgenden Richtlinien. Diese Richtlinien gelten auch für Managed Server.

  • Sie können jede beliebige Port-Nummer angeben. Wenn Sie jedoch Port 80 angeben, können Sie die Port-Nummer bei der HTTP-Anforderung weglassen, die für den Zugriff auf Ressourcen über HTTP verwendet wird. Beispiel: Wenn Sie Port 80 als Listening-Port festlegen, können Sie den URL http://hostname:portnumber/myfile.html verwenden.
  • Bei einigen Betriebssystemen ist Port 80 nur für solche Prozesse zugänglich, die unter einer berechtigten Benutzer- oder Gruppen-ID ausgeführt werden. In diesem Fall können Sie die Serverinstanz einem UNIX-Rechner zuweisen, auf dem eine Post-Bind-UID oder -GID definiert ist.
