Cluster konfigurieren

Hier können Sie Cluster hinzufügen, konfigurieren und löschen.

Ein Cluster ist eine Gruppe von WebLogic Server-Instanzen, die gemeinsam für Skalierbarkeit und hohe Verfügbarkeit von Anwendungen sorgen. Durch die Erstellung von Clustern können Sie Managed Server in Gruppen zusammenfassen, so dass sie als ein einzelner Host für Anwendungen und Ressourcen fungieren.

  1. Prüfen Sie die aktuelle Cluster-Konfiguration. Die Standardwerte hängen von der gewählten Domain-Quelle ab.

    Hinweis: Der Assistent bietet zwei Ansichten: eine tabellarische Übersicht aller Cluster und eine Einzelansicht jedes einzelnen Clusters, bei der jeder Server über eine eigene Registerkarte dargestellt wird. Sie können den Anzeigemodus zwischen Tabellen- und Registerkartenansicht hin- und herwechseln, indem Sie auf Anzeige umschalten klicken.

  2. Je nach den Erfordernissen Ihrer Domain können Sie Cluster hinzufügen oder löschen oder die Einstellungen für bestehende Cluster ändern.

    In der folgenden Tabelle werden die einzelnen Felder auf diesem Bildschirm beschrieben. Mit einem Sternchen (*) markierte Felder müssen ausgefüllt werden.

    Feld Beschreibung
    Name* Geben Sie einen gültigen Namen für das Cluster ein.

    Der Name des Clusters muss unter allen Komponentennamen innerhalb der Domain eindeutig sein. Der Standardwert in diesem Feld ist new_Cluster_n, wobei n ein numerischer Wert ist, mit dem die einzelnen Standard-Clusternamen unterschieden werden. Der Wert von n für das erste Cluster ist 1. Für jedes weitere Cluster wird der Wert um 1 erhöht.
    Cluster-Messaging-Modus Wählen Sie als Messaging-Modus "Unicast" oder "Multicast".
    Multicast-Adresse Dieses Feld ist nur aktiviert, wenn Sie den Messaging-Modus "Multicast" ausgewählt haben.

    Geben Sie die Multicast-Adresse für das Cluster ein. Diese Adresse wird von Cluster Members für die Kommunikation untereinander verwendet. Der Standardwert ist 239.192.0.0. Der Bereich gültiger Multicast-Adressen liegt zwischen 224.0.0.1 und 239.255.255.255.
    Multicast Port Dieses Feld ist nur aktiviert, wenn Sie den Messaging-Modus "Multicast" ausgewählt haben.

    Geben Sie den Multicast Port für das Cluster ein. Der Multicast Port wird von Cluster Members für die Kommunikation untereinander verwendet. Der Standardwert ist 7001.
    Cluster-Adresse Geben Sie die Adressen der Managed Server im Cluster ein. Eine Cluster-Adresse kann folgendermaßen aussehen:
    • Durch Komma getrennte Liste von IP-Adressen oder DNS-Namen und -Ports (Beispiel: dns_name:port, dns_name:port)
    • DNS-Name, der mehreren IP-Adressen zugeordnet ist
    • localhost, DNS-Name oder IP-Adresse, wenn die Listening-Adresse aller Managed Server auf dieselbe Listening-Adresse mit eindeutigen Port-Nummern eingestellt ist

    Die Cluster-Adresse wird in Entity- und Stateless-EJBs verwendet, um den Host-Namen-Anteil von URLs zu erstellen. Ist die Cluster-Adresse nicht eingestellt, funktionieren die EJB-Handles möglicherweise nicht ordnungsgemäß.
  3. Wenn Sie alle erforderlichen Einstellungen und Einträge vorgenommen haben, klicken Sie auf die Schaltfläche Weiter.

Um ein Cluster zu löschen, wählen Sie den Server und klicken auf Löschen. Beim Löschen eines Clusters löschen Sie nicht die ihm zugeordneten Server. Die Server werden lediglich aus dem Cluster entfernt und können dann einem anderen Cluster hinzugefügt werden. Sie können Cluster nur einzeln löschen.

Um ein Cluster hinzufügen, klicken Sie auf Hinzufügen und konfigurieren Sie die Einstellungen für den neuen Server. Der Standardname für ein neues Cluster lautet new_Cluster_n, wobei n bei 1 beginnt und sich für jedes weitere Cluster schrittweise erhöht. Zur Änderung des Standards geben Sie den gewünschten Namen in der Spalte "Name" ein.

Um die Anzeige von dem standardmäßigen tabellarischen Format in ein Registerkartenformat zu ändern (mit einer Registerkarte pro Cluster), klicken Sie auf Anzeige umschalten. Um zum tabellarischen Format zurückzukehren, klicken Sie erneut auf Anzeige umschalten.
