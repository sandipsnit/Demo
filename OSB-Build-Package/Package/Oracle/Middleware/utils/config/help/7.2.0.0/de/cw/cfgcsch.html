JDBC-Komponentenschema konfigurieren

Für einige Fusion Middleware-Komponenten (Beispiel: SOA und Web Center Spaces) können JDBC-Datenquellen als Teil des Datenbankschemas der Komponente definiert werden. Sie werden dann während der Installation mit dem Utility für das Erstellen des Repositorys (RCU) geladen.

Wenn Sie mit dem Konfigurationsassistenten eine WebLogic Domain für solche Komponenten erstellen, können Sie die Schemaeinstellungen für die JDBC-Komponente, wie Datenbanktreiber, Schemaeigentümer, Kennwort usw., konfigurieren.

In der unteren Bildschirmhälfte sind die JDBC-Komponentenschemas aufgelistet, die mit den Produkten verknüpft sind, für die Sie die Domain erstellen (Beispiel: SOA).

Hinweis: Wenn Sie eines oder mehrere der Schemas als RAC-Datenquellen konfigurieren möchten, aktivieren Sie die Kontrollkästchen neben dem Namen des jeweiligen Schemas, und aktivieren Sie das Kontrollkästchen Konfigurieren Sie gewählte Komponentenschemas als RAC-Multidatenquellschemas im nächsten Fensterbereich. Klicken Sie dann auf die Schaltfläche Weiter.

  1. Wählen Sie die Schemas aus, für die Sie Datenquelleinstellungen vornehmen möchten, indem Sie die Kontrollkästchen neben den Schemanamen aktivieren.

    Hinweis: Wenn Sie mehrere Komponentenschemas auswählen, wird möglicherweise der Text "Variiert unter Komponentenschemas" in bestimmten Feldern angezeigt. Dies weist darauf hin, dass die aktuellen Werte dieser Felder nicht in allen ausgewählten Komponentenschemas identisch sind. Wenn Sie die Werte in solchen Feldern ändern, werden die neuen Werte einheitlich auf die gewählten Komponentenschemas angewendet.

    Hinweis: Die Standardwerte von Komponentenschemaparametern wie Hersteller, Treiber, Hostname und Port-Nummer sind abhängig von den Werten, die in den Anwendungsvorlagen festgelegt sind.

  2. Prüfen Sie die aktuellen Konfigurationseinstellungen, und ändern Sie sie nach Bedarf.

    Hinweis: Die Standardwerte von Schemaparametern wie Hersteller, Treiber, Hostname und Port-Nummer sind abhängig von den Werten, die in den Anwendungsvorlagen festgelegt sind.

    Feld Beschreibung
    Hersteller Wählen Sie den Datenbankhersteller aus.
    Treiber Wählen Sie den JDBC-Treiber aus, mit dem Sie sich bei der Datenbank anmelden wollen.

    Die Liste beinhaltet die JDBC-Treiber für den gewählten Datenbankhersteller.
    Schema-Eigentümer Geben Sie den Anmeldenamen für die Anmeldung bei der Datenbank ein.
    Schemakennwort Geben Sie das Anmeldekennwort für den angegebenen Schema-Eigentümer ein.
    DBMS/Service Geben Sie einen Datenbank-DBMS-Namen oder einen Service-Namen ein, wenn Sie einen Treiber für einen Service-Typ ausgewählt haben.
    Host-Name Geben Sie den Namen des Servers ein, auf dem sich die Datenbank befindet.
    Port Geben Sie die Port-Nummer ein, über die Sie sich beim Server anmelden wollen, auf dem sich die Datenbank befindet.

    Die von Ihnen angegebenen Werte werden für die gewählten Schemas in den entsprechenden Spalten in der Schemaliste angezeigt.

  3. Wenn Sie alle erforderlichen Einstellungen und Einträge vorgenommen haben, klicken Sie im Assistenten auf die Schaltfläche Weiter, um die Komponentenschemas zu testen.

Weitere Informationen finden Sie in der Dokumentations-Library für Oracle Fusion Middleware im Oracle Technology Network.
