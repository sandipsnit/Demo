JDBC-Datenquellen konfigurieren

Hier können Sie die JDBC-Datenquellen konfigurieren, die in der Domainquelle definiert sind.

Eine JDBC-Datenquelle enthält einen Pool mit Datenbankverbindungen, die bei der Erstellung der Datenquelleninstanz geschaffen werden, also wenn sie eingesetzt oder als Ziel definiert wird oder beim Serverstart. Anwendungen suchen eine Datenquelle aus dem JNDI-Baum heraus und fordern dann eine Verbindung an. Benötigen die Anwendungen die Verbindungen nicht länger, geben Sie die Verbindungen an den Connection Pool in der Datenquelle zurück.

In der unteren Bildschirmhälfte sind die JDBC-Datenquellen aufgelistet, die mit den Produkten verknüpft sind, für die Sie die Domain erstellen.

Hinweis: Wenn Sie eine oder mehrere der Datenquellen als RAC-Datenquellen konfigurieren möchten, aktivieren Sie die Kontrollkästchen neben dem Namen der jeweiligen Datenquelle, und aktivieren Sie das Kontrollkästchen Konfigurieren Sie gewählte Datenquellen als RAC-Multidatenquellen im nächsten Fensterbereich. Klicken Sie dann auf die Schaltfläche Weiter.

  1. Wählen Sie die Datenquelle aus, für die Sie Einstellungen vornehmen möchten, indem Sie das Kontrollkästchen neben dem Namen der Datenquelle aktivieren.

    Hinweis: Wenn Sie mehrere Datenquellen auswählen, wird möglicherweise der Text "Variiert unter Datenquellen" in bestimmten Feldern angezeigt. Dies weist darauf hin, dass die aktuellen Werte dieser Felder nicht in allen ausgewählten Datenquellen identisch sind. Wenn Sie die Werte in solchen Feldern ändern, werden die neuen Werte einheitlich auf die gewählten Datenquellen angewendet.

    Hinweis: Die Standardwerte von Datenquellenparametern wie Hersteller, Treiber, Hostname und Port-Nummer sind abhängig von den Werten, die in den Anwendungsvorlagen festgelegt sind.

  2. Prüfen Sie die aktuellen Konfigurationseinstellungen, und ändern Sie sie nach Bedarf.

    In der folgenden Tabelle werden die Felder auf diesem Bildschirm beschrieben.

    Hinweis: Die Standardwerte von Datenquellenparametern wie Hersteller, Treiber, Hostname und Port-Nummer sind abhängig von den Werten, die in den Anwendungsvorlagen festgelegt sind.

    Feld Beschreibung
    Hersteller Wählen Sie den Datenbankhersteller aus.
    Treiber Wählen Sie den JDBC-Treiber aus, mit dem Sie sich bei der Datenbank anmelden wollen.

    Die Liste beinhaltet die allgemeinen JDBC-Treiber für den gewählten Datenbankhersteller.
    Benutzername Geben Sie den Anmeldenamen für die Anmeldung bei der Datenbank ein.
    Kennwort Geben Sie das Anmeldekennwort für den angegebenen Benutzernamen ein.
    DBMS/Service Geben Sie eine DBMS-SID oder einen Service-Namen ein. Der eingegebene Wert hängt von dem ausgewählten Treiber ab.
    • Wenn der Name des ausgewählten Oracle Treibers die Wörter "für Instanzverbindungen" enthält, müssen Sie eine SID eingeben.
    • Wenn der Name des ausgewählten Oracle Treibers die Wörter "für Service-Verbindungen" enthält, müssen Sie einen Service-Namen eingeben.
    • Weitere Informationen zur Konfiguration von DataDirect-Treibern finden Sie in der DataDirect-Dokumentation.
    Host-Name Geben Sie den Namen des Servers ein, auf dem sich die Datenbank befindet.
    Port Geben Sie den Port ein, über den Sie sich beim Server anmelden wollen, auf dem sich die Datenbank befindet.

    Die von Ihnen angegebenen Werte werden für die gewählten Datenquellen in den entsprechenden Spalten in der Datenquellenliste angezeigt.

  3. Wenn Sie alle erforderlichen Einstellungen und Einträge vorgenommen haben, klicken Sie auf Weiter, um die Verbindungen zu den JDBC-Datenquellen zu testen.

Weitere Informationen finden Sie in der Dokumentations-Library für Oracle Fusion Middleware im Oracle Technology Network.
