JMS-Dateispeicher konfigurieren

Ein JMS-Speicher ist eine datei- oder datenbankbasierte Datei, in der persistente Nachrichten gespeichert werden können.

Prüfen Sie die aktuellen Einstellungen für JMS-Dateispeicher, und nehmen Sie nach Bedarf Änderungen vor. Die Standardwerte hängen von der zuvor gewählten Domain-Quelle ab.

Hinweis: Der Assistent bietet zwei Anzeigemodi: eine tabellarische Übersicht aller definierten Komponenten und eine Einzelansicht, in der jede Komponente in einer eigenen Registerkarte dargestellt wird. Sie können den Anzeigemodus zwischen Tabellen- und Registerkartenansicht hin- und herwechseln, indem Sie auf Anzeige umschalten klicken.

In der folgenden Tabelle werden die einzelnen Felder auf diesem Bildschirm beschrieben. Mit einem Sternchen (*) markierte Felder müssen ausgefüllt werden.

Feld Beschreibung
Name* Geben Sie einen Namen für den JMS-Dateispeicher ein.

Der Name muss eine Zeichenfolge sein und darf Leerzeichen beinhalten. Der Name des JMS-Dateispeichers muss unter sämtlichen Komponentennamen innerhalb der Domain eindeutig sein.
Verzeichnis Geben Sie den Pfad des Verzeichnisses (in Ihrem System) ein, in dem sich der JMS-Dateispeicher befindet.
Synchrone Speicher-Policy Wählen Sie eine der folgenden synchronen Speicher-Policys aus, um festzulegen, auf welche Weise der Dateispeicher Daten auf den Datenträger schreiben soll:
  • Cache-Flush: Transaktionen können nicht abgeschlossen werden, bevor all ihre Schreibvorgänge auf den Datenträger geleert worden sind.
  • Direct-Write: Schreibvorgänge werden direkt auf den Datenträger ausgeführt. Diese Policy wird auf Solaris und Windows unterstützt. Ist diese Policy auf einer nicht unterstützten Plattform aktiv, wechselt der Dateispeicher automatisch zur Policy "Cache-Flush".

    Hinweis: Die Policy "Direct-Write" ist in Solaris-Systemen verlässlich. Bei Windows-Systemen bleiben jedoch möglicherweise Transaktionsdaten im Datenträger-Cache zurück, ohne dass diese umgehend auf den Datenträger geschrieben werden. Dies gilt als unzuverlässig, da ein Stromausfall zu einem Verlust von Daten im Datenträger-Cache führen kann, der möglicherweise einen Verlust oder ein Duplizieren von Meldungen verursacht. Um mit der Policy "Direct-Write" auf Windows verlässliche Schreibvorgänge zu gewährleisten, deaktivieren Sie entweder sämtliche Schreib-Caches für den Datenträger (diese sind standardmäßig aktiviert), oder verwenden Sie einen Datenträger mit einem batteriegestützten Cache. Einige Dateisysteme lassen jedoch eine Änderung dieses Wertes nicht zu (Beispiel: ein RAID-System, welches einen verlässlichen Cache hat).

  • Deaktiviert: Transaktionen sind abgeschlossen, sobald die zu schreibenden Daten im Cache gespeichert sind. Ist diese Policy aktiv, wird mit der Beendigung von Transaktionen nicht gewartet, bis alle Daten auf Datenträger geschrieben sind.

Diese Einstellung hat Auswirkungen auf Performance, Skalierbarkeit und Verlässlichkeit.

Hinweis: Wird der JMS-Dateispeicher ausschließlich für das Paging nicht-persistenter Meldungen auf den Datenträger verwendet, wird die synchrone Speicher-Policy ignoriert.

Wenn Sie alle erforderlichen Einstellungen und Einträge vorgenommen haben, klicken Sie auf die Schaltfläche Weiter.
