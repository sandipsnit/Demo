Rechner konfigurieren

In einer WebLogic-Domain dienen Rechnerdefinitionen zur Identifizierung physikalischer Hardwareeinheiten und werden den von ihnen gehosteten WebLogic Server-Instanzen zugeordnet.

Hier können Sie Rechner hinzufügen oder löschen oder die Einstellungen für einen vorhandenen Rechner ändern.

Wählen Sie die Registerkarte Rechner (für Windows) bzw. die Registerkarte "Unix-Rechner" (für UNIX).

Sie können Rechnerdefinitionen in den folgenden Situationen erstellen:

  • Der Administrationsserver verwendet die Rechnerdefinitionen zusammen mit dem Node Manager, um Remote-Server zu starten.
  • WebLogic Server verwendet konfigurierte Rechnernamen, um in einem Cluster den Server festzulegen, der bestimmte Tasks, wie HTTP-Sessionreplikation, am besten verarbeiten kann. Anschließend delegiert WebLogic Server diese Tasks an den angegebenen Server.

Hinweis: Sie müssen Rechner für jede Produktinstallation konfigurieren, bei der ein Node Manager-Prozess ausgeführt wird. Die Rechnerkonfiguration muss Werte für die Listening-Adresse und Port-Nummer-Parameter enthalten.

Für jeden Rechner bestehen die folgenden Konfigurationseinstellungen. Pflichtfelder werden mit einem Sternchen (*) markiert.

  1. Wählen Sie die Registerkarte Rechner (für Windows) bzw. die Registerkarte Unix-Rechner (für UNIX).
  2. Prüfen Sie die aktuelle Liste der Konfigurationen, und fügen Sie Einstellungen wie für Ihre Domain erforderlich hinzu, bzw. löschen oder ändern Sie sie. In der folgenden Tabelle werden die Felder auf diesem Bildschirm beschrieben. Mit Sternchen (*) markierte Felder müssen ausgefüllt werden.

    Feld Beschreibung
    Name* Geben Sie einen gültigen Namen für den Rechner ein.

    Mit dem Rechnernamen wird der Rechner innerhalb der WebLogic Server-Domain identifiziert. Dieser Name muss nicht mit dem Netzwerknamen des Rechners übereinstimmen. Der Name muss unter allen Komponentennamen innerhalb der Domain eindeutig sein. Der Standardwert in diesem Feld ist new_[Unix]Machine_n, wobei n ein numerischer Wert ist, mit dem die einzelnen Standard-Rechnernamen unterschieden werden. Der Wert von n für den ersten Rechner ist 1. Für jeden weiteren Rechner wird der Wert um 1 erhöht.
    Listening-Adresse des Node Managers Wählen Sie aus der Liste einen Wert für die Adresse aus, die der Node Manager zur Überwachung von Verbindungsanforderungen verwenden soll.

    Standardmäßig werden in der Liste die für das lokale System definierten Adressen und localhost angezeigt. Der Standardwert ist localhost. Wenn Sie eine IP-Adresse für einen Rechner angeben, auf dem sich der Administrationsserver befindet, und Sie auf den Node Manager von WebLogic Server zugreifen müssen, müssen Sie die Verifikation des Hostnamens deaktivieren.

    Weitere Informationen finden Sie unter Hostnamenüberprüfung verwenden in Oracle WebLogic Server sichern.
    Listening-Port des Node Managers Geben Sie einen gültigen Wert für den Listening-Port ein, den der Node Manager auf Verbindungsanforderungen überwachen soll.

    Der Bereich gültiger Port-Nummern für den Listening-Port des Node Managers liegt zwischen 1 und 65534. Der Standardwert ist 5556.
    Post-Bind-GID aktiviert (Nur für UNIX-Rechner) Aktivieren Sie dieses Kontrollkästchen, damit ein auf diesem Rechner ausgeführter Server nach Abschluss aller privilegierten Startprozesse mit einer UNIX-Gruppen-ID (UID) verbunden werden kann (siehe unten die Beschreibung des Feldes Post-Bind-UID). Standardmäßig ist dieses Feld deaktiviert.
    Post-Bind-GID (Nur für UNIX-Rechner) Geben Sie eine gültige UNIX-Gruppen-ID (GID) ein, die ein auf diesem Rechner befindlicher Server nach Abschluss aller privilegierten Startprozesse ausführen soll. Ansonsten verwendet der Server weiterhin die Gruppen-ID, mit der er gestartet wurde.

    Diese Einstellung wird nur dann wirksam, wenn Sie das Kontrollkästchen Post-bind-GID aktiviert aktivieren.

    Post-Bind-UID aktiviert (Nur für UNIX-Rechner) Aktivieren Sie dieses Kontrollkästchen, damit ein auf diesem Rechner ausgeführter Server nach Abschluss aller privilegierten Startprozesse mit einer UNIX-Benutzer-ID (UID) verbunden werden kann (siehe unten die Beschreibung des Feldes Post-Bind-UID). Standardmäßig ist dieses Feld deaktiviert.
    Post-Bind-UID (Nur für UNIX-Rechner) Geben Sie eine gültige UNIX-Benutzer-ID (UID) ein, die ein auf diesem Rechner befindlicher Server nach Abschluss aller privilegierten Startprozesse ausführen soll. Ansonsten verwendet der Server weiterhin die Benutzer-ID, mit der er gestartet wurde.

    Diese Einstellung wird nur dann wirksam, wenn Sie das Kontrollkästchen Post-bind-UID aktiviert aktivieren.
  3. Wenn Sie alle erforderlichen Einstellungen und Einträge vorgenommen haben, klicken Sie auf die Schaltfläche Weiter.

Um einen neuen Rechner hinzuzufügen, klicken Sie auf Hinzufügen. Der Standardname für einen neuen Rechner lautet new_[Unix]Machine_n, wobei n bei 1 beginnt und sich für jeden weiteren Rechner um 1 erhöht.

Um einen vorhandenen Rechner zu löschen, klicken Sie auf Löschen.

Weitere Informationen finden Sie in der Dokumentations-Library für Oracle Fusion Middleware im Oracle Technology Network.
