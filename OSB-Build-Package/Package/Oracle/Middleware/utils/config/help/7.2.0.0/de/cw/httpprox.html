HTTP-Proxyanwendungen erstellen

Auf diesem Bildschirm können Sie eine HTTP-Proxyanwendung für jedes Cluster erstellen und den Managed Server festlegen, auf dem die Proxyanwendung bereitgestellt werden soll. Eine HTTP-Proxyanwendung dient als Zwischenstation für HTTP-Anforderungen.

Dieser Bildschirm wird nur angezeigt, wenn die beiden folgenden Anweisungen richtig sind:

  • Mindestens ein Managed Server ist einem Cluster zugeordnet.
  • Mindestens ein Managed Server ist keinem Cluster zugeordnet.
  1. Aktivieren Sie das Kontrollkästchen HTTP-Proxy erstellen neben dem Cluster, für das Sie die HTTP-Proxyanwendung erstellen wollen.

    Eine Liste der Managed Server, die keinem Cluster zugeordnet sind, wird in der Liste Proxyserver angezeigt.

  2. Wählen Sie aus der Liste Proxyserver einen Managed Server, auf dem die Proxyanwendung bereitgestellt werden soll.

    Daraufhin wird eine Proxyanwendung namens OracleProxy4_<clustername>_<servername> erstellt und auf dem gewählten Managed Server bereitgestellt.

  3. Wiederholen Sie die Schritte 1 und 2 für jedes Cluster, für das Sie HTTP-Proxyanwendungen erstellen wollen.
  4. Wenn Sie alle erforderlichen Einstellungen vorgenommen haben, klicken Sie auf die Schaltfläche Weiter.

Weitere Informationen finden Sie in der Dokumentations-Library für Oracle Fusion Middleware im Oracle Technology Network.
