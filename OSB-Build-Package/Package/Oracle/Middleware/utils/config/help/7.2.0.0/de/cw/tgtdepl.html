Ziel-Deployments auf Clustern oder Servern

Hier können Sie Anwendungen und Librarys für das Deployment auf Servern oder Clustern definieren.

Anwendungen und Librarys, die mit dem Produkt verknüpft sind, für das Sie die Domain konfigurieren (Beispiel: SOA), werden automatisch als Ziel-Deployments festgelegt: entweder auf dem Managed Server, der für dieses Produkt erstellt wurde, oder auf dem Cluster, dem dieser Managed Server zugewiesen ist. Auf diesem Bildschirm können Sie Anwendungen und Librarys für das Deployment auf zusätzlichen Server und Clustern festlegen.

  1. Wählen Sie im Bereich Ziel das Cluster oder den Server aus, auf dem Sie Anwendungen oder Librarys bereitstellen wollen.

    Der Name des gewählten Zieles wird als Titel des rechten Fensterbereiches angezeigt.

  2. Aktivieren Sie im rechten Fensterbereich die Kontrollkästchen der jeweiligen Anwendungen oder Librarys, für die ein Deployment auf das gewählte Ziel durchgeführt werden soll.

    Die hier angezeigten Anwendungen und Librarys hängen von den Produkten ab, die Sie auf dem Bildschirm Domainquelle wählen weiter vorne im Assistenten ausgewählt haben.

    Hinweis: Wenn Sie im Bereich Ziel einen Managed Server ausgewählt haben, sind einige der Kontrollkästchen im rechten Bereich möglicherweise deaktiviert. Dies weist darauf hin, dass die Anwendungen und Librarys bereits für das Deployment auf dem Cluster, das den Managed Server enthält, festgelegt wurden.

    Nachdem Sie Anwendungen und Librarys ausgewählt haben, erscheinen die Namen der als Ziel definierten Cluster und Server in der Spalte Ziel im rechten Fensterbereich.

  3. Wiederholen Sie die Schritte 1 und 2 nach Bedarf für weitere Cluster und Server.
  4. Wenn Sie alle erforderlichen Einstellungen vorgenommen haben, klicken Sie auf die Schaltfläche Weiter.

Wenn Sie eine Domain erweitern und einen Managed Server oder ein Cluster löschen, auf dem das Deployment von Anwendungen derzeit festgelegt ist, ordnet der Konfigurationsassistent automatisch die Ziel-Deployments dieser Anwendungen neu zu, und zwar folgendermaßen:

  • Wenn die Ziel-Deployments der Anwendungen ursprünglich nur für die Managed Server oder Cluster festgelegt waren, die nun gelöscht werden (das heißt, nach dem Löschen des Managed Servers oder Clusters hätte die Anwendung in der geänderten Domain kein Ziel), dann ordnet der Konfigurationsassistent automatisch die Anwendungen allen berechtigen Zielen neu zu.

    Berechtigte Ziele sind alle Cluster oder Managed Server, die nicht in den Konfigurationsgruppenspezifikationen (config-groups.xml) einer enthaltenen Vorlage definiert sind. In der Datei config-groups.xml angegebene Server oder Cluster gehören zu der Vorlage und sind daher nicht für automatische Ziel-Deployments berechtigt.

  • Falls die Deployments der Anwendungen ursprünglich für mehrere Ziele festgelegt waren (wie Managed Server, Cluster und Administrationsserver) und ein als Ziel definierter Managed Server oder Cluster gelöscht wird, werden die verbleibenden Zielverknüpfungen in der erweiterten Domain beibehalten. Der Konfigurationsassistent nimmt keine Neuzuordnung der Anwendungen vor.

Weitere Informationen finden Sie in der Dokumentations-Library für Oracle Fusion Middleware im Oracle Technology Network.
