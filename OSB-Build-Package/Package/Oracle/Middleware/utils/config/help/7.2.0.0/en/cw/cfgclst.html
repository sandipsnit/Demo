Configure Clusters

Use this screen to add, configure, and delete clusters.

A cluster is a group of WebLogic Server instances that work together to provide scalability and high-availability for applications. By creating clusters, you can group managed servers such that they operate as a single unit for hosting applications and resources.

  1. Review the current cluster configuration. Default values might vary based on the domain source you have selected.

    Note: The wizard provides two views: a concise tabular view of all the clusters, and an individual view of each cluster with each server represented by a tab. To toggle the view mode between the table and tab views, click Switch Display.

  2. Add or delete clusters, or change the settings for existing clusters, as required for your domain.

    The following table describes the fields in this screen. The fields marked with an asterisk (*) are mandatory fields.

    Field Description
    Name* Enter a valid name for the cluster.

    The name of the cluster must be unique among all component names within the domain. The default value in this field is new_Cluster_n, where n is a numeric value that is used to differentiate among all the default cluster names; the value of n for the first cluster is 1. The value is incremented by 1 for each cluster that you add.
    Cluster messaging mode Select unicast or multicast messaging mode.
    Multicast address This field is enabled only if you select the multicast messaging mode.

    Enter the multicast address for the cluster. This address is used by cluster members to communicate with each other. The default value is 239.192.0.0. The valid multicast address range is from 224.0.0.1 to 239.255.255.255.
    Multicast port This field is enabled only if you select the multicast messaging mode.

    Enter the multicast port for the cluster. The multicast port is used by cluster members to communicate with each other. The default value is 7001.
    Cluster address Enter the addresses to identify the managed servers in the cluster. A cluster address can be one of the following:
    • Comma-separated list of IP addresses or DNS names and ports (for example: dns_name:port, dns_name:port)
    • DNS name that maps to multiple IP addresses
    • localhost, DNS name, or IP address if the listen address of all managed servers is listening to the same address with unique port numbers

    The cluster address is used in entity and stateless EJBs to construct the host name portion of URLs. If the cluster address is not set, EJB handles might not work properly.
  3. After making the required selections and entries, click Next.

To delete a cluster, select the server and click Delete. When you delete a cluster, you do not delete the servers assigned to it; the servers are merely removed from the cluster and can then be added to another cluster. You can delete only one cluster at a time.

To add a cluster, click Add and configure the settings for the new cluster. The default name for a new cluster is new_Cluster_n, where n starts at 1 and increments for each new cluster you add. To change the default, type the desired name in the Name column

To switch the display from the default tabular format to a tabbed format (with one tab for each cluster), click Switch Display. Click Switch Display again to return to tabular format.
