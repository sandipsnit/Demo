Configure Machines

In a WebLogic domain, the machine definitions identify physical units of hardware and are associated with the WebLogic Server instances that they host.

Use this screen to add or delete machines, or to modify the settings for an existing machine.

Select the Machine tab (for Windows) or the UNIX Machine tab (for UNIX).

You might want to create machine definitions in situations such as the following:

  • The Administration Server uses the machine definition, with the Node Manager application, to start remote servers.
  • WebLogic Server uses configured machine names when determining the server in a cluster that is best able to handle certain tasks, such as HTTP session replication. WebLogic Server then delegates those tasks to the identified server.

Note: You must configure machines for each product installation that runs a Node Manager process. The machine configuration must include values for the listen address and port number parameters.

Each machine has the following configuration settings. Required fields are marked with an asterisk (*).

  1. Select the Machine tab (for Windows) and Unix Machine tab (for UNIX).
  2. Review the current list of configurations, and add, delete, or change settings as required for your domain. The following table describes the fields in this screen. The fields marked with an asterisk (*) are mandatory fields.

    Field Description
    Name* Enter a valid name for the machine.

    The machine name is used to identify the machine within the WebLogic Server domain; it is not required to match the network name for the machine. The name must be unique among all component names within the domain. The default value in this field is new_[Unix]Machine_n, where n is a numeric value that is used to differentiate among all default machine names; the value of n for the first machine is 1. The value is incremented by 1 for each machine that you add.
    Node manager listen address Select a value from the list, for the address used by node manager to listen for connection requests.

    By default, the IP addresses defined for the local system and localhost are shown in the list. The default value is localhost. If you specify an IP address for a machine that hosts the administration server and you need to access the WebLogic Server node manager, then you must disable host name verification.

    For more information, see Using Host Name Verification in Securing Oracle WebLogic Server
    Node manager listen port Enter a valid value for the listen port used by node manager to listen for connection requests.

    The valid port number range for the listen port of the node manager is from 1 to 65534. The default value is 5556.
    Post bind GID enabled (UNIX machines only) Select this check box to enable a server running on this machine to bind to a UNIX group ID (GID) after it finishes all privileged startup actions (see the description of the Post bind GID field below). By default, this check box is not selected.
    Post bind GID (UNIX machines only) Enter a valid UNIX group ID (GID), using which a server hosted on this machine will run after it finishes all privileged startup actions. Otherwise, the server continues to run by using the group ID with which it was started.

    For this setting to take effect, you must select the Post-bind GID enabled check box.

    Post bind UID enabled (UNIX machines only) Select this check box to enable a server hosted on this machine to bind to a UNIX user ID (UID) after it finishes all privileged startup actions (see the description of the Post bind UID field below). By default, this check box is not selected.
    Post bind UID (UNIX machines only) Enter a valid UNIX user ID (UID), using which a server hosted on this machine will run after it finishes all privileged startup actions. Otherwise, the server continues to run by using the user ID with which it was started.

    For this setting to take effect, you must select the Post-bind UID enabled check box.
  3. After making the required selections and entries, click Next.

Click Add to add a new machine. The default name for a new machine is new_[Unix]Machine_n, where n starts at 1 and increments by 1 for each machine that you add.

Click Delete to delete an existing machine.
