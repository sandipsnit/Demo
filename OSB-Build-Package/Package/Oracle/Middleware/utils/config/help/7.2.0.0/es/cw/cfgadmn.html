Configurar Servidor de Administración

El servidor de administración es el punto central desde el que se gestiona el dominio. Para acceder al servidor de administración, utilice la URLprotocolo://dirección-recepción:puerto-recepción de la URL. El protocolo puede ser cualquiera de los siguientes: t3, t3s, http, https.

Utilice esta pantalla para configurar o cambiar los siguientes valores del servidor de administración. Los campos obligatorios están marcados con un asterisco (*).

Campo Descripción
*Nombre El nombre por defecto es AdminServer. Los nombres de servidor válidos son una cadena de caracteres (alfabéticos y numéricos).

Cada instancia de servidor del entorno de producción debe tener un nombre único con independencia del dominio o cluster en el que reside y con independencia de si se trata de un servidor de administración o un servidor gestionado. Además, el nombre del servidor de administración debe ser único entre todos los nombres de componentes del dominio.

Nota: Este valor se especifica únicamente con fines de identificación. No se utiliza como parte de la dirección URL para las aplicaciones desplegadas en el servidor. El nombre del servidor se muestra en la Consola de Administración de WebLogic Server. Además, si utiliza utilidades de línea de comandos de WebLogic Server o API, debe especificar este nombre para identificar el servidor.

*Dirección de Recepción En la lista desplegable, seleccione un valor para la dirección de recepción.

Si selecciona localhost como la dirección de recepción de una instancia de servidor, los procesos remotos no se pueden conectar a dicha instancia de servidor. Solo los procesos de la máquina que aloja la instancia de servidor se pueden conectar a esta instancia. Si la instancia de servidor debe ser accesible como un host local (por ejemplo, si crea scripts administrativos que se conectan al host local) y también debe ser accesible para los procesos remotos, seleccione Todas las Direcciones Locales. La instancia de servidor determina la dirección de la máquina y recibe en ella.

Consulte Especificación de la Dirección de Recepción para obtener más información.
Puerto de Recepción Introduzca un valor válido para el puerto de recepción que se va a utilizar para las solicitudes normales no seguras (mediante protocolos como HTTP y T3).

El valor por defecto es 7001 para el servidor de administración. Si deja este campo en blanco, se utiliza el valor por defecto.

El rango de puertos de recepción válido está comprendido entre 1 y 65534.

Consulte Especificación del Puerto de Recepción para obtener más información.
SSL Activado Active esta casilla de control para activar el puerto de recepción SSL. Por defecto, SSL está desactivado para todos los servidores nuevos.
Puerto de Recepción SSL Este campo se activa únicamente si la casilla de control SSL Activado está activada.

Introduzca un valor válido para las solicitudes seguras (mediante protocolos como HTTPS y T3S). El valor por defecto es 7002. Si deja este campo en blanco, se utiliza el valor por defecto.

El rango de puertos de recepción válido está comprendido entre 1 y 65535.

Nota: Por defecto, una instancia de servidor utiliza certificados de demostración para autenticar las solicitudes desde un puerto seguro. En un entorno de producción, debe configurar SSL para utilizar los certificados de una autoridad de certificación.

Después de realizar las selecciones y entradas requeridas, haga clic en Siguiente.

Para obtener más información, consulte la biblioteca de documentación de Oracle Fusion Middleware en Oracle Technology Network.

Especificación de la Dirección de Recepción

Esta sección proporciona las instrucciones para especificar la dirección de recepción para el servidor de administración. Estas directrices también se aplican a todos los servidores gestionados.

Tipo de Dirección de Recepción Comportamiento
Todas las Direcciones Locales o Nombre DNS En máquinas de Windows de varios directorios raíz, una instancia de servidor se enlaza a todas las direcciones IP disponibles.
Dirección IP o Nombre DNS
  • Para conectarse a la instancia del servidor, los procesos pueden especificar la dirección IP o el nombre DNS correspondiente.
  • Los procesos que especifican el host local no pueden conectarse.
  • Debe actualizar los procesos existentes que utilizan el host local para conectarse a la instancia del servidor.
  • Para las conexiones que especifican la dirección IP de la dirección de recepción y un puerto seguro para el puerto de recepción, se debe desactivar la verificación de nombres de host.

Nota:Para resolver un nombre DNS en una dirección IP, WebLogic Server debe poder contactar con un servidor DNS adecuado u obtener la asignación de dirección IP de forma local. Por lo tanto, si especifica un nombre DNS para la dirección de recepción, debe dejar un puerto abierto el tiempo suficiente para que la instancia de WebLogic Server conecte con un servidor DNS y almacene en caché su asignación, o bien debe especificar la asignación de dirección IP en un archivo local. Si especifica una dirección IP para la dirección de recepción y, a continuación, una solicitud de cliente especifica un nombre DNS, WebLogic Server intentará resolver el nombre DNS. Si no puede acceder a la asignación de nombre DNS, la solicitud fallará.
Host local
  • Los procesos deben especificar el host local para conectarse a la instancia del servidor.
  • Los procesos que residen en la máquina que aloja la instancia del servidor (procesos locales) son los únicos que pueden conectarse a la instancia del servidor.

Especificación del Puerto de Recepción

Tenga en cuenta las siguientes directrices a la hora de especificar el puerto de recepción del servidor de administración. Estas directrices también se aplican a los servidores gestionados.

  • Aunque puede especificar cualquier número de puerto válido, si especifica el puerto 80, puede omitir el número de puerto en la solicitud HTTP que se utiliza para acceder a los recursos sobre HTTP. Por ejemplo, si define el puerto 80 como el puerto de recepción, puede utilizar la URL http://nombrehost:númeropuerto/miarchivo.html.
  • En algunos sistemas operativos, al puerto 80 sólo pueden acceder los procesos que se ejecutan con un usuario o identificador de grupo con privilegios. En este caso, puede asignar la instancia del servidor a una máquina UNIX en la cual hay definido un GID o UID posterior a enlace.
