Configurar Clusters

Utilice esta pantalla para agregar, configurar y suprimir clusters.

Un cluster es un grupo de instancias de WebLogic Server que funcionan de forma conjunta para proporcionar escalabilidad y una alta disponibilidad para las aplicaciones. La creación de clusters permite agrupar servidores gestionados de modo que funcionen como una sola unidad para alojar aplicaciones y recursos.

  1. Revise la configuración de clusters actual. Los valores por defecto pueden variar según el origen de dominio seleccionado.

    Nota: El asistente proporciona dos vistas (una vista tabular concisa de todos los clusters y una vista individual de cada cluster con cada servidor representado por un separador). Para alternar entre las vistas de tabla y separador, haga clic en Cambiar Visualización.

  2. Agregue o suprima clusters, o cambie la configuración de los clusters existentes, según sea necesario para el dominio.

    En la siguiente tabla se describen los campos de esta pantalla. Los campos marcados con un asterisco (*) son obligatorios.

    Campo Descripción
    Nombre* Introduzca un nombre válido para el cluster.

    El nombre del cluster debe ser único entre todos los nombres de componentes del dominio. El valor por defecto de este campo es new_Cluster_n, donde n es un valor numérico que se utiliza para diferenciar todos los nombres de cluster por defecto; el valor de n para el primer cluster es 1. El valor se incrementa en 1 para cada cluster agregado.
    Modo de Mensajes de Cluster Seleccione el modo de mensajes de unidifusión o multidifusión.
    Dirección de Multidifusión Este campo se activa únicamente si selecciona el modo de mensajes de multidifusión.

    Introduzca la dirección de multidifusión para el cluster. Esta dirección la utilizan los miembros del cluster para comunicarse entre sí. El valor por defecto es 239.192.0.0. El rango de direcciones de multidifusión válido está comprendido entre 224.0.0.1 y 239.255.255.255.
    Puerto de Multidifusión Este campo se activa únicamente si selecciona el modo de mensajes de multidifusión.

    Introduzca el puerto de multidifusión para el cluster. El puerto de multidifusión lo utilizan los miembros del cluster para comunicarse entre sí. El valor por defecto es 7001.
    Dirección de Cluster Introduzca las direcciones para identificar los servidores gestionados del cluster. Una dirección de cluster puede ser:
    • Una lista separada por comas de direcciones IP o nombres DNS y puertos (por ejemplo, nombre_dns:puerto, nombre_dns:puerto).
    • Un nombre DNS asignado a varias direcciones IP.
    • Un host local, un nombre DNS o una dirección IP si la dirección de recepción de todos los servidores gestionados recibe en la misma dirección con números de puerto únicos.

    La dirección de recepción se utiliza en EJB de entidades y sin estado para crear la parte del nombre del host de las direcciones URL. Si no se ha establecido la dirección del cluster, es posible que los manejadores de EJB no funcionen correctamente.
  3. Después de realizar las selecciones y entradas requeridas, haga clic en Siguiente.

Para suprimir un cluster, seleccione el servidor y haga clic en Suprimir. Cuando se suprime un servidor, no se suprimen los servidores asignados a él; estos simplemente se eliminan del cluster y luego se pueden agregar a otro cluster. Sólo se puede eliminar un cluster a la vez.

Para agregar un cluster, haga clic en Agregar y configure los valotes para el nuevo servidor. El nombre por defecto del cluster nuevo es new_Cluster_n, donde n comienza en 1 y se incrementa con cada cluster que se agrega. Para cambiar el nombre por defecto, escriba el nombre que desea en la columna Nombre.

Para cambiar la visualización de formato tabular por defecto (por un separador por cada cluster), haga clic en Cambiar Visualización. Vuelva a hacer clic en Cambiar Visualización para regresar al formato tabular.
