Configurar Esquema de Componentes de JDBC

Para algunos componentes de Fusion Middleware (por ejemplo, SOA y Web Center Spaces), puede que haya que definir orígenes de datos JDBC como parte del esquema de base de datos del componente, que se carga (durante la instalación ) mediante la Utilidad de Creación de Repositorios (RCU).

Al crear un dominio de WebLogic para dichos componentes mediante el Asistente de Configuración, puede configurar los valores del esquema del componente JDBC (por ejemplo, el controlador de la base de datos, el propietario del esquema, la contraseña, etc).

Los esquemas de componentes de JDBC asociados a los productos para los que se va a crear el dominio (por ejemplo, SOA) se muestran en la mitad inferior de la pantalla.

Nota: Si desea configurar uno o más esquemas como orígenes de datos de RAC, active las casillas de control junto al nombre de los esquemas requeridos y active la casilla de control Configure los esquemas de componentes seleccionados como esquemas de orígenes de datos múltiples de RAC en el panel siguiente. A continuación, haga clic en el botón Siguiente.

  1. Para seleccionar los esquemas para los que desea especificar la configuración de orígenes de datos, active la casilla de control junto a los nombres de los esquemas.

    Nota: Si se seleccionan varios esquemas de componentes, es posible que se muestre el texto “Varía entre los esquemas de componentes” en determinados campos, lo que indica que los valores actuales de estos campos no son los mismos en los componentes de esquemas seleccionados. Si continúa y cambia los valores de estos campos, los valores nuevos se aplican de manera uniforme a los esquemas de componentes seleccionados.

    Nota: Los valores por defecto de los parámetros del esquema del componente (por ejemplo, proveedor, controlador, nombre del host y número de puerto) dependen de los valores especificados en las plantillas de aplicación.

  2. Revise los valores de configuración actuales y modifíquelos según sea necesario.

    Nota: Los valores por defecto de los parámetros de esquema (por ejemplo, proveedor, controlador, nombre del host y número de puerto) dependen de los valores especificados en las plantillas de aplicación.

    Campo Descripción
    Proveedor Seleccione el proveedor de bases de datos.
    Controlador Seleccione el controlador JDBC que desee utilizar para conectarse a la base de datos.

    En la lista se incluyen los controladores JDBC para el proveedor de bases de datos seleccionado.
    Propietario del Esquema Introduzca el nombre de conexión para conectarse a la base de datos.
    Contraseña del Esquema Introduzca la contraseña de conexión para el propietario del esquema seleccionado.
    DBMS/Servicio Introduzca un nombre de DBMS de base de datos o un nombre de servicio si ha seleccionado un controlador de servicio.
    Nombre del Host Introduzca el nombre del servidor que aloja la base de datos.
    Puerto Introduzca el número de puerto que se va a utilizar para establecer la conexión con el servidor que aloja la base de datos.

    Los valores especificados se muestran en las columnas correspondientes de la lista de esquemas para los esquemas seleccionados.

  3. Después de realizar las selecciones y entradas requeridas, haga clic en el botón Siguiente del asistente para probar los esquemas de componentes.
