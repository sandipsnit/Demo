Configurar Orígenes de Datos JDBC

Utilice esta pantalla para configurar los orígenes de datos JDBC definidos en el origen del dominio.

Un origen de datos JDBC contiene un pool de conexiones de base de datos establecidas al crear la instancia del origen de datos (al desplegarla o dirigirla, o al iniciar el servidor). Las aplicaciones consultan un origen de datos en el árbol de JNDI y, a continuación, solicitan una conexión. Si las aplicaciones ya no necesitan las conexiones, las devuelven al pool de conexiones del origen de datos.

Los orígenes de datos JDBC asociados a los productos para los que se va crear el dominio se muestran en la mitad inferior de la pantalla.

Nota: Si desea configurar uno o más orígenes de datos como orígenes de datos de RAC, active las casillas de control junto al nombre de los orígenes de datos requeridos y active la casilla de control Configure los orígenes de datos seleccionados como orígenes de datos múltiples de RAC en el panel siguiente. A continuación, haga clic en el botón Siguiente.

  1. Para seleccionar el origen de datos para el que desea especificar la configuración, active la casilla de control junto al nombre del origen de datos.

    Nota: Si se seleccionan varios orígenes de datos, es posible que se muestre el texto “Varía entre los orígenes de datos” en determinados campos, lo que indica que los valores actuales de estos campos no son los mismos en los orígenes de datos seleccionados. Si continúa y cambia los valores de estos campos, los valores nuevos se aplican de manera uniforme a los orígenes de datos seleccionados.

    Nota: Los valores por defecto de los parámetros de origen de datos (por ejemplo, proveedor, controlador, nombre del host y número de puerto) dependen de los valores especificados en las plantillas de aplicación.

  2. Revise los valores de configuración actuales y cámbielos según sea necesario.

    En la siguiente tabla se describen los campos de esta pantalla.

    Nota: Los valores por defecto de los parámetros de origen de datos (por ejemplo, proveedor, controlador, nombre del host y número de puerto) dependen de los valores especificados en las plantillas de aplicación.

    Campo Descripción
    Proveedor Seleccione el proveedor de bases de datos.
    Controlador Seleccione el controlador JDBC que desee utilizar para conectarse a la base de datos.

    En la lista se incluyen los controladores JDBC comunes para el proveedor de bases de datos seleccionado.
    Usuario Introduzca el nombre de conexión para conectarse a la base de datos.
    Contraseña Introduzca la contraseña de conexión para el usuario especificado.
    DBMS/Servicio Introduzca un nombre de servicio o SID de DBMS. El valor que introduzca depende del controlador que haya seleccionado.
    • Si el nombre del controlador de Oracle que ha seleccionado contiene las palabras "para conexiones de instancia", debe introducir el SID.
    • Si el nombre del controlador de Oracle contiene las palabras "para conexiones de servicio", debe introducir el nombre del servicio.
    • Para obtener más información acerca de la configuración de un controlador de DataDirect, consulte la documentación de DataDirect.
    Nombre del Host Introduzca el nombre del servidor que aloja la base de datos.
    Puerto Introduzca el puerto que se va a utilizar para establecer la conexión con el servidor que aloja la base de datos.

    Los valores especificados se muestran en las columnas correspondientes de la lista de orígenes de datos para los orígenes de datos seleccionados.

  3. Después de realizar las selecciones y entradas requeridas, haga clic en Siguiente para probar las conexiones a los orígenes de datos JDBC.

Para obtener más información, consulte la biblioteca de documentación de Oracle Fusion Middleware en Oracle Technology Network.
