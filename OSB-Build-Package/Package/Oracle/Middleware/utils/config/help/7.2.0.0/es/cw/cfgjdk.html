Configurar Modo de Inicio de Servidor y JDK

En esta pantalla, especifique si desea ejecutar WebLogic en modo de producción o de desarrollo. También se especifica la ubicación de los JDK que se utilizan en este dominio.

  1. Seleccione el modo de inicio del dominio de WebLogic.
    • Modo de Desarrollo
      En este modo, la configuración de seguridad es relativamente fácil, lo que le permite desplegar las aplicaciones automáticamente.
    • Modo de Producción
      En este modo, la configuración de seguridad es relativamente estricta, lo que requiere el uso de un nombre de usuario y una contraseña para desplegar aplicaciones. Antes de poner en producción un dominio, familiarícese con los procedimientos para proteger un entorno de producción. Para obtener más información, consulte Cómo Garantizar la Seguridad del Entorno de Producción en Protección de un Entorno de Producción para Oracle WebLogic Server.

    En la siguiente tabla se proporciona información para facilitar la selección del modo de inicio que mejor se adapte a sus necesidades.

    Función Modo de Desarrollo Modo de Producción
    SSL Puede usar los certificados digitales y los almacenes de claves de demostración proporcionados por los servicios de seguridad de WebLogic Server. Con estos certificados, puede diseñar la aplicación para que funcione en entornos protegidos por SSL.

    Para obtener más información, consulte Configuración de SSL en Protección de Oracle WebLogic Server.

    		 No debe usar los certificados digitales ni los almacenes de claves de demostración. En caso contrario, se mostrará un mensaje de advertencia.
    Despliegue de Aplicaciones Las instancias de WebLogic Server pueden desplegar y actualizar automáticamente aplicaciones que residen en el directorio nombre_dominio/autodeploy.

    Oracle sólo recomienda utilizar este método en un entorno de desarrollo de un solo servidor.

    Para obtener más información, consulte Despliegue de Aplicaciones y Módulos con weblogic.deployer en Despliegue de Aplicaciones en Oracle WebLogic Server.

    		 La función de despliegue automático está desactivada; por lo tanto, debe utilizar la Consola de Administración de WebLogic Server, la herramienta weblogic.Deployer o la herramienta de generación de scripts de WebLogic.
    Rotación del Archivo Log Por defecto, al iniciar la instancia de WebLogic Server, el servidor cambia automáticamente el nombre (rota) del archivo log del servidor local a SERVER-NAME.log.n. Durante el resto de la sesión del servidor, los mensajes se acumulan en el archivo log hasta alcanzar un tamaño de 500 KB.

    El valor por defecto de Limitar Número de Archivos Retenidos en Configuración de Registro es true. Este valor limita el número de archivos de registro que crea la instancia de servidor para almacenar mensajes antiguos.

    		 El servidor rota el archivo log local cuando alcanza un tamaño de 5.000 KB. Si el servidor está configurado para el modo de producción, se conservan todas las versiones de los archivos log por defecto. Es posible que los administradores deseen personalizar el número de archivos log conservados.
    Recurso del Sistema de JDBC La capacidad por defecto es de 15 conexiones. La capacidad por defecto es de 25 conexiones.
  2. Seleccione el JDK.

    En el panel Selección de JDK, seleccione el JDK para el modo de inicio elegido en el panel Modo de Inicio de Dominio de WebLogic. El Asistente de Configuración muestra una lista de los JDK incluidos en el instalador. Puede seleccionar estos JDK o un JDK instalado en el sistema.

    Notas:

    • Si selecciona un JDK incluido en el instalador, el Asistente de Configuración crea scripts de inicio del servidor para llamar al JDK. Si selecciona un JDK no suministrado por JDK, el Asistente de Configuración no configura los scripts de inicio y debe cambiarlos manualmente.
    • Seleccione solo los JDK admitidos en la plataforma que se va a utilizar. La selección por defecto refleja el JDK que mejor se adapta a las necesidades del entorno según la plataforma en la que se va a instalar el dominio.
  3. Después de realizar las selecciones y entradas requeridas, haga clic en el botón Siguiente del asistente.

Para obtener más información, consulte la biblioteca de documentación de Oracle Fusion Middleware en Oracle Technology Network.
