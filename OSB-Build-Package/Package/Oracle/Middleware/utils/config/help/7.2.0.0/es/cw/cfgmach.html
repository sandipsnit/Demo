Configurar Máquinas

En un dominio de WebLogic, las definiciones de máquina identifican unidades físicas de hardware y están asociadas a las instancias de WebLogic Server que alojan.

Utilice esta pantalla para agregar o suprimir máquinas o para modificar la configuración de una máquina existente.

Seleccione el separador Máquina (para Windows) o el separador Máquina UNIX (para UNIX).

Puede que desee craer definiciones de máquinas en situaciones como la siguiente:

  • El servidor de administración utiliza la definición de máquina, con la aplicación Gestor de Nodos, para iniciar servidores remotos.
  • WebLogic Server utiliza nombres de máquina configurados para determinar el servidor de un cluster que está más capacitado para manejar determinadas tareas, como la replicación de sesiones HTTP. WebLogic Server delega esas tareas en el servidor identificado.

Nota: Debe configurar las máquinas para cada instalación del producto que ejecute un proceso del Gestor de Nodos. La configuración de la máquina debe incluir valores para los parámetros de la dirección de recepción y el número de puerto.

Cada máquina tiene la siguiente configuración. Los campos obligatorios están marcados con un asterisco (*).

  1. Seleccione el separador Máquina (para Windows) y el separador Máquina Unix (para UNIX).
  2. Revise la lista de configuraciones actual y agregue, suprima o cambie la configuración según sea necesario para el dominio. En la siguiente tabla se describen los campos de esta pantalla. Los campos marcados con un asterisco (*) son obligatorios.

    Campo Descripción
    Nombre* Introduzca un nombre válido para la máquina.

    El nombre de la máquina se utiliza para identificar la máquina del dominio de WebLogic Server y no es necesario que coincida con el nombre de red de la máquina. Este nombre debe ser único entre todos los nombres de componentes del dominio. El valor por defecto de este campo es new_[Unix]Machine_n, donde n es un valor numérico utilizado para diferenciar todos los nombres de máquina por defecto; el valor de n de la primera máquina es 1. Este valor se incrementa en 1 para cada máquina agregada.
    Dirección de Recepción del Gestor de Nodos Seleccione un valor de la lista para la dirección utilizada por el gestor de nodos para recibir las solicitudes de conexión.

    Por defecto, las direcciones IP definidas para el sistema local y localhost se muestran en la lista. El valor por defecto es localhost. Si especifica una dirección IP para una máquina que aloja el servidor de administración y necesita acceder al gestor de nodos de WebLogic Server, debe desactivar la verificación de nombre de host.

    Para obtener más información, consulte Uso de la Verificación del Nombre de Host en Protección de Oracle WebLogic Server.
    Puerto de Recepción del Gestor de Nodos Introduzca un valor válido para el puerto de recepción utilizado por el gestor de nodos para recibir las solicitudes de conexión.

    El rango de números de puerto válido para el puerto de recepción del gestor de nodos está comprendido entre 1 y 65534. El valor por defecto es 5556.
    Activar GID Posterior a Enlace (Sólo máquinas UNIX) Active esta casilla de control para activar un servidor que se ejecute en esta máquina para establecer un enlace con un identificador de grupo (GID) UNIX una vez finalizadas todas las acciones de inicio con privilegios (consulte la descripción del campo GID Posterior a Enlace que se incluye a continuación). Por defecto, esta casilla de control no está activada.
    GID Posterior a Enlace (Sólo máquinas UNIX) Introduzca un identificador de grupo (GID) UNIX válido para ejecutar un servidor alojado en esta máquina una vez finalizadas todas las acciones de inicio con privilegios. En caso contrario, el servidor sigue ejecutándose con el identificador de grupo con el que se inició.

    Para aplicar esta configuración, debe activar la casilla de control Activar GID Posterior a Enlace.

    Activar UID Posterior a Enlace (Sólo máquinas UNIX) Active esta casilla de control para activar un servidor alojado en esta máquina para establecer un enlace con un identificador de usuario (UID) UNIX una vez finalizadas todas las acciones de inicio con privilegios (consulte la descripción del campo UID Posterior a Enlace que se incluye a continuación). Por defecto, esta casilla de control no está activada.
    UID Posterior a Enlace (Sólo máquinas UNIX) Introduzca un identificador de usuario (UID) UNIX válido para ejecutar un servidor alojado en esta máquina una vez finalizadas todas las acciones de inicio con privilegios. En caso contrario, el servidor sigue ejecutándose con el identificador de usuario con el que se inició.

    Para aplicar esta configuración, debe activar la casilla de control Activar UID Posterior a Enlace.
  3. Después de realizar las selecciones y entradas requeridas, haga clic en Siguiente.

Haga clic en Agregar para agregar una máquina nueva. El nombre por defecto de la nueva máquina es new_[Unix]Machine_n, donde n comienza en 1 y se incrementa en 1 por cada máquina que se agrega.

Haga clic Suprimir para suprimir una máquina existente.
