Ejecutar Archivos de Comandos de Base de Datos

La plantilla de dominio puede contener un juego de archivos SQL organizados por tipo de base de datos. Si la plantilla de dominio contiene archivos SQL, puede ejecutarlos mientras crea el dominio de WebLogic en la pantalla Ejecutar Scripts de Base de Datos. El contenido de la base de datos de cada uno de los orígenes de datos definidos en el dominio de WebLogic se configura mediante los archivos de carga de base de datos o SQL preexistentes.

El contenido de la base de datos de cada uno de los orígenes de datos definidos en el dominio se configura mediante los archivos SQL preexistentes o los archivos de carga de base de datos.

  1. En el panel Orígenes de Datos JDBC Disponibles, seleccione el origen de datos para el que desee ejecutar los scripts.

    Los scripts que se pueden ejecutar se muestran en el panel Archivos SQL y Opciones de Carga de Base de Datos Disponibles.

  2. Seleccione la versión de la base de datos en la lista Versión de la Base de Datos.
  3. Haga clic en Ejecutar Archivos de Comandos.

    Se ejecutan todos los scripts mostrados en el panel Archivos SQL y Opciones de Carga de Base de Datos Disponibles para el origen de datos seleccionado y los resultados se muestran en el panel Resultados.

    Si desea capturar la salida de la prueba en un archivo log, active la casilla de control Archivo Log y especifique la ubicación de este archivo.

  4. Repita los pasos del 1 al 3 para cada origen de datos para el que desee ejecutar scripts de base de datos.
  5. Después de ejecutar los scripts de base de datos, haga clic en Siguiente.

Para obtener más información, consulte la biblioteca de documentación de Oracle Fusion Middleware en Oracle Technology Network.
