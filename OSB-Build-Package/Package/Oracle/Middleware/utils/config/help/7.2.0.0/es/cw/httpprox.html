Crear Aplicaciones de Proxy HTTP

Utilice esta pantalla para crear una aplicación de proxy HTTP para cada cluster y especificar el servidor gestionado en el que se debe desplegar la aplicación de proxy. La aplicación de proxy HTTP funciona como un intermediario para las solicitudes HTTP.

Esta pantalla sólo aparece si se cumplen las dos siguientes condiciones:

  • Se ha asignado al menos un servidor gestionado a un cluster.
  • No se ha asignado al menos un servidor gestionado a un cluster.
  1. Active la casilla de control Crear Proxy HTTP para el cluster para el que desee crear la aplicación de proxy HTTP.

    Se muestra una lista de los servidores gestionados no asignados a ningún cluster en la lista Servidor Proxy.

  2. En la lista Servidor Proxy, seleccione el servidor gestionado en el que se debe desplegar la aplicación de proxy.

    Se crea y despliega una aplicación de proxy llamada OracleProxy4_<nombreDeCluster>_<nombreDeServidor> en el servidor gestionado seleccionado.

  3. Repita los pasos 1 y 2 para cada cluster para el que desee crear aplicaciones de proxy HTTP.
  4. Después de realizar las selecciones requeridas, haga clic en Siguiente.
