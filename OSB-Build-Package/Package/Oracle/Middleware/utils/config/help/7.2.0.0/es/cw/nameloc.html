Especificar Nombre y Ubicación de Dominio

Especifique los siguientes elementos.

  • Nombre de Dominio
    Nombre del dominio. El Asistente de Configuración almacena en este directorio, bajo el directorio Dominio especificado, el archivo config.xml y todos los demás componentes generados para el dominio. Por ejemplo, si introduce midominio, sus archivos de dominio se almacenan (por defecto) en MW_HOME\user_projects\domains\midominio.

    Nota: El nombre de dominio no debe comenzar con un número. Esta restricción tiene como finalidad evitar potenciales problemas con nombres de tablas de almacenes de JDBC generados de manera interna, que deben comenzar por un carácter alfabético.

  • Ubicación de Dominio
    Ubicación en la que se almacenará el directorio del dominio. La ubicación por defecto es MW_HOME\user_projects\domains. La ubicación del dominio puede ser cualquiera de la unidad local o la red. No es necesario que su ubicación sea MW_HOME.

    En Windows, debe incluir la letra de la unidad en esta ruta de acceso.

  • Ubicación de la Aplicación
    Este campo sólo se muestra si ha seleccionado una plantilla de dominio o un producto que contiene aplicaciones. Ésta es la ubicación en la que residirán sus aplicaciones. La ubicación por defecto es MW_HOME\user_projects\applications.

    En Windows, debe incluir la letra de la unidad en esta ruta de acceso.

Después de realizar las selecciones y entradas requeridas, haga clic en Siguiente.
