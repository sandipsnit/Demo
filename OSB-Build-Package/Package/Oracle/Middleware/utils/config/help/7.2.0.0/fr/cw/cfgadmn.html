Configurer le serveur d'administration

Le serveur d'administration centralise la gestion de votre domaine. Vous pouvez accéder à ce serveur à l'aide de l'URL protocol://listen-address:listen-port. Il peut s'agir de n'importe lequel des formats suivants : t3, t3s, http, https.

Utilisez cet écran pour configurer ou modifier les paramètres ci-après du serveur d'administration. Les champs obligatoires sont signalés par un astérisque (*).

Champ Description
*Nom Le nom par défaut est AdminServer. Les noms de serveur valides consistent dans une chaîne de caractères (alphabétiques et numériques).

Chaque instance de serveur de l'environnement de production doit avoir un nom unique, quel que soit le domaine ou le cluster dans lequel elle réside, et qu'il s'agisse d'un serveur d'administration ou d'un serveur géré. Le nom du serveur d'administration doit en outre être unique parmi tous les noms de composant du domaine.

Remarque : cette valeur est fournie à des fins d'identification uniquement. Elle n'est pas utilisée comme composant de l'URL pour les applications déployées sur le serveur. Le nom du serveur est affiché dans la console d'administration WebLogic Server. En outre, si vous employez des utilitaires de ligne de commande WebLogic Server ou des API, vous devez indiquer ce nom pour identifier le serveur.

*Adresse d'écoute Dans la liste déroulante, sélectionnez une valeur pour l'adresse d'écoute.

Si vous sélectionnez l'adresse d'écoute localhost pour une instance de serveur, les processus distants ne peuvent pas se connecter à cette instance. Seuls les processus exécutés sur l'ordinateur qui héberge l'instance peuvent s'y connecter. Si l'instance de serveur doit être accessible en tant qu'hôte local (par exemple, si vous créez des scripts d'administration qui se connectent à l'hôte local) et accessible par les processus distants, sélectionnez Toutes les adresses locales. L'instance de serveur détermine l'adresse de l'ordinateur et écoute sur cette dernière.

Pour plus d'informations, reportez-vous à Spécification de l'adresse d'écoute.
Port d'écoute Entrez une valeur valide pour le port d'écoute à utiliser pour les demandes normales non sécurisées (via des protocoles tels que HTTP et T3).

La valeur par défaut est 7001 pour le serveur d'administration. Si vous laissez ce champ vide, la valeur par défaut est utilisée.

La plage de ports d'écoute valides s'étend de 1 à 65534.

Pour plus d'informations, reportez-vous à Spécification du port d'écoute.
SSL activé Cochez cette case pour activer le port d'écoute SSL. Par défaut, SSL est désactivé pour tous les nouveaux serveurs.
Port d'écoute SSL Ce champ est activé uniquement si la case SSL activé est cochée.

Entrez une valeur valide à utiliser pour les demandes sécurisées (via des protocoles tels que HTTPS et T3S). La valeur par défaut est 7002. Si vous laissez ce champ vide, la valeur par défaut est utilisée.

La plage de ports d'écoute valides s'étend de 1 à 65535.

Remarque : par défaut, une instance de serveur utilise des certificats de démonstration pour authentifier les demandes provenant d'un port sécurisé. Dans un environnement de production, vous devez configurer SSL pour l'utilisation de certificats provenant d'une autorité de certification.

Une fois les sélections et entrées obligatoires créées, cliquez sur Suivant.

Pour plus d'informations, reportez-vous à la bibliothèque de documentation Oracle Fusion Middleware sur Oracle Technology Network.

Spécification de l'adresse d'écoute

Cette section fournit des instructions concernant la spécification de l'adresse d'écoute du serveur d'administration. Ces instructions s'appliquent également à tous les serveurs gérés.

Type d'adresse d'écoute Description
Toutes les adresses locales ou un nom DNS Sur les ordinateurs Windows offrant un hébergement multiple, une instance de serveur est liée à toutes les adresses IP disponibles.
Une adresse IP ou un nom DNS
  • Pour se connecter à l'instance de serveur, les processus peuvent indiquer l'adresse IP ou le nom DNS correspondant.
  • Les processus indiquant localhost ne peuvent pas se connecter.
  • Vous devez mettre à jour les processus existants utilisant localhost pour se connecter à l'instance de serveur.
  • La vérification du nom d'hôte doit être désactivée pour les connexions indiquant l'adresse IP pour l'adresse d'écoute et un port sécurisé pour le port d'écoute.

Remarque : pour convertir un nom DNS en adresse IP, WebLogic Server doit pouvoir contacter un serveur DNS approprié ou obtenir la correspondance d'adresse IP localement. Par conséquent, si vous indiquez un nom DNS pour l'adresse d'écoute, vous devez soit laisser un port ouvert assez longtemps pour que l'instance WebLogic Server puisse se connecter à un serveur DNS et mettre sa correspondance en mémoire cache, soit spécifier la correspondance d'adresse IP dans un fichier local. Si vous indiquez une adresse IP pour l'adresse d'écoute et qu'une demande client indique un nom DNS, WebLogic Server tente de convertir le nom DNS. S'il ne parvient pas à accéder à la correspondance de nom DNS, la demande échoue.
localhost
  • Les processus doivent indiquer localhost pour se connecter à l'instance de serveur.
  • Seuls les processus installés sur l'ordinateur hébergeant l'instance de serveur (processus locaux) peuvent se connecter à cette dernière.

Spécification du port d'écoute

Suivez les instructions suivantes lorsque vous indiquez le port d'écoute du serveur d'administration. Ces instructions s'appliquent également aux serveurs gérés.

  • Bien que vous puissiez indiquer n'importe quel numéro de port valide, si vous indiquez le port 80, vous pouvez omettre le numéro de port de la demande HTTP utilisée pour accéder aux ressources sur HTTP. Par exemple, si vous définissez le port 80 comme port d'écoute, vous pouvez vous servir de l'URL http://hostname:portnumber/myfile.html.
  • Sur certains systèmes d'exploitation, le port 80 n'est accessible qu'aux processus exécutés sous un ID utilisateur ou de groupe doté de privilèges. Dans ce cas, vous pouvez associer l'instance de serveur à un ordinateur UNIX sur lequel un GID ou UID d'authentification POST est défini.
