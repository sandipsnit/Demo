Configurer les clusters

Utilisez cet écran pour ajouter, configurer et supprimer des clusters.

Un cluster est un groupe d'instances WebLogic Server qui fonctionnent simultanément pour fournir évolutivité et haute disponibilité aux applications. En créant des clusters, vous pouvez regrouper des serveurs gérés de sorte qu'ils fonctionnent comme une unité unique pour l'hébergement d'applications et de ressources.

  1. Passez en revue la configuration de cluster en cours. Les valeurs par défaut peuvent varier selon la source de domaine sélectionnée.

    Remarque : l'assistant propose deux vues : une vue tabulaire concise de tous les clusters et une vue individuelle de chaque cluster, chaque serveur étant représenté par un onglet. Pour passer d'une vue à l'autre, cliquez sur Changer d'affichage.

  2. Ajoutez ou supprimez des clusters, ou modifiez les paramètres de clusters existants, selon les besoins du domaine.

    Le tableau suivant décrit les champs de cet écran. Les champs accompagnés d'un astérisque (*) sont obligatoires.

    Champ Description
    Nom* Entrez un nom valide pour le cluster.

    Le nom du cluster doit être unique parmi tous les noms de composant du domaine. La valeur par défaut de ce champ est new_Cluster_n, n étant une valeur numérique utilisée pour différencier tous les noms de cluster par défaut ; la valeur de n pour le premier cluster est 1. Cette valeur est incrémentée de 1 pour chaque cluster que vous ajoutez.
    Mode de messagerie du cluster Sélectionnez le mode de messagerie Monodiffusion ou Multidiffusion.
    Adresse de multidiffusion Ce champ est activé uniquement si vous sélectionnez le mode de messagerie Multidiffusion.

    Entrez l'adresse de multidiffusion pour le cluster. Cette adresse est utilisée par les membres du cluster pour communiquer entre eux. La valeur par défaut est 239.192.0.0. La plage d'adresses de multidiffusion valides s'étend de 224.0.0.1 à 239.255.255.255.
    Port de multidiffusion Ce champ est activé uniquement si vous sélectionnez le mode de messagerie Multidiffusion.

    Entrez le port de multidiffusion pour le cluster. Ce port est utilisé par les membres du cluster pour communiquer entre eux. La valeur par défaut est 7001.
    Adresse de cluster Entrez les adresses permettant d'identifier les serveurs gérés dans le cluster. L'adresse d'un cluster peut se présenter comme suit :
    • Liste d'adresses IP ou de ports et noms DNS séparés par des virgules (par exemple : dns_name:port, dns_name:port)
    • Nom DNS mis en correspondance avec plusieurs adresses IP
    • localhost, nom DNS ou adresse IP si l'adresse d'écoute de tous les serveurs gérés écoute sur la même adresse avec des numéros de port uniques

    L'adresse de cluster est utilisée dans les EJB d'entité et sans conservation de statut (stateless) pour construire la partie correspondant au nom d'hôte dans les URL. Si l'adresse de cluster n'est pas définie, les descripteurs EJB risquent de ne pas fonctionner correctement.
  3. Une fois les sélections et entrées obligatoires créées, cliquez sur Suivant.

Pour supprimer un cluster, sélectionnez-le, puis cliquez sur Supprimer. Lorsque vous supprimez un cluster, cette opération ne porte pas sur les serveurs associés ; ces derniers sont simplement enlevés du cluster et ajoutés à un autre cluster. Vous ne pouvez supprimer qu'un seul cluster à la fois.

Pour ajouter un cluster, cliquez sur Ajouter, puis configurez les paramètres du nouveau serveur. Le nom par défaut d'un nouveau cluster est new_Cluster_n, où n commence à 1 et est incrémenté pour chaque nouveau cluster ajouté. Pour modifier le nom par défaut, saisissez le nom souhaité dans la colonne Nom.

Pour passer de l'affichage par défaut sous forme de tableau à un affichage à onglets (un onglet par cluster), cliquez sur Changer d'affichage. Cliquez de nouveau sur Changer d'affichage pour revenir au format tabulaire.
