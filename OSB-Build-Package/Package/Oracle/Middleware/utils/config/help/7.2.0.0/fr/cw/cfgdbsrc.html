Configurer les sources de données JDBC

Utilisez cet écran pour configurer les sources de données JDBC définies dans votre source de domaine.

Une source de données JDBC contient un pool de connexions de base de données créées en même temps que l'instance de source de données (lorsqu'elle est déployée ou ciblée, ou au démarrage du serveur). Les applications recherchent une source de données dans l'arborescence JNDI, puis demandent une connexion. Lorsque les applications n'ont plus besoin des connexions, elles les renvoient vers le pool de connexions de la source de données.

Les sources de données JDBC associées aux produits pour lesquels vous créez le domaine sont répertoriées dans la partie inférieure de l'écran.

Remarque : pour configurer des sources de données en tant que sources de données RAC, cochez les cases situées en regard du nom des sources requises, puis cochez la case Configurer les sources de données sélectionnées en tant que sources de données multiples RAC dans le panneau suivant. Cliquez ensuite sur le bouton Suivant.

  1. Pour sélectionner la source de données pour laquelle indiquer des paramètres, cochez la case située en regard de son nom.

    Remarque : si vous sélectionnez plusieurs sources de données, le texte "Varie selon les sources de données" peut apparaître dans certains champs, indiquant que les valeurs en cours de ces champs ne sont pas identiques dans les sources de données sélectionnées. Si vous continuez et modifiez les valeurs de ces champs, les nouvelles valeurs sont appliquées de façon uniforme dans les sources de données sélectionnées.

    Remarque : les valeurs par défaut des paramètres de source de données tels que Fournisseur, Pilote, Nom d'hôte et Numéro de port dépendent des valeurs indiquées dans les modèles d'application.

  2. Passez en revue les paramètres de configuration en cours et modifiez-les si nécessaire.

    Le tableau suivant décrit les champs de cet écran.

    Remarque : les valeurs par défaut des paramètres de source de données tels que Fournisseur, Pilote, Nom d'hôte et Numéro de port dépendent des valeurs indiquées dans les modèles d'application.

    Champ Description
    Fournisseur Sélectionnez le fournisseur de base de données.
    Pilote Sélectionnez le pilote JDBC à utiliser pour la connexion à la base de données.

    La liste comprend les pilotes JDBC standard correspondant au fournisseur de base de données sélectionné.
    Nom utilisateur Entrez le nom de connexion à la base de données.
    Mot de passe Entrez le mot de passe de connexion correspondant au nom utilisateur indiqué.
    SGBD/Service Entrez un nom de service ou un SID SGBD. La valeur saisie dépend du pilote que vous avez sélectionné.
    • Si le nom du pilote Oracle sélectionné contient les mots "pour connexions d'instance", vous devez entrer le SID.
    • Si le nom du pilote Oracle contient les mots "pour connexions de service", vous devez entrer le nom de service.
    • Pour plus d'informations sur la configuration d'un pilote DataDirect, reportez-vous à la documentation DataDirect.
    Nom d'hôte Entrez le nom du serveur hébergeant la base de données.
    Port Entrez le port à utiliser pour la connexion au serveur qui héberge la base de données.

    Les valeurs que vous indiquez sont affichées dans les colonnes appropriées de la liste de sources de données, pour les sources de données sélectionnées.

  3. Après avoir effectué les sélections et entrées requises, cliquez sur Suivant pour tester les connexions aux sources de données JDBC.

Pour plus d'informations, reportez-vous à la bibliothèque de documentation Oracle Fusion Middleware sur Oracle Technology Network.
