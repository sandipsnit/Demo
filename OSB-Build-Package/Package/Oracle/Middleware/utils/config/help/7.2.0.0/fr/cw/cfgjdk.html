Configurer le mode de démarrage du serveur et JDK

Sur cet écran, indiquez si vous souhaitez exécuter WebLogic en mode Production ou Développement. Vous indiquez également l'emplacement des kits JDK à utiliser dans ce domaine.

  1. Sélectionnez le mode de démarrage du domaine WebLogic.
    • Mode Développement
      Dans ce mode, la configuration de la sécurité est relativement souple, ce qui permet le déploiement automatique d'applications.
    • Mode Production
      Dans ce mode, la configuration de la sécurité est relativement stricte et exige des nom utilisateur et mot de passe pour le déploiement d'applications. Avant de passer votre domaine en mode Production, prenez connaissance des procédures de sécurisation d'un environnement de production. Pour plus d'informations, reportez-vous à Mise en oeuvre de la sécurité de votre environnement de production dans Sécurisation d'un environnement de production pour Oracle WebLogic Server.

    Le tableau suivant fournit des informations permettant de sélectionner le mode de démarrage correspondant le mieux à vos besoins.

    Fonction Mode Développement Mode Production
    SSL Vous pouvez utiliser les certificats numériques et les fichiers de clés de démonstration fournis par les services de sécurité WebLogic Server. Avec ces certificats, vous pouvez concevoir une application de sorte qu'elle fonctionne dans les environnements sécurisés par SSL.

    Pour plus d'informations, reportez-vous à Configuration de SSL dans Sécurisation d'Oracle WebLogic Server.

    		 Vous ne pouvez pas utiliser les certificats numériques et les fichiers de clés de démonstration sous peine de voir apparaître un message d'avertissement.
    Déploiement d'applications Les instances WebLogic Server peuvent déployer et mettre à jour automatiquement les applications qui résident dans le répertoire domain_name/autodeploy.

    Oracle recommande de n'utiliser cette méthode que dans un environnement de développement monoserveur.

    Pour plus d'informations, reportez-vous à Déploiement d'applications et de modules avec weblogic.deployer dans Déploiement d'applications vers Oracle WebLogic Server.

    		 La fonction de déploiement automatique étant désactivée, vous devez utiliser la console d'administration WebLogic Server, l'outil weblogic.Deployer ou l'outil de génération de script WebLogic.
    Rotation des fichiers journaux Par défaut, lorsque vous démarrez l'instance WebLogic Server, le serveur renomme automatiquement (change) le fichier journal du serveur local en SERVER-NAME.log.n. Pendant le reste de la session de serveur, les messages s'accumulent dans le fichier journal jusqu'à ce que la taille de ce dernier atteigne 500 ko.

    La valeur par défaut du paramètre Limiter le nombre de fichiers conservés dans Configuration de la journalisation est True. Elle limite le nombre de fichiers journaux que l'instance de serveur crée pour stocker les anciens messages.

    		 Le serveur change de fichier journal local lorsque la taille de ce dernier atteint 5 000 ko. Lorsque le serveur est configuré pour le mode Production, par défaut toutes les versions des fichiers journaux sont conservées. Les administrateurs peuvent personnaliser le paramètre déterminant le nombre de fichiers journaux conservés.
    Ressource système JDBC La capacité par défaut est de 15 connexions. La capacité par défaut est de 25 connexions.
  2. Sélectionnez le kit JDK.

    Dans le panneau Sélection de kit JDK, sélectionnez le kit JDK correspondant au mode de démarrage sélectionné dans le panneau Mode de démarrage du domaine WebLogic. L'assistant Configuration présente la liste des kits JDK inclus dans le programme d'installation. Vous pouvez choisir l'un de ces kits JDK ou en choisir un autre que vous avez installé sur votre système.

    Remarque :

    • Si vous sélectionnez un kit JDK inclus dans le programme d'installation, l'assistant Configuration crée des scripts de démarrage de serveur pour appeler ce kit. Si vous sélectionnez un kit JDK non fourni par JDK, l'assistant Configuration ne configure pas les scripts de démarrage ; vous devez modifier ces derniers manuellement.
    • Sélectionnez uniquement les kits JDK pris en charge sur la plate-forme que vous utilisez. La sélection par défaut indique le kit JDK répondant le mieux aux exigences de votre environnement, selon la plate-forme sur laquelle vous installez le domaine.
  3. Après avoir effectué les sélections et entrées requises, cliquez sur le bouton Suivant de l'assistant.

Pour plus d'informations, reportez-vous à la bibliothèque de documentation Oracle Fusion Middleware sur Oracle Technology Network.
