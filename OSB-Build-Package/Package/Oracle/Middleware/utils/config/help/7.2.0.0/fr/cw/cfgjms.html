Configurer des emplacements de stockage des fichiers JMS

Un emplacement de stockage de fichiers JMS est un fichier basé sur un disque dans lequel les messages persistants peuvent être enregistrés.

Passez en revue les paramètres en cours des emplacements de stockage des fichiers JMS et modifiez-les si nécessaire. Les valeurs par défaut peuvent varier selon la source de domaine sélectionnée précédemment.

Remarque : l'assistant propose deux modes d'affichage : une vue tabulaire concise de tous les composants définis et une vue individuelle dans laquelle chaque composant est représenté par un onglet. Pour passer d'un mode à l'autre, cliquez sur Changer d'affichage.

Le tableau suivant décrit les champs de cet écran. Les champs accompagnés d'un astérisque (*) sont obligatoires.

Champ Description
Nom* Entrez le nom de l'emplacement de stockage des fichiers JMS.

Le nom de l'emplacement de stockage des fichiers JMS doit être une chaîne de caractères, il peut comprendre des espaces et doit être unique parmi tous les noms de composant du domaine.
Répertoire Entrez le chemin du répertoire (sur votre système) dans lequel réside l'emplacement de stockage des fichiers JMS.
Stratégie d'écriture synchrone Sélectionnez l'une des stratégies d'écriture synchrone suivantes pour déterminer comment l'emplacement de stockage des fichiers écrit les données sur le disque :
  • Vidage du cache : les transactions ne peuvent être exécutées qu'une fois toutes leurs opérations d'écriture transférées vers le disque.
  • Ecriture directe : les opérations d'écriture sont réalisées directement sur le disque. Cette stratégie est prise en charge sous Solaris et Windows. Si cette stratégie est active sur une plate-forme non prise en charge, l'emplacement de stockage des fichiers opte automatiquement pour la stratégie Vidage du cache.

    Remarque : la stratégie Ecriture directe est fiable sur les systèmes Solaris, mais les systèmes Windows peuvent laisser des données de transaction dans le cache sur disque sans les écrire immédiatement sur le disque. Cette méthode est peu fiable car une panne de courant peut entraîner la perte des données figurant dans le cache sur disque, et éventuellement la perte ou la duplication de messages. Pour garantir la fiabilité des écritures réalisées à l'aide de la stratégie Ecriture directe sous Windows, désactivez toute mise en cache d'écritures pour le disque (activée par défaut) ou utilisez un disque doté d'un cache avec batterie de sauvegarde. Toutefois, certains systèmes de fichiers n'autorisent pas la modification de cette valeur (par exemple, un système RAID doté d'un cache fiable).

  • Désactivé : les transactions sont terminées une fois les écritures mises en mémoire cache. Si cette stratégie est active, les transactions peuvent être terminées avant que les écritures n'atteignent le disque.

Ce paramètre affecte les performances, l'évolutivité et la fiabilité.

Remarque : si l'emplacement de stockage des fichiers JMS est utilisé exclusivement pour la pagination des messages non persistants sur le disque, la stratégie d'écriture synchrone n'est pas prise en compte.

Une fois les sélections et entrées obligatoires créées, cliquez sur Suivant.
