Configurer les ordinateurs

Dans un domaine WebLogic, les définitions d'ordinateur identifient les unités physiques matérielles et sont associées aux instances WebLogic qu'elles hébergent.

Utilisez cet écran pour ajouter ou supprimer des ordinateurs, ou modifier les paramètres d'un ordinateur existant.

Sélectionnez l'onglet Ordinateur (pour Windows) ou l'onglet Ordinateur Unix (pour UNIX).

Vous pouvez créer des définitions d'ordinateur dans les cas suivants :

  • Le serveur d'administration utilise la définition d'ordinateur, ainsi que l'application de gestionnaire de noeuds, pour démarrer les serveurs distants.
  • WebLogic Server emploie des noms d'ordinateur configurés lorsqu'il définit le serveur dans le cluster le plus approprié pour traiter certaines tâches, telles que la réplication de session HTTP. Il délègue ensuite ces tâches au serveur identifié.

Remarque : vous devez configurer les ordinateurs pour chaque installation de produit exécutant un processus de gestionnaire de noeuds. Cette configuration doit inclure les valeurs des paramètres d'adresse d'écoute et de numéro de port.

Chaque ordinateur présente les paramètres de configuration ci-après. Les champs obligatoires sont marqués d'un astérisque (*).

  1. Sélectionnez l'onglet Ordinateur (pour Windows) ou l'onglet Ordinateur Unix (pour UNIX).
  2. Passez en revue la liste des configurations, et ajoutez, supprimez ou modifiez les paramètres selon les besoins du domaine. Le tableau suivant décrit les champs de cet écran. Les champs accompagnés d'un astérisque (*) sont obligatoires.

    Champ Description
    Nom* Entrez un nom valide pour l'ordinateur.

    Le nom d'ordinateur permet d'identifier l'ordinateur au sein du domaine WebLogic Server ; il ne doit pas nécessairement correspondre au nom réseau de l'ordinateur. Ce nom doit être unique parmi tous les noms de composant au sein du domaine. La valeur par défaut de ce champ est new_[Unix]Machine_n, n étant une valeur numérique utilisée pour différencier tous les noms d'ordinateur par défaut ; la valeur de n pour le premier ordinateur est 1. Cette valeur est incrémentée de 1 pour chaque ordinateur que vous ajoutez.
    Adresse d'écoute du gestionnaire de noeuds Sélectionnez dans la liste la valeur correspondant à l'adresse utilisée par le gestionnaire de noeuds pour écouter les demandes de connexion.

    Par défaut, les adresses IP définies pour le système local et localhost apparaissent dans la liste. La valeur par défaut est localhost. Si vous indiquez une adresse IP pour un ordinateur qui héberge le serveur d'administration et que vous devez accéder au gestionnaire de noeuds WebLogic Server, vous devez désactiver la fonction de vérification du nom d'hôte.

    Pour plus d'informations, reportez-vous à Utilisation de la vérification de nom d'hôte dans Sécurisation d'Oracle WebLogic Server.
    Port d'écoute du gestionnaire de noeuds Entrez une valeur valide pour le port d'écoute utilisé par le gestionnaire de noeuds pour écouter les demandes de connexion.

    La plage de numéros de port valides pour le port d'écoute du gestionnaire de noeuds s'étend de 1 à 65534. La valeur par défaut est 5556.
    GID d'authentification POST activé (Ordinateurs UNIX uniquement) Cochez cette case pour permettre à un serveur exécuté sur cet ordinateur d'établir une liaison avec un ID de groupe UNIX (GID) une fois toutes les opérations de démarrage privilégiées terminées (reportez-vous à la description du champ GID d'authentification POST ci-dessous). Par défaut, cette case est désélectionnée.
    GID d'authentification POST (Ordinateurs UNIX uniquement) Entrez un ID de groupe (GID) UNIX valide grâce auquel un serveur hébergé sur cet ordinateur sera exécuté une fois toutes les opérations de démarrage privilégiées terminées. Sinon, le serveur poursuit son exécution en utilisant l'ID de groupe avec lequel il a été démarré.

    Pour que ce paramètre soit appliqué, vous devez cocher la case GID d'authentification POST activé.

    UID d'authentification POST activé (Ordinateurs UNIX uniquement) Cochez cette case pour permettre à un serveur hébergé sur cet ordinateur d'établir une liaison avec un ID utilisateur UNIX (UID) une fois toutes les opérations de démarrage privilégiées terminées (reportez-vous à la description du champ UID d'authentification POST ci-dessous). Par défaut, cette case est désélectionnée.
    UID d'authentification POST (Ordinateurs UNIX uniquement) Entrez un ID utilisateur (UID) UNIX valide grâce auquel un serveur hébergé sur cet ordinateur sera exécuté une fois toutes les opérations de démarrage privilégiées terminées. Sinon, le serveur poursuit son exécution en utilisant l'ID utilisateur avec lequel il a été démarré.

    Pour que ce paramètre soit appliqué, vous devez cocher la case UID d'authentification POST activé.
  3. Une fois les sélections et entrées obligatoires créées, cliquez sur Suivant.

Cliquez sur Ajouter pour ajouter un autre ordinateur. Le nom par défaut de ce dernier est new_[Unix]Machine_n, où n commence à 1 et incrémente de 1 pour chaque ordinateur ajouté.

Cliquez sur Supprimer pour supprimer un ordinateur existant.
