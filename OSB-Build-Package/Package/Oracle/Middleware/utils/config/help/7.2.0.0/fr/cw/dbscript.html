Exécuter des scripts de base de données

Un modèle de domaine peut contenir un ensemble de fichiers SQL organisés par type de base de données. Si ce modèle inclut des fichiers SQL, vous pouvez les exécuter lors de la création du domaine WebLogic, dans l'écran Exécuter des scripts de base de données. Le contenu de base de données de chacune des sources de données définies dans votre domaine WebLogic est configuré via les fichiers SQL ou de chargement de base de données existants.

Le contenu de la base de données pour chacune des sources de données définies dans votre domaine est configuré à l'aide d'un code SQL préexistant ou de fichiers de chargement de base de données.

  1. Dans le panneau Sources de données JDBC disponibles, sélectionnez la source de données pour laquelle exécuter les scripts.

    Les scripts pouvant être exécutés sont affichés dans le panneau Fichiers SQL disponibles et options de chargement de base de données.

  2. Sélectionnez la version de la base de données dans la liste Version de base de données.
  3. Cliquez sur Exécuter des scripts.

    Tous les scripts affichés dans le panneau Fichiers SQL disponibles et options de chargement de base de données pour la source de données sont exécutés et les résultats apparaissent dans le panneau Résultats.

    Pour capturer le résultat du test dans un fichier journal, cochez la case Fichier journal et indiquez l'emplacement du fichier journal.

  4. Répétez les étapes 1 à 3 pour chaque source de données pour laquelle exécuter des scripts de base de données.
  5. Une fois les scripts de base de données exécutés, cliquez sur Suivant.

Pour plus d'informations, reportez-vous à la bibliothèque de documentation Oracle Fusion Middleware sur Oracle Technology Network.
