Créer des applications de proxy HTTP

Cet écran permet de créer une application de proxy HTTP pour chaque cluster et d'indiquer le serveur géré sur lequel cette application doit être déployée. Une application de proxy HTTP agit comme un intermédiaire pour les demandes HTTP.

Cet écran n'apparaît que si les deux affirmations suivantes sont avérées :

  • Au moins un serveur géré est affecté à un cluster.
  • Au moins un serveur géré n'est pas affecté à un cluster.
  1. Cochez la case Créer un proxy HTTP correspondant au cluster pour lequel créer l'application de proxy HTTP.

    Le nom des serveurs gérés qui ne sont affectés à aucun cluster apparaît dans la liste Serveur proxy.

  2. Dans la liste Serveur proxy, sélectionnez un serveur géré sur lequel déployer l'application de proxy.

    Une application de proxy nommée OracleProxy4_<clustername>_<servername> est créée et déployée dans le serveur géré sélectionné.

  3. Répétez les étapes 1 et 2 pour chaque cluster pour lequel créer des applications de proxy HTTP.
  4. Une fois les sélections obligatoires créées, cliquez sur Suivant.

Pour plus d'informations, reportez-vous à la bibliothèque de documentation Oracle Fusion Middleware sur Oracle Technology Network.
