Indiquer le nom et l'emplacement du domaine

Indiquez les éléments suivants :

  • Nom du domaine
    Nom du domaine. L'assistant Configuration stocke le fichier config.xml et tous les autres composants générés pour le domaine dans ce répertoire, sous le répertoire de domaine indiqué. Par exemple, si vous saisissez mydomain, les fichiers du domaine sont stockés (par défaut) dans le répertoire MW_HOME\user_projects\domains\mydomain.

    Remarque : le nom de domaine ne doit pas commencer par un chiffre. Cette restriction vise à empêcher les problèmes potentiels avec les noms de table de stockage JDBC générés en interne, qui doivent commencer par un caractère alphabétique.

  • Emplacement du domaine
    Emplacement dans lequel le répertoire de domaine sera stocké. Le répertoire par défaut est MW_HOME\user_projects\domains. L'emplacement du domaine peut faire partie de votre réseau ou lecteur local. Il ne doit pas se trouver dans le répertoire MW_HOME.

    Sous Windows, vous devez indiquer la lettre de lecteur correspondante dans ce chemin.

  • Emplacement de l'application
    Ce champ n'apparaît que si vous avez sélectionné un modèle de domaine ou un produit contenant des applications. Il s'agit de l'emplacement dans lequel vos applications seront installées. Le répertoire par défaut est MW_HOME\user_projects\applications.

    Sous Windows, vous devez indiquer la lettre de lecteur correspondante dans ce chemin.

Une fois les sélections et entrées obligatoires créées, cliquez sur Suivant.
