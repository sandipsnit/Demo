Cibler les déploiements sur des clusters ou des serveurs

Utilisez cet écran pour cibler des applications et des bibliothèques afin de les déployer sur des serveurs ou des clusters.

Les applications et les bibliothèques associées au produit pour lequel vous configurez le domaine (par exemple, SOA) sont ciblées automatiquement sur le serveur géré créé pour ce produit ou sur le cluster auquel ce serveur géré est affecté. Dans cet écran, vous pouvez cibler les applications et les bibliothèques sur des serveurs et clusters supplémentaires.

  1. Dans le panneau Target, sélectionnez le cluster ou le serveur sur lequel déployer les applications ou les bibliothèques.

    Le nom de la cible sélectionnée apparaît dans le titre du panneau de droite.

  2. Dans le panneau de droite, cochez les cases correspondant aux applications ou aux bibliothèques à déployer sur la cible sélectionnée.

    Les applications et bibliothèques affichées ici varient selon les produits sélectionnés précédemment dans l'écran Sélectionner une source de domaine de l'assistant.

    Remarque : lorsque vous sélectionnez un serveur géré dans le panneau Target, certaines des cases du panneau de droite peuvent être désactivées ; elles correspondent aux applications et bibliothèques déjà ciblées sur le cluster contenant le serveur géré sélectionné.

    Une fois que vous avez sélectionné les applications et les bibliothèques voulues, les noms des clusters et serveurs ciblés apparaissent dans la colonne Target du panneau de droite.

  3. Répétez les étapes 1 et 2 pour les autres clusters et serveurs requis.
  4. Une fois les sélections obligatoires créées, cliquez sur Suivant.

Si vous étendez un domaine et que vous supprimez un serveur géré ou un cluster sur lequel les applications sont ciblées, l'assistant Configuration cible de nouveau automatiquement ces dernières comme suit :

  • Si les applications n'ont été initialement ciblées que sur le serveur géré ou le cluster que vous supprimez (au terme de la suppression, elles ne sont pas ciblées dans le domaine modifié), l'assistant Configuration cible de nouveau automatiquement ces applications sur toutes les cibles admissibles.

    Une cible admissible est un cluster ou un serveur géré non défini dans la spécification de groupes de configuration (fichier config-groups.xml) d'un modèle inclus. Les serveurs ou les clusters indiqués dans le fichier config-groups.xml appartiennent essentiellement au modèle et ne peuvent donc pas faire l'objet d'un ciblage automatique.

  • Si les applications ont été initialement ciblées sur plusieurs cibles (y compris les serveurs gérés, les clusters et le serveur d'administration), et que l'un des serveurs gérés ou clusters ciblés est supprimé, dans le domaine étendu, l'assistant Configuration laisse les autres associations de cibles intactes et ne tente pas de cibler de nouveau ces applications.
