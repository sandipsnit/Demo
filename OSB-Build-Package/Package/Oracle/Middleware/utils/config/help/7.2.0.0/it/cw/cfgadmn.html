Configura server di amministrazione

Il server di amministrazione costituisce il punto centrale da cui viene gestito il dominio. È possibile accedere al server di amministrazione utilizzando l'URL protocollo://indirizzo-di-ascolto:porta-di-ascolto. Il protocollo può essere uno dei seguenti: t3, t3s, http e https.

Utilizzare questa schermata per configurare o modificare le impostazioni del server di amministrazione riportate di seguito. I campi obbligatori sono indicati da un asterisco (*).

Campo Descrizione
*Nome Il nome predefinito è AdminServer. I nomi di server validi sono costituiti da una stringa di caratteri (alfabetici e numerici).

Ogni istanza del server nell'ambiente di produzione deve avere un nome univoco, indipendentemente dal dominio o dal cluster in cui si trova e dal fatto che sia un server di amministrazione o un server gestito. Inoltre, il nome del server di amministrazione deve essere univoco tra tutti i nomi di componente all'interno del dominio.

Nota: questo valore viene specificato solo a scopi di identificazione. Non viene usato come parte dell'URL per le applicazioni distribuite nel server. Il nome del server viene visualizzato nella console di amministrazione del server WebLogic. Inoltre, se vengono usate le utility o le API della riga di comando del server WebLogic, è necessario specificare questo nome per identificare il server.

*Indirizzo di ascolto Selezionare un valore per l'indirizzo di ascolto dall'elenco a discesa.

Se si seleziona localhost come indirizzo di ascolto per un'istanza del server, i processi remoti non possono effettuare la connessione a tale istanza del server. Solo i processi sul computer che ospitano l'istanza del server possono effettuare la connessione. Se l'istanza del server deve essere accessibile come host locale (ad esempio, se vengono creati degli script amministrativi che consentono la connessione all'host locale) ed essere anche accessibile ai processi remoti, selezionare Tutti gli indirizzi locali. L'istanza del server determina l'indirizzo del computer e lo usa per l'ascolto.

Per ulteriori informazioni, vedere Specifica dell'indirizzo di ascolto.
Porta di ascolto Immettere un valore valido per la porta di ascolto da usare per le richieste regolari non sicure (attraverso protocolli quali HTTP e T3).

Il valore predefinito per il server di amministrazione è 7001. Se questo campo viene lasciato vuoto, viene usato il valore predefinito.

L'intervallo valido delle porte di ascolto è compreso tra 1 e 65534.

Per ulteriori informazioni, vedere Specifica della porta di ascolto.
SSL abilitato Selezionare questa casella di controllo per abilitare la porta di ascolto SSL. Per impostazione predefinita, SSL viene disabilitato per tutti i nuovi server.
Porta di ascolto SSL Questo campo è abilitato solo se è selezionata la casella di controllo SSL abilitato.

Immettere un valore valido da usare per le richieste sicure (attraverso protocolli quali HTTPS e T3S). Il valore predefinito per il server di amministrazione è 7002. Se questo campo viene lasciato vuoto, viene usato il valore predefinito.

L'intervallo valido delle porte di ascolto è compreso tra 1 e 65535.

Nota: per impostazione predefinita, un'istanza del server usa i certificati di dimostrazione per autenticare le richieste da una porta sicura. In un ambiente di produzione, è necessario configurare SSL per usare i certificati di un'autorità di certificazione.

Dopo aver effettuato le selezioni e le immissioni necessarie, fare clic su Successivo.

Per ulteriori informazioni, vedere la libreria della documentazione di Oracle Fusion Middleware su Oracle Technology Network.

Specifica dell'indirizzo di ascolto

Questa sezione fornisce le istruzioni per specificare l'indirizzo di ascolto per il server di amministrazione. Queste istruzioni sono valide anche per tutti i server gestiti.

Tipo di indirizzo di ascolto Funzionamento
Tutti gli indirizzi locali o un nome DNS Sui computer Windows multihomed, un'istanza di server è associata a tutti gli indirizzi IP disponibili.
Un indirizzo IP o un nome DNS
  • Per connettersi all'istanza del server, i processi possono specificare l'indirizzo IP o il nome DNS corrispondente.
  • La connessione non riesce se i processi specificano localhost.
  • Per connettersi all'istanza del server, è necessario aggiornare i processi esistenti che usano localhost.
  • Per le connessioni che specificano l'indirizzo IP come indirizzo di ascolto e una porta protetta come porta di ascolto, è necessario disabilitare la verifica del nome host.

Nota: per risolvere un nome DNS in un indirizzo IP, il server WebLogic deve contattare un server DNS appropriato oppure ottenere il mapping dell'indirizzo IP a livello locale. Pertanto, se si specifica un nome DNS per l'indirizzo di ascolto, è necessario lasciare aperta una porta per un periodo di tempo sufficientemente lungo per consentire all'istanza del server WebLogic di connettersi a un server DNS e inserire nella cache il mapping del server oppure specificare il mapping dell'indirizzo IP in un file locale. Se si specifica un indirizzo IP come indirizzo di ascolto e successivamente una richiesta di un client specifica un nome DNS, il server WebLogic tenterà di risolvere il nome DNS. Se il server WebLogic non ha accesso al mapping del nome DNS, la richiesta non riesce.
localhost
  • I processi devono specificare localhost per connettersi all'istanza del server.
  • Solo i processi che risiedono sul computer che ospita l'istanza del server (processi locali) possono connettersi all'istanza del server.

Specifica della porta di ascolto

Tenere presenti le istruzioni riportate di seguito quando si specifica la porta di ascolto per il server di amministrazione. Queste istruzioni sono valide anche per i server gestiti.

  • Sebbene sia possibile specificare qualsiasi numero di porta valido, se si specifica la porta 80 è possibile omettere il numero di porta dalla richiesta HTTP utilizzata per accedere alle risorse su HTTP. Ad esempio, se si definisce la porta 80 come porta di ascolto, è possibile utilizzare l'URL http://nomehost:numeroporta/myfile.html.
  • In alcuni sistemi operativi l'accesso alla porta 80 è consentito solo ai processi eseguiti da un utente o un ID gruppo con privilegi. In questo caso, è possibile assegnare l'istanza del server a un computer UNIX sul quale è definito un UID o un GID di post-autenticazione.
