Configura cluster

Utilizzare questa schermata per aggiungere, modificare ed eliminare cluster.

Un cluster è un gruppo di istanze del server WebLogic che vengono eseguite simultaneamente per fornire scalabilità e alta disponibilità per le applicazioni. La creazione dei cluster consente di raggruppare i server gestiti in modo che possano funzionare come una singola unità per ospitare applicazioni e risorse.

  1. Rivedere la configurazione del cluster corrente. I valori predefiniti possono variare in base all'origine del dominio selezionata.

    Nota: la procedura guidata fornisce due viste, una vista tabellare concisa di tutti i cluster e una singola vista di ogni cluster con ogni server rappresentato da una scheda. Per spostarsi tra le modalità di visualizzazione delle viste tabellari e a scheda, fare clic su Cambia visualizzazione.

  2. Aggiungere o eliminare i cluster o modificare le impostazioni dei cluster esistenti, come richiesto per il dominio.

    Nella tabella seguente vengono descritti i campi disponibili in questa schermata. I campi contrassegnati da un asterisco (*) sono obbligatori.

    Campo Descrizione
    Nome* Immettere un nome valido per il cluster.

    Il nome del cluster deve essere univoco tra tutti i nomi di componente all'interno del dominio. Il valore predefinito per questo campo è new_Cluster_n, dove n è un valore numerico usato per effettuare una differenziazione tra tutti i nomi cluster predefiniti; il valore n per il primo cluster è 1. Il valore viene aumentato di 1 per ogni cluster che viene aggiunto.
    Modalità di messaggistica cluster Selezionare la modalità di messaggistica Unicast o Multicast.
    Indirizzo Multicast Questo campo è abilitato solo se si seleziona la modalità di messaggistica Multicast.

    Immettere l'indirizzo Multicast per il cluster. Questo indirizzo viene usato dai membri del cluster per comunicare tra loro. Il valore predefinito è 239.192.0.0. L'intervallo di indirizzi Multicast valido è compreso tra 224.0.0.1 e 239.255.255.255.
    Porta Multicast Questo campo è abilitato solo se si seleziona la modalità di messaggistica Multicast.

    Immettere la porta Multicast per il cluster. Questa porta Multicast viene usata dai membri del cluster per comunicare tra loro. Il valore predefinito è 7001.
    Indirizzo cluster Immettere gli indirizzi per identificare i server gestiti nel cluster. Di seguito è riportata una lista di possibili indirizzi cluster.
    • Liste di indirizzi IP separati da virgole o nomi e porte DNS (ad esempio: dns_name:port, dns_name:port)
    • Nome DNS mappato a più indirizzi IP
    • Host locale, nome DNS o indirizzo IP se l'indirizzo di ascolto di tutti i server gestiti è lo stesso con numeri di porta univoci

    L'indirizzo cluster viene usato in entità e EJB senza conservazione dello stato per costruire la parte di nome host degli URL. Se l'indirizzo cluster non è impostato, gli handle EJB potrebbero non funzionare correttamente.
  3. Dopo aver effettuato le selezioni e le immissioni necessarie, fare clic su Successivo.

Per eliminare un cluster, selezionare il server e fare clic su Elimina. Quando si elimina un cluster, il server assegnato ad esso non viene eliminato; i server vengono rimossi semplicemente dal cluster e possono quindi essere aggiunti a un altro cluster. È possibile eliminare un solo cluster alla volta.

Per aggiungere un cluster, fare clic su Aggiungi e configurare le impostazioni per il nuovo server. Il nome predefinito per un nuovo cluster è new_Cluster_n, dove n ha inizialmente il valore 1 che viene incrementato di un'unità per ogni nuovo cluster aggiunto. Per modificare l'impostazione predefinita, digitare il nome desiderato nella colonna Nome.

Per passare dal formato di visualizzazione tabulare predefinito al formato a schede (con una scheda per ogni cluster), fare clic su Cambia visualizzazione. Fare di nuovo clic su Cambia visualizzazione per tornare al formato tabulare.
