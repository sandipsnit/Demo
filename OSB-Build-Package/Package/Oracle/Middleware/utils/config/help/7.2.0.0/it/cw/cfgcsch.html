Configura schema dei componenti JDBC

Per alcuni componenti di Fusion Middleware (ad esempio, SOA e WebCenter Spaces), è possibile che le origini dati JDBC siano definite come parte dello schema di database del componente e vengano caricate durante l'installazione utilizzando Repository Creation Utility (RCU).

Quando si crea un dominio WebLogic per tali componenti utilizzando la Configurazione guidata, è possibile configurare le impostazioni degli schemi dei componenti JDBC, come ad esempio il driver di database, il proprietario dello schema, la password e così via.

Gli schemi dei componenti JDBC associati ai prodotti per cui si sta creando il dominio (ad esempio, SOA) sono elencati nella parte inferiore della schermata.

Nota: se si desidera configurare uno o più schemi come origini dati RAC, selezionare le caselle di controllo accanto al nome degli schemi richiesti e la casella di controllo Configurare nel pannello successivo gli schemi dei componenti selezionati come schemi di origini dati multiple RAC. Quindi fare clic sul pulsante Successivo.

  1. Scegliere gli schemi per cui si desidera specificare le impostazioni delle origini dati selezionando la casella di controllo accanto ai nomi degli schemi.

    Nota: quando si selezionano più schemi dei componenti, in alcuni campi potrebbe essere visualizzato il testo "Varia a seconda degli schemi dei componenti", indicante che i valori correnti di questi campi non sono gli stessi per tutti gli schemi dei componenti selezionati. Se si prosegue e si modificano i valori in tali campi, i nuovi valori vengono applicati in modo uniforme agli schemi dei componenti selezionati.

    Nota: i valori predefiniti dei parametri degli schemi dei componenti quali fornitore, driver, nome host e numero di porta dipendono dai valori specificati nei modelli di applicazione.

  2. Rivedere le impostazioni di configurazione correnti e modificarle come richiesto.

    Nota: i valori predefiniti dei parametri degli schemi quali fornitore, driver, nome host e numero di porta dipendono dai valori specificati nei modelli di applicazione.

    Campo Descrizione
    Fornitore Selezionare il fornitore del database.
    Driver Selezionare il driver JDBC da usare per connettersi al database.

    La lista include i driver JDBC per il fornitore del database selezionato.
    Proprietario schema Immettere il nome di login per la connessione al database.
    Password schema Immettere la password di login per il proprietario dello schema specificato.
    DBMS/Servizio Immettere un nome DBMS per il database o il nome del servizio se è stato selezionato un driver del tipo di servizio.
    Nome host Immettere il nome del server su cui si trova il database.
    Porta Immettere il numero di porta da usare per la connessione al server su cui si trova il database.

    I valori specificati vengono visualizzati nelle colonne appropriate della lista di schemi per gli schemi selezionati.

  3. Dopo aver effettuato le selezioni e le immissioni necessarie, fare clic sul pulsante Successivo nella procedura guidata per eseguire il test degli schemi dei componenti.
