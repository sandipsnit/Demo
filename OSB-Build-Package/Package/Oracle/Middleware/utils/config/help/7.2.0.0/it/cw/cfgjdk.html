Configura modalità di avvio del server e JDK

In questa schermata specificare se si desidera eseguire WebLogic in modalità di produzione o di sviluppo. È anche possibile specificare la posizione dei kit JDK da usare in questo dominio.

  1. Selezionare la modalità di avvio del dominio WebLogic.
    • Modalità di sviluppo
      In questa modalità la configurazione di sicurezza non è particolarmente restrittiva in quanto consente la distribuzione delle applicazioni in modo automatico.
    • Modalità di produzione
      In questa modalità la configurazione di sicurezza è relativamente restrittiva in quanto è necessario specificare un nome utente e una password per distribuire le applicazioni. Prima di inserire un dominio nell'ambiente di produzione, familiarizzare con le procedure per la protezione di un ambiente di produzione. Per ulteriori informazioni, vedere Garantire la sicurezza nell'ambiente di produzione in Protezione di un ambiente di produzione per Oracle WebLogic Server.

    La tabella riportata di seguito fornisce informazioni per la selezione di una modalità di avvio adatta alle proprie esigenze.

    Funzione Modalità di sviluppo Modalità di produzione
    SSL È possibile usare i certificati digitali e i keystore di dimostrazione forniti dai servizi di sicurezza del server WebLogic. Questi certificati consentono di progettare l'applicazione da usare all'interno di ambienti protetti da SSL.

    Per ulteriori informazioni, vedere Configurazione di SSL in Protezione di Oracle WebLogic Server.

    		 I certificati digitali e i keystore di dimostrazione non devono essere usati. Nel caso in cui ciò avvenga, viene visualizzato un messaggio di avvertenza.
    Distribuzione applicazioni Le istanze del server WebLogic possono distribuire e aggiornare in modo automatico le applicazioni che si trovano nella directory nome_dominio/autodeploy/.

    Oracle raccomanda di usare questo metodo solo in un ambiente di sviluppo a server singolo.

    Per ulteriori informazioni, vedere Distribuzione di applicazioni e moduli con weblogic.deployer in Distribuzione di applicazioni su Oracle WebLogic Server.

    		 La funzione di distribuzione automatica è disabilitata e quindi è necessario usare la console di amministrazione del server WebLogic, lo strumento weblogic.Deployer o lo strumento di script di WebLogic.
    Rotazione dei file di log Per impostazione predefinita, quando viene avviata l'istanza del server WebLogic, il server rinomina (ruota) automaticamente il file di log del server locale come SERVER-NAME.log.n. Per il resto della sessione del server, i messaggi si accumulano nel file di log finché questi non raggiunge una dimensione di 500 kilobyte.

    Il valore predefinito dell'impostazione Limita il numero di file conservati in Configurazione di log è true. Questo valore limita il numero di file di log creati dell'istanza del server per memorizzare i messaggi meno recenti.

    		 Il server ruota il file di log locale dopo che questi ha raggiunto una dimensione di 5000 kilobyte. Quando il server viene configurato per la modalità di produzione, per impostazione predefinita vengono conservate tutte le versioni dei file di log. Gli amministratori possono personalizzare il numero di file di log conservati.
    Risorsa di sistema JDBC La capienza predefinita è di 15 connessioni. La capienza predefinita è di 25 connessioni.
  2. Selezionare il JDK.

    Nel riquadro Selezione JDK, selezionare il JDK per la modalità di avvio selezionata nel riquadro Modalità di avvio dominio WebLogic. La Configurazione guidata presenta una lista di JDK inclusi nell'Installer. È possibile scegliere tra uno di questi JDK o un altro JDK installato nel sistema.

    Note:

    • se si seleziona un JDK incluso nell'Installer, la Configurazione guidata crea gli script di avvio del server per richiamare il JDK. Se si seleziona un JDK non fornito da JDK, la Configurazione guidata non configura gli script di avvio e quindi è necessario modificare tali script manualmente;
    • selezionare solo i JDK supportati sulla piattaforma in uso. La selezione predefinita riflette il JDK che meglio soddisfa i requisiti dell'ambiente, a seconda della piattaforma su cui viene installato il dominio.
  3. Dopo aver effettuato le selezioni e le immissioni necessarie, fare clic sul pulsante Successivo nella procedura guidata.
