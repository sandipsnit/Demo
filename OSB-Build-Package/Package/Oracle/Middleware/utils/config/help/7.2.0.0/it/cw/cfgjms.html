Configura aree memorizzazione file JMS

Un'area di memorizzazione file JMS è un file basato su disco in cui è possibile salvare i messaggi persistenti.

Rivedere le impostazioni correnti per le aree di memorizzazione file JMS e modificarle come richiesto. I valori predefiniti possono variare in base all'origine del dominio selezionata in precedenza.

Nota: la procedura guidata fornisce due modalità di visualizzazione: una vista tabellare concisa di tutti i componenti definiti e una singola vista in cui ogni componente è rappresentato da una scheda. Per spostarsi tra le modalità di visualizzazione delle viste tabellari e a scheda, fare clic su Cambia visualizzazione.

Nella tabella seguente vengono descritti i campi disponibili in questa schermata. I campi contrassegnati da un asterisco (*) sono obbligatori.

Campo Descrizione
Nome* Immettere un nome per l'area di memorizzazione JMS.

Il nome deve essere una stringa di caratteri e può includere gli spazi. Il nome dell'area di memorizzazione JMS deve essere univoco tra tutti i nomi di componente all'interno del dominio.
Directory Immettere il percorso della directory (nel sistema) in cui si trova l'area di memorizzazione JMS.
Criteri scrittura sincrona Selezionare uno dei seguenti criteri di scrittura sincrona per determinare la modalità in cui l'area di memorizzazione dei file scrive i dati sul disco:
  • Svuotamento cache: le transazioni non possono essere completate finché tutte le relative operazioni di scrittura non sono state svuotate sul disco.
  • Scrittura diretta: le operazioni di scrittura vengono eseguite direttamente sul disco. Questo criterio è supportato su Solaris e Windows. Se questo criterio è attivo su una piattaforma non supportata, l'area di memorizzazione dei file passa automaticamente al criterio di svuotamento della cache.

    Nota: il criterio di scrittura diretta è affidabile sui sistemi Solaris ma sui sistemi Windows potrebbe lasciare i dati della transazione nella cache su disco senza scriverli immediatamente sul disco. Questa operazione è considerata non affidabile perché una caduta di tensione può provocare la perdita dei dati delle cache su disco e, di conseguenza, una possibile perdita o duplicazione dei messaggi. Per una scrittura affidabile tramite il criterio di scrittura diretta su Windows, disabilitare l'inserimento dei dati scritti nella cache su disco (abilitato per impostazione predefinita) oppure usare un disco con una cache dotata di batteria. Alcuni file system, tuttavia, non consentono la modifica di questo valore (ad esempio, un sistema RAID che ha una cache affidabile).

  • Disabilitato: le transazioni vengono completate appena le scritture vengono inserite nella cache in memoria. Quando questo criterio è attivo, il completamento delle transazioni non dipende dal tempo di attesa delle scritture per raggiungere il disco.

Quest'impostazione influisce sulle prestazioni, sulla scalabilità e sull'affidabilità.

Nota: se l'area di memorizzazione dei file JMS viene usata esclusivamente per il paging dei messaggi non persistenti sul disco, il criterio di scrittura sincrona viene ignorato.

Dopo aver effettuato le selezioni e le immissioni necessarie, fare clic su Successivo.
