Configura computer

In un dominio WebLogic le definizioni di computer identificano le unità hardware fisiche e sono associate alle istanze del server WebLogic che si trovano su tali unità.

Utilizzare questa schermata per aggiungere o eliminare computer oppure per modificare le impostazioni di un computer esistente.

Selezionare la scheda Computer per Windows e la scheda Computer Unix per UNIX.

È possibile creare definizioni di computer nei casi riportati di seguito.

  • Il server di amministrazione e l'applicazione Node Manager utilizzano la definizione di computer per avviare i server remoti.
  • Il server WebLogic utilizza i nomi dei computer configurati per determinare il server in un cluster più idoneo a gestire determinati task, come ad esempio la replica della sessione HTTP. Il server WebLogic delega quindi l'esecuzione di questi task al server identificato.

Nota: è necessario configurare i computer per ciascuna installazione di prodotto che esegue un processo di Node Manager. Nella configurazione del computer devono essere inclusi i valori dei parametri per l'indirizzo di ascolto e il numero di porta.

A ogni computer sono assegnate le impostazioni di configurazione riportate di seguito. I campi obbligatori sono contrassegnati da un asterisco (*).

  1. Selezionare la scheda Computer per Windows e la scheda Computer Unix per UNIX.
  2. Rivedere la lista corrente di configurazioni e aggiungere, eliminare o modificare le impostazioni come necessario per il dominio. Nella tabella seguente vengono descritti i campi disponibili in questa schermata. I campi contrassegnati da un asterisco (*) sono obbligatori.

    Campo Descrizione
    Nome* Immettere un nome valido per il computer.

    Il nome del computer viene usato per identificare il computer all'interno del dominio del server WebLogic e non è necessario che corrisponda al nome di rete per il computer. Il nome deve essere univoco tra tutti i nomi di componente all'interno del dominio. Il valore predefinito per questo campo è new_[Unix]Machine_n, dove n è un valore numerico usato per effettuare una differenziazione tra tutti i nomi di computer predefiniti; il valoren per il primo computer è 1. Il valore viene aumentato di 1 per ogni computer che viene aggiunto.
    Indirizzo di ascolto di Node Manager Selezionare un valore dalla lista per l'indirizzo usato dal Node Manager per l'ascolto delle richieste di connessione.

    Per impostazione predefinita, nella lista vengono visualizzati gli indirizzi IP definiti per il sistema locale e il localhost. Il valore predefinito è localhost. Se si specifica un indirizzo IP per un computer che ospita il server di amministrazione ed è necessario accedere al Node Manager del server WebLogic, disabilitare la verifica del nome host.

    Per ulteriori informazioni, vedere Uso della verifica del nome host in Protezione di Oracle WebLogic Server.
    Porta di ascolto di Node Manager Immettere un valore valido per la porta di ascolto usata dal Node Manager per l'ascolto delle richieste di connessione.

    L'intervallo valido dei numeri di porta di ascolto di Node Manager è compreso tra 1 e 65534. Il valore predefinito è 5556.
    GID post-autenticazione abilitato (Solo computer UNIX) Selezionare questa casella di controllo per abilitare l'autenticazione di un server in esecuzione su questo computer a un ID gruppo (GID) UNIX dopo il completamento di tutte le azioni di avvio con privilegi (vedere la descrizione del campo GID post-autenticazione di seguito). Per impostazione predefinita, questa casella di controllo non è selezionata.
    GID post-autenticazione (Solo computer UNIX) Immettere un ID gruppo (GID) UNIX valido con il quale un server che si trova su questo computer verrà eseguito dopo aver completato tutte le azioni di avvio con privilegi. Altrimenti, il server continuerà a essere eseguito con l'ID gruppo con cui è stato avviato.

    Perché questa impostazione diventi effettiva, è necessario selezionare la casella di controllo GID post-autenticazione abilitato.

    GID post-autenticazione abilitato (Solo computer UNIX) Selezionare questa casella di controllo per abilitare l'autenticazione di un server che si trova su questo computer a un ID utente (UID) UNIX dopo aver completato tutte le azioni di avvio con privilegi (vedere la descrizione del campo UID post-autenticazione riportata di seguito). Per impostazione predefinita, questa casella di controllo non è selezionata.
    UID post-autenticazione (Solo computer UNIX) Immettere un ID utente (UID) UNIX valido con il quale un server che si trova su questo computer verrà eseguito dopo aver completato tutte le azioni di avvio con privilegi. Altrimenti, il server continuerà a essere eseguito con l'ID utente con cui è stato avviato.

    Perché questa impostazione diventi effettiva, è necessario selezionare la casella di controllo UID post-autenticazione abilitato.
  3. Dopo aver effettuato le selezioni e le immissioni necessarie, fare clic su Successivo.

Fare clic su Aggiungi per aggiungere un nuovo computer. Il nome predefinito per un nuovo computer è new_[Unix]Machine_n, dove n ha inizialmente il valore 1 che viene incrementato di un'unità per ogni computer aggiunto.

Fare clic su Elimina per eliminare un computer esistente.
