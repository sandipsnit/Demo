Indirizza servizi a cluster o server

Utilizzare questa schermata per indirizzare i servizi ai server gestiti o ai cluster appropriati.

I servizi associati al prodotto per cui si sta configurando il dominio (ad esempio, SOA) vengono indirizzati automaticamente al server gestito creato per questo prodotto o al cluster a cui è assegnato tale server gestito. In questa schermata è possibile indirizzare i servizi a server e cluster aggiuntivi.

  1. Nel riquadro Destinazione selezionare il cluster o server su cui si desidera distribuire i servizi.

    Il nome della destinazione selezionata viene visualizzato come titolo del riquadro destro.

  2. Nel riquadro destro selezionare le caselle di controllo corrispondenti ai servizi che si desidera distribuire sulla destinazione selezionata.

    I servizi visualizzati qui variano in base ai prodotti selezionati in precedenza nella schermata Seleziona origine dominio della procedura guidata.

    Quando si seleziona un server gestito nel riquadro Destinazione, alcune delle caselle di controllo del riquadro destro potrebbero essere disabilitate indicando i servizi già indirizzati al cluster contenente il server gestito selezionato.

    Dopo aver selezionato i servizi, i nomi dei cluster e dei server di destinazione vengono visualizzati nella colonna Destinazione nel riquadro destro.

  3. Ripetere i passi 1 e 2 per gli altri cluster e server come richiesto.
  4. Dopo aver effettuato le selezioni necessarie, fare clic su Successivo.

Quando si estende un dominio, se si elimina un server gestito o un cluster a cui vengono attualmente indirizzati i servizi, la Configurazione guidata reindirizza in modo automatico questi servizi come indicato di seguito.

  • Se i servizi sono stati indirizzati inizialmente solo al server gestito o al cluster che si sta eliminando (ovvero, dopo l'eliminazione del server gestito o del cluster, i servizi non saranno più indirizzati nel dominio modificato), la Configurazione guidata reindirizza in modo automatico i servizi a tutte le destinazioni idonee.

    Una destinazione idonea è un cluster o un server gestito non definito nella specifica dei gruppi di configurazione (file config-groups.xml) di un modello incluso. I server o i cluster specificati nel file config-groups.xml sono essenzialmente di proprietà del modello e, pertanto, non sono idonei per l'indirizzamento automatico.

  • Se i servizi sono stati indirizzati inizialmente a più destinazioni (inclusi server gestiti, cluster e il server di amministrazione) e uno dei server gestiti o dei cluster indirizzati viene eliminato, nel dominio esteso la Configurazione guidata lascia le restanti associazioni di destinazione intatte e non tenta di reindirizzare i servizi.
