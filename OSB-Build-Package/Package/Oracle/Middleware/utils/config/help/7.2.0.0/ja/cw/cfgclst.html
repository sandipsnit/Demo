クラスタの構成

この画面では、クラスタの追加、構成および削除を行います。

クラスタはWebLogic Serverインスタンスのグループであり、互いに連携してアプリケーションのスケーラビリティと高可用性を実現します。クラスタを作成することで、アプリケーションまたはリソースをホストする1つの単位として動作するように管理対象サーバーをグループ化できます。

  1. 現在のクラスタ構成を確認します。選択したドメイン・ソースによってデフォルト値は異なります。

    注意: 2種類の表示方法があります。すべてのクラスタを簡潔な表形式で表示する方法と、各クラスタを個別に表示（1つのクラスタを1つのタブに表示）する方法です。個別表示で他のクラスタに表示を切り替えるには、該当するタブを選択します。表形式とタブ形式の表示モードを切り替えるには、「表示の切替え」をクリックします。

  2. ドメインの要求に応じて、クラスタを追加または削除する、あるいは既存のクラスタの設定を変更します。

    次の表では、この画面のフィールドについて説明します。アスタリスク（*）が付いているフィールドは必須フィールドです。

    フィールド 説明
    名前* 有効なクラスタ名を入力します。

    クラスタの名前は、ドメイン内のすべてのコンポーネント名の間で一意であることが必要です。このフィールドのデフォルト値はnew_Cluster_nであり、nはすべてのデフォルトのクラスタ名を区別する数値を表します。1つ目のクラスタのnは1になります。値はクラスタを追加するたびに1ずつ増えます。
    Cluster messaging mode ユニキャストまたはマルチキャスト・メッセージング・モードを選択します。
    Multicast address このフィールドはマルチキャスト・メッセージング・モードを選択したときにのみ有効です。

    クラスタのマルチキャスト・アドレスを入力します。このアドレスはクラスタ・メンバーが相互に通信するために使用されます。デフォルトは239.192.0.0です。有効なマルチキャスト・アドレスの範囲は224.0.0.1から239.255.255.255です。
    Multicast port このフィールドはマルチキャスト・メッセージング・モードを選択したときにのみ有効です。

    クラスタのマルチキャスト・ポートを入力します。このマルチキャスト・ポートはクラスタ・メンバーが相互に通信するために使用されます。デフォルトは7001です。
    Cluster address クラスタ内の管理対象サーバーを識別するクラスタ・アドレスを入力します。クラスタ・アドレスには次のいずれかを指定できます。
    • IPアドレスまたはDNS名とポートのカンマ区切りリスト（例: dns_name:port, dns_name:port）
    • 複数のIPアドレスにマップするDNS名
    • すべての管理対象サーバーのリスニング・アドレスが、一意なポート番号を持つ同一アドレスをリスニングしている場合は、localhost、DNS名、またはIPアドレス

    クラスタ・アドレスはエンティティEJBおよびステートレスEJBで使用され、URLのホスト名の部分を構成します。クラスタ・アドレスを設定しないと、EJBハンドルが適切に動作しない場合があります。
  3. 必要な選択と入力が完了した後、「次へ」をクリックします。

クラスタを削除するには、サーバーを選択し、「削除」をクリックします。クラスタの削除では、割り当てられているサーバーは削除されません。サーバーは単にクラスタから解除され、別のクラスタに追加できます。一度に1つのクラスタを削除できます。

クラスタを追加するには、「追加」をクリックし、新しいサーバーの設定を構成します。新しいクラスタのデフォルトの名前はnew_Cluster_nです。nは1から始まり、新しいクラスタが追加されるたびに増加します。デフォルトを変更するには、「名前」列に目的の名前を入力します。

デフォルトの表形式からタブ化形式（各クラスタごとに1つのタブを使用）に表示を切り替えるには、「表示の切替え」をクリックします。表形式に戻るには、「表示の切替え」をもう一度クリックします。
