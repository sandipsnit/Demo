マシンの構成

WebLogicドメインでは、マシン定義によって物理ハードウェア単位が識別され、それらの定義はホストするWebLogic Serverインスタンスに関連付けられます。

この画面では、マシンの追加と削除、および既存マシンの設定の変更を行います。

「マシン」タブ（Windowsの場合）および「Unixマシン」タブ（UNIXの場合）を選択します。

次のような場合は、マシン定義を作成する必要があります。

  • 管理サーバーがリモート・サーバーを開始するために、ノード・マネージャ・アプリケーションとともにマシン定義を使用する。
  • WebLogic Serverが、HTTPセッションのレプリケーションなどの特定のタスクを処理するのに最適なクラスタ内のサーバーを判断するために、構成されたマシン名を使用する。WebLogic Serverは識別されたサーバーにそれらのタスクを委任します。

注意: ノード・マネージャ・プロセスを実行する各製品インストールごとに、マシンを構成する必要があります。マシンの構成には、リスニング・アドレスおよびポート番号のパラメータ値が含まれている必要があります。

各マシンの構成の設定は、次のとおりです。アスタリスク（*）は必須フィールドを示します。

  1. 「マシン」タブ（Windowsの場合）および「Unixマシン」タブ（UNIXの場合）を選択します。
  2. 構成の現在のリストを確認し、ドメインの必要に応じて追加、削除、または設定を変更します。次の表では、この画面のフィールドについて説明します。アスタリスク（*）が付いているフィールドは必須フィールドです。

    フィールド 説明
    名前* 有効なマシン名を入力します。

    マシン名は、WebLogic Serverドメイン内のマシンを識別するために使用されます。マシンのネットワーク名と一致している必要はありません。ドメイン内のすべてのコンポーネント名の間で一意であることが必要です。このフィールドのデフォルト値はnew_[Unix]Machine_nであり、nはすべてのデフォルトのマシン名を区別する数値を表します。1つ目のマシンのnは1になります。値はマシンを追加するたびに1ずつ増えます。
    ノード・マネージャ・リスニング・アドレス リストからノード・マネージャで接続要求のリスニングに使用するリスニング・アドレスの値を選択します。

    デフォルトでは、ローカル・システムに定義されているIPアドレスとlocalhostがリストに表示されます。デフォルトはlocalhostです。管理サーバーをホストするマシンのIPアドレスを指定し、WebLogic Serverノード・マネージャにアクセスする必要がある場合は、ホスト名の検証を無効にする必要があります。

    詳細は、Oracle WebLogic Serverの保護のホスト名の検証の使用を参照してください。
    ノード・マネージャ・リスニング・ポート ノード・マネージャで接続要求のリスニングに使用するリスニング・ポートの有効な値を入力します。

    ノード・マネージャの有効なリスニング・ポート番号の範囲は1から65534までです。デフォルトは5556です。
    バインド後のGIDの有効化 （UNIXマシンのみ）このチェック・ボックスを選択すると、このマシンで実行されるサーバーは、すべての権限付起動アクションの完了後に、UNIXグループID（GID）にバインドされます（詳細は次の「バインド後のGID」フィールドを参照）。デフォルトでは、このチェック・ボックスは選択されていません。
    バインド後のGID （UNIXマシンのみ）このマシンにホストされるサーバーが、すべての権限付起動アクションの完了後に動作する際に使用する有効なUNIXグループID （GID）を入力します。このGIDを入力しない場合、サーバーは起動時のグループIDを使用して動作し続けます。

    この設定を有効にするには、「バインド後のGIDの有効化」チェック・ボックスをチェックする必要があります。

    バインド後のUIDの有効化 （UNIXマシンのみ）このチェック・ボックスを選択すると、このマシンにホストされるサーバーは、すべての権限付起動アクションの完了後に、UNIXユーザーID（UID）にバインドされます（詳細は次の「バインド後のUID」フィールドを参照）。デフォルトでは、このチェック・ボックスは選択されていません。
    バインド後のUID （UNIXマシンのみ）このマシンにホストされるサーバーが、すべての権限付起動アクションの完了後に動作する際に使用する有効なUNIXユーザーID （UID）を入力します。このUIDを入力しない場合、サーバーは起動時のユーザーIDを使用して動作し続けます。

    この設定を有効にするには、「バインド後のUIDの有効化」チェック・ボックスをチェックする必要があります。
  3. 必要な選択と入力が完了した後、「次へ」をクリックします。

新しいマシンを追加するには、「追加」をクリックします。新しいマシンのデフォルト名はnew_[Unix]Machine_nです。nは1から始まり、マシンを追加するたびに1ずつ増加します。

既存のマシンを削除するには、「削除」をクリックします。

詳細は、Oracle Technology NetworkのOracle Fusion Middlewareのドキュメント・ライブラリを参照してください。
