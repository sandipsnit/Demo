RDBMSセキュリティ・ストア・データベースの構成

この画面では、RDBMSセキュリティ・ストアの設定を定義します。

次の表で、この画面のフィールドについて説明します。

フィールド 説明
Database Type RDBMSセキュリティ・ストアとして使用するデータベースのタイプをドロップダウン・リストから選択します。
Driver データベースで使用するドライバを選択します。使用可能なドライバのリストは、選択したデータベース・タイプによって異なります。
Class Name アクションは必要ありません。選択したドライバに基づいて、クラス名が自動的に表示されます。
DBMS SID データベースのSIDを入力します。
DBMS Host データベースをホストするマシンの名前を入力します。
DBMS Port サーバーへの接続に使用するポートを入力します。選択したデータベース・タイプに関連するデフォルトのポート番号が自動的に表示されます。
URL アクションは必要ありません。選択したドライバに基づいて、URLが自動的に表示されます。
User Name データベースに接続するログイン名を入力します。
User Password データベースにアクセスするパスワードを入力します。

有効な値は英数字文字列で、ハイフン（-）と下線（_）を使用できます。
値は暗号化されます。
Confirm User Password パスワードを再入力します。
Known Properties アクションは必要ありません。選択したドライバに基づいて、データベースの既知のプロパティが自動的に表示されます。
Additional PropertiesAdditional Properties ドライバに渡す追加のプロパティがあれば入力します。

注意: 「接続のテスト」をクリックすることでデータベースへの接続をテストできます。サーバーを起動する前に、RDBMSセキュリティ・ストアに必要なSQLスクリプトをロードする必要があります。クラスタ化されたドメインでRDBMSセキュリティ・ストアを使用する場合は、JMS構成（JNDI名およびJMSトピック）を使用することをお薦めします。

必要な選択と入力を行った後、ウィザードで「次へ」ボタンをクリックします。

詳細は、Oracle Technology NetworkのOracle Fusion Middlewareのドキュメント・ライブラリを参照してください。
