HTTPプロキシ・アプリケーションの作成

この画面を使用して、各クラスタにHTTPプロキシ・アプリケーションを作成し、このプロキシ・アプリケーションをデプロイする必要がある管理対象サーバーを指定できます。HTTPプロキシ・アプリケーションはHTTPリクエストの仲介役として動作します。

この画面は、次の両方に当てはまる場合にのみ表示されます。

  • 少なくとも1つの管理対象サーバーがクラスタに割り当てられている。
  • 少なくとも1つの管理対象サーバーがどのクラスタにも割り当てられていない。
  1. HTTPプロキシ・アプリケーションを作成するクラスタの「HTTPプロキシの作成」チェック・ボックスをチェックします。

    「プロキシ・サーバー」リストには、クラスタに割り当てられていない管理対象サーバーが表示されます。

  2. 「プロキシ・サーバー」リストから、プロキシ・アプリケーションをデプロイする必要がある管理対象サーバーを選択します。

    OracleProxy4_<clustername>_<servername>という名前のプロキシ・アプリケーションが作成されて、管理対象サーバーに割り当てられます。

  3. HTTPプロキシ・アプリケーションを作成する各クラスタに対して、手順1から2を繰り返します。
  4. 必要な選択が完了した後、「次へ」をクリックします。

詳細は、Oracle Technology NetworkのOracle Fusion Middlewareのドキュメント・ライブラリを参照してください。
