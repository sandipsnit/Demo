관리 서버 구성

관리 서버는 도메인을 관리하는 중앙 지점입니다. 관리 서버는 URL protocol://listen-address:listen-port를 사용하여 액세스할 수 있습니다. 프로토콜은 t3, t3s, http, https 중 하나가 될 수 있습니다.

이 화면에서는 다음과 같은 관리 서버 설정을 구성하거나 변경할 수 있습니다. 별표(*)로 표시된 필드는 필수 필드입니다.

필드 설명
*이름 기본 이름은 AdminServer입니다. 적합한 서버 이름은 문자열(영숫자)입니다.

운영 환경의 각 서버 인스턴스는 해당 인스턴스가 도메인에 상주하는지 클러스터에 상주하는지 또는 관리 서버인지에 상관없이 고유한 이름을 가져야 합니다. 또한 관리 서버 이름은 도메인 내의 모든 구성 요소 이름 중에서 고유해야 합니다.

주: 이 값은 식별용으로만 지정되며 서버에 배치된 애플리케이션에 대한 URL의 일부로는 사용되지 않습니다. 서버 이름은 WebLogic Server 관리 콘솔에 표시됩니다. 또한 WebLogic Server 명령행 유틸리티 또는 API를 사용할 경우에는 이 이름을 지정하여 서버를 식별해야 합니다.

*수신 주소 드롭다운 목록에서 수신 주소 값을 선택합니다.

서버 인스턴스의 수신 주소로 localhost를 선택하면 원격 프로세스는 해당 서버 인스턴스에 연결할 수 없습니다. 서버 인스턴스를 호스트하는 시스템의 프로세스만 서버 인스턴스에 연결할 수 있습니다. 서버 인스턴스가 로컬 호스트에 액세스할 수 있고(예를 들어, 로컬 호스트에 연결하는 관리 스크립트를 생성한 경우), 원격 프로세스에도 액세스할 수 있도록 하려면 모든 로컬 주소를 선택합니다. 서버 인스턴스는 시스템 주소를 확인하고 해당 주소에서 수신합니다.

자세한 내용은 수신 주소 지정을 참조하십시오.
수신 포트 보안이 되지 않는 일반 요청(HTTP 및 T3 등의 프로토콜을 통한 요청)에 사용되는 수신 포트의 적합한 값을 입력합니다.

관리 서버에 대한 기본값은 7001입니다. 이 필드를 공백으로 두면 기본값이 사용됩니다.

적합한 수신 포트 범위는 1-65534입니다.

자세한 내용은 수신 포트 지정을 참조하십시오.
SSL 사용으로 설정 SSL 수신 포트를 사용으로 설정하려면 이 체크 박스를 선택합니다. 기본적으로 모든 새 서버에 대해 SSL은 사용 안함으로 설정됩니다.
SSL 수신 포트 SSL 사용으로 설정 체크 박스를 선택한 경우에만 이 필드를 사용할 수 있습니다.

보안 요청(HTTPS 및 T3S 등의 프로토콜을 통한 요청)에 사용되는 적합한 값을 입력합니다. 기본값은 7002입니다. 이 필드를 공백으로 두면 기본값이 사용됩니다.

적합한 수신 포트 범위는 1-65535입니다.

주: 기본적으로 서버 인스턴스는 데모 인증서를 사용하여 보안 포트의 요청을 인증합니다. 운영 환경의 경우 인증 기관의 인증서를 사용하려면 SSL을 구성해야 합니다.

필수 항목을 선택하고 입력한 후 다음을 누릅니다.

자세한 내용은 Oracle Technology Network의 Oracle Fusion Middleware 설명서 라이브러리를 참조하십시오.

수신 주소 지정

이 섹션에서는 관리 서버에 대한 수신 주소 지정 지침을 제공합니다. 이러한 지침은 모든 관리되는 서버에도 적용됩니다.

수신 주소 유형 동작
모든 로컬 주소 또는 DNS 이름 다중 홈 Windows 시스템에서 서버 인스턴스가 모든 사용 가능한 IP 주소에 바인드됩니다.
IP 주소 또는 DNS 이름
  • 서버 인스턴스에 접속하기 위해 프로세스가 IP 주소 또는 해당하는 DNS 이름을 지정할 수 있습니다.
  • 로컬 호스트를 지정하는 프로세스는 접속을 실패합니다.
  • 서버 인스턴스에 접속하려면 로컬 호스트를 사용하는 기존 프로세스를 갱신해야 합니다.
  • 수신 주소에 대해 IP 주소를 지정하고 수신 포트에 대해 보안 포트를 지정하는 접속의 경우 호스트 이름 확인을 사용 안함으로 설정해야 합니다.

주: DNS 이름을 IP 주소로 지정하기 위해 WebLogic Server는 해당 DNS 서버에 접속하거나 IP 주소 매핑을 로컬에 가져올 수 있어야 합니다. 따라서 수신 주소의 DNS 이름을 지정한 경우 WebLogic Server 인스턴스가 DNS 서버 및 캐시 해당 매핑에 접속할 수 있는 충분한 시간만큼 포트를 열어 두거나 IP 주소 매핑을 로컬 파일에 지정해야 합니다. 수신 주소에 대해 IP 주소가 지정되고 클라이언트 요청에서 DNS 이름을 지정한 경우, WebLogic Server가 DNS 이름을 확인하려고 하지만 DNS 이름 매핑에 액세스할 수 없으면 요청을 실패합니다.
로컬 호스트
  • 서버 인스턴스에 접속하려면 프로세스가 로컬 호스트를 지정해야 합니다.
  • 서버 인스턴스를 호스트하는 시스템에 상주하는 프로세스(로컬 프로세스)만 서버 인스턴스에 접속할 수 있습니다.

수신 포트 지정

관리 서버에 대한 수신 포트를 지정할 때 다음 지침을 따르십시오. 이러한 지침은 관리되는 서버에도 적용됩니다.

  • 모든 적합한 포트 번호를 지정할 수 있지만, 포트 80을 지정하는 경우 HTTP를 통해 리소스에 액세스하는 데 사용되는 HTTP 요청에서 포트 번호를 생략할 수 있습니다. 예를 들어, 포트 80을 수신 포트로 정의할 경우 URL http://hostname:portnumber/myfile.html을 사용할 수 있습니다.
  • 일부 운영 체제에서 포트 80은 권한이 부여된 사용자 또는 그룹 ID로 실행된 프로세스에서만 액세스할 수 있습니다. 이 경우 호스트 바인드 UID 또는 GID가 정의된 UNIX 시스템에 서버 인스턴스를 할당할 수 있습니다.
