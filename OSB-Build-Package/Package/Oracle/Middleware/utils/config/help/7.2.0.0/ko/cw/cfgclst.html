클러스터 구성

이 화면에서는 클러스터를 추가, 구성 및 삭제할 수 있습니다.

클러스터는 WebLogic Server 인스턴스 그룹으로, 이들 인스턴스가 함께 작동하여 응용 프로그램에 확장성 및 고가용성을 제공합니다. 클러스터를 생성하여 응용 프로그램과 리소스를 호스트하는 단일 단위로 작동할 수 있도록 관리 서버를 그룹화할 수 있습니다.

  1. 현재 클러스터 구성을 검토합니다. 기본값은 선택한 도메인 소스에 준해 달라질 수 있습니다.

    주: 마법사는 두 가지 보기를 제공합니다. 하나는 모든 클러스터가 표시되는 간결한 표 형식의 보기이고, 다른 하나는 각각의 서버가 탭으로 표시되는 개별 보기입니다. 보기 모드를 표 보기와 탭 보기 간에 토글하려면 표시 전환을 누릅니다.

  2. 사용자의 도메인에 필요한 대로 클러스터를 추가 또는 삭제하거나 기존 클러스터의 설정을 변경합니다.

    다음 표에서는 이 화면의 필드를 설명합니다. 별표(*)가 있는 필드는 필수 필드입니다.

    필드 설명
    이름* 클러스터의 적합한 이름을 입력합니다.

    클러스터 이름은 도메인 내의 모든 구성 요소 이름 중에서 고유해야 합니다. 이 필드의 기본값은 new_Cluster_n입니다. 여기서 n은 모든 기본 클러스터 이름을 구분하는 데 사용되는 숫자 값입니다. 첫 번째 클러스터의 n 값은 1입니다. 이 값은 추가하는 클러스터마다 1씩 증가합니다.
    클러스터 메시징 모드 unicast 또는 multicast 메시징 모드를 선택합니다.
    멀티캐스트 주소 multicast 메시징 모드를 선택한 경우에만 이 필드가 사용으로 설정됩니다.

    클러스터의 멀티캐스트 주소를 입력합니다. 이 주소는 클러스터 멤버들이 서로 통신하는 데 사용됩니다. 기본값은 239.192.0.0입니다. 적합한 멀티캐스트 주소 범위는 224.0.0.1-239.255.255.255입니다.
    멀티캐스트 포트 multicast 메시징 모드를 선택한 경우에만 이 필드가 사용으로 설정됩니다.

    클러스터의 멀티캐스트 포트를 입력합니다. 멀티캐스트 포트는 클러스터 멤버들이 서로 통신하는 데 사용됩니다. 기본값은 7001입니다.
    클러스터 주소 클러스터의 관리 서버를 식별하는 주소를 입력합니다. 클러스터 주소는 다음 중 하나가 될 수 있습니다.
    • 콤마로 구분된 IP 주소 목록 또는 DNS 이름 및 포트 목록(예: dns_name:port, dns_name:port)
    • 다중 IP 주소에 매핑되는 DNS 이름
    • 모든 관리 서버의 수신 주소가 고유 포트 번호를 갖는 동일한 주소를 수신하는 경우, localhost, DNS 이름 또는 IP 주소

    클러스터 주소는 엔티티 및 stateless EJB에서 URL의 호스트 이름 부분을 생성하는 데 사용됩니다. 클러스터 주소가 설정되지 않으면 EJB 핸들이 제대로 작동하지 않을 수 있습니다.
  3. 필수 항목을 선택하고 입력한 후 다음을 누릅니다.

클러스터를 삭제하려면 서버를 선택하고 삭제를 누릅니다. 클러스터를 삭제할 때 할당된 서버는 삭제되지 않습니다. 서버는 단지 클러스터에서 제거되며, 다른 클러스터에 추가할 수 있습니다. 한 번에 하나의 클러스터만 삭제할 수 있습니다.

클러스터를 추가하려면 추가를 누르고 새 서버에 대한 설정을 구성합니다. 새 클러스터에 대한 기본 이름은 new_Cluster_n입니다. 여기서 n은 1부터 시작하고 추가하는 클러스터마다 1씩 증가합니다. 기본값을 변경하려면 원하는 이름을 [이름] 열에 입력합니다.

표시를 기본 표 형식에서 탭 형식(각 클러스터에 대해 하나의 탭)으로 전환하려면 표시 전환을 누릅니다. 표시 전환을 다시 누르면 표 형식으로 돌아갑니다.
