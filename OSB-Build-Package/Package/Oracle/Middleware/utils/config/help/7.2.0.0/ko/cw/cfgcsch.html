JDBC 구성 요소 스키마 구성

일부 Fusion Middleware 구성 요소(예: SOA 및 Web Center Spaces)의 경우 JDBC 데이터 소스가 RCU(저장소 생성 유틸리티) 사용으로 로드(설치 중)되는 구성 요소 데이터베이스 스키마의 일부로 정의될 수 있습니다.

구성 마법사를 사용하여 각 구성 요소에 대한 WebLogic 도메인을 생성할 때 데이터베이스 드라이버, 스키마 소유자, 비밀번호 등과 같은 JDBC 구성 요소 스키마 설정을 구성할 수 있습니다.

도메인을 생성하는 제품과 연관된 JDBC 구성 요소 스키마(예를 들어, SOA)는 화면의 아래쪽에 표시됩니다.

주: RAC 데이터 소스로 하나 이상의 스키마를 구성하려면 필수 스키마 이름 옆에 있는 체크 박스를 선택하고 다음 패널에서 선택한 구성 요소 스키마를 RAC 다중 데이터 소스 스키마로 구성합니다 체크 박스를 선택합니다. 다음 단추를 누릅니다.

  1. 스키마 이름 옆의 체크 박스를 선택하여 데이터 소스 설정을 지정하려는 스키마를 선택합니다.

    주: 다중 구성 요소 스키마를 선택하면 "구성 요소 스키마에 따라 다릅니다"라는 텍스트가 특정 필드에 표시되며, 이는 이러한 필드의 현재 값이 선택된 구성 요소 스키마 간에 같지 않다는 뜻입니다. 계속 진행하여 이러한 필드의 값을 변경하면 새 값이 선택된 구성 요소 스키마 간에 동일하게 적용됩니다.

    주: 공급업체, 드라이버, 호스트 이름 및 포트 번호와 같은 구성 요소 스키마 매개변수의 기본값은 응용 프로그램 템플리트에 지정된 값에 따라 다릅니다.

  2. 현재 구성 설정을 검토하고 필요한 경우 이를 수정합니다.

    주: 공급업체, 드라이버, 호스트 이름 및 포트 번호와 같은 스키마 매개변수의 기본값은 응용 프로그램 템플리트에 지정된 값에 따라 다릅니다.

    필드 설명
    공급업체 데이터베이스 공급업체를 선택합니다.
    드라이버 데이터베이스에 연결하는 데 사용할 JDBC 드라이버를 선택합니다.

    이 목록은 선택된 데이터베이스 공급업체의 JDBC 드라이버를 포함합니다.
    스키마 소유자 데이터베이스 연결을 위한 로그인 이름을 입력합니다.
    스키마 비밀번호 지정된 스키마 소유자의 로그인 비밀번호를 입력합니다.
    DBMS/서비스 데이터베이스 DBMS 이름을 입력하거나 서비스 이름을 입력합니다(서비스 유형 드라이버를 선택한 경우).
    호스트 이름 데이터베이스를 호스트하는 서버 이름을 입력합니다.
    포트 데이터베이스를 호스트하는 서버에 연결하는 데 사용되는 포트 번호를 입력합니다.

    선택된 스키마에 대해 지정한 값은 스키마 목록의 해당 열에 표시됩니다.

  3. 필수 선택 사항을 선택하고 입력한 다음 마법사의 다음 단추를 눌러 구성 요소 스키마를 테스트합니다.
