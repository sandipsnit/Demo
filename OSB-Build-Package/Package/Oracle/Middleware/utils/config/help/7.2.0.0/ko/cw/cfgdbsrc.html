JDBC 데이터 소스 구성

이 화면에서는 도메인 소스에서 정의한 JDBC 데이터 소스를 구성할 수 있습니다.

JDBC 데이터 소스에는 데이터베이스 접속 풀이 포함되어 있습니다. 이 데이터베이스 접속 풀은 데이터 소스 인스턴스를 생성할 때, 즉 데이터 소스 인스턴스를 배치하거나 대상으로 지정할 때 또는 서버를 시작할 때 생성됩니다. 응용 프로그램은 JNDI 트리에서 데이터 소스를 조회한 다음 접속을 요청합니다. 응용 프로그램에서 접속이 필요하지 않으면 데이터 소스의 접속 풀로 접속을 반환합니다.

도메인을 생성하는 제품과 연관된 JDBC 데이터 소스는 화면의 아래쪽에 표시됩니다.

주: RAC 데이터 소스로 하나 이상의 데이터 소스를 구성하려면 필수 데이터 소스 이름 옆에 있는 체크 박스를 선택하고 다음 패널에서 선택한 데이터 소스를 RAC 다중 데이터 소스로 구성합니다 체크 박스를 선택합니다. 다음 단추를 누릅니다.

  1. 데이터 소스 이름 옆의 체크 박스를 선택하여 설정을 지정할 데이터 소스를 선택합니다.

    주: 다중 데이터 소스를 선택하면 "데이터 소스에 따라 다릅니다"라는 텍스트가 특정 필드에 표시되며, 이는 이러한 필드의 현재 값이 선택된 데이터 소스 간에 같지 않다는 뜻입니다. 계속 진행하여 이러한 필드의 값을 변경하면 새 값이 선택된 데이터 소스 간에 동일하게 적용됩니다.

    주: 공급업체, 드라이버, 호스트 이름 및 포트 번호와 같은 데이터 소스 매개변수의 기본값은 응용 프로그램 템플리트에 지정된 값에 따라 다릅니다.

  2. 현재 구성 설정을 검토하고 필요한 경우 이를 변경합니다.

    다음 표에서는 이 화면의 필드를 설명합니다.

    주: 공급업체, 드라이버, 호스트 이름 및 포트 번호와 같은 데이터 소스 매개변수의 기본값은 응용 프로그램 템플리트에 지정된 값에 따라 다릅니다.

    필드 설명
    공급업체 데이터베이스 공급업체를 선택합니다.
    드라이버 데이터베이스에 연결하는 데 사용할 JDBC 드라이버를 선택합니다.

    이 목록은 선택된 데이터베이스 공급업체의 공통 JDBC 드라이버를 포함합니다.
    사용자 이름 데이터베이스 연결을 위한 로그인 이름을 입력합니다.
    비밀번호 지정된 사용자 이름의 로그인 비밀번호를 입력합니다.
    DBMS/서비스 DBMS SID 또는 서비스 이름을 입력합니다. 입력하는 값은 선택한 드라이버에 따라 달라집니다.
    • 선택한 Oracle 드라이버의 이름에 "인스턴스 접속용"이라는 단어가 포함되어 있으면 SID를 입력해야 합니다.
    • Oracle 드라이버의 이름에 "서비스 접속용"이라는 단어가 포함되어 있으면 서비스 이름을 입력해야 합니다.
    • DataDirect 드라이버에 대한 자세한 내용은 DataDirect 설명서를 참조하십시오.
    호스트 이름 데이터베이스를 호스트하는 서버 이름을 입력합니다.
    포트 데이터베이스를 호스트하는 서버에 연결하는 데 사용되는 포트를 입력합니다.

    선택된 데이터 소스에 대해 지정한 값은 데이터 소스 목록의 해당 열에 표시됩니다.

  3. 필수 항목을 선택하고 입력한 후 다음을 눌러 JDBC 데이터 소스에 대한 접속을 테스트합니다.
