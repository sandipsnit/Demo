JMS 파일 저장소 구성

JMS 파일 저장소는 영구 메시지를 저장할 수 있는 디스크 기반 파일입니다.

JMS 파일 저장소의 현재 설정을 검토하고 필요한 경우 이를 수정합니다. 기본값은 이전에 선택한 도메인 소스에 준해 달라질 수 있습니다.

주: 마법사는 두 가지 표시 모드를 제공합니다. 하나는 모든 정의된 구성 요소가 표시되는 간결한 표 형식의 보기이고, 다른 하나는 각각의 구성 요소가 탭으로 표시되는 개별 보기입니다. 보기 모드를 표 보기와 탭 보기 간에 토글하려면 표시 전환을 누릅니다.

다음 표에서는 이 화면의 필드를 설명합니다. 별표(*)가 있는 필드는 필수 필드입니다.

필드 설명
이름* JMS 파일 저장소의 이름을 입력합니다.

이름은 문자열이어야 하며 공백을 포함할 수 있습니다. JMS 파일 저장소 이름은 도메인 내의 모든 구성 요소 이름 중에서 고유해야 합니다.
디렉토리 JMS 파일 저장소가 상주하는 디렉토리(시스템 내) 경로를 입력합니다.
동기화 쓰기 정책 다음 동기화 쓰기 정책 중 하나를 선택하여 파일 저장소가 디스크에 데이터를 쓰는 방법을 결정합니다.
  • 캐시 비우기: 해당하는 모든 쓰기 작업이 디스크에 비워질 때까지 트랜잭션을 완료할 수 없습니다.
  • 직접 쓰기: 쓰기 작업이 디스크에 직접 수행됩니다. 이 정책은 Solaris와 Windows에서 지원됩니다. 지원되지 않는 플랫폼에서 이 정책이 활성화되면 파일 저장소가 자동으로 캐시 비우기 정책으로 전환됩니다.

    주: Solaris 시스템에서는 직접 쓰기 정책이 안정적이지만 Windows 시스템은 디스크에 즉시 쓰지 않은채 디스크 캐시에 트랜잭션 데이터를 남길 수 있습니다. 정전으로 인해 디스크의 캐시 데이터가 손상되어 메시지가 손실되거나 중복될 수 있기 때문에 이는 안정적이지 않습니다. Windows에서 직접 쓰기 정책을 사용하여 안정적으로 쓰기 작업을 하려면 디스크에 대한 모든 쓰기 캐시 작업을 사용 안함으로 설정하거나 (기본적으로 사용으로 설정됨) 배터리가 지원되는 캐시를 사용하는 디스크를 사용합니다. 하지만 일부 파일 시스템(예를 들어, 안정적인 캐시를 가지고 있는 RAID 시스템)은 이 값의 변경을 허용하지 않습니다.

  • 사용 안함으로 설정됨: 쓰기 작업이 메모리에 캐시되는 즉시 트랜잭션이 완료됩니다. 이 정책이 활성이면 디스크에 쓸 때까지 기다리지 않고 트랜잭션을 완료할 수 있습니다.

이 설정은 성능, 확장성 및 신뢰성에 영향을 줍니다.

주: 지속적이지 않은 메시지를 디스크에 페이징하는 데 JMS 파일 저장소가 배타적으로 사용되는 경우에는 동기화 쓰기 정책이 무시됩니다.
