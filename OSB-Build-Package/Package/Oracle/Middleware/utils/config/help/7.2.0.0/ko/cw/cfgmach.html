시스템 구성

WebLogic 도메인에서 시스템 정의는 하드웨어의 물리적 단위를 식별하고 호스트하는 WebLogic Server 인스턴스와 연관됩니다.

이 화면에서는 시스템을 추가 또는 삭제하거나 기존 시스템의 설정을 수정할 수 있습니다.

시스템 탭(Windows의 경우) 또는 [UNIX 시스템] 탭(UNIX의 경우)을 선택합니다.

다음과 같은 상황에서 시스템 정의를 생성할 수 있습니다.

  • 관리 서버에서 노드 관리자 응용 프로그램과 함께 시스템 정의를 사용하여 원격 서버를 시작합니다.
  • HTTP 세션 복제와 같은 특정 작업을 가장 잘 처리할 수 있는 클러스터의 서버를 결정할 때 WebLogic Server에서 구성된 시스템 이름을 사용합니다. 그러면 WebLogic Server는 이러한 작업을 식별된 서버에 위임합니다.

주: 노드 관리자 프로세스를 실행하는 각 제품 설치에 대해 시스템을 구성해야 합니다. 시스템 구성에는 수신 주소 및 포트 번호 매개변수에 대한 값이 포함되어야 합니다.

각 시스템에는 다음 구성 설정이 있어야 합니다. 별표(*)로 표시된 필드는 필수 필드입니다.

  1. 시스템 탭(Windows의 경우) 및 Unix 시스템 탭(UNIX의 경우)을 선택합니다.
  2. 구성의 현재 목록을 검토하고 도메인에 필요한 경우 설정을 추가, 삭제 또는 변경합니다. 다음 표에서는 이 화면의 필드를 설명합니다. 별표(*)가 있는 필드는 필수 필드입니다.

    필드 설명
    이름* 시스템의 적합한 이름을 입력합니다.

    시스템 이름은 WebLogic Server 도메인 내에서 시스템을 식별하는 데 사용됩니다. 이 이름이 시스템의 네트워크 이름과 일치할 필요는 없습니다. 이름은 도메인 내의 모든 구성 요소 이름 중에서 고유해야 합니다. 이 필드의 기본값은 new_[Unix]Machine_n입니다. 여기서 n은 모든 기본 시스템 이름을 구분하는 데 사용되는 숫자 값입니다. 첫 번째 시스템의 n 값은 1입니다. 이 값은 추가하는 시스템마다 1씩 증가합니다.
    노드 관리자 수신 주소 노드 관리자가 연결 요청을 수신하는 데 사용하는 주소 값을 목록에서 선택합니다.

    기본적으로 로컬 시스템 및 localhost에 정의된 IP 주소가 목록에 표시됩니다. 기본값은 localhost입니다. 관리 서버를 호스트하는 시스템의 IP 주소를 지정하고 WebLogic Server 노드 관리자에 액세스해야 하는 경우 호스트 이름 확인을 사용 안함으로 설정합니다.

    자세한 내용은 Securing Oracle WebLogic Server의 Using Host Name Verification을 참조하십시오.
    노드 관리자 수신 포트 노드 관리자가 연결 요청을 수신하는 데 사용하는 적합한 수신 포트 값을 입력합니다.

    노드 관리자의 수신 포트에 대한 적합한 포트 번호 범위는 1 - 65534입니다. 기본값은 5556입니다.
    사후 바인드 GID 사용으로 설정 (UNIX 시스템만 해당) 이 체크 박스를 선택하면 이 시스템에서 실행되는 서버가 권한을 가진 모든 시작 작업을 마친 후 UNIX 그룹 ID(GID)에 바인드(아래 사후 바인드 GID 필드의 설명 참조)할 수 있습니다. 기본적으로 이 체크 박스는 선택이 해제되어 있습니다.
    사후 바인드 GID (UNIX 시스템만 해당) 이 시스템에서 호스트되는 서버가 권한을 가진 모든 시작 작업을 마친 후 실행될 때 사용할 적합한 UNIX 그룹 ID(GID)를 입력합니다. 그렇지 않으면 서버는 시작될 때 사용한 그룹 ID를 사용하여 계속 실행됩니다.

    이 설정을 적용하려면 사후 바인드 GID 사용으로 설정 체크 박스를 선택해야 합니다.

    사후 바인드 UID 사용으로 설정 (UNIX 시스템만 해당) 이 체크 박스를 선택하면 이 시스템에서 호스트되는 서버가 권한을 가진 모든 시작 작업을 마친 후 UNIX 사용자 ID(UID)에 바인드(아래 사후 바인드 UID 필드의 설명 참조)할 수 있습니다. 기본적으로 이 체크 박스는 선택이 해제되어 있습니다.
    사후 바인드 UID (UNIX 시스템만 해당) 이 시스템에서 호스트되는 서버가 권한을 가진 모든 시작 작업을 마친 후 실행될 때 사용할 적합한 UNIX 사용자 ID(UID)를 입력합니다. 그렇지 않으면 서버는 시작될 때 사용한 사용자 ID를 사용하여 계속 실행됩니다.

    이 설정을 적용하려면 사후 바인드 UID 사용으로 설정 체크 박스를 선택해야 합니다.
  3. 필수 항목을 선택하고 입력한 후 다음을 누릅니다.

새 시스템을 추가하려면 추가를 누릅니다. 새 시스템의 기본 이름은 new_[Unix]Machine_n입니다. 여기서 n은 1에서 시작하고 추가하는 시스템마다 1씩 증가합니다.

기존 시스템을 삭제하려면 삭제를 누릅니다.
