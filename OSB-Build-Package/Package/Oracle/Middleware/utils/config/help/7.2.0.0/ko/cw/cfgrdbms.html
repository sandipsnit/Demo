RDBMS 보안 저장소 데이터베이스 구성

이 화면에서는 RDBMS 보안 저장소 설정을 정의할 수 있습니다.

다음 표에서는 이 화면의 필드를 설명합니다.

필드 설명
데이터베이스 유형 드롭다운 목록에서 RDBMS 보안 저장소로 사용할 데이터베이스 유형을 선택합니다.
드라이버 데이터베이스용으로 사용할 드라이버를 선택합니다. 사용 가능한 드라이버 목록은 선택한 데이터베이스 유형에 따라 다릅니다.
클래스 이름 작업이 필요하지 않습니다. 선택한 드라이버에 준해 클래스 이름이 자동으로 표시됩니다.
DBMS SID 데이터베이스 SID를 입력합니다.
DBMS 호스트 데이터베이스를 호스트할 시스템의 이름을 입력합니다.
DBMS 포트 서버에 연결할 때 사용할 포트를 입력합니다. 선택한 데이터베이스 유형과 연관된 기본 포트 번호가 자동으로 표시됩니다.
URL 작업이 필요하지 않습니다. 선택한 드라이버에 준해 URL이 자동으로 표시됩니다.
사용자 이름 데이터베이스 연결을 위한 로그인 이름을 입력합니다.
사용자 비밀번호 데이터베이스 액세스에 필요한 비밀번호를 입력합니다.

적합한 값은 영숫자 문자로 구성됩니다. 하이픈(-)과 밑줄( _ )은 사용할 수 있습니다.
값은 비밀번호화됩니다.
사용자 비밀번호 확인 비밀번호를 다시 입력합니다.
알려진 속성 작업이 필요하지 않습니다. 선택한 드라이버에 준해 데이터베이스의 알려진 속성이 자동으로 표시됩니다.
추가 속성 드라이버로 전달되는 추가 속성(있는 경우)을 입력합니다.

주: 연결 테스트를 눌러 데이터베이스에 대한 연결을 테스트할 수 있습니다. 서버를 시작하기 전에 RDBMS 보안 저장소에 대해 필요한 SQL 스크립트를 로드해야 합니다. 클러스터화된 도메인에서 RDBMS 보안 저장소를 사용하는 경우 JMS 구성(JNDI 이름 및 JMS 항목)과 함께 사용하는 것이 권장됩니다.

필수 선택 사항을 선택하고 입력한 다음 마법사의 다음 단추를 누릅니다.
