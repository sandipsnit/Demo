클러스터 또는 서버에 배치 대상 지정

이 화면에서는 서버나 클러스터에 배치할 응용 프로그램 및 라이브러리의 대상을 지정할 수 있습니다.

도메인(예를 들어, SOA)을 구성 중인 제품과 연관된 응용 프로그램 및 라이브러리는 관리 서버가 할당되는 클러스터나 해당 제품에 대해 생성된 관리 서버에 자동으로 대상이 지정됩니다. 이 화면에서는 추가 서버 및 클러스터에 응용 프로그램 및 라이브러리의 대상을 지정할 수 있습니다.

  1. 대상 창에서 응용 프로그램이나 라이브러리를 배치할 클러스터 또는 서버를 선택합니다.

    선택한 대상의 이름은 오른쪽 창에 제목으로 표시됩니다.

  2. 오른쪽 창에서 선택한 대상에 배치할 응용 프로그램이나 라이브러리에 해당하는 체크 박스를 선택합니다.

    여기에 표시된 응용 프로그램 및 라이브러리는 마법사의 이전 도메인 소스 선택 화면에서 선택한 제품에 따라 다릅니다.

    주: 대상 창에서 관리되는 서버를 선택하면 오른쪽 창의 일부 체크 박스가 사용 안함으로 설정되며, 이는 선택한 관리되는 서버를 포함하는 클러스터에서 이미 응용 프로그램 및 라이브러리가 대상으로 지정되었음을 나타냅니다.

    응용 프로그램 및 라이브러리를 선택하면 대상으로 지정된 클러스터 및 서버의 이름이 오른쪽 창의 대상 열에 표시됩니다.

  3. 필요한 경우 다른 클러스터 및 서버에 대해 1단계와 2단계를 반복합니다.
  4. 필수 항목을 선택한 후 다음을 누릅니다.

도메인을 확장할 때 응용 프로그램이 현재 대상으로 지정된 관리되는 서버나 클러스터를 삭제할 경우 구성 마법사가 해당 응용 프로그램 대상을 다음과 같이 자동으로 재지정합니다.

  • 응용 프로그램이 현재 삭제하려는 관리되는 서버나 클러스터에만 원래 대상으로 지정된 경우(즉, 관리되는 서버나 클러스터를 삭제한 후 응용 프로그램이 수정된 도메인에서 대상으로 지정되지 않음) 구성 마법사가 응용 프로그램을 모든 자격 있는 대상에 자동으로 재지정합니다.

    자격 있는 대상은 포함된 템플리트의 구성 그룹 사양(config-groups.xml 파일)에 정의되지 않은 모든 클러스터 또는 관리되는 서버입니다. config-groups.xml에 지정된 서버나 클러스터는 원래 템플리트의 소유이므로 자동 대상 지정 자격이 없습니다.

  • 응용 프로그램이 원래 여러 대상(관리되는 서버, 클러스터 및 관리 서버 포함)에 대상으로 지정되고, 대상 지정된 관리되는 서버나 클러스터 중 하나가 확장된 도메인에서 삭제되면 구성 마법사는 나머지 대상 연관을 그대로 두고 응용 프로그램 대상 재지정을 시도하지 않습니다.
