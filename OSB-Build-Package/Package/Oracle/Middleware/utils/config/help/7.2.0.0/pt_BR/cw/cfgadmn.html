Configurar o Servidor de Administração

O Servidor de Administração é o ponto central com base no qual você gerencia seu domínio. É possível acessá-lo usando o URL protocol://listen-address:listen-port. O protocolo pode ser um dos seguintes t3, t3s, http, https.

Use esta tela para configurar ou alterar as seguintes definições do Servidor de Administração. Os campos necessários são indicados por um asterisco (*).

Campo Descrição
*Nome O nome default é AdminServer. Os nomes do servidor válidos são uma string de caracteres (alfabéticos e numéricos).

Cada instância do servidor no ambiente de produção deve ter um nome exclusivo, independentemente do domínio ou cluster em que ela reside e se é um servidor de administração ou um servidor gerenciado. Além disso, o nome do servidor de administração deve estar entre todos os nomes do componente dentro do domínio.

Observação: Este valor é especificado para fins de comunicação apenas. Ele não é usado como parte do URL para aplicações que são implantadas no servidor. O nome do servidor é exibido no console de administração do WebLogic Server. Além disso, se você utilizar os utilitários da linha de comando ou APIs do WebLogic Server, você deve especificar este nome para identificar o servidor.

*Endereço de listening Na lista suspensa, selecione um valor para o endereço de listening.

Se você selecionar host localcomo o endereço de listening para uma instância do servidor, os processos remotos não poderão conectar-se a esta instância do servidor. Apenas processos na máquina que hospeda a instância do servidor podem conectar-se à instância do servidor. Caso a instância do servidor esteja acessível como um host local (por exemplo, se você criar scripts administrativos que se conectam ao host local) e também esteja acessível para processos remotos, selecione Todos os Endereços Locais. A instância do servidor determina o endereço da máquina e executa o listening nela.

Consulte Especificando o Endereço de Listening para obter mais informações.
Porta do listening Informe um valor válido para a porta de listening a ser usada para solicitações regulares e inseguras (utilizando protocolos como o HTTP e o T3).

O valor default é 7001 para o servidor de administração. Se você deixar este campo em branco, o valor default será usado.

A faixa da porta de listening vai de 1 a 65534.

Consulte Especificando a Porta de Listening para obter mais informações.
SSL ativada Selecione esta caixa de seleção para ativar a porta de listening SSL. Por default, o SSL é desativado para todos os servidores novos.
Porta de Listening SSL Este campo é ativado somente se a caixa de seleção SSL ativada for selecionada.

Informe um valor válido a ser usado para solicitações seguras (utilizando protocolos como HTTPS e T3S). O valor default é 7002. Se você deixar este campo em branco, o valor default será usado.

A faixa da porta de listening vai de 1 a 65535.

Observação: Por default, uma instância do servidor usa certificados de demonstração para autenticar solicitações de uma porta segura. Em um ambiente de produção, você deve configurar o SSL para usar certificados de uma autoridade de certificação.

Após efetuar as seleções e entradas necessárias, clique em Próximo.

Para obter mais informações, consulte a biblioteca de documentação do Oracle Fusion Middleware no Oracle Technology Network.

Especificando o Endereço de Listening

Esta seção fornece diretrizes para especificar o endereço de listening do Servidor de Administração. Estas diretrizes também são aplicadas a todos os Servidores Gerenciados.

Tipo de Endereço de Listening Comportamento
Todos os Endereços Locais ou um nome DNS Em máquinas Windows com várias conexões de rede, uma instância do servidor é associada a todos os endereços IP disponíveis.
Um endereço IP ou nome DNS
  • Para estabelecer conexão com a instância do servidor, os processos podem especificar o endereço IP ou o nome DNS correspondente.
  • Processos que especificam a falha de conexão do host local.
  • Você deve atualizar os processos existentes que usam o host local para estabelecer conexão com a instância do servidor.
  • Para conexões que especificam o endereço IP do endereço de listening e uma porta segura da porta de listening, a verificação de nome do host deve ser desativada.

Observação: Para resolver um nome de DNS para um endereço IP, o WebLogic Server deve contatar um servidor DNS apropriado ou obter o mapeamento do endereço IP localmente. Portanto, se você especificar um nome de DNS para o endereço de listening, deve deixar uma porta aberta por um tempo suficiente para que a instância do WebLogic Server estabeleça conexão com um servidor DNS e armazene em cache o mapeamento ou especificar o mapeamento do endereço IP em um arquivo local. Se você especificar um endereço IP para o endereço de listening e depois, de uma solicitação do cliente, especificar um nome de DNS, o WebLogic Server tentará resolver o nome de DNS, mas, se não conseguir acessar o mapeamento do nome de DNS, a solicitação falhará.
host local
  • Os processos devem especificar localhost para estabelecer conexão com a instância do servidor.
  • Somente os processos que residem na máquina que hospeda a instância do servidor (processos locais) podem estabelecer conexão com a instância do servidor.

Especificando a Porta de Listening

Observe as diretrizes a seguir ao especificar a porta de listening para o Servidor de Administração. Essas diretrizes também são aplicadas aos Servidores Gerenciados.

  • Embora você possa especificar qualquer número de porta válido, se você especificar a porta 80, é possível omitir o número da porta da solicitação HTTP usada para acessar recursos do HTTP. Por exemplo, se você definir a porta 80 como a porta de listening, você pode usar o URL http://hostname:portnumber/myfile.html.
  • Em alguns sistemas operacionais, a porta 80 pode ser acessada somente por meio da execução dos processos em um usuário privilegiado ou ID do grupo. Neste caso, você pode designar a instância do servidor a uma máquina UNIX na qual um UID ou GID pós-bind foi definido.
