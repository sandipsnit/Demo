Configurar Clusters

Use esta tela para adicionar, configurar e deletar clusters.

Um cluster é um grupo das instâncias do WebLogic Server que funcionam juntas para fornecer escalabilidade e alta disponibilidade para aplicações. Ao criar clusters, é possível agrupar servidores gerenciados de forma que eles operem como uma unidade exclusiva para a hospedagem de aplicações e recursos.

  1. Revise a configuração do cluster atual. Os valores default podem variar baseados na origem do domínio que você selecionou.

    Observação: O assistente fornece duas views: uma view tabular concisa de todos os clusters e uma view individual de cada cluster com cada servidor representado por uma guia. Para alternar o modo da view entre as views da tabelas e da guia, clique em Trocar Exibição.

  2. Adicionar ou deletar clusters ou alterar as definições para clusters existentes, conforme o exigido para o seu domínio.

    A tabela a seguir descreve os campos nesta tela. Os campos marcados com um asterisco (*) são campos obrigatórios.

    Campo Descrição
    Nome* Informe um nome válido para o cluster.

    O nome do cluster deve ser exclusivo entre todos os nomes do componente dentro do domínio. O valor default neste campo é new_Cluster_n, no qual n é um valor numérico que é usado para diferenciar entre todos os nomes do cluster default, o valor de n para o primeiro cluster é 1. O valor é incrementado por 1 para cada cluster adicionado.
    Modo de mensagem do cluster Selecione o modo de mensagem unicast ou multicast.
    Endereço multicast Este campo é ativado apenas se você selecionar o modo de mensagem multicast.

    Informe o endereço multicast para o cluster. Este endereço é usado pelos membros do cluster para estabelecer comunicação entre eles. O valor default é 239.192.0.0. A faixa do endereço multicast válida vai de 224.0.0.1 até 239.255.255.255.
    Porta multicast Este campo é ativado apenas se você selecionar o modo de mensagem multicast.

    Informe a porta multicast para o cluster. A porta multicast é usada pelos membros do cluster para estabelecer comunicação entre eles. O valor default é 7001.
    Endereço do cluster Informe os endereços para identificar os servidores gerenciados no cluster. Um endereço do cluster pode ser um dos listados a seguir:
    • Uma lista separada por vírgulas de endereços IP ou nomes e portas DNS (por exemplo: dns_name:port, dns_name:port)
    • O nome DNS que mapeia os vários endereços IP
    • host local, nome DNS, ou endereço IP caso o endereço de listening de todos os servidores gerenciados estiver efetuando o listening no mesmo endereço com números de porta exclusivos

    O endereço do cluster é usado na entidade e nos EJBs stateless para construir a parte do nome do host dos URLs. Se o endereço do cluster não estiver definido, os handles do EJB podem não funcionar adequadamente.
  3. Após efetuar as seleções e entradas necessárias, clique em Próximo.

Para deletar um cluster, selecione o servidor e clique em Deletar. Quando você deleta um cluster, você não deleta os servidores designados a ele; os servidores são simplesmente removidos do cluster e podem ser adicionados posteriormente a outro cluster. Você pode deletar somente um cluster de cada vez.

Para adicionar um cluster, clique em Adicionar e configure as definições do novo servidor. O nome default do novo cluster é new_Cluster_n, no qual n começa com 1 e aumenta conforme um novo cluster é adicionado. Para alterar o default, digite o nome desejado na coluna Nome.

Para trocar a exibição do formato tabular default para um formato com guias (com uma guia para cada cluster), clique em Trocar Exibição. Clique em Trocar Exibição novamente para retornar ao formato tabular.
