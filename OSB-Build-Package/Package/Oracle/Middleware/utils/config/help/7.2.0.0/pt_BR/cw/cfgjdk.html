Configurar o Modo de Inicialização do Servidor e o JDK

Nesta tela, especifique se você deseja executar o WebLogic no modo de Produção ou de Desenvolvimento. Você também pode especificar a localização de JDK(s) para usar neste domínio.

  1. Seleciona o modo de inicialização do domínio WebLogic.
    • Modo de Desenvolvimento
      Neste modo, a configuração de segurança é relativamente reduzida, permitindo que você implante automaticamente as aplicações.
    • Modo de Produção
      Neste modo, a configuração de segurança é relativamente rigorosa, exigindo um nome do usuário e senha para implantar as aplicações. Antes de colocar um domínio em produção, familiarize-se com os procedimentos para proteger um ambiente de produção. Para obter mais informações, consulte Garantindo a Segurança do Ambiente de Produção em Protegendo um Ambiente de Produção do Oracle WebLogic Server.

    A tabela a seguir fornece informações para ajudá-lo a selecionar um modo de inicialização que atenda aos seus requisitos.

    Função Modo de desenvolvimento Modo de produção
    SSL É possível utilizar os certificados digitais de demonstração as áreas de armazenamento de chaves de demonstração fornecidas pelos serviços de segurança do WebLogic Server. Com estes certificados, é possível projetar a sua aplicação para funcionar dentro dos ambientes assegurados pelo SSL.

    Para obter mais informações, consulte Configurando o SSL em Protegendo o Oracle WebLogic Server

    		 Você não deve usar os certificados digitais de demonstração e as áreas de armazenamento de chaves. Caso venha a fazê-lo, uma mensagem de advertência será exibida.
    Implantando aplicações As instâncias do WebLogic Server podem implantar e atualizar aplicações que residem no diretório domain_name/autodeploy automaticamente.

    A Oracle recomenda que você use este método somente em um ambiente de desenvolvimento de servidor único.

    Para obter mais informações, consulte Implantando Aplicações e Módulos com weblogic.deployer em Implantando Aplicações no Oracle WebLogic Server;

    		 O recurso de implantação automático está desativado, portanto, você deve usar o console de administração do WebLogic Server , a ferramenta weblogic.Deployer ou a Ferramenta de Script do WebLogic.
    Rotação do arquivo de log Por default, ao inicializar uma instância do WebLogic Server, o servidor renomeia automaticamente (gira) seu arquivo de log do servidor local como SERVER-NAME.log.n. Para o restante da sessão do servidor, mensagens serão acumuladas no arquivo de log até que o arquivo cresça a um tamanho de 500 kilobytes.

    O valor default da definição do Número limite de arquivos retidos na Configuração de Log é verdadeiro. Isso limita o número de arquivos de log que a instância do servidor cria para armazenar mensagens antigas.

    		 O servidor gira o arquivo de log local após o tamanho de 5000 kilobytes ser atingido. Quando o servidor é configurado para o modo de produção, por default, todas as versões dos arquivos de log são mantidas. Os administradores podem desejar personalizar o número dos arquivos de log retidos.
    Recurso do sistema JDBC A capacidade default é de 15 conexões. A capacidade default é de 25 conexões.
  2. Selecione o JDK.

    No painel Seleção JDK, selecione o JDK para o modo de inicialização que você selecionou no painel Modo de Inicialização do Domínio do WebLogic. O Assistente de Configuração apresenta uma lista de JDKs incluídos no installer. É possível escolher um destes JDKs ou outro JDK que você tenha instalado no seu sistema.

    Observações:

    • Se você selecionar um JDK que esteja incluído no installer, o Assistente de Configuração cria scripts de inicialização do servidor para chamar o JDK. Se você selecionar um JDK que não seja fornecido pelo JDK, o Assistente de Configuração não configura os scripts de inicialização, você deve alterar os scripts de inicialização manualmente.
    • Selecione um destes JDKs que são suportados na plataforma que você está usando. A seleção default reflete o JDK que melhor atende aos requisitos do seu ambiente, baseando-se na plataforma na qual você está instalando o domínio.
  3. Após a marcação das seleções e entradas solicitadas, clique no botão Próximo no assistente.
