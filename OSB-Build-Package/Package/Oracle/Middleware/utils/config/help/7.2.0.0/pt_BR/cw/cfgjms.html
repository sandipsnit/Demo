Configurar Armazenamentos de Arquivo JMS

Um armazenamento de arquivos JMS é um arquivo baseado em disco no qual as mensagens persistentes podem ser salvas.

Revise as definições atuais para os armazenamentos do arquivo JMS e as modifique conforme o exigido. Os valores default podem variar baseando-se na origem do domínio selecionado anteriormente.

Observação: o assistente fornece dois modos de exibição: uma view tabular concisa de todos os componentes definidos e uma view individual na qual cada componente é representado por uma guia. Para alternar o modo da view entre as views da tabela e da guia, clique em Trocar Exibição.

A tabela a seguir descreve os campos nesta tela. Os campos marcados com um asterisco (*) são campos obrigatórios.

Campo Descrição
Nome* Informe um nome para o armazenamento do arquivo JMS.

O nome deve ser uma string de caracteres e pode incluir espaços. O nome do armazenamento do arquivo JMS deve ser exclusivo entre todos os nomes do componente dentro do domínio.
Diretório Informe o caminho do diretório (no seu sistema) no qual o armazenamento do arquivo JMS reside.
Política de gravação síncrona Selecione uma das políticas de gravação síncronas a seguir para determinar como o armazenamento do arquivo é gravado no disco:
  • Descarga de Cache: as transações não podem ser concluídas até que todas as suas operações de gravação sejam descarregadas no disco.
  • Gravação Direta: operações de gravação são executadas diretamente no disco. Esta política é suportada no Solaris e no Windows. Se esta política estiver ativa em uma plataforma não suportada, o armazenamento do arquivo é alternado automaticamente para a política de descarga de cache.

    Observação: A política de gravação direta é confiável em sistemas do Solaris, mas sistemas do Windows podem deixar os dados de transação no cache de disco sem serem gravados no disco imediatamente. Isso é considerado não confiável, porque um queda de energia pode causar a perda dos dados do cache de disco, possivelmente resultando em perda ou duplicação de mensagens. Para gravações confiáveis usando a política de gravação direta no Windows, ou desative a gravação de armazenamento em cache para o disco (ativado por default) ou use um disco com um cache com back up de bateria. Alguns sistemas de arquivo, no entanto, não permitem que este valor seja alterado (por exemplo, um sistema RAID que tenha um cache confiável).

  • Desativado: Transações são concluídas assim que as gravações forem armazenadas em cache na memória. Quando esta política estiver ativa, a conclusão das transações não dependerão em esperar para que as gravações atinjam o disco.

Esta definição afeta a execução, escalabilidade e confiabilidade.

Observação: Se o armazenamento do arquivo JMS for usado exclusivamente para a paginação de mensagens não persistentes para o disco, a política de gravação síncrona é ignorada.

Após efetuar as seleções e entradas necessárias, clique em Próximo.
