Configurar Máquinas

Em um domínio do WebLogic, as definições da máquina identificam unidades físicas de hardware e estão associadas às instâncias do WebLogic Server que elas hospedam.

Use esta tela para adicionar ou deletar máquinas ou para modificar as definições de uma máquina existente.

Selecione a guia Máquina (para o Windows) ou a guia Máquina UNIX (para UNIX).

É conveniente criar definições da máquina em situações como as seguintes:

  • O Servidor de Administração usa a definição da máquina, com a aplicação do Gerenciador de Nó, para iniciar os servidores remotos.
  • O WebLogic Server usa os nomes de máquina configurados ao determinar o servidor em um cluster que tem capacidade de tratar melhor determinadas tarefas, como replicação da sessão de HTTP. O WebLogic Server delega estas tarefas ao servidor identificado.

Observação: Você deve configurar máquinas para cada instalação do produto que executa um processo do Gerenciador de Nós. A configuração da máquina deve incluir valores para os parâmetros de endereço de listening e número da porta.

Cada máquina tem as seguintes definições de configuração. Os campos obrigatórios são marcados com um asterisco (*).

  1. Selecione a guia Máquina (para o Windows) e a guia Máquina (para o UNIX).
  2. Revise a lista de configurações atual e adicione, delete ou altere as configurações conforme o exigido para o seu domínio. A tabela a seguir descreve os campos nesta tela. Os campos marcados com um asterisco (*) são campos obrigatórios.

    Campo Descrição
    Nome* Informe um nome válido para a máquina.

    O nome da máquina é usado para identificar a máquina dentro do domínio do WebLogic Server, esse nome não é obrigado a corresponder ao nome da rede para a máquina. O nome deve ser exclusivo entre todos os nomes dos componentes dentro do domínio. O valor default neste campo é new_[Unix]Machine_n, no qual n é um valor numérico usado para diferenciar dentre todos os nomes de máquina default, o valor de n para a primeira máquina é 1. O valor é incrementado por 1 para cada máquina que você adicionar.
    Endereço de listening do gerenciador do nó Selecione um valor da lista para o endereço usado pelo gerenciador do nó para executar o listening para solicitações de conexão.

    Por default, o endereço IP definido para o sistema local e o host local estão exibidos na lista. O valor default é o host local. Se você especificar um endereço IP para uma máquina que hospede o servidor de administração e você precisar acessar o gerenciador do nó do WebLogic Server, você deve desativar a verificação do nome do host.

    Para obter mais informações, consulte Usando a Verificação do Nome do Host em Protegendo o Oracle WebLogic Server
    Porta de listening do gerenciador do nó Informe um valor válido para a porta de listening usada pelo gerenciador do nó para executar o listening para as solicitações de conexão.

    A faixa do número de porta válido para a porta de listening do gerenciador do nó vai de 1 até 65534. O valor default é 5556.
    GID Pós-bind ativado (Somente máquinas UNIX) Selecione esta caixa de seleção para ativar um servidor em execução nesta máquina para associar-se a um ID do grupo UNIX (GID) após o término de todas as ações de inicialização privilegiadas (consulte a descrição do campo GID Pós-bindabaixo). Por default, esta caixa de seleção não é selecionada.
    GID Pós-bind (Somente máquinas UNIX) Informe um ID do grupo UNIX válido (GID), usando aquele hospedado pelo servidor nesta máquina que será executado após o término de todas as ações de inicialização privilegiadas. Caso contrário, o servidor continua a ser executado, usando o ID do grupo com o qual ele foi inicializado.

    Para que esta definição entre em vigor, você deve selecionar a caixa de seleção GID Pós-bind ativado.

    UID Pós-bind ativado (Somente máquinas UNIX) Selecione a caixa de seleção para ativar um servidor hospedado nesta máquina para associar-se ao ID do usuário UNIX (UID) após o término das ações de inicialização privilegiadas (consulte a descrição do campo UID Pós-bind ativado abaixo). Por default, esta caixa de seleção não é selecionada.
    UID Pós-bind (Somente máquinas UNIX) Informe um ID do usuário UNIX (UID) válido, usando aquele hospedado nesta máquina que será executado após o término de todas as ações de inicialização privilegiadas. Caso contrário, o servidor continuará sendo executado, usando o ID do grupo com o qual ele foi inicializado.

    Para que esta definição entre em vigor, você deve selecionar a caixa de seleção UID Pós-bind ativado.
  3. Após efetuar as seleções e entradas necessárias, clique em Próximo.

Clique em Adicionar para adicionar uma nova máquina. O nome default de uma nova máquina é new_[Unix]Machine_n, no qual n começa com 1 e é aumentado, conforme cada máquina é adicionada.

Clique em Deletar para deletar uma máquina existente.
