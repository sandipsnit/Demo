Criar Aplicações de Proxy HTTP

Use esta tela para criar uma aplicação de proxy HTTP para cada cluster e especificar o Servidor Gerenciado no qual a aplicação de proxy deve ser implantada. Uma aplicação de proxy HTTP age como um intermediário para solicitações de HTTP.

Esta tela é exibida somente se ambas declarações a seguir forem verdadeiras:

  • Pelo menos um Servidor Gerenciado foi designado a um cluster.
  • Pelo menos um Servidor Gerenciado não foi designado a nenhum cluster.
  1. Selecione a caixa de seleção Criar Proxy HTTP no cluster em que você deseja criar uma aplicação de proxy HTTP.

    Uma lista de todos os Servidores Gerenciados que não estão designados a nenhum cluster será exibida na lista Servidor de Proxy.

  2. Na lista Servidor de Proxy, selecione um Servidor Gerenciado no qual a aplicação de proxy deve ser implantada.

    Uma aplicação de proxy chamada OracleProxy4_<clustername>_<servername> foi criada e implantada no Servidor Gerenciado selecionado.

  3. Repita as etapas 1 e 2 para cada cluster em que você desejar criar aplicações de proxy HTTP.
  4. Após efetuar as seleções e entradas necessárias, clique em Próximo.
