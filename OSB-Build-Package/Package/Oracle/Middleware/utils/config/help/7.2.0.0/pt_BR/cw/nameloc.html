Especificar um Nome e uma Localização do Domínio

Especifique os seguintes itens.

  • Nome do domínio
    O nome do domínio. O Assistente de Configuração armazena o arquivo config.xml e todos os outros componentes gerados para o domínio neste diretório, no diretório de Domínio especificado. Por exemplo, se você informar mydomain, os arquivos de domínio são armazenados (por default) em MW_HOME\user_projects\domains\mydomain.

    Observação: O nome do domínio não deve começar com um número. Esta restrição tem como objetivo evitar possíveis problemas com os nomes da tabela de armazenamento de JDBC gerados internamente, que devem começar com um caractere alfabético.

  • Localização do domínio
    A localização na qual o diretório de domínio será armazenado. O default é MW_HOME\user_projects\domains. A localização do domínio pode ser em qualquer lugar de sua unidade local ou rede. Não deve estar localizada em MW_HOME.

    No Windows, você deve incluir a letra da unidade deste caminho.

  • Localização da aplicação
    Este campo é exibido somente se você selecionou um modelo de domínio ou produto que contém aplicações. Esta é a localização na qual reside sua aplicação. O default é MW_HOME\user_projects\applications.

    No Windows, você deve incluir a letra da unidade deste caminho.

Após efetuar as seleções e entradas necessárias, clique em Próximo.
