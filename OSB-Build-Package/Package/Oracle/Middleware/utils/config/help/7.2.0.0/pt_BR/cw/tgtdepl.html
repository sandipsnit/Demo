Direcionar Implantações para Clusters ou Servidores

Use esta tela para direcionar aplicações e bibliotecas para implantação em servidores ou clusters.

Aplicações e bibliotecas associadas com o produto para o qual você está configurando o domínio (por exemplo, SOA) são direcionadas automaticamente para o Servidor Gerenciado criado para aquele produto ou para o cluster para o qual aquele Servidor Gerenciado é designado. Nesta tela, é possível direcionar aplicações e bibliotecas para servidores e clusters adicionais.

  1. No painel Destino, selecione o cluster ou servidor em que você deseja implantar as aplicações ou bibliotecas.

    O nome do destino selecionado foi exibido como o título do painel à direita.

  2. No painel à direita, selecione as caixas de seleção correspondentes às aplicações ou bibliotecas que você deseja implantar no destino selecionado.

    As aplicações e bibliotecas implantadas·aqui variam, dependendo dos produtos que você selecionou na tela Origem do Domínio anteriormente no assistente.

    Observação: Ao selecionar um servidor gerenciado no painel Direcionar, algumas das caixas de seleção no painel à esquerda podem ser desativadas, indicando aplicações e bibliotecas que já estão direcionadas para o cluster que contiver o servidor gerenciado selecionado.

    Após a seleção das aplicações e bibliotecas, os nomes dos clusters e servidores direcionados são exibidos na coluna Destino no painel à esquerda.

  3. Repita as etapas 1 e 2 para os outros clusters e servidores, conforme o solicitado.
  4. Após fazer a seleção necessária, clique em Próximo.

Quando você estende um domínio, se você deletar um Servidor Gerenciado ou cluster para o qual as aplicações estão sendo direcionadas no momento, o Assistente de Configuração redireciona automaticamente essas aplicações da seguinte forma:

  • Se as aplicações foram direcionadas originalmente somente ao Servidor Gerenciado ou cluster que você está deletando agora (isto é, após você deletar o Servidor Gerenciado ou cluster, as aplicações se tornariam não direcionadas no domínio modificado), então o Assistente de Configuração redireciona automaticamente as aplicações a todos os alvos apropriados.

    Um alvo apropriado é qualquer cluster ou Servidor Gerenciado que não é definido na especificação dos grupos de configuração (arquivo config-groups.xml) de um modelo incluído. Os servidores ou clusters que são especificados no config-groups.xml pertencem basicamente ao modelo e, por isso, não são apropriados para direcionamento automático.

  • Se as aplicações foram direcionadas originalmente a vários alvos (incluindo Servidores Gerenciados, clusters e o Servidor de Administração) e um dos Servidores Gerenciados ou clusters direcionados foram deletados, então no domínio estendido, o Assistente de Configuração deixa as associações de alvo restantes intactas e não tenta redirecionar as aplicações.
