Direcionar Serviços para Servidores ou Clusters

Use esta tela para direcionar serviços aos Servidores Gerenciados ou clusters apropriados.

Os serviços que estão associados ao produto para o qual você está configurando o domínio (por exemplo, SOA) estão direcionados automaticamente para o servidor gerenciado criado para aquele produto ou para o cluster ao qual aquele servidor gerenciado está designado. Nesta tela, é possível direcionar serviços para servidores e clusters adicionais.

  1. No painel Destino, selecione o cluster ou servidor em que você deseja implantar serviços.

    O nome do destino selecionado foi exibido como o título do painel à direita.

  2. No painel à esquerda, selecione as caixas de seleção correspondentes aos serviços que você deseja implantar·no destino selecionado.

    Os serviços exibidos aqui variam, dependendo dos produtos selecionado na tela Selecionar Origem do Domínio anteriormente no assistente.

    Ao selecionar o servidor gerenciado no painelDestino, algumas das caixas de seleção no painel à esquerda podem estar desativadas, indicando serviços que já estão direcionados no cluster que contém o servidor gerenciado selecionado.

    Após a seleção de serviços, os nomes dos clusters e servidores direcionados são exibidos na coluna Destino no painel à direita.

  3. Repita as etapas 1 e 2 para os outros clusters e servidores, conforme necessário.
  4. Após efetuar as seleções e entradas necessárias, clique em Próximo.

Quando você estende um domínio, se você deletar um Servidor Gerenciado ou cluster para o qual os serviços estão direcionados no momento, o Assistente de Configuração redireciona automaticamente esses serviços conforme a seguir:

  • Se os serviços foram direcionados originalmente somente ao Servidor Gerenciado ou cluster que você está deletando agora (isto é, após você deletar o Servidor Gerenciado ou cluster, os serviços se tornariam não direcionados no domínio modificado), então o Assistente de Configuração redireciona automaticamente as aplicações a todos os alvosapropriados.

    Um alvo apropriado é qualquer cluster ou Servidor Gerenciado que não é definido na especificação dos grupos de configuração (arquivo config-groups.xml) de um modelo incluído. Os servidores ou clusters que são especificados no config-groups.xml pertencem basicamente ao modelo e, por isso, não são apropriados para direcionamento automático.

  • Se os serviços foram direcionados originalmente a vários alvos (incluindo Servidores Gerenciados, clusters e o Servidor de Administração) e um dos Servidores Gerenciados ou clusters direcionados foram deletados, então no domínio estendido, o Assistente de Configuração deixa as associações de alvo restantes intactas e não tenta redirecionar os serviços.
