配置集群

使用此屏幕可以添加、配置和删除集群。

集群是一组 WebLogic Server 实例，这些实例一起工作，为应用程序提供可伸缩性和高可用性。通过创建集群，可以对受管服务器进行分组，使它们作为单个单元运行来承载应用程序和资源。

  1. 检查当前集群配置。默认值因所选的域源而异。

    注：该向导提供了两种视图：一种是简洁明了的表格式视图，其中包含所有集群；另一种是每个集群的单个视图，每个服务器由一个选项卡来表示。要在表视图和选项卡视图之间切换视图模式，请单击切换显示。

  2. 根据域的需要，添加或删除集群或者更改现有集群的设置。

    下表描述了此屏幕中的各个字段。带星号 (*) 标记的字段是必填字段。

    字段 说明
    名称* 输入有效的集群名。

    集群的名称在域内的所有组件名中必须是唯一的。此字段中的默认值为 new_Cluster_n，其中 n 是一个用于区分所有默认集群名的数字值；第一个集群的 n 值为 1。每添加一个集群，该值便递增 1。
    集群消息传送模式 选择单点传送或多点传送消息传送模式。
    多播地址 仅当选择多点传送消息传送模式时，才会启用此字段。

    输入集群的多播地址。集群成员使用此地址相互通信。默认值为 239.192.0.0。有效的多播地址范围介于 224.0.0.1 与 239.255.255.255 之间。
    多播端口 仅当选择多点传送消息传送模式时，才会启用此字段。

    输入集群的多播端口。集群成员使用多播端口相互通信。默认值为 7001。
    集群地址 输入用于标识集群中的受管服务器的地址。集群地址可采用下列任一种形式：
    • 用逗号分隔的 IP 地址或 DNS 名称和端口的列表 (例如：dns_name:port, dns_name:port)
    • 映射到多个 IP 地址的 DNS 名称
    • 如果所有受管服务器的监听地址要使用唯一端口号监听同一地址，则可以为 localhost、DNS 名称或 IP 地址

    在实体 EJB 和无状态 EJB 中，可使用集群地址构建 URL 的主机名部分。如果未设置集群地址，EJB 句柄可能无法正常工作。
  3. 做出所需选择和输入所需内容之后，单击下一步。

要删除集群，请选择服务器并单击删除。删除某个集群时，不会删除分配到该集群的服务器；服务器只是从集群中移去，然后可以添加到另一个集群。每次只能删除一个集群。

要添加集群，请单击添加并配置新服务器的设置。新集群的默认名称为 new_Cluster_n，其中 n 从 1 开始，在每次添加新集群时都会递增。要更改默认名称，请在“名称”列中键入所需的名称

要将显示方式从默认的表格格式切换到选项卡格式 (每个集群对应一个选项卡)，请单击切换显示。再次单击切换显示则恢复到表格格式。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
