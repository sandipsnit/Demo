配置 JDBC 组件方案

对于某些 Fusion Middleware 组件 (例如，SOA 和 Web Center Spaces)，可能会将 JDBC 数据源定义为组件的数据库方案的一部分，这些数据源是在安装期间使用资料档案库创建实用程序 (RCU) 加载的。

使用配置向导为这样的组件创建 WebLogic 域时，可以配置 JDBC 组件方案设置，例如数据库驱动程序、方案所有者、口令等。

此屏幕的下半部分列出了与要为其创建域的产品 (例如 SOA) 关联的 JDBC 组件方案。

注：要将一个或多个方案配置为 RAC 数据源，请选中所需方案的名称旁的复选框，然后选中在下一个面板中将选定的组件方案配置为 RAC 多数据源方案复选框。接下来，单击下一步按钮。

  1. 选中方案名称旁的复选框，选择要为其指定数据源设置的方案。

    注：选择多个组件方案时，某些字段中可能显示“各个组件方案之间不同”文本，指示在选定组件方案中这些字段的当前值是不同的。如果仍要更改此类字段的值，将在选定组件方案中统一应用新值。

    注：供应商、驱动程序、主机名和端口号等组件方案参数的默认值取决于在应用程序模板中指定的值。

  2. 检查当前配置设置并根据需要进行修改。

    注：供应商、驱动程序、主机名和端口号等方案参数的默认值取决于在应用程序模板中指定的值。

    字段 说明
    供应商 选择数据库供应商。
    驱动程序 选择连接到数据库要使用的 JDBC 驱动程序。

    该列表包含适用于选定数据库供应商的 JDBC 驱动程序。
    方案所有者 输入连接到数据库所使用的登录名。
    方案口令 输入指定方案所有者的登录口令。
    DBMS/服务 输入数据库 DBMS 名或服务名 (如果选定了服务类型驱动程序)。
    主机名 输入承载数据库的服务器的名称。
    端口 输入用于连接到承载数据库的服务器的端口号。

    在方案列表中，为选定方案指定的值将显示在相应列中。

  3. 设置所需选项和条目后，单击该向导中的下一步按钮测试这些组件方案。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
