配置 JDBC 数据源

使用此屏幕可以配置在您的域源中定义的 JDBC 数据源。

JDBC 数据源包含在创建数据源实例时 (部署或定位数据源实例时)，或启动服务器时创建的数据库连接池。应用程序会在 JNDI 树中查找数据源，然后请求连接。如果应用程序不再需要这些连接，会将这些连接返回到数据源中的连接池。

此屏幕的下半部分列出了与要为其创建域的产品关联的 JDBC 数据源。

注：要将一个或多个数据源配置为 RAC 数据源，请选中所需数据源的名称旁的复选框，然后选中在下一个面板中将选定的数据源配置为 RAC 多数据源复选框。接下来，单击下一步按钮。

  1. 通过选中数据源名称旁的复选框，来选择要为其指定设置的数据源。

    注：选择多个数据源时，某些字段中可能显示“各个数据源之间不同”文本，指示在选定数据源中这些字段的当前值是不同的。如果仍要更改此类字段的值，将在选定数据源中统一应用新值。

    注：供应商、驱动程序、主机名和端口号等数据源参数的默认值取决于在应用程序模板中指定的值。

  2. 检查当前配置设置并根据需要进行更改。

    下表描述了此屏幕中的各个字段。

    注：供应商、驱动程序、主机名和端口号等数据源参数的默认值取决于在应用程序模板中指定的值。

    字段 说明
    供应商 选择数据库供应商。
    驱动程序 选择连接到数据库要使用的 JDBC 驱动程序。

    该列表中包括适用于所选数据库供应商的公用 JDBC 驱动程序。
    用户名 输入连接到数据库所使用的登录名。
    口令 输入指定用户名的登录口令。
    DBMS/服务 输入 DBMS SID 或服务名。输入的值取决于所选的驱动程序。
    • 如果所选 Oracle 驱动程序的名称中包含“用于实例连接”这样的文字，则必须输入 SID。
    • 如果 Oracle 驱动程序的名称中包含“用于服务连接”这样的文字，则必须输入服务名称。
    • 有关配置 DataDirect 驱动程序的信息，请参阅 DataDirect 文档。
    主机名 输入承载数据库的服务器的名称。
    端口 输入用于连接到承载数据库的服务器的端口。

    在数据源列表中，为选定数据源指定的值将显示相应列中。

  3. 做出所需选择和输入所需内容之后，单击下一步以测试与 JDBC 数据源的连接。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
