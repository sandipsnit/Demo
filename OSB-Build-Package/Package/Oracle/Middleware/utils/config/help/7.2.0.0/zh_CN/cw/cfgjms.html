配置 JMS 文件存储

JMS 文件存储是可以保存持久性消息的基于磁盘的文件。

检查 JMS 文件存储的当前设置，并根据需要进行修改。默认值因之前选择的域源而异。

注：该向导提供了两种显示模式：一种是简洁明了的表格式视图，其中包含所有已定义的组件；一种是每个组件由一个选项卡表示的单个视图。要在表视图和选项卡视图之间切换视图模式，请单击切换显示。

下表描述了此屏幕中的各个字段。带星号 (*) 标记的字段是必填字段。

字段 说明
名称* 输入 JMS 文件存储的名称。

该名称必须是由字符组成的字符串，可含空格。JMS 文件存储的名称在域内的所有组件名中必须是唯一的。
目录 输入您的系统中 JMS 文件存储所在目录的路径。
同步写入策略 选择下列同步写入策略之一，确定文件存储如何将数据写入磁盘：
  • 高速缓存刷新：只有将事务的所有写入操作刷新到磁盘后，才能完成事务处理。
  • 直接写入：直接执行到磁盘的写入操作。Solaris 和 Windows 支持此策略。如果在不支持的平台上激活此策略，文件存储会自动切换到高速缓存刷新策略。

    注：在 Solaris 系统中，直接写入策略很可靠，但 Windows 系统可能会将事务处理数据保留在磁盘高速缓存中，不立即将它写入磁盘。这是不可靠的，因为电源故障可能导致磁盘高速缓存数据丢失，并可能造成消息丢失或重复。要在 Windows 中使用直接写入策略可靠地写入数据，请禁用磁盘的所有写入高速缓存 (默认情况下启用)，或使用带有由电池支持高速缓存的磁盘。不过，某些文件系统不允许更改此值 (例如具有可靠高速缓存的 RAID 系统)。

  • 已禁用：一旦将写入操作高速缓存到内存中即完成事务处理。激活此策略后，事务处理的完成将与等待写入磁盘无关。

此设置影响性能、可伸缩性和可靠性。

注：如果 JMS 文件存储专用于将非持久性消息分页存储到磁盘，将忽略同步写入策略。

做出所需选择和输入所需内容之后，单击下一步。
