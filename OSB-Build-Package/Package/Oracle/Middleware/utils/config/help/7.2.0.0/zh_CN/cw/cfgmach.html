配置计算机

在 WebLogic 域中，计算机定义用于标识硬件的物理单元，并与它们承载的 WebLogic Server 实例相关联。

使用此屏幕可以添加或删除计算机，或修改现有计算机的设置。

选择计算机选项卡 (对于 Windows) 或“UNIX 计算机”选项卡 (对于 UNIX)。

在诸如以下的情形中，您可能希望创建计算机定义：

  • 管理服务器使用带有节点管理器应用程序的计算机定义来启动远程服务器。
  • WebLogic Server 在确定集群中最适合执行特定任务 (如 HTTP 会话复制) 的服务器时使用已配置的计算机名称。然后，WebLogic Server 会将这些任务委派给标识的服务器。

注：必须为每个运行节点管理器进程的产品安装都配置计算机。计算机配置必须包含监听地址和端口号参数的值。

每个计算机都有以下配置设置。必需的字段标有星号 (*)。

  1. 选择计算机选项卡 (对于 Windows) 和 Unix 计算机选项卡 (对于 UNIX)。
  2. 检查当前配置列表，并根据域的需要添加、删除或更改设置。下表描述了此屏幕中的各个字段。带星号 (*) 标记的字段是必填字段。

    字段 说明
    名称* 输入有效的计算机名。

    该计算机名用于在 WebLogic Server 域中标识计算机，不必与计算机的网络名一致。该名称在域内的所有组件名中必须是唯一的。此字段中的默认值为 new_[Unix]Machine_n，其中 n 是一个用于区分所有默认计算机名的数字值；第一台计算机的 n 值为 1。每添加一台计算机，该值便递增 1。
    节点管理器监听地址 从节点管理器监听连接请求所使用的地址列表中选择一个值。

    默认情况下，为本地系统定义的 IP 地址和 localhost 显示在该列表中。默认值为 localhost。如果为承载管理服务器的计算机指定 IP 地址，而且需要访问 WebLogic Server 节点管理器，则必须禁用主机名验证。

    有关详细信息，请参阅 Securing Oracle WebLogic Server 中的 Using Host Name Verification
    节点管理器监听端口 输入有效的监听端口值，节点管理器将使用该端口监听连接请求。

    节点管理器监听端口的有效端口号介于 1 与 65534 之间。默认值为 5556。
    已启用绑定后 GID (仅限 UNIX 计算机) 选中此复选框可使在该计算机上运行的服务器，在完成所有授权启动操作后绑定到 UNIX 组 ID (GID) (请参阅下面的绑定后 GID 字段说明)。默认情况下不选中此复选框。
    绑定后 GID (仅限 UNIX 计算机) 输入有效的 UNIX 组 ID (GID)，此计算机上承载的服务器在完成所有授权启动操作后，将使用该 ID 运行。否则，该服务器将继续使用启动时所用的组 ID 运行。

    为使此设置生效，必须选中已启用绑定后 GID 复选框。

    已启用绑定后 UID (仅限 UNIX 计算机) 选中此复选框可使该计算机上承载的服务器，在完成所有授权启动操作后绑定到 UNIX 用户 ID (UID) (请参阅下面的绑定后 UID 字段说明)。默认情况下不选中此复选框。
    绑定后 UID (仅限 UNIX 计算机) 输入有效的 UNIX 用户 ID (UID)，此计算机上承载的服务器在完成所有授权启动操作后，将使用该 ID 运行。否则，该服务器将继续使用启动时所用的用户 ID 运行。

    为使此设置生效，必须选中已启用绑定后 UID 复选框。
  3. 做出所需选择和输入所需内容之后，单击下一步。

单击添加可添加一个新的计算机。新计算机的默认名称为 new_[Unix]Machine_n，其中 n 从 1 开始，每次添加一个新计算机将递增 1。

单击删除可删除现有计算机。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库
