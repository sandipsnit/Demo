配置 RAC 多数据源组件方案

使用此屏幕可以将包含在 WebLogic 域中的组件方案配置为 Oracle RAC 多数据源。

向导上一屏幕 (配置 JDBC 组件方案) 的下半部分列出了您决定为其配置 RAC 数据源的组件方案。

  1. 在该屏幕下半部分的方案列表中，选中方案名称旁的复选框，选择要配置为 RAC 数据源的方案。

    注：选择多个数据源方案时，某些字段中可能显示“各个组件方案之间不同”文本，指示在选定方案中这些字段的当前值是不同的。如果仍要更改此类字段的值，将在选定方案中统一应用新值。

  2. 指定 RAC 数据源的设置。

    下表描述了此屏幕中的各个字段。

    字段 说明
    驱动程序 选择要连接到数据库所使用的驱动程序。
    服务名 输入数据库 Oracle RAC 服务名。
    用户名 输入连接到数据库所使用的登录名。
    口令 输入指定用户名的登录口令。
    主机名 输入承载 RAC 数据库实例的服务器的名称。
    实例名 输入每个 Oracle 数据库实例的名称。
    端口 输入用于连接到承载数据库实例的服务器的端口号。

    注：至少必须指定一个数据库实例的主机名、实例名和端口号。

    要添加数据库实例，请单击添加，然后指定主机名、数据库实例名和端口号。

    在方案列表中，为选定方案指定的值将显示在相应列中。

  3. 做出所需选择和输入所需内容之后，单击下一步测试组件方案。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库
