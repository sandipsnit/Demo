配置 RAC 多数据源

使用此屏幕可以将域中包含的数据源配置为 Oracle RAC 数据源。

向导上一屏幕 (配置 JDBC 数据源) 的下半部分列出了在该屏幕中选择配置为 RAC 数据源的数据源。

  1. 通过选中数据源名称旁的复选框，来选择要为其指定数据源设置的数据源。

    注：选择多个数据源时，某些字段中可能显示“各个数据源之间不同”文本，指示在选定数据源中这些字段的当前值是不同的。如果仍要更改此类字段的值，将在选定数据源中统一应用新值。

  2. 指定 RAC 数据源的设置。

    下表描述了此屏幕中的各个字段。

    字段 说明
    驱动程序 选择连接到数据库要使用的 JDBC 驱动程序。
    服务名 输入 Oracle RAC 数据库服务的名称。
    用户名 输入连接到数据库所使用的登录名。
    口令 输入指定用户名的登录口令。
    主机名 输入承载 Oracle RAC 数据库实例的服务器的名称。
    实例名 输入每个 Oracle 数据库实例的名称。
    端口 输入用于连接到承载数据库实例的服务器的端口号。

    注：至少必须指定一个数据库实例的主机名、实例名和端口号。

    要添加数据库实例，请单击添加，然后指定主机名、实例名和端口号。

    在数据源列表中，为选定数据源指定的值将显示相应列中。

  3. 做出所需选择和输入所需内容之后，单击下一步测试与数据源的连接。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
