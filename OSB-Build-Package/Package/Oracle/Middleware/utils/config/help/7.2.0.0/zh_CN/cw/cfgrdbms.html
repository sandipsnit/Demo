配置 RDBMS 安全存储数据库

使用此屏幕可以定义 RDBMS 安全存储设置。

下表介绍了此屏幕上的字段。

字段 说明
数据库类型 从下拉列表中，选择要用作 RDBMS 安全存储的数据库的类型。
驱动程序 选择要用于数据库的驱动程序。可用驱动程序列表视所选的数据库类型而定。
类名称 不必执行任何操作。系统将根据选定的驱动程序自动显示类名。
DBMS SID 输入数据库的 SID。
DBMS 主机 输入承载数据库的计算机的名称。
DBMS 端口 输入用于连接到服务器的端口。系统将自动显示与所选数据库类型关联的默认端口号。
URL 不必执行任何操作。系统将根据选定的驱动程序自动显示 URL。
用户名 输入连接到数据库所使用的登录名。
用户口令 输入用于访问数据库的口令。

有效值由字母数字字符串组成。允许使用连字符 (-) 和下划线 ( _ ) 字符。
该值将被加密。
确认用户口令 重新输入口令。
已知属性 不必执行任何操作。系统将根据选定的驱动程序自动显示数据库的已知属性。
其他属性 输入要传递给驱动程序的其他属性 (如果有)。

注：可单击测试连接来测试与数据库的连接。启动服务器之前，必须加载 RDBMS 安全存储所需的 SQL 脚本。如果在集群域中使用 RDBMS 安全存储，建议将其与 JMS 配置 (JNDI 名称和 JMS 主题) 一起使用。

设置所需选项和条目后，单击该向导中的下一步按钮。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
