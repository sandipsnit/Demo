运行数据库脚本

域模板可能包含一组按数据库类型组织的 SQL 文件。如果域模板包含 SQL 文件，则在创建 WebLogic 域时，可以在运行数据库脚本屏幕中运行这些文件。使用预先存在的 SQL 或数据库加载文件，可以设置在 WebLogic 域中定义的每个数据源的数据库内容。

通过预先存在的 SQL 或数据库加载文件，可设置域中定义的每个数据源的数据库内容。

  1. 在可用 JDBC 数据源窗格中，选择要针对其运行脚本的数据源。

    可执行脚本显示在可用 SQL 文件和数据库加载选项窗格中。

  2. 从数据库版本列表中选择数据库版本。
  3. 单击运行脚本。

    此时将针对选定数据源，执行可用 SQL 文件和数据库加载选项窗格中显示的所有脚本，执行结果显示在结果窗格中。

    要在日志文件中捕获测试输出，请选中日志文件复选框并指定日志文件的位置。

  4. 针对要执行数据库脚本的每个数据源，重复执行步骤 1 - 3。
  5. 运行数据库脚本后，单击下一步。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
