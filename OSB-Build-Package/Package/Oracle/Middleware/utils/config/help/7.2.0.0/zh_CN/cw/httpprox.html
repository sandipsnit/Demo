创建 HTTP 代理应用程序

使用此屏幕可以为每个集群创建 HTTP 代理应用程序，并指定必须部署代理应用程序的受管服务器。HTTP 代理应用程序充当 HTTP 请求的中介。

仅在以下两个条件都得到满足时才会显示此屏幕：

  • 至少有一个受管服务器已分配给集群。
  • 至少有一个受管服务器尚未分配给任何集群。
  1. 为要创建 HTTP 代理应用程序的集群选中创建 HTTP 代理复选框。

    代理服务器列表中显示了一系列未分配给任何集群的受管服务器。

  2. 在代理服务器列表中，选择必须部署代理应用程序的受管服务器。

    此时将创建名为 OracleProxy4_<clustername>_<servername> 的代理应用程序，并将它部署在所选受管服务器中。

  3. 为要创建 HTTP 代理应用程序的每个集群重复执行步骤 1 和 2。
  4. 做出所需选择之后，单击下一步。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
