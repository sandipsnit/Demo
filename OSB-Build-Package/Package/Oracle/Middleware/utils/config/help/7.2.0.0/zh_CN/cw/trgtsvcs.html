将服务定位到服务器或集群

使用此屏幕可以将服务定位到适当的受管服务器或集群。

与要配置域的产品 (例如 SOA) 关联的服务将自动定位到为该产品创建的受管服务器，或自动定位到为其分配该受管服务器的集群。可在此屏幕中将服务定位到其他服务器和集群。

  1. 在目标窗格中，选择要在其中部署服务的集群或服务器。

    选定目标的名称将显示为右窗格的标题。

  2. 在右窗格中，选中要在选定目标上部署的服务的对应复选框。

    此处显示的服务视在该向导前面的选择域源屏幕中选择的产品而定。

    在目标窗格中选择受管服务器时，可能会禁用右窗格中的某些复选框，指示已将服务定位到包含选定受管服务器的集群。

    选择服务后，所定位的集群和服务器的名称将显示在右窗格的目标列中。

  3. 如有必要，对其他集群和服务器重复执行步骤 1 和 2。
  4. 做出所需选择之后，单击下一步。

扩展某个域之后，如果删除服务当前定位到的受管服务器或集群，则配置向导会自动按照如下方式重新定位这些服务：

  • 如果服务最初仅定位到当前正在删除的受管服务器或集群 (即，删除受管服务器或集群之后，将取消服务在所修改域中的定位)，则配置向导会自动将服务重新定位到所有符合条件的目标。

    符合条件的目标是指未在已包含的模板的配置组规范 (config-groups.xml 文件) 中定义的任何集群或受管服务器。在 config-groups.xml 中指定的服务器或集群实质上由模板拥有，因此不符合自动定位的条件。

  • 如果服务最初定位到多个目标 (包括受管服务器、集群和管理服务器)，并且定位到的其中一个受管服务器或集群被删除，则在扩展域中，配置向导将保持其余的目标关联不变，并且不会尝试重新定位服务。

有关详细信息，请参阅 Oracle 技术网中的 Oracle Fusion Middleware 文档库。
