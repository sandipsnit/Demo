設定叢集

使用此畫面以新增、設定和刪除叢集。

叢集是共同運作的一組 WebLogic Server 執行處理，為應用程式提供擴充性和高可用性。您可以建立叢集來群組受管理伺服器，讓它們可以當成主控應用程式和資源的單一單元來操作。

  1. 複查目前叢集組態。預設值可能會根據您選取的網域來源而不同。

    注意：此精靈提供兩種視圖：所有叢集的簡要表格式視圖，以及每個叢集的個別視圖，其中每個伺服器都是以頁籤代表。若要切換表格和頁籤視圖這兩種視圖模式，請按一下「切換顯示」。

  2. 新增或刪除叢集，或變更現有叢集的設定 (依網域需要)。

    下表描述此畫面中的欄位。標上星號 (*) 的欄位是必要欄位。

    欄位 描述
    Name* 輸入叢集的有效名稱。

    叢集的名稱在網域的所有元件名稱中必須是唯一的。此欄位的預設值是 new_Cluster_n，其中 n 是用來區別所有預設叢集名稱的數值；第一個叢集的 n 值是 1。每個所新增叢集的這個值的遞增量都是 1。
    Cluster messaging mode 選取單點傳播或多點傳播訊息傳遞模式。
    Multicast address 只有在您選取多點傳播訊息傳遞模式時，才會啟用此欄位。

    輸入叢集的多點傳播位址。叢集成員會使用此位址在彼此間進行通訊。預設值為 239.192.0.0。有效的多點傳播位址介於 224.0.0.1 與 239.255.255.255 之間。
    Multicast port 只有在您選取多點傳播訊息傳遞模式時，才會啟用此欄位。

    輸入叢集的多點傳播連接埠。叢集成員會使用多點傳播連接埠在彼此間進行通訊。預設值為 7001。
    叢集位址 輸入地址，以識別叢集中的受管理伺服器。叢集位址可以是下列其中一項：
    • 以逗號分隔之 IP 位址或 DNS 名稱及連接埠的清單 (例如：dns_name:port, dns_name:port)
    • 對應至多個 IP 位址的 DNS 名稱
    • localhost、DNS 名稱或 IP 位址 (如果所有受管理伺服器的監聽地址正在使用唯一的連接埠號碼監聽同一位址)

    叢集位址用於實體和無狀態 EJB，以建構 URL 的主機名稱部分。如果未設定叢集位址，則 EJB 處理可能無法正常運作。
  3. 進行必要的選取和輸入之後，請按一下「下一步」。

若要刪除叢集，請選取伺服器，然後按一下「刪除」。刪除叢集時，並不會刪除指派至叢集的伺服器，僅會將伺服器由叢集移除，之後可新增至其他叢集。您一次只能刪除一個叢集。

若要新增叢集，請按一下「新增」並設定新伺服器的設定。新叢集的預設名稱為 new_Cluster_n，其中 n 從 1 開始，隨著您新增的每個新叢集遞增。若要變更預設，請在「名稱」欄位中鍵入想要的名稱。

若要將預設的顯示從表格格式切換為頁籤式格式 (一個頁籤代表一個叢集)，請按一下「切換顯示」。再按一下「切換顯示」，會回到表格格式。
