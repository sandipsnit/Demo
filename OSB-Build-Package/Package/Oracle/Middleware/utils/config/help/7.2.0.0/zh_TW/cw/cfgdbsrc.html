設定 JDBC 資料來源

使用此畫面以設定網域來源中定義的 JDBC 資料來源。

JDBC 資料來源包含在建立資料來源執行處理時 (建置或定位資料來源執行處理)，或伺服器啟動時所建立的資料庫連線集區。應用程式會在 JNDI 樹狀圖上尋找資料來源，然後請求連線。應用程式不再需要連線時，會將連線傳回到資料來源的連線集區中。

與正在建立網域之產品相關聯的 JDBC 資料來源，會列在畫面的下半部。

注意：如果您想要設定一或多個資料來源作為 RAC 資料來源，請選取與必要資料來源名稱相鄰的核取方塊，並選取「在下一個面板中，將選定資料來源設定為 RAC 多重資料來源」核取方塊。然後按「下一步」按鈕。

  1. 選取與資料來源名稱相鄰的核取方塊，以選取想要指定設定的資料來源。

    注意：當您選取多個資料來源時，可能會在特定欄位中顯示「因資料來源而異」文字，指出在選定資料來源中，那些欄位的目前值會不同。如果您繼續進行並變更這類欄位中的值，則會將新值統一套用至選定資料來源。

    注意：資料來源參數 (如供應商、驅動程式、主機名稱和連接埠號碼) 的預設值取決於應用程式樣板中指定的值。

  2. 複查目前組態設定，並依需要進行變更。

    下表描述此畫面中的欄位。

    注意：資料來源參數 (如供應商、驅動程式、主機名稱和連接埠號碼) 的預設值取決於應用程式樣板中指定的值。

    欄位 描述
    供應商 選取資料庫供應商。
    驅動程式 選取想要用來連線至資料庫的 JDBC 驅動程式。

    此清單包含選定資料庫供應商的一般 JDBC 驅動程式。
    使用者名稱 輸入用於連線至資料庫的登入名稱。
    密碼 輸入所指定使用者名稱的登入密碼。
    DBMS/服務 輸入 DBMS SID 或服務名稱。您輸入的值取決於您選取的驅動程式。
    • 如果您選取的 Oracle 驅動程式名稱包含 "for Instance connections" 字樣，就必須輸入 SID。
    • 如果 Oracle 驅動程式名稱包含 "for Service connections" 字樣，就必須輸入服務名稱。
    • 有關設定 DataDirect 驅動程式的詳細資訊，請參閱 DataDirect 文件。
    主機名稱 輸入主控資料庫的伺服器名稱。
    連接埠 輸入用來連線至主控資料庫的伺服器連接埠。

    所指定的值會顯示在選定資料來源之資料來源清單中的適當資料欄。

  3. 進行必要的選取和輸入之後，請按一下「下一步」，以測試與 JDBC 資料來源的連線。

有關詳細資訊，請參閱 Oracle 技術網路上的 Oracle Fusion Middleware 文件庫。
