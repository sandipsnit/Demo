設定伺服器啟動模式和 JDK

在此畫面中，請指定您要在生產模式或開發模式中執行 WebLogic。您也可以指定要在此網域中使用的 JDK 位置。

  1. 選取 WebLogic 網域啟動模式。
    • 開發模式
      在此模式中，安全組態相對比較彈性，可讓您自動建置應用程式。
    • 生產模式
      在此模式中，安全組態相對比較嚴謹，需要有使用者名稱和密碼才能建置應用程式。將網域運用到生產之前，請您先熟悉如何保護生產環境的安全。有關詳細資訊，請參閱「Oracle WebLogic Server 保護生產環境的安全」中的確保您的生產環境的安全。

    下表提供資訊，協助您選取符合您需求的啟動模式。

    功能 開發模式 生產模式
    SSL 您可以使用 WebLogic Server 安全服務所提供的展示數位憑證和展示金鑰儲存庫。使用這些憑證，您可以設計應用程式，讓它在 SSL 保護的環境中運作。

    有關詳細資訊，請參閱「保護 Oracle WebLogic Server」中的設定 SSL。

    		 您不能使用展示數位憑證和展示金鑰儲存庫。如果這樣做，則會顯示警告訊息。
    建置應用程式 WebLogic Server 執行處理可以自動建置和更新位在 domain_name/autodeploy 目錄中的應用程式。

    Oracle 建議您僅在單一伺服器開發環境中使用此方法。

    有關詳細資訊，請參閱「將應用程式建置至 Oracle WebLogic Server」中的以 weblogic.deployer 建置應用程式和模組。

    		 已停用自動建置功能；因此您必須使用 WebLogic Server 管理主控台、weblogic.Deployer 工具或 WebLogic 命令碼工具。
    日誌檔循環 當您啟動 WebLogic Server 執行處理時，伺服器預設會自動將它的本機伺服器日誌檔重新命名 (循環) 為 SERVER-NAME.log.n。針對伺服器階段作業的其餘部分，訊息在日誌檔的大小成長到 500 KB 之前都會累積在此日誌檔中。

    「記錄日誌組態」中「限制保留的檔案數」設定的預設值為 true。此預設值限制了伺服器執行處理為了儲存舊訊息所建立的日誌檔數量。

    		 伺服器會在本機日誌檔的大小達到 5000 KB 之後循環本機日誌檔。將伺服器設定為生產模式時，預設會保留日誌檔的所有版本。而管理員可能會想要自訂保留的日誌檔數目。
    JDBC 系統資源 預設容量為 15 個連線。 預設容量為 25 個連線。
  2. 選取 JDK。

    在「JDK 選取」窗格中，選取「網域啟動模式」窗格中所選取之啟動模式的 JDK。「組態精靈」會呈現安裝程式中所含的 JDK 清單。您可以選擇其中一個 JDK 或安裝在系統上的其他 JDK。

    注意：

    • 如果您選取安裝程式中所含的 JDK，則「組態精靈」會建立伺服器啟動命令碼來叫用 JDK。如果您選取 JDK 未提供的 JDK，則「組態精靈」不會設定啟動命令碼；您必須手動變更啟動命令碼。
    • 僅選取您使用的平台所支援的那些 JDK。預設選取會根據您安裝網域的平台，反映最能滿足環境要求的 JDK。
  3. 進行必要的選取和輸入之後，請按一下精靈中的「下一步」按鈕。

有關詳細資訊，請參閱 Oracle 技術網路上的 Oracle Fusion Middleware 文件庫。
