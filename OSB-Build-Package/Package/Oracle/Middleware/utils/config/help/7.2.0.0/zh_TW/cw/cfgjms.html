設定 JMS 檔案儲存

JMS 檔案儲存是磁碟型的檔案儲存，其中可儲存持續性訊息。

複查 JMS 檔案儲存的目前設定，並依需要進行修改。預設值可能會根據您先前選取的網域來源而不同。

注意：此精靈提供兩種顯示模式：所有已定義元件的簡要表格式視圖，以及個別視圖，其中每個元件都是以頁籤代表。若要切換表格和頁籤視圖這兩種視圖模式，請按一下「切換顯示」。

下表描述此畫面中的欄位。標上星號 (*) 的欄位是必要欄位。

欄位 描述
Name* 輸入 JMS 檔案儲存的名稱。

此名稱必須是字元字串，而且可以包含空格。JMS 檔案儲存的名稱在網域的所有元件名稱中必須是唯一的。
目錄 輸入 JMS 檔案儲存所在目錄 (位於系統中) 的路徑。
同步寫入原則 選取下列其中一個同步寫入原則，以確定檔案儲存將資料寫入磁碟的方式：
  • Cache-Flush：將交易在快取中的所有寫入操作都存入磁碟之後，交易才可完成。
  • Direct-Write：直接對磁碟執行寫入操作。Solaris 和 Windows 都支援此原則。如果在不受支援的平台上此原則為作用中，則檔案儲存會自動切換至 Cache-Flush 原則。

    注意：在 Solaris 系統中，Direct-Write 原則是可靠的，但 Windows 系統可能會將交易資料保留在磁碟快取中，而未立即寫入至磁碟。這視為不可靠，因為停電可能會導致磁碟快取資料遺失，而這可能會導致訊息遺失或重複。若要在 Windows 上使用 Direct-Write 來進行可靠的寫入，請停用磁碟的所有寫入快取 (預設為啟用)，或使用具有備用電池快取的磁碟。然而，部分檔案系統不允許變更此值 (例如，具有可靠快取的 RAID 系統)。

  • 已停用：只要將寫入快取至記憶體中，交易即可完成。當此原則為作用中時，不需等待寫入到磁碟，交易便可完成。

此設定會影響效能、擴充性和可靠性。

注意：如果將 JMS 檔案儲存專用於將非持續性訊息分頁存入磁碟中，則會忽略同步寫入原則。

進行必要的選取和輸入之後，請按一下「下一步」。
