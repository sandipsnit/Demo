設定機器

在 WebLogic 網域中，機器定義可識別硬體的實體單元，並且和其承載的 WebLogic Server 執行處理相關聯。

使用此畫面以新增或刪除機器，或者修改現有機器的設定。

選取「機器」頁籤 (針對 Windows) 或「Unix 機器」頁籤 (針對 UNIX)。

下列情況中，您可能想要建立機器定義：

  • 「管理伺服器」使用機器定義以及「節點管理程式」應用程式，來啟動遠端伺服器。
  • WebLogic Server 會使用已設定的機器名稱，來判斷叢集中處理特定工作 (例如 HTTP 階段作業複製) 的最佳伺服器。然後 WebLogic Server 就會委派那些工作給識別出來的伺服器。

注意：您必須針對執行「節點管理程式」程序的每項產品安裝設定機器。機器設定必須包含監聽位址和連接埠號碼參數的值。

每台機器都有下列的組態設定。必要欄位會標上星號 (*)。

  1. 選取「機器」頁籤 (針對 Windows) 和「Unix 機器」頁籤 (針對 UNIX)。
  2. 複查目前的組態清單，並依需要新增、刪除或變更您網域的組態。下表描述此畫面中的欄位。標上星號 (*) 的欄位是必要欄位。

    欄位 描述
    Name* 輸入機器的有效名稱。

    機器名稱是用來識別 WebLogic Server 網域內的機器，並不需要符合機器的網路名稱。此名稱在網域的所有元件名稱中必須是唯一的。此欄位的預設值是 new_[Unix]Machine_n，其中 n 是用來區別所有預設機器名稱的數值；第一個機器的 n 值是 1。每個所新增機器的這個值的遞增量都是 1。
    Node manager listen address 從清單中選取節點管理程式用於監聽連線請求之地址的值。

    預設會在清單中顯示針對本機系統和 localhost 所定義的 IP 位址。預設值為 localhost。如果您指定要主控管理伺服器之機器的 IP 位址，而且需要存取 WebLogic Server 節點管理程式，則必須停用主機名稱驗證。

    有關詳細資訊，請參閱「保護 Oracle WebLogic Server」中的使用主機名稱驗證
    Node manager listen port 輸入節點管理程式用於監聽連線請求之監聽連接埠的有效值。

    節點管理程式監聽連接埠的有效連接埠號碼範圍介於 1 與 65534 之間。預設值為 5556。
    Post bind GID enabled (僅限 UNIX 機器) 選取此核取方塊，可讓此機器上所執行的伺服器在完成所有授權的啟動動作之後，連結到 UNIX 群組 ID (GID) (請參閱下面「Post bind GID」欄位中的描述)。預設不會選取此核取方塊。
    Post bind GID (僅限 UNIX 機器) 輸入有效的 UNIX 群組 ID (GID)，而此機器上主控的伺服器在完成所有授權的啟動操作之後，會使用此 ID 執行。否則，該伺服器會繼續使用啟動時的群組 ID 執行。

    若要讓此設定生效，您必須選取「Post-bind GID enabled」核取方塊。

    Post bind UID enabled (僅限 UNIX 機器) 選取此核取方塊，可讓此機器上所主控的伺服器在完成所有授權的啟動動作之後，連結到 UNIX 使用者 ID (UID) (請參閱下面「Post bind UID」欄位中的描述)。預設不會選取此核取方塊。
    Post bind UID (僅限 UNIX 機器) 輸入有效的 UNIX 使用者 ID (UID)，而此機器上主控的伺服器在完成所有授權的啟動操作之後，會使用此 ID 執行。否則，該伺服器會繼續使用啟動時的群組 ID 執行。

    若要讓此設定生效，您必須選取「Post-bind UID enabled」核取方塊。
  3. 進行必要的選取和輸入之後，請按一下「下一步」。

按一下「新增」以新增新機器。新機器的預設名稱為 new_[Unix]Machine_n，其中 n 從 1開始，隨著您新增的每台機器，一次遞增一個數字。

按一下「刪除」以刪除現有的機器。

有關詳細資訊，請參閱 Oracle 技術網路上的 Oracle Fusion Middleware 文件庫。
