設定 RAC 多重資料來源

使用此畫面將網域中所包含的資料來源設為 Oracle RAC 資料來源。

在精靈的前一個畫面中，您選擇為其設定為 RAC 資料來源的資料來源 (設定 JDBC 資料來源) 會列在畫面的下半部。

  1. 選取與資料來源名稱相鄰的核取方塊，以選取想要為其指定資料來源設定的資料來源。

    注意：當您選取多個資料來源時，可能會在特定欄位中顯示「因資料來源而異」文字，指出在選定資料來源中，那些欄位的目前值會不同。如果您繼續進行並變更這類欄位中的值，則會將新值統一套用至選定資料來源。

  2. 指定 RAC 資料來源的設定。

    下表描述此畫面中的欄位。

    欄位 描述
    驅動程式 選取想要用來連線至資料庫的 JDBC 驅動程式。
    服務名稱 輸入 Oracle RAC 資料庫服務名稱。
    使用者名稱 輸入用於連線至資料庫的登入名稱。
    密碼 輸入所指定使用者名稱的登入密碼。
    主機名稱 輸入主控 Oracle RAC 資料庫執行處理的伺服器名稱。
    執行處理名稱 輸入每個 Oracle 資料庫執行處理的名稱。
    連接埠 輸入用來連線至主控資料庫執行處理之伺服器的連接埠號碼。

    注意：您至少必須指定一個資料庫執行處理的主機名稱、執行處理名稱和連接埠號碼。

    若要新增資料庫執行處理，請按一下「新增」，然後指定主機名稱、執行處理名稱和連接埠號碼。

    所指定的值會顯示在選定資料來源之資料來源清單中的適當資料欄。

  3. 進行必要的選取和輸入之後，請按一下「下一步」，以測試與資料來源的連線。

有關詳細資訊，請參閱 Oracle 技術網路上的 Oracle Fusion Middleware 文件庫。
