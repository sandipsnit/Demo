設定 RDBMS 安全儲存資料庫

使用此畫面以定義 RDBMS 安全儲存設定

下表描述此畫面中的欄位。

欄位 描述
Database Type 從下拉清單中，選取想要當做 RDBMS 安全儲存使用的資料庫類型。
Driver 選取想要用於資料庫的驅動程式。可用驅動程式清單會根據選取的資料庫類型而不同。
Class Name 不需要採取任何動作。會根據選取的驅動程式來自動顯示類別名稱。
DBMS SID 輸入資料庫的 SID。
DBMS Host 輸入主控資料庫的機器名稱。
DBMS Port 輸入用來連線至伺服器的連接埠。會自動顯示與選定資料庫類型相關聯的預設連接埠號碼。
URL 不需要採取任何動作。會根據選取的驅動程式來自動顯示 URL。
User Name 輸入用於連線至資料庫的登入名稱。
User Password 輸入用來存取資料庫的密碼。

有效值由英數字元的字串組成。支援連字號 (-) 和底線 ( _ ) 字元。
會加密此值。
Confirm User Password 重新輸入密碼。
Known Properties 不需要採取任何動作。會根據選取的驅動程式來自動顯示資料庫的已知特性。
Additional Properties 輸入要傳遞給驅動程式的其他特性 (如果有的話)。

注意：您可以按一下「測試連線」，來測試與資料庫的連線。啟動伺服器之前，您必須載入 RDBMS 安全儲存的必要 SQL 命令碼。如果您在叢集網域中使用 RDBMS 安全儲存，則建議您將它與 JMS 組態 (JNDI 名稱和 JMS 主題) 搭配使用。

進行必要的選取和輸入之後，請按一下精靈中的「下一步」按鈕。

有關詳細資訊，請參閱 Oracle 技術網路上的 Oracle Fusion Middleware 文件庫。
