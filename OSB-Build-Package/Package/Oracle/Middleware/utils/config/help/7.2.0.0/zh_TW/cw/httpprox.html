建立 HTTP 代理應用程式

使用此畫面以建立每個叢集的 HTTP 代理應用程式，並指定必須在其上建置代理應用程式的「受管理伺服器」。HTTP 代理應用程式的運作方式，有如 HTTP 要求的中繼者。

下列兩個敘述皆為真時，此畫面才會顯示：

  • 至少指派一個「受管理伺服器」至叢集。
  • 至少有一個「受管理伺服器」未指派至任何叢集。
  1. 針對想要建立 HTTP 代理應用程式的叢集，選取「建立 HTTP 代理」核取方塊。

    未指派至任何叢集的「受管理伺服器」清單會顯示在「代理伺服器」清單中。

  2. 在「代理伺服器」清單中，選取必須在其上建置代理應用程式的「受管理伺服器」。

    會在選取的「受管理伺服器」上建立和建置 OracleProxy4_<clustername>_<servername> 代理應用程式。

  3. 針對每個要建立 HTTP 代理應用程式的叢集，重複步驟 1 和 2。
  4. 進行必要的選取之後，請按一下「下一步」。
