將建置定位至叢集或伺服器

使用此畫面以針對伺服器或叢集的建置，定位應用程式和程式庫。

會將與設定網域之產品 (例如，SOA) 相關聯的應用程式和程式庫自動定位至為該產品所建立的「受管理伺服器」，或自動定位至「受管理伺服器」指派到的叢集。在此畫面中，您可以將應用程式和程式庫定位至其他伺服器和叢集。

  1. 在「Target」窗格中，選取想要在其上建置應用程式或程式庫的叢集或伺服器。

    選定目標的名稱會顯示為右側窗格的標題。

  2. 在右側窗格中，選取對應至想要建置於選定目標之應用程式或程式庫的核取方塊。

    這裡顯示的應用程式和程式庫會根據您先前在精靈的「選取網域來源」畫面中選取的產品而不同。

    注意：當您在「Target」窗格中選取「受管理伺服器」時，右側窗格中的部分核取方塊可能會停用，表示應用程式和程式庫已定位至的叢集，包含選取的「受管理伺服器」。

    選取應用程式和程式庫之後，已定位叢集和伺服器的名稱會顯示在右側窗格的「Target」資料欄。

  3. 依需要針對其他叢集和伺服器，重複步驟 1 和 2。
  4. 進行必要的選取之後，請按一下「下一步」。

當您擴充網域時，如果您刪除應用程式目前所定位的「受管理伺服器」或叢集，「組態精靈」會依下列方式自動重新定位應用程式：

  • 如果應用程式之前只定位到您刪除的「受管理伺服器」或叢集 (即在您刪除「受管理伺服器」或叢集之後，應用程式在修改後的網域中會變成未定位)，「組態精靈」會自動重新定位應用程式至所有合適的目標。

    合適的目標就是未在包含樣板之組態群組規格 (config-groups.xml 檔案) 中定義的任何叢集或「受管理伺服器」。config-groups.xml 檔案中指定的伺服器或叢集基本上是樣板所擁有，因此不適合用來自動定位。

  • 如果應用程式之前定位到多個目標 (包括「受管理伺服器」、叢集和「管理伺服器」)，刪除其中一個所定位的「受管理伺服器」或叢集時，「組態精靈」會維持其他的目標關聯不變，而且不會嘗試重新定位應用程式。
