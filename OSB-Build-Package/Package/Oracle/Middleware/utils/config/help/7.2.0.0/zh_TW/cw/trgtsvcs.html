將服務定位至伺服器或叢集

使用此畫面將服務定位至適當的「受管理伺服器」或叢集。

會將與設定網域之產品 (例如，SOA) 相關聯的服務自動定位至為該產品所建立的受管理伺服器，或自動定位至已指派該受管理伺服器的叢集。在此畫面中，您可以將服務定位至其他伺服器和叢集。

  1. 在「Target」窗格中，選取想要在其上建置服務的叢集或伺服器。

    選定目標的名稱會顯示為右側窗格的標題。

  2. 在右側窗格中，選取對應至想要建置於選定目標之服務的核取方塊。

    這裡顯示的服務會根據您先前在精靈的「選取網域來源」畫面中選取的產品而不同。

    當您在「Target」窗格中選取受管理伺服器時，可能會停用右側窗格中的部分核取方塊，以指出已定位至含有選定受管理伺服器之叢集的服務。

    選取服務之後，已定位叢集和伺服器的名稱會顯示在右側窗格的「Target」資料欄。

  3. 依需要針對其他叢集和伺服器，重複步驟 1 和 2。
  4. 進行必要的選取之後，請按一下「下一步」。

當您擴充網域時，如果您刪除服務目前所定位的「受管理伺服器」或叢集，「組態精靈」會依下列方式自動重新定位服務：

  • 如果服務之前只定位到您刪除的「受管理伺服器」或叢集 (即在您刪除「受管理伺服器」或叢集之後，服務在修改後的網域中會變成未定位)，「組態精靈」會自動重新定位服務至所有「合適」的目標。

    合適的目標就是未在包含樣板之組態群組規格 (config-groups.xml 檔案) 中定義的任何叢集或「受管理伺服器」。config-groups.xml 檔案中指定的伺服器或叢集基本上是樣板所擁有，因此不適合用來自動定位。

  • 如果服務之前定位到多個目標 (包括「受管理伺服器」、叢集和「管理伺服器」)，刪除其中一個所定位的「受管理伺服器」或叢集時，「組態精靈」會維持其他的目標關聯不變，而且不會嘗試重新定位服務。
