WebLogic Server-Administrationskonsole

Die Oracle WebLogic Server-Administrationskonsole ist eine webbrowserbasierte grafische Benutzeroberfläche, über die Sie eine WebLogic Server-Domain verwalten. Eine WebLogic Server-Domain ist eine logisch zusammengehörende Gruppe von WebLogic Server-Ressourcen, die Sie als Einheit verwalten. Eine Domain umfasst einen oder mehrere WebLogic-Server, darüber hinaus möglicherweise auch WebLogic Server-Cluster. Cluster sind Gruppen von WebLogic-Servern, die zusammenarbeiten und so Skalierbarkeit und High-Availability für Anwendungen zur Verfügung stellen. Sie verwalten und stellen Ihre Anwendungen als Teil einer Domain bereit.

Eine Instanz von WebLogic Server in jeder Domain wird als Administrationsserver konfiguriert. Der Administrationsserver stellt einen zentralen Punkt für die Verwaltung einer WebLogic Server-Domain bereit. Alle anderen WebLogic Server-Instanzen in einer Domain werden als verwaltete Server bezeichnet. In einer Domain mit nur einer WebLogic Server-Instanz fungiert dieser Server sowohl als Administrationsserver als auch als verwalteter Server. Der Administrationsserver hostet die Administrationskonsole, eine Webanwendung, auf die mit einem beliebigen unterstützten Webbrowser mit Netzwerkzugriff auf den Administrationsserver zugegriffen werden kann.

Sie können die Administrationskonsole für folgende Aufgaben verwenden:

  • WebLogic Server-Instanzen konfigurieren, starten und stoppen
  • WebLogic Server-Cluster konfigurieren
  • WebLogic Server-Services konfigurieren, beispielsweise Datenbankkonnektivität (JDBC) und Messaging (JMS)
  • Sicherheitsparameter konfigurieren sowie Benutzer, Gruppen und Rollen verwalten
  • Anwendungen konfigurieren und bereitstellen
  • Performance von Servern und Anwendungen überwachen
  • Log-Dateien für Server und Domains anzeigen
  • Deskriptoren für das Deployment von Anwendungen anzeigen
  • Ausgewählte Laufzeit-Deskriptorelemente für das Deployment von Anwendungen bearbeiten
