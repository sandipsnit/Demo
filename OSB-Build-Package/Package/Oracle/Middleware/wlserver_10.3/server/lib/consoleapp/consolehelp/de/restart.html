Änderungen werden wirksam, nachdem Sie das Modul erneut bereitgestellt oder den Server neu gestartet haben.

Wenn dieses Attribut ein von Ihnen bereitgestelltes Modul, z.B. eine Anwendung oder eine JDBC-Datenquelle als Teil einer Anwendung, oder eine Systemressource konfiguriert, deren Konfiguration in einer Deskriptor-Datei statt in der Datei config.xml gespeichert wird, z.B. wie bei der JDBC-Datenquelle, deren Geltungsbereich die Systemebene ist, dann kann das Modul bzw. die Ressource die Veränderung nicht verarbeiten, bis Sie es bzw. sie erneut bereitstellen oder den Host-Server neu starten. Wenn das Modul eine Komponente in einer Anwendung ist, empfiehlt Oracle, dass Sie die gesamte Anwendung neu bereitstellen, um Probleme zu vermeiden, die aus den Beziehungen zwischen den Anwendungen entstehen könnten.

Wenn dieses Attribut einen anderen Teil der Domain konfiguriert, z.B. einen Server, Cluster oder EJB-Container, dann kann das System diese Änderung nicht verarbeiten, bis Sie den Server bzw. Cluster neu starten.
