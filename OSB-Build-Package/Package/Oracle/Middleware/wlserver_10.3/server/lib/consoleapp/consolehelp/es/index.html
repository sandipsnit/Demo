Consola de Administración de WebLogic Server

La Consola de Administración de Oracle WebLogic Server es una interfaz gráfica de usuario basada en explorador web que se utiliza para gestionar un dominio de WebLogic Server. Un dominio de WebLogic Server es un grupo de recursos de WebLogic Server relacionado de forma lógica que el usuario gestiona como una unidad. Un dominio incluye uno o más servidores WebLogic Server y puede incluir además clusters WebLogic Server. Los clusters son grupos de servidores WebLogic Server que funcionan conjuntamente para proporcionar escalabilidad y una alta disponibilidad de aplicaciones. Las aplicaciones se despliegan y gestionan como parte de un dominio.

Las instancias de WebLogic Server se configuran en cada dominio como un servidor de administración. El servidor de administración proporciona un punto central de gestión para un dominio de WebLogic Server. El resto de instancias de WebLogic Server de un dominio se llaman servidores gestionados. En un dominio con una única instancia de WebLogic Server, el servidor funciona como servidor de administración y servidor gestionado. El servidor de administración aloja la Consola de Administración, que es una aplicación web accesible desde cualquier explorador compatible con la Web con acceso de red al servidor de administración.

Puede utilizar la Consola de Administración para:

  • Configurar, iniciar y parar instancias de WebLogic Server.
  • Configurar clusters de WebLogic Server.
  • Configurar servicios de WebLogic Server, como conectividad de base de datos (JDBC) y mensajería (JMS).
  • Configurar parámetros de seguridad, que incluyen la gestión de usuarios, grupos y roles.
  • Configurar y desplegar aplicaciones.
  • Supervisar el rendimiento del servidor y las aplicaciones.
  • Ver archivos log de servidor y dominio.
  • Ver descriptores de despliegue de aplicación.
  • Editar elementos de descriptor de despliegue de aplicación seleccionada en tiempo de ejecución.
