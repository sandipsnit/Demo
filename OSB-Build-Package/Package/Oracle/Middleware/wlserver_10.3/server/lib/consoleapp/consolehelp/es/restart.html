Los cambios se aplicarán después de volver a desplegar el módulo o reiniciar el servidor.

Si este atributo configura un módulo que despliega (por ejemplo, una aplicación o un origen de datos de JDBC que es parte de una aplicación) o un recurso del sistema cuya configuración se guarda en un archivo descriptor en lugar de en el archivo config.xml del dominio (como un origen de datos de JDBC con ámbito en el nivel del sistema), el módulo o el recurso no puede procesar el cambio hasta que lo vuelva a desplegar o reinicie su servidor de host. Si el módulo es un componente de una aplicación, Oracle recomienda que vuelva a desplegar toda la aplicación para evitar complicaciones debidas a las dependencias dentro de aplicación.

Si este atributo configura alguna otra parte del dominio (como un servidor, un cluster o un contenedor EJB), el sistema no puede procesar el cambio hasta que reinicie el servidor o el cluster.
