Console d'administration WebLogic Server

La console d'administration Oracle WebLogic Server est une interface utilisateur graphique, basée sur un navigateur Web, qui permet de gérer un domaine WebLogic Server. Un domaine WebLogic Server est un groupe de ressources WebLogic Server ayant un lien logique, que vous gérez en tant qu'unité. Il contient au moins un serveur WebLogic et peut aussi contenir des clusters de serveurs WebLogic. Les clusters sont des groupes de serveurs WebLogic qui fonctionnent ensemble pour fournir évolutivité et haute disponibilité aux applications. Vous déployez et gérez vos applications dans le cadre d'un domaine.

Dans chaque domaine, une instance WebLogic Server est configurée comme serveur d'administration. Celui-ci sert de point de référence pour la gestion du domaine. Toutes les autres instances WebLogic Server d'un domaine sont appelées des serveurs gérés. Si un domaine ne possède qu'une seule instance WebLogic Server, elle sert à la fois de serveur d'administration et de serveur géré. Le serveur d'administration héberge la console d'administration, une application Web accessible à partir de tous les navigateurs Web pris en charge avec accès réseau au serveur d'administration.

Vous pouvez utiliser la console d'administration pour :

  • Configurer, démarrer et arrêter des instances WebLogic Server.
  • Configurer des clusters WebLogic Server.
  • Configurer des services WebLogic Server, tels que la connectivité à la base de données (JDBC) et la messagerie (JMS).
  • Configurer des paramètres de sécurité, ce qui inclut la gestion des utilisateurs, des groupes et des rôles.
  • Configurer et déployer vos applications.
  • Surveiller les performances des serveurs et des applications.
  • Visualiser les fichiers journaux de serveur et de domaine.
  • Visualiser les descripteurs de déploiement d'application.
  • Modifier les éléments du descripteur de déploiement d'application d'exécution sélectionné.
