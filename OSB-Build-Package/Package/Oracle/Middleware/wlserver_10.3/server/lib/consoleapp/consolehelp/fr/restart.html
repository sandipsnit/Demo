Les modifications sont prises en compte une fois le module redéployé ou le serveur relancé.

Si cet attribut configure un module que vous déployez (par exemple, une application ou une source de données JDBC qui fait partie d'une application) ou une ressource système dont la configuration est enregistrée dans un fichier de descripteur plutôt que dans le fichier config.xml du domaine (par exemple, une source de données JDBC étendue au niveau du système), le module ou la ressource ne peut pas traiter les modifications tant que vous ne l'avez pas redéployé ou n'avez pas redémarré son serveur hôte. Si le module est un composant d'une application, Oracle vous recommande de redéployer entièrement l'application pour éviter toute complication due à des dépendances au sein de celle-ci.

Si cet attribut configure une autre partie du domaine (par exemple un serveur, un cluster ou un conteneur EJB), le système ne peut pas traiter les modifications tant que le serveur ou le cluster n'est pas redémarré.
