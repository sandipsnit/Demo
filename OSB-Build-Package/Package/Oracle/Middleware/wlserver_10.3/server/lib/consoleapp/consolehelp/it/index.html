La console di amministrazione di WebLogic Server

La console di amministrazione di Oracle WebLogic Server è un'interfaccia grafica basata su browser Web utilizzata per gestire un dominio WebLogic Server. Un dominio WebLogic Server è un gruppo di risorse di WebLogic Server correlato logicamente e gestito come unità. Un dominio include uno o più WebLogic Server e potrebbe includere anche i cluster WebLogic Server. I cluster sono gruppi di WebLogic Server che funzionano insieme per aumentare la scalabilità e l'affidabilità delle applicazioni. Le applicazioni vengono distribuite e gestite come parte di un dominio.

In ogni dominio viene configurata una sola istanza di WebLogic Server come server di amministrazione. Il server di amministrazione rappresenta il fulcro della gestione di un dominio di WebLogic Server. Tutte le altre istanze di WebLogic Server di un dominio vengono denominate server gestiti. In un dominio con un'unica istanza di WebLogic Server il server funziona sia come server di amministrazione che come server gestito. Il server di amministrazione ospita la console di amministrazione, ovvero un'applicazione Web accessibile da qualsiasi browser Web supportato con accesso di rete al server di amministrazione.

È possibile utilizzare la console di amministrazione per effettuare le operazioni riportate di seguito.

  • Configurare, avviare e arrestare le istanze di WebLogic Server.
  • Configurare i cluster di WebLogic Server.
  • Configurare i servizi di WebLogic Server, ad esempio la connettività al database (JDBC) e la messaggistica (JMS).
  • Configurare i parametri di sicurezza, inclusa la gestione di utenti, gruppi e ruoli.
  • Configurare e distribuire le applicazioni.
  • Monitorare le prestazioni di server e applicazioni.
  • Visualizzare i file di log di server e dominio.
  • Visualizzare descrittori di distribuzione delle applicazioni.
  • Modificare gli elementi dei descrittori di distribuzione delle applicazioni di runtime selezionati.
