Le modifiche avranno effetto dopo la ridistribuzione del modulo o il riavvio del server.

Se questo attributo configura un modulo che si distribuisce, ad esempio un'applicazione o un'origine dati JDBC che fa parte di un'applicazione, o una risorsa di sistema la cui configurazione è salvata in un file descrittore anziché nel file config.xml del dominio, ad esempio un'origine dati JDBC con ambito a livello di sistema, la risorsa o il modulo non può elaborare la modifica finché non viene ridistribuito o non si riavvia il server host. Se il modulo è un componente in un'applicazione, Oracle consiglia di ridistribuire l'intera applicazione in modo da evitare complicazioni a causa delle dipendenze tra le applicazioni.

Se questo attributo configura altre parti del dominio, ad esempio un server, un cluster o un contenitore EJB, il sistema non può elaborare la modifica finché non si riavvia il server o il cluster.
