모듈을 재배치하거나 서버를 재시작한 후에 변경 사항이 적용됩니다.

이 속성이 사용자가 배치한 모듈(예: 응용 프로그램 또는 응용 프로그램의 일부인 JDBC 데이터 소스) 또는 도메인의 config.xml 파일(예: 시스템 레벨 범위의 JDBC 데이터 소스) 대신 기술자 파일에 구성이 저장된 시스템 리소스를 구성하는 경우, 이 모듈이나 리소스는 변경 사항을 재배치하거나 호스트 서버를 재시작할 때까지 변경 사항을 처리할 수 없습니다. 모듈이 응용 프로그램의 구성 요소인 경우, 응용 프로그램 간 종속성으로 인한 혼선을 방지하기 위해 전체 응용 프로그램을 재배치하는 것이 좋습니다.

이 속성이 일부 도메인의 다른 부분(예: 서버, 클러스터 또는 EJB 컨테이너)을 구성하는 경우, 시스템은 서버나 클러스터를 재시작할 때까지 변경 사항을 처리할 수 없습니다.
