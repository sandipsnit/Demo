O Console de Administração do WebLogic Server

O Console de Administração do Oracle WebLogic Server é uma interface gráfica de usuário baseada em Web browser usada para gerenciar um domínio do WebLogic Server. Um domínio do WebLogic Server é um grupo relacionado logicamente de recursos do WebLogic Server gerenciados como uma unidade. Um domínio inclui um ou mais WebLogic Servers e também pode incluir clusters do WebLogic Server. Clusters são grupos de WebLogic Servers que funcionam em conjunto para oferecer escalabilidade e alta disponibilidade para aplicações. Implante e gerencie suas aplicações como parte de um domínio.

Uma instância do WebLogic Server em cada domínio é configurada como um Servidor de Administração. O Servidor de Administração oferece um ponto central para gerenciamento de um domínio do WebLogic Server. Todas as outras instâncias do WebLogic Server em um domínio são chamados de Servidores Gerenciados. Em um domínio com apenas uma única instância do WebLogic Server, esse servidor funciona como Servidor de Administração e Servidor Gerenciado. O Servidor de Administração hospeda a Console de Administração, que é uma Aplicação Web acessível em qualquer Web browser suportado com acesso de rede ao Servidor de Administração.

É possível usar a Console de Administração para:

  • Configurar, iniciar e parar instâncias do WebLogic Server.
  • Configurar clusters do WebLogic Server.
  • Configurar serviços do WebLogic Server, como conectividade do banco de dados (JDBC) e mensagens (JMS).
  • Configurar parâmetros de segurança, incluindo o gerenciamento de usuários, grupos e atribuições.
  • Configurar e implantar suas aplicações.
  • Monitorar desempenho do servidor e da aplicação.
  • Exibir arquivos de log do servidor e do domínio.
  • Exibir descritores de implantação da aplicação.
  • Editar elementos do descritor de implantação da aplicação de run-time.
