As alterações têm efeito após você reimplantar o módulo ou reiniciar o servidor.

Se este atributo configurar um módulo que você implantar (como um aplicativo ou uma origem de dados JDBC que faça parte de um aplicativo) ou um recurso do sistema cuja configuração está salva em um arquivo descritor em vez do arquivo config.xml do domínio (como uma origem de dados JDBC com escopo no nível do sistema), o módulo ou recurso não poderá processar a alteração até você reimplantá-lo ou reiniciar seu servidor host. Se o módulo for um componente em um aplicativo, a Oracle recomenda que você reimplante todo o aplicativo para evitar complicações devidas a dependências entre os aplicativos.

Se este atributo configurar alguma outra parte do domínio (como um servidor, um cluster ou um container EJB), o sistema não poderá processar a alteração até você reiniciar o servidor ou cluster.
