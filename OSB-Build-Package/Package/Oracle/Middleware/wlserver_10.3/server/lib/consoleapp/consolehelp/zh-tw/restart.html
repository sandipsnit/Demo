在您重新建置模組或重新啟動伺服器之後，變更才會生效。

如果此屬性設定您建置的模組 (如應用程式或應用程式一部分的 JDBC 資料來源)，或此屬性設定一個系統資源，而此系統資源的組態已儲存在描述元檔案而不是網域之 config.xml 檔案 (如系統層級作用領域的 JDBC 資料來源)，則該模組或資源無法處理變更，直到您重新建置它或重新啟動其主機伺服器為止。如果模組是應用程式中的元件，則 Oracle 建議您重新建置整個應用程式，以避免由於內部應用程式相依性而使得應用程式更為複雜。

如果此屬性設定網域的某些其他部分 (如伺服器、叢集或 EJB 容器)，則系統無法處理變更，直到您重新啟動伺服器或叢集為止。
