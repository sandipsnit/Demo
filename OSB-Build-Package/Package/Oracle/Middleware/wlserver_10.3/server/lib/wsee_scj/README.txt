There are the following files in the directory (i.e. wsee_scj)
wsee_scj/
    - README.txt
    - wseejb.jar (Dynamically generated by SCJ)
    - resDef.xml (Dynamically copied from SCJ)
    - benchmarks/
        - jaxws-assembly.xml (Manually generated by SCJ Framework, for JAX-WS client)
        - jaxrpc-assembly.xml (Manually generated by SCJ Framework, for JAX-RPC client)
        - all-assembly.xml (Manually generated by SCJ Framework, for both JAX-WS and JAX-RPC client)
        
According to our client requirement, we can pack a standalone client jars with below command:

  java -jar wseejb.jar options assemblyFile

  Options:
    -D wl.home="your wl_home"  (Mandatory) Define the WL_HOME
    -D bea.home="your bea_home"  (Mandatory) Define the BEA_HOME
    -r resFile Define an resource definition file
    -d destDir Where to generate the jar(s), default is current directory
    -v version Define the wls.version, default it is 10.3.2.0
    -h help info   
    
E.g. if our WLS is installed on d:/wls1034, and we want only JAX-WS client, we can use the command:

  java -jar wseejb.jar \
    -D wl.home="d:/wls1033/wlserver_10.3" \
    -D bea.home="d:/wls1033" \
    -v 10.3.4.0
    -r resDef.xml \
    -d d:/jaxws_scj \
    benchmarks/jaxws-assembly.xml
